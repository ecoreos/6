DDNS

DDNS (Dynamic Domain Name Service) vereinfacht die Verbindung von DiskStation über das Internet durch Zuweisung eines Hostnamens zur IP-Adresse. DDNS erlaubt Ihnen z.B. den Zugriff auf DiskStation unter Verwendung eines Domänennamens (z.B. www.john.synology.me), ohne dass Sie sich an die IP-Adresse (z.B. 172.16.254.1) erinnern müssen.

DDNS einrichten

Sie können einen existierenden Hostnamen auf die IP-Adresse von DiskStation richten oder einen neuen, von Synology oder verschiedenen anderen DDNS Providern bereitgestellten, registrieren. Bitte wenden Sie sich an den jeweiligen Anbieter, um mehr über die Registrierung von Hostnamen zu erfahren.

Einen DDNS Hostnamen einrichten:

  1. Klicken Sie auf Hinzufügen.
  2. Ein Dialogfenster wird angezeigt und Sie auffordern, die folgenden Einstellungen zu ändern:
    • Serviceanbieter: Wählen Sie einen Service-Anbieter. Um einen kostenlosen Hostnamen von Synology zu registrieren, wählen Sie Synology aus dem Dropdown-Menü, klicken Sie auf Jetzt registrieren und geben Sie die Details wie gefordert ein.
    • Hostname: Geben Sie einen registrierten DDNS Hostnamen ein, zum Beispiel john.synology.me.
    • Benutzername: Geben Sie den Benutzernamen für Ihren DDNS Anbieter ein.
    • Kennwort: Geben Sie das Kennwort für Ihren DDNS Anbieter ein.
    • Externe Adresse: Geben Sie die externe IP-Adresse von DiskStation ein, der den Hostnamen verwenden wird.
    • Heartbeat: Aktivieren Sie diese Option, um Alarme in Bezug auf den Status des zugewiesenen Hostnamens zu erhalten (nur für von Synology bereitgestellte Hostnamen).
  3. Klicken Sie auf Verbindung testen, um zu prüfen, ob die Einstellungen korrekt sind.
  4. Klicken Sie zum Speichern und Fertigstellen auf OK.

Nach dem Abschluss der Einrichtung können Sie auf DiskStation über das Internet durch Eingabe des DDNS Hostnamens in einem Webbrowser zugreifen.

Anforderungen und Einschränkungen

  • Die Server des DDNS Dienstes funktionieren.
  • DiskStation kann sich mit dem Internet verbinden.
  • DDNS funktioniert in einer Netzwerkumgebung, in der kein Proxyserver benötigt wird.
  • Sie können nur einen Hostnamen für jeden DDNS-Anbieter eingeben.

Hostnamen verwalten

Verwenden Sie die folgenden Schaltflächen, um bestehende Hostnameneinträge zu verwalten:

  • Löschen: Entfernt den ausgewählten Hostnameneintrag aus der Liste.
  • Bearbeiten: Ändert Einstellungen des ausgewählten Hostnameneintrags.
  • Jetzt aktualisieren: Aktualisiert den Status aller Hostnameneinträge, für die der DDNS Dienst aktiviert ist.

Benutzerdefinierten DDNS-Anbieter einrichten

Sie können einen benutzerdefinierten DDNS-Anbieter hinzufügen, wenn Ihr Anbieter nicht bereits in der Liste der Serviceanbieter angezeigt wird.

Benutzerdefinierten DDNS-Anbieter einrichten:

  1. Klicken Sie auf Anpassen.
  2. Ein Dialogfenster wird angezeigt und fordert Sie auf, die nachstehenden Einstellungen einzugeben:
    • Serviceanbieter: Geben Sie den Namen Ihres benutzerdefinierten Serviceanbieters ein.
    • Abfrage-URL: Geben Sie die URL Ihres benutzerdefinierten Serviceanbieters ein.
  3. Klicken Sie auf Speichern, um den benutzerdefinierten Serviceanbieter zur Liste der DDNS-Serviceanbieter hinzuzufügen.

Anforderungen und Einschränkungen

  • Die Abfrage-URL muss dem von Ihrem Serviceanbieter angegebenen Protokoll entsprechen. Diese URL wird verwendet, um aktuelle IP-Informationen zum DDNS-Server Ihres Anbieters zusenden.
  • Eine Abfrage-URL enthält grundsätzlich die vier folgenden Parameter. Ersetzen Sie bei der Eingabe der Abfrage-URL die folgenden Parameter durch die entsprechende Variable.
    • Benutzername: __USERNAME__
    • Kennwort: __PASSWORD__
    • Hostname: __HOSTNAME__
    • Externe Adresse: __MYIP__
  • Die Abfrage-URL darf keine Leerzeichen oder Semikolons enthalten.

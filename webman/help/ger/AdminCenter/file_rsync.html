rsync

Sie können es anderen Benutzern erlauben, über rsync von einem Synology NAS oder von rsync-kompatiblen Clients Daten auf Ihrem DiskStation zu sichern.

rsync-Sicherungsdienst aktivieren

rsync-Sicherungsdienst aktivieren:

Gehen Sie zu Systemsteuerung > Dateidienste > rsync und wählen Sie rsync-Dienst aktivieren.

rsync-Sicherungsberechtigungen zuweisen:

Weisen Sie Benutzern die richtigen rsync-Berechtigungen zu, damit sie Daten von einem Quell-Synology NAS oder von einem rsync-kompatiblen Server auf dem Ziel-Synology NAS sichern können.

  1. Gehen Sie zu Systemsteuerung > Berechtigungen und klicken Sie auf rsync (Synchronisierung freigegebener Ordner, LUN-Sicherung auf unterstützten Modellen) > Bearbeiten.
  2. Weisen Sie den gewünschten Benutzern (DSM-Benutzer für verschlüsseltes rsync oder rsync-Konten für unverschlüsseltes rsync) Berechtigungen des Typs Zulassen zu oder oder geben Sie die zugelassenen IP-Adressen für bestimmte Benutzer an.

Hinweis:

Um die rsync-Sicherung von einem DiskStation mit einer DSM-Version vor 3.0 aus durchzuführen oder von einem Client aus, der nicht ein DiskStation ist, und um die Informationen des Besitzers und der Gruppe als Quelldaten beizubehalten, müssen Sie die rsync-Konten zur Gruppe administrators hinzufügen und die Daten im freigegebenen Ordner NetBackup sichern.

rsync-Konten aktivieren

Wenn Sie eine rsync-Sicherung von einem Nicht-Synology NAS-Client durchführen, müssen Sie auf dem Ziel-Synology NAS rsync-Konten erstellen, damit die rsync-Benutzer Daten auf Ihrem Synology NAS sichern können.

rsync-Konten aktivieren:

  1. Gehen Sie zu Systemsteuerung > Dateidienste > rsync und wählen Sie rsync-Konto aktivieren.
  2. Klicken Sie auf rsync-Konto bearbeiten, um Benutzer und entsprechende Kennwörter für die Authentifizierung einzustellen.
  3. Klicken Sie auf Übernehmen.

Hinweis:

  • Wenn der rsync-Dienst verschlüsselt ist (d. h. wenn ein SSH-Verschlüsselungsport angegeben ist), setzt das System direkt die Benutzernamen und Kennwörter von DSM-Benutzern als deren rsync-Konten ein.
  • Wenn der rsync-Dienst nicht verschlüsselt ist, müssen Sie die entsprechenden rsync-Konten für sie erstellen.

Verwenden der benutzerdefinierten rsync-Konfiguration

Sie können eine benutzerdefinierte rsync-Konfiguration verwenden und Benutzerberechtigungen sowie das Datensicherungsziel auf Ihrem DiskStation konfigurieren. Daher können andere Personen eine rsync-Sicherung von einem DiskStation oder einem rsync-kompatiblen Client in freigegebene Zielordner (oder „Datensicherungsmodule“) auf dem DiskStation durchführen.

Benutzerdefinierte rsync-Konfiguration verwenden:

  1. Melden Sie sich beim DiskStation über Telnet oder SSH an.
  2. Geben Sie das Datensicherungsmodul und die autorisierten Benutzer in der rsync-Konfigurationsdatei "/etc/rsyncd.conf" an. Detaillierte Informationen finden Sie in der Anleitung von rsyncd.conf auf Ihrem rsync-Client oder hier.

Hinweis:

Wenn das Datensicherungsmodul und der freigegebene Ordner denselben Namen haben, wird die Einstellung des Datensicherungsmoduls verwendet.

Geschwindigkeitseinschränkung aktivieren

Sie können die maximal verwendbare Bandbreite für die rsync-Sicherung für Benutzer und Gruppen überwachen.

Geschwindigkeitseinschränkung aktivieren:

  1. Gehen Sie zu Systemsteuerung > Dateidienste > rsync. Wählen Sie eine der folgenden Optionen aus:
    • Keine Geschwindigkeitseinschränkung
    • Geschwindigkeitsbeschränkung aktivieren
    • Geschwindigkeitseinschränkung nach einem Zeitplan aktivieren (Klicken Sie auf Zeitplan einstellen, um weitere Änderungen vorzunehmen.)

    Hinweis:

    Dies ist eine serverweite Einstellung, die die Einstellungen der einzelnen Benutzer und Gruppen überschreibt. Beispielsweise deaktiviert die Auswahl von Keine Geschwindigkeitseinschränkung die Einstellungen für die Geschwindigkeitseinschränkung aller Benutzer und Gruppen.

  2. Klicken Sie, wenn Sie die Geschwindigkeitseinschränkung aktiviert haben, auf Geschwindigkeitseinschränkungseinstellungen und anschließend auf den gewünschten Benutzer bzw. die gewünschte Gruppe, um die Einstellungen zu ändern:
    • Ohne: Es wird keine Geschwindigkeitseinschränkung angewendet, wenn Gruppenmitglieder Dateien mit dem Dienst übertragen.
    • Gruppeneinstellungen übernehmen: Für den Benutzer gelten dieselben Geschwindigkeitseinschränkungseinstellungen wie für die zugehörige Gruppe.
    • Konstant: Für alle Benutzer und Gruppenmitglieder gilt eine feste, konstante Geschwindigkeitseinschränkung. Upload- und Download-Geschwindigkeitseinschränkungen können in den Feldern links angegeben werden.
    • Benutzerdefiniert: Für Benutzer oder Gruppenmitglieder können zwei unterschiedliche Geschwindigkeitseinschränkungen gemäß einem Zeitplan festgelegt werden. Klicken Sie auf Anpassen, um die Geschwindigkeitseinschränkungseinstellungen zu ändern und einen Zeitplan festzulegen.
  3. Klicken Sie auf Übernehmen.

Hinweis:

Im Abschnitt Ergebnis (UL/DL) wird das tatsächliche Upload- und Download-Limit gemäß seinen eigenen und gemäß den Gruppeneinstellungen angezeigt. Falls beispielsweise ein Benutzer „Achilles“ ein individuelles Download-Limit von 100 KB/s hat, aber zu einer Benutzergruppe mit einem Download-Limit von 500 KB/s gehört, wird im Bereich Ergebniss (UL/DL) „0/500“ angezeigt.

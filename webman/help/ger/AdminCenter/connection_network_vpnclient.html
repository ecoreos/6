Verbindung zu VPN

Mit VPN können Sie aus dem DiskStation ganz einfach einen VPN-Client machen und eine Verbindung zu einem bestehenden VPN-Server (Virtual Private Network-Server) über PPTP, OpenVPN oder L2TP/IPSec herstellen. Falls Sie mehrere VPN-Server besitzen, können Sie spezifische VPN-Profile für die einzelnen Server erstellen, sodass Sie schnell umschalten und mit nur einem Klick auf andere VPN-Server wechseln können.

Was ist VPN?

Ein VPN, oder „virtuelles privates Netzwerk“, ist eine Lösung, mit der Sie über das Internet sicher auf Ihr privates Netzwerk zugreifen können. Für Unternehmen oder Privatpersonen, die ihre Netzwerkfunktionalität in immer kürzeren Abständen erweitern müssen, nehmen die Kosten für dieses physische Netzwerksystem und deren technischen Support exponentiell zu. Angesichts der Kosteneffizienz und des geringen langfristigen Wartungsaufwands ist VPN eine intelligente und immer attraktiver werdende Lösung. Dank Verschlüsselung und anderer Sicherheitsmechanismen können Mitarbeiter mithilfe der VPN-Technologie mühelos auf das zentrale Netzwerk ihres Unternehmens zugreifen und dessen Ressourcen so nutzen, wie wenn sie sich im LAN befänden. Außerdem können Privatpersonen auch von unterwegs aus auf Ihr Heimnetzwerk zugreifen.

Hinweis:

  • DiskStation kann sich nur mit OpenVPN Servern verbinden, die Tunnel im „tun-Stil“ auf Ebene 3 unterstützen.
  • Bevor Sie das VPN-Paket installieren, müssen Sie einen VPN-Server haben, mit dem Sie sich verbinden können. Zur Verbesserung der Kompatibilität empfehlen wir, durch Ausführen des Pakets VPN Server die Verbindung zu VPN-Diensten herzustellen, die auf einer DiskStation gehostet werden. Zum Einrichten einer weiteren DiskStation als VPN-Server siehe Hilfe zu DSM > VPN Server > VPN Server einrichten.

VPN-Profile erstellen:

  1. Gehen Sie zu Systemsteuerung > Netzwerk.
  2. Klicken Sie auf der Registerkarte Netzwerkschnittstelle auf Erstellen und wählen Sie VPN-Profil erstellen aus.
  3. Für die Erstellung eines neuen VPN-Profils stehen die folgenden Verbindungstypen zur Verfügung:
    • PPTP: PPTP (Point-to-Point Tunneling Protocol) ist eine häufig genutzte VPN-Lösung, die von den meisten Clients (einschließlich Windows, Mac, Linux und Mobilgeräte) unterstützt wird.
    • OpenVPN: OpenVPN ist eine Open-Source-Lösung für die Implementierung des VPN-Dienstes. Sie schützt die VPN-Verbindung mit dem Verschlüsselungsmechanismus SSL/TLS.
    • L2TP/IPSec: L2TP (Layer 2 Tunneling Protocol) über IPSec erstellt virtuelle private Netzwerke mit verbesserter Sicherheit und wird von den meisten Clients (z. B. Windows, Mac, Linux und Mobilgeräte) unterstützt.
  4. In den folgenden Abschnitten finden Sie Anweisungen zur Einrichtung einer der drei Typen von VPN-Verbindungen.

PPTP-Profil erstellen:

  1. Wählen Sie, wenn Sie aufgefordert werden, eine VPN-Verbindungsmethode auszuwählen, PPTP aus und klicken Sie auf Weiter.
  2. Geben Sie dem Profil einen Namen, geben Sie die Adresse des VPN-Servers an, mit dem Sie eine Verbindung herstellen möchten, und geben Sie den Benutzernamen und das Kennwort für den Server ein. Klicken Sie auf Weiter, um fortzufahren.

    3. Hinweis:

    • Benutzername, Kennwort und Adresse des VPN-Servers sollten vom Administrator des VPN-Servers zur Verfügung gestellt werden.
  3. Wählen Sie eines der folgenden Authentifizierungsprotokolle im Dropdown-Menü Authentifizierung aus, um das Kennwort des VPN-Clients während der Authentifizierung zu schützen:
    • PAP: Das Kennwort wird während der Authentifizierung nicht verschlüsselt.
    • CHAP: Das Kennwort wird mit CHAP (Challenge-Handshake Authentication Protocol) verschlüsselt.
    • MS CHAP: Das Kennwort wird mit Microsoft CHAP Version 1 verschlüsselt.
    • MS CHAP v2: Das Kennwort wird mit Microsoft CHAP Version 2 verschlüsselt.
  4. Wählen Sie, wenn Sie MS-CHAP oder MS-CHAP v2 ausgewählt haben, eine der folgenden Optionen im Menü Verschlüsselung aus, um die VPN-Verbindung zu verschlüsseln:
    • Kein MPPE: VPN-Verbindung wird nicht durch Microsoft Point to Point to Point-Verschlüsselung geschützt.
    • Erfordert MPPE (40/128 Bit): VPN-Verbindung wird durch eine 40-Bit- oder 128-Bit Microsoft Point to Point-Verschlüsselung geschützt.
    • Maximale MPPE (128 Bit): Die VPN-Verbindung wird durch 128-Bit Microsoft Point to Point to Point-Verschlüsselung geschützt, was die höchste Sicherheitsstufe bietet.

    5. Hinweis:

    • Die Einstellungen für Authentifizierung und Verschlüsselung müssen mit den Einstellungen auf dem VPN-Server identisch sein. Einzelheiten erhalten Sie vom Administrator des VPN-Servers.
  5. Aktivieren Sie je nach Ihren Anforderungen eines der folgenden Kontrollkästchen:
    • Standard-Gateway auf Remote-Netzwerk verwenden: Aktivieren Sie diese Option, um den Netzwerkverkehr des DiskStation zum angegebenen VPN-Server zu leiten.
    • Anderen Netzwerkgeräten ermöglichen, Verbindung über die Internetverbindung dieses Synology Servers herzustellen: Aktivieren Sie diese Option, um es Netzwerkgeräten, die sich im selben lokalen Netzwerk befinden wie Ihr DiskStation, zu ermöglichen, eine Verbindung zum selben VPN-Server herzustellen.
    • Wieder verbinden, wenn die VPN-Verbindung verloren wurde: Wenn die VPN-Verbindung unerwartet verloren geht, versucht das System, die Verbindung fünf Mal alle 30 Sekunden wiederherzustellen.
  6. Klicken Sie auf Übernehmen.

OpenVPN-Profil erstellen:

  1. Wählen Sie, wenn Sie aufgefordert werden, eine VPN-Verbindungsmethode auszuwählen, OpenVPN aus und klicken Sie auf Weiter.
  2. Geben Sie dem Profil einen Namen, geben Sie die Adresse des VPN-Servers an, mit dem Sie eine Verbindung herstellen möchten, und geben Sie den Benutzernamen und das Kennwort für den Server ein.
  3. Klicken Sie im Feld Zertifikat auf die Schaltfläche Durchsuchen, um die Zertifikatdatei (z. B. ca.crt) zu suchen und zu importieren, die vom VPN-Server exportiert wurde. Klicken Sie auf Weiter, um fortzufahren.

    4. Hinweis:

    • Benutzername, Kennwort, Adresse und Zertifikat des VPN-Servers sollten vom Administrator des VPN-Servers zur Verfügung gestellt werden.
  4. Aktivieren Sie je nach Ihren Anforderungen eines der folgenden Kontrollkästchen:
    • Komprimierung auf dem VPN-Link verwenden: Aktivieren Sie diese Option, um Daten zu komprimieren, bevor Sie sie über VPN übertragen. Diese Option kann die Übertragungsgeschwindigkeit steigern, verbraucht aber mehr Systemressourcen.
    • Standard-Gateway auf Remote-Netzwerk verwenden: Aktivieren Sie diese Option, um den Netzwerkverkehr des DiskStation zum angegebenen VPN-Server zu leiten.
    • Anderen Netzwerkgeräten ermöglichen, Verbindung über die Internetverbindung dieses Synology Servers herzustellen: Aktivieren Sie diese Option, um es Netzwerkgeräten, die sich im selben lokalen Netzwerk befinden wie Ihr DiskStation, zu ermöglichen, eine Verbindung zum selben VPN-Server herzustellen.
    • Wieder verbinden, wenn die VPN-Verbindung verloren wurde: Wenn die VPN-Verbindung unerwartet verloren geht, versucht das System, die Verbindung fünf Mal alle 30 Sekunden wiederherzustellen.
  5. Klicken Sie auf Übernehmen.

OpenVPN-Profil (via .ovpn) erstellen:

  1. Wählen Sie, wenn Sie zur Auswahl einer VPN-Verbindungsmethode aufgefordert werden, OpenVPN (über den Import einer .ovpn-Datei) aus und klicken Sie auf Weiter.
  2. Wählen Sie einen Namen für das neue Profil aus und klicken Sie im Feld .ovpn-Datei importieren auf Durchsuchen, um eine vom VPN Server exportierte .ovpn-Datei auszuwählen und zu importieren.
  3. Geben Sie die vom Administrator des VPN Servers erhaltenen Informationen ein: Ihren Benutzernamen, Ihr Passwort. Importieren Sie die CA-Zertifikatdatei (z. B. ca.crt).
  4. Wenn Sie vom Anbieter des VPN Servers auch TLS-auth Schlüssel, Client-Schlüssel, Client-Zertifikat oder Zertifikatsentzugsliste erhalten haben, klicken Sie bitte auf Erweiterte Optionen und importieren Sie diese Daten in die entsprechenden Felder.
  5. Klicken Sie auf Weiter, um fortzufahren.

    6. Hinweis:

    • Benutzername, Passwort, Adresse und sonstige Dateien, wie Zertifikate und Schlüssel, sollten durch den Administrator des VPN Servers zur Verfügung gestellt werden.
  6. Aktivieren Sie je nach Ihren Anforderungen eines der folgenden Kontrollkästchen:
    • Komprimierung auf dem VPN-Link verwenden: Aktivieren Sie diese Option, um Daten zu komprimieren, bevor Sie sie über VPN übertragen. Diese Option kann die Übertragungsgeschwindigkeit steigern, verbraucht aber mehr Systemressourcen.
    • Standard-Gateway auf Remote-Netzwerk verwenden: Aktivieren Sie diese Option, um den Netzwerkverkehr des DiskStation zum angegebenen VPN-Server zu leiten.
    • Anderen Netzwerkgeräten ermöglichen, Verbindung über die Internetverbindung dieses Synology Servers herzustellen: Aktivieren Sie diese Option, um es Netzwerkgeräten, die sich im selben lokalen Netzwerk befinden wie Ihr DiskStation, zu ermöglichen, eine Verbindung zum selben VPN-Server herzustellen.
    • Wieder verbinden, wenn die VPN-Verbindung verloren wurde: Wenn die VPN-Verbindung unerwartet verloren geht, versucht das System, die Verbindung fünf Mal alle 30 Sekunden wiederherzustellen.
  7. Klicken Sie auf Übernehmen.

L2TP/IPSec-Profil erstellen:

  1. Wählen Sie, wenn Sie aufgefordert werden, eine VPN-Verbindungsmethode auszuwählen, L2TP/IPSec aus und klicken Sie auf Weiter.
  2. Geben Sie dem Profil einen Namen, geben Sie die Adresse des VPN-Servers an, mit dem Sie eine Verbindung herstellen möchten, und geben Sie den Benutzernamen und das Kennwort für den Server ein. Sie müssen auch den Pre-Shared Key für den VPN-Server eingeben. Klicken Sie anschließend auf Weiter.

    3. Hinweis:

    • Benutzername, Kennwort, Adresse und Pre-Shared Key des VPN-Servers sollten vom Administrator des VPN-Servers zur Verfügung gestellt werden.
  3. Wählen Sie eines der folgenden Authentifizierungsprotokolle im Dropdown-Menü Authentifizierung aus, um das Kennwort des VPN-Clients während der Authentifizierung zu schützen:
    • PAP: Das Kennwort wird nicht verschlüsselt.
    • CHAP: Das Kennwort wird mit CHAP (Challenge-Handshake Authentication Protocol) verschlüsselt.
    • MS CHAP: Das Kennwort wird mit Microsoft CHAP Version 1 verschlüsselt.
    • MS CHAP v2: Das Kennwort wird mit Microsoft CHAP Version 2 verschlüsselt.

    4. Hinweis:

    • Die Einstellungen für Authentifizierung und Verschlüsselung müssen mit den Einstellungen auf dem VPN-Server identisch sein. Einzelheiten erhalten Sie vom Administrator des VPN-Servers.
  4. Aktivieren Sie je nach Ihren Anforderungen eines der folgenden Kontrollkästchen:
    • Standard-Gateway auf Remote-Netzwerk verwenden: Aktivieren Sie diese Option, um den Netzwerkverkehr des DiskStation zum angegebenen VPN-Server zu leiten.
    • Anderen Netzwerkgeräten ermöglichen, Verbindung über die Internetverbindung dieses Synology Servers herzustellen: Aktivieren Sie diese Option, um es Netzwerkgeräten, die sich im selben lokalen Netzwerk befinden wie Ihr DiskStation, zu ermöglichen, eine Verbindung zum selben VPN-Server herzustellen.
    • Server befindet sich hinter NAT-Gerät: Standardmäßig lässt das L2TP/IPSec-Protokoll keine Verbindung zu einem VPN-Server hinter einem NAT-Gerät zu. Aktivieren Sie diese Option, um diese Einschränkung zu umgehen, damit der DiskStation die Verbindung zu einem L2TP/IPSec-VPN-Server hinter einem NAT-Gerät herstellen kann.
    • Wieder verbinden, wenn die VPN-Verbindung verloren wurde: Wenn die VPN-Verbindung unerwartet verloren geht, versucht das System, die Verbindung fünf Mal alle 30 Sekunden wiederherzustellen.
  5. Klicken Sie auf Übernehmen.

Verbindung zu einem VPN-Server herstellen oder trennen:

  1. Wählen Sie ein VPN-Profil aus.
  2. Klicken Sie auf Verbinden oder Trennen.

Hinweis:

  • Es kann nicht mehr als ein Profil gleichzeitig verbunden werden.

VPN-Profil ändern:

  1. Wählen Sie ein VPN-Profil aus.
  2. Klicken Sie auf Bearbeiten.

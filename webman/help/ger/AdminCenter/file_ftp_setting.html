FTP/FTPS

Unter Systemsteuerung > Dateidienste > FTP können Sie FTP-Dienste aktivieren, um Clients den Zugriff auf Ordner und Daten des Systems zu erlauben. Bevor Sie FTP aktivieren, müssen Sie sicherstellen, dass die folgenden TCP-Ports Ihres Routers zum DiskStation weitergeleitet werden: 21 (Standard-Steuerungsverbindung), 20 (Datenverbindung für aktiven Modus) und 55536-55567 (Datenverbindung für passiven Modus).

FTP-Dienst aktivieren:

  1. Aktivieren Sie eine der folgenden Optionen:
    • FTP-Dienst aktivieren (keine Verschlüsselung): Dies ist das Standard-Netzwerkprotokoll, das für die Übertragungsdateien verwendet wird. FTP bietet während der Übertragungssitzung keine Verschlüsselung zum Schutz von Informationen, z. B. von Kennwörtern, Benutzernamen oder Dateien. Dafür sind die Übertragungsgeschwindigkeiten höher und benötigen weniger Systemressourcen.
    • FTP SSL/TLS-Verschlüsselungsdienst (FTPS) aktivieren: Eine Weiterentwicklung von Standard-FTP mit zusätzlicher Unterstützung der Verschlüsselungsprotokollen Transport Layer Security (TLS) und Secure Sockets Layer (SSL), die Informationen wie z. B. Kennwörter, Benutzernamen oder Dateien während der Übertragungssitzung schützen. Die Übertragungsgeschwindigkeiten sind allerdings geringer und wegen der Verschlüsselung werden mehr Systemressourcen benötigt.
  2. Klicken Sie auf Übernehmen.

Timeout

Der FTP-Server trennt die Verbindung zu Benutzern ohne Aktivität nach dem angegebenem Zeitraum.

Portnummer-Einstellung des FTP-Dienstes

In diesem Bereich können Sie den für den FTP-Steuerkanal verwendeten Port angeben.

Hinweis:

Die voreingestellte Portnummer für den FTP-Steuerkanal ist 21. Er kann auf 1 bis 65535 eingestellt werden, ausgenommen folgende Zahlen: 20, 22, 23, 25, 80, 110, 137, 138, 139, 143, 199, 443, 445, 515, 543, 548, 587, 873, 993, 995, 3306, 3689, 5000, 5001, 5005, 5006, 5335, 5432, 6881, 8080, 7000, 7001, 8081, 9997, 9998, 9999, 50001, 50002 und eMule-Standard-Ports: 4662 (TCP), 4672 (UDP).

Portbereich des passiven FTP

In diesem Bereich können Sie den Portbereich festlegen, der für die passiven FTP-Verbindungen verwendet wird. Sie können den Standard-Portbereich verwenden oder einen benutzerdefinierten Bereich angeben. Der Portbereich kann zwischen 1025 und 65535 eingestellt werden und bis zu 128 Ports enthalten.

Externe IP im PASV-Modus melden

Durch Aktivieren dieser Option meldet der Server seine externe IP-Adresse an die FTP-Clients. Die gemeldete IP-Adresse wird im FTP-Client-Verbindungsprotokoll angezeigt. Nach der Aktivierung können Sie die automatisch erkannte externe IP-Adresse auswählen oder die externe IP-Adresse der DiskStation manuell eingeben.

Diese Option funktioniert nur, wenn sich die DiskStation hinter einem NAT befindet und sich die FTP-Clients in einem anderen Teilnetz als die DiskStation befinden. Meistens ist diese Option nicht erforderlich, aber falls FTP-Clients keine Verbindung zur DiskStation herstellen können, können Sie diese Option aktivieren und es erneut versuchen.

FXP aktivieren

FXP steht für File eXchange Protocol. Durch Aktivieren dieser Option unterstützt der FTP-Dienst die Server-zu-Server-Dateiübertragungsfunktion.

Aktivierung des FIPS-Verschlüsselungsmoduls

FIPS wird beim FTPS-Dienst unter Verwendung des nach FIPS 140-2 validierten OpenSSL-Verschlüsselungsmoduls (Zertifikat Nr. 1747) unterstützt. Durch Aktivieren dieser Option werden zusätzliche sichere Verschlüsselungsalgorithmen vom FTPS-Dienst verwendet, um die Sitzungen noch sicherer zu machen.

Unterstützung des ASCII-Übertragungsmodus

Aktivieren Sie diese Option, um den ASCII-Übertragungsmodus zu unterstützen, damit die DiskStation mit ASCII formatierte Textdateien korrekt über FTP überträgt.

UTF-8-Codierung

Diese Option bewirkt, dass FTP-Clients Dateien mit mehrsprachigen Namen korrekt anzeigen und behandeln. Um die Anforderungen unterschiedlicher FTP-Clients zu erfüllen, stehen die folgenden drei Optionen zur Verfügung:

  • Deaktivieren: UTF-8-Codierung wird nicht verwendet. Diese Option ist für Clients vorgesehen, die Windows File Explorer als FTP-Client verwenden, um Dateien mit mehrsprachigen Namen hoch-/herunterzuladen. Für die normale Verwendung wird sie nicht empfohlen.
  • Automatisch: DSM bestimmt automatisch die Behandlung der UTF-8-Codierung. Diese Option wird für die normale Verwendung empfohlen.
  • Erzwingen: Ermöglicht das Erzwingen der Verwendung der UTF-8-Codierung. Wählen Sie diese Option aus, wenn Sie die UTF-8-Codierung auf Ihrem FTP-Client aktiviert haben und Dateien mit mehrsprachigen Namen noch immer nicht korrekt angezeigt werden.

Verbindungseinschränkungen

Dieser Bereich enthält die folgenden Optionen.

  • Verbindungen pro IP beschränken: Wenn diese Funktion aktiviert ist, beschränkt der Server die maximalen Verbindungen pro IP. Aktivieren Sie diese Funktion und die FTP-Bandbreitenbeschränkung, um eine effektive Bandbreitenverwaltung zu erreichen.
  • FTP-Bandbreitenbeschränkung aktivieren: Aktivieren Sie diese Funktion, um die Bandbreite einer einzelnen FTP-Verbindung zu beschränken.

Hinweis:

  • Das „guest“-Benutzerkonto kann sich nicht über FTP beim Server anmelden.
  • Die Einstellung für die Codepage auf dem FTP-Client muss für einen korrekten Zugriff auf die Daten die gleiche wie beim DiskStation sein.
  • Bei einigen bestimmten Modellen unterstützt der FTP-Dienst die Verbindung mit SSL oder TLS.
  • Die Firewall-Einstellungen Ihres FTP-Client und der zwei FTP-Server müssen richtig konfiguriert sein, da die FXP-Funktion ansonsten nicht gut funktioniert.
  • Bestimmte FTP-Clients, wie z. B. solche, die Internet Explorer 8/9 verwenden, müssen sich mit ihrem Benutzernamen und Kennwort anmelden. Beispiel: ftp://username:password@your.ftp.com.

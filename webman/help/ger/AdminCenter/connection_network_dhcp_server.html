DHCP Server

Ihre DiskStation kann als DHCP-Server fungieren und den DHCP-Clients innerhalb Ihres lokalen Netzwerks dynamische IP-Adressen zuweisen.

DHCP-Server aktivieren:

  1. Gehen Sie zu Systemsteuerung > DHCP Server > Netzwerkschnittstelle.
  2. Wählen Sie eine Netzwerkschnittstelle (z. B. LAN 1) aus, um ihr IP-Adressen zuzuweisen, und klicken Sie auf Bearbeiten.
  3. Klicken Sie im Popup-Fenster auf die Registerkarte DHCP Server und wählen Sie DHCP-Server aktivieren aus.
  4. Nehmen Sie die folgenden DHCP-Einstellungen vor:
    • Lease-Time von Adresse (Minute)
    • Primäres DNS
    • Sekundäres DNS (optional)
    • Domainname (optional)
    • Automatische Web Proxy-Erkennung aktivieren (optional)
  5. Richten Sie mindestens ein Subnetz ein. Klicken Sie zum Einrichten eines Subnetzes auf Hinzufügen, um folgende Informationen anzugeben:
    • Start-IP-Adresse
    • End-IP-Adresse
    • Netzmaske
    • Gateway
  6. Klicken Sie auf Erstellen, um das erstellte Subnetz in der Liste anzuzeigen.
  7. Nachdem Sie ausreichend viele Subnetze eingerichtet haben, müssen Sie mindestens eines aktivieren.
  8. Klicken Sie auf OK.

Hinweis:

  • Sie können so viele Subnetze erstellen, wie Sie möchten.
  • Sofern mit einem DHCP Relay-Server koordiniert, kann Ihre DiskStation als DHCP-Server Geräten in einem anderen Subnetz IP-Adressen zuweisen.

DHCP Clients

Sie können eine Liste mit DHCP-Clients und deren Netzwerkkonfigurationen (z. B. MAC-Adresse, IP-Adresse, Hostname, die Frist bis zum Ablauf des DHCP-Leasing und den Reservierungsstatus) anzeigen.

  1. Gehen Sie zu Systemsteuerung > DHCP Server > Netzwerkschnittstelle und wählen Sie die gewünschte Schnittstelle aus.
  2. Klicken Sie auf Bearbeiten > DHCP Clients, um die Clients und Details zu finden.

IP-Adressen für DHCP-Clients verwalten:

Sie können einstellen, dass ein Client bei jeder bei Erneuerung des DHCP-Leasings die gleiche IP-Adresse erhält, und können diese Clients verwalten.

  • DHCP-Clients hinzufügen: Klicken Sie auf Hinzufügen, um dem Client-Gerät die IP-Adresse zuzuweisen.
  • IP-Adresse reservieren: Klicken Sie auf das Schlosssymbol, um die IP-Adresse für das aktuelle Gerät zu reservieren.
  • Reservierung abbrechen: Klicken Sie auf das Schlosssymbol, um es zuzulassen, dass die aktuelle IP-Adresse nach Ablauf von deren Lease-Zeit von einem anderen Gerät verwendet wird.

Hinweis:

Wenn ein Gerät sowohl IPv4- als auch IPv6-Adressen haben kann, können Sie eine IPv4-Adresse und eine IPv6-Adresse für das Gerät reservieren.

DHCP-Clients suchen:

  1. Klicken Sie rechts oben auf das Trichtersymbol.
  2. Wählen Sie eines der nachstehenden Suchkriterien aus:
    • Alle: Listet alle DHCP-Clients und DHCP-Reservierungen auf.
    • DHCP-Clients: Listet alle DHCP-Clients mit reservierten und nicht reservierten IP-Adressen auf.
    • DHCP-Reservierungen: Listet alle reservierten IP-Adressen auf. Eine reservierte IP-Adresse ist möglicherweise noch nicht mit einem Gerät gepaart.

DHCP-Option

Nachdem Sie die DHCP-Optionen festgelegt haben, kann Ihre DiskStation als DHCP-Server den DHCP-Clients in einem bestimmten Subnetz zugehörige Informationen senden.

DHCP-Optionen für ein Subnetz festlegen:

  1. Gehen Sie zu Systemsteuerung > DHCP Server > Netzwerkschnittstelle.
  2. Wählen Sie die gewünschte Netzwerk-Schnittstelle aus und klicken Sie auf Bearbeiten.
  3. Gehen Sie zu DHCP Server und zur Teilnetz-Liste.
  4. Führen Sie einen der folgenden Schritte durch:
    • DHCP-Optionen für ein bestehendes Subnetz festlegen: Wählen Sie die gewünschte Schnittstelle und klicken Sie auf Bearbeiten.
    • Ein Subnetz erstellen und seine DHCP-Optionen festlegen: Klicken Sie auf Hinzufügen und legen Sie die Subnetzeinstellungen fest.
  5. Wählen Sie die gewünschten DHCP-Optionen aus; klicken Sie auf die leeren Felder, um sie zu bearbeiten und die entsprechenden Werte festzulegen.
  6. Klicken Sie auf OK.

Anbieter

DHCP-Clients verschiedener Anbieter benötigen unter Umständen noch weitere DHCP-Informationen. Wenn Sie die DHCP-Optionen für den jeweiligen Anbieter festlegen, kann Ihre DiskStation den Clients diese Informationen übermitteln.

Bitte kontaktieren Sie den jeweiligen Anbieter, um die von diesem unterstützten DHCP-Optionen zu erfahren.

Anbieter hinzufügen:

  1. Gehen Sie zu Systemsteuerung > DHCP Server > Anbieter.
  2. Klicken Sie auf Erstellen, um den Anbieter und seine DHCP-Optionen festzulegen:
    • Name: Geben Sie den Namen des Anbieters an.
    • Code: Geben Sie den numerischen Code der DHCP-Option an.
    • Wert: Geben Sie den Wert für die Option ein.
    • +: Eine DHCP-Option hinzufügen.
    • -: Eine DHCP-Option löschen.
  3. Klicken Sie auf Erstellen, um das Setup abzuschließen.

DHCP-Optionen für Anbieter bearbeiten:

  1. Gehen Sie zu Systemsteuerung > DHCP Server > Anbieter.
  2. Wählen Sie den gewünschten Anbieter aus und klicken Sie auf Bearbeiten, um seine DHCP-Optionen zu verwalten.
  3. Klicken Sie auf OK.

PXE

Ihre DiskStation kann als DHCP-Server fungieren, um die Einrichtung von PXE (Pre-Boot Execution Environment) zu implementieren. PXE ist eine Netzwerkumgebung, die es Client-Geräten ermöglicht, über eine Netzwerkschnittstelle anstatt über ein Datenspeichergerät (z. B. eine Festplatte) zu starten.

Wenn PXE implementiert ist, kann ein Bootloader auf Ihrer DiskStation oder einem Remote-TFTP-Server gespeichert werden und Client-Geräte können über das Netzwerk mit dem Bootloader gestartet werden.

Ihre DiskStation als Bootloader-Server verwenden:

  1. Vergewissern Sie sich, dass TFTP-Dienst aktivieren ausgewählt ist (unter Systemsteuerung > Dateidienste > TFTP).
  2. Wählen Sie einen Ordner für die Bootloader-Datei als TFTP-Root-Ordner aus.
  3. Vergewissern Sie sich, dass mindestens ein DHCP-Subnetz aktiviert ist (unter Systemsteuerung > DHCP Server > Netzwerkschnittstelle).
  4. Gehen Sie zu Systemsteuerung > DHCP Server > PXE.
  5. Wählen Sie PXE aktivieren (Pre-boot Execution Environment) aus.
  6. Wählen Sie Lokaler TFTP-Server und die Bootloader-Datei aus.
  7. Klicken Sie auf Übernehmen, um die Einstellungen zu speichern.

Einen Remote-TFTP-Server als Bootloader-Server verwenden:

Wenn Sie einen Remote-TFTP-Server als Bootloader-Server verwenden, können Sie Clients das Hochfahren erlauben, ohne dass Sie die aktuelle Netzwerkinfrastruktur ändern müssen.

  1. Vergewissern Sie sich, dass mindestens ein DHCP-Subnetz aktiviert ist (unter Systemsteuerung > DHCP Server > Netzwerkschnittstelle).
  2. Gehen Sie zu Systemsteuerung > DHCP Server > PXE.
  3. Wählen Sie PXE aktivieren (Pre-boot Execution Environment) aus.
  4. Wählen Sie Remote-TFTP-Server aus und geben Sie die IP-Adresse des Servers und den Dateipfad der Bootloader-Datei an.
  5. Klicken Sie auf Übernehmen, um die Einstellungen zu speichern.

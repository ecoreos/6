Volume

Volumes sind grundlegende Speicherbereiche, in denen der Benutzer gemeinsame Ordner, iSCSI LUN (reguläre Datei) erstellen, Daten speichern oder Pakete installieren kann. Vergewissern Sie sich vor der Erstellung von Volumes, dass Sie bereits eine RAID-Gruppe erstellt haben. Weitere Informationen zu RAID-Gruppen finden Sie unter RAID-Gruppe.

Volume erstellen:

  1. Stellen Sie sicher, dass Ihr DiskStation über eine bestehende RAID-Gruppe mit zuweisbarem Speicher verfügt.
  2. Klicken Sie auf die Schaltfläche Erstellen.
  3. Wählen Sie eine RAID-Gruppe aus, in der Sie das Volume erstellen möchten.
  4. Ändern Sie die gewünschten Einstellungen.
  5. Stellen Sie die Zuordnungsgröße für das Volume ein. (Die Einstellung der Größenzuordnung ist nicht verfügbar, wenn Volumes in RAID-Gruppen für einzelne Volumes oder iSCSI LUN erstellt wurden.)

Hinweis:

  • Die maximal zuzuordnende Volume-Größe beträgt 200 TB.
  • Einem Btrfs-Volume muss eine Größe von mindestens 10 GB zugewiesen werden.
  • Btrfs-Volumes reservieren 4 % des Speicherplatzes für Metadaten.

Volume entfernen:

  1. Wählen Sie das Volume aus, das Sie löschen möchten.
  2. Klicken Sie auf die Schaltfläche Entfernen und folgen Sie dem Assistenten, um den Vorgang durchzuführen.

Hinweis:

  • Durch Entfernen eines Volumes werden sämtliche Daten auf dem Volume gelöscht.

Volume zwangsweise anhängen:

Wenn der Cache auf einem Volume fehlt, wird das Volume beim Hochfahren vom System nicht angehängt. Wenn der Cache nicht wiederherstellbar ist und Sie auf das Volume zugreifen möchten, können Sie auf Zwangsweise installieren klicken.

Hinweis:

  • Diese Funktion wird nur bei bestimmten Modellen unterstützt.

Volume reparieren:

Gehen Sie, wenn der Volume-Status Fehlerhaft ist, zur Registerkarte RAID-Gruppe und suchen Sie die entsprechende RAID-Gruppe. Folgen Sie anschließend den vorgeschlagenen Anweisungen, um es zu reparieren.

Größe eines Volumes erweitern:

  • Für Volumes, die in einer RAID-Gruppe für mehrere Volumes oder iSCSI LUNs (Block-Niveau) erstellt wurden:
    1. Wählen Sie das Volume aus, das Sie erweitern möchten.
    2. Klicken Sie auf Bearbeiten.
    3. Geben Sie die neue Größe ein und klicken Sie zum Erweitern auf OK.
  • Bei Volumes, die in einer RAID-Gruppe für einzelne Volumes oder iSCSI LUN (Block-Niveau) erstellt wurden, lesen Sie bitte den Abschnitt Größe einer RAID-Gruppe erweitern unter RAID-Gruppe. Die Größe des Volumes wird erweitert, wenn die betreffende RAID-Gruppe erweitert wurde.

SSD TRIM

Wenn ein Volume ausschließlich aus SSDs (Solid State Drives) besteht, empfehlen wir die Aktivierung von SSD TRIM. Diese Funktion verbessert die Lese- und Schreibleistung von Volumes, die auf SSDs erstellt wurden, wodurch die Effizienz Ihrer SSDs gesteigert und deren Lebensdauer verlängert wird.

SSD TRIM aktivieren:

  1. Öffnen Sie Speicher-Manager und gehen Sie zu Volume.
  2. Wählen Sie ein Volume aus, das ausschließlich aus SSDs besteht, und klicken Sie auf die Schaltfläche Konfigurieren.
  3. Gehen Sie im angezeigten Fenster zur Registerkarte SSD TRIM und aktivieren Sie das Kontrollkästchen TRIM aktivieren.
  4. Klicken Sie auf die Schaltfläche Zeitplan einstellen, um festzulegen, wann das System SSD TRIM ausführt.
  5. Klicken Sie auf OK.

Hinweis:

  • SSD TRIM ist nur auf bestimmten DiskStation-Modellen und RAID-Typen verfügbar.
  • Einige SSD-Modelle unterstützen die Ausführung von SSD TRIM auf RAID 5- und RAID 6-Volumes nicht. Beachten Sie die Kompatibilitätsliste auf www.synology.com.
  • SSD TRIM kann in den folgenden Situationen nicht aktiviert werden:
    • Das Volume wurde mit SSD Cache angehängt.
    • Das Paket Time Backup ist installiert.
    • Ein High-Availability-Cluster wurde mit dem High Availability Manager erstellt.
  • SSD TRIM kann nicht auf Block-Niveau-iSCSI LUNs aktiviert werden.
  • Die Systemleistung wird durch das Ausführen von SSD TRIM beeinträchtigt. Die benötigte Zeit bis zum Ende der TRIM-Ausführung variiert je nach Volume-Größe und SSD-Modell.
  • Wenn TRIM auf einem Volume aktiviert ist, wird es auch auf anderen Volumes der gleichen Diskgruppe oder RAID-Gruppe aktiviert.

Dateisystembereinigung*

Bei der Dateisystembereinigung handelt es sich um eine Wartungsfunktion für Dateien, bei der Volumes oder Diskgruppen überprüft werden. Wenn Daten gefunden werden, die nicht mit der Prüfsumme übereinstillen, wird der Dateipfad im Protokoll-Center festgehalten und es wird empfohlen, eine Sicherung zu verwenden, um die Daten zu reparieren.

Wir empfehlen, die Dateisystembereinigung regelmäßig durchzuführen, um die Datenkonsistenz sicherzustellen.

Dateisystembereinigung durchführen:

  1. Öffnen Sie Speicher-Manager und gehen Sie zu Volume.
  2. Wählen Sie das Volume aus, auf dem die Dateisystembereinigung durchgeführt werden soll.
  3. Klicken Sie auf Verwalten.
  4. Wählen Sie Dateisystembereinigung aus.

Hinweis:

  • Die Dateisystembereinigung kann nur auf Volumes im Btrfs-Dateisystem durchgeführt werden.
  • Die Dateisystembereinigung kann nur durchgeführt werden, wenn das Volume den Status Normal hat.
  • Fahren Sie das System nicht herunter, bevor die Dateisystembereinigung abgeschlossen ist. Wenn das System vor der Fertigstellung heruntergefahren wird, müssen Sie die Dateisystembereinigung manuell von vorn beginnen.

Dateisystem-Defragmentierung*

Wenn Ihre Daten kontinuierlich geändert werden, nimmt die Anzahl der fragmentierten Dateien im Dateisystem zu, wodurch der Zugriff auf die Dateien auf den Festplatten verlangsamt wird. Mit der Dateisystem-Defragmentierung können Sie die Leistung verbessern.

Dateisystem-Defragmentierung durchführen:

  1. Öffnen Sie Speicher-Manager und gehen Sie zu Volume.
  2. Wählen Sie das Volume aus, auf dem die Dateisystem-Defragmentierung durchgeführt werden soll.
  3. Klicken Sie auf Verwalten.
  4. Wählen Sie Dateisystem-Defragmentierung aus.

Hinweis:

  • Die Dateisystem-Defragmentierung kann nur auf Volumes im Btrfs-Dateisystem durchgeführt werden.
  • Die Dateisystem-Defragmentierung kann nur durchgeführt werden, wenn das Volume den Status Normal hat.
  • Fahren Sie das System nicht herunter, bevor die Dateisystem-Defragmentierung abgeschlossen ist. Wenn das System vor der Fertigstellung heruntergefahren wird, müssen Sie die Dateisystem-Defragmentierung manuell von vorn beginnen.
  • Die Volume-Auslastung könnte sich erhöhen, wenn das Volume Schnappschüsse von freigegebenen Ordnern enthält.

iSCSI LUN*

iSCSI ist ein Speichernetzwerkdienst, der bei der Übertragung von Daten über lokale Netzwerke oder lange Distanzen behilflich ist. Bei der Zuordnung von iSCSI LUNs zu iSCSI-Ziele haben Clients möglicherweise Zugriff auf Speicherbereiche, z. B. die lokalen Laufwerke. Eine iSCSI LUN (logische Einheitsnummer) kann einem oder mehreren iSCSI-Zielen zugewiesen werden, um iSCSI-Speicherdienste anzubieten. Gehen Sie zu Speicher-Manager > iSCSI LUN, um iSCSI LUNs zu verwalten.

Hinweis:

Die Anzahl unterstützter iSCSI LUNs unterscheidet sich je nach DiskStation-Modell. Detaillierte Produktspezifikationen finden Sie auf www.synology.com.

Der DiskStation unterstützt derzeit die folgenden iSCSI LUN-Typen:

  • iSCSI LUN (reguläre Dateien): Die iSCSI LUN wird erstellt, indem ein Teil des Volume-Speicherplatzes für ihre Nutzung zugewiesen wird. Sie können die Größe später bei Bedarf erhöhen. Dieser iSCSI LUN-Typ bietet die Flexibilität des dynamischem Kapazitätsmanagements mit Thin Provisioning, außerdem unterstützt er VMware VAAI Hardware-Beschleunigung, Windows ODX, LUN-Schnappschuss und LUN-Klon.
  • iSCSI LUN (Block-Niveau) – Eine LUN auf RAID: Die iSCSI LUN wird erstellt, indem der gesamte Speicherplatz eines Volumes verwendet wird. Sie können sie reparieren oder erweitern oder seinen RAID-Typ ändern, als würden Sie ein Volume benutzen. Weitere Informationen finden Sie auf der Seite Volume.
  • iSCSI LUN (Block-Niveau) - Mehrere LUNs auf RAID: Die iSCSI LUN wird erstellt, indem ein bestimmter Speicherplatz einer Diskgruppe zugewiesen wird. Sie können die iSCSI LUN später erweitern, wenn auf der Diskgruppe noch Speicherplatz verfügbar ist. Weitere Informationen finden Sie auf der Seite Diskgruppe.
  • iSCSI LUN (Block-Niveau): Dieser iSCSI LUN-Typ wird auf einer RAID-Gruppe erstellt und bietet zur optimierten Zugriffsleistung zusätzlich die Flexibilität des dynamischen Kapazitätsmanagements. Sie können diese iSCSI LUN später ggf. durch Nutzung nicht zugewiesenen Speichers in der RAID Group erweitern. Weitere Informationen finden Sie auf der Seite RAID-Gruppe.

Hinweis:

iSCSI LUNs mit erweiterten LUN-Funktionen sind nur im ext4/Btrfs-Dateisystem verfügbar.

iSCSI LUN erstellen:

  1. Klicken Sie auf Erstellen.
  2. Wählen Sie den zu erstellenden iSCSI LUN-Typ aus.
  3. Folgen Sie dem Assistenten, um den Vorgang abzuschließen.

Hinweis:

Erweiterte LUN-Funktionen können nur während der erstmaligen iSCSI LUN-Erstellung aktiviert werden. Diese Einstellung kann später nicht geändert werden.

Die erweiterte LUN-Größe der Zuordnungseinheit kann für bestimmte Funktionen für die Anwendungsserver, wie Microsoft Exchange oder SQL Server, oder Virtualisierungs-Server und Speicherbeschleunigungstechnologien, wie zum Beispiel Microsoft Windows ODX oder VMware VAAI, optimiert werden, indem Sie eine der folgenden Möglichkeiten ausführen:

  • Legen Sie die Größe der Zuordnungseinheit während der Erstellung der ersten erweiterten LUN eines Volumes fest.
  • Legen Sie die Größe der Zuordnungseinheit während der Erstellung der ersten erweiterten LUN eines Volumes unter Konfigurieren fest.

Wir empfehlen Ihnen, die Größe der Zuordnungseinheit für Ihre Anwendung festzulegen, wenn Sie die direkte Verbindung von Ihrem Anwendungsserver oder die Pass-Through-Methoden (RDM) mit Advanced iSCSI LUNs auf dem DiskStation verwenden. Um die Synology Advanced LUN-Technologie für das Klonen auf Windows ODX auszunutzen, empfehlen wir Ihnen, das NTFS-Volume mit einer Größe der Zuordnungseinheit zu formatieren, die gleich der ist (oder größer), die Sie auf DSM festgelegt haben.

Hinweis:

Die Advanced LUN-Zuordnungseinheit kann für jedes Volume nur einmal festgelegt werden.

iSCSI LUN entfernen:

Wenn Sie eine iSCSI LUN entfernen, werden alle Daten in der iSCSI LUN gelöscht. Verbindungen zwischen der iSCSI LUN und zugeordneten iSCSI-Zielen gehen verloren.

  1. Wählen Sie die zu entfernende iSCSI LUN aus.
  2. Klicken Sie auf die Schaltfläche Entfernen und folgen Sie dem Assistenten, um den Vorgang abzuschließen.

Hinweis:

Wenn VAAI LUNs auf dem DiskStation gelöscht werden, ist nicht sofort mehr Speicherplatz verfügbar. Die Daten werden in einen vom System reservierten Ordner namens @EP_trash verschoben und nach und nach gelöscht. Wenn zum Beispiel eine VAAI LUN mit 100 GB vom DiskStation gelöscht wird, dauert es ungefähr 15-20 Minuten, um die 100 GB Speicherplatz auf dem Datenträger-Volume freizugeben.

iSCSI LUN (Block-Niveau) zwangsweise installieren:

Wenn der Cache auf einer iSCSI LUN (Block-Niveau) fehlt, wird die iSCSI LUN (Block-Niveau) beim Hochfahren vom System nicht bereitgestellt. Wenn der Cache nicht wiederherstellbar ist und Sie auf die iSCSI LUN (Block-Niveau) zugreifen möchten, können Sie auf Zwangsweise installieren klicken.

Hinweis:

Diese Funktion wird nur bei bestimmten Modellen unterstützt.

iSCSI LUN bearbeiten:

  1. Wählen Sie die zu bearbeitende iSCSI LUN aus.
  2. Klicken Sie auf Bearbeiten.
  3. Bearbeiten Sie die gewünschten Eigenschaften.
  4. Klicken Sie auf OK, um die Einstellungen zu speichern.

Hinweis:

Um einen Datenverlust beim Bearbeiten der iSCSI LUN-Kapazität zu vermeiden, können Sie die Kapazität nur vergrößern.

iSCSI LUN-Schnappschuss erstellen:

Ein Schnappschuss einer iSCSI LUN ist eine zu einem bestimmten Zeitpunkt erstellte schreibgeschützte Kopie einer iSCSI LUN.

  1. Wählen Sie die iSCSI LUN aus, für die Sie Schnappschüsse erstellen möchten.
  2. Klicken Sie auf Schnappschuss und wählen Sie Einen Schnappschuss machen aus.
  3. Geben Sie eine Beschreibung für diesen Schnappschuss ein.
  4. Wählen Sie je nach Bedarf einen der folgenden Schnappschuss-Typen aus:
    • Anwendungskonsistent: Dieser Schnappschuss-Typ kann nur erstellt werden, wenn Synology Snapshot Manager je nach Ihrer Betriebsumgebung für die Kommunikation mit DSM auf Ihrem Windows-Server oder dem VMware vCenter-Server installiert ist. Wenn ein anwendungskonsistenter Schnappschuss in DSM ausgelöst wird, werden Anwendungen benachrichtigt, die daraufhin alle Daten aus dem Speicher zur LUN übertragen, auf der der Schnappschuss erstellt wird, um die Konsistenz der Daten zu gewährleisten.
    • Absturzkonsistent: Wenn auf keinem Ihrer Clients Synology Snapshot Manager installiert ist, können Sie nur absturzkonsistente Schnappschüsse von LUNs erstellen. Wenn diese ausgelöst werden, werden Anwendungen nicht angewiesen, Daten vom Speicher zur LUN zu übertragen, sodass die Datenkonsistenz nicht gewährleistet ist.
  5. Klicken Sie auf OK.

iSCSI LUN wiederherstellen:

Wählen Sie einen Schnappschuss zu einem bestimmten Zeitpunkt aus, um die Daten auf einer iSCSI LUN wiederherzustellen.

  1. Wählen Sie die wiederherzustellende iSCSI LUN aus.
  2. Klicken Sie auf Schnappschuss und wählen Sie Snapshot Manager aus.
  3. Wählen Sie einen Schnappschuss aus, den Sie wiederherstellen möchten, und klicken Sie auf Wiederherstellen.

iSCSI LUN oder Schnappschuss klonen:

Erstellen Sie eine beschreibbare Kopie einer iSCSI LUN oder eines iSCSI LUN-Schnappschusses.

  • Wählen Sie eine iSCSI LUN aus der Liste aus und klicken Sie auf Klonen.
  • Wählen Sie einen iSCSI LUN-Schnappschuss aus dem Schnappschuss Manager aus und klicken Sie auf Klonen.

Hinweis:

  • Bei bestimmten Modellen müssen Sie zum Paketzentrum gehen und Schnappschuss & Replikation installieren, um Funktionen, wie etwa das Wiederherstellen von iSCSI LUNs oder das Aufnehmen und Klonen von iSCSI LUN-Schnappschüssen nutzen zu können.
  • Wenn mindestens eine verbundene iSCSI-Sitzung mit der LUN verknüft ist und vom Windows-Server oder vom VMware ESXi-Server verwendet wird, empfehlen wir die Installation des Plugins Synology Snapshot Manager, um mögliche Inkonsistenzen zu vermeiden. Andernfalls müssen Sie alle Anwendungen beenden und diese Sitzungen von den Initiatoren abmelden, bevor Sie fortfahren können.
  • Die Funktionen "LUN-Schnappschuss", "Wiederherstellen" und "Klonen" sind nur auf iSCSI LUNs mit erweiterten LUN-Funktionen verfügbar.
  • Wenn Sie einen Schnappschuss oder einen Klon erstellt haben und der Status Abnormal/Abgestürzt ist, müssen Sie diesen Schnappschuss/Klon löschen und einen anderen erstellen.
  • Alle an einer LUN vorgenommenen Änderungen gehen nach der Erstellung eines Schnappschusses verloren, wenn Sie die LUN mit diesem Schnappschuss wiederherstellen.
  • Wenn der Status der iSCSI LUN zu Nicht verfügbar wechselt, bedeutet dies, dass datenbezogene Fehler während des iSCSI LUN-Zugriffs aufgetreten sind. Um einen Datenverlust auf Ihrem System zu verhindern, ist diese iSCSI LUN vorübergehend nicht verfügbar. Wenden Sie sich bitte so schnell wie möglich an Synology, um Hilfe zu erhalten.

Glossar:

  • Thin Provisioning: Thin Provisioning ist eine Methode für die Optimierung der Speicherauslastung durch die dynamische und bedarfsabhängige Zuweisung von Speicherplatz. Diese Einstellung ist nur für iSCSI LUN (reguläre Dateien) verfügbar.
  • VMware VAAI: Die VMware vStorage APIs für Array Integration (VAAI) können die Arbeit der Standardfunktionen auf den virtuellen Speicher-Arrays auf das iSCSI-Gerät in der VMware vSphere-Umgebung auslagern und die Speicherleistung optimieren.
  • Windows ODX: Eine neue Datenübertragungstechnologie von Windows auf Windows Server 2012 und 8, die Offload Data Transfer (Offload-Datenübertragung – ODX) genannt wird. ODX verbessert die Übertragungsleistung durch Verlagerung eines Teils der Arbeitslast auf das Speicher-Array.
    Wenn die Quell- und Zieldaten sich auf LUNs auf demselben Volume befinden, wird das Klonen von der Synology Advanced LUN-Technologie verarbeitet, die bis zu 99,9 % des Speicherplatzbedarfs einspart und gleichzeitig das Klonen beschleunigt.
  • Zuordnung iSCSI-Ziel: Ordnet die iSCSI LUN einem oder mehreren iSCSI-Zielen zu.
  • Synology Snapshot Manager: Synology Snapshot Manager ist ein Software-Plugin, mit dem Sie anwendungskonsistente Schnappschüsse direkt von DiskStation Manager (DSM) aus erstellen können.
    • Für VMware-Umgebungen müssen Sie Synology Snapshot Manager auf dem Windows-Server installieren, auf dem der VMware vCenter-Server installiert ist. Wenn ein Schnappschuss in DSM ausgelöst wird, wird der VMware vCenter-Server benachrichtigt und überträgt alle Daten aus dem Speicher auf die LUN, auf dem der Schnappschuss erstellt wird, um die Konsistenz der Daten zu gewährleisten. Nachdem der Schnappschuss erstellt wurde, setzt der VMware vCenter-Server die normale E/A-Funktion des VMware-Datenspeichers fort. Synology Snapshot Manager für VMware vCenter-Server unterstützt vSphere 5.1 oder neuere Versionen.
    • Für eine Windows-Umgebung müssen Sie Synology Snapshot Manager auf dem Windows-Server installieren. Wenn ein Schnappschuss in DSM ausgelöst wird, nutzt Synology Snapshot Manager die Microsoft Volume Shadow Copy Service (VSS)-Technologie, um konsistente zeitpunktbezogene Kopien der Daten zu erstellen. Synology Snapshot Manager für Windows unterstützt Windows-Server 2008 R2, 2012 und 2012 R2.
    • Synology Snapshot Manager kann kostenlos von www.synology.com heruntergeladen werden.

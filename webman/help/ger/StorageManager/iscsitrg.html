iSCSI Target*

iSCSI ist eine Art Storage Area Networking (SAN)-Dienst, der den Zugriff auf konsolidierten Datenspeicher auf Block-Ebene ermöglicht. Er erleichtert die Datenübertagungen über Intranets und die Speicherverwaltung über lange Distanzen. Werden iSCSI LUNs iSCSI Targets zugeordnet, können Client-Server über ihren iSCSI-Initiator so auf Speicherbereiche zugreifen, als wären es lokale Festplatten. Je nach Modell unterstützt das Synology-Produkt die Erstellung von mehreren iSCSI Targets, die iSCSI LUNs zugewiesen werden können.

Hinweis: Die Anzahl unterstützter iSCSI Targets unterscheidet sich je nach Ihren Synology-Produkten. Detaillierte Produktspezifikationen finden Sie auf der Synology-Website.

iSCSI Target erstellen:

  1. Klicken Sie auf die Schaltfläche Erstellen.
  2. Geben Sie die Informationen zum iSCSI Target ein, wie Name, IQN und ob die CHAP-Authentifizierung aktiviert werden soll.
  3. Klicken Sie auf Weiter.
  4. Richten Sie die iSCSI LUN-Zuordnung ein. Wählen Sie Neue iSCSI LUN erstellen oder Vorhandene iSCSI LUNs zuordnen aus.
  5. Klicken Sie auf Weiter.
  6. Klicken Sie auf Zurück, um eine Änderung vorzunehmen, oder auf Übernehmen, um die Einstellungen zu bestätigen.

Erklärung der Begriffe

  • IQN: Bei IQN (iSCSI Qualified Name) handelt es sich um eine Gruppe eindeutiger Namen für ein iSCSI Target. Es darf nicht mit iSCSI Target IQNs auf anderen Servern in Konflikt stehen. IQN muss weniger als 128 Zeichen umfassen und darf Buchstaben (Groß- oder Kleinschreibung), Ziffern, Doppelpunkte, Bindestriche und Punkte enthalten. Das vorgeschlagene Benennungsformat ist iqn.[date].[domain]:[device identifier], also z. B.: iqn.2000-04.com.synology:storage.diskarray.1.
  • CHAP-Authentifizierung: Wenn Sie CHAP auswählen, kann der Server die Identität des Clients überprüfen. Der Benutzername kann 1-12 Buchstaben und Ziffern umfassen und das Kennwort kann aus 12-16 Buchstaben und Ziffern bestehen.
  • Gegenseitige CHAP-Authentifizierung: Unter Gegenseitiger CHAP authentifizieren sich Client und Server gegenseitig. Ein zur Client-Authentifizierung verwendetes CHAP-Kennwort darf nicht für die Authentifizierung eines Servers konfiguriert werden und ein zur Server-Authentifizierung verwendetes CHAP-Kennwort darf nicht für die Authentifizierung eines Clients konfiguriert werden.

iSCSI Target entfernen:

Sie können ein iSCSI Target und zugeordnete iSCSI LUNs entfernen. Denken Sie daran, dass bei der Entfernung einer iSCSI LUN alle Daten in der iSCSI LUN gelöscht werden.

  1. Wählen Sie das iSCSI Target aus, das Sie entfernen möchten.
  2. Klicken Sie auf Entfernen.
  3. Wählen Sie, wenn Sie zugeordnete iSCSI LUNs entfernen möchten, iSCSI LUN-Zuordnungen ebenfalls löschen aus und wählen Sie diejenigen aus, die Sie entfernen möchten.
  4. Klicken Sie auf OK.

iSCSI Target bearbeiten:

  1. Wählen Sie das iSCSI Target aus, das Sie bearbeiten möchten.
  2. Klicken Sie auf Bearbeiten.
  3. Bearbeiten Sie die Einstellungen nach Ihren Wünschen. Klicken Sie auf OK, um die Änderungen zu speichern.

Erklärung der Begriffe

  • CRC-Prüfsumme: Aktivieren Sie die CRC Prüfsumme, um zu prüfen, ob ein Fehler bei der Datenübertragung aufgetreten ist.
  • Mehrere Sitzungen: Ermöglicht mehrere Sitzungen von einem oder mehreren Initiatoren, die zur selben Zeit mit dem selben iSCSI Target verbunden sind. Um Ihre Dateien vor Beschädigung oder Verlust zu schützen, sollten Sie ein freigegebenes Datensystem oder Cluster-System wie z. B. VMware Virtual Machine File System (VMFS) und Oracle Cluster File System (OCFS) verwenden.
  • Maskierung: Mittels Maskierung kann auf ein iSCSI Target exklusiv mit angegebenen iSCSI Initiator IQNs zugegriffen werden.

iSCSI Target aktivieren oder deaktivieren:

  1. Wählen Sie das iSCSI Target aus, das Sie aktivieren oder deaktivieren möchten.
  2. Klicken Sie auf Aktivieren oder Deaktivieren.

iSCSI Targets bei einem iSNS-Server registrieren:

iSNS (Internet Storage Name Service) bietet den Mechanismus der zentralisierten Verwaltung. Sie können alle iSCSI Targets bei iSNS-Server registrieren.

  1. Klicken Sie auf die Schaltfläche iSNS und aktivieren Sie das Kontrollkästchen iSNS aktivieren.
  2. Geben Sie die IP-Adresse des iSNS-Servers ein. Klicken Sie auf OK, um die Einstellung zu übernehmen.

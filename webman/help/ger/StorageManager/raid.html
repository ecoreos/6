RAID Group*

Mit dem DiskStation können Sie mehrere Festplatten in einer als RAID Group bezeichneten Speichereinheit organisieren. Volumes oder iSCSI LUNs (Block-Ebene) können auf RAID Groups erstellt werden. Volumes oder iSCSI LUNs können erweitert werden, wenn eine RAID Group über Speicherplatz verfügt, der zugeordnet werden kann. Verschiedene RAID-Typen bieten unterschiedliche Ebenen des Datenschutzes. Eine allgemeine Liste über die von Ihrem DiskStation unterstützten RAID-Typen finden Sie unter Allgemein.

RAID Group erstellen:

  1. Klicken Sie auf die Schaltfläche Erstellen.
  2. Wählen Sie eine der folgenden Optionen aus:
    • RAID Group für einzelnes Volume oder iSCSI LUN (Block-Ebene): Bietet bessere Leistung.
    • RAID Group für mehrere Volumes oder iSCSI LUNs (Block-Ebene): Bietet bessere Flexibilität bei der Verwaltung.
  3. Folgen Sie dem Assistenten, um den Vorgang abzuschließen.

Hinweis:

  • Eine RAID Group muss aus Festplatten desselben Typs bestehen. Sie können weder SATA- und SAS-Festplatten noch SSDs und Festplatten gleichzeitig verwenden, um ein RAID zu erstellen.
  • Eine RAID Group für mehrere Volumes oder iSCSI LUNs (Block-Ebene), die RAID 5 oder RAID 6 verwendet, kann mehrere RAID Arrays enthalten. Jedes RAID Array, das zur selben RAID Group gehört, ist vom selben RAID-Typ (z. B. RAID 5 oder RAID 6). Die maximale Anzahl der Festplatten pro RAID Array kann 6, 12 oder 24 sein. Sie können die maximale Anzahl der Festplatten beim Erstellen der RAID Group festlegen. Diese Anzahl kann später nicht mehr geändert werden. Wurde einem RAID Array die maximale Anzahl an Festplatten zugewiesen, kann die nächste zur RAID Group hinzugefügte Festplatte einem neuen RAID Array zugewiesen werden. Das neue RAID Array muss die minimale Anzahl an Festplatten enthalten, die für diesen bestimmten RAID-Typ benötigt werden (d. h. drei Festplatten für RAID 5 und vier Festplatten für RAID 6).
  • Wenn Sie eine RAID Group auf einer bestimmten Festplatte erstellen möchten, die Festplatte aber unter RAID Group-Erstellungsassistent nicht erscheint, können Sie den Status der Festplatte unter Festplattenverwaltung überprüfen.

RAID Group entfernen:

  1. Wählen Sie die RAID Group aus, die Sie entfernen möchten.
  2. Klicken Sie auf die Schaltfläche Entfernen und folgen Sie dem Assistenten, um den Vorgang durchzuführen.

Hinweis:

  • Volumes und iSCSI LUNs auf der RAID Gruppe werden gelöscht, wenn Sie die RAID Gruppe entfernen. Stellen Sie sicher, dass keine wichtigen Daten gelöscht werden.

RAID Group reparieren:

Eine RAID Group, deren Status Fehlerhaft ist, kann repariert werden, wenn die RAID Group einer der folgenden Typen ist: RAID 1, RAID 5, RAID 6 und RAID 10. In der RAID Group vorhandene Daten werden nicht gelöscht.

Gehen Sie zum Reparieren einer RAID Group gemäß den folgenden Schritten vor:

  1. Schalten Sie den DiskStation aus. (Sie können diesen Schritt auslassen, wenn Ihr Modell Hot-Swapping unterstützt.)
  2. Tauschen Sie die unter Disk-Informationen im RAID Group-Status angezeigte fehlerhafte Festplatte aus.
  3. Klicken Sie auf Verwalten.
  4. Wählen Sie Reparieren aus.
  5. Folgen Sie dem Assistenten, um den Vorgang abzuschließen.

Hinweis:

  • Die Ersatzfestplatte muss größer oder genauso groß sein wie die kleinste Festplatte der RAID Group.
  • Der Status der Ersatzfestplatte muss "Initialisiert" oder "Nicht initialisiert" sein.

RAID-Typ einer RAID Group ändern:

Um den RAID-Typ einer RAID Group zu ändern, müssen verfügbare Festplatten auf Ihrem DiskStation installiert sein. Derzeit werden die folgenden RAID-Umstellungen unterstützt:

  • Basic auf RAID 1 oder RAID 5
  • RAID 1 auf RAID 5
  • RAID 5 auf RAID 6
  • Fügen Sie dem RAID 1 eine Mirror-Disk hinzu

Führen Sie die nachstehenden Schritte durch, um den RAID-Typ Ihrer RAID Group zu ändern:

  1. Wählen Sie die RAID Group aus, die Sie ändern möchten.
  2. Klicken Sie auf Verwalten.
  3. Wählen Sie RAID-Typ ändern aus.
  4. Folgen Sie dem Assistenten, um den Vorgang abzuschließen.

Hinweis:

  • Der Status einer RAID Group muss Normal sein, bevor ihr RAID-Typ geändert werden kann. Andernfalls kommt es zum Datenverlust. Wenn die RAID Group den Status Fehlerhaft hat, müssen Sie sie zuerst reparieren.
  • Um den RAID-Typ einer RAID Group zu ändern, müssen genügend Festplatten auf Ihrem DiskStation installiert und die Größe der neuen Festplatte muss größer als die kleinste Festplatte in der RAID Group sein.
  • Der Status der neuen Festplatte muss „initialisiert“ oder „nicht initialisiert“ sein.
  • Der RAID Typ einer RAID Group mit mehreren RAID Arrays kann nicht geändert werden.

Größe einer RAID Group erweitern:

Sie können die Größe einer RAID Group erweitern, indem Sie vorhandene Festplatten durch größere ersetzen oder Festplatten hinzufügen.

Hinweis:

  • Um eine RAID Group mit Festplatten mit 3 TB oder mehr zu erweitern, muss Ihre RAID Group mit DSM 3.0 oder später erstellt worden sein.

RAID Group durch Ersetzen vorhandener Festplatten durch größere erweitern:

Sie können eine RAID Group erweitern, indem Sie vorhandene Festplatten in den folgenden RAID-Typen durch größere ersetzen: RAID 1, RAID 5 und RAID 6. Zusammenhängende Volumes oder iSCSI LUN in RAID Group für einzelne Volumes oder iSCSI LUNs werden automatisch erweitert. Führen Sie die folgenden Schritte durch:

  1. Schalten Sie den DiskStation aus. Sie können diesen Schritt auslassen, wenn Ihr Modell Hot-Swapping unterstützt.
  2. Tauschen Sie eine Festplatte gegen eine größere aus.
  3. Schalten Sie den DiskStation ein und reparieren Sie die RAID Group.
  4. Wiederholen Sie die Schritte 1 bis 3, bis alle Festplatten durch größere ersetzt sind.
  5. Das System erweitert die RAID Group automatisch.

Hinweis:

  • Alle Dienste des Speicher-Managers werden bei der Erweiterung vorübergehend angehalten.
  • Sie können die Festplatte nur erweitern, wenn die RAID Group den Status Normal hat; andernfalls gehen Daten verloren. Wenn die RAID Group den Status Fehlerhaft hat, müssen Sie sie zuerst reparieren.

RAID Group durch Hinzufügen von Festplatten erweitern:

Diese Funktion gilt für die RAID-Typen JBOD, RAID 5 und RAID 6. Gegen Sie wie folgt vor:

  1. Wählen Sie die RAID Group aus, die Sie erweitern möchten.
  2. Klicken Sie auf Verwalten.
  3. Wählen Sie Festplatte hinzufügen aus.
  4. Folgen Sie dem Assistenten, um die Installation abzuschließen.

Hinweis:

  • Der RAID Group muss den Status Normal haben.
  • Bei den RAID Group-Typen RAID 5 und RAID 6 muss die Festplatte größer sein als die kleinste Festplatte der RAID Group.
  • Der Status der ausgewählten Festplatte muss „Initialisiert“ oder „Nicht initialisiert“ sein.

RAID Group-Beschreibung bearbeiten:

  1. Wählen Sie die RAID Group aus, die Sie bearbeiten möchten.
  2. Klicken Sie auf Verwalten.
  3. Wählen Sie Beschreibung bearbeiten aus.
  4. Bearbeiten Sie die Beschreibung und klicken Sie auf OK, um die Änderungen zu speichern.

RAID-Bereinigung

Die RAID-Bereinigung ist eine Wartungsfunktion für Daten, die RAID Groups überprüft und erkannte Dateninkonsistenzen repariert. Diese Funktion kann bei RAID Groups des Typs RAID 5 oder RAID 6 verwendet werden.

Wir empfehlen eine regelmäßige Ausführung der RAID-Bereinigung, um die Datenkonsistenz zu erhalten und den Verlust wichtiger Daten im Fall eines Festplattendefekts zu vermeiden.

RAID-Bereinigung durchführen:

  1. Wählen Sie die RAID Group aus, in der die RAID-Bereinigung ausgeführt werden soll.
  2. Klicken Sie auf Verwalten.
  3. Wählen Sie RAID-Bereinigung aus.

Hinweis:

  • Die RAID-Bereinigung kann nur ausgeführt werden, wenn der Status der RAID Group Normal ist.
  • Fahren Sie das System nicht herunter, bevor die RAID-Bereinigung abgeschlossen ist. Wenn das System vorher heruntergefahren wird, muss die RAID-Bereinigung neu gestartet werden.

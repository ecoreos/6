Ihren Synology NAS sichern

Überblick

Unerwarteter Datenverlust infolge eines Festplattenfehlers oder einer Naturkatastrophe stellt eine potenzielle Bedrohung Ihrer wichtigsten Daten dar, insbesondere wenn keine zuverlässige Datensicherungslösung verfügbar ist. Hyper Backup ist eine intuitive und integrierte Lösung für die Sicherung und Wiederherstellung von Kopien Ihrer wertvollen Daten und die Wiederherstellung früherer Anwendungen und Systemkonfigurationen. Die folgenden Speicherziele werden unterstützt:

  • Lokale freigegebene Ordner (einschließlich an Ihrem DiskStation angeschlossene externe Speichergeräte)
  • Remote Synology NAS
  • Remote-rsync-Server
  • Remote OpenStack Swift-Server
  • Öffentliche Clouds (Amazon S3, Microsoft Azure, OpenStack Swift usw.)

Hinweis:

Hyper Backup sichert nur bestimmte Systemkonfigurationen und Anwendungen. Weitere Informationen finden Sie in der Hilfe und Dokumentation zu Hyper Backup.

1. Aufgaben für die lokale Datensicherung in einem lokalen freigegebenen Ordner oder auf einem externen Speichergerät erstellen

Hyper Backup bietet für unterschiedliche Zieltypen ähnliche Verfahren für die Datensicherung und Wiederherstellung. Nachstehend wird die lokale Sicherung und Wiederherstellung erläutert, die auch für andere Aufgabentypen zurate gezogen werden kann.

  1. Melden Sie sich bei DSM an und suchen Sie nach dem Paket Hyper Backup.
  2. Klicken Sie unten links auf + und wählen Sie Datensicherungsaufgabe aus.
  3. Wählen Sie zum Einrichten einer lokalen Datensicherungsaufgabe Lokaler gemeinsamer OrdnerLokaler aus.
  4. Wählen Sie Datensicherungsaufgabe erstellen aus und wählen Sie einen lokalen Ordner oder ein an Ihrem DiskStation angeschlossenes externes Speichergerät aus.
  5. Wählen Sie den (die) Quellordner mit den zu sichernden Daten aus. Verschlüsselte freigegebene Ordner müssen, wenn sie ausgewählt werden, während der Datensicherung angehängt bleiben. Wenn sich Quelle und Ziel auf demselben Volume befinden, werden die Quellordner vom System mit orangefarbenen Ausrufezeichen markiert.
  6. Wählen Sie die Anwendung(en) aus, die Sie sichern möchten.
  7. Legen Sie die nachstehenden Datensicherungseinstellungen gemäß Ihren Anforderungen fest:
    • Aufgabe: Geben Sie der Datensicherungsaufgabe einen Namen.
    • Aufgabenbenachrichtigung aktivieren: Geben Sie hier an, dass Sie per E-Mail, SMS oder Push-Dienst über den Aufgabenstatus informiert werden möchten.
    • Konfigurationssicherung aktivieren: Über diese Option können Sie die Systemkonfigurationen (z. B. Benutzerdaten) sichern. Diese Einstellung wird standardmäßig bei allen Datensicherungsaufgaben erzwungen.
    • Gesicherte Daten komprimieren: Sie können gesicherte Daten komprimieren, um Speicherplatz am ausgewählten Datensicherungsziel zu sparen. Wenn diese Option aktiviert ist, dauert die Datensicherung länger.
    • Übertragungsverschlüsselung aktivieren (nur für Remote-Datensicherung): Aktivieren Sie diese Option, um die Sicherungsdaten während der Übertragung zu einem Remote-Ziel (z. B. ein rsync-Server oder eine öffentliche Cloud) zu verschlüsseln.
    • Teilgröße bei mehrteiligem Upload (nur für S3-kompatible Server): Über diese Option können Sie die Sicherungsdaten entsprechend der angegebenen Teilgröße aufteilen.
    • Client-seitige Verschlüsselung aktivieren: Wählen Sie diese Option aus, um die Sicherungsdaten mit einem Kennwort vor dem ungewünschten Zugriff auf der Zielseite zu schützen.
    • Sicherungszeitplan aktivieren: Über diese Option können Sie eine Zeit für die automatische Durchführung der Datensicherungsaufgabe angeben.
  8. Wählen Sie Sicherungsrotation aktivieren, um das Rotationsschema einzurichten:
    • Von den frühesten Versionen: Löscht die frühesten vorhandenen Sicherungsversionen, wenn die angegebene Anzahl der Versionen überschritten wird.
    • Smart Recycle: Das System verwahrt jede Sicherungsversion, bis die angegebene Anzahl der Versionen überschritten ist. Wenn die Rotation ausgelöst wird, rotiert das System zuerst die Versionen, die keine der Bedingungen erfüllen; wenn alle vorhandenen Versionen die nachstehenden Bedingungen erfüllen, rotiert das System die früheste Version:
      • Stündliche Versionen der letzten 24 Stunden: Das System bewahrt die früheste in jeder Stunde erstellte Version auf.
      • Tägliche Versionen vom vergangenen 1 Tag bis zu 1 Monat: Das System bewahrt die früheste an jedem Tag erstellte Version auf.
      • Wöchentliche Version älter als 1 Monat: Das System bewahrt die früheste in jeder Woche erstellte Version auf.
    • Anzahl der Versionen: Legt die Obergrenze möglicher verwahrter Sicherungsversionen fest.
    • Zeitachse: Zeigt das mögliche Ergebnis für die Aufbewahrung von Sicherungen gemäß Ihrem Sicherungszeitplan und Rotationsschema an.
  9. Klicken Sie zum Beenden des Setups auf Übernehmen.
  10. Folgen Sie den Anweisungen des Sicherungsassistenten und klicken Sie auf Ja, um die Datensicherungsaufgabe sofort durchzuführen.

2. Einzelne Dateien aus einem lokalen freigegebenen Ordner bzw. von einem externen Speichergerät wiederherstellen

  1. Gehen Sie zu Hyper Backup, wählen Sie im linken Fenster die gewünschte Datensicherungsaufgabe aus und klicken Sie auf Datensicherungs-Explorer (das Lupensymbol). Geben Sie das Kennwort ein, wenn die Aufgabe durch clientseitige Verschlüsselung geschützt ist.
  2. Gehen Sie zur Datei, die Sie wiederherstellen möchten, und wählen Sie sie aus. Über die Zeitachse können Sie auch die wiederherzustellende Version auswählen.
  3. Klicken Sie auf Wiederherstellen, um die ausgewählte Datei bzw. den ausgewählten Ordner wiederherzustellen.

3. Vollständige Aufgaben zur gewünschten Zeit wiederherstellen

  1. Klicken Sie unten links auf Wiederherstellen und dann auf Daten. Geben Sie das Kennwort ein, wenn die Aufgabe durch clientseitige Verschlüsselung geschützt ist.
  2. Wählen Sie eine wiederherzustellende Aufgabe aus. Um Daten von anderen Repositories wiederherzustellen, wählen Sie Von anderen Repositories wiederherstellen aus und gehen zu der wiederherzustellenden Aufgabe.
  3. Wählen Sie aus, ob die Systemkonfigurationen wiederhergestellt werden sollen. Wenn Sie Systemkonfiguration wiederherstellen auswählen, müssen Sie die betreffenden wiederherzustellenden Konfigurationen in der nachstehenden Liste auswählen.
  4. Gehen Sie zu der (den) Datei(en) bzw. zu der Aufgabe, die Sie wiederherstellen möchten, und wählen Sie sie aus. Über die Zeitachse können Sie auch die wiederherzustellende Version auswählen.
  5. Wählen Sie die wiederherzustellenden Anwendungen aus.
  6. Es werden grundlegende Informationen zur Wiederherstellung angezeigt.
  7. Klicken Sie auf Übernehmen, um die Wiederherstellung zu starten.

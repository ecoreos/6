Netzwerkeinstellungen

Diese Seite enthält drei Registerkarten, welche die Netzwerkeinstellungen des high-availability Cluster, den aktiven Mitgliedsserver und den passiven Mitgliedsserver anzeigen.

HA Cluster Registerkarte

Wenn Sie einen high-availability Cluster erstellen, erstellt das System eine gemeinsame IP-Adresse für den high-availability Cluster IP, um eingehende Verbindungen zu empfangen. Zusätzliche high-availability Cluster IP-Adresse können nach Bedarf erzeugt werden. Die IP-Adressen aktivierter Netzwerkschnittstellen auf dem aktiven Server werden auf dieser Registerkarte angezeigt (die von der Heartbeat-Verbindung angezeigte Schnittstelle wird nicht auf dieser Seite angezeigt, da die entsprechende Schnittstelle für die Datenreplikation zwischen den beiden Servern verwendet wird). Diese Registerkarte zeigt die unten stehenden Informationen:

  • Netzwerk-Schnittstelle: Die aktivierte(n) Netzwerk-Schnittstelle(n) auf dem aktiven Server.
  • Verbindung: Zeigt an, ob diese Schnittstelle angeschlossen ist oder nicht.
  • IP-Adresse: Wenn Sie einen High-availability Cluster erstellen, erstellt das System eine gemeinsame IP-Adresse für die High-availability Cluster-IP, um eingehende Verbindungen zu empfangen. Sie können die Verbindung mit dem High-availability Cluster über diese IP-Adresse herstellen, wenn Sie sich bei DiskStation Manager (DSM) anmelden möchten, unabhängig davon, welcher Synology Server der aktive Server ist. Dies vermeidet die Verbindung zu anderen IP-Adressen, wenn eine Übergabe stattfindet.
  • Teilnetzmaske: Die entsprechende Teilnetzmaske für die zugewiesene Teilnetzmaske high-availability Cluster IP-Adresse.

Bearbeiten von IP-Adressen und Teilnetzmaske:

  1. Wählen Sie die IP-Adresse oder Teilnetzmaske, die Sie bearbeiten möchten, und klicken Sie auf Bearbeiten oder klicken Sie doppelt auf das Feld, um mit der Bearbeitung zu beginnen.
  2. Markieren Sie die Option Aktivieren, um jede Schnittstelle zu aktivieren oder deaktivieren. Sie können mehr als eine Cluster Netzwerkschnittstelle aktivieren, aber die während der Cluster Erstellung konfigurierte Netzwerkschnittstelle kann nicht deaktiviert werden.

Änderung des high-availability Cluster Servernamens:

  1. Klicken Sie auf Netzwerkeinstellungen ändern, um den high-availability Cluster Namen anzuzeigen und die Einstellungen zu ändern.
  2. Klicken Sie auf Übernehmen, um den aktuellen high-availability Cluster Namen und die IP-Adresseinstellungen zu speichern.
  3. Wenn Sie die aktuellen Änderungen verwerfen möchten, klicken Sie auf Abbrechen.

Aktiver und Passiver Server Registerkarten (angezeigt mit ihrem Servernamen)

Diese Registerkarten zeigen den aktuellen Nutzungsstatus der Netzwerkschnittstellen auf dem aktiven oder passiven Server an. Unter ihnen wird die Heartbeat-Verbindung jeweils eine Netzwerkschnittstelle auf dem aktiven und passiven Server belegen. Diese Registerkarten zeigen die folgenden Informationen an:

  • Netzwerk-Schnittstelle: Die physische Netzwerk-Schnittstelle am Server.
  • IP Adresse: Die der Netzwerkschnittstelle zugewiesene IP-Adresse.
  • Teilnetzmaske: Die entsprechende Teilnetzmaske für die zugewiesene IP-Adresse.
  • Beschreibung: Zeigt Informationen über die entsprechende Netzwerkschnittstelle und die Verfügbarkeit der zugewiesenen IP-Adresse an. Diese Feld wird die folgenden möglichen Bedingungen anzeigen:
    1. Erreichbare originale IP-Adresse: Die originale IP-Adresse, die dieser Schnittstelle am aktiven Server zugewiesen wurde, ist für Daten und Dienste verfügbar.
    2. Nicht erreichbare originale IP-Adresse: Die originale IP-Adresse, die dieser Schnittstelle am passiven Server zugewiesen wurde, ist nicht verfügbar.
    3. Heartbeat-Verbindung IP, nicht erreichbar aus dem Intranet: Die originale IP-Adresse, die automatisch der Heartbeat-Verbindung zugewiesen wurde, ist nicht für Daten und Dienste verfügbar.

Herunterfahren und Zurücksetzen

Die unten stehenden Anweisungen führen Sie durch den Prozess, einen High-availability Cluster herunterzufahren, zu entfernen oder zurückzusetzen.

High-availability Cluster mit Synology High Availability herunterfahren:

Wenn Sie einen High-availability Cluster herunterfahren möchten, um die Server zu einem neuen Standort zu transportieren oder eine Wartung durchzuführen, aber nicht die High-availability Cluster-Konfigurationen entfernen möchten, folgen Sie bitte diesen Anweisungen.

  1. Melden Sie sich bei DSM an.
  2. Öffnen Sie Synology High Availability.
  3. Klicken Sie auf die Schaltfläche Verwaltung.
  4. Wählen Sie HA Cluster herunterfahren.
  5. Eine Bestätigungsmeldung wird angezeigt. Klicken Sie zur Bestätigung auf Ja.

So fahren Sie einen High-availability Cluster manuell herunter:

Der High-availability Cluster kann auch über die Ein/Aus-Tasten an jedem Server manuell heruntergefahren werden.

  1. Halten Sie die Ein/Aus-Taste an einem Server (aktiv oder passiv) gedrückt, bis ein Signalton ausgegeben wird.
  2. Warten Sie nach dem vollständigen Herunterfahren eines Servers mindestens 30 Sekunden lang (der Server, der noch online ist, benötigt diese Zeit zum Speichern der Einstellungen).
  3. Halten Sie die Ein/Aus-Taste am anderen Server gedrückt, bis ein Signalton ausgegeben wird.
  4. Warten Sie, bis der Server vollständig heruntergefahren ist.

Server einschalten:

Wenn Sie die Server wieder einschalten möchten, müssen der aktive und passive Server zur selben Zeit eingeschaltet werden (oder innerhalb von 5 Minuten). Wenn Sie nur einen der Server einschalten, wird er versuchen, den anderen Server durch Senden der Meldung "Aufwachen bei LAN-Verbindung" einzuschalten.

Anormales Herunterfahren:

Wenn ein High-availability Cluster erstellt wird, ist die Funktion für den Neustart nach einem Stromausfall in der Systemsteuerung automatisch aktiviert. Wenn die Server aufgrund eines Stromausfalls nicht normal herunterfahren, werden sie automatisch neu gestartet, wenn die Stromversorgung wiederhergestellt ist.

So entfernen Sie einen High-availability Cluster:

Wenn Sie beide Server und den High-availability Cluster entfernen möchten, um einen anderen Server einzubinden, folgen Sie bitte diesen Anweisungen.

  1. Gehen Sie zur Seite Überblick.
  2. Klicken Sie auf die Schaltfläche Verwaltung.
  3. Wählen Sie HA Cluster entfernen.

Nach der Bestätigung entfernt das System den High-availability Cluster. Das Statusfeld zeigt den Fortschritt des Vorgangs an. Wenn der High-availability Cluster erfolgreich entfernt wurde, werden Sie zur ursprünglichen IP-Adresse des aktiven Servers geleitet. Der Status der beiden Servern ändert sich zu Einzeln.

So setzen Sie einen fehlerhaften High-availability Cluster zurück:

Wenn Sie aufgrund eines Split Brain-Fehlers oder einer anderen Fehlfunktion nicht mehr auf DSM oder Synology High Availability zugreifen können, können Sie versuchen, ein Entfernen des High-availability Clusters zu erzwingen, indem Sie auf die Reset-Taste an den Servern drücken.

  1. Halten Sie die Reset-Taste (gewöhnlich an der Rückseite eines Servers) gedrückt, bis der Server einen Signalton ausgibt.
  2. Das System entfernt den High-availability Cluster und der Status der beiden Server ändert sich zu Einzeln.
  3. Wenn Sie auf die Reset-Taste drücken, werden einige Einstellungen, z. B. IP-Adresse, DNS und Admin-Passwort zusätzlich zu folgenden Optionen auf die Standardwerte zurückgesetzt:
    • Das Admin-Passwort wird gelöscht.
    • Das Gastkonto wird deaktiviert.
    • DHCP wird in den IP, DNS, Gateway und anderen Netzwerkschnittstellen aktiviert.
    • JumboFrames werden deaktiviert und der MTU-Wert auf 1500 eingestellt.
    • Der Verwaltungs-Port wird auf 5000/5001 zurückgesetzt.
    • PPPoE wird auf Standard zurückgesetzt.
    • Automatische IP-Blockierung wird deaktiviert (die blockierte IP-Adresse wird nicht gelöscht).
    • Link Aggregation wird deaktiviert.
    • Firewall-Regeln werden deaktiviert (die Regeln werden nicht gelöscht).

Hinweis:

Während des Entfernens eines High-availability Clusters könnten einige unvollständige Datenschreibvorgänge stattfinden. Um die Vollständigkeit der Daten zu gewährleisten, führen Sie bitte eine Datenreplikation basierend auf dem aktiven Server aus, da der passive Server einige Datenaktualisierungen verpassen könnte.

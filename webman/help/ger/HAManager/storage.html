Speicherstatus

Diese Seite beschreibt den Status der Speicher-Volumes und iSCSI LUNs (Block-Ebene) auf dem aktiven und passiven Server des High Availability-Clusters.

Da die Kapazität und Konfigurationen auf den entsprechenden Volumes und iSCSI LUNs (Block-Ebene) auf dem aktiven und passiven Servern identisch sein muss, werden die Volumes und iSCSI LUN (Block-Ebene) auf den beiden Servern zusammen als einzelne Einheiten angezeigt. Wenn Sie daher den Status von Volumes und iSCSI LUNs (Block-Ebene) des aktiven und passiven Servers anzeigen möchten, müssen Sie die Seite Speicherstatus aufrufen [wenn Sie auf einen High Availability-Cluster zugreifen, sind nur iSCSI LUNs (Block-Ebene) auf dem aktiven Server im Speicher-Manager sichtbar].

Dieser Abschnitt zeigt die folgenden Informationen an:

  • Name: Zeigt die Namen von Volumes und iSCSI LUNs (Blockebene) an.
  • Benutzt: Zeigt den Prozentsatz der genutzten Kapazität von Volumes und iSCSI LUNs (Blockebene) an.
  • Benutzt/Gesamtgröße: Zeigt die gesamte und genutzte Kapazität von Volumes und iSCSI LUNs (Blockebene) an.
  • Status: Zeigt den Status von Volumes, iSCSI-LUNs (Block-Level) und SSD-Cache an.
  • Beschreibung: Zeigt die Beschreibung von Fehlern oder anderer Informationen des Volumes, iSCSI LUNs (Blockebene) und SSD Caches an.

Reparatur eines fehlerhaften Volumes oder iSCSI LUNs (Blockebene):

  • Wenn eine Festplatte in einem der Server abgestürzt ist, können Sie zur Seite Festplattenstatus wechseln, um zu erfahren, welche Festplatte abgestürzt ist und ausgetauscht werden muss. Machen Sie in diesem Fall Folgendes:
    1. Tauschen Sie die abgestürzte Festplatte aus (für die Schritte hinsichtlich der Festplatteninstallation lesen Sie bitte die Schnellinstallationsanleitung für Ihr Synology Produkt).
    2. Gehen Sie zur Seite Speicherstatus, und klicken Sie auf Reparieren.
  • Wenn der passive Server offline ist, wird der Status von Volumes und iSCSI LUNs (Blockebene) das Auftreten eines Fehlers anzeigen, und keine der Festplatten im passiven Server wird auf der Seite Festplattenstatus angezeigt. In diesem Fall können Sie versuchen, das Problem folgendermaßen zu lösen:
    1. Stellen Sie sicher, dass der passive Server eingeschaltet ist.
    2. Überprüfen Sie sämtliche Stromversorgungen.
    3. Überprüfen Sie sämtliche Anschlüsse.
  • Wenn ein Dateisystemfehler auf einem Volume oder iSCSI LUN (Blockebene) auf einem der Server auftritt, tun Sie bitte Folgendes:
    • Gehen Sie zur Seite Speicherstatus.
    • Klicken Sie auf Reparieren.

Hinweis: Das Limit von Volumes und iSCSI LUNs (Blockebene) auf einem High Availability-Cluster ist 64.

Protokoll senden

Wenn Sie einen Syslog-Server besitzen, können Sie mithilfe der nachstehenden Anleitung Ihren DiskStation als Syslog-Client konfigurieren und die lokalen Systemprotokolle Ihres DiskStation zur zentralen Überwachung und Verwaltung an einen Syslog-Server senden.

Protokolle an einen Syslog-Server senden:

  1. Gehen Sie zu Protokoll-Center > Protokoll senden.
  2. Aktivieren Sie das Kontrollkästchen Protokolle an einen Syslog-Server senden.
  3. Geben Sie in Server den Hostnamen oder die IP-Adresse des Syslog-Servers ein.
  4. Geben Sie für den Empfang von Syslog-Nachrichten in Port eine Portnummer an.
  5. Wählen Sie aus Transferprotokoll UDP oder TCP aus. Dies muss mit dem Protokoll übereinstimmen, das vom Syslog-Server verwendet wird.
  6. Wählen Sie aus Protokollformat BSD (RFC 3164) oder IETF (RFC 5424) aus, um Protokolle im ausgewählten Format zu senden.

    7. Hinweis:

    • Die Standard-Portnummer ist 514. Es müssen entsprechende Portweiterleitungsregeln festgelegt werden, wenn dem Syslog Server eine andere Portnummer zugewiesen werden soll.
    • UDP (User Datagram Protocol) ist ein einfacheres auf Nachrichten basierendes Verbindungsprotokoll, das schneller ist, aber über keine Sicherheits- und Fehlerkorrekturfunktionen verfügt. TCP (Transmission Control Protocol) garantiert das erfolgreiche Senden von Paketen, da es mittels Flusssteuerung prüft, ob alle Pakete erfolgreich gesendet wurden, bevor neue Pakete verarbeitet werden.
    • Eine sichere Verbindung (SSL) ist nur verfügbar, wenn Sie TCP verwenden.
    • Wenn Sie TCP auswählen und die Syslog-Verbindung mit SSL sichern möchten, müssen Sie die Option Sichere Verbindung aktivieren (SSL) aktivieren. Klicken Sie dann auf Zertifikat importieren, und wählen Sie das Zertifikat für die Protokollübertragung an den Syslog-Server aus.
  7. Klicken Sie auf Testprotokoll senden, um zu überprüfen, ob der Syslog-Server das Protokoll erfolgreich empfangen hat. Überprüfen Sie, wenn der Syslog-Server das Testprotokoll nicht empfangen hat, ob die Einstellungen korrekt sind.
  8. Klicken Sie auf Übernehmen, um die Einstellungen zu speichern.

Filtern von Protokollen

Auf der Registerkarte Filter können Sie nach Kategorie (z. B. System, Verbindung, Datensicherung) oder Schweregrad (z. B. Warnung, Fehler, Information) festlegen, welche Arten von Protokollen an den Syslog-Server gesendet werden.

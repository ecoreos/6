Remote-Ordner bereitstellen

Sie können Remote-Ordner von Remote-Servern (z. B. Netzwerkcomputer oder andere Synology NAS) bereitstellen, sodass Sie eingelegte Ordner mit File Station navigieren und verwalten können.

Einen freigegebenen Remote-CIFS-Ordner anhängen:

  1. Klicken Sie auf Extras > Remote-Ordner anhängen > Freigegebener CIFS-Ordner.
  2. Geben Sie im Feld Ordner den Remote-Ordnerpfad im Format \\Remote_Server\Remote_Folder ein, z. B. \\192.168.0.2\share oder \\chad\cheeseburger\pictures.
  3. Geben Sie, wenn der Remote-Server Zugriffsberechtigung für den Remote-Ordner benötigt, Ihre Anmeldedaten für den Remote-Server in die Felder Kontoname und Passwort ein.
  4. Klicken Sie auf Durchsuchen, um einen leeren Zielordner auf Ihrem DiskStation für das Anhängen des Remote-Ordners auszuwählen oder zu erstellen.
  5. Aktivieren Sie Beim Start automatisch anhängen, wenn der DiskStation den Remote-Ordner bei jedem Systemstart bzw. Neustart anhängen soll.
  6. Klicken Sie auf Anhängen, um den Remote-Ordner an den Zielordner anzuhängen. Sie können nun den Remote-Ordner über Ihren DiskStation navigieren und verwalten.

Einen freigegebenen Remote-NFS-Ordner anhängen:

  1. Klicken Sie auf Extras > Remote-Ordner anhängen > Freigegebener NFS-Ordner.
  2. Geben Sie im Feld Ordner den Remote-Ordnerpfad im Format Remote_Server:/Remote_Folder_Path ein, z. B. 192.168.0.2:/volume1/share.
  3. Wählen Sie die gewünschte NFS-Version und das gewünschte Übertragungsprotokoll aus.
  4. Klicken Sie auf Durchsuchen, um einen leeren Zielordner auf Ihrem DiskStation für das Anhängen des Remote-Ordners auszuwählen oder zu erstellen.
  5. Aktivieren Sie Beim Start automatisch anhängen, wenn Ihr DiskStation den Remote-Ordner bei jedem Systemstart bzw. Neustart anhängen soll.
  6. Klicken Sie auf Anhängen, um den Remote-Ordner an den Zielordner anzuhängen. Sie können nun den Remote-Ordner über Ihren DiskStation navigieren und verwalten.

Hinweis:

NFSv4 unterstützt ausschließlich das TCP-Protokoll.

Remote-Ordner trennen:

  1. Wählen Sie den Remote-Ordner aus, den Sie trennen möchten, und klicken Sie auf Extras > Trennen.
  2. Sie können auch einfach auf den Remote-Ordner rechtsklicken und Trennen auswählen.

Hinweis:

  • Die maximale Anzahl von gleichzeitigen Remote-Ordnern ist 100.
  • Sie dürfen nur Remote-Ordner eines Remote-Servers anhängen, der das CIFS- oder NFS-Protokoll unterstützt.
  • Remote-Ordner können nur an Unterordner unter freigegebenen Ordnern angehängt werden.
  • Das Anhängen von Remote-Ordnern wird nur im Modus hoher Priorität unterstützt. Gehen Sie, um diese Funktion zu verwenden, zum Docker-Paket des Host-DSM, wählen Sie das Docker DSM aus, für das Sie verschlüsselte freigegebene Ordner aktivieren möchten, klicken Sie auf Bearbeiten und aktivieren Sie das Kontrollkästchen neben Container mit hoher Priorität ausführen.

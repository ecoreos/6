Einschränkungen und Systemanforderungen

Der Einsatz von Synology High Availability erfordert zwei identische Synology Server, die als aktiver und passiver Server dienen. Bevor die beiden Server zu einem High Availability-Cluster kombiniert werden, überprüft das System die folgenden Hardware- und Software-Einschränkungen, um zu gewährleisten, dass ein High Availability-Cluster erstellt werden kann.

Systemanforderungen

Beachten Sie die folgenden Leitlinien hinsichtlich Software und Hardware:

  • Synology Server: Beim aktiven und beim passiven Server muss es sich um identische Modelle handeln.
  • Expansionseinheit(en): Der aktive und der passive Server müssen korrekt mit denselben Expansionseinheiten verbunden sein.
  • Anzahl und Typ der Laufwerke: Es dürfen nur SAS-Laufwerke verwendet werden. Vor der Installation von DSM müssen mindestens drei Laufwerke in die Expansionseinheit(en) eingebaut werden.
  • Externes USB-Gerät: Nur USB-Speicher wird unterstützt. Sonstige externe USB-Geräte (z. B. UPS, WLAN-Dongle, Lautsprecher, Drucker, Bluetooth-Dongle und DVB-Dongle) werden nicht unterstützt. USB-Speicher kann nur im aktiven Server verwendet und nicht an den passiven Server übergeben werden.

Netzwerkumgebung

Beachten Sie bei der Erstellung eines High Availability-Clusters die folgenden Leitlinien zu den Netzwerkeinstellungen.

  • Netzwerkeinstellungen: Die Heartbeat-Verbindung zwischen den beiden Servern muss korrekt eingerichtet sein. Beide Server müssen zum selben Teilnetz gehören und nach der Installation müssen statische IP-Adressen konfiguriert werden.
  • Anzahl der LAN-Ports: Beide Server müssen über dieselbe Anzahl von LAN-Ports verfügen. Wenn eines Ihrer DiskStation-Modelle mit zusätzlichen Netzwerkkarten-Schnittstellen ausgerüstet ist, muss der andere dieselben Schnittstellen besitzen.
  • Verbindungseinschränkungen: WLAN wird vom High Availability-Cluster nicht unterstützt.

Hinweis:

  • Der NTP-Server wird auf dem aktiven Server automatisch aktiviert.

Paket- und Software-Einschränkungen

Beachten Sie die folgenden für Pakete und Software geltenden Einschränkungen:

  • Die folgenden Pakete und Funktionen werden vom High Availability-Cluster nicht unterstützt:
    • DVBLink-Server
    • Die Funktion zum Neustart/Herunterfahren des High-Availability-Clusters in CMS
    • Time Backup
  • High Availability-Cluster mit gemeinsamen Speicher können nicht als replikationsbasierte High Availability-Cluster konfiguriert oder in Verbindung mit diesem verwendet werden.

Spezielle Expansionseinheiten

Der aktive und der passive Server müssen mit den speziellen Expansionseinheiten verbunden sein. Jede Expansionseinheit verfügt über zwei Module.

Expansionseinheiten einrichten/hinzufügen:

  • Ein Modul enthält zwei Ports: den Eingangsport auf der linken und den Ausgangsport auf der rechten Seite. Verbinden Sie den Ausgangsport eines Moduls mit dem Eingangsport des Moduls der nächsten Expansionseinheit mit einem SAS-Kabel.
  • Die Expansionseinheiten müssen mit den Modulen auf den entsprechenden Seiten für jeden Server in Reihe geschaltet sein.
  • Der aktive und der passive Server müssen mit den gemeinsamen Expansionseinheiten in derselben Reihenfolge verbunden sein.

Hinweise:

Ausführliche Informationen zur Installation von Expansionseinheiten finden Sie in der Installationskurzanleitung für Ihre Server und Expansionseinheiten.

Wiederherstellung

Auf dieser Seite können Systemadministratoren Wiederherstellungen durchführen. Damit auf Schnappschüsse von freigegebenen Ordnern zugegriffen werden kann, muss für jeden freigegebenen Ordner die Option Schnappschuss sichtbar machen aktiviert sein, damit über File Station oder Windows File Explorer auf die Schnappschüsse zugegriffen werden kann.

Freigegebener Ordner

Einzelne Dateien/Ordner in einem freigegebenen Ordner wiederherstellen:

  1. Wählen Sie den freigegebenen Ordner aus, den Sie wiederherstellen möchten.
  2. Klicken Sie auf Wiederherstellen.
  3. Wählen Sie eine Schnappschussversion aus, aus der Sie wiederherstellen möchten.
  4. Klicken Sie auf Durchsuchen und kopieren Sie die Dateien/Ordner, die Sie wiederherstellen möchten, zum Ziel.

Freigegebenen Ordner wiederherstellen:

  1. Wählen Sie den freigegebenen Ordner aus, den Sie wiederherstellen möchten.
  2. Klicken Sie auf Wiederherstellen.
  3. Wählen Sie eine Schnappschussversion aus, aus der Sie wiederherstellen möchten.
  4. Klicken Sie auf Aktion und wählen Sie Am Speicherort wiederherstellen aus der Dropdown-Liste aus.
  5. Eine Bestätigungsmeldung wird angezeigt. Klicken Sie auf Ja, um die Aktion zu bestätigen.

Schnappschuss in einen neuen freigegebenen Ordner klonen:

  1. Wählen Sie den freigegebenen Ordner aus, den Sie klonen möchten.
  2. Klicken Sie auf Wiederherstellen.
  3. Wählen Sie eine Schnappschussversion aus, die Sie in einen neuen freigegebenen Ordner klonen möchten.
  4. Klicken Sie auf Aktion und wählen Sie Mit neuem Namen klonen aus der Dropdown-Liste aus.
  5. Sie werden zur Seite Klonen unter Systemsteuerung > Gemeinsamer Ordner > Erstellen weitergeleitet. Geben Sie den Namen des geklonten freigegebenen Ordners ein und klicken Sie auf OK, um den Vorgang abzuschließen.

Hinweis:

  • Die „Homes“ eines freigegebenen Ordners können nicht wiederhergestellt werden.
  • Der geklonte freigegebene Ordner kann nicht verschlüsselt werden.
  • Wird ein Schnappschuss aufgenommen, während Dateien übertragen werden, kann die Datenkonsistenz nicht garantiert werden.
  • Einschränkungen für verschlüsselte freigegebene Ordner:
    • Für bereitgestellte verschlüsselte freigegebene Ordner ist keine Art der Wiederherstellung verfügbar.
    • Mit neuem Namen klonen ist für verschlüsselte freigegebene Ordner nicht verfügbar.
    • Getrennte, verschlüsselte freigegebene Ordner können mithilfe von Am Speicherort wiederherstellen aus zuvor aufgenommenen Schnappschüssen wiederhergestellt werden.

Replikation eines freigegebenen Ordners übergeben:

  1. Wählen Sie den replizierten freigegebenen Ordner aus, den Sie übergeben möchten.
  2. Klicken Sie auf Übergabe.
  3. Wählen Sie den neuen Zielserver aus, wenn für den freigegebenen Ordner mehrere Replikationsaufgaben existieren.
  4. Klicken Sie auf Übergabe, um die Aktion zu starten.

Failover einer Replikation eines freigegebenen Ordners:

  1. Wählen Sie den replizierten freigegebenen Ordner aus, auf den Sie das Failover anwenden möchten.
  2. Klicken Sie auf Failover > Failover.
  3. Wählen Sie die wiederherzustellende Version des Schnappschusses aus.
  4. Klicken Sie auf Failover, um die Aktion zu starten.

Hinweis:

  • Sie können einen Failover nur nach Ausfall des Quellservers initiieren.

Schutz der Replikation eines freigegebenen Ordners erneuern:

Während des Failover zum Ziel-Server stoppt die Replikationsaufgabe die Synchronisierung von Daten zwischen den beiden Servern. Sie müssen den Schutz der Replikationsaufgabe erneuern, nachdem der ursprüngliche Ziel-Server wiederhergestellt wurde.

  1. Wählen Sie den freigegebenen Failover-Ordner aus, dessen Schutz Sie erneuern möchten.
  2. Klicken Sie auf Failover > Schutz erneuern.
  3. Wählen Sie den neuen Quellserver aus.
  4. Wählen Sie den neuen Zielserver aus, wenn für den freigegebenen Ordner mehrere Replikationsaufgaben existieren.
  5. Klicken Sie auf Schutz erneuern, um die Aktion zu starten.

Hinweis:

  • Die auf dem neuen Zielserver vorhandenen Daten des replizierten freigegebenen Ordners werden durch die auf dem neuen Quellserver überschrieben.
  • Wenn die ausgewählte Schnappschussversion für den Failover auf dem Partnerserver nicht vorhanden war, kann die Erneuerung des Schutzes länger dauern.

Test-Failover für einen replizierten freigegebenen Ordner durchführen:

  1. Wählen Sie den replizierten freigegebenen Ordner aus, den Sie testen möchten.
  2. Klicken Sie auf Test-Failover > Test-Failover.
  3. Wählen Sie den Test-Server aus, wenn für den freigegebenen Ordner mehrere Replikationsaufgaben existieren.
  4. Geben Sie den Namen für Test-Failover des freigegebenen Ordners ein und wählen Sie die zu testende Schnappschussversion aus.
  5. Klicken Sie auf Test-Failover, um die Aktion zu starten.

Hinweis:

  • Durch die Übergabe bzw. den Failover einer Replikation wird der zugehörige Test-Failover bereinigt.
  • Sie können eine Replikationsaufgabe für einen Test-Failover eines freigegebenen Ordners nicht erstellen.

Test-Failover für einen replizierten freigegebenen Ordner bereinigen:

  1. Wählen Sie den replizierten freigegebenen Ordner aus, für den Sie den Test-Failover bereinigen möchten.
  2. Klicken Sie auf Test-Failover > Test-Failover bereinigen.
  3. Aktivieren Sie den Test-Failover, den Sie bereinigen möchten.
    Hinweis: Aktivieren Sie Test-Failover-Daten löschen, wenn Sie den für den Test-Failover erstellten freigegebenen Ordner nicht behalten möchten.
  4. Klicken Sie auf Test-Failover bereinigen, um die Aktion zu starten.

iSCSI LUN

iSCSI LUN wiederherstellen:

  1. Wählen Sie die wiederherzustellende iSCSI LUN aus.
  2. Klicken Sie auf Wiederherstellen.
  3. Wählen Sie die Schnappschussversion aus, aus der Sie wiederherstellen möchten.
  4. Klicken Sie auf Aktion und wählen Sie Am Speicherort wiederherstellen aus der Dropdown-Liste aus.
  5. Eine Bestätigungsmeldung wird angezeigt. Klicken Sie auf Ja, um die Aktion zu bestätigen.

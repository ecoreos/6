Split-brain

Wanneer een high-availability cluster normaal functioneert, moet slechts een van de deel uitmakende servers de rol van actieve server op zich nemen. In dit geval detecteert de passieve server de aanwezigheid van de actieve server via de Heartbeat-verbinding en de gegevensverbinding met de gedeelde uitbreidingsunits zodat de overschakeling niet wordt geactiveerd. Bovendien detecteert de passieve server de status van de gedeelde opslag om een foutieve overschakeling naar de rol van actieve server te voorkomen zo lang de huidige actieve server nog steeds normaal contact heeft met de gedeelde uitbreidingseenheden.

Bij verlies van alle gegevens- en Heartbeatverbinding of onstabiele status van gedeelde opslag en systeem kunnen beide servers de rol van actieve server overnemen. Deze situatie wordt omschreven als een "split-brain". Om in dergelijke gevallen beschadiging van gegevens te voorkomen door gelijktijdige toegang van beide servers, zal een server doorgaan met het leveren van services en de andere server opnieuw opstarten en vervolgens een verbindingscontrole uitvoeren. Gedurende deze situatie kunnenn verbindingen van clientgebruikers tijdelijk onstabiel worden.

High Availability Manager

High Availability verwijst naar een serverlay-outoplossing ontwikkeld om onderbrekingen van services die het gevolg zijn van een slecht werkende server te beperken. Synology High Availability maakt gebruik van twee servers om een "high-availability cluster" te vormen, daarbij neemt een server de rol van "actieve server" op zich en fungeert de andere server als "passieve server". Om een high-availability cluster te implementeren hebt u twee compatibele Synologyservers nodig.

Een high-availability cluster gebruikt de volgende componenten en concepten:

  • Concept: Implementatie van Synology High Availability vereist twee Synologyservers om een high-availability cluster te vormen. Een server fungeert als "actieve server" en de andere als een "passieve server". Zowel de actieve als de passieve server in de high-availability cluster zijn verbonden met dezelfde uitbreidingseenheden en delen dezelfde opslag. Alleen de actieve server neemt de werkbelasting op zich om de gegevensintegriteit te garanderen. Bij een storing van de actieve server staat de passieve server klaar om alle services over te nemen. De high-availability cluster zal dan doorgaan met de normale levering van services en de serverdowntime beperken. Meer informatie over overschakelen vindt u bij Overschakelen.
  • Actieve server:De actieve server zorgt voor de levering van alle services. Als er een fout optreedt op de actieve server, kunt u of het systeem een overschakeling naar de passieve server initialiseren. In dergelijke gevallen wordt de voorziening van deze services overgedragen aan de passieve server en wordt de passieve server de actieve server.
  • Passieve server: in normale omstandigheden verkeert de passieve server in stand-bymodus. Als de actieve server geen services meer kan leveren, kunt u of kan het systeem een overschakeling naar de passieve server initialiseren.
  • Heartbeatverbinding: De actieve en passieve servers van een high-availability cluster zijn gekoppeld door een speciaal daartoe voorzien private-netwerkverbinding die men aanduidt als de "Heartbeatverbinding." Als het contact tussen de servers gedurende een bepaalde tijdspanne is verloren, zal het systeem nagaan of een overschakeling nodig is.
  • High-availability cluster gedeelde opslag: Zowel de actieve als de passieve server in de high-availability cluster zijn verbonden met dezelfde uitbreidingseenheden en delen dezelfde opslag. Alleen de actieve server heeft schrijfmachtigingen voor de opslag.
  • Servernaam en netwerkinstelling van de high-availability cluster: de servernaam en netwerkinstellingen van de High-availability cluster worden bepaald door de desbetreffende instellingen in DSM. In het netwerk kunt u een verbinding maken met deze servernaam en de services worden geleverd door de actieve server. Meer informatie over netwerkinstellingen vindt u hier.

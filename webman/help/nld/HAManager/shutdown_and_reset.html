Uitschakelen en opnieuw instellen

De onderstaande instructies begeleiden u doorheen het proces van het afsluiten, verwijderen of opnieuw instellen van een high-availability cluster.

Om een high-availability cluster uit te schakelen met Synology High Availability:

Volg de onderstaande richtlijnen als u een high-availability cluster wilt uitschakelen om de servers naar een nieuwe fysieke locatie te vervoeren of onderhoudswerkzaamheden wilt uitvoeren, maar de high-availability cluster-configuraties niet wilt verwijderen.

  1. Meld u aan bij DSM.
  2. Open Synology High Availability.
  3. Klik op de knop Beheer.
  4. Selecteer HA cluster uitschakelen.
  5. Er verschijnt een bevestigingsmelding. Klik op Ja om te bevestigen.

Om een high-availability cluster handmatig uit te schakelen:

High-availability cluster kan ook handmatig worden uitgeschakeld via de voedingsknoppen die zich op elke server bevinden.

  1. Houd de aan/uit-knop op een server (actief of passief) ingedrukt tot een pieptoon weerklinkt.
  2. Wacht nadat een server volledig is uitgeschakeld minstens 30 seconden (de server die nog online is, heeft tijd nodig om de instellingen op te slaan).
  3. Houd de aan/uit-knop op de andere server ingedrukt tot een pieptoon weerklinkt.
  4. Wacht tot de server volledig is uitgeschakeld.

Servers opstarten:

Als u de servers opnieuw wilt inschakelen, moeten de actieve en passieve servers tegelijk worden ingeschakeld (of met een interval van minder dan vijf minuten). Als u slechts een van de servers inschakelt; zal deze server met een Wake-on-LAN-melding proberen om de andere server op te starten.

Abnormale uitschakeling:

Als er een high-availability cluster is aangemaakt, wordt de optie herstarten na stroomstoring in het Configuratiescherm automatisch ingeschakeld. Als de servers door een stroomstoring worden uitgeschakeld, zullen ze automatisch bij herstel van de stroomvoorziening worden opgestart.

Een high-availability cluster verwijderen:

Volg onderstaande richtlijnen als u beide servers wilt ontkoppelen en een high-availability cluster wilt verwijderen om een andere server te koppelen.

  1. Ga naar de pagina Overzicht.
  2. Klik op de knop Beheer.
  3. Selecteer HA cluster verwijderen.

Het systeem zal na bevestiging de high-availability cluster verwijderen. Het statusveld toont de voortgang van het verwijderingsproces. Als de high-availability cluster is verwijderd, wordt u omgeleid naar het oorspronkelijke IP-adres van de actieve server. De status van beide servers zal veranderen in Standalone.

Om een slecht werkende high-availability cluster opnieuw in te stellen:

Als u geen toegang krijgt tot DSM of Synology High Availability ten gevolge van een split-brainfout of andere defecten, kunt u proberen om de high-availability cluster te ontkoppelen door de Reset-knop op uw servers in te drukken.

  1. Houd de Reset-knop (gewoonlijk op het achterpaneel van de server) tot de server een pieptoon laat weerklinken.
  2. het systeem zal de high-availability cluster verwijderen en de status van beide servers zal veranderen in Standalone
  3. Door op de Reset-knop te drukken, worden een aantal standaardinstellingen teruggezet, zoals IP-adres, DNS en admin-wachtwoord, naast de volgende zaken:
    • Het admin-wachtwoord zal opnieuw leeg worden gemaakt.
    • De guest-account zal uitgeschakeld zijn.
    • DHCP zal ingeschakeld zijn op IP, DNS, Gateway en andere netwerkinterfaces.
    • JumboFrames zal uitgeschakeld zijn, en MTU zal ingesteld zijn op 1500.
    • Beheerpoort zal opnieuw ingesteld worden op 5000/5001.
    • De standaardinstellingen van PPPoE zullen worden teruggezet.
    • IP automatisch blokkeren zal worden uitgeschakeld (het geblokkeerde IP-adres wordt niet gewist).
    • Link aggregation zal uitgeschakeld zijn.
    • Firewallregels worden uitgeschakeld (de regels worden niet gewist).

Opmerking:

tijdens het verwijderen van een high-availability cluster zal het wegschrijven van gegevens mogelijk niet worden voltooid. Om de volledigheid van de gegevens te garanderen, voert u best een gegevensreplicatie uit op de actieve server omdat er mogelijk enkele gegevensupdates op de passieve server ontbreken.

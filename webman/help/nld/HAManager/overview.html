Overzicht

Deze pagina toont basisinformatie over de high-availability cluster en servers die er deel van uitmaken. Deze pagina biedt daarnaast hulpmiddelen om te creëren, te verwijderen of onderhoud uit te voeren op high-availability clusters en op de servers die er deel van uitmaken.

High Availability Cluster-status

Dit veld toont de globale status van uw high-availability cluster en wijst u op mogelijke fouten. Hieronder volgt een tabel van mogelijke condities die in dit veld worden weergegeven:

Status Beschrijving
Standalone De servers worden niet gecombineerd in een high-availability cluster.
Gezond De algemene conditie van uw high-availability cluster is goed. Het spiegelen van gegevens van de actieve server naar de passieve server werkt normaal.
Bezig met voorbereiding De servers zijn gecombineerd in een high-availability cluster, maar de initiële gegevensreplicatie is nog niet voltooid. Er kan geen overschakeling plaatsvinden op dit moment. Meer informatie vindt u in Overschakelen
Bezig met laden Het systeem is bezig met het laden van de clusterstatus. Overschakelen kan niet worden uitgevoerd tot de status laden is voltooid.
Waarschuwing De status van HA-cluster wijkt af, maar de services zijn nog steeds operationeel.
Gevaar Zowel de actieve als de passieve server werken niet naar behoren. De high-availability cluster kan niet normaal werken. Controleer de status van alle opslagruimtes (bijv. RAID-groepen, schijfgroepen, volumes en iSCSI LUN's).

Waarschuwingsmeldingen

De volgende tabel toont een overzicht van waarschuwingsmeldingen die systeemfouten kunnen worden weergegeven. Wanneer er meer dan een waarschuwingsmelding verschijnt, kunt u met de knoppen voorwaarts/achterwaarts doorheen de meldingen bladeren.

Waarschuwing/fout Beschrijving
Bewaakte servicefout Er is een fout opgetreden in een of meer van de gecontroleerde services.
Opslagruimte beschadigd Een opslagruimte (bijv. RAID-groep, schijfgroep, volume, iSCSI LUN en SSD-cache) op de actieve of passieve server is beschadigd. Controleer de pagina Opslagstatus.
Opslagruimte gecrasht Een opslagruimte (bijv. RAID-groep, schijfgroep, volume, iSCSI LUN en SSD-cache) op de actieve of passieve server is vastgelopen. Controleer de pagina Opslagstatus.
Detectie van de passieve server is mislukt Deze fout kan optreden omdat de passieve server offline is of omdat de actieve en passieve servers geen verbinding kunnen maken via de Heartbeatverbinding. In dit geval is overschakelen niet beschikbaar.
Gegevens kunnen niet gerepliceerd worden naar passieve server Deze fout kan optreden als gevolg van een defect in het opslagvolume of van de schijf. In dit geval is overschakelen niet beschikbaar. Controleer de pagina Opslagstatus.
Passieve server is niet geselecteerd Deze status zal worden getoond als de passieve server van de high-availability cluster is ontkoppeld. In dit geval is overschakelen niet beschikbaar. Klik op de vervolgkeuzelijst Behouden en klik op Koppelen als u de passieve server opnieuw wilt koppelen. Volledige datareplicatie is vereist voor het opnieuw koppelen van de passieve server.

High Availability Cluster-informatie

Dit veld bevindt zich onder de high-availability cluster-status en toont basisinformatie van de cluster. Als er geen high-availability cluster bestaat, zal dit veld enkel informatie bevatten over de functie van high-availability clusters.

Dit veld bevat de volgende gegevens:

  • Clusterservernaam: de servernaam die door de high-availability cluster wordt gebruikt.
  • Cluster-IP-adres: het IP-adres dat door de high-availability cluster wordt gebruikt. Ongeacht op welke server de services worden uitgevoerd, kunnen gebruikers met behulp van dit IP-adres een verbinding maken met de high-availability cluster.
  • Opstellingstijd: de datum waarop de high-availability cluster is gemaakt.

Bedieningsknoppen van de High Availability Cluster

De volgende twee knoppen bevinden zich onder de high-availability cluster-informatie:

  1. HA cluster maken: als er geen high-availability cluster is, kunt u op deze knop klikken om twee servers te combineren en een high-availability cluster creëren. Meer informatie over het aanmaken van een high-availability cluster vindt u in Synology High Availability-wizard.
  2. Beheer: deze knop is enkel beschikbaar als er al een high-availability cluster bestaat. Het bevat de volgende opties:
    • Beheerwizard: start een wizard met instructies voor de verschillende beheerprocedures.
    • Overschakelen: initieert overschakelen van de actieve server naar de passieve server. Tijdens dit proces stopt de actieve server met het verlenen van services en wordt hij de nieuwe passieve server. De oude passieve server verandert vervolgens in de nieuwe actieve server en begint met de verlening van alle services van de high-availability cluster. Deze optie is niet beschikbaar als de passieve server de status fout heeft of niet is geselecteerd.
    • DSM updaten: opent de pagina DSM updaten van het Configuratiescherm. Hier kunt u beide servers bijwerken.
    • HA cluster verwijderen: verwijdert de high-availability cluster.
    • Passieve server ontkoppelen: verwijdert de passieve server van de cluster. Wanneer de server opnieuw aan de high-availability cluster wordt gekoppeld na ontkoppelen, zal een volledige datareplicatie nodig zijn. We raden aan om deze optie alleen te gebruiken als u de bestaande passieve server door een nieuwe wilt vervangen.
    • Passieve server koppelen: voegt een passieve server toe aan de cluster. Deze optie is alleen beschikbaar als er geen passieve server is in de high-availability cluster.
    • HA cluster uitschakelen: schakelt zowel de actieve als de passieve servers uit.
    • Actieve server uitschakelen: schakel de actieve server uit.
    • Passieve server uitschakelen: schakelt de passieve server uit.
    • Actieve server opnieuw opstarten: voor het opnieuw opstarten van de actieve server.
    • Passieve server opnieuw opstarten: start de passieve server opnieuw op.
    • Pieptoon op passieve server deactiveren: stopt de pieptoon van de passieve server als er een fout optreedt. De passieve server geeft nog altijd een pieptoon als er een andere fout optreedt.

Opmerking:

  • zodra een high-availability cluster is verwijderd, moet een volledige datareplicatie worden uitgevoerd als u een nieuwe cluster wilt maken.
  • Schakelt u de passieve server uit of koppelt u deze los dan kunnen de services niet worden overgeschakeld wanneer er een fout op de actieve server optreedt.

Server- en gegevensreplicatiestatus

Dit gedeelte bevat afbeeldingen met de status van de actieve en passieve servers, en van de stroom van gegevensreplicatie tussen de servers.

De afbeelding van gegevensreplicatie duidt op de volgende mogelijke condities:

  • gegevensreplicatie verloopt normaal: dit wordt aangeduid door gegevens die van de actieve server naar de passieve server stromen.
  • Geen datareplicatie: dit wordt aangeduid door de afwezigheid van de afbeelding van de gegevensstroom.

Servergegevens

Dit gedeelte bevindt zich onder de serverafbeeldingen. Informatie over de actieve en passieve servers wordt in dit gedeelte weergegeven. Als er geen passieve server is geselecteerd, zal in dit gedeelte Nvt worden weergegeven.

  • Servernaam: Synologyservernaam.
  • Modelnaam: Synologyservermodel.
  • Serienummer: serienummer van de Synologyserver.
  • Ventilatorstatus: status van de ventilatoren van de server. Als een ventilator slecht werkt, verschijnt er een waarschuwing. Neem in dat geval contact op met uw plaatselijke verdeler voor ondersteuning.
  • Temperatuur: temperatuur van de Synologyserver.
  • Vermogenstatus: geeft aan of server de voeding van een server normaal werkt. AIs het servermodel is uitgerust met redundante stroombronnen, verschijnt er een waarschuwing als een van beide een storing ondervindt.

Web Uygulamaları

Yerleşik Web Station özelliği ve PHP desteği ile DiskStation ürününüzde kolayca bir web sitesi barındırabilir ve işletmeniz için ya da hobi olarak dinamik web siteleri oluşturabilirsiniz. Buna ek olarak Paket Merkezi, İçerik Yönetimi, Müşteri İlişkileri Yönetimi ve e-Ticaret sistemi gibi çeşitli 3. taraf paketleri de sunar. Web sitenizin dosyalarını depolamak üzere web adı verilen varsayılan bir paylaşımlı klasör oluşturulacaktır.

Web Station'ı etkinleştirmek için:

  1. Denetim Masası > Web Hizmetleri öğesine gidin.
  2. Web Station'ı etkinleştir seçeneğini işaretleyin.
  3. Uygula'ya tıklayın.

Not:

  • Daha iyi esneklik ve güvenlik için, görevleri yürütmek için Web Station http kullanıcı grubunu kullanır. Bu yüzden, web sayfalarınız için erişim izinlerini kontrol etmek için, lütfen http grubunun erişim izinlerini her bir ilgili klasör ve dosya için değiştirin. Örneğin, web paylaşımlı klasöründeki sample.htm dosyasının Web Station yoluyla erişilebilir olmasını istiyorsanız, http grubunun web paylaşımlı klasörü ile birlikte sample.htm dosyası için okuma/yazma izinlerine sahip olduğundan emin olmanız gerekir.
  • Ana sayfa dosyası index.htm, index.html veya index.php olarak adlandırılmalıdır.
  • PHP posta() işlevini kullandığınızda, sistem otomatik olarak Bildirim sayfasındaki SMTP sunucu bilgisini alacak ve kullanacaktır. E-posta bildirimlerini etkinleştirdiğinizden ve doğru SMTP sunucusu bilgilerini girdiğinizden emin olun.
  • web paylaşımlı klasörünün gerçek yolu /volume1/web'dir. Bazı uygulamalar kullanıcının yapılandırma dosyasında yolu ayarlamasını gerektirir.
  • blog, photo, photosrc ve webdefault adları ayrılmış klasör adlarıdır. Bu adlar Web Station tarafından kullanılamaz, dolayısıyla lütfen web paylaşımlı klasörü altında bu adlar bulunan klasörleri oluşturmayın.
  • Bir missing.html, missing.htm veya missing.php dosyası oluşturarak hata sayfasını özelleştirebilirsiniz. İlk önce Web Station'ı etkinleştirin. Ardından missing.html, missing.htm veya missing.php dosyasını sunucuda web paylaşımlı klasörü altına yerleştirin. Kullanıcılar bulunmayan bir sayfaya link verdiklerinde, bu özelleştirilmiş hata sayfasını inceleyeceklerdir.
  • En fazla etkin HTTP bağlantısı sayısı: 256
  • En fazla sanal ana bilgisayar sayısı: 30

Web Sitelerini Barındırmak için Sanal Ana Makine'yi Etkinleştirin

Sanal ana bilgisayar özelliğiyle, tek bir sunucu üzerinde birden fazla web sitesi barındırabilirsiniz. Başlamak için birkaç etki alanı adı kaydetmeniz ve aşağıdaki adımları takip ederek sanal barındırmayı yapılandırmanız gerekir.

Sanal Ana Makineyi etkinleştirmek için:

  1. Sanal Ana Makine'ye tıklayın.
  2. Oluştur'a tıklayın.
  3. Web sitesi dosyalarını saklayacak olan web paylaşımlı klasörünün alt klasörü için bir ad girin.
  4. Web sitesi için bir ana sistem adı girin.
  5. Web sitesi için bir port girin.
  6. Tamam'a tıklayın.

Kişisel Web Sitesini Etkinleştir

Kişisel web sitesi işlevi, yerel kullanıcıların, etki alanı kullanıcılarının ve LDAP kullanıcılarının kendi kişisel web sitelerini oluşturmalarına olanak sağlayan güvenilir bir yoldur.

Yerel kullanıcılar için

Her yerel kullanıcının özgün bir web sitesi adresi olacaktır sunucu-IP/~kullanıcıadı veya sunucu-adı/~kullanıcıadı. Örnek:

  1. 192.168.1.1/~ken
  2. SynologyNAS/~ken

Etki alanı kullanıcıları için

Her etki alanı kullanıcısının özgün bir web sitesi adresi olacaktır sunucu-IP/~etkialanıadı%5ckullanıcıadı veya sunucu-adı/~etkialanıadı%5ckullanıcıadı. Örnek:

  1. 192.168.1.1/~testetkialanı%5cken
  2. SynologyNAS/~testetkialanı%5cken

LDAP kullanıcıları için

Her LDAP kullanıcısının özgün bir web sitesi adresi olacaktır sunucu-IP/~kullanıcıadı@etkialanıadı veya sunucu-adı/~kullanıcıadı@etkialanıadı. Örnek:

  1. 192.168.1.1/~ken@testetkialanı
  2. SynologyNAS/~ken@testetkialanı

Not:

Bu özelliği etkinleştirmeden önce lütfen kullanıcı ana sayfaları'nın etkinleştirildiğinden emin olun. Yerel kullanıcılar için Denetim Masası > Kullanıcı > Gelişmiş öğesine; etki alanı kullanıcıları için Denetim Masası > Dizin Hizmeti > Etki Alanı Kullanıcıları > Kullanıcı Ana Sayfası öğesine; LDAP kullanıcıları için Denetim Masası > Dizin Hizmeti > LDAP Kullanıcıları > Kullanıcı Ana Sayfası öğesine gidin.

Not:

Bu özelliği etkinleştirmeden önce lütfen kullanıcı ana sayfaları'nın etkinleştirildiğinden emin olun. Yerel kullanıcılar için, Denetim Masası > Kullanıcı > Gelişmiş öğesine gidin.

Kişisel bir web sitesi oluşturmak için:

  1. Kişisel web sitesini etkinleştir seçeneğini işaretlediyseniz www kullanıcının ana dizininde otomatik olarak oluşturulur.
  2. Kullanıcı artık web siteyle ilgili dosyaları (örn. web sayfaları ve ses dosyaları gibi) www klasörüne yükleyebilir.
  3. Kullanıcının dosya yüklemek için uygun erişim izinlerine sahip olduğundan emin olun.

Hata sayfasını özelleştirmek için:

  1. Hata sayfasını missing.html, missing.htm veya missing.php olarak kaydedin.
  2. Dosyayı www klasörüne yükleyin.

Kullanıcılar yerel kullanıcı web sitesinde bulunmayan bir sayfaya link verirlerse, özelleştirilen hata sayfası gösterilecektir.

Destek ve sınırlamalar:

Yerel kullanıcıların sadece şu durumlardaki verilere erişim sağlamasına izin verilmiştir:

  • Kendi ana sayfaları
  • /var/tmp
  • /tmp
  • /volumeX/@tmp

HTTP Hizmeti

80 (HTTP için) veya 443 (HTTPS için) varsayılan bağlantı noktası numaralarına ek olarak Photo Station ve Web Station'a erişmek için ilave bağlantı noktası numaraları eklenebilir. HTTP veya HTTPS için 1'den 65535'e kadar ilave bağlantı noktası numarası ekleyebilirsiniz.

HTTP veya HTTPS erişimi için ilave bağlantı noktası numarası eklemek için:

  1. İlave HTTP bağlantı noktası ekle veya İlave HTTPS bağlantı noktası ekle'yi işaretleyin (HTTPS için bir bağlantı noktası eklemeden önce Web hizmetleri için HTTPS bağlantısını etkinleştirin seçeneğinin etkinleştirilmesi gerekir).
  2. Alana istenen bağlantı noktası numarasını girin.

Not:

  • Aşağıdaki bağlantı noktaları hariç tutulmalıdır: 20, 21, 22, 23, 25, 110, 137, 138, 139, 143, 199, 443, 445, 515, 543, 548, 587, 873, 993, 995, 3306, 3689, 5000, 5001, 5005, 5006, 5335, 5432, 6881, 8080, 7000, 7001, 8081, 9997, 9998, 9999, 50001, 50002, eMule varsayılan bağlantı noktaları: 4662 (TCP), 4672(UDP) ve FTP varsayılan bağlantı noktası aralığı: gerçek aralık farklı modellere bağlı olarak değişiklik gösterebilir.

常规设置

在控制面板 > 文件服务 > FTP 中，您可以在常规区域启用速度限制、文件传输日志、安全性设置及其它高级设置。

启用 FTP 文件传输日志

单击高级设置来启用此项功能，以记录下载或上传文件的日志。

注：

启用 FTP 文件传输日志可能会影响系统性能。

更改用户根目录

单击高级设置，然后勾选更改用户根目录。这样，所选的用户只有在通过 FTP 登录 DSM 时才能访问指定的文件夹。

若要创建用目录更改规则：

  1. 在高级设置中，勾选选择用户。
  2. 单击添加。
  3. 在用户或群组中,选择要对其应用此规则的用户。
  4. 选择当该用户登录时可用的根目录：
    • 用户主目录：用户登录后进入其主目录。
    • 其他目录：选择一个共享文件夹作为根目录并单击选择。请注意，选择前应先勾选用户或群组对该文件夹的访问权限。文件夹访问权限不足的用户或群组将无法登录。
  5. 单击确定。
  6. 添加所有规则后，单击应用来保存规则。

注：

  • 规则的优先顺序将根据在列表中排序的位置。
  • 您可通过拖放操作来重新排序列表中的规则。

启用 Anonymous FTP

单击高级设置，勾选该选项以允许任何人登录到 DiskStation 设备的 FTP 服务（通过匿名或 ftp 帐户）。

变更匿名根目录

单击高级设置，并勾选此选项来变更匿名 FTP 用户的登录目录。从下拉菜单中选择一个共享文件夹。

应用默认的 UNIX 权限

单击高级设置并勾选此选项，在上传或创建文件或文件夹时应用默认的 UNIX 权限。所应用的权限将与 UNIX umask 命令所应用的权限相同。启用此选项后，文件的 UNIX 权限为 644，而文件夹为 755。禁用此选项后，文件的 UNIX 权限为 666，而文件夹为 777。默认 umask 值为 022。

注：

  • 对于启用 Windows ACL 的共享文件夹（外部设备上的“photo”和共享文件除外），请在 Linux 或 FTP 客户端上运行 chmod 命令，以将 Windows ACL 上的文件夹和文件权限类型更改到 UNIX。
  • 启动此选项可能会导致不同协议之间发生权限不一致的问题。为避免不一致问题，建议您保持禁用此选项。

启用速度限制

启用速度限制可控制用户和群组通过 FTP 传输数据的最大可用带宽。

若要启用速度限制：

  1. 如果您只要在特定时间启用速度限制，请勾选标示为启用速度限制或根据计划启用速度限制的复选框。此将应用至整个服务器，并覆盖服务器中的各用户和群组的设置。选择无速度限制可禁用所有用户和群组的速度限制设置。
  2. 单击速度限制设置可修改单个用户或群组的速度限制。您可为每个用户和群组选择下列其中一个选项：
    • 应用群组设置：速度限制设置将与用户所属群组的设置相同。
    • 无：当群组成员用此服务传输文件时不应用速度限制。
    • 恒定：一个固定速度限制常量将应用到用户或群组成员。可在左侧栏中指定上传与下载的速度限制。
    • 已自定义：可根据计划指定两种不同的速度限制并应用到用户或群组成员。单击自定义按钮可修改速度限制设置并设置计划。
  3. 单击应用。

注：

  • 要了解有关群组速度限制动态详情，请参阅此处。
  • 设置用户的速度限制后，结果区域会根据单个和群组设置显示用户的实际速度限制。例如，如果用户 “Achilles” 有 100 KB/秒的具体下载限制，但该用户所属的群组的下载限制为 500 KB/秒，则结果区域将显示 “0/500”。

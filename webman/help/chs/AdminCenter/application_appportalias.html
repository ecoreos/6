应用程序门户

应用程序门户让您配置各种应用程序的连接设置，使您在独立浏览器选项卡或窗口中直接访问和运行这些应用程序 (如 File Station)。

注：

您可为应用程序 (如 Audio Station、Download Station、Surveillance Station、Video Station、File Station) 编辑访问设置。

自定义别名

您可为每个 Synology 开发的应用程序 (如 File Station) 指定别名。通过自定义别名，您可通过特定的 URL 快速打开该应用程序。

若要自定义应用程序别名：

  1. 请进入控制面板 > 应用程序门户 > 应用程序。
  2. 请选择一个应用程序。
  3. 单击编辑 > 常规。
  4. 在弹出窗口中，选择启用自定义别名并指定别名。
  5. 单击确定来保存设置。

注：

  • 部分应用程序有默认别名：Audio Station (audio)、Download Station (download)、File Station (file)、Surveillance Station (cam) 和 Video Station (video)。
  • 别名不能与系统和浏览器保留的或其他应用程序使用的相同，且其字符数必须在 2 到 20 个字符之间。
  • 别名只能由字母、数字字符和两个特殊字符 (-) 和 (_) 混合构成。
  • 别名不可以特定字符 (-) 或 (_) 开头和结尾。

若要通过别名访问应用程序：

在为应用程序 (如 File Station) 设置别名后，您可输入以下格式的 URL 来快速打开应用程序：http://DS_IP_OR_SERVER_NAME/ALIAS/ 或 https://DS_IP_OR_SERVER_NAME/ALIAS/

例如，您可通过 URL 快速访问 File Station：http://192.168.xx.xx/file/ 或 https://MySynologyNAS/file/

自定义 HTTP/HTTPS 端口

您可为每个 Synology 开发的应用程序 (如 File Station) 指定 HTTP/HTTPS 端口。通过自定义端口，您可通过特定的 URL 快速打开该应用程序。

若要自定义应用程序 HTTP/HTTPS 端口：

  1. 请进入控制面板 > 应用程序门户 > 应用程序。
  2. 请选择一个应用程序。
  3. 单击编辑 > 常规。
  4. 在弹出的窗口中，选择启用自定义端口 (HTTP) 或启用自定义端口 (HTTPS)。
  5. 指定自定义端口号。

注：

  • 端口号范围可以是 1 和 65535 之间。
  • 以下端口不可使用，因这它们是为系统使用而保留的：
    • 20, 21, 22, 23, 25, 80, 110, 137, 138, 139, 143, 199, 443, 445, 515, 543, 548, 587, 873, 993, 995, 3306, 3689, 5000, 5001, 5005, 5006, 5335, 5432, 6881, 8080, 7000, 7001, 8081, 9997, 9998, 9999, 50001, 50002
    • eMule 默认端口：4662 (TCP)、4672(UDP)
    • FTP 默认端口范围：实际范围视不同的型号而定。

若要通过 HTTP/HTTPS 端口访问应用程序：

在为应用程序 (如 File Station) 设置别名后，您可输入以下格式的 URL 来快速打开应用程序：http://DS_IP_OR_SERVER_NAME:HTTP_PORT 或 https://DS_IP_OR_SERVER_NAME:HTTPS_PORT

例如，您可通过 URL 快速访问 File Station：http://192.168.xx.xx:7000 或 https://MySynologyNAS:7001

自定义域

您可将域名与 Synology 开发的应用程序 (如 File Station) 进行匹配。通过自定义域，您可通过特定的 URL 快速打开该应用程序。

若要自定义应用程序域：

  1. 请进入控制面板 > 应用程序门户 > 应用程序。
  2. 请选择一个应用程序。
  3. 单击编辑 > 常规。
  4. 在弹出窗口中，选择启用自定义域。
  5. 指定自定义域。
  6. 要进行其它设置，您可选择启用 HSTS 或 启用 HTTP/2 以满足您的需要。

注：

  • 设置此功能之前，您需向域供应商申请域名，以使用户可从 Internet 访问该服务。
  • 对于与文件上传/下载相关联的应用程序 (如 File Station)，我们建议您不要选择启用 HTTP/2，因为此类应用程序的上传和下载速度可能较慢。
  • 每个域名只能用于 DiskStation 中一个不同的应用程序。
  • 如果域名的长度未超出 15 个字符并符合 NetBIOS 命名规则，系统将自动注册和播送域名。
  • 当通过使用域名的 URL 访问应用程序时，您会通过标准的 HTTP 端口 (80) 或标准的 HTTPS 端口 (443)。

若要通过域访问应用程序：

在为应用程序 (如 File Station)设置域 (如 file.example.com) 后，您可输入以下格式的 URL 来快速打开应用程序：http://APP_DOMAIN 或 https://APP_DOMAIN

例如，您可通过 URL 快速访问 File Station：http://file.example.com 或 https://file.example.com

如果应用程序 (如 File Station) 的域名 (如 FileFile) 符合 NetBIOS 命名规则，Windows 用户可在有类似 URL 的同一本地网络中的 Synology NAS 上访问它：http://FileFile 或 https://FileFile

自定义反向代理服务器规则

您的 DiskStation 可用作反向代理服务器，将请求从 Internet 传输到本地网络中的设备。反向代理服务器规则可帮助您对潜在威胁隐藏敏感端口，如以下两种情况：

情况 1：假设敏感端口为 80，根据防火墙规则它不可让外部访问。您可设置反向代理服务器规则，让受信任的用户通过另一个开放端口 (如 81) 从 Internet 到达敏感端口 80。在此方式中，受信任的用户可绕过防火墙并仍可访问端口 80。

情况 2：假话敏感端口为 80，除特定设备外 (如名为“MyTrustee”)，该端口不允许外部访问。通过反向代理服务器规则，您可仅让来自 MyTrustee 的流量到达端口 80，而来自其它设备的流量则不会到达此端口。

若要设置反向代理服务器规则：

  1. 请进入控制面板 > 应用程序门户 > 反向代理服务器。
  2. 单击创建并指定以下设置：
    • 说明：指定名称有助于您识别规则功能。
    • 为来源 (从 Internet 发送请求的设备) 和目的地 (本地网络中的设备) 指定规则：
      • 协议：来源/目的地使用的 HTTP 或 HTTPS 协议
      • 主机名：来源/目的地设备的名称
      • 端口：来源/目的地设备使用的端口
      • 启用 HSTS 和 启用 HTTP/2 (仅适用于来源)
  3. 单击确定来保存设置。

注：

如果您输入 * 作为源主机名或留空，该规则将被应用任何来源。

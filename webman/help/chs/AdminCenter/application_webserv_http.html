网页应用程序

通过内置的 Web Station 和 PHP 功能，您可轻松在 DiskStation 上架设网站，并为企业或个人爱好建立动态网站。此外，“套件中心”提供各种第三方套件，如内容管理、客户关系管理和电子商务系统。将创建名为 web 的默认共享文件夹，存储您网站的文件。

若要启用 Web Station：

  1. 请进入控制面板 > 网页服务。
  2. 勾选启用 Web Station。
  3. 单击应用。

注：

  • 为了获得更好的灵活性和安全性，Web Station 使用了 http 用户组来执行任务。因此，为了控制网页的访问权限，请为各相应文件夹和文件更改 http 组的访问权限。例如，如果要让 web 共享文件夹中的 sample.htm 文件可通过 Web Station 访问，需要确认 http 组有 web 共享文件夹及 sample.htm 文件适当的读/写权限。
  • 主页文件的名称必须为 index.htm、index.html 或 index.php。
  • 当您使用 PHP mail() 功能，系统会自动采集并使用通知页面的 SMTP 服务器信息。请确认您已启用电子邮件通知并输入正确的 SMTP 服务器信息。
  • web 共享文件夹的实际路径是 /volume1/web。有些应用程序会要求用户在配置文件设置这个路径。
  • blog、photo、photosrc 和 webdefault 为系统保留的文件夹名称。因为这些目录下的文件将无法被 Web Station 使用，所以请避免在 web 共享文件夹内创建这些目录名称。
  • 您可以通过创建 missing.html 文件、missing.htm 文件或 missing.php 文件来自定义错误页面。请先启用 Web Station。然后将 missing.html、missing.htm 或 missing.php 文件置于服务器的 web 共享文件夹下。则当用户尝试链接到一个不存在的页面时，即会看见您自定义的错误页面。
  • 最大活动 HTTP 连接数： 256
  • 虚拟主机的最大数： 30

启用虚拟主机来架设许多网站

虚拟主机的支持功能，可让您在一台服务器上架设多个网站。若要着手进行，您需要按照下列步骤注册一系列域名并配置虚拟主机。

若要启用虚拟主机：

  1. 请单击虚拟主机。
  2. 单击创建。
  3. 输入 web 共享文件夹的子文件夹名称，该子文件夹将用来存放网站文件。
  4. 输入网站的主机名。
  5. 输入网站的端口。
  6. 单击确定。

启用个人网站

个人网站功能可让本地用户、域用户和 LDAP 用户方便地创建其自己的个人网站。

对于本地用户

每位本地用户将有一个唯一的网址，格式为服务器-IP/~用户帐号或服务器名称/~用户帐号。例如：

  1. 192.168.1.1/~ken
  2. SynologyNAS/~ken

对于域用户

每位域用户将拥有一个唯一的网址，格式为服务器-IP/~域名%5c用户帐号 或 服务器名称/~域名%5c用户帐号。例如：

  1. 192.168.1.1/~testdomain%5cken
  2. SynologyNAS/~testdomain%5cken

对于 LDAP 用户

每位 LDAP 用户将拥有一个唯一的网址，格式为服务器-IP/~用户帐号@域名或服务器名称/~用户帐号@域名。例如：

  1. 192.168.1.1/~ken@testdomain
  2. SynologyNAS/~ken@testdomain

注：

启用此项功能之前，请确认已启用用户主目录。对于本地用户，请进入控制面板 > 用户 > 高级；对于域用户，请进入控制面板 > 目录服务 > 域用户 > 用户主目录；对于 LDAP 用户，请进入控制面板 > 目录服务 > LDAP 用户 > 用户主目录。

注：

启用此项功能之前，请确认已启用用户主目录。对于本地用户，请进入控制面板 > 用户 > 高级。

若要创建个人网站：

  1. 如果您已勾选启用个人网站，用户的主目录中将自动创建一个名为 www 的文件夹。
  2. 用户现在可上传网页相关文件（如，页面和声音文件）至 www 文件夹。
  3. 请确认用户有上传文件的适当访问权限。

若要自定义错误页面：

  1. 请保存错误页面，如 missing.html、missing.htm 或 missing.php。
  2. 将文件上传至 www 文件夹。

当用户尝试链接到本地用户网站中不存在的页面时，将显示自定义的错误页面。

支持及限制：

本地用户仅允许访问以下数据：

  • 其自己的用户主目录
  • /var/tmp
  • /tmp
  • /volumeX/@tmp

HTTP 服务

除默认端口号 80（适用于 HTTP）或 443（适用于 HTTPS）外，可添加其它端口号用于访问 Photo Station 和 Web Station。您可以额外添加端口号（1 到 65535）供 HTTP 或 HTTPS 连接使用。

若要添加额外的 HTTP 或 HTTPS 访问端口号：

  1. 请勾选添加额外的 HTTP 端口或添加额外的 HTTPS 端口（在为 HTTPS 添加端口之前必须启用为网页服务启用 HTTPS 连接选项）。
  2. 在栏中输入所需的端口号。

注：

  • 下列端口不可使用：20、21、22、23、25、110、137、138、139、143、199、443、445、515、543、548、587、873、993、995、3306、3689、5000、5001、5005、5006、5335、5432、6881、8080、7000、7001、8081、9997、9998、9999、50001、50002、eMule 默认端口：4662（TCP）、4672（UDP）和 FTP 默认端口范围：实际范围视不同的型号而定。

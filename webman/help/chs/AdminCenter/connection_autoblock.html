自动封锁

设置自动封锁可以防止未经允许的登录。启用本服务后，登录失败次数过多的 IP 地址都会遭到封锁。通过 SSH、Telnet、rsync、网络备份、共享文件夹同步、FTP、WebDAV、Synology 行动应用程序、File Station 或 DSM 的登录失败次数都会被加总起来。您还可以拥有一个“允许列表”，它包含从不封锁的 IP 地址。

设置自动封锁：

  1. 进入常规选项卡并勾选启用自动封锁。
  2. 在登录尝试和几分钟内后面输入数字，系统将会封锁在设置分钟内登录失败超过尝试登录次数的 IP 地址。
  3. 勾选启用封锁过期并输入数字来在指定天数后删除封锁的 IP 地址。若不勾选此服务或封锁天数设置为 0，被封锁的 IP 地址将会永远留在封锁名单上，除非管理者从封锁名单手动删除。
  4. 单击应用。

管理已封锁的 IP 地址：

单击封锁名单选项卡，并从创建下拉菜单中选择下列任何项目：

  • 添加 IP 地址

    • 输入 IP 地址并设置到期时间。

  • 导入 IP 地址名单

    • 导入列有 IP 地址的文本文件以一次添加多个 IP 地址并设置到期时间。如果您要覆盖已有的 IP 地址，请勾选相应的复选框。如果该选项未被选取，则重复的 IP 地址将被跳过。

    要导入文件，该文件必须符合以下条件：

    • 该文件必须为纯文本文件。
    • 该文件的每一行必须仅包含一个 IP 地址。
    • 命令行必须以 #（井号）开头。

      • 如果您选择的文件格式正确，您将在预览区域看到所有的 IP 地址。单击确定导入。

删除已封锁的 IP 地址：

  1. 单击封锁名单选项卡可查看被封锁的 IP 地址列表。
  2. 从名单中选择要删除的 IP 地址，然后单击删除。

添加允许的 IP 地址：

单击允许名单选项卡，并从创建下拉菜单中选择下列任何项目：

  • 添加 IP 地址

    • 输入 IP 地址。

  • 导入 IP 地址名单

    • 导入列有 IP 地址的文本文件以一次添加多个 IP 地址。如果您要覆盖已有的 IP 地址，请勾选相应的复选框。如果该选项未被选取，则重复的 IP 地址将被跳过。

    要导入文件，该文件必须符合以下条件：

    • 该文件必须为纯文本文件。
    • 该文件的每一行必须仅包含一个 IP 地址。
    • 命令行必须以 #（井号）开头。

      • 如果您选择的文件格式正确，您将在预览区域看到所有的 IP 地址。单击确定导入。

删除允许的 IP 地址：

  1. 单击允许名单选项卡可查看允许的 IP 地址列表。
  2. 从名单中选择要删除的 IP 地址，然后单击删除。

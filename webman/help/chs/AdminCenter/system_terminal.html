终端机

启用控制面板 > 终端机和 SNMP > 终端机中的选项后，您的 DiskStation 就可支持 Telnet 和 SSH 命令行界面的服务。您还可以更改 SSH 加密算法的安全级别。

若要启用 Telnet/SSH 服务：

  1. 请勾选您想要启用的通讯协议 (SSH 或 Telnet) 旁的复选框。
  2. 单击应用。

若要更改 SSH 服务端口：

  1. 在端口中，输入新的端口号。
  2. 单击应用。

各种不同安全等级的加密算法用于保护 SSH 连接的安全。您可自行决定最适合您需要的算法。

若要更改 SSH 加密算法安全等级：

  1. 单击高级设置。
  2. 为加密算法选择安全等级。如果您要指定使用的确切算法，请选择自定义从这三个群组中进行选择：加密、KEX (密钥交换) 和 MAC (消息验证代码)。
  3. 单击应用。

部分型号支持特定加密算法的硬件加速。您可勾选仅使用硬件加速密码可提高 SSH 相关服务的传输速度 (包括加密网络备份和 SFTP 服务)。在高级页面，以红色显示使用硬件加速的算法性能。

注：

  • 部分服务 (例如网络备份、SFTP) 会自动启用 SSH，并打开端口 22。在此情况下，您必须在控制面板 > 终端机和 SNMP > 终端机中启用 SSH 服务以使用 SSH 登录 DSM。
  • SSH/Telnet 仅支持属于 administrators 群组的帐户登录系统。若要切换到root帐户，请通过 SSH/Telnet 以属于 administrators 群组的用户身份登录系统，运行命令 sudo -i，然后输入用于登录的帐户密码。
  • 以 SSH 或 Telnet 登录系统时，密码不能留空。
  • 禁用 SSH 服务后，所有的 SSH 连接都将终止，包括任何正在执行的加密网络备份任务。建议您在禁用 SSH 服务后，重新执行网络备份任务。
  • 任何不当操作或修改 DiskStation 都可能造成产品毁损或数据丢失。
  • 选择更高的安全等级或仅使用硬件加速密码可能会造成部分 SSH 客户端因兼容性问题而无法连接。
  • 根据所选安全等级，部分支持硬件加速的算法可能无法使用。

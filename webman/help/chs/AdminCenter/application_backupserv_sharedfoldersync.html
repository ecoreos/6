共享文件夹同步

“共享文件夹同步”服务可让用户将共享文件夹从一台 Synology NAS 设备（来源）同步至局域网或 Internet 上的另一台 Synology NAS 设备（目的地）上。

第一次进行同步时，共享文件夹同步功能会将所选的共享文件夹进行完整的同步。之后则会仅同步文件修改过的部分。在来源端删除的文件，在目标端也会被删除。

注：

  • “共享文件夹同步”是一个单向同步解决方案，将来源文件中的文件同步到目的地。对于双向同步解决方案，请使用 Cloud Station ShareSync。
  • 如果在目的地存在与来源共享文件夹相同名称的共享文件夹 (如 SharedFolder)，则目标文件夹将被命名为“SharedFolder_1”。
  • 如果在目的地不存在与来源共享文件夹相同名称的共享文件夹，则将使用相应或首个可用卷中的名称来创建文件夹。
  • 默认没有用户有访问目的地上共享文件夹的权限。请联系目的地 admin 以获得权限。

启用此服务

若要启用共享文件夹同步：

您必须先选在目的地 Synology NAS 上启用此服务。

  1. 请进入控制面板 > 文件服务 > rsync。
  2. 选择启用同步服务以启用“共享文件夹同步”服务。
  3. 单击应用。

若要取消共享文件夹与目的地 Synology NAS 的链接：

如果不同的来源同时将相同的共享文件夹同步到一个目的地 DiskStation，将发生冲突。在另一个来源可将其文件夹同步到目的地之前，您需要取消与来源地共享文件夹的链接。

  1. 登录到目的地 Synology NAS，进入控制面板 > 共享文件夹同步 > 连接列表。
  2. 从来源 Synology NAS 选择冲突连接，并单击取消链接以将其删除。

现在另一个来源就能将该共享文件夹同步到目的地 DiskStation 了。

管理同步任务

若要创建同步任务：

  1. 登录到来源 Synology NAS，进入控制面板 > 共享文件夹同步 > 任务列表。
  2. 单击创建。
  3. 输入任务的名称。
  4. 选择您要同步至目的地的共享文件夹。
  5. 指定目的地 Synology NAS 及下列同步选项：
    • 为加密共享文件夹同步自定义 SSH 加密端口：为 SSH 传输加密指定加密端口。
    • 启用 SSH 传输加密：传输过程中对数据加密。提供较高的安全性，但非加密传输的性能较佳。您可按需要选择。
    • 启用传输压缩：传输过程中对数据压缩。可节省网络带宽，但会增加 CPU 负担。
    • 启用段落分块同步：仅传输不同的数据而非整个文件。可节省网络带宽，但会增加 CPU 负担。
  6. 提示后，选择以下选项之一以决定什么时候从来源同步到目的地。
    • 数据异动时自动进行同步：一旦来源共享文件夹内有任何变化，立即运行同步任务。
    • 手动运行同步：从来源共享文件夹手动运行同步任务。
    • 高级计划：单击进度计划可指定执行同步任务的时间。
  7. 单击应用。现在您可以在任务列表上看到同步任务，系统会按所指定的计划管理运行任务。

若要管理同步任务：

  1. 在来源 Synology NAS 中，进入控制面板 > 共享文件夹同步，单击任务列表，并选择所需的任务。
  2. 执行下列任何操作：
    • 单击编辑可编辑任务。
    • 单击删除可删除任务。
    • 如果该同步任务尚未进行，单击立即同步可立即运行任务。
    • 如果该同步任务正在进行，单击取消可停止正在进行的任务。

      • 注：

      如果同步任务的计划设置为数据异动时自动进行同步，单击取消会停止进行中的同步任务。然而，如果同步任务监控的任何文件夹内容有发生异动，[共享文件夹同步] 会继续进行同步任务。

    • 单击全部同步以同步所有数据。“共享文件夹同步”第一次运行同步任务时会运行完整同步，并在第一次同步完成之后仅同步有修改的部分。单击此按钮可辅助您以手动方式再度同步所有的数据。

更多信息

  1. 任务名称必须设置为 1 到 32 个 Unicode 字符，但不可为下列字符：
    ! " # $ % & ' ( ) * + , / : ; <= > ? @ [ ] \ ^ ` { } | ~
  2. 如果您要将 ext4 文件系统的共享文件夹同步至 ext3 的文件系统，同步任务的文件夹数量若超过 32,000 个时可能会发生错误。
  3. 若目的地 DiskStation 上的共享文件夹曾与其他 DiskStation 进行同步，“共享文件夹同步”在进行任务时会做完整的同步。
  4. 要运行共享文件夹同步任务，您必须拥有目的地 DiskStation 的管理权限。您必须拥有 admin 帐户的密码，或为管理群组内的一员。
  5. 当您通过 Internet 同步到目的地 DiskStation 时，在服务器名称栏您可输入其外部 IP 地址或主机名来链接。然而，如果您选择以主机名来链接，此主机名必须符合 FQDN（Fully Qualified Domain Name）格式，例如“www.synology.com”。
  6. “共享文件夹同步”在端口转送时，使用端口 22 进行加密传输和使用端口 873 进行非加密传输。
  7. “共享文件夹同步”不支持外接存储设备。
  8. 当“共享文件夹同步”任务运行后，将自动在目的地装载加密共享文件夹；但要访问其中的数据，必须先在目的地装载加密共享文件夹。
  9. 如果对所选共享文件夹没有更改，则“计划的共享文件夹同步”将不会在特定时间运行。
  10. 为获得最佳性能，我们建议每个同步任务包含的共享文件夹不超过 200 个。
  11. 如果 homes 共享文件夹从源 DiskStation 同步到目的地 DiskStation，则用户主目录功能在目的地 DiskStation 将被禁用。
  12. 对于带非 Intel CPU 的型号，您可使用共享文件夹同步最多 100,000 个共享文件夹（包括子文件夹）。
  13. “共享文件夹同步”在目的地的文件系统为 ext4 时，可以备份 Windows ACL 设置。
  14. 如果您要同步到与 DiskStation 共享相同名称的目标服务器，共享文件夹同步将失败。
  15. 来源中的高级共享文件夹权限将被同步到目的地。此外，在目的地将创建 administrators 群组的只读规则。

Windows 文件服务

Windows 文件服务可让支持 SMB/CIFS 的 Windows、Mac 和 Linux 客户端访问存储在 DiskStation 中的数据。如果您要通过 Windows 资源管理器浏览 DiskStation 上的文件或装载共享文件夹作为网络驱动器，请进入控制面板 > 文件服务 > Win/Mac/NFS 启用此选项。

若要启用 Windows 文件服务：

  1. 请勾选启用 Windows 文件服务。
  2. 单击应用。

注：

启用 Windows 文件服务后，您可通过 SMB 将全面建索文件夹装载到 Mac 计算机，并使用 Finder 上的 Spotlight 搜索已装载文件夹中的文件内容。

工作群组

此选项可让您将系统指定为一系列特定工作群组以进行网络管理。

注：

  • 工作群组的名称可包含 1 至 15 个字符。但名称不得包含以下字符： [ ] ; : " < > * + = \ / | ? ,
  • 如果您 Mac 的系统版本为 OS X 10.9，且 DiskStation 已加入 Windows 域，则您可在 DiskStation 本地用户名前添加“localhost\”进行登录，通过 Windows 文件服务功能访问存储在 DiskStation 上的文件。

不可访问以前版本

在 Windows 资源管理器中，用户可在共享文件夹中单击以前版本选项卡来访问文件夹或文件的快照，允许还原或复制数据而无需管理员的帮助。如果您选择不可访问以前版本，则“以前版本”页面将不会显示任何信息。

注：

  • 用户仅在以之前版本拍摄快照、其用户帐户已创建并已授予拍摄相应快照所在共享文件夹的读取权限后才可访问之前的版本。

启用传输日志

勾选这个选项，系统将会记录通过 CIFS/SMB 协议，即所谓的 Windows 文件传输协议，访问文件的记录，包含对文件及文件夹的创建、删除、读取、写入、移动及重命名。

高级设置

单击高级设置按钮可查看以下选项。

  • WINS 服务器：Microsoft Windows Internet 名称服务 (WINS) 是一个 NetBIOS 名称对 IP 地址的映射服务。如果系统配置使用 WINS 服务器来进行注册，Windows 的用户可以更容易地将系统设置在 TCP/IP 网络上。
  • Windows 文件服务的最大协议数：根据网络配置，您可设置 DiskStation 支持的最高协议级别。
    • 启用 SMB 2：SMB (Server Message Block) 2 自从 Windows Vista 发布以来即被支持，且为 SMB 的增强版本。SMB 2 增加了将多重 SMB 操作功能组合到单个请求的功能，以减少网络数据包的数量并提高性能。
    • 启用 SMB 2 和 Large MTU：最大传输单元 (MTU) 是指可通过通讯协议的最大数据单元。为利用最快的网络接口，如 1- 或 10-gigabit 以太网，Large MTU 将最大传输单元提高至 1 megabyte (MB)。启用 Large MTU 可提高大文件传输的速度和效率，同时降低需处理的数据包数量。
    • 启用 SMB 3：SMB 3 自 Windows 8 和 Windows Server 2012 以来得到支持。它是 SMB 2 的增强版。SMB 3 支持基于 AES 的文件加密传输，从而提高了对等文件传输的安全性。

    注：

    • Windows Vista、Windows Server 2008 R2、Windows 7 和以上的版本支持 SMB 2。
    • Windows Server 2008 R2、Windows 7 和以上的版本支持 Large MTU。
    • 您需要修改计算机的注册设置来使 Windows 计算机的 Large MTU 选项生效。详情请参见 Microsoft 技术支持 (如本文章)。
  • 传输加密模式：启用 SMB 3 后，Windows 文件服务将增加传输加密以加强文件传输的安全性。
    • 禁用：不应用传输加密。
    • 自动：传输加密将只应用到支持此功能的客户端。
    • 强制：始终应用传输加密。此选项将造成不支持传输加密的客户端无法使用 Windows 文件服务。

    • 注：

    • Windows 8、Windows Server 2012 和以上的版本支持传输加密。
    • 启用传输加密后，不支持匿名登录 Windows 文件服务。
    • 启用传输加密后，请取消勾选禁用 Opportunistic Locking 以免应用程序超时。
  • 禁用 Opportunistic Locking：勾选此选项将禁用 CIFS 中的 "机会锁定"，并可提升数据库程序经由网络访问 DiskStation 设备共享文件夹上数据库文件的性能。
  • 启用 Local Master Browser：当多台 Windows 计算机存在于同一子网络时，它们将协议并选择一台计算机作为 "local master browser"。Local Master Browser 将维护子网中的其它计算机及其它共享资源的列表，并与其它计算机共享这些列表。此选项可让 DiskStation 作为 Local Master Browser 使用。

    • 注：

    启用此选项将禁用硬盘休眠功能，且不需要密码就能激活 guest 帐户。

  • 启用 DirSort VFS 模块：按字母顺序排列文件夹名称。启用此选项会影响系统性能。
  • 禁止文件：根据特定条件隐藏文件。条件可包括统配符 (*)，并使用斜线号 (/) 分隔多项条目。例如：/abc*/*.txt/*.conf/。启用此选项会影响系统性能。
  • 允许在共享文件夹中使用符号链接：启用此选项可允许符号链接指向位在同一共享文件夹中的文件或文件夹。符号链接可以使用 DSM 终端机来创建。
  • 允许跨共享文件夹的符号链接：启用此选项可允许符号链接指向不同共享文件夹中的文件或文件夹。符号链接可以使用 DSM 终端机来创建。请注意，启用此选项可能会让无访问权限的用户查看并访问符号链接指向的共享文件夹。
  • 启用 MSDFS VFS 模块：允许符号链接指向位于其他 SMB 服务器中的文件或文件夹。对于旧版本的 Windows，如 Windows XP，客户端和 DiskStation 需要加入 Windows 域后才能启用该选项。如需更多有关创建 MSDFS 链接的信息，请参阅此处。请注意，禁用此选项后将会清除客户端设备的缓存，以避免影响正常的操作性能。
  • 禁用同一 IP 地址的多个连接：启用此选项后，DiskStation 限制客户端一次只能一个 IP 地址使用 Windows 文件服务。使用 Windows 文件服务且使用同一 IP 地址的多个客户端，DiskStation 将保留最新连接而中断所有旧连接。在 NAT 环境下，客户端可能只使用一个 IP 地址，在此情况下启用此选项会影响正常的操作性能。
  • 收集调试日志：启用此选项后将保存更详细的日志供调试之用。启用此选项会影响系统性能。
  • 应用默认的 UNIX 权限：启用此选项可在上传或创建文件和文件夹时应用默认的 UNIX 权限。文件的 UNIX 权限为 744，而文件夹的 UNIX 权限为 755。禁用此选项后，文件与文件夹的 UNIX 权限均为 777。

    • 注：

    • 启动此选项可能会导致不同协议之间发生权限不一致的问题。为避免不一致问题，建议您保持禁用此选项。

    注：

    • 对于启用 Windows ACL 的共享文件夹（外部设备上的 "照片" 和共享文件除外），请在 Linux 或 FTP 客户端上运行 chmod 命令，以将 Windows ACL 上的文件夹和文件权限类型更改到 UNIX。
    • 启动此选项可能会导致不同协议之间发生权限不一致的问题。为避免不一致问题，建议您保持禁用此选项。
  • 请勿在创建文件时保留硬盘空间：启用此选项可在 DiskStation 上创建文件时拒绝硬盘空间被保留。
  • 清除 SMB 缓存：单击此按钮可删除以下 SMB 缓存文件以排除故障：
    • /var/cache/samba/netsamlogon_cache.tdb
    • /var/lib/samba/winbindd_cache.tdb
    • /var/lib/samba/share_info.tdb
    • /var/lib/samba/registry.tdb
    • /run/samba/connections.tdb
    • /run/samba/gencacche.tdb
    • /run/samba/gencache_notran.tdb

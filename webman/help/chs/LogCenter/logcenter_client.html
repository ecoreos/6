日志发送

如果您有 syslog 服务器，您可按以下指示将 DiskStation 配置为syslog 客户端，将 DiskStation 上的本地系统日志传送至 syslog 服务器来集中监控及管理。

若要发送日志至 syslog 服务器：

  1. 请进入日志中心 > 日志发送。
  2. 勾选将日志发送至 syslog 服务器复选框。
  3. 在服务器栏中输入 syslog 服务器的主机名或 IP 地址。
  4. 在端口栏中指定接收 syslog 信息的端口号。
  5. 从传输协议中选择 UDP 或 TCP。它应匹配 syslog 服务器所用的协议。
  6. 从日志格式中选择 BSD (RFC 3164) 或 IETF (RFC 5424) 以使用所选格式发送日志。

    7. 注：

    • 按照默认，端口为 514。如果您要为 syslog 服务器重新指定不同的端口，请先设置适当的端口转发规则。
    • UDP（用户数据报协议）是一种基于信息的简易连接协议，传输速度较快，但缺乏安全性和错误纠正能力。TCP (传输控制协议) 提供有保证的传送，它使用流量控制来确定所有的数据包是否成功发送，然后处理新的进程。
    • 安全连接 (SSL) 仅在使用 TCP 时可用。
    • 如果您选择 TCP 并要确保 syslog 与 SSL 的连接安全，可勾选启用安全连接（SSL）选项。然后单击导入证书并选择证书来将日志安全传输到 syslog 服务器。
  7. 单击发送测试日志来确认 syslog 服务器可成功接收日志。如果 syslog 服务器未收到测试日志，请确认您的设置是否正确。
  8. 单击应用来保存设置。

过滤日志

在过滤器选项卡中，您可根据类别（如系统、连接、备份）或严重性（如警告、错误、信息）来指定发送到 syslog 服务器的日志类型。

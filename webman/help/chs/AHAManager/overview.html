总览

本页显示 high-availability 集群和成员服务器的基本信息。另外，本页提供在 high-availability 集群和成员服务器上执行维护的工具。

High-Availability 集群状态

本栏显示 high-availability 集群的整体状态，提醒您任何错误的发生。下表为本栏中可能显示的状况：

状态 描述
良好 High-availability 集群整体状况良好。
正在加载 系统正在加载集群状态。在状态加载完成后才能执行移转。请参见移转。
正在配置 系统正在配置。在配置完成后才能执行移转。
警告 HA 集群的状态异常，但服务仍在运行。移转在此情况下可能无法执行。
危险 活动和无源服务器都发生故障，或存储器损毁。High-availability 集群无法正常运作。请检查所有存储空间（例如 RAID Group、硬盘组、卷和 iSCSI LUN）的状态。移转在此情况下无法执行。

警告信息

下表列示了系统出现错误时可能显示的警告信息。当出现多个警告时，您可用向前/向后按钮浏览它们。

警告/错误 描述
此 high-availability 集群中没有无源服务器 无源服务器的信息无法获得。请确认无源服务器正确连接。在这种情形下，无法使用移转。
无法检测无源服务器 该错误可能是由于无源服务器离线或活动服务器与无源服务器无法通过 Heartbeat 连接进行通信。在这种情形下，无法使用移转。
无法执行故障移转 在活动服务器上触发了故障移转，但移转失败。在这种情况下，活动服务器不会尝试执行另一个故障移转。请按以下步骤修复此错误：
  1. 尝试解决 high-availability 集群中的问题。
  2. 重启无源服务器。
  3. 单击管理 > 移转以执行移转。
监控的目标错误 在一个或多个监控的目标中出现错误，包括系统状态、服务或网络接口。
在扩充设备和活动/无源服务器之间的连接中出现错误。 扩充设备未正确连接到活动或无源服务器。请确认其正确连接到活动服务器，并对至无源服务器的连接作相同的设置。在这种情形下，无法使用移转。
卷处于堪用状态 存储空间（例如 RAID Group、硬盘组、卷、iSCSI LUN 和 SSD 缓存）已处于堪用状态。请检查存储状态页面。
卷已损毁 存储空间（例如 RAID Group、硬盘组、卷、iSCSI LUN 和 SSD 缓存）已损毁。请检查存储状态页面。如果无源服务器尝试识别活动服务器无法访问的硬盘，将自动执行故障移转。否则，将无法进行切换。
Heartbeat 连接丢失。 该错误可能是由于 Heartbeat 连接失败造成的。在这种情形下，无法使用移转。
硬件故障 此错误可能是由于活动服务器或无源服务器上的硬件故障（例如电源设备或系统风扇）所造成的。
不同的物理内存大小 此错误可能是由于活动服务器或无源服务器上的物理内存大小不匹配所造成的。在这种情形下，无法使用移转。
未对此 high-availability 集群指定静态 IP 地址 此错误可能由于 high-availaility 集群缺少静态 IP 地址所造成的。在这种情形下，无法使用移转。
监控接口上没有静态 IP 地址 此错误可能由于受监控接口缺少静态 IP 地址所造成的。在这种情形下，无法使用移转。
还原无源服务器 无源服务器还原进行中。在这种情形下，无法使用移转。
正在升级无源服务器 系统正在升级无源服务器。在这种情形下，无法使用移转。

High-Availability 集群“管理”按钮

此按钮位于 high-availability 集群信息下方。其包含以下操作：

  • 移转：进行从活动服务器到无源服务器的移转。在该过程中，活动服务器将停止提供服务并成为新的无源服务器。然后，旧的无源服务器将成为新的活动服务器，并开始提供 high-availability 集群的所有服务。当无源服务器发生错误或未被选定时，此选项将无法使用。
  • 更新 DSM：打开控制面板的 DSM 更新。两个服务器都可在此进行更新。
  • 关闭 high-availability 集群：关闭活动和无源服务器。
  • 关闭活动服务器：关闭活动服务器。如果无源服务器正常，服务将移转。
  • 关闭无源服务器：关闭无源服务器。
  • 重启活动服务器：重启活动服务器。如果无源服务器正常，服务将移转。
  • 重启无源服务器：重启无源服务器。
  • 还原未知服务器：如果无源服务器被更新或仍被识别为未知，则还原无源服务器。
  • 系统哔声关闭：发生错误时关闭活动服务器和无源服务器的哔声。如果以后发生其它错误，系统仍会在活动服务器和无源服务器上发出哔声。

注：

  • 如果您关闭无源服务器或没有无源服务器，一旦活动服务器发生错误，将无法移转服务。

服务器图解和信息

服务器图解显示服务器状态。与服务器连接的模块侧面相对应的位置。将鼠标放在其上可获得更改细节。

其包含以下情况：

  • 此 high-availability 集群中没有无源服务器：显示空白框。
  • 无法检测无源服务器：显示模糊的服务器图像。
  • 感叹号：硬件可能出现故障，或服务器信息因 Heartbeat 连接失败而无法获得。

注：

  • 如果在另一个服务器上显示“未知”，这意味着该服务器不属于 high-availability 集群。单击管理然后选择还原无源服务器以修复此问题。

服务器信息包括下列条目。如果未检测到无源服务器，它会显示无。

  • 序列号：Synology 服务器的序列号。
  • 风扇状态：服务器冷却风扇的状态。如果一个风扇发生故障，将出现警告。在这种情况下，请联系本地经销商获得帮助。
  • 温度：Synology 服务器的温度。
  • 电源状态：指出服务器电源是否工作正常。如果该服务器配有冗余电源，则当其中一个发生故障时，将出现警告。
  • 物理内存：Synology 服务器的物理内存大小。如果两台服务器没有相同的物理内存大小，切换无法执行。

服务器之间的连接状态

此区域包含活动和无源服务器之间的图像。其包含以下情况：

  • 无图像：表示未检测到无源服务器。
  • 静态图像：表示 Heartbeat 连接不正确。
  • 感叹号：表示两台服务器之间连接异常，或 Heartbeat 连接不正确。

网络接口状态

此区域包含服务器和数据网络之间的图像。其包含以下情况：

  • 无感叹号：表示未检测到网络故障。
  • 感叹号：表示在监控的网络接口中发生故障。

共享存储器图解和信息

共享存储器图解显示了服务器和共享存储器之间的连接状态。将鼠标放在其上可获得基本信息。

其包含以下情况：

  • 无感叹号：表示未检测到错误。
  • 感叹号：表示扩充设备的异常连接状态。

共享存储器基本信息包括下列条目。请进入存储管理器来了解更多信息。

  • 风扇状态：显示扩充设备风扇的状态。如果发生风扇故障，将出现警告。在这种情况下，请联系本地经销商获得帮助。
  • 电源状态：指出服务器电源是否工作正常。如果其中一个发生故障，将出现警告。

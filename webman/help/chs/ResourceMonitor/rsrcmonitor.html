资源监视器

资源监视器让您可以监控 CPU 使用率、内存使用率、硬盘利用率和网络流量。您可选择实时监视或查看之前的数据。

资源监视器让您可以监控 CPU 使用率、内存使用率和网络流量。您可选择实时监视或查看之前的数据。

注：

  • 如果在设置选项卡中勾选了启用使用历史记录，将仅显示之前的数据。
  • 对于 Docker DSM，CPU 和内存表示有关 Docker Host。

性能

单击性能选项卡可查看资源利用率数据。鼠标移至其上时将显示图表的详细数据、时间和信息。

CPU

您可看到用户、系统和 I/O 等待的整体 CPU 使用率。初次启动资源监视器时，CPU 负载可能会很高，因为系统除了正在收集系统资源的统计数据外，同时还需负载显示页面。

用户显示运行用户应用程序使用的 CPU 百分比。系统显示在系统核心和硬件/软件 IRQ 中使用的 CPU 百分比。I/O 等待显示 I/O 等待中消耗的 CPU 百分比。

利用率是用户和系统 CPU 使用率的总和。

显示 1、5 和 15 分钟以内的负载平均，并显示运行队列长度 (即当前进程总数加上等待运行数)。

名词解释

  • I/O 等待：“I/O 等待”表示 CPU 空闲且未被任何程序所使用。

    • I/O 等待为常规行为。由于 CPU 的速度远快于 I/O，因此 CPU 经常要等待硬盘或网络的传输的完成。但如果 I/O 等待的百分比非常高，表明 CPU 在 I/O 等待上花子大部分时间，且可能未很好地利用。您可以使用效率更高的磁盘组配置 (如 SHR 或 RAID 5) 或网络配置 (如绑定) 来获得更好的系统性能。

内存容量

将显示物理内存的整体内存和交换空间使用率。内存使用率维持很高是因为系统将缓存数据放置在快速缓冲贮存区，所以数据可以很快从快速缓冲贮存区被取得而不用真的去读取硬盘上的数据。当内存不够时，快速缓冲贮存区将会释放内存。

较高交换空间使用率显示系统内存不足，并将影响到系统性能。您可选择顶部下拉菜单中的交换以查看换入和换出的速率。

网络

网络流量图会以每秒多少 KB 的方式显示数据传送与接收的流量。若有启用 PPPoE 服务，亦会显示 PPPoE 的网络流量。

硬盘

您可单击图表顶部的下拉菜单选择查看整体磁盘利用率、传输率或 IOPS。您可单击底部的查看全部，通过勾选复选框选择更多磁盘以将其数据添加到图表。

卷

此页面显示了卷的整体传输状态。单击图表上方的下拉菜单可查看传输利用率、传输率或 IOPS。单击查看全部按钮可选择单个卷并将其添加到图表中。

iSCSI

此页面显示了当前运行 iSCSI LUN 的整体状态。单击图表上方的下拉菜单可选择 LUN 并查看其状态。默认显示三个图表，分别为传输性能、IOPS 和 延迟.要获得更详细的信息，单击选择图表按钮以选择要查看的其它图表。

进程

单击进程选项卡可按 CPU 负载的降序查看各进程的 CPU 和内存使用率。

进程状态包括正在运行、正在休眠和已停止。进程状态对应至 Linux 则为 running、stopped/tracing，sleeping。

正在休眠后跟一个字母以表示详细信息，如“D”表示硬盘休眠，“Z”表示僵停，“X”表示不活动。

目前连接用户

您可以在此选项卡查看并管理目前连接到系统以及正在访问资源的客户端。此区块会显示通过 HTTP、TELNET、SSH、SMB/CIFS、AFP、FTP 和 NFS 连接的客户端。

若要管理当前连接，请执行下列任一操作：

  • 单击保存导出当前列表为 HTML 文件。
  • 单击刷新刷新列表。
  • 单击终止连接停止用户访问资源。断开连接的用户必须重新授权才能访问资源。
  • 单击禁用用户将用户禁用。要重新启用该用户，进入主菜单 > 控制面板 > 用户。

注：

  • 终止连接选项在 NFS 客户端无法使用。
  • 如果 NFS 客户端未正确卸载共享文件夹，它们仍会出现在已连接用户列表中。
  • 如果 DiskStation 重启，之前连接的 NFS 客户端将不会出现在已连接用户列表中。

速度限制

在速度限制选项卡下，您可查看和管理在 DiskStation 上应用速度限制的文件传输进程列表。

若要刷新列表：

请单击刷新来重新加载页面，并获得最新的记录。

若要停止文件传输进程：

请单击终止连接来停止文件传输。您必须重新开始文件传输才能成功完成该进程。

设置

单击设置可设置以下选项：

  • 从下拉菜单中选择实际持续时间 (按分钟)。
  • 如果要查看之前的数据，勾选启用使用历史记录。当它启用后，您可在查看性能图表时，从右上角下拉菜单中的实时、1 天、1 周、1 个月和1 年之间切换。

单击应用以保存该设置。

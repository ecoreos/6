关闭和重置

以下指示将引导您完成关闭、删除或重置 high-availability 集群的整个过程。

若要使用 Synology High Availability 关闭 high-availability 集群：

如果您想要关闭 high-availability 集群来将服务器搬至新的位置或进行维护，但是您不想删除 high-availability 集群配置，请遵循这些指示。

  1. 请登录 DSM。
  2. 打开 Synology High Availability。
  3. 单击标记为管理的按钮。
  4. 选择关闭 HA 集群。
  5. 将显示确认信息。点击是进行确认。

若要手动关闭 high-availability 集群：

High-availability 集群也可以使用每台服务器上的电源按钮来手动关闭。

  1. 请按住一台服务器（活动或无源）上的电源按钮，直至发出哔声。
  2. 一台服务器完全关闭后，等待至少 30 秒（仍然联机的服务器需要时间保存设置）。
  3. 按住另一台服务器上的电源按钮，直至发出哔声。
  4. 等待服务器完全关闭。

若要开启服务器：

如果您想要再次打开服务器，应同时打开活动和无源服务器（或在 5 分钟内打开两台服务器）。如果您仅开启一台服务器，它会尝试通过播送网络唤醒信息来开启另一台服务器。

异常关机：

创建 high-availability 集群后，控制面板中的“断电自动重启”选项会自动启用。如果服务器由于断电而导致异常关机，服务器将在恢复供电后自动开机。

若要删除 high-availability 集群：

如果您想取消绑定两台服务器并删除 high-availability 集群来绑定另一台服务器，请遵循这些指示。

  1. 前往总览页面。
  2. 单击管理按钮。
  3. 选择删除 HA 集群。

确认后，系统将删除 high-availability 集群。状态栏将会显示删除进程。成功删除 high-availability 集群后，您将被引导至活动服务器的原始 IP 地址。两台服务器的状态将更改为独立。

若要重置发生故障的 high-availability 集群：

如果因为 split-brain 错误或其他故障而无法访问 DSM 或 Synology High Availability，您可以按下服务器上的 Reset 按钮来尝试强行取消 high-availability 集群绑定。

  1. 按下 Reset 按钮（通常位于后面板上），直至服务器发出哔声。
  2. 系统将删除 high-availability 集群，而两个服务器的状态将更改为独立
  3. 按下 Reset 按钮也会将某些设置恢复到默认，例如 IP 地址、DNS 和 Admin 密码，还有以下选项：
    • Admin 密码将重置为空。
    • Guest 帐户将禁用。
    • IP、DNS、网关和其他网络接口将启用 DHCP。
    • JumboFrames 将禁用，且 MTU 将设置到 1500。
    • 管理端口将重置为 5000/5001。
    • PPPoE 将重置为默认。
    • IP 自动封锁将禁用（已封锁 IP 地址不会删除）。
    • Link Aggregation 将禁用。
    • 防火墙规则将禁用（不会删除规则）。

注：

删除 high-availability 集群过程中，可能有一些不完整的数据写入操作。若要确保数据完整性，请在活动服务器上执行数据复制，因为无源服务器可能丢失一些数据更新。

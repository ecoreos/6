还原

此页面可让系统管理员执行还原。要让用户访问共享文件夹快照，请为各共享文件夹启用让快照可见，以使用户可通过 File Station 或 Windows 文件资源管理器访问快照。

共享文件夹

若要还原共享文件夹中的各个文件/文件夹：

  1. 选择您要还原的共享文件夹。
  2. 单击还原。
  3. 选择要还原的快照版本。
  4. 单击浏览，并复制要还原到目的地的文件/文件夹。

若要还原共享文件夹：

  1. 选择您要还原的共享文件夹。
  2. 单击还原。
  3. 选择要还原的快照版本。
  4. 从下拉列表中单击操作，然后选择还原到适当位置。
  5. 您将看到确认信息。单击是来确认操作。

若要将快照克隆到新的共享文件夹：

  1. 选择您要克隆的共享文件夹。
  2. 单击还原。
  3. 选择要克隆到新共享文件夹的快照版本。
  4. 从下拉列表中单击操作，然后选择以新名称克隆。
  5. 您将被重新引导到克隆页面（在控制面板 > 共享文件夹 > 创建）。输入克隆共享文件夹的名称，并单击确定完成。

注：

  • “homes”共享文件夹无法还原。
  • 克隆的共享文件夹无法加密。
  • 如果在任何文件尚在传输时拍摄快照，则无法保证数据一致性。
  • 加密共享文件夹的限制：
    • 已装载加密共享文件夹无法执行还原。
    • 加密共享文件夹无法执行以新名称克隆。
    • 已卸载的加密共享文件夹可使用还原到适当位置，从之前拍摄的快照还原。

若要切换共享文件夹的复制：

  1. 选择您要切换的复制共享文件夹。
  2. 单击切换。
  3. 如果有多个共享文件夹的复制任务，请选择新的目标服务器。
  4. 单击切换可开始操作。

若要故障移转共享文件夹的复制：

  1. 选择您要故障移转的复制共享文件夹。
  2. 单击故障移转 > 故障移转。
  3. 选择要还原的快照版本。
  4. 单击故障移转可开始操作。

注：

  • 您只能在源服务器发生故障时启动故障移转。

若要重新保护共享文件夹的复制：

当复制任务被故障移转到目标服务器后，复制任务将停止在两个服务器之间的同步数据。您需要在原始源服务器还原后重新保护复制任务。

  1. 选择您要重新保护的故障移转共享文件夹。
  2. 单击故障移转 > 重新保护。
  3. 选择新源服务器。
  4. 如果有多个共享文件夹的复制任务，请选择新的目标服务器。
  5. 单击重新保护可开始操作。

注：

  • 在新目标服务器上已有的复制共享文件夹数据将被新源服务器上的数据覆盖。
  • 如果所选的故障移转快照版本未在伙伴服务器上，可能需要较长的时间来重新保护。

若要为复制的共享文件夹执行测试故障移转：

  1. 选择您要测试的复制共享文件夹。
  2. 单击测试故障移转 > 测试故障移转。
  3. 如果有多个共享文件夹的复制任务，请选择测试服务器。
  4. 输入共享文件夹的测试故障移转名称，并选择要测试的快照版本。
  5. 单击测试故障移转可开始操作。

注：

  • 执行复制的切换/故障移转将清除其测试故障移转。
  • 您无法为测试故障移转共享文件夹创建复制任务。

若要为复制的共享文件夹清除测试故障移转：

  1. 选择您要清除其测试故障移转的复制共享文件夹。
  2. 单击测试故障移转 > 清除测试故障移转。
  3. 勾选您要清除的测试故障移转。
    注：如果您不再要保存为测试故障移转创建的共享文件夹，勾选删除测试故障移转数据。
  4. 单击清除测试故障移转可开始操作。

iSCSI LUN

若要恢复 iSCSI LUN：

  1. 选择您要恢复的 iSCSI LUN。
  2. 单击还原。
  3. 选择要从中还原的快照版本。
  4. 从下拉列表中单击操作，然后选择还原到适当位置。
  5. 您将看到确认信息。单击是来确认操作。

若要将快照克隆到新的 iSCSI LUN：

  1. 选择您想要克隆的 iSCSI LUN。
  2. 单击还原。
  3. 选择要克隆到新 iSCSI LUN 的快照版本。
  4. 从下拉列表中单击操作，然后选择以新名称克隆。
  5. 您将被重新引导到克隆页面（在存储管理器 > iSCSI LUN 中）。输入克隆 iSCSI LUN 的名称，并单击确定完成。

注：

  • 如果在任何文件尚在传输时拍摄快照，则无法保证数据一致性。

若要切换共享文件夹的复制：

  1. 选择您要切换的复制 iSCSI LUN。
  2. 单击切换。
  3. 如果有多个 iSCSI LUN 的复制任务，请选择新的目标服务器。
  4. 单击切换可开始操作。

若要故障移转 iSCSI LUN 的复制：

  1. 选择您要故障移转的复制 iSCSI LUN。
  2. 单击故障移转 > 故障移转。
  3. 选择要还原的快照版本。
  4. 单击故障移转可开始操作。

注：

  • 您只能在源服务器发生故障时启动故障移转。

若要重新保护 iSCSI LUN 的复制：

当复制任务被故障移转到目标服务器后，复制任务将停止在两个服务器之间的同步数据。您需要在原始源服务器还原后重新保护复制任务。

  1. 选择您要重新保护的故障移转 iSCSI LUN。
  2. 单击故障移转 > 重新保护。
  3. 选择新源服务器。
  4. 如果有多个 iSCSI LUN 的复制任务，请选择新的目标服务器。
  5. 单击重新保护可开始操作。

注：

  • 在新目标服务器上已有的复制 iSCSI LUN 数据将被新源服务器上的数据覆盖。
  • 如果所选的故障移转快照版本未在伙伴服务器上，可能需要较长的时间来重新保护。

若要为复制的 iSCSI LUN 执行测试故障移转：

  1. 选择您要测试的复制 iSCSI LUN。
  2. 单击测试故障移转 > 测试故障移转。
  3. 如果有多个 iSCSI LUN 的复制任务，请选择测试服务器。
  4. 输入 iSCSI LUN 的测试故障移转名称，并选择要测试的快照版本。
  5. 单击测试故障移转可开始操作。

注：

  • 执行复制的切换/故障移转将清除其测试故障移转。
  • 您无法为测试故障移转 iSCSI LUN 创建复制任务。

若要为复制的 iSCSI LUN 清除测试故障移转：

  1. 选择您要清除其测试故障移转的复制 iSCSI LUN。
  2. 单击测试故障移转 > 清除测试故障移转。
  3. 勾选您要清除的测试故障移转。
    注：如果您不再要保存为测试故障移转创建的 iSCSI LUN，勾选删除测试故障移转数据。
  4. 单击清除测试故障移转可开始操作。

远程连接

在 File Station 中，您可通过各种不同的协议 (FTP、STFP、WebDAV 和 WebDAV HTTPS) 连接到公共云服务 (Box、Dropbox、Google Drive 和 Microsoft OneDrive) 和文件服务器。

若要连接至云服务：

  1. 进入工具 > 远程连接 > 连接设置。
  2. 单击该图标并单击下一步以选择所需的云服务。
  3. 在弹出窗口中，按向导的说明进行帐户认证。
  4. 设置完成后，目标云服务将显示在左侧面板中。

云特定限制：

  • Box：
    • 与其它合作者共享的类别将以拥有者的用户名添加后缀。
    • 如果与其它合作者共享的类别使用相同的名称 (以拥有者的用户名为后缀的类别名称)，此类别将只显示一个。
    • 您只有通过 Box 才能打开这些扩展名的文件。.gdoc、.gsheet 和 .boxnote。
    • 文件名有大小写之分。
    • 要了解有关正确的防火墙设置，请参见本文章。
  • Dropbox：
    • 如果 Dropbox 类别含有超过 25000 个文件，则该类别无法显示。
    • 如果 Dropbox 类别含有超过 10000 个文件，则该类别无法重命名。
    • 超过 10 GB 的文件无法上传。
    • 如果目标文件上传超过 24 小时，则上传任务将失败。
    • 文件名有大小写之分。
    • 要了解有关正确的防火墙设置，请参见本文章。
  • Google 云端硬盘：
    • 路径超过 2048 个字符的文件/文件夹将不显示。
    • 文件名超过 255 个字符的文件将不显示。
    • 相同名称的文件/类别将被标记相同的创建时间。
    • 如果您通过 File Station 打开 Google 特定文件 (如 Google 文档和 Google 幻灯片)，将显示一个 URL 以让您访问目标文件。
    • 在 File Station 显示的我的云端硬盘和已加星标文件夹分别与 Google 云端硬盘中“My Drive”和“Starred”文件夹相对应。
    • 要了解有关正确的防火墙设置，请参见本文章。
  • Microsoft OneDrive：
    • OneNote 记事本文件将不会显示。
    • 超过 10 GB 的文件无法上传。
    • 文件 (例如，.png、.jpg、.docx、.pdf) 可能显示较大的文件大小 (原始大小加缩略图大小)。这不会影响其被下载时的原始文件大小。
    • 文件名有大小写之分。

要了解更多限制，我们建议您访问官方网站以获取有关云服务的信息。

若要连接至文件服务器：

  1. 进入工具 > 远程连接 > 连接设置。
  2. 单击该图标并单击下一步以选择所需的服务器协议。
  3. 提供以下信息以完成设置：
    • 主机名或 IP：输入目标文件服务器的主机名或 IP。
    • 端口：输入目标文件服务器的端口号。
    • 用户和密码：如果目标文件服务器需要帐户/密码，请输入此登录信息。
    • 代码页：选择目标文件服务器的字符编码以正确显示文件名。
    • 配置文件名称：输入独特的名称以轻松识别该文件服务器。
    • 使用 WebDAV HTTPS 连接：本选项仅在您设置 WebDAV 协议以及当客户端和文件服务器支持 WebDAV HTTPS 时可用。
    • 一个用户只允许一个连接：此选项仅在您设置 FTP 或 SFTP 协议时可用。当服务器有连接数量限制时，勾选此选项可优化连接。
  4. 单击应用以保存该设置。左侧面板中将显示目标文件服务器。

若要修改现有的云服务或文件服务器配置文件：

  1. 进入工具 > 远程连接 > 连接列表。
  2. 单击目标云服务或文件服务器以进行以下管理：
    • 编辑：单击编辑，更改当前配置文件的配置，然后单击确定。
    • 断开连接：单击断开连接可终止连接。
    • 重新连接：单击连接以恢复继续。(或者，单击 File Station 左侧面板中的目标以重新连接。)
    • 删除：单击删除，然后确认放弃该配置文件。(或者，单击 File Station 左侧面板中的目标并单击删除以进行删除。)

注：

  • 每个用户可同时连接不超过 100 个云服务和文件服务器。
  • 请确认您选择了正确的服务器代码页以避免意外错误。如果您不知道服务器代码面，建议选择 Unicode。
  • 某些 File Station 功能 (例如，压缩、解压缩、预览等) 无法应用于存储在云服务或文件服务器中的文件。
  • 各连接的云服务或文件服务器传输速度取决于用户速度限制设置。
  • 各用户可在所有云服务和文件服务器之间最多同时执行三个文件传输任务。
  • 若要配置 WebDAV、WebDAV HTTPS 和云服务代理服务器设置，请进入控制面板 > 网络 > 常规 > 代理服务器。如果您在连接到云服务或文件服务器的过程中已更改代理服务器设置，请断开连接然后重新连接。
  • 修改所连接云服务或文件服务器的信息不会影响到连接状态。
  • 若要发布连接资源，如果所含的文件超过 30 分钟未接触，则云服务/文件服务器将自动断开连接。单击云服务/文件服务器以恢复连接。
  • 系统不显示采用以下命名格式的文件/文件夹：
    • 以 ._ 或 .SYNOPPSDB 开头
    • 包括 /
    • 确切命名为 .    ..    '    "    .DS_Store    @eaDir    @sharebin    @tmp    .SynologyWorkingDirectory
  • 只有在高权限模式下才支持远程连接。若要使用此功能，请进入主 DSM 的 Docker 套件，选择您要为其启用加密共享文件夹的 Docker DSM，单击编辑， 然后勾选使用高权限执行容器旁的复选框。

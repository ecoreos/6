上传文件或文件夹

使用下列任何方法将文件或文件夹从计算机上传至 DiskStation。

若要全面访问下文显示的上传功能，请先安装最新的 Java Runtime Environment (JRE) 和 Flash Player。

方法 1：

  1. 在左侧面板的上半部中，选择 DiskStation 上的一个目的地文件夹。
  2. 单击上传，选择上传 - 跳过或上传 - 覆盖，然后从本地计算机中选择文件或文件夹。

方法 2：

  1. 进入设置，勾选通过 File Station 浏览本地计算机中的文件，然后刷新网页以在左侧面板的下半部分查看本地计算机中的文件夹。

    2. 注：

    通过 File Station 浏览本地计算机中的文件这一选项只有在您的计算机中安装并启用 JRE 才有效。

  2. 在左侧面板的下半部中，请选择本地计算机上的一个文件夹。主要部分现在会显示所选文件夹的内容。
  3. 在主要区块中选择您要的文件或文件夹，然后单击上传。
  4. 在显示的窗口中，选择 DiskStation 上的一个目的地文件夹。
  5. 在该窗口的右上角，如果您想要覆盖目的地文件夹中名称相同的文件或文件夹，请选择覆盖。否则，选择跳过。
  6. 单击确定可开始上传。

方法 3：

  1. 进入设置，勾选通过 File Station 浏览本地计算机中的文件，然后刷新网页以在左侧面板的下半部分查看本地计算机中的文件夹。
  2. 在左侧面板上，请将文件夹从您的计算机（下半部）拖放至 DiskStation（上半部）上的任何文件夹。
  3. 在显示的弹出式菜单中，选择覆盖或跳过目的地文件夹中名称相同的文件或文件夹。

方法 4：

  1. 在左侧面板的上半部中，选择 DiskStation 上的一个目的地文件夹。
  2. 选择计算机桌面或资源管理器（例如，Windows 资源管理器或 Mac Finder）中的文件，然后将它们拖放到网页浏览器的 File Station 中。
  3. 在显示的弹出式菜单中，选择覆盖或跳过目的地文件夹中名称相同的文件或文件夹。

    4. 注：

    • 方法 4 仅特定浏览器支持此功能：Chrome、Firefox、Safari 5.1 或以上。但对于 Mac 用户，当通过 File Station 浏览本地计算机中的文件被勾选时，不支持此功能。
    • 您还可以对文件夹或空文件进行拖放操作, 但上列浏览器只有在勾选通过 File Station 浏览本地计算机中的文件时才有此功能。否则，您无法采用方法 4 上传文件夹或空文件。
    • 如果您使用 Linux 版 Chrome 并通过 HTTPS 通讯协议登录至 DSM，则需要导入信任的证书（在控制面板 > 安全性 > 证书）才能用方法 4 进行上传。

注：

  1. 您最多可向上传队列添加 1000 个文件。
  2. 如果您使用 Google Chrome、Firefox、Safari 5.1 或以上版本，或者通过 File Station 浏览本地计算机中的文件被勾选，则您可以上传大小超过 4GB 的文件。否则，您一次最多仅可上传 2GB 的文件。
  3. 在非加密共享文件夹中，文件/文件夹的名称必须在 255 个字符以内（非拉丁语系至多 80 个字符）；在加密共享文件夹中，文件/文件夹的名称必须在 143 个字符以内（非拉丁语系至多 47 个字符）。
  4. 文件/文件夹的名称不可包含下列字符：\ / : * ? " > |.
  5. 文件/文件夹的名称不能以âœ._â开头。
  6. 安装 JRE 之后，您可以上传 0KB 的文件。

疑难解答

如果您无法用 Safari 正确浏览 Mac 机上存储的本地项目，请进行以下操作：

  1. 在 Mac 机上，打开 Safari 并选择 Safari 菜单中的偏好设置。
  2. 单击安全性选项卡中的管理网站设置。
  3. 在 Java 下，选择 DSM “网站”。
  4. 选择以不安全模式运行。

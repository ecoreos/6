创建文件请求

文件请求是发送给非 DSM 用户的文件上传邀请。通过文件请求链接，非 DSM 用户可将文件直接上传到 DiskStation 上的 File Station，而无需拥有相应的用户帐户和文件上传权限。

若要创建文件请求链接：

  1. 进入 File Station 并选择一个文件夹作为上传目的地。
  2. 单击操作或右键单击目的地文件夹，并选择创建文件请求。
  3. 在弹出窗口中，您可找到/修改以下信息：
    • 文件路径：列示目标目录路径。
    • 链接：列示文件请求链接以与目标用户共享。
    • 您的名称：输入名称以表示请求者身份。默认值为您的用户名。
    • 信息：为目标用户自定义信息。
    • 启用密码保护：选择此选项以使只有拥有密码的用户才能通过创建的链接上传文件。在以下框中指定密码。
    • 有效期：单击此按钮可自定义此文件请求链接的持续时间：
      • 设置停止时间：选择此选项可确认链接停止文件上传的日期和时间。
      • 设置开始时间：选择此选项可确认链接停止文件启动的日期和时间。
      • 允许访问的数量：选择此选项可确定链接可被访问的次数。
  4. 单击保存完成设置。

若要共享文件请求链接：

若要找到文件请求 URL，请进入 File Station > 工具 > 共享链接管理器，然后单击所需的链接。您可通过以下方式之一共享该链接：

  • 只需将 URL 复制到目标用户即可。
  • 通过 默认电子邮件 (您的 DSM 电子邮件帐户) 或通过 Mail (第三方电子邮件应用程序) 发送链接。对于首次使用的用户，请按以下向导说明设置要使用的电子邮件帐户。

若要上传文件：

如果要求您通过链接上传文件，请执行以下操作：

  1. 打开在本地计算机或移动设备 (Android or iOS) 上获得的 URL。如需求，输入密码。
  2. 在上传页面中，输入表示您身份的名称。
  3. 只需拖放文件即可上传，或单击添加文件以选择文件。
  4. 单击上传完成请求。

注：

  • 无法通过文件请求链接上传文件夹。
  • 在“web”共享文件夹中的文件夹无法被指定为目的地文件夹。
  • 上传功能根据网页浏览器而有所不同：
    • Internet Explorer 8 和 Internet Explorer 9 不支持通过拖放操作进行文件上传。
    • iOS 设备上的 Chrome 仅图像上传。
    • iOS 设备上的 Safari 仅图像和视频上传。
    • Android 设备上的网页浏览器不受任何文件类型的限制。

若要找到上传的文件：

同一用户上传的文件都将出现的目的地的同一文件夹中。该文件夹名称默认为 Uploader_Name (Requestor_Name)。

例如，如果文件被 Watson 上传到 Holmes，您可在名为 Watson (Holmes) 的文件夹中找到文件。

管理文件请求链接

若要修改文件请求链接：

创建文件请求链接后，您可在 File Station > 工具 > 共享链接管理器中修改其设置。

  1. 双击所需链接，或选择链接并单击编辑。
  2. 在弹出窗口中，修改相关设置。
  3. 单击保存来应用新设置。

若要删除文件请求链接：

  1. 请进入 File Station > 工具 > 共享链接管理器。
  2. 选择所需的链接，然后单击删除。相应的文件请求 URL 将立即停止工作。

若要指定请求归档权限：

作为 admin 或属于 administrators 组的用户，您可指定用户的权限，以使这些用户可创建文件请求链接。

  1. 请进入 File Station > 设置 > 共享链接 > 文件请求链接。
  2. 选择用户类型以给予这些用户请求归档权限。
  3. 单击确定来保存设置。

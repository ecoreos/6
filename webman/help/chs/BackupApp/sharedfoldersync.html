共享文件夹同步

“共享文件夹同步”服务可让用户将共享文件夹从一个 Synology 产品（来源）同步至局域网或 Internet 上的另一个 Synology 产品（目的地）上。本功能是以共享文件夹为工作基础。同步时，如果已在目的地 Synology 产品存在相同名称的文件夹 (如 SharedFolder)，则该目的地中的文件夹将被重命名为“SharedFolder_1”。若尚未有同名文件夹，共享文件夹同步功能会于相应的卷或第一个可用的卷内添加该文件夹。该文件夹的访问权限将会按照 DiskStation 预设，即未指定任何访问权限。

第一次进行同步时，共享文件夹同步功能会将所选的共享文件夹进行完整的同步。之后则会仅同步文件修改过的部分。在来源端删除的文件，在目标端也会被删除。

注：

  • 共享文件夹同步是一种同步解决方案，意指源文件将同步到目的地，但不能反方向操作。如果您要寻找双向的同步解决方案，请使用 Cloud Station。

启用此服务

若要启用共享文件夹同步：

您必须在目的地 DiskStation 上启用此服务。若要进行此操作，请进入备份和复制 > 共享文件夹同步。在服务器选项卡下，勾选启用共享文件夹同步服务可启用此功能。

若要将共享文件夹从来源连接至目的地 Synology 产品：

您可从来源端将共享文件夹同步到多个不同的目标端服务器。此外，从 DSM 4.1 开始，“共享文件夹同步”提供了“多源共享文件夹同步”功能，该功能可让多个客户端将其共享文件夹同步到一个 DiskStation。如果不同的客户端同时将同一个共享文件夹同步到一个 DiskStation，将发生冲突，您需在另一个客户端能够将此文件夹同步到目标位置之前取消链接共享文件夹。如果您要取消链接共享文件夹，请进入共享文件夹同步 > 服务器，单击连接列表，选择连接，并单击目标侧的取消链接。

现在您就可以开始将共享文件夹从来源同步至目的地 DiskStation。请按照以下步骤继续进行。

管理同步任务

通过共享文件夹同步向导，您可轻松创建同步任务。

若要创建同步任务：

  1. 进入备份和复制 > 共享文件夹同步 > 客户端。
  2. 单击创建。
  3. 输入任务的名称。
  4. 选择您要同步至目的地的共享文件夹。勾选下方的复选框，并单击下一步以继续。
  5. 设置目的地及下列同步选项：
    • 启用传输加密：传输过程中对数据加密。提供较高的安全性，但非加密传输的性能较佳。您可按需要选择。
    • 启用传输压缩：传输过程中对数据压缩。可节省网络带宽，但会增加 CPU 负担。
    • 启用段落分块同步：仅传输不同的数据而非整个文件。可节省网络带宽，但会增加 CPU 负担。
  6. 设置任务的计划管理：
    • 每 [数字] 小时运行同步或每 [数字] 分钟运行同步：每隔特定分钟或小时间隔运行同步任务。时间由午夜开始计算，最大间隔不可超过 24 小时。
    • 每天的（小时：分钟）自动运行同步：每天的特定时间运行同步任务。
    • 手动运行同步：仅手动运行同步任务。
    • 数据异动时自动进行同步：一旦来源共享文件夹内有任何变化，立即运行同步任务。
  7. 单击应用。现在您可以在任务列表上看到同步任务，系统会按所指定的计划管理运行任务。

若要管理同步任务：

  1. 请在任务列表上选择该任务。
  2. 执行下列任何操作：
    • 单击编辑可编辑任务。
    • 单击删除可删除任务。
    • 如果该同步任务尚未进行，单击立即同步可立即运行任务。
    • 如果该同步任务正在进行，单击取消可停止正在进行的任务。

      • 注：

      如果同步任务的计划设置为数据异动时自动进行同步，单击取消会停止进行中的同步任务。然而，如果同步任务监控的任何文件夹内容有发生异动，[共享文件夹同步] 会继续进行同步任务。

    • 单击全部同步以同步所有数据。“共享文件夹同步”第一次运行同步任务时会运行完整同步，并在第一次同步完成之后仅同步有修改的部分。单击此按钮可辅助您以手动方式再度同步所有的数据。

更多信息

  1. 任务名称必须设置为 1 到 32 个 Unicode 字符，但不可为下列字符：
    ! " # $ % & ' ( ) * + , / : ; <= > ? @ [ ] \ ^ ` { } | ~
  2. 如果您要将 ext4 文件系统的共享文件夹同步至 ext3 的文件系统，同步任务的文件夹数量若超过 32,000 个时可能会发生错误。
  3. 若目的地 DiskStation 上的共享文件夹曾与其他 DiskStation 进行同步，“共享文件夹同步”在进行任务时会做完整的同步。
  4. 要运行共享文件夹同步任务，您必须拥有目的地 DiskStation 的管理权限。您必须拥有 admin 帐户的密码，或为管理群组内的一员。
  5. 当您通过 Internet 同步到 DiskStation 时，在服务器名称栏您可输入其外部 IP 地址或主机名来链接。然而，如果您选择以主机名来链接，此主机名必须符合 FQDN（Fully Qualified Domain Name）格式，例如 www.synology.com。
  6. “共享文件夹同步”在端口转送时，加密传输使用端口 22，非加密传输使用端口 873。
  7. “共享文件夹同步”不支持外接存储设备。
  8. 目标端的加密共享文件夹需要手动使用加密密钥才能装载。
  9. 如果对所选共享文件夹没有更改，则“计划的共享文件夹同步”将不会在特定时间运行。
  10. 为获得最佳性能，我们建议每个同步任务包含的共享文件夹不超过 200 个。
  11. 如果 homes 共享文件夹从源 DiskStation 同步到目的地 DiskStation，则用户主目录功能在目的地 DiskStation 将被禁用。
  12. 对于带非 Intel CPU 的型号，您可使用共享文件夹同步最多 100,000 个共享文件夹（包括子文件夹）。
  13. “共享文件夹同步”在目的地的文件系统为 ext4 时，可以备份 Windows ACL 权限设置。
  14. 如果您要同步到与 DiskStation 共享相同名称的目标服务器，共享文件夹同步将失败。
  15. 来源中的高级共享文件夹权限将被同步到目的地。此外，在目的地将创建管理员组用户的只读规则。

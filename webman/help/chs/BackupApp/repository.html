备份目的地

在备份和复制的备份目的地选项卡下，您可在创建备份任务之前指定本地、网络或云备份目的地。

若要创建本地备份目的地：

如果您要从 DiskStation 备份数据到本地共享文件夹、卷或外接 USB/eSATA 硬盘，则需要创建本地备份目的地。备份目的地必须与数据来源不同，且备份目的地存储容量必须大于所备份数据的大小。

  1. 单击创建，选择本地备份目的地，然后单击下一步。
  2. 为此备份目的地命名，选择用来存储备份数据的目的地以及是否仅要保留最新备份的副本或保留所有备份版本。

注：

  • 若备份目的地的数据在任务进行时被删除，则下一次执行备份任务时将会执行完整备份。
  • 新外接硬盘在作为备份目的地使用前可能需要先进行格式化。
  • 创建备份任务之前，请先确认您已将外接设备正确连接到 DiskStation。您可进入控制面板 > 外接设备来检查 USB/eSATA 设备的状态。如果外接硬盘正在初始化、正在格式化或出现故障，您无法将其选为备份目的地。
  • 您无法将远程装载的共享文件夹法选作为备份来源或目的地。
  • 如果加密共享文件夹已卸载，您无法将其选为备份目的地。
  • 当备份目的地的文件系统为 NTFS/FAT 时，您无法备份图像缩略图。

若要创建网络备份目的地：

如果您要通过网络从 DiskStation 备份数据到另一台 DiskStation 或 rsync 兼容服务器中，则需要创建网络备份目的地。创建网络备份任务时，您有多种选项可使用，如传输加密、压缩、段落分块备份等功能。请注意，部分选项（如传输加密）可能会降低备份任务的速度。

  1. 在创建网络备份目的地之前，请先登录目的地服务器，进入备份和复制 > 备份服务并启用网络备份服务和/或网络备份卷目的地。
  2. 返回 DiskStation，进入备份和复制 > 备份目的地。单击创建，选择网络备份目的地，然后按向导完成。
  3. 通过网络备份目的地（DiskStation 服务器），您可将备份数据保到共享文件夹来保留最新的备份副本，或将备份数据存储到卷中来保留多个备份版本。通过网络备份卷目的地(rsync 兼容服务器)，您仅可保留一份最新的备份副本。

注：

  • 创建任务来备份数据到 rsync 兼容服务器时，有以下模式可用：当您从备份模块旁的下拉菜单中选择一个备份模块（rsync 兼容服务器上的共享文件夹），系统会使用 rsync daemon 模式。仅当您在备份模块中输入绝对路径（以“/”开头）时，系统才会启用 remote shell 模式。
  • 在 Linux 平台上，仅 admin、root 和 rsync 帐户可执行网络备份。
  • 如果您要通过 Internet 使用主机名称访问目的地服务器，主机名称必须符合 FQDN（完全合格域名），例如 abc.bcd.com。
  • 当从一台 DiskStation 设备备份到另一台 DiskStation 设备时，系统将提示您输入一组目的地服务器上的用户名和密码。特别注意，如果您要备份到另一台 DiskStation 的卷时，您需要指定一个属于 administrators 群组的帐户。备份到另一台 DiskStation 上的共享文件夹时，您可根据文件夹访问权限设置来备份共享文件夹。仅拥有访问权限的特定用户帐户才能看到该共享文件夹。如果您无法目的地服务器中找到想要的共享文件夹，请检查您的访问权限。
  • 若要使用加密备份，目的地服务器必须支持加密网络备份功能且用户认证的密码不得为空。
  • Remote shell 模式仅支持加密备份。
  • 备份到 rsync 兼容服务器时，您将无法在创建 rsync 备份任务后更改 rsync 加密模式。若要更改配置，请创建新的 rsync 备份任务。
  • 如果您通过 rsync 客户端将文件备份至 DiskStation，该文件名必须为 UTF-8 编码。
  • 如果您使用 rsync 兼容客户端备份至 DiskStation，则只有该 DiskStation 的 admin、root 和 rsync 帐户用户可执行备份。
  • 如果您使用 DiskStation 备份至 rsync 兼容服务器时，您需要输入正确的 rsync 备份帐户名称和密码。您同时需要选择具有写入权限的备份模块。

若要创建 Amazon S3 备份目的地：

如果您要将数据从 DiskStation 备份到 Amazon S3 服务器，需要创建 Amazon S3 备份目的地。

  1. 单击创建，选择公共云备份目的地，单击 Amazon S3 备份目的地，然后单击下一步。
  2. 为此备份的目的地命名，输入登录凭据并填入所有必填栏来创建备份任务。

注：

  • 如果您创建以 Amazon S3 服务器为备份目的地的备份任何时，您有多种选项可使用，如传输加密、备份至 Amazon S3 RRS（低冗余存储）等。请注意，部分选项（如传输加密）可能会降低备份任务的速度。
  • 已备份的文件会存储在 Amazon S3 服务器上所选的 Bucket，而 Set KEY 值的开头为您所选的目录名称，结尾为该文件的完整路径。
  • Amazon S3 的数据结构包括一对 Bucket 和 KEY，并不含文件夹结构。目前不支持备份空的文件夹。
  • 如果备份的文件没有修改，则不会再备份到 Amazon S3 服务器上同一个 Bucket 中。
  • Amazon S3 上不可存在相同名称的 Bucket。此外，名称应：
    • 介于 3 至 63 个字符。
    • 只能包含小写字母、数字、句点（.）和短划线（-）。
    • 开头和结尾必须为数字或字母，结尾不能为短划线（-）、不能含有两个相邻的句点或句点旁边有短划线（如：“my-.bucket.com”和“my.-bucket”均无效）。
  • Amazon S3 KEY 值的长度上限为 1024 个字符。若要确保成功备份到 Amazon S3，路径长度上限应为 1024 减去创建任务时指定的目的地文件夹名称长度。
  • 如果您启用 Amazon S3 RRS（低冗余存储）支持，仅有启用选项后所修改的数据才会备份至 RRS 存储器，这意味着已备份到 Amazon S3 标准存储器的数据将不会再次备份到 RRS。
  • 不支持将已备份文件从 Amazon S3 移动至 Amazon Glacier。如果通过 AWS 管理控制台来将 Amazon S3 的文件移动至 Amazon Glacier，则相关备份任务可能会失败。
  • 不支持通过 SSL 备份文件至虚拟主机型的 bucket。
  • 传输文件时，该文件将被分割后传输。各部分的大小限制可在任务设置中的分块上传拆分大小中设置，默认值为 512MB。如果网络环境不稳定，建议您将该选项设置为较小数字。
  • 每次上传部分的最大数为 10000。一个文件大小的上限为 Multipart Upload 文件拆分大小（MB）乘以 10000。
  • 文件名称不可含有任何非 UTF-8 或控制字符。

若要创建 Microsoft Azure 备份目的地：

如果要将数据从 DiskStation 备份数据到 Microsoft Azure blob 服务器，您需要创建 Microsoft Azure 备份目的地。

  1. 单击创建，选择公共云备份目的地，单击 Microsoft Azure 备份目的地，然后单击下一步。
  2. 为您的备份的目的地命名，输入登录凭据并填入所有必填栏来创建备份任务。

注：

  • 创建以 Microsoft Azure 服务器为网络备份目的地的备份任何时，您有多个选项可用(如传输加密)。请注意，部分选项（如传输加密）可能会降低备份任务的速度。
  • 已备份的文件会存储在 Microsoft Azure 服务器上所选的 Container 中，而设置 Blob 名称的开头为您所选的目录名称，结尾为该文件的完整路径。
  • Microsoft Azure 的数据结构包括一对 Container 和 Blob，并不含文件夹结构。目前不支持备份空的文件夹。
  • 如果备份的文件没有修改，则不会再备份到 Microsoft Azure 服务器上同一个 Container 中。
  • Microsoft Azure 上不可存在相同名称的 Container。此外，名称应：
    • 介于 3 至 63 个字符。
    • 只能包含小写字母、数字和短划线（-）。
    • 开头和结尾为数字或字母，不能含有两个相邻的短划线。
  • Microsoft Azure Blob 名称的长度上限为 1024 个字符。若要确保成功备份到 Microsoft Azure，路径长度上限应为 1024 减去创建任务时指定的目的地文件夹名称长度。
  • 文件名称不可含有任何非 UTF-8 或控制字符。
  • 不正确的验证信息、网络连接问题或 DiskStation 和远程服务器的时间不相符可能导致出现验证失败警告信息。要检查 DiskStation 上的时间设置，请进入控制面板 > 区域选项 > 时间以同步与 NTP 服务器的时间。

若要创建 hicloud S3 备份目的地：

  2. 为此备份的目的地命名，输入登录凭据并填入所有必填栏来创建备份任务。

注：

  • 创建以 hicloud S3 服务器为网络备份目的地的备份任何时，您有多个选项可用(如传输加密)。请注意，部分选项（如传输加密）可能会降低备份任务的速度。
  • 已备份的文件会存储在 hicloud S3 服务器上所选的 Bucket，而 Set KEY 值的开头为您所选的目录名称，结尾为该文件的完整路径。
  • hicloud S3 的数据结构包括一对 Bucket 和 KEY，并不含文件夹结构。目前不支持备份空的文件夹。
  • 如果备份的文件没有修改，则不会再备份到 hicloud S3 服务器上同一个 Bucket 中。
  • hicloud S3 上不可存在相同名称的 Bucket。此外，名称应：
    • 介于 3 至 63 个字符。
    • 只能包含小写字母、数字、句点（.）和短划线（-）。
    • 开头和结尾必须为数字或字母，结尾不能为短划线（-）、不能含有两个相邻的句点或句点旁边有短划线（如：“my-.bucket.com”和“my.-bucket”均无效）。
  • hicloud S3 KEY 值的长度上限为 255 个字符。若要确保 hicloud S3 备份任务能顺利运行，路径长度上限应为 255 减去创建任务时指定的目的地文件夹名称长度。
  • 传输文件时，该文件将被分割后传输。各部分的大小限制可在任务设置中的分块上传拆分大小中设置，默认值为 512MB。如果网络环境不稳定，建议您将该选项设置为较小数字。
  • Hicloud 每次传输只支持最多 1000 个分块。这意味着文件总大小不得超过分块大小乘以 1000。例如，如果分块上传拆分大小设置为 8MB，则超过 8000MB 的单个文件就无法上传。
  • 不支持加密共享文件夹备份。
  • 文件名称不可含有任何非 UTF-8 或控制字符。

  • 文件名称不可含有任何非 UTF-8 或控制字符。

若要管理备份目的地：

选择已有的备份目的地，然后单击下列按钮：

  • 编辑：修改所选备份目的地的设置。
  • 删除：删除所选备份目的地。

备份到共享文件夹

此类的备份目的地只会保留一份最新的备份副本。按照默认，完成初始备份后，下一次执行任务时，已在来源位置中删除的文件也将从目的地中删除。但如果您要在来源位置删除文件后保留目的地的文件，请在创建备份任务时启用在目的地保留备份的文件。

备份到卷

此类备份目的地必须是 DiskStation 设备中的本地或网络卷 (不包括外接存储设备)。它功能会归档多个备份版本，因此您可以轻松从任何时间点还原文件夹。

若要删除备份任务的已保存数据：

  1. 进入备份目的地选项卡，并扩充包含您要删除任务的备份目的地。
  2. 单击所选备份任务上的删除图标，然后单击对话框中的删除。

若要删除备份版本的已保存数据：

  1. 进入备份目的地选项卡，并扩充包含您要删除任务版本的备份目的地。
  2. 单击所选备份任务上的备份版本信息图标。
  3. 选择您要删除的备份版本，并单击删除图标。

若要锁定备份版本以防止其在备份循环过程中被删除：

  1. 进入备份目的地选项卡，并扩充包含您要锁定任务版本的备份目的地。
  2. 单击所选备份任务上的备份版本信息图标。
  3. 选择您要锁定的备份版本，并单击锁定图标。

若要管理备份目的地的权限：

  1. 选择与本地卷相关联的备份目的地，单击编辑。
  2. 单击允许列表，然后单击添加来将用户或群组添加到列表。

若要设置所有者的备用目的地：

  1. 选择与本地卷相关联的备份目的地，单击所选备份任务上的属性图标。
  2. 在备份权限下，选择下拉菜单中的所有者。

若要让 DiskStation 的上用户或群组阅读和还原存储的备份任务数据：

  1. 选择与本地卷相关联的备份目的地，单击所选备份任务上的属性图标。
  2. 单击允许列表，然后单击添加来将用户或群组添加到列表。

注：

  • 备份任务的默认所有者是创建任务的用户。
  • 仅备份任务所有者能执行该任务。但 DSM 5.1 之前版本创建的备份任务无此限制，因此属于 administrators 群组的用户能够执行此备份任务。
  • 属于 administrators 群组的任务所有者和用户能删除储存的备份任务数据。
  • 属于 administrators 群组的用户和允许列表中的用户都能还原存储的备份任务数据。
  • 只有属于 administrators 群组的用户能通过访问权限设置本地备份目的地，并以所有者的身份和允许列表在本地备份目的地中备份任务。
  • 存储的备份任务数据会消耗任务所有者的存储配额。
  • 如果一个卷没有备份目的地，则只有属于 administrators 群组的用户能够在该卷上创建第一个备份目的地。
  • 只有属于 administrators 群组的用户可删除存储的整个备份目的地数据。
  • 各备份包含系统配置和文件元数据。
  • 不支持加密共享文件夹备份。
  • 在 DSM 5.1 或以下版本中不支持版本锁定。

修复卷或硬盘组

导致卷出现错误的原因有很多，如系统磁区损坏或硬盘故障。在某些情况下，您可按下列步骤来诊断错误并修复问题。

注：

  • 在此所述的修复功能仅适用于具有数据冗余功能的 RAID 类型（如 RAID 1、RAID 5、RAID 6）。请注意，此功能无法修复损毁的硬盘，也无法保证能够挽救损毁卷上的数据。
  • 对于已损毁卷，建议您联系我们的技术支持来寻求帮助。

系统磁区错误

当系统磁区出现错误时，您会看到一则信息，告知您系统分磁无法访问。

修复系统磁区：

  1. 打开存储管理器。
  2. 进入概述并单击标记为修复的链接。系统应修复系统磁区。

卷处于堪用状态

当硬盘出错时，卷或硬盘组会变为堪用，但数据不会丢失。这只会发生在有数据冗余功能的 RAID 类型中，如 SHR、RAID 1、RAID 5、RAID 6 和 RAID 10。

堪用的卷或硬盘组可修复，方法是以状态良好的硬盘来替换故障硬盘并修复卷或硬盘组。

注：

  • 用于更换的硬盘容量必须大于或等于卷中的最小硬盘容量。
  • 用于更换的硬盘状态必须为已初始化或未初始化。

修复堪用的卷或硬盘组：

  1. 进入存储管理器 > 卷或硬盘组来确认确认状态已更改为堪用。
  2. 如果是，进入 HDD/SSD 选项卡来查看发生故障的硬盘。
  3. 卸下故障硬盘并用新硬盘进行替换。

    4. 注：

    • 如果您的 DiskStation 不支持热插拔功能，卸下和安装任何硬盘之前请务必先关机。
    • 用于更换的硬盘中的已有数据都将擦除。
  4. 安装完替换硬盘后，回到卷或硬盘组选项卡。
  5. 选择堪用的卷或硬盘组。
  6. 单击管理。
  7. 选择修复并单击下一步。
  8. 选择要添加到卷的替换硬盘。然后单击下一步。
  9. 按照设置向导的指示完成操作。

修复卷

若要修复堪用状态的卷/硬盘组/iSCSI LUN，您需要用符合下列条件的硬盘来更换损毁的硬盘:

  • 对于 RAID 1、RAID 5 或 RAID 6 卷，堪用的卷可用等于或大于 RAID 卷中最小硬盘的硬盘来修复。
  • 对于 SHR-1 卷，可用的硬盘必须符合以下条件之一：(1) 其大小必须等于或大于损毁硬盘的大小；(2) 由于不包括 RAID 卷中的最大硬盘，其大小必须等于或大于余下的最大硬盘。
    1. 例 1：SHR-1 由 3TB、2TB、2TB 和 1TB 硬盘组成。如果 3TB 硬盘损坏，可用等于或大于 2TB 的硬盘修复。
    2. 例 2：SHR-1 由 4TB、4TB、2TB 和 2TB 硬盘组成。如果 4TB 硬盘损坏，可用等于或大于 4TB 的硬盘修复。
    3. 例 3：SHR-1 由 4TB、4TB、2TB 和 2TB 硬盘组成。如果 2TB 硬盘损坏，可用等于或大于 2TB 的硬盘修复。
  • 对于 SHR-2 卷，可用的硬盘必须符合以下条件之一：(1) 其大小必须等于或大于损毁硬盘的大小；(2) 由于不包括 RAID 卷中的两个最大硬盘，其大小必须等于或大于余下的最大硬盘。

如果您将硬盘配置为符合上述条件的备援，则它们可用作修复卷/硬盘组/RAID 组/iSCSI LUN。

解决损毁的卷

当卷状态变更为已损毁时，您无法自行修复该卷。若要解决此问题，建议您联系我们的技术支持来寻求帮助。

您联系技术支持前：

  • 通过 File Station、Windows File Service、FTP 或其它文件服务将文件复制到计算机或外接设备可抢救 DiskStation 上的数据。
  • 保持硬盘的排序与卷损毁时相同。
  • 请对损毁硬盘执行 S.M.A.R.T. 检测。若要进行此操作，请进入存储管理器 > HDD/SSD。选择有问题的硬盘，然后单击 S.M.A.R.T.测试。

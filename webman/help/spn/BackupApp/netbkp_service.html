Servicios de copia de seguridad

Las opciones de Copia de seguridad y replicación > Servicios de copia de seguridad permiten a los usuarios realizar copias de seguridad de datos desde sus dispositivos DiskStation o clientes compatibles con rsync a su DiskStation.

Habilitar servicio de copia de seguridad de red

Si desea realizar copias de seguridad de datos de un DiskStation o de un servidor compatible con rsync a otro DiskStation, o permitir que otro dispositivo DiskStation realice tareas de copia de seguridad remotas y almacene los datos de las copias de seguridad en un volumen de su dispositivo DiskStation, deberá habilitar el servicio de copias de seguridad de red en el servidor de destino antes de crear una tarea de copia de seguridad de red. Para hacer esto, vaya a Panel de control > Privilegios y otorgue el privilegio de Destino de la copia de seguridad de red a un usuario. A continuación, podrá crear un usuario de DSM con los privilegios adecuados de aplicaciones y carpetas compartidas en su DiskStation. Una vez finalizada la configuración, los demás podrán utilizar los privilegios de usuario para realizar copias de seguridad de red desde sus DiskStation a las carpetas compartidas de destino de su DiskStation.

Para habilitar el servicio de copia de seguridad de versión única:

  1. Vaya a Servicios de copia de seguridad > Destino de la copia de seguridad de red y a continuación haga clic en Habilitar servicio de copia de seguridad de red.
  2. Haga clic en Aplicar.
  3. Vaya a Panel de control > Carpeta compartida, seleccione una carpeta compartida, haga clic en Editar > Permisos, y luego otorgue el privilegio Lectura/Escritura a un usuario. Repita este paso para incluir más carpetas compartidas que podrían utilizarse como destinos de copia de seguridad.

Para habilitar el servicio de copia de seguridad de versión múltiple:

  1. Vaya a Servicios de copia de seguridad > Destino del volumen de la copia de seguridad de red y marque la casilla junto a Habilitar el destino del volumen de la copia de seguridad de red.
  2. Vaya a Destino de la copia de seguridad y seleccione un destino de copia de seguridad con un volumen local.
  3. Haga clic en Editar > Lista de permiso > Agregar para agregar el usuario o grupo a la lista.

Nota:

  • Si otros usuarios desean realizar una copia de seguridad de red desde un cliente que no sea DiskStation, solo podrán hacerlo con privilegios de admin o rsync en su DiskStation. Si no desea compartir sus credenciales de admin con otras personas, puede ir al Panel de control > Usuario para crear un usuario denominado rsync, modificar sus privilegios y luego compartir sus credenciales con otras personas.
  • Si realiza una copia de seguridad de red desde un DiskStation con una versión de DSM anterior a 3.0 o un cliente que no sea DiskStation, y desea conservar la información sobre el propietario y grupo de los datos de origen, debe añadir la cuenta de copia de seguridad al grupo de administrators y realizar una copia de seguridad de los datos a la carpeta compartida NetBackup.

Habilitar el servicio de copia de seguridad de red utilizando una configuración personalizada de rsync

Puede habilitar el servicio de copia de seguridad de red usando una configuración personalizada de rsync, y configurar los privilegios de usuario y el destino de la copia de seguridad en su DiskStation. Una vez finalizada la configuración, los demás podrán realizar copias de seguridad rsync desde sus dispositivos DiskStation o clientes compatibles con rsync a las carpetas compartidas de destino (o "módulos de copia de seguridad") en su DiskStation.

Para habilitar el servicio de copia de seguridad de red utilizando una configuración personalizada de rsync:

  1. Vaya a Servicios de copia de seguridad > Destino de la copia de seguridad de red.
  2. Marque Habilitar servicio de copia de seguridad de red.
  3. Marque Utilizar configuración personalizada de rsync.
  4. Haga clic en Aplicar.

Para configurar privilegios de usuario y módulos de copia de seguridad en su DiskStation:

  1. Inicie sesión en su dispositivo DiskStation a través de Telnet o SSH.
  2. Especifique el módulo de copia de seguridad y los usuarios autorizados en el archivo de configuración rsync "/etc/rsyncd.conf". Para más información, consulte la página del manual de rsyncd.conf en su cliente de rsync o haga clic aquí.

Nota:

  • Si se utiliza una configuración personalizada de rsync, DiskStation no podrá ser el servidor de destino para Sincronización de carpetas compartidas, Time Backup o Copia de seguridad LUN.

Habilitar el límite de velocidad

Puede habilitar el límite de velocidad para controlar el ancho de banda útil máximo para usuarios y grupos al transferir datos a través de la copia de seguridad de red.

Para habilitar el límite de velocidad:

  1. Vaya a Servicios de copia de seguridad > Destino de la copia de seguridad de red. Marque la casilla con la indicación de Habilitar límite de velocidad o Habilitar el límite de velocidad de acuerdo con un programa si desea imponer un límite de velocidad solo en determinados momentos. Este ajuste se realiza a nivel de servidor y anulará los ajustes de usuarios individuales y de grupos. Si elige Sin límite de velocidad deshabilitará los ajustes del límite de velocidad para todos los usuarios y grupos.
  2. Haga clic en Ajustes del límite de velocidad para modificar la configuración para determinados usuarios o grupos. Para cada usuario y grupo puede elegir una de las siguientes opciones:
    • Aplicar configuración de grupo: los ajustes del límite de velocidad serán los mismos que los del grupo al que pertenece el usuario.
    • Ninguno: no se aplica ningún límite de velocidad cuando miembros del grupo transfieren archivos con el servicio.
    • Constante: se aplicará un límite de velocidad fijo constante para el usuario o los miembros del grupo. Los límites de velocidad de carga y descarga se pueden especificar en los campos a la izquierda.
    • Variable: se pueden especificar dos límites de velocidad diferentes al usuario o los miembros del grupo de acuerdo con un programa. Haga clic en el botón Personalizar para modificar los ajustes del límite de velocidad y definir un programa.
  3. Haga clic en Aplicar.

Nota:

Cuando esté definiendo el límite de velocidad para un usuario, la sección Resultado mostrará el límite de velocidad actual del usuario de acuerdo con el ajuste individual y de grupo. Por ejemplo, si el usuario "Aquiles" tiene un límite individual de descarga de 100 kB/s, pero pertenece a un grupo de usuarios con un límite de descarga de 500 KB/s, entonces la sección de Resultados mostrará "0/500".

Habilitar servicio Time Backup

El servicio Time Backup permite a los usuarios realizar copias de seguridad de varias versiones de carpetas desde otros DiskStation a este DiskStation de forma remota. Para esto es necesario habilitar el servicio de copia de seguridad de red y el servicio Time Backup.

Para habilitar el servicio de copia de seguridad de red y el servicio Time Backup:

Vaya a Servicios de copia de seguridad > Time Backup. Marque la casilla de verificación adecuada.

Nota:

Solo se permite la cuenta admin para crear la conexión de copia de seguridad para Time Backup remoto.

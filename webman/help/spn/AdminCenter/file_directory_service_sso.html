Cliente de SSO

Si su DiskStation se ha unido al mismo dominio o servicio LDAP que el servidor SSO, puede configurar su DiskStation como cliente de SSO. Los usuarios pueden acceder a los servicios proporcionados por su DiskStation una vez inician sesión en el servidor SSO con sus credenciales.

Antes de comenzar, asegúrese de que su DiskStation se ha unido al mismo dominio o servicio LDAP que el servidor SSO.

Para configurar su DiskStation como cliente de SSO:

  1. Acceda a Panel de control > Dominio/LDAP > Cliente de SSO.
  2. Active la casilla de verificación Habilitar servicio SSO.
  3. Especifique la URL del servidor SSO y el ID de aplicación en los siguientes campos debajo de la casilla de verificación.

    Nota:

    Para obtener el ID de aplicación, escoja uno de los siguientes métodos:

    • Método 1:
      1. Inicie sesión en el DSM que actúa como el servidor SSO.
      2. Añada su DiskStation, que hará las veces de aplicación cliente en la lista de aplicaciones de SSO Server. Para obtener información detallada, consulte el artículo de ayuda para SSO Server > Lista de aplicaciones.
      3. Copie y pegue el ID de aplicación asignado por el servidor SSO una vez su DiskStation se haya añadido al servidor SSO.
    • Método 2:
      1. Tras introducir la URL del servidor SSO*, haga clic en Registro rápido junto al campo del ID de la aplicación.
      2. En el cuadro de registro, introduzca las credenciales de una cuenta de administrador del servidor SSO en los campos Administrador del servidor y Contraseña.
      3. Especifique el nombre de la aplicación cliente en el campo Nombre de la aplicación.
      4. Asegúrese de que Redirigir URI hace referencia exactamente a la página de inicio de sesión de su DiskStation.
      5. Haga clic en OK y se añadirá su DiskStation a SSO Server, mientras que el ID de la aplicación se rellenará automáticamente.

    ______

    * Se le pedirá que introduzca la URL idéntica del servidor SSO (nombre de host o IP) al configurar cada cliente SSO.

  4. Haga clic en Aplicar para guardar la configuración.

Nota:

  • Si el servidor SSO utiliza el protocolo HTTPS, puede que los exploradores que incumplen un certificado de confianza bloqueen el inicio de sesión SSO.
  • Al acceder al servidor SSO mediante el protocolo HTTP, si su DiskStation (establecido como cliente de SSO) utiliza conexión HTTPS, los usuarios necesitan ajustar la configuración de sus exploradores para permitir el uso de una conexión no segura.
  • La dirección IP del servidor SSO en formato IPv6 no será compatible si ha marcado la casilla de verificación Mejorar la seguridad con el encabezado HTTP de la Directiva de seguridad de contenido (CSP) en Panel de control > Seguridad > Seguridad en su DSM.

General

Opciones genéricas de red, como pasarela predeterminada, DNS server, proxy y tunelización IPv6 están disponibles en Panel de control > Red > General.

Cambiar el nombre del dispositivo DiskStation

Para cambiar el nombre del dispositivo DiskStation, introduzca el nuevo nombre del servidor y haga clic en Aplicar.

El nombre del servidor distingue entre mayúsculas y minúsculas, y puede tener entre 1 y 15 caracteres, que pueden ser letras, números, caracteres de subrayado y signos menos. El primer carácter del nombre debe ser una letra.

Especificar la pasarela predeterminada

Sencillamente introduzca la dirección IP para la pasarela predeterminada y haga clic en Aplicar para guardar la configuración.

Especificar la dirección IP del DNS Server

Puede optar por configurar manualmente la dirección IP de su DNS Server. En ese caso, se debe introducir la dirección IP de su DNS Server preferido. También se puede introducir una dirección IP de DNS Server que servirá de respaldo si el servidor DNS preferido experimenta problemas. Una vez que haya configurado las direcciones IP del DNS Server como desee, haga clic en Aplicar para guardar su configuración.

Nota:

Los campos DNS Server y Puerta de enlace predeterminada se configurarán automáticamente y no podrán cambiarse si todas sus interfaces se configuran automáticamente o si está utilizando la conexión PPPoE.

Conectar a través de un servidor proxy

Para permitir que su dispositivo DiskStation conecte a través de un servidor proxy, marque Conectar a través de un servidor proxy, introduzca la Dirección y Puerto y a continuación haga clic en Aplicar.

Nota:

  • Haga clic en Configuración avanzada para configurar el proxy HTTP, el proxy HTTPS y la información de autenticación.
  • De manera predeterminada, la configuración del proxy HTTPS es la misma que la del proxy HTTP.
  • Si se marca Conectar a través de un servidor proxy, Actualización de DSM y Centro de paquetes, además de otros paquetes, se conectarán siempre a través de Internet a través de un proxy.
  • Si Omitir servidor proxy para direcciones locales se ha marcado, asegúrese de que el DNS server utilizado por el DiskStation puede resolver consultas de dispositivos en la red local.

Habilitar la tunelización IPv6

Hasta que IPv6 sustituya completamente a IPv4, la tunelización IPv6 es necesaria para permitir que hosts con direcciones IPv6 alcancen los servicios IPv4. Esta técnica proporciona una manera de llevar tráfico IPv6 dentro de redes IPv4 existentes.
Si no está seguro sobre el modo de configurar IPv6 o afronta problemas de conexión, póngase en contacto con su Proveedor de servicios de Internet.

Para habilitar IPv6 Tunnel:

  1. Marque la casilla de verificación Habilitar túnel para habilitar la compatibilidad con el túnel.
  2. En Dirección de servidor, escriba el nombre de host o la dirección de IP del proveedor de servicios de túnel, por ejemplo: anonymous.freenet6.net.
  3. Seleccione si desea Conectar de forma anónima o Conectar con una cuenta existente.

Nota:

Es posible que su dirección IPv6 sea actualizada automáticamente por el proveedor de servicios. No obstante, DiskStation no quitará inmediatamente la antigua dirección IPv6.

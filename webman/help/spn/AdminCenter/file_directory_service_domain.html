Dominio

Su DiskStation puede unirse a un dominio de Windows. Tras unirse, los usuarios del dominio pueden conectarse a DiskStation con su cuenta de dominio y contraseña, que les permiten acceder a los archivos y utilizar aplicaciones de DSM sin necesidad de tener que recordar otro nombre de usuario y contraseña.

Para unirse a un dominio:

  1. Vaya a Panel de control >Servicio de directorio > Dominio.
  2. Marque la casilla Unirse a dominio.
  3. Introduzca el nombre de dominio y especifique el servidor DNS en los campos correspondientes.
  4. Si fuera necesario, marque la casilla Opciones avanzadas de dominio. (Para más información, consulte más abajo).
  5. Haga clic en Aplicar.
  6. En la ventana emergente, introduzca el nombre del usuario y la contraseña del administrador del servidor del dominio.
  7. Haga clic en OK.

DNS Server

Se recomienda que especifique un servidor DNS para ayudar a DiskStation a buscar el dominio. Sin embargo, si el dominio no está registrado en el servidor DNS, es posible que falle el procedimiento de unirse al dominio.

Tipo de servidor de dominio

Este campo muestra el tipo de dominio cuando su DiskStation se une a un dominio. El tipo de dominio será "Dominio AD".

Modo de administración

Esta opción determinará la manera de administrar los privilegios de usuarios y grupos dominio.

  • Dominio de confianza: puede administrar usuarios y grupos en el dominio unido, además de en los dominios de confianza. Los usuarios y grupos se pueden filtrar utilizando la lista desplegable de dominios.
  • Dominio único con UO: en este modo solo se mostrarán los usuarios y grupos en el dominio al que se haya unido. Este modo le permite filtrar usuarios y grupos de dominio con la lista desplegable de la UO (unidad organizativa).

Opciones avanzadas de dominio

En la mayoría de los casos, no es necesario rellenar ninguna de las opciones de dominio avanzadas. Las opciones avanzadas de dominio únicamente se necesitan para determinados entornos de dominio.

  • DC IP/FQDN: especifique una dirección IP de DC (controlador de dominio) o FQDN para que DiskStation intente comunicarse con ella. Si hay más de una dirección IP o FQDN, inserte una coma (,) entre ellas. Si lo desea, añada un asterisco (*) después de la última dirección IP de DC o FQDN y DiskStation intentará comunicarse con el otro DC cuando fallen todos los que haya especificado. Tendrá que añadir una coma entre el asterisco y la última dirección IP o FQDN.
  • Nombre NetBIOS de dominio: especifique el nombre de NetBIOS del dominio.
  • FQDN de dominio (nombre DNS): especifique el FQDN (nombre DNS) del dominio.
  • Registrar interfaz DNS: al unirse a un dominio, la NIC (tarjeta interfaz de red) especificada aquí se registrará en el servidor DNS. Tenga en cuenta que si el nombre de host de DiskStation incluye un guion bajo (_), el registro no se completará porque los guiones bajos (_) no se pueden utilizar en DNS.
  • Actualizar la lista de usuarios/grupos: especifique con qué frecuencia actualiza automáticamente su DiskStation el nombre de usuario/la lista de grupos. Puede personalizar el periodo de tiempo para realizar actualizaciones diaria, semanal o mensualmente. Además, las listas de grupo/usuarios del dominio se pueden actualizar manualmente si va a la pestaña Usuario del dominio y hace clic en Actualizar todos los dominios. Tenga en cuenta que las actualizaciones automáticas afectarán a la hibernación del sistema.

Unirse a un controlador de dominio de solo lectura

Para unir su DiskStation a un controlador de dominio de solo lectura (RODC), deberá especificar el "nombre completo del equipo" (FQDN) del controlador de dominio de lectura escritura (RWDC) en el campo DC IP/FQDN. El nombre completo del equipo se debe introducir con el siguiente formato: Nombredelequipo.Dominio, por ejemplo "ad01.synology.com".

Nota:

  • Si su nombre de usuario del dominio incluye “%” o “$”, es posible que no pueda acceder a su carpeta home. Solicite un nuevo nombre de usuario al administrador de su dominio.
  • Los grupos de dominios Administradores de dominio y Administradores de empresa se añadirán automáticamente al grupo local administrators. Por tanto, los usuarios del dominio en estos grupos tendrán privilegios administrativos en DiskStation, incluido el acceso a servicios de archivos como CIFS/FTP/AFP/WebDAV y otras aplicaciones de DSM.
  • También puede configurar privilegios de acceso para que los usuarios del dominio puedan compartir las carpetas compartidas de DiskStation. Consulte aquí para obtener más información.

Opciones de dominio

Esta sección incluye las siguientes opciones adicionales de dominio.

  • DiskStation sincronizará con un servidor NTP cada vez que un usuario de dominio inicie sesión: habilitar esta opción forzará la sincronización de tiempo entre DiskStation y el servidor NTP.
  • Habilitar el inicio de sesión en los servidores: si el cliente ha habilitado el inicio de sesión de cliente, el inicio de sesión en los servidores también debe habilitarse en su DiskStation para que los usuarios puedan utilizar los servicios del dominio con normalidad.
  • Conseguir listas de usuarios/grupos en modo compatible con NT4: habilitar esta opción permite al sistema obtener las listas de usuarios/grupos utilizando el modo NT4 RPC. Esta opción se puede habilitar si ciertas listas de usuarios/grupos del dominio no se pueden obtener con la configuración predeterminada.
  • Deshabilitar los permisos de los Administradores de dominio y de los Administradores de empresa: un usuario con permisos de administrador puede tener control absoluto de DiskStation y su archivos. Habilitar esta opción le permitirá restringir los permisos de Administradores de dominio y Administradores de empresa, para que sean iguales a los de los usuarios normales, heredando así todos los permisos de archivos/carpetas de los usuarios normales.
  • Enumerar los niveles de grupos anidados: especifique el número de miembros de grupo de dominio anidados que se pueden enumerar.

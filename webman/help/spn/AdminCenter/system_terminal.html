Terminal

Habilitar opciones en Panel de control > Terminal y SNMP > Terminal permite a su DiskStation ser compatible con servicios de interfaz de la línea de comandos Telnet y SSH. También puede cambiar el nivel de seguridad del algoritmo de cifrado SSH.

Para habilitar el servicio Telnet/SSH:

  1. Marque la casilla junto a los protocolos deseados (SSH o Telnet).
  2. Haga clic en Aplicar.

Para cambiar el puerto de servicio SSH:

  1. En Puerto escriba un nuevo número de puerto.
  2. Haga clic en Aplicar.

Para proteger las conexiones SSH se usan multitud de algoritmos de cifrado con distintos niveles de seguridad. Puede decidir utilizar los algoritmos que mejor se adapten a sus necesidades.

Para cambiar el nivel de seguridad del algoritmo de cifrado:

  1. Haga clic en Configuración avanzada.
  2. Seleccione un nivel de seguridad para los algoritmos de cifrado. Si desea indicar exactamente los algoritmos que se van a utilizar, seleccione Personalizar para elegir de entre tres grupos: Cipher, KEX (intercambio de claves) y MAC (código de autenticación de mensajes).
  3. Haga clic en Aplicar.

Algunos modelos son compatibles con la aceleración de hardware para ciertos algoritmos de cifrado. Puede marcar la opción Usar solo codificadores con aceleración de hardware para mejorar las velocidades de transferencia de los servicios relacionados con SSH, incluidos los servicios SFTP y Copia de seguridad de red cifrados. En la página avanzada, los algoritmos capaces de utilizar la aceleración de hardware se muestran en rojo.

Nota:

  • Algunos servicios (por ejemplo, copia de seguridad de red, SFTP) habilitan SSH automáticamente y abren el puerto 22. En este caso, deberá habilitar el servicio SSH en Panel de control > Terminal y SNMP > Terminal Para iniciar sesión en DSM con SSH.
  • SSH/Telnet solo permite iniciar sesión en el sistema con cuentas pertenecientes al grupo administrators. Para cambiar a una cuenta root, inicie sesión en el sistema con SSH/Telnet como usuario perteneciente al grupo administrators, ejecute el comando sudo -i y, a continuación, introduzca la contraseña de la cuenta que ha usado para iniciar sesión.
  • La contraseña no se puede dejar en blanco cuando se inicia sesión en el sistema con SSH/Telnet.
  • Cuando se desactiva el servicio SSH, se interrumpen todas las conexiones SSH, incluida cualquier tarea de copia de seguridad cifrada de red que se esté ejecutando en esos momentos. Sugerimos realizar las tareas de copia de seguridad de red de nuevo más tarde, cuando el servicio SSH se haya deshabilitado.
  • La manipulación o modificación incorrecta de DiskStation puede provocar averías o pérdidas de datos.
  • Si selecciona un nivel de seguridad más alto o solo utiliza codificadores con aceleración de hardware, puede provocar que algunos clientes SSH no puedan conectar debido a problemas de compatibilidad.
  • Según el nivel de seguridad seleccionado, es posible que algunos algoritmos compatibles con la aceleración de hardware no estén disponibles.

Actualización de DSM

De manera periódica, Synology publica actualizaciones gratuitas para DSM. Las actualizaciones pueden incluir características nuevas, mejoras a funciones y mejoras de rendimiento.

Importante: Normalmente, la actualización de DSM no afecta a los datos de DiskStation. Sin embargo, para mayor seguridad de los datos, se recomienda encarecidamente hacer una copia de seguridad de los datos antes de empezar.

Realizar la actualización manual de DSM

Puede descargar el archivo de actualización .pat desde el Centro de descargas de Synology y realizar la actualización manual de DSM.

Para realizar una actualización manual de DSM:

  1. Visite www.synology.com para descargar la actualización de DSM más reciente a su equipo local. Descomprima el archivo para obtener un archivo de actualización .pat.
  2. Haga clic en Actualización DSM manual para acceder a la página Actualización DSM manual
  3. Haga clic en Examinar para seleccionar el archivo de actualización.
  4. Haga clic en Aplicar para comenzar a actualizar DSM. El sistema se reiniciará de forma automática cuando finalice la actualización.

Resolución de problemas:

Si no puede actualizar DSM, las siguientes sugerencias pueden ayudarle a solucionar el problema:

  • Compruebe si el proceso de actualización ha sido interrumpido por los usuarios, o si se han producido otros eventos inesperados, como un corte eléctrico.
  • Compruebe la versión de DSM. Sólo puede actualizar el sistema con una versión más reciente de DSM.
  • Descargue de nuevo la actualización de DSM, pues es posible que el archivo de actualización esté dañado.

Si continúa sin resolver el problema, póngase en contacto con el Soporte técnico de Synology.

Realizar la actualización automática de DSM

Aparte de la actualización manual, cada vez que vaya a Actualización de DSM, el sistema comprobará con Synology para ver si está disponible un nuevo DSM para su descarga y mostrará a continuación el resultado en texto verde o rojo. Si los resultados indican que el DSM más reciente está listo para su descarga, puede hacer que el sistema descargue automáticamente el último DSM y, a continuación, realizar la actualización.

Para que el sistema descargue el DSM más reciente y realice la actualización:

  1. Si el DSM más reciente está listo para descargar, haga clic en el vínculo Descargar para que el sistema descargue el DSM más reciente.
  2. El sistema mostrará cualquiera de las siguientes notificaciones de escritorio para indicarle si la descarga es correcta o no: Descarga DSM completada o Descarga DSM errónea.
  3. Haga clic en la ventana de notificaciones de escritorio para acceder a Actualización de DSM.
  4. Realice cualquiera de las siguientes operaciones:
    • Si la descarga de DSM se ha completado, haga clic en Actualizar ahora para empezar a actualizar DSM de forma inmediata.
    • Si la descarga de DSM ha sido errónea, haga clic en Reintentar para que el sistema vuelva a descargar el DSM más reciente de Synology.

Para comprobar actualizaciones de forma regular:

  1. Marque Buscar actualizaciones de DSM automáticamente.
  2. Seleccione un plan de actualización y configure el Programa de instalación.
  3. Haga clic en OK.

Se mostrará una notificación en el escritorio cuando la actualización de DSM se haya descargado y esté lista para la instalación. Para evitar que DSM se actualice automáticamente, puede cancelar esta opción en las notificaciones de escritorio o cambiando la Configuración de actualizaciones. Las actualizaciones de DSM se instalarán antes de los apagados programados si ha habilitado Apagar en Programa de energía.

Resolución de problemas:

Si ve mensajes de error (en el texto en rojo) durante la comprobación o descarga del DSM más reciente, los siguientes consejos pueden ayudarle a solucionar el problema:

  • Compruebe que su DiskStation esté conectado a Internet.
  • Es posible que el servidor de descargas de Synology esté ocupado en este momento. Vuelva a intentar la descarga más adelante.

Si continúa sin resolver el problema, póngase en contacto con el Soporte técnico de Synology.

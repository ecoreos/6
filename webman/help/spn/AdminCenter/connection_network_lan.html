LAN/Internet

Según el modelo de DiskStation y la configuración de red, puede configurar los parámetros DHCP para LAN o Internet en Panel de control > Red > Interfaz de red. DHCP es un protocolo de red para la configuración automática de IP. Para dejar que DHCP asigne un IP para el sistema, haga clic en Editar y elija Obtener la configuración de la red automáticamente (DHCP) en la pestaña IPv4.

Si utiliza una dirección IP específica de su proveedor de servicios de red, elija Utilizar configuración manual y escriba manualmente la dirección IP y la máscara de subred. Si no están disponibles un DNS Server o una pasarela predeterminada, deje el campo en blanco.

Nota:

  1. Para estar seguro de que las notificaciones por correo electrónico se envían correctamente a través del servidor SMTP, debe especificarse un IP de DNS Server válido.
  2. Los Jumbo Frames son marcos de Ethernet con más de los 1500 bytes convencionales de la Unidad de transmisión máxima (MTU), permitiendo la transmisión por Ethernet de archivos grandes para ser más eficientes. Solamente puede habilitarse en el entorno de red Gigabit. Para asegurarse de que Jumbo Frame funciona correctamente, es necesario que todos los equipos y dispositivos de la red que accedan a DiskStation lo admitan y utilicen el mismo valor de MTU.

VLAN

Cuando el entorno de red requiera la configuración de una VLAN, puede habilitar la opción VLAN e introducir una ID de VLAN para el entorno de red en la pestaña IPv4 en la ventana de Editar.

Nota:

  • Asegúrese de que el interruptor de red está bien configurado, ya que, de lo contrario, es posible que no se pueda acceder a DiskStation.
  • Si no puede conectar con la unidad DiskStation tras modificar la configuración anterior, pulse el botón RESET ubicado en la unidad DiskStation para restaurar la configuración predeterminada.

Configuración de IPv6

Los dispositivos de DiskStation son compatibles con los esquemas de pila dual IPv4/IPv6. Esto significa que tanto IPv4 como IPv6 pueden trabajar simultáneamente en el DiskStation. La configuración predeterminada para IPv6 es Automático. Seleccione una interfaz de red existente, haga clic en el botón Editar y vaya a la pestaña IPv6 para cambiar la configuración.

Para ajustar la configuración de IPv6:

  1. Vaya al Panel de control > Red > Interfaz de red.
  2. Seleccione una interfaz de red existente, haga clic en Editar.
  3. En la pestaña IPv6 seleccione un tipo de configuración desde Configuración de IPv6:
    • Automático: El DiskStation generará automáticamente una dirección de IPv6 según la publicidad del enrutador LAN (RA).
      • Modo sin estado: Permite que dispositivos conectados al enrutador configuren automáticamente una dirección IPv6 sin configurar el DNS Server de IPv6.
      • Modo sin estado DHCPv6: Permite que dispositivos conectados al enrutador configuren automáticamente una dirección IPv6 y obtengan la información DNS a través de DHCPv6.
      • Modo con estado: Permite que dispositivos conectados al enrutador obtener una dirección IPv6 y la información DNS a través de DHCPv6. En este modo tiene que habilitar el DHCPv6 server y configurar la Inicio de dirección IP y la Fin de dirección IP del DHCPv6 server.
    • Manual: Configure IPv6 manualmente introduciendo la información en Dirección IPv6, Longitud de Prefix y Pasarela predeterminada.
    • Desactivado: Desactiva la compatibilidad con IPv6.

Hasta que IPv6 sustituya completamente a IPv4, los mecanismos de transición de IPv6 son necesarios para habilitar los hosts con direcciones IPv4 y alcanzar los servicios IPv6. Elija entre los diferentes mecanismos de transición IPv6:

  • 6in4: Conecte a la red IPv6 con un agente de tunelización. Deberá rellenar Dirección IPv6, Longitud de Prefix, Prefix y Dirección IPv4 de agente de túnel. Esto requiere una dirección IPv4 pública.
  • 6to4: Conecte a la red IPv6 con un agente de tunelización con anycast. Esto requiere una dirección IPv4 pública.
  • DHCP-PD: Obtenga un prefijo IPv6 del DHCP Server mediante la delegación de prefijo dentro de LAN. Esto requiere la compatibilidad con ISP y solo funciona en el modo de enrutador.
  • Tunelización IPv6 (TSP): Vaya a Panel de control > Red > General para configurar esta opción.

802.1X

802.1X es el estándar de autenticación desarrollado por la IEEE para redes de conexión de usuarios. 802.1X puede usarse en redes para autenticar el cliente y denegar el acceso a la red protegida a los clientes que no superen correctamente la autenticación.

Puede permitir a su Synology NAS conectarse a la red protegida mediante autenticación 802.1X.

Para habilitar la autenticación 802.1X:

  1. Vaya al Panel de control > Red > Interfaz de red.
  2. Seleccione una interfaz de red cableada (por ejemplo LAN) y haga clic en Editar.
  3. Vaya a la ficha 802.1X y marque Habilitar autenticación 802.1X.
  4. Seleccione uno de los siguientes protocolos en el menú desplegable Autenticación. Si no está seguro de qué protocolo es el más adecuado, seleccione Auto.
    • EAP protegido (PEAP): usa un certificado CA (opcional) para establecer una conexión segura. Esta conexión segura se usará para la autenticación interna.
    • TLS tunelizado (TTLS): usa un certificado CA (opcional) para establecer una conexión segura. Esta conexión segura se usará para la autenticación interna.
    • TLS: usa TLS (Transport Layer Security) para la autenticación.
  5. Si selecciona TTLS o PEAP, seleccione uno de los protocolos de autenticación interna del menú desplegable.
    • MS CHAP v2: la información de autenticación se cifrará con Microsoft CHAP v2.
    • GTC: la información de autenticación usará GTC (Generic Token Card) para el cifrado.
  6. Escriba el nombre de usuario y la contraseña.
  7. Si fuera necesario, haga clic en Importar y, a continuación, Examinar para buscar e importar un certificado exportado del servidor Radius (por ejemplo ca.crt).
  8. Haga clic en OK para guardar su configuración.

Nota:

  • Para configurar la autenticación 802.1X para redes inalámbricas, vaya a Panel de control > Inalámbrico > Wi-Fi.
  • La configuración de autenticación debe ser la misma que la configurada en el servidor Radius. Para obtener más información, póngase en contacto con el administrador del servidor Radius.

LAN inalámbrica*

Cuando DiskStation actúa como enrutador inalámbrico, puede configurar los siguientes parámetros para LAN inalámbrico en Panel de control > Red > Interfaz de red.

Configuración IPv4

Dirección IP y máscara de subred

Puede configurar la dirección de IP y la máscara de subred para DiskStation aquí.

DHCP Server

Marque la opción Habilitar DHCP Server para permitir que DiskStation asigne direcciones IP a los dispositivos cliente que se conecten a la red local proporcionada por DiskStation.

Transmisión PPPoE

Habilite la Transmisión PPPoE para permitir que los dispositivos conectados a DiskStation se conecten a Internet mediante una conexión PPPoE.

Configurar como pasarela de Internet UPnP

Habilite el servicio UPnP para permitir que su DiskStation se convierta en un dispositivo de pasarela de Internet UPnP, de manera que los dispositivos conectados a la red local de DiskStation puedan configurar parámetros de reenvío de puertos con UPnP.

Configuración IPv6

Puede conseguir una dirección IPv6 y un prefijo de LAN/Internet con 6in4, 6to4 o DHCP-PD. Con esta información, puede configurar aquí las opciones de enrutamiento IPv6 del DiskStation.

  • Dirección IPv6: Introduzca la dirección IPv6 obtenida de LAN/Internet.
  • Longitud de Prefix: Introduzca la longitud de prefijo de la dirección IPv6 obtenida de LAN/Internet, normalmente 64.
  • DNS primario: El DNS server primario para los dispositivos conectados al enrutador. No es necesario configurar esto en el Modo sin estado.
  • DNS secundario: El DNS server secundario para los dispositivos conectados al enrutador. No es necesario configurar esto en el Modo sin estado.

Existen tres modos de publicidad de enrutador (RA) para seleccionar entre:

  • Modo sin estado: Permite que dispositivos conectados al enrutador configuren automáticamente una dirección IPv6 sin configurar el DNS Server de IPv6.
  • Modo sin estado DHCPv6: Permite que dispositivos conectados al enrutador configuren automáticamente una dirección IPv6 y obtengan la información DNS a través de DHCPv6.
  • Modo con estado: Permite que dispositivos conectados al enrutador obtener una dirección IPv6 y la información DNS a través de DHCPv6. En este modo tiene que habilitar el DHCPv6 server y configurar la Inicio de dirección IP y la Fin de dirección IP del DHCPv6 server.

Clientes DHCP

Aquí puede ver una lista de clientes DHCP y sus configuraciones de red (por ejemplo, dirección MAC, dirección IP, nombre de host y la cantidad de tiempo que queda antes de que expire el uso de DHCP).

Puede filtrar la lista desplegable de clientes IPv4/IPv6 para mostrar los clientes que utilizan IPv4 y IPv6 respectivamente. Elija un cliente y haga clic en Agregar a la lista de direcciones reservadas para agregar el cliente a la lista de direcciones reservadas.

Reserva de DHCP

Si desea que un cliente reciba siempre la misma dirección IP al renovar el uso de DHCP, puede agregar el cliente a la lista de direcciones reservadas.

Puede filtrar la lista desplegable de direcciones reservadas de IPv4/IPv6. Las direcciones IPv4 se identifican con direcciones MAC, mientras que las direcciones IPv6 se identifican con DUID.

Para agregar un cliente a la lista de direcciones reservadas:

  • Seleccione un cliente y haga clic en Agregar.
  • Especifique la dirección MAC o DUID del cliente, la dirección IPv4/IPv6 que quiere asignar al cliente y el nombre de host del cliente.
  • Haga clic en OK para guardar su configuración.

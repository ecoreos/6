Aplicaciones web

Con la característica incorporada de Web Station y la compatibilidad PHP, podrá alojar fácilmente un sitio web en su DiskStation y crear sitios web dinámicos para su empresa o afición. Además, el Centro de paquetes ofrece una gran variedad de paquetes de terceros, como Administración de contenido, Administración de relaciones con clientes y Sistema de comercio electrónico. Una carpeta compartida predeterminada llamada web se creará para almacenar archivos para su sitio web.

Para habilitar Web Station:

  1. Vaya al Panel de control > Servicios web.
  2. Marque Habilitar Web Station.
  3. Haga clic en Aplicar.

Nota:

  • Para una mayor flexibilidad y seguridad, Web Station utiliza el grupo de usuarios http para realizar las tareas. Por lo tanto, para controlar los permisos de acceso de sus páginas web, cambie los permisos de acceso del grupo http para cada carpeta y fichero correspondiente. Por ejemplo, si desea que el fichero sample.htm en la carpeta compartida web sea accesible con Web Station, necesitará asegurarse de que el grupo http tiene los permisos de lectura/escritura para la carpeta compartida web además del archivo sample.htm.
  • El archivo de la página principal debe denominarse index.htm, index.html o index.php.
  • Cuando utilice la función PHP mail(), el sistema recuperará automáticamente y utilizará la información de servidor SMTP en la página Notificación. Asegúrese de que ha habilitado las notificaciones por correo electrónico y ha introducido la información correcta del servidor SMTP.
  • La ruta real de la carpeta compartida web es /volume1/web. Algunas aplicaciones requieren que el usuario defina la ruta en el archivo de configuración.
  • Los nombre blog, photo, photosrc, y webdefault son nombres de carpeta reservados. Estos nombres no se pueden usar en Web Station, por lo que no debe crear carpetas con estos nombres en la carpeta compartida web.
  • Puede personalizar la página de error creando un archivo missing.html, missing.htm o missing.php. Primero habilite Web Station. A continuación, coloque el archivo missing.html, missing.htm o missing.php en la carpeta compartida web del servidor. Cuando los usuarios se enlacen a una página que no existe, verán esta página de error personalizada.
  • Número máximo de conexiones HTTP activas: 256
  • Número máximo de hosts virtuales: 30

Habilitar host virtual para alojar sitios web

Puede alojar varios sitios web en un único servidor con la función del host virtual. Para comenzar, deberá registrar un par de nombres de dominio y configurar el alojamiento virtual siguiendo los pasos que se indican a continuación.

Para habilitar el host virtual:

  1. Haga clic en Host virtual.
  2. Haga clic en Crear.
  3. Escriba un nombre para la subcarpeta o carpeta compartida web, que almacenará los archivos del sitio web.
  4. Especifique un nombre de host para el sitio web.
  5. Especifique un puerto para el sitio web.
  6. Haga clic en OK.

Habilitar sitio web personal

La función sitio web personal es una forma práctica de permitir que los usuarios locales, los usuarios de dominio y los usuarios de LDAP puedan crear su propio sitio web personal.

Para usuarios locales

Cada usuario local tendrá una dirección exclusiva para su sitio web, como server-IP/~username o server-name/~username. Por ejemplo:

  1. 192.168.1.1/~ken
  2. SynologyNAS/~ken

Para usuarios de dominio:

Cada usuario de dominio tendrá una dirección exclusiva para su sitio web, como server-IP/~domainname%5cusername o server-name/~domainname%5cusername. Por ejemplo:

  1. 192.168.1.1/~testdomain%5cken
  2. SynologyNAS/~testdomain%5cken

Para usuarios de LDAP

Cada usuario de LDAP tendrá una dirección exclusiva para su sitio web, como server-IP/~username@domainname o server-name/~username@domainname. Por ejemplo:

  1. 192.168.1.1/~ken@testdomain
  2. SynologyNAS/~ken@testdomain

Nota:

Antes de habilitar esta característica, asegúrese de que homes de usuarios se han habilitado. Para usuarios locales, vaya a Panel de control > Usuario > Avanzado; para usuarios de dominio, vaya a Panel de control > Servicio de directorio > Usuarios de dominio > Inicio de usuario; para usuarios de LDAP, vaya a Panel de control > Servicio de directorio > Usuarios de LDAP > Inicio de usuario.

Nota:

Antes de habilitar esta característica, asegúrese de que homes de usuarios se han habilitado. Para locales, vaya a Panel de control > Usuario > Avanzado.

Para crear un sitio web personal:

  1. Si ha marcado Habilitar sitio web personal, se creará una carpeta www automáticamente en el directorio inicial del usuario.
  2. Ahora el usuario podrá cargar archivos relacionados con el sitio web (por ejemplo, páginas web y archivos de sonido) en la carpeta www.
  3. Asegúrese de que el usuario tiene permisos de acceso adecuados en los archivos cargados.

Para personalizar la página de error:

  1. Guarde la página de error como missing.html, missing.htm o missing.php.
  2. Cargue el archivo a la carpeta www.

Si los usuarios se enlazan a una página que no existe en el sitio web del usuario local, se mostrará la página de error personalizada.

Compatibilidad y limitaciones:

Los usuarios locales solo pueden acceder a datos debajo de lo siguiente:

  • Su propio sitio de inicio
  • /var/tmp
  • /tmp
  • /volumeX/@tmp

Servicio HTTP

Aparte de los números de puertos predeterminados 80 (para HTTP) o 443 (para HTTPS), se pueden agregar números de puertos adicionales para acceder a Photo Station y Web Station. Puede agregar un número de puerto adicional de 1 a 65535 para HTTP o HTTPS.

Para agregar un número de puerto adicional para acceso HTTP o HTTPS:

  1. Marque Agregar puerto HTTP adicional o Agregar puerto HTTPS adicional (la opción Habilitar conexión HTTPS para servicios web debe estar habilitada antes de agregar un puerto para HTTPS).
  2. Escriba el número de puerto deseado en el campo.

Nota:

  • Deben excluirse los siguientes puertos: 20, 21, 22, 23, 25, 110, 137, 138, 139, 143, 199, 443, 445, 515, 543, 548, 587, 873, 993, 995, 3306, 3689, 5000, 5001, 5005, 5006, 5335, 5432, 6881, 8080, 7000, 7001, 8081, 9997, 9998, 9999, 50001, 50002, puertos predeterminados de eMule: 4662 (TCP), 4672 (UDP), e intervalo de puertos por defecto para FTP: el intervalo concreto puede variar dependiendo de los distintos modelos.

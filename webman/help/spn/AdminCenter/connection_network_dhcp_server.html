DHCP Server

Su DiskStation puede actuar como un servidor DHCP y asigna direcciones IP dinámicas a clientes DHCP dentro de su red de área local.

Para habilitar un servidor DHCP:

  1. Vaya al Panel de control > Servidor DHCP > Interfaz de red.
  2. Seleccione un interfaz de red (por ejemplo LAN 1) para asignar direcciones IP y haga clic en Editar.
  3. En la ventana emergente, haga clic en la pestaña DHCP Server y seleccione Habilitar servidor DHCP.
  4. Ajuste la siguiente configuración de DHCP:
    • Tiempo de uso de dirección (minutos)
    • DNS primario
    • DNS secundario (opcional)
    • Nombre de dominio (opcional)
    • Habilitar el descubrimiento automático de proxy web (opcional)
  5. Configure al menos una subred. Para configurar la subred, haga clic en Agregar para especificar la siguiente información:
    • Inicio de dirección IP
    • Fin de dirección IP
    • Máscara de subred
    • Pasarela
  6. Haga clic en Crear para mostrar la subred creada en la lista.
  7. Después de crear suficientes subredes, recuerde activar al menos una de ellas.
  8. Haga clic en OK.

Nota:

  • Puede crear tantas subredes como desee.
  • Si se coordina con un servidor de retransmisión DHCP, su DiskStation como servidor DHCP puede asignar direcciones IP a dispositivos en otra subred.

Clientes DHCP

Puede ver una lista de clientes DHCP y sus configuraciones de red (p. ej., dirección MAC, dirección IP, nombre de host y la cantidad de tiempo que queda antes de que expire el uso de DHCP y el estado de reserva).

  1. Vaya al Panel de control > Servidor DHCP > Interfaz de red y seleccione la interfaz que desee.
  2. Haga clic en Editar > Clientes DHCP para localizar los clientes y los detalles.

Para administrar direcciones IP para clientes DHCP:

Si lo desea, puede hacer que un cliente reciba siempre la misma dirección IP al renovar el uso de DHCP y administrar estos clientes.

  • Para agregar clientes DHCP: Haga clic en Agregar para asignar la dirección IP al dispositivo cliente.
  • Para reservar la dirección IP: Haga clic en el icono de bloqueo para reservar la dirección IP para el dispositivo actual.
  • Para cancelar la reserva: Haga clic en el icono de bloqueo para permitir que la dirección IP actual sea utilizada por otro dispositivo una vez que expire el tiempo de uso de dirección.

Nota:

Si un dispositivo puede tener direcciones IPv4 e IPv6, puede reservar una dirección IPv4 y una dirección IPv6 para el dispositivo.

Para buscar clientes DHCP:

  1. Haga clic en el icono de embudo de la esquina superior derecha.
  2. Seleccione uno de los siguientes criterios de búsqueda:
    • Todo: muestra todos los clientes DHCP y las reservas de DHCP.
    • Clientes DHCP: muestra todos los clientes DHCP con direcciones IP reservadas o sin reservar
    • Reservas de DHCP: muestra todas las direcciones IP reservadas. Una dirección IP reservada no puede emparejarse con ningún otro dispositivo.

Opciones de DHCP

Una vez que haya especificado las opciones de DHCP, su DiskStation podrá enviar, como el servidor DHCP, información relacionada a los clientes DHCP en una determinada subred.

Para especificar las opciones de DHCP de una subred:

  1. Vaya al Panel de control > Servidor DHCP > Interfaz de red.
  2. Seleccione la interfaz de red que desee y haga clic en Editar.
  3. Acceda a Servidor DHCP y, a continuación, a la Lista de subredes.
  4. Realice una de las siguientes dos acciones:
    • Para especificar las opciones de DHCP de una subred existente: seleccione la interfaz que desee y haga clic en Editar.
    • Para crear una subred y especificar sus opciones de DHCP: haga clic en Agregar y defina la configuración de la subred.
  5. Seleccione las opciones de DHCP que desee; para especificar los valores correspondientes, haga clic en los campos en blanco para editarlos.
  6. Haga clic en OK.

Proveedor

Los clientes DHCP de diferentes proveedores pueden requerir más información sobre DHCP. Al configurar las opciones de DHCP para los proveedores, su DiskStation puede proporcionar dichos datos a los clientes.

Para descubrir qué opciones de DHCP son compatibles con cada uno de los proveedores, póngase en contacto con ellos para obtener más información.

Para agregar proveedores:

  1. Vaya a Panel de control > Servidor DHCP > Proveedor.
  2. Haga clic en Crear para especificar el proveedor y sus opciones de DHCP:
    • Nombre: especifique el nombre del proveedor.
    • Código: especifique el código numérico para la opción de DHCP.
    • Valor: especifique el valor de la opción.
    • +: agregue una opción de DHCP.
    • -: elimine una opción de DHCP.
  3. Haga clic en Crear para terminar la configuración.

Para editar las opciones de DHCP para los proveedores:

  1. Vaya a Panel de control > Servidor DHCP > Proveedor.
  2. Seleccione el proveedor que desee y haga clic en Editar para gestionar sus opciones de DHCP.
  3. Haga clic en OK.

PXE

Su DiskStation puede actuar como un servidor DHCP para implementar la configuración de PXE (entorno de ejecución previo al arranque). PXE es un entorno de red que permite que los dispositivos cliente arranquen utilizando una interfaz de red en lugar de un dispositivo de almacenamiento de datos (p. ej., discos duros).

Al implementar PXE, se puede almacenar un cargador de arranque en su DiskStation o un servidor TFTP remoto, y los dispositivos cliente pueden arrancar a través de la red utilizando el cargador de arranque.

Para utilizar DiskStation como servidor del cargador de arranque:

  1. Asegúrese de que ha seleccionado Habilitar servicio TFTP (en Panel de control > Servicios de archivos > TFTP).
  2. Seleccione una carpeta que aloje el archivo del cargador de arranque como la carpeta root TFTP.
  3. Asegúrese de que al menos una subred DHCP está habilitada (en Panel de control > Servidor DHCP > Interfaz de red).
  4. Vaya al Panel de control > Servidor DHCP > PXE.
  5. Seleccione Habilitar PXE (entorno de ejecución previo al arranque).
  6. Pulse Servidor TFTP remoto y, a continuación, seleccione el archivo del cargador de arranque.
  7. Haga clic en Aplicar para guardar la configuración.

Para utilizar un servidor TFTP remoto como servidor del cargador de arranque:

Al utilizar un servidor TFTP remoto como un servidor de cargador de arranque, los clientes podrán arrancar sin tener que cambiar la infraestructura de red actual.

  1. Asegúrese de que al menos una subred DHCP está habilitada (en Panel de control > Servidor DHCP > Interfaz de red).
  2. Vaya al Panel de control > Servidor DHCP > PXE.
  3. Seleccione Habilitar PXE (entorno de ejecución previo al arranque).
  4. Seleccione Servidor TFTP remoto, y especifique la dirección IP del servidor y la ruta del archivo del cargador de arranque.
  5. Haga clic en Aplicar para guardar la configuración.

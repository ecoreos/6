Control de tráfico

El objetivo del control de tráfico es controlar el tráfico saliente de los servicios que se ejecutan en DiskStation. Una vez que seleccione un servicio, debe configurar el Ancho de banda garantizado y el Ancho de banda máximo para limitar su tráfico saliente.

El Ancho de banda garantizado representa el tráfico saliente que garantiza este servicio cuando basta con el ancho de banda de todo el servicio.

El Ancho de banda máximo representa el tráfico saliente que puede utilizar este servicio cuando basta con el ancho de banda de todo el sistema y hay un Ancho de banda del sistema restante. Se recomienda que el administrador del sistema calcule primero el Ancho de banda de salida del sistema y luego se asegure de que la suma del Ancho de banda garantizado de cada servicio no supera el Ancho de banda de salida del sistema, porque de lo contrario puede que el ajuste no funcione correctamente.

Fórmula de ajuste:

Ancho de banda de salida del sistema = Suma del Ancho de banda garantizado de cada servicio + Ancho de banda restante del sistema

Ancho de banda garantizado de cada servicio ≤ Ancho de banda máximo de cada servicio

Nota: Si los paquetes salientes no cumplen ninguna regla de control de tráfico, el comportamiento predeterminado será ninguna limitación. Puede añadir su propia regla de control de tráfico predeterminada al final de todas las reglas seleccionando Todos los puertos e indicando Ancho de banda garantizado y Ancho de banda máximo.

Para crear una regla de control de tráfico:

  1. Vaya a Panel de control > Cortafuegos > Control de tráfico y seleccione la interfaz de red de la columna izquierda según el tipo de conexión de red.
  2. Haga clic en Crear.
  3. En la sección Puertos, seleccione una de las siguientes opciones para la regla:
    • Elija Todos para aplicar la reglas a todo los puertos.
    • Elija Seleccione de una lista de aplicaciones incorporadas.
    • Elija Personalizado y haga clic en Personalizado y escriba hasta 15 puertos separados por comas o especifique un intervalo de puertos.
    • También puede especificar los protocolos.

Ahora puede ver la regla de control de tráfico creada en la lista de reglas.

Para eliminar reglas de control de tráfico, haga lo siguiente:

  • Desactive la casilla de verificación de la regla en la columna Habilitado de la lista
  • Elimine la regla de control de tráfico.
    Nota:
  1. Puede crear hasta 100 reglas para una interfaz de red.
  2. Puede reordenar las reglas en la lista arrastrando y colocando. Las reglas se priorizan en función de sus posiciones en la lista.
  3. Si el sistema tiene 2 puertos de red conectados a la misma subred, es posible que las reglas de cortafuegos no funcionen correctamente.*

____
* Sólo se aplica a determinados modelos.

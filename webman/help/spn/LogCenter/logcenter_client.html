Envío de registros

Si tiene un Syslog Server, puede seguir estas instrucciones para configurar DiskStation como un cliente Syslog y transmitir los registros del sistema local de su DiskStation a un Syslog Server para realizar un seguimiento y administración centralizados.

Para enviar registros a un syslog server:

  1. Vaya a Centro de registros > Envío de registros.
  2. Marque la casilla Enviar registros a un syslog server.
  3. Escriba el nombre de host o la dirección IP del syslog server en Servidor.
  4. Indique un número de puerto para recibir mensajes de syslog en Puerto.
  5. Seleccione UDP o TCP en Protocolo de transferencia. Esto debería coincidir con el protocolo utilizado por el syslog server.
  6. Seleccione BSD (RFC 3164) o IETF (RFC 5424) en Formato de registro para enviar registros con el formato seleccionado.

    7. Nota:

    • De forma predeterminada, el puerto utilizado es 514. Se deben definir reglas de reenvío de puertos adecuadas si desea reasignar un número de puerto diferente al syslog server.
    • UDP (Protocolo de datagrama de usuario) es un protocolo de conexiones basado en mensajes más sencillo, que es más rápido pero ofrece una seguridad y una corrección de errores menores. TCP (Protocolo de control de transmisiones) ofrece garantía de entrega, para la cual utiliza un control de flujo con el propósito de determinar si todos los paquetes se han enviado correctamente antes de que se procesen otros nuevos.
    • La conexión segura (SSL) solo está disponible cuando se utiliza TCP.
    • Si elige TCP y desea asegurar la conexión de Syslog con SSL, marque la casilla Habilitar conexión segura (SSL). A continuación haga clic en Importar certificado y seleccione el certificado para transferir registros de manera segura al syslog server.
  7. Haga clic en Enviar registro de prueba para confirmar que el syslog server puede recibir registros correctamente. Si el syslog server no recibe el registro de prueba, compruebe que la configuración es correcta.
  8. Haga clic en Aplicar para guardar la configuración.

Filtrar registros

En la pestaña Filtra, puede especificar los tipos de registros que se enviarán al syslog server, según la categoría (por ejemplo, conexión del sistema, copia de seguridad) o severidad (por ejemplo, advertencia, error, información).

Descripción general

Esta página muestra información básica sobre el clúster High-Availability y sus servidores. Además, esta página ofrece herramientas para realizar tareas de mantenimiento en clústeres High-Availability y en sus servidores.

Estado del clúster High-Availability

Este campo muestra el estado general de su clúster High-Availability y le avisa de cualquier error. A continuación se muestra una tabla de los estados que pueden aparecer en este campo:

Estado Descripción
En buen estado El estado general de su clúster High-Availability es bueno.
Cargando El sistema está cargando el estado del clúster. No se puede realizar un Cambio hasta que no se haya completado la carga del estado. Consulte Cambio.
Configurando El sistema se está configurando. No se puede realizar un Cambio hasta que no se haya completado la configuración.
Advertencia El estado del clúster HA es anormal, pero los servicios siguen funcionando. Tal vez no se pueda realizar el Cambio en esta situación.
Peligro Tanto el servidor activo como el pasivo funcionan incorrectamente o el almacenamiento se ha averiado. El clúster High-Availability no puede funcionar normalmente. Compruebe el estado de todos los espacios de almacenamiento (p. ej., RAID Groups, grupos de discos, volúmenes e iSCSI LUNs). En esta situación no se pueda realizar el Cambio.

Mensajes de advertencia

La siguiente tabla muestra los mensajes de advertencia que pueden aparecer cuando se produce un error en el sistema. Cuando aparezca más de un mensaje de advertencia, puede desplazarse entre ellos con los botones de avance/retroceso.

Advertencia/Error Descripción
No hay ningún servidor pasivo en este clúster High-Availability La información sobre el servidor pasivo no está disponible. Asegúrese de que el servidor pasivo esté conectado correctamente. En este caso, el cambio no estará disponible.
No se ha podido detectar el servidor pasivo Este error podría producirse porque el servidor pasivo está desconectado o los servidores activo y pasivo no se pueden comunicar a través de la conexión Heartbeat. En este caso, el cambio no estará disponible.
No se ha podido realizar la conmutación por error Se ha activado una conmutación por error en el servidor activo pero ha fallado. En este caso, el servidor activo no volverá a intentar otra conmutación por error. Para eliminar este error, siga estos pasos:
  1. Intente resolver los problemas en el clúster High-Availability.
  2. Reiniciar el servidor pasivo.
  3. Realice una conmutación por error haciendo clic en Administrar > Cambio.
Error de destino supervisado Se ha producido un error en uno o más de los destinos supervisados, incluido el estado del sistema, servicio o interfaz de red.
Se han producido errores en la conexión entre las unidades de expansión y el servidor activo/pasivo Las unidades de expansión no están conectadas correctamente con el servidor pasivo o con el activo. Asegúrese de que su conexión con el servidor activo es correcta y configure la misma conexión con el servidor pasivo. En este caso, el cambio no estará disponible.
Se ha degradado un volumen Un espacio de almacenamiento (p. ej., RAID Group, grupo de discos, volumen, iSCSI LUN y caché de SSD) está degradado. Consulte la página Estado de almacenamiento.
Se ha averiado un volumen Un espacio de almacenamiento (p. ej., RAID Group, Grupo de discos, volumen, iSCSI LUN y caché de SSD) está averiado. Consulte la página Estado de almacenamiento. La conmutación por error se realizará automáticamente si el servidor pasivo consigue reconocer los discos a los que el servidor activo no ha conseguido acceder. De lo contrario, el cambio no estará disponible.
Se ha perdido la conexión Heartbeat Este error podría deberse a un fallo en la conexión Heartbeat. En este caso, el cambio no estará disponible.
Avería de hardware Este error se puede producir debido a un fallo de hardware (por ejemplo, la unidad de potencia o el ventilador del sistema) en el servidor activo o pasivo.
Diferente tamaño de RAM física Este error se puede deber a que el tamaño de RAM física del servidor activo no coincide con el del servidor pasivo. En este caso, el cambio no estará disponible.
No se ha asignado una IP estática a este clúster High-Availability Este error se puede producir por una falta de dirección IP estática para el clúster High-Availability. En este caso, el cambio no estará disponible.
No hay ninguna IP estática en la interfaz supervisada Este error se puede producir debido a la falta de una IP estática para la interfaz supervisada. En este caso, el cambio no estará disponible.
Recuperación del servidor pasivo La recuperación del servidor pasivo está en curso. En este caso, el cambio no estará disponible.
Actualizando el servidor pasivo El sistema está actualizando el servidor pasivo. En este caso, el cambio no estará disponible.

Botón "Administrar" del clúster High-Availability

Este botón está situado debajo de la información del clúster High-Availability. Incluye las siguientes operaciones:

  • Cambio: inicia un cambio de servicios del servidor activo al servidor pasivo. Durante este proceso, el servidor activo deja de suministrar servicios y se convierte en el nuevo servidor pasivo. Entonces, el servidor pasivo anterior pasa a ser el nuevo servidor activo y empieza a suministrar todos los servicios del clúster High-Availability. Esta opción no está disponible cuando el servidor pasivo está en estado de error o no está seleccionado.
  • Actualizar DSM: abre la página Actualizar DSM del Panel de control. Desde aquí se pueden actualizar ambos servidores.
  • Apagar el clúster High-Availability: apaga los servidores activos y pasivos.
  • Apagar el servidor activo: apaga el servidor activo. Los servicios cambiarán si el servidor pasivo está en buen estado.
  • Apagar el servidor pasivo: apaga el servidor pasivo.
  • Reiniciar el servidor activo: reinicia el servidor activo. Los servicios cambiarán si el servidor pasivo está en buen estado.
  • Reiniciar el servidor pasivo: reinicia el servidor pasivo.
  • Recuperar el servidor desconocido: recupera el servidor pasivo si se sustituye o si todavía está reconocido como desconocido.
  • Pitido del sistema desactivado: detiene el sonido del pitido del servidor activo y del servidor pasivo cuando se produce un error. El sistema seguirá emitiendo un pitido en el servidor activo o en el servidor pasivo si se produce otro error más tarde.

Nota:

  • Si apaga el servidor pasivo o si no hay servidor pasivo, los servicios no se pueden cambiar cuando se produce un error en el servidor activo.

Información y diagrama del servidor

El diagrama del servidor muestra el estado del servidor. La ubicación se corresponde con el lado de los módulos a los que está conectado el servidor. Pase el cursor del ratón por encima para obtener más detalles.

Incluye las siguientes situaciones:

  • No hay ningún servidor pasivo en este clúster high-availability: muestra un marco en blanco.
  • No se ha podido detectar el servidor pasivo: muestra una imagen borrosa del servidor.
  • Signo de exclamación: puede haber una avería del hardware o la información del servidor no está disponible debido a un fallo de la conexión Heartbeat.

Nota:

  • Si muestra "desconocido" en el otro servidor, significa que el servidor no pertenece al clúster High-Availability. Haga clic en Administrar y seleccione Recuperar el servidor desconocido para resolver este problema.

La información del servidor incluye las siguientes entradas. Muestra N/A si no se detecta el servidor pasivo.

  • Número de serie: número de serie del servidor de Synology.
  • Estado del ventilador: estado de los ventiladores de refrigeración del servidor. Si un ventilador falla, aparecerá una advertencia. En este caso, póngase en contacto con su distribuidor local para obtener ayuda.
  • Temperatura: temperatura del servidor de Synology.
  • Estado de energía: indica si los suministros de energía de los servidores funcionan con normalidad. Si el servidor cuenta con fuentes de alimentación redundantes, aparecerá una advertencia si una de ellas no funciona correctamente.
  • Memoria física: tamaño de RAM física del servidor de Synology. El cambio no se puede realizar si los dos servidores no tienen el mismo tamaño de RAM física.

Estado de conectividad entre servidores

Esta sección contiene imágenes entre el servidor activo y el servidor pasivo. Incluye las siguientes situaciones:

  • Sin imagen: indica que no se detecta ningún servidor pasivo.
  • Imagen fija: indica una conexión Heartbeat incorrecta.
  • Signo de exclamación: indica una conexión anormal entre los dos servidores o una conexión Heartbeat incorrecta.

Estado de la interfaz de red

Esta sección contiene imágenes entre los servidores y la red de datos. Incluye las siguientes situaciones:

  • Sin signo de exclamación: indica que no se ha detectado ningún fallo de red.
  • Signo de exclamación: indica que existe un fallo en la interfaz de red supervisada.

Información y diagrama del almacenamiento compartido

El diagrama de almacenamiento compartido muestra el estado de conectividad entre los servidores y el almacenamiento compartido. Pase el cursor del ratón por encima para obtener información básica.

Incluye las siguientes situaciones:

  • Sin signo de exclamación: indica que no se ha detectado ningún error.
  • Signo de exclamación: indica un estado de conectividad anormal en las unidades de expansión.

La información básica del almacenamiento compartido incluye las siguientes entradas. Vaya al Administrador de almacenamiento para obtener información más detallada.

  • Estado del ventilador: muestra el estado de los ventiladores de las unidades de expansión. Si un ventilador falla, aparecerá una advertencia. En este caso, póngase en contacto con su distribuidor local para obtener ayuda.
  • Estado de energía: indica si los suministros de energía de los servidores funcionan con normalidad. Aparecerá una advertencia si alguno de ellos no funciona con normalidad.

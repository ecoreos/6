High Availability Manager

High Availability se refiere a una solución de distribución de servidores diseñada para reducir las interrupciones de servicio causadas por fallos de funcionamiento del servidor. Synology High Availability utiliza dos servidores para formar un "clúster High-Availability" en el que un servidor asume el papel de "servidor activo" y otro servidor actúa como "servidor pasivo" en espera. Para crear e implementar un clúster High-Availability, necesitará dos servidores Synology compatibles.

Un clúster High-Availability utiliza los siguientes componentes y conceptos:

  • Concepto: La implementación de Synology High Availability requiere dos servidores de Synology para crear un clúster High-Availability. Un servidor actúa como "servidor activo" y el otro actúa como "servidor pasivo". Los servidores activo y pasivo del clúster High-Availability están conectados a las mismas unidades de expansión y comparten el mismo almacenamiento entre ellos. Para garantizar la integridad de los daos, solo el servidor activo puede manejar la carga de trabajo. En caso de que se produzca una avería del servidor activo, el servidor pasivo estará listo para hacerse cargo de todos los servicios. Por tanto, el clúster High-Availability seguirá ofreciendo servicios normalmente, minimizando el tiempo de inactividad del servidor. Para obtener más información sobre el cambio, consulte Cambio.
  • Servidor activo: El servidor activo gestiona el suministro de todos los servicios. En caso de que se produzcan errores en el servidor activo, usted o el sistema pueden iniciar el cambio al servidor pasivo. En ese caso, la prestación de servicios se transferirá al servidor pasivo, momento en el cual el servidor activo se convertirá en servidor pasivo.
  • Servidor pasivo: En condiciones normales, el servidor pasivo permanece en modo de espera. En caso de que el servidor activo no pueda prestar servicios, usted o el sistema puede iniciar un cambio al servidor pasivo.
  • Conexión Heartbeat: los servidores activo y pasivo de un clúster High-Availability están conectados entre sí mediante una conexión de red privada exclusiva denominada "conexión Heartbeat". Si se pierde el contacto entre los servidores durante un periodo de tiempo concreto, el sistema determinará si es necesario un cambio.
  • Almacenamiento compartido del clúster High-Availability: Los discos en las unidades de expansión conectadas por los servidores activo y pasivo simultáneamente actúan como almacenamiento compartido en el clúster High-Availability. Solo el servidor activo tiene permisos de escritura en el almacenamiento.
  • Configuración de red y nombre de servidor del clúster High-Availability: La configuración de red y el nombre de servidor del clúster High-Availability estarán determinados por la configuración correspondiente en DSM. En la red, puede conectarse utilizando este nombre de servidor y el servidor activo prestará los servicios. Para obtener más información acerca de la configuración de red consulte aquí.

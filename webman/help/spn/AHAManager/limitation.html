Limitaciones y requisitos del sistema

La implementación de Synology High Availability necesita dos servidores Synology idénticos que actúen como servidores activo y pasivo. Antes de que los dos servidores se combinen formando un clúster high-availability, el sistema comprobará las siguientes limitaciones de hardware y software para garantizar que puede crearse un clúster high-availability.

Requisitos del sistema

Tenga en cuenta las siguientes directrices sobre software y hardware:

  • Servidor Synology: los servidores activo y pasivo deben ser modelos idénticos.
  • Unidades de expansión: tanto el servidor activo como el pasivo deben estar conectados a las mismas unidades de expansión correctamente.
  • Número y tipo de discos: el tipo de discos debe ser SAS solo. Antes de instalar DSM tiene que instalar al menos 3 discos en las unidades de expansión.
  • Dispositivo USB externo: solo el almacenamiento USB es compatible. Otros dispositivos USB externos (por ejemplo, SAI, dongles de Wi-Fi, altavoces, impresoras, dongle de Bluetooth y dongle de DVB) no son compatibles. El almacenamiento USB se puede utilizar solo en el servidor activo y no se puede cambiar al servidor pasivo.

Entorno de red

Observe las siguientes directrices referentes a las configuraciones de red cuando intente formar un clúster de high-availability.

  • Configuración de red: la conexión Heartbeat entre los dos servidores se debe configurar correctamente. Ambos servidores deben pertenecer a la misma subred y es necesario configurar las direcciones IP estáticas después de la instalación.
  • Número de puertos LAN: Ambos servidores deben tener el mismo número de puertos LAN. Si uno de los modelos de DiskStation está equipado con interfaces de tarjeta de red, el otro debe estar equipado con las mismas interfaces de tarjeta de red.
  • Restricciones de conexión: el clúster de high-availability no es compatible con la red Wi-Fi.

Nota:

  • el servidor NTP se habilitará automáticamente en el servidor activo.

Limitaciones del software y de paquetes

Vea las limitaciones relacionadas con los paquetes y el software a continuación:

  • la función de clúster high-availability no es compatible con los siguientes paquetes y características:
    • DVBLink Server
    • La característica para reiniciar/apagar el clúster de high-availability en CMS
    • Time Backup
  • El clúster high-availability con almacenamiento compartido no se puede configurar como un clúster high-availability basado en la replicación ni se puede utilizar en combinación con un clúster de ese tipo.

Unidades de expansión específicas

Tanto el servidor activo como el pasivo deben estar conectados a unidades de expansión específicas. Existen dos módulos en cada unidad de expansión.

Para configurar/agregar unidades de expansión:

  • un módulo contiene dos puertos; el puerto de entrada está en el lado izquierdo y el de salida en el lado derecho. Conecte un cable SAS desde el puerto de salida de un módulo hasta el puerto de entrada del módulo de la siguiente unidad de expansión.
  • Las unidades de expansión deben estar conectadas en serie con los módulos en los lados correspondientes para cada servidor.
  • Tanto el servidor activo como el pasivo deben estar conectados a unidades de expansión compartidas en el mismo orden.

Notas:

para obtener información detallada sobre la instalación de unidades de expansión, no olvide consultar la Guía de instalación rápida de los servidores y las unidades de expansión.

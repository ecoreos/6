Apagado y reinicio

Las siguientes instrucciones le guiarán en el proceso de apagado, eliminación o reinicio de un clúster high-availability.

Para apagar el clúster high-availability utilizando Synology High Availability:

Si desea apagar un clúster high-availability para transportar los servidores a una nueva ubicación física o realizar labores de mantenimiento, pero no quiere eliminar las configuraciones del clúster high-availability, siga estas instrucciones.

  1. Inicie sesión en DSM.
  2. Abra Synology High Availability.
  3. Haga clic en el botón Gestionar.
  4. Seleccione Apagar clúster HA.
  5. Aparecerá un mensaje de confirmación. Haga clic en Sí para confirmar.

Para apagar manualmente un clúster high-availability:

El clúster high-availability también se puede apagar pulsando los botones de encendido de cada servidor.

  1. Mantenga pulsado el botón de encendido de un servidor (activo o pasivo) hasta que emita un pitido.
  2. Una vez que se haya apagado por completo un servidor, espere al menos 30 segundos (el servidor que sigue en línea necesita tiempo para guardar la configuración).
  3. Mantenga pulsado el botón de encendido del otro servidor hasta que emita un pitido.
  4. Espere a que el servidor se apague por completo.

Encender servidores:

Cuando quiera volver a encender los servidores, debería encender el servidor activo y el servidor pasivo al mismo tiempo (o con un intervalo de cinco minutos entre ellos). Si solo enciende uno de los servidores, este intentará encender el otro servidor enviando un mensaje de Despertar con LAN.

Apagado anormal:

Cuando se crea un clúster high-availability, se habilita automáticamente la opción de reinicio tras corte de electricidad en el Panel de control. Si los servidores se apagan de forma anormal debido a un corte de electricidad, se encenderán automáticamente una vez que se restablezca la electricidad.

Para eliminar un clúster de high-availability:

Cuando desee desvincular ambos servidores y eliminar un clúster high-availability para vincular otro servidor, siga estas instrucciones.

  1. Vaya a la página Visión general.
  2. Haga clic en el botón Gestionar.
  3. Seleccione Eliminar clúster HA.

Tras la confirmación, el sistema eliminará el clúster high-availability. El campo de estado muestra el progreso del proceso de eliminación. Cuando el clúster high-availability se haya eliminado correctamente, se le redirigirá a la dirección IP original del servidor activo. El estado de ambos servidores cambiará a Independiente.

Para reiniciar un clúster high-availability que funciona mal:

En caso de que no pueda acceder a DSM o al High Availability de Synology, debido a un error de "split brain" o a otro tipo de fallo, puede intentar forzar el clúster high-availability para que se desvincule pulsando el botón Restablecer ubicado en sus servidores.

  1. Mantenga pulsado el botón Restablecer (normalmente ubicado en el panel posterior de un servidor) hasta que el servidor emita un pitido.
  2. El sistema eliminará el clúster high-availability, y el estado de ambos servidores cambiará a Independiente
  3. Al pulsar el botón Restablecer, también se restaurarán algunos valores de configuración predeterminados, como la dirección IP, el DNS y la contraseña de admin, además de lo siguiente:
    • La contraseña de admin quedará vacía.
    • La cuenta de invitado se deshabilitará.
    • Se habilitará DHCP en IP, DNS, pasarela y otras interfaces de red.
    • Se deshabilitará JumboFrames y MTU se ajustará en 1500.
    • El puerto de administración se restablecerá en 5000/5001.
    • PPPoE se restablecerá a los valores predeterminados.
    • El bloqueo automático de IP se deshabilitará (la dirección IP bloqueada no se borrará).
    • Link Aggregation se deshabilitará.
    • Las reglas de cortafuegos se deshabilitarán (pero no se borrarán).

Nota:

Durante el proceso de eliminación de un clúster high-availability, pueden quedar algunas operaciones de escritura de datos incompletas. Para garantizar la integridad de los datos, realice la replicación de datos basada en el servidor activo, ya que puede que los datos del servidor pasivo no estén del todo actualizados.

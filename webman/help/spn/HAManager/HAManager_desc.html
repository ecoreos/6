Administrador de High Availability

High Availability se refiere a una solución de distribución de servidores diseñada para reducir las interrupciones de servicio causadas por fallos de funcionamiento del servidor. Synology High Availability utiliza dos servidores para formar un "clúster de high-availability" en el que un servidor asume el papel de "servidor activo" y otro servidor actúa como "servidor pasivo" en espera. Para crear e implementar un clúster de high-availability, necesitará dos servidores Synology compatibles.

Un clúster high-availability utiliza los siguientes componentes y conceptos:

  • Concepto: la implementación de Synology High Availability requiere dos servidores de Synology para crear un clúster high-availability. Un servidor actúa como "servidor activo" y el otro actúa como "servidor pasivo". Una vez que se crea el clúster high-availability, los datos están replicándose continuamente desde el servidor activo al servidor pasivo. Todos los archivos del servidor activo se duplicarán en el servidor pasivo. En caso de que falle el servidor activo, el servidor pasivo estará listo para tomar el control de todos los servicios con una copia espejo de los datos del servidor activo. Por tanto, el clúster high-availability seguirá ofreciendo servicios normalmente, minimizando el tiempo de inactividad del servidor. Para más información sobre el cambio, consulte Cambio
  • Servidor activo: el servidor activo gestiona el suministro de todos los servicios. En caso de que se produzcan errores en el servidor activo, usted o el sistema pueden iniciar un cambio al servidor pasivo. En ese caso, el suministro de servicios se transferirá al servidor pasivo y el servidor activo se convertirá en servidor pasivo.
  • Servidor pasivo: en condiciones normales, el servidor pasivo permanece en modo de espera. Los datos del servidor activo están replicándose continuamente en el servidor pasivo. En caso de que el servidor activo no pueda prestar servicios, usted o el sistema puede iniciar un cambio al servidor pasivo.
  • Conexión Heartbeat: los servidores activo y pasivo de un clúster high-availability están conectados entre sí mediante una conexión de red privada dedicada llamada "conexión Heartbeat". Una vez que se haya creado el clúster high-availability, la conexión Heartbeat facilita la sincronización de los datos y la replicación entre los servidores activo y pasivo. Si se pierde el contacto entre los servidores durante un periodo de tiempo concreto, el sistema determinará si es necesario un cambio.
  • Volumen de almacenamiento principal del clúster high-availability: el volumen de almacenamiento del servidor activo se convertirá en el volumen de almacenamiento principal del clúster high-availability. Los datos se replicarán en el volumen de almacenamiento del servidor pasivo a través de la conexión Heartbeat.
  • Volumen de almacenamiento de recambio del clúster high-availability: el volumen de almacenamiento del servidor pasivo se convertirá en el volumen de almacenamiento de recambio del clúster high-availability. Los datos se replicarán continuamente desde el volumen de almacenamiento del servidor activo al volumen de almacenamiento del servidor pasivo a través de la conexión Heartbeat. Esto garantiza la coherencia del contenido de archivos cuando se produce el cambio.
  • Nombre de servidor del clúster high-availability: los servidores del clúster high-availability compartirán un nombre de servidor. En la red, puede conectarse utilizando este nombre de servidor y los servicios serán suministrados por el servidor activo.
  • IP del clúster high-availability: dado que los servidores activo y pasivo del clúster high-availability tienen cada uno su propia dirección IP, para evitar tener que cambiar la dirección IP cuando se realice un cambio, el clúster high-availability generará direcciones IP comunes para el acceso. Para acceder a cualquier servicio de DSM, sea cual sea el servidor de Synology que esté actuando como servidor activo, puede conectarse a través de estas direcciones IP.

Control de servicios

Esta página le permite controlar algunos servicios. Si un servicio controlado se avería en el servidor activo, el sistema conmutará al servidor pasivo en modo de espera (siempre que el servidor pasivo funcione con normalidad). En esta página aparecen las siguientes secciones:

  • Servicio: Muestra los servicios que se pueden controlar.
  • Estado: Muestra si los servicios están en funcionamiento o no.
  • Conmutación por error si el servicio falla: Marque la casilla de verificación junto a un servicio para comenzar a controlarlo. El servicio conmutará si el servicio seleccionado deja de funcionar.

Para controlar un servicio:

  1. Encuentre el servicio que desea controlar. Marque la casilla de verificación junto a él.
  2. Haga clic en Aplicar para guardar la configuración.

Servicios compatibles

Los servicios que se pueden controlar son los siguientes: Servicios de archivos Windows, iSCSI Target, FTP, Servicio de archivos Mac, Servicio NFS.

Conmutación cuando falla la conexión a la red

Si se habilita esta opción, el sistema conmutará al servidor pasivo cuando se interrumpa o averíe la conexión de un servidor activo con la red, y la conexión del servidor pasivo a la red todavía funcione.

  • La conmutación se disparará solo si el servidor pasivo tiene una mejor conexión. Por ejemplo, si el servidor activo tiene tres conexiones y una falla, el sistema activará una conmutación solo si el servidor pasivo tiene tres o más conexiones buenas.
  • Las conexiones conectadas con Link Aggregation cuentan como una conexión. Por ejemplo, si dos conexiones se han combinado y una de ellas falla, pero la otra todavía funciona, la conexión todavía se considera válida.

Configuración de red

Esta página contiene tres fichas que muestran la configuración de red del clúster high-availability, el servidor activo y el servidor pasivo.

Ficha Clúster HA

Al crear un clúster high-availability, el sistema genera una dirección IP común para que la IP del clúster high-availability reciba conexiones entrantes. Si es necesario, se pueden crear direcciones IP adicionales para el clúster high-availability. Las direcciones IP de las interfaces de red activadas del servidor activo se mostrarán en esta ficha (la interfaz de red usada por la conexión Heartbeat no se mostrará en esta página, ya que dicha interfaz se usará para la replicación de los datos entre los dos servidores). Esta ficha muestra la siguiente información:

  • Interfaz de red: indica la interfaz o las interfaces de red activadas en el servidor activo.
  • Conexión: Muestra si la interfaz está conectada o no.
  • Dirección IP: al crear un clúster high-availability, el sistema genera una dirección IP común para que la IP del clúster high-availability reciba conexiones entrantes. Puede conectarse al clúster high-availability utilizando esta dirección IP cuando quiera iniciar sesión en DiskStation Manager (DSM) independientemente de qué servidor de Synology sea el servidor activo. Esto evita tener que conectar con distintas direcciones IP después de un cambio.
  • Máscara de subred: la máscara de subred asociada a la dirección IP del clúster high-availability en cuestión.

Para editar la Dirección IP y la Máscara de subred:

  1. Elija una dirección IP o máscara de subred que desee editar y haga clic en Editar o bien haga doble clic en el campo para iniciar la edición.
  2. Active las casillas de verificación Habilitar para habilitar o deshabilitar cada interfaz. Puede habilitar más de una interfaz de red de clúster, pero la interfaz de red original configurada al crear el clúster no se puede deshabilitar.

Para cambiar el nombre del servidor del clúster high-availability:

  1. Haga clic en Cambie la configuración de red para ver el nombre del clúster high-availability y cambiar la configuración.
  2. Haga clic en Aplicar para guardar el nombre y la configuración de dirección IP del clúster high-availability actual.
  3. Para descartar los cambios actuales, haga clic en Cancelar.

Fichas del Servidor activo y pasivo (mostradas con el nombre de servidor)

Estas fichas muestran el estado de las interfaces de red del servidor activo o pasivo. De estas, la conexión Heartbeat ocupará una interfaz de red en cada uno de los servidores activo y pasivo. Estas fichas muestran la siguiente información:

  • Interfaz de red: la interfaz de red física del servidor.
  • Dirección IP: la dirección IP asignada a la interfaz de red en cuestión.
  • Máscara de subred: la máscara de subred asociada a la dirección IP en cuestión.
  • Descripción: muestra la información de la interfaz de red en cuestión y la accesibilidad de su dirección IP asignada. Este campo mostrará las siguientes posibles condiciones:
    1. Dirección IP original accesible: la dirección IP original asignada a la interfaz en el servidor activo está accesible para datos y servicios.
    2. Dirección IP original no accesible: la dirección IP original asignada a la interfaz en el servidor pasivo no está accesible.
    3. IP de conexión Heartbeat, no accesible desde la intranet: la dirección IP original asignada automáticamente a la conexión Heartbeat no está accesible para datos y servicios.

iSCSI LUN*

iSCSI es un servicio de conexión a red de almacenamiento que facilita la transferencia de datos a través de redes locales o largas distancias. Asignando iSCSI LUN a iSCSI Target, los clientes pueden acceder a los espacios de almacenamiento como si fueran discos locales. Un iSCSI LUN (número de unidad lógica) se puede asignar a uno o varios iSCSI Target para proporcionar servicios de almacenamiento iSCSI. Vaya a Administrador de almacenamiento > iSCSI LUN para administrar los iSCSI LUN.

Nota:

El número de iSCSI LUN compatibles varía según el modelo de DiskStation. Para obtener las especificaciones detalladas del producto, visite www.synology.com.

DiskStation actualmente es compatible con los siguientes tipos de iSCSI LUN:

  • iSCSI LUN (archivos normales): el iSCSI LUN se crea designando una porción del espacio de volumen para su uso. Puede ampliar su tamaño más tarde cuando sea necesario. Este tipo de iSCSI LUN ofrece la flexibilidad de una administración de capacidad dinámica con Thin Provisioning, compatibilidad con la aceleración de hardware VMware VAAI, compatibilidad con Windows ODX, instantánea de LUN y clon de LUN.
  • iSCSI LUN (nivel de bloques): LUN único en RAID: el iSCSI LUN se crea utilizando todo el espacio de un volumen. Puede repararlo o expandirlo, o bien cambiar su tipo de RAID, igual que usar un volumen. Consulte la página Volumen para obtener más información.
  • iSCSI LUN (nivel de bloques): LUN múltiple en RAID: el iSCSI LUN se crea designando un espacio específico de un Grupo de discos. Puede ampliar el iSCSI LUN más tarde si hay espacio disponible en el Grupo de discos. Consulte la página Grupo de discos para obtener más información.
  • iSCSI LUN (nivel de bloques): este tipo de iSCSI LUN está creado en un RAID Group y ofrece la flexibilidad de una administración de capacidad dinámica además de un rendimiento de acceso optimizado. Este tipo de iSCSI LUN se puede ampliar más adelante usando espacio sin asignar en el RAID Group. Consulte la página RAID Group para obtener más información.

Nota:

Los iSCSI LUN con características de LUN avanzadas solo están disponibles en el sistema de archivos ext4/Btrfs.

Para crear un iSCSI LUN:

  1. Haga clic en Crear.
  2. Seleccione el tipo de iSCSI LUN que desee crear.
  3. Siga el asistente para completar el proceso.

Nota:

Las características de LUN avanzadas solo se pueden habilitar durante la creación inicial de iSCSI LUN. Este ajuste no se puede modificar más adelante.

El tamaño de la unidad de asignación de LUN avanzada se puede optimizar para ciertas características del servidor de aplicaciones, como por ejemplo Microsoft Exchange o SQL Server, o servidores de visualización y tecnologías de aceleración de almacenamiento, como Microsoft Windows ODX o VMware VAAI, haciendo una de las siguientes acciones:

  • Configure el tamaño de la unidad de asignación durante la creación del primer LUN avanzado de un volumen.
  • Configure el tamaño de la unidad de asignación en Configurar antes de crear el primer LUN avanzado de un volumen.

Le recomendamos que especifique el tamaño de la unidad de asignación para su aplicación cuando utilice la conexión directa desde su servidor de aplicaciones o métodos RDM (pass-through) para trabajar con iSCSI LUN avanzados en DiskStation. Si desea aprovechar la tecnología de LUN avanzados de Synology para realizar la clonación en Windows ODX, le recomendamos formatear el volumen NTFS con el tamaño de la unidad de asignación igual (o mayor que) el que ha especificado en DSM.

Nota:

La unidad de asignación de LUN avanzado de cada volumen se puede configurar solo una vez.

Para quitar un iSCSI LUN:

Al quitar un iSCSI LUN, se eliminarán todos los datos del iSCSI LUN. También se perderán las conexiones entre el iSCSI LUN y cualquier iSCSI Target asignado.

  1. Seleccione el iSCSI LUN que desea quitar.
  2. Haga clic en el botón Quitar y siga el asistente para completar el proceso.

Nota:

Al borrar VAAI LUN de DiskStation, no se libera espacio de disco inmediatamente. Los datos se trasladan a una carpeta reservada del sistema llamada @EP_trash y se eliminan uno por uno. Por ejemplo, si se elimina un VAAI LUN de 100 GB de DiskStation, se puede tardar unos 15-20 minutos en liberar los 100 GB de espacio en el volumen del disco.

Para realizar un montaje forzoso de iSCSI LUN (nivel de bloques):

Si falta la caché en un iSCSI LUN (nivel de bloques), el sistema no montará el iSCSI LUN (nivel de bloques) al arrancar. Si no se puede recuperar la caché y desea acceder a iSCSI LUN (nivel de bloques), puede hacer clic en Montaje forzoso.

Nota:

Esta función solo está disponible en determinados modelos.

Para editar un iSCSI LUN:

  1. Seleccione el iSCSI LUN que desea editar.
  2. Haga clic en Editar.
  3. Edite las propiedades que desee.
  4. Haga clic en OK para guardar la configuración.

Nota:

Para evitar la pérdida de datos al editar la capacidad de iSCSI LUN, solo podrá hacerla mayor del tamaño actual.

Para tomar una instantánea de iSCSI LUN:

Una instantánea de un iSCSI LUN es una copia puntual de solo lectura de un iSCSI LUN.

  1. Seleccione el iSCSI LUN del que desea sacar instantáneas.
  2. Haga clic en Instantánea y elija Tomar una instantánea.
  3. Introduzca una descripción para esta instantánea.
  4. Elija uno de los siguientes tipos de instantáneas de acuerdo con sus necesidades:
    • Consistente con aplicaciones: este tipo de instantánea solo se puede tomar cuando se ha instalado Synology Snapshot Manager en su servidor Windows o VMware vCenter Server, según su entorno operativo para comunicarse con DSM. Cuando se activan instantáneas consistentes con aplicaciones en DSM, las aplicaciones recibirán una notificación y pasarán todos los datos de la memoria al LUN en el que se toma la instantánea para garantizar la consistencia de los mismos.
    • Consistente con averías: si no se ha instalado Synology Snapshot Manager en ninguno de sus clientes, solo podrá capturar instantáneas consistentes con averías de los LUN. Cuando se activan, las aplicaciones no recibirán notificaciones para pasar los datos de la memoria al LUN y por tanto no se garantizará la consistencia de los datos.
  5. Haga clic en OK.

Para restaurar un iSCSI LUN:

Seleccione una instantánea puntual para restaurar los datos de un iSCSI LUN.

  1. Seleccione el iSCSI LUN que desea restaurar.
  2. Haga clic en Instantánea y elija Administrador de instantáneas.
  3. Seleccione la instantánea que desea restaurar y haga clic en Restaurar.

Para clonar un iSCSI LUN o una instantánea:

Realice una copia grabable de un iSCSI LUN o de una instantánea de iSCSI LUN.

  • Seleccione un iSCSI LUN de la lista y haga clic en Clonar.
  • Seleccione una instantánea de iSCSI LUN de Snapshot Manager y haga clic en Clonar.

Nota:

  • En determinados modelos, deberá ir a Centro de paquetes e instalar Instantánea y replicación para funciones como la restauración de iSCSI LUN, o la realización y clonación de instantáneas de iSCSI LUN.
  • Si hay al menos una sesión iSCSI conectada asociada a este LUN y Windows Server o VMware ESXi Server la utilizan, recomendamos instalar el complemento Synology Snapshot Manager para evitar posibles inconsistencias de datos, de lo contrario deberá detener todas las aplicaciones y cerrar estas sesiones desde los iniciadores antes de proceder.
  • Las funciones de instantánea, restauración y clonación de LUN solo están disponibles en iSCSI LUN con características de LUN avanzadas.
  • Si ha tomado una instantánea o ha creado un clon, y el estado ha pasado a ser Anormal/Averiado, elimine dicha instantánea/clon y cree otra/o.
  • Todos los cambios realizados en un LUN hasta después de crear una instantánea se perderán si restaura el LUN en esta instantánea.
  • Si el estado de un iSCSI LUN cambia a No disponible, significa que se han producido algunos errores relacionados con el sistema al acceder a iSCSI LUN. Para evitar una grave pérdida de datos en su sistema, este iSCSI LUN no estará accesible temporalmente. Póngase en contacto con Synology para solicitar asistencia lo antes posible.

Glosario:

  • Thin Provisioning: Thin Provisioning es un método que permite optimizar el uso del almacenamiento asignando espacio de almacenamiento de modo dinámico y bajo demanda. Esta configuración solo está disponible para iSCSI LUN (archivos normales).
  • VMware VAAI: las VMware vStorage APIs for Array Integration (VAAI) pueden liberar la carga de las operaciones estándar en matrices de almacenamiento virtual trasladándola al dispositivo iSCSI en el entorno VMware vSphere y optimizar así el rendimiento de almacenamiento.
  • ODX de Windows: una nueva tecnología de transferencia de datos desarrollada por Windows en Windows Server 2012 y 8 llamada Offload Data Transfer (ODX). ODX mejora la transferencia de datos descargando parte de la carga en la matriz de almacenamiento.
    Si el origen y el destino de datos están situados en los LUN situados en el mismo volumen, la clonación se procesará mediante la tecnología Synology Advanced LUN, que ahorrará hasta un 99,9 % del consumo de espacio de almacenamiento y acelerará la clonación al mismo tiempo.
  • Asignación de iSCSI Target: asigna el iSCSI LUN a uno o varios iSCSI Target.
  • Synology Snapshot Manager: Synology Snapshot Manager es un complemento de software que permite tomar instantáneas consistentes con las aplicaciones directamente desde DiskStation Manager (DSM).
    • En el entorno VMware, necesitará instalar Synology Snapshot Manager en el servidor Windows donde esté instalado VMware vCenter Server. Cuando se active una instantánea en DSM, VMware vCenter Server recibirá una notificación y pasará todos los datos de la memoria al LUN en el que se toma la instantánea para garantizar la consistencia de los mismos. Cuando se haya completado la instantánea, VMware vCenter Server continuará con el funcionamiento E/S normal del almacén de datos VMware. Synology Snapshot Manager for VMware vCenter Server es compatible con vSphere 5.1 o versiones más recientes.
    • En entornos Windows, necesitará instalar Synology Snapshot Manager en su servidor Windows. Cuando se active una instantánea en DSM, Synology Snapshot Manager aprovechará la tecnología de Microsoft Volume Shadow Copy Service (VSS) para producir copias puntuales de datos consistentes. Synology Snapshot Manager for Windows es compatible con Windows Server 2008 R2, 2012 y 2012 R2.
    • Synology Snapshot Manager se puede descargar de manera gratuita desde www.synology.com.

Conexión remota

File Station le permite conectarse a servicios de nube pública (Box, Dropbox, Google Drive y Microsoft OneDrive) y servidores de archivos a través de varios protocolos diferentes (FTP, STFP, WebDAV y WebDAV HTTPS).

Para conectar a un servicio de nube:

  1. Vaya a Herramientas > Conexión remota > Configuración de conexión.
  2. Para seleccionar el servicio de nube que prefiera, haga clic en el icono y haga clic en Siguiente.
  3. En la ventana emergente, siga las instrucciones del asistente para autenticar la cuenta.
  4. Cuando se haya completado la configuración, el servicio de nube de destino aparecerá en el panel de la izquierda.

Limitaciones específicas de cada nube:

  • Box:
    • Un catálogo compartido con otros colaboradores recibirá un sufijo con el nombre de usuario del propietario.
    • Si los catálogos compartidos con otros colaboradores utilizan un nombre idéntico (un nombre de catálogo con un sufijo con el nombre de usuario del propietario), solo se mostrará uno de dichos catálogos.
    • Los archivos con las siguientes extensiones solo se podrán abrir a través de Box: .gdoc, .gsheet y .boxnote.
    • Los nombres de archivos distinguen entre mayúsculas y minúsculas.
    • Para obtener la configuración correcta del cortafuegos consulte este artículo.
  • Dropbox:
    • Un catálogo de Dropbox no se puede mostrar si contiene más de 25 000 archivos.
    • Un catálogo de Dropbox no se puede renombrar si contiene más de 10 000 archivos.
    • No se pueden cargar archivos de más de 10 GB.
    • Una tarea de carga no se completará correctamente si el archivo necesita más de 24 horas para cargarse.
    • Los nombres de archivos distinguen entre mayúsculas y minúsculas.
    • Para obtener la configuración correcta del cortafuegos consulte este artículo.
  • Google Drive:
    • Los archivos/carpetas con una ruta superior a 2048 caracteres no se mostrarán.
    • Los archivos con un nombre superior a 255 caracteres no se mostrarán.
    • Los archivos/catálogos con nombres idénticos se marcarán con horas de creación individuales.
    • Si abre un archivo específico de Google (por ejemplo, Google Docs y Google Slides) a través de File Station, se mostrará una URL para que acceda al archivo de destino.
    • Las carpetas My Drive y Starred que se muestran en File Station corresponden respectivamente a "Mi unidad" y "Destacados" en Google Drive.
    • Para obtener la configuración correcta del cortafuegos consulte este artículo.
  • Microsoft OneDrive:
    • Los archivos del bloc de notas OneNote no se mostrarán.
    • No se pueden cargar archivos de más de 10 GB.
    • Los archivos (por ejemplo, .png, .jpg, .docx, .PDF) pueden mostrar tamaño de archivo mayores (tamaños originales + tamaños de miniaturas). Esto no afectará a los tamaños de los archivos originales cuando se descarguen.
    • Los nombres de archivos distinguen entre mayúsculas y minúsculas.

Para obtener información acerca de otras limitaciones, le sugerimos que visite el sitio web oficial del servicio de nube correspondiente.

Para conectar a un servidor de archivos:

  1. Vaya a Herramientas > Conexión remota > Configuración de conexión.
  2. Para elegir el protocolo de servidor deseado, haga clic en el icono y haga clic en Siguiente.
  3. Proporcione la siguiente información para completar la configuración:
    • Nombre de host o IP: escriba el nombre de host o IP del servidor de archivos de destino.
    • Puerto: escriba el número de puerto del servidor de archivos de destino.
    • Usuario y Contraseña: introduzca esta información de inicio de sesión si el servidor de archivos de destino requiere cuenta/contraseña.
    • Página de código: seleccione la codificación de caracteres del servidor de archivos de destino para mostrar los nombres de los archivos correctamente.
    • Nombre del perfil: escriba un nombre único para identificar este servidor de archivos fácilmente.
    • Utilizar conexión WebDAV HTTPS: esta opción está disponible solo cuando configura un protocolo WebDAV y tanto el cliente como el servidor de archivos son compatibles con WebDAV HTTPS.
    • Permitir solo una conexión por usuario: esta opción está disponible solo cuando configura un protocolo FTP o SFTP. Marque esta opción para optimizar la conectividad cuando el servidor tiene limitaciones en cuanto al número de conexiones.
  4. Haga clic en Aplicar para guardar la configuración. El servidor de archivos de destino aparecerá en el panel de la izquierda.

Para modificar el perfil del servidor de archivos o del servicio de nube existentes:

  1. Vaya a Herramientas > Conexión remota > Lista de conexiones.
  2. Haga clic en el servicio de nube o servidor de archivos de destino para administrar lo siguiente:
    • Edición: haga clic en Editar, cambie la configuración actual del perfil y haga clic en OK.
    • Desconexión: haga clic en Desconectar para finalizar la conexión.
    • Reconexión: haga clic en Conectar para recuperar la conexión (también puede hacer clic en el destino del panel de la izquierda en File Station para reconectar).
    • Eliminación: haga clic en Eliminar y a continuación confirme para abandonar el perfil (también puede hacer clic en el destino del panel de la izquierda en File Station y hacer clic en Eliminar para borrarlo).

Nota:

  • Cada usuario puede conectar simultáneamente a un máximo de 100 servicios de nube y servidores de archivos.
  • Asegúrese de seleccionar la página de códigos del servidor correcto para evitar errores inesperados. Se recomienda Unicode si no conoce la página de códigos del servidor.
  • Algunas funciones de File Station (por ejemplo, Comprimir a, Extraer, Vista previa, etc.) no se pueden aplicar a archivos almacenados en servicios en la nube o servidores de archivos.
  • Las velocidades de transferencia individuales para cada servicio de nube o servidor de archivos conectado están sujetas a las configuraciones de límite de velocidad de los usuarios.
  • Cada usuario está limitado a tres transferencias de archivos simultáneas a través de todos los servicios de nube y servidores de archivos conectados.
  • Para configurar los parámetros del proxy de WebDAV, WebDAV HTTPS y del servicio de nube vaya a Panel de control > Red > General > Proxy. Si ha cambiado la configuración del proxy durante la conexión al servicio de nube o al servidor de archivos, desconéctese y vuélvase a conectar.
  • Si modifica la información del servicio de nube o los servidores de archivos conectados el estado de conexión no se verá afectado.
  • Para liberar recursos de conexión, un servicio de nube/servidor de archivos se desconectará automáticamente si los archivos que contiene permanecen inalterados durante más de 30 minutos. Haga clic en el servicio de nube/servidor de archivos para recuperar conexión.
  • El sistema no puede mostrar archivos/carpetas que adoptan los siguientes formatos de nomenclatura:
    • Comienzan por ._ o .SYNOPPSDB
    • Incluyen /
    • Nombrados exactamente como .    ..    '    "    .DS_Store    @eaDir    @sharebin    @tmp    .SynologyWorkingDirectory
  • La conexión remota solo es compatible en el modo con privilegios altos. Para usar esta función vaya al paquete Docker de su DSM host, seleccione el Docker DSM para el que desea habilitar las carpetas compartidas cifradas, haga clic en Editar y marque la casilla junto a Ejecutar contenedor usando privilegios altos.

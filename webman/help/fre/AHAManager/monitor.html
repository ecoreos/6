Surveiller

Cette page comporte trois onglets qui affichent la surveillance de l'état du système, du service et de l'interface réseau. Cette page vous permet de surveiller certaines cibles. Si une cible surveillée est défaillante sur le serveur actif, le système va basculer sur le serveur en standby passif (à condition que le serveur passif fonctionne normalement). Les onglets suivants apparaissent sur cette page :

Onglet État du système

L'état du système est surveillé par défaut et ne peut pas être désactivé. Un basculement est déclenché lorsque des pannes système surviennent sur le serveur actif. Les sections suivantes apparaissent dans cet onglet :

  • État du serveur actif : Basculement en cas de panne du serveur actif pour fournir les services. Par exemple, en cas de panne des deux alimentations du serveur actif.
  • Connectivité du stockage : Basculement lors d'une situation dans laquelle un disque est absent sur le serveur actif mais présent sur le serveur passif.

Onglet Service

Affiche les services qui peuvent faire l'objet d'une surveillance ainsi que leur état. Le basculement est déclenché lorsque les services sont anormaux. Les sections suivantes apparaissent dans cet onglet :

  • Service : Affiche les services qui peuvent être surveillés.
  • État : Indique si les services fonctionnent ou non.
  • Surveiller : Cochez la case à côté d'un service pour commencer à le surveiller. Le système va basculer en cas de panne du service.

Afin de surveiller un service :

  1. Trouver le service que vous désirez surveiller. Assurez-vous que l'état du service est normal.
  2. Cochez la case à côté de celui-ci.
  3. Cliquez sur Appliquer pour enregistrer les paramètres.

Remarque :

Les service qui peuvent être surveillés sont les suivants : Windows File Service, iSCSI Target, FTP, Mac File Service, NFS Service.

Onglet Interface réseau

Affiche toutes les interfaces réseaux logiques qui peuvent faire l'objet d'une surveillance. Le basculement est déclenché lorsque la connexion réseau est anormale. (Reportez-vous aux Remarques suivantes.) Les sections suivantes apparaissent dans cet onglet :

  • Interface réseau : Affiche les interfaces qui peuvent faire l'objet d'une surveillance.
  • Surveiller : Cochez la case à côté d'une interface pour commencer à la surveiller.

Remarque :

Si la case Surveiller est cochée, le système va basculer vers le serveur passif lorsque la connexion réseau du serveur actif est défaillante, et que la connexion réseau du serveur passif fonctionne encore normalement.

  • Le basculement ne sera déclenché que si le serveur passif possède des connexion plus saines. Par exemple, si le serveur actif possède trois connexions et que l'une est défaillante, le système ne déclenchera déclencher un basculement que si le serveur passif dispose de trois connexions en meilleur état.
  • Les connexions combinées avec Link Aggregation comptent comme une seule connexion. Par exemple, si deux connexion sont combinées, et que l'une d'elle est défaillante mais que l'autre fonctionne toujours, la connexion est toujours considérée comme en bon état.

Pour surveiller une interface :

  1. Trouvez l'interface que vous désirez surveiller. Assurez-vous que les câbles sont correctement connectés aux interfaces sur les serveurs actif et passif.
  2. Cochez la case à côté de celui-ci.
  3. Cliquez sur Appliquer pour enregistrer les paramètres.

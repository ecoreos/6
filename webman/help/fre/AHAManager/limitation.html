Limites et configuration système requise

La mise en place de la technologie Synology High Availability nécessite deux serveurs Synology identiques agissant en tant que serveur actif et serveur passif. Avant que les deux serveurs Synology ne soient combinés dans un cluster high-availability, le système vérifie les conditions matérielles et logicielles suivantes afin de s'assurer qu'un cluster high-availability peut être créé.

Configuration système requise

Veuillez noter les consignes suivantes concernant le logiciel et le matériel :

  • Serveur Synology : Le serveur actif et le serveur passif doivent être des modèles identiques.
  • Unité(s) d'expansion : Les serveurs actif et passif doivent se connecter correctement aux mêmes unités d'expansion.
  • Nombre et type de disques : Les disques doivent être uniquement du type SAS. Avant l'installation du DSM, vous devez installer au moins 3 disques dans les unités d'expansion.
  • Périphérique externe USB : Seul le stockage USB est pris en charge. Les autres périphériques externes USB (par ex. onduleur, dongle Wi-Fi, haut-parleur, imprimante, dongle Bluetooth et dongle DVB) ne sont pas pris en charge. Le stockage USB ne peut être utilisé que sur le serveur actif et ne peut pas être basculé sur le serveur passif.

Environnement réseau

Veuillez prendre note des directives suivantes concernant la configuration du réseau lorsque que vous tentez de former un cluster high-availability.

  • Paramètres réseau : La connexion Heartbeat entre les deux serveurs doit être correctement configurée. Les deux serveurs doivent appartenir au même sous-réseau et il est nécessaire de configurer des adresses IP statiques après l'installation.
  • Nombres de ports de réseau local : Les deux serveurs doivent disposer des mêmes nombres de ports de réseau local. Si l'un de vos modèles DiskStation est équipé d'interfaces de carte réseau supplémentaires, l'autre doit être équipé des mêmes interfaces de carte réseau.
  • Limites de connexion : Le réseau Wi-Fi n'est pas pris en charge par le cluster high-availability.

Remarque :

  • Le serveur NTP est automatiquement activé sur le serveur actif.

Limitations du paquet et du logiciel

Veuillez prendre connaissance des limitations liées au paquet et au logiciel ci-dessous :

  • Le cluster high-availability ne prend pas en charge les paquets ni les fonctionnalités suivantes :
    • Serveur DVBLink
    • La propriété pour redémarrer/arrêter le cluster high-availability dans CMS
    • Time Backup
  • Il est impossible de configurer un cluster high-availability comme ou utilisé en association avec un cluster high-availability basé sur la réplication.

Unités d'expansion spécifiques

Les serveurs actif et passif doivent être connectés aux unités d'expansion spécifiques. Il existe deux modules sur chaque unité d'expansion.

Pour configurer/ajouter des unités d'expansion :

  • Un module comporte deux port ; le port d'entrée se trouve à gauche et le port de sortie à droite. Connectez un câble SAS entre le port de sortie d'un module et le port d'entrée du module de l'unité d'expansion suivante.
  • Les unités d'expansion doivent être connectées en série avec les modules sur les côtés correspondants pour chaque serveur.
  • Les serveurs actif et passif doivent être connectés aux unités d'expansion partagées dans le même ordre.

Remarques :

Pour des informations détaillées relatives à l'installation des unités d'expansion, n'oubliez pas de consulter le guide d'installation rapide de vos serveurs et de vos unités d'expansion.

iSCSI LUN*

iSCSI est un service de stockage en réseau qui facilite le transfert de données sur des réseaux locaux ou de longues distances. En mappant des iSCSI LUN vers des iSCSI Target, les clients peuvent accéder aux espaces de stockage comme s'il s'agissait de disques locaux. Un iSCSI LUN (logical unit number) peut être mappé vers un ou plusieurs iSCSI Target pour fournir des services de stockage iSCSI. Allez dans Gestionnaire de stockage > iSCSI LUN pour gérer les iSCSI LUN.

Remarque :

Le nombre d'iSCSI LUN pris en charge varie selon les modèles de DiskStation. Pour des spécifications produit détaillées, veuillez consulter www.synology.com.

DiskStation prend actuellement en charge les types d'iSCSI LUN suivants :

  • iSCSI LUN (fichiers courants) : L'iSCSI LUN est créé en désignant une portion de l'espace du volume pour son usage. Vous pouvez étendre sa taille plus tard si nécessaire. Ce type d'iSCSI LUN offre une gestion de la capacité dynamique et flexible avec Thin Provisioning, la prise en charge de l'accélération matérielle VAAI VMware, la compatibilité Windows ODX, l'instantané du LUN et clonage du LUN.
  • iSCSI LUN (niveau bloc) - LUN unique sur RAID : L'iSCSI LUN est créé en utilisant tout l'espace d'un volume. Vous pouvez le réparer ou l'étendre, ou changer son type de RAID, comme pour un volume. Reportez-vous à la page Volume pour plus d'informations.
  • iSCSI LUN (niveau bloc) - LUN multiples sur RAID : L'iSCSI LUN est créé en désignant un espace spécifique d'un groupe de disques. Vous pouvez étendre l'iSCSI LUN ultérieurement si de l'espace s'avère disponible sur le groupe de disques. Reportez-vous à la page Groupe de disques pour plus d'informations.
  • iSCSI LUN (niveau bloc) Ce type d'iSCSI LUN est créé sur un RAID Group et offre une souplesse de gestion de la capacité dynamique, en plus de performances d'accès optimisées. Vous pouvez étendre ce type d'iSCSI LUN ultérieurement en utilisant de l'espace non alloué sur le RAID Group. Veuillez consulter la page Groupe RAID pour plus d'informations.

Remarque :

Les iSCSI LUN avec fonctions LUN avancées ne sont disponibles que sur le système de fichiers ext4/Btrfs.

Pour créer un iSCSI LUN :

  1. Cliquez sur Créer.
  2. Sélectionnez le type d'iSCSI LUN que vous souhaitez créer.
  3. Suivez l'assistant pour terminer le processus.

Remarque :

Les fonctions LUN avancées ne peuvent être activées que pendant la création initiale des iSCSI LUN. Ce paramètre ne peut être modifié par la suite.

La taille de l'unité d'allocation LUN avancée peut être optimisée pour certaines fonctions pour le serveur d'application tel que Microsoft Exchange ou SQL Server ou les serveurs de virtualisation et les technologies d'accélération du stockage telles que Microsoft Windows ODX ou VMware VAAI en réalisant l'une des actions suivantes :

  • Paramétrez la taille de l'unité d'allocation pendant la création du premier LUN avancé d'un volume.
  • Paramétrez la taille de l'unité d'allocation à partir de Configurer avant de créer le premier LUN avancé d'un volume.

Nous vous recommandons de spécifier la taille de l'unité d'allocation pour votre application en utilisant directement la connexion directe de votre serveur d'application ou des méthodes (pass-through) RDM de travail avec les iSCSI LUN avancés sur DiskStation. Pour tirer parti de la technologie Synology LUN avancée pour le clonage pour Windows ODX, nous vous suggérons de formater le volume NTFS avec l'unité d'allocation de taille égale (ou supérieure à) celle que vous avez spécifiée sur DSM.

Remarque :

L'unité d'allocation LUN avancée de chaque volume peut être configurée une seule fois.

Pour supprimer un iSCSI LUN :

Lors de la suppression d'un iSCSI LUN, toutes les données qui s'y trouvent sont supprimées. Les connexions entre l'iSCSI LUN et les iSCSI Target éventuellement mappées seront également perdues.

  1. Sélectionnez l'iSCSI LUN que vous souhaitez supprimer.
  2. Cliquez sur le bouton Supprimer et suivez l'assistant pour terminer le processus.

Remarque :

Supprimer les VAAI LUN sur le DiskStation ne libère pas de l'espace disque immédiatement. Les données sont déplacées dans un dossier réservé du système appelé @EP_trash et supprimées morceau par morceau. Par exemple si un VAAI LUN de 100 Go est supprimé du DiskStation, cela peut prendre environ 15-20 minutes pour libérer l'espace de 100 Go sur le volume du disque.

Pour forcer un iSCSI LUN (niveau bloc) à monter :

Si le cache est absent sur un iSCSI LUN (niveau bloc), le système ne montera pas le iSCSI LUN (niveau bloc) lors du démarrage. Si le cache n'est pas récupérable, et que vous voulez accéder au iSCSI LUN (niveau bloc), vous pouvez cliquer sur Montage en force.

Remarque :

Cette fonction n'est disponible que sur certains modèles.

Pour modifier un iSCSI LUN :

  1. Sélectionnez l'iSCSI LUN que vous souhaitez modifier.
  2. Cliquez sur Modifier.
  3. Modifiez les priorités désirées.
  4. Cliquez sur OK pour enregistrer les paramètres.

Remarque :

Pour éviter toute perte de données lors de la modification de la capacité d'un iSCSI LUN, vous ne pouvez qu'augmenter sa taille.

Pour prendre l'instantané d'un iSCSI LUN :

Un instantané d'un iSCSI LUN est une copie en lecture seule à un instant donné d'un iSCSI LUN.

  1. Sélectionnez l'iSCSI LUN duquel vous souhaitez prendre des instantanés.
  2. Cliquez sur Instantané et choisissez Prendre un instantané.
  3. Saisissez la description de cet instantané.
  4. Choisissez l'un des types d'instantané suivants selon vos besoins :
    • Cohérent avec l'application : Ce type d'instantané ne peut être pris que lorsque Synology Snapshot Manager est installé sur votre serveur Windows ou sur VMware vCenter Server selon votre environnement pour communiquer avec DSM. Lorsqu'un instantané cohérent pour l'application est déclenché sur DSM, les applications sont notifiées et transfèrent toutes les données depuis la mémoire vers le LUN sur lequel l'instantané est pris pour garantir la cohérence des données.
    • Cohérent avec le plantage : Si aucun Synology Snapshot Manager n'est installé sur l'un de vos clients, vous ne pouvez capturer que des instantanés cohérents avec le plantage des LUN. Lorsqu'ils sont déclenchés, les applications ne sont pas notifiées de transférer les données depuis la mémoire sur les LUN et par conséquent cela ne garantit aucune cohérence des données.
  5. Cliquez sur OK.

Pour restaurer un iSCSI LUN :

Sélectionnez un instantané d'un point dans le temps pour restaurer les données sur un iSCSI LUN.

  1. Sélectionnez l'iSCSI LUN que vous souhaitez restaurer.
  2. Cliquez sur Instantané et choisissez Gestionnaire d'instantané.
  3. Sélectionnez l'instantané que vous souhaitez restaurer et cliquez sur Restaurer.

Pour cloner un iSCSI LUN ou un instantané :

Effectuez une copie accessible en écriture d'un iSCSI LUN ou un instantané d'un iSCSI LUN.

  • Sélectionnez un iSCSI LUN dans la liste et cliquez sur Cloner.
  • Sélectionnez un instantané iSCSI LUN dans le Gestionnaire d'instantané et cliquez sur Cloner.

Remarque :

  • Sur certains modèles, vous devez accéder à Centre de paquets et installer Instantané et réplication pour les fonctions qui comprennent la restauration de iSCSI LUN, qui prennent et qui clonent des instantanés de iSCSI LUN.
  • S'il y a au moins une séance iSCSI associée avec le LUN et qui est utilisée par Windows Server ou VMware ESXi Server, nous recommandons d'installer le plug-in Synology Snapshot Manager pour éviter des incohérences potentielles de données, autrement vous devez arrêter toutes les applications et déconnecter les sessions de ces initiateurs avant de continuer.
  • Les fonctions d'instantané/restauration et de clonage du LUN ne sont disponibles que sur les iSCSI LUN dotés de fonctions LUN avancées.
  • Si vous avez pris un instantané ou créé un clone, et que le statut est devenu Anormal/En panne, veuillez supprimer cet instantané/clone et en créer un autre.
  • Toutes les modifications apportées à un LUN après la création d'un instantané seront perdues lorsque vous restaurez le LUN à cet instantané.
  • Si le statut d'un iSCSI LUN devient Indisponible, cela signifie que des erreurs liées aux données se sont produites lors de l'accès à l'iSCSI LUN. Afin d'éviter les graves pertes de données sur votre système, cet iSCSI LUN sera temporairement inaccessible. Veuillez contacter Synology pour obtenir une assistance dès que possible.

Glossaire :

  • Thin Provisioning : Le Thin Provisioning est une méthode permettant d'optimiser l'utilisation de l'espace de stockage en allouant de l'espace de manière dynamique et « à la demande ». Ce paramètre n'est disponible qu'avec un iSCSI LUN (fichiers courants).
  • VMware VAAI : Les VAAI (API for Array Integration) VMware vStorage peuvent décharger le travail des opérations standards sur les matrices de stockage virtuel vers le périphérique iSCSI dans l'environnement VMware vSphere et optimiser les performances de stockage.
  • Windows ODX : Une nouvelle technologie de transfert des données développée par Windows Server 2012 et 8 appelée Offload Data Transfer (ODX). ODX améliore les performances de transfert en déchargeant une partie de la charge de travail vers les matrices de stockage.
    Si les source et destination des données se trouvent sur des LUN situés sur le même volume, la technologie LUN avancée de Synology procède au clonage, ce qui économise 99,9 % de votre consommation d'espace de stockage et accélère le clonage en même temps.
  • Mappage d'iSCSI Target : Mapper l'iSCSI LUN à un ou plusieurs iSCSI Targets.
  • Synology Snapshot Manager : Synology Snapshot Manager est un plug-in logiciel vous permettant de prendre des instantanés cohérents avec l'application directement depuis DiskStation Manager (DSM).
    • Pour un environnement VMware, vous devez installer Synology Snapshot Manager sur les serveurs Windows où VMware vCenter Server est installé. Lorsqu'un instantané est déclenché sur DSM, VMware vCenter Server est notifié et transfère toutes les données depuis la mémoire vers le LUN sur lequel l'instantané est pris pour garantir la cohérence des données. Une fois que l'instantané est complet, VMware vCenter Server reprendre les opérations d'E/S normales de l'entrepôt de données VMware. Synology Snapshot Manager for VMware vCenter Server prend en charge vSphere 5.1 ou les versions ultérieures.
    • Pour un environnement Windows, vous devez installer Synology Snapshot Manager sur votre serveur Windows. Lorsqu'un instantané est déclenché sur DSM, Synology Snapshot Manager s'appuie sur la technologie Microsoft Volume Shadow Copy Service (VSS) pour produire des copies de données « instantanées » cohérentes. Synology Snapshot Manager for Windows prend en charge Windows Server 2008 R2, 2012 et 2012 R2.
    • Vous pouvez télécharger gratuitement Synology Snapshot Manager depuis www.synology.com.

iSCSI Target*

iSCSI est un type de réseau de stockage (SAN) qui donne accès au stockage de données consolidé, au niveau des blocs. Il facilite les transferts de données sur les intranets et la gestion du stockage sur de longues distances. En mappant des iSCSI LUN vers des iSCSI Target, les serveurs clients peuvent accéder aux espaces de stockage par son initiateur iSCSI comme s'ils étaient des disques locaux. Selon le modèle, le produit Synology prend en charge la création de multiples iSCSI Targets sur lesquels les iSCSI LUN peuvent être mappés.

Remarque : Le nombre d'iSCSI Targets pris en charge varie selon les produits Synology. Pour des caractéristiques techniques détaillées, veuillez consulter le site Web Synology.

Pour créer un iSCSI Target :

  1. Cliquez sur le bouton Créer.
  2. Entrez les informations de l'iSCSI Target, notamment le Nom, l'IQN, et s'il faut activer l'Authentification CHAP.
  3. Cliquez sur Suivant.
  4. Configurer les propriétés d'iSCSI LUN. Sélectionnez Créer un nouveau iSCSI LUN ou Mapper les iSCSI LUN existants.
  5. Cliquez sur Suivant.
  6. Cliquez sur Retour pour modifier les paramètres ou sur Appliquer pour les confirmer.

Explication des termes

  • IQN : IQN (iSCSI Qualified Name) est un ensemble de noms uniques pour un iSCSI Target. Il ne devrait pas provoquer de conflits avec d'autres IQN d'iSCSI Target sur d'autres serveurs. L'IQN doit comporter moins de 128 caractères et peut inclure des lettres (sensible à la casse), des numéros, des double-points, des tirets et des points. Le format de nomination suggéré est iqn.[date].[domaine]:[identifiant périphérique], par exemple : iqn.2000-04.com.synology:storage.diskarray.1.
  • Authentification CHAP : Si vous sélectionnez CHAP, le serveur pourra vérifier l'identité du client. Le nom d'utilisateur peut comporter 1 à 12 lettres et numéros, et le mot de passe peut compter 12 à 16 lettres et numéros.
  • Authentification CHAP mutuel : Dans CHAP mutuel, le client et le serveur s'authentifieront l'un l'autre. Tout mot de passe CHAP utilisé pour l'authentification de client ne doit pas être configuré pour l'authentification d'un serveur, et tout mot de passe CHAP utilisé pour l'authentification de serveur ne doit pas être configuré pour l'authentification d'un client.

Pour supprimer un iSCSI Target :

Vous pouvez supprimer un iSCSI Target ainsi que tous les iSCSI LUN mappés. Notez qu'en supprimant un iSCSI LUN, toutes les données qui s'y trouvent seront supprimées.

  1. Sélectionnez l'iSCSI Target que vous souhaitez supprimer.
  2. Cliquez sur Supprimer.
  3. Si vous voulez supprimer des iSCSI LUN mappés, sélectionnez Supprimer également les iSCSI LUN mappés et sélectionnez ceux que vous voulez supprimer.
  4. Cliquez sur OK.

Pour modifier un iSCSI Target :

  1. Sélectionnez l'iSCSI Target que vous souhaitez modifier.
  2. Cliquez sur Modifier.
  3. Modifiez les paramètres à votre convenance. Cliquez sur OK pour enregistrer les modifications.

Explication des termes

  • Somme de contrôle CRC : Activez la somme de contrôle CRC pour vérifier si une erreur s'est produite au cours de la transmission de données.
  • Sessions multiples : Autorisez plusieurs sessions d'un ou plusieurs initiateurs à se connecter au même iSCSI Target en même temps. Pour protéger vos fichiers contre toute altération ou perte de données, nous vous conseillons fortement d'utiliser un système de fichiers partagés ou un système à clusters tel que Virtual Machine File System (VMFS) de VMware et Oracle Cluster File System (OCFS).
  • Masquage : Le masquage permet à des IQN d'initiateur iSCSI définis d'accédede manière exclusive à un iSCSI Target.

Pour activer ou désactiver un iSCSI Target :

  1. Sélectionnez l'iSCSI Target que vous souhaitez activer ou désactiver.
  2. Cliquez sur Activer ou Désactiver.

Pour enregistrer les iSCSI Target sur un serveur iSNS :

iSNS (Internet Storage Name Service) fournit le mécanisme des gestions centralisées. Vous pouvez enregistrer toutes les iSCSI Targets avec le serveur iSNS.

  1. Cliquez sur le bouton iSNS et cochez la case Activer iSNS.
  2. Entrez l'adresse IP du serveur iSNS. Cliquez sur OK pour appliquer le paramètre.

Réplication

Cette page permet aux administrateurs système de gérer des réplications. Vous pouvez créer, modifier et supprimer des tâches de réplication. Vous pouvez également exécuter/arrêter la synchronisation et consulter les informations de vos tâches de réplication.

Dossier partagé

Pour créer une tâche de réplication pour un dossier partagé :

  1. Accédez à l'onglet Dossier partagé.
  2. Cliquez sur Créer.
  3. Vous verrez l'assistant Créer une tâche de réplication. Suivez les étapes de l'assistant pour créer une réplication de dossier partagé.

Remarque :

  • Il est impossible de répliquer le dossier partagé « homes ».
  • Il est impossible de répliquer un dossier partagé crypté.
  • Le dossier partagé doit être situé sur un volume Btrfs.
  • Vous pouvez créer des réplications une-plusieurs/étendues pour un dossier partagé sur différents serveurs de destination. Toutefois, vous ne pouvez créer qu'une seule réplication sur un serveur de destination pour chaque dossier partagé.

Pour supprimer une réplication de dossier partagé :

  1. Sélectionnez la tâche de réplication du dossier partagé que vous souhaitez supprimer.
  2. Cliquez sur Supprimer.
  3. Un message de confirmation s'affiche. Cliquez sur Supprimer pour confirmer la suppression.

Pour modifier une réplication de dossier partagé :

  1. Sélectionnez la tâche de réplication du dossier partagé que vous souhaitez modifier.
  2. Cliquez sur Modifier.
  3. Une fenêtre contenant les onglets suivant s'affiche :
    • Serveur partenaire
    • Planification
    • Conservation
  4. Dans l'onglet Serveur partenaire, vous pouvez modifier les informations du serveur partenaire, notamment le nom de l'hôte, le port, le nom d'utilisateur et le mot de passe.
  5. Dans l'onglet Programmer, cochez la case Activer le calendrier d'instantané et configurez le moment et la fréquence de prise des instantanés. Vous pouvez décocher la case pour annuler les instantanés programmés pour le dossier partagé.
  6. Dans l'onglet Conservation, choisissez l'une des options suivantes :
    • Conserver les 256 derniers instantanés : Lorsque le nombre maximum d'instantanés (256) est atteint, les instantanés les plus anciens sont remplacés par les nouveaux instantanés pris.
    • Toujours conserver les instantanés : Lorsque le nombre maximum d'instantanés est atteint, Instantané et réplication cesse de prendre de nouveaux instantanés sans supprimer ceux qui ont déjà été pris.
    • Conserver les instantanés selon la règle suivante : Vous pouvez configurer des règles de conservation pour des plages horaires spécifiées.
  7. Cliquez sur OK pour appliquer les paramètres.

Remarque :

  • Vous ne pouvez modifier que le calendrier de la tâche de réplication sur le serveur source.
  • La modification des paramètres de conservation a une incidence sur les paramètres équivalents de la page Instantané.

Pour afficher les informations d'une réplication de dossier partagé :

  1. Sélectionnez la tâche de réplication du dossier partagé que vous souhaitez consulter.
  2. Cliquez sur Infos.
  3. Une fenêtre contenant les informations suivantes s'affiche :
    • État de cette réplication.
    • Statistiques de cette réplication.

Pour exécuter la synchronisation d'une réplication de dossier partagé :

  1. Sélectionnez la réplication du dossier partagé que vous souhaitez synchroniser.
  2. Cliquez sur Exécuter.
  3. Saisissez la description de l'instantané à prendre.
  4. Cliquez sur OK pour confirmer l'action.

Pour arrêter la synchronisation d'une réplication de dossier partagé :

  1. Sélectionnez la synchronisation de réplication du dossier partagé que vous souhaitez arrêter.
  2. Cliquez sur Arrêt.
  3. Un message de confirmation s'affiche. Cliquez sur Arrêt pour confirmer l'action.

iSCSI LUN

Pour créer une tâche de réplication pour un iSCSI LUN :

  1. Accédez à l'onglet iSCSI LUN.
  2. Cliquez sur Créer.
  3. Vous verrez l'assistant Créer une tâche de réplication. Suivez les étapes de l'assistant pour créer une réplication d'iSCSI LUN.

Remarque :

  • L'iSCSI LUN doit avoir été créé en tant que LUN avancé.
  • Vous pouvez créer des réplications une-plusieurs/étendues pour un iSCSI LUN sur différents serveurs de destination. Toutefois, vous ne pouvez créer qu'une seule réplication sur un serveur de destination pour chaque iSCSI LUN.

Pour supprimer une réplication d'iSCSI LUN :

  1. Sélectionnez la tâche de réplication d'iSCSI LUN que vous souhaitez supprimer.
  2. Cliquez sur Supprimer.
  3. Un message de confirmation s'affiche. Cliquez sur Supprimer pour confirmer l'action.

Pour modifier une réplication d'iSCSI LUN :

  1. Sélectionnez la tâche de réplication d'iSCSI LUN que vous souhaitez modifier.
  2. Cliquez sur Modifier.
  3. Une fenêtre contenant les onglets suivant s'affiche :
    • Serveur partenaire
    • Planification
    • Conservation
  4. Dans l'onglet Serveur partenaire, vous pouvez modifier les informations du serveur partenaire, notamment le nom de l'hôte, le port, le nom d'utilisateur et le mot de passe.
  5. Dans l'onglet Programmer, cochez la case Activer le calendrier d'instantané et configurez le moment et la fréquence de prise des instantanés. Vous pouvez décocher la case pour annuler les instantanés programmés pour l'iSCSI LUN.
  6. Dans l'onglet Conservation, choisissez l'une des options suivantes :
    • Conserver les 256 derniers instantanés : Lorsque le nombre maximum d'instantanés (256) est atteint, les instantanés les plus anciens sont remplacés par les nouveaux instantanés pris.
    • Toujours conserver les instantanés : Lorsque le nombre maximum d'instantanés est atteint, Instantané et réplication cesse de prendre de nouveaux instantanés sans supprimer ceux qui ont déjà été pris.
    • Conserver les instantanés selon la règle suivante : Vous pouvez configurer des règles de conservation pour des plages horaires spécifiées.
  7. Dans l'onglet Application, cochez Activer des instantanés conformes à l'application pour activer les instantanés conformes à l'application. Pour annuler les instantanés conformes à l'application, décochez la case.
  8. Cliquez sur OK pour appliquer les paramètres.

Remarque :

  • Vous ne pouvez que modifier le calendrier sur la source de la réplication.
  • La modification des paramètres de conservation/d'application a une incidence sur les paramètres équivalents de la page Instantané.

Pour consulter les informations d'une réplication d'iSCSI LUN :

  1. Sélectionnez la tâche de réplication d'iSCSI LUN que vous souhaitez consulter.
  2. Cliquez sur Infos.
  3. Une fenêtre contenant les informations suivantes s'affiche :
    • État de cette réplication.
    • Statistiques de cette réplication.

Pour exécuter la synchronisation d'une réplication d'iSCSI LUN :

  1. Sélectionnez la tâche de réplication d'iSCSI LUN que vous souhaitez synchroniser.
  2. Cliquez sur Exécuter.
  3. Saisissez la description de l'instantané à prendre.
  4. Cliquez sur OK pour confirmer l'action.

Pour arrêter la synchronisation d'une réplication d'iSCSI LUN :

  1. Sélectionnez la synchronisation de la tâche de réplication d'iSCSI LUN que vous souhaitez arrêter.
  2. Cliquez sur Arrêt.
  3. Un message de confirmation s'affiche. Cliquez sur Arrêt pour confirmer l'action.

Réplication Une-plusieurs/Étendue

Vous pouvez créer plusieurs réplications pour un dossier partagé/iSCSI LUN sur plusieurs serveurs de destination et former un schéma un-plusieurs. Vous pouvez également créer une réplication étendue pour un dossier partagé/iSCSI LUN répliqué, et former un schéma étendu.

Remarque :

  • Qu'il s'agisse d'un schéma un-plusieurs ou étendu, toutes les tâches de réplication ne peuvent avoir que trois serveurs de destination au maximum.

Politique de conservation

Cette option offre une solution idéale pour vous aider à spécifier le nombre maximum de versions d'instantanés enregistrées dans l'espace de votre volume, mais vous pouvez être amené à conserver vos instantanés pendant une période plus longue.

Synology utilise la politique de conservation GFS, ou Grandfather-Father-Son. Vous pouvez configurer le nombre maximum de versions d'instantanés à conserver pour les plages horaires suivantes, respectivement : horaire, quotidienne, hebdomadaire, mensuelle et annuel.

Le Gestionnaire de protection des données conserve autant d'instantanés que le nombre maximum configuré pour chaque plage horaire. Si plus d'une version d'un instantané existe dans une plage horaire, seule la plus ancienne est conservée. Par exemple, si vous avez configuré une politique de 10 instantané(s) hebdomadaire(s), le Gestionnaire de protection des données conserve l'instantané le plus ancien (si plus d'un instantané est pris au cours d'une semaine) pour chacune des 10 dernières semaines.

En outre, toutes les versions des instantanés dans une heure, depuis la prise de l'instantané le plus récent, sont conservées par défaut, pour vous aider à procéder commodément à la recherche et à la restauration parmi les versions récentes des instantanés. Vous pouvez également verrouiller un instantané afin d'empêcher sa suppression automatique selon vos règles de conservation.

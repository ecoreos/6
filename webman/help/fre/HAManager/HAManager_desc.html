Gestionnaire High Availability

High Availability est une solution de configuration de serveurs conçue pour réduire les interruptions des services causées par des défaillances du serveur. Synology High Availability utilise deux serveurs pour former une « grappe de serveurs High Availability » dans lequel un serveur assume le rôle de « serveur actif » et un autre serveur agit comme « serveur passif » en veille. Afin de mettre en place et de créer une grappe de serveurs high-availability, deux serveurs Synology compatibles sont nécessaires.

Un cluster high-availability utilise les composants et concepts suivants :

  • Concept : La mise en place de la technologie Synology High Availability nécessite deux serveurs Synology pour créer un cluster high-availability. Un serveur agit comme « serveur actif » et un l'autre agit comme « serveur passif ». Une fois le cluster high-availability créé, les données sont répliquées en permanence du serveur actif vers le serveur passif. Tous les fichiers sur le serveur actif existeront en double sur le serveur passif. En cas de dysfonctionnement du serveur actif, le serveur passif sera prêt à prendre en charge tous les services avec une copie complète des données en miroir sur le serveur actif. Ainsi, le cluster high-availability continuera à fournir les services normalement, ce qui minimise les interruptions du serveur. Pour plus de détails sur le basculement, veuillez consulter Basculement
  • Serveur actif : Le serveur actif gère la fourniture de tous les services. Si des erreurs se produisent sur ​​le serveur actif, vous, ou le système, pouvez initier un basculement vers le serveur passif. Dans ce cas, la fourniture des services est transférée vers le serveur passif, et à ce point le serveur actif devient le serveur passif.
  • Serveur passif : Dans des conditions normales, le serveur passif reste en mode veille. Les données du serveur actif sont répliquées en permanence vers le serveur passif. Si ​​le serveur actif ne peut pas fournir les services, vous, ou le système, pouvez initier un basculement vers le serveur passif.
  • Connexion de pulsation : Le serveur actif et le serveur passif d'un cluster high-availability sont reliés par une connexion réseau privée et dédiée appelée « connexion de pulsation ». Une fois le cluster high-availability créé, la connexion de pulsation facilite la synchronisation des données et la réplication entre le serveur actif et le serveur passif. Si le contact entre les serveurs est perdu pendant une durée de temps donnée, le système détermine qu'un basculement est nécessaire.
  • Volume de stockage principal du cluster high-availability : Le volume de stockage du serveur actif deviendra le volume de stockage principal du cluster high-availability. Les données sont répliquées avec le volume de stockage du serveur passif via la connexion de pulsation.
  • Volume de stockage de secours du cluster high-availability : Le volume de stockage sur le serveur passif deviendra le volume de stockage de secours du cluster high-availability. Les données sont répliquées en permanence depuis le volume de stockage du serveur actif au volume de stockage passif via la connexion de pulsation. Cela assure la cohérence du contenu des fichiers lorsqu'un basculement se produit.
  • Nom de serveur du cluster high-availability : Les serveurs du cluster high-availability partageront un nom de serveur. Dans le réseau, vous pouvez vous connecter en utilisant ce nom de serveur, et les services seront fournis par le serveur actif.
  • IP du cluster high-availability : Le serveur actif et le serveur passif du cluster high-availability ayant chacun sa propre adresse IP, afin d'éviter d'avoir à passer à une autre adresse IP lorsqu'un basculement est effectué, le cluster high-availability générera des adresses IP communes pour l'accès. Pour accéder à tous les services DSM, quel que soit le serveur Synology qui agit en tant que serveur actif, vous pouvez vous connecter via ces adresses IP.

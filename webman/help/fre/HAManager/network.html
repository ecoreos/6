Paramètres réseau

Cette page contient trois onglets qui affichent les paramètres réseau du cluster high-availability, le serveur membre actif, et le serveur membre passif.

Onglet Cluster HA

Lors de la création d'un cluster high-availability, le système génère une adresse IP commune pour que l'IP du cluster high-availability reçoive les connexions entrantes. Des adresses IP du cluster high-availability supplémentaires peuvent être créées si nécessaire. Les adresses IP des interfaces réseau activées sur le serveur actif seront affichées sur cet onglet (l'interface réseau utilisée par la connexion de pulsation ne sera pas affichée sur cette page, car cette interface sera utilisée pour la réplication des données entre les deux serveurs). Cet onglet affiche les informations ci-dessous :

  • Interface réseau : La ou les interfaces réseau activées sur le serveur actif.
  • Connexion : Affiche l'état de connexion (ou non) de cette interface.
  • Adresse IP : Lorsque l'on crée un cluster high-availability, le système génère une adresse IP commune de cluster high-availability IP pour recevoir les connexions entrantes. Vous pouvez vous connecter au cluster high-availability en utilisant cette adresse IP lorsque vous souhaitez vous connecter à DiskStation Manager (DSM), quel que soit le serveur Synology qui est le serveur actif. Cela évite d'avoir à se connecter à des adresses IP différentes après un basculement.
  • Masque de sous réseau : Le masque de sous réseau associé à l'adresse IP du cluster high-availability donnée.

Pour modifier l'Adresse IP et le Masque de sous réseau :

  1. Choisissez l'adresse IP et le masque de sous réseau que vous souhaitez modifier et cliquez sur Modifier ou double cliquez sur le champ pour commencer la modification.
  2. Cochez les case Activer pou activer ou désactiver chaque interface. Vous pouvez activer plus d'une interface réseau de cluster, mais l'interface réseau d'origine qui a été configurée lors de la création du cluster ne peut pas être désactivée.

Pour changer le nom de serveur du cluster high-availability :

  1. Cliquez sur Modifier les paramètres réseau pour afficher le nom du cluster high-availability et changer ses paramètres.
  2. Cliquez sur Appliquer pour enregistrer le nom du cluster high-availability et ses paramètres d'adresse IP.
  3. Pour abandonner les modifications actuelles, cliquez sur Annuler.

Onglets Serveur actif et Serveur passif (affiché comme nom de serveur)

Ces onglets affichent l'état des interfaces réseau sur le serveur actif ou le serveur passif. De celles-ci, la Connexion de pulsation occupera une interface réseau sur chacun des serveurs, actif et passif. Ces onglets affichent les informations suivantes :

  • Interface réseau : L'interface réseau physique sur le serveur.
  • Adresse IP : L'adresse IP attribuée pour l'interface réseau donnée.
  • Masque de sous réseau : Le masque de sous réseau associé à l'adresse IP donnée.
  • Description : Affiche les informations de l'interface réseau donnée et l'accessibilité de son adresse IP attribuée. Ce champ affiche les conditions suivantes possibles :
    1. Adresse IP originale accessible : L'adresse IP originale attribuée à l'interface sur le serveur actif est accessible pour les données et les services.
    2. Adresse IP originale non-accessible : L'adresse IP originale attribuée à l'interface sur le serveur passif n'est pas accessible.
    3. IP de connexion de pulsation, non accessible depuis l'intranet : L'adresse IP originale attribuée automatiquement à la connexion de pulsation n'est pas accessible pour les données et les services.

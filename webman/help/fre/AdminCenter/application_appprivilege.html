Privilèges

Les privilèges utilisateur et groupe pour les services et les applications individuels peuvent être vus et modifiés. DSM fournit des paramètres flexibles pour le contrôle des privilèges d'application des utilisateurs et des groupes, ce qui facilite la gestion de l'accès des utilisateurs et des groupes aux différentes applications.

Pour modifier les privilèges d'applications pour les utilisateurs et les groupes :

  1. Allez à Panneau de configuration > Privilèges.
  2. Sélectionnez l'application que vous désirez modifier. Cliquez sur Modifier.
  3. Cliquez sur Utilisateur pour voir les privilèges individuels ou Groupe pour voir les privilèges de groupe.
  4. Pour chaque utilisateur ou groupe, cochez Autoriser pour permettre l'accès ou Refuser pour refuser l'accès à l'application sélectionnée. Selon l'application, vous pouvez également sélectionner Par IP pour autoriser/refuser l'accès basée sur l'adresse IP.
  5. Cliquez sur OK pour enregistrer vos modifications.

Pour voir les privilèges d'application pour les utilisateurs ayant différent adresses IP réseau :

  1. Allez à Panneau de configuration > Privilèges.
  2. Cliquez sur Visionneuse d'autorisation.
  3. Sélectionnez un utilisateur à partir du Nom d’utilisateur et ensuite entrez l'adresse IP que vous voulez pour voir les autorisations pour l'adresse IP.

Pour configurer les privilèges d'applications par défaut pour tous les utilisateurs :

  1. Allez à Panneau de configuration > Privilèges.
  2. Sélectionnez l'application que vous désirez modifier. Cliquez sur Modifier.
  3. Cliquez sur Privilèges par défaut et cochez Accorder ce privilège à tout le monde.
  4. Cliquez sur OK pour enregistrer vos modifications.

Remarque :

  • Les privilèges des admin et invité ne peuvent pas être modifiés ici.
  • Lorsque vous rencontrez des conflits de privilèges, la priorité des privilèges est comme suit : Refuser > Autoriser.
  • Les paquets et les services ne sont pas tous pris en charge par les configurations de privilège d'accès Par IP.

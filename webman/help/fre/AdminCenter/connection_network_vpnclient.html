Connexion à VPN

Vous pouvez facilement basculer votre DiskStation afin de devenir un client VPN et vous connecter aux serveurs VPN (Virtual Private Network) via les protocoles PPTP, OpenVPN ou L2TP/IPSec. Si vous avez plusieurs VPN Server, vous pouvez également créer des profiles VPN spécifiques pour chaque serveur, vous permettant de vous connecter aux différents VPN Server d'un simple clic.

Qu'est-ce qu'un VPN ?

Un VPN, ou (Virtual Private Network), est une solution pour répondre au besoin d'accès aux ressources en toute sécurité sur votre réseau privée à partir d'Internet. Pour les entreprises ou certains privés, avec un besoin d'étendre les capacités de réseaux sociaux croissantes, le coût de ce système de réseau social physique et leur prise en charge technique augmentera de manière exponentielle. Lorsque l'on prend en compte l'efficacité des coûts et les maintenances à long terme, VPN est une solution intelligente et incroyablement attractive. Grâce au cryptage et aux autres mécanismes de sécurité, la technologie VPN permet aux membres de l'entreprise d'accéder facilement au réseau central de l'entreprise et de tirer parti des ressources comme dans un port réseau. Les membres individuels peuvent également accéder aux ressources sur leur port réseau accueil lorsqu'ils sont loin de chez eux.

Remarque :

  • DiskStation peut se connecter seulement aux serveurs OpenVPN qui prennent en charge des tunnels de style tun sur Layer 3.
  • Avant d'installer le paquet VPN, assurez-vous que vous avez un serveur VPN sur lequel vous connecter. Pour une meilleure compatibilité, il est recommandé de se connecter à des services VPN hébergés sur un DiskStation avec le paquet VPN Server installé. Pour configurer un autre DiskStation comme un serveur VPN, veuillez consulter Aide de DSM > VPN Server > Configurer VPN Server pour les instructions.

Pour créer des profils VPN :

  1. Accédez à Panneau de configuration > Réseau.
  2. Dans l'onglet Interface réseau, cliquez sur Créez et choisissez Créer un profil VPN.
  3. Lors de la création d'un nouveau profil VPN, vous pouvez choisir l'un des types de connexion suivants :
    • PPTP : Le protocole PPTP (Point-to-Point Tunneling Protocol) est une solution VPN couramment utilisée prise en charge par la plupart des clients (notamment Windows, Mac, Linux et les appareils mobiles).
    • OpenVPN : OpenVPN est une solution open source pour la mise en œuvre du service VPN. Il protège la connexion VPN avec le mécanisme de cryptage SSL/TLS.
    • L2TP/IPSec : L2TP (Layer 2 Tunneling Protocol) sur IPSec fournit des réseaux privés virtuels avec une sécurité accrue et est pris en charge par la plupart des clients (tels que Windows, Mac, Linux et les appareils mobiles).
  4. Référez-vous aux sections ci-dessous pour les instructions pour le paramétrage de l'un des trois types de connexions VPN.

Pour créer un profil de connexion PPTP :

  1. Lorsque vous êtes invité à choisir une méthode de connexion VPN, choisissezPPTP et ensuite cliquez sur Suivant.
  2. Nommez le nouveau profil, spécifiez l'adresse du serveur VPN, avec lequel vous souhaitez établir une connexion, puis entrez votre nom d'utilisateur et votre mot de passe pour le serveur. Cliquez sur Suivant pour continuer.

    3. Remarque :

    • Le nom d'utilisateur, le mot de passe, l'adresse et le serveur VPN seront fournis par l'administrateur des serveurs VPN.
  3. Choisissez l'un des protocoles d'authentification suivants dans le menu déroulant Authentification pour protéger le mot de passe du client VPN pendant l'authentification :
    • PAP: Le mot de passe ne sera pas crypté lors de l'authentification.
    • CHAP : Le mot de passe sera crypté à l'aide de l’authentification CHAP (Protocole d'identification de négociation par défi).
    • MS CHAP : Le mot de passe sera crypté à l'aide de l’authentification Microsoft CHAP version 1.
    • MS CHAP v2 : Le mot de passe sera crypté à l'aide de l’authentification Microsoft CHAP version 2.
  4. Si vous choisissez MS CHAP ou MS CHAP v2, alors sélectionnez un des éléments suivants à partir du menu Cryptage pour crypter la connexion VPN :
    • Pas de MPPE : La connexion VPN ne sera pas protégée avec le cryptage Microsoft étape par étape.
    • Nécessite MPPE (40/128 bits) : La connexion VPN sera protégée par un mécanisme de cryptage 40 bits ou 128 bits par le cryptage Microsoft étape par étape.
    • MPPE maximum (128 bits) : La connexion VPN sera protégée par un mécanisme de cryptage Microsoft Point to Point 128 bits, ce qui fournit le plus haut niveau de sécurité.

    5. Remarque :

    • Les paramètres d'authentification et de cryptage ici doivent être identiques aux paramètres spécifiés sur le serveur VPN. Pour plus de détails, veuillez contacter l'administrateur du seveur VPN.
  5. Cochez l’une des cases suivantes selon vos besoins :
    • Utiliser la passerelle par défaut sur le réseau distant : Activez cette option pour router le trafic réseau du DiskStation vers le serveur VPN spécifié.
    • Autoriser d'autres appareils du réseau à se connecter via la connexion Internet de ce serveur Synology : Activez cette option pour autoriser les appareils réseau qui sont dans le même réseau local que votre DiskStation à se connecter au même serveur VPN.
    • Reconnecter quand la connexion VPN est perdue : Si la connexion VPN est perdue inopinément, le système tentera de rétablir la connexion cinq fois, à 30 secondes d'intervalle.
  6. Cliquez sur Appliquer.

Pour créer un profil de connexion OpenVPN :

  1. Lorsque vous êtes invité à choisir une méthode de connexion VPN, choisissezOpenVPN et ensuite cliquez sur Suivant.
  2. Nommez le nouveau profil, spécifiez l'adresse du serveur VPN, avec lequel vous souhaitez établir une connexion, puis entrez votre nom d'utilisateur et votre mot de passe pour le serveur.
  3. Dans le champ du Certificat, cliquez sur le bouton Naviguer pour trouver et importer le fichier du certificat (par ex. ca.crt) qui ont été exportés du serveur VPN. Cliquez sur Suivant pour continuer.

    4. Remarque :

    • Le nom d'utilisateur, le mot de passe, l'adresse et le certificat du serveur VPN seront fournis par l'administrators des serveurs VPN.
  4. Cochez l’une des cases suivantes selon vos besoins :
    • Activer la compression sur la liaison VPN : Activez cette option pour effectuer la compression des données avant le transfert via VPN. Cette option peut augmenter la vitesse de transfert, mais peut consommer davantage de ressources système.
    • Utiliser la passerelle par défaut sur le réseau distant : Activez cette option pour router le trafic réseau du DiskStation vers le serveur VPN spécifié.
    • Autoriser d'autres appareils du réseau à se connecter via la connexion Internet de ce serveur Synology : Activez cette option pour autoriser les appareils réseau qui sont dans le même réseau local que votre DiskStation à se connecter au même serveur VPN.
    • Reconnecter quand la connexion VPN est perdue : Si la connexion VPN est perdue inopinément, le système tentera de rétablir la connexion cinq fois, à 30 secondes d'intervalle.
  5. Cliquez sur Appliquer.

Pour créer un profil de connexion OpenVPN (via .ovpn) :

  1. Lorsque vous êtes invité à choisir une méthode de connexion VPN, choisissez OpenVPN (via l'importation d'un fichier .ovpn), puis cliquez sur Suivant.
  2. Choisissez un nom pour le nouveau profil et, dans le champ Importer le fichier .ovpn, cliquez sur Parcourir pour sélectionner et importer un fichier .ovpn depuis le serveur VPN.
  3. Saisissez les informations suivantes fournies par votre administrateur de serveurs VPN. vos nom d'utilisateur et mot de passe, puis importez le fichier du certificat CA (par ex., ca.crt).
  4. Si le fournisseur du serveur VPN vous a également fourni une clé TLS-auth, une clé client, un certificat client ou une liste de révocation de certificats, cliquez sur Options avancées et importez-les dans les champs correspondants.
  5. Cliquez sur Suivant pour continuer.

    6. Remarque :

    • Le nom d'utilisateur, le mot de passe, l'adresse et tous les fichiers supplémentaires, tels que les certificats ou les clés seront fournis par l'administrateur du serveur VPN.
  6. Cochez l’une des cases suivantes selon vos besoins :
    • Activer la compression sur la liaison VPN : Activez cette option pour effectuer la compression des données avant le transfert via VPN. Cette option peut augmenter la vitesse de transfert, mais peut consommer davantage de ressources système.
    • Utiliser la passerelle par défaut sur le réseau distant : Activez cette option pour router le trafic réseau du DiskStation vers le serveur VPN spécifié.
    • Autoriser d'autres appareils du réseau à se connecter via la connexion Internet de ce serveur Synology : Activez cette option pour autoriser les appareils réseau qui sont dans le même réseau local que votre DiskStation à se connecter au même serveur VPN.
    • Reconnecter quand la connexion VPN est perdue : Si la connexion VPN est perdue inopinément, le système tentera de rétablir la connexion cinq fois, à 30 secondes d'intervalle.
  7. Cliquez sur Appliquer.

Pour créer un profil de connexion L2TP/IPSec :

  1. Lorsque vous êtes invité à choisir une méthode de connexion VPN, choisissezL2TP/IPSec et ensuite cliquez sur Suivant.
  2. Nommez le nouveau profil, spécifiez l'adresse du serveur VPN, avec lequel vous souhaitez établir une connexion, puis entrez votre nom d'utilisateur et votre mot de passe pour le serveur. Vous devez aussi entrer la clé pré-partagée pour le serveur VPN. Puis cliquez sur Suivant.

    3. Remarque :

    • Le nom d'utilisateur, le mot de passe, l'adresse et la clé pré-partagée du serveur VPN seront fournis par l'administrateur des serveurs VPN.
  3. Choisissez l'un des protocoles d'authentification suivants dans le menu déroulant Authentification pour protéger le mot de passe du client VPN pendant l'authentification :
    • PAP: Le mot de passe ne sera pas crypté.
    • CHAP : Le mot de passe sera crypté à l'aide de l’authentification CHAP (Protocole d'identification de négociation par défi).
    • MS CHAP : Le mot de passe sera crypté à l'aide de l’authentification Microsoft CHAP version 1.
    • MS CHAP v2 : Le mot de passe sera crypté à l'aide de l’authentification Microsoft CHAP version 2.

    4. Remarque :

    • Les paramètres d'authentification et de cryptage ici doivent être identiques aux paramètres spécifiés sur le serveur VPN. Pour plus de détails, veuillez contacter l'administrateur de votre serveur VPN.
  4. Cochez l’une des cases suivantes selon vos besoins :
    • Utiliser la passerelle par défaut sur le réseau distant : Activez cette option pour router le trafic réseau du DiskStation vers le serveur VPN spécifié.
    • Autoriser d'autres appareils du réseau à se connecter via la connexion Internet de ce serveur Synology : Activez cette option pour autoriser les appareils réseau qui sont dans le même réseau local que votre DiskStation à se connecter au même serveur VPN.
    • Le serveur est derrière le périphérique NAT : Par défaut, le protocole L2TP/IPSec ne permet pas de connecter un serveur VPN derrière un appareil NAT. Activez cette option pour ignorer cette limitation, permettant au DiskStation de connecter un VPN Server L2TP/IPSec derrière un appareil NAT.
    • Reconnecter quand la connexion VPN est perdue : Si la connexion VPN est perdue inopinément, le système tentera de rétablir la connexion cinq fois, à 30 secondes d'intervalle.
  5. Cliquez sur Appliquer.

Pour se connecter à ou se déconnecter d'un serveur VPN :

  1. Sélectionnez un profil VPN.
  2. Cliquez sur Connecter ou Déconnecter.

Remarque :

  • On ne peut connecter qu'un seul profil à la fois.

Pour modifier un profil VPN :

  1. Sélectionnez un profil VPN.
  2. Cliquez sur Modifier.

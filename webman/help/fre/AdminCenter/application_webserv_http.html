Applications Web

Avec la caractéristique intégrée Web Station et la prise en charge PHP, vous pouvez facilement abriter un site web sur votre DiskStation et créer des sites web dynamiques pour votre entreprise ou vos loisirs. En plus, le centre de paquet fournit une variété de paquets tiers tels que la Gestion de contenu, la gestion de relation client et le système e-Commerce. Un dossier partagé par défaut appelé web sera créé pour stocker les fichiers de votre site web.

Pour activer Web Station :

  1. Accédez à Panneau de configuration > Services Web.
  2. Cochez Activer Web Station.
  3. Cliquez sur Appliquer.

Remarque :

  • Pour une meilleure flexibilité et sécurité, Web Station utilise le groupe utilisateur http pour exécuter les tâches. Donc, pour contrôler les autorisations d'accès de vos pages Web, veuillez modifier les autorisations d'accès du groupe http pour chaque dossier et fichier correspondant. Par exemple, si vous voulez rendre un fichier sample.htm accessible dans le dossier partagé web via Web Station, vous devez vous assurer que le groupe http dispose des autorisations d'écriture/lecture appropriées pour le dossier partagé web ainsi que pour le fichier sample.htm.
  • Le fichier de la page d'accueil de votre site Web doit être nommé index.htm, index.html ou index.php.
  • Lorsque vous utilisez la fonction PHP mail(), le système récupérera automatiquement et utilisera les informations du serveur SMTP sur la page Notification. Assurez-vous d'avoir activé les notifications par e-mail et entré l'information adéquate pour le serveur SMTP.
  • Le vrai chemin du dossier partagé web est /volume1/web. Certaines applications nécessitent que l'utilisateur définisse le chemin dans le fichier de configuration.
  • Les noms blog, photo, photosrc et webdefault sont des noms de dossiers réservés. Ces noms ne peuvent pas être utilisés par Web Station. Ne créez donc aucun dossier sous ces noms dans le dossier partagé Web.
  • Vous pouvez personnaliser la page d'erreur en créant un fichier missing.html, missing.htm ou missing.php. Activez d'abord Web Station. Puis placez le fichier missing.html, missing.htm ou missing.php dans le dossier partagé web sur le serveur. Lorsque des utilisateurs font appel à une page qui n'existe pas, la page d'erreur personnalisée s'affiche.
  • Nombre maximum de connexions HTTP actives : 256
  • Nombre maximum d'hôtes virtuels : 30

Activation de l’Hôte virtuel pour les sites Web

Vous pouvez héberger plusieurs sites Web sur un seul serveur avec la fonction d’hôte virtuel. Pour commencer, vous devez enregistrer un couple de noms de domaine et configurer un hôte virtuel en suivant les étapes ci-dessous.

Pour activer l'hôte virtuel :

  1. Cliquez sur Hôte virtuel.
  2. Cliquez sur Créer.
  3. Entrez un nom pour le sous-dossier du dossier partagé « web » où seront stockés les fichiers du site Web.
  4. Entrez un nom d'hôte pour le site Web.
  5. Entrez un port pour le site Web.
  6. Cliquez sur OK.

Activation d'un site Web personnel

La fonction site Web personnel est un moyen pratique de permettre à un utilisateur local, un utilisateur du domaine et un utilisateur LDAP de créer son propre site Web personnel.

Pour les utilisateurs locaux

Chaque utilisateur local possèdera une adresse de site Web unique, server-IP/~nom utilisateur ou server-nom/~nom utilisateur. Par exemple :

  1. 192.168.1.1/~ken
  2. SynologyNAS/~ken

Pour les utilisateurs du domaine

Chaque utilisateur de domaine possèdera une adresse de site Web unique, server-IP/~domainname%5cusername ou server-name/~domainname%5cusername. Par exemple :

  1. 192.168.1.1/~testdomain%5cken
  2. SynologyNAS/~testdomain%5cken

Pour les utilisateurs LDAP

Chaque utilisateur LDAP une adresse de site Web unique, server-IP/~username@domainname ou server-name/~username@domainname. Par exemple :

  1. 192.168.1.1/~ken@testdomain
  2. SynologyNAS/~ken@testdomain

Remarque :

Avant d'activer cette fonction, veuillez vous assurer que les Accueils utilisateur sont activés. Pour les utilisateurs locaux, allez dans Panneau de configuration > Utilisateur > Avancé; pour les utilisateurs de domaine, allez dans Panneau de configuration > Service d'annuaire > Utilisateurs de domaine > Accueil utilisateur; pour les utilisateurs LDAP, allez dans Panneau de configuration > Service d'annuaire > Utilisateurs LDAP > Accueil utilisateur.

Remarque :

Avant d'activer cette fonction, veuillez vous assurer que les Accueils utilisateur sont activés. Pour les utilisateurs locaux, allez dans Panneau de configuration > Utilisateur > Avancé.

Pour créer un site Web personnel :

  1. Si vous avez coché Activer un site Web personnel, un dossier www sera automatiquement créé dans l'annuaire accueil de l'utilisateur.
  2. Maintenant l'utilisateur peut charger les fichiers relatifs au site web (par ex. les pages web et fichiers sonores) dans le dossier www.
  3. Assurez-vous que l'utilisateur a ses propres autorisations d'accès vers les fichiers chargés.

Pour personnaliser la page d'erreur :

  1. Enregistrez la page d'erreur sous le nom missing.html, missing.htm ou missing.php.
  2. Chargez le fichier dans le dossier www.

Lorsque les utilisateurs font un lien vers une page inexistante sur le site Web de l'utilisateur local, la page d'erreur personnalisée s'affichera.

Prises en charge et limites :

Les utilisateurs locaux sont uniquement autorisés à accéder aux données de :

  • Leur propre accueil
  • /var/tmp
  • /tmp
  • /volumeX/@tmp

Service HTTP

En plus des numéros de port par défaut, 80 (pour HTTP) ou 443 (pour HTTPS), des numéros de port supplémentaires peuvent être ajoutés pour l'accès à Photo Station et Web Station. Vous pouvez ajouter un numéro de port supplémentaire compris entre 1 et 65 535 pour l'accès HTTP ou HTTPS.

Pour ajouter un numéro de port supplémentaire pour l'accès HTTP ou HTTPS :

  1. Cochez Ajouter un port HTTP supplémentaire ou Ajouter un port HTTPS supplémentaire (l'option Activer la connexion HTTPS pour les services Web doit être activée avant d'ajouter un port pour HTTPS).
  2. Entrez le numéro de port souhaité dans le champ.

Remarque :

  • Les ports suivants doivent être exclus : 20, 21, 22, 23, 25, 110, 137, 138, 139, 143, 199, 443, 445, 515, 543, 548, 587, 873, 993, 995, 3306, 3689, 5000, 5001, 5005, 5006, 5335, 5432, 6881, 8080, 7000, 7001, 8081, 9997, 9998, 9999, 50001, 50002, les ports par défaut eMule : 4662 (TCP), 4672 (UDP), et la plage de ports FTP par défaut : la plage actuelle peut varier selon les modèles.

FTP/FTPS

Dans Panneau de configuration > Services de fichiers > FTP, vous pouvez activer les services FTP pour autoriser les clients à accéder aux dossiers et aux données sur le système. Avant d'activer FTP, assurez-vous que les ports TCP suivants de votre routeur sont transmis au DiskStation : 21 (connexion de contrôle par défaut), 20 (connexion des données pour le mode actif) et 55536-55567 (connexions des données pour le mode passif).

Pour activer le service FTP :

  1. Activez une des options suivantes :
    • Activer le service FTP (pas de chiffrage) : Il s'agit du protocole réseau standard utilisé pour transférer des fichiers. FTP ne fournit pas de chiffrement pour protéger les informations pendant les sessions de transfert, comme les mots de passe, noms d'utilisateur ou fichiers. Cependant, les vitesses de transfert sont plus rapides et nécessitent moins de ressources système.
    • Activer le service de chiffrage FTP SSL/TLS (FTPS) : Un prolongement du standard FTP avec la prise en charge supplémentaire des protocoles cryptographiques TLS (Transport Layer Security) et SSL (Secure Sockets Layer), qui protègent les informations pendant les sessions de transfert comme les mots de passe, noms d'utilisateur ou fichiers. Cependant, les vitesses de transfert sont plus lentes et utilisent plus de ressources système en raison du chiffrement.
  2. Cliquez sur Appliquer.

Délai d'expiration

Le serveur FTP déconnecte les utilisateurs inactifs au bout de la période de temps définie.

Paramètre de numéro de port du service FTP

Cette section vous permet de spécifier le port utilisé pour le canal de contrôle FTP.

Remarque :

21 est le numéro de port par défaut pour le canal de contrôle FTP. Vous pouvez choisir un numéro entre 1 et 65535, sauf ceux ci-dessous : 20, 22, 23, 25, 80, 110, 137, 138, 139, 143, 199, 443, 445, 515, 543, 548, 587, 873, 993, 995, 3306, 3689, 5000, 5001, 5005, 5006, 5335, 5432, 6881, 8080, 7000, 7001, 8081, 9997, 9998, 9999, 50001, 50002, et les ports par défaut d'eMule : 4662 (TCP), 4672 (UDP).

Plage de ports du FTP passif

Cette section vous permet de spécifier le port utilisé pour les connexions FTP passif. Vous pouvez utiliser la plage de ports par défaut ou utiliser une plage personnalisée. Le plage de ports peut être définie de 1025 à 65535 et peut contenir jusqu'à 128 ports.

Renvoyer l'IP externe en mode PASV

En activant cette option, le serveur envoie son adresse IP externe aux clients FTP. L'adresse IP externe est affichée dans le journal de connexion du client FTP. Une fois la fonction activée, vous pouvez sélectionner l'adresse IP détectée automatiquement ou saisir manuellement l'adresse IP externe du DiskStation.

Cette option ne fonctionne que lorsque le DiskStation est derrière un NAT, et que les clients FTP appartiennent à un sous-réseau différent du DiskStation. Dans la plupart des cas, cette option n'est pas nécessaire, mais si les clients FTP ne peuvent se connecter au DiskStation, vous pouvez alors activer cette option et réessayer.

Activation FXP

FXP est l'acronyme de File eXchange Protocol. En activant cette option, le service FTP prend en charge la fonction de transfert de fichiers de serveur à serveur.

Activation module cryptographique FIPS

FIPS est pris en charge sur le service FTPS à l'aide du module de chiffrement validé OpenSSL FIPS 140-2 (certificat n°1747). En activant cette option, d'autres algorithmes de chiffement sécurisés sont utilisés par les services FTPS pour offrir des sessions plus sûres.

Prise en charge du mode de transfert ASCII

Cette option permet de prendre en charge le mode de transfert ASCII, ce qui permet au DiskStation de transférer correctement les fichiers de texte simple formaté via FTP.

Codage UTF-8

Vous pouvez utiliser cette option pour aider les clients FTP à afficher et à gérer correctement des fichiers avec des noms de fichiers multilingues. Les trois options suivantes sont fournies afin de répondre aux besoins des divers clients FTP :

  • Désactiver : Le codage UTF-8 n'est pas utilisé. Cette option est destinée aux clients qui utilisent l'explorateur de fichiers Windows comme clients FTP pour transférer/télécharger des fichiers avec des noms de fichiers multilingues. Elle n'est pas recommandée pour un usage normal.
  • Auto : Permet à DSM de déterminer automatiquement comment gérer le codage UTF-8. Cette option est recommandée pour un usage normal.
  • Forcer : Force l'utilisation du codage UTF-8. Si vous avez activé le codage UTF-8 sur votre client FTP et que vous ne parvenez toujours pas à afficher et à gérer correctement les fichiers avec des noms de fichiers multilingues, sélectionnez cette option.

Limites de connexion

Cette section inclut les options suivantes :

  • Limiter les connexions par adresse IP :Lorsque cette fonction est activée, le serveur limite le nombre maximal de connexions par adresse IP. Activez cette fonction et limite de débit FTP pour gérer efficacement le débit.
  • Activer la limite de bande passante FTP : Activez cette fonction pour limiter le débit d'une connexion FTP unique.

Remarque :

  • Le compte « guest » ne peut se connecter au serveur via le FTP.
  • Les paramètres de la page de code sur le FTP de l'ordinateur client doivent être les mêmes que ceux de DiskStation pour pouvoir accéder aux données correctement.
  • Sur certains modèles, le service FTP accepte les connexions par SSL ou TLS.
  • Les paramètres de pare-feu de votre client FTP et de deux serveurs FTP doivent être configurés correctement, sinon la fonction FXP ne peut pas fonctionner correctement.
  • Certains clients FTP, tels que ceux qui utilisent Internet Explorer 8/9, doivent se connecter avec leur nom d'utilisateur et mot de passe. Par exemple : ftp://nomutilisateur:motdepasse@votre.ftp.com.

Réseau local/Internet

Selon le modèle de votre DiskStation et les paramètres réseau, vous pouvez configurer les paramètres DHCP pour le réseau local ou Internet dans Panneau de configuration > Réseau > Interface réseau. DHCP est un protocole réseau pour la configuration automatique des adresses IP. Pour laisser le DHCP attribuer une adresse IP pour le système, cliquez sur Modifier, puis choisissez Définir la configuration réseau automatiquement (DHCP) dans l'onglet IPv4.

Si vous utilisez une adresse IP spécifique de votre fournisseur de services réseau, choisissez Utiliser la configuration manuelle et saisissez manuellement l'adresse IP et le masque de sous-réseau. Si aucun serveur DNS ou passerelle par défaut n'est disponible, laissez le champ vide.

Remarque :

  1. Pour être sûr que les courriers électroniques de notification sont correctement envoyés par le serveur SMTP, vous devez entrer une IP valide pour le serveur DNS.
  2. Les Jumbo Frames sont des trames Ethernet dont l'unité de transmission maximale (MTU, Maximum Transmission Unit) excède 1500 octets. Ainsi la transmission de fichiers de grande taille via Ethernet en est améliorée. Elles ne peuvent être activées que dans un environnement réseau Gigabit. Pour vous assurer que Jumbo Frame fonctionne correctement, tous les ordinateurs et périphériques sur le réseau accédant à votre DiskStation doivent le prendre en charge et utiliser la même valeur de MTU.

VLAN

Lorsque l'environnement réseau nécessite des paramètres VLAN, vous pouvez activer l'option VLAN et saisir un ID VLAN pour l'environnement réseau dans l'onglet IPv4 de la fenêtre Modifier.

Remarque :

  • Assurez-vous que votre commutateur réseau est correctement configuré, sinon le service du DiskStation pourrait devenir inaccessible.
  • Si vous ne pouvez pas vous connecter au DiskStation après avoir modifié les paramètres ci-dessus, veuillez appuyer sur le bouton RESET situé sur votre DiskStation pour pouvoir restaurer les paramètres par défaut.

Configuration d’IPv6

DiskStation prend en charge le schéma de fonctionnement IPv4/IPv6 dual stack. Cela signifie que l’IPv4 et l’IPv6 peuvent fonctionner simultanément sur votre DiskStation. Le paramètre par défaut de l'IPv6 est Auto. Sélectionnez une interface réseau existante, cliquez sur le bouton Modifier et aller dans l'onglet IPv6 pour modifier les paramètres.

Pour configurer les paramètres IPv6 :

  1. Accédez dans Panneau de configuration > Réseau > Interface réseau.
  2. Sélectionnez une interface réseau existante et cliquez sur Modifier.
  3. Sous l'onglet IPv6, sélectionnez un type de configuration depuis configuration IPv6 :
    • Auto : Votre DiskStation créera automatiquement une adresse IPv6 selon le LAN Router Advertisement (RA).
      • Mode sans état : Permettre aux appareils connectés au routeur de configurer automatiquement une adresse IPv6 sans configurer de serveur IPv6 DNS.
      • Mode DHCPv6 sans état : Permettre aux appareils connectés au routeur de configurer automatiquement une adresse IPv6 et d'obtenir des informations DNS via DHCPv6.
      • Mode d'état : Permettre aux appareils connectés au routeur d'obtenir une adresse IPv6 et des informations DNS via DHCPv6. Dans ce mode, vous devez activer le serveur DHCPv6 et définissez l'Adresse IP de début et l'Adresse IP de fin du serveur DHCPv6.
    • Manuel : Configurez manuellement l'IPv6 en entrant des informations dans Adresse IPv6, Longueur du préfixe, et Passerelle par défaut.
    • Désactivé : Désactiver la prise en charge d’IPv6.

Jusqu'à ce que IPv6 supplante complètement IPv4, les mécanismes de transition IPv6 sont nécessaires pour permettre aux hôtes avec des adresse IPv6 d'atteindre des services IPv6. Choisissez entre les différents types de mécanismes de transition IPv6 :

  • 6in4 : Connectez vous au réseau IPv6 à travers un agent de tunnel. Vous devrez remplir l'Adresse IPv6, la Longueur de préfixe, le Préfixe et l'Adresse IPv4 de l'agent de tunnel. Ceci nécessite une adresse IPv4 publique.
  • 6to4 : Connectez vous au réseau IPv6 à travers un agent de tunnel utilisant anycast. Ceci nécessite une adresse IPv4 publique.
  • DHCP-PD : Obtenez un préfixe IPv6 depuis le serveur DHCP par délégation de préfixe dans le LAN. Ceci requiert une prise en charge ISP et fonctionne uniquement en mode routeur.
  • Tunnellisation IPv6 (TSP) : Allez dans Panneau de configuration > Réseau > Général pour configurer cette option.

802.1X

802.1X est une norme d'authentification développée par l'IEEE pour les réseaux de connexions utilisateur. 802.1X peut être utilisé sur des réseaux pour authentifier le client, en n'autorisant pas les clients ne répondant avec succès aux critères d'authentification à accéder au réseau protégé.

Vous pouvez autoriser votre Synology NAS à se connecter au réseau protégé par authentification 802.1X.

Pour activer l'authentification 802.1X :

  1. Accédez dans Panneau de configuration > Réseau > Interface réseau.
  2. Sélectionnez une interface de réseau câblé (par ex. LAN) et cliquez sur Modifier.
  3. Accédez à l'onglet 802.1X et cochez Activer l'authentification 802.1X.
  4. Choisissez l'un des protocoles suivants dans le menu déroulant Authentification : Si vous n'êtes pas certain du protocole le plus adapté, sélectionnez Auto.
    • Protected EAP (PEAP) : Utiliser un certificat CA (facultatif) pour établir une connexion sécurisée. Cette connexion sécurisée est utilisée par l'authentification interne.
    • Tunneled TLS (TTLS) : Utiliser un certificat CA (facultatif) pour établir une connexion sécurisée. Cette connexion sécurisée est utilisée par l'authentification interne.
    • TLS : Utiliser TLS (Transport Layer Security) pour l'authentification.
  5. Si vous choisissez TTLS ou PEAP, vous devez choisir l'un des protocoles d'authentification interne dans le menu déroulant.
    • MS CHAP v2 : Les informations d'authentification sont chiffrées avec Microsoft CHAP v2.
    • GTC : Les informations d'authentification utilisent GTC (Generic Token Card) pour le chiffrement.
  6. Saisissez votre nom d'utilisateur et votre mot de passe.
  7. Si nécessaire, cliquez sur Importer, puis Parcourir pour rechercher et importer un certificat exporté depuis un serveur Radius (par exemple ca.crt).
  8. Cliquez sur OK pour enregistrer vos paramètres.

Remarque :

  • Pour configurer l'authentification 802.1X pour des réseaux sans fil, accédez à Panneau de configuration > Sans fil > Wi-Fi.
  • Les paramètres d'authentification doivent être identiques à ceux de Radius Server. Pour plus d'informations, veuillez contacter l'administrateur de Radius Server.

Réseau local sans fil*

Lorsque votre DiskStation fait office de routeur sans fil, vous pouvez configurer les paramètres suivants pour le réseau local sans fil dans Panneau de configuration > Réseau > Interface réseau.

Paramètres IPv4

Adresse IP et masque de sous-réseau

Vous pouvez définir l'adresse IP et le masque de sous-réseau pour le DiskStation ici.

DHCP Server

Cochez l'option Activer le serveur DHCP pour permettre à votre DiskStation d'attribuer des adresses IP aux appareils client qui se connectent au réseau local fourni par votre DiskStation.

Relais PPPoE

Activer Relais PPPoE pour permettre aux appareils qui sont connectés à votre DiskStation de se connecter à Internet via la connexion PPPoE.

Configurer un port Internet UPnP

Activez UPnP pour permettre à votre DiskStation de devenir un périphérique UPnP Internet Gateway, de sorte que les appareils connectés au réseau local du DiskStation puissent configurer les paramètres de transmission de port avec UPnP.

Paramètres IPv6

Vous pouvez obtenir une adresse et un préfixe IPv6 depuis LAN/Internet avec 6in4, 6to4, ou DHCP-PD. Avec ces informations, vous pouvez configurer les options de routage IPv6 pour votre DiskStation ici.

  • Adresse IPv6 : Entrez l'adresse IPv6 obtenue depuis LAN/Internet.
  • Longueur du préfixe : Entrez la longueur du préfixe de l'adresse IPv6 obtenue depuis LAN/Internet, généralement 64.
  • DNS principal : Le serveur DNS principal pour les appareils connectés au routeur. Ceci ne doit pas nécessairement être défini dans Mode sans état.
  • DNS secondaire : Le serveur DNS secondaire pour les appareils connectés au routeur. Ceci ne doit pas nécessairement être défini dans Mode sans état.

Vous pouvez choisir entre trois Router Advertisement (RA) :

  • Mode sans état : Permettre aux appareils connectés au routeur de configurer automatiquement une adresse IPv6 sans configurer de serveur IPv6 DNS.
  • Mode DHCPv6 sans état : Permettre aux appareils connectés au routeur de configurer automatiquement une adresse IPv6 et d'obtenir des informations DNS via DHCPv6.
  • Mode d'état : Permettre aux appareils connectés au routeur d'obtenir une adresse IPv6 et des informations DNS via DHCPv6. Dans ce mode, vous devez activer le serveur DHCPv6 et définissez l'Adresse IP de début et l'Adresse IP de fin du serveur DHCPv6.

Clients DHCP

Ici, vous pouvez voir une liste des clients DHCP et leur configuration réseau (comme adresse MAC, adresse IP, nom d'hôte, et temps restant avant l'expiration du bail DHCP).

Vous pouvez filtrer la liste déroulante des clients IPv4/IPv6 à afficher aux clients qui utilisent respectivement IPv4 et IPv6. Choisissez un client et cliquez sur Ajouter à la réservation d'adresse pour ajouter le client à la liste de réservation d'adresse.

Réservation DHCP

Si vous voulez qu'un client reçoive toujours la même adresse IP lors du renouvellement du bail DHCP, vous pouvez ajouter le client à la liste de réservation d'adresse.

Vous pouvez filtrer la liste déroulante de réservation d'adresse IPv4/IPv6. Les adresses IPv4 sont identifiées par l'adresse MAC, tandis que les adresses IPv6 sont identifiées par DUID.

Pour ajouter un client à la liste de réservation d'adresses :

  • Sélectionnez un client et cliquez sur Ajouter.
  • Spécifiez l'adresse MAC du client ou DUID, l'adresse IPv4/IPv6 que vous désirez attribuer au client et le nom d'hôte du client.
  • Cliquez sur OK pour sauvegarder les paramètres.

Gérer les permissions avancées sur le dossier partagé

La page Panneau de configuration > Dossier partagé > Modifier > Avancé inclut les options ci-dessous afin de régler plus finement les permissions d'accès d'un dossier partagé.

Permissions du dossier partagé

Les permissions du dossier partagé offrent une couche supplémentaire de contrôle pour gérer les permissions d'accès des dossiers partagés. Lorsqu'elles sont activées, les utilisateurs et les groupes peuvent voir ou modifier le contenu d'un dossier partagé uniquement si l'utilisateur ou le groupe a reçu à la fois les permissions de partage avancées et les permissions Windows ACL (situées dans Dossier partagé > Modifier > Permissions).

Pour activer les permissions de partage avancées :

  1. Cochez la case Activer les permissions de partage avancées.
  2. Cliquez sur Permissions de partage avancées pour modifier les permissions de partage avancées pour le dossier partagé.

Remarque :

  • Les permissions de partage avancées sont appliquées lorsqu'un utilisateur accède à un dossier partagé via les services suivants : Windows File Sharing, Apple File Sharing, File Station, FTP et WebDAV.

Paramètres avancés

Si nécessaire, vous pouvez placer d'autres restrictions sur les utilisateurs lorsqu'ils accèdent à un dossier partagé via File Station, FTP ou WebDAV.

  • Désactiver la navigation dans le répertoire : Cette option empêche les utilisateurs de voir le contenu du dossier partagé.
  • Désactiver la modification des fichiers existants : Cette option empêche les utilisateurs de déplacer, détruire ou modifier les fichiers dans le dossier partagé. Veuillez noter que les utilisateurs seront capables de visualiser, télécharger/charger, copier ou décompresser les contenus du dossier partagé.
  • Désactiver le téléchargement de fichiers : Cette option empêche les utilisateurs de télécharger le contenu du dossier partagé.

Remarque :

Veuillez noter que ces restrictions ne s'appliquent pas aux utilisateurs qui appartiennent au groupe administrators.

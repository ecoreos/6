Service de fichiers Windows

Le service de fichiers Windows permet aux clients Windows, Mac et Linux prenant en charge SMB/CIFS d'accéder aux données stockées sur le DiskStation. Si vous désirez parcourir les fichiers sur le DiskStation via Windows Explorer ou monter des dossiers partagés en tant que disques de réseaux, activez cette option dans Panneau de configuration > Services de fichiers > Win/Mac/NFS.

Pour activer le service de fichiers Windows :

  1. Cochez Activer le service de fichiers Windows.
  2. Cliquez sur Appliquer.

Remarque :

Après avoir activé le service de fichiers Windows, vous pouvez monter des dossiers entièrement indexés sur votre ordinateur Mac via SMB et utiliser Spotlight dans Finder pour rechercher le contenu de fichiers dans les dossiers montés.

Groupe de travail

Cette option vous permet de désigner le système comme membre d'un groupe de travail spécifique à des fins de gestion de réseau.

Remarque :

  • Le nom d'un groupe de travail peut inclure de 1 à 15 caractères. Le nom ne peut pas inclure les caractères suivants : [ ] ; : " < > * + = \ / | ? ,
  • Si vous utilisez un MAC sous OS X 10.9 et que votre DiskStation a rejoint un domaine Windows, vous pouvez vous connecter en ajoutant localhost\ avant votre nom d'utilisateur local de DiskStation pour accéder aux fichiers stockés sur le DiskStation via le service de fichiers Windows.

Refuser l'accès aux versions précédentes

Dans Windows Explorer, les utilisateurs peuvent accéder aux instantanés des dossiers ou des fichiers en cliquant sur l'onglet Versions précédentes dans un dossier partagé, ce qui leur permet de restaurer ou de copier des données sans passer par un administrateur. Si vous sélectionnez Refuser l'accès aux versions précédentes, la page « Versions précédentes » n'affiche aucune information.

Remarque :

  • Les utilisateurs sont autorisés à accéder à la version précédente uniquement si, au moment où l'instantané correspondant à la version précédente a été pris, leurs comptes utilisateurs ont été créés et qu'on leur a octroyé la permission en lecture sur le dossier partagé dont l'instantané correspondant a été pris.

Activation de l'accès au transfert

Cette option permet d'activer l'enregistrement des opérations d'accès aux fichiers, y compris créer, supprimer, lire, écrire, déplacer et renommer les fichiers ou des dossiers par le biais du protocole CIFS/SMB, également appelé protocole de transfert de fichiers Windows.

Paramètres avancés

Cliquez sur le bouton Paramètres avancés pour voir les options ci-dessous.

  • Serveur WINS : Microsoft Windows Internet Name Service (WINS) est un service de mappage nom NetBIOS - adresse IP. Les utilisateurs Windows localiseront le système plus facilement sur les réseaux TCP/IP si le système a été configuré pour s'enregistrer sur un serveur WINS.
  • Protocole max pour le service de fichiers Windows : Selon votre configuration réseau, vous pouvez configurer le niveau de protocole le plus élevé pris en charge par votre DiskStation.
    • Activer SMB 2 : SMB (Server Message Block) 2 est pris en charge depuis Windows Vista et représente la version améliorée de SMB. SMB 2 ajoute la capacité à composer plusieurs actions SMB dans une seule demande pour réduire le nombre de paquets réseau et pour améliorer la performance.
    • Activer SMB 2 et Large MTU : MTU (maximum transmission unit) fait référence à la plus grande unité de données qui peut être transmise via un protocole de communication. Afin de tirer parti des interfaces réseau plus rapides, tels que l'Ethernet 1 ou 10 gigabits, Grand MTU augmente l'unité de transmission maximale à 1 mégaoctet (Mo). L'activation de Grand MTU améliore la rapidité et l'efficacité des transferts de fichiers volumineux, tout en réduisant le nombre de paquets qui doivent être traités.
    • Activer SMB 3 : SMB 3 est pris en charge depuis Windows 8 et Windows Server 2012. Il s'agit de la version améliorée de SMB 2. SMB 3 prend en charge la transmission de chiffrement de fichiers AES, ce qui permet d'améliorer la sécurité des transferts de fichiers en mode pair à pair.

    Remarque :

    • SMB 2 est pris en charge sur Windows Vista, Windows Server 2008 R2, Windows 7, et versions ultérieures.
    • Grand MTU est pris en charge sur Windows Server 2008 R2, Windows 7, et versions ultérieures.
    • Afin de rendre le grand MTU opérationnel sur votre ordinateur Windows, vous devez modifier les paramètres de registre de votre ordinateur. Pour plus de détails, veuillez vous reporter au support technique Microsoft (comme cet article).
  • Mode de transport du chiffrement : Lorsque SMB 3 est activé, le service de fichiers Windows ajoute le chiffrement du transport afin de renforcer la sécurité de la transmission des fichiers.
    • Désactiver : Aucun chiffrement de transport n'est appliqué.
    • Auto : Le chiffrement de transport n'est appliqué qu'aux clients qui prennent en charge cette fonctionnalité.
    • Forcer : Le chiffrement du transport est toujours appliqué. Cela empêche les clients qui ne prennent pas en charge le chiffrement du transport d'utiliser le service de fichiers Windows.

    • Remarque :

    • Le chiffrement du transport pris en charge sous Windows 8, Windows Server 2012 et les versions ultérieures.
    • La connexion anonyme pour le service de fichiers Windows n'est pas prise en charge lorsque le chiffrement du transport est activé.
    • Si le chiffrement de transport est activé, laissez la case Désactiver le verrouillage opportuniste décochée pour éviter les dépassements de délai de l'application.
  • Désactiver le verrouillage opportuniste : Si vous cochez cette option, cela désactive la fonction de « verrouillage opportuniste » de CIFS, entraînant une amélioration des performances quand des applications de base de données accèdent aux fichiers de la base de données sur le dossier partagé du DiskStation via le réseau.
  • Activer Local Master Browser : Lorsque plusieurs ordinateurs Windows existent dans le même sous-réseau, ils négocient et élisent un ordinateur en tant que « local master browser ». Le local master browser maintient les listes des autres ordinateurs du sous-réseau et de leurs ressources partagées, et partage ces listes avec les autres ordinateurs. Cette option permet au DiskStation d'assumer le rôle de local master browser.

    • Remarque :

    L'activation de cette fonction désactive l'hibernation du disque dur et active le compte guest sans mot de passe.

  • Activer le module VFS DirSort : Trie les noms de dossier et les classe par ordre alphabétique. Cette option affecte la performance du système lorsqu'elle est activée.
  • Omettre des fichiers : Masque des fichiers selon des critères spécifiques. Ces critères peuvent inclure des caractères de remplacement (*) ; les entrées multiples doivent être séparées par une barre oblique (/). Par exemple : /abc*/*.txt/*.conf/. Cette option affecte la performance du système lorsqu'elle est activée.
  • Permettre les liens symboliques au sein de dossiers partagés : L'activation de cette option permet aux liens symboliques de pointer vers des fichiers ou des dossiers situés dans le même dossier partagé. Les liens symboliques peuvent être créés à l'aide d'un terminal DSM.
  • Autoriser les liens symboliques entre différents dossiers partagés : Permet que les liens symboliques pointent vers des fichiers ou des dossiers situés dans des dossiers partagés différents. Les liens symboliques peuvent être créés à l'aide d'un terminal DSM. Veuillez noter que l'activation de cette option pourrait permettre à des utilisateurs sans privilèges d'accès de voir et d'accéder à des dossiers partagés qui sont la cible de liens symboliques.
  • Activer le module VFS MSDFS : Permet aux liens symboliques de pointer vers des fichiers ou des dossiers situés sur d'autres serveurs SMB. Sur les anciennes versions de Windows, telles que Windows XP, le client et le DiskStation doivent rejoindre un domaine Windows avant d'activer cette option. Pour en savoir davantage sur la création des liens MSDFS, veuillez voir ici. Veuillez noter que le cache de l'appareil client doit être effacé après la désactivation de cette option, autrement la performance normale pourrait en être affectée.
  • Désactiver les connexions multiples à partir de la même adresse IP : Lorsque cette option est activée, le DiskStation autorise une seule connexion par adresse IP pour les clients utilisant le Service de fichiers Windows. Lorsque plusieurs clients avec la même adresse IP utilisent Windows File Service, le DiskStation conserve la dernière connexion et interrompt toutes les connexions plus anciennes. Dans un environnement NAT, le client pourrait utiliser une adresse IP unique, auquel cas l'activation de cette option pourrait affecter la performance normale.
  • Activer les journaux de débogage : Lorsque cette option est activée, le système crée des journaux détaillés pour aider au débogage. Cette option affecte la performance du système lorsqu'elle est activée.
  • Application des permissions UNIX par défaut : L'activation de cette option applique les permissions UNIX par défaut lors du chargement ou de la création de fichiers et dossiers. La permission UNIX est de 744 pour les fichiers et 755 pour les dossiers. Lorsque cette option est désactivée, la permission UNIX est 777 pour les fichiers et les dossiers.

    • Remarque :

    • L'activation de cette option pourrait provoquer des problèmes d'incohérence de permission entre différents protocoles. Afin d'éviter les incohérences, nous suggérons de laisser cette option désactivée.

    Remarque :

    • Pour les dossiers partagés avec Windows ACL activé (tous les partages sauf les « photos » et les partages sur des disques externes), veuillez exécuter la commande chmod sur votre client Linux ou FTP pour changer les types de permissions de dossier et de fichier depuis Windows ACL vers UNIX.
    • L'activation de cette option pourrait provoquer des problèmes d'incohérence de permission entre différents protocoles. Afin d'éviter les incohérences, nous suggérons de laisser cette option désactivée.
  • Ne pas réserver d'espace disque lors de la création de fichiers : Activez cette option pour empêcher la réservation d'espace disque lors de la création de fichiers sur votre DiskStation.
  • Vider le cache SMB : Cliquez sur ce bouton pour supprimer les fichiers de cache SMB dans un but de dépannage :
    • /var/cache/samba/netsamlogon_cache.tdb
    • /var/lib/samba/winbindd_cache.tdb
    • /var/lib/samba/share_info.tdb
    • /var/lib/samba/registry.tdb
    • /run/samba/connections.tdb
    • /run/samba/gencacche.tdb
    • /run/samba/gencache_notran.tdb

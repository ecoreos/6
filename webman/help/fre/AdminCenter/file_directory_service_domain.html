Domaine

Votre DiskStation peut rejoindre un domaine Windows. Après avoir rejoint un domaine, les utilisateurs du domaine peuvent se connecter sur le DiskStation à l'aide de leurs comptes et mots de passe de domaine, ce qui leur permet d'accéder et d'utiliser les applications DSM sans devoir mémoriser d'autres noms d'utilisateurs et mots de passe.

Pour rejoindre un domaine :

  1. Accédez à Panneau de configuration > Service d'annuaire > Domaine.
  2. Cochez la case Rejoindre le domaine.
  3. Entrez le nom de domaine et spécifiez le serveur DNS dans les champs appropriés.
  4. Si nécessaire, cochez la case Options de domaine avancées. (Voir ci-dessous pour plus de détails.)
  5. Cliquez sur Appliquer.
  6. Dans la fenêtre déroulante, saisissez le nom d'utilisateur et le mot de passe de l'administrateur du serveur de domaine.
  7. Cliquez sur OK.

DNS Server

Il est conseillé de spécifier un serveur DNS pour aider le DiskStation à rechercher le domaine. Toutefois, si le domaine n'est pas enregistré sur le serveur DNS, la procédure pour rejoindre le domaine peut échouer.

Type de serveur de domaine

Ce champ affiche le type de domaine lorsque le DiskStation rejoint un domaine. Le type de domaine sera « Domaine AD ».

Mode de gestion

Cette option déterminera la façon dont vous gérer les privilèges des utilisateurs et groupes de domaine.

  • Trusted Domain: Vous pouvez gérer les utilisateurs et les groupes dans le domaine joint ainsi que les domaines de confiance. Les utilisateurs et les groupes peuvent être filtrés avec la liste déroulante de domaine.
  • Domaine seul avec OU: Seuls les utilisateurs et les groupes dans le domaine joint seront affichés dans ce mode. Ce mode vous permet de filtrer les utilisateurs de domaine/groupes de domaine avec la liste déroulante OU (Unité organisationnelle).

Options avancées du domaine

Dans la plupart des cas, vous n'avez pas besoin de remplir les options avancées du domaine. Les options avancées du domaine ne sont nécessaires que dans des environnements de domaine spécifiques.

  • DC IP/FQDN: Spécifiez une adresse IP pour le DC (contrôleur de domaine) ou FQDN et le DiskStation essaiera de communiquer avec celui-ci. S'il y a plusieurs adresses IP ou FQDN, insérez une virgule (,) entre chacune d'entre elles. Ajoutez un astérisque (*) après la dernière adresse IP de DC ou FQDN si vous voulez, et DiskStation essaiera de communiquer avec un autre DC quand tous ceux qui ont été spécifiés ont échoué. Vous devez ajouter une virgule entre l'astérisque et la dernière adresse IP ou FQDN.
  • Nom de domaine NetBIOS : Spécifiez le nom NetBIOS du domaine.
  • FQDN du domaine (nom DNS) : Spécifiez le FQDN (nom DNS) du domaine.
  • Enregistrer l'interface DNS : Lorsque vous rejoingez un domaine, la NIC (Network Interface Card) spécifiée ici est enregistrée sur le DNS server. Veuillez noter que si le nom d'hôte du DiskStation comprend un trait de soulignement (_), l'enregistrement échouera car les traits de soulignement (_) ne peuvent pas être utilisés avec DNS.
  • Mettre à jour la liste utilisateur/groupe : Spécifiez la fréquence avec laquelle le DiskStation doit mettre à jour automatiquement la liste des utilisateurs/groupes du domaine. Vous pouvez définir une fréquence de mise à jour quotidienne, hebdomadaire ou mensuelle. De plus, les listes d'utilisateurs du domaine/groupes peuvent être mises à jour manuellement dans l'onglet Utilisateur du domaine, en cliquant sur Mettre à jour tous les domaines. Veuillez noter que les mises à jour automatiques affecteront l'hibernation du système.

Comment rejoindre un Contrôleur de domaine en lecture seule

Pour joindre votre DiskStation à un contrôleur de domaine en lecture seulement (RODC), vous devez spécifier le "nom complet de l'ordinateur" (FQDN) de votre contrôleur de domaine en lecture seulement (RWDC) dans le champ DC IP/FQDN. Le nom complet de l'ordinateur doit être au format suivant : ComputerName.Domain, p. ex. « ad01.synology.com ».

Remarque :

  • Si votre nom d'utilisateur de domaine comprend les caractères « % » ou « $ », il se peut que vous ne puissiez pas accéder à votre dossier d'accueil. Demandez à votre administrateur de domaine de vous donner un nouveau nom d'utilisateur.
  • Les groupes de domaine Domain Admins et Enterprise Admins seront automatiquement ajoutés au groupe local administrateurs. Par conséquent, les utilisateurs de domaine appartenant à ces groupes posséderont des privilèges administratifs sur le DiskStation, notamment l'accès aux services de fichiers comme CIFS/FTP/AFP/WebDAV et d'autres applications DSM.
  • Vous pouvez également configurer les privilèges d'accès aux dossiers partagés du DiskStation pour les utilisateurs du domaine Voir ici pour plus d’informations.

Options du domaine

Cette section inclut les options de domaine supplémentaires ci-dessous.

  • Le DiskStation se synchronisera avec un serveur NTP chaque fois qu'un utilisateur de domaine se connecte : Cette option force le moment de synchronisation entre le DiskStation et le serveur NTP.
  • Activer la signature serveur : Si la connexion client est activée sur le client, il faut alors activer la connexion serveur sur le DiskStation, de sorte que les utilisateurs puissent utiliser le domaine normalement.
  • Obtenir les listes utilisateurs/groupes avec le mode compatible NT4 Cette option permet au système d'obtenir les listes d'utilisateurs/groupes à l'aide du mode NT4 RPC. Cette option peut être activée si certaines listes d'utilisateurs/groupes du domaine ne sont pas accessibles à l'aide des paramètres par défaut.
  • Désactiver les permissions administrateur pour les admins de domaine et d'entreprise : Un utilisateur avec des autorisations administrator peut avoir un contrôle complet de votre DiskStation et de ses fichiers. L'activation de cette option vous permettra de restreindre les autorisations des Domain Admins et Enterprise Admins pour être de même à un utilisateur normal
  • Énumérer les niveaux de groupes imbriqués : Spécifiez le nombre de membres de groupe de domaine imbriqués qui peuvent être énumérés.

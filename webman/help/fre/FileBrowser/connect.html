Connexion à distance

Dans File Station, vous pouvez vous connecter à des services cloud publics (Box, Dropbox, Google Drive et Microsoft OneDrive) et à des serveurs de fichiers via toute une gamme de différents protocoles (FTP, STFP, WebDAV et WebDAV HTTPS).

Pour vous connecter à un service cloud :

  1. Accéder à Outils > Connexion à distance > Configuration de la connexion.
  2. Choisissez le service cloud souhaité en cliquant sur l'icône, puis sur Suivant.
  3. Dans la fenêtre contextuelle, suivez les instructions de l'assistant pour l'authentification du compte.
  4. Une fois la configuration terminée, le service cloud cible s'affiche dans le volet de gauche.

Limitations propres au cloud :

  • Box :
    • Un catalogue partagé avec d'autres collaborateurs prend un suffixe composé du nom d'utilisateur du propriétaire.
    • Si des catalogues partagés avec d'autres collaborateurs utilisent un nom identique (un nom de catalogue avec pour suffixe le nom d'utilisateur du propriétaire), un seul de ces catalogues est affiché.
    • Vous ne pouvez ouvrir des fichiers portant ces extensions qu'avec Box : .gdoc, .gsheet et .boxnote.
    • Les noms de fichier sont sensibles à la casse.
    • Pour une configuration appropriée du pare-feu, veuillez vous reporter à cet article.
  • Dropbox :
    • Il est impossible d'afficher un catalogue Dropbox s'il comporte plus de 25 000 fichiers.
    • Il est impossible de renommer un catalogue Dropbox s'il comporte plus de 10 000 fichiers.
    • Il est impossible de transférer des fichiers de plus de 10 Go.
    • La tâche de chargement échoue si le transfert du fichier cible prend plus de 24 heures.
    • Les noms de fichier sont sensibles à la casse.
    • Pour une configuration appropriée du pare-feu, veuillez vous reporter à cet article.
  • Google Drive :
    • Les fichiers/dossiers dont le chemin d'accès comporte plus de 2048 caractères ne s'affichent pas.
    • Les fichiers dont le nom comporte plus de 255 caractères ne s'affichent pas.
    • Les fichiers/catalogues portant un nom identique sont indiqués avec des heures de création individuelles.
    • Si vous ouvrez un fichier propre à Google (par ex., Google Docs et Google Slides) via File Station, une URL s'affiche et vous permet d'accéder au fichier cible.
    • Les dossiers My Drive et Starred affichés dans File Station correspondent respectivement aux dossiers « Mon Drive » et « Suivi » de Google Drive.
    • Pour une configuration appropriée du pare-feu, veuillez vous reporter à cet article.
  • Microsoft OneDrive :
    • Les fichiers de carnet OneNote ne s'affichent pas.
    • Il est impossible de transférer des fichiers de plus de 10 Go.
    • Il est possible que les fichiers (e.g., .png, .jpg, .docx, .pdf) affichent des tailles plus importantes (La taille originale plus la taille de la miniature). Cela n'a aucun effet sur la taille des fichiers originaux lorsque vous les téléchargez.
    • Les noms de fichier sont sensibles à la casse.

Pour prendre connaissance de davantage de limitations, nous vous suggérons d'accéder au site Web officiel du service cloud concerné.

Pour se connecter à un serveur de fichiers :

  1. Accéder à Outils > Connexion à distance > Configuration de la connexion.
  2. Choisissez le protocole du serveur souhaité en cliquant sur l'icône, puis sur Suivant.
  3. Fournissez les informations suivantes pour compléter la configuration :
    • Nom d'hôte ou IP : Saisissez le nom d'hôte ou l'adresse IP du serveur de fichiers cible.
    • Port : Saisissez le numéro de port du serveur de fichiers cible.
    • Utilisateur et Mot de passe : Saisissez ces informations de connexion si le serveur de fichiers cible nécessite un compte/mot de passe
    • Page de code : Sélectionnez le codage des caractères du serveur de fichiers cible pour afficher correctement les noms de fichiers.
    • Nom du profile : Saisissez un nom unique afin d'identifier facilement ce serveur de fichiers.
    • Utiliser une connexion WebDAV HTTPS : Cette option n'est disponible que lorsque vous configurez un protocole WebDAV et que le client et le serveur de fichiers prennent tous deux en charge WebDAV HTTPS.
    • Autoriser une seule connexion par utilisateur : Cette option n'est disponible que si vous configurez un protocole FTP ou SFTP. Cochez cette option pour optimiser la connectivité si le serveur possède des limitations relatives au nombre de connexions.
  4. Cliquez sur Appliquer pour enregistrer les paramètres. Le serveur de fichiers cible s'affiche dans le volet de gauche.

Pour modifier un service cloud existant ou un profil de serveur de fichiers :

  1. Accéder à Outils > Connexion à distance > Liste de connexion.
  2. Cliquez sur le service cloud ou le serveur de fichiers cible pour les gestions suivantes :
    • Modification : Cliquez sur Modifier, modifiez la configuration du profil actuel, et cliquez sur OK.
    • Déconnexion : Cliquez sur Déconnecter pour mettre un terme à la connexion.
    • Reconnexion : Cliquez sur Connecter pour rétablir la connexion. (Vous pouvez également cliquer sur la cible dans le volet de gauche de File Station pour la reconnexion.)
    • Suppression : Cliquez sur Supprimer, puis confirmez l'abandon du profil. (Vous pouvez également cliquer sur la cible dans le volet de gauche de File Station et cliquez sur Supprimer pour la suppression.)

Remarque :

  • Chaque utilisateur peut se connecter simultanément à un maximum de 100 services cloud et serveurs de fichiers.
  • Veillez à sélectionner le bon code de page afin d'éviter les erreurs imprévues. Unicode est recommandé si vous ne connaissez pas la page de code du serveur.
  • Certaines fonctions de File Station (par exemple, compresser dans, extraire, aperçu, etc.) ne peuvent pas être appliquées aux fichiers stockés sur des services cloud ou des serveurs de fichiers.
  • Les vitesses de transfert de chaque service cloud ou serveur de fichiers connecté sont soumises aux paramètres de limite de vitesse de l'utilisateur.
  • Chaque utilisateur est limité à trois transferts de fichiers simultanés à travers tous les services cloud ou serveurs de fichiers connectés.
  • Pour configurer les paramètres WebDAV, WebDAV HTTPS et des services cloud, accédez à Panneau de configuration > Réseau > Général > Proxy. Si vous avez modifié les paramètres de proxy pendant la connexion à un service cloud ou un serveur de fichiers, déconnectez-vous, puis reconnectez-vous.
  • La modification des informations des services cloud ou des serveurs de fichiers connectés n'a aucune incidence sur l'état de la connexion.
  • Pour de libérer des ressources de connexion, un service cloud/serveur de fichiers est automatiquement déconnecté si les fichiers contenus demeurent inchangés pendant plus de 30 minutes. Cliquez sur le service cloud/serveur de fichiers pour rétablir la connexion.
  • Le système ne peut pas afficher des fichiers/dossiers qui adoptent les formats de nom suivants :
    • Commençant par ._ ou .SYNOPPSDB
    • Comprenant /
    • Nommés exactement .    ..    '    "    .DS_Store    @eaDir    @sharebin    @tmp    .SynologyWorkingDirectory
  • La connexion distante n'est prise en charge qu'en mode à privilèges élevés. Pour utiliser cette fonction, accédez au paquet Docker sur votre DSM hôte, sélectionnez le Docker DSM pour lequel vous souhaitez activer des dossiers partagés chiffrés, cliquez sur Modifier et cochez la case en regard de Exécuter le conteneur à l'aide de privilèges élevés.

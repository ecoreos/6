Envio de Registos

Se dispõe de um servidor syslog, pode seguir as instruções abaixo para configurar o seu DiskStation como um cliente syslog e transmitir registos de sistema local do seu DiskStation para um servidor syslog para monitorização e gestão centralizadas.

Para enviar registos para um servidor syslog:

  1. Aceda a Centro de Registos > Envio de Registos.
  2. Assinale a caixa Enviar registos para um servidor syslog.
  3. Introduza o nome de anfitrião ou o endereço IP do servidor syslog em Servidor.
  4. Especifique um número de porta para a receção de mensagens do syslog em Porta.
  5. Selecione UDP ou TCP em Protocolo de transferência. Isto deve ser coincidente com o protocolo utilizado pelo servidor syslog.
  6. Selecione BSD (RFC 3164) ou IETF (RFC 5424) em Formato do Registo para enviar os registos utilizando o formato selecionado.

    7. Nota:

    • Por predefinição, a porta é 514. Deve definir as regras de encaminhamento de porta adequadas se pretender atribuir de novo um número de porta diferente a um servidor syslog.
    • O UDP (Protocolo Datagram do Utilizador) é um protocolo de ligações baseadas em mensagem mais simples, que é mais rápido mas tem falhas a nível de segurança e de correção de erros. O TCP (Protocolo de Controlo de Transmissão) oferece garantia da entrega, em que utiliza um controlo de fluxo para determinar se todos os pacotes foram enviados com êxito até serem processados novos pacotes.
    • A ligação segura (SSL) apenas está disponível com a utilização do TCP.
    • Se escolher o TCP e pretender proteger a ligação syslog com SSL, pode assinalar a opção Ativar ligação segura (SSL). Em seguida, clique em Importar certificado e selecione o certificado para transferir registos para o servidor syslog em segurança.
  7. Clique em Enviar registo de teste para confirmar se o servidor syslog recebe o registo com êxito. Se o servidor syslog não receber o registo de teste, verifique se as definições estão corretas.
  8. Clique em Aplicar para guardar as definições.

Filtrar Registos

No separador Filtrar, pode especificar que tipos de registos são enviados para o servidor syslog, de acordo com a categoria (por exemplo, sistema, ligação, cópia de segurança) ou a gravidade (por exemplo, aviso, erro, informação).

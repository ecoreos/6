Gestor de High Availability

High Availability refere-se a uma solução de esquema de servidor concebida para reduzir as interrupções de serviços provocadas por avarias no servidor.O Synology High Availability utiliza dois servidores para formar um “cluster de high-availability” em que um servidor funciona como “servidor activo” e outro servidor funciona como “servidor passivo” em espera.Para implementar e criar um cluster de high-availability, são necessários dois servidores Synology compatíveis.

Um grupo High-availability tira partido dos seguintes componentes e conceitos:

  • Conceito: A implementação de Synology High Availability requer dois servidores Synology para criar um cluster de high-availability.Um servidor funciona como “servidor activo” e outro funciona como “servidor passivo”.Quer os servidores activo e passivo no cluster high-availability estão ligados às mesmas unidades de expansão, e neles partilham o mesmo armazenamento.Para garantir a integridade de dados, apenas o servidor activo pode gerir o fluxo de trabalho.Caso ocorram avarias no servidor activo, o servidor passivo estará pronto a assumir controlo de todos os serviços.Assim, o grupo High-availability continuará a assegurar os serviços normalmente, minimizando o tempo de inactividade do servidor.Para mais detalhes sobre a mudança, consulte Mudança.
  • Servidor activo: O servidor activo gere a provisão de todos os serviços.No caso de ocorrerem erros no servidor activo, o utilizador ou o sistema podem iniciar uma mudança para o servidor passivo.Neste caso, a provisão dos serviços será transferida para o servidor passivo e, a partir desse momento, o servidor activo passa a ser o servidor passivo.
  • Servidor passivo: Em condições normais, o servidor passivo continua em modo de espera.No caso do servidor activo não conseguir providenciar os serviços, o utilizador ou o sistema podem iniciar uma mudança para o servidor passivo.
  • Ligação Heartbeat: Os servidores activo e passivo de um cluster de high-availability estão ligados por uma ligação de rede privada dedicada mencionada como "ligação Heartbeat".Se o contacto entre os servidores for interrompido durante um período de tempo específico, o sistema determinará se é necessária uma mudança.
  • Armazenamento partilhado do cluster high-availability: As drives nas unidades de expansão, ligadas pelos servidores activo e passivo, funcionam ao mesmo tempo como o armazenamento partilhado no cluster high-availability.Apenas o servidor activo tem permissões de escrita relativamente ao armazenamento.
  • Nome do servidor do cluster high-availability e definição de rede: O nome do servidor e definições de rede do cluster high-availability será determinado pelas definições correspondentes na DSM.Na rede, pode estabelecer ligação utilizando este nome de servidor e os serviços serão providenciados pelo servidor activo.Para obter mais informações acerca do das definições de rede, consulte aqui.

Encerrar e Substituir o Servidor

As seguintes instruções ajudá-lo-ão durante o procedimento de encerramento, ou mudança de um servidor num cluster de high-availability.

Para encerrar o cluster de high-availability com Gestor de High Availability:

Quando desejar encerrar um cluster de high-availability para transportar os servidores para um novo local físico ou realizar manutenção, siga estas instruções.

  1. Inicie a sessão no DSM.
  2. Vá até ao Gestor de High Availability.
  3. Clique no botão marcado Gerir.
  4. Seleccione Encerrar cluster de high-availability.
  5. É exibida uma mensagem de confirmação.Clique em OK para confirmar.

Para encerrar manualmente um cluster de high-availability:

O cluster de high-availability também pode ser encerrado usando manualmente os botões de energia em cada servidor.

  1. Prima longamente o botão de energia no servidor passivo até emitir um som, e aguarde até que o servidor encerre completamente.
  2. Prima longamente o botão de energia no servidor activo até emitir um som, e aguarde até que o servidor encerre completamente.

Nota:

  • Ligar Servidores:
    Quando pretender ligar de novo os servidores, deve ligar tanto o servidor activo como passivo ao mesmo tempo (ou com um intervalo de cinco minutos entre si).Se apenas ligar um dos servidores, este irá tentar ligar o outro servidor emitindo uma mensagem Despertar pela LAN.
  • Encerramento Anormal:
    No cluster de high-availability a opção de reiniciar após falha de energia está activada.Se os servidores encerrarem de forma anormal devido a uma falha de energia, ligar-se-ão automaticamente assim que for restaurada a energia.

Para substituir um servidor num cluster de high-availability:

Quando desejar remover um servidor e configurar outro servidor, siga estas instruções.

  1. Aceda à página Visão geral.
  2. Clique no botão Gerir.
  3. Seleccione Encerrar Servidor Activo ou Encerrar Servidor Passivo para encerrar o servidor que deseja substituir.
  4. Verifique as ligações do novo servidor.Depois de ligado o novo servidor, será marcado como “desconhecido” no Gestor de High Availability.
  5. Aceda novamente à página Visão geral.
  6. Clique no botão Gerir.
  7. Seleccione Recuperar servidor desconhecido para que “servidor desconhecido” seja adicionado ao cluster de high-availability e reconhecido no servidor passivo.

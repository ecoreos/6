Mudança

Para garantir o desempenho contínuo e minimizar o tempo de inatividade de serviços, pode mudar os serviços do servidor ativo de um cluster high-availability para o servidor em espera, passivo. Esta funcionalidade chama-se "Mudança" e, em alguns casos "Activação após-falha". Esta secção contém informações básicas destes dois conceitos.

Mudança

Mudança refere-se a uma mudança de um servidor ativo para servidor passivo, em espera de um high-availability cluster. Em condições normais, o servidor ativo gere todos os serviços. No entanto, no caso do servidor ativo avariar, você pode iniciar uma mudança e transferir todos os serviços para o servidor passivo, desde que o servidor passivos esteja a funcionar normalmente. Neste caso o servidor activo assume o papel do servidor passivo, e vice-versa.

Para iniciar manualmente uma mudança:

  1. Aceda à página Visão geral.
  2. Clique em Gerir > Mudar.

Nota:

Antes da mudança manual, recomendamos que configure um IP estático e inicie a sessão na DSM com este IP estático, para evitar perda da página DSM devido à alteração e IP depois de uma mudança.

Nota:

Quando uma mudança é iniciada, o servidor ativo assume primeiro o papel de servidor passivo e depois o servidor passivo passa a ser o servidor ativo. Assim, haverá um curto período de tempo no qual ambos os servidores são passivos e não poderão providenciar os serviços normalmente. O tempo para concluir a mudança varia em função do número e tamanho dos volumes ou iSCSI LUN (nível do bloco) ou o número e carga total de serviços e pacotes do cluster high-availability.

Ativação após-falha

Quando ocorrerem determinados erros, o sistema pode iniciar automaticamente uma mudança de serviços para minimizar o tempo sem serviço. Este evento chama-se "ativação após falha". O sistema poderá iniciar uma falha nas seguintes situações, assumindo que o servidor passivo está a funcionar normalmente:

  • Falha de energia:
    • O servidor activo foi encerrado ou reiniciado.
    • Ambas unidades de alimentação no servidor activo avariaram.
    • Sem fonte de alimentação.
  • Erro na unidade: Número de caminhos de unidade no servidor activo é inferior ao do servidor passivo.
  • Erro de serviço: Ocorreu um erro num serviço monitorizado.
  • Erro na ligação: Ocorreu um erro de ligação numa interface monitorizada.

Falha da Mudança

A mudança de serviços ou ativação após falha poderá falhar nas seguintes situações:

  • Ligação Heartbeat anormal: Estado da ligação do servidor activo-passivo não está normal.
  • Tamanho de memória diferente: Os servidores activo e passivo não têm o mesmo tamanho de RAM.
  • Falha de energia:
    • O servidor passivo foi encerrado ou reiniciado.
    • Ambas unidades de alimentação no servidor passivo avariaram.
    • Sem fonte de alimentação.
  • Erro na ligação: Ocorreu um erro de ligação numa interface monitorizada no servidor passivo.
  • Erro de serviço: Existem erros nos serviços monitorizados no servidor passivo.
  • Erro de ligação na unidade de expansão: A ligação de unidades de expansão nos dois servidores não está pela mesma ordem.
  • Erro na unidade: Número de caminhos de unidade no servidor passivo é inferior ao do servidor activo.
  • Atualizar o DSM: Quando instala uma atualização DSM, todos os serviços serão parados e irão retomar após a instalação da atualização DSM estar completa.

Se iniciar manualmente uma mudança, mas não for possível mudar os serviços para o servidor passivo, então o sistema não tentará mudar os serviços de novo para o servidor ativo.

Fonte de alimentação ininterrupta

Os high-availability clusters podem ser ligados a fontes de alimentação ininterruptas (UPS) para reduzir o risco de falhas de energia inesperadas. Se ocorrer falha de energia, os servidores receberão energia auxiliar de qualquer unidade UPS ligada. Quando a unidade UPS ligada ao servidor activo começar a ficar sem energia, o servidor passivo irá encerrar, e o servidor activo passa ao modo de segurança. Depois de retomada a energia, o servidor activo reiniciará e tentará ligar o outro servidor emitindo uma mensagem Wake-on-LAN.

  • Recomendamos que ligue ambos servidores e unidades de expansão à mesma UPS a que estão ligados os switches de rede.

Nota:

DiskStation é compatível com a ligação apenas a dispositivos UPS de rede. Para mais informações, consulte a ajuda em Painel de Controlo > Hardware > UPS.

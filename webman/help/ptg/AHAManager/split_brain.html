Split-brain

Quando um cluster de high-availability estiver a funcionar normalmente, só um dos servidores membros deverá ser o servidor activo. Neste caso, o servidor passivo detecta a presença do servidor activo através da ligação Heartbeat bem como a ligação de dados com as unidades de expansão partilhadas, para que a mudança não seja activada. Para além disso, o servidor passivo detecta o estado do armazenamento partilhado, para evitar assumir de forma errónea o papel do servidor activo quando o servidor activo actual ainda acede às unidades de expansão partilhadas de forma normal.

Com a perda das ligações Heartbeat e de todos os dados, ou o estado instável do armazenamento partilhado e do sistema, cada um dos dois servidores poderá assumir o papel de servidor activo. Esta situação é denominada como "split-brain". Neste caso, para evitar a corrupção de dados devido ao acesso simultâneo de ambos servidores, um servidor irá tentar continuar a fornecer serviços, ao passo que o outro servidor irá reiniciar e solicitar uma verificação de ligação posteriormente. As ligações dos utilizadores clientes podem ficar instáveis durante algum tempo nesta situação.

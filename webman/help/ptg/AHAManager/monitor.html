Monitorizar

Esta página contém três separadores que mostram o monitor do estado de sistema, serviço e interface de rede. Esta página permite-lhe monitorizar determinados alvos. Se um alvo monitorizado falhar no servidor activo, o sistema irá entrar em pós-falha no modo de servidor passivo em espera (desde que o servidor passivo esteja a funcionar normalmente). São apresentados nesta página os seguintes separadores:

Separador do Estado do Sistema

O estado de sistema é monitorizado por predefinição, e não pode ser desactivado. Será activada uma falha quando ocorram avarias de sistema no servidor activo. São apresentadas neste separador as seguintes secções:

  • Estado do servidor activo: Falha no caso de avaria no servidor activo, impedindo-o de fornecer serviços. Por exemplo, ambas unidades de alimentação do servidor avariam.
  • Conectividade do armazenamento: Falha quando uma unidade está ausente no servidor activo, mas presente no servidor passivo.

Separador de serviço

Exibe os serviços que podem ser monitorizados e respectivo estado. Falha será activada quando os serviços não funcionarem correctamente. São apresentadas neste separador as seguintes secções:

  • Serviço: Exibe os serviços que podem ser monitorizados.
  • Estado: Mostra se os serviços estão ou não em execução.
  • Monitorizar: Assinale a caixa de verificação junto a um serviço para começar a monitorizá-lo. O sistema irá entrar em pós-falha, caso o serviço seleccionado falhe.

Para monitorizar um serviço:

  1. Encontre o serviço que pretende monitorizar. Certifique-se que o estado do serviço está normal.
  2. Assinale a caixa de verificação junto ao mesmo.
  3. Clique em Aplicar para guardar as definições.

Nota:

Os serviços que podem ser monitorizados são os seguintes: Serviço de Ficheiros Windows, iSCSI Target, FTP, Serviço de Ficheiros Mac, Serviço NFS.

Separador Interface de Rede

Apresenta todas as interfaces de rede lógica que possam ser monitorizadas. Falha será activada caso a ligação de rede seja anormal. (Consulte as Notas da seguinte forma.) São apresentadas neste separador as seguintes secções:

  • Interface de Rede: Exibe as interfaces que podem ser monitorizadas.
  • Monitorizar: Assinale a caixa de verificação junto a uma interface para começar a monitorizá-la.

Nota:

Se a caixa de verificação Monitor estiver assinalada, o sistema irá entrar em modo de pós-falha no servidor passivo quando a ligação de rede do servidor activo falhar ou funcionar incorrectamente e a ligação de rede do servidor passivo estiver a funcionar normalmente.

  • A pós-falha será activada apenas se o servidor passivo tiver mais ligações saudáveis. Por exemplo, se o servidor activo tiver mais do que três ligações e uma falhar, o sistema irá activar uma pós-falha apenas se o servidor passivo possuir três ou mais ligações saudáveis.
  • As ligações combinadas com Link Aggregation contam como uma ligação. Por exemplo, se estiverem combinadas duas ligações e uma delas falhar, mas a outra ainda funcionar, a ligação irá ainda assim contar como saudável.

Para monitorizar uma interface:

  1. Encontra interface que pretende monitorizar. Certifique-se que os cabos foram correctamente ligados às interfaces quer no servidor activo, quer no servidor passivo.
  2. Assinale a caixa de verificação junto ao mesmo.
  3. Clique em Aplicar para guardar as definições.

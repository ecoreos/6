Limitações e Requisitos do Sistema

A implementação do Synology High Availability requer dois servidores Synology idênticos para funcionarem como servidor ativo e passivo. Antes que os dois servidores sejam combinados num cluster de high-availability, o sistema verificará as seguintes limitações de hardware e de software para garantir a criação de um cluster de high-availability.

Requisitos do Sistema

Tenha em atenção as seguintes diretrizes sobre software e hardware:

  • Servidor Synology: Quer os servidores activo e passivo têm de ser modelos idênticos.
  • Unidade(s) de expansão: Quer os servidores activo e passivo têm de estar ligados correctamente às mesmas unidades de expansão.
  • Número e Tipo de Unidades: O tipo de unidade deve ser apenas SAS. Antes de instalar o DSM, tem de instalar pelo menos 3 unidade na(s) unidade(s) de expansão.
  • Dispositivo USB Externo: Apenas o armazenamento USB é suportado. Outros dispositivos USB externos (por ex., UPS, dongle Wi-Fi, Altifalante, Impressora, dongle Bluetooth e dongle DVB) não são suportados. O armazenamento USB pode ser usado apenas no servidor activo, e não pode ser mudado para o servidor passivo.

Ambiente de Rede

Tenha em atenção as seguintes diretrizes sobre definições de rede quando tentar formar um cluster de high-availability.

  • Definições de Rede: A ligação Heartbeat entre os dois servidores tem de estar configurada correctamente. Ambos os servidores têm de pertencer à mesma subrede, e é necessário configurar endereços IP estáticos após a instalação.
  • Número de Portas LAN: Ambos os servidores devem ter o mesmo número de portas LAN. Se um dos seus modelos DiskStation estiver equipado com interfaces de placas de rede adicionais, a outra tem de estar equipada com as mesmas interfaces de placa de rede.
  • Restrições de Ligação: Rede Wi-Fi não é compatível com o cluster high-availability.

Nota:

  • O servidor NTP será automaticamente activado no servidor activo.

Limitações de Pacote e Software

Consulte as limitações relacionadas com o pacote e software abaixo:

  • O cluster high-availability não é compatível com os seguintes pacotes e funcionalidades:
    • Servidor DVBLink
    • A funcionalidade para reiniciar/encerrar o cluster de high-availability no CMS
    • Time Backup
  • O cluster high-availability com armazenamento partilhado não pode ser configurado como nem usado juntamente com um cluster high-availability com base em replicação.

Unidades de Expansão Específicas

Quer os servidores activo e passivo têm de estar ligados às unidades de expansão específicas. Há dois modelos em cada unidade de expansão.

Para configurar/adicionar unidades de expansão:

  • Um módulo contém duas portas; a porta de entrada, no lado esquerdo, e a porta de saída no lado direito. Ligue um cabo SAS da porta de saída de um módulo à porta de entrada do módulo da unidade de expansão seguinte.
  • As unidades de expansão têm de estar ligados em série com os módulos nos lados correspondentes para cada servidor.
  • Quer os servidores activo e passivo têm de estar ligados às unidades de expansão partilhadas pela mesma ordem.

Notas:

Para obter informações detalhadas sobre a instalação da unidade de expansão, não se esqueça de consultar o Guia de Instalação para os seus servidores e unidades de expansão.

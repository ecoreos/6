Visão Geral

Esta página apresenta a informação básica sobre o cluster de high-availability e dos servidores membros. Além disso, esta página oferece ferramentas para realizar manutenção de clusters de high-availability e dos servidores membros.

Estado do Cluster de High-Availability

Este campo apresenta o estado geral do seu cluster de high-availability, alertando-o para qualquer erro. A tabela seguinte apresenta as possíveis condições identificadas neste campo:

Estado Descrição
Saudável O estado geral do seu cluster de high-availability é bom.
A carregar O sistema está a carregar o estado do cluster. Mudança não pode ser realizado até que esteja concluído o carregamento de estado. Consulte Mudança.
A Configurar O sistema está a ser configurado. Mudança não pode ser realizado até que esteja concluída a configuração.
Atenção O estado do cluster HA é anormal, mas os serviços ainda estão a funcionar. Mudança pode não ter sido realizada nesta situação.
Perigo Quer o servidor activo e passivo estão a avariar, ou o armazenamento encravou. O cluster de high-availability não poderá funcionar normalmente. Consulte o estado de todos os espaços de armazenamento (por exemplo, RAID Groups, Grupos de Discos, volumes e iSCSI LUNs). Mudança não pode ser realizada nesta situação.

Mensagens de Aviso

A tabela que se segue apresenta mensagens de aviso que podem aparecer quando ocorre um erro no sistema. Quando aparece mais do que uma mensagem de aviso, pode navegar entre as mesmas com os botões avançar/recuar.

Aviso/Erro Descrição
Não existe servidor passivo neste cluster de high-availability. A informação do servidor passivo não está disponível. Certifique-se que o servidor passivo está ligado correctamente. Neste caso, a mudança não estará disponível.
Falha ao detectar servidor passivo Este erro poderá ocorrer porque o servidor passivo está off-line ou os servidores ativo e passivo não podem ligar-se através da ligação Heartbeat. Neste caso, a mudança não estará disponível.
Falha ao realizar failover Foi activado um failover no servidor activo, mas falhou. Neste caso, o servidor activo não irá tentar novo failover. Para eliminar este erro, proceda da seguinte forma:
  1. Experimente solucionar os problemas no high-availability cluster.
  2. Reinicie o servidor passivo.
  3. Realize uma mudança clicando em Gerir > Switchover.
Erro de alvo monitorizado Ocorreu um erro num ou mais alvos monitorizados, incluindo estado de sistema, serviço ou interface de rede.
Ocorreram erros na ligação entre as unidades de expansão e servidor activo/passivo. As unidades de expansão não estão ligadas correctamente ao servidor activo ou passivo. Certifique-se que a ligação ao servidor activo está correcta, e que configura a mesma ligação ao servidor passivo. Neste caso, a mudança não estará disponível.
Um volume está degradado Existe um espaço de armazenamento (por exemplo, RAID Group, Grupo de Discos, volume, iSCSI LUN e cache SSD) está degradado. Verifique a página Estado de Armazenamento.
Um volume envravou Existe um espaço de armazenamento (por exemplo, RAID Group, Grupo de Discos, volume, iSCSI LUN e cache SSD) que encravou. Verifique a página Estado de Armazenamento. Pós-falha será automaticamente realizada caso o servidor passivo consiga reconhecer as unidades que o servidor activo não conseguiu aceder. Caso contrário, a mudança ficará indisponível.
Perdeu-se a ligação de pulsação Este erro poderá ocorrer devido a uma falha na ligação Pulsação. Neste caso, a mudança não estará disponível.
Avaria no hardware Este erro poderá ocorrer devido a uma falha no hardware (por ex., unidade de alimentação ou ventoinha de sistem) no servidor activo ou servidor passivo...
RAM física de tamanho diferente Este erro poderá ocorrer devido a uma diferença física no tamanho da RAM nos servidores activo e passivo. Neste caso, a mudança não estará disponível.
Nenhum IP estático atribuído a este cluster de high-availability Este erro poderá ocorrer devido à ausência de um IP estático no cluster high-availability. Neste caso, a mudança não estará disponível.
Não existe IP estático na interface monitorizada Este erro poderá ocorrer devido à ausência de um IP estático na interface monitorizada. Neste caso, a mudança não estará disponível.
Recuperar servidor passivo Está em curso a recuperação do servidor passivo. Neste caso, a mudança não estará disponível.
A atualizar o servidor passivo O sistema está a actualizar o servidor passivo. Neste caso, a mudança não estará disponível.

Botão "Gerir" do Cluster de High Availability

Este botão está localizado sob a informação do cluster high-availability. Inclui as seguintes operações:

  • Mudança: Inicia uma mudança do servidor activo para o servidor passivo. Durante este processo, o servidor ativo interrompe os serviços e passa a ser o novo servidor passivo. Em seguida, o servidor passivo antigo torna-se no novo servidor ativo e começa a fornecer todos os serviços do cluster de high-availability. Esta opção não está disponível quando o servidor passivo tiver o estado erro ou não estiver selecionado.
  • Atualizar o DSM: Abre a página Actualização do DSM do Painel de Controlo. A partir daqui, os dois servidores podem ser atualizados.
  • Encerrar cluster de high-availability: Encerra os servidores activo e passivo.
  • Encerrar servidor ativo: Encerra o servidor activo. Os serviços serão mudados caso o servidor passivo esteja saudável.
  • Encerrar servidor passivo: Encerra o servidor passivo.
  • Reiniciar servidor activo: Reinicia o servidor activo. Os serviços serão mudados caso o servidor passivo esteja saudável.
  • Reiniciar servidor passivo: Reinicia o servidor passivo.
  • Recuperar o servidor desconhecido: Recupera o servidor passivo caso o servidor passivo serja substituído ou ainda seja identificado como desconhecido.
  • Beep de sistema desligado: Pára o sinal sonoro do servidor activo e passivo quando ocorre um erro. O sistema ainda emitirá um bip no servidor activo ou servidor passivo caso ocorra outro erro mais tarde.

Nota:

  • Se encerrar o servidor passivo ou não exista um servidor passivo, os serviços não podem ser mudados quando ocorrem erros no servidor activo.

Diagrama e Informação do Servidor

O diagrama do servido mostra o estado do servidor. A localização corresponde à lateral dos módulos aos quais o servidor está ligado. Passe o rato por cima para obter mais detalhes.

Inclui as seguintes situações:

  • Não existe servidor passivo neste cluster de high-availability: Apresenta uma imagem em branco.
  • Falha ao detectar servidor passivo: Apresenta uma imagem de servidor desfocada.
  • Ponto de exclamação: Possível avaria de hardware, ou informação de servidor indisponível devido a falha da ligação Pulsação.

Nota:

  • Se for mostrado "desconhecido" no outro servidor, significa que o servidor não pertence ao cluster high-availability. Clique em Gerir e seleccione Recuperar servidor desconhecido para corrigir este problema.

A informação do servidor inclui as seguintes entradas. Mostra N/A caso não seja detectado qualquer servidor passivo.

  • Número de Série: Número de série do servidor Synology.
  • Estado da Ventoinha: Estado das ventoinhas de arrefecimento do servidor. Se uma ventoinha avariar, surgirá um aviso. Neste caso, contacte o seu distribuidor local para obter assistência.
  • Temperatura: Temperatura do servidor Synology.
  • Estado da Fonte de Alimentação: Indica se as fontes de alimentação do servidor estão todas a funcionar normalmente. Se o servidor estiver equipado com fontes de alimentação redundantes, surgirá um aviso se uma delas avariar.
  • Memória física: O tamanho da RAM física do servidor Synology. Mudança não pode ser realizada caso os dois servidores não tenham o mesmo tamanho físico de RAM.

Estado de ligação entre servidores

Esta secção contém imagens entre os servidores activo e passivo. Inclui as seguintes situações:

  • Sem imagem: Indica que nenhum servidor passivo foi detectado.
  • Imagem estática: Indica uma ligação Pulsação correcta.
  • Ponto de exclamação: Indica uma comunicação anormal entre os dois servidores, ou uma ligação Pulsação incorrecta.

Estado da interface de rede

Esta secção contém imagens entre os servidores e rede de dados. Inclui as seguintes situações:

  • Nenhum ponto de exclamação: Indica que não foi detectada qualquer falha de rede.
  • Ponto de exclamação: Indica uma falha na interface de rede monitorizada.

DIagrama e Informação do Armazenamento Partilhado

O diagrama de armazenamento partilhado mostra o estado da ligação entre os servidores e o armazenamento partilhado. Passe o rato por cima para obter informações básicas.

Inclui as seguintes situações:

  • Nenhum ponto de exclamação: Indica que nenhum erro foi detectado.
  • Ponto de exclamação: Indica um estado de ligação anormal nas unidades de expansão.

As informações básicas do armazenamento partilhado incluem as seguintes entradas. Aceda ao Gestor de Armazenamento para informações mais detalhadas.

  • Estado da Ventoinha: Mostra o estado das ventoinhas das suas unidades de expansão. Se uma ventoinha avariar, surgirá um aviso. Neste caso, contacte o seu distribuidor local para obter assistência.
  • Estado da energia: Indica se as fontes de alimentação do servidor estão todas a funcionar normalmente. Aparecerá um aviso caso uma delas avarie.

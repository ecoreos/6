iSCSI LUN*

iSCSI é um serviço de rede de armazenamento que facilita a transferência de dados em redes locais ou em longas distâncias. Ao mapear iSCSI LUNs para iSCSI Targets, os clientes podem aceder aos espaços de armazenamento como se fossem discos locais. Um iSCSI LUN (número de unidade lógica) pode ser mapeado num ou em vários iSCSI Targets para providenciar serviços de armazenamento iSCSI. Aceda a Gestor de Armazenamento > iSCSI LUN para gerir iSCSI LUNs.

Nota:

O número de iSCSI LUNs suportados varia nos modelos DiskStation. Para obter uma especificação detalhada do produto, consulte www.synology.com.

DiskStation suporta actualmente os seguintes tipos de iSCSI LUNs:

  • iSCSI LUN (Ficheiros Normais): O iSCSI LUN é criado designando uma parte do espaço do volume para a sua utilização. Poderá expandir o seu tamanho quando for necessário. Este tipo de iSCSI LUN fornece flexibilidade de gestão de capacidade dinâmica com Thin Provisioning, suporte de aceleração de hardware VMware VAAI, suporte Windows ODX, imagem instantânea LUN e clonagem LUN.
  • iSCSI LUN (Nível de Bloqueio) - LUN Único em RAID: O iSCSI LUN é criado utilizando todo o espaço de um volume. Pode reparar ou expandi-lo ou alterar o tipo de RAID, tal como utilizar um volume. Consulte a página Volume para obter mais informações.
  • iSCSI LUN (Nível de Bloqueio) - Múltiplos LUNs em RAID O iSCSI LUN é criado designando uma parte específica do espaço de um Grupo de Discos. Pode expandir o iSCSI LUN posteriormente se existir espaço suficiente no Grupo de Discos. Consulte a página Grupos de Discos para obter mais informações.
  • iSCSI LUN (Nível de Bloqueio): Este tipo de iSCSI LUN é criado num RAID Group e fornece flexibilidade da gestão de capacidade dinâmica, para além de um desempenho de acesso otimizado. Pode expandir este tipo de iSCSI LUN mais tarde utilizando espaço não atribuído no RAID Group. Consulte a página do RAID Group para obter mais informações.

Nota:

iSCSI LUNs com funcionalidades LUN avançadas apenas estão disponíveis no sistema de ficheiros ext4/Btrfs.

Para criar um iSCSI LUN:

  1. Clique em Criar.
  2. Selecione o tipo de iSCSI LUN que pretende criar.
  3. Siga o assistente para concluir o processo.

Nota:

As funcionalidades Advanced LUN só podem ser ativadas durante a criação inicial de iSCSI LUN. Esta definição não pode ser alterada depois.

O tamanho da unidade de alocação Advanced LUN pode ser otimizado para determinadas funcionalidades do servidor da aplicação, como o Microsoft Exchange ou SQL Server ou servidores de virtualização e tecnologias de aceleração de armazenamento, como o Microsoft Windows ODX ou VMware VAAI, procedendo de uma das seguintes formas:

  • Defina o tamanho da unidade de alocação durante a criação da primeira LUN avançada de um volume.
  • Defina o tamanho da unidade de alocação em Configurar antes de criar a primeira LUN avançada de um volume.

Recomendamos que especifique o tamanho da unidade de alocação para a sua aplicação quando utilizar os métodos de trabalho da ligação direta do seu servidor da aplicação ou RDM (pass-through) com Advanced iSCSI LUNs no DiskStation. Para desfrutar da tecnologia Synology Advanced LUN para clonagem no Windows ODX, sugerimos que formate o volume NTFS com o tamanho da unidade de alocação igual (ou maior do que) aquele que especificou no DSM.

Nota:

A unidade de alocação do Advanced LUN de cada volume apenas pode ser definida uma vez.

Para remover um iSCSI LUN:

Quando remover um iSCSI LUN, todos os dados no iSCSI LUN serão eliminados. As ligações entre o iSCSI LUN e qualquer iSCSI Targets mapeado também serão perdidas.

  1. Selecione o iSCSI LUN que pretende remover.
  2. Clique no botão Remover e siga o assistente para concluir o processo.

Nota:

Eliminar VAAI LUNs no DiskStation não liberta espaço de disco de imediato. Os dados são movidos para uma pasta reservada ao sistema designada @EP_trash e eliminados aos bocados. Por exemplo, se um VAAI LUN de 100 GB fo apagado de DiskStation, irá demorar cerca de 15-20 minutos até libertar os 100GB de espaço no volume de disco.

Para forçar a montagem de iSCSI LUN (Nível de Bloqueio):

Se a cache num iSCSI LUN (Nível de Bloqueio) estiver em falta, o sistema não irá montar o iSCSI LUN (Nível de Bloqueio) quando arrancar. Se a cache não for recuperável e pretender aceder ao iSCSI LUN (Nível de Bloqueio), pode clicar em Forçar montagem.

Nota:

Esta função apenas está disponível em determinados modelos.

Para editar um iSCSI LUN:

  1. Selecione o iSCSI LUN que pretende editar.
  2. Clique em Editar.
  3. Edite as propriedades pretendidas.
  4. Clique em OK para guardar as definições.

Nota:

Para evitar a perda de dados quando editar a capacidade do iSCSI LUN, apenas o pode tornar maior do que o tamanho atual.

Para obter uma imagem instantânea de iSCSI LUN:

Uma imagem instantânea de iSCSI LUN é uma cópia só de leitura de um iSCSI LUN, de um momento específico.

  1. Selecione o iSCSI LUN da qual deseja obter uma imagem instantânea.
  2. Clique em Imagem Instantânea e escolha Obter uma Imagem Instantânea.
  3. Introduza uma descrição para esta imagem instantânea.
  4. Escolha um dos seguintes tipos de imagens instantâneas, de acordo com as suas necessidades:
    • Consistente com a Aplicação Este tipo de imagem instantânea só pode ser obtida com o Synology Snapshot Manager instalado no seu servidor Windows ou VMware vCenter Server, dependendo do seu ambiente operativo para comunicar com o DSM. Quando são acionadas imagens instantâneas consistentes com a aplicação no DSM, são notificadas as aplicações e todos os dados da memória são enviados para o LUN onde a imagem instantânea é obtida, para garantir a consistência dos dados.
    • Consistente com Avaria Se o Synology Snapshot Manager não estiver instalado em qualquer dos seus clientes, apenas pode capturar imagens instantâneas consistentes com avaria de LUNs. Quando são acionadas, as aplicações não serão notificadas para enviar os dados da memória para o LUN e, dessa forma, não é garantida a consistência dos dados.
  5. Clique em OK.

Para restaurar um iSCSI LUN:

Selecione uma imagem instantânea de um momento específico para restaurar os dados num iSCSI LUN.

  1. Selecione o iSCSI LUN que pretende restaurar.
  2. Clique em Imagem Instantânea e escolha Gestor de Imagens Instantâneas.
  3. Selecione uma imagem instantânea para a qual deseja restaurar e clique em Restaurar.

Para clonar um iSCSI LUN ou uma imagem instantânea:

Crie uma cópia gravável de um iSCSI LUN ou de uma imagem instantânea de iSCSI LUN.

  • Selecione um iSCSI LUN na lista e clique em Clonar.
  • Selecione uma imagem instantânea de iSCSI LUN em Gestor de Imagens Instantâneas e clique em Clonar.

Nota:

  • Em certos modelos, terá de aceder ao Centro de Pacotes e instalar Instantâneo e Replicação para funções como o restauro de iSCSI LUNs, torar e clonar instantâneos iSCSI LUN.
  • Se existir pelo menos uma sessão iSCSI ligada associada ao LUN e for utilizado pelo Windows Server ou VMware ESXi Server, recomendamos a instalação da extensão do Synology Snapshot Manager para evitar possíveis inconsistências de dados, caso contrário, deve parar quaisquer aplicações e terminar essas sessões nos iniciadores antes de continuar.
  • As funções de imagem instantânea, restaurar e clonagem LUN apenas estão disponíveis em iSCSI LUNs com funcionalidades LUN avançadas.
  • Se obteve uma imagem instantânea ou criou um clone e o estado foi Anormal/Avariado, elimine esta imagem instantânea/clone e crie um outro.
  • Todas as alterações efetuadas a um LUN depois de criado uma imagem instantânea perdem-se quando restaura o LUN para esta imagem instantânea.
  • Se o estado de iSCSI LUN mudar para Indisponível, significa que alguns erros relacionados com dados ocorreram ao aceder a iSCSI LUN. Para evitar a perda grave de dados no seu sistema, este iSCSI LUN ficará temporariamente inacessível. Contacte a Synology para obter suporte assim que possível.

Glossário:

  • Thin Provisioning: Thin Provisioning é um método para a otimização da utilização de armazenamento atribuindo espaço de armazenamento de uma forma dinâmica e a pedido. Esta definição apenas está disponível para iSCSI LUN (Ficheiros Normais).
  • VMware VAAI: O VMware vStorage APIs for Array Integration (VAAI) pode aliviar o trabalho de operações normais em matrizes de armazenamento virtuais para o dispositivo iSCSI no ambiente VMware vSphere e otimizar o desempenho de armazenamento.
  • Windows ODX: Uma nova tecnologia de transferência de dados desenvolvida pelo Windows no Windows Server 2012 e 8 designada como Offload Data Transfer (ODX). O ODX melhora o desempenho de transferência aliviando alguma carga de trabalho na matriz de armazenamento.
    Se a origem e o destino dos dados estiverem localizados nos LUNs localizados no mesmo volume, a clonagem será processada pela tecnologia Synology Advanced LUN que poupa até 99,9% do consumo do espaço de armazenamento e acelera a clonagem ao mesmo tempo.
  • Mapeamento iSCSI Target: Mapear o iSCSI LUN para um ou mais iSCSI Targets.
  • Synology Snapshot Manager: O Synology Snapshot Manager é uma extensão de software que permite a obtenção de imagens instantâneas consistentes com a aplicação, diretamente do DiskStation Manager (DSM).
    • Para o ambiente VMware, será necessário instalar o Synology Snapshot Manager no servidor Windows onde o VMware vCenter Server está instalado. Quando uma imagem instantânea é acionada no DSM, o VMware vCenter Server será notificado e envia todos os dados da memória para o LUN onde a imagem instantânea é obtida, para garantir a consistência dos dados. Depois de a imagem instantânea estar concluída, o VMware vCenter Server retoma o funcionamento I/O normal do armazenamento de dados VMware. O Synology Snapshot Manager for VMware vCenter Server suporta vSphere 5.1 ou versões mais recentes.
    • Para o ambiente Windows, será necessário instalar o Synology Snapshot Manager no seu servidor Windows. Quando uma imagem instantânea é acionada no DSM, o Synology Snapshot Manager aproveita a tecnologia do Microsoft Volume Shadow Copy Service (VSS) para produzir cópias de dados consistentes naquele momento. O Synology Snapshot Manager for Windows suporta o Windows Server 2008 R2, 2012 e 2012 R2.
    • O Synology Snapshot Manager pode ser transferido gratuitamente em www.synology.com.

Geral

O Gestor de Armazenamento permite a utilizadores gerir todos os espaços de armazenamento nos seus produtos Synology. Para simplificar a gestão de armazenamento e criar espaços de armazenamento flexíveis com protecção de dados, o Synology DSM suporta vários tipos de grupos RAID.

RAID Group

Com DiskStation, pode combinar múltiplos discos rígidos numa única unidade de armazenamento chamada RAID Group. DiskStation fornece os seguintes tipos de RAID Groups:

  • Grupo RAID para volume único ou iSCSI LUN (Nível do Bloco):
    • Atribui toda a capacidade disponível a um único volume ou iSCSI LUN (Nível de Bloqueio).
    • Oferece o melhor desempenho mas menor flexibilidade de gestão do armazenamento.
    • Permite criar múltiplos iSCSI LUNs (Ficheiro Normal) nos volumes.
    • Suporta um máximo de um Conjunto RAID.
  • Grupo RAID para múltiplos volumes ou iSCSI LUNs (Nível do Bloco):
    • Suporta a criação de múltiplos volumes ou iSCSI LUNs (Nível do Bloco) num Grupo RAID.
    • Oferece melhor flexibilidade de gestão do armazenamento.
    • Permite criar múltiplos iSCSI LUNs (Ficheiro Normal) nos volumes.
    • Suporta a combinação de múltiplos Conjuntos RAID num RAID Group (quando configurado como RAID 5 ou RAID 6).

Tipo RAID

Quando criar um grupo RAID, primeiro seleccione o tipo de RAID. Diferentes tipos de RAID oferecem diferentes níveis de protecção de dados, funcionalidades de armazenamento e desempenho. DiskStation actualmente suporta os seguintes tipos RAID:

  • Básico: Usa um disco rígido para criar um grupo RAID. Grupos RAID básicos não oferecem redundância de dados.
  • JBOD*: Combina, pelo menos, 2 discos rígidos para criar um grupo RAID. Grupos RAID JBOD não oferecem redundância de dados. A capacidade disponível de um grupo RAID JBOD equivale à capacidade total de todos os discos incluídos no grupo. JBOD suporta a combinação de discos rígidos de vários tamanhos.
  • RAID 0*: Combina, pelo menos, 2 discos rígidos para criar um grupo RAID. RAID 0 não oferece redundância de dados. Se um único disco falhar, os dados no grupo RAID perdem-se. A capacidade disponível do um grupo RAID 0 é idêntica à capacidade total de todos os discos membro. RAID 0 suporta a combinação de discos rígidos de vários tamanhos.
  • RAID 1*: Combina 2-4 discos rígidos para criar um grupo RAID. O sistema irá escrever dados idênticos em cada disco rígido. RAID 1 oferece protecção de dados enquanto, pelo menos, um disco funcionar normalmente. A capacidade disponível do Grupo RAID 1 é idêntica ao tamanho do disco mais pequeno.
  • RAID 5*: Combina pelo menos três discos rígidos para criar um grupo RAID, registando tanto os dados como a informação de paridade em todos os discos. RAID 5 oferece redundância de dados. Se um disco falhar, o sistema poderá ser reconstruído usando a informação de paridade dos restantes discos. Suporta a combinação de múltiplos Conjuntos RAID quando criados num RAID Group para múltiplos volumes ou iSCSI LUNs (Nível do Bloco). A capacidade disponível de um Grupo RAID 5 = Capacidade total dos Conjuntos RAID. Capacidade dos Conjuntos RAID = (número de discos rígidos - 1) x (Capacidade do disco rígido mais pequeno)
  • RAID 6*: Combina pelo menos quatro discos rígidos para criar um grupo RAID, registando tanto os dados como a informação de paridade em todos os discos. RAID 6 oferece redundância de dados. Mesmo que dois discos falhem, o sistema poderá ser reconstruído usando a informação de paridade nos outros discos. Suporta a combinação de múltiplos Conjuntos RAID quando criados num RAID Group para múltiplos volumes ou iSCSI LUNs (Nível do Bloco). A capacidade disponível de um Grupo RAID 6 = Capacidade total dos Conjuntos RAID. Capacidade dos Conjuntos RAID = (número de discos rígidos - 2) x (Capacidade do disco rígido mais pequeno)
  • RAID 10*: Combina, pelo menos, quatro discos rígidos para criar um grupo RAID. O número de discos deve ser par. RAID 10 tem o desempenho de RAID 0 para além do mesmo nível de proteção de dados de RAID 1. A capacidade disponível do Grupo RAID 10 = (capacidade do disco rígido mais pequeno) X (número de discos rígidos / 2)

Nota: Tipos RAID com “*” só estão disponíveis em modelos NAS com baías de disco rígido suficientes.

Tipo de Volume

Dependendo do tipo de grupo RAID subjacente, existem dois tipos de volumes:

  • Volume no RAID Group para volume único ou iSCSI LUN (Nível de Bloqueio)
    • Atribui toda a capacidade de armazenamento num Grupo RAID para um único volume. Oferece o melhor desempenho mas menor flexibilidade de gestão.
    • Permite criar múltiplos iSCSI LUNs (Ficheiro Normal) nos volumes.
  • Volume do RAID Group para múltiplos volumes ou iSCSI LUNs (Nível de Bloqueio)
    • Atribui um parte designada da capacidade de armazenamento num RAID Group para criar um volume. Disponibiliza melhor flexibilidade de gestão, permitindo expandir o tamanho do volume ou criar volumes adicionais, conforme necessário.
    • Permite criar múltiplos iSCSI LUNs (Ficheiro Normal) nos volumes.

Formato do volume

Dependendo do sistema de ficheiro em utilização, há dois formatos de volume:

  • Formato do novo volume: Este formato utiliza o sistema de ficheiros Btrfs. O tamanho mínimo de um volume neste formato deverá ser pelo menos 10 GB. Suporta as seguintes novas funcionalidades com as quais pode gerir os seus dados mais facilmente:
    • Instantâneo de pasta partilhada: O instantâneo regista o estado completo dos dados no momento em que foi tirado. Se um utilizador modificar ou eliminar acidentalmente dados num volume com instantâneos, poderá rapidamente recuperar os dados ao momento anterior, quando foi tirado o instantâneo.
    • Quota de pasta partilhada: Pode configurar a capacidade máxima para cada pasta.
    • Quota de utilizador com base em pasta partilhada: Pode controlar quanto espaço dentro de uma pasta partilhada pode ser usada por cada utilizador.
    • Protecção avançada da integridade de dados: A pasta partilhada está protegida por estratégias CRC32 checksum e copy-on-write.
  • Formato tradicional do volume: Este formato utiliza o sistema de ficheiros ext4. O tamanho mínimo de um volume neste formato deverá ser pelo menos 10 GB. Uma vez que o sistema de ficheiros ext4 é suportado pela maioria dos produtos Synology, pode migrar facilmente o seu volume neste formato para outro produto Synology.

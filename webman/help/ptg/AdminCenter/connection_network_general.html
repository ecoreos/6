Geral

As opções genéricas de rede, como gateway predefinido, DNS server, proxy e IPv6 tunneling estão disponíveis em Painel de Controlo > Rede > Geral.

Renomear o Dispositivo DiskStation

Para renomear o seu dispositivo DiskStation, introduza o novo nome do servidor e clique em Aplicar.

O nome do servidor é sensível a maiúsculas e minúsculas e pode incluir de 1 a 15 caracteres, que podem ser letras, números, sinais de sublinhado e de subtração. O primeiro caractere do nome deve ser uma letra.

Especificação do Gateway Predefinido

Basta introduzir o endereço IP para o gateway predefinido e clicar em Aplicar para guardar a definição.

Especificação do Endereço IP do DNS Server

Pode optar por configurar manualmente o endereço IP do seu servidor DNS. Neste caso, o endereço IP do seu servidor DNS preferido tem de ser introduzido. Pode também ser introduzido um endereço IP de servidor DNS alternativo, para salvaguarda caso o servidor DNS tenha problemas. Depois de configurados os endereços IP do servidor DNS como desejado, clique em Aplicar para guardar as suas definições.

Nota:

Os campos Servidor DNS e gateway predefinido serão configurados automaticamente e não podem ser alterados se todos as interfaces da sua rede forem configuradas automaticamente ou caso utilize uma ligação PPPoE.

Ligação via Servidor Proxy

Para permitir que o seu dispositivo DiskStation seja ligado através de um servidor proxy, assinale Ligar via servidor proxy, introduza o Endereço e a Porta e depois clique em Aplicar.

Nota:

  • Clique em Definições avançadas para configurar o proxy HTTP, o proxy HTTPS e a informação de autenticação.
  • Por predefinição, as definições do proxy HTTPS são iguais às definições do proxy HTTP.
  • Se estiver assinalada a opção Ligar via servidor proxy, a Atualização DSM e o Centro de Pacotes ligam-se sempre à Internet através de um proxy.
  • Se estiver assinalada a opção Contornar servidor de proxy para endereços locais, certifique-se de que o servidor DNS utilizado pelo DiskStation pode resolver os pedidos de dispositivos na rede local.

Ativação de IPv6 Tunneling

Enquanto o IPv6 não substituir completamente o IPv4, o IPv6 tunneling é necessário para permitir aos anfitriões com endereços IPv6 acederem aos serviços de IPv4. Esta técnica proporciona uma forma de encaminhar o tráfego do IPv6 nas redes IPv4 existentes.
Se não tiver a certeza acerca da configuração do IPv6 ou encontrar alguns problemas de ligação, contacte o seu Fornecedor do Serviço de Internet.

Para ativar um Tunnel IPv6:

  1. Assinale a caixa de verificação Ativar tunnel para ativar o suporte de tunnel.
  2. Em Endereço do Servidor, introduza o nome do anfitrião ou o endereço IP do fornecedor do serviço de tunnel, por exemplo: anonymous.freenet6.net.
  3. Selecione se pretende Ligar anonimamente ou Ligar com uma conta existente.

Nota:

O seu endereço IPv6 externo pode ser atualizado automaticamente pelo fornecedor do serviço. No entanto, o dispositivo DiskStation não irá remover o endereço IPv6 antigo imediatamente.

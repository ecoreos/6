Atribui permissões NFS

Pode atribuir permissões NFS para qualquer pasta partilhada, permitindo que clientes Linux tenham acesso à mesma.

Para atribuir permissões NFS a uma pasta partilhada:

  1. Selecione a pasta partilhada que pretende editar na lista de pastas partilhadas.
  2. Clique em Editar > Permissões NFS.
  3. Clique em Criar para adicionar uma regra NFS.
  4. Aparece uma janela de pop-up. Defina as opções abaixo.
  5. Nome de anfitrião ou IP: Introduza o endereço IP do cliente NFS que terá acesso à pasta partilhada. Pode especificar um anfitrião de três formas:
    • Anfitrião Único: O nome do domínio totalmente qualificado ou um endereço IP.
    • Caracteres universais: *,* .synology.com
    • Redes IP: 203.74.205.32/255.255.252.0, /24
  6. Privilégio: Selecione as permissões de leitura/escrita para o cliente NFS.
  7. Segurança: Especifique o tipo de segurança a implementar.
    • AUTH_SYS: Utilize o UID (identificador do utilizador) do cliente NFS e GID (identificador do grupo) para verificar as permissões de acesso.
    • Autenticação Kerberos: Efetue a autenticação Kerberos quando o cliente NFS estabelecer ligação à pasta partilhada. O cliente só pode aceder à pasta partilhada depois de passar na autenticação Kerberos.
    • Integridade Kerberos: Efetue a autenticação Kerberos e assegure a integridade dos pacotes durante a transferência de dados.
    • Privacidade Kerberos: Efetue a autenticação Kerberos e encripte os pacotes NFS durante a transferência de dados, deste modo, evita a adulteração por parte de entidades mal intencionadas com o tráfego NFS.
  8. Squash: Este campo permite-lhe controlar os privilégios de acesso do utilizador do cliente NFS. Selecione um dos seguintes:
    • Sem mapeamento: Permite que todos os utilizadores do cliente NFS, incluindo os utilizadores root, mantenham os privilégios de acesso originais.
    • Mapear root para admin: Atribui privilégios de acesso aos utilizadores root do cliente NFS equivalentes aos privilégios de acesso do utilizador admin no seu sistema.
    • Mapear root para guest: Atribui privilégios de acesso aos utilizadores root do cliente NFS equivalentes aos privilégios de acesso do guest no seu sistema.
    • Mapear todos os utilizadores para admin: Atribui privilégios de acesso a todos os utilizadores do cliente NFS equivalentes aos privilégios de acesso do utilizador admin no seu sistema.
  9. Activar assíncrono: Marcar esta opção permite que DiskStation responda a pedidos de clientes NFS antes da conclusão de qualquer alteração nos ficheiros, permitindo um melhor desempenho.
  10. Permitir ligações a partir de portas sem privilégios (portas superiores a 1024): A marcação desta opção permite que os clientes NFS utilizem as portas sem privilégios (por exemplo, as portas superiores a 1024) quando estabelecem ligação a DiskStation.
  11. Permite aos utilizadores acederem a sub-pastas montadas： Marcar esta opção permite aos clientes NFS acederem a sub-pastas montadas.
  12. Clique em OK para terminar.
  13. Clique em OK para aplicar as permissões NFS.

Nota:

  • Quando o formato do nome do servidor é *.domain, o endereço IP do cliente NFS deve ter um registo DNS PTR correspondente, para permitir que o DiskStation encontre o nome *.domain ao pesquisar pelo correspondente endereço IP.

Tipos de Segurança:

Quando aceder a uma pasta partilhada através de NFS com uma conta de utilizador específica:

  • Se o tipo de segurança AUTH_SYS for implementado: O cliente tem de ter exatamente o mesmo UID (identificador do utilizador) e GID (identificador do grupo) numéricos no cliente NFS e DiskStation ou serão atribuídas ao cliente as permissões others quando aceder à pasta partilhada. Para evitar quaisquer conflitos de permissões, pode selecionar Mapear todos os utilizadores para admin de Squash ou conceder permissões "Everyone" (Todos) à pasta partilhada.
  • Se o tipo de segurança Kerberos (krb5, krb5i, krb5p) for implementado: Deve aceder a Win/Mac/NFS > Serviço NFS > Definições Kerberos > Mapeamento de ID para mapear o cliente NFS para um utilizador específico ou aderir a um domínio Windows/LDAP com a conta de utilizador correspondente, caso contrário, serão atribuídas ao cliente as permissões guest quando aceder à pasta partilhada.
  • Se o sistema de ficheiros do dispositivo externo no qual as pastas partilhadas são criadas for NTFS ou FAT, a opção Mapear todos os utilizadores para admin será aplicada de forma forçada.

Para utilizar os tipos de segurança Kerberos para ligar ao DiskStation, tem de ser configurada a autenticação Kerberos indo para Win/Mac/NFS > Serviço NFS > Definições Kerberos.

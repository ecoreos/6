LAN/Internet

Dependendo do seu modelo de DiskStation e das definições de rede, pode configurar as definições DHCP para LAN ou Internet em Painel de Controlo > Rede > Interface de rede. DHCP é um protocolo de rede para configuração automática de IP. Para permitir ao DHCP atribuir um IP para o sistema, clique em Editar e escolha Obter a configuração da rede automaticamente (DHCP) no separador IPv4.

Se estiver a utilizar um endereço IP do seu fornecedor de serviço de rede, escolha Utilizar a configuração manual e introduza manualmente o endereço IP e a máscara de sub-rede. Se não existir um servidor DNS ou gateway predefinido disponível, deixe o campo em branco.

Nota:

  1. Para garantir que as notificações por e-mail são entregues corretamente através do servidor SMTP, deve ser introduzido um IP do servidor DNS válido.
  2. Os Jumbo Frames são frames Ethernet com não mais do que os 1500 bytes padrão de Unidade Máxima de Transmissão (MTU), que permitem que a transmissão por Ethernet de ficheiros grandes seja mais eficiente. Só pode estar ativado em ambiente de redes Gigabit. Para garantir que o Jumbo Frame funciona corretamente, todos os computadores e dispositivos na rede a aceder à sua DiskStation devem suportar e utilizar o mesmo valor MTU.

VLAN

Quando o ambiente de rede requer definições VLAN, pode ativar a opção VLAN e introduzir um ID VLAN para o ambiente de rede no separador IPv4, na janela de Edição.

Nota:

  • Certifique-se de que o switch de rede está devidamente configurado, caso contrário o serviço da DiskStation poderá ficar inacessível.
  • Se não conseguir ligar ao DiskStation depois de modificar as definições acima, prima o botão RESET localizado no DiskStation para restaurar as predefinições.

Configuração do IPv6

Os dispositivos DiskStation suportam o esquema de conjugação dupla IPv4/IPv6. Isto significa que o IPv4 e o IPv6 podem funcionar simultaneamente no seu DiskStation. A definição predefinida para o IPv6 é Automático. Selecione uma interface de rede existente, clique no botão Editar e aceda ao separador IPv6 para alterar as definições.

Para configurar as definições de IPv6:

  1. Aceda a Painel de Controlo > Rede > Interface de rede.
  2. Selecione uma interface de rede existente e clique em Editar.
  3. No separador IPv6, selecione um tipo de configuração em Configuração do IPv6:
    • Automático: O seu DiskStation gera automaticamente um endereço IPv6, dependendo do Anúncio do router (RA, Router Advertisement) da LAN.
      • Modo sem monitorização de estado: Permite que os dispositivos ligados ao router configurem automaticamente um endereço IPv6 sem definir o servidor DNS de IPv6.
      • Modo sem monitorização de estado de DHCPv6: Permite que os dispositivos ligados ao router configurem automaticamente um endereço IPv6 e obtenham informações de DNS através de DHCPv6.
      • Modo de monitorização de estado: Permite que os dispositivos ligados ao router obtenham um endereço IPv6 e informações de DNS através de DHCPv6. Neste modo, tem de ativar o servidor DHCPv6 e definir o Endereço IP inicial e o Endereço IP final do servidor DHCPv6.
    • Manual: Configure o IPv6 manualmente introduzindo as informações em endereço de IPv6, Comprimento do prefixo e Gateway predefinido.
    • Desligar: Desativar o suporte IPv6.

Enquanto o IPv6 não substituir completamente o IPv4, os mecanismos de transição de IPv6 são necessários para permitir aos anfitriões com endereços IPv4 acederem aos serviços de IPv6. Escolha entre os diferentes mecanismos de transição do IPv6:

  • 6em4: Estabeleça ligação à rede IPv6 através de um mediador de tunnel. Tem de preencher em Endereço de IPv6, Comprimento do prefixo, Prefixo e Endereço de IPv4 de mediador de tunnel. Para isso, é necessário um endereço público de IPv4.
  • 6para4: Estabeleça ligação à rede IPv6 através de um mediador de tunnel, utilizando anycast. Para isso, é necessário um endereço público de IPv4.
  • DHCP-PD: Obtenha um prefixo de IPv6 do DHCP Server através de delegação de prefixo na LAN. Para isso, é necessário suporte de ISP e só funciona no modo de router.
  • Tunneling de IPv6 (TSP): Aceda a Painel de controlo > Rede > Geral para configurar esta opção.

802.1X

802.1X é uma norma de autenticação desenvolvido pela IEEE para redes de ligação de utilizadores. 802.1X pode ser usado em redes para autenticar o cliente, impedindo que clientes que não tenham passado com êxito a autenticação acedam à rede protegida.

Pode permitir que o seu Synology NAS se ligue à rede protegida por autenticação 802.1X.

Para ativar a autenticação 802.1X:

  1. Aceda a Painel de Controlo > Rede > Interface de rede.
  2. Selecione uma interface de rede com fios (por exemplo, LAN) e clique em Editar.
  3. Aceda ao separador 802.1X e assinale Ativar autenticação 802.1X.
  4. Selecione um dos seguintes protocolos do menu pendente Autenticação. Se não tiver a certeza do protocolo mais adequado, selecione Auto. Â Â Â Â 
    • Protected EAP (PEAP): Use um certificado CA (opcional) para estabelecer uma ligação protegida. Esta ligação protegida será usada pela autenticação interna.
    • Tunneled TLS (TTLS): Use um certificado CA (opcional) para estabelecer uma ligação protegida. Esta ligação protegida será usada pela autenticação interna.
    • TLS: usar TLS (Transport Layer Security) para autenticar.
  5. Se escolher TTLS ou PEAP, seleciona um dos protocolos de autenticação internos do menu de contexto.
    • MS CHAP v2: A informação de autenticação será encriptada com Microsoft CHAP v2.
    • GTC: A informação de autentivação usará GTC (Generic Token Card) para encriptação.
  6. Introduza o seu nome de utilizador e palavra-passe.
  7. Se necessário, clique em Importar, depois Percorrer para encontrar e importar um certificado exportado do servidor Radius (Por exemplo ca.crt).
  8. Clique em OK para guardar as suas definições.

Nota:

  • Para configurar a autenticação 802.1X para redes sem fios, aceda ao Painel de Controlo > Sem fios > Wi-Fi.
  • As definições de autenticação têm de ser as mesmas que o Radius Server definido. Para mais informações, contacte o administrador do Radius Server.

LAN Sem Fios*

Quando o DiskStation funciona como router sem fios, pode configurar as seguintes definições para LAN Sem fios, em Painel de controlo > Rede > Interface de rede.

Definições de IPv4

Endereço IP e máscara de sub-rede

Pode definir o endereço IP e a máscara de sub-rede do seu DiskStation aqui.

DHCP Server

Marque a opção Ativar DHCP Server para permitir que o seu DiskStation atribua endereços IP a dispositivos cliente que estabelecem ligação à rede local fornecida pelo seu DiskStation.

Retransmissão de PPPoE

Ative Retransmissão de PPPoE para permitir que os dispositivos que estão ligados a DiskStation estabeleçam ligação à Internet através da ligação PPPoE.

Configurar como Gateway para a Internet UPnP

Ative o serviço UPnP para permitir que DiskStation se torne num dispositivo UPnP de Gateway para a Internet, de modo a que os dispositivos que estão ligados à rede local do seu DiskStation possam configurar as definições de reencaminhamento de portas com UPnP.

Definições de IPv6

Pode obter um endereço e um prefixo de IPv6 na LAN/Internet com 6em4, 6para4 ou DHCP-PD. Com estas informações, pode configurar as opções de encaminhamento de IPv6 para o seu DiskStation aqui.

  • Endereço de IPv6: Insira o endereço de IPv6 obtido a partir de LAN/Internet.
  • Comprimento do prefixo: Insira o comprimento do prefixo do endereço de IPv6 obtido a partir de LAN/Internet, normalmente 64.
  • DNS primário: O servidor DNS primário para os dispositivos ligados ao router. Não precisa de ser definido no Modo sem monitorização de estado.
  • DNS secundário: O servidor DNS secundário para os dispositivos ligados ao router. Não precisa de ser definido no Modo sem monitorização de estado.

Existem três modos de Anúncio do router (RA, Router Advertisement) a selecionar de:

  • Modo sem monitorização de estado: Permite que os dispositivos ligados ao router configurem automaticamente um endereço IPv6 sem definir o servidor DNS de IPv6.
  • Modo sem monitorização de estado de DHCPv6: Permite que os dispositivos ligados ao router configurem automaticamente um endereço IPv6 e obtenham informações de DNS através de DHCPv6.
  • Modo de monitorização de estado: Permite que os dispositivos ligados ao router obtenham um endereço IPv6 e informações de DNS através de DHCPv6. Neste modo, tem de ativar o servidor DHCPv6 e definir o Endereço IP inicial e o Endereço IP final do servidor DHCPv6.

Clientes DHCP

Aqui, pode ver uma lista de clientes de DHCP e as respetivas configurações de rede (como endereço MAC, endereço IP, nome de anfitrião e quanto tempo resta até expirar a concessão DHCP).

Pode filtrar a lista pendente de clientes IPv4/IPv6 para que mostre clientes que estão a utilizar IPv4 e IPv6, respetivamente. Escolha um cliente e clique Adicionar a reserva de endereço para adicionar o cliente à lista de reservas de endereço.

Reserva DHCP

Se pretende que um cliente receba sempre o mesmo endereço IP durante a renovação da concessão DHCP, pode adicionar o cliente à lista de reservas de endereços.

Pode filtrar a lista pendente de reservas de endereços de IPv4/IPv6. Os endereços de IPv4 identificam-se pelo endereço MAC, enquanto os endereços de IPv6 são identificados por DUID.

Para adicionar um cliente à lista de reserva de endereços:

  • Selecione um cliente e clique em Adicionar.
  • Especifique o endereço MAC ou DUID do cliente, o endereço IPv4/IPv6 que pretende atribuir ao cliente e o nome de anfitrião do cliente.
  • Clique em OK para guardar as definições.

Privilégios

Os privilégios de utilizador e de grupo para aplicações e serviços individuais podem ser vistos e editados. O DSM proporciona definições flexíveis para controlar os privilégios da aplicação, tornando mais fácil gerir o acesso dos utilizadores e aos grupos a diferentes aplicações.

Para modificar os privilégios da aplicação para os utilizadores e grupos:

  1. Aceda a Painel de Controlo > Privilégios.
  2. Selecione a aplicação que pretende modificar. Clique em Editar.
  3. Clique em Utilizador para ver os privilégios individuais ou em Grupo para ver os privilégios de grupo.
  4. Para cada utilizador ou grupo, marque Permitir para permitir o acesso ou Negar para negar o acesso para a aplicação selecionada. Dependendo da aplicação, também pode selecionar Por IP para permitir/negar o acesso com base no endereço IP.
  5. Clique em OK para guardar as suas alterações.

Para ver os privilégios da aplicação para os utilizadores que tenham endereços IP de rede diferentes:

  1. Aceda a Painel de Controlo > Privilégios.
  2. Clique em Visualizador de permissões.
  3. Selecione um utilizador a partir de Nome do utilizador e introduza o endereço IP para o qual pretende ver as permissões em Endereço IP.

Para definir os privilégios da aplicação predefinidos para todos os utilizadores:

  1. Aceda a Painel de Controlo > Privilégios.
  2. Selecione a aplicação que pretende modificar. Clique em Editar.
  3. Clique em Privilégios predefinidos e marque Conceder este privilégio a todos.
  4. Clique em OK para guardar as suas alterações.

Nota:

  • Os privilégios das contas admin e guest não podem ser modificados aqui.
  • Quando ocorrerem conflitos de privilégios, a prioridade do privilégio é a seguinte: Negar > Permitir.
  • Nem todos os pacotes e serviços suportam as definições de privilégios de acesso Por IP.

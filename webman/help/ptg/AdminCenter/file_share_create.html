Criar uma Pasta Partilhada

As pastas partilhadas são os directórios básicos onde pode armazenar e gerir ficheiros e pastas em DiskStation. Antes de armazenar ficheiros, é necessário criar, pelo menos, uma pasta partilhada no DSM.

Para criar uma pasta partilhada:

  1. Vá para Painel de Controlo > Pasta Partilhada e clique em Criar.
  2. Especifique um nome e uma descrição da pasta partilhada.
  3. Selecione um armazenamento onde irá criar a pasta partilhada (ignore este passo se só existir um armazenamento).
  4. Modifique as seguintes opções, se necessário:
    • Ocultar esta pasta partilhada em "Os Meus Locais na Rede": Impede que a pasta partilhada apareça em “Rede” no Explorador de Ficheiros do Windows. Esta opção não afecta os privilégios de acesso da pasta partilhada.

      Nota:

      Ocultar uma pasta partilhada em "Os Meus Locais na Rede" não afecta o direito de acesso à pasta. O direito de acesso à pasta partilhada oculta permanecerá inalterado. Os utilizadores que possuam os direitos de acesso adequados para a pasta partilhada podem aceder, introduzindo "\\nome do servidor\nome da pasta partilhada".

    • Ocultar subpastas e ficheiros de utilizadores sem permissões: Quando Ocultar subpastas e ficheiros de utilizadores sem permissões estiver ativado, se um utilizador sem privilégios de leitura tentar aceder a uma pasta partilhada através de um protocolo de Partilha de Ficheiros do Windows, o utilizador não conseguirá ver pastas nem ficheiros dentro da pasta partilhada. Isto impede os utilizadores de ficarem confusos quando se ligarem aos seus produtos Synology e encontrarem um grande número de ficheiros e subpastas que não podem abrir.

      Nota:

      • Esta funcionalidade só é suportada com o protocolo de Partilha de Ficheiros do Windows.
      • Pode modificar as definições de privilégios de cada pasta e ficheiro na File Station, clicando nele com o botão direito do rato. Selecione Propriedades no menu pendente e clique no separador Permissão.
      • A pasta partilhada, em si própria, não ficará oculta para os utilizadores sem permissões ao ativar Ocultar subpastas e ficheiros de utilizadores sem permissões. Apenas as subpastas e ficheiros nesta pasta partilhada ficarão ocultos.
  5. Ativar Reciclagem: Quando os ficheiros da pasta partilhada forem eliminados, serão movidos para uma pasta com o nome #recycle. O acesso à reciclagem pode estar limitado aos utilizadores pertencentes ao grupo administrators.
  6. Ativar quota da pasta partilhada: Para especificar a capacidade máxima para cada pasta partilhada, assinale Activar quota da pasta partilhada e introduza a capacidade máxima (por ex. 10 GB) no campo seguinte.

    Nota:

    • Apenas quando o armazenamento está a usar o sistema de ficheiros Btrfs é que a opção Ativar quota da pasta partilhada ficam disponíveis para as pastas partilhadas.
    • A utilização do armazenamento conjugada entre os directórios home de todos os utilizadores não pode exceder a quota de "homes".
  7. Ativar protecção avançada da integridade de dados: Para proteger uma pasta partilhada com as estratégias CRC32 checksum e copy-on-write, pode ativar proteção avançada da integridade de dados durante a criação da pasta partilhada. CRC32 checksum é aplicado para verificar se os dados são idênticos ao momento em que foram originalmente escritos, e irá notificá-lo assim que seja detetada alguma corrupção. Copy-on-write ajuda a melhorar a consistência dos dados quando haja um encerramento anormal do sistema.

    Nota:

    • Esta opção está apenas disponível para volumes com o sistema de ficheiros Btrfs.
    • Pode ativar/desativar esta opção para cada pasta partilhada durante a sua criação. Esta opção não pode ser modificada assim que tenha sido criada a pasta partilhada.
    • Se ficheiros como bases de dados forem editados e atualizados de forma constante, a quantidade de ficheiros fragmentados poderá aumentar devido a copy-on-write. Recomenda-se que desative esta opção para melhorar o desempenho.
  8. Ativar compressão de ficheiros: Os dados na pasta partilhada criada serão automaticamente comprimidos pelo sistema de ficheiros Btrf para poupar espaço de armazenamento. Quando os recolher para utilização, os dados serão automaticamente descomprimidos.

    Nota:

    • Esta opção só está disponível quando criar uma pasta partilhada num volume de Btrfs ou quando ativar Proteção avançada da integridade de dados.
    • Esta opção não pode ser modificada depois de criar a pasta partilhada.
    • Os rácios de compressão variam com os tipos de dados: Os ficheiros de texto têm o rácio de compressão mais alto, enquanto que os ficheiros multimédia (por exemplo, imagens, músicas e vídeos) poderão não serão tão comprimidos uma vez que já são comprimidos para serem transmitidos.
  9. Encriptar esta pasta partilhada: Se deseja encriptar a pasta partilhada, proceda da seguinte forma:
    1. Especifique e confirme uma nova chave de encriptação. A chave de encriptação não pode incluir vírgulas (,) ou sinais de igual (=).
    2. Se necessário, assinale a caixa Montar automaticamente ao iniciar para montar automaticamente a pasta encriptada quando o sistema for iniciado. Se esta opção for desativada, a pasta partilhada encriptada será desmontada na próxima vez que DiskStation for reiniciado. Para ver ou aceder a uma pasta partilhada desmontada, terá de montar a pasta manualmente importando ou introduzindo a chave de encriptação.
  10. Clique em OK para terminar.

Regras de Nomeação da Pasta Partilhada:

  • Os nomes das pastas partilhadas não são sensíveis a maiúsculas e minúsculas e podem incluir 1 a 32 caracteres. Os nomes não podem incluir os seguintes caracteres: ! " # $ % & ' ( ) * + , / : ; <= > ? @ [ ] \ ^ ` { } | ~
  • Os seguintes nomes estão reservados para utilização do sistema: ., .., global, home, homes, printers, public, satashare, surveillance, usbbackup, usbshare.
  • O primeiro caractere do nome de uma pasta partilhada não pode ser um sinal de subtração ou um espaço e o último caractere não pode ser um espaço.
  • A descrição da pasta partilhada é sensível a maiúsculas e minúsculas e tem o limite até 64 caracteres Unicode apresentáveis.

Encriptar Pastas Partilhadas:

A encriptação AES de 256 bits protege dados bloqueando todas as tentativas de acesso não autorizadas. Assim que uma pasta partilhada for encriptada, os utilizadores que não tiverem a chave de encriptação não poderão aceder ou modificar os dados encriptados mesmo que removam os discos rígidos de DiskStation e os montem noutro dispositivo.

Nota:

  • As pastas partilhadas só podem ser encriptadas no momento da sua criação.
  • Os nomes dos ficheiros e pastas em pastas partilhadas encriptadas não podem exceder 143 caracteres em inglês ou 43 caracteres asiáticos.
  • As pastas partilhadas encriptadas não podem ser acedidas através de NFS.
  • Se forem restauradas as predefinições das palavras-passe utilizando o botão de reposição localizado em DiskStation, as pastas partilhadas encriptadas serão desmontadas e a opção para montar automaticamente pastas partilhadas encriptadas será desativada.
  • Pastas partilhadas encriptadas são apenas suportadas no modo de privilégio elevado. Para usar esta funcionalidade, aceda ao pacote Docker no seu DSM anfitrião, selecione o Docker DSM que deseja para ativar pastas partilhadas encriptadas, clique em Editar e assinale a caixa junto de Executar contentor usando privilégios elevados.

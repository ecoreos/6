FTP/FTPS

Em Painel de Controlo > Serviços de Ficheiros > FTP, pode ativar os serviços FTP para permitir que os clientes acedam a pastas e dados do sistema. Antes de ativar o FTP, certifique-se de que as seguintes portas TCP do router estão a ser reencaminhadas para o dispositivo DiskStation: 21 (ligação de controlo predefinida), 20 (ligação de dados para o modo ativo) e 55536-55567 (ligação de dados para o modo passivo).

Para ativar o serviço FTP:

  1. Ative um dos seguintes:
    • Ativar serviço FTP (Sem encriptação): Isto é o protocolo de rede padrão utilizado para transferir ficheiros. O FTP não fornece qualquer tipo de encriptação para proteger as informações durante sessões de transferência, como palavras-passe, nomes de utilizador ou ficheiros. No entanto, as velocidades de transferência são mais rápidas e requerem menos recursos de sistema.
    • Ativar serviço de encriptação FTP SSL/TLS (FTPS): Uma série de protocolos criptográficos FTP padrão com suporte adicional para Transport Layer Security (TLS) e Secure Sockets Layer (SSL), que protege informação durante sessões de transferência, como palavras-passe, nomes de utilizador ou ficheiros. No entanto, as velocidades de transferência são menores e consomem mais recursos de sistema devido a encriptação.
  2. Clique em Aplicar.

Tempo Limite

O servidor FTP desliga os utilizadores inativos após um período de tempo especificado.

Definição de Número da Porta do Serviço FTP

Esta secção permite especificar a porta utilizada do canal de controlo FTP.

Nota:

O número da porta predefinida para o canal de controlo FTP é 21. Pode ser definida de 1 a 65535, excluindo os seguintes números: 20, 22, 23, 25, 80, 110, 137, 138, 139, 143, 199, 443, 445, 515, 543, 548, 587, 873, 993, 995, 3306, 3689, 5000, 5001, 5005, 5006, 5335, 5432, 6881, 8080, 7000, 7001, 8081, 9997, 9998, 9999, 50001, 50002 e portas predefinidas do eMule: 4662 (TCP), 4672 (UDP).

Intervalo de Portas de FTP Passivo

Esta secção permite especificar o intervalo de portas utilizado para ligações de FTP Passivo. Pode utilizar o intervalo de portas predefinido ou especificar um intervalo personalizado. O intervalo de portas pode ser definido de 1025 a 65535 e pode conter até 128 portas.

Comunicar o IP Externo em Modo PASV

Ao ativar esta opção, o servidor irá comunicar o respetivo endereço IP externo aos clientes FTP. O endereço IP comunicado será apresentado no registo de ligação do cliente FTP. Após a ativação, pode escolher o endereço IP externo detetado automaticamente ou introduzir manualmente o endereço IP externo de DiskStation.

Esta opção apenas funciona quando DiskStation estiver atrás de um NAT e os clientes FTP pertencerem a uma sub-rede diferente da que é utilizada por DiskStation. Na maior parte dos casos, esta opção não é necessária mas se os clientes FTP falharem na ligação a DiskStation, pode ativar esta opção e tentar novamente.

Ativar o FXP

FXP significa File eXchange Protocol. Se ativar esta opção, o serviço FTP suporta a função de transferência de ficheiros servidor a servidor.

Ativação do Módulo Criptográfico FIPS

O FIPS é suportado pelo serviço FTPS que utiliza o módulo de criptografia validado por OpenSSL FIPS 140-2 (Certificado #1747). Se ativar esta opção, serão utilizados algoritmos adicionais de criptografia segura pelo serviço FTPS para garantir sessões mais seguras.

Suportar o Modo de Transferência ASCII

Ative esta opção para suportar o modo de transferência ASCII, permitindo que DiskStation transfira de modo adequado os ficheiros de texto simples formatados em ASCII através de FTP.

Codificação UTF-8

Esta opção pode ser usada para ajudar clientes FTP a mostrar e gerir correctamente ficheiros com nomes multilingues. Para ir de encontro às necessidades dos vários clientes FTP, são disponibilizadas as três seguintes opções:

  • Desativar: A codificação UTF-8 não será usada. Esta opção está direccionada a clientes com o Explorador de Ficheiros do Windows como cliente FTP para carregar/transferir ficheiros com nomes multilingue. Não se recomenda para utilização normal.
  • Automático: Deixar que o DSM determine automaticamente como gerir a codificação UTF-8. Esta opção é recomendada para utilização normal.
  • Forçar: Forçar a utilização da codificação UTF-8. Se tiver ativado UTF-8 no seu cliente de FTP e se ainda assim não exibir corretamente os nomes dos ficheiros nem gerir ficheiros com nomes multilingue, selecione esta opção.

Restrição de Ligação

Esta secção inclui as seguintes opções.

  • Limitar ligações por IP: Quando esta função está activada, o servidor limita o número máximo de ligações por IP. Ative esta função e a restrição de largura de banda de FTP, para uma gestão mais eficaz da largura de banda.
  • Ativar restrição de largura de banda de FTP: Active esta opção para restringir a largura de banda da ligação única de FTP.

Nota:

  • A conta "guest" não pode iniciar sessão no servidor através do FTP.
  • A configuração da página de código no computador do cliente de FTP deve ser a mesma do DiskStation, para poder aceder correctamente aos dados.
  • Em alguns modelos específicos, o serviço FTP suporta a ligação com SSL ou TLS.
  • As definições de firewall do seu cliente FTP e dos dois servidores FTP devem ser configuradas corretamente ou a função FXP poderá não funcionar adequadamente.
  • Alguns clientes FTP, tais como os que utilizam o Internet Explorer 8/9, têm de iniciar sessão com o nome de utilizador e a palavra-passe. Por exemplo: ftp://nomedeutilizador:palavra-passe@oseu.ftp.com.

Aplicações da Internet

Com a funcionalidade Web Station incorporada e o suporte PHP, pode alojar facilmente um website no DiskStation e criar websites dinâmicos para o seu negócio ou passatempo. Além disso, o Centro de Pacotes fornece vários pacotes de terceiros, tais como Gestão de Conteúdos, Gestão de Relacionamento de Clientes e sistema de Comércio eletrónico. Uma pasta partilhada predefinida designada como web será criada para armazenar os ficheiros para o seu website.

Para ativar a Web Station:

  1. Aceda a Painel de Controlo > Serviços na Internet.
  2. Assinale a opção Ativar a Web Station.
  3. Clique em Aplicar.

Nota:

  • Para maior flexibilidade e segurança, a Web Station utiliza o grupo de utilizador http para executar tarefas. Por este motivo, para controlar as permissões de acesso às suas páginas de Internet, altere as permissões de acesso do grupo http para cada uma das pastas e ficheiros correspondentes. Por exemplo, se pretender que o ficheiro sample.htm na pasta partilhada web fique acessível através da Web Station, será necessário que se certifique que o grupo http dispõe das permissões adequadas de leitura/escrita para a pasta partilhada web, tal como para o ficheiro sample.htm.
  • O ficheiro da página inicial deve ter o nome index.htm, index.html ou index.php.
  • Quando utiliza a função mail() PHP, o sistema irá obter e utilizar automaticamente as informações do servidor SMTP na página Notificação. Certifique-se de que ativou as notificações por e-mail e introduziu as informações corretas do servidor SMTP.
  • O caminho real da pasta partilhada web é /volume1/web. Algumas aplicações requerem que o utilizador defina o caminho no ficheiro de configuração.
  • Os nomes blog, photo, photosrc e webdefault são nomes de pastas reservados. Estes nomes não podem ser utilizados pela Web Station, assim, não crie pastas com estes nomes na pasta partilhada web.
  • Pode personalizar a página de erro ao criar um ficheiro missing.html, um ficheiro missing.htm ou um ficheiro missing.php. Em primeiro lugar, ative a Web Station. Em seguida, coloque o ficheiro missing.html, missing.htm ou missing.php na pasta partilhada web no servidor. Quando os utilizadores selecionam uma ligação para uma página que não existe, irão visualizar esta página de erro personalizada.
  • Número máximo de ligações HTTP ativas: 256
  • Número máximo de anfitriões virtuais: 30

Ativar Anfitrião Virtual para Alojar Websites

Pode alojar vários websites num único servidor com a funcionalidade de anfitrião virtual. Para começar, é necessário registar alguns nomes de domínio e configurar o alojamento virtual seguindo os passos abaixo.

Para ativar o Anfitrião Virtual:

  1. Clique em Anfitrião Virtual.
  2. Clique em Criar.
  3. Introduza um nome para a subpasta da pasta partilhada web, que irá armazenar os ficheiros do website.
  4. Introduza um nome de anfitrião para o website.
  5. Introduza uma porta para o website.
  6. Clique em OK.

Ativar o Website Pessoal

A funcionalidade de website pessoal é uma forma conveniente de permitir que utilizadores locais, utilizadores de domínio e utilizadores LDAP criem o seu próprio website pessoal.

Para utilizadores locais

Cada utilizador local terá um endereço de website exclusivo server-IP/~username ou server-name/~username. Por exemplo:

  1. 192.168.1.1/~ken
  2. SynologyNAS/~ken

Para utilizadores de domínio

Cada utilizador de domínio terá um endereço de website exclusivo server-IP/~domainname%5cusername ou server-name/~domainname%5cusername. Por exemplo:

  1. 192.168.1.1/~testdomain%5cken
  2. SynologyNAS/~testdomain%5cken

Para utilizadores LDAP

Cada utilizador LDAP terá um endereço de website exclusivo server-IP/~username@domainname ou server-name/~username@domainname. Por exemplo:

  1. 192.168.1.1/~ken@testdomain
  2. SynologyNAS/~ken@testdomain

Nota:

Antes de ativar esta funcionalidade, certifique-se de que as pastas homes do utilizador estão ativadas. Para os utilizadores locais, aceda a Painel de Controlo > Utilizador > Avançadas, para os utilizadores de domínio, aceda a Painel de Controlo > Serviço de Diretório > Utilizadores de Domínio > Home do Utilizador, para utilizadores LDAP, aceda a Painel de Controlo > Serviço de Diretório > Utilizadores LDAP > Home do Utilizador.

Nota:

Antes de ativar esta funcionalidade, certifique-se de que as pastas homes do utilizador estão ativadas. Para utilizadores locais, aceda a Painel de Controlo > Utilizador > Avançado.

Para criar um website pessoal:

  1. Se tiver marcado Ativar o website pessoal, será automaticamente criada uma pasta www no diretório home do utilizador.
  2. Depois, o utilizador já pode carregar os ficheiros relacionados com o website (por exemplo, ficheiros de som e de páginas de Internet) para a pasta www.
  3. Certifique-se de que o utilizador dispõe das permissões de acesso adequadas aos ficheiros carregados.

Para personalizar a página de erro:

  1. Guarde a página de erro como missing.html, missing.htm ou missing.php.
  2. Carregue o ficheiro para a pasta www.

Quando os utilizadores selecionam uma ligação para uma página que não existe no website do utilizador local, será apresentada a página de erro personalizada.

Suporte e limitações:

Os utilizadores locais só poderão aceder aos dados nas seguintes circunstâncias:

  • As suas próprias páginas iniciais
  • /var/tmp
  • /tmp
  • /volumeX/@tmp

Serviço HTTP

Para além dos números de portas predefinidos 80 (para HTTP) ou 443 (para HTTPS), é possível adicionar números de portas adicionais para aceder à Photo Station e à Web Station. Pode adicionar um número de porta adicional de 1 a 65535 para HTTP ou HTTPS.

Para adicionar um número de porta adicional para acesso HTTP ou HTTPS:

  1. Assinale a opção Adicionar uma porta HTTP suplementar ou Adicionar uma porta HTTPS suplementar (a opção Ativar ligação HTTPS para serviços na Internet tem de estar ativada antes de adicionar uma porta para HTTPS).
  2. Introduza o número da porta pretendido no campo.

Nota:

  • As seguintes portas devem ser excluídas: 20, 21, 22, 23, 25, 110, 137, 138, 139, 143, 199, 443, 445, 515, 543, 548, 587, 873, 993, 995, 3306, 3689, 5000, 5001, 5005, 5006, 5335, 5432, 6881, 8080, 7000, 7001, 8081, 9997, 9998, 9999, 50001, 50002, portas predefinidas do eMule: 4662 (TCP), 4672 (UDP) e gama predefinida da porta FTP: a gama real poderá variar, dependendo de diferentes modelos.

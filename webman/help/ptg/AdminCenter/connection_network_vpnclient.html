Ligar a VPN

Pode facilmente tornar o seu DiskStation num cliente VPN e estabelecer ligação a servidores VPN (Rede Privada Virtual) através dos protocolos PPTP, OpenVPN ou L2TP/IPSec. Se dispõe de vários servidores VPN, também pode criar perfis VPN específicos para cada servidor para rapidamente alterar e ligar a diferentes servidores VPN com um simples clique.

O que é o VPN?

Um VPN ou rede privada virtual, é uma solução para responder à necessidade de aceder em segurança aos recursos na sua rede privada a partir da Internet. Para negócios ou alguns indivíduos, com a necessidade de expandir o crescimento das capacidades de rede, o custo deste sistema de rede físico e o seu suporte técnico irá aumentar exponencialmente. Quando considera as relações custo-eficácia e as manutenções a longo prazo, VPN é uma solução inteligente e cada vez mais atrativa. Com a encriptação e outros mecanismos de segurança, a tecnologia VPN permite aos membros empresariais acederem facilmente à rede central da empresa e nivelar os recursos na mesma, como em LAN. Podem também aceder aos recursos na sua LAN doméstica, quando estão longe de casa.

Nota:

  • O DiskStation apenas pode ligar a servidores OpenVPN que suportem túneis estilo tun em Layer 3.
  • Antes de instalar o pacote VPN, certifique-se de que tem um VPN Server para ligar. Para melhor compatibilidade, sugerimos que ligue a serviços VPN alojados num DiskStation com o pacote VPN Server. Para configurar outro DiskStation como VPN server, consulte Ajuda DSM > VPN Server > Configurar o VPN Server para obter instruções.

Para criar perfis VPN:

  1. Aceda a Painel de Controlo > Rede.
  2. No separador Interface de Rede, clique Criar e escolha Criar perfil VPN.
  3. Ao criar um novo perfil VPN, pode escolher um dos seguintes tipos de ligação:
    • PPTP: PPTP (Point-to-Point Tunneling Protocol) é uma solução VPN comummente usada pela maioria dos clientes (incluindo Windows, Mac, Linux e dispositivos móveis).
    • OpenVPN: OpenVPN é uma solução open source para implementar o serviço VPN. Protege a ligação VPN com o mecanismo de encriptação SSL/TLS.
    • L2TP/IPSec: L2TP (Layer 2 Tunneling Protocol) em IPSec fornece redes privadas virtuais com maior segurança e é suportado pela maior parte dos clientes (tais como, Windows, Mac, Linux e dispositivos móveis).
  4. Consulte as secções abaixo para obter instruções para configurar qualquer um dos três tipos de ligações VPN.

Para criar um perfil PPTP:

  1. Quando indicado para escolher um método de ligação VPN, escolha PPTP e clique Seguinte.
  2. Escolha um nome para o novo perfil, especifique o endereço do servidor VPN com o qual pretende estabelecer ligação e introduza o seu nome de utilizador e a palavra-passe para o servidor. Clique em Seguinte para continuar.

    3. Nota:

    • O nome de utilizador, a palavra-passe e o endereço do servidor VPN devem ser fornecidos pelo administrador do servidor VPN.
  3. Escolha um dos seguintes protocolos de autenticação do menu Autenticação para proteger a palavra-passe do cliente VPN durante a autenticação:
    • PAP: A palavra-passe não será encriptada durante a autenticação.
    • CHAP: A palavra-passe será encriptada utilizando o CHAP (Protocolo de Autenticação Challenge-Handshake).
    • MS CHAP: A palavra-passe será encriptada utilizando Microsoft CHAP versão 1.
    • MS CHAP v2: A palavra-passe será encriptada utilizando Microsoft CHAP versão 2.
  4. Se escolher MS CHAP ou MS CHAP v2, então seleccione uma das seguintes opções no menu Encriptação para encriptar a ligação VPN:
    • Sem MPPE: A ligação VPN não será protegida com a Encriptação Ponto a Ponto da Microsoft.
    • Exigir MPPE (40/128 bit): A ligação VPN será protegida com a Encriptação Ponto a Ponto da Microsoft de 40 bits ou 128 bits.
    • MPPE máximo (128 bits): A ligação VPN será protegida com um mecanismo de encriptação de 128 bits Point to Point da Microsoft, que fornece o mais alto nível de segurança.

    5. Nota:

    • Estas definições de autenticação e encriptação devem ser idênticas às definições especificadas no servidor VPN. Para obter detalhes, contacte o administrador do servidor VPN.
  5. Assinale qualquer uma das seguintes caixas de verificação, dependendo das suas necessidades:
    • Utilizar gateway predefinido na rede remota: Active esta opção para encaminhar o tráfego de rede do DiskStation para o servidor VPN especificado.
    • Permitir a outros dispositivos de rede a ligar através da ligação da Internet do servidor Synology: Active esta opção para permitir que dispositivos de rede que se encontrem na mesma rede local do seu DiskStation se liguem ao mesmo servidor VPN.
    • Ligue de novo quando a ligação a VPN for interrompida: Se perder a ligação VPN inesperadamente, o sistema irá tentar restabelecer a ligação cinco vezes, uma tentativa a cada 30 segundos.
  6. Clique em Aplicar.

Para criar um perfil OpenVPN:

  1. Quando indicado para escolher um método de ligação VPN, escolha OpenVPN e clique Seguinte.
  2. Escolha um nome para o novo perfil, especifique o endereço do servidor VPN com o qual pretende estabelecer ligação e introduza o seu nome de utilizador e a palavra-passe para o servidor.
  3. No campo Certificado, clique no botão Pesquisar para encontrar e importar o ficheiro certificado (por exemplo, ca.crt) que foi exportado do servidor VPN. Clique em Seguinte para continuar.

    4. Nota:

    • O nome de utilizador, a palavra-passe, o endereço e o certificado do servidor VPN devem ser fornecidos pelo administrador do servidor VPN.
  4. Assinale qualquer uma das seguintes caixas de verificação, dependendo das suas necessidades:
    • Activar compressão na ligação VPN: Active esta opção para realizar a compressão de dados antes de transferir os dados através da VPN. Esta opção pode aumentar a velocidade de transferência mas irá consumir mais recursos do sistema.
    • Utilizar gateway predefinido na rede remota: Active esta opção para encaminhar o tráfego de rede do DiskStation para o servidor VPN especificado.
    • Permitir a outros dispositivos de rede a ligar através da ligação da Internet do servidor Synology: Active esta opção para permitir que dispositivos de rede que se encontrem na mesma rede local do seu DiskStation se liguem ao mesmo servidor VPN.
    • Ligue de novo quando a ligação a VPN for interrompida: Se perder a ligação VPN inesperadamente, o sistema irá tentar restabelecer a ligação cinco vezes, uma tentativa a cada 30 segundos.
  5. Clique em Aplicar.

Para criar um perfil OpenVPN (via .ovpn):

  1. Quando indicado para escolher um método de ligação VPN, escolha OpenVPN (através da importação de um ficheiro .ovpn) e clique Seguinte.
  2. Escolha um nome para o novo perfil, e no campo Importar ficheiro .ovpn, clique em Pesquisar para selecionar e importar um ficheiro .ovpn exportado do servidor VPN.
  3. Introduza as seguintes informações fornecidas pelo administrador do seu servidor VPN: o seu nome de utilizador, palavra-passe e importe o ficheiro certificado CA (por ex., ca.crt).
  4. Se o fornecedor do servidor VPN tiver também fornecido uma chave TLS-auth, chave de cliente, certificado de cliente, ou uma lista de revogação de certificados, clique em Opções avançadas e importe-as para os respetivos campos.
  5. Clique em Seguinte para continuar.

    6. Nota:

    • O nome de utilizador, a palavra-passe, o endereço e quaisquer ficheiros adicionais, como certificados ou chaves devem ser fornecidos pelo administrador do servidor VPN.
  6. Assinale qualquer uma das seguintes caixas de verificação, dependendo das suas necessidades:
    • Activar compressão na ligação VPN: Active esta opção para realizar a compressão de dados antes de transferir os dados através da VPN. Esta opção pode aumentar a velocidade de transferência mas irá consumir mais recursos do sistema.
    • Utilizar gateway predefinido na rede remota: Active esta opção para encaminhar o tráfego de rede do DiskStation para o servidor VPN especificado.
    • Permitir a outros dispositivos de rede a ligar através da ligação da Internet do servidor Synology: Active esta opção para permitir que dispositivos de rede que se encontrem na mesma rede local do seu DiskStation se liguem ao mesmo servidor VPN.
    • Ligue de novo quando a ligação a VPN for interrompida: Se perder a ligação VPN inesperadamente, o sistema irá tentar restabelecer a ligação cinco vezes, uma tentativa a cada 30 segundos.
  7. Clique em Aplicar.

Para criar um perfil L2TP/IPSec:

  1. Quando indicado para escolher um método de ligação VPN, escolha L2TP/IPSec e clique Seguinte.
  2. Escolha um nome para o novo perfil, especifique o endereço do servidor VPN com o qual pretende estabelecer ligação e introduza o seu nome de utilizador e a palavra-passe para o servidor. Também será necessário introduzir a chave pré-partilhada para o servidor VPN. Depois, clique em Seguinte.

    3. Nota:

    • O nome de utilizador, a palavra-passe, o endereço e a chave pré-partilhada do servidor VPN devem ser fornecidos pelo administrador do servidor VPN.
  3. Escolha um dos seguintes protocolos de autenticação do menu Autenticação para proteger a palavra-passe do cliente VPN durante a autenticação:
    • PAP: A palavra-passe não será encriptada.
    • CHAP: A palavra-passe será encriptada utilizando o CHAP (Protocolo de Autenticação Challenge-Handshake).
    • MS CHAP: A palavra-passe será encriptada utilizando Microsoft CHAP versão 1.
    • MS CHAP v2: A palavra-passe será encriptada utilizando Microsoft CHAP versão 2.

    4. Nota:

    • Estas definições de autenticação e encriptação devem ser idênticas às definições especificadas no servidor VPN. Para obter detalhes, contacte o administrador do seu servidor VPN.
  4. Assinale qualquer uma das seguintes caixas de verificação, dependendo das suas necessidades:
    • Utilizar gateway predefinido na rede remota: Active esta opção para encaminhar o tráfego de rede do DiskStation para o servidor VPN especificado.
    • Permitir a outros dispositivos de rede a ligar através da ligação da Internet do servidor Synology: Active esta opção para permitir que dispositivos de rede que se encontrem na mesma rede local do seu DiskStation se liguem ao mesmo servidor VPN.
    • O servidor está atrás de um dispositivo NAT: Por predefinição, o protocolo L2TP/IPSec não permite a ligação a um servidor VPN atrás de um dispositivo NAT: Active esta opção para contornar esta limitação, permitindo que o DiskStation seja ligado a um servidor VPN L2TP/IPSec atrás de um dispositivo NAT:
    • Ligue de novo quando a ligação a VPN for interrompida: Se perder a ligação VPN inesperadamente, o sistema irá tentar restabelecer a ligação cinco vezes, uma tentativa a cada 30 segundos.
  5. Clique em Aplicar.

Para ligar ou desligar de um servidor VPN:

  1. Seleccione um perfil VPN.
  2. Clique em Ligar ou Desligar.

Nota:

  • Apenas pode ser ligado um perfil de cada vez.

Para modificar um perfil VPN:

  1. Seleccione um perfil VPN.
  2. Clique em Editar.

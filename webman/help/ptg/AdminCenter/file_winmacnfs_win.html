Serviço de Ficheiros Windows

O serviço de ficheiros Windows permite aos clientes de Windows, Mac e Linux com suporte SMB/CIFS acederem aos dados armazenados em DiskStation. Se pretende pesquisar ficheiros em DiskStation através do Explorador do Windows ou montar pastas partilhadas como unidades de rede, ative esta opção em Painel de Controlo > Serviços de Ficheiros > Win/Mac/NFS.

Para ativar o serviço de ficheiros Windows:

  1. Assinale Ativar serviço de ficheiros Windows.
  2. Clique em Aplicar.

Nota:

Depois de ativar o serviço de ficheiros Windows, pode montar as pastas totalmente indexadas no seu computador Mac com SMB e usar o Spotlight no Finder para procurar os conteúdos de ficheiros nas pastas montadas.

Grupo de Trabalho

Esta opção permite-lhe designar o sistema como membro de um grupo de trabalho específico para fins de gestão de rede.

Nota:

  • O nome de um grupo de trabalho pode incluir 1 a 15 caracteres. O nome não pode incluir os seguintes caracteres: [ ] ; : " < > * + = \ / | ? ,
  • Se estiver a utilizar um Mac com o OS X 10.9 e o seu DiskStation estiver associado a um domínio Windows, pode iniciar a sessão adicionando localhost\ antes do nome de utilizador local DiskStation para aceder aos ficheiros armazenados em DiskStation através do serviço de ficheiros Windows.

Cancelar acesso a Versões Anteriores

No Explorador do Windows, utilizadores podem aceder aos instantâneos de pastas ou ficheiros clicando no separador Versões Anteriores dentro de uma pasta partilhada, permitindo o restauro ou cópia de dados sem a ajuda de um administrador. Se escolher Cancelar o acesso a Versões Anteriores, a Página "Versões Anteriores" não apresentará qualquer informação.

Nota:

  • Os utilizadores têm acesso a uma versão anterior apenas se, no momento em que o instantâneo correspondeu à versão anterior, as suas contas de utilizador tiverem sido criadas e tendo permissão de leitura da pasta partilhada à qual corresponde o instantâneo tirado.

Ativação de Registo de Transferência

Marcar esta opção ativará operações de acesso a registo do ficheiro, incluindo criar, eliminar, ler, escrever, mover e renomear ficheiros ou pastas através do protocolo CIFS/SMB, também conhecido como protocolo de Transferência de Ficheiros Windows.

Definições avançadas

Clique no botão Definições Avançadas para ver as opções abaixo.

  • Servidor WINS: O Microsoft Windows Internet Name Service (WINS) é um serviço para mapear o nome NetBIOS para endereços IP. Os utilizadores Windows localizarão o sistema mais facilmente nas redes TCP/IP se o sistema tiver sido configurado para se registar com um servidor WINS.
  • Protocolo máximo para serviço de ficheiros Windows: Dependendo da configuração da sua rede, pode definir o nível mais elevado de protocolo que será suportado pelo seu DiskStation.
    • Ativar SMB 2: O SMB (Bloqueio da mensagem do servidor) 2 tem sido suportado desde o Windows Vista e é a versão melhorada do SMB. O SMB 2 adiciona a capacidade de compor múltiplas ações SMB num só pedido, para reduzir o número de pacotes de rede e melhorar o desempenho.
    • Ativar SMB 2 e Large MTU: A unidade máxima de transmissão (MTU) refere-se à unidade máxima de dados que pode ser transmitida num protocolo de comunicação. Para tirar partido de interfaces de rede mais rápidas, como Ethernet de 1 ou 10 gigabits, Large MTU aumenta a unidade máxima de transmissão para 1 megabyte (MB). Ativar Large MTU melhora a velocidade e eficiência de transferências de ficheiros grandes, enquanto reduz o número de pacotes que necessitam de ser processados.
    • Ativar SMB 3: SMB 3 é suportado desde o Windows 8 e Windows Server 2012. É a versão melhorada de SMB 2. SMB 3 suporta transmissão de encriptação de ficheiros com base em AES, melhorando assim a segurança de transferências peer-to-peer.

    Nota:

    • O SMB 2 é suportado no Windows Vista, Windows Server 2008 R2, Windows 7 e superior.
    • Large MTU é suportado no Windows Server 2008 R2, Windows 7 e superior.
    • Para que Large MTU seja eficaz no computador Windows, terá de modificar as definições de registo do computador. Para mais detalhes, consulte o suporte técnico da Microsoft (tal como este artigo).
  • Modo de encriptação de transporte: Quando SMB 3 está ativado, o Serviço de Ficheiros do Windows adicionará encriptação do transporte para segurança na transmissão.
    • Desativar: Não será aplicada qualquer encriptação de transporte.
    • Automático: A encriptação de transporte será apenas aplicada aos clientes que suportem esta funcionalidade.
    • Forçar: A encriptação de transporte será sempre aplicada. Isto levará a que os clientes que não suportem a encriptação do transporte não consigam usar o Serviço de Ficheiros do Windows.

    • Nota:

    • A encriptação de transporte é suportada no Windows 8, Windows Server 2012 e superior.
    • Não é suportado o início de sessão anónimo para o Serviço de Ficheiros do Windows quando ativada a encriptação de transporte.
    • Quando ativada a encriptação de transporte, deixe desmarcado Desativar Opportunistic Locking para evitar inatividade da aplicação.
  • Desativar Opportunistic Locking: Ao marcar esta opção irá desativar o "Opportunistic Locking" do CIFS, resultando num desempenho melhorado quando as aplicações de base de dados acedem aos ficheiros de base de dados na pasta partilhada do dispositivo DiskStation através da rede.
  • Ativar Local Master Browser: Quando existem vários computadores Windows na mesma subrede, negociam e elegem um computador como "local master browser". O local master browser mantém listas dos outros computadores dentro da sub-rede e respetivos recursos partilhados, e partilha estas listas com os outros computadores. Esta opção permite a DiskStation assumir a função de local master browser.

    • Nota:

    Ativar esta função desativa a hibernação de disco rígido e ativa a conta guest sem uma palavra-passe.

  • Ativar módulo DirSort VFS: Ordena os nomes das pastas e lista-as por ordem alfabética. A ativação desta opção afeta o desempenho do sistema.
  • Vetar ficheiros: Oculta ficheiros de acordo com os critérios especificados. Os critérios podem incluir caracteres universais (*) e as várias entradas devem estar separadas por uma barra (/). Por exemplo: /abc*/*.txt/*.conf/. A ativação desta opção afeta o desempenho do sistema.
  • Permitir ligações simbólicas dentro de pastas partilhadas: Ativar esta opção permite que as ligações simbólicas apontem para ficheiros ou pastas na mesma pasta partilhada. As ligações simbólicas podem ser criadas utilizando o terminal DSM.
  • Permitir ligações simbólicas em pastas partilhadas: Ativar esta opção permite que as ligações simbólicas apontem para ficheiros ou pastas em diferentes pastas partilhadas. As ligações simbólicas podem ser criadas utilizando o terminal DSM. Tenha em conta que ativar esta opção pode permitir que os utilizadores sem privilégios de acesso possam visualizar e aceder a pastas partilhadas que são o alvo das ligações simbólicas.
  • Ativar módulo MSDFS VFS: Permite que as ligações simbólicas apontem para ficheiros ou pastas localizados noutros servidores SMB. Em versões antigas do Windows, tais como o Windows XP, o cliente e o DiskStation têm de aderir a um domínio Windows antes de ativar esta opção. Para mais informações sobre como criar ligações MSDFS, consulte aqui. Tenha em conta que a cache do dispositivo cliente deve ser limpa após desativar esta opção, caso contrário, o desempenho normal pode ser afetado.
  • Desativar múltiplas ligações a partir do mesmo endereço IP: Quando esta opção está ativada, o DiskStation apenas permite uma ligação por endereço IP de clientes que utilizem o Serviço de Ficheiros do Windows. Quando múltiplos clientes com o mesmo endereço IP utilizam o Serviço de Ficheiros do Windows, o DiskStation mantém o último e elimina ligações antigas. Num ambiente NAT, o cliente pode estar a utilizar um endereço IP único. Neste caso, ao ativar esta opção, o desempenho normal pode ser afetado.
  • Reunir registos de depuração: Quando esta opção está ativada, são armazenados registos detalhados para fins de depuração. A ativação desta opção afeta o desempenho do sistema.
  • Aplicar permissões de UNIX predefinidas: Ative esta opção para aplicar as permissões de UNIX predefinidas quando carregar ou criar ficheiros e pastas. A permissão de UNIX será 744 para ficheiros e 755 para pastas. Quando esta opção estiver desativada, a permissão de UNIX é 777 para ficheiros e pastas.

    • Nota:

    • Ativar esta opção pode provocar questões de inconsistência de permissões entre diferentes protocolos. Para evitar inconsistências, sugerimos que deixe esta opção desativada.

    Nota:

    • Para pastas partilhadas com ativação ACL do Windows (todas as partilhas, excluindo "photo" e partilhas em discos externos) execute o comando chmod no seu Linux ou cliente FTP para alterar os tipos de permissão de pasta e ficheiro do ACL do Windows para UNIX.
    • Ativar esta opção pode provocar questões de inconsistência de permissões entre diferentes protocolos. Para evitar inconsistências, sugerimos que deixe esta opção desativada.
  • Não reservar espaço de disco quando cria ficheiros: Ative esta opção para proibir que mais espaço de disco seja reservado quando cria ficheiros no seu DiskStation .
  • Limpar cache SMB: Clique neste botão para eliminar os seguintes ficheiros da cache SMB para fins de resolução de problemas:
    • /var/cache/samba/netsamlogon_cache.tdb
    • /var/lib/samba/winbindd_cache.tdb
    • /var/lib/samba/share_info.tdb
    • /var/lib/samba/registry.tdb
    • /run/samba/connections.tdb
    • /run/samba/gencacche.tdb
    • /run/samba/gencache_notran.tdb

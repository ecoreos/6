Portal de Aplicações

Portal de Aplicações permite-lhe pode configurar as definições de ligação de várias aplicações, para que possa aceder e executar diretamente estas aplicações (por exemplo, File Station) em janelas ou separadores independentes do navegador.

Nota:

Pode editar as definições de acesso para aplicações, tais como a Audio Station, a Download Station, a Surveillance Station, a Video Station, a File Station, assim como muitas outras.

Personalizar Pseudónimos

Pode atribuir um pseudónimo a cada aplicação desenvolvida para Synology, como File Station. Com o pseudónimo personalizado, pode abrir rapidamente a aplicação com um URL especializado.

Para personalizar pseudónimos para aplicações:

  1. Vá para Painel de Controlo > Portal de Aplicações > Aplicação.
  2. Seleccione uma aplicação.
  3. Clique em Editar > Geral.
  4. Na janela de contexto, selecione Ativar pseudónimo personalizado e especifique o pseudónimo.
  5. Clique em OK para guardar as definições.

Nota:

  • Algumas aplicações têm os seus pseudónimos predefinidos: Audio Station (áudio), Download Station (transferir), File Station (ficheiro), Surveillance Station (cam), e Video Station (vídeo).
  • Os nomes de alias não podem ser iguais aos reservados para o sistema, navegadores ou utilizados por outras aplicações e o respectivo número de carateres tem de ser limitado entre 2 e 20.
  • Um pseudónimo só pode ser constituído por letras maiúsculas e minúsculas, carateres numéricos e dois carateres especiais (-) e (_).
  • Um pseudónimo não pode começar ou terminar com os carateres especiais (-) ou (_).

Para aceder a aplicações com pseudónimos:

Depois de configurar um pseudónimo para uma aplicação (por exemplo, File Station), pode abrir rapidamente a aplicação introduzindo um URL nos seguintes formatos: http://DS_IP_OR_SERVER_NAME/ALIAS/ ou https://DS_IP_OR_SERVER_NAME/ALIAS/

Por exemplo, pode aceder facilmente a File Station com os URL: http://192.168.xx.xx/file/ ou https://MySynologyNAS/file/

Portas HTTP/HTTPS personalizadas

Pode atribuir uma porta HTTP/HTTPS a cada aplicação desenvolvida para Synology, como File Station. Com a porta personalizada, pode abrir rapidamente a aplicação com um URL especializado.

Para personalizar portas HTTP/HTTPS para aplicações:

  1. Vá para Painel de Controlo > Portal de Aplicações > Aplicação.
  2. Seleccione uma aplicação.
  3. Clique em Editar > Geral.
  4. Na janela de contexto, selecione Ativar porta personalizada (HTTP) ou Ativar porta personalizada (HTTPS).
  5. Especifique um número de porta personalizada.

Nota:

  • O número da porta deve ser entre 1 e 65535.
  • As seguintes portas não podem ser utilizadas, pois estão reservadas para utilização pelo sistema:
    • 20, 21, 22, 23, 25, 80, 110, 137, 138, 139, 143, 199, 443, 445, 515, 543, 548, 587, 873, 993, 995, 3306, 3689, 5000, 5001, 5005, 5006, 5335, 5432, 6881, 8080, 7000, 7001, 8081, 9997, 9998, 9999, 50001, 50002
    • Portas predefinidas para eMule: 4662 (TCP), 4672 (UDP)
    • Intervalo de portas FTP predefinido: O intervalo efetivo poderá depender dos diferentes modelos.

Para aceder a aplicações com portas HTTP/HTTPS:

Depois de configurar um pseudónimo para uma aplicação (por exemplo, File Station), pode abrir rapidamente a aplicação introduzindo um URL nos seguintes formatos: http://DS_IP_OR_SERVER_NAME:HTTP_PORT ou https://DS_IP_OR_SERVER_NAME:HTTPS_PORT

Por exemplo, pode aceder facilmente a File Station com os URL: http://192.168.xx.xx:7000 ou https://MySynologyNAS:7001

Personalizar Domínios

Pode fazer corresponder um nome de domínio com uma aplicação desenvolvida para Synology, como File Station. Com o domínio personalizado, pode abrir rapidamente a aplicação com um URL especializado.

Para personalizar domínios para aplicações:

  1. Vá para Painel de Controlo > Portal de Aplicações > Aplicação.
  2. Seleccione uma aplicação.
  3. Clique em Editar > Geral.
  4. Na janela de contexto, selecione Ativar domínio personalizado.
  5. Especifique um domínio personalizado.
  6. Para definições adicionais, pode selecionar Ativar HSTS ou Ativar HTTP/2 consoante as suas necessidades.

Nota:

  • Antes de configurar esta funcionalidade, deve solicitar nomes de domínios dos fornecedores de domínios para que os utilizadores consigam aceder ao servidor da Internet.
  • Para aplicações (por exemplo, File Station) associadas a carregamento/transferência de ficheiros, sugerimos que não selecione Ativar HTTP/2 porque as velocidades de carregamento e de transferência dessas aplicações podem ser baixas.
  • Cada nome de domínio pode ser usado apenas para uma aplicação em separado no seu DiskStation.
  • Se o nome de domínio não exceder os 15 caracteres de comprimento, e cumprir as convenções de nome NetBIOS, o sistema irá registar e transmitir o nome de domínio automaticamente.
  • Quando acede a aplicações através de URLs que utiliza nomes de domínio, irá percorrer a porta HTTP padrão (80) ou a porta HTTPS padrão (443).

Para aceder a aplicações através de domínios:

Depois de configurar um domínio (por exemploi, ficheiro.exemplo.com) para uma aplicação (por exemplo, File Station), pode abrir rapidamente a aplicação introduzindo um URL nos seguintes formatos: http://APP_DOMAIN ou https://APP_DOMAIN

Por exemplo, pode aceder facilmente a File Station com os URL: http://file.example.com ou https://file.example.com

Se o nome de domínio (por ex., FileFile) para uma aplicação (por ex., File Station) obedecer às convenções de nome NetBIOS, os utilizadores Windows podem aceder ao mesmo num Synology NAS na mesma rede local com um URL parecido: http://FileFile ou https://FileFile

Personalizar regras de proxy inverso

O seu DiskStation pode servir como servidor de proxy inverso que transfere pedidos da Internet para dispositivos na rede local. As regras de proxy inverso podem ajudá-lo a ocultar portas sensíveis de potenciais ameaças como nos dois seguintes cenários:

Cenário 1 Suponha que a porta sensível é 80, que não deve autorizar acesso externo de acordo com as regras da firewall. Pode configurar uma regra de proxy inverso para autorizar utilizadores de confiança na Internet a utilizar a porta sensível 80 através de outra porta aberta (por exemplo, 81). Deste modo, os utilizadores de confiança podem entrar na firewall e ainda conseguir aceder à porta 80.

Cenário 2 Suponha que a porta sensível é 80, que não deve autorizar acesso externo exceto através de um dispositivo específico (por exemplo, um servidor chamado "MyTrustee"). Com as regras de proxy inverso, pode autorizar que apenas o tráfego de MyTrustee utilize a porta 80 enquanto que os restantes dispositivos não o poderão fazer.

Para configurar regras de proxy inverso:

  1. Vá para Painel de Controlo > Portal de Aplicações > Proxy Inverso.
  2. Clique em Criar e configure as seguintes definições:
    • Descrição: Especifique um nome que o ajude a identificar a função da regra.
    • Especifique as regras da Origem (o dispositivo que envia pedidos da Internet) e do Destino (o dispositivo na rede local):
      • Protocolo: Os protocolos de HTTP ou de HTTPS utilizados pela origem/destino
      • Nome do anfitrião: O nome do dispositivo de origem/destino:
      • Porta: A porta utilizada pelo dispositivo de origem/destino:
      • Ativar HSTS e Ativar HTTP/2 (apenas para a origem)
  3. Clique em OK para guardar as definições.

Nota:

Se introduzir * como nome de anfitrião de origem ou deixar em branco, a regra aplica-se a todas as origens.

Visão Geral

Esta página apresenta a informação básica sobre o cluster de high-availability e dos servidores membros. Além disso, esta página oferece ferramentas para criar, remover ou efetuar a manutenção de clusters de high-availability e dos servidores membros.

Estado do Cluster de High Availability

Este campo apresenta o estado geral do seu cluster de high-availability, alertando-o para qualquer erro. A tabela seguinte apresenta as possíveis condições identificadas neste campo:

Estado Descrição
Autónomo Os servidores não são combinados num cluster de high-availability.
Saudável O estado geral do seu cluster de high-availability é bom. A duplicação de dados do servidor ativo para o servidor passivo funciona normalmente.
Preparação Os servidores foram combinados com êxito num cluster de high-availability mas a duplicação inicial dos dados ainda não terminou. Não é possível efetuar a mudança nesta altura. Para mais detalhes, consulte Mudança.
A carregar O sistema está a carregar o estado do cluster. Mudança não pode ser realizado até que esteja concluído o carregamento de estado.
Atenção O estado do cluster HA é anormal, mas os serviços ainda estão a funcionar.
Perigo Os servidores ativo e passivo estão avariados. O cluster de high-availability não poderá funcionar normalmente. Consulte o estado de todos os espaços de armazenamento (por exemplo, RAID Groups, Grupos de Discos, volumes e iSCSI LUNs).

Mensagens de Aviso

A tabela que se segue apresenta mensagens de aviso que podem aparecer quando ocorre um erro no sistema. Quando aparece mais do que uma mensagem de aviso, pode navegar entre as mesmas com os botões avançar/recuar.

Aviso/Erro Descrição
Erro de serviço monitorizado Ocorreu um erro num ou mais serviços monitorizados.
Espaço de armazenamento degradado Existe um espaço de armazenamento (por exemplo, RAID Group, Grupo de Discos, volume, iSCSI LUN e cache SSD) no servidor ativo ou passivo que está degradado. Verifique a página Estado de Armazenamento.
Espaço de armazenamento encravado Existe um espaço de armazenamento (por exemplo, RAID Group, Grupo de Discos, volume, iSCSI LUN e cache SSD) no servidor ativo ou passivo que está avariado. Verifique a página Estado de Armazenamento.
Erro ao detetar servidor passivo Este erro poderá ocorrer porque o servidor passivo está off-line ou os servidores ativo e passivo não podem ligar-se através da ligação Heartbeat. Neste caso, a mudança não estará disponível.
Não é possível duplicar dados para o servidor passivo Este erro poderá ocorrer devido a falha do volume de armazenamento ou do disco. Neste caso, a mudança não estará disponível. Verifique a página Estado de Armazenamento.
O servidor passivo não está selecionado Se o servidor passivo for desassociado do cluster de high-availability, este estado será apresentado. Neste caso, a mudança não estará disponível. Se pretender associar de novo o servidor passivo, clique no menu pendente Manter e clique em Associar. Quando associar de novo o servidor passivo, será necessária a duplicação total dos dados.

Informação sobre o Cluster de High Availability

Este campo encontra-se sob o estado do cluster de high-availability e apresenta informação básica sobre o cluster. Se um cluster de high-availability não existir, este campo só apresenta uma explicação da função de clusters de high-availability.

Este campo inclui a seguinte informação:

  • Nome do Servidor do Cluster: O nome do servidor utilizado pelo cluster de high-availability.
  • Endereço IP do Cluster: O endereço IP utilizado pelo cluster de high-availability. Independentemente do servidor que executa os serviços, os utilizadores podem ligar-se ao cluster de high-availability utilizando este endereço IP.
  • Data de construção: A data em que o cluster de high-availability foi criado.

Botões de Operação do Cluster de High Availability

Os dois botões seguintes encontram-se sob a informação do cluster de high-availability:

  1. Criar Cluster HA: Quando nenhum cluster de high-availability existir, pode clicar neste botão para combinar dois servidores e criar um cluster de high-availability. Para obter mais informações sobre como criar um cluster de high-availability, consulte Assistente de Synology High Availability.
  2. Gestão: Este botão só está disponível quando já existe um cluster de high-availability. Inclui as seguintes opções:
    • Assistente de gestão: Inicia um assistente que inclui instruções para vários procedimentos de gestão.
    • Mudança: Inicia uma mudança do servidor activo para o servidor passivo. Durante este processo, o servidor ativo interrompe os serviços e passa a ser o novo servidor passivo. Em seguida, o servidor passivo antigo torna-se no novo servidor ativo e começa a fornecer todos os serviços do cluster de high-availability. Esta opção não está disponível quando o servidor passivo tiver o estado erro ou não estiver selecionado.
    • Atualizar o DSM: Abre a página Actualização do DSM do Painel de Controlo. A partir daqui, os dois servidores podem ser atualizados.
    • Remover cluster HA: Remove o cluster de high-availability.
    • Desassociar servidor passivo: Remove o servidor passivo do cluster. Quando o servidor for adicionado ao cluster de high-availability de novo depois de desassociar, a duplicação total dos dados será necessária. Assim, recomendamos esta opção só quando pretender substituir o servidor passivo existente por um novo.
    • Associar servidor passivo: Adiciona um servidor passivo ao cluster. Esta opção só está disponível quando não existe qualquer servidor passivo no cluster de high-availability.
    • Encerrar cluster HA: Encerra os servidores activo e passivo.
    • Encerrar servidor ativo: Encerra o servidor activo.
    • Encerrar servidor passivo: Encerra o servidor passivo.
    • Reiniciar servidor activo: Reinicia o servidor activo.
    • Reiniciar servidor passivo: Reinicia o servidor passivo.
    • Desativar sinal sonoro no servidor passivo: Pára o sinal sonoro do servidor passivo quando ocorre um erro. O servidor passivo ainda emite um sinal sonoro se ocorrer outro erro mais tarde.

Nota:

  • Quando um cluster de high-availability for removido, será necessária uma duplicação total dos dados quando pretender criar um novo.
  • Se encerrar ou desassociar o servidor passivo, não é possível mudar os serviços se ocorrerem erros no servidor ativo.

Estado do Servidor e da Duplicação de Dados

Esta secção contém imagens que indicam o estado dos servidores ativo e passivo, assim como o fluxo da duplicação de dados entre os servidores.

A imagem da duplicação de dados indica as seguintes condições possíveis:

  • A duplicação dos dados está a funcionar normalmente: Representado por dados que fluem do servidor activo para o servidor passivo.
  • Nenhuma duplicação de dados: Representado pela ausência de imagens de fluxo de dados.

Informação do Servidor

Esta secção encontra-se sob as imagens do servidor. A informação sobre os servidores ativo e passivo é apresentada nesta secção. Se nenhum servidor passivo for selecionado, esta secção indicará N/A.

  • Nome de Servidor: Nome do servidor Synology.
  • Nome do Modelo: Modelo do servidor Synology.
  • Número de Série: Número de série do servidor Synology.
  • Estado da Ventoinha: Estado das ventoinhas de arrefecimento do servidor. Se uma ventoinha avariar, surgirá um aviso. Neste caso, contacte o seu distribuidor local para obter assistência.
  • Temperatura: Temperatura do servidor Synology.
  • Estado da Fonte de Alimentação: Indica se as fontes de alimentação de um servidor estão ou não a funcionar normalmente. Se o modelo do servidor estiver equipado com fontes de alimentação redundantes, surgirá um aviso se uma delas avariar.

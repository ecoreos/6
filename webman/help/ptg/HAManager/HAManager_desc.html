Gestor de High Availability

High Availability refere-se a uma solução de esquema de servidor concebida para reduzir as interrupções de serviços provocadas por avarias no servidor. O Synology High Availability utiliza dois servidores para formar um “cluster de high-availability” em que um servidor funciona como “servidor activo” e outro servidor funciona como “servidor passivo” em espera. Para implementar e criar um cluster de high-availability, são necessários dois servidores Synology compatíveis.

Um cluster de alta disponibilidade tira partido dos seguintes componentes e conceitos:

  • Conceito: A implementação de Synology High Availability requer dois servidores Synology para criar um cluster de high-availability. Um servidor serve como "servidor activo" e outro serve como "servidor passivo". Quando o cluster de high-availability for criado, os dados são continuamente duplicados do servidor activo para o servidor passivo. Todos os ficheiros no servidor activo existirão em duplicado no servidor passivo. Em caso de avaria do servidor activo, o servidor passivo estará pronto para tomar conta de todos os serviços com uma cópia espelhada completa dos dados do servidor activo. Assim, o cluster de high-availability continuará a providenciar os serviços normalmente, minimizando o tempo de inactividade do servidor. Para mais detalhes sobre a mudança, consulte Mudança
  • Servidor activo: O servidor activo gere a provisão de todos os serviços. No caso de ocorrer um evento no servidor activo, você ou o sistema podem iniciar uma mudança para o servidor passivo. Neste caso, a provisão dos serviços será transferida para o servidor passivo, e a partir desse momento o servidor activo passa a ser servidor passivo.
  • Servidor passivo: Em condições normais, o servidor passivo continua em modo de espera. Os dados do servidor activo serão continuamente duplicados para o servidor passivo. No caso do servidor activo não conseguir providenciar os serviços, você ou o sistema podem iniciar uma mudança para o servidor passivo.
  • Ligação heartbeat: Os servidores activo e passivo de um cluster de high-availability estão ligados por uma ligação de rede privada dedicada mencionada como "ligação Heartbeat". Quando o cluster de high-availability for criado, a ligação Heartbeat facilita a sincronização de dados e a duplicação entre os servidores activo e passivo. Se o contacto entre os servidores for interrompido durante um período de tempo específico, o sistema determinará se uma mudança é necessária.
  • Volume de armazenamento principal do cluster de high-availability: O volume de armazenamento do servidor activo passará a ser o volume de armazenamento principal do cluster de high-availability. Os dados são duplicados para o volume de armazenamento do servidor passivo através da ligação Heartbeat.
  • Volume suplementar de armazenamento do cluster de alta disponibilidade: O volume de armazenamento no servidor passivo torna-se no volume suplementar de armazenamento do cluster de alta disponibilidade. Os dados são continuamente duplicados do volume de armazenamento do servidor activo para o volume de armazenamento passivo através da ligação Heartbeat. Isto garante a consistência do conteúdo dos ficheiros quando ocorrer uma mudança.
  • Nome do servidor do cluster de alta disponibilidade: Os servidores no cluster de high-availability partilham um nome de servidor. Na rede, pode ligar utilizando este nome de servidor e os serviços serão providenciados pelo servidor activo.
  • IP do cluster de alta disponibilidade: Uma vez que o servidor activo e passivo no cluster de alta disponibilidade têm o seu próprio endereço IP, para evitar a mudança para outro endereço IP quando ocorrer uma mudança, o cluster de alta disponibilidade gerará endereços IP comuns para acesso. Para aceder a qualquer serviço DSM, independentemente do servidor Synology que é o servidor activo, pode ligar através destes endereços IP.

Mudança

Para garantir o desempenho contínuo e minimizar o tempo de inatividade de serviços, pode mudar os serviços do servidor ativo de um high-availability cluster para o servidor passivo redundante. Esta funcionalidade chama-se "Mudança" e, em alguns casos "Ativação após-falha". Esta secção descreve os princípios básicos destes dois conceitos.

Mudança

Mudança refere-se a uma mudança de um servidor ativo para servidor passivo redundante de um high-availability cluster. Em condições normais, o servidor ativo gere todos os serviços. No entanto, no caso do servidor ativo avariar, você pode iniciar uma mudança e transferir todos os serviços para o servidor passivo, desde que o servidor passivos esteja a funcionar normalmente. Neste caso, o servidor ativo assume o papel de servidor passivo e o servidor passivo assume o papel de servidor ativo.

Para iniciar manualmente uma mudança:

  1. Aceda ao separador Visão geral.
  2. Clique em Gestão > Mudança.

Nota:

Quando uma mudança é iniciada, o servidor ativo assume primeiro o papel de servidor passivo e depois o servidor passivo passa a ser o servidor ativo. Assim, haverá um curto período de tempo no qual ambos os servidores são passivos e não poderão providenciar os serviços normalmente. O tempo para concluir a mudança varia em função do número e tamanho dos volumes ou iSCSI LUN (nível do bloco) ou o número e carga total de serviços do high-availability cluster.

Ativação após-falha

Quando ocorrerem determinados erros, o sistema pode iniciar automaticamente uma mudança de serviços para garantir o desempenho. Este evento chama-se "ativação após falha". O sistema poderá iniciar uma ativação após falha nas seguintes situações:

  • Espaço de armazenamento encravado: Um espaço de armazenamento (por exemplo volume, Grupo de Discos, RAID group, cache SSD, etc.) no servidor ativo encravou, mas o espaço de armazenamento correspondente no servidor passivo está a funcionar normalmente. Tome nota que o sistema não inicia uma ativação após falha se não existirem volumes ou iSCSI LUN (nível do bloco) no espaço de armazenamento encravado.
  • Erro de serviço: Ocorreu um erro num serviço monitorizado.
  • Falha de energia: O servidor ativo é desligado ou reiniciado; ambas as unidades de energia do servidor ativo falharam ou ocorreu uma quebra de energia.

Falha da Mudança

A mudança de serviços ou ativação após falha poderá falhar nas seguintes situações:

  • Duplicação de dados incompleta: Quando combinar primeiro dois servidores para criar um cluster high-availability, o sistema necessita de tempo para duplicar todos os dados do servidor ativo para o servidor passivo.
  • Espaço de armazenamento no servidor passivo está encravado: Um espaço de armazenamento (por exemplo volume, Grupo de Discos, RAID group, etc.) no servidor passivo encravou.
  • Falha de energia: O servidor passivo é desligado ou reiniciado; ambas as unidades de energia do servidor passivo falharam ou ocorreu uma quebra de energia.
  • Atualizar o DSM: Quando instala atualizações DSM, todos os serviços serão parados e irão retomar após a instalação da atualização DSM estar completa.

Se iniciar manualmente uma mudança, mas não for possível mudar os serviços para o servidor passivo, então o sistema não tentará mudar os serviços de novo para o servidor ativo.

No caso da ativação após-falha ser iniciada mas nenhum servidor assumir o papel de servidor ativo por causa de erros, o high-availability cluster não poderá providenciar o serviço normalmente. Se o estado dos servidores não passar automaticamente para Autónomo, prima o botão Repor em ambos os servidores para restaurar as predefinições. Quando as predefinições forem restauradas, pode combinar os servidores e criar de novo um high-availability cluster.

Fonte de alimentação ininterrupta

Os high-availability clusters podem ser ligados a fontes de alimentação ininterruptas (UPS) para reduzir o risco de falhas de energia inesperadas. Se ocorrer falha de energia, os servidores receberão energia auxiliar de qualquer unidade UPS ligada. Quando a unidade UPS ligada ao servidor ativo começar a ficar sem energia, o sistema tentará uma mudança de serviços para o servidor passivo. Veja as diretrizes em baixo.

  • Se os servidores ativo e passivo estiverem anexados ao mesmo interruptor de rede, recomenda-se que ligue ambos os servidores à mesma fonte de alimentação que o interruptor de rede.
  • Se os servidores ativo e passivo estiverem anexados a diferentes interruptores de rede que utilizam diferentes fontes de alimentação (por ex. eletricidade de diferentes companhias elétricas), recomenda-se que ligue os servidores ativo e passivo às fontes de alimentação correspondentes, iguais às dos interruptores de rede anexos.
  • Se os servidores ativo e passivo estiverem anexados a diferentes interruptores de rede que utilizem diferentes unidades UPS, recomenda-se que ligue ambos os servidores a uma das unidades UPS em vez de a duas unidades UPS diferentes. Quando a unidade UPS perdem potência, o servidor ativo será notificado e entra em modo de segurança e o servidor passivo desliga-se devidamente. Se os servidores estiverem ligados a diferentes unidades UPS de outro modo, o servidor ativo não poderá detetar se o UPS do servidor passivo está a ficar sem carga e o servidor passivo pode então inesperadamente desligar-se.

Nota:

DiskStation suporta a ligação a dispositivos locais e SNMP UPS. Para mais informações, consulte a ajuda em Painel de Controlo > Hardware > UPS.

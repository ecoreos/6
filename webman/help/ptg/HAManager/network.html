Definições de rede

Esta página contém três separadores que apresentam as definições de rede do cluster de high-availability, o servidor membro activo e o servidor membro passivo.

Separador Cluster HA

Quando criar um cluster de high-availability, o sistema gera um endereço IP comum para o IP do cluster de high-availability para receber ligações de entrada. Se necessário, podem ser criados endereços IP adicionais para cluster de alta disponibilidade. OS endereços IP das interfaces de rede activadas no servidor activo serão apresentadas neste separador (a interface de rede utilizada pela ligação Heartbeat não será apresentada nesta página, porque a dita interface será utilizada para duplicação de dados entre dois servidores). Este separador mostra a seguinte informação.

  • Interface de Rede: A(s) interface(s) de rede activada(s) no servidor activo.
  • Ligação: Mostra se esta interface está ou não ligada.
  • Endereço IP: Quando criar um cluster de high-availability, o sistema gera um endereço IP comum para o IP do cluster de high-availability receber ligações de entrada. Pode ligar-se ao cluster de high-availability utilizando este endereço IP quando pretender iniciar sessão no DiskStation Manager (DSM), independentemente de qual seja o servidor Synology activo. Isto evita que seja necessário ligar a endereços IP diferentes quando ocorrer uma mudança.
  • Máscara Subrede: A máscara de subrede associada para um determinado endereço IP do cluster de alta disponibilidade.

Para editar Endereço IP e Máscara Subnet:

  1. Seleccione o endereço IP ou máscara subnet que pretende editar e clique em Editar ou clique duas vezes no campo para começar a editar.
  2. Marque as caixas de verificaçãoo Activar para activar ou desactivar cada interface. Pode activar mais do que uma interface de rede de cluster mas a interface de rede original configurada durante a criação do cluster não pode ser desactivada.

Para alterar o nome do servidor do cluster de alta disponibilidade:

  1. Clique em Alterar definições de rede para ver o nome do cluster de high-availability e alterar as suas definições.
  2. Clique em Aplicar para guardar o actual nome e as actuais definições do endereço IP do cluster de alta disponibilidade.
  3. Para ignorar as actuais alterações, clique em Cancelar.

Separadores Servidor Activo e Passivo (apresentado como nome de servidor)

Estes separadores apresentam o estado das interfaces de rede no servidor activo ou passivo. Destas, a ligação Heartbeat ocupará uma interface de rede em cada um dos servidores activo e passivo. Estes separadores apresentam a seguinte informação:

  • Interface de rede: A interface de rede física no servidor.
  • Endereço IP:O endereço IP atribuído a uma determinada interface de rede.
  • Máscara Subnet:A máscara de subrede associada a um determinado endereço IP.
  • Descrição: Apresenta informação detalhada de uma determinada interface de rede e a acessibilidade do seu endereço IP atribuído. Este campo apresenta as seguintes condições possíveis:
    1. Endereço IP original acessível: O endereço IP original atribuído à interface do servidor activo é acessível para dados e serviços.
    2. Endereço IP original não acessível: O endereço IP original atribuído à interface no servidor passivo não é acessível.
    3. IP de ligação Heartbeat, não acessível da Intranet: O endereço IP original atribuído automaticamente à ligação Heartbeat não é acessível para dados e serviços.

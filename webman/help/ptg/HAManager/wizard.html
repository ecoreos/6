Assistente do High Availability Manager

Este assistente ajuda-o durante o processo de criação e de configuração de um cluster de high-availability.

Para iniciar este assistente:

  1. Abra o Gestor de High Availability.
  2. Aceda à página Visão geral.
  3. Clique em Criar cluster HA.

Nota:

  • Conclua o assistente de configuração do cluster de high-availability no limite de tempo fornecido. Se o limite de tempo for excedido, terá que recomeçar o processo de configuração.
  • Se for necessário identificar qual o servidor ativo num cluster de high-availability, recomendamos que transfira a aplicação DS finder da Synology e utilize a respetiva função de sinal sonoro, que faz com que o servidor ativo emita um breve sinal sonoro.

Para configurar o servidor passivo:

Designe um servidor Synology como servidor passivo. Em condições normais, o servidor passivo não providencia serviços. No entanto, em caso de avaria, os serviços podem ser transferidos do servidor ativo para o servidor passivo.

  1. Endereço IP: Introduza o endereço IP do servidor passivo.
  2. Nome do utilizador: Introduza uma conta de utilizador pertencente ao grupo administrator no servidor de destino.
  3. Palavra-passe: Introduza a palavra-passe para a conta pertencente ao grupo administrator.

Nota:

  • Quando o cluster de high-availability for criado, todos os dados no servidor passivo serão eliminados. Certifique-se de que não existem dados importantes armazenados no servidor passivo.

Para configurar a ligação Heartbeat:

Ligue os dois servidores entre si. Esta ligação deve funcionar como ligação "Heartbeat" entre os dois servidores, facilitando a comunicação e permitindo a duplicação dos dados do servidor ativo para o servidor passivo. Para garantir a qualidade da ligação Heartbeat, o sistema vai verificar se esta ligação está conforme os seguintes critérios quando cria o high-availability cluster:

  • Entre os dois servidores, não há outros dispositivos com endereços IP encontrados no mesmo domínio de emissão. Se esta ligação Heartbeat passar por algum switch, tem de garantir que os switches são compatíveis com multicast e Jumbo Frame.
  • Ter uma ligação de rede de baixa latência. O tempo de resposta do Ping entre os dois servidores deve ser inferior a 1 ms, e a velocidade de transmissão deve ser de pelo menos 500 Mbps. MTU será configurada automaticamente para 9000.
  • Utilize a mesma interface de rede nos dois servidores. Por exemplo, se uma extremidade estiver ligada a LAN 1 num servidor, a outra extremidade tem de estar ligada a LAN 1 no outro servidor.
  • Utilize a interface de rede mais rápida nos dois servidores. Se os servidores estiverem equipados com placas de interface suplementar de 10GbE, esta ligação terá de utilizá-las.

Depois de confirmar os procedimentos acima, vá para a secção Interface da Ligação HA (Heartbeat) e escolha a interface de rede nos servidores para funcionar como ligação Heartbeat.

Nota:

  • As definições da Ligação Heartbeat irão produzir dois conjuntos de endereços IP (por exemplo, 169.254.1.1 e 169.254.1.2) para efeitos de duplicação de dados entre o servidor ativo e o servidor passivo. Certifique-se de que estes dois endereços IP não são utilizados por qualquer outro equipamento na rede.
  • Activar VLAN ID nas portas da ligação Heartbeat do seu DiskStation não é suportado.

Configurar Link Aggregation nas ligações de dados ou Heartbeat:

  • Para configurar o Link Aggregation na ligação de dados ou Heartbeat, configure o Link Aggregation em duas ou mais portas LAN em Painel de Controlo > Rede > Interface de Rede antes de iniciar o assistente de criação de high availability. Certifique-se de que as definições de Link Aggregation são idênticas em ambos os servidores (por exemplo, se configurar o Link Aggregation nas portas 1 e 2 do servidor ativo, o Link Aggregation também tem de estar configurado nas portas 1 e 2 do servidor passivo).
  • O Heartbeat está equipado com capacidades para recuperação após falha e equilíbrio de carga, mas as ligações de dados dependem do modo da Link Aggregation que escolher.

Para configurar as definições da interface de rede do cluster de high-availability:

O cluster de high-availability deverá utilizar um endereço de IP para cluster de high-availability como interface de ligação para o exterior. Pode ligar-se utilizando este endereço IP do cluster de high-availability quando pretender utilizar serviços DSM, independentemente do Synology que seja o servidor ativo. Isto impede que seja necessário ligar a diferentes endereços de IP depois de fazer a mudança do servidor ativo para o servidor passivo.

  1. Nome do Servidor do Cluster: O nome do cluster de high-availability. Consulte as regras de atribuição de nome ao servidor (em Ajuda do DSM > Painel de Controlo > Conectividade > Rede > Geral) para obter informações sobre as restrições do nome.
  2. Interface de Rede: A interface de rede a utilizar pelo cluster de high-availability. Deve selecionar uma porta LAN diferente daquela que foi designada para duplicação de dados na fase anterior.
  3. Endereço IP: O endereço IP a utilizar pelo cluster de high-availability. Independentemente do servidor que providencia ativamente os serviços, o cluster pode ser ligado através deste IP.
  4. Máscara de Sub-rede: A máscara de sub-rede a utilizar pelo cluster de high-availability.

Atribuir um ID de VLAN à ligação de dados:

  • Para atribuir um ID de VLAN à ligação de dados, vá para Painel de Controlo > Rede > Interface de Rede antes de iniciar o assistente de criação de high-availability. Certifique-se de que as definições de VLAN são idênticas nos dois servidores (por exemplo, se LAN 1 estiver definido com VLAN ID 1 no servidor ativo, o LAN 1 do servidor passivo também tem de estar definido como VLAN 1).

Para confirmar definições:

Confirme as definições a utilizar pelo cluster de high-availability. O sistema inicia a associação do cluster de high-availability assim que clicar em Aplicar.

Nota:

  • Quando criar um cluster de high-availability, todos os dados do servidor ativo têm de ser duplicados para o servidor passivo. Se perder a ligação ao servidor passivo durante mais de 30 minutos antes de concluir o processo de duplicação inicial, o servidor passivo irá regressar ao modo Autónomo. Neste caso, será necessário reassociar o servidor e efetuar novamente uma duplicação completa dos dados.

Encerramento e Reiniciação

As seguintes instruções ajudá-lo-ão durante o procedimento de encerramento, remoção ou reiniciação de um cluster de high-availability.

Para encerrar o cluster de high-availability com Synology High Availability:

Quando pretender encerrar um cluster de high-availability para transportar os servidores para uma nova localização física ou efectuar manutenção, mas não pretende remover as configurações do cluster de high-availability, siga as instruções em baixo.

  1. Inicie a sessão no DSM.
  2. Abra o Synology High Availability.
  3. Clique no botão marcado como Gestão.
  4. Seleccione Encerrar cluster HA.
  5. Aparece uma mensagem de confirmação. Clique em Sim para confirmar.

Para encerrar manualmente um cluster de high-availability:

O cluster de high-availability também pode ser encerrado usando manualmente os botões de energia localizadas em cada servidor.

  1. Mantenha premido o botão de energia num servidor (activo ou passivo) até emitir um sinal sonoro.
  2. Depois do servidor ser completamente encerrado, aguarde pelo menos 30 segundos (o servidor que ainda está online demora algum tempo a guardar as definições).
  3. Mantenha premido o botão de energia no outro servidor até emitir um sinal sonoro.
  4. Aguarde até o servidor encerrar completamente.

Ligar Servidores:

Quando pretender ligar de novo os servidores, deve ligar tanto o servidor activo como passivo ao mesmo tempo (ou com um intervalo de cinco minutos entre si). Se apenas ligar um dos servidores, este irá tentar ligar o outro servidor emitindo uma mensagem Despertar pela LAN.

Encerrar anormal:

Quando é criado um cluster high-availability, a opção reiniciar após falha de energia no Painel de Controlo é activada automaticamente. Se os servidores encerrarem de forma anormal devido a uma falha de energia, ligar-se-ão automaticamente assim que for restaurada a energia.

Para remover um cluster de high-availability:

Quando pretender desassociar ambos os servidores e remover um cluster de high-availability para poder associar um servidor diferente, siga estas instruções.

  1. Vá para a página Visão geral.
  2. Clique no botão Gestão.
  3. Seleccione em Remover cluster HA.

Após confirmação, o sistema removerá o cluster de high-availability. O campo de estado apresenta o progresso do processo de remoção. Quando o cluster de high-availability for finalmente removido, será encaminhado para o endereço de IP original do servidor activo. O estado de ambos os servidores muda para Autónomo.

Para reiniciar um cluster de high-availability avariado:

No caso de não conseguir aceder ao DSM ou Synology High Availability, devido a um erro de split-brain ou outras avarias, pode tentar forçar o cluster de high-availability a desassociar premindo o botão Repor nos servidores.

  1. Mantenha premido o botão Repor (normalmente no painel traseiro de um servidor) até o servidor emitir um sinal sonoro.
  2. O sistema removerá o cluster de high-availability e o estado de ambos os servidores muda para Autónomo
  3. Se premir o botão Repor também restaura algumas predefinições, como endereço de IP, DNS e palavra-passe de admin, para além do seguinte:
    • A palavra-passe de admin será reposta como vazia.
    • A conta de convidado será desactivada.
    • DHCP será activado em IP, DNS, Gateway e outras interfaces de rede.
    • JumboFrames será desactivado e MTU será definido como 1500.
    • Porta de Gestão será reposta como 5000/5001.
    • PPPoE será reconfigurado para a predefinição.
    • Bloqueio automática de IP será desactivado (o endereço de IP bloqueado não será apagado).
    • Link Aggregation será desactivada.
    • As regras de firewall serão desactivadas (as regras não serão apagadas).

Nota:

Durante o procedimento de remoção de um cluster de high-availability, algumas operações de escrita de dados poderão ficar incompletas. Para garantir a plenitude dos dados, efectue a duplicação de dados com base no servidor activo porque poderão faltar algumas actualizações de dados no servidor passivo.

Salvaguardar o seu Synology NAS

Visão Geral

Perdas inesperadas de dados devido a falhas de discos rígidos ou desastres naturais constituem potenciais ameaças aos seus importantes dados, em especial quando não está disponível qualquer solução fiável de cópia de segurança. Hyper Backup proporciona uma solução intuitiva e integrada de cópia de segurança/restauro para o ajudar a reter/obter cópias dos seus preciosos dados e restaurar aplicações e configurações de sistema a partir de um ponto no tempo. São compatíveis os seguintes destinos de armazenamento:

  • Pastas partilhadas locais (incluindo dispositivos de armazenamento externo ligados ao seu DiskStation)
  • Synology NAS remoto
  • Servidores rsync Remotos
  • Servidores OpenStack Swift Remotos
  • Nuvens públicas (Amazon S3, Microsoft Azure, e OpenStack Swift, etç.)

Nota:

Hyper Backup salvaguarda apenas certas configurações de sistema e aplicações. Para mais informações, consulte a documentação de ajuda para Hyper Backup.

1. Crie tarefas de cópia de segurança local numa pasta partilhada local/dispositivo de armazenamento externo

Hyper Backup proporciona semelhantes procedimentos de cópia de segurança e restauro em diferentes tipos de destinos. Abaixo mostramos como salvaguardar e restaurar dados localmente, como referência de outros tipos de tarefas.

  1. Inicie a sessão no DSM e localize o pacote Hyper Backup.
  2. Clique em + no canto inferior esquerdo e selecione Tarefa de cópia de segurança de dados.
  3. Para configurar uma tarefa de cópia de segurança local, selecione Pasta Partilhada Local.
  4. Selecione Criar tarefa de cópia de segurança e selecione uma pasta partilhada local ou um dispositivo de armazenamento externo ligado ao seu DiskStation.
  5. Selecione a(s) pasta(s) de origem para salvaguardar os dados contidos. Pastas partilhadas encriptadas, se selecionadas, devem permanecer montadas durante o processo de cópia de segurança. Se a origem e o destino estiverem no mesmo volume, o sistema irá assinalar tais pastas de origem com pontos de exclamação a laranja.
  6. Selecione a(s) aplicação(ões) a salvaguardar.
  7. Especifique abaixo as definições de cópia de segurança para se ajustarem às suas necessidades:
    • Tarefa: Dê um nome à tarefa de cópia de segurança.
    • Ativar notificação de tarefa: Selecione para ser informado do estado da tarefa por email, SMS ou serviço push.
    • Ativar a cópia de segurança de configuração: Salvaguardar configurações de sistema (por ex., credenciais de utilizador). Esta definição é aplicada a todas as tarefas de cópia de segurança por predefinição.
    • Comprimir dados de cópia de segurança: Comprimir dados de cópia de segurança para reduzir a utilização do armazenamento de destino. É necessário mais tempo para a cópia de segurança caso esta opção esteja selecionada.
    • Ativar encriptação de transferência (apenas para cópia de segurança remota): Selecione para encriptar os dados de cópia de segurança durante a transferência para um destino remoto (por ex., um servidor rsync ou nuvem pública).
    • Tamanho de parte de Carregamento Multipartes (apenas para servidores compatíveis com S3): Dividir dados de cópia de segurança pelo tamanho de bloco especificado.
    • Ativa a encriptação do lado do cliente: Selecione para proteger os dados de cópia de segurança com uma palavra-passe, para evitar acesso indesejado do lado do destino.
    • Ativar horário da cópia de segurança: Selecione para agendar uma tarefa de cópia de segurança automática.
  8. Selecione Ativar rotação de cópia de segurança para configurar o esquema de rotação:
    • De versões mais antigas: Elimina as versões de cópias de segurança mais antigas existentes assim que o Número de versões for ultrapassado.
    • Smart Recycle: O sistema irá reter todas as versões de cópia de segurança até ser excedido o número especificado de versões. Quando é acionada a rotação, o sistema irá primeiro rodar as versões que não cumpram quaisquer das condições; se todas as versões existentes cumprirem as condições abaixo, o sistema irá rodar a versão mais antiga:
      • Versões de hora a hora das últimas 24 horas: O sistema mantém a versão mais antiga criada a cada hora.
      • Versões diárias do último 1 dia a 1 mês: O sistema mantém a versão mais antiga criada a cada dia.
      • Versão semanal após 1 mês: O sistema mantém a versão mais antiga criada cada semana.
    • Número de versões: Defina o limite superior de versões possíveis de cópias de segurança retidas.
    • Cronologia: Apresenta o possível resultado da retenção de cópia de segurança de acordo com o seu horário e esquema de rotação de cópia de segurança:
  9. Clique em Aplicar para concluir a configuração.
  10. Para realizar de imediato a tarefa de cópia de segurança, siga as instruções do Assistente de Cópia de Segurança e clique em Sim.

2. Restaurar ficheiros individuais de uma pasta partilhada local/dispositivo de armazenamento externo.

  1. Aceda a Hyper Backup, selecione a tarefa de cópia de segurança desejada no painel da esquerda, e clique em Explorador de Cópias de Segurança (o ícone da lupa). Introduza a palavra-passe caso a tarefa esteja protegida por encriptação do lado do cliente.
  2. Localize e selecione o ficheiro desejado, para restaurar. Pode também selecionar a versão desejada para restaurar com a barra de cronologia.
  3. Clique em Restaurar para restaurar o ficheiro/pasta selecionado.

3. Restaurar tarefas completas aos momentos desejados

  1. Clique em Restauro no canto inferior esquerdo e clique em Dados. Introduza a palavra-passe caso a tarefa esteja protegida por encriptação do lado do cliente.
  2. Selecione uma tarefa a restaurar. Para restaurar dados de outros repositórios, selecione Restaurar de outros repositórios e localize a tarefa a restaurar.
  3. Escolha entre restaurar ou não as configurações de sistema. Se selecionar Restaurar configuração de sistema, selecione configurações específicas na lista abaixo, para restaurar.
  4. Localize e selecione o(s) ficheiro(s)/tarefa de destino, para restaurar. Pode também selecionar a versão desejada para restaurar com a barra de cronologia.
  5. Selecionar aplicações para restaurar.
  6. Serão apresentadas informações básicas de restauro.
  7. Clique em Aplicar para iniciar o restauro.

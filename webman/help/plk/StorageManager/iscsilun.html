iSCSI LUN*

iSCSI to usługa sieciowa pamięci masowej, która ułatwia przesyłanie danych za pośrednictwem sieci lokalnych i na dalekie odległości. Po odwzorowaniu jednostek iSCSI LUN do obiektów iSCSI Target klienty mogą uzyskiwać dostęp do miejsc w pamięci masowej jak do dysków lokalnych. iSCSI LUN (numer jednostki logicznej) można odwzorować na jeden lub wiele obiektów iSCSI Target w celu udostępniania usług pamięci masowej iSCSI. Wybierz opcję Zarządzanie przechowywaniem > iSCSI LUN, aby zarządzać jednostkami iSCSI LUN.

Uwaga:

Liczba obsługiwanych jednostek iSCSI LUN zależy od modeli serwerów DiskStation. Więcej informacji o danych technicznych produktu można znaleźć w witrynie www.synology.com.

DiskStation obsługuje obecnie następujące typy jednostek iSCSI LUN:

  • iSCSI LUN (zwykłe pliki): jednostkę iSCSI LUN tworzy się, przeznaczając na nią część miejsca woluminu. Można ją później powiększać stosownie do potrzeb. Ten typ jednostek iSCSI LUN zapewnia elastyczność dynamicznego zarządzania pojemnością przy użyciu funkcji Thin Provisioning, obsługi przyspieszenia sprzętowego VMware VAAI, funkcji Windows ODX, migawek jednostek LUN i klonowania jednostek LUN.
  • iSCSI LUN (poziom bloku) — pojedyncza LUN w RAID: Jednostkę iSCSI LUN tworzy się z wykorzystaniem całego miejsca na woluminie. Można ją naprawiać lub powiększać albo zmieniać jej typ RAID, jak w przypadku woluminu. Więcej informacji można znaleźć na stronie Wolumin.
  • iSCSI LUN (poziom bloku) — wiele LUN w RAID: Jednostkę iSCSI LUN tworzy się przez wyznaczenie określonego miejsca w grupie dysków. Jednostkę iSCSI LUN można później powiększyć jeżeli w grupie dysków jest dostępne miejsce. Więcej informacji można znaleźć na stronie Grupa dysków.
  • iSCSI LUN (poziom bloku): ten typ jednostki iSCSI LUN tworzy się w grupie RAID Group. Oferuje on funkcje dynamicznego zarządzania pojemnością w połączeniu ze zoptymalizowaną wydajnością dostępu. Jednostkę iSCSI LUN tego typu można później powiększyć, wykorzystując nieprzydzielone miejsce w grupie RAID Group. Więcej informacji można znaleźć na stronie Grupa RAID Group.

Uwaga:

Jednostki iSCSI LUN z zaawansowanymi funkcjami LUN są obsługiwane tylko w systemie plików ext4/Btrfs.

Aby utworzyć jednostkę iSCSI LUN:

  1. Kliknij przycisk Utwórz.
  2. Wybierz typ iSCSI LUN, który ma być utworzony.
  3. Postępuj zgodnie z instrukcjami kreatora, aby zakończyć proces.

Uwaga:

Zaawansowane funkcje LUN można włączyć tylko podczas początkowego tworzenia jednostki iSCSI LUN. Tego ustawienia nie można później zmieniać.

Rozmiar jednostki alokacji zaawansowanej jednostki LUN można zoptymalizować pod kątem pewnych funkcji serwera aplikacji, np. Microsoft Exchange lub SQL Server albo serwerów wirtualizacji i technologii przyspieszających pamięć masową, takich jak Windows ODX lub VMware VAAI. Aby to zrobić, wykonaj jedną z następujących czynności:

  • Ustaw rozmiar jednostki alokacji podczas tworzenia pierwszej zaawansowanej jednostki LUN woluminu.
  • Ustaw rozmiar jednostki alokacji w obszarze Konfiguruj przed utworzeniem pierwszej zaawansowanej jednostki LUN woluminu.

Zaleca się określenie rozmiaru jednostki alokacji na potrzeby aplikacji podczas używania jednej z następujących metod pracy z zaawansowanymi jednostkami iSCSI LUN na serwerze DiskStation: bezpośredniego połączenia z serwerem aplikacji lub RDM (pass-through). Aby wykorzystać technologię Synology Advanced LUN do klonowania na potrzeby funkcji Windows ODX, zaleca się sformatowanie woluminu NTFS z jednostką alokacji o rozmiarze równym (lub większym) temu, który został określony w systemie DSM lub większym.

Uwaga:

Jednostkę alokacji zaawansowanej jednostki LUN można ustawić tylko raz.

Aby usunąć jednostkę iSCSI LUN:

Usunięcie jednostki iSCSI LUN spowoduje usunięcie wszystkich danych przechowywanych na tej jednostce iSCSI LUN. Zostaną utracone także połączenia między jednostką iSCSI LUN a wszystkimi odwzorowanymi obiektami iSCSI Target.

  1. Wybierz jednostkę iSCSI LUN, która ma być usunięta.
  2. Kliknij przycisk Usuń i postępuj zgodnie z instrukcjami kreatora, aby ukończyć proces.

Uwaga:

Usuwanie jednostek VAAI LUN na serwerze DiskStation nie powoduje natychmiastowego zwolnienia miejsca na dyskach. Dane są przenoszone do zarezerwowanego przez system folderu @EP_trash i stopniowo usuwane. Jeżeli na przykład jednostka VAAI LUN o wielkości 100 GB zostanie usunięta z serwera DiskStation, zwolnienie 100 GB miejsca na woluminie może potrwać około 15–20 minut.

Aby wymusić montowanie jednostki iSCSI LUN (na poziomie bloku):

Jeżeli na jednostce iSCSI LUN (na poziomie bloku) nie ma pamięci podręcznej, jednostka iSCSI LUN (na poziomie bloku) nie zostanie zamontowana przez system podczas uruchamiania. Jeżeli pamięci podręcznej nie można odzyskać, dostęp do jednostki iSCSI LUN (na poziomie bloku) można uzyskać, klikając przycisk Wymuś montowanie.

Uwaga:

Ta funkcja jest dostępna tylko w niektórych modelach.

Aby edytować jednostkę iSCSI LUN:

  1. Wybierz jednostkę iSCSI LUN, która ma być edytowana.
  2. Kliknij Edytuj.
  3. Dokonaj edycji odpowiednich właściwości.
  4. Kliknij OK, aby zapisać ustawienia.

Uwaga:

Aby zapobiec utracie danych podczas edycji wielkości jednostki iSCSI LUN, można ją tylko powiększać.

Aby zrobić migawkę jednostki iSCSI LUN:

Migawkę jednostki iSCSI LUN to kopia jednostki iSCSI LUN odzwierciedlająca określony punkt w czasie i przeznaczona tylko do odczytu.

  1. Wybierz jednostkę iSCSI LUN, której migawki mają być utworzone.
  2. Kliknij przycisk Migawka i wybierz polecenie Zrób migawkę.
  3. Wprowadź opis migawki.
  4. Wybierz jeden z typów migawki stosownie do potrzeb:
    • Spójne z aplikacjami: Migawki tego typu można tworzyć wyłącznie przy użyciu programu Synology Snapshot Manager zainstalowanego na serwerze Windows lub VMware vCenter Server (w zależności od środowiska operacyjnego komunikującego się z systemem DSM). Po uruchomieniu procesu tworzenia migawki spójnej z aplikacjami w systemie DSM zostanie przesłane odpowiednie powiadomienie do aplikacji o tym, aby wszystkie dane z pamięci zostały zapisane w jednostce LUN, której ma być utworzona migawka. Zapewni to spójność danych.
    • Spójne przy awarii: Jeżeli narzędzie Synology Snapshot Manager nie jest zainstalowane na żadnym z klientów, można tworzyć wyłącznie migawki jednostek LUN zachowujące spójność przy awarii. Po przesłaniu sygnału uruchamiającego wykonanie takiego zrzutu obrazu aplikacje nie otrzymają powiadomienia o konieczności zapisania stanu pamięci w jednostce LUN. Może to spowodować niespójność danych.
  5. Kliknij przycisk OK.

Aby przywrócić jednostkę iSCSI LUN:

Wybierz migawkę z określonego punktu w czasie, aby przywrócić dane w jednostce iSCSI LUN.

  1. Wybierz jednostkę iSCSI LUN, która ma być przywrócona.
  2. Kliknij przycisk Migawkę i wybierz polecenie Snapshot Manager.
  3. Wybierz migawkę, który ma być użyty do przywrócenia danych, a następnie kliknij Przywróć.

Aby sklonować jednostkę iSCSI LUN lub migawkę:

Utwórz kopię jednostki iSCSI LUN do zapisu lub migawkę jednostki iSCSI LUN.

  • Z listy wybierz jednostkę iSCSI LUN i kliknij przycisk Klonuj.
  • W oknie narzędzia Snapshot Manager wybierz migawkę jednostki iSCSI LUN i kliknij przycisk Klonuj.

Uwaga:

  • W niektórych modelach należy przejść do Centrum pakietów i zainstalować narzędzie Migawka i replikacja, które zawiera funkcje umożliwiające przywracanie jednostek iSCSI LUN oraz wykonywanie i klonowanie migawek iSCSI LUN.
  • Jeżeli występuje co najmniej jedna połączona sesja iSCSI skojarzona z jednostką LUN i jest używana przez Windows Server lub VMware ESXi Server, zaleca się zainstalowanie dodatku Synology Snapshot Manager, aby uniknąć ewentualnych niespójności danych. W przeciwnym razie trzeba najpierw zatrzymać wszelkie aplikacje i wylogować te sesje z inicjatorów.
  • Funkcje zrzutów obrazu, przywracania i klonowania jednostek LUN są dostępne tylko w przypadku jednostek iSCSI LUN z zaawansowanymi funkcjami LUN.
  • Jeżeli zostanie zrobiona migawka lub utworzony klon jednostki, która ma stan Nienormalny/Uszkodzony, usuń ten migawkę lub klon i utwórz nowe.
  • Wszystkie zmiany jednostki LUN dokonane po utworzeniu migawki zostaną utracone po przywróceniu jednostki LUN do stanu migawki.
  • Jeżeli stan jednostki iSCSI LUN zmieni się na Niedostępna, będzie to oznaczać, że wystąpiły błędy związane z danymi podczas uzyskiwania dostępu do jednostki iSCSI LUN. Aby uniknąć poważnej utraty danych w systemie, ta jednostka iSCSI LUN będzie tymczasowo niedostępna. Skontaktuj się jak najszybciej z firmą Synology w celu uzyskania pomocy.

Słowniczek:

  • Thin Provisioning: Thin Provisioning to metoda optymalizacji wykorzystania pamięci przez alokowanie miejsca w pamięci dynamicznie i na żądanie. To ustawienie jest dostępne tylko w przypadku typu iSCSI LUN (zwykłe pliki).
  • VMware VAAI: technologia VMware vStorage APIs for Array Integration (VAAI) umożliwia przenoszenie zadań związanych ze standardowymi operacjami na wirtualnych macierzach pamięci masowej na urządzenie iSCSI w środowisku VMware vSphere i optymalizowanie wydajności pamięci masowej.
  • Windows ODX: nowa technologia transferu danych w systemach Windows Server 2012 i Windows 8 o nazwie Offload Data Transfer (ODX). ODX umożliwia zwiększenie wydajności transferu przez przeniesienie części obciążenia na macierz pamięci masowej.
    Jeżeli zarówno źródło danych, jak i miejsce docelowe znajdują się w jednostkach LUN na tym samym woluminie, klonowanie odbywa się przy użyciu technologii Synology Advanced LUN, która pozwala oszczędzić do 99,9% używanego miejsca, a jednocześnie przyspiesza klonowanie.
  • Mapowanie iSCSI Target: mapowanie iSCSI LUN do jednego lub większej liczby obiektów iSCSI Target.
  • Synology Snapshot Manager: Synology Snapshot Manager to dodatek umożliwiający tworzenie migawek spójnych z aplikacjami bezpośrednio z systemu DiskStation Manager (DSM).
    • W środowisku VMware należy zainstalować narzędzie Synology Snapshot Manager na serwerze Windows, na którym zainstalowano VMware vCenter Server. Po uruchomieniu procesu tworzenia migawki w systemie DSM zostanie przesłane odpowiednie powiadomienie do VMware vCenter Server o tym, aby wszystkie dane z pamięci zostały zapisane w jednostce LUN, której ma być utworzona migawka. Zapewni to spójność danych. Po ukończeniu tworzenia migawki na serwerze VMware vCenter Server zostanie przywrócone normalne działanie we/wy magazynu danych VMware. Synology Snapshot Manager for VMware vCenter Server obsługuje oprogramowanie vSphere 5.1 i nowsze wersje.
    • W środowisku Windows należy zainstalować narzędzie Synology Snapshot Manager na serwerze Windows. Po uruchomieniu procesu tworzenia migawki w systemie DSM narzędzie Synology Snapshot Manager wykorzysta technologię Microsoft Volume Shadow Copy Service (VSS) do tworzenia spójnych kopii danych z punktu w czasie. Synology Snapshot Manager for Windows obsługuje systemy Windows Server 2008 R2, 2012 i 2012 R2.
    • Synology Snapshot Manager można pobrać bezpłatnie z witryny www.synology.com.

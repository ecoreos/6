Ustawienia DSM

W części Panel sterowania > Sieć > Ustawienia DSM możesz zmienić numer portu, za pośrednictwem którego klienci uzyskują dostęp do DiskStation.

Aby zmienić domyślny numer portu:

Połączenia HTTP i HTTPS w systemie DSM są zawsze włączone. Domyślnym portem HTTP jest 5000, a domyślny port HTTPS to 5001.

  1. Wprowadź numer portu w polu HTTP lub HTTPS.
  2. Kliknij przycisk Zastosuj.

Aby przekierować połączenia HTTP do HTTPS:

Gdy klienci próbują uzyskać dostęp do serwera DiskStation przez port HTTP, możesz przekierować ich do portu HTTPS.

  1. Wybierz opcję Automatycznie przekieruj połączenia HTTP na HTTPS (nie dotyczy Web Station i Photo Station).
  2. Kliknij przycisk Zastosuj.

Porty używane do połączeń HTTPS:

  • DSM: Domyślnym numerem portu jest 5001. Aby uzyskać bezpieczny dostęp do DSM, należy wprowadzić następujący adres: https://server-hostname:5001/
  • Web Station i Photo Station: Domyślnym numerem portu jest 443. Aby uzyskać bezpieczny dostęp do Web Station i Photo Station, należy wprowadzić następujący adres: https://server-hostname/

Aby odnaleźć w sieci serwer DiskStation:

Serwer DiskStation można odnaleźć przy użyciu funkcji wykrywania sieci Windows lub wykrywania usługi Bonjour.

  1. Aby odnaleźć w sieci serwer DiskStation, wybierz usługę:
    • Włącz odnajdowanie sieci systemu Windows
    • Włącz usługę wykrywania Bonjour
  2. Kliknij przycisk Zastosuj.

Domena

W polu Domena można określić nazwę niestandardowej domeny wykorzystywanej w celu uzyskania dostępu do DiskStation.

Aby włączyć domenę niestandardową:

  1. Zaznacz opcję Włącz domenę niestandardową.
  2. Określ nazwę domeny stosowanej dla DiskStation.
  3. Ponadto możesz zaznaczyć opcję Włącz HSTS, aby wymusić w tej domenie połączenia szyfrowane przy użyciu protokołu HTTPS.
  4. Kliknij przycisk Zastosuj.

Uwaga:

  • Każda nazwa domeny może zostać użyta tylko w stosunku do jednej odrębnej aplikacji na serwerze DiskStation.
  • Jeśli nazwa domeny jest krótsza niż 15 znaków i jest zgodna z konwencjami nazewniczymi NetBIOS, system zarejestruje domenę i nada jej nazwę automatycznie (np. MyDSM). Użytkownicy systemu Windows mogą uzyskiwać dostęp do pulpitu DSM w tej samej lokalnej dzięki podobnemu adresowi URL: http://MyDSM lub https://MyDSM
  • Gdy uzyskujesz dostęp do aplikacji przy użyciu adresów URL, które korzystają z nazw domen, połączenia będą realizowane za pośrednictwem jednego ze standardowych portów: HTTP (80) lub HTTPS (443).

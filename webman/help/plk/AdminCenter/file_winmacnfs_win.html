Usługa plików systemu Windows

Usługa plików systemu Windows umożliwia klientom z systemami Windows, Mac OS i Linux z obsługą protokołów SMB/CIFS dostęp do danych przechowywanych na serwerze DiskStation. Aby przeglądać pliki na serwerze DiskStation za pośrednictwem programu Eksplorator Windows lub montować foldery współdzielone jako dyski sieciowe, włącz tę opcję na stronie Panel sterowania > Usługi plików > Win/Mac/NFS.

Aby włączyć obsługę plików Windows:

  1. Zaznacz pole wyboru Włącz obsługę plików Windows.
  2. Kliknij przycisk Zastosuj.

Uwaga:

Po włączeniu usługi plików systemu Windows można zamontować w pełni indeksowane foldery w komputerze Mac za pośrednictwem SMB i wyszukiwać zawartość plików w zamontowanych folderach przy użyciu opcji Spotlight w aplikacji Finder.

Grupa robocza

Ta opcja umożliwia przypisanie systemu jako członka określonej grupy roboczej do celów zarządzania siecią.

Uwaga:

  • Nazwa grupy roboczej może zawierać od 1 do 15 znaków. Nazwa nie może zawierać następujących znaków: [ ] ; : ” < > * + = \ / | ? ,
  • Jeżeli używany jest komputer Mac z systemem OS X 10.9, a DiskStation dołączył do domeny Windows, można się zalogować, dodając localhost\ przed nazwą użytkownika lokalnego DiskStation, aby uzyskać dostęp do plików przechowywanych w urządzeniu DiskStation za pośrednictwem usługi plików Windows.

Nie zezwalaj na dostęp do poprzednich wersji

W programie Eksplorator Windows użytkownicy mogą uzyskiwać dostęp do migawek folderów lub plików, klikając we właściwościach folderu współdzielonego kartę Poprzednie wersje, co pozwala na przywracanie lub kopiowanie danych bez udziału administratora. Jeżeli opcja Nie zezwalaj na dostęp do poprzednich wersji zostanie zaznaczona, na karcie [Poprzednie wersje] nie będzie żadnych informacji.

Uwaga:

  • Użytkownicy mogą uzyskać dostęp do wcześniejszej wersji tylko wtedy, gdy w momencie tworzenia migawki odpowiadającej wcześniejszej wersji ich konta użytkowników były utworzone i miały uprawnienia do odczytu folderu współdzielonego, którego migawka była tworzona.

Włączanie rejestrowania transferu w logu

Zaznaczenie tej opcji spowoduje włączenie rejestrowania operacji dostępu do plików, w tym tworzenia, usuwania, odczytu, zapisu, przenoszenia i zmiany nazw plików lub folderów za pośrednictwem protokołu CIFS/SMB, znanego także jako protokół Windows File Transfer.

Ustawienia zaawansowane

Kliknij przycisk Ustawienia zaawansowane, aby wyświetlić poniższe opcje.

  • Serwer WINS: Microsoft Windows Internet Name Service (WINS) to nazwa usługi odwzorowania nazw NetBIOS na adresy IP. Użytkownicy systemu Windows znajdą system w sieciach TCP/IP znacznie łatwiej, jeżeli zostanie on skonfigurowany tak, aby rejestrować się na serwerze WINS.
  • Maks. protokół usługi plików systemu Windows: w zależności od konfiguracji sieci można określić najwyższy poziom protokołu, który będzie obsługiwany przez urządzenie DiskStation.
    • Włącz SMB 2: protokół SMB (Server Message Block) 2 jest obsługiwany od wersji Windows Vista i jest on udoskonaloną wersją SMB. SMB 2 umożliwia łączenie wielu działań SMB w jedno żądanie w celu zmniejszenia liczby pakietów sieciowych i poprawy wydajności.
    • Włącz SMB 2 z Large MTU: Maksymalna jednostka transmisji (MTU — Maximum Transmission Unit) to największa jednostka danych, którą można przesłać przy użyciu protokołu telekomunikacyjnego. Aby umożliwić wykorzystanie szybszych interfejsów sieciowych, takich jak Ethernet 1 Gb lub 10 Gb, w funkcji Large MTU zwiększono MTU do 1 megabajta (MB). Włączenie funkcji Large MTU zwiększa szybkość i wydajność transferów dużych plików, zmniejszając jednocześnie liczbę pakietów do przetworzenia.
    • Włącz SMB 3: protokół SMB 3 jest obsługiwany od wersji Windows 8 i Windows Server 2012. Jest to rozszerzona wersja protokołu SMB 2. Protokół SMB 3 obsługuje transmisję plików szyfrowanych algorytmem AES, co pozwala poprawić bezpieczeństwo transferu plików między komputerami.

    Uwaga:

    • Protokół SMB 2 jest obsługiwany w systemach Windows Vista, Windows Server 2008 R2, Windows 7 i nowszych.
    • Funkcja Large MTU jest obsługiwana w systemach Windows Server 2008 R2, Windows 7 i nowszych.
    • W celu uzyskania efektywnej obsługi dużych MTU na komputerze z systemem Windows należy zmodyfikować ustawienia rejestru komputera. Więcej informacji można znaleźć w materiałach pomocy technicznej firmy Microsoft (takich jak ten artykuł).
  • Tryb szyfrowania transferu: po włączeniu protokołu SMB 3 w usłudze plików systemu Windows zostanie włączone szyfrowanie transferu, aby zwiększyć bezpieczeństwo transmisji plików.
    • Wyłącz: wyłączenie szyfrowania transferu.
    • Auto: włączenie szyfrowania transferu na klientach obsługujących tę funkcję.
    • Wymuś: wymuszenie szyfrowania transferu. Po wybraniu tej opcji klienci nieobsługujący szyfrowania transferu nie będą mogli korzystać z usługi plików systemu Windows.

    • Uwaga:

    • Funkcja szyfrowania transferu jest obsługiwana przez system Windows 8, Windows Server 2012 i nowsze wersje.
    • W przypadku włączenia szyfrowania transferu logowanie anonimowe w usłudze plików systemu Windows nie będzie obsługiwane.
    • Po włączeniu szyfrowania należy pozostawić opcję Wyłącz Opportunistic Locking niezaznaczoną, aby uniknąć przekraczania limitów czasu oczekiwania aplikacji.
  • Wyłącz Opportunistic Locking: Zaznaczenie tej opcji wyłączy Opportunistic Locking CIFS, zwiększając wydajność, kiedy aplikacje bazy danych uzyskują dostęp do plików bazy w folderze współdzielonym urządzenia DiskStation przez sieć.
  • Włącz Local Master Browser: kiedy w tej samej podsieci istnieje wiele komputerów z systemem Windows, w wyniku negocjacji jeden z nich zostanie wybrany jako Local Master Browser (główna przeglądarka lokalna). Komputer pełniący funkcję Local Master Browser przechowuje listy pozostałych komputerów z podsieci oraz ich udostępnionych zasobów i udostępnia im te listy. Ta opcja umożliwia DiskStation objęcie roli Local Master Browser (nadrzędnej przeglądarki lokalnej).

    • Uwaga:

    Włączenie tej funkcji spowoduje wyłączenie funkcji hibernacji dysku twardego i aktywowanie konta użytkownika guest (gość) niezabezpieczonego hasłem.

  • Włącz moduł DirSort VFS: sortowanie nazw plików i wyświetlanie ich na liście w kolejności alfabetycznej. Włączenie tej opcji obciąża system.
  • Odrzuć pliki: ukrywanie plików według określonych kryteriów. Kryteria mogą obejmować symbole wieloznaczne (*), a większą liczbę kryteriów należy rozdzielić ukośnikami (/). Na przykład: /abc*/*.txt/*.conf/. Włączenie tej opcji obciąża system.
  • Zezwalaj na łącza symboliczne w folderach współdzielonych: zezwolenie na wskazywanie plików lub folderów docelowych przez łącza symboliczne w tym samym folderze współdzielonym. Łącza symboliczne można tworzyć przy użyciu terminala systemu DSM.
  • Zezwalaj na łącza symboliczne między folderami współdzielonymi: zezwolenie na wskazywanie plików i folderów przez łącza symboliczne w różnych folderach współdzielonych. Łącza symboliczne można tworzyć przy użyciu terminala systemu DSM. Włączenie tej opcji może umożliwić użytkownikom bez uprawnień dostępu wyświetlanie zawartości folderów współdzielonych, do których kierują łącza symboliczne, oraz uzyskiwanie do nich dostępu.
  • Włącz moduł MSDFS VFS: zezwolenie na wskazywanie przez łącza symboliczne plików lub folderów znajdujących się na innych serwerach SMB. W przypadku starszych wersji systemu Windows, takich jak Windows XP, przed włączeniem tej opcji należy dołączyć klienta i DiskStation do domeny Windows. Więcej informacji o tworzeniu łączy MSDFS można uzyskać tutaj. Pamiętaj, że po wyłączeniu tej opcji należy wyczyścić pamięć podręczną urządzenia klienckiego. W przeciwnym razie wydajność może ulec pogorszeniu.
  • Wyłącz wiele połączeń z tego samego adresu IP: kiedy ta opcja jest włączona, DiskStation zezwala na tylko jedno połączenie z każdego adresu IP w przypadku klientów używających usługi plików systemu Windows. Jeżeli wiele klientów z tego samego adresu IP używa usługi plików systemu Windows, DiskStation utrzyma najnowsze połączenie i odrzuci wszystkie starsze. W środowisku NAT klient może używać jednego adresu IP. W takim przypadku włączenie tej opcji może wpływać negatywnie na normalne działanie.
  • Gromadź logi debugowania: przechowywanie szczegółowych dzienników do celów debugowania. Włączenie tej opcji obciąża system.
  • Zastosuj domyślne uprawnienia UNIX: Włączenie tej opcji powoduje zastosowanie domyślnych uprawnień systemu UNIX podczas przesyłania lub tworzenia plików i folderów. Uprawnienia systemu UNIX mają wartość 744 w przypadku plików i 755 w przypadku folderów. Po wyłączeniu tej opcji uprawnienia systemu UNIX mają wartość 777 w przypadku plików i folderów.

    • Uwaga:

    • Włączenie tej opcji może spowodować problemy z niespójnością uprawnień między różnymi protokołami. Aby uniknąć niespójności, nie zaleca się włączania tej opcji.

    Uwaga:

    • W przypadku folderów współdzielonych z obsługą listy kontroli dostępu systemu Windows (ACL) (wszystkie udziały z wyłączeniem udziału photo i udziałów na zewnętrznych dyskach) należy uruchomić polecenie chmod w kliencie Linux lub FTP, aby zmienić rodzaje uprawnień dotyczące folderów i plików z lista kontroli dostępu systemu Windows (ACL) na UNIX.
    • Włączenie tej opcji może spowodować problemy z niespójnością uprawnień między różnymi protokołami. Aby uniknąć niespójności, nie zaleca się włączania tej opcji.
  • Nie rezerwuj przestrzeni na dysku podczas tworzenia plików: Włącz tę opcję, aby zapobiec rezerwowaniu przestrzeni na dysku podczas tworzenia plików na serwerze DiskStation.
  • Wyczyść pamięć podręczną SMB: kliknięcie tego przycisku pozwala usunąć następujące pliki pamięci podręcznej SMB do celów rozwiązywania problemów:
    • /var/cache/samba/netsamlogon_cache.tdb
    • /var/lib/samba/winbindd_cache.tdb
    • /var/lib/samba/share_info.tdb
    • /var/lib/samba/registry.tdb
    • /run/samba/connections.tdb
    • /run/samba/gencacche.tdb
    • /run/samba/gencache_notran.tdb

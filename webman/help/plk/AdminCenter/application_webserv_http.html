Aplikacje www

Korzystając z wbudowanej funkcji Web Station oraz obsługi PHP, można łatwo udostępniać witrynę internetową na serwerze DiskStation i tworzyć dynamiczne witryny do użytku służbowego lub w ramach hobby. W Centrum pakietów można także znaleźć wiele różnych pakietów innych producentów, m.in. do zarządzania treścią, zarządzania relacjami z klientami (CRM) czy handlu elektronicznego. W celu przechowywania plików witryny internetowej zostanie utworzony domyślny folder współdzielony web.

Aby włączyć aplikację Web Station:

  1. Wybierz Panel sterowania > Usługi WWW.
  2. Zaznacz pole wyboru Włącz Web Station.
  3. Kliknij przycisk Zastosuj.

Uwaga:

  • Aby osiągnąć wyższą elastyczność i lepsze bezpieczeństwo, program Web Station do realizacji zadań używa grupy http. Z tego powodu w celu kontroli nad uprawnieniami dostępu do stron należy zmienić uprawnienia dostępu grupy http do odpowiednich folderów i plików. Jeżeli na przykład plik sample.htm ma być dostępny w folderze współdzielonym web dostępnym z programu Web Station, należy sprawdzić, czy grupa http ma odpowiednie uprawnienia odczytu i zapisu do folderu współdzielonego web oraz pliku sample.htm.
  • Plik strony głównej powinien nazywać się index.htm, index.html lub index.php.
  • Kiedy użyjesz funkcji PHP mail(), system automatycznie pobierze i wykorzysta informacje serwera SMTP ze strony Powiadomienia. Sprawdź, czy zostały włączone powiadomienia pocztą e-mail i wprowadzone zostały poprawne informacje serwera SMTP.
  • Prawdziwa ścieżka folderu współdzielonego web to /volume1/web. Niektóre aplikacje wymagają od użytkownika podania ścieżki w pliku konfiguracji.
  • Nazwy blog, photo, photosrc i webdefault są zastrzeżonymi nazwami folderów. Nazwy te nie mogą być używane przez Web Station. Nie twórz folderów o takich nazwach w folderze współdzielonym web.
  • Możesz dostosować stronę błędu, tworząc plik missing.html, missing.htm lub missing.php. Najpierw włącz Web Station. Następnie umieść plik missing.html, missing.htm lub missing.php w folderze współdzielonym web na serwerze. Kiedy użytkownik przejdzie na nieistniejącą stronę, zobaczy dostosowaną stronę błędu.
  • Maksymalna liczba aktywnych połączeń HTTP: 256
  • Maksymalna liczba hostów wirtualnych: 30

Udostępnianie witryn internetowych na hoście wirtualnym

Przy użyciu funkcji Wirtualny host można udostępniać wiele witryn internetowych na jednym serwerze. Na początek musisz zarejestrować kilka nazw domen i skonfigurować hosting wirtualny. W tym celu wykonaj poniższe czynności.

Aby włączyć funkcję Wirtualny host:

  1. Kliknij Wirtualny host.
  2. Kliknij przycisk Utwórz.
  3. Wprowadź nazwę podfolderu w folderze współdzielonym web, który będzie przechowywał pliki witryny internetowej.
  4. Wprowadź nazwę hosta witryny internetowej.
  5. Wprowadź port witryny internetowej.
  6. Kliknij przycisk OK.

Włączanie osobistej witryny internetowej

Funkcja osobistej witryny internetowej umożliwia użytkownikom lokalnym, użytkownikom domeny i użytkownikom LDAP tworzenie ich własnych witryn internetowych.

W przypadku użytkowników lokalnych

Każdy użytkownik lokalny będzie miał unikatowy adres www IP-serwera/~nazwaużytkownika lub nazwa-serwera/~nazwaużytkownika. Na przykład:

  1. 192.168.1.1/~marek
  2. SynologyNAS/~marek

W przypadku użytkowników domeny

Każdy użytkownik domeny będzie miał unikatowy adres witryny internetowej IP-serwera/~nazwadomeny%5cnazwaużytkownika lub nazwa-serwera/~nazwadomeny%5cnazwaużytkownika. Na przykład:

  1. 192.168.1.1/~domenatestowa%5cmarek
  2. SynologyNAS/~domenatestowa%5cmarek

W przypadku użytkowników LDAP

Każdy użytkownik LDAP będzie miał unikatowy adres witryny internetowej IP-serwera/~nazwaużytkownika@nazwadomeny lub nazwa-serwera/~nazwaużytkownika@nazwadomeny. Na przykład:

  1. 192.168.1.1/~marek@domenatestowa
  2. SynologyNAS/~marek@domenatestowa

Uwaga:

Przed włączeniem tej funkcji należy włączyć funkcję użytkownik domowy. W przypadku użytkowników lokalnych należy przejść do pozycji Panel sterowania > Użytkownik > Zaawansowane. W przypadku użytkowników domeny należy przejść do pozycji Panel sterowania > Usługa katalogowa > Użytkownicy domeny > Użytkownik domowy. W przypadku użytkowników LDAP należy przejść do pozycji Panel sterowania > Usługa katalogowa > Użytkownicy LDAP > Użytkownik domowy.

Uwaga:

Przed włączeniem tej funkcji należy włączyć funkcję użytkownik domowy. W przypadku użytkowników lokalnych należy przejść do strony Panel sterowania > Użytkownik > Zaawansowane.

Aby utworzyć osobistą witrynę internetową:

  1. Jeżeli zostało zaznaczone pole wyboru Włącz osobistą stronę internetową, zostanie automatycznie utworzony folder www w katalogu „home” użytkownika.
  2. Użytkownik może przesyłać do folderu www pliki powiązane z witryną internetową (np. strony internetowe i pliki dźwiękowe).
  3. Użytkownik musi mieć odpowiednie uprawnienia dostępu do przesyłanych plików.

Aby dostosować stronę błędu:

  1. Zapisz stronę błędu jako missing.html, missing.htm lub missing.php.
  2. Prześlij plik do folderu www.

Kiedy użytkownik wejdzie na stronę nieistniejącą w witrynie internetowej lokalnego użytkownika, wyświetli się dostosowana strona błędu.

Obsługa i ograniczenia:

Lokalni użytkownicy mogą mieć dostęp do danych jedynie w następujących katalogach:

  • Ich własny katalog domowy
  • /var/tmp
  • /tmp
  • /volumeX/@tmp

Usługa HTTP

Oprócz domyślnych numerów portów 80 (w przypadku HTTP) lub 443 (w przypadku HTTPS) można dodać kolejne numery portów w celu uzyskiwania dostępu do aplikacji Photo Station i Web Station. Do celów dostępu przez HTTP i HTTPS można dodać numer portu z zakresu od 1 do 65535.

Aby dodać kolejny numer portu w celu uzyskiwania dostępu przez HTTP i HTTPS:

  1. Zaznacz pole wyboru Dodaj dodatkowy port HTTP lub Dodaj dodatkowy port HTTPS (przed dodaniem portu protokołu HTTPS opcja Włącz połączenia HTTP dla usług WWW musi być włączona).
  2. Wprowadź w polu żądany numer portu.

Uwaga:

  • Następujące porty należy pominąć: 20, 21, 22, 23, 25, 110, 137, 138, 139, 143, 199, 443, 445, 515, 543, 548, 587, 873, 993, 995, 3306, 3689, 5000, 5001, 5005, 5006, 5335, 5432, 6881, 8080, 7000, 7001, 8081, 9997, 9998, 9999, 50001, 50002, domyślne porty eMule: 4662 (TCP), 4672 (UDP) i domyślny zakres portów FTP: zakres może zależeć od modelu.

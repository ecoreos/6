Ustawienia sieci

Ta strona zawiera trzy karty, które wyświetlają ustawienia sieci klastra high-availability, serwera aktywnego i serwera pasywnego.

Karta Klaster HA

Podczas tworzenia klastra high-availability system generuje wspólny adres IP klastra high-availability w celu odbierania połączeń przychodzących. W razie potrzeby można utworzyć dodatkowe adresy IP klastra high-availability. Adresy IP włączonych interfejsów sieciowych na serwerze aktywnym są wyświetlane na tej karcie (nie jest natomiast wyświetlany interfejs sieciowy używany przez połączenie pulsu, ponieważ jest on używany do replikacji danych pomiędzy oboma serwerami). Ta karta przedstawia następujące informacje:

  • Interfejs sieciowy: Włączone interfejsy sieciowe na serwerze aktywnym.
  • Połączenie: Wskazuje, czy dany interfejs jest podłączony.
  • Adres IP: Podczas tworzenia klastra high-availability system generuje wspólny adres IP klastra high-availability w celu odbierania połączeń przychodzących. Używając tego adresu IP, można łączyć się z klastrem high-availability, aby zalogować się do systemu DiskStation Manager (DSM) niezależnie od tego, który serwer firmy Synology jest serwerem aktywnym. Zapobiega to konieczności łączenia się z różnymi adresami IP po przełączeniu usług.
  • Maska podsieci: Maska podsieci powiązana z danym adresem IP klastra high-availability.

Aby edytować ustawienia Adres IP i Maska podsieci:

  1. Wybierz adres IP i maskę podsieci, które chcesz edytować, a następnie kliknij Edytuj lub kliknij dwukrotnie pole, aby zacząć edycję.
  2. Zaznacz pola wyboru Włącz, aby włączyć lub wyłączyć poszczególne interfejsy. Można włączyć więcej niż jeden interfejs sieciowy klastra, ale nie można wyłączyć interfejsu sieciowego skonfigurowanego podczas tworzenia klastra.

Aby zmienić nazwę serwera klastra high-availability:

  1. Kliknij Zmień ustawienia sieci, aby wyświetlić nazwę klastra high-availability i zmienić jej ustawienie.
  2. Kliknij Zastosuj, aby zapisać ustawienie bieżącej nazwy klastra high-availability i jego adresu IP.
  3. Aby odrzucić bieżące zmiany, kliknij przycisk Anuluj.

Serwer aktywny i pasywny (wyświetlane jako nazwa serwera)

Te karty przedstawiają aktualny stan różnych interfejsów sieciowych na serwerze aktywnym lub pasywnym. Połączenie pulsu będzie zajmować jeden interfejs sieciowy na serwerze aktywnym i pasywnym. Na tych kartach są wyświetlane następujące informacje:

  • Interfejs sieciowy: Fizyczny interfejs sieciowy na serwerze.
  • Adres IP: Adres IP przypisany do danego interfejsu sieciowego.
  • Maska podsieci: Maska podsieci powiązana z danym adresem IP.
  • Opis: Przedstawia informacje o danym interfejsie sieciowym oraz dostępności przypisanego mu adresu IP. To pole przedstawia następujące możliwe stany:
    1. Dostępny oryginalny adres IP: Oryginalny adres IP przypisany do interfejsu na serwerze aktywnym jest dostępny dla danych i usług.
    2. Niedostępny oryginalny adres IP: Oryginalny adres IP przypisany do interfejsu na serwerze pasywnym jest niedostępny.
    3. Adres IP połączenia pulsu, niedostępny z intranetu: Oryginalny adres IP przypisany automatycznie do połączenia pulsu jest niedostępny dla danych i usług.

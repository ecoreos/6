DDNS

DDNS (Dynamic Domain Name Service) semplifica la connessione di DiskStation tramite Internet eseguendo la mappatura del nome host sul relativo indirizzo IP. Ad esempio, DDNS consente di accedere a DiskStation utilizzando un nome dominio (ad esempio www.john.synology.me), senza dover ricordare l'indirizzo IP (ad esempio 172.16.254.1).

Impostare il DDNS

È possibile puntare un nome host esistente sull'indirizzo IP di DiskStation, o registrarne uno nuovo fornito da Synology o altri fornitore del servizio DDNS. Consultare ciascun fornitore del servizio per informazioni relative alla registrazione di un nome host.

Per impostare il nome host DDNS:

  1. Fare clic sul pulsante Aggiungi.
  2. Viene visualizzata una finestra di dialogo che consente di modificare le seguenti impostazioni:
    • Fornitore di servizio: selezionare un fornitore del servizio. Per registrare un nome host gratuito fornito da Synology, scegliere Synology dal menu a discesa, fare clic su Registrati adesso, e inserire le informazioni necessarie.
    • Nome host: inserire il nome host DDNS registrato, ad esempio john.synology.me.
    • Nome utente: inserire il nome utente del fornitore DDNS.
    • Password: inserire la password del fornitore DDNS.
    • Indirizzo esterno: inserire l'indirizzo IP esterno di DiskStation che utilizzerà il nome host.
    • Heartbeat: abilitare questa opzione per ricevere gli avvisi relativi allo stato del nome host mappato (solo per i nomi host forniti da Synology).
  3. Fare clic su Test di connessione per vedere se le impostazioni sono corrette.
  4. Fare clic su OK per salvare e terminare.

Una volta completata l'impostazione, è possibile accedere a DiskStation tramite Internet inserendo il nome host DDNS nel browser web.

Requisiti e limitazioni

  • Il server del fornitore del servizio DDNS sono funzionanti.
  • DiskStation può collegarsi a Internet.
  • DDNS funziona solo in ambienti di rete dove non sono necessari server proxy.
  • È anche possibile inserire un nome host per ciascun fornitore DDNS.

Gestire i nomi host

Usare i seguenti pulsanti per gestire le voci del nome host esistente:

  • Elimina: rimuove la voce del nome host selezionato dall'elenco.
  • Modifica: cambia le impostazioni della voce del nome host selezionato.
  • Aggiorna adesso: aggiorna lo stato di tutte le voci del nome host che hanno il servizio DDNS abilitato.

Configurare di un provider DDNS personalizzato

È possibile aggiungere un provider DDNS personalizzato se il provide non appare ancora nell'elenco dei provider di servizi.

Per configurare un provider DDNS personalizzato:

  1. Cliccare su Personalizza.
  2. Viene visualizzata una finestra di dialogo che richiede di inserire le seguenti impostazioni:
    • Fornitore di servizio: inserire il nome del provider di servizi personalizzati.
    • URL query: inserire il nome del provider di servizi personalizzati.
  3. Fare clic su Salva per aggiungere il provider di servizi personalizzati all'elenco dei provider di servizi DDNS.

Requisiti e limitazioni

  • L'URL query deve corrispondere al protocollo specificato dal provider di servizi. L'URL è usato per aggiornare le informazioni IP sul server DDNS del provider.
  • Un'URL query in genere contiene i seguenti quattro parametri. Quando si inserire l'URL query, sostituire i seguenti parametri con la variabile corrispondente.
    • Nome utente: __USERNAME__
    • Password: __PASSWORD__
    • Nome host: __HOSTNAME__
    • Indirizzo esterno: __MYIP__
  • L'URL query non deve contenere spazi o punti e virgola.

Applicazioni Web

Con la funzione Web Station integrata e il supporto PHP, è possibile ospitare facilmente un sito Web su DiskStation e creare siti Web dinamici per uso aziendale o hobby. Inoltre, Centro pacchetti offre una varietà di pacchetti di terze parti come Gestione dei contenuti, Gestione delle relazioni con il cliente e sistema di e-Commerce. Una cartella condivisa predefinita chiamata web sarà creata per archiviare i file del proprio sito Web.

Per attivare Web Station:

  1. Andare su Pannello di controllo > Servizi Web.
  2. Selezionare Abilita Web Station.
  3. Fare clic su Applica.

Nota.

  • Per una migliore flessibilità e sicurezza, Web Station utilizza il gruppo utente http per eseguire le attività. Quindi, per controllare i permessi di accesso alle pagine Web, modificare i permessi di accesso del gruppo http per ogni cartella e file corrispondente. Ad esempio, se si vuole rendere il file sample.htm nella cartella condivisa Web accessibile via Web Station, è necessario assicurare che il gruppo http disponga dei corretti permessi in scrittura/lettura per la cartella condivisa Web nonché il file sample.htm.
  • Il file della pagina Home deve essere chiamato index.htm, index.html o index.php.
  • Quando si utilizza la funzione PHP mail(), il sistema consentirà di recuperare e utilizzare automaticamente le informazioni del server SMTP nella pagina Notifica. Assicurarsi di aver abilitato le notifiche e-mail e di aver inserito le informazioni corrette per il server SMTP.
  • Il percorso effettivo della cartella condivisa web è /volume1/web. Alcune applicazioni richiedono l'impostazione del percorso da parte dell'utente nel file di configurazione.
  • I nomi blog, photo, photosrc e webdefault sono riservati ai nomi delle cartelle. Questi nomi non possono essere usati dalla Web Station, non creare cartelle con questi nomi nella cartella condivisa web.
  • La pagina di errore può essere personalizzata creando un proprio file missing.html, missing.htm o missing.php. Attivare prima la Web Station. Quindi posizionare il file missing.html, missing.htm o missing.php nella cartella condivisa web sul server. Quando gli utenti tentano di accedere ad una pagina inesistente, vedranno questa pagina di errore personalizzata.
  • Numero massimo di connessioni HTTP attive: 256
  • Numero massimo di host virtuali: 30

Abilitare host virtuale per siti Web host

È possibile eseguire l'hosting di più siti Web su un singolo server utilizzando la funzione di host virtuale. Per iniziare, è necessario registrare una coppia di nomi dominio e configurare l'hosting virtuale seguenti i seguenti passaggi.

Per abilitare Host virtuale:

  1. Fare clic su Host Virtuale.
  2. Fare clic su Crea.
  3. Inserire un nome per la sotto-cartella della cartella condivisa web, che memorizzerà i file del sito Web.
  4. Inserire un nome host per il sito Web.
  5. Inserire una porta per il sito Web.
  6. Fare clic su OK.

Abilitare il sito Web personale

La funzione sito Web personale consente agli utenti locali, utenti di dominio e utenti LDAP di creare il proprio sito Web personale.

Per gli utenti locali

Ogni utente locale ha un indirizzo Web unico server-IP/~username o server-name/~username. Ad esempio:

  1. 192.168.1.1/~ken
  2. SynologyNAS/~ken

Per gli utenti di dominio

Ogni utente dominio ha un indirizzo Web unico server-IP/~domainname%5cusername o server-name/~domainname%5cusername. Ad esempio:

  1. 192.168.1.1/~testdomain%5cken
  2. SynologyNAS/~testdomain%5cken

Per gli utenti LDAP

Ogni utente LDAP ha un indirizzo Web unico server-IP/~username@domainname o server-name/~username@domainname. Ad esempio:

  1. 192.168.1.1/~ken@testdomain
  2. SynologyNAS/~ken@testdomain

Nota.

Prima di abilitare questa funzione, assicurarsi che homes utente siano abilitate. Per utenti locali, andare su Pannello di controllo > Utente > Avanzate; per utenti del dominio, andare su Pannello di controllo > Servizio directory > Utenti dominio > Home utente; per utenti LDAP, andare su Pannello di controllo > Servizio directory > Utenti LDAP > Home utente.

Nota.

Prima di abilitare questa funzione, assicurarsi che homes utente siano abilitate. Per utenti locali, andare su Pannello di controllo > Utente > Avanzate.

Per creare un sito Web personale:

  1. Se Abilita sito Web personale ha un segno di spunta, verrà automaticamente creata una cartella www nella directory principale.
  2. È ora possibile caricare i file relativi al sito Web (per es. pagine Web e i file audio) nella cartella www.
  3. Assicurarsi di disporre delle autorizzazioni di accesso adatte per i file caricati.

Per personalizzare la pagina di errore:

  1. Salvare la pagina di errore come missing.html, missing.htm o missing.php.
  2. Caricare il file nella cartella www.

Quando gli utenti si collegano a una pagina che non esiste all'interno del sito Web dell'utente locale, verrà visualizzata la pagina di errore personalizzata.

Compatibilità e limitazioni:

Gli utenti locali possono accedere ai dati come segue:

  • La loro sede
  • /var/tmp
  • /tmp
  • /volumeX/@tmp

Servizio HTTP

Oltre ai numeri di porta predefiniti 80 (per HTTP) o 443 (per HTTPS), numeri porta aggiuntivi possono essere aggiunti per accedere a Photo Station e Web Station. È possibile aggiungere un numero di porta supplementare da 1 a 65535 per HTTP o HTTPS.

Per aggiungere un numero di porta supplementare per l'accesso a HTTP o HTTPS:

  1. Spuntare Aggiungi porta HTTP aggiuntiva o Aggiungi porta HTTPS aggiuntiva (l'opzione Attiva connessione HTTPS per i servizi Web deve essere abilitata prima di aggiungere una porta HTTPS).
  2. Immettere il numero porta desiderato nel campo.

Nota.

  • Sono escluse le seguenti porte: 20, 21, 22, 23, 25, 110, 137, 138, 139, 143, 199, 443, 445, 515, 543, 548, 587, 873, 993, 995, 3306, 3689, 5000, 5001, 5005, 5006, 5335, 5432, 6881, 8080, 7000, 7001, 8081, 9997, 9998, 9999, 50001, 50002, porte predefinite di eMule: Intervallo porta predefinito 4662 (TCP), 4672 (UDP) e FTP: l'intervallo effettivo può variare in relazione al modello.

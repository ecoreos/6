Dominio

È possibile aggiungere DiskStation a un dominio Windows. Una volta aggiunto il dominio, gli utenti di dominio possono accedere alla DiskStation con i propri account e password di dominio, consentendo di accedere ai file e usare applicazioni DSM senza la necessità di ricordare un altro set di nomi utente e password.

Per unirsi a un dominio:

  1. Andare su Pannello di controllo > Servizio directory > Dominio.
  2. Spuntare la casella Accedi al dominio.
  3. Inserire il nome dominio e specificare il DNS Server nelle caselle appropriate.
  4. Se necessario, spuntare la casella Opzioni dominio avanzate. Per i dettagli, vedere di seguito.
  5. Fare clic su Applica.
  6. Nella finestra popup, inserire il nome utente e la password dell'amministratore sul server di dominio.
  7. Fare clic su OK.

DNS Server

Si consiglia di specificare un server DNS per aiutare DiskStation a cercare il dominio. Tuttavia, se il dominio non è registrato sul server DNS, la procedura di accesso al dominio potrebbe non andare a buon fine.

Tipo di server dominio

Questo campo mostra il tipo di dominio quando la DiskStation è collegata a un dominio. Il tipo di dominio sarà "Dominio AD".

Modalità gestione

Questa opzione determinerà come gestire i privilegi degli utenti e gruppi dominio.

  • Dominio fidato: è possibile gestire utenti e gruppi in un dominio congiunto nonché domini fidati. Utenti e gruppi possono essere filtrati con l'elenco a discesa del dominio.
  • Dominio singolo con OU: solo utenti e gruppi nel dominio congiunto visualizzeranno in questa modalità. Questa modalità consente di filtrare utenti dominio/gruppi dominio con l'elenco a discesa OU (Unità organizzativa).

Opzioni di dominio avanzate

Nella maggior parte dei casi, non è necessario riempire nessuna delle opzioni avanzate del dominio. Le opzioni avanzate del dominio sono necessarie solo per determinati ambienti di dominio.

  • DC IP/FQDN: Specificare un indirizzo IP DC (controller di dominio) o FQDN, e DiskStation cercherà di comunicare con esso. In presenza di più di un indirizzo IP o FQDN, inserire una virgola (,) tra ciascun indirizzo. Aggiungere un asterisco (*) dopo l'ultimo indirizzo IP DC o FQDN, quindi DiskStation proverà a comunicare con l'altro DC se tutti quelli specificati hanno esito negativo. Sarà necessario aggiungere una virgola tra l'asterisco e l'ultimo indirizzo IP o FQDN.
  • Nome del dominio NetBIOS: Indicare il nome del dominio NetBIOS.
  • Dominio FQDN (nome DNS): Indicare l'FQDN (nome DNS) del dominio.
  • Registrare l'interfaccia DNS: Quando si accede a un dominio, la NIC (Network Interface Card, scheda di interfaccia di rete) specificata qui sarà registrata con il server DNS. Si noti che se il nome host di DiskStation include un sottolineato (_), la registrazione fallisce in quanto i sottolineati (_) non possono essere usati con DNS.
  • Aggiorna elenco utenti/gruppo: specificare con quale frequenza DiskStation aggiorna automaticamente l'elenco utente/gruppo dominio. È possibile personalizzare il periodo di tempo per eseguire aggiornamenti quotidiani, settimanali o mensili. Inoltre, gli elenchi utente/gruppo dominio possono essere aggiornati manualmente andando alla scheda Utente di dominio e facendo clic su Aggiorna tutti i domini. Gli aggiornamenti automatici influiranno sulla sospensione del sistema.

Accesso al controller dominio di sola lettura

Per far accedere DiskStation a un controller dominio di sola lettura (RODC), è necessario specificare il "nome intero del computer" (FQDN) del proprio controller di sola lettura (RWDC) nel campo DC IP/FQDN. Il nome del computer intero deve essere nel seguente formato: NomeComputer.Dominio, ad es. "ad01.synology.com".

Nota:

  • se il proprio nome utente dominio include “%” o “$”, potrebbe non essere possibile accedere alla propria cartella principale. Richiedere un nuovo nome utente all'amministratore del dominio.
  • I gruppi di dominio Domain Admins e Enterprise Admins verranno aggiunti al gruppo locale administrators automaticamente. Quindi, gli utenti del dominio in questi gruppi dispongono dei privilegi di amministrazione su DiskStation, incluso l'accesso ai servizi di file come CIFS/FTP/AFP/WebDAV e altre applicazioni DSM.
  • È anche possibile configurare i privilegi di accesso degli utenti del dominio per le cartelle condivise su DiskStation. Vedere qui per maggiori informazioni.

Opzioni dominio

Questa sezione include le opzioni di dominio aggiuntive seguenti.

  • DiskStation eseguirà la sincronizzazione con un server NTP ogni volta che un utente dominio esegue l'accesso: abilitando questa opzione si forzerà la sincronizzazione dell'ora tra DiskStation e il server NTP.
  • Abilita accesso server: Se il client ha accesso client abilitato, deve essere anche abilitato l'accesso server su DiskStation, quindi gli utenti possono utilizzare i servizi di dominio normalmente.
  • Ottieni elenchi utenti/gruppo con modalità compatibile con NT4: abilitando questa opzione si consente al sistema di ottenere elenchi utente/gruppo usando la modalità NT4 RPC. Questa opzione può essere abilitata se alcuni elenchi utente/gruppo non possono essere ottenuti usando le impostazioni predefinite.
  • Disabilita permessi amministratore per Domain Admins ed Enterprise Admins: un utente con permessi amministratore ha il controllo completo della propria DiskStation e dei suoi file. Abilitando questa opzione è possibile limitare i permessi Domain Admins ed Enterprise Admins per essere gli stessi di un utente normale, ereditando quindi tutti i permessi cartella/file di un utente normale.
  • Enumera livelli gruppo compatto: specificare il numero di membri del gruppo dominio compatto che possono essere enumerati.

Invio registro

In caso di server syslog, è possibile seguire le istruzioni di seguito per configurare il proprio DiskStation come un client syslog e trasmettere i registri di sistema locali di DiskStation a un server syslog per monitoraggio e gestione centralizzati.

Per inviare registri a un syslog server:

  1. Andare su Log Center > Invio registro.
  2. Spuntare la casella Invia registri a un syslog server.
  3. Immettere un nome host o un indirizzo IP del server syslog in Server.
  4. Specificare un numero di porta per la ricezione di messaggi syslog in Porta.
  5. Selezionare UDP o TCP in Protocollo trasferimento. Deve essere corrispondente al protocollo utilizzato dal server syslog.
  6. Selezionare BSD (RFC 3164) o IETF (RFC 5424) in Formato registro per inviare i registri utilizzando il formato selezionato.

    7. Nota:

    • Per impostazione predefinita la porta è 514. Corrette regole di inoltro porta devono essere impostate per riassegnare un numero porta diverso al server syslog.
    • UDP (User Datagram Protocol) è un protocollo di connessione basato su messaggi più semplici e più rapido, ma manca di sicurezza e correzione degli errori. TCP (Transmission Control Protocol) offre una trasmissione garantita, in cui viene usato un controllo di flusso per determinare se tutti i pacchetti sono stati inviati con successo prima che di elaborarne di nuovi.
    • Connessione sicura (SSL) è solo disponibile quando si utilizza TCP.
    • Se si sceglie TCP e si intende assicurare la connessione syslog con SSL, è possibile mettere il segno di spunta sulla casella Abilita connessione sicura (SSL). Quindi spuntare Importa certificato e selezionare il certificato per trasferire i log in modo sicuro al server syslog.
  7. Fare clic su Invia registro di prova per confermare il server syslog è in grado di ricevere il registro con successo. Se il server syslog non riceve il registro di prova, controllare che le impostazioni siano corrette.
  8. Fare clic su Applica per salvare le impostazioni.

Filtraggio registri

Sulla scheda Filtro, è possibile specificare i tipi di registri inviati al server syslog, secondo categoria (ad es. sistema, connessione, backup) o gravità (ad es. preavviso, errore, informazioni).

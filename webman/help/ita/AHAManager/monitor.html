Monitor

In questa pagina sono riportate tre schede che visualizzano il monitoraggio dello stato di sistema, del servizio e dell'interfaccia di rete. Questa pagina consente di monitorare alcune destinazioni. Se una destinazione monitorata non funziona sul server attivo, il sistema effettuerà il failover del server passivo in standby (finché il server passivo sta funzionando normalmente). In questa pagina, sono visualizzate le schede seguenti:

Scheda sullo stato del sistema

Lo stato di sistema viene monitorato per impostazione predefinita e non può essere disattivato. Il failover verrà attivato quando si verificano degli errori di sistema nel server attivo. In questa scheda, sono visualizzate le sezioni seguenti:

  • Stato del server attivo: Failover in caso di guasto del server attivo nella fornitura di servizi. Ad esempio, entrambe le unità di del server attivo hanno esito negativo.
  • Connettività di archiviazione: Failover in caso di unità assente nel server attivo ma presente nel server passivo.

Scheda di servizio

Visualizza i servizi che possono essere monitorati e il relativo stato. Il failover verrà attivato quando i servizi sono anomali. In questa scheda, sono visualizzate le sezioni seguenti:

  • Servizio: Visualizza i servizi che possono essere monitorati.
  • Stato: Visualizza se i servizi stanno funzionando o meno.
  • Monitor: Spuntare la casella di controllo accanto a un servizio per avviare il monitoraggio. Il sistema effettua il failover se il servizio selezionato fallisce.

Per monitorare un servizio:

  1. Trovare il servizio che si desidera monitorare. Verificare che lo stato del servizio sia normale.
  2. Spuntare la casella di controllo accanto.
  3. Fare clic su Applica per salvare le impostazioni.

Nota:

I servizi che possono essere monitorati sono i seguenti: Windows File Service, iSCSI Target, FTP, Mac File Service, NFS Service.

Scheda dell'Interfaccia di rete

Visualizza tutte le interfacce di rete logiche che è possibile monitorare. Il failover verrà attivato se la connessione di rete è anomala. (Consultare le note seguenti) In questa scheda, sono visualizzate le sezioni seguenti:

  • Interfaccia di rete: Visualizza le interfacce che possono essere monitorate.
  • Monitor: Spuntare la casella di controllo accanto a un'interfaccia per avviare il monitoraggio.

Nota:

Se la casella di controllo Monitor è spuntata, il sistema effettuerà il failover del server passivo quando la connessione di rete del server attivo fallisce o non funziona correttamente, e la connessione di rete del server passivo funziona ancora normalmente.

  • Il failover sarà attivato solo se il server passivo ha più connessioni funzionanti. Ad esempio, se il server attivo ha tre connessioni e una non funziona, il sistema attiverà un failover solo se il server passivo ha tre o più connessioni funzionanti.
  • Connessioni combinate con Link Aggregation contano come una connessione. Ad esempio, se sono combinate due connessioni, e una di esse non funziona, ma l'altra sta ancora funzionando, la connessione sarà considerata ancora funzionante.

Per monitorare un'interfaccia:

  1. Trovare l'interfaccia da monitorare. Verificare che i cavi siano stati collegati correttamente alle interfacce su entrambi i server attivo e passivo.
  2. Spuntare la casella di controllo accanto.
  3. Fare clic su Applica per salvare le impostazioni.

Panoramica

Questa pagina visualizza le informazioni di base relative al cluster con high-availability e ai server membri. Fornisce, inoltre, gli strumenti per eseguire la manutenzione dei cluster con high-availability e server membri.

Stato di cluster con high-availability

Questo campo visualizza lo stato globale del cluster high-availability, avvisando in caso di errori. Di seguito una tabella delle possibile condizioni visualizzate nel campo:

Stato Descrizione
Integrità La condizione globale del cluster high-availability è buona.
Caricamento Il sistema sta caricando lo stato del cluster. Non è possibile eseguire il Passaggio fino a quando non è stato completato il caricamento dello stato. Consultare Passaggio.
Configurazione Il sistema è in corso di configurazione. Non è possibile eseguire il Passaggio fino a quando non è stata completata la configurazione.
Avvertenza Lo stato del cluster HA è anomalo,ma i servizi sono ancora funzionanti. In questa situazione il Passaggio potrebbe non essere eseguito.
Pericolo Entrambi i server attivo e passivo non funzionano correttamente oppure l'archiviazione è danneggiata. Il cluster high-availability non funziona normalmente. Verificare lo stato di tutti gli spazi di archiviazione (ad esempio RAID Group, Gruppi disco, volumi e iSCSI LUN). In questa situazione il Passaggio non può essere eseguito

Messaggi di avviso

La tabella di seguito visualizza i messaggi di avviso che possono essere visualizzati quando si verifica un errore del sistema. Quando viene visualizzato più di un messaggio di avviso, è possibile scorrere i messaggi utilizzando i pulsanti avanti/indietro.

Avvertenza/Errore Descrizione
Non è presente alcun server passivo nel cluster con high-availability Le informazioni del server passivo non sono disponibili. Accertarsi che il server passivo sia connesso correttamente. In questo caso, il passaggio non sarà disponibile.
Impossibile rilevare il server passivo Questo errore si potrebbe verificare perché il server passivo è offline o i server attivo e passivo non possono comunicare tramite la connessione Heartbeat. In questo caso, il passaggio non sarà disponibile.
Impossibile eseguire il failover. Un failover è stato attivato nel server attivo, ma avuto esito negativo. In questo caso, il server attivo non tenterà un altro failover. Per eliminare l'errore, seguire i passi indicati:
  1. Provare a ordinare i problemi nel cluster high-availability.
  2. Riavvia il server passivo.
  3. Eseguire uno switchover facendo clic su Gestisci > Passaggio.
Errore destinazione monitorata Si è verificato un errore in una o più destinazioni monitorate, incluso lo stato di sistema, servizio o interfaccia di rete.
Si sono verificati degli errori nella connessione tra le unità di espansione e il server attivo/passivo Le unità di espansione non sono connesse correttamente al server attivo o passivo. Verificare che la relativa connessione al server attivo sia corretta e configurare la stessa connessione sul server passivo. In questo caso, il passaggio non sarà disponibile.
Un volume è stato danneggiato Uno spazio di archiviazione (ad esempio RAID Group, Gruppo disco, volume, iSCSI LUN e cache SSD) è danneggiato. Controllare la pagina Stato archiviazione.
Un volume si è bloccato Uno spazio di archiviazione (ad esempio RAID Group, Gruppo disco, volume, LUN iSCSI e cache SSD) è danneggiato. Controllare la pagina Stato archiviazione. Il failover verrà eseguito automaticamente se il server passivo prevede il riconoscimento delle unità a cui il server attivo non è riuscito ad accedere. In caso contrario, il passaggio non sarà disponibile.
Connessione Heartbeat persa Questo errore si potrebbe verificare a causa di un errore di connessione Heartbeat. In questo caso, il passaggio non sarà disponibile.
Malfunzionamento dell'hardware Questo errore potrebbe verificarsi a causa di un errore dell'hardware (es., unità di alimentazione o ventola del sistema) nel server attivo o passivo.
Dimensioni RAM fisica differenti Questo errore potrebbe verificarsi a causa della mancata corrispondenza delle dimensioni della RAM fisica nei server attivo e passivo. In questo caso, il passaggio non sarà disponibile.
Nessun IP statico è stato assegnato a questo cluster con high-availability Questo errore potrebbe verificarsi a causa della mancanza di un IP statico per il cluster con high-availability. In questo caso, il passaggio non sarà disponibile.
Non è presente alcun IP statico nell'interfaccia monitorata Questo errore potrebbe verificarsi a causa della mancanza di un IP statico per l'interfaccia monitorata. In questo caso, il passaggio non sarà disponibile.
Recupero server passivo Il recupero del server passivo è in corso. In questo caso, il passaggio non sarà disponibile.
Aggiornamento server passivo Questo sistema sta aggiornando il server passivo. In questo caso, il passaggio non sarà disponibile.

Pulsante "Gestisci" Cluster High-Availability

Questo pulsante si trova al di sotto delle informazioni sul cluster con high-availability. Comprende le seguenti operazioni:

  • Passaggio: inizializza il passaggio dal server attivo al server passivo. Durante questo processo, il server attivo smette di fornire i servizi e diventa il nuovo server passivo. Quindi, il server passivo vecchio diventa il server attivo nuovo e inizia a fornire tutti i servizi del cluster high-availability. Questa opzione non è disponibile se il server passivo è in stato errore o non è selezionato.
  • Aggiorna DSM: Apre la pagina Aggiornamento DSM del Pannello di controllo. Da qui, entrambi i server possono essere aggiornati.
  • Spegni cluster con high-availability: spegne entrambi i server attivi e passivi.
  • Spegni server attivo: spegne il server attivo. I servizi saranno passati se lo stato del server passivo è integro.
  • Spegni server passivo: spegne il server passivo.
  • Riavvio server attivo: riavvia il server attivo. I servizi saranno passati se lo stato del server passivo è integro.
  • Riavvia server passivo: riavvia il server passivo.
  • Recupera server sconosciuto: recupera il server passivo se il server passivo viene sostituito o se viene ancora riconosciuto come sconosciuto.
  • Avviso acustico sistema disattivato: disattiva il suono bip del server passivo e di quello passivo quando si verifica un errore. Il sistema emetterà il suono bip nel server attivo o nel server passivo se, successivamente, si verifica un altro errore.

Nota:

  • se si arresta il server passivo o se non è presente alcun server passivo, i servizi non possono essere passati se si verificano errori sul server attivo.

Schema del server e informazioni

Nello schema del server viene mostrato lo stato del server. La posizione corrisponde al lato dei moduli a cui è collegato il server. Per maggiori dettagli, posizionarvi sopra il puntatore del mouse.

Comprende le seguenti situazioni:

  • Non è presente alcun server passivo nel cluster con high-availability: visualizza un fotogramma vuoto.
  • Impossibile rilevare il server passivo: visualizza un'immagine offuscata del server.
  • Punto esclamativo: l'hardware potrebbe non funzionare correttamente oppure le informazioni sul server non sono disponibili a causa di un errore della connessione Heartbeat.

Nota:

  • se viene visualizzato "sconosciuto" nell'altro server, il server non appartiene al cluster con high-availability. Fare clic su Gestisci e selezionare Recupera server sconosciuto per correggere l'errore.

Le informazioni sul server comprendono le seguenti voci. Mostra N/D se non viene rilevato alcun server passivo.

  • Numero di serie: il numero di serie del server Synology.
  • Stato ventola: lo stato delle ventole di raffreddamento del server. In caso di malfunzionamento di una ventola viene visualizzato un avviso. In questo caso, contattare il distributore locale per assistenza.
  • Temperatura: la temperatura del server Synology.
  • Stato alimentazione: indica se le alimentazioni del server funzionano tutte normalmente. Se il server è dotato di due sorgenti di alimentazione ridondanti, in caso di malfunzionamento di una delle due, viene visualizzato un messaggio di avviso.
  • Memoria fisica: dimensioni della RAM fisica del server Synology. Il passaggio non può essere eseguito se i due server non dispongono delle stesse dimensioni di RAM fisica.

Stato di connettività tra i server

in questa sezione sono contenute le immagini tra i server attivo e passivo. Comprende le seguenti situazioni:

  • Nessuna immagine: indica che non è stato rilevato alcun server passivo.
  • Fermo immagine: indica un'errata connessione Heartbeat.
  • Punto esclamativo: indica una comunicazione anomala tra i due server o un'errata connessione Heartbeat.

Stato dell'Interfaccia di rete

In questa sezione sono contenute le immagini tra i server e la rete dati. Comprende le seguenti situazioni:

  • Nessun punto esclamativo: indica che non è stato rilevato alcun errore di rete.
  • Punto esclamativo: indica un errore che si verifica nell'interfaccia di rete monitorata.

Schema di archiviazione condivisa e Informazioni

Nello schema di archiviazione condivisa viene mostrato lo stato di connettività tra i server e l'archiviazione condivisa. Per le informazioni di base, usare il puntatore del mouse.

Comprende le seguenti situazioni:

  • Nessun punto esclamativo: indica che non è stato rilevato alcun errore.
  • Punto esclamativo: indica uno stato di connettività anomalo delle unità di espansione.

Le informazioni di base sull'archiviazione condivisa comprendono le seguenti voci. Per informazioni dettagliate, andare su Gestore archiviazione.

  • Stato ventola: visualizza lo stato delle ventole delle unità di espansione. In caso di malfunzionamento di una ventola viene visualizzato un avviso. In questo caso, contattare il distributore locale per assistenza.
  • Stato alimentazione: indica se le alimentazioni del server funzionano tutte normalmente. In caso di malfunzionamento di una di queste, viene visualizzato un avviso.

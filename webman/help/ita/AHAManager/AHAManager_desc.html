High Availability Manager

High Availability è una soluzione layout server progettata per ridurre le interruzioni dei servizi causate da malfunzionamenti del server. Synology High Availability utilizza due server per creare un "cluster high-availability" nel quale un server assume il ruolo di "server attivo" mentre l'altro server agisce come "server passivo" in standby. Per implementare e creare un cluster high-availability, sono necessari due server Synology compatibili.

Un cluster high-availability richiede l'uso dei seguenti componenti e concetti:

  • Concetto: L'implementazione di Synology High Availability richiede due server Synology per creare un cluster high-availability. Un server agisce da "server attivo" mentre l'altro agisce da "server passivo". Entrambi i server attivo e passivo nel cluster high-availability sono collegati alle stesse unità di espansione e condividono la stessa archiviazione. Per garantire l'integrità dei dati, solo il server attivo potrà gestire il carico di lavoro. In caso di malfunzionamento del server attivo, il server passivo sarà pronto ad acquisire tutti i servizi. Pertanto, il cluster high-availability continuerà a offrire normalmente i servizi, riducendo il periodo di inattività. Per ulteriori informazioni sul passaggio, fare riferimento a Passaggio.
  • Server attivo: il server attivo gestisce l'offerta di tutti i servizi. In caso di errori sul server attivo, l'utente o il sistema può inizializzare il passaggio al server passivo. In questo caso, la fornitura dei servizi sarà trasferita al server passivo, e al contempo il server attivo diventa server passivo.
  • Server passivo: in condizioni normali, il server passivo rimane in modalità standby. Nel caso in cui il server attivo non riesca a fornire i servizi, l'utente o il sistema può inizializzare il passaggio al server passivo.
  • Connessione heartbeat: i server attivo e passivo del cluster high-availability sono collegati tramite una connessione di rete dedicata privata che prende il nome di "Connessione heartbeat." Se la connessione tra i server è assente per un dato periodo di tempo, il sistema determina la necessità di eseguire il passaggio.
  • Archiviazione condivisa del cluster high-availability: Le unità nelle unità di espansione collegate da entrambi i server attivo e passivo funzionano nello stesso tempo come l'archiviazione condivisa nel cluster high-availability. Solo il server attivo dispone delle autorizzazioni di scrittura per l'archiviazione.
  • Nome del server del cluster high-availability e impostazione di rete: Il nome del server e le impostazioni di rete del cluster high-availability saranno determinati dalle impostazioni corrispondenti in DSM. Nella rete è possibile connettersi utilizzando questo nome server, e i servizi saranno forniti dal server attivo. Per maggiori informazioni sulle impostazioni di rete, vedere qui.

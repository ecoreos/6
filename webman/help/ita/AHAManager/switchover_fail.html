Passaggio

Per garantire prestazioni continue e ridurre il tempo di interruzione del servizio, i servizi possono essere passati dal server attivo del cluster high-availability al server passivo in standby. Questa funziona prende il nome di "Passaggio", o in molti casi "Failover". In questa sezione sono riportate le informazioni di base di questi due concetti.

Passaggio

Il passaggio consiste nel trasferire i servizi dal server attivo al server passivo in standby di un cluster high-availability. In condizioni normali, il server attivo gestisce tutti i servizi. Tuttavia, nel caso il server attivo non funzioni correttamente, l'utente può inizializzare il passaggio e il trasferimento di tutti i servizi al server passivo, presumendo che il server passivo funzioni correttamente. In questo caso, il server attivo assume il ruolo di server passivo e viceversa.

Per inizializzare manualmente il passaggio:

  1. Andare alla pagina Panoramica.
  2. Fare clic su Gestisci > Passaggio.

Nota:

Prima del passaggio manuale, si consiglia di configurare un IP statico e accedere a DSM con questo IP statico, al fine di evitare la perdita della pagina DSN causata da un cambio di IP dopo il passaggio.

Nota:

quando viene inizializzato un passaggio, prima il server attivo assume il ruolo di server passivo, quindi il server passivo diventa il server attivo. Pertanto, per un breve periodo di tempo entrambi i server saranno passivi e i servizi potrebbero non essere forniti in modo normale. Il tempo necessario per completare il passaggio varia in relazione al numero e alla dimensione dei volumi o iSCSI LUN (Livello blocco), o al numero e al carico totale di servizi e pacchetti sul cluster high-availability.

Failover

Quando si verificano alcuni errori, il sistema può inizializzare automaticamente un passaggio di servizi in modo da ridurre al minimo i tempi di inattività del servizio. Questo evento prende il nome di "failover". Il sistema può inizializzare un failover nelle seguenti situazioni, presumendo che il server passivo funzioni normalmente:

  • Interruzione di corrente:
    • il server attivo è stato arrestato o riavviato.
    • Entrambe le unità di alimentazione nel server attivo hanno avuto esito negativo.
    • Nessuna alimentazione.
  • Errore unità: il numero di percorsi unità nel server attivo è inferiore a quello nel server passivo.
  • Errore servizio: si è verificato un errore nel servizio monitorato.
  • Errore di connessione: si è verificato un errore di connessione nell'interfaccia monitorata.

Errore nel passaggio

Il passaggio o failover potrebbe non funzionare nelle seguenti situazioni:

  • Connessione heartbeat anomala: lo stato della connessione del server attivo-passivo non è normale.
  • Dimensioni memoria non corrispondenti: I server attivo e passivo non hanno dimensioni RAM identiche.
  • Interruzione di corrente:
    • il server passivo è stato arrestato o riavviato.
    • Entrambe le unità di alimentazione nel server passivo hanno avuto esito negativo.
    • Nessuna alimentazione.
  • Errore di connessione: si è verificato un errore di connessione nell'interfaccia del server passivo.
  • Errore servizio: si verificano errori nei servizi monitorati sul server passivo.
  • Errore di connessione dell'unità di espansione: La connessione delle unità di espansione nei due server non è dello stesso ordine.
  • Errore unità: il numero di percorsi unità nel server attivo è passivo è inferiore a quello nel server attivo.
  • Aggiornamento DSM: quando si installa un aggiornamento DSM, tutti i servizi saranno interrotti e riprenderanno dopo che l'installazione dell'aggiornamento DSM è completata.

Nel caso venga inizializzato manualmente un passaggio ma i servizi non possono essere passati al server passivo, il sistema tenta di ripassare i servizi al server attivo.

UPS (gruppo di continuità)

I cluster high-availability possono essere collegati a unità UPS (gruppi di continuità di rete) in modo da ridurre il rischio di interruzioni di corrente. In caso di interruzione, i server riceveranno alimentazione di backup dalle unità UPS a cui sono collegati. Quando l'unità UPS collegata al server attivo inizia a funzione a bassa potenza, il server passivo si arresta e il server attivo passa alla modalità di sicurezza. Dopo la ripresa di potenza, il server attivo viene riavviato e tenta di attivare l'altro server trasmettendo un messaggio Wake on LAN.

  • Si consiglia di collegare entrambi i server e le unità di espansione nella stessa UPS come switch di rete a cui sono collegati.

Nota:

DiskStation supporta soltanto la connessione a dispositivi UPD di rete. Per maggiori informazioni, consultare la guida in Pannello di controllo > Hardware > UPS.

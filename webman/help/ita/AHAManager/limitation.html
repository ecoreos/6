Limitazioni e requisiti di sistema

L'implementazione di Synology High Availability richiede due server Synology identici da utilizzare come server attivo e passivo. Prima di creare un cluster high-availability con due server, il sistema verifica le seguenti limitazioni hardware e software per garantire la creazione di un cluster high-availability.

Requisiti di sistema

Consultare le seguenti linee guida relative a software e hardware:

  • Server Synology: I server attivi e passivi devono essere modelli identici.
  • Unità di espansione Entrambi i server attivo e passivo devono connettersi correttamente alle stesse unità di espansione.
  • Numero e tipo di unità: Il tipo di unità deve essere soltanto SAS. Prima di installare DSM, occorre installare almeno 3 unità nelle unità di espansione.
  • Dispositivo esterno USB: È supportata soltanto l'archiviazione USB. Altri dispositivi esterni USB (es., UPS, dongle Wi-Fi, Microfono, Stampante, dongle Bluetooth e dongle DVB) non sono supportati. L'archiviazione USB può essere usata soltanto nel server attivo e non può essere passata al server passivo.

Ambiente di rete

Consultare le seguenti linee guida relative alle impostazioni di rete quando si tenta di formare un cluster high-availability.

  • Impostazioni di rete: La connessione Heartbeat tra i due server deve essere configurata correttamente. Entrambi i server devono appartenere alla stessa subnet, che è necessaria per configurare gli indirizzi IP statici dopo l'installazione.
  • Numero di porte LAN: entrambi i server devono avere lo stesso numero di porte LAN. Se uno dei modelli DiskStation è fornito di altre interfacce di scheda di rete, l'altro dovrà essere fornito delle stesse interfacce di scheda di rete.
  • Limitazioni connessione: la rete Wi-Fi non è supportata dal cluster high-availability.

Nota:

  • il server NTP verrà abilitato automaticamente nel server attivo.

Limitazioni pacchetto e software

Consultare le limitazioni relative a pacchetto e software riportate di seguito:

  • Il cluster high-availability non supporta i seguenti pacchetti e funzioni:
    • Server DVBLink
    • La funzione per riavviare/spegnere il cluster high-availability in CMS
    • Time Backup
  • Il cluster high-availability con archiviazione condivisa non può essere configurato come o usato in combinazione con un cluster high-availability basato su replica.

Unità di espansione specifiche

Entrambi i server attivo e passivo devono essere connessi a specifiche unità di espansione. In ciascuna unità di espansione, sono presenti due moduli.

Per configurare/aggiungere le unità di espansione:

  • Un modulo contiene due porte: la porta di ingresso, ubicata a sinistro, e la porta di uscita, ubicata a destra. Collegare un cavo SAS dalla porta di uscita di un modulo alla porta di ingresso del modulo dell'unità di espansione successiva.
  • Le unità di espansione devono essere collegate in modo seriale con i moduli sui lati corrispondenti per ciascun server.
  • Entrambi i server attivo e passivo devono essere connessi alle unità di espansione condivise nello stesso ordine.

Nota:

Per informazioni dettagliate sull'installazione dell'unità di espansione, consultare la guida di installazione rapida per i server e le unità di espansione.

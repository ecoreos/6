Arresto e sostituzione del server

Le istruzioni di seguito illustrano la procedura di spegnimento di modifica di un server in un cluster high-availability.

Disattivare il cluster high-availability utilizzando High Availability Manager:

Quando si desidera spegnere un cluster high-availability per spostare i server in una nuova posizione fisica o eseguire la manutenzione, procedere come segue.

  1. Accedere al DSM.
  2. Andare su High Availability Manager.
  3. Fare clic sul pulsante indicante Gestisci.
  4. Selezionare Spegni cluster high-availability.
  5. Viene visualizzato un messaggio di conferma. Fare clic su OK per confermare.

Per spegnere manualmente un cluster high-availability:

Il cluster high-availability può anche essere spento manualmente utilizzando i pulsanti di accensione su ciascun server.

  1. Tenere premuto il pulsante di alimentazione sul server passivo fino a quando emette un avviso acustico, quindi attendere che il server si arresti completamente.
  2. Tenere premuto il pulsante di alimentazione sul server attivo fino a quando emette un avviso acustico, quindi attendere che il server si arresti completamente.

Nota:

  • Accendere i server:
    Per attivare nuovamente i server, sia il server attivo che il server passivo devono essere accesi contemporaneamente (o entro cinque minuti uno dall'altro). Se si accende solo uno dei server, questo tenterà di alimentare l'altro server trasmettendo un messaggio Wake on LAN.
  • Arresto anomalo:
    Nel cluster high-availability, viene abilitata l'opzione di riavvio dopo un'interruzione di potenza. Se i server si spengono a causa di una interruzione dell'alimentazione, si riavvieranno automaticamente una volta ripristinata l'alimentazione.

Per sostituire un server in un cluster high-availability:

Per rimuovere un server e configurare un altro server, seguire le istruzioni riportate di seguito.

  1. Andare alla pagina Panoramica.
  2. Fare clic sul pulsante Gestisci.
  3. Selezionare Spegni server attivo o Spegni server passivo per arrestare il server da sostituire.
  4. Controllare le connessioni del nuovo server. Dopo avere attivato il nuovo server, questo verrà contrassegnato come "sconosciuto" in High Availaibility Manager.
  5. Andare di nuovo alla pagina Panoramica.
  6. Fare clic sul pulsante Gestisci.
  7. Selezionare Recupera server sconosciuto in modo che il "server sconosciuto" venga aggiunto al cluster high-availability e riconosciuto come server passivo.

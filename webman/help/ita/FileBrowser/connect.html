Connessione remota

In File Station, è possibile connettersi ai servizi cloud pubblici (Box, Dropbox, Google Drive e Microsoft OneDrive) e ai file server tramite svariati e differenti protocolli (FTP, STFP, WebDAV e WebDAV HTTPS).

Per collegarsi a un servizio cloud:

  1. Andare su Strumenti > Connessione remota > Configurazione di connessione.
  2. Scegliere un servizio cloud facendo clic sull'icona e fare clic su Avanti.
  3. Nella finestra a comparsa, seguire le istruzioni della procedura guida per l'autenticazione dell'account.
  4. Al termine della configurazione, il servizio cloud di destinazione apparirà nel riquadro a sinistra.

Limitazioni specifiche del cloud:

  • Box:
    • un catalogo condiviso con altri collaboratori verrà fornito del suffisso del nome utente del proprietario.
    • Se i cataloghi condivisi con altri collaboratori utilizzano un nome identico (un nome di catalogo con il suffisso del nome utente del proprietario), verrà visualizzato solo un catalogo di questo tipo.
    • I file con queste estensioni possono essere aperti solo tramite Box: .gdoc, .gsheet e .boxnote.
    • I nomi file non fanno distinzione tra maiuscole e minuscole.
    • Per le corrette impostazioni del firewall, consultare questo articolo.
  • Dropbox:
    • un catalogo Dropbox non può essere visualizzato se contiene oltre 25.000 file.
    • Un catalogo Dropbox non può essere rinominato se contiene oltre 10.000 file.
    • I file che superano 10 GB non possono essere caricati.
    • Un'attività di caricamento avrà esito negativo se il file di destinazione impiega più di 24 ore per il caricamento.
    • I nomi file non fanno distinzione tra maiuscole e minuscole.
    • Per le corrette impostazioni del firewall, consultare questo articolo.
  • Google Drive:
    • i file/cartelle con un percorso superiore a 2048 non saranno visualizzati.
    • I file/cartelle con un nome file superiore a 255 non saranno visualizzati.
    • I file/cataloghi con nome identico verranno contrassegnati con i singoli orari di creazione.
    • Se si apre un file specifico di Google (es., Google Docs e Google Slides) tramite File Station, verrà visualizzato un URL che consentirà di accedere al file di destinazione.
    • Le cartelle My Drive e Starred visualizzate su File Station corrispondono rispettivamente alle cartelle "My Drive" e "Con asterisco" in Google Drive.
    • Per le corrette impostazioni del firewall, consultare questo articolo.
  • Microsoft OneDrive:
    • I file di notebook OneNot non verranno visualizzati.
    • I file che superano 10 GB non possono essere caricati.
    • I file (es., .png, .jpg, .docx, .pdf) potrebbero visualizzare dimensioni di file maggiori (le dimensioni originali e le dimensioni delle miniature). Questa condizione non influirà sulle dimensioni dei file originali quando vengono scaricati.
    • I nomi file non fanno distinzione tra maiuscole e minuscole.

Per conoscere più limitazioni, si consiglia di visitare il sito Web ufficiale del servizio cloud interessato.

Per collegarsi a un file server:

  1. Andare su Strumenti > Connessione remota > Configurazione di connessione.
  2. Scegliere un protocollo server facendo clic sull'icona e fare clic su Avanti.
  3. Per completare la configurazione, fornire le informazioni seguenti:
    • Nome host o IP: inserire il nome host o l'IP del file server di destinazione.
    • Porta: Inserire il numero di porta del file server di destinazione.
    • Utente e Password: inserire le informazioni di accesso se il file server di destinazione potrebbe richiedere un account/password.
    • Codice pagina: Selezionare la codifica di carattere del file server di destinazione per visualizzare correttamente i nomi dei file.
    • Nome profilo: immettere un nome univoco per identificare facilmente questo file server.
    • Usa connessione WebDAV HTTPS: quest'opzione è disponibile solo quando si configura un protocollo WebDAV e quando entrambi il client e il file server supportano WebDAV HTTPS.
    • Consenti una sola connessione per utente: quest'opzione è disponibile solo quando si configura un protocollo FTP o SFTP. Spuntare quest'opzione per ottimizzare la connettività quando il server presenta delle limitazioni sul numero di connessioni.
  4. Fare clic su Applica per salvare le impostazioni. Il file server di destinazione apparirà nel riquadro di sinistra.

Per modificare un servizio cloud o un profilo file server esistente:

  1. Andare su Strumenti > Connessione remota > Elenco connessioni.
  2. Fare clic sul servizio cloud o il file server di destinazione per le seguenti gestioni:
    • Modifica: fare clic su Modifica, modificare la configurazione del profilo corrente e fare clic su OK.
    • Disconnessione in corso: fare clic su Disconnetti per terminare la connessione.
    • Riconnessione in corso: fare clic su Connetti per recuperare la connessione. In alternativa, fare clic sulla destinazione nel riquadro di sinistra di File Station per la riconnessione.
    • Eliminazione in corso: fare clic su Elimina e confermare per uscire dal profilo. In alternativa, fare clic sulla destinazione nel riquadro di sinistra di File Station e fare clic su Elimina per eliminare.

Nota:

  • Ogni utente può connettersi contemporaneamente a un massimo di 100 servizi cloud e file server.
  • Verificare di selezionare il codice del server corretto per evitare errori imprevisti. Unicode è consigliato se non si conosce il codice pagina del server.
  • Alcune funzioni di File Station (es. Comprimi su, Estrai, Anteprima, ecc.) non possono essere applicate ai file memorizzati su servizi cloud o file server.
  • Le singole velocità di trasferimento per ciascun servizio cloud o file server connesso sono soggette a impostazioni dei limiti di velocità utente.
  • È possibile eseguire un massimo di tre trasferimenti di file contemporaneamente in tutti i servizi cloud e file server connessi.
  • Per configurare le impostazioni del proxy di WebDAV, WebDAV HTTPS e del servizio cloud, andare su Pannello di controllo > Rete > Generale > Proxy. Se le impostazioni proxy sono state modificate durante la connessione a un servizio cloud o un file server, disconnettersi e riconnettersi.
  • La modifica delle informazioni di servizi cloud o file server connessi non influirà sullo stato della connessione.
  • Per rilasciare la risorsa di connessione, un servizio cloud/server file si disconnetterà automaticamente se i file contenuti restano intatti per oltre 30 minuti. Fare clic sul servizio cloud/server file per ripristinare la connessione.
  • Il sistema non è in grado di visualizzare file/cartelle che adottano i seguenti formati di denominazione:
    • Iniziano con ._ o .SYNOPPSDB
    • Includono /
    • Denominato esattamente come .    ..    '    "    .DS_Store    @eaDir    @sharebin    @tmp    .SynologyWorkingDirectory
  • La connessione remota è supportata solo nella modalità privilegi elevati. Per usare questa funzione, andare nel pacchetto Docker del DSM host, selezionare il Docker DSM per cui abilitare le cartelle condivise crittografate, fare clic su Modifica e spuntare le caselle accanto Esegui contenitore utuilizzando i privilegi elevati.

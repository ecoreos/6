iSCSI Target*

iSCSI è un tipo di SAN (storage area networking) che fornisce accesso all'archiviazione dei dati consolidata a livello di blocco. Semplifica il trasferimento dei dati tramite Intranet e la gestione dell'archiviazione su lunghe distanze. Eseguendo la mappatura iSCSI LUN su iSCSI Target, i server client possono accedere agli spazi di archiviazione tramite i relativi iniziatori iSCSI allo stesso modo dei dischi locali. In relazione al modello, il prodotto Synology supporta la creazione di più iSCSI Target su cui possono essere mappati gli iSCSI LUN.

Nota: il numero di iSCSI Target supportati varia in relazione ai prodotti Synology. Per le specifiche dettagliate dei prodotti, consultare il sito web Synology.

Per creare una iSCSI Target:

  1. Fare clic sul pulsante Crea.
  2. Inserire le informazioni iSCSI Target, incluso Nome, IQN e se abilitare l'Autenticazione CHAP.
  3. Fare clic su Avanti.
  4. Impostare la mappatura iSCSI LUN. Selezionare Crea nuovo iSCSI LUN o Mappare iSCSI LUN esistenti.
  5. Fare clic su Avanti.
  6. Fare clic su Indietro per modificare o su Applica per confermare le impostazioni.

Spiegazione dei termini

  • IQN: IQN (Nome qualificato iSCSI) è un gruppo di nomi unici per un iSCSI Target. Esso non deve essere in conflitto con iSCSI Target IQN su altri server. Il nome qualificato iSCSI (IQN) non deve essere inferiore a 128 caratteri e può includere lettere (senza maiuscole/minuscole), numeri, due punti, trattini e punti. Il formato del nome suggerito è iqn.[data].[dominio]:[identificatore del dispositivo], ad esempio: iqn.2000-04.com.synology:storage.diskarray.1.
  • Autenticazione CHAP: Selezionando CHAP, il server sarà in grado di verificare l'identità del cliente. Il nome utente può essere composto da 1 a 12 lettere e numeri, e la password può essere composta da 12 a 16 lettere e numeri.
  • Autenticazione CHAP mutua: In CHAP mutua, client e server devono eseguire l'autenticazione reciproca. Qualsiasi password CHAP utilizzata per l'autenticazione del client non deve essere configurata per l'autenticazione di alcun server, e qualsiasi password CHAP utilizzata per l'autenticazione del server non deve essere configurata per l'autenticazione di alcun client.

Per eliminare un iSCSI Target:

È possibile rimuovere un iSCSI Target e ogni iSCSI LUN mappata. Quando si rimuove un iSCSI LUN, tutti i dati su iSCSI LUN saranno cancellati.

  1. Selezionare l'iSCSI Target da eliminare.
  2. Fare clic su Rimuovi.
  3. Per rimuovere le iSCSI LUN, selezionare Elimina anche le iSCSI LUN mappate e selezionare quelle da rimuovere.
  4. Fare clic su OK.

Per modificare un iSCSI Target:

  1. Selezionare l'iSCSI Target che si desidera modificare.
  2. Fare clic su Modifica.
  3. Modificare le impostazioni secondo le preferenze. Fare clic su OK per salvare le modifiche.

Spiegazione dei termini

  • Somma di controllo CRC: Abilitare la Somma di controllo CRC per controllare la presenza di errori durante la trasmissione dei dati.
  • Sessioni multiple: consente sessioni multiple da uno o più iniziatori connessi alla stessa iSCSI Target allo stesso momento. Per proteggere i file da corruzione o perdita dei dati, si consiglia di utilizzare un file di sistema condiviso o un cluster come VMware Virtual Machine File System (VMFS) e Oracle Cluster File System (OCFS).
  • Mascheramento: il mascheramento consente l'accesso esclusivo a una iSCSI Target da iniziatori iSCSI specificati IQN.

Per abilitare o disabilitare una iSCSI Target:

  1. Selezionare l'iSCSI Target da abilitare o disabilitare.
  2. Fare clic su Abilita o Disabilita.

Per registrare gli iSCSI Target con un server iSNS

iSNS (Internet Storage Name Service) offre un meccanismo di gestione centralizzato. È possibile registrare tutti gli iSCSI Target con un server iSNS.

  1. Fare clic sul pulsante iSNS quindi spuntare la casella Attiva iSNS.
  2. Inserire l'indirizzo IP del server iSNS. Fare clic su OK per applicare le impostazioni.

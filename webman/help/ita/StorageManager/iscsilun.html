iSCSI LUN*

iSCSI è un servizio di archiviazione di rete che semplifica il trasferimento dei dati su reti locali o lunghe distanze. La mappatura degli iSCSI LUN su iSCSI Target consente ai client di accedere agli spazi di archiviazione come i dischi locali. Un iSCSI LUN (logical unit number) può essere mappato su uno o più iSCSI Target per offrire i servizi di archiviazione iSCSI. Andare su Gestione archiviazione > iSCSI LUN per gestire iSCSI LUN.

Nota:

Il numero di iSCSI LUN supportati varia in relazione ai modelli di DiskStation. Per specifiche dettagliate sui prodotti, consultare www.synology.com.

Attualmente, DiskStation supporta i seguenti tipi di iSCSI LUN:

  • iSCSI LUN (file normali): iSCSI LUN è creato destinando una porzione dello spazio sul volume al suo utilizzo. Se necessario, è possibile espanderne le dimensioni. Questo tipo di iSCSI LUN offre una gestione della capacità dinamica più flessibile con Thin Provisioning, il supporto di accelerazione hardware VMware VAAI, il supporto Windows ODX, le istantanee LUN e il clone LUN.
  • iSCSI LUN (livello blocco) - LUN singolo su RAID: iSCSI LUN viene creato utilizzando tutto lo spazio di un volume. È possibile ripararlo o espanderlo oppure cambiare il tipo di RAID, come quando utilizzo di un volume. Fare riferimento alla pagina Volume per ulteriori informazioni.
  • iSCSI LUN (livello blocco) - LUN multiplo su RAID: iSCSI LUN viene creato designando una quantità specifica di spazio di un Gruppo disco. In seguito sarà possibile espandere l'iSCSI LUN, in caso di spazio disponibile sul Gruppo disco. Fare riferimento alla pagina Gruppo Disco per ulteriori informazioni.
  • iSCSI LUN (livello blocco): questo tipo di iSCSI LUN viene creato su un RAID Group e offre una gestione delle capacità flessibile e dinamica, con prestazioni di accesso ottimizzate. È possibile espandere questo tipo di iSCSI LUN in seguito utilizzando lo spazio non allocato sul RAID Group. Fare riferimento alla pagina RAID Group per ulteriori informazioni.

Nota:

Gli iSCSI LUN con funzioni LUN avanzate sono disponibili solo su file system ext4/Btrfs.

Per creare un iSCSI LUN:

  1. Fare clic su Crea.
  2. Selezionare il tipo di iSCSI LUN che si desidera creare.
  3. Seguire la procedura guidata per completare il processo.

Nota:

le funzioni LUN avanzate possono essere abilitate solamente durante la creazione iniziale di iSCSI LUN. Questa impostazione non può essere modificata in un secondo momento.

L'unità di allocazione LUN avanzata può essere ottimizzata per alcune funzionalità per il server dell'applicazione, come Microsoft Exchange o SQL Server oppure per i server di virtualizzazione e le tecnologie di accelerazione di archiviazione, come Microsoft Windows ODS o VMware VAAI, in questo modo:

  • Impostare le dimensioni dell'unità di allocazione durante la creazione del primo LUN avanzato di un volume.
  • Impostare le dimensioni dell'unità di allocazione in Configura prima di creare il primo LUN avanzato di un volume.

Si consiglia di specificare la dimensione dell'unità di allocazione per l'applicazione quando si utilizza il metodo di collegamento diretto dal server dell'applicazione o RDM (pass-through) per lavorare con iSCSI LUN avanzati su DiskStation. Per collegare la tecnologia Synology Advanced LUN per la clonazione di Windows ODS, si consiglia di formattare il volume NTFS con dimensioni dell'unità di allocazione uguali (o superiori) a uno dei DSM specificati.

Nota:

l'unità di allocazione Advanced LUN di ciascun volume può essere impostata solo una volta.

Per rimuovere un iSCSI LUN:

quando si rimuove un iSCSI LUN, tutti i dati su iSCSI LUN saranno cancellati. Verrà persa anche la connessione tra iSCSI LUN e qualsiasi iSCSI Target mappato.

  1. Selezionare la iSCSI LUN che si desidera eliminare.
  2. Fare clic su Rimuovi e seguire la procedura guidata per completare il processo.

Nota:

l'eliminazione dei VAAI LUN su DiskStation non libera subito lo spazio sul disco. I dati vengono spostati in una cartella riservata al sistema con il nome @EP_trash e eliminati uno alla volta. Ad esempio, se un VAAI LUN da 100 GB viene eliminato dalla DiskStation, potrebbero essere necessari circa 15-20 minuti per liberare 100 GB di spazio dal volume del disco.

Forzare il montaggio di iSCSI LUN (Livello blocco):

Se la cache su un iSCSI LUN (livello blocco) è mancante, il sistema non monterà l'iSCSI LUN (livello blocco) al riavvio. Se la cache non è rilevabile, e si vuole accedere all'iSCSI LUN (livello blocco), è possibile fare clic su Forza montaggio.

Nota:

questa funzione è disponibile solo su modelli specifici.

Per modificare un iSCSI LUN:

  1. Selezionare l'iSCSI LUN che si desidera modificare.
  2. Fare clic su Modifica.
  3. Modificare le proprietà.
  4. Fare clic su OK per salvare le impostazioni.

Nota:

per evitare la perdita di dati durante la modifica della capacità iSCSI LUN, è solo possibile ampliarla rispetto alla sua dimensione corrente.

Per scattare un'istantanea iSCSI LUN:

un'istantanea di un iSCSI LUN è una copia in sola lettura in un punto nel tempo di un iSCSI LUN.

  1. Selezionare l'iSCSI LUN del quale scattare le istantanee.
  2. Fare clic su Istantanea e scegliere Scatta istantanea.
  3. Immettere la descrizione per questa istantanea.
  4. Scegliere uno dei seguenti tipi di istantanea in base alle proprie necessità:
    • Coerente con l'applicazione: questo tipo di istantanea può essere acquisita solo quando è installato Synology Snapshot Manager sul server Windows o VMware vCenter Server in uso, a seconda dell'ambiente operativo per comunicare con DSM. Quando istantanee consistenti all'applicazione sono effettuate su DSM, le applicazioni saranno notificate e passeranno tutti i dati dalla memoria al LUN su cui l'istantanea è effettuata per garantire la consistenza di dati.
    • Consistente al blocco: se nessun Synology Snapshot Manager è installato su uno dei client in uso, è possibile acquisire solo istantanee consistenti al blocco delle LUN. Quando sono attivate, le applicazioni non saranno notificate per il flusso dei dati dalla memoria alla LUN, quindi non sarà possibile garantire alcuna coerenza dei dati.
  5. Fare clic su OK.

Per ripristinare un iSCSI LUN:

selezionare un'istantanea di un punto nel tempo per ripristinare i dati su un iSCSI LUN.

  1. Selezionare l'iSCSI LUN che si desidera ripristinare.
  2. Fare clic su Istantanea e scegliere Gestione istantanee.
  3. Selezionare un'istantanea da ripristinare e fare clic su Ripristina.

Per clonare un iSCSI LUN o un'istantanea:

creare una copia scrivibile di un iSCSI LUN o un'istantanea di un iSCSI LUN.

  • Selezionare un iSCSI LUN dall'elenco e fare clic su Clona.
  • Selezionare un'istantanea di iSCSI LUN da Gestione istantanea e fare clic su Clona.

Nota:

  • in alcuni modelli, sarà necessario andare su Centro pacchetti e installare Istantanea e Replica per le funzioni che includono il ripristino di iSCSI LUN, l'acquisizione e la clonazione di istantanee iSCSI LUN.
  • Se è presente almeno una sessione iSCSI collegata associata al LUN ed è utilizzata da Windows Server o VMware ESXi Server, è possibile installare il plugin Synology Snapshot Manager per evitare possibili inconsistenze dei dati o si deve arrestare qualsiasi applicazione e uscire da queste sessioni dagli iniziati prima di procedere.
  • Le funzioni di ripristino e clonazione dell'istantanea LUN sono disponibili solo su iSCSI LUN con funzioni LUN avanzate.
  • Se viene scattata un'istantanea o creato un clone, e lo stato è su Anomalo/Bloccato, eliminare l'istantanea/clone e crearne uno nuovo.
  • Tutte le modifiche apportate a un LUN dopo la creazione dell'istantanea andranno perse quando il LUN viene ripristinato dall'istantanea.
  • Se lo stato di iSCSI LUN passa su Non disponibile, si sono verificati alcuni errori relativi ai dati durante l'accesso a iSCSI LUN. Per evitare la perdita di dati sul sistema, l'iSCSI LUN sarà temporaneamente inaccessibile. Contattare Synology immediatamente per richiedere assistenza.

Glossario:

  • Thin Provisioning: Thin Provisioning è un metodo di ottimizzazione dell'utilizzo dello spazio che sfrutta l'allocazione dello spazio di archiviazione in maniera dinamica e a richiesta. Questa impostazione è disponibile solo per iSCSI LUN (file normali).
  • VMware VAAI: VMware vStorage API per Array Integration (VAAI) può scaricare il lavoro delle operazioni standard su array di archiviazione virtuale nel dispositivo iSCSI nell'ambiente VMware vSphere e ottimizzare le prestazioni della memoria.
  • Windows ODX: una nuova tecnologia di trasferimento dati è stata sviluppata da Windows in Windows Server 2012 e 8 chiamata Offload Data Transfer (ODX). ODX migliora le prestazioni di trasferimento mediante l'offload del carico di lavoro dell'array di archiviazione.
    Se i dati di origine e di destinazione si trovano su LUN posti nello stesso volume, la clonazione sarà eseguita dalla tecnologia Synology Advanced LUN che risparmia fino al 99,9% di consumo dello spazio di archiviazione e al contempo accelera la clonazione.
  • Mappatura iSCSI LUN: per mappare l'iSCSI LUN su uno o più iSCSI Target.
  • Synology Snapshot Manager: Synology Snapshot Manager è un plug-in software, che consente di effettuare istantanee consistenti all'applicazione direttamente da DiskStation Manager (DSM).
    • In ambiente VMware, sarà necessario installare Synology Snapshot Manager sul server Windows in cui VMware vCenter Server è installato. Quando un'istantanea è effettuata su DSM, VMware vCenter Server saranno notificati e passeranno tutti i dati dalla memoria al LUN su cui l'istantanea è effettuata per garantire la coerenza dei dati. Una volta completata l'istantanea, VMware vCenter Server riprenderà il funzionamento I/O normale del datastore VMware. Synology Snapshot Manager for VMware vCenter Server supporta vSphere 5.1 o versioni più recenti.
    • In ambiente Windows, sarà necessario installare Synology Snapshot Manager sul server Windows. Quando un'istantanea è effettuata su DSM, Synology Snapshot Manager rafforzerà la tecnologia Microsoft Volume Shadow Copy Service (VSS) per produrre le copie di dati coerenti in un punto nel tempo. Synology Snapshot Manager for Windows supporta Windows Server 2008 R2, 2012 e 2012 R2.
    • Synology Snapshot Manager può essere scaricato gratis da www.synology.com.

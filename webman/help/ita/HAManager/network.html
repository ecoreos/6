Impostazioni di rete

Questa pagina contiene tre schede che visualizzano le impostazioni di rete del cluster high-availability, il server membro attivo e il server membro passivo.

Scheda cluster HA

Durante la creazione di un cluster high-availability, il sistema genera un indirizzo IP comune per l'IP del cluster high-availability in modo da ricevere le connessioni in arrivo. Se necessario possono essere creati altri indirizzi IP del cluster high-availability. Gli indirizzi IP delle interfacce di rete attivate saranno visualizzati in questa scheda (l'interfaccia di rete utilizzata dalla connessione heartbeat non sarà visualizzata in questa pagina poiché questa interfaccia sarà utilizzata per la replica dei dati tra i due server). La scheda mostra le seguenti informazioni:

  • Interfaccia di rete: le interfacce di rete attivate sul server attivo.
  • Connessione: visualizza se questa interfaccia è connessa o meno.
  • Indirizzo IP: durante la creazione di un cluster high-availability, il sistema genera un indirizzo IP comune per l'IP del cluster high-availability in modo da ricevere le connessioni in arrivo. È possibile connettersi al cluster high-availability utilizzando questo indirizzo IP quando si desidera accedere a DiskStation Manager (DSM) senza considerare quale server Synology è il server attivo. In questo modo si evita la connessione a indirizzi IP diversi quando si verifica il passaggio.
  • Subnetmask: la subnetmask associata per fornire l'indirizzo IP del cluster high-availability.

Per modificare Indirizzo IP e Subnetmask:

  1. Scegliere l'indirizzo IP o subnet mask da modificare e fare clic su Modifica o fare doppio clic sul campo per iniziare la modifica.
  2. Selezionare le caselle di controllo Abilita per abilitare o disabilitare ciascuna interfaccia. È possibile abilitare più di una interfaccia di rete cluster, mentre l'interfaccia di rete originale configurata durante la creazione del cluster non può essere disabilitata.

Per cambiare il nome server del cluster high-availability:

  1. Fare clic su Cambia impostazioni di rete per visualizzare il nome del cluster high-availability e cambiarne le impostazioni.
  2. Fare clic su Applica per salvare il nome corrente del cluster high-availability e le impostazioni dell'indirizzo IP.
  3. Per ignorare le modifiche correnti, fare clic su Annulla.

Schede server attivo e passivo (visualizzate come nome server)

Queste schede mostrano lo stato dell'interfaccia di rete sul server attivo o passivo. Di questi, la Connessione heartbeat occuperà una delle interfacce di rete su ciascun server attivo e passivo. Le schede visualizzano le seguenti informazioni:

  • Interfaccia di rete: l'interfaccia di rete fisica sul server.
  • Indirizzo IP: l'indirizzo IP assegnato per l'interfaccia di rete specificata.
  • Subnetmask: la subnetmask associata per l'indirizzo IP specificato.
  • Descrizione: visualizza le informazioni dettagliate dell'interfaccia di rete specificata e l'accessibilità del relativo indirizzo IP assegnato. Questo campo mostra le seguenti condizioni possibili:
    1. Indirizzo IP originale accessibile: l'indirizzo IP originale assegnato all'interfaccia sul server attivo è accessibile per i dati e i servizi.
    2. Indirizzo IP originale non accessibile: l'indirizzo IP originale assegnato all'interfaccia sul server passivo non è accessibile.
    3. IP connessione heartbeat non accessibile dall'Intranet: l'indirizzo IP originale assegnato automaticamente alla connessione heartbeat non è accessibile per dati e servizi.

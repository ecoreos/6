Monitor servizio

Questa pagina consente di monitorare alcuni servizi. Se un servizio monitorato non funziona sul server attivo, il sistema effettuerà il failover del server passivo in standby (finché il server passivo sta funzionando normalmente). Le sezioni seguenti sono visualizzate su questa pagina:

  • Servizio: visualizza i servizi che possono essere monitorati.
  • Stato: visualizza se i servizi stanno funzionando o meno.
  • Failover su malfunzionamento del servizio: Spuntare la casella di controllo accanto a un servizio per avviare il monitoraggio. Il sistema effettua il failover se il servizio selezionato fallisce.

Per monitorare un servizio:

  1. Trovare il servizio che si desidera monitorare. Spuntare la casella di controllo di fianco a esso.
  2. Fare clic su Applica per salvare le impostazioni.

Servizi supportati

I servizi che possono essere monitorati sono i seguenti: Windows File Service, iSCSI Target, FTP, Mac File Service, NFS Service.

Failover quando la connessione di rete fallisce

Se questa opzione è abilitata, il sistema effettuerà il failover del server passivo quando la connessione di rete del server attivo fallisce o non funziona correttamente, e la connessione di rete del server passivo funziona ancora normalmente.

  • Il failover sarà attivato solo se il server passivo ha più connessioni funzionanti. Ad esempio, se il server attivo ha tre connessioni e una non funziona, il sistema attiverà un failover solo se il server passivo ha tre o più connessioni funzionanti.
  • Connessioni combinate con Link Aggregation contano come una connessione. Ad esempio, se sono combinate due connessioni, e una di esse non funziona, ma l'altra sta ancora funzionando, la connessione sarà considerata ancora funzionante.

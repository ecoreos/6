Recupero

Questa pagina consente agli amministratori di sistema di eseguire il recupero. Per abilitare gli utenti all'accesso a istantanee della cartella condivisa, abilitare Rendi istantanea visibile per ogni cartella condivisa, in modo che gli utenti possano accedere alle istantanee tramite File Station o Windows File Explorer.

Cartella condivisa

Per ripristinare file/cartelle in una cartella condivisa:

  1. Selezionare una cartella condivisa da ripristinare.
  2. Fare clic su Recupera.
  3. Selezionare versione di istantanea da ripristinare.
  4. Fare clic su Sfoglia e copiare i file/cartelle da ripristinare nella destinazione.

Per ripristinare una cartella condivisa:

  1. Selezionare una cartella condivisa da ripristinare.
  2. Fare clic su Recupera.
  3. Selezionare versione di istantanea da ripristinare.
  4. Fare clic su Azione e selezionare Ripristino sul posto dall'elenco a discesa.
  5. Verrà visualizzato un messaggio di conferma. Fare clic su Sì per confermare l'azione.

Per clonare un'istantanea in una nuova cartella condivisa:

  1. Selezionare una cartella condivisa da ripristinare.
  2. Fare clic su Recupera.
  3. Selezionare una versione dell'istantanea da clonare in una nuova cartella condivisa.
  4. Fare clic su Azione e selezionare Clona con un nuovo nome dall'elenco a discesa.
  5. L'utente verrà reindirizzato alla pagina Clona in Pannello di controllo > Cartella condivisa > Crea. Inserire il nome della cartella condivisa clonata e fare clic su OK per finire.

Nota:

  • Le sezioni "homes" della cartella condivisa non possono essere ripristinate.
  • Le cartelle condivise non possono essere crittografate.
  • Se un'istantanea viene acquisita quando il trasferimento di qualsiasi file è ancora in corso, la coerenza dei dati non sarà garantita.
  • Limitazioni sulle cartelle condivise crittografate:
    • Nessun tipo di ripristino è disponibile per le cartelle condivise crittografate montate.
    • Clona con un nuovo nome non è disponibile per le cartelle condivise crittografate.
    • Le cartelle condivise crittografate non montate possono essere ripristinate da istantanee acquisite in precedenza utilizzando Ripristino sul posto.

Per eseguire la sincronizzazione di una replica di cartella condivisa:

  1. Selezionare la cartella condivisa replicata da cambiare.
  2. Fare clic su Passaggio.
  3. Selezionare il nuovo server di destinazione se sono presenti più attività di replica della cartella condivisa.
  4. Fare clic su Passaggio per avviare l'azione.

Per eseguire il failover di una replica di cartella condivisa:

  1. Selezionare una cartella condivisa di cui eseguire il failover.
  2. Fare clic su Failover > Failover.
  3. Selezionare la versione dell'istantanea da ripristinare.
  4. Fare clic su Failover per avviare l'azione.

Nota:

  • è possibile iniziare soltanto un failover del server di origine.

Per riproteggere la replica di cartella condivisa:

quando viene eseguito il failover nel server di destinazione, l'attività di replica arresterà la sincronizzazione dei dati tra i due server. Sarà necessario riproteggere l'attività di replica dopo il ripristino del server sorgente originale.

  1. Selezionare una cartella condivisa da ripristinare.
  2. Fare clic su Failover > Riproteggi.
  3. Selezionare il nuovo server sorgente.
  4. Selezionare il nuovo server di destinazione se sono presenti più attività di replica della cartella condivisa.
  5. Fare clic su Riproteggi per avviare l'azione.

Nota:

  • I dati della cartella condivisa replicata esistenti nel nuovo server di destinazione saranno sovrascritti da quelli nel nuovo server sorgente.
  • Se la versione istantanea selezionata per il failover non esiste nel server partner, potrebbe richiedere più tempo per ripetere la protezione.

Per eseguire un failover di test per una cartella condivisa replicata:

  1. Selezionare la cartella condivisa replicata di cui eseguire il test.
  2. Fare clic su Failover di test > Failover di test.
  3. Selezionare il Server di test se sono presenti più attività di replica della cartella condivisa.
  4. Inserire il Nome per il failover di test della cartella condivisa e selezionare una versione istantanea di cui eseguire il test.
  5. Fare clic su Failover di test per avviare l'azione.

Nota:

  • l'esecuzione del passaggio/failover di una replica ne pulirà il failover di test.
  • Non è possibile creare un'attività di replica per una cartella condivisa di failover di test.

Per pulire un failover di test per una cartella condivisa replicata:

  1. Selezionare la cartella condivisa replicata di cui si desidera pulire il failover di test.
  2. Fare clic su Failover di test > Pulizia failover di test.
  3. Selezionare il failover di test da pulire.
    Nota: Spuntare Elimina dati failover di test se non si desidera più mantenere la cartella condivisa creata per il failover di test.
  4. Fare clic su Pulizia failover di test per avviare l'azione.

iSCSI LUN

Per ripristinare un iSCSI LUN:

  1. Selezionare l'iSCSI LUN che si desidera ripristinare.
  2. Fare clic su Recupera.
  3. Selezionare versione di istantanea da ripristinare.
  4. Fare clic su Azione e selezionare Ripristino sul posto dall'elenco a discesa.
  5. Verrà visualizzato un messaggio di conferma. Fare clic su Sì per confermare l'azione.

Per clonare un'istantanea in una nuova iSCSI LUN:

  1. Selezionare la iSCSI LUN che si desidera clonare.
  2. Fare clic su Recupera.
  3. Selezionare una versione dell'istantanea da clonare in una nuova iSCSI LUN.
  4. Fare clic su Azione e selezionare Clona con un nuovo nome dall'elenco a discesa.
  5. L'utente verrà reindirizzato alla pagina Clona in Gestore archiviazione > iSCSI LUN. Inserire il nome della iSCSI LUN e fare clic su OK per finire.

Nota:

  • Se un'istantanea viene acquisita quando il trasferimento di qualsiasi file è ancora in corso, la coerenza dei dati non sarà garantita.

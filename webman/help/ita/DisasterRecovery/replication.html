Replica

Questa pagina consente agli amministratori di sistema di gestire le repliche. È possibile creare, modificare ed eliminare le attività di replica. Inoltre, è possibile seguire/arrestare la sincronizzazione e visualizzare le informazioni delle attività di replica.

Cartella condivisa

Per creare un'attività di replica per una cartella condivisa:

  1. Andare alla scheda Cartella condivisa:
  2. Fare clic su Crea.
  3. Verrà visualizzata la procedura guidata Crea attività di replica. Seguire i passaggi della procedura guidata per creare una replica della cartella condivisa.

Nota:

  • Le sezioni "homes" della cartella condivisa non possono essere replicate.
  • La cartella condivisa crittografata non può essere replicata.
  • La cartella condivisa deve essere posizionata in un volume Btrfs.
  • È possibile creare repliche uno-a-molti/estesi per una cartella condivisa in differenti server di destinazione. Tuttavia, è possibile creare soltanto una replica su un server di destinazione per ciascuna cartella condivisa.

Per eliminare una replica di cartella condivisa:

  1. Selezionare l'attività di replica della cartella condivisa da eliminare.
  2. Fare clic su Elimina.
  3. Verrà visualizzato un messaggio di conferma. Fare clic su Elimina per confermare l'eliminazione.

Per modificare una replica di cartella condivisa:

  1. Selezionare l'attività di replica della cartella condivisa da modificare.
  2. Fare clic su Modifica.
  3. Verrà visualizzata una finestra contenente le schede seguenti:
    • Server partner
    • Programmazione
    • Conservazione
  4. Nella scheda Server partner, è possibile modificare le informazioni del server partner, inclusi nome host, porta, nome utente e password.
  5. Nella scheda Programmazione, spuntare la casella di controllo Abilita la programmazione istantanee e impostare l'ora e la frequenza di scatto delle istantanee. È possibile deselezionare la casella di controllo per annullare le istantanee programmate per la cartella condivisa.
  6. Nella scheda Conservazione, scegliere una delle seguenti opzioni:
    • Conservare 256 ultime istantanee: Quando viene raggiunto il numero massimo di istantanee (256), le istantanee più antecedenti verranno sovrascritte da quelle scattate più di recente.
    • Conservare sempre le istantanee: Quando viene raggiunto il numero massimo di istantanee, Snapshot e Replica interromperà lo scatto di nuove istantanee senza rimuovere quelle già scattate.
    • Conservare le istantanee con il seguente criterio: È possibile configurare le regole di conservazione per intervalli di tempo specificati.
  7. Fare clic su OK per applicare le impostazioni.

Nota:

  • È possibile modificare solo la programmazione dell'attività di replica nel server di origine.
  • La modifica delle impostazioni di conservazione influirà sulle impostazioni equivalenti nella pagina Snapshot.

Per visualizzare le informazioni di una replica di cartella condivisa:

  1. Selezionare l'attività di replica della cartella condivisa da visualizzare.
  2. Fare clic su Informazioni.
  3. Verrà visualizzata una finestra contenente le informazioni seguenti:
    • Stato di questa replica.
    • Statistiche di questa replica.

Per eseguire la sincronizzazione di una replica di cartella condivisa:

  1. Selezionare la replica di cartella condivisa da sincronizzare.
  2. Fare clic su Esegui.
  3. Inserire la descrizione dell'istantanea da acquisire.
  4. Fare clic su OK per confermare l'azione.

Per arrestare la sincronizzazione di una replica di cartella condivisa:

  1. Selezionare la replica di cartella condivisa di sincronizzazione da arrestare.
  2. Fare clic su Stop.
  3. Verrà visualizzato un messaggio di conferma. Fare clic su Stop per confermare l'azione.

iSCSI LUN

Per creare un'attività di replica per una iSCSI LUN:

  1. Andare alla scheda iSCSI LUN.
  2. Fare clic su Crea.
  3. Verrà visualizzata la procedura guidata Crea attività di replica. Seguire i passaggi della procedura guidata per creare una replica iSCSI LUN.

Nota:

  • l'iSCSI LUN deve essere stata creata come LUN avanzata.
  • È possibile creare repliche uno-a-molti/estesi per una iSCSI LUN in differenti server di destinazione. Tuttavia, è possibile creare soltanto una replica su un server di destinazione per ciascuna iSCSI LUN.

Per eliminare una replica di iSCSI LUN:

  1. Selezionare l'attività di replica dell'iSCSI LUN da eliminare.
  2. Fare clic su Elimina.
  3. Verrà visualizzato un messaggio di conferma. Fare clic su Elimina per confermare l'azione.

  1. Selezionare l'attività di replica dell'iSCSI LUN da modificare.
  2. Fare clic su Modifica.
  3. Verrà visualizzata una finestra contenente le schede seguenti:

    • Programmazione
    • Conservazione
  4. Nella scheda Server partner, è possibile modificare le informazioni del server partner, inclusi nome host, porta, nome utente e password.
  5. Nella scheda Programmazione, spuntare la casella di controllo Abilita la programmazione istantanee e impostare l'ora e la frequenza di scatto delle istantanee. È possibile deselezionare la casella di controllo per annullare le istantanee programmate per l'iSCSI LUN.
  6. Nella scheda Conservazione, scegliere una delle seguenti opzioni:
    • Conservare 256 ultime istantanee: Quando viene raggiunto il numero massimo di istantanee (256), le istantanee più antecedenti verranno sovrascritte da quelle scattate più di recente.
    • Conservare sempre le istantanee: Quando viene raggiunto il numero massimo di istantanee, Snapshot e Replica interromperà lo scatto di nuove istantanee senza rimuovere quelle già scattate.
    • Conservare le istantanee con il seguente criterio: È possibile configurare le regole di conservazione per intervalli di tempo specificati.
  7. Nella scheda Applicazione, spuntare l'opzione Abilitare le istantanee coerenti con l'applicazione per abilitare le istantanee coerenti con l'applicazione. Per annullare le istantanee coerenti con l'applicazione, deselezionare la casella di controllo.
  8. Fare clic su OK per applicare le impostazioni.

Nota:

  • è possibile modificare la programmazione sull'origine della replica.
  • La modifica delle impostazioni di conservazione/applicazione influirà sulle impostazioni equivalenti nella pagina Snapshot.

Replica uno-a-molti/estesa

È possibile creare più repliche di una cartella condivisa/iSCSI LUN su più server di destinazione, formando uno schema uno-a-molti. È possibile anche creare una replica estesa di una cartella condivisa/iSCSI LUN replicata, formando uno schema esteso.

Nota:

  • sia in uno schema uno-a-molti che esteso, qualsiasi attività di replica può avere soltanto tre server di destinazione al massimo.

Politica di conservazione

Quest'opzione fornisce una soluzione ideale per contribuire a specificare le quantità massime delle versioni di istantanee per salvare lo spazio su volume, ma potrebbe essere necessario conservare le istantanee per periodi di tempo più lunghi.

Synology impiega la politica di conservazione GFS, o Grandfather-Father-Son. È possibile configurare le quantità massime di versioni di istantanee da conservare per i seguenti intervalli di tempo rispettivamente: orario, giornaliero, settimanale, mensile e annuale.

Data Protection Manager conserverà il maggior numero possibile di istantanee configurate per ogni intervallo di tempo. Se in un intervallo di tempo è presente più di una versione di istantanea, verrà conservata solo la più vecchia. Ad esempio, se si imposta una politica su 10 istantanee settimanali, Data Protection Manager conserverà l'istantanea più vecchia (se viene acquisita più di un'istantanea a settimana) per ciascuna delle ultime 10 settimane.

Inoltre, tutte le versioni di istantanee entro un'ora dall'acquisizione dell'istantanea più recente verranno conservate per impostazione predefinita, per semplificare la ricerca e il ripristino da versioni di istantanee recenti. È anche possibile bloccare un'istantanea per evitarne la rimozione automatica prevista dalle regole di conservazione in uso.

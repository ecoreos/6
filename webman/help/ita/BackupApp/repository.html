Destinazione di backup

Nella scheda Destinazione di backup di Backup e Replica, è possibile definire una destinazione di backup locale, di rete o cloud creando un'attività di backup.

Per creare una destinazione di backup locale:

Una destinazione di backup locale è necessaria per il backup di dati da DiskStation a una cartella locale condivisa, volume o unità esterna USB/eSATA. La destinazione del backup deve essere diversa dalla sorgente e la capacità della destinazione deve essere maggiore della dimensione totale dei dati di backup.

  1. Fare clic su Crea, seleziona Destinazione di backup locale, quindi fare clic su Avanti.
  2. Assegnare un nome alla destinazione di backup, selezionare la destinazione in cui salvare i dati di backup e se si desidera conservare solo la copia di backup più recente o tutte le versioni di backup.

Note.

  • Se i dati nella destinazione vengono rimossi durante la procedura di backup, il backup successivo sarà un backup completo.
  • Le unità esterne nuove possono avere la necessità di essere formattate prima di servire come destinazione di backup.
  • Prima di creare un'attività di backup, verificare di avere collegato le unità esterne correttamente a DiskStation. È possibile andare su Pannello di controllo > Dispositivi esterni per controllare lo stato delle unità USB/eSATA. Le unità esterne non possono essere selezionate come destinazione per il backup se l'unità esterna è stata inizializzata, formattata o non funziona correttamente.
  • Cartelle condivise remote non possono essere selezionate come sorgente o destinazione di backup.
  • Se una cartella condivisa crittografata non è attiva, non può essere scelta come destinazione di backup.
  • Non può essere effettuato il backup di miniature dell'immagine quando il file system della destinazione è NTFS/FAT.

Per creare una destinazione di backup di rete:

Una destinazione di backup di rete è necessaria se si esegue il backup di dati da DiskStation a un altro DiskStation o server compatibile con rsync sulla rete. Quando si creano attività di backup di rete, sono disponibili varie opzioni, quali codifica del trasferimento, compressione, backup livello di blocco, ecc. Alcune opzioni, come la codifica trasferimento, potrebbero ridurre la velocità dell'attività di backup.

  1. Prima di creare un'attività di backup di rete, accedere al server di destinazione, andare su Backup e Replica > Servizi di backup e abilitare il servizio di backup di rete e/o destinazione volume del backup di rete.
  2. Per il backup su DiskStation, andare su Backup e Replica > Destinazione di backup. Fare clic su Crea, selezionare Destinazione di backup di rete e seguire la procedura guidata per completare.
  3. Con Destinazione di backup di rete (server DiskStation) è possibile archiviare i dati di backup in una cartella condivisa per conservare la copia di backup più aggiornata o archiviare i dati di backup su un volume per conservare più versioni delle copie di backup. Con Destinazione di backup di rete (server compatibile rsync), è possibile conservare solo la copia di backup più recente.

Note.

  • Le seguenti modalità sono disponibili quando si crea un'attività di backup con server compatibili rsync come destinazione di backup di rete: modalità rsync daemon, che è utilizzata quando si seleziona un modulo di backup (cartella condivisa su server compatibile rsync) dal menu a discesa accanto a Moduli backup; e modalità shell remota, che viene abilitata solo quando si inserisce un percorso assoluto (che inizia con "/") nei Moduli backup.
  • Per gli utenti Linux, solo gli account admin, root e rsync possono svolgere le attività di backup.
  • Per accedere a un server di destinazione su Internet usando il suo nome host, il nome host deve essere del formato FQDN (Nome di Dominio completo), ad esempio abc.bcd.com.
  • Quando si effettua il backup da un dispositivo Synology DiskStation all'altro, viene chiesto di inserire il nome utente e la password per un account di destinazione. Soprattutto, se si vuole effettuare il backup su un volume in un'altro DiskStation, è necessario specificare un account appartenente al gruppo administrators. Quando si esegue il backup di cartelle condivise su un'altra DiskStation, è possibile farlo in base alle impostazioni dei privilegi di accesso delle cartelle. Solo le cartelle condivise alle quali un account utente specifico ha privilegi di accesso saranno visualizzate. Se non si trova la cartella condivisa desiderata come destinazione, controllare i privilegi di accesso.
  • La codifica del backup viene supportata solo quando il server di destinazione supporta il backup di rete codificato e la password di autenticazione non può essere vuota.
  • La modalità shell remota supporta solo il backup crittografato.
  • Quando si effettua un backup su un server compatibile con rsync, non si sarà in grado di modificare la modalità di crittografia rsync dopo aver creato le attività di backup di rsync. Per modificare la configurazione, creare una nuova attività di rsync.
  • In caso di backup dei file in DiskStation tramite un client rsync, i nomi dei file devono essere nella codifica UTF-8.
  • In caso di backup dei file in DiskStation tramite un client compatibile rsync, è possibile eseguire le operazioni soltanto dagli account admin, root e rsync della DiskStation.
  • In caso di backup dei file da un DiskStation in un server compatibile rsync, sarà necessario inserire il nome e la password corretti dell'account rsync di backup. Inoltre , sarà necessario scegliere il modulo di backup corretto con il privilegio Scrittura.

Per creare una destinazione di Backup Amazon S3:

È necessaria una destinazione di backup Amazon S3 per il backup di dati da DiskStation a un server Amazon S3.

  1. Fare clic su Crea, selezionare Destinazione di backup cloud pubblico, quindi Destinazione Backup Amazon S3 e fare clic su Avanti.
  2. Assegnare un nome alla destinazione di backup, immettere le credenziali di accesso e quindi compilare tutti i campi necessari per configurare l'attività di backup.

Note.

  • Quando si creano attività di backup con server Amazon S3 come destinazione di backup di rete, sono disponibili varie opzioni, quali la codifica del trasferimento, il backup su Amazon S3 RRS (archiviazione a ridondanza ridotta), ecc. Alcune opzioni, come codifica trasferimento, potrebbero ridurre la velocità delle attività di backup.
  • I file di backup vengono archiviati nel Bucket selezionato sul server Amazon S3 e un Valore Set Key inizia con un nome directory a propria scelta e termina con il percorso completo del file.
  • La struttura dei dati di Amazon S3 include un paio di Bucket e KEY senza alcuna struttura a cartella. Il backup di cartella vuota non è supportato.
  • Se un file sottoposto a backup non è stato modificato, non verrà sottoposto di nuovo a backup sul server Amazon S3 nello stesso Bucket.
  • Ogni Bucket su Amazon S3 deve avere un nome univoco. Inoltre, il nome deve:
    • Essere composto da 3 a 63 caratteri.
    • Contenere soltanto lettere minuscole, numeri, punti (.) e trattini (-).
    • Iniziare e finire con un numero o una lettera e non può finire con un trattino (-), contenere due punti adiacenti o contenere trattini dopo punti (ad es., "my-.bucket.com" e "my.-bucket" non sono validi).
  • La lunghezza massima di una Amazon S3 KEY è di 1024 caratteri. Per assicurare il successo di un backup su Amazon S3, la lunghezza massima del percorso è di 1024 meno la lunghezza del nome della cartella di destinazione specificato durante la creazione dell'attività.
  • Abilitando il supporto Amazon S3 RRS (archiviazione a ridondanza ridotta) verrà eseguito il backup sull'archivio RRS solo dei dati modificati dopo l'abilitazione dell'opzione, il backup su RRS non sarà disponibile per i dati di cui è già stato eseguito il backup sull'archivio standard di Amazon S3.
  • Lo spostamento dei file di backup da Amazon S3 a Amazon Glacier non è supportato. Se i file sono spostati da Amazon S3 a Amazon Glacier mediante la console di Gestione AWS, l'attività di backup collegata potrebbe non venire eseguita in modo corretto.
  • Il backup di file su bucket stile host virtuale con SSL non è supportato.
  • Quando si trasferisce un file sarà diviso in parti. La dimensione della parte può essere impostata dall'opzione Dimensione parte Multipart Upload in impostazioni attività con 512MB come dimensione predefinita. Se l'ambiente di rete non è stabile, è bene che l'opzione sia un valore più piccolo.
  • Il numero massimo di parti per upload è 10000. Le dimensioni massime di un file corrispondono alla Dimensione parte Multipart Upload moltiplicata per 10000.
  • I nomi dei file non possono contenere caratteri non UTF-8 o di controllo.

Per creare una destinazione di backup Microsoft Azure:

È necessaria una destinazione di backup Microsoft Azure per il backup di dati da DiskStation a un server Microsoft Azure.

  1. Fare clic su Crea, selezionare Destinazione di backup cloud pubblico, quindi Destinazione di Backup Microsoft Azure e fare clic su Avanti.
  2. Denominare la destinazione di backup, immettere le credenziali di accesso e quindi compilare tutti i campi necessari per configurare l'attività di backup.

Note.

  • Quando si creano attività di backup con il server Microsoft Azure come destinazione di backup di rete, sono disponibili varie opzioni (ad es. codifica trasferimento). Alcune opzioni, come codifica trasferimento, potrebbero ridurre la velocità delle attività di backup.
  • I file di backup sono archiviati nel Container selezionato sul server Microsoft Azure e il nome Blob impostato inizia con un nome directory a propria scelta e termina con il percorso completo del file.
  • La struttura di dati di Microsoft Azure include un'associazione Contenitore e Blob senza alcuna struttura a cartella. Il backup di cartella vuota non è supportato.
  • Se un file sottoposto a backup non è stato modificato, non verrà sottoposto di nuovo a backup sul server Microsoft Azure nello stesso Contenitore.
  • Ogni Contenitore su Microsoft Azure deve avere un nome univoco. Inoltre, il nome deve:
    • Essere composto da 3 a 63 caratteri.
    • Contenere soltanto lettere minuscole, numeri e trattini (-).
    • Iniziare e finire con un numero o una lettera e non può contenere due trattini adiacenti.
  • La lunghezza massima di un nome Blob di Microsoft Azure è di 1024 caratteri. Per assicurare il successo di un'attività di backup su Microsoft Azure, la lunghezza massima del percorso è di 1024 meno la lunghezza del nome della cartella di destinazione specificato durante la creazione dell'attività.
  • I nomi dei file non possono contenere caratteri non UTF-8 o di controllo.
  • Il messaggio di avviso di autenticazione non riuscita può essere dovuto a un'informazione di autenticazione non corretta, a un problema di connessione di rete o all'ora del DiskStation e del server remoto che non corrispondono. Per controllare le impostazioni sull'orario in DiskStation, andare su Pannello di controllo > Opzioni regionali > Ora per sincronizzare l'ora con il server NTP.

Per gestire la destinazione di backup:

Selezionare una destinazione di backup esistente e fare clic su uno dei pulsanti seguenti:

  • Modifica: per modificare le impostazioni della destinazione di backup selezionata.
  • Elimina: per eliminare la destinazione di backup selezionata.

Backup alla cartella condivisa

Questi due tipi di destinazioni di backup conserveranno solo la copia di backup più recente. Come impostazione predefinita, dopo aver completato il backup iniziale, i file che sono stati rimossi dall'origine sono rimossi dalla destinazione quando un'attività è eseguita successivamente. Tuttavia, se di desidera conservare i file nella destinazione anche dopo averli cancellati dall'origine, abilitare l'opzione Conserva file di backup nella destinazione quando si creano attività di backup.

Backup al Volume

Ce type de destination de sauvegarde doit être un volume local ou réseau sur un périphérique DiskStation (à l'exception des périphériques de stockage externes). Esso archivierà più versioni di copie di backup, in modo da poter ripristinare facilmente qualsiasi cartella da un dato momento.

Per eliminare i dati archiviati da un'attività di backup:

  1. Andare sulla scheda Destinazione di backup ed espandere la destinazione di backup contenente l'attività che si vuole eliminare.
  2. Fare clic sull'icona Elimina in Attività di backup selezionata e quindi Elimina nella finestra di dialogo.

Per eliminare i dati archiviati di una versione di backup:

  1. Andare sulla scheda Destinazione di backup ed espandere la destinazione di backup contenente la versione dell'attività che si vuole eliminare.
  2. Fare clic sull'icona Informazioni versione di backup sull'Attività di Backup selezionata.
  3. Selezionare la versione di backup che si vuole eliminare e fare clic sull'icona Elimina.

Per bloccare una versione di backup e impedirne la rimozione durante la rotazione di backup:

  1. Andare sulla scheda Destinazione di backup ed espandere la destinazione di backup contenente la versione dell'attività che si vuole bloccare.
  2. Fare clic sull'icona Informazioni versione di backup sull'Attività di Backup selezionata.
  3. Selezionare la versione di backup che si vuole bloccare e fare clic sull'icona Blocca.

Per gestire i privilegi di una destinazione di backup:

  1. Selezionare una destinazione di backup associata a un volume locale, fare clic su Modifica.
  2. Fare clic su Elenco consentiti e poi su Aggiungi per aggiungere l'utente o il gruppo all'elenco.

Per impostare una destinazione di backup con un proprietario:

  1. Selezionare una destinazione di backup associata a un volume locale e fare clic sull'icona Proprietà sull'Attività di backup selezionata.
  2. In Privilegio di backup, selezionare un Proprietario nel menu a discesa.

Per consentire a un utente o a un gruppo sulla DiskStation di leggere e ripristinare i dati archiviati da un'attività di backup:

  1. Selezionare una destinazione di backup associata a un volume locale e fare clic sull'icona Proprietà sull'Attività di backup selezionata.
  2. Fare clic su Elenco consentiti e poi su Aggiungi per aggiungere l'utente o il gruppo all'elenco.

Note.

  • Il proprietario predefinito di un'attività di backup è l'utente che ha creato l'attività.
  • Solo il proprietario dell'attività di backup può eseguire l'attività. Tuttavia, le attività di backup create prima di DSM 5.1 non hanno tale limitazione, quindi qualsiasi utente appartenente al gruppo administrators è in grado di eseguire questa attività di backup.
  • Il proprietario dell'attività e gli utenti che appartengono al gruppo administrators possono eliminare i dati archiviati di un'attività di backup.
  • Il proprietario dell'attività e gli utenti che appartengono al gruppo administrators, e gli utenti che fanno parte di Elenco consentiti possono tutti ripristinare i dati archiviati di un'attività di backup.
  • Solo gli utenti che appartengono al gruppo administrators possono impostare una destinazione di backup locale con privilegi di accesso e un'attività di backup su una destinazione di backup locale con un proprietario e un elenco consentiti.
  • I dati archiviati di un'attività di backup utilizzeranno la quota di archiviazione del proprietario dell'attività.
  • Se un volume non ha una destinazione di backup, solo gli utenti che appartengono al gruppo administrators possono creare la prima destinazione di backup sul volume.
  • Solo gli utenti che appartengono al gruppo administrators possono eliminare i dati archiviati di un'intera destinazione di backup.
  • Ogni backup comprende una configurazione del sistema e metadati del file.
  • Crittografia cartella condivisa cifrata non supportata.
  • Il blocco versioni non è supportato in DSM 5.1 o inferiori.

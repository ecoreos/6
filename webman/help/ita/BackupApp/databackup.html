Archiviazione

Backup e Ripristino consente di eseguire il backup di vari tipi dati (configurazioni di sistema, cartelle condivise e applicazioni/pacchetti) in DiskStation, manualmente in base alla programmazione. I dati di backup possono archiviati nelle cartelle condivise locali, nei server remoti e nei cloud pubblici (Amazon S3, Microsoft Azure, hicloud S3 e SFR NAS BACKUP). Con Backup e Ripristino, ogni attività di backup consente di conservare più versioni di backup e di ruotarle con i criteri Smart Recycle.

Backup dei dati

Backup e Ripristino offre varie opzioni di destinazioni e metodi di backup per configurare un'attività di backup dati.

Per creare un'attività di backup dati:

  1. Fare clic su + nell'angolo in basso a sinistra e selezionare Attività di backup dati per lanciare la procedura guidata di backup.
  2. Selezionare il tipo desiderato di destinazione di backup:
    • Cartelle condivise locali/remote su dispositivi di archiviazione esterni
    • Server rsync remoti
    • Cloud pubblici (es. Amazon S3 e Microsoft Azure)
  3. Configurare il tipo desiderato di attività di backup:
    • Per creare un'attività completamente nuova, selezionare Crea attività di backup.
    • Per riutilizzare i dati di backup esistenti su qualsiasi destinazione, selezionare Ricollega ad attività esistente. Il ricollegamento consente di utilizzare direttamente i dati di backup da un'attività differente.
  4. Seguire le istruzioni della procedura guidata per completare le impostazioni. Selezionare Abilita crittografia lato client pere crittografare i dati prima della trasmissione e garantire la sicurezza dei dati nella destinazione di backup.

Note:

  • I nomi delle attività possono includere da 1 a 32 caratteri Unicode, ma non possono contenere i seguenti caratteri:
    ! " # $ % & ' ( ) * + , / : ; < = > ? @ [ ] \ ^ ` { } | ~
  • La lunghezza massima del percorso completo della cartella per il backup è 2048 caratteri.
  • Se si caricano più di 32.000 file o cartelle via AFP in una cartella condivisa EXT3, tutte le attività di backup della cartella condivisa non verranno eseguite in modo corretto.
  • Una cartella condivisa crittografata può essere sottoposta a backup quando viene montata e può essere ripristinata in una cartella crittografata/decrittografata. Tuttavia, verrà sempre ripristinata come decrittografata /////////e non sarà montata automaticamente all'avvio. Verrà richiesto di inserire o importare la chiave di crittografia della cartella condivisa crittografata per montarla.
  • Il backup di una cartella crittografata a un'altra cartella crittografata non è supportato.
  • L'utilizzo di cartelle remote come sorgente di backup o destinazione di backup non è supportato.
  • Il backup di determinate applicazioni è supportato. Per verificare se un pacchetto specifico supporta il backup, e il contenuto del backup e le limitazioni, fare riferimento alla guida di ciascuna applicazione.
  • DiskStation non supporta il backup di applicazioni senza un dispositivo esterno EXT3/EXT4 scrivibile.
  • In DiskStation è solo possibile eseguire il backup delle cartelle condivise create nei dispositivi esterni EXT3/EXT4.

può essere anche effettuato il backup dei file delle impostazioni di Windows ACL su DiskStation a seconda dei file system di destinazione:

  • Tutte le destinazioni di backup possono eseguire il backup delle impostazioni di Windows ACL indipendentemente dal file system di destinazione

Per esportare i dati i backup in cloud pubblici o NAS Synology remoto:

Quando si esegue il backup di una grande quantità di dati su un cloud pubblico, si consiglia di esportare la versione iniziale in un dispositivo di archiviazione esterno (es. unità USB) e fornire il dispositivo al provider di servizi cloud. Il dispositivo potrà connettersi e caricare i dati direttamente nel cloud. In questo modo, sarà possibile ridurre notevolmente la durata del primo backup.

  1. Fare clic su + nell'angolo in basso a sinistra e selezionare Synology NAS remoto o il provider di servizi cloud desiderati.
  2. Selezionare Esportare in una cartella condivisa locale (incluso un dispositivo di archiviazione esterno) e completare le impostazioni. Quando richiesto, selezionare Abilita crittografia lato client per il backup di dati protetto.
  3. Consegnare il dispositivo di archiviazione esterno al NAS Synology remoto o al provider di servizi cloud. Importare i dati esportati dal dispositivo di archiviazione esterno.
  4. Dopo la conferma, tornare a Backup e Ripristino e fare clic su Ricollega ora.
  5. Seguire le istruzioni della procedura guidata di ricollegamento per collegare DiskStation al repository. Se i dati esportati sono criptati, è necessaria la password/chiave di crittografia.

Nota:

  • la crittografia lato client non può essere disabilitata dopo la configurazione dell'attività, né è possibile modificare la password.
  • Un repository di backup può essere collegato solo con un'attività di backup.
  • Se la cartella di origine backup è una cartella condivisa crittografata o la relativa sotto-cartella, la cartella di origine può essere montata o smontata quando si crea l'attività di backup.
  • Le cartelle condivise crittografate non possono essere la destinazione di backup.
  • Se si seleziona Abilita crittografia lato client, la password/chiave di crittografia è richiesta per ripristinare i dati in questa attività. La perdita della password/chiave di crittografia comporterà la perdita definitiva dei dati.

Gestire attività di backup:

Selezionare un'attività di backup nel pannello sinistro e fare clic sui pulsanti seguenti per la gestione corrispondente:

  • Backup ora: backup immediato dei dati dell'attività selezionata.
  • Backup di Explorer (icona lente di ingrandimento): visualizzazione di tutte le versioni di backup disponibili e copia/ripristino/download dei file contenuti.
  • Elenco versioni: visualizzare dell'ora e dello stato di creazione di tutte le versioni di backup, eliminazione di versioni specifiche o blocco di alcune specifiche per la prenotazione forzata.
  • Modifica: modifica dell'origine di backup, della destinazione, della programmazione e di altre impostazioni.
  • Elimina: eliminare l'attività di backup.
  • Annulla: arresto dell'attività di backup in esecuzione.

Per ruotare le versioni:

La rotazione backup è un sistema di gestione delle versioni. Consente di gestire le versioni di backup eliminando automaticamente le versioni più vecchie, quelle non necessarie e, potenzialmente, liberando spazio di archiviazione. Questa funzione può essere abilitata su Impostazioni attività oppure configurata quando si crea un'attività di backup.

  1. Selezionare l'attività di backup desiderata nel pannello di sinistra e fare clic su Modifica o Impostazioni> Impostazioni attività.
  2. Selezionare Attiva rotazione backup e scegliere la modalità di rotazione desiderata:
    • Dalle versioni più recenti: eliminare le versioni di backup più recenti quando le versioni disponibili superano il numero definito.
    • Smart Recycle: mantenere le versioni di backup mediante i criteri:
      • Versioni orarie dalle 24 ore passate: mantenere la versione più recente creata ogni ora.
      • Versioni giornaliere da 1 giorno trascorso a 1 mese: mantenere la versione più recente creata ogni giorno.
      • Versione settimanale dopo 1 mese: mantenere la versione più recente creata ogni settimana.
  3. Specificare il numero di versioni massimo. Quando il numero di versioni supera questo numero, la versione meno recente viene ruotata.
  4. Se è stata configurata una programmazione di backup ripetuta su Programmazione, è possibile visualizzare in anteprima il modo in cui le versioni di backup saranno conservate e il punto di ripristino più recente nella timeline.
  5. Fare clic su OK per salvare le impostazioni.

Nota:

  • Le versioni bloccate non saranno mai rimosse.
  • Solo le attività di backup multiversione forniscono la rotazione delle versioni.
  • La rotazione versioni non è supportata in DSM 5.1 o inferiori.
  • La rotazione sarà attivata subito dopo la modifica dell'impostazione o al termine dell'attività di backup.
  • Le versioni bloccate sono escluse quando si calcola il numero massimo di versioni.

Note sulle attività di backup dei dati

Per ogni tipo di attività di backup, leggere le note seguenti per importanti dettagli di configurazione e suggerimenti sulla risoluzione dei problemi.

Attività di backup locale:

  • Se i dati nella destinazione vengono rimossi durante la procedura di backup, il backup successivo sarà un backup completo.
  • Le unità esterne nuove possono avere la necessità di essere formattate prima di servire come destinazione di backup.
  • Prima di creare un'attività di backup, verificare di avere collegato le unità esterne correttamente a DiskStation. È possibile andare su Pannello di controllo > Dispositivi esterni per controllare lo stato delle unità USB/eSATA. Le unità esterne non possono essere selezionate come destinazione per il backup se l'unità esterna è stata inizializzata, formattata o non funziona correttamente.
  • Cartelle condivise remote non possono essere selezionate come sorgente o destinazione di backup.
  • Se una cartella condivisa crittografata non è attiva, non può essere scelta come destinazione di backup.
  • Non può essere effettuato il backup di miniature dell'immagine quando il file system della destinazione è NTFS/FAT.

Attività di backup rsync remote:

  • Le seguenti modalità sono disponibili quando si crea un'attività di backup con server compatibili rsync come destinazione di backup di rete: modalità rsync daemon, che è utilizzata quando si seleziona un modulo di backup (cartella condivisa su server compatibile rsync) dal menu a discesa accanto a Moduli backup; e modalità shell remota, che viene abilitata solo quando si inserisce un percorso assoluto (che inizia con "/") nei Moduli backup.
  • Per gli utenti Linux, solo gli account admin, root e rsync possono svolgere le attività di backup.
  • Per accedere a un server di destinazione su Internet usando il suo nome host, il nome host deve essere del formato FQDN (Nome di Dominio completo), ad esempio abc.bcd.com.
  • Quando si effettua un backup da un dispositivo Synology NAS all'altro, viene chiesto di inserire il nome utente e la password per un account di destinazione. Quando si esegue il backup di cartelle condivise su un'altra DiskStation, è possibile farlo in base alle impostazioni dei privilegi di accesso delle cartelle. Solo le cartelle condivise alle quali un account utente specifico ha privilegi di accesso saranno visualizzate. Se non si trova la cartella condivisa desiderata come destinazione, controllare i privilegi di accesso.
  • La codifica del trasferimento viene supportata solo quando il server di destinazione supporta il backup di rete codificato e la password di autenticazione non può essere vuota.
  • La modalità shell remota supporta solo la codifica di trasferimento.
  • In caso di backup dei file in un dispositivo Synology NAS tramite un client rsync, i nomi dei file devono essere nella codifica UTF-8.
  • In caso di backup dei file in un dispositivo Synology NAS tramite un client compatibile rsync, è possibile eseguire le operazioni soltanto dagli account admin, root e rsync della DiskStation.
  • Se si sta eseguendo il backup dei file da una DiskStation verso un server compatibile rsync, sarà necessario inserire il nome e la password corretti dell'account rsync di backup. Inoltre , sarà necessario scegliere il modulo di backup corretto con il privilegio Scrittura.

Attività di backup di Amazon S3:

  • Quando si creano attività di backup con server Amazon S3 come destinazione di backup di rete, sono disponibili varie opzioni, quali la codifica del trasferimento, il backup su Amazon S3 RRS (archiviazione a ridondanza ridotta), ecc. Alcune opzioni, come codifica trasferimento, potrebbero ridurre la velocità delle attività di backup.
  • I file di backup vengono archiviati nel Bucket selezionato sul server Amazon S3 e un Valore Set Key inizia con un nome directory a propria scelta e termina con il percorso completo del file.
  • La struttura dei dati di Amazon S3 include un paio di Bucket e KEY senza alcuna struttura a cartella. Il backup di cartella vuota non è supportato.
  • Se un file sottoposto a backup non è stato modificato, non verrà sottoposto di nuovo a backup sul server Amazon S3 nello stesso Bucket.
  • Ogni Bucket su Amazon S3 deve avere un nome univoco. Inoltre, il nome deve:
    • Essere composto da 3 a 63 caratteri.
    • Contenere soltanto lettere minuscole, numeri, punti (.) e trattini (-).
    • Iniziare e finire con un numero o una lettera e non può finire con un trattino (-), contenere due punti adiacenti o contenere trattini dopo punti (ad es., "my-.bucket.com" e "my.-bucket" non sono validi).
  • La lunghezza massima di una Amazon S3 KEY è di 1024 caratteri. Per assicurare il successo di un backup su Amazon S3, la lunghezza massima del percorso è di 1024 meno la lunghezza del nome della cartella di destinazione specificato durante la creazione dell'attività.
  • Abilitando il supporto Amazon S3 RRS (archiviazione a ridondanza ridotta) verrà eseguito il backup sull'archivio RRS solo dei dati modificati dopo l'abilitazione dell'opzione, il backup su RRS non sarà disponibile per i dati di cui è già stato eseguito il backup sull'archivio standard di Amazon S3.
  • Lo spostamento dei file di backup da Amazon S3 a Amazon Glacier non è supportato. Se i file sono spostati da Amazon S3 a Amazon Glacier mediante la console di Gestione AWS, l'attività di backup collegata potrebbe non venire eseguita in modo corretto.
  • Il backup di file su bucket stile host virtuale con SSL non è supportato.
  • Quando si trasferisce un file sarà diviso in parti. La dimensione della parte può essere impostata dall'opzione Dimensione parte Multipart Upload in impostazioni attività con 512MB come dimensione predefinita. Se l'ambiente di rete non è stabile, è bene che l'opzione sia un valore più piccolo.
  • Il numero massimo di parti per upload è 10000. Le dimensioni massime di un file corrispondono alla Dimensione parte Multipart Upload moltiplicata per 10000.
  • I nomi dei file non possono contenere caratteri non UTF-8 o di controllo.

Attività di backup Microsoft Azure:

  • Quando si creano attività di backup con il server Microsoft Azure come destinazione di backup di rete, sono disponibili varie opzioni (ad es. codifica trasferimento). Alcune opzioni, come codifica trasferimento, potrebbero ridurre la velocità delle attività di backup.
  • I file di backup sono archiviati nel Container selezionato sul server Microsoft Azure e il nome Blob impostato inizia con un nome directory a propria scelta e termina con il percorso completo del file.
  • La struttura di dati di Microsoft Azure include un'associazione Contenitore e Blob senza alcuna struttura a cartella. Il backup di cartella vuota non è supportato.
  • Se un file sottoposto a backup non è stato modificato, non verrà sottoposto di nuovo a backup sul server Microsoft Azure nello stesso Contenitore.
  • Ogni Contenitore su Microsoft Azure deve avere un nome univoco. Inoltre, il nome deve:
    • Essere composto da 3 a 63 caratteri.
    • Contenere soltanto lettere minuscole, numeri e trattini (-).
    • Iniziare e finire con un numero o una lettera e non può contenere due trattini adiacenti.
  • La lunghezza massima di un nome Blob di Microsoft Azure è di 1024 caratteri. Per assicurare il successo di un'attività di backup su Microsoft Azure, la lunghezza massima del percorso è di 1024 meno la lunghezza del nome della cartella di destinazione specificato durante la creazione dell'attività.
  • I nomi dei file non possono contenere caratteri non UTF-8 o di controllo.
  • Il messaggio di avviso di autenticazione non riuscita può essere dovuto a un'informazione di autenticazione non corretta, a un problema di connessione di rete o all'ora del DiskStation e del server remoto che non corrispondono. Per controllare le impostazioni sull'orario in DiskStation, andare su Pannello di controllo > Opzioni regionali > Ora per sincronizzare l'ora con il server NTP.

Attività di backup di hicloud S3:

  • Quando si creano attività di backup con il server hicloud S3 come destinazione di backup di rete, sono disponibili varie opzioni (ad es. codifica trasferimento). Alcune opzioni, come codifica trasferimento, potrebbero ridurre la velocità delle attività di backup.
  • I file di backup sono archiviati nel Bucket selezionato sul server hicloud S3 e un Valore Set KEY inizia con un nome directory a propria scelta e termina con il percorso completo del file.
  • La struttura di dati di hicloud S3 include un paio di Bucket e KEY senza alcuna struttura a cartella. Il backup di cartella vuota non è supportato.
  • Se un file sottoposto a backup non è stato modificato, non verrà sottoposto di nuovo a backup sul server hicloud S3 nello stesso Bucket.
  • Ogni Bucket su hicloud S3 deve avere un nome univoco. Inoltre, il nome deve:
    • Essere composto da 3 a 63 caratteri.
    • Contenere soltanto lettere minuscole, numeri, punti (.) e trattini (-).
    • Iniziare e finire con un numero o una lettera e non può finire con un trattino (-), contenere due punti adiacenti o contenere trattini dopo punti (ad es., "my-.bucket.com" e "my.-bucket" non sono validi).
  • La lunghezza massima di una hicloud S3 KEY è 255 caratteri. Per assicurare il successo di un backup su hicloud S3, la lunghezza massima di percorso è 255 meno la lunghezza del nome della cartella di destinazione specificato durante la creazione dell'attività.
  • Quando si trasferisce un file sarà diviso in parti. La dimensione della parte può essere impostata dall'opzione Dimensione parte Multipart Upload in impostazioni attività con 512MB come dimensione predefinita. Se l'ambiente di rete non è stabile, si raccomanda di impostare un numero più piccolo.
  • Hicloud supporta solo un massimo di 1000 parti per trasferimento. Quindi, le dimensioni totali dei file non devono superare 1000 moltiplicato per le dimensioni della parte. Ad esempio, se le dimensioni della parte di caricamento multiparte sono impostate su 8MB, non sarà possibile calcolare singoli file con dimensioni superiori a 8000MB.
  • Crittografia cartella condivisa cifrata non supportata.
  • I nomi dei file non possono contenere caratteri non UTF-8 o di controllo.

Attività di backup SFR NAS BACKUP:

  • Quando si creano attività di backup con il server SFR NAS BACKUP come destinazione di backup di rete, sono disponibili varie opzioni, come la codifica del trasferimento e altre. Alcune opzioni, come codifica trasferimento, potrebbero ridurre la velocità delle attività di backup.
  • I file di backup sono archiviati nel Bucket selezionato sul server SFR NAS BACKUP, e un Valore Set Key inizia con un nome directory a propria scelta e termina con il percorso completo del file.
  • La struttura di dati di SFR NAS BACKUP include un paio di Bucket e Key senza alcuna struttura a cartella. Il backup di cartella vuota non è supportato.
  • Se un file sottoposto a backup non è stato modificato, non verrà sottoposto di nuovo a backup sul server SFR NAS BACKUP nello stesso Bucket.
  • Ogni Bucket sul server SFR NAS BACKUP deve avere un nome univoco. Inoltre, il nome deve:
    • Essere composto da 3 a 63 caratteri.
    • Contenere soltanto lettere minuscole, numeri, punti (.) e trattini (-).
    • Iniziare e finire con un numero o una lettera e non può finire con un trattino (-), contenere due punti adiacenti o contenere trattini dopo punti (ad es., "my-.bucket.com" e "my.-bucket" non sono validi).
  • La lunghezza massima di una SFR NAS BACKUP KEY è di 1024 caratteri. Per assicurare il successo di un backup su SFR NAS BACKUP, la lunghezza massima di percorso è 1024 meno la lunghezza del nome della cartella di destinazione specificato durante la creazione dell'attività.
  • Quando si trasferisce un file sarà diviso in parti. La dimensione della parte può essere impostata dall'opzione Dimensione parte Multipart Upload in impostazioni attività con 64MB come dimensione predefinita. Se l'ambiente di rete non è stabile, si raccomanda di impostare un numero più piccolo.
  • I nomi dei file non possono contenere caratteri non UTF-8 o di controllo.

Altre attività di backup compatibili con S3:

  • Per utilizzare altri cloud compatibili con S3, inserire l'indirizzo IP/dominio del cloud compatibile S3 desiderato nel campo Server S3 quando viene richiesto.
  • In caso di problemi di utilizzo, consultare la documentazione pertinente o contattare il provider di servizi cloud compatibili con S3.

Backup LUN*

È possibile utilizzare Backup LUN locale per eseguire il backup delle LUN iSCSI in DiskStation nelle cartelle condivise locali. È possibile utilizzare anche Backup LUN di rete (server Synology) per eseguire il backup su un altro DiskStation.

Per eseguire il backup di un iSCSI LUN:

  1. Fare clic su +, nell'angolo in basso a sinistra, e selezionare Attività di backup LUN.
  2. Quando viene chiesto di scegliere il tipo di destinazione, scegliere una delle seguenti:
    • Backup LUN locale: Scegliere questa opzione per eseguire il backup dei LUN da DiskStation a una cartella condivisa locale o dispositivo di archiviazione esterno connesso alla porta USB o eSATA della DiskStation.
    • Backup LUN di rete (server Synology): Scegliere questa opzione per eseguire il backup dei LUN da DiskStation a un altra DiskStation.
  3. Quando chiesto, specificare la destinazione di backup e l'origine LUN, quindi impostare la programmazione del backup.
  4. Seguire la procedura guidata per completare la configurazione.

Una volta creata l'attività di backup, DiskStation esegue l'attività di backup nel momento stabilito e salva il backup dei LUN sulla destinazione. Il backup dei LUN è utilizzato esclusivamente per il backup. Per accedere al backup dei LUN, vedere Per ripristinare il backup dei LUN per ulteriori informazioni.

Nota:

  • Il backup di livelli di blocco LUN e file regolari sarà effettuato come file immagine alla destinazione. Se ripristinato, tutti i livelli di blocco LUN saranno modificati a file LUN.
  • Ciascuna attività di backup può includere un solo LUN come origine di backup.
  • Non è possibile effettuare i backup di LUN su un disco esterno nel formato FAT, FAT32, HFS, HFS Plus o NTFS.
  • Prima di eseguire il Backup LUN di rete, accedere alla DiskStation di destinazione per assicurarsi che il servizio backup di rete sia abilitato in Pannello di controllo > Servizio di backup.
  • Se la DiskStation si connette alla rete tramite un router, assicurarsi di aver specificato una regola di invio porta per il servizio iSCSI (porta TCP 3260) sul router prima di eseguire il Backup LUN di rete.

Informazioni su Backup completo e Backup incrementale:

La prima volta DiskStation esegue un backup completo dei LUN selezionati, mentre successivamente esegue dei backup incrementali. Tuttavia, se il nome della cartella condivisa o il nome della directory della destinazione del backup viene cambiato, DiskStation esegue nuovamente un backup completo per l'attività.

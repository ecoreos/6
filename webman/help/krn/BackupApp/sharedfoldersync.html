공유 폴더 동기화

공유 폴더 동기화 서비스는 사용자가 LAN이나 인터넷을 통해 하나의 Synology 제품(소스)에서 다른 Synology 제품(대상)으로 공유 폴더를 동기화할 수 있게 해줍니다. 이것은 공유된 폴더를 기반으로 작동합니다. 동기화할 때, 대상 Synology 제품에 동일한 이름을 가진 폴더(예: SharedFolder)가 이미 있으면, 대상에 있는 폴더 이름이 "SharedFolder_1"로 변경됩니다. 그런 이름의 폴더가 없으면 해당 대상 또는 첫 번째 사용 가능한 대상에 폴더가 생성됩니다. 대상 DiskStation에서 이 폴더를 액세스할 수 있는 권한은 기본적으로 설정됩니다. 따라서 이 폴더에 어떤 권한도 지정할 필요가 없습니다.

처음에는 공유 폴더 동기화가 선택한 공유 폴더에 대해 전체 동기화를 수행합니다. 그러면 변경된 파일만 동기화됩니다. 소스에서 제거된 파일은 대상에서도 제거됩니다.

참고:

  • 공유 폴더 동기화는 일방향 동기화 솔루션입니다. 즉, 소스의 파일이 대상으로 동기화되지만 그 반대로는 동기화되지 않습니다. 양방향 동기화 솔루션이 필요하면 Cloud Station을 사용하십시오.

서비스 활성화

공유 폴더 동기화 활성화하기:

대상 DiskStation에서 서비스를 활성화해야 합니다. 이렇게 하려면 백업 및 복제 > 공유 폴더 동기화로 이동합니다. 서버 탭에서 공유 폴더 동기화 서비스 활성화를 선택하여 활성화합니다.

공유 폴더를 소스에서 대상 Synology 제품으로 연결하기:

공유 폴더를 소스에서 여러 대상 서버로 동기화할 수 있습니다. 또한 DSM 4.1 이후부터, 공유 폴더 동기화는 여러 클라이언트가 공유 폴더를 하나의 DiskStation으로 동기화할 수 있는 다중 소스 공유 폴더 동기화 기능을 제공합니다. 여러 클라이언트가 동일한 공유 폴더를 하나의 DiskStation에 동시에 동기화하면 충돌이 발생합니다. 먼저 공유 폴더의 연결을 해제해야 다른 클라이언트가 이 폴더를 대상에 동기화할 수 있습니다. 공유 폴더의 연결을 해제하려면 대상 측에서 공유 폴더 동기화 > 서버로 이동하고, 연결 목록을 클릭하고, 연결을 선택한 다음 링크 끊기를 클릭합니다.

이제 소스에서 대상 DiskStation으로 공유 폴더를 동기화할 수 있습니다. 아래 섹션의 지침을 따라 작업을 계속합니다.

동기화 작업 관리

공유 폴더 동기화 마법사를 통해 쉽게 동기화 작업을 생성할 수 있습니다.

동기화 작업 생성하기:

  1. 백업 및 복제 > 공유 폴더 동기화 > 클라이언트로 이동합니다.
  2. 생성을 클릭합니다.
  3. 작업에 대한 이름을 입력합니다.
  4. 대상에 동기화할 공유된 폴더를 선택합니다. 아래 선택 상자를 체크 표시하여 확정하고 다음을 클릭하여 작업을 계속합니다.
  5. 대상과 다음 동기화 옵션을 설정하십시오.
    • 전송 암호화 활성화: 전송 중 데이터를 암호화합니다. 이 기능은 보다 안전한 보안을 제공하지만, 암호화하지 않고 전송하면 성능이 향상됩니다. 필요에 따라 이 옵션을 선택할 수 있습니다.
    • 전송 압축 활성화: 전송 중 데이터를 압축합니다. 이 기능은 네트워크 대역폭을 절약할 수 있지만 CPU 부하가 커집니다.
    • 블록 수준 동기화 활성화: 전체 파일이 아니라 차이가 있는 데이터만 전송합니다. 이 기능은 네트워크 대역폭을 절약할 수 있지만 CPU 부하가 커집니다.
  6. 작업의 스케줄을 설정합니다.
    • 동기화를 [number]시간마다 실행 또는 동기화를 [number]분마다 실행: 특정한 시간 간격을 두고 동기화 작업을 실행합니다. 시간은 자정부터 카운트를 시작합니다. 또한 간격은 24 시간을 초과하지 않습니다.
    • 다음 시간마다 자동으로 동기화 실행(hh:mm): 매일 특정 시간에 동기화 작업을 실행합니다.
    • 수동으로 동기화 실행: 수동으로 수행할 때만 동기화를 실행합니다.
    • 수정 시 동기화 실행: 소스 공유 폴더에서 변경 내용이 발생하면 즉시 동기화 작업을 실행합니다.
  7. 적용을 클릭합니다. 이제 작업 목록에 동기화 작업이 표시됩니다. 이 작업은 지정된 일정에 따라 수행됩니다.

동기화 작업을 관리하려면 다음과 같이 하십시오.

  1. 작업 목록에서 작업을 선택하십시오.
  2. 다음 중 하나를 수행하십시오.
    • 편집을 클릭하여 작업을 편집하십시오.
    • 삭제를 클릭하여 작업을 삭제하십시오.
    • 동기화 작업이 진행되지 않는 경우 지금 동기화를 클릭하여 작업을 즉시 실행하십시오.
    • 동기화 작업이 실행되는 중이면 취소를 클릭하여 진행 작업을 중지하십시오.

      • 참고:

      동기화 작업 일정을 수정 시 동기화 실행으로 설정한 경우 취소를 클릭하면 진행 중인 동기화 작업이 중지됩니다. 동기화 작업에서 모니터링하는 공유 폴더의 콘텐츠가 변경된 경우 공유 폴더 동기화에서 작업을 다시 시작합니다.

    • 전체 동기화를 클릭하여 모든 데이터를 동기화하십시오. 공유 폴더 동기화에서 처음에는 전체 동기화를 실행한 다음 그 이후에는 수정된 부분만 동기화합니다. 이 단추를 클릭하면 전체 데이터를 수동으로 다시 동기화할 수 있습니다.

추가 정보

  1. 작업 이름은 1-32자의 Unicode 문자이며 다음 문자는 포함시키지 않아야 합니다:
    ! " # $ % & ' ( ) * + , / : ; <= > ? @ [ ] \ ^ ` { } | ~
  2. 공유 폴더를 ext4에서 ext3 파일 시스템으로 동기화하면 작업에 32,000개 이상의 폴더가 포함될 때 에러가 발생할 수 있습니다.
  3. 이전에 대상 DiskStation에 있는 공유 폴더가 다른 DiskStation와(과) 동기화된 경우, 공유 폴더 동기화는 작업에 대해 전체 동기화를 수행합니다.
  4. 공유 폴더 동기화 작업을 수행하려면 대상 DiskStation의 관리 권한이 필요합니다. admin 계정 패스워드가 있거나 관리 그룹 사용자로 등록되어 있어야 합니다.
  5. 인터넷을 통해 DiskStation을(를) 동기화하는 경우, 외부 IP 주소나 호스트 이름을 사용하여 서버 이름을 입력할 수 있습니다. 그러나, 호스트 이름을 입력할 때는 해당 이름이 FQDN(Fully Qualified Domain Name) 형식(예. "www.synology.com")이어야 합니다.
  6. 포트 전달의 경우, 공유 폴더 동기화는 암호화된 전송의 경우 포트 22, 암호화되지 않은 전송의 경우 포트 873을 사용합니다.
  7. 공유 폴더 동기화에서는 외부 저장 장치를 지원하지 않습니다.
  8. 암호화된 공유 폴더를 사용하려면 대상에서 암호화된 키로 수동으로 마운트해야 합니다.
  9. 선택한 공유 폴더에 변경 내용이 없을 경우, 예약된 공유 폴더 동기화가 지정된 시간에 실행되지 않습니다.
  10. 최상의 성능을 위해서는 각 동기화 작업에 200개 미만의 공유된 폴더를 포함하는 것이 좋습니다.
  11. homes 공유 폴더가 소스 DiskStation에서 대상 DiskStation으로 동기화되는 경우, 대상 DiskStation에서 사용자 홈 기능이 비활성화됩니다.
  12. Intel CPU가 아닌 모델의 경우, 공유 폴더 동기화를 사용하여 최대 100,000개의 공유 폴더(하위 폴더 포함)를 동기화할 수 있습니다.
  13. 공유 폴더 동기화는 대상 파일 시스템이 ext4일 때 Windows ACL 설정을 백업할 수 있습니다.
  14. DiskStation와(과) 동일한 이름을 공유하는 대상 서버에 동기화하면 공유 폴더 동기화가 실패합니다.
  15. 소스의 고급 공유 폴더 권한은 대상으로 동기화되지 않습니다. 뿐만 아니라 관리자 그룹 사용자에 대한 읽기 전용 규칙이 대상에서 생성됩니다.

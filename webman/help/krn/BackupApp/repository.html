백업 대상

백업 작업을 생성하기 전에 백업 및 복제의 백업 대상 탭 아래에서, 로컬, 네트워크 또는 클라우드 백업 대상을 정의할 수 있습니다.

로컬 백업 대상 생성하기:

DiskStation에서 로컬 공유 폴더, 볼륨 또는 외부 USB/eSATA 드라이브로 데이터를 백업하려면 로컬 백업 대상이 필요합니다. 백업 대상은 소스와 달라야 하며, 대상의 용량은 백업할 데이터의 총 크기보다 커야 합니다.

  1. 생성을 클릭하고, 로컬 백업 대상을 선택한 후 다음을 클릭합니다.
  2. 이 백업 대상에 이름을 지정한 다음, 백업 데이터를 저장할 대상을 선택하고 가장 최신 상태의 백업 사본을 보관할지 또는 모든 백업 버전을 보관할지 선택합니다.

참고:

  • 백업 작업을 진행하는 도중 대상에 있는 데이터를 삭제한 경우 다음에 작업을 수행할 때 전체 백업이 실행됩니다.
  • 새 외부 드라이브는 백업 대상으로 이용하기 전에 포맷해야 할 수 있습니다.
  • 백업 작업을 생성하기 전에, 외부 드라이브를 적절하게 DiskStation에 연결했는지 확인합니다. 제어판 > 외부 장치로 이동하여 USB/eSATA 드라이브 상태를 확인할 수 있습니다. 외부 드라이브가 초기화 및 포맷 중이거나 오작동하는 경우 이를 백업 대상으로 선택할 수 없습니다.
  • 원격 마운트된 공유는 백업 소스 또는 대상으로 선택할 수 없습니다.
  • 암호화된 공유 폴더의 마운트가 해제되면 해당 폴더를 백업 대상으로 선택할 수 없습니다.
  • 이미지 섬네일은 대상의 파일 시스템이 NTFS/FAT인 경우 백업되지 않습니다.

네트워크 백업 대상 생성하기:

네트워크에서 데이터를 DiskStation에서 다른 DiskStation 또는 rsync 호환 가능 서버로 백업할 경우 네트워크 백업 대상이 필요합니다. 네트워크 백업 대상을 사용하여 백업 작업을 생성할 때 전송 암호화, 압축, 블록 수준 백업 등과 같은 다양한 옵션을 사용할 수 있습니다. 전송 암호화 같은 일부 옵션의 경우 백업 작업 속도가 감소될 수 있음에 유의하십시오.

  1. 네트워크 백업 대상을 생성하기 전에 먼저 대상 서버에 로그인하고 백업 및 복제 > 백업 서비스로 이동한 다음 네트워크 백업 서비스 및/또는 네트워크 백업 볼륨 대상을 활성화합니다.
  2. DiskStation에 돌아와, 백업 및 복제 > 백업 대상으로 이동합니다. 생성을 클릭하고, 네트워크 백업 대상을 선택한 후 마법사를 따라 완료합니다.
  3. 네트워크 백업 대상(DiskStation 서버)을 통해, 공유 폴더에 백업 데이터를 저장하여 최신 백업 사본을 보유하거나 볼륨에 백업 데이터를 저장하여 복수 버전의 백업 사본을 보유할 수 있습니다. 네트워크 백업 대상(rsync 호환 서버)을 사용하여 최신 상태의 백업 사본만 보유할 수 있습니다.

참고:

  • 네트워크 백업 대상으로 rsync 호환 가능 서버를 사용하여 백업 작업을 생성할 때 다음과 같은 모드를 사용할 수 있습니다: rsync 데몬(daemon) 모드는 백업 모듈 옆에 있는 드롭다운 메뉴에서 백업 모듈(rsync 호환 서버의 공유 폴더)을 선택한 경우에 사용됩니다. 원격 셸(remote shell) 모드는 백업 모듈에 절대 경로("/"로 시작)를 입력한 경우에만 활성화됩니다.
  • Linux 사용자의 경우 admin, root 및 rsync 계정만 백업 작업을 수행할 수 있습니다.
  • 대상 서버의 호스트 이름을 사용하여 인터넷에서 이 서버에 액세스하는 경우, 호스트 이름을 FQDN(정규화된 도메인 이름) 형식(예: abc.bcd.com)으로 입력해야 합니다.
  • DiskStation 장치 간에 백업할 경우 대상에 있는 계정에 대한 사용자 이름과 패스워드를 입력하라는 메시지가 표시됩니다. 특히, 다른 DiskStation의 볼륨에 백업하려면 administrators 그룹에 속한 계정을 지정해야 합니다. 다른 DiskStation의 공유 폴더에 백업할 때 이 폴더의 액세스 권한 설정에 따라 수행할 수 있습니다. 접근 권한이 있는 특정 사용자 계정에 대한 공유 폴더만 표시됩니다. 대상에서 원하는 공유 폴더를 찾을 수 없으면 접근 권한을 확인하십시오.
  • 백업 암호화는 대상 서버가 암호화된 네트워크 백업을 지원하고 인증 패스워드가 비어 있지 않은 경우에만 지원됩니다.
  • 원격 셸(remote shell) 모드는 암호화된 백업만 지원합니다.
  • rsync 호환 서버에 백업하는 경우 rsync 백업 작업이 생성된 후에는 rsync 암호화 모드를 변경할 수 없게 됩니다. 구성을 변경하려면 새 rsync 작업을 생성하십시오.
  • 파일을 rsync 클라이언트를 통해 DiskStation에 백업할 경우 파일 이름은 UTF-8 인코딩 형식이어야 합니다.
  • Rsync 호환 클라이언트를 통해 파일을 DiskStation에 백업하는 경우 DiskStation의 admin, root 및 rsync 계정으로만 작업을 수행할 수 있습니다.
  • DiskStation에서 rsync 호환 서버로 파일을 백업하는 경우 올바른 rsync 백업 계정 이름과 패스워드를 입력해야 합니다. 또한 쓰기 권한이 있는 올바른 백업 모듈을 선택해야 합니다.

Amazon S3 백업 대상 생성하기:

데이터를 DiskStation에서 Amazon S3 서버로 백업할 경우 Amazon S3 백업 대상이 필요합니다.

  1. 생성을 클릭하고 공용 클라우드 백업 대상, Amazon S3 백업 대상을 차례로 선택한 후 다음을 클릭합니다.
  2. 이 백업 대상에 이름을 지정한 다음 로그인 자격 증명을 입력하고 나서 백업 작업을 설정하는 데 필요한 모든 필드를 채웁니다.

참고:

  • 네트워크 백업 대상으로 Amazon S3 서버를 사용하여 백업 작업을 생성할 때, 전송 암호화, Amazon S3 RRS(감소된 중복 저장소)에 백업 등과 같은 다양한 옵션을 사용할 수 있습니다. 전송 암호화 같은 일부 옵션의 경우 백업 작업 속도가 감소될 수 있음에 유의하십시오.
  • 백업 파일은 Amazon S3 서버에서 선택한 Bucket에 저장되고, 설정된 Key 값은 사용자가 선택한 디렉토리 이름으로 시작하여 이 파일의 전체 파일 경로로 끝납니다.
  • Amazon S3 데이터 구조는 폴더 구조 없이 Bucket과 KEY 쌍으로 이루어집니다. 빈 폴더 백업은 지원되지 않습니다.
  • 백업된 파일이 수정되지 않은 경우, 동일 Bucket에 있는 Amazon S3 서버로 다시 백업되지 않습니다.
  • Amazon S3의 각 Bucket에는 고유한 이름을 지정해야 합니다. 또한, 이름은 다음과 같아야 합니다:
    • 3-63자 사이의 문자이어야 함.
    • 소문자, 숫자, 마침표(.), 대시(-)만 포함해야 합니다.
    • 숫자 또는 문자로 시작 및 끝내고, 대시(-)로 끝나지 말고, 마침표를 연달아 두 개 사용하지 말고, 마침표 옆에 대시가 오지 않아야 합니다(예: "my-.bucket.com" 및 "my.-bucket"은 잘못된 이름).
  • Amazon S3 KEY의 최대 길이는 1024자입니다. Amazon S3으로의 백업을 성공하기 위해서는 최대 경로 길이가 1024에서 작업 생성 과정에서 지정한 대상 폴더 이름을 뺀 값이어야 합니다.
  • Amazon S3 RRS(감소된 중복 저장소) 지원을 활성화할 경우, 이 옵션을 활성화한 후 수정한 데이터만 RRS 저장소로 백업됩니다. 즉, Amazon S3의 표준 저장소로 이미 백업된 데이터는 다시 RRS로 백업되지 않습니다.
  • Amazon S3에서 Amazon Glacier로 백업된 파일 이동은 지원되지 않습니다. AWS 관리 콘솔을 통해 Amazon S3에서 Amazon Glacier로 파일이 이동되면 관련 백업 작업이 실패할 수 있습니다.
  • SSL를 사용한 가상 호스팅 스타일 Bucket으로의 파일 백업은 지원되지 않습니다.
  • 파일을 전송할 때 파일은 부분들로 나뉩니다. 부분 크기는 기본 크기가 512MB로 작업 설정에서 멀티파트 업로드 부분 크기 옵션을 사용하여 설정할 수 있습니다. 네트워크 환경이 불안정할 경우 이 옵션을 작게 설정하는 것이 권장됩니다.
  • 업로드 당 최대 부분 수는 10000입니다. 최대 파일 크기는 멀티파트 업로드 부분 크기 곱하기 10000입니다.
  • 파일 이름은 비 UTF-8 또는 제어 문자를 포함하지 않아야 합니다.

Microsoft Azure 백업 대상 생성하기:

데이터를 DiskStation에서 Microsoft Azure blob 서버로 백업할 경우 Microsoft Azure 백업 대상이 필요합니다.

  1. 생성을 클릭하고 공용 클라우드 백업 대상, Microsoft Azure 백업 대상을 차례로 선택한 후 다음을 클릭합니다.
  2. 이 백업 대상에 이름을 지정한 다음 로그인 자격 증명을 입력하고 나서 백업 작업을 설정하는 데 필요한 모든 필드를 채웁니다.

참고:

  • 네트워크 백업 대상으로 Microsoft Azure 서버에 백업 작업을 생성할 때 다양한 옵션을 사용할 수 있습니다(예: 전송 암호화). 전송 암호화 같은 일부 옵션의 경우 백업 작업 속도가 감소될 수 있음에 유의하십시오.
  • 백업 파일은 Microsoft Azure 서버에서 선택한 Container에 저장되고, 설정된 Blob 이름은 사용자가 선택한 디렉토리 이름으로 시작하여 이 파일의 전체 파일 경로로 끝납니다.
  • Microsoft Azure 데이터 구조는 폴더 구조 없이 Container 및 Blob 쌍으로 이루어집니다. 빈 폴더 백업은 지원되지 않습니다.
  • 백업된 파일이 수정되지 않은 경우, 동일 Container에 있는 Microsoft Azure 서버로 다시 백업되지 않습니다.
  • Microsoft Azure의 각 Container에는 고유한 이름을 지정해야 합니다. 또한, 이름은 다음과 같아야 합니다:
    • 3-63자 사이의 문자이어야 함.
    • 소문자, 숫자, 대시(-)만 포함합니다.
    • 숫자 또는 문자로 시작 및 끝내고, 두 개의 인접한 대시를 포함할 수 없습니다.
  • Microsoft Azure Blob 이름의 최대 길이는 1024자입니다. Microsoft Azure로의 백업을 성공하기 위해서는 최대 경로 길이가 1024에서 작업 생성 과정에서 지정한 대상 폴더 이름을 뺀 값이어야 합니다.
  • 파일 이름은 비 UTF-8 또는 제어 문자를 포함하지 않아야 합니다.
  • 인증 실패 경고 메시지는 잘못된 인증 정보, 네트워크 연결 문제 또는 DiskStation와(과) 원격 서버의 시간이 일치하지 않을 때 발생할 수 있습니다. DiskStation에서 시간 설정을 확인하려면 제어판 > 지역 옵션 > 시간으로 이동하여 NTP 서버와 시간을 동기화합니다.

hicloud S3 백업 대상 생성하기:

데이터를 DiskStation에서 hicloud S3 서버로 백업할 경우 hicloud S3 백업 대상이 필요합니다.

  1. 생성을 클릭하고 공용 클라우드 백업 대상, hicloud S3 백업 대상을 차례로 선택합니다.
  2. 이 백업 대상에 이름을 지정한 다음 로그인 자격 증명을 입력하고 나서 백업 작업을 설정하는 데 필요한 모든 필드를 채웁니다.

참고:

  • 네트워크 백업 대상으로 hicloud S3 서버에 백업 작업을 생성할 때 다양한 옵션을 사용할 수 있습니다(예: 전송 암호화). 전송 암호화 같은 일부 옵션의 경우 백업 작업 속도가 감소될 수 있음에 유의하십시오.
  • 백업 파일은 hicloud S3 서버에서 선택한 Bucket에 저장되고, Set KEY 값은 사용자가 선택한 디렉토리 이름으로 시작하여 이 파일의 전체 파일 경로로 끝납니다.
  • hicloud S3 데이터 구조는 폴더 구조 없이 Bucket과 KEY 쌍으로 이루어집니다. 빈 폴더 백업은 지원되지 않습니다.
  • 백업된 파일이 수정되지 않은 경우, 동일 Bucket에 있는 hicloud S3 서버로 다시 백업되지 않습니다.
  • hicloud S3의 각 Bucket에는 고유한 이름을 지정해야 합니다. 또한, 이름은 다음과 같아야 합니다:
    • 3-63자 사이의 문자이어야 함.
    • 소문자, 숫자, 마침표(.), 대시(-)만 포함해야 합니다.
    • 숫자 또는 문자로 시작 및 끝내고, 대시(-)로 끝나지 말고, 마침표를 연달아 두 개 사용하지 말고, 마침표 옆에 대시가 오지 않아야 합니다(예: "my-.bucket.com" 및 "my.-bucket"은 잘못된 이름).
  • hicloud S3 KEY의 최대 길이는 255자입니다. hicloud S3으로의 백업을 성공하기 위해서는 최대 경로 길이가 255에서 작업 생성 과정에서 지정한 대상 폴더 이름을 뺀 값이어야 합니다.
  • 파일을 전송할 때 파일은 부분들로 나뉩니다. 부분 크기는 기본 크기가 512MB로 작업 설정에서 멀티파트 업로드 부분 크기 옵션을 사용하여 설정할 수 있습니다. 네트워크 환경이 불안정할 경우 보다 작게 설정하는 것이 권장됩니다.
  • Hicloud는 전송당 최대 1000개의 파트만 지원합니다. 이는 총 파일 크기가 1000 곱하기 파트 크기를 초과할 수 없음을 의미합니다. 예를 들어, 멀티파트 업로드 부분 크기가 8MB로 설정되면, 8000MB를 넘는 단일 파일은 업로드할 수 없습니다.
  • 암호화된 공유 폴더 백업은 지원되지 않습니다.
  • 파일 이름은 비 UTF-8 또는 제어 문자를 포함하지 않아야 합니다.

SFR NAS BACKUP 백업 대상 생성하기:

데이터를 DiskStation에서 SFR NAS BACKUP 서버로 백업할 경우 SFR NAS BACKUP 백업 대상이 필요합니다.

  1. 생성을 클릭하고 공용 클라우드 백업 대상, SFR NAS BACKUP 백업 대상을 차례로 선택한 후 다음을 클릭합니다.
  2. 이 백업 대상에 이름을 지정한 다음 로그인 자격 증명을 입력하고 나서 백업 작업을 설정하는 데 필요한 모든 필드를 채웁니다.

참고:

  • 네트워크 백업 대상으로 SFR NAS BACKUP 서버에 백업 작업을 생성할 때 전송 암호화 등과 같은 다양한 옵션을 사용할 수 있습니다. 전송 암호화 같은 일부 옵션의 경우 백업 작업 속도가 감소될 수 있음에 유의하십시오.
  • 백업 파일은 SFR NAS BACKUP 서버에서 선택한 Bucket에 저장되고, 설정된 Key 값은 사용자가 선택한 디렉토리 이름으로 시작하여 이 파일의 전체 파일 경로로 끝납니다.
  • SFR NAS BACKUP 데이터 구조는 폴더 구조 없이 Bucket과 Key 쌍으로 이루어집니다. 빈 폴더 백업은 지원되지 않습니다.
  • 백업된 파일이 수정되지 않은 경우, 동일 Bucket에 있는 SFR NAS BACKUP 서버로 다시 백업되지 않습니다.
  • SFR NAS BACKUP의 각 Bucket에는 고유한 이름을 지정해야 합니다. 또한, 이름은 다음과 같아야 합니다:
    • 3-63자 사이의 문자이어야 함.
    • 소문자, 숫자, 마침표(.), 대시(-)만 포함해야 합니다.
    • 숫자 또는 문자로 시작 및 끝내고, 대시(-)로 끝나지 말고, 마침표를 연달아 두 개 사용하지 말고, 마침표 옆에 대시가 오지 않아야 합니다(예: "my-.bucket.com" 및 "my.-bucket"은 잘못된 이름).
  • SFR NAS BACKUP KEY의 최대 길이는 1024자입니다. SFR NAS BACKUP으로의 백업을 성공하기 위해서는 최대 경로 길이가 1024에서 작업 생성 과정에서 지정한 대상 폴더 이름을 뺀 값이어야 합니다.
  • 파일을 전송할 때 파일은 부분들로 나뉩니다. 부분 크기는 기본 크기가 64MB로 작업 설정에서 멀티파트 업로드 부분 크기 옵션을 사용하여 설정할 수 있습니다. 네트워크 환경이 불안정할 경우 보다 작게 설정하는 것이 권장됩니다.
  • 파일 이름은 비 UTF-8 또는 제어 문자를 포함하지 않아야 합니다.

백업 대상 관리하기:

기존 백업 대상을 선택하고 아래 단추 중 하나를 클릭합니다:

  • 편집: 선택한 백업 대상의 설정을 수정합니다.
  • 삭제: 선택한 백업 대상을 삭제합니다.

공유 폴더에 백업

이 유형의 백업 대상은 최신 백업 사본만 보관합니다. 기본적으로, 초기 백업이 완료된 후 다음 작업을 수행할 때 소스에서 제거된 파일이 대상에서 제거됩니다. 그러나 파일이 소스에서 삭제된 후에도 그대로 대상에 남도록 하려면, 백업 작업을 생성할 때 대상에서 백업된 파일 보존 옵션을 활성화하십시오.

볼륨에 백업

이 유형의 백업 대상은 DiskStation 장치(외부 저장소 장치 제외)에 있는 로컬 또는 네트워크 볼륨이여 합니다. 이는 특정 시간의 어떤 폴더든 손쉽게 복원할 수 있도록 여러 버전의 백업 사본을 보관합니다.

백업 작업의 저장된 데이터 삭제하기:

  1. 백업 대상 탭으로 이동하고 삭제할 작업이 포함된 백업 대상을 확장합니다.
  2. 선택한 백업 작업에서 삭제를 클릭한 다음 대화 상자에서 삭제를 선택합니다.

백업 버전의 저장된 데이터 삭제하기:

  1. 백업 대상 탭으로 이동하고 삭제할 작업의 버전이 포함된 백업 대상을 확장합니다.
  2. 선택한 백업 작업에서 백업 버전 정보를 클릭합니다.
  3. 삭제하려는 백업 버전을 선택하고 삭제 아이콘을 클릭합니다.

백업 회전 동안 제거되지 않도록 방지하기 위해 백업 버전을 잠그려면:

  1. 백업 대상 탭으로 이동하고 잠그려는 작업의 버전이 포함된 백업 대상을 확장합니다.
  2. 선택한 백업 작업에서 백업 버전 정보를 클릭합니다.
  3. 잠그려는 백업 버전을 선택하고 잠금 아이콘을 클릭합니다.

백업 대상의 권한 관리하기:

  1. 로컬 볼륨과 연관된 백업 대상을 선택하고 편집을 클릭합니다.
  2. 허용 목록, 추가를 차례로 클릭하여 사용자 또는 그룹을 목록에 추가합니다.

소유자가 있는 백업 대상 설정하기:

  1. 로컬 볼륨과 연관된 백업 대상을 선택하고, 선택한 백업 작업에서 속성 아이콘을 클릭합니다.
  2. 백업 권한 아래의 드롭다운 메뉴에서 소유자를 선택합니다.

DiskStation의 사용자 또는 그룹이 백업 작업의 저장된 데이터를 읽고 복원할 수 있게 하기:

  1. 로컬 볼륨과 연관된 백업 대상을 선택하고, 선택한 백업 작업에서 속성 아이콘을 클릭합니다.
  2. 허용 목록, 추가를 차례로 클릭하여 사용자 또는 그룹을 목록에 추가합니다.

참고:

  • 백업 작업의 기본 소유자는 작업을 생성한 사용자입니다.
  • 백업 작업 소유자만 작업을 수행할 수 있습니다. 그러나 DSM 5.1 이전에 생성한 백업 작업은 이런 제한이 없으므로 administrators 그룹에 속하는 사용자는 누구나 이 백업 작업을 수행할 수 있습니다.
  • 작업 소유자와 administrators 그룹에 속하는 사용자는 백업 작업의 저장된 데이터를 삭제할 수 있습니다.
  • 작업 소유자와 administrators 그룹에 속하는 사용자, 허용 목록에 있는 사용자 모두 백업 작업의 저장된 데이터를 복원할 수 있습니다.
  • administrators 그룹에 속하는 사용자만 액세스 권한을 사용하여 로컬 백업 대상을 설정하고, 소유자 및 허용 목록을 사용하여 로컬 백업 대상에 백업 작업을 설정할 수 있습니다.
  • 백업 작업의 저장된 데이터는 작업 소유자의 저장소 할당량을 사용합니다.
  • 볼륨에 백업 대상이 없으면 administrators 그룹에 속하는 사용자만 이 볼륨에 첫 번째 백업 대상을 생성할 수 있습니다.
  • administrators 그룹에 속하는 사용자만 전체 백업 대상의 저장된 데이터를 삭제할 수 있습니다.
  • 각 백업은 시스템 구성과 파일 메타데이터를 포함합니다.
  • 암호화된 공유 폴더 백업은 지원되지 않습니다.
  • DSM 5.1 이하에서는 버전 잠금이 지원되지 않습니다.

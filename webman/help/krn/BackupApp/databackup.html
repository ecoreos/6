백업

백업 및 복원을 사용하면 DiskStation에서 수동으로 또는 일정에 따라 다양한 종류의 데이터(시스템 구성, 공유 폴더 및 응용 프로그램/패키지)를 백업할 수 있습니다. 로컬 공유 폴더, 원격 서버 및 공개 클라우드(Amazon S3, Microsoft Azure, hicloud S3 및 SFR NAS BACKUP)에 백업 데이터를 저장할 수 있습니다. 백업 및 복원을 사용하면 각 백업 작업은 여러 백업 버전을 유지하고 Smart Recycle 정책을 사용해 이러한 버전을 회전할 수 있습니다.

데이터 백업

백업 및 복원은 다양한 백업 대상 옵션 및 데이터 백업 작업을 설정하는 방법을 제공합니다.

데이터 백업 작업 생성하기:

  1. 왼쪽 아래 구석에서 +를 클릭하고 데이터 백업 작업을 선택하여 백업 마법사를 실행합니다.
  2. 다음과 같은 원하는 백업 대상 유형을 선택합니다.
    • 로컬/원격 공유 폴더 또는 외부 저장 장치
    • 원격 rsync 서버
    • 공개 클라우드(예: Amazon S3 및 Microsoft Azure)
  3. 원하는 백업 작업 유형 설정하기:
    • 완전히 새로운 작업을 생성하려면 백업 작업 생성을 선택합니다.
    • 대상에서 기존 백업 데이터를 재사용하려면 기존 작업으로 재링크를 선택합니다. 재링크하면 다른 작업에서 백업 데이터를 직접 활용할 수 있습니다.
  4. 마법사의 지침에 따라 설정을 마칩니다. 클라이언트측 암호화 활성화를 선택하여 백업 대상에서 데이터 안전을 보장하기 위해 전송 전에 암호화합니다.

참고:

  • 작업 이름은 1-32자의 Unicode 문자를 포함할 수 있지만 다음 문자는 포함할 수 없습니다:
    ! " # $ % & ' ( ) * + , / : ; < = > ? @ [ ] \ ^ ` { } | ~
  • 백업의 전체 폴더 경로의 최대 길이는 2048자입니다.
  • AFP를 통해 32000개 이상의 파일 또는 폴더를 EXT3 공유 폴더에 업로드하는 경우 이 공유 폴더를 포함한 모든 백업 작업이 실패합니다.
  • 암호화된 공유 폴더는 마운트 시 백업할 수 있으며 암호화/비암호화된 폴더로 복원할 수 있습니다. 그러나 복원은 항상 암호화되지 않은 상태로 수행되며///////// 시작할 때 자동으로 마운트되지 않습니다. 암호화된 공유 폴더를 마운트하기 위해서는 암호화 키를 가져오거나 입력해야 합니다.
  • 암호화된 폴더를 다른 암호화된 폴더로 백업하는 것은 지원되지 않습니다.
  • 백업 소스 또는 백업 대상으로 원격 폴더를 사용하는 것은 지원되지 않습니다.
  • 특정 응용 프로그램에 대한 백업이 지원됩니다. 특정 패키지가 백업을 지원하는지, 백업 내용 및 제한에 대해서 알아보려면 각 응용 프로그램의 도움말을 참조하십시오.
  • DiskStation은(는) 쓰기 가능한 EXT3/EXT4 외부 장치가 없으면 응용 프로그램 백업을 지원하지 않습니다.
  • DiskStation은(는) EXT3/EXT4 외부 장치에서 생성된 공유 폴더만 백업할 수 있습니다.

Windows ACL 설정 백업:

DiskStation에 있는 파일의 Windows ACL 설정 또한 대상의 파일 시스템에 따라 백업될 수 있습니다:

  • 모든 백업 대상은 대상의 파일 시스템과 관계 없이 Windows ACL 설정을 백업할 수 있습니다.

공개 클라우드 또는 원격 Synology NAS로 백업 데이터 내보내기:

대용량 데이터를 공개 클라우드로 백업할 때 초기 버전을 외부 저장 장치(예: USB 드라이브)로 내보내고 해당 장치를 클라우드 서비스 제공업체에 전달하는 것이 좋습니다. 장치를 클라우드에 연결해 데이터를 직접 업로드할 수 있습니다. 이렇게 하면 첫 번째 백업에 걸리는 시간을 크게 줄일 수 있습니다.

  1. 왼쪽 아래 구석에서 +를 클릭하고 원격 Synology NAS 또는 원하는 클라우드 서비스 제공업체를 선택합니다.
  2. 로커 공유 폴더로 내보내기(외부 저장 장치 포함)을 선택하고 설정을 마칩니다. 메시지가 나타나면 보호된 데이터 백업에 대해 클라이언트측 암호화 활성화를 선택합니다.
  3. 외부 저장 장치를 원격 Synology NAS 또는 클라우드 서비스 제공업체에 전달합니다. 외부 저장 장치에서 내보낸 데이터를 가져옵니다.
  4. 확인된 후, 백업 및 복원으로 돌아가 지금 재링크를 클릭합니다.
  5. 재링크 마법사의 지침에 따라 DiskStation을(를) 리포지토리에 링크합니다. 데이터가 암호화된 상태로 내보내지면 패스워드/암호화 키가 필요합니다.

참고:

  • 작업이 설정되었거나 패스워드를 변경할 수 없는 경우 클라이언트측 암호화를 비활성화할 수 없습니다.
  • 백업 리포지토리는 한 개의 백업 작업에만 연결할 수 있습니다.
  • 백업 소스 폴더가 암호화된 공유 폴더 또는 그 하위 폴더이면 백업 작업 생성 시 소스 폴더를 마운트 또는 마운트 해제할 수 있습니다.
  • 암호화된 공유 폴더는 백업 대상이 될 수 없습니다.
  • 클라이언트측 암호화 활성화를 선택하면 이 작업에서 데이터를 복원하는 데 패스워드/암호화 키가 필요합니다. 패스워드/암호화 키를 잊어 버리면 데이터가 영구히 손실될 수 있습니다.

백업 작업 관리하기:

왼쪽 패널에서 원하는 백업 작업을 선택하고 아래 중에서 해당 관리에 맞는 단추를 클릭합니다.

  • 지금 백업: 선택한 작업의 데이터를 즉시 백업합니다.
  • 백업 탐색기(돋보기 아이콘): 사용 가능한 모든 백업 버전을 확인하고 포함된 파일을 복사/복원/다운로드할 수 있습니다.
  • 버전 목록: 모든 백업 버전의 생성 시간과 상태를 확인하고, 특정 백업 버전을 삭제하거나 예약을 적용하기 위해 특정 버전을 잠글 수 있습니다.
  • 편집: 백업 소스, 대상, 일정 및 기타 설정을 수정합니다.
  • 삭제: 백업 작업을 삭제합니다.
  • 취소: 현재 실행 중인 백업 작업을 중지합니다.

버전을 회전하려면:

백업 회전은 버전 관리 메커니즘입니다. 이 기능은 오래되고 불필요한 버전을 자동 삭제하여 잠재적인 저장소 공간을 확보함으로써 백업 버전 관리하는 것을 도와줍니다. 작업 설정에서 이 기능을 활성화하거나 백업 작업을 생성할 때 이를 구성할 수 있습니다.

  1. 왼쪽 패널에서 원하는 백업 작업을 선택하고 편집 또는 설정> 작업 설정을 클릭합니다.
  2. 백업 회전 활성화를 선택하고 원하는 회전 모드를 선택합니다.
    • 가장 초기 버전에서: 사용 가능한 버전이 정의된 개수를 초과한 경우 가장 초기 백업 버전을 삭제합니다.
    • Smart Recycle: 다음 정책에 따라 백업 버전을 보존합니다.
      • 과거 24시간부터 매시간 버전: 매 시간 생성된 가장 초기 버전을 보존합니다.
      • 과거 1일에서 1개월 사이의 매일 버전: 매일 생성된 가장 초기 버전을 보존합니다.
      • 1개월 이상된 매주 버전: 매주 생성된 가장 초기 버전을 보존합니다.
  3. 최대 버전 수를 지정하십시오. 버전 수가 이 숫자를 초과하면 가장 오래된 버전이 회전됩니다.
  4. 일정에서 반복되는 백업 일정을 구성한 경우 백업 버전이 유지되는 방법과 시작표에서 가장 빠른 복구 지점을 미리 볼 수 있습니다.
  5. 확인을 클릭하여 설정을 저장합니다.

참고:

  • 잠근 버전은 절대로 제거되지 않습니다.
  • 복수 버전 백업 작업만이 버전 회전 기능을 제공합니다.
  • DSM 5.1 이하에서는 버전 회전이 지원되지 않습니다.
  • 회전은 설정이 변경되거나 백업 작업이 완료된 직후 트리거됩니다.
  • 잠근 버전은 최대 버전 수 계산에서 제외됩니다.

데이터 백업 작업에 대한 주의 사항

각 유형의 백업 작업에 대해 중요한 설정 세부 사항 및 문제 해결 정보에 대한 다음 주의 사항을 읽으십시오.

로컬 백업 작업:

  • 백업 작업을 진행하는 도중 대상에 있는 데이터를 삭제한 경우 다음에 작업을 수행할 때 전체 백업이 실행됩니다.
  • 새 외부 드라이브는 백업 대상으로 이용하기 전에 포맷해야 할 수 있습니다.
  • 백업 작업을 생성하기 전에, 외부 드라이브를 적절하게 DiskStation에 연결했는지 확인합니다. 제어판 > 외부 장치로 이동하여 USB/eSATA 드라이브 상태를 확인할 수 있습니다. 외부 드라이브가 초기화 및 포맷 중이거나 오작동하는 경우 이를 백업 대상으로 선택할 수 없습니다.
  • 원격 마운트된 공유 폴더는 백업 소스 또는 대상으로 선택할 수 없습니다.
  • 암호화된 공유 폴더의 마운트가 해제되면 해당 폴더를 백업 대상으로 선택할 수 없습니다.
  • 이미지 섬네일은 대상의 파일 시스템이 NTFS/FAT인 경우 백업되지 않습니다.

원격 rsync 백업 작업:

  • 네트워크 백업 대상으로 rsync 호환 가능 서버를 사용하여 백업 작업을 생성할 때 다음과 같은 모드를 사용할 수 있습니다: rsync 데몬(daemon) 모드는 백업 모듈 옆에 있는 드롭다운 메뉴에서 백업 모듈(rsync 호환 서버의 공유 폴더)을 선택한 경우에 사용됩니다. 원격 셸(remote shell) 모드는 백업 모듈에 절대 경로("/"로 시작)를 입력한 경우에만 활성화됩니다.
  • Linux 사용자의 경우 admin, root 및 rsync 계정만 백업 작업을 수행할 수 있습니다.
  • 대상 서버의 호스트 이름을 사용하여 인터넷에서 이 서버에 액세스하는 경우, 호스트 이름을 FQDN(정규화된 도메인 이름) 형식(예: abc.bcd.com)으로 입력해야 합니다.
  • Synology NAS 장치 간에 백업할 경우 대상에 있는 계정에 대한 사용자 이름과 패스워드를 입력하라는 메시지가 표시됩니다. 다른 DiskStation의 공유 폴더에 백업할 때 이 폴더의 액세스 권한 설정에 따라 수행할 수 있습니다. 접근 권한이 있는 특정 사용자 계정에 대한 공유 폴더만 표시됩니다. 대상에서 원하는 공유 폴더를 찾을 수 없으면 접근 권한을 확인하십시오.
  • 전송 암호화는 대상 서버가 암호화된 네트워크 백업을 지원하고 인증 패스워드가 비어 있지 않은 경우에만 지원됩니다.
  • 원격 셸(remote shell) 모드는 전송 암호화만 지원합니다.
  • 파일을 rsync 클라이언트를 통해된 Synology NAS 장치에 백업할 경우 파일 이름은 UTF-8 인코딩 형식이어야 합니다.
  • Rsync 호환 클라이언트를 통해 파일을 Synology NAS 장치에 백업하는 경우 DiskStation의 admin, root 및 rsync 계정으로만 작업을 수행할 수 있습니다.
  • DiskStation에서 rsync 호환 서버로 파일을 백업하는 경우 올바른 rsync 백업 계정 이름과 패스워드를 입력해야 합니다. 또한 쓰기 권한이 있는 올바른 백업 모듈을 선택해야 합니다.

Amazon S3 백업 작업:

  • 네트워크 백업 대상으로 Amazon S3 서버를 사용하여 백업 작업을 생성할 때, 전송 암호화, Amazon S3 RRS(감소된 중복 저장소)에 백업 등과 같은 다양한 옵션을 사용할 수 있습니다. 전송 암호화 같은 일부 옵션의 경우 백업 작업 속도가 감소될 수 있음에 유의하십시오.
  • 백업 파일은 Amazon S3 서버에서 선택한 Bucket에 저장되고, 설정된 Key 값은 사용자가 선택한 디렉토리 이름으로 시작하여 이 파일의 전체 파일 경로로 끝납니다.
  • Amazon S3 데이터 구조는 폴더 구조 없이 Bucket과 KEY 쌍으로 이루어집니다. 빈 폴더 백업은 지원되지 않습니다.
  • 백업된 파일이 수정되지 않은 경우, 동일 Bucket에 있는 Amazon S3 서버로 다시 백업되지 않습니다.
  • Amazon S3의 각 Bucket에는 고유한 이름을 지정해야 합니다. 또한, 이름은 다음과 같아야 합니다:
    • 3-63자 사이의 문자이어야 함.
    • 소문자, 숫자, 마침표(.), 대시(-)만 포함해야 합니다.
    • 숫자 또는 문자로 시작 및 끝내고, 대시(-)로 끝나지 말고, 마침표를 연달아 두 개 사용하지 말고, 마침표 옆에 대시가 오지 않아야 합니다(예: "my-.bucket.com" 및 "my.-bucket"은 잘못된 이름).
  • Amazon S3 KEY의 최대 길이는 1024자입니다. Amazon S3으로의 백업을 성공하기 위해서는 최대 경로 길이가 1024에서 작업 생성 과정에서 지정한 대상 폴더 이름을 뺀 값이어야 합니다.
  • Amazon S3 RRS(감소된 중복 저장소) 지원을 활성화할 경우, 이 옵션을 활성화한 후 수정한 데이터만 RRS 저장소로 백업됩니다. 즉, Amazon S3의 표준 저장소로 이미 백업된 데이터는 다시 RRS로 백업되지 않습니다.
  • Amazon S3에서 Amazon Glacier로 백업된 파일 이동은 지원되지 않습니다. AWS 관리 콘솔을 통해 Amazon S3에서 Amazon Glacier로 파일이 이동되면 관련 백업 작업이 실패할 수 있습니다.
  • SSL를 사용한 가상 호스팅 스타일 Bucket으로의 파일 백업은 지원되지 않습니다.
  • 파일을 전송할 때 파일은 부분들로 나뉩니다. 부분 크기는 기본 크기가 512MB로 작업 설정에서 멀티파트 업로드 부분 크기 옵션을 사용하여 설정할 수 있습니다. 네트워크 환경이 불안정할 경우 이 옵션을 작게 설정하는 것이 권장됩니다.
  • 업로드 당 최대 부분 수는 10000입니다. 최대 파일 크기는 멀티파트 업로드 부분 크기 곱하기 10000입니다.
  • 파일 이름은 비 UTF-8 또는 제어 문자를 포함하지 않아야 합니다.

Microsoft Azure 백업 작업:

  • 네트워크 백업 대상으로 Microsoft Azure 서버에 백업 작업을 생성할 때 다양한 옵션을 사용할 수 있습니다(예: 전송 암호화). 전송 암호화 같은 일부 옵션의 경우 백업 작업 속도가 감소될 수 있음에 유의하십시오.
  • 백업 파일은 Microsoft Azure 서버에서 선택한 Container에 저장되고, 설정된 Blob 이름은 사용자가 선택한 디렉토리 이름으로 시작하여 이 파일의 전체 파일 경로로 끝납니다.
  • Microsoft Azure 데이터 구조는 폴더 구조 없이 Container 및 Blob 쌍으로 이루어집니다. 빈 폴더 백업은 지원되지 않습니다.
  • 백업된 파일이 수정되지 않은 경우, 동일 Container에 있는 Microsoft Azure 서버로 다시 백업되지 않습니다.
  • Microsoft Azure의 각 Container에는 고유한 이름을 지정해야 합니다. 또한, 이름은 다음과 같아야 합니다:
    • 3-63자 사이의 문자이어야 함.
    • 소문자, 숫자, 대시(-)만 포함합니다.
    • 숫자 또는 문자로 시작 및 끝내고, 두 개의 인접한 대시를 포함할 수 없습니다.
  • Microsoft Azure Blob 이름의 최대 길이는 1024자입니다. Microsoft Azure로의 백업을 성공하기 위해서는 최대 경로 길이가 1024에서 작업 생성 과정에서 지정한 대상 폴더 이름을 뺀 값이어야 합니다.
  • 파일 이름은 비 UTF-8 또는 제어 문자를 포함하지 않아야 합니다.
  • 인증 실패 경고 메시지는 잘못된 인증 정보, 네트워크 연결 문제 또는 DiskStation와(과) 원격 서버의 시간이 일치하지 않을 때 발생할 수 있습니다. DiskStation에서 시간 설정을 확인하려면 제어판 > 지역 옵션 > 시간으로 이동하여 NTP 서버와 시간을 동기화합니다.

  • 파일 이름은 비 UTF-8 또는 제어 문자를 포함하지 않아야 합니다.

LUN 백업*

로컬 LUN 백업을 사용하여 DiskStation의 iSCSI LUN을 로컬 공유 폴더에 백업할 수 있습니다. 또한 네트워크 LUN 백업(Synology Server)을 사용하여 다른 DiskStation에 백업할 수도 있습니다.

iSCSI LUN 백업하기:

  1. 왼쪽 아래 구석에서 +를 클릭한 다음 LUN 백업 작업을 선택합니다.
  2. 대상 유형을 선택하라는 메시지가 표시되면 다음 중 하나를 선택합니다:
    • 로컬 LUN 백업: 이 옵션을 선택하여 LUN을 DiskStation에서 DiskStation의 USB 또는 eSATA 포트에 연결된 외부 저장소 및 로컬 공유 폴더에 백업합니다.
    • 네트워크 LUN 백업(Synology Server): 이 옵션을 선택하여 LUN을 DiskStation에서 다른 DiskStation에 백업합니다.
  3. 메시지가 표시되면 백업 대상과 소스 LUN을 지정하고 백업 일정을 설정합니다.
  4. 마법사를 따라 설치를 완료하십시오.

백업 작업이 생성된 후 DiskStation이(가) 일정에 따라 백업 작업을 수행하고 대상에 백업 LUN을 저장합니다. 백업 LUN은 백업 목적으로만 사용됩니다. 백업 LUN에 접근하려면 "백업 LUN 복원하기"에서 자세한 내용을 참조하십시오.

참고:

  • 일반 파일 및 블록 수준 LUN은 둘다 대상에 파일 이미지로 백업됩니다. 복원하면 모든 블록 수준 LUN이 파일 LUN으로 변경됩니다.
  • 각 백업 작업은 백업 소스로 하나의 LUN만 포함할 수 있습니다.
  • LUN은 FAT, FAT32, HFS, HFS PLUS 또는 NTFS 형식의 외부 디스크에 백업되지 않습니다.
  • 네트워크 LUN 백업을 수행하기 전에 대상 DiskStation에 로그인하여 해당 네트워크 백업 서비스가 제어판 > 백업 서비스에서 활성화되어 있는지 확인하십시오.
  • 라우터를 통해 DiskStation가 네트워크에 연결된 경우, 네트워크 LUN 백업을 수행하기 전에 라우터에서 iSCSI 서비스를 위한 포트 전달 규칙을 지정했는지 확인합니다(TCP 포트 3260).

전체 백업 및 점진적 백업 정보:

DiskStation은 먼저 선택한 LUN의 전체 백업을 수행한 다음 이후에 점진적 백업을 수행합니다. 그러나, 백업 대상의 공유 폴더 이름이나 디렉토리 이름이 변경되면 DiskStation이 작업에 대한 전체 백업을 다시 수행합니다.

허용 / 거부

내장 방화벽은 승인되지 않은 로그인을 차단하고 서비스 액세스를 제어할 수 있습니다. 또한 특정 IP 주소의 특정 네트워크 포트 접근을 허용하거나 거부하도록 선택할 수 있습니다.

방화벽 규칙 생성하기:

  1. 제어판 > 방화벽으로 이동한 다음 네트워크 연결 유형에 따라 왼쪽 열에서 네트워크 인터페이스를 선택합니다.
  2. 생성을 클릭합니다.
  3. 포트 섹션에서 규칙에 맞게 다음 중 하나를 선택합니다:
    • 규칙을 모든 포트에 적용하려면 모두를 선택합니다.
    • 내장된 응용 프로그램 목록에서 선택을 선택합니다.
    • 사용자 지정을 선택하고 사용자 지정을 클릭한 다음, 쉼표로 구분하여 최대 15개의 포트를 입력하거나 포트 범위를 지정합니다.
    • 또한 프로토콜도 지정할 수 있습니다.
  4. 소스 IP 섹션에서 규칙에 맞게 다음 중 하나를 선택합니다:
    • 모든 IP 주소를 허용하거나 거부하려면 모두를 선택합니다.
    • 단일 호스트 IP 주소를 입력합니다.
    • 서브넷을 입력합니다.
  5. 작업 섹션에서 규칙에 맞게 다음 중 하나를 선택합니다:
    • 위의 포트와 소스 IP 설정을 만족시키는 액세스를 허용하려면 허용을 선택합니다.
    • 위의 포트와 소스 IP 설정을 만족시키는 액세스를 거부하려면 거부를 선택합니다.

이제 규칙 목록에서 생성한 방화벽 규칙을 확인할 수 있습니다.

방화벽 규칙을 삭제하려면 다음을 수행하십시오:

  • 활성화 열 목록에서 규칙의 선택 상자 선택을 해제합니다
  • 방화벽 규칙 삭제

규칙 목록 아래에서 액세스 허용 또는 액세스 거부를 선택하여 규칙과 일치하지 않는 액세스를 허용하거나 거부할 수 있습니다.

    참고:
  1. 네트워크 인터페이스에 대해 최대 100개의 규칙을 생성할 수 있으며 드래드 앤 드롭을 사용하여 규칙의 순서를 변경할 수 있습니다.
  2. 규칙은 목록에서의 위치에 따라 우선 순위가 부여됩니다.
  3. 시스템에 동일한 서브넷에 연결된 여러 네트워크 포트가 있는 경우 방화벽 규칙이 제대로 작동하지 않을 수 있습니다.*
  4. Link Aggregation을 사용하여 여러 개의 LAN 포트를 결합하면 방화벽이 첫 번째 네트워크 인터페이스에 적용되고 두 번째 네트워크 인터페이스의 규칙은 그대로 유지됩니다.

____
* 일부 모델에서만 가능.

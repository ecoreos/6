SSO 클라이언트

DiskStation이(가) SSO 서버와 동일한 도메인 또는 LDAP 서비스에 가입한 경우, DiskStation을(를) SSO 클라이언트로 설정할 수 있습니다. 사용자는 자격 증명을 사용하여 SSO 서버에 로그인한 후 DiskStation이(가) 제공하는 서비스에 액세스할 수 있습니다.

시작하기 전에, DiskStation이(가) SSO 서버와 동일한 도메인 또는 LDAP 서비스에 가입했는지 확인하십시오.

DiskStation을(를) SSO 클라이언트로 설정하기:

  1. 제어판 > 도메인/LDAP > SSO 클라이언트로 갑니다.
  2. SSO 서비스 활성화 선택 상자를 선택합니다.
  3. 선택 상자 아래의 필드에 SSO 서버 URL 및 응용 프로그램 ID를 지정합니다.

    참고:

    응용 프로그램 ID를 가져오려면, 다음 방법 중 하나를 선택합니다:

    • 방법 1:
      1. SSO 서버로 실행하는 DSM에 로그인합니다.
      2. 클라이언트 응용 프로그램 역할을 할 DiskStation을(를) SSO Server의 응용 프로그램 목록에 추가합니다. 자세한 내용은 SSO Server > 응용 프로그램 목록에 대한 도움말 항목을 참조하십시오.
      3. DiskStation이(가) SSO 서버에 추가된 후 SSO Server에 의해 할당된 응용 프로그램 ID를 복사하여 붙여넣습니다.
    • 방법 2:
      1. SSO 서버 URL*을 입력한 후, 응용 프로그램 ID 필드 옆의 빠른 등록을 클릭합니다.
      2. 등록 상자에서, 서버 관리자 및 패스워드 필드에 SSO 서버 관리자 계정의 자격 증명을 입력합니다.
      3. 응용 프로그램 이름 필드에 클라이언트 응용 프로그램의 이름을 지정합니다.
      4. 리디렉션 URI는 정확하게 DiskStation의 로그인 페이지를 가리켜야 합니다.
      5. 확인을 클릭하면, DiskStation이(가) SSO Server에 추가되고 응용 프로그램 ID가 자동으로 채워집니다.

    ______

    * 사용자는 모든 SSO 클라이언트를 구성할 때 동일한 SSO 서버 URL(호스트 이름 또는 IP)을 입력하도록 요구됩니다.

  4. 적용을 클릭하여 설정을 저장합니다.

참고:

  • SSO 서버가 HTTPS 프로토콜을 사용하는 경우, SSO 로그인은 트러스트된 인증서의 기본값으로 브라우저에 의해 차단될 수 있습니다.
  • SSO 서버가 HTTP 프로토콜을 통해 액세스되는 경우, DiskStation (SSO 클라이언트로 설정됨)에서 HTTPS 연결을 사용하면, 사용자는 안전하지 않은 연결의 사용을 허용하도록 브라우저의 설정을 구성해야 합니다.
  • DSM의 제어판 > 보안 > 보안에서 HTTP 콘텐츠 보안 정책(CSP) 헤더로 보안 향상 확인란을 선택한 경우 IPv6 형식인 SSO 서버의 IP 주소는 지원되지 않습니다.

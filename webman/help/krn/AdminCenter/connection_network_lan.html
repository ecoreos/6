LAN/인터넷

DiskStation 모델과 네트워크 설정에 따라 제어판 > 네트워크 > 네트워크 인터페이스에서 LAN 또는 인터넷의 DHCP 설정을 구성할 수 있습니다. DHCP는 자동 IP 구성을 위한 네트워크 프로토콜입니다. DHCP가 시스템에 IP를 할당하도록 하려면 편집을 클릭하고 IPv4 탭 아래에서 자동으로 네트워크 구성 얻기(DHCP)를 선택합니다.

네트워크 서비스 제공업체에서 특정 IP 주소를 이용하고 있을 경우 수동 구성 이용을 선택하고 IP 주소와 서브넷 마스크를 수동으로 입력합니다. DNS 서버 또는 기본 게이트웨이를 사용할 수 없을 경우 해당 필드를 비워 두십시오.

참고:

  1. SMTP 서버를 통해서 이메일 알림이 올바로 전달되도록 하려면 올바른 DNS 서버 IP를 입력해야 합니다.
  2. Jumbo Frame은 표준 1500바이트 이상의 MTU(Maximum Transmission Unit)가 있는 이더넷 프레임으로, 대용량 파일의 이더넷 전송을 보다 효율적으로 처리할 수 있습니다. 이 프레임은 기가비트 네트워크 환경에서만 활성화할 수 있습니다. Jumbo Frame을 올바르게 작동하려면 DiskStation에 접근하는 네트워크 상의 모든 컴퓨터와 장치가 MTU를 지원하고 동일한 MTU 값을 사용해야 합니다.

VLAN

네트워크 환경에 VLAN 설정이 필요한 경우, VLAN 옵션을 활성화하고 편집 창의 IPv4 탭 아래에서 네트워크 환경에 대한 VLAN ID를 입력할 수 있습니다.

참고:

  • 네트워크 스위치가 적절하게 구성되었는지 확인하십시오. 그렇지 않으면 DiskStation 서비스를 이용하지 못할 수도 있습니다.
  • 위 설정을 수정한 후 DiskStation에 연결되지 않으면 DiskStation에 있는 RESET 단추를 눌러 기본 설정을 복원하십시오.

IPv6 설정

DiskStation은(는) IPv4/IPv6 이중 스택 구성표를 지원합니다. 즉, DiskStation에서 IPv4와 IPv6를 동시에 작동할 수 있습니다. IPv6의 기본 설정은 자동입니다. 기존 네트워크 인터페이스를 선택하고, 편집 단추를 클릭하고, IPv6 탭으로 이동하여 설정을 변경합니다.

IPv6 설정 구성하기:

  1. 제어판 > 네트워크 > 네트워크 인터페이스로 이동합니다.
  2. 기존 네트워크 인터페이스를 선택하고 편집을 클릭합니다.
  3. IPv6 탭 아래의 IPv6 설정에서 설정 유형을 선택합니다:
    • 자동: DiskStation이(가) LAN 라우터 광고(RA)에 따라 IPv6 주소를 자동으로 생성합니다.
      • 스테이트리스 모드: 라우터에 연결된 장치가 IPv6 DNS 서버를 설정하지 않고 자동으로 IPv6 주소를 구성할 수 있게 합니다.
      • 스테이트리스 DHCPv6 모드: 라우터에 연결된 장치가 DHCPv6를 통해 자동으로 IPv6 주소를 구성하고 DNS 정보를 얻을 수 있게 합니다.
      • 스테이트풀 모드: 라우터에 연결된 장치가 DHCPv6를 통해 IPv6 주소 및 DNS 정보를 얻을 수 있게 합니다. 이 모드에서 DHCPv6 서버를 활성화하고 DHCPv6 서버의 시작 IP 주소 및 끝 IP 주소를 설정해야 합니다.
    • 수동: IPv6 주소, 접두어 길이 및 기본 게이트웨이에 정보를 입력해서 IPv6을 수동으로 설정합니다.
    • 끄기: IPv6 지원을 비활성화합니다.

IPv6 전환 매커니즘은 IPv6가 IPv4를 완전히 대체할 때까지 IPv4 주소를 가진 호스트가 IPv6 서비스에 도달하도록 하는데 필요합니다. 여러 IPv6 전환 매커니즘 중에서 선택합니다:

  • 6in4: 터널 브로커를 통해 IPv6 네트워크에 연결합니다. IPv6 주소, 접두어 길이, 접두어 및 터널 브로커 IPv4 주소를 채워야 합니다. 이것은 공용 IPv4 주소를 요구합니다.
  • 6to4: 임의 캐스트를 사용하는 터널 브로커를 통해 IPv6 네트워크에 연결합니다. 이것은 공용 IPv4 주소를 요구합니다.
  • DHCP-PD: LAN 내에서 접두어 위임을 통해 DHCP 서버로부터 IPv6 접두어를 얻습니다. 이것은 ISP 지원을 요구하며 라우터 모드에서만 작동합니다.
  • IPv6 Tunneling(TSP): 제어판 > 네트워크 > 일반으로 이동하여 이 옵션을 구성합니다.

802.1X

802.1X는 사용자 연결 네트워크르 위해 IEEE에서 개발한 인증 표준입니다. 802.1X는 클라이언트 인증을 위해 네트워크에서 사용할 수 있지만 보호된 네트워크에 액세스하려고 했으나 인증을 성공적으로 전달하지 못한 클라이언트에는 사용할 수 없습니다.

Synology NAS를 사용하면 802.1X 인증으로 보호된 네트워크에 연결할 수 있습니다.

802.1X 인증 활성화하기:

  1. 제어판 > 네트워크 > 네트워크 인터페이스로 이동합니다.
  2. 유선 네트워크 인터페이스(즉, LAN)를 선택하고 편집을 클릭합니다.
  3. 802.1X 탭으로 이동하고 802.1X 인증 활성화를 선택합니다.
  4. 인증 드롭다운 메뉴에서 다음 프로토콜 중 하나를 선택합니다. 어떤 프로토콜이 가장 적합한지 잘 모르면 자동을 선택합니다.     
    • PEAP(보호된 EAP): CA 인증서(옵션)를 사용하여 보안 연결을 설정합니다. 이 보안 연결은 내부 인증에 사용됩니다.
    • TTLS(Tunneled TLS): CA 인증서(옵션)를 사용하여 보안 연결을 설정합니다. 이 보안 연결은 내부 인증에 사용됩니다.
    • TLS: TLS(전송 레이어 보안)를 사용하여 인증합니다.
  5. TTLS 또는 PEAP를 선택하면 드롭다운 메뉴에서 내부 인증 프로토콜 중 하나를 선택합니다.
    • MS CHAP v2: 인증 정보는 Microsoft CHAP v2로 암호화됩니다.
    • GTC: 인증 정보는 암호화에 GTC(Generic Token Card)를 사용합니다.
  6. 사용자 이름과 패스워드를 입력합니다.
  7. 필요한 경우 가져오기 및 찾아보기를 차례로 클릭하여 Radius Server에서 내보낸 인증서(예: ca.crt)를 찾아서 가져옵니다.
  8. 확인을 클릭하여 설정을 저장합니다.

참고:

  • 무선 네트워크에 필요한 802.1X 인증을 구성하려면 제어판 > 무선 > Wi-Fi로 이동합니다.
  • 인증 설정은 Radius Server 설정과 동일해야 합니다. 자세한 내용은 Radius Server 관리자에게 문의하십시오.

무선 LAN*

DiskStation이(가) 무선 라우터 역할을 할 때, 제어판 > 네트워크 > 네트워크 인터페이스에서 다음 무선 LAN 설정을 구성할 수 있습니다.

IPv4 설정

IP 주소와 서브넷 마스크

여기에서 DiskStation에 대한 IP 주소와 서브넷 마스크를 설정할 수 있습니다.

DHCP Server

DHCP 서버 활성화 옵션을 선택하여 DiskStation이(가) IP 주소를 DiskStation에서 제공하는 로컬 네트워크에 연결된 클라이언트 장치에 할당하도록 할 수 있습니다.

PPPoE 릴레이

PPPoE 릴레이를 활성화하여 DiskStation에 연결된 장치가 PPPoE 연결을 통해 인터넷에 연결될 수 있도록 합니다.

UPnP 인터넷 게이트웨이로 구성

UPnP 서비스를 활성화하여 DiskStation의 로컬 네트워크에 연결된 장치가 UPnP로 포트 전달 설정을 구성할 수 있도록 DiskStation이(가) UPnP 인터넷 게이트웨이 장치가 되도록 합니다.

IPv6 설정

6in4, 6to4 또는 DHCP-PD으로 LAN/인터넷에서 IPv6 주소와 접두어를 얻을 수 있습니다. 이 정보로 여기에서 DiskStation의 IPv6 라우팅 옵션을 구성할 수 있습니다.

  • IPv6 주소: LAN/인터넷에서 얻은 IPv6 주소를 입력합니다.
  • 접두어 길이: LAN/인터넷에서 얻은 IPv6 주소의 접두어 길이, 보통은 64를 입력합니다.
  • 기본 DNS: 라우터에 연결된 장치의 기본 DNS 서버입니다. 이것은 스테이트리스 모드에서 설정할 필요가 없습니다.
  • 보조 DNS: 라우터에 연결된 장치의 보조 DNS 서버입니다. 이것은 스테이트리스 모드에서 설정할 필요가 없습니다.

다음 3가지 라우터 광고(RA) 모드 중에서 선택할 수 있습니다:

  • 스테이트리스 모드: 라우터에 연결된 장치가 IPv6 DNS 서버를 설정하지 않고 자동으로 IPv6 주소를 구성할 수 있게 합니다.
  • 스테이트리스 DHCPv6 모드: 라우터에 연결된 장치가 DHCPv6를 통해 자동으로 IPv6 주소를 구성하고 DNS 정보를 얻을 수 있게 합니다.
  • 스테이트풀 모드: 라우터에 연결된 장치가 DHCPv6를 통해 IPv6 주소 및 DNS 정보를 얻을 수 있게 합니다. 이 모드에서 DHCPv6 서버를 활성화하고 DHCPv6 서버의 시작 IP 주소 및 끝 IP 주소를 설정해야 합니다.

DHCP 클라이언트

여기에서 DHCP 클라이언트 및 네트워크 구성 목록(예: MAC 주소, IP 주소, 호스트 이름 및 DHCP 임대 만료까지 남은 시간)을 확인할 수 있습니다.

IPv4/IPv6 클라이언트 드롭다운 목록을 필터링하여 IPv4 및 IPv6을 사용하는 클라이언트를 각각 표시할 수 있습니다. 클라이언트를 선택하고 주소 예약에 추가를 클릭하여 주소 예약 목록에 해당 클라이언트를 추가합니다.

DHCP 예약

클라이언트가 DHCP 임대 갱신 동안 항상 동일한 IP 주소를 받도록 하려면 클라이언트를 주소 예약 목록에 추가합니다.

IPv4/IPv6 주소 예약 드롭다운 목록을 필터링할 수 있습니다. IPv4 주소는 MAC 주소로, IPv6 주소는 DUID로 식별됩니다.

클라이언트를 주소 예약 목록에 추가하기:

  • 클라이언트를 선택하고 추가를 클릭합니다.
  • 클라이언트의 MAC 주소 또는 DUID, 클라이언트에 할당할 IPv4/IPv6 주소 및 클라이언트의 호스트 이름을 지정합니다.
  • 확인을 클릭하여 설정을 저장합니다.

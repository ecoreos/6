일반

기본 게이트웨이, DNS 서버, 프록시 및 IPv6 Tunneling 같은 일반 네트워크 옵션은 제어판 > 네트워크 > 일반에서 사용할 수 있습니다.

DiskStation 장치 이름 바꾸기

DiskStation 장치의 이름을 바꾸려면 새 서버 이름을 입력하고 적용을 클릭합니다.

서버 이름은 대소문자를 구분하지 않으며 1~15자 사이로 설정되어야 하고 문자, 숫자, 밑줄, 빼기(-) 기호로 구성될 수 있습니다. 이름의 첫 문자는 알파벳이어야 합니다.

기본 게이트웨이 지정

기본 게이트웨이의 IP 주소를 입력하고 적용을 클릭하여 설정을 저장합니다.

DNS 서버의 IP 주소 지정

DNS 서버의 IP 주소를 수동으로 구성할 수 있습니다. 이 경우, 기본 DNS 서버의 IP 주소를 입력해야 합니다. 또한 기본 DNS 서버에 문제가 발생할 경우 백업을 제공할 수 있도록 대체 DNS 서버 IP 주소를 입력할 수도 있습니다. DNS 서버 IP 주소를 원하는 대로 구성한 후 적용을 클릭하여 설정을 저장합니다.

참고:

모든 네트워크 인터페이스가 자동으로 구성되거나 PPPoE 연결을 사용하는 경우에는 DNS 서버와 기본 게이트웨이 필드가 자동으로 설정되며 변경이 불가능합니다.

프록시 서버를 통해 연결

DiskStation 장치가 프록시 서버를 통해 연결되도록 하려면 프록시 서버를 통해 연결을 선택한 다음 주소 및 포트를 입력하고 적용을 클릭합니다.

참고:

  • 고급 설정을 클릭하여 HTTP 프록시, HTTPS 프록시 및 인증 정보를 구성합니다.
  • 기본적으로, HTTPS 프록시 설정은 HTTP 프록시와 동일합니다.
  • 프록시 서버를 통해 연결을 선택하면 DSM 업데이트와 패키지 센터가 다른 패키지와 함께 항상 프록시를 통해 인터넷에 연결됩니다.
  • 프록시 서버에서 로컬 주소 바이패스가 선택된 경우, DiskStation이(가) 사용하는 DNS 서버가 로컬 영역 네트워크에 있는 장치에 대한 조회를 해결할 수 있는지 확인하십시오.

IPv6 Tunneling 활성화

IPv6가 IPv4를 완전히 대체할 때까지, IPv6 tunneling은 IPv6 주소를 가진 호스트가 IPv4 서비스에 도달할 수 있도록 하는데 필요합니다. 이 기술은 기존 IPv4 네트워크 내에서 IPv6 트래픽을 운반하는 방법을 제공합니다.
IPv6 구성 방법에 대해 잘 모르거나 연결 문제가 발생한 경우 인터넷 서비스 제공업체로 문의하십시오.

IPv6 Tunnel 활성화하기:

  1. Tunnel 활성화 선택 상자를 선택하여 tunnel 지원을 활성화합니다.
  2. 서버 주소에 Tunnel 서비스 제공업체의 호스트명 또는 IP 주소를 입력합니다(예: anonymous.freenet6.net).
  3. 익명으로 연결 또는 기존 계정으로 연결 중에서 원하는 옵션을 선택합니다.

참고:

서비스 제공업체에 의해 외부 IPv6 주소가 자동으로 업데이트될 수 있습니다. 그러나 DiskStation 장치는 이전의 IPv6 주소를 즉시 제거하지 않습니다.

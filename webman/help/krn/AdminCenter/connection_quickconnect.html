QuickConnect

QuickConnect는 포트 전달 규칙을 설정하는 번거로움 없이 클라이언트 응용 프로그램이 인터넷을 통해 DiskStation에 연결할 수 있도록 해줍니다. QuickConnect는 또한 Audio Station, Video Station, Download Station, Surveillance Station, Photo Station, File Station, Note Station, CMS, Cloud Station, 모바일 응용 프로그램 같은 Synology가 개발한 패키지와 작동할 수 있습니다.

QuickConnect 활성화하기:

  1. 제어판 > QuickConnect > 일반으로 이동합니다.
  2. QuickConnect 활성화를 선택합니다.
  3. 기존 Synology 계정 정보를 입력하거나 새 계정을 등록합니다.
  4. 새 QuickConnect ID를 지정합니다. 설정하여 언제든지 DiskStation에 액세스할 수 있도록 기억하기 쉬운 ID를 만드십시오.
  5. 적용을 클릭합니다.

참고:

사용자 지정 QuickConnect ID는 문자, 숫자, 하이픈("-")을 포함할 수 있으며 문자로 시작해야 합니다.

QuickConnect 연결성 향상하기:

DiskStation은(는) 서로 다른 네트워크 환경에 QuickConnect 액세스가 더 잘 적용되도록 하는 고급 옵션을 제공합니다.

  1. 제어판 > QuickConnect > 고급으로 이동합니다.
  2. 필요에 따라 아래 옵션을 선택합니다:
    • QuickConnect 릴레이 서비스 활성화: 현재 네트워크 환경 아래에서 DiskStation에 대한 직접 QuickConnect 액세스가 불가능할 때 QuickConnect 연결이 Synology 릴레이 서버를 통해 중계됩니다.
    • 자동으로 포트 전달 규칙 생성: DiskStation이(가) UPnP 가능 라우터 아래에 있을 때, QuickConnect를 위한 포트 전달 규칙을 생성하도록 라우터에게 알립니다.
  3. 적용을 클릭합니다.

참고:

  • 중계된 QuickConnect 연결은 더 긴 네트워크 지연 시간으로 인해 느릴 수도 있습니다.
  • 라우터가 QuickConnect를 위한 포트 전달 규칙을 생성하도록 허용되는 경우 DiskStation이(가) 보안 위험에 노출될 수 있습니다.

특정 응용 프로그램/서비스에 대해 QuickConnect 활성화/비활성화하기:

  1. 제어판 > QuickConnect > 고급으로 이동합니다.
  2. QuickConnect를 활성화할 응용 프로그램/서비스를 선택합니다.
  3. 적용을 클릭합니다.

참고:

QuickConnect 성능을 좋게 하려면 제어판 > 외부 액세스 > 라우터 구성으로 이동하여 각 서비스의 포트 전달을 구성하는 것이 좋습니다.

  • DSM: 5000(HTTP); 5001(HTTPS)
  • Photo Station: 80(HTTP); 443(HTTPS)
  • Cloud Station: 6690

문제 해결

네트워크 오류로 인해 QuickConnect 서비스가 중단되었음을 나타내는 네트워크 오류 발생 메시지가 표시되면 다음을 확인하십시오.

  • DiskStation이(가) 활성화된 네트워크에 연결되어 있는지(제어판 > 네트워크 > 네트워크 인터페이스).
  • DNS 서버 및 기본 게이트웨이 설정이 올바르게 구성되어 있는지(제어판 > 네트워크 > 일반).
  • DiskStation이(가) 다음 대상 포트를 사용하여 인터넷에 접근할 수 있는지 확인합니다: 80, 443, 8888.

네트워크 오류가 수정되면 QuickConnect 서비스가 몇 분 안에 자동으로 다시 시작됩니다.

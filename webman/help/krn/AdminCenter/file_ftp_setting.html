FTP/FTPS

제어판 > 파일 서비스 > FTP에서 FTP 서비스를 활성화하여 클라이언트가 시스템에 있는 폴더와 데이터에 액세스하도록 할 수 있습니다. FTP를 활성화하기 전에 라우터의 다음 TCP 포트가 DiskStation 장치에 전달되는지 확인합니다: 21(기본 컨트롤 연결), 20(액티브 모드에 대한 데이터 연결), 55536-55567(패시브 모드에 대한 데이터 연결).

FTP 서비스 활성화하기:

  1. 다음 중 하나를 활성화하십시오:
    • FTP 서비스 활성화(암호화 없음): 이는 파일을 전송하는 데 사용되는 표준 네트워크 프로토콜입니다. FTP는 전송 세션 중 패스워드, 사용자 이름 또는 파일 같은 정보를 보호하는 암호화를 제공하지 않습니다. 그러나 전송 속도가 빠르고 시스템 리소스를 덜 필요로 합니다.
    • FTP SSL/TLS 암호화 서비스(FTPS) 활성화: 전송 레이어 보안(TLS) 및 보안 소켓 레이어(SSL) 암호화 프로토콜이 추가로 지원되는 표준 FTP로, 전송 세션 중 패스워드, 사용자 이름 또는 파일 같은 정보를 보호합니다. 그러나 암호화로 인해 전송 속도가 느리고 시스템 리소스를 많이 소모합니다.
  2. 적용을 클릭합니다.

시간 초과

FTP 서버가 일정 시간이 지나면 유휴 사용자의 연결을 끊습니다.

FTP 서비스의 포트 번호 설정

이 섹션에서는 FTP 제어 채널에 사용되는 포트를 지정할 수 있습니다.

참고:

FTP 제어 채널의 기본 포트 번호는 21입니다. 다음 번호를 제외하고 1 ~ 65535 사이의 번호로 설정할 수 있습니다: 20, 22, 23, 25, 80, 110, 137, 138, 139, 143, 199, 443, 445, 515, 543, 548, 587, 873, 993, 995, 3306, 3689, 5000, 5001, 5005, 5006, 5335, 5432, 6881, 8080, 7000, 7001, 8081, 9997, 9998, 9999, 50001, 50002, eMule 기본 포트: 4662 (TCP), 4672(UDP).

패시브 FTP의 포트 범위

이 섹션에서는 패시브 FTP 연결에 사용되는 포트 범위를 지정할 수 있습니다. 기본 포트 범위를 사용하거나 사용자 지정 범위를 지정할 수 있습니다. 포트 범위는 1025 ~ 65535 사이에서 설정할 수 있으며 최대 128개의 포트를 포함할 수 있습니다.

PASV 모드에서 외부 IP 보고

이 옵션을 활성화하면 서버가 외부 IP 주소를 FTP 클라이언트에게 보고합니다. 보고된 IP 주소는 FTP 클라이언트의 연결 로그에 표시됩니다. 활성화하고 나면, 자동으로 감지된 외부 IP 주소를 선택하거나 수동으로 DiskStation의 외부 IP 주소를 입력할 수 있습니다.

이 옵션은 DiskStation이(가) NAT 뒤에 있고 FTP 클라이언트가 DiskStation과 다른 서브넷에 속할 때에만 작동합니다. 대부분 경우 이 옵션은 불필요하지만, FTP 클라이언트가 DiskStation에 연결되지 않을 경우 사용자가 이 옵션을 활성화하고 다시 시도해 볼 수 있습니다.

FXP 활성화

FXP는 파일 교환 프로토콜(File eXchange Protocol)을 나타냅니다. 이 옵션을 활성화하면 FTP 서비스가 서버 간 파일 전송 기능을 지원합니다.

FIPS 암호화 모듈 활성화

OpenSSL FIPS 140-2 유효한 암호화 모듈(인증서 #1747)을 사용하는 FTPS 서비스에서는 FIPS가 지원됩니다. 이 옵션을 활성화하면 FTPS 서비스에서 추가 보안 암호화 알고리즘을 사용하여 보다 안전한 세션이 되도록 보호합니다.

ASCII 전송 모드 지원

ASCII 전송 모드를 지원하기 위해 이 옵션을 활성화하면 DiskStation이(가) ASCII 형식의 일반 텍스트 파일을 FTP를 통해 적절하게 전송할 수 있습니다.

UTF-8 인코딩

이 옵션을 사용하면 FTP 클라이언트가 파일 이름이 다국어로 작성된 파일을 올바르게 표시하고 처리할 수 있습니다. 다양한 FTP 클라이언트의 필요를 충족하기 위해 다음 3가지 옵션이 제공됩니다.

  • 비활성화: UTF-8 인코딩이 사용되지 않습니다. 이 옵션은 파일 이름이 다국어로 작성된 파일을 업로드/다운로드하기 위해 Windows 파일 탐색기를 FTP 클라이언트로 사용하는 고객을 위해 마련되었습니다. 이 옵션은 일반 사용에는 권장되지 않습니다.
  • 자동: DSM에서 UTF-8 인코딩을 처리하는 방법을 자동으로 결정하도록 합니다. 이 옵션은 일반적인 사용에 권장됩니다.
  • 강제: UTF-8 인코딩을 강제로 사용합니다. FTP 클라이언트에서 UTF-8 인코딩을 활성화했는데도 파일 이름이 다국어로 작성된 파일이 제대로 표시 및 처리되지 않는 경우 이 옵션을 선택합니다.

연결 제한

이 섹션은 다음 옵션을 포함합니다.

  • IP당 연결 제한: 이 기능이 활성화되면 서버가 IP당 최대 연결을 제한합니다. 이 기능과 FTP 대역폭 제한을 활성화하면 효과적으로 대역폭을 관리할 수 있습니다.
  • FTP 대역폭 제한 활성화: 이 기능을 활성화하면 단일 FTP 연결의 대역폭을 제한합니다.

참고:

  • "guest" 계정은 FTP를 통해서 서버에 로그인할 수 없습니다.
  • 데이터를 올바로 이용하기 위해서 FTP 클라이언트 컴퓨터에서 설정된 코드 페이지는 반드시 DiskStation의 코드 페이지와 동일해야 합니다.
  • 특정 모델에서 FTP 서비스는 SSL 또는 TLS을 이용한 연결만 지원합니다.
  • FTP 클라이언트와 두 FTP 서버의 방화벽 설정을 올바르게 구성해야 합니다. 그렇지 않으면 FXP 기능이 제대로 작동되지 않습니다.
  • Internet Explorer 8/9를 사용하는 경우처럼 일부 FTP 클라이언트는 사용자 이름과 패스워드를 사용하여 로그인해야 합니다. 예: ftp://username:password@your.ftp.com.

외부 디스크

외부 디스크를 시스템에 연결함으로써 디스크 용량을 usbshare[number]라고 하는 시스템이 생성한 공유 폴더를 통해 공유할 수 있습니다(USB 디스크의 경우). 공유 폴더는 외부 디스크를 시스템에서 연결 해제 할 때 자동으로 제거됩니다.

외부 디스크를 시스템에 연결하면 디스크 용량을 usbshare[number](USB 디스크) 또는 satashare[number](eSATA 디스크)라는 시스템이 생성한 공유 폴더를 통해 공유할 수 있습니다. 공유 폴더는 외부 디스크를 시스템에서 연결 해제 할 때 자동으로 제거됩니다.

DiskStation에서는 다음 형식을 인식합니다: Btrfs, ext3, ext4, FAT, FAT32, HFS, HFS Plus 및 NTFS 인식되지 않은 외부 디스크는 시스템에서 사용하기 전에 먼저 포맷해야 합니다.

참고:

  1. 일부 모델은 읽기 전용으로 HFS/HFS Plus를 지원합니다.
  2. 저널은 HFS/HFS Plus에서 지원되지 않습니다.
  3. 연결을 분리하기 전에 외부 디스크를 꺼냈는지 확인하십시오.

디스크 포맷하기:

포맷을 클릭하고 마법사를 따릅니다. 외부 디스크를 DiskStation에서 ext4 및 FAT32로 포맷할 수 있습니다.

참고:

  1. 외부 디스크를 포맷하기 전에 반드시 데이터를 백업하십시오. 이 절차 중에는 모든 데이터가 삭제되며 다시 복원할 수 없습니다.
  2. 외부 디스크를 ext3 또는 ext4 형식으로 포맷하려면 디스크 크기가 1GB 이상이어야 합니다.
  3. 디스크를 ext3 또는 ext4 형식으로 포맷하면 DiskStation에서만 인식 가능합니다. PC 및 Mac으로 외부 디스크에서 직접 데이터를 읽으려면 FAT32 형식을 선택합니다.

S.M.A.R.T. 테스트 수행하기:

디스크 S.M.A.R.T. 테스트는 하드 디스크 상태를 검사한 후 보고하여 급박한 디스크 오류에 대해 조기 경고를 보냅니다. 오류가 발견되면 디스크를 바꾸는 것이 좋습니다.
S.M.A.R.T. 테스트를 수행하려면 S.M.A.R.T. 테스트 단추를 클릭하고 마법사를 따릅니다.
테스트 결과를 확인하려면 S.M.A.R.T. 정보를 클릭합니다.

참고:

테스트 시간은 시스템 환경에 따라 달라질 수 있습니다.

제한 사항

다음 상황에서는 디스크 포맷을 사용할 수 없습니다:

  • 외부 디스크가 초기화 중인 경우(예: 외부 장치가 아직 연결되지 않은 경우).
  • 외부 디스크 용량이 1GB 이하인 경우(ext3 또는 ext4 형식).
  • 외부 디스크가 포맷 중인 경우.
  • 시스템이 파일을 외부 디스크에 백업 중인 경우.
  • 외부 디스크가 손상된 경우.

문제 해결

  1. 외부 디스크 포맷이 실패했습니다. 그 이유는 무엇입니까?
    • 외부 디스크가 올바로 연결되었는지 확인해 주십시오.
    • 외부 디스크에 쓰기 금지가 설정되었는지 확인해 주십시오. 그럴 경우 포맷 작업 전에 제거해 주십시오.
  2. 외부 디스크의 파일 시스템이 지원되지 않음으로 표시되거나 시스템이 외부 디스크에 대한 usbshare[number] 또는 satashare[number] 공유 폴더를 만들지 않습니다. 어떻게 해야 합니까?
    • 현재 시스템에서는 Btrfs, ext3, ext4, FAT, FAT32, HFS, HFS Plus 및 NTFS 형식을 인식할 수 있습니다. 위의 형식으로 포맷되지 않은 외부 디스크는 인식되지 않으며 따라서 상태가 지원되지 않음으로 표시됩니다.
  3. usbshare[number] 또는 satashare[number] 공유 폴더에 액세스할 수 없습니다. 어떻게 해야 합니까?
    • 이것은 외부 디스크의 파일 시스템에 있는 문제 때문에 발생할 수 있습니다. 이를 고치려면 외부 디스크를 PC에 연결한 후 파일 시스템 검사를 수행하십시오.
  4. ext3 또는 ext4 형식의 내 외부 디스크를 내 PC 및 Mac에 연결하여 직접 액세스할 수 있습니까?
    • 불가능합니다. ext3 또는 ext4는 DiskStation에서만 인식 가능합니다. PC 및 Mac으로 외부 디스크에서 직접 데이터를 읽으려면 FAT32 형식을 선택합니다.
  5. 외부 디스크에 "lost+found"라는 이름의 폴더가 있는 이유는 무엇입니까?
    • 이 폴더는 ext3/ext4 파일 시스템을 사용할 때 자동으로 생성되는 시스템 기본 폴더입니다. 외부 디스크에 이 폴더가 표시되는 것은 정상적인 현상입니다. 관리자가 해당 폴더 내의 내용에 액세스하여 관리할 수 있는 권한이 있더라도 폴더를 수정하지 않고 그대로 두는 것이 권장됩니다.

모든 외부 장치에 대한 기본 권한 설정하기:

기본 권한을 클릭하고 외부 장치 및 해당 내용을 액세스, 보기 또는 수정할 수 있는 사용자나 그룹을 지정하여 기본 권한을 설정할 수 있습니다. 외부 장치를 DiskStation에 삽입하면 사용자가 지정한 권한이 자동으로 이 외부 장치에 할당됩니다.

권한 창에서 다음을 수행하여 기본 권한을 설정할 수 있습니다:

  1. 드롭다운 메뉴에서 다음 중 하나를 선택합니다:
    • 시스템 내부 사용자: Anonymous FTP/WebDAV 사용자 같이 기본 시스템 사용자에 액세스 권한을 할당합니다.
    • 로컬 사용자: 로컬 사용자에 권한을 할당합니다(guest 포함).
    • 로컬 그룹: 로컬 그룹에 권한을 할당합니다.
  2. 각 사용자 또는 그룹에 대한 적절한 상자를 선택 또는 선택 해제하여 외부 장치에 대한 액세스 권한을 사용자 지정합니다:
    • 읽기/쓰기: 사용자 또는 그룹이 외부 장치에 있는 파일과 폴더에 접근하여 변경할 수 있습니다.
    • 읽기 전용: 사용자 또는 그룹이 외부 장치에 있는 파일과 폴더에 접근할 수 있습니다.
    • 접근 금지: 사용자 또는 그룹이 외부 장치에 있는 파일과 폴더에 접근할 수 없습니다.

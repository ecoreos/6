DHCP Server

DiskStation은(는) DHCP 서버 역할을 하고 동적 IP 주소를 로컬 영역 네트워크 내의 DHCP 클라이언트에 할당할 수 있습니다.

DHCP 서버 활성화하기:

  1. 제어판 > DHCP Server > 네트워크 인터페이스로 이동합니다.
  2. IP 주소를 할당할 네트워크 인터페이스(예: LAN 1)를 선택하고 편집을 클릭합니다.
  3. 팝업 창에서 DHCP Server 탭을 클릭하고 DHCP 서버 활성화를 선택합니다.
  4. 다음 DHCP 설정을 지정합니다:
    • 주소 임대 시간(분)
    • 기본 DNS
    • 보조 DNS(선택 사항)
    • 도메인 이름(선택사항)
    • 웹 프록시 자동 검색 활성화(선택 사항)
  5. 서브넷을 최소한 하나 이상 설정합니다. 서브넷 설정의 경우 추가를 클릭하여 다음 정보를 지정합니다:
    • 시작 IP 주소
    • 끝 IP 주소
    • 넷마스크
    • 게이트웨이
  6. 생성을 클릭하면 생성된 서브넷이 목록에 표시됩니다.
  7. 충분한 서브넷을 생성한 후에는 서브넷을 최소한 하나 이상 활성화해야 합니다.
  8. 확인을 클릭합니다.

참고:

  • 서브넷은 원하는 만큼 많이 생성할 수 있습니다.
  • DHCP 릴레이 서버로 조정된 경우 DHCP 서버인 DiskStation이(가) 다른 서브넷의 장치에 IP 주소를 할당할 수 있습니다.

DHCP 클라이언트

DHCP 클라이언트 및 네트워크 구성 목록(예: MAC 주소, IP 주소, 호스트 이름, DHCP 임대 만료까지 남은 시간 및 예약 상태)을 확인할 수 있습니다.

  1. 제어판 > DHCP Server > 네트워크 인터페이스로 이동한 다음 원하는 인터페이스를 선택합니다.
  2. 편집 > DHCP 클라이언트를 클릭하여 클라이언트와 세부 정보를 찾습니다.

DHCP 클라이언트의 IP 주소 관리하기:

DHCP 임대 갱신 동안 클라이언트가 항상 동일한 IP 주소를 받고 이러한 클라이언트들을 관리하도록 할 수 있습니다.

  • DHCP 클라이언트 추가하기: 추가를 클릭하여 IP 주소를 클라이언트에 할당합니다.
  • IP 주소 예약하기: 잠금 아이콘을 클릭하여 현재 장치의 IP 주소를 예약할 수 있습니다.
  • 예약 취소하기: 잠금 아이콘을 클릭하여 주소 임대 시간이 지난 후 다른 장치에서 현재 IP 주소를 사용하도록 허용할 수 있습니다.

참고:

장치에 IPv4와 IPv6 주소가 둘 다 있는 경우에는 장치에 대해 IPv4 주소 및 IPv6 주소를 각각 하나씩 예약할 수 있습니다.

DHCP 클라이언트 검색하기:

  1. 오른쪽 위 구석에 있는 깔대기 아이콘을 클릭합니다.
  2. 아래 검색 조건 중 하나를 선택합니다:
    • 모두: 모든 DHCP 클라이언트 및 DHCP 예약을 나열합니다.
    • DHCP 클라이언트: 예약 여부와 상관 없이 IP 주소와 함께 DHCP 클라이언트를 모두 나열합니다.
    • DHCP 예약: 예약된 IP 주소를 모두 나열합니다. 예약된 IP 주소는 어떠한 장치와도 아직 페어링되지 않았을 수 있습니다.

DHCP 옵션

DHCP 옵션을 지정한 후, DiskStation은(는) DHCP 서버로서 특정 서브넷에 있는 DHCP 클라이언트에게 관련 정보를 전송할 수 있습니다.

서브넷에 대해 DHCP 옵션 지정하기:

  1. 제어판 > DHCP Server > 네트워크 인터페이스로 이동합니다.
  2. 원하는 네트워크 인터페이스를 선택하고 편집을 클릭합니다.
  3. DHCP Server로 이동하고 서브넷 목록으로 갑니다.
  4. 아래 작업 중 하나를 수행합니다:
    • 기존 서브넷에 대해 DHCP 옵션 지정하기: 원하는 인터페이스를 선택하고 편집을 클릭합니다.
    • 서브넷을 생성하고 해당 DHCP 옵션 지정하기: 추가를 클릭하고 서브넷 설정을 지정합니다.
  5. 원하는 DHCP 옵션을 선택합니다. 상응하는 값을 지정하기 위해 빈 필드를 클릭하여 편집합니다.
  6. 확인을 클릭합니다.

공급업체

다른 공급업체의 DHCP 클라이언트는 추가적인 DHCP 정보를 필요로 할 수 있습니다. 공급업체에 대해 DHCP 옵션을 설정함으로써 DiskStation이(가) 그러한 상세 정보를 클라이언트에 제공할 수 있습니다.

각 공급업체가 지원하는 DHCP 옵션을 알려면 해당 공급업체에 상세 정보를 문의하십시오.

공급업체 추가하기:

  1. 제어판 > DHCP Server > 공급업체로 이동합니다.
  2. 생성을 클릭하여 공급업체 및 해당 DHCP 옵션을 지정합니다:
    • 이름: 공급업체의 이름을 지정합니다.
    • 코드: DHCP 옵션의 숫자 코드를 지정합니다.
    • 값: 옵션 값을 지정합니다.
    • +: DHCP 옵션을 추가합니다.
    • -: DHCP 옵션을 삭제합니다.
  3. 생성을 클릭하여 설정을 마칩니다.

공급업체에 대한 DHCP 옵션 편집하기:

  1. 제어판 > DHCP Server > 공급업체로 이동합니다.
  2. 원하는 공급업체를 선택하고 편집을 클릭하여 DHCP 옵션을 관리합니다.
  3. 확인을 클릭합니다.

PXE

DiskStation은(는) DHCP 서버 역할을 하여 PXE(Pre-Boot Execution Environment) 설정을 구성합니다. PXE는 데이터 저장 장치(예: 하드 디스크)가 아니라 네트워크 인터페이스를 사용하여 클라이언트 장치를 부팅할 수 있도록 하는 네트워크 환경입니다.

PXE가 구현될 때, 부트 로더가 DiskStation 또는 원격 TFTP 서버에 저장될 수 있고, 클라이언트 장치는 네트워크를 통해 부트 로더를 사용하여 부팅될 수 있습니다.

DiskStation을(를) 부트 로더 서버로 사용하기:

  1. TFTP 서비스 활성화 (제어판 > 파일 서비스 > TFTP)를 선택했는지 확인합니다.
  2. 부트 로더 파일을 TFTP 루트 폴더로 호스팅하는 폴더를 선택합니다.
  3. 최소 하나의 DHCP 서브넷이 활성화되어 있는지 확인합니다(제어판 > DHCP Server > 네트워크 인터페이스에서).
  4. 제어판 > DHCP Server > PXE로 이동합니다.
  5. PXE(Pre-boot Execution Environment) 활성화를 선택합니다.
  6. 로컬 TFTP 서버를 선택한 다음 부트 로더 파일을 선택합니다.
  7. 적용을 클릭해서 설정을 저장합니다.

원격 TFTP 서버를 부트 로더 서버로 사용하기:

원격 TFTP 서버를 부트 로더 서버로 사용하여, 클라이언트가 현재 네트워크 인프라를 변경하지 않고 부팅하도록 할 수 있습니다.

  1. 최소 하나의 DHCP 서브넷이 활성화되어 있는지 확인합니다(제어판 > DHCP Server > 네트워크 인터페이스에서).
  2. 제어판 > DHCP Server > PXE로 이동합니다.
  3. PXE(Pre-boot Execution Environment) 활성화를 선택합니다.
  4. 원격 TFTP 서버를 선택하고 부트 로더 파일의 파일 경로와 서버 IP 주소를 지정합니다.
  5. 적용을 클릭해서 설정을 저장합니다.

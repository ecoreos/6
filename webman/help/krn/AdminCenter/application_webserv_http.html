웹 응용 프로그램

내장된 Web Station 기능과 PHP 지원으로, DiskStation에 웹사이트를 손쉽게 호스팅하고 동적 웹사이트를 만들어 업무용이나 취미 생활을 하는 데 사용할 수 있습니다. 추가로, 패키지 센터는 콘텐츠 관리, 고객 관계 관리 및 전자 상거래 시스템 같은 다양한 타사 패키지를 제공합니다. 기본 공유 폴더 web이 생성되어 사용자의 웹사이트 파일을 저장합니다.

Web Station 활성화하기:

  1. 제어판 > 웹 서비스로 이동합니다.
  2. Web Station 활성화를 선택합니다.
  3. 적용을 클릭합니다.

참고:

  • 보다 나은 유연성과 보안을 위해 Web Station이 http 사용자 그룹을 사용하여 작업을 수행합니다. 따라서, 웹 페이지에 대한 액세스 권한을 제어하려면 각 해당 폴더 및 파일에 대한 http 그룹의 액세스 권한을 변경하십시오. 예를 들어, web 공유 폴더의 sample.htm 파일을 Web Station을 통해 액세스할 수 있도록 하려면, http 그룹이 sample.htm 파일뿐 아니라 web 공유 폴더에 대한 적절한 읽기/쓰기 권한을 가지고 있는지 확인해야 합니다.
  • 홈 페이지 파일 이름은 index.htm, index.html 또는 index.php로 지정해야 합니다.
  • PHP mail() 함수를 이용하면 시스템이 자동으로 알림 페이지에서 SMTP 서버 정보를 가져와서 이용합니다. 이메일 알림을 활성화했고 올바른 SMTP 서버 정보를 입력했는지 확인하십시오.
  • 공유 폴더 web의 실제 경로는 /volume1/web입니다. 일부 응용 프로그램에서는 사용자가 구성 파일에서 경로를 설정하도록 요청합니다.
  • blog, photo, photosrc 및 webdefault 이름은 폴더 이름으로 예약됩니다. 이 이름들은 Web Station 에서 이용할 수 없기 때문에 web 공유 폴더 아래에서 이 이름으로 폴더를 생성하지 마십시오.
  • missing.html 파일, missing.htm 파일, 또는 missing.php 파일을 생성해서 에러 페이지를 사용자 지정할 수 있습니다. 먼저 Web Station을 활성화합니다. 그 다음 missing.html, missing.htm, 또는 missing.php 파일을 서버의 web 공유 폴더에 위치시킵니다. 사용자가 존재하지 않는 페이지로 연결되면 이 사용자 지정된 에러 페이지가 나타납니다.
  • 활성 HTTP 연결의 최대 수: 256
  • 가상 호스트의 최대 수: 30

가상 호스트 활성화를 통한 웹사이트 호스팅

가상 호스트 기능을 사용하여 단일 서버에 여러 웹사이트를 호스팅할 수 있습니다. 시작하려면 두 개의 도메인 이름을 등록하고 아래 단계에 따라 가상 호스팅을 구성해야 합니다.

가상 호스트 활성화하기:

  1. 가상 호스트를 클릭합니다.
  2. 생성을 클릭합니다.
  3. 웹사이트 파일이 저장되는 web 공유 폴더의 하위 폴더 이름을 입력합니다.
  4. 이 웹사이트에 대한 호스트 이름을 입력합니다.
  5. 이 웹사이트에 대한 포트를 입력합니다.
  6. 확인을 클릭합니다.

개인 웹사이트 활성화

로컬 사용자, 도메인 사용자 및 LDAP 사용자는 개인 웹사이트 기능을 사용하여 자신의 개인 웹사이트를 생성할 수 있습니다.

로컬 사용자

각 로컬 사용자는 서버-IP/~사용자 이름 또는 서버-이름/~사용자 이름과 같은 고유의 웹사이트 주소를 갖게 됩니다. 예:

  1. 192.168.1.1/~ken
  2. SynologyNAS/~ken

도메인 사용자

각 도메인 사용자는 서버-IP/~도메인 이름%5c사용자 이름 또는 서버-이름/~도메인 이름%5c사용자 이름과 같은 고유의 웹사이트 주소를 갖게 됩니다. 예:

  1. 192.168.1.1/~testdomain%5cken
  2. SynologyNAS/~testdomain%5cken

LDAP 사용자

각 LDAP 사용자는 서버-IP/~사용자 이름@도메인 이름 또는 서버-이름/~사용자 이름@도메인 이름과 같은 고유의 웹사이트 주소를 갖게 됩니다. 예:

  1. 192.168.1.1/~ken@testdomain
  2. SynologyNAS/~ken@testdomain

참고:

이 기능을 활성화하기 전에 사용자 홈을 활성화하십시오. 로컬 사용자는 제어판 > 사용자 > 고급으로 이동; 도메인 사용자는 제어판 > 디렉터리 서비스 > 도메인 사용자 > 사용자 홈으로 이동; LDAP 사용자는 제어판 > 디렉터리 서비스 > LDAP 사용자 > 사용자 홈으로 각각 이동합니다.

참고:

이 기능을 활성화하기 전에 사용자 홈을 활성화하십시오. 로컬 사용자의 경우, 제어판 > 사용자 > 고급으로 이동합니다.

개인 웹사이트 생성하기:

  1. 개인 웹사이트 활성화를 선택한 경우 사용자의 홈 디렉토리에 www 폴더가 자동으로 생성됩니다.
  2. 이제 사용자가 웹 사이트 관련 파일(예: 웹 페이지 및 사운드 파일)을 www 폴더에 업로드할 수 있습니다.
  3. 사용자가 업로드된 파일에 대해 적절한 액세스 권한을 갖고 있는지 확인합니다.

오류 페이지 사용자 지정하기:

  1. 오류 페이지를 missing.html, missing.htm 또는 missing.php로 저장합니다.
  2. 파일을 www 폴더에 업로드합니다.

사용자가 로컬 사용자 웹사이트에 존재하지 않는 페이지에 연결된 경우 사용자 지정된 오류 페이지가 표시됩니다.

지원 및 제한:

로컬 사용자는 다음과 같은 조건에서만 데이터에 액세스할 수 있습니다.

  • 사용자 자신의 홈
  • /var/tmp
  • /tmp
  • /volumeX/@tmp

HTTP 서비스

기본 포트 번호 80(HTTP) 또는 443(HTTPS) 이외에, Photo Station 및 Web Station에 액세스하기 위한 추가 포트 번호를 지정할 수 있습니다. 1~65535 중에서 HTTP 또는 HTTPS를 위한 포트 번호를 추가할 수 있습니다.

HTTP 또는 HTTPS 액세스를 위해 포트 번호 추가하기:

  1. 추가로 HTTP 포트 추가 또는 추가로 HTTPS 포트 추가를 선택합니다(웹 서비스를 위한 HTTPS 연결 활성화 옵션을 활성화한 후에 HTTPS 포트를 추가해야 합니다).
  2. 필드에 원하는 포트 번호를 입력합니다.

참고:

  • 다음 포트는 제외되어야 합니다. 20, 21, 22, 23, 25, 110, 137, 138, 139, 143, 199, 443, 445, 515, 543, 548, 587, 873, 993, 995, 3306, 3689, 5000, 5001, 5005, 5006, 5335, 5432, 6881, 8080, 7000, 7001, 8081, 9997, 9998, 9999, 50001, 50002, eMule 기본 포트: 4662 (TCP), 4672(UDP), 및 FTP 기본 포트 범위: 실제 범위는 모델에 따라 다를 수 있습니다.

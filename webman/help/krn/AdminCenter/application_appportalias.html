응용 프로그램 포털

응용 프로그램 포털에서는 다양한 응용 프로그램의 연결 설정을 구성하여 독립 브라우저 탭이나 창에서 직접 이러한 응용 프로그램(예: File Station)에 액세스하여 실행할 수 있습니다.

참고:

Audio Station, Download Station, Surveillance Station, Video Station, File Station 등과 같은 다양한 응용 프로그램에 대한 액세스 설정을 편집할 수 있습니다.

별칭 사용자 지정

File Station과 같이 Synology에서 개발한 각 응용 프로그램에 별칭을 할당할 수 있습니다. 사용자 지정 별칭을 사용하면 특화된 URL을 통해 응용 프로그램을 신속하게 열 수 있습니다.

응용 프로그램 별칭 사용자 지정하기:

  1. 제어판 > 응용 프로그램 포털 > 응용 프로그램으로 이동합니다.
  2. 응용 프로그램을 선택합니다.
  3. 편집 > 일반을 클릭합니다.
  4. 팝업 창에서 사용자 지정 별칭 활성화를 선택하고 별칭을 지정합니다.
  5. 확인을 클릭하여 설정을 저장합니다.

참고:

  • 일부 응용 프로그램에는 다음과 같은 기본 별칭이 있습니다: Audio Station(audio), Download Station(download), File Station(file), Surveillance Station(cam) 및 Video Station(video).
  • 별칭 이름은 시스템 또는 브라우저용으로 예약되어 있거나 다른 응용 프로그램에서 사용하고 있는 것과 동일한 이름이 될 수 없으며, 문자 수는 2 에서 20 자 이내여야 합니다.
  • 별칭에는 대소문자, 숫자 및 2개의 특수 문자인 (-)와 (_)만 사용할 수 있습니다.
  • 별칭은 특수 문자인 (-) 또는 (_)로 시작하거나 끝날 수 없습니다.

별칭을 사용하여 응용 프로그램에 액세스하기:

응용 프로그램(예:File Station)에 대한 별칭을 설정한 후에는 다음 형식으로 URL을 입력하여 응용 프로그램을 신속하게 열 수 있습니다: http://DS_IP_OR_SERVER_NAME/ALIAS/ 또는 https://DS_IP_OR_SERVER_NAME/ALIAS/

예를 들면 다음 URL을 통해 File Station에 신속하게 접근할 수 있습니다: http://192.168.xx.xx/file/ 또는 https://MySynologyNAS/file/

HTTP/HTTPS 포트 사용자 지정

File Station과 같이 Synology가 개발한 각 응용 프로그램에 HTTP/HTTPS 포트를 할당할 수 있습니다. 사용자 지정 포트를 사용하면 특화된 URL을 통해 응용 프로그램을 신속하게 열 수 있습니다.

응용 프로그램 HTTP/HTTPS 포트 사용자 지정하기:

  1. 제어판 > 응용 프로그램 포털 > 응용 프로그램으로 이동합니다.
  2. 응용 프로그램을 선택합니다.
  3. 편집 > 일반을 클릭합니다.
  4. 팝업 창에서 사용자 지정 포트(HTTP) 활성화 또는 사용자 지정 포트(HTTPS) 활성화를 선택합니다.
  5. 사용자 지정 포트 번호를 지정합니다.

참고:

  • 포트 번호 범위는 1 ~ 65535 사이일 수 있습니다.
  • 다음 포트는 시스템 사용으로 예약되어 있으므로 사용할 수 없습니다:
    • 20, 21, 22, 23, 25, 80, 110, 137, 138, 139, 143, 199, 443, 445, 515, 543, 548, 587, 873, 993, 995, 3306, 3689, 5000, 5001, 5005, 5006, 5335, 5432, 6881, 8080, 7000, 7001, 8081, 9997, 9998, 9999, 50001, 50002
    • eMule 기본 포트: 4662 (TCP), 4672 (UDP)
    • FTP 기본 포트 범위: 실제 범위는 모델에 따라 다를 수 있습니다.

HTTP/HTTPS 포트를 사용하여 응용 프로그램에 액세스하기:

응용 프로그램(예:File Station)에 대한 별칭을 설정한 후에는 다음 형식으로 URL을 입력하여 응용 프로그램을 신속하게 열 수 있습니다: http://DS_IP_OR_SERVER_NAME:HTTP_PORT 또는 https://DS_IP_OR_SERVER_NAME:HTTPS_PORT

예를 들면 다음 URL을 통해 File Station에 신속하게 접근할 수 있습니다: http://192.168.xx.xx:7000 또는 https://MySynologyNAS:7001

도메인 사용자 지정

File Station과 같이 Synology에서 개발한 응용 프로그램과 도메인 이름을 일치시킬 수 있습니다. 사용자 지정 도메인을 사용하면 특화된 URL을 통해 응용 프로그램을 신속하게 열 수 있습니다.

응용 프로그램 도메인 사용자 지정하기:

  1. 제어판 > 응용 프로그램 포털 > 응용 프로그램으로 이동합니다.
  2. 응용 프로그램을 선택합니다.
  3. 편집 > 일반을 클릭합니다.
  4. 팝업 창에서 사용자 지정된 도메인 활성화를 선택합니다.
  5. 사용자 지정 도메인을 지정합니다.
  6. 추가 설정의 경우 필요에 따라 HSTS 활성화 또는 HTTP/2 활성화를 선택할 수 있습니다.

참고:

  • 사용자가 인터넷에서 이 서비스에 액세스할 수 있으려면 이 기능을 설정하기 전에 도메인 제공업체에 도메인 이름을 신청해야 합니다.
  • 파일 업로드/다운로드와 관련된 응용 프로그램(예: File Station)의 경우 업로드 및 다운로드 속도를 저하시킬 수 있으므로 HTTP/2 활성화를 선택하지 않는 것이 좋습니다.
  • 각 도메인 이름은 DiskStation에서 별개의 한 응용 프로그램에 대해서만 사용될 수 있습니다.
  • 도메인 이름의 길이가 15자를 초과하지 않고 NetBIOS 명명 규칙을 준수하면, 시스템이 자동으로 도메인 이름을 등록하고 브로드캐스트합니다.
  • 도메인 이름을 사용하는 URL을 통해 응용 프로그램에 액세스할 때, 표준 HTTP 포트(80) 또는 표준 HTTPS 포트(443)를 통과합니다.

도메인을 통해 응용 프로그램에 액세스하기:

응용 프로그램(예:File Station)에 대한 도메인(예: file.example.com)을 설정한 후에는 다음 형식으로 URL을 입력하여 응용 프로그램을 신속하게 열 수 있습니다: http://APP_DOMAIN 또는 https://APP_DOMAIN

예를 들면 다음 URL을 통해 File Station에 신속하게 접근할 수 있습니다: http://file.example.com 또는 https://file.example.com

응용 프로그램(예: File Station)의 도메인 이름(예: FileFile)이 NetBIOS 명명 규칙을 따를 경우, Windows 사용자는 다음과 유사한 URL을 사용하여 동일한 로컬 네트워크의 Synology NAS에 있는 응용 프로그램에 액세스할 수 있습니다: http://FileFile 또는 https://FileFile

역방향 프록시 규칙 사용자 지정

DiskStation은(는) 인터넷에서 로컬 네트워크의 장비로 요청을 전송하는 역방향 프록시 서버 역할을 할 수 있습니다. 역방향 프록시 규칙은 아래의 두 시나리오에서처럼 잠재적인 위협으로부터 중요한 포트를 숨기는 데 도움이 될 수 있습니다:

시나리오 1: 중요한 포트가 80이라고 가정합니다. 이 포트는 방화벽 규칙에 따라 외부 액세스를 허용하면 안 됩니다. 신뢰할 수 있는 사용자가 인터넷에서 다른 열린 포트(예: 81)를 통해 중요한 포트 80에 도달하도록 허용하는 역방향 프록시 규칙을 설정할 수 있습니다. 이렇게 하면 신뢰할 수 있는 사용자가 방어벽을 회피하여 계속해서 포트 80에 액세스할 수 있습니다.

시나리오 2: 중요한 포트가 80이라고 가정합니다. 이 포트는 특정 장치(예: "MyTrustee"라는 서버)를 제외한 외부 액세스를 허용하면 안 됩니다. 역방향 프록시 규칙을 통해 MyTrustee의 트래픽만 포트 80에 도달할 수 있도록 허용하고 반면에 다른 장치의 트래픽은 이 포트에 도달할 수 없도록 할 수 있습니다.

역방향 프록시 규칙 설정하기:

  1. 제어판 > 응용 프로그램 포털 > 역방향 프록시로 이동합니다.
  2. 생성을 클릭하고 다음 설정을 지정합니다:
    • 설명: 규칙 기능을 식별하는 데 도움이 되는 이름을 지정합니다.
    • 소스(인터넷에서 요청을 보내는 장치) 및 대상(로컬 네트워크의 장치)에 대해 다음 규칙을 지정합니다:
      • 프로토콜: 소스/대상에 사용된 HTTP 또는 HTTPS 프로토콜
      • 호스트 이름: 소스/대상 장치의 이름
      • 포트: 소스/대상 장치에서 사용하는 포트
      • HSTS 활성화 및 HTTP/2 활성화(소스에만 해당)
  3. 확인을 클릭하여 설정을 저장합니다.

참고:

소스 호스트 이름으로 *를 입력하거나 이 이름을 비워두면 규칙이 모든 소스에 적용됩니다.

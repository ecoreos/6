네트워크 라우팅

DSM은 다양한 네트워크 구성 요구 사항을 위한 몇 가지 네트워크 라우팅 옵션을 제공합니다.

동일한 LAN에서 네트워크 인터페이스의 네트워크 흐름 구분하기:

  1. 제어판 > 네트워크 > 고급으로 이동합니다.
  2. 일반 아래에서 대상 IP 주소가 들어오는 인터페이스에 구성된 로컬 주소인 경우 ARP 요청에 응답을 선택합니다.
  3. 적용을 클릭하여 설정을 저장합니다.

이것은 각 네트워크 인터페이스에 기본 라우팅 규칙을 설정하여 해당되는 개별 물리적인 네트워크 인터페이스에서 각 인터페이스의 네트워크 패킷이 전송되도록 합니다.

또한, 동일한 LAN에 두 개 이상의 네트워크 인터페이스가 있을 경우 이 옵션을 설정하면 각 인터페이스에서 네트워크 흐름이 개별적으로 전송됩니다. 그런 다음 개별 방화벽 규칙을 각 인터페이스에 적용할 수 있습니다.

각 네트워크 인터페이스를 해당되는 개별 게이트웨이를 통해 라우팅하기:

  1. 제어판 > 네트워크로 이동합니다.
  2. 일반 아래에서 다중 게이트웨이 활성화를 선택합니다.
  3. 적용을 클릭하여 설정을 저장합니다.

이렇게 하면 네트워크에서 각 네트워크 인터페이스가 해당되는 개별 게이트웨이를 통과하도록 라우팅합니다.

기본 게이트웨이 변경하기:

  1. 제어판 > 네트워크로 이동합니다.
  2. 네트워크 인터페이스 아래에서 서비스 순서를 클릭합니다.
  3. 게이트웨이를 선택하고 위 또는 아래를 클릭하여 게이트웨이를 재정렬합니다.
  4. 확인을 클릭합니다.

첫 번째 순위의 게이트웨이가 LAN의 기본 게이트웨이로 작동합니다.

정적 라우팅 설정하기:

  1. 제어판 > 네트워크로 이동합니다.
  2. 정적 라우트 아래에서 생성을 클릭합니다.
  3. 대상 네트워크에서 대상의 IP 주소를 입력합니다.
  4. 넷마스크에서 대상의 넷마스크를 입력합니다.
  5. 게이트웨이에서 대상의 게이트웨이 주소를 입력합니다.
  6. 인터페이스에서 대상의 인터페이스를 선택합니다.
  7. 확인을 클릭합니다.

IP 라우팅 테이블 보기:

  1. 제어판 > 네트워크로 이동합니다.
  2. 정적 라우트 아래에서 IP 라우팅 테이블을 클릭합니다.
  3. 네트워크 인터페이스를 선택하여 왼쪽 위 드롭다운 목록에서 해당 IP 라우팅 테이블을 봅니다.

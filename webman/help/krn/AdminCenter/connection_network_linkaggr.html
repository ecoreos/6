Link Aggregation*

DiskStation에서 다중 LAN을 지원하므로, Link Aggregation 기술을 사용하여 그러한 LAN 인터페이스들을 결합할 수 있습니다. Link Aggregation은 여러 네트워크 인터페이스를 종합함으로써 DiskStation의 대역폭을 늘리고 트래픽 대체 작동을 제공하여 연결이 끊길 경우 네트워크 연결을 유지합니다.

LAN 인터페이스들이 결합된 후 제어판 > 네트워크 > 네트워크 인터페이스에 Bond라는 이름의 새 인터페이스가 나타납니다. 이것은 다른 인터페이스(예: LAN 또는 PPPoE)와 마찬가지로 구성할 수 있습니다.

참고:

  • Link Aggregation이 활성화되면, 각 연결된 장치에 대해 네트워크 트래픽이 자동으로 조정되고 균형이 맞추어 집니다.
  • 네트워크 인터페이스가 여러 스위치에 연결된 경우, 스위치 포트가 동일한 LAN 또는 VLAN에 있는지 확인하십시오.

Link Aggregation을 사용하여 다중 LAN 결합하기:

  1. 제어판 > 네트워크 > 네트워크 인터페이스로 이동합니다. 생성 > Bond 생성을 클릭합니다.
  2. 원하는 모드를 선택합니다.

    모드 설명:

    • 적응 로드 밸런싱: 이 모드는 스위치가 Link Aggregation을 지원하는지 여부에 관계없이 DiskStation에 의해 수신 및 전송되는 네트워크 트래픽을 최적화합니다. 예기치 않은 오류를 피하려면, Link Aggregation이 지원되는 경우에도 이를 활성화하지 마십시오.
    • IEEE 802.3ad 동적 Link Aggregation: 이 모드는 DiskStation에 의해 수신 및 송신되는 네트워크 트래픽을 최적화하며, 이 경우 스위치에서 IEEE 802.3ad (동적) Link Aggregation (LACP, 802.1AX)을 활성화해야 합니다. 여러 스위치를 사용하는 경우, 그러한 스위치들은 스택 가능하고 적절하게 구성되어야 합니다.
    • 균형 XOR: 이 모드는 DiskStation에 의해 수신 및 송신되는 네트워크 트래픽의 균형을 맞추며, 이 경우 스위치에서 정적 Link Aggregation을 활성화해야 합니다. 여러 스위치를 사용하는 경우, 그러한 스위치들은 스택 가능하고 적절하게 구성되어야 합니다. 스위치가 Link Aggregation을 지원하지 않는 경우, DiskStation에서 전송하는 트래픽은 여전히 균형이 조정될 수 있습니다. 이 모드를 사용할 때, 손실된 링크가 감지될 수 있지만 원인이 되지는 않습니다(예: 이더넷 케이블이 연결되지 않았거나 스위치가 잘못 구성되었을 때).
    • 활성/대기: 이 기능은 DiskStation에 네트워크 오류 해결 기능을 제공합니다. 즉, 활성 네트워크 인터페이스의 작동이 중단되면, 다른 LAN 포트 중 하나가 임무를 넘겨 받아 네트워크 연결을 유지합니다. 스위치를 구성하지 않은 채로 또는 IEEE 802.3ad Link Aggregation을 지원하는 스위치에서 이 모드를 선택할 수 있습니다.
  3. Link Aggregation을 생성할 인터페이스를 선택합니다.
  4. IP 설정을 구성합니다. 필요에 따라 VLAN을 활성화할 수도 있습니다. 적용을 클릭하여 설정을 완료합니다.
  5. 마치고 나면, 네트워크 인터페이스 목록에서 Bond 1이라는 새 인터페이스를 볼 수 있습니다.

Link Aggregation 설정 편집하기:

  1. Link Aggregation 모드를 변경하거나 물리적 장치의 정보를 보려면 제어판 > 네트워크 > 네트워크 인터페이스로 이동합니다.
  2. 인터페이스 Bond 1을 선택하고 편집 단추를 클릭합니다.
  3. 해당 탭으로 이동해서 원하는 설정을 수정합니다.

Link Aggregation 제거하기:

  1. 제어판 > 네트워크 > 네트워크 인터페이스로 이동합니다.
  2. 인터페이스 Bond 1을 선택하고 삭제 단추를 클릭합니다.
  3. 마치고 나면, 이제 Bond 1이 두 개의 LAN 인터페이스로 교체된 것을 알 수 있습니다.

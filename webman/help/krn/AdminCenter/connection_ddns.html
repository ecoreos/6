DDNS

DDNS(동적 도메인 이름 서비스)는 호스트 이름을 IP 주소에 매핑함으로써 인터넷을 통해 DiskStation에 연결하기 쉽게 해줍니다.예를 들어, DDNS는 도메인 이름(예: www.john.synology.me)을 사용하여 DiskStation에 액세스할 수 있게 하므로 IP 주소(예: 172.16.254.1)를 외울 필요가 없습니다.

DDNS 설정

기존 호스트 이름을 DiskStation의 IP 주소를 가리키도록 설정할 수 있고, Synology나 다른 여러 DDNS 제공업체가 제공하는 새로운 호스트 이름을 등록할 수도 있습니다.호스트 이름 등록에 대한 자세한 내용은 각 제공업체에 문의하십시오.

DDNS 호스트 이름을 설정하려면:

  1. 추가 단추를 클릭합니다.
  2. 다음 설정을 편집하라고 알리는 대화 상자가 나타납니다:
    • 서비스 제공업체: 서비스 제공업체를 선택합니다.Synology가 제공하는 무료 호스트 이름을 사용하기 위해 등록하려면 드롭다운 메뉴에서 Synology를 선택하고 지금 등록을 클릭하고 지시된 대로 세부 정보를 입력합니다.
    • 호스트 이름: 등록한 DDNS 호스트 이름(예: john.synology.me)을 입력합니다.
    • 사용자 이름: DDNS 제공업체의 사용자 이름을 입력합니다.
    • 패스워드: DDNS 제공업체의 패스워드를 입력합니다.
    • 외부 주소: 호스트 이름을 사용할 DiskStation의 외부 IP 주소를 입력합니다.
    • 심장박동: 매핑된 호스트 이름의 상태에 관한 경고를 수신하려면 이 옵션을 활성화합니다 (Synology에서 제공한 호스트 이름에만 해당).
  3. 연결 테스트를 클릭하여 설정이 올바른지 확인합니다.
  4. 확인을 클릭하여 저장하고 마칩니다.

설정이 완료된 후, 웹 브라우저에 DDNS 호스트 이름을 입력하여 인터넷을 통해 DiskStation에 액세스할 수 있습니다.

요구 사항 및 제한

  • DDNS 서비스 제공업체의 서버가 작동 중이어야 합니다.
  • DiskStation이(가) 인터넷에 연결할 수 있어야 합니다.
  • DDNS는 프록시 서버가 필요하지 않은 네트워크 환경에서만 작동합니다.
  • 각 DDNS 제공업체에 대해 호스트 이름을 하나만 입력할 수 있습니다.

호스트 이름 관리

다음 단추를 사용하여 기존 호스트 이름 항목을 관리합니다:

  • 삭제:목록에서 기존 호스트 이름 항목을 제거합니다.
  • 편집: 선택된 호스트 이름 항목의 설정을 변경합니다.
  • 지금 업데이트: DDNS 서비스가 활성화되어 있는 모든 호스트 이름 항목의 상태를 업데이트합니다.

사용자 지정된 DDNS 제공업체 설정

사용 중인 제공업체가 서비스 제공업체 목록에 아직 나타나지 않으면 사용자 지정된 DDNS 제공업체를 추가할 수 있습니다.

사용자 지정된 DDNS 제공업체를 설정하려면:

  1. 사용자 지정 을 클릭합니다.
  2. 다음 설정을 입력하라고 알리는 대화 상자가 나타납니다:
    • 서비스 제공업체: 사용자 지정된 서비스 제공업체의 이름을 입력합니다.
    • 쿼리 URL: 사용자 지정된 서비스 제공업체의 URL을 입력합니다.
  3. 저장을 클릭하여 DNS 서비스 제공업체 목록에 사용자 지정된 서비스 제공업체를 추가합니다.

요구 사항 및 제한

  • 쿼리 URL은 서비스 제공업체에서 지정한 프로토콜을 준수해야 합니다.이 URL은 제공업체의 DDNS 서버에 대한 IP 정보를 업데이트하는 데 사용됩니다.
  • 쿼리 URL에는 일반적으로 아래와 같은 4가지 매개변수가 들어 있습니다.쿼리 URL을 입력하는 경우 다음 매개변수를 해당하는 변수로 바꾸십시오.
    • 사용자 이름:__USERNAME__
    • 패스워드:__PASSWORD__
    • 호스트 이름:__HOSTNAME__
    • 외부 주소:__MYIP__
  • 쿼리 URL에는 공백이나 세미콜론을 포함할 수 없습니다.

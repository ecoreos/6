로그 수신

DiskStation은(는) 네트워크를 통해 클라이언트 장치에서 로그를 수신하기 위한 syslog 서버 역할을 할 수 있습니다. 로그를 수신하는 데 다음 3개의 형식이 지원됩니다: BSD (RFC 3164), IETF (RFC 5424) 및 사용자 지정. 수신된 모든 로그는 로그 데이터베이스에 저장됩니다.

시작하기 전에:

로그 센터 > 아카이브 설정으로 이동하고 아카이브 로그를 저장할 위치를 선택합니다.

표준 로그 수신 설정하기:

  1. 로그 센터 > 로그 수신 > 표준으로 이동합니다.
  2. 필요한 형식 아래에서 다른 장치에서 로그 받기 옵션을 선택하여 BSD 및 IETF 로그 수신을 활성화할 수 있습니다.
  3. 전송 프로토콜에서 UDP 또는 TCP를 선택합니다.
  4. 포트에 syslog 메시지를 수신하는 데 필요한 포트 번호를 지정합니다. 기본 포트 번호는 514입니다.
  5. 전송 로그를 안전하게 보호하려면 보안 연결(SSL) 활성화 옵션을 선택합니다. 이 옵션은 TCP를 선택할 때만 사용할 수 있습니다.
  6. 적용을 클릭하여 설정을 저장합니다.

사용자 지정 로그 수신 설정하기:

  1. 로그 센터 > 로그 수신 > 사용자 지정으로 이동합니다.
  2. 생성을 클릭합니다.
  3. 이름에 고유한 규칙 이름을 입력합니다.
  4. 규칙 매개변수에 구문 분석 규칙을 입력합니다. 패턴 구문 분석기 생성에 대한 자세한 내용은 아래 참고를 참조하십시오.
  5. 전송 프로토콜에서 UDP 또는 TCP를 선택합니다.
  6. 포트에 syslog 메시지를 수신하는 데 필요한 포트 번호를 지정합니다. 기본 포트 번호는 514입니다.
  7. 전송 로그를 안전하게 보호하려면 보안 연결(SSL) 활성화 옵션을 선택합니다. 이 옵션은 TCP를 선택할 때만 사용할 수 있습니다.
  8. 확인을 클릭하여 설정을 저장합니다.

참고:

패턴 구문 분석기에 대한 자세한 내용은 여기에서 찾을 수 있습니다. 참고를 위해 샘플 패턴을 다운로드할 수도 있습니다.

사용자 지정 규칙 관리하기:

  1. 기존 사용자 지정 규칙을 선택합니다.
  2. 다음 중에서 선택합니다:
    • 편집: 선택한 사용자 지정 규칙을 편집합니다.
    • 삭제: 선택한 사용자 지정 규칙을 삭제합니다.
    • 활성화: 선택한 사용자 지정 규칙을 활성화합니다.
    • 비활성화: 선택한 사용자 지정 규칙을 비활성화합니다.

인증서 내보내기

보안 연결(SSL) 활성화 옵션을 활성화하면 인증서 내보내기 단추를 클릭하여 DiskStation에 로그를 전송할 때, 다른 장치 인증에 사용되는 인증서를 내보낼 수 있습니다.

로그 전송

syslog 서버가 있으면 아래 지침에 따라 DiskStation을(를) syslog 클라이언트로 구성하고 DiskStation의 로컬 시스템 로그를 syslog 서버에 전송하여 모니터링 및 관리를 중앙화할 수 있습니다.

syslog 서버로 로그 보내기:

  1. 로그 센터 > 로그 전송으로 이동합니다.
  2. syslog 서버로 로그 보내기 상자를 선택합니다.
  3. 서버에 syslog 서버의 호스트 이름 또는 IP 주소를 입력합니다.
  4. 포트에 syslog 메시지를 수신하는 데 필요한 포트 번호를 지정합니다.
  5. 전송 프로토콜에서 UDP 또는 TCP를 선택합니다. 이것은 syslog 서버에서 사용하는 프로토콜과 일치해야 합니다.
  6. 로그 형식에서 BSD(RFC 3164) 또는 IETF(RFC 5424)를 선택하여 선택한 형식을 사용하는 로그를 보냅니다.

    7. 참고:

    • 기본적으로, 포트는 514입니다. 다른 포트 번호를 syslog 서버에 다시 할당하려는 경우 적절한 포트 전달 규칙을 설정해야 합니다.
    • UDP(사용자 데이터그램 프로토콜)는 보다 단순한 메시지 기반 연결 포로토콜로 더 빠르기는 하지만 안전성이 부족하며 오류 정정 기능이 없습니다. TCP(전송 제어 프로토콜)는 플로우 컨트롤을 사용하여 새 것이 처리되기 전에 모든 패킷이 성공적으로 전달되었는지 여부를 알아내는 방식으로 성능을 보장하는 프로토콜입니다.
    • 보안 연결(SSL)은 TCP 프로토콜에서만 사용할 수 있습니다.
    • TCP를 선택하고 syslog 연결을 SSL로 안전하게 보호하려면, 보안 연결(SSL) 활성화 옵션을 선택합니다. 인증서 가져오기를 클릭하고 로그를 안전하게 syslog 서버에 전송하기 위한 인증서를 선택합니다.
  7. 테스트 로그 보내기를 클릭하여 syslog 서버가 로그를 성공적으로 수신할 수 있는지 확인합니다. syslog 서버가 테스트 로그를 수신하지 못한 경우 설정이 올바른지 확인하십시오.
  8. 적용을 클릭하여 설정을 저장합니다.

로그 필터

필터 탭에서, 범주(예: 시스템, 연결, 백업) 또는 심각도(예: 경고, 오류, 정보)를 기준으로 syslog 서버에 전송될 로그 유형을 지정할 수 있습니다.

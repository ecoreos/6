리소스 모니터

리소스 모니터를 사용하면 CPU 사용, 메모리 사용, 디스크 사용 및 네트워크 흐름을 모니터링할 수 있습니다. 실시간으로 모니터링하거나 이전 데이터를 보도록 선택할 수 있습니다.

리소스 모니터를 사용하면 CPU 사용, 메모리 사용 및 네트워크 흐름을 모니터링할 수 있습니다. 실시간으로 모니터링하거나 이전 데이터를 보도록 선택할 수 있습니다.

참고:

  • 설정 탭에서 사용 기록 활성화를 선택한 경우에만 이전 데이터가 표시됩니다.
  • Docker DSM의 경우, CPU 및 메모리는 Docker 호스트에 관한 사용 정보를 표시합니다.

성능

성능 탭을 클릭하면 리소스 사용 데이터가 표시됩니다. 상세한 날짜, 시간 및 차트의 정보는 마우스를 가져가면 표시됩니다.

CPU

사용자, 시스템 및 I/O 대기의 전체 CPU 사용량을 볼 수 있습니다. 리소스 모니터를 처음으로 실행하는 경우 시스템이 리소스 데이터를 수집하는 동시에 UI 페이지를 로드하기 때문에 CPU 부하가 높을 수 있습니다.

사용자는 사용자 응용 프로그램 실행에 사용된 CPU의 백분율을 표시합니다. 시스템은 주요 시스템과 하드웨어/소프트웨어 IRQ에 사용된 CPU 비율을 표시합니다. I/O 대기는 I/O 대기에 사용된 CPU 비율을 표시합니다.

이용은 사용자와 시스템에서 사용한 CPU 합계입니다.

1, 5 및 15분 내에 평균 로드가 표시되어, 실행-대기열 길이(예: 현재 실행 중인 작업 수 + 실행을 기다리는 수의 합)를 보여줍니다.

용어 설명

  • I/O 대기: I/O 대기는 CPU가 유휴이고 다른 프로그램에서 사용하지 않는 것을 나타냅니다.

    • I/O 대기는 정상적인 동작입니다. CPU 속도는 I/O보다 빠르기 때문에 CPU에서 일반적으로 디스크 또는 네트워크의 전송이 완료될 때까지 기다립니다. I/O 대기율이 너무 높으면 I/O 대기에 대부분의 CPU 시간이 소요되어 활용률이 저하됩니다. 보다 나은 시스템 성능을 구현하려면 더 효율적인 디스크 그룹 구성(예: SHR 또는 RAID 5) 또는 네트워크 구성(예: 본딩)을 사용하십시오.

메모리

물리적 메모리의 전체 메모리 사용량과 스왑 공간이 표시됩니다. 하드 디스크에 접근하지 않고도 데이터를 빠르게 확보할 수 있도록 시스템이 자주 이용하는 데이터를 캐시에 저장하기 때문에 메모리 사용은 높은 상태로 유지됩니다. 전체 메모리가 부족한 경우 캐시 메모리가 해제됩니다.

스왑 공간 사용량이 많은 것은 시스템 메모리가 부족함을 나타내며 시스템 성능에도 영향을 미칩니다. 상단의 드롭다운 메뉴에서 스왑을 선택하여 스왑 인과 스왑 아웃의 비율을 볼 수 있습니다.

네트워크

네트워크 흐름 차트는 초당 KB 비율로 송/수신된 데이터를 보여줍니다. PPPoE 가 활성화된 경우 차트에 전송 속도도 표시됩니다.

디스크

차트 위쪽의 드롭다운 메뉴를 클릭하여 전체 디스크 사용, 전송율 또는 IOPS 를 볼 수 있습니다. 아래쪽에서 모두 보기를 클릭하고 데이터를 차트에 추가하는 선택 상자를 눌러 좀 더 많은 디스크를 선택할 수 있습니다.

볼륨

이 페이지는 볼륨의 전반적인 전송 상태를 표시합니다. 차트 위의 드롭다운 메뉴를 클릭하여 전송 이용률, 전송 속도 또는 IOPS를 봅니다. 모두 보기 단추를 클릭하고 개별 볼륨을 선택하여 차트에 추가합니다.

iSCSI

이 페이지는 현재 실행 중인 iSCSI LUN의 전반적인 상태를 표시합니다. 차트 위의 드롭다운 메뉴를 클릭하여 LUN을 선택하고 해당 상태를 봅니다. 기본적으로 다음 3개의 차트가 표시됩니다: 처리량, IOPS 및 지연 시간. 보다 자세한 정보를 알려면 차트 선택 단추를 클릭하고 보고 싶은 다른 차트를 선택합니다.

프로세스

프로세스 탭을 클릭하여 CPU 부하를 줄이기 위해 각 프로세스의 CPU와 메모리 사용을 확인할 수 있습니다.

프로세스 상태는 실행 중, 수면 중 및 정지됨으로 표시됩니다. Linux의 프로세스 상태는 running, stopped/tracing, sleeping이 사용됩니다.

수면 중 뒤에는 자세한 정보를 나타내는 문자가 올 수 있습니다(예: 'D'는 디스크 수면, 'Z'는 좀비, 'X'는 오류).

연결된 사용자

이 탭에서는 현재 시스템에 연결되어 있고 리소스에 액세스되어 있는 클라이언트를 보고 관리할 수 있습니다. 이 섹션은 HTTP, TELNET, SSH, SMB/CIFS, AFP, FTP 및 NFS를 통해 연결되어 있는 클라이언트를 표시합니다.

현재 연결을 관리하려면 다음 중에서 수행하십시오:

  • 저장을 클릭하여 현재 목록을 HTML 파일로 내보냅니다.
  • 새로 고침을 클릭하여 목록을 새로 고칩니다.
  • 연결 끊기를 클릭하여 사용자가 리소스에 액세스하는 것을 중지합니다. 연결이 끊긴 사용자의 경우 리소스에 액세스하려면 인증을 다시 받아야 합니다.
  • 사용자 비활성화를 클릭하여 사용자를 비활성화합니다. 사용자를 다시 활성화하려면 메인 메뉴 > 제어판 > 사용자로 이동합니다.

참고:

  • 연결 끊기 옵션은 NFS 클라이언트에 작동하지 않습니다.
  • NFS 클라이언트가 공유 폴더를 적절하게 마운트 해제하지 않을 경우, 공유 폴더는 연결된 사용자 목록에 여전히 표시될 수 있습니다.
  • DiskStation을(를) 다시 시작하면, 이전에 연결된 NFS 클라이언트가 연결된 사용자 목록에 나타나지 않습니다.

속도 제한

속도 제한 탭 아래에서, DiskStation에서 속도 제한이 적용된 파일 전송 작업 목록을 보고 관리할 수 있습니다.

목록 새로 고치기:

페이지를 다시 로드하고 최근에 업데이트된 기록을 가져오려면 새로 고침을 클릭합니다.

파일 전송 작업 중지하기:

연결 끊기를 클릭하여 파일 전송을 중지합니다. 성공적으로 작업을 완료하려면 파일 전송을 다시 시작해야 합니다.

설정

설정을 클릭하고 다음 옵션을 설정합니다.

  • 드롭다운 메뉴에서 실시간 경과를 분으로 선택합니다.
  • 이전 데이터를 보려면 사용 기록 활성화를 선택합니다. 이 옵션을 활성화하면, 성능 차트를 볼 때 오른쪽 위 구석에 있는 드롭다운 메뉴에서 실시간, 1일, 1주, 1개월 및 1년 사이를 전환할 수 있습니다.

적용을 클릭해서 설정을 저장합니다.

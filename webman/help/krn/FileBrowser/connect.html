원격 연결

File Station에서는 다양한 프로토콜(FTP, STFP, WebDAV 및 WebDAV HTTPS)을 통해 공개 클라우드 서비스(Box, Dropbox, Google Drive 및 Microsoft OneDrive) 및 파일 서버에 연결할 수 있습니다.

클라우드 서비스에 연결하기:

  1. 도구 > 원격 연결 > 연결 설정으로 이동합니다.
  2. 아이콘을 클릭하여 원하는 클라우드 서비스를 선택하고 다음을 클릭합니다.
  3. 팝업 창에서 계정 인증에 대한 마법사의 지침을 따릅니다.
  4. 설정이 완료되면 대상 클라우드 서비스가 왼쪽 창에 나타납니다.

클라우드별 제한 사항:

  • Box:
    • 다른 공동 작업자와 공유한 카탈로그에는 소유자의 사용자 이름이 접미어로 붙습니다.
    • 다른 공동 작업자와 공유한 카탈로그가 동일한 이름(소유자의 사용자 이름이 접미어로 붙은 카탈로그 이름)을 사용하는 경우 이러한 카탈로그 하나만 표시됩니다.
    • Box를 통해서는 확장자가 다음과 같은 파일만 열 수 있습니다. .gdoc, .gsheet, 및 .boxnote.
    • 파일 이름은 대소문자를 구분합니다.
    • 적절한 방화벽 설정에 대해서는 이 게시물을 참조하십시오.
  • Dropbox:
    • 파일이 25,000개가 넘게 들어 있으면 Dropbox 카탈로그를 표시할 수 없습니다.
    • 파일이 10,000개가 넘게 들어 있으면 Dropbox 카탈로그의 이름을 바꿀 수 없습니다.
    • 10GB가 넘는 파일은 업로드할 수 없습니다.
    • 대상 파일을 업로드하는 데 24시간 이상이 걸리는 경우 업로드 작업에 실패합니다.
    • 파일 이름은 대소문자를 구분합니다.
    • 적절한 방화벽 설정에 대해서는 이 게시물을 참조하십시오.
  • Google 드라이브:
    • 경로 이름이 2048자 이상인 파일/폴더는 표시되지 않습니다.
    • 파일 이름이 255자 이상인 파일은 표시되지 않습니다.
    • 파일/카탈로그의 이름이 같더라도 생성 시간은 별도로 표시됩니다.
    • File Station을 통해 Google 관련 파일(예: Google 문서 및 Google 슬라이드)을 열면 대상 파일에 액세스할 수 있도록 URL이 표시됩니다.
    • File Station에 표시된 내 드라이브 및 별표함 폴더는 각각 Google 드라이브의 "내 드라이브" 및 "별표함" 폴더에 상응합니다.
    • 적절한 방화벽 설정에 대해서는 이 게시물을 참조하십시오.
  • Microsoft OneDrive:
    • OneNote 메모장 파일이 표시되지 않습니다.
    • 10GB가 넘는 파일은 업로드할 수 없습니다.
    • 파일(예: .png, .jpg, .docx, .pdf)이 더 큰 파일 크기로 표시될 수 있습니다(원래 크기에 섬네일 크기 추가). 이는 다운로드 시 원래 파일 크기에 영향을 미치지 않습니다.
    • 파일 이름은 대소문자를 구분합니다.

제한 사항에 대해 자세히 알고 싶으면 해당 클라우드 서비스의 공식 웹사이트를 방문하십시오.

파일 서버에 연결하기:

  1. 도구 > 원격 연결 > 연결 설정으로 이동합니다.
  2. 아이콘을 클릭하여 원하는 서버 프로토콜을 선택하고 다음을 클릭합니다.
  3. 다음 정보를 입력하여 설정을 완료합니다.
    • 호스트 이름 또는 IP: 대상 파일 서버의 호스트 이름 또는 IP를 입력합니다.
    • 포트: 대상 파일 서버의 포트 번호를 입력합니다.
    • 사용자 및 패스워드: 대상 파일 서버에 계정/패스워드가 필요한 경우 이러한 로그인 정보를 입력합니다.
    • 코드페이지: 파일 이름을 올바르게 표시하려면 대상 파일 서버의 문자 인코딩을 선택합니다.
    • 프로파일 이름: 파일 서버를 쉽게 식별하려면 고유한 이름을 입력합니다.
    • WebDAV HTTPS 연결 사용: 이 옵션은 WebDAV 프로토콜을 설정하는 경우와 클라이언트 및 파일 서버에서 둘 다 WebDAV HTTPS를 지원하는 경우에만 사용할 수 있습니다.
    • 사용자당 하나의 연결만 허용: 이 옵션은 FTP 또는 SFTP 프로토콜을 설정하는 경우에만 사용할 수 있습니다. 이 옵션을 선택하면 서버에 연결 수 제한이 있는 경우 연결성을 최적화합니다.
  4. 적용을 클릭해서 설정을 저장합니다. 대상 파일 서버가 왼쪽 패널에 나타납니다.

기존 클라우드 서비스 또는 파일 서버 프로파일 수정하기:

  1. 도구 > 원격 연결 > 연결 목록으로 이동합니다.
  2. 다음 관리에 필요한 대상 클라우드 서비스 또는 파일 서버를 클릭합니다.
    • 편집: 편집을 클릭하고, 현재 프로파일 구성을 변경한 다음 확인을 클릭합니다.
    • 연결 해제: 연결 해제를 클릭하면 연결이 종료됩니다.
    • 다시 연결: 다시 연결을 클릭하면 연결을 복구합니다. 또는 다시 연결하려면 File Station의 왼쪽 패널에서 대상을 클릭합니다.
    • 삭제: 삭제를 클릭한 다음 파일을 버릴지 확인합니다. (또는, 삭제하려면 File Station의 왼쪽 패널에서 대상을 클릭한 다음 삭제를 클릭합니다.)

참고:

  • 각 사용자는 동시에 100개 이상의 클라우드 서비스 및 파일 서버에 연결할 수 없습니다
  • 올바른 서버 코드 페이지를 선택하여 예기치 않은 오류가 발생하지 않도록 하십시오. 서버 코드 페이지를 모르는 경우 유니코드를 사용하는 것이 좋습니다.
  • 특정 File Station 기능(예: 압축 대상, 압축 풀기, 미리보기 등)은 클라우드 서비스 또는 파일 서버에 저장된 파일에 대해서는 적용할 수 없습니다.
  • 연결된 각 클라우드 서비스 또는 파일 서버의 개별 전송 속도는 사용자 속도 제한 설정에 따라 다릅니다.
  • 각 사용자는 연결된 모든 클라우드 서비스 및 파일 서버에서 동시에 3개의 파일을 전송하도록 제한됩니다.
  • WebDAV 및 WebDAV HTTPS 및 클라우드 서비스 프록시 설정을 구성하려면 제어판 > 네트워크 > 일반 > 프록시로 이동합니다. 클라우드 서비스 또는 파일 서버에 연결 중 프록시 설정을 변경한 경우 연결 해제한 다음 다시 연결하십시오.
  • 연결된 클라우드 서비스 또는 파일 서버의 정보를 수정하더라도 연결 상태에는 아무런 영향을 미치지 않습니다.
  • 연결 리소스를 확보하기 위해, 클라우드 서비스/파일 서버는 포함된 파일이 30분 이상 손대지 않은 채 있으면 자동으로 연결이 끊어집니다. 클라우드 서비스/파일 서버를 클릭하여 연결을 복구합니다.
  • 다음 이름 지정 형식을 따르는 파일/폴더는 표시되지 않습니다.
    • ._ 또는 .SYNOPPSDB로 시작하는 경우
    • /가 포함된 경우
    • 정확하게 다음으로 명명됩니다 .    ..    '    "    .DS_Store    @eaDir    @sharebin    @tmp    .SynologyWorkingDirectory
  • 원격 연결은 높은 권한 모드에서만 지원됩니다. 이 기능을 사용하려면, 호스트 DSM의 Docker 패키지로 가서, 암호화된 공유 폴더를 활성화할 Docker DSM을 선택하고, 편집을 클릭하고 높은 권한을 사용하여 컨테이너 실행 옆의 상자를 선택합니다.

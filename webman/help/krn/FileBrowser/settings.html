File Station 설정 관리

File Station 설정을 관리하려면 File Station을 열고 설정 단추를 클릭합니다.

일반

여기에서 File Station 로그를 활성화하여 모든 사용자 활동을 기록할 수 있습니다. 사용자 활동은 폴더 생성, 업로드, 다운로드, 삭제, 이름 바꾸기, 이동, 복사 및 속성 설정을 포함합니다.

File Station 로그 활성화하기:

  1. 모든 사용자의 활동을 기록하려면 File Station 로그 활성화를 선택합니다.
  2. 확인을 클릭합니다.

File Station 로그 보기:

  1. 로그 보기를 클릭하면 로그 센터로 안내합니다.
  2. 파일 전송 탭을 클릭합니다.

끌어다 놓기 업로딩 또는 키보드 바로가기를 사용하여 파일을 자동으로 덮어쓰기:

파일을 컴퓨터 바탕 화면에서 웹 브라우저로 끌어다 놓거나, 키보드 바로가기를 사용하여 File Station 내에 파일을 붙여넣을 때 동일한 이름을 가진 파일을 자동으로 덮어쓰고 싶으면 파일을 웹 브라우저로 끌어 놓거나 키보드의 바로가기를 사용하여 붙여넣을 때 항상 덮어씁니다.를 선택합니다.

스마트 끌어 놓기 활성화하기:

File Station 내에서 파일을 끌어 놓을 때, 이 기능은 전송이 복사 동작인지 또는 이동 동작인지를 자동으로 결정합니다.

  1. 스마트 끌어 놓기 활성화를 선택합니다.
  2. 우연히 다른 사용자가 동일한 이름을 가진 파일을 동일한 위치에 전송하는 경우 항상 두 동작 중 하나가 수행되도록 선택합니다:
    • 덮어쓰기: 전송된 파일이 같은 이름을 가진 파일을 덮어씁니다.
    • 건너뛰기: 전송된 파일은 건너뛰고 같은 이름을 가진 파일을 덮어쓰지 않습니다.
  3. 확인을 클릭합니다.

참고:

스마트 끌어 놓기에 의해 결정된 복사 또는 이동 동작을 변경하려면, 필요에 맞춰 다음 핫키를 적용합니다:

  • 복사 동작으로 전환하기: 끌어 놓기를 하는 동안 Ctrl을 누릅니다.
  • 이동 동작으로 전환하기: 끌어 놓기를 하는 동안 Shift를 누릅니다.

File Station으로 로컬 컴퓨터에서 파일 검색 및 관리하기:

  1. File Station으로 로컬 컴퓨터에서 파일 검색을 누릅니다.
  2. 확인을 클릭합니다.

DSM 사용자 목록을 admin이 아닌 사용자와 공유하기:

이 옵션을 이용하면 admin이 아닌 사용자가 특정 DSM 사용자/그룹이 소유한 파일을 검색하고자 하거나 다른 DSM 사용자와 링크를 공유하고자 할 때 전체 DSM 사용자 목록에 액세스할 수 있습니다.

  1. DSM 사용자 목록을 admin이 아닌 사용자와 공유를 선택합니다.
  2. 확인을 클릭합니다.

기본 UNIX 권한 적용

고급 설정을 클릭하고 파일과 폴더를 업로드하거나 생성할 때 기본 UNIX 권한을 적용하는 옵션을 선택합니다. 적용된 권한은 UNIX command umask에 의해 적용된 권한과 동일합니다. 이 옵션을 활성화하면, UNIX 권한은 파일에 대해서는 644이고 폴더에 대해서는 755입니다. 이 옵션을 비활성화하면, UNIX 권한은 파일에 대해서는 666이고 폴더에 대해서는 777입니다. 기본 umask 값은 022입니다.

참고:

  • Windows ACL 가능 공유 폴더("photo"를 제외한 모든 공유 및 외부 드라이브의 공유)의 경우, Linux 또는 FTP 클라이언트에서 chmod 명령을 실행하여 폴더 및 파일 권한 유형을 Windows ACL에서 UNIX로 변경하십시오.
  • 이 옵션을 활성화하면 다른 프로토콜 사이에서 권한 문제가 일관되지 않을 수 있습니다. 일관성을 유지하려면 이 옵션을 비활성화 상태를 유지하는 것이 좋습니다.

마운트/연결

기본적으로, 모든 사용자는 원격 폴더 또는 가상 드라이브를 마운트할 수 있습니다. 여기에서 원격 폴더 또는 가상 드라이브 관리에 대한 접근을 관리자로만 제한할 수 있습니다.

참고:

관리자가 아닌 사용자는 자신의 home 폴더 안에서만 원격 폴더를 마운트할 수 있습니다.

서로 다른 사용자에게 원격 폴더 또는 가상 드라이브를 마운트할 수 있도록 허용하기:

  1. 원격 폴더 또는 Administrators 섹션에서 모든 사용자나 관리자를 선택합니다.
  2. 확인을 클릭합니다.

서로 다른 사용자에게 서버 및 클라우드 서비스를 마운트하도록 허용하기:

  1. 모든 사용자, 관리자 또는 특정 사용자에서 선택합니다.
  2. 특정 사용자를 선택하면 여러 사용자의 마운트 권한을 구성할 수 있는 창이 열립니다. 특정 사용자에 대한 액세스를 활성화하려면 활성화됨 열 아래에 이 상자가 선택되어 있는지 확인합니다. 특정 사용자에 대한 액세스를 비활성화하려면 활성화됨 열 아래에 이 상자의 선택이 해제되어 있는지 확인합니다. 작업을 마치면 확인을 클릭하여 이 창을 닫습니다.

    3. 참고:

    • administrators 그룹에 속하는 사용자는 이 기능을 사용할 수 있지만 변경할 수는 없습니다. guest로 로그인한 사용자는 이 기능을 사용할 수도 변경할 수도 없습니다. 다른 모든 사용자의 권한을 여기에서 설정할 수 있습니다.
  3. 확인을 클릭합니다.

공유 링크

기본적으로 모든 사용자는 파일 링크를 공유할 수 있습니다. 여기에서 공유된 링크 관리에 대한 접근을 관리자로만 제한할 수 있습니다.

서로 다른 사용자에게 파일 링크를 공유하도록 허용하기:

  1. 모든 사용자, 관리자 또는 특정 사용자를 선택합니다.
  2. 확인을 클릭합니다.

HTML 파일을 일반 텍스트로 열기:

  1. HTML 파일을 항상 일반 텍스트 형식으로 열기 상자를 선택합니다. 이 옵션을 활성화하면 사용자가 파일 공유 링크를 통해 HTML 파일(예: HTM, HTML, JSP, XHTML 파일)을 열 때 일반 텍스트 형식을 사용하여 표시됩니다. 이 옵션은 악의적인 사용자가 HTML 파일에 포함된 스크립트를 이용하여 로그인 세션을 가로채는 것을 막아줍니다.
  2. 확인을 클릭합니다.

속도 제한

File Station을 통해 파일을 전송하는 사용자 또는 그룹이 사용할 수 있는 최대 대역폭을 제어하기 위해 속도 제한을 활성화할 수 있습니다.

속도 제한 활성화하기:

  1. 속도 제한 활성화 상자를 선택하거나, 특정 시간에만 속도 제한을 구현하려는 경우에는 예약에 따라 속도 제한 활성화를 선택합니다. 이는 서버 수준 설정이며, 개별 사용자 및 그룹의 설정에 우선합니다. 속도 제한 없음을 선택하면 개별 사용자 및 그룹의 속도 제한 설정이 비활성화됩니다.
  2. 속도 제한 설정을 클릭하여 개별 사용자 또는 그룹에 대한 설정을 수정합니다. 각 사용자 및 그룹에 대해 다음 중 하나를 선택할 수 있습니다:
    • 그룹 설정 적용: 속도 제한 설정은 사용자가 속하는 그룹의 설정과 동일하게 됩니다.
    • 속도 제한 없음: 그룹 구성원이 서비스를 사용하여 파일을 전송할 때 속도 제한 없음이 적용됩니다.
    • 상시: 사용자 및 그룹 구성원에게 고정된 일정한 속도 제한이 적용됩니다. 왼쪽의 필드에 업로드 및 다운로드 속도 제한을 지정할 수 있습니다.
    • 가변: 서로 다른 두 가지 속도 제한을 지정하여 사용자 또는 그룹 구성원에게 예약에 따라 적용할 수 있습니다. 사용자 지정 단추를 클릭하여 속도 제한 설정을 수정하고 일정을 설정합니다.
  3. 적용을 클릭합니다.

참고:

  • 그룹 속도 제한에 대한 자세한 내용은 여기를 참조하십시오.
  • 사용자에 대해 속도 제한을 설정할 때, 결과 섹션에는 사용자의 개인 및 그룹 설정에 따라 실제 속도 제한이 표시됩니다. 예를 들어, 사용자 "Achilles"의 개인 다운로드 제한이 100 KB/s지만, 다운로드 제한이 500 KB/s인 그룹에 속하는 경우, 결과 섹션에는 "0/500"이 표시됩니다.

HTML 파일을 일반 텍스트로 열기:

  1. HTML 파일을 항상 일반 텍스트 형식으로 열기 상자를 선택합니다. 이 옵션을 활성화하면 File Station에서 HTML 파일(예: HTM, HTML, JSP, XHTML 파일)을 열 때 이 파일이 일반 텍스트 형식을 사용하여 표시됩니다. 이 옵션은 악의적인 사용자가 HTML 파일에 포함된 스크립트를 이용하여 로그인 세션을 가로채는 것을 막아줍니다.

    2. 참고:

    • 이것은 전역 설정이 아닙니다. 각 사용자는 개인의 필요에 따라 이 옵션을 활성화할지 여부를 선택할 수 있습니다.
  2. 확인을 클릭합니다.

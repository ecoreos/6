종료 및 재설정

아래 지침에서는 high-availability 클러스터를 종료, 제거 또는 재설정하는 과정을 안내합니다.

Synology High Availability를 사용하여 high-availability 클러스터 종료하기:

새로운 실제 위치에 서버를 이동하거나 유지 보수를 수행하기 위해 high-availability 클러스터를 종료할 때 high-availability 클러스터 구성을 제거하지 않으려면 이 지침을 따르십시오.

  1. DSM에 로그인합니다.
  2. Synology High Availability를 엽니다.
  3. 관리라고 표시된 단추를 클릭합니다.
  4. HA 클러스터 종료를 선택합니다.
  5. 확인 메시지가 나타납니다. 예를 클릭하여 확인합니다.

High-availability 클러스터 수동으로 종료하기:

High-availability 클러스터는 각 서버에 있는 전원 단추를 사용하여 수동으로 종료할 수도 있습니다.

  1. 신호음이 울릴 때까지 하나의 서버(활성 또는 수동)에서 전원 단추를 길게 누릅니다.
  2. 하나의 서버가 완전히 종료되면 30초 이상 기다립니다(아직 온라인 상태인 서버는 설정을 저장하는 데 시간이 걸립니다).
  3. 신호음이 울릴 때까지 다른 서버의 전원 단추를 길게 누릅니다.
  4. 서버가 완전히 종료될 때까지 기다립니다.

서버 켜기:

서버를 다시 켜려면, 활성 서버와 수동 서버를 동시에 켜거나 5분 간격 이내에 켜야 합니다. 서버 중 하나만 켜면 LAN 깨우기 메시지가 브로드캐스트되어 상대편 서버의 전원을 켜려고 시도합니다.

비정상 종료:

high-availability 클러스터가 생성되면 제어판에 있는 전원 차단 후 다시 시작 옵션이 자동으로 활성화됩니다. 전원 차단으로 인해 서버가 비정상적으로 종료될 경우 전원이 복구되면 자동으로 서버가 켜집니다.

high-availability 클러스터 제거:

다른 서버와 연결하기 위해 두 서버의 연결을 해제하고 high-availability 클러스터를 제거하려면 이 지침을 따르십시오.

  1. 개요 페이지로 이동합니다.
  2. 관리 단추를 클릭합니다.
  3. HA 클러스터 제거를 선택합니다.

확인하고 나면 시스템에서 high-availability 클러스터를 제거합니다. 상태 필드에 제거 프로세스 진행률이 표시됩니다. High-availability 클러스터가 제거되면 활성 서버의 원래 IP 주소로 연결됩니다. 두 서버의 상태가 독립 실행형으로 변경됩니다.

오작동하는 high-availability 클러스터 재설정하기:

브레인 분할 또는 다른 오작동으로 인해 DSM 또는 Synology High Availability에 액세스할 수 없는 경우 서버에 있는 Reset 단추를 눌러 high-availability 클러스터에서 연결을 해제하도록 시도할 수 있습니다.

  1. 서버에서 신호음이 울릴 때까지 일반적으로 서버의 후면 패널에 있는 Reset 단추를 길게 누릅니다.
  2. 시스템에서 high-availability 클러스터가 제거되고 두 서버의 상태가 독립 실행형으로 변경됩니다
  3. Reset 단추를 누르면 다음과 더불어 IP 주소, DNS 및 admin 패스워드와 같은 일부 설정이 기본값으로 복원됩니다.
    • Admin 패스워드가 공백으로 재설정됩니다.
    • Guest 계정이 비활성화됩니다.
    • IP, DNS, 게이트웨이 및 다른 네트워크 인터페이스에서 DHCP가 활성화됩니다.
    • JumboFrames가 비활성화되고 MTU가 1500으로 설정됩니다.
    • 관리 포트가 5000/5001로 재설정됩니다.
    • PPPoE가 기본값으로 재설정됩니다.
    • IP 자동 차단이 비활성화됩니다(차단된 IP 주소는 지워지지 않음).
    • Link Aggregation이 비활성화됩니다.
    • 방화벽 규칙이 비활성화됩니다(규칙은 지워지지 않음).

참고:

High-availability 클러스터를 제거하는 동안 일부 데이터 쓰기 작업이 완료되지 않을 수 있습니다. 수동 서버에는 일부 데이터 업데이트가 누락되었을 수 있기 때문에 완전한 데이터를 유지하려면 활성 서버를 토대로 데이터 복제를 수행하십시오.

네트워크 설정

이 페이지에는 high-availability 클러스터, 활성 구성원 서버, 수동 구성원 서버의 네트워크 설정을 표시하는 3개의 탭이 있습니다.

HA 클러스터 탭

High-availability 클러스터를 생성하면 시스템에서 high-availability 클러스터 IP용 공통 IP 주소를 생성하여 들어오는 연결을 수신합니다. 필요에 따라 high-availability 클러스터 IP 주소를 추가로 생성할 수 있습니다. 활성 서버에서 활성화된 네트워크 인터페이스의 IP 주소는 이 탭에 표시됩니다. 심장박동 연결에 사용된 네트워크 인터페이스는 두 서버 간의 데이터 복제에 사용되므로 이 페이지에 표시되지 않습니다. 이 탭에는 아래 정보가 표시됩니다.

  • 네트워크 인터페이스: 활성 서버에서 활성화된 네트워크 인터페이스입니다.
  • 연결: 이 인터페이스가 연결되는지 여부를 표시합니다.
  • IP 주소: High-availability 클러스터를 생성하면 시스템에서 high-availability 클러스터 IP용 공통 IP 주소를 생성하여 들어오는 연결을 수신합니다. DiskStation Manager(DSM)에 로그인할 때 어느 Synology 서버가 활성 서버인지 여부에 관계 없이 이 IP 주소를 사용하여 high-availability 클러스터에 연결할 수 있습니다. 이것은 전환이 발생한 후에 다른 IP 주소로 연결하지 못하도록 합니다.
  • 서브넷 마스크: 지정된 high-availability 클러스터 IP 주소와 연결된 서브넷 마스크입니다.

IP 주소와 서브넷 마스크 편집하기:

  1. 편집할 IP 주소와 서브넷 마스크를 선택하고 편집을 클릭하거나 필드를 더블 클릭하여 편집을 시작합니다.
  2. 활성화 확인란을 선택하여 각 인터페이스를 활성화 또는 비활성화할 수 있습니다. 둘 이상의 클러스터 네트워크 인터페이스를 활성화할 수 있지만 클러스터 생성 중에 구성한 원래 네트워크 인터페이스는 비활성화할 수 없습니다.

High-availability 클러스터 서버 이름 변경하기:

  1. 네트워크 설정 변경을 클릭하여 high-availability 클러스터 이름을 보고 설정을 변경합니다.
  2. 적용을 클릭하여 현재 high-availability 클러스터 이름과 IP 주소 설정을 저장합니다.
  3. 현재 변경 사항을 취소하려면 취소를 클릭합니다.

활성 및 수동 서버 탭(서버 이름으로 표시됨)

이 탭에는 활성 서버 또는 수동 서버의 네트워크 인터페이스 상태가 표시됩니다. 심장박동 연결은 각 활성 서버와 수동 서버에서 하나의 네트워크 인터페이스로 구성됩니다. 이 탭에는 다음 정보가 표시됩니다.

  • 네트워크 인터페이스: 서버 상의 물리적 네트워크 인터페이스입니다.
  • IP 주소: 지정된 네트워크 인터페이스에 할당된 IP 주소입니다.
  • 서브넷 마스크: 지정된 IP 주소와 연결된 서브넷 마스크입니다.
  • 설명: 지정된 네트워크 인터페이스에 대한 정보와 할당된 IP 주소에 대한 액세스 가능 여부가 표시됩니다. 이 필드에는 다음과 같은 항목의 가능 여부 상태가 나타납니다.
    1. 원래 IP 주소에 액세스할 수 있음: 활성 서버의 인터페이스에 할당된 원래 IP 주소에 액세스하여 데이터와 서비스를 사용할 수 있습니다.
    2. 원래 IP 주소에 액세스할 수 없음: 수동 서버의 인터페이스에 할당된 원래 IP 주소에 액세스할 수 없습니다.
    3. 심장박동 연결 IP, 인트라넷에서 액세스할 수 없음: 데이터와 서비스 사용을 위해 심장박동 연결에 자동으로 할당된 원래 IP 주소에 액세스할 수 없습니다.

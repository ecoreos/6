브레인 분할

High-availability 클러스터가 정상적으로 작동하는 경우 한 개의 구성원 서버만 활성 서버 역할을 담당합니다. 이 경우, 수동 서버는 Heartbeat 연결 및 데이터 연결을 통해 활성 서버가 있는지 감지합니다.

모든 Heartbeat 및 데이터 연결이 손실된 경우, 두 서버 모두 활성 서버의 역할을 가정하려고 시도할 수 있습니다. 이런 상황을 "브레인 분할" 오류라고 합니다. 이 경우, high-availability 클러스터의 IP 주소 연결은 두 서버 중 하나로 리디렉션되며, 일관성이 없는 데이터는 두 서버에서 업데이트되거나 쓰여질 수 있습니다.

Heartbeat 또는 데이터 연결 중 하나가 다시 연결되면, 시스템이 두 서버 간에 브레인 분할 오류 및 데이터 불일치를 감지하고 high-availability 안전 모드로 들어갑니다.

브레인 분할 오류가 발생하는 경우:

  1. high-availability 클러스터의 IP 주소 및 두 서버 모두에 있는 서비스는 브레인 분할 오류가 해결되어야 사용할 수 있습니다.
  2. 두 서버가 high-availability 안전 모드에 들어가면 브레인 분할이라는 새 탭이 왼쪽 패널에 표시됩니다. 이 탭에 다음 정보가 나열됩니다: 두 서버의 공유 폴더에 있는 파일 간 차이, 서버가 활성 서버로 된 시간, 최종 iSCSI Target 연결 정보. 다른 모든 탭은 읽기 전용으로 유지됩니다.
  3. high-availability 안전 모드에서, File Station은 읽기 전용 모드가 되고, 파일을 다운로드하거나 볼 수 있습니다.
  4. 개요 탭에서는 브레인 분할 오류를 해결하거나 현재 로그인 서버를 종료하는 것만 가능합니다. 브레인 분할 오류를 해결하려면 다음을 수행합니다:
    • 한 서버를 high-availability 클러스터의 활성 서버로 선택하고 나머지 서버를 수동 서버로 선택합니다. 두 서버가 모두 재부팅된 후, 활성 서버의 다른 모든 데이터 및 설정이 수동 서버로 동기화됩니다. 브레인 분할 오류 동안 수동 서버에 있는 업데이트된 데이터가 손실된다는 점에 유의합니다.
    • 한 서버를 high-availability 클러스터의 활성 서버로 선택하고 나머지 서버의 연결을 해제합니다. 두 서버가 모두 재부팅된 후, 활성 서버는 high-availability 클러스터에 남고, 연결이 해제된 서버는 데이터를 보존하고 독립 실행형 상태로 돌아갑니다. 나중에 새 수동 서버를 연결하려면 전체 복제가 필요하다는 점에 유의하십시오.
    • 두 서버 모두를 연결 해제하고, 이들의 개별 데이터를 보존하고, 독립 실행형 상태로 돌아가게 합니다.

참고:

  • 공유 폴더에 파일이 많을수록, 차이점을 나열하는 데 시간이 오래 걸립니다.
  • 서버를 활성/수동 서버로 선택하기 전에, 두 서버 모두 전원이 켜져 있는지 확인하십시오.

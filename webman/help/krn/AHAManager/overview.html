개요

이 페이지에는 high-availability 클러스터와 구성원 서버에 대한 기본 정보가 표시됩니다. 또한 high-availability 클러스터와 구성원 서버에서 유지 보수를 수행하기 위한 도구도 제공됩니다.

High-Availability 클러스터 상태

이 필드에는 high-availability 클러스터의 전체 상태가 표시되어 사용자에게 오류를 알려줍니다. 다음은 이 필드에 표시되는 가능한 상태 표입니다.

상태 설명
정상 High-availability 클러스터의 전체 상태가 양호합니다.
로드 중 시스템이 클러스터 상태를 로드 중입니다. 상태 로드를 마치기 전에는 전환을 수행할 수 없습니다. 전환을 참조하십시오.
구성 중 시스템을 구성하는 중입니다. 구성을 마치기 전에는 전환을 수행할 수 없습니다.
경고 HA 클러스터의 상태는 비정상이지만, 서비스가 여전히 기능을 하고 있습니다. 이러한 상황에서는 전환을 수행할 수 없습니다.
위험 활성 및 수동 서버가 정상적으로 작동하지 않거나 저장소에서 문제가 발생했습니다. High-availability 클러스터를 정상적으로 작동할 수 없습니다. 모든 저장소 공간(RAID Group, 디스크 그룹, 볼륨 및 iSCSI LUN)의 상태를 확인하십시오. 이러한 상황에서는 전환을 수행할 수 없습니다.

경고 메시지

다음 표에는 시스템에서 오류가 발생할 경우 나타날 수 있는 경고 메시지가 나와 있습니다. 경고 메시지가 여러 개 나타난 경우에는 앞으로/뒤로 단추를 사용하여 메시지 간에 이동할 수 있습니다.

경고/오류 설명
High-availability 클러스터에 수동 서버가 없습니다 수동 서버의 정보를 사용할 수 없습니다. 수동 서버가 올바르게 연결되어 있는지 확인하십시오. 이 경우 전환을 수행할 수 없습니다.
수동 서버를 검색하지 못함 이 오류는 수동 서버가 오프라인이거나 Heartbeat 연결을 통해 활성 서버와 수동 서버가 통신할 수 없는 경우에 발생할 수 있습니다. 이 경우 전환을 수행할 수 없습니다.
대체 작동 수행 실패 활성 서버에서 대체 작동이 트리거되었지만 실패했습니다. 이 경우, 활성 서버는 또다른 대체 작동을 시도하지 않습니다. 이 오류를 없애려면 아래 단계를 수행하십시오:
  1. High-availability 클러스터에서 문제를 처리하도록 시도해 봅니다.
  2. 수동 서버를 재부팅합니다.
  3. 관리 > 전환을 클릭하여 전환을 수행합니다.
모니터링되는 대상 오류 시스템 상태, 서비스 또는 네트워크 인터페이스를 비롯하여 모니터링되는 대상 하나 이상에서 오류가 발생했습니다.
확장 유니트와 활성/수동 서버 간의 연결에서 오류가 발생했습니다. 확장 유니트가 활성 또는 수동 서버에 올바르게 연결되어 있지 않습니다. 활성 서버에 대한 연결이 올바른지 확인하고 수동 서버에 대해서도 동일한 연결을 설정하십시오. 이 경우 전환을 수행할 수 없습니다.
볼륨의 성능이 저하되었습니다. 저장소 공간(예: RAID Group, 디스크 그룹, 볼륨, iSCSI LUN 및 SSD 캐시)이 저하되었습니다. 저장소 상태 페이지를 확인하십시오.
볼륨에서 문제가 발생했습니다. 저장소 공간(예: RAID Group, 디스크 그룹, 볼륨, iSCSI LUN 및 SSD 캐시)에서 문제가 발생했습니다. 저장소 상태 페이지를 확인하십시오. 수동 서버가 활성 서버에서 액세스하지 못한 드라이브를 인식한 경우 대체 작동이 자동으로 수행됩니다. 그렇지 않은 경우 전환을 사용할 수 없습니다.
Heartbeat 연결이 끊겼습니다. 이 오류는 Heartbeat 연결 실패로 인해 발생할 수 있습니다. 이 경우 전환을 수행할 수 없습니다.
하드웨어 장애 이 오류는 활성 서버 또는 수동 서버의 하드웨어(예: 전원 장치 또는 시스템 팬) 장애로 인해 발생할 수 있습니다.
다른 물리적 RAM 크기 이 오류는 활성 서버 및 수동 서버에서의 물리적 RAM 크기 불일치로 인해 발생할 수 있습니다. 이 경우 전환을 수행할 수 없습니다.
이 high-availability 클러스터에 정적 IP가 할당되지 않았습니다. 이 오류는 high-availability 클러스터에 정적 IP가 없기 때문에 발생할 수 있습니다. 이 경우 전환을 수행할 수 없습니다.
모니터링되는 인터페이스에 대한 정적 IP가 없습니다. 이 오류는 모니터링되는 인터페이스에 대한 정적 IP가 없기 때문에 발생할 수 있습니다. 이 경우 전환을 수행할 수 없습니다.
수동 서버 복구 중 수동 서버 복구가 진행 중입니다. 이 경우 전환을 수행할 수 없습니다.
수동 서버 업그레이드 중 시스템에서 수동 서버를 업그레이드 중입니다. 이 경우 전환을 수행할 수 없습니다.

High-Availability 클러스터 "관리" 단추

이 단추는 high-availability 클러스터 정보 바로 아래에 있습니다. 다음과 같은 작업이 포함되어 있습니다.

  • 전환: 활성 서버에서 수동 서버로의 전환을 시작합니다. 이 프로세스 중에는 활성 서버가 서비스 제공을 중지하고 새로운 수동 서버가 됩니다. 그런 다음 이전 수동 서버가 새로운 활성 서버로 되어 high-availability 클러스터의 모든 서비스를 제공하기 시작합니다. 이 옵션은 수동 서버가 오류 상태에 있거나 선택되지 않은 경우에는 사용할 수 없습니다.
  • DSM 업데이트: 제어판의 DSM 업데이트 페이지를 엽니다. 이곳에서 두 서버를 업데이트할 수 있습니다.
  • High-availability 클러스터 종료: 활성 서버 및 수동 서버를 둘다 종료합니다.
  • 활성 서버 종료: 활성 서버를 종료합니다. 수동 서버가 정상 상태인 경우 서비스가 전환됩니다.
  • 수동 서버 종료: 수동 서버를 종료합니다.
  • 활성 서버 재부팅: 활성 서버를 재부팅합니다. 수동 서버가 정상 상태인 경우 서비스가 전환됩니다.
  • 수동 서버 재부팅: 수동 서버를 재부팅합니다.
  • 알 수 없는 서버 복구: 수동 서버가 교체되었거나 여전히 알 수 없는 서버로 인식되는 경우 수동 서버를 복구합니다.
  • 시스템 음소거: 오류가 발생할 때 활성 서버 및 수동 서버의 신호음이 정지됩니다. 나중에 다른 오류가 발생하면 활성 서버 또는 수동 서버에서는 계속해서 신호음이 울립니다.

참고:

  • 수동 서버를 종료했거나 수동 서버가 없는 경우 활성 서버에서 오류가 발생하면 서비스를 전환할 수 없습니다.

서버 다이어그램 및 정보

서버 다이어그램에는 서버 상태가 표시됩니다. 위치는 서버가 연결된 모듈의 측면에 해당합니다. 마우스를 올려 놓으면 자세한 정보를 확인할 수 있습니다.

다음과 같은 상황이 포함되어 있습니다.

  • High-availability 클러스터에 수동 서버가 없음: 빈 프레임이 표시됩니다.
  • 수동 서버를 검색하지 못함: 흐릿한 서버 이미지가 표시됩니다.
  • 느낌표: 하드웨어가 정상적으로 작동하지 않을 수 있습니다. 또는 Heartbeat 연결 실패로 인해 서버 정보를 확인할 수 없습니다.

참고:

  • 다른 서버에 대해 "알 수 없음"이 표시되면 해당 서버가 high-availability 클러스터에 속해 있지 않음을 나타냅니다. 이 문제를 해결하려면 관리를 클릭하고 알 수 없는 서버 복구를 선택합니다.

서버 정보에는 다음 항목이 포함됩니다. 감지된 수동 서버가 없는 경우 해당 없음이 표시됩니다.

  • 시리얼 번호: Synology 서버의 일련 번호입니다.
  • 팬 상태: 서버의 냉각 팬 상태입니다. 팬 중 하나가 정상적으로 작동하지 않는 경우 경고가 나타납니다. 이 경우에는 지역 대리점으로 도움을 요청하십시오.
  • 온도: Synology 서버의 온도입니다.
  • 전원 상태: 서버의 전원 공급 장치가 모두 정상적으로 작동하는지 여부를 나타냅니다. 서버에 여유 전원 공급 장치가 포함된 경우 둘 중 하나가 오작동하면 경고가 표시됩니다.
  • 물리적 메모리: Synology 서버의 물리적 RAM 크기입니다. 두 서버의 물리적 RAM 크기가 동일하지 않은 경우에는 전환을 수행할 수 없습니다.

서버 간 연결 상태

이 섹션에는 활성 서버와 수동 서버 사이의 이미지가 포함되어 있습니다. 다음과 같은 상황이 포함되어 있습니다.

  • 이미지 없음: 감지된 수동 서버가 없음을 나타냅니다.
  • 정지 이미지: 잘못된 Heartbeat 연결을 나타냅니다.
  • 느낌표: 두 서버 간의 비정상적인 통신 또는 잘못된 Heartbeat 연결을 나타냅니다.

네트워크 인터페이스 상태

이 섹션에는 서버와 데이터 네트워크 사이의 이미지가 포함되어 있습니다. 다음과 같은 상황이 포함되어 있습니다.

  • 느낌표 없음: 감지된 네트워크 장애가 없음을 나타냅니다.
  • 느낌표: 모니터링된 네트워크 인터페이스에서 장애가 발생했음을 나타냅니다.

공유 저장소 다이어그램 및 정보

공유 저장소 다이어그램에는 서버와 공유 저장소 간의 연결 상태가 표시됩니다. 마우스를 올려 놓으면 기본 정보를 확인할 수 있습니다.

다음과 같은 상황이 포함되어 있습니다.

  • 느낌표 없음: 감지된 오류가 없음을 나타냅니다.
  • 느낌표: 확장 유니트의 연결 상태가 비정상적임을 나타냅니다.

공유 저장소 기본 정보에는 다음 항목이 포함됩니다. 자세한 정보를 확인하려면 저장소 관리자로 이동하십시오.

  • 팬 상태: 확장 유니트의 팬 상태를 표시합니다. 팬이 오작동하면 경고가 나타납니다. 이 경우에는 지역 대리점으로 도움을 요청하십시오.
  • 전원 상태: 서버의 전원 공급 장치가 모두 정상적으로 작동하는지 여부를 나타냅니다. 전원 공급 장치 중 하나가 정상적으로 작동하지 않는 경우 경고가 나타납니다.

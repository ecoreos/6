High-availability 클러스터 상태

이 필드에는 다음과 같이 high-availability 클러스터의 상태가 표시됩니다.

  • 독립 실행형: Synology NAS가 high-availability 클러스터의 일부가 아닙니다.
  • 정상: High-availability 클러스터가 구성되었습니다. 활성 서버와 수동 서버의 데이터가
    • 동기화되고 모든 서비스가 정상적으로 작동합니다.
  • 준비 중: High-availability 클러스터가 구성되었습니다. 활성 서버와 수동 서버의 데이터 동기화가 아직 완료되지 않았습니다.
  • 경고： High-availability 클러스터가 구성되었습니다. 그러나 특정 서비스나 데이터 동기화가 정상적으로 작동하지 않습니다. 이 상황에서는 활성 서버에서 제공하는 서비스는 정상적으로 유지되지만 수동 서버로 전달할 수 없습니다. 경고 상태를 유발하는 원인은 다음과 같습니다.
    • 활성 서버 상태 오류: 이것은 일시적인 상황입니다. 이 경우에는 high-availability 클러스터에서 수동 서버로 서비스를 전송하고 서버 역할이 바뀝니다. 활성 서버 오작동은 다음이 원인일 수 있습니다.
      1. 모니터링한 서비스의 오류.
      2. 활성 서버 저장소 볼륨이 문제 발생 또는 충돌함. 저장소 상태 UI를 확인하십시오.
      3. 활성 서버 디스크 오류. 디스크 상태 UI의 활성 서버 탭을 확인하십시오.
    • 수동 서버 상태 오류. 저장소 상태 및 디스크 상태 UI를 확인하여 오작동 원인을 파악하십시오.
      1. 수동 서버 저장소 볼륨이 충돌함 또는 문제 발생. 저장소 상태 UI를 확인하십시오.
      2. 수동 서버 디스크 오류. 디스크 상태 UI의 수동 서버 탭을 확인하십시오.
    • 수동 서버를 검색하지 못한 경우 수동 서버 전원이 꺼져 있거나 심장박동 연결을 통해 활성 서버와 수동 서버를 연결하지 못한 것이 원인일 수 있습니다.
    • 수동 서버로 데이터를 동기화하지 못한 경우 저장소 볼륨 또는 디스크 오류일 수 있습니다. 저장소 상태 UI를 확인하십시오.
  • 수동 서버를 선택하지 않음: High-availability 클러스터에서 수동 서버를 제거하고 클러스터는 그대로 남아 있는 경우 클러스터에 이 상태가 표시될 수 있습니다. 일반적으로 이 상황은 유지 관리를 위해 수동 서버 연결을 해제한 경우에 발생합니다. 유지 관리가 완료되면 유지 관리 > 연결을 클릭하여 수동 서버를 high-availability 클러스터로 복귀시키십시오.
  • 위험: 활성 서버와 수동 서버가 모두 오작동하는 경우 high-availability 클러스터를 정상적으로 작동할 수 없습니다. 작동이 중단된 경우 high-availability 클러스터에서 서비스를 신속하게 다시 시작할 수 없습니다.

High-availability 클러스터 정보

High-availability 클러스터는 활성화한 후에 활성 서버와 수동 서버로 구성됩니다. 클러스터에 대한 정보는 high-availability 클러스터 상태 아래에 표시됩니다. High-availability 클러스터가 활성화되지 않은 경우 high-availability 클러스터 상태가 독립 실행형으로 표시됩니다. 이 필드는 클러스터 정보가 아니라 high-availability 클러스터의 기능만 나타냅니다.

  • 클러스터 서버 이름: High-availability 클러스터에서 사용하는 서버 이름입니다.
  • 클러스터 IP 주소:High-availability 클러스터에서 사용하는 IP 주소입니다. 어느 서버에서 서비스를 실행하는지에 관계 없이 사용자는 이 IP 주소를 사용하여 high-availability 클러스터에 연결할 수 있습니다.
  • HA 클러스터 작동 시간： High-availability 클러스터를 활성화한 후로 경과된 시간입니다.

High-availability 클러스터 작동 단추

High-availability 클러스터 정보에는 클러스터 작동과 관련된 두 개의 단추가 있습니다.

  • 서비스 전환: 이 단추를 클릭하면 현재 실행하는 서비스가 수동 서버로 전환됩니다. 전환이 완료되면 활성 서버가 수동 서버가 되고 수동 서버는 활성 서버가 됩니다. 이 옵션은 수동 서버가 오작동하거나 수동 서버를 선택하지 않은 경우에는 사용할 수 없습니다.
  • HA 클러스터 활성화/비활성화: 이 단추는 현재 상태에 따라 다른 작업을 수행합니다. High-availability 클러스터가 비활성화된 경우 이 단추를 사용하여 HA 클러스터 활성화를 수행합니다. High-availability 클러스터가 활성화된 경우 이 단추를 사용하여 HA 클러스터 비활성화를 수행합니다. HA 클러스터 활성화 기능을 수행하는 경우 단추를 클릭하면 High Availability Manager 마법사가 high-availability 클러스터 설정을 안내합니다.

서버 동기화 상태

Synology NAS 이미지와 데이터 흐름 이미지에는 현재 서버 상태가 나타납니다.

  • Synology NAS 상태 이미지： 활성 서버와 수동 서버 모두 표시됩니다. 상태는 다음과 같이 나타납니다.
    1. 상태: 독립 실행형. 한쪽 서버에는 파란색 표시등이 켜지고 다른 서버에 점선 윤곽선이 나타납니다.
    2. 상태: 정상. 두 서버 모두에 녹색 표시등이 나타납니다.
    3. 상태: 경고. 문제가 있는 서버에 노란색 표시등이 나타납니다.
    4. 상태: 위험. 문제가 있는 서버에 빨간색 표시등이 나타납니다.
    5. 상태: 수동 서버를 검색하지 못했습니다. 오프라인 서버가 반투명 이미지로 나타납니다.
    6. 상태: 수동 서버를 선택하지 않음. 수동 서버에 점선 윤곽선이 나타납니다.
  • 데이터 흐름 방향 이미지: 데이터 동기화 방향을 나타냅니다.
    1. 상태: 데이터 동기화가 정상적으로 작동합니다. 활성 서버에서 수동 서버로 데이터가 흐르는 것으로 나타납니다.
    2. 상태: 데이터 동기화 안됨. 데이터 흐름이 없는 이미지로 나타납니다.

서버 정보

활성 서버와 수동 서버에 대한 정보는 이 섹션에 표시됩니다. 활성 서버와 수동 서버는 모두 개별적인 Synology NAS입니다. Synology NAS를 수동 서버로 선택하지 않은 경우 해당 정보는 해당 없음으로 표시됩니다.

  • 서버 이름: Synology NAS 서버 이름입니다.
  • 모델 이름: Synology NAS 모델입니다.
  • 일련 번호: 서버의 일련 번호입니다.
  • 팬 상태: 오작동이 발생하면 경고가 표시됩니다. 팬 상태는 CPU 팬 실패 및 시스템 팬 실패로 나타납니다. 팬이 오작동하면 지역 대리점으로 도움을 요청하십시오.
  • 온도: Synology NAS의 현재 시스템 온도입니다.
  • 전원 상태: Synology NAS 전원 공급 장치가 정상적으로 작동하는지 여부를 나타냅니다. Synology NAS 모델에 전원 소수가 두 개 포함된 경우 둘 중 하나가 오작동하면 경고가 표시됩니다.

유지 관리 작업 단추

유지 관리 드롭다운 메뉴는 각 Synology NAS 서버 아래 있습니다. 드롭다운 메뉴에는 수동 서버 유지 관리를 위한 작업 단추가 포함되어 있습니다. 따라서 수동 서버 아래 메뉴만 사용할 수 있습니다. High-availability 클러스터가 비활성화되면 두 개의 유지 관리 메뉴도 모두 비활성화됩니다.

  • 종료: 현재 수동 서버를 종료합니다. 수동 서버를 종료한 후에는 유지 관리할 수 있습니다. 종료 후에도 서버는 high-availability 클러스터의 일부로 유지됩니다. 이 단추는 수동 서버가 이미 꺼져 있거나 수동 서버를 선택하지 않은 경우에는 비활성화됩니다.
  • 연결/연결 해제:현재 high-availability 클러스터에 수동 서버가 포함되어 있는 경우 이 단추가 연결 해제로 표시됩니다. 연결 해제를 클릭하면 high-availability 클러스터에서 수동 서버가 제거됩니다. High-availability 클러스터에 수동 서버가 없는 경우 이 단추가 연결로 표시됩니다. 이 단추를 클릭하면 수동 서버의 정보로 들어가서 high-availability 클러스터에 추가할 수 있습니다.

참고: 수동 서버를 종료하거나 연결 해제하면 high-availability 클러스터에 수동 서버가 포함되지 않거나 활성 서버에서 수동 서버로 서비스를 전환할 수 없게 됩니다.

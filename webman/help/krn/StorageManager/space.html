볼륨

볼륨은 사용자가 공유 폴더, iSCSI LUN(일반 파일)을 생성하고, 데이터를 저장하고, 패키지를 설치할 수 있는 기본 저장소 공간입니다. 볼륨을 생성하기 전에 RAID Group을 미리 생성하십시오. RAID Group에 대한 자세한 정보는 RAID Group을 참조하십시오.

볼륨 생성하기:

  1. DiskStation이(가) 할당 가능한 공간이 있는 기존 RAID Group을 가지고 있는지 확인하십시오.
  2. 생성 단추를 클릭합니다.
  3. 볼륨을 생성할 RAID Group을 선택합니다.
  4. 원하는 설정을 수정합니다.
  5. 볼륨에 크기 할당을 설정합니다. (단일 볼륨 또는 iSCSI LUN용 RAID Group에서 생성한 볼륨에서는 크기 할당 설정을 사용할 수 없습니다)

참고:

  • 할당할 수 있는 최대 볼륨 크기는 200 TB입니다.
  • Btrfs 볼륨에 할당되는 최소 크기는 적어도 10GB라야 합니다.
  • Btrfs 볼륨은 메타데이터 공간 중 4%를 차지합니다.

볼륨 제거하기:

  1. 제거할 볼륨을 선택합니다.
  2. 제거 버튼을 클릭하고 마법사를 따라 프로세스를 완료합니다.

참고:

  • 볼륨을 제거하면 해당 볼륨의 기존 데이터가 모두 지워집니다.

볼륨 강제 마운트하기:

볼륨의 캐시가 손실된 경우 시스템이 부팅할 때 볼륨을 마운트하지 않습니다. 캐시 복원이 불가능할 때 볼륨에 액세스하고 싶으면 강제 마운트를 클릭할 수 있습니다.

참고:

  • 이 기능은 특정 모델에서만 사용할 수 있습니다.

볼륨 복구하기:

볼륨 상태가 문제 발생으로 표시되면 RAID Group 탭으로 이동한 다음 알맞은 RAID Group을 찾습니다. 권장 지침에 따라 볼륨을 복구합니다.

볼륨의 크기 확장하기

  • 여러 볼륨 또는 iSCSI LUN(블록 수준)용 RAID Group에서 생성한 볼륨.
    1. 확장할 볼륨을 선택합니다.
    2. 편집을 클릭합니다.
    3. 새 크기를 입력하고 확인을 클릭하여 확장합니다.
  • 단일 볼륨/iSCSI LUN(블록 수준)용 RAID Group에서 생성한 볼륨의 경우 RAID Group의 RAID Group의 크기 확장하기 섹션을 참조하십시오. 볼륨 크기는 기본 RAID Group이 확장된 후에 확장됩니다.

SSD TRIM

볼륨이 완전히 SSD(Solid State Drive)로 구성된 경우 SSD TRIM을 활성화할 것을 권장합니다. 이 기능은 SSD에 생성된 볼륨의 읽기 및 쓰기 성능을 향상시켜 SSD의 효율성을 높이고 수명을 연장합니다.

SSD TRIM 활성화하기:

  1. 저장소 관리자를 열고 볼륨으로 이동합니다.
  2. 전체가 SSD로 구성된 볼륨을 선택한 다음 구성 단추를 클릭합니다.
  3. 나타나는 창에서 SSD TRIM 탭으로 이동하고 TRIM 활성화 상자를 선택합니다.
  4. 예약 설정 단추를 클릭하고 시스템에 언제 SSD TRIM을 수행할지 선택합니다.
  5. 확인을 클릭합니다.

참고:

  • SSD TRIM은 특정 DiskStation 모델 및 RAID 유형에서만 사용 가능합니다.
  • 일부 SSD 모델은 RAID 5와 RAID 6 볼륨에서 SSD TRIM을 수행할 수 없습니다. www.synology.com에서 호환성 목록을 참조하십시오.
  • 다음 상황에서는 SSD TRIM을 활성화할 수 없습니다:
    • 볼륨이 SSD 캐시로 마운트되어 있습니다.
    • Time Backup 패키지가 설치되어 있습니다.
    • high-availability 클러스터가 High Availability 관리자를 사용하여 생성되었습니다.
  • SSD TRIM을 블록 수준 iSCSI LUN에서 활성화할 수 없습니다.
  • SSD TRIM을 수행할 때 시스템 성능이 영향을 받습니다. TRIM 수행을 완료하는 데 필요한 시간은 볼륨 크기와 SSD 모델에 따라 다릅니다.
  • 볼륨에서 TRIM이 활성화되어 있을 때, 이는 같은 디스크 그룹 또는 RAID Group에 만든 다른 볼륨에서도 활성화됩니다.

파일 시스템 스크럽*

파일 시스템 스크럽은 볼륨을 검사하는 데이터 유지 보수 기능입니다. 체크섬과 일치하지 않는 데이터가 감지되면 파일 경로가 로그 센터에 기록되고 백업을 사용하여 데이터를 복구하는 것이 좋습니다.

파일 시스템 스크럽을 정기적으로 수행하여 데이터 일관성을 보장하는 것이 좋습니다.

파일 시스템 스크럽 수행하기:

  1. 저장소 관리자를 열고 볼륨으로 이동합니다.
  2. 파일 시스템 스크럽을 수행할 볼륨을 선택합니다.
  3. 관리를 클릭합니다.
  4. 파일 시스템 스크럽을 선택합니다.

참고:

  • 파일 시스템 스크럽은 Btrfs 파일 시스템의 볼륨에서만 수행할 수 있습니다.
  • 볼륨의 상태가 정상일 때만 파일 시스템 스크럽을 수행할 수 있습니다.
  • 파일 시스템 스크럽이 완료되기 전에 시스템을 종료하지 마십시오. 완료 전에 시스템이 종료된 경우 수동으로 처음부터 파일 시스템 스크럽을 다시 시작해야 합니다.

파일 시스템 단편화 제거*

데이터가 지속적으로 수정되는 경우, 파일 시스템에서 단편화된 파일의 양이 증가하여 결과적으로 하드 드라이브의 파일에 대한 액세스 성능이 느려집니다. 파일 시스템 단편화 제거를 수행하여 성능을 향상시킬 수 있습니다.

파일 시스템 단편화 제거 수행:

  1. 저장소 관리자를 열고 볼륨으로 이동합니다.
  2. 파일 시스템 단편화 제거를 수행할 볼륨을 선택합니다.
  3. 관리를 클릭합니다.
  4. 파일 시스템 단편화 제거를 선택합니다.

참고:

  • 파일 시스템 단편화 제거는 Btrfs 파일 시스템의 볼륨에만 사용할 수 있습니다.
  • 볼륨의 상태가 정상일 때만 파일 시스템 단편화 제거를 수행할 수 있습니다.
  • 파일 시스템 단편화 제거가 완료되기 전에 시스템을 종료하지 마십시오. 완료 전에 시스템이 종료된 경우 수동으로 처음부터 파일 시스템 단편화 제거를 다시 시작해야 합니다.
  • 볼륨에 공유 폴더의 스냅샷이 포함된 경우 볼륨 사용량이 증가할 수 있습니다.

*일부 모델에서만 사용할 수 있습니다.

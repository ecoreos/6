일반

저장소 관리자를 통해 사용자가 Synology 제품의 모든 저장소 공간을 관리할 수 있습니다. Synology DSM에서는 저장소 관리를 단순화하고 데이터가 보호되는 유연한 저장소 공간을 만들 수 있도록 여러 가지 유형의 RAID Groups을 지원합니다.

RAID Group

DiskStation에서는 여러 개의 하드 디스크를 RAID Group이라고 하는 단일 저장소 유니트로 결합할 수 있습니다. DiskStation은(는) 다음 유형의 RAID Group을 제공합니다:

  • 단일 볼륨 또는 iSCSI LUN(블록 수준)용 RAID Group:
    • 사용 가능한 모든 용량을 단일 볼륨 또는 iSCSI LUN(블록 수준)에 할당합니다.
    • 성능은 향상되지만 저장소 관리 유연성이 저하됩니다.
    • 볼륨에 여러 개의 iSCSI LUN(일반 파일)을 생성할 수 있습니다.
    • 최대 한 개의 RAID 어레이를 지원합니다.
  • 여러 볼륨 또는 iSCSI LUN(블록 수준)용 RAID Group:
    • RAID Group에서 여러 볼륨 또는 iSCSI LUN(블록 수준) 생성을 지원합니다.
    • 보다 유연하고 향상된 저장소 관리 기능을 제공합니다.
    • 볼륨에 여러 개의 iSCSI LUN(일반 파일)을 생성할 수 있습니다.
    • RAID Group 아래 여러 RAID 어레이를 결합하도록 지원합니다(RAID 5 또는 RAID 6으로 구성된 경우).

RAID 유형

RAID Group을 생성하는 경우 먼저 RAID 유형을 선택해야 합니다. RAID 유형마다 데이터 보호 수준, 저장소 기능 및 성능이 달라집니다. DiskStation에서는 현재 다음 RAID 유형을 지원합니다:

  • 기본: 한 개의 하드 디스크를 사용하여 RAID Group을 생성합니다. Basic RAID Groups은 데이터 중복 기능을 지원하지 않습니다.
  • JBOD*: 2개 이상의 하드 디스크를 결합하여 RAID Group을 생성합니다. JBOD RAID Groups은 데이터 중복 기능을 지원하지 않습니다. JBOD RAID Group의 사용 가능한 용량은 그룹에 포함된 모든 디스크의 전체 용량과 같습니다. JBOD에서는 다른 크기의 하드 디스크를 결합할 수 있습니다.
  • RAID 0*: 2개 이상의 하드 디스크를 결합하여 RAID Group을 생성합니다. RAID 0은 데이터 중복 기능을 지원하지 않습니다. 하나의 멤버 디스크에 오류가 발생하면 RAID Group에서 해당 데이터가 손실됩니다. RAID 0 Group의 사용 가능한 용량은 모든 멤버 디스크의 전체 용량과 같습니다. RAID 0에서는 다른 크기의 하드 디스크를 결합할 수 있습니다.
  • RAID 1*: 2-4개의 하드 디스크를 결합하여 하나의 RAID Group을 생성합니다. 시스템은 각 하드 디스크에 동일한 데이터를 기록합니다. RAID 1에서 하나의 디스크가 정상적으로 작동하는 동안에는 데이터 보호 기능이 사용됩니다. RAID 1 Group의 사용 가능한 용량은 가장 작은 멤버 디스크의 크기와 같습니다.
  • RAID 5*: 3개 이상의 하드 디스크를 결합하여 하나의 RAID Group을 생성한 다음 모든 멤버 디스크에 두 데이터와 패리티 정보를 스트라이핑합니다. RAID 5는 데이터 중복 기능을 지원합니다. 한 개의 멤버 디스크에 오류가 발생하면 다른 멤버 디스크의 패리티 정보를 사용하여 시스템이 다시 구축됩니다. 여러 볼륨 또는 iSCSI LUN(블록 수준)용 RAID Group에서 생성할 때 여러 개의 RAID 어레이 결합을 지원합니다. RAID 5 Group의 사용 가능한 용량 = 결합된 RAID 어레이의 전체 용량. RAID 어레이 용량 = (하드 디스크 수 - 1) x (가장 작은 하드 디스크의 용량)
  • RAID 6*: 4개 이상의 하드 디스크를 결합하여 하나의 RAID Group을 생성한 다음 모든 멤버 디스크에 두 데이터와 패리티 정보를 스트라이핑합니다. RAID 6는 데이터 중복 기능을 지원합니다. 두 개의 멤버 디스크에 오류가 발생하는 경우에도 다른 멤버 디스크의 패리티 정보를 사용하여 시스템이 다시 구축됩니다. 여러 볼륨 또는 iSCSI LUN(블록 수준)용 RAID Group에서 생성할 때 여러 개의 RAID 어레이 결합을 지원합니다. RAID 6 Group의 사용 가능한 용량 = 결합된 RAID 어레이의 전체 용량. RAID 어레이 용량 = (하드 디스크 수 - 2) x (가장 작은 하드 디스크의 용량)
  • RAID 10*: 4개 이상의 하드 디스크를 결합하여 RAID Group을 생성합니다. 디스크 수는 짝수여야 합니다. RAID 10은 RAID 0의 성능 및 RAID 1과 같은 데이터 보호 수준을 제공합니다. RAID 10 Group의 사용 가능한 용량 = (가장 작은 하드 디스크의 용량) X (하드 디스크 수 / 2)

참고: "*" 표시가 있는 RAID 유형은 하드 디스크 베이가 충분이 갖춰진 NAS 모델에서만 사용할 수 있습니다.

볼륨 유형

기본 RAID Group 유형에 따라 다음 두 가지 볼륨 유형을 사용할 수 있습니다.

  • 단일 볼륨 또는 iSCSI LUN(블록 수준)용 RAID Group의 볼륨
    • RAID Group의 모든 저장소 용량을 단일 볼륨에 할당합니다. 성능은 향상되지만 관리 유연성이 저하됩니다.
    • 볼륨에 여러 개의 iSCSI LUN(일반 파일)을 생성할 수 있습니다.
  • 여러 볼륨 또는 iSCSI LUN(블록 수준)용 RAID Group의 볼륨
    • RAID Group에서 지정된 저장소 용량을 할당하여 볼륨을 생성합니다. 보다 유연하고 향상된 관리 기능을 제공하여 볼륨 크기를 확장하고 추가 볼륨을 필요한 만큼 생성할 수 있도록 합니다.
    • 볼륨에 여러 개의 iSCSI LUN(일반 파일)을 생성할 수 있습니다.

볼륨 형식

사용 중인 파일 시스템에 따라 두 가지 볼륨 형식이 있습니다.

  • 새 볼륨 형식: 이 형식은 Btrfs 파일 시스템을 사용합니다. 이 형식에서 볼륨의 최소 크기는 적어도 10 GB라야 합니다. 이 형식은 데이터를 더 쉽게 관리할 수 있는 다음과 같은 새로운 기능을 지원합니다:
    • 공유 폴더 스냅샷: 스냅샷은 캡처 당시의 모든 데이터 상태를 기록합니다. 사용자가 스냅샷이 있는 볼륨에서 실수로 데이터를 수정하거나 삭제할 경우, 스냅샷을 사용하면 데이터를 스냅샷을 촬영한 이전 시점으로 빠르게 다시 복원할 수 있습니다.
    • 공유 폴더 할당량: 각 폴더에 대한 최대 용량을 설정할 수 있습니다.
    • 공유 폴더에 기반한 사용자 할당량: 모든 사용자가 공유 폴더 내의 공간을 얼마나 사용할 수 있는지 제어할 수 있습니다.
    • 고급 데이터 무결성 보호: 공유 폴더는 CRC32 체크섬 및 쓰기 시 복사 전략으로 보호됩니다.
  • 전통 볼륨 형식: 이 형식은 ext4 파일 시스템을 사용합니다. 이 형식에서 볼륨의 최소 크기는 적어도 10 GB라야 합니다. ext4 파일 시스템은 대부분의 Synology 제품에서 지원되므로, 이 형식의 볼륨은 쉽게 다른 Synology 제품으로 마이그레이션할 수 있습니다.

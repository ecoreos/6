iSCSI LUN*

iSCSI는 로컬 네트워크 또는 장거리 데이터 전송을 위한 저장소 네트워크 서비스입니다. iSCSI LUN을 iSCSI Target에 매핑하면 클라이언트가 저장소 공간을 로컬 디스크처럼 액세스할 수 있습니다. iSCSI LUN(논리 단위 번호)을 하나 또는 여러 iSCSI Target에 매핑하여 iSCSI 저장소 서비스를 제공할 수 있습니다. 저장소 관리자 > iSCSI LUN으로 이동하여 iSCSI LUN을 관리합니다.

참고:

지원되는 iSCSI LUN 수는 DiskStation 모델에 따라 다릅니다. 자세한 제품 사양에 대해서는 www.synology.com을 참조하십시오.

DiskStation에서는 현재 다음 유형의 iSCSI LUN을 지원합니다:

  • iSCSI LUN (일반 파일): 사용할 볼륨의 일부를 지정하는 방식으로 iSCSI LUN 이 생성됩니다. 필요한 경우 나중에 크기를 확장할 수 있습니다. 이 유형의 iSCSI LUN은 Thin Provisioning, VMware VAAI 하드웨어 가속화 지원, Windows ODX 지원, LUN 스냅샷 및 LUN 복제로 동적 용량 관리의 유연성을 제공합니다.
  • iSCSI LUN(블록 수준) - RAID에 단일 LUN: iSCSI LUN은 볼륨 전체 공간을 사용하여 만듭니다. 볼륨을 사용할 때와 같이 복구 또는 확장하거나 RAID 유형을 변경할 수 있습니다. 자세한 정보는 볼륨 페이지를 참조하십시오.
  • iSCSI LUN (블록 수준) - RAID에 복수 LUN: iSCSI LUN은 디스크 그룹의 특정 공간을 지정하여 만듭니다. 디스크 그룹에 사용 가능한 공간이 있으면 나중에 iSCSI LUN을 확장할 수 있습니다. 자세한 정보는 디스크 그룹 페이지를 참조하십시오.
  • iSCSI LUN(블록 수준): RAID Group에서 생성되는 이 유형의 iSCSI LUN은 유연한 동적 용량 관리와 최적화된 액세스 성능을 제공합니다. 이 유형의 iSCSI LUN은 나중에 RAID Group에서 할당되지 않은 공간을 사용하여 확장할 수 있습니다. 자세한 정보는 RAID Group 페이지를 참조하십시오.

참고:

고급 LUN 기능을 가진 iSCSI LUN은 ext4/Btrfs 파일 시스템에서만 사용할 수 있습니다.

iSCSI LUN 생성하기:

  1. 생성을 클릭합니다.
  2. 생성할 iSCSI LUN 유형을 선택합니다.
  3. 마법사를 따라 프로세스를 완료합니다.

참고:

고급 LUN 기능은 초기 iSCSI LUN 생성 중에만 활성화할 수 있습니다. 그 후에는 이 설정을 변경할 수 없습니다.

고급 LUN 할당 유니트 크기는 다음 중 하나를 실행하여 Microsoft Exchange 또는 SQL Server 같은 응용 프로그램 서버의 특정 기능이나 Microsoft Windows ODX 또는 VMware VAAI 같은 가상화 서버 및 저장소 가속 기술에 맞게 최적화할 수 있습니다:

  • 볼륨의 첫 번째 고급 LUN을 생성하는 동안 할당 유니트 크기를 설정합니다.
  • 볼륨의 첫 번째 고급 LUN을 생성하기 전에 구성에서 할당 유니트 크기를 설정합니다.

DiskStation에서 고급 iSCSI LUN과 작동하는 RDM(pass-through) 방법 또는 응용 프로그램 서버에서 직접 연결을 사용할 때 응용 프로그램의 할당 유니트 크기를 지정하는 것이 좋습니다. Windows ODX에 대한 복제에 Synology 고급 LUN 기술을 사용하려면, DSM에 지정한 것과 크기가 동일한(또는 더 큰) 할당 유니트 크기로 NTFS 볼륨을 포맷할 것을 권장합니다.

참고:

각 볼륨의 고급 LUN 할당 유니트는 한번만 설정할 수 있습니다.

iSCSI LUN 제거하기:

iSCSI LUN을 제거하면 해당 iSCSI LUN에 있는 모든 데이터가 삭제됩니다. iSCSI LUN과 매핑된 모든 iSCSI Target 간의 연결도 끊어집니다.

  1. 제거할 iSCSI LUN을 선택합니다.
  2. 제거 단추를 클릭하고 마법사를 따라 프로세스를 완료합니다.

참고:

DiskStation에서 VAAI LUN을 삭제하는 경우 즉시 디스크 공간이 늘어나지는 않습니다. 데이터는 @EP_trash라는 시스템 예약 폴더로 이동되어 조금씩 삭제됩니다. 예를 들어, 100 GB의 VAAI LUN은 DiskStation에서 삭제할 경우, 디스크 볼륨에서 100 GB의 공간을 되찾은 데 15-20분이 걸릴 수 있습니다.

iSCSI LUN(블록 수준) 강제 마운트하기:

iSCSI LUN(블록 수준)의 캐시가 손실된 경우 시스템이 부팅할 때 iSCSI LUN(블록 수준)을 마운트하지 않습니다. 캐시 복원이 불가능할 때 iSCSI LUN(블록 수준)에 액세스하려는 경우 강제 마운트를 클릭할 수 있습니다.

참고:

이 기능은 특정 모델에서만 사용할 수 있습니다.

iSCSI LUN 편집하기:

  1. 편집할 iSCSI LUN을 선택합니다.
  2. 편집을 클릭합니다.
  3. 원하는 속성을 편집합니다.
  4. 확인을 클릭하여 설정을 저장합니다.

참고:

iSCSI LUN 용량을 편집할 때 데이터 손실을 방지하기 위해 현재 크기보다 더 큰 용량으로만 설정할 수 있습니다.

iSCSI LUN 스냅샷 촬영하기:

iSCSI LUN 스냅샷은 iSCSI LUN에 대한 특정 시점의 읽기 전용 복사본입니다.

  1. 스냅샷을 촬영할 iSCSI LUN을 선택합니다.
  2. 스냅샷을 클릭하고 스냅샷 촬영을 선택합니다.
  3. 이 스냅샷에 대한 설명을 입력합니다.
  4. 필요에 따라 다음 스냅샷 유형 중에서 선택하십시오:
    • 응용 프로그램 일관: 이 유형의 스냅샷은 DSM과 통신하는 운영 환경에 따라 Synology Snapshot Manager가 Windows 서버 또는 VMware vCenter Server에 설치되어 있을 때만 촬영할 수 있습니다. 응용 프로그램 일관 스냅샷이 DSM에서 트리거되면, 응용 프로그램에 통지되고 여기에서 데이터 일관성을 보장하기 위해 메모리의 데이터를 모두 스냅샷이 촬영된 LUN으로 플러시합니다.
    • 충돌 일관: Synology Snapshot Manager가 어떤 클라이언트에도 설치되어 있지 않으면 LUN의 충돌 일관 스냅샷만 촬영할 수 있습니다. 이들이 트리거될 때 메모리에서 LUN으로 데이터를 플러시하도록 응용 프로그램에 통지되지 않아 데이터 일관성이 보장되지 않습니다.
  5. 확인을 클릭합니다.

iSCSI LUN 복원하기:

특정 시점 스냅샷을 선택하고 iSCSI LUN에 데이터를 복원합니다.

  1. 복원할 iSCSI LUN을 선택합니다.
  2. 스냅샷을 클릭하고 스냅샷 관리자를 선택합니다.
  3. 복원하려는 스냅샷을 선택하고 복원을 클릭합니다.

iSCSI LUN 또는 스냅샷 복제하기:

iSCSI LUN 또는 iSCSI LUN 스냅샷의 쓰기 가능한 사본을 만듭니다.

  • 목록에서 iSCSI LUN을 선택하고 복제를 클릭합니다.
  • 스냅샷 관리자에서 iSCSI LUN 스냅샷을 선택하고 복제를 클릭합니다.

참고:

  • 특정 모델에서는 패키지 센터로 이동한 다음 iSCSI LUN 복원과 iSCSI LUN 스냅샷 촬영 및 복제를 비롯한 기능에 대해 스냅샷 및 복제를 설치합니다.
  • LUN과 관련된 iSCSI 세션이 최소한 하나 연결되어 있고, 이 세션을 Windows 서버 또는 VMware ESXi 서버에서 사용하는 경우, Synology Snapshot Manager 플러그인을 설치하여 가능한 데이터 불일치를 방지하는 것이 좋습니다. 그렇지 않으면 작업을 계속하기 전에 응용 프로그램을 중지하고 개시자에서 이러한 세션을 로그오프해야 합니다.
  • LUN 스냅샷, 복원 및 복제 기능은 고급 LUN 기능을 가진 iSCSI LUNs에서만 사용할 수 있습니다.
  • 스냅샷을 촬영했거나 복제를 생성했고 상태가 비정상/충돌함으로 변한 경우, 이 스냅샷/복제를 삭제하고 새로 생성하십시오.
  • 스냅샷을 생성한 후 LUN에서 변경된 모든 내용은 LUN을 이 스냅샷에 복원할 때 손실됩니다.
  • iSCSI LUN의 상태가 사용 불가로 변경되는 경우에는 iSCSI LUN에 액세스하는 동안 일부 데이터 관련 오류가 발생한 것입니다. 시스템에서 심각한 데이터 손실을 방지하기 위해서 이 iSCSI LUN에 일시적으로 액세스할 수 없습니다. Synology에 즉시 지원을 요청하십시오.

용어:

  • Thin Provisioning: Thin Provisioning 은 동적 및 온디맨드 방식으로 저장 공간을 할당하여 저장소 이용을 최적화하는 방법입니다. 이 설정은 iSCSI LUN(일반 파일)에 대해서만 사용할 수 있습니다.
  • VMware VAAI: VMware VAAI(어레이 통합을 위한 vStorage API)는 가상 저장소 어레이에서 표준 작동의 작업을 VMware vSphere 환경의 iSCSI 장치로 넘겨 저장소 성능을 최적화할 수 있습니다.
  • Windows ODX: Microsoft가 Windows Server 2012 및 8에서 새로 개발한 ODX(오프로드 데이터 전송)라는 새로운 데이터 전송 기술입니다. ODX는 작업 부하의 일부를 저장소 어레이에 분담하여 전송 성능을 향상시킵니다.
    데이터 소스와 대상이 동일한 볼륨의 LUN에 있는 경우, 복제는 Synology 고급 LUN 기술을 통해 처리되며, 이 기술은 저장소 공간 소비를 최고 99.9%까지 줄이고 복제 속도록 가속화합니다.
  • iSCSI Target 매핑: iSCSI LUN을 하나 이상의 iSCSI Target에 매핑합니다.
  • Synology Snapshot Manager: Synology Snapshot Manager는 DiskStation Manager(DSM)에서 직접 응용 프로그램 일관 스냅샷을 촬영할 수 있도록 해주는 소프트웨어 플러그인입니다.
    • VMware 환경의 경우, VMware vCenter Server가 설치된 Windows 서버에 Synology Snapshot Manager를 설치해야 합니다. 스냅샷이 DSM에서 트리거되면, VMware vCenter Server에 통지되고 여기에서 데이터 일관성을 보장하기 위해 메모리의 데이터를 모두 스냅샷이 촬영된 LUN으로 플러시합니다. 스냅샷이 완료되면 VMware vCenter Server가 VMware 데이터베이스의 정상 I/O 작업을 계속합니다. Synology Snapshot Manager for VMware vCenter Server는 vSphere 5.1 이상을 지원합니다.
    • Windows 환경에서는 Windows 서버에 Synology Snapshot Manager를 설치해야 합니다. 스냅샷이 DSM에서 트리거되면, Synology Snapshot Manager가 Microsoft 볼륨 섀도 복사 서비스(VSS) 기술을 사용하여 데이터에 대해 일관된 특정 시점 복사를 생성합니다. Synology Snapshot Manager for Windows는 Windows Server 2008 R2, 2012 또는 2012 R2를 지원합니다.
    • Synology Snapshot Manager는 www.synology.com에서 무료로 다운로드할 수 있습니다.

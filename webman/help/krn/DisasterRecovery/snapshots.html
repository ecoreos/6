스냅샷

이 페이지에서는 스냅샷을 촬영, 제어 및 편집하고 스냅샷 예약 및 보존 설정을 구성할 수 있습니다.

각각의 공유 폴더 또는 iSCSI LUN에 대해 한 번에 최대 256개의 스냅샷 복사본을 저장할 수 있습니다. 스냅샷은 전체 파일이 아닌 개별 블록을 보존하여 디스크 소비량을 최소화합니다. 공유 폴더의 파일 또는 iSCSI LUN의 블록을 업데이트하거나 제거할 때 추가 공간이 사용될 수 있습니다.

공유 폴더

공유 폴더의 스냅샷 촬영하기:

  1. 스냅샷을 촬영할 공유 폴더를 선택합니다.
  2. 스냅샷 촬영을 클릭합니다.
  3. 설명에 촬영할 스냅샷에 대한 설명을 입력합니다.
  4. 잠금 확인란을 선택/선택 취소하여 자동 제거가 트리거될 경우 스냅샷에서 보존 설정을 무시할 것인지 여부를 결정합니다.
  5. 확인을 클릭하여 스냅샷 촬영을 시작합니다.

참고:

  • 제어판 > 공유 폴더 > 편집 페이지의 설정 중에, Windows ACL 권한만 스냅샷에 기록됩니다.

공유 폴더의 스냅샷 찾아보기:

  1. 스냅샷을 검색할 공유 폴더를 선택합니다.
  2. 스냅샷 목록을 클릭합니다.
  3. 검색할 스냅샷을 선택하고 찾아보기를 클릭합니다.

참고:

  • 암호화된 공유 폴더에는 스냅샷 찾아보기를 사용할 수 없습니다.
  • 스냅샷을 검색하지 않으려면 제어판 > 공유 폴더 > 편집으로 이동하여 스냅샷을 보이게 만들기를 활성화하십시오.

공유 폴더의 스냅샷 제거하기:

  1. 스냅샷을 제거할 공유 폴더를 선택합니다.
  2. 스냅샷 목록을 클릭합니다.
  3. 제거할 스냅샷을 하나 이상 선택하고 제거를 클릭합니다. (여러 항목을 선택하려면 Shift 또는 Ctrl 키를 누른 상태에서 선택하십시오.)
  4. 확인 메시지가 표시됩니다. 삭제를 클릭하여 제거를 확인합니다.

공유 폴더의 스냅샷 편집하기:

  1. 스냅샷을 편집할 공유 폴더를 선택합니다.
  2. 스냅샷 목록을 클릭합니다.
  3. 편집할 스냅샷을 선택하고 편집을 클릭합니다.
  4. 설명에서 스냅샷에 대한 설명을 수정하고 잠금 확인란을 선택/선택 취소하여 자동 제거가 트리거될 경우 스냅샷에서 보존 설정을 무시할 것인지 여부를 결정합니다.
  5. 확인을 클릭하여 설정을 저장합니다.

예약된 스냅샷 구성하기:

예약된 스냅샷을 사용하면 스냅샷 및 복제가 미리 정의된 시간과 빈도로 스냅샷을 자동으로 촬영하므로, 스냅샷을 수동으로 촬영하는 작업 부하를 줄일 수 있습니다.

  1. 공유 폴더를 선택합니다.
  2. 예약 설정을 클릭합니다.
  3. 예약 탭에서 스냅샷 예약 활성화 확인란을 선택하고 스냅샷을 촬영할 시간과 빈도를 설정합니다. 이 공유 폴더에 대해 예약된 스냅샷을 취소하려면 확인란을 선택 취소하십시오.
  4. 확인을 클릭하여 설정을 저장합니다.
  5. 확인 메시지가 표시됩니다. 확인을 클릭하여 구성을 확인합니다.

참고:

  • 제어판 > 업데이트 및 복원 > 구성 백업 기능에는 예약된 스냅샷의 구성이 포함되지 않습니다.

예약된 스냅샷의 보존 설정 구성하기:

  1. 공유 폴더를 선택합니다.
  2. 예약 설정을 클릭합니다.
  3. 보존 탭을 선택합니다.
  4. 보존 설정을 구성합니다. 다음 옵션 중에서 선택합니다.
    • 제한에 도달하면 이전 스냅샷 제거: 최대 스냅샷 개수에 도달하면 가장 오래된 스냅샷이 새로 촬영된 스냅샷으로 덮어쓰입니다.
    • 항상 스냅샷 보존: 최대 스냅샷 개수에 도달하면 스냅샷 및 복제는 이미 촬영된 스냅샷을 제거하지 않고 새 스냅샷 촬영을 중지합니다.
    • 다음 정책에 따라 스냅샷을 보존합니다. 지정된 시간 범위에 대한 보존 규칙을 구성할 수 있습니다. 자세한 내용은 보존 정책 노트를 참조하십시오.
  5. 확인을 클릭하여 설정을 저장합니다.

참고:

  • 보존 설정을 변경하면 복제 페이지의 동일한 설정도 변경됩니다.

iSCSI LUN

iSCSI LUN의 스냅샷 촬영하기:

  1. 스냅샷을 촬영할 iSCSI LUN을 선택합니다.
  2. 스냅샷 촬영을 클릭합니다.
  3. 설명에 촬영할 스냅샷에 대한 설명을 입력합니다.
  4. 잠금 드롭다운 목록에서 예/아니오를 선택하여 자동 제거가 트리거될 경우 스냅샷에서 보존 설정을 무시할 것인지 여부를 결정합니다.
  5. 확인을 클릭하여 스냅샷 촬영을 시작합니다.

iSCSI LUN의 스냅샷 제거하기:

  1. 스냅샷을 제거할 iSCSI LUN을 선택합니다.
  2. 스냅샷 목록을 클릭합니다.
  3. 제거할 스냅샷을 하나 이상 선택하고 제거를 클릭합니다. (여러 항목을 선택하려면 Shift 또는 Ctrl 키를 누른 상태에서 선택하십시오.)
  4. 확인 메시지가 표시됩니다. 예를 클릭하여 제거를 확인합니다.

iSCSI LUN의 스냅샷 편집하기:

  1. 스냅샷을 편집할 iSCSI LUN을 선택합니다.
  2. 스냅샷 목록을 클릭합니다.
  3. 편집할 스냅샷을 선택하고 편집을 클릭합니다.
  4. 다음을 수행하여 스냅샷을 편집할 수 있습니다.
    • 설명에서 스냅샷에 대한 설명을 변경합니다.
    • 응용 프로그램 일치 스냅샷이 활성화되어 있는 경우 일관된 상태를 선택합니다.
    • 잠금 드롭다운 목록에서 예/아니오 간에 전환하여 자동 제거가 트리거될 경우 스냅샷에서 보존 설정을 무시할 것인지 여부를 결정합니다.
  5. 확인을 클릭하여 설정을 저장합니다.

예약된 스냅샷 구성하기:

예약된 스냅샷을 사용하면 스냅샷 및 복제가 미리 정의된 시간과 빈도로 스냅샷을 자동으로 촬영하므로, 스냅샷을 수동으로 촬영하는 작업 부하를 줄일 수 있습니다.

  1. iSCSI LUN을 선택합니다.
  2. 예약 설정을 클릭합니다.
  3. 예약 탭에서 스냅샷 예약 활성화 확인란을 선택하고 스냅샷을 촬영할 시간과 빈도를 설정합니다. 이 공유 폴더에 대해 예약된 스냅샷을 취소하려면 확인란을 선택 취소하십시오.
  4. 확인을 클릭하여 설정을 저장합니다.

예약된 스냅샷의 보존 설정 구성하기:

  1. iSCSI LUN을 선택합니다.
  2. 예약 설정을 클릭합니다.
  3. 보존 탭을 선택합니다.
  4. 보존 설정을 구성합니다. 다음 옵션 중에서 선택합니다.
    • 제한에 도달하면 이전 스냅샷 제거: 최대 스냅샷 개수에 도달하면 가장 오래된 스냅샷이 새로 촬영된 스냅샷으로 덮어쓰입니다.
    • 항상 스냅샷 보존: 최대 스냅샷 개수에 도달하면 스냅샷 및 복제는 이미 촬영된 스냅샷을 제거하지 않고 새 스냅샷 촬영을 중지합니다.
    • 다음 정책에 따라 스냅샷을 보존합니다. 지정된 시간 범위에 대한 보존 규칙을 구성할 수 있습니다. 자세한 내용은 보존 정책 노트를 참조하십시오.
  5. 확인을 클릭하여 설정을 저장합니다.
  6. 확인 메시지가 표시됩니다. 확인을 다시 클릭하여 구성을 확인합니다.

iSCSI LUN 스냅샷의 일관된 상태 구성하기:

  1. iSCSI LUN을 선택합니다.
  2. 예약 설정을 클릭합니다.
  3. 응용 프로그램 탭을 선택합니다.
  4. 응용 프로그램 일치 스냅샷 활성화 확인란을 선택/선택 취소하여 다음 일관된 상태 간에 전환합니다.
    • 응용 프로그램 일관: 이러한 일관된 상태의 스냅샷은 DSM과 통신하는 운영 환경에 따라 Synology Snapshot Manager가 Windows 서버 또는 VMware vCenter Server에 설치되어 있을 때만 촬영할 수 있습니다. 응용 프로그램 일치 스냅샷이 DSM에서 트리거되면, 응용 프로그램에 통지되고 여기에서 데이터 일관성을 보장하기 위해 메모리의 데이터를 모두 스냅샷이 촬영되는 LUN으로 플러시합니다.

      참고:

      • 운영 환경에 Synology Snapshot Manager가 설치되어 있지 않은 경우에는 확인란을 선택해도 응용 프로그램 일치 스냅샷이 비활성화된 상태로 유지됩니다.
    • 충돌 일관: Synology Snapshot Manager가 어떤 클라이언트에도 설치되어 있지 않으면 LUN의 충돌 일관 스냅샷만 캡처할 수 있습니다. 충돌 일관 스냅샷이 트리거될 때에는 메모리에서 LUN으로 데이터를 플러시하도록 응용 프로그램에 통지되지 않아 데이터 일관성이 보장되지 않습니다.
  5. 확인을 클릭하여 설정을 저장합니다.

참고:

  • 일관된 상태 설정을 변경하면 복제 페이지의 동일한 설정도 변경됩니다.

보존 정책

이 옵션을 사용하면 볼륨 공간에 저장할 스냅샷 버전의 최대 용량을 이상적인 방법으로 지정할 수 있지만, 스냅샷을 더 오래 보존해야 하는 경우가 있을 수 있습니다.

Synology에서는 GFS 또는 Grandfather-Father-Son 보존 정책을 적용합니다. 보존할 스냅샷 버전의 최대 용량을 다음과 같은 시간 범위로 각각 구성할 수 있습니다: 시간별, 일별, 주별, 월별 및 년별.

스냅샷 및 복제는 모든 시간 범위에 대해 구성된 최대 스냅샷 수를 최대한 많이 보존합니다. 시간 범위 내에 스냅샷 버전이 둘 이상 있는 경우 가장 먼저 생성된 버전만 유지됩니다. 예를 들어, 정책을 10 주별 스냅샷으로 설정하는 경우 스냅샷 및 복제가 최근 10주 동안 각각 가장 먼저 생성된 스냅샷을 보존합니다(일주일에 스냅샷이 둘 이상 촬영된 경우).

또한 최근 스냅샷 버전을 쉽게 찾고 복원할 수 있도록 기본적으로 가장 최근에 스냅샷을 촬영한 이후 1시간 내에 속하는 모든 스냅샷 버전이 유지됩니다. 보존 규칙에 의해 자동으로 제거되지 않도록 스냅샷을 잠글 수도 있습니다.

복구

이 페이지에서는 시스템 관리자가 복구를 수행할 수 있습니다. 사용자가 공유 폴더 스냅샷에 액세스할 수 있도록 하려면 사용자가 File Station 또는 Windows 파일 탐색기를 통해 스냅샷에 액세스할 수 있도록 각 공유 폴더에 대해 스냅샷을 보이게 만들기를 활성화하십시오.

공유 폴더

공유 폴더에서 개별 파일/폴더 복원하기:

  1. 복원할 공유 폴더를 선택합니다.
  2. 복구를 클릭합니다.
  3. 복원할 스냅샷 버전을 선택합니다.
  4. 찾아보기를 클릭하고 복원할 파일/폴더를 대상에 복사합니다.

공유 폴더 복원하기:

  1. 복원할 공유 폴더를 선택합니다.
  2. 복구를 클릭합니다.
  3. 복원할 스냅샷 버전을 선택합니다.
  4. 작업을 클릭하고 드롭다운 목록에서 제자리에서 복원을 선택합니다.
  5. 확인 메시지가 표시됩니다. 예를 클릭하여 작업을 확인합니다.

스냅샷을 새 공유 폴더에 복제하기:

  1. 복제할 공유 폴더를 선택합니다.
  2. 복구를 클릭합니다.
  3. 새 공유 폴더에 복제할 스냅샷 버전을 선택합니다.
  4. 작업을 클릭하고 드롭다운 목록에서 새 이름으로 복제를 선택합니다.
  5. 제어판 > 공유 폴더 > 생성에서 복제 페이지로 리디렉션됩니다. 복제한 공유 폴더의 이름을 입력하고 확인을 클릭하여 완료합니다.

참고:

  • 공유 폴더 “homes”는 복원할 수 없습니다.
  • 복제한 공유 폴더를 암호화할 수 없습니다.
  • 파일 전송이 진행 중인 상태에서 스냅샷을 촬영한 경우에는 데이터 일관성이 보장되지 않습니다.
  • 암호화된 공유 폴더에 대한 제한:
    • 마운트된 암호화된 공유 폴더에 대해서는 어떠한 유형의 복구도 수행할 수 없습니다.
    • 암호화된 공유 폴더에는 새 이름으로 복제를 사용할 수 없습니다.
    • 제자리에서 복원을 사용하여 이전에 촬영한 스냅샷에서 마운트가 해제된 암호화된 공유 폴더를 복원할 수 있습니다.

공유 폴더의 복제 전환하기:

  1. 전환하려는 복제된 공유 폴더를 선택합니다.
  2. 전환을 클릭합니다.
  3. 공유 폴더의 복제 작업이 여러 개이면 새 대상 서버를 선택합니다.
  4. 전환을 클릭하여 작업을 시작합니다.

공유 폴더의 복제 대체 작동하기:

  1. 대체 작동하려는 복제된 공유 폴더를 선택합니다.
  2. 대체 작동 > 대체 작동을 클릭합니다.
  3. 복원할 스냅샷의 버전을 선택합니다.
  4. 대체 작동을 클릭하여 작업을 시작합니다.

참고:

  • 소스 서버가 실패하는 경우에만 대체 작동을 시작할 수 있습니다.

공유 폴더의 복제 재보호하기:

대상 서버로 대체 작동되는 경우 복제 작업이 두 서버 간의 데이터 동기화를 중지합니다. 원래 소스 서버가 복구된 다음 복제 작업을 재보호해야 합니다.

  1. 재보호하려는 대체 작동된 공유 폴더를 선택합니다.
  2. 대체 작동 > 재보호를 클릭합니다.
  3. 새 소스 서버를 선택합니다.
  4. 공유 폴더의 복제 작업이 여러 개이면 새 대상 서버를 선택합니다.
  5. 재보호를 클릭하여 작업을 시작합니다.

참고:

  • 새 대상 서버에 있는 복제된 공유 폴더의 데이터는 새 소스 서버의 데이터로 덮어쓰게 됩니다.
  • 대체 작동을 위해 선택된 스냅샷 버전이 파트너 서버에 없으면 재보호에 더 많은 시간이 걸릴 수도 있습니다.

복제된 공유 폴더에 대해 테스트 대체 작동 수행하기:

  1. 테스트하려는 복제된 공유 폴더를 선택합니다.
  2. 테스트 대체 작동 > 테스트 대체 작동을 클릭합니다.
  3. 공유 폴더의 복제 작업이 여러 개이면 테스트 서버를 선택합니다.
  4. 공유 폴더의 테스트 대체 작동의 이름을 입력하고 테스트할 스냅샷 버전을 선택합니다.
  5. 테스트 대체 작동을 클릭하여 작업을 시작합니다.

참고:

  • 복제의 전환/대체 작동을 수행하면 테스트 대체 작동이 지워집니다.
  • 테스트 대체 작동 공유 폴더에 대해서는 복제 작업을 생성할 수 없습니다.

복제된 공유 폴더에 대해 테스트 대체 작동 지우기:

  1. 테스트 대체 작동을 지우려는 복제된 공유 폴더를 선택합니다.
  2. 테스트 대체 작동 > 테스트 대체 작동 지우기를 클릭합니다.
  3. 지우려는 테스트 대체 작동을 선택합니다.
    참고: 테스트 대체 작동을 위해 생성된 공유 폴더를 더 이상 보존하지 않으려면 테스트 대체 작동 데이터 삭제를 선택합니다.
  4. 테스트 대체 작동 지우기를 클릭하여 작업을 시작합니다.

iSCSI LUN

  1. 복원할 iSCSI LUN을 선택합니다.

  5. 확인 메시지가 표시됩니다. 예를 클릭하여 작업을 확인합니다.

스냅샷을 새 iSCSI LUN에 복제하기:

  1. 복제할 iSCSI LUN을 선택합니다.
  2. 복구를 클릭합니다.
  3. 새 iSCSI LUN에 복제할 스냅샷 버전을 선택합니다.
  4. 작업을 클릭하고 드롭다운 목록에서 새 이름으로 복제를 선택합니다.
  5. 저장소 관리자 > iSCSI LUN에서 복제 페이지로 리디렉션됩니다. 복제한 iSCSI LUN의 이름을 입력하고 확인을 클릭하여 완료합니다.

참고:

  • 파일 전송이 진행 중인 상태에서 스냅샷을 촬영한 경우에는 데이터 일관성이 보장되지 않습니다.

공유 폴더의 복제 전환하기:

  1. 전환하려는 복제된 iSCSI LUN을 선택합니다.
  2. 전환을 클릭합니다.
  3. iSCSI LUN의 복제 작업이 여러 개이면 새 대상 서버를 선택합니다.
  4. 전환을 클릭하여 작업을 시작합니다.

iSCSI LUN의 복제 대체 작동하기:

  1. 대체 작동하려는 복제된 iSCSI LUN을 선택합니다.
  2. 대체 작동 > 대체 작동을 클릭합니다.
  3. 복원할 스냅샷의 버전을 선택합니다.
  4. 대체 작동을 클릭하여 작업을 시작합니다.

참고:

  • 소스 서버가 실패하는 경우에만 대체 작동을 시작할 수 있습니다.

iSCSI LUN의 복제 재보호하기:

대상 서버로 대체 작동되는 경우 복제 작업이 두 서버 간의 데이터 동기화를 중지합니다. 원래 소스 서버가 복구된 다음 복제 작업을 재보호해야 합니다.

  1. 재보호하려는 대체 작동된 iSCSI LUN을 선택합니다.
  2. 대체 작동 > 재보호를 클릭합니다.
  3. 새 소스 서버를 선택합니다.
  4. iSCSI LUN의 복제 작업이 여러 개이면 새 대상 서버를 선택합니다.
  5. 재보호를 클릭하여 작업을 시작합니다.

참고:

  • 새 대상 서버에 있는 복제된 iSCSI LUN의 데이터는 새 소스 서버의 데이터로 덮어쓰게 됩니다.
  • 대체 작동을 위해 선택된 스냅샷 버전이 파트너 서버에 없으면 재보호에 더 많은 시간이 걸릴 수도 있습니다.

복제된 iSCSI LUN에 대해 테스트 대체 작동 수행하기:

  1. 테스트하려는 복제된 iSCSI LUN을 선택합니다.
  2. 테스트 대체 작동 > 테스트 대체 작동을 클릭합니다.
  3. iSCSI LUN의 복제 작업이 여러 개이면 테스트 서버를 선택합니다.
  4. iSCSI LUN의 테스트 대체 작동의 이름을 입력하고 테스트할 스냅샷 버전을 선택합니다.
  5. 테스트 대체 작동을 클릭하여 작업을 시작합니다.

참고:

  • 복제의 전환/대체 작동을 수행하면 테스트 대체 작동이 지워집니다.
  • 테스트 대체 작동 iSCSI LUN에 대해서는 복제 작업을 생성할 수 없습니다.

복제된 iSCSI LUN에 대해 테스트 대체 작동 지우기:

  1. 테스트 대체 작동을 지우려는 복제된 iSCSI LUN을 선택합니다.
  2. 테스트 대체 작동 > 테스트 대체 작동 지우기를 클릭합니다.
  3. 지우려는 테스트 대체 작동을 선택합니다.
    참고: 테스트 대체 작동을 위해 생성된 iSCSI LUN을 더 이상 보존하지 않으려면 테스트 대체 작동 데이터 삭제를 선택합니다.
  4. 테스트 대체 작동 지우기를 클릭하여 작업을 시작합니다.

계정

계정 탭은 설명, 패스워드, 이메일, 2 단계 검증 같은 기본 계정 설정을 편집하는 옵션을 제공합니다. 이 탭을 보려면 바탕화면 오른쪽 위에 있는 사람 모양의 아이콘을 클릭하고 개인용을 선택합니다.

계정 설정 수정

다음 계정 설정을 편집할 수 있습니다:

  • 설명: 계정에 대한 설명을 입력합니다. 사용자 설명에는 최대 64개의 표시 가능한 Unicode 문자가 포함될 수 있습니다.
  • 새 패스워드: 계정의 패스워드를 변경하려면 여기에 새 패스워드를 입력하고 아래 패스워드 확인 필드에서 확인합니다. 패스워드는 대소문자를 구분하며 문자, 숫자, 기호, 공백을 포함하여 최대, 표시 가능한 127자까지 입력 가능합니다.
  • 이메일: 이메일 주소를 입력합니다(관리자가 아닌 사용자만).
  • 표시 언어: 계정의 표시 언어를 지정합니다. 여기에서 지정한 표시 언어 설정은 제어판 > 지역 옵션 > 언어에서 지정한 언어보다 우선합니다.

계정 설정을 수정한 후에 확인을 클릭하여 변경 사항을 저장합니다.

2 단계 검증

2 단계 검증은 DSM 계정에 대한 향상된 보안을 제공합니다. 2 단계 검증이 활성화되면 DSM에 로그인할 때 패스워드 외에 1회용 검증 코드를 입력해야 합니다. 모바일 기기에 설치된 인증기 앱에서 검증 코드를 얻을 수 있습니다. 따라서 사용자의 계정에 접근하고 싶은 사람은 사용자 이름과 패스워드 뿐 아니라 사용자의 모바일 기기도 필요합니다.

요구 사항:

2 단계 검증은 모바일 기기와 TOTP(Time-based One-Time Password 시간 기반 1회용 패스워드) 프로토콜을 지원하는 인증기 앱을 필요로 합니다. 인증기 앱에는 Google Authenticator (Android/iPhone/BlackBerry) 또는 Authenticator (Windows Phone)가 있습니다.

2 단계 검증 활성화하기:

  1. 개인용 > 계정 탭으로 이동하고 2 단계 검증 활성화 상자를 선택하여 2 단계 검증 설정 마법사를 실행합니다.
  2. 첫 화면에 이메일 주소를 입력합니다. 모바일 기기를 잃어버릴 경우 이 이메일 주소로 긴급 검증 코드를 수신할 수 있습니다.
  3. Google Authenticator (Android/iPhone/BlackBerry) 또는 Authenticator (Windows Phone) 같은 인증기 앱을 다운로드하여 설치합니다.
  4. 다음 화면에서 인증기 앱을 열고 QR 코드를 스캔합니다. 또는 링크를 클릭하여 수동으로 비밀 키를 입력할 수도 있습니다.
  5. 그런 다음 인증기 앱이 6자리 검증 코드를 생성합니다. 구성이 올바른지 확인하기 위해 이 코드를 마법사 텍스트 필드에 입력합니다. 에러가 발생할 경우 모바일 기기의 시스템 시간이 DSM의 시스템 시간과 동기화되어 있는지 확인하십시오. 또한 검증 코드를 정기적으로 업데이트하여 입력한 코드가 만료되지 않도록 하십시오.
  6. 설정 마법사를 마친 후 확인을 클릭하여 설정을 저장합니다.

2 단계 검증으로 DSM에 로그인하기:

2 단계 검증이 활성화되면 DSM에 로그인할 때 6자리 검증 코드를 입력하라는 메시지가 표시됩니다.

  1. DSM 로그인 화면에서 평상시처럼 사용자 이름과 패스워드를 입력합니다.
  2. 검증 코드를 입력하라는 요청을 받으면 모바일 기기에서 인증기 앱을 엽니다.
  3. 계정에 대한 6자리 검증 코드를 찾아서 입력합니다. 모바일 기기를 잃어버린 경우 폰을 잃어버리셨습니까? 링크를 클릭하면 이메일 주소로 긴급 코드가 전송됩니다.

참고:

  • SMTP 설정: 이메일을 통해 긴급 코드를 받으려면 제어판 > 알림에 있는 SMTP 서버 설정이 적절하게 지정되어 있어야 합니다.
  • 긴급 코드 제한: 각 사용자는 5개로 제한된 긴급 코드를 갖습니다. 제한된 수를 초과한 경우 2 단계 검증을 비활성화하고 나서 활성화해야 긴급 코드를 더 받을 수 있습니다.
  • 2단계 검증은 DSM 및 모든 관련 Synology 모바일 응용 프로그램에 적용될 수 있습니다(예: DSM mobile 및 DS file).

사용자의 2 단계 검증 설정 재설정하기:

administrators 그룹에 속하는 사용자는 다른 사용자의 2 단계 검증 설정을 재설정할 수 있습니다.

  1. 제어판 > 사용자로 이동합니다.
  2. 2 단계 검증을 재설정할 사용자를 더블 클릭합니다.
  3. 사용자 정보 탭에서 2 단계 검증 재설정 단추를 클릭합니다.

참고:

administrators 그룹에 속한 사용자가 없는 경우 DiskStation에서 물리적 재설정 단추를 눌러 DSM 설정을 재설정하고 2 단계 검증을 비활성화할 수 있습니다.

계정 활동

계정 활동은 현재 연결, 트러스트된 장치 및 로그인 기록 등 DSM 계정의 로그인 활동을 표시합니다.

계정 활동 확인하기:

계정 활동 단추를 클릭합니다. 아래와 같은 탭을 표시하는 팝업이 나타납니다:

  • 연결된 사용자: 이 탭에는 다음 정보가 표시됩니다:
    • 날짜와 시간: 계정에 접근한 시간입니다.
    • 클라이언트 이름 및 IP: 계정에 접근한 장치의 이름과 IP 주소입니다. 클라이언트 이름은 CIFS 프로토콜을 사용하여 계정에 접근했을 때만 표시됩니다.
    • 서비스: 계정에 접근하는 데 사용된 프로토콜입니다.
    • 리소스: 계정이 사용한 시스템 리소스입니다.
  • 트러스트된 장치: 이 탭에는 트러스트된 장치 관리 페이지가 표시됩니다.
    • 현재 장치가 트러스트 해제/트러스트되었습니다. 현재 장치를 트러스트하도록 선택한 경우 이 장치 트러스트를 클릭합니다. 그러면 이 장치에서 DSM에 로그인할 때 더 이상 2단계 검증이 필요하지 않습니다.
    • 다른 장치 트러스트 해제: 트러스트된 다른 장치(예: 다른 컴퓨터 또는 모바일 기기)를 트러스트 해제할 수 있습니다. 이는 이러한 장치의 DSM에 로그인할 때 2단계 검증이 요구됨을 의미합니다.
  • 로그인 기록: 이 탭에는 최근 로그인 세션 및 다음 정보가 표시됩니다:
    • 날짜와 시간: 계정에 접근한 시간입니다.
    • 이벤트: 사용자 이름과 IP 주소가 표시됩니다.

참고:

트러스트된 것으로 표시한 장치는 동일한 브라우저에서 동일한 계정으로 로그인하는 경우에만 기억됩니다. 이전에 장치를 트러스트했더라도 다음 시나리오에서는 2단계 검증을 위해 OTP(1회용 패스워드)를 입력하라는 메시지가 표시됩니다.

  • 다른 브라우저에서 동일한 계정으로 로그인한 경우.
  • 브라우저에서 쿠키가 제거된 경우.
  • 사용자 A로 로그인했을 때 장치를 트러스트한 다음 사용자 B로 로그인하여 동일한 장치를 트러스트합니다. 이러한 경우 다시 사용자 A로 로그인하면 2단계 검증을 위해 패스워드를 입력해야 합니다.

Synology NAS 백업

개요

하드 드라이브 오류나 자연 재해로 인한 예기치 못한 데이터 손실은 중요한 데이터에 위협이 될 수 있습니다. 특히 안정적인 백업 솔루션이 없을 때는 더욱 그렇습니다. Hyper Backup은 이전 데이터 사본을 보유/검색하고 특정 시점에서 응용 프로그램 및 시스템 구성을 복원하는 데 유용한 직관적인 통합 백업/복원 솔루션을 제공합니다. 다음 저장소 대상이 지원됩니다:

  • 로컬 공유 폴더(DiskStation에 연결된 외부 저장소 장치 포함)
  • 원격 Synology NAS
  • 원격 rsync 서버
  • 원격 OpenStack Swift 서버
  • 공개 클라우드(Amazon S3, Microsoft Azure 및 OpenStack Swift 등)

참고:

Hyper Backup은 특정 시스템 구성 및 응용 프로그램만 백업합니다. 자세한 내용은 Hyper Backup의 도움말 문서를 참조하십시오.

1. 로컬 공유 폴더/외부 저장소 장치에 로컬 백업 작업 생성

Hyper Backup은 서로 다른 유형의 대상에서 비슷한 백업 및 복원 절차를 제공합니다. 아래에서는 참고로 서로 다른 유형의 작업에 대해 데이터를 로컬로 백업하고 복원하는 방법을 보여 줍니다.

  1. DSM에 로그인하고 Hyper Backup 패키지를 찾습니다.
  2. 왼쪽 아래 구석에서 +를 클릭하고 데이터 백업 작업을 선택합니다.
  3. 로컬 백업 작업을 설정하려면 로컬 공유 폴더를 선택합니다.
  4. 백업 작업 생성을 선택하고 로컬 공유 폴더 또는 DiskStation에 연결할 외부 저장소 장치를 선택합니다.
  5. 포함된 데이터를 백업할 소스 폴더를 선택합니다. 암호화된 공유 폴더를 선택한 경우에는 백업하는 동안 해당 폴더가 마운트되어 있어야 합니다. 소스 및 대상이 동일한 볼륨에 있는 경우에는 소스 폴더에 주황색 느낌표가 표시됩니다.
  6. 백업할 응용 프로그램을 선택합니다.
  7. 필요에 따라 아래와 같은 백업 설정을 지정합니다.
    • 작업: 백업 작업의 이름을 지정합니다.
    • 작업 알림 활성화: 이메일, SMS 또는 푸시 서비스를 통해 작업 상태에 대한 알림을 받으려는 경우에 선택합니다.
    • 구성 백업 활성화: 사용자 자격 증명과 같은 시스템 구성을 백업합니다. 이 설정은 모든 백업 작업에 대해 기본적으로 적용됩니다.
    • 백업 데이터 압축: 백업 데이터를 압축하여 대상 저장소의 사용량을 줄입니다. 이 옵션을 선택하면 백업 시간이 더 필요합니다.
    • 전송 암호화 활성화(원격 백업에만 해당): rsync 서버 또는 공용 클라우드 같은 원격 대상에 전송하는 동안 백업 데이터를 암호화하려는 경우에 선택합니다.
    • 멀티파트 업로드 부분 크기(S3 호환 서버에만 해당): 백업 데이터를 지정된 청크 크기로 나눕니다.
    • 클라이언트측 암호화 활성화: 패스워드를 사용하여 대상 측의 원치 않는 액세스로부터 백업 데이터를 보호하려는 경우에 선택합니다.
    • 백업 일정 활성화: 백업 작업을 자동으로 수행할 시간을 예약하려는 경우에 선택합니다.
  8. 회전 구성표를 설정하려면 백업 회전 활성화를 다음과 같이 선택합니다.
    • 가장 초기 버전에서: 지정한 버전 수를 초과한 경우 가장 초기의 기존 백업 버전을 삭제합니다.
    • Smart Recycle: 지정된 개수의 버전을 초과하기 전까지 모든 백업 버전이 보관됩니다. 회전이 트리거되면 조건에 맞지 않는 버전이 먼저 회전됩니다. 기존의 버전이 모두 아래 조건을 충족하는 경우에는 가장 초기의 버전이 먼저 회전됩니다.
      • 과거 24시간부터 매시간 버전: 시스템이 매 시간 가장 초기에 생성된 버전을 보관합니다.
      • 과거 1일에서 1개월 사이의 매일 버전: 시스템이 매일 가장 초기에 생성된 버전을 보관합니다.
      • 1개월 이상된 매주 버전: 시스템이 매 주 가장 초기에 생성된 버전을 보관합니다.
    • 버전 개수: 보관할 수 있는 백업 버전의 최고 한도를 설정합니다.
    • 시각표: 사용자의 백업 일정 및 회전 구성표에 따른 백업 보유의 가능한 결과를 표시합니다.
  9. 적용을 클릭하여 설정을 완료합니다.
  10. 백업 작업을 즉시 수행하려면 백업 마법사의 지침에 따르고 예를 클릭합니다.

2. 로컬 공유 폴더/외부 저장소 장치에서 단일 파일 복원

  1. Hyper Backup으로 가서 왼쪽 패널에서 원하는 백업 작업을 선택하고 백업 탐색기(돋보기 아이콘)를 클릭합니다. 작업이 클라이언트측 암호화로 보호되는 경우에는 패스워드를 입력합니다.
  2. 복원할 파일을 찾아 선택합니다. 시각표 표시줄을 사용하여 복원할 버전을 선택할 수도 있습니다.
  3. 복원을 클릭하여 선택한 파일/폴더를 복원합니다.

3. 원하는 시점으로 전체 작업 복원

  1. 왼쪽 아래 구석에서 복원을 클릭하고 데이터를 클릭합니다. 작업이 클라이언트측 암호화로 보호되는 경우에는 패스워드를 입력합니다.
  2. 복원할 작업을 선택합니다. 다른 리포지토리의 데이터를 복원하려면 다른 리포지토리에서 복원을 선택하고 복원할 작업을 찾습니다.
  3. 시스템 구성을 복원할지 여부를 선택합니다. 시스템 구성 복원을 선택한 경우 아래 목록에서 복원할 구성을 선택합니다.
  4. 복원할 대상 파일/작업을 찾아 선택합니다. 시각표 표시줄을 사용하여 복원할 버전을 선택할 수도 있습니다.
  5. 복원할 응용 프로그램을 선택하십시오.
  6. 기본 복원 정보가 표시됩니다.
  7. 적용을 클릭하여 복원을 시작합니다.
  • 이 항목의 이전: Cloud Station을 사용하여 PC/Mac의 데이터 백업

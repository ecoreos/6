ログ受信

ネットワークを介してクライアント デバイスからログを受信するために、DiskStation を syslog サーバーとして機能させることができます。ログ受信には、3 つのフォーマットがサポートされています。BSD (RFC 3164)、IETF (RFC 5424)、カスタム です。受信したすべてのログは、ログ データベースに保管されます。

はじめる前に：

[ログ センター] > [アーカイブの設定] を選択し、ログを保管し、アーカイブする場所を選択します。

標準のログ受信の設定：

  1. [ログ センター] > [ログ受信] > [標準] を選択します。
  2. 必要なフォーマットで [別のデバイスからログを受け取る] オプションにチェックマークを付けると、BSD および IETF ログ受信を有効にすることができます。
  3. [転送プロトコル] から [UDP] または [TCP] を選択します。
  4. syslog メッセージを受信するために、[ポート] にポート番号を入力します。デフォルトのポート番号は「514」です。
  5. ログを安全に転送するには、[セキュア接続（SSL）を有効にする] オプションにチェックマークを付けてください。このオプションは、[TCP] が選択されている場合に限り有効です。
  6. [適用] をクリックして、設定を保存します。

カスタム ログ受信の設定：

  1. [ログ センター] > [ログ受信] > [カスタム] を選択します。
  2. [作成] をクリックします。
  3. [名前] 欄に、規則に独自の名前を付けてください。
  4. [規則のパラメータ] 欄に解析規則を入力します。パターン パーサーの作成については、下記の情報をご参照ください。
  5. [転送プロトコル] から [UDP] または [TCP] を選択します。
  6. syslog メッセージを受信するために、[ポート] にポート番号を入力します。デフォルトのポート番号は「514」です。
  7. ログを安全に転送するには、[セキュア接続（SSL）を有効にする] オプションにチェックマークを付けてください。このオプションは、[TCP] が選択されている場合に限り有効です。
  8. [OK] をクリックして、設定を保存します。

注意：

パターン パーサーについては、こちらをお読みください。参照用にサンプル パターンもダウンロードできます。

カスタム規則を管理する：

  1. 既存のカスタム規則を選択します。
  2. 次の中から選択してください。
    • 編集： 選択したカスタム規則を編集します。
    • 削除： 選択したカスタム規則を削除します。
    • 有効： 選択したカスタム規則を有効にします。
    • 無効： 選択したカスタム規則を無効にします。

エクスポートする証明書

[セキュア接続（SSL）を有効にする] オプションにチェックマークを付けたとき、[証明書をエクスポート] ボタンをクリックすると、DiskStation にログを送信するときに他のデバイスを認証するための証明書がエクスポートされます。

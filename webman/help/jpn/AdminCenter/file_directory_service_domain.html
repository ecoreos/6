ドメイン

DiskStation を Windows ドメインに加えることができます。ドメインに加えると、わざわざ別のユーザー名とパスワードの組み合わせを覚えなくても、ドメイン ユーザーは各自のドメイン アカウントとパスワードを使って DiskStation にログインし、ファイルにアクセスしたり、DSM アプリケーションを使用したりすることができます。

ドメインに加入する：

  1. [コントロール パネル] > [ディレクトリ サービス] > [ドメイン] を選択します。
  2. [ドメインに参加] オプションにチェックマークを付けます。
  3. 相当する欄にドメイン名を入力し、DNS サーバーを指定します。
  4. 必要であれば、[詳細ドメインオプション] にチェックマークを付けます（詳細は、次の説明をお読みください）。
  5. [適用] をクリックします。
  6. 開いたウィンドウで、ドメインサーバーの管理者用ユーザー名とパスワードを入力します。
  7. [OK] をクリックします。

DNS Server

DiskStation でドメイン検索をできるようにするため、DNS サーバーを指定するよう推奨しています。ただし、ドメインが DNS サーバーに登録されていない場合は、ドメイン加入処理ができない場合があります。

ドメインサーバーの種類

DiskStation がドメインに加入すると、この欄にはドメイン タイプが表示されます。ドメインタイプは「AD ドメイン」になります。

管理モード

このオプションは、ドメイン ユーザーとグループの権限を管理する方法を決定します。

  • 信頼できるドメイン：結合したドメインと信頼できるドメインのユーザーとグループを管理できます。ユーザーとグループは、ドメイン ドロップダウン リストでフィルタ化できます。
  • OUのシングルドメイン：このモードでは、結合したドメインのユーザーとグループのみ表示されます。このモードでは、ドメイン ユーザー/ドメイン グループを OU (Organizational Unit) ドロップダウン リストでフィルタ化できます。

詳細ドメイン オプション

ほとんどの場合、詳細ドメインオプションの設定は必要ありません。詳細ドメインオプションは、特定のドメイン環境でのみ必要です。

  • DC IP/FQDN：DC（ドメインコントローラー）IP アドレスまたは FQDN を指定すると、DiskStation がその情報を使用して通信しようとします。複数の IP アドレスまたは FQDN が存在する場合は、IP の間にコンマを入力します。必要に応じて最後の DC IP アドレスまたは FQDN の後にアスタリスク(*)を追加すると、指定したすべての IP アドレスで失敗したときに DiskStation が別の DC との通信を試みます。このとき、最後の IP アドレスまたは FQDN とアスタリスクの間には必ずコンマを追加してください。
  • ドメインNetBIOS名：ドメインの NetBIOS 名を指定します。
  • Domain FQDN (DNS name)：ドメインの FQDN (DNS名)を指定します。
  • DNS インターフェイスの登録： ドメインを加えると、ここで指定した NIC (Network Interface Card) は DNS サーバーに登録されます。DiskStation のホスト名にアンダースコア（_）が含まれていると、DNS にはアンダースコアは使用できないため、登録できません。
  • ユーザー/グループ リストを更新：DiskStation がドメイン ユーザー／グループ リストを自動更新する間隔を指定します。この間隔は、毎日、毎週、毎月のいずれかで設定できます。また、[ドメイン ユーザー] タブを選択し、[すべてのドメインを更新] をクリックすると、ドメイン ユーザー／グループ リストを手動で更新できます。自動更新はシステムのハイバネーションに影響しますのでご注意ください。

読み取り専用ドメイン コントローラに加入する

DiskStation を読み取り専用ドメイン コントローラ (RODC) に加入するには、[DC IP/FQDN] 欄で読み書き可能ドメイン コントローラ (RWDC) の「コンピュータのフルネーム」(FQDN) を指定することができます。フルコンピュータ名は次のフォーマットに従う必要があります。ComputerName.Domain 例：「ad01.synology.com」

注意：

  • ドメイン ユーザー名に「%」や「$」が含まれていると、ホームフォルダにアクセスできない場合があります。その場合は、ドメイン管理者に新しいユーザー名を要請してください。
  • ドメイングループ「Domain Admins」と「Enterprise Admins」は、自動的に「Administrators」という名前のローカルグループに追加されます。要するに、これらのグループのドメイン ユーザーは、CIFS/FTP/AFP/WebDAV、その他の DSM アプリケーションなどのファイル サービスへのアクセスをはじめ、DiskStation についての管理権を有することになります。
  • また、DiskStation の共有フォルダへアクセスする権限をドメインユーザーに設定することもできます。詳しくは、「こちら」を参照してください。

ドメイン オプション

このセクションには、次のようなドメインの追加オプションがあります。

  • DiskStation はドメインユーザーがログインすると、毎回 NTP サーバーと同期を取る： DiskStation と NTP サーバーの間で強制的に同期させます。
  • サーバーの署名を有効にする： クライアントでクライアントの署名を有効にしてある場合は、ユーザーがドメイン機能を正常に使用できるように、サーバーの署名を DiskStation で有効にする必要があります。
  • NT4 対応モードでユーザー/グループ リストを取得： NT4 RPC モードでシステムにユーザー／グループ リストを取得させます。このオプションは、デフォルトの設定であるドメイン ユーザー／グループ リストを取得できないときに有効にしてください。
  • Domain Admins と Enterprise Admins の管理者権限を無効にする： 管理者の権限を持つユーザーには、DiskStation とそのファイルを管理する権限があります。このオプションを有効にすると、Domain Admins と Enterprise Admins の権限を普通のユーザーと同じにして、すべての普通のユーザー ファイル/フォルダの権限を継承させることができます。
  • ネスト構造のグループレベルを列挙する： ネスト構造の数と番号を付けられるドメイン グループのメンバーを指定します。

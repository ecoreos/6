FTP / FTPS

[コントロール パネル] > [ファイル サービス] > [FTP] を選択すると、FTP サービスを有効にしてクライアントがシステム上のフォルダやデータにアクセスできるようにすることができます。FTP を有効にする前に、ルーター上の次の TCP ポートが DiskStation デバイスに転送されていることを確認してください。21（デフォルトのコントロール接続）、20（アクティブ モードでのデータ接続）、55536-55567（受動モードでのデータ接続）。

FTP サービスを有効にする：

  1. 以下のいずれかのオプションを有効にします。
    • FTP サービスを有効にする（暗号化しない）： ファイル転送に使用される、標準のネットワーク プロトコルです。FTP は転送セッション中にパスワード、ユーザー名、ファイルといった、情報を保護するための暗号化を行いません。ただし、転送速度はより速く、システム リソースも少ない量で済みます。
    • FTP SSL/TLS 暗号化サービスを有効にする (FTPS)： TLS (Transport Layer Security) と SSL (Secure Sockets Layer) 暗号化プロトコルを組み合わせた標準 FTP の継続型で、パスワード、ユーザー名、ファイル等の情報を暗号化します。暗号化を行う分、転送速度は遅くなりますし、必要なシステム リソースも増えます。
  2. [適用] をクリックします。

タイムアウト

指定した時間が過ぎると、無作動のユーザーは FTP サーバーへの接続が解除されます。

FTP サービスのポート番号の設定

このセクションでは、FTP コントロール チャンネルで使用されるポートを指定します。

注意：

FTP コントロール チャンネルのデフォルトポート番号は 21 です。次の番号を除き、1 から 65535 の範囲で設定できます。20、22、23、25、80、110、137、138、139、143、199、443、445、515、543、548、587、873、993、995、3306、3689、5000、5001、5005、5006、5335、5432、6881、8080、7000、7001、8081、9997、9998、9999、50001、50002、eMule デフォルトポート：4662 (TCP)、 4672(UDP)

受動 FTP のポート範囲

このセクションでは、受動 FTP 接続で使用されるポート範囲を指定します。デフォルトのポート範囲を使用するか、自分で範囲を指定することができます。ポート範囲は 1025 から 65535 の範囲で設定でき、128 個のポートが含まれています。

PASV モードで外部 IP をレポートする

このオプションを有効にすると、サーバーは外部 IP アドレスを FTP クライアントにレポートします。レポートされた IP アドレスは、FTP クライアントの接続ログに表示されます。このオプションを有効にすると、外部 IP アドレスの自動検出を選択するか、DiskStation の外部 IP アドレスを手動で入力してください。

このオプションは、DiskStation が NAT の背後にあり、FTP クライアントが DiskStation 以外の別のサブネットに属している場合にのみ作動します。ほとんどの場合、このオプションを設定する必要はありませんが、FTP クライアントが DiskStation に接続できない場合は、このオプションを使ってやり直すことができます。

FXP を有効にする

FXP とは、File eXchange Protocol の略です。このオプションを有効にすると、FTP サービスがサーバーからサーバーへのファイル転送機能をサポートします。

FIPS 暗号化モジュールを有効にする

FIPS は OpenSSL FIPS 140-2 により有効化された暗号モジュールを使用したとき、FTPS サービスでサポートされます（認証 #1747）。このオプションを有効にすると、より安全なセッションを提供するために、FTPS サービスは追加のセキュア暗号化アルゴリズムを使用します。

ASCII 転送モードへの対応

このオプションを有効にすると、ASCII 転送モードをサポートします。このようにすると、DiskStation はASCII フォーマットの書式なしテキスト ファイルを FTP で転送できるようになります。

UTF-8 エンコーディング

このオプションは、FTP クライアントを正しく表示し、多言語のファイル名を持つファイルを処理するために使用します。さまざまな FTP クライアントの需要を満たすために、次の3つのオプションがあります。

  • 無効：UTF-8 エンコードは使用されません。このオプションは、クライアントが FTP クライアントとして Windows File Explorer を使用し、多言語のファイル名を持つファイルをアップロード/ダウンロードするためのものです。通常の使用にはお薦めしません。
  • 自動：DSM に UTF-8 エンコードをどのように処理させるかを自動的に決定させます。このオプションは、通常の使用にはお薦めしません。
  • 強制：UTF-8 エンコードを強制的に使用します。FTP クライアントで UTF-8 エンコードを有効にしたのに、多言語のファイル名を持つファイルが正しく表示されない場合は、このオプションを有効にしてください。

接続制限

このセクションには、次のオプションがあります。

  • IP 別に接続を制限する： この機能を利用すると、サーバーは、IP 別に最大接続数を制限します。この機能と FTP 帯域幅制限を有効にして幅広帯域を効率良く管理します。
  • FTP 帯域幅の制限を有効にする： この機能を有効にすると、1 つの FTP 接続の帯域幅を制限することができます。

注意：

  • 「guest」 アカウントは FTP からサーバーにログインできません。
  • FTP クライアントのコンピュータのコードページ設定は、正しくデータにアクセスするためには DiskStation である必要があります。
  • モデルによっては、FTP サービスは SSL または TLS を使用する接続をサポートします。
  • FTP クライアントのファイアウォール設定と、2 つの FTP サーバーは正しく設定しなければ、FXP 機能が正しく作動しない場合があります。
  • 一部の FTP クライアント（Internet Explorer 8/9 を使用しているユーザーなど）は、それぞれのユーザー名とパスワードでログインする必要があります。例えば：ftp://username:password@your.ftp.com

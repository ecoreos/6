Create a User

You can create and manage multiple user accounts on your DiskStation.

To create a user:

  1. Go to Control Panel > User.
  2. Click the Create button and then Create user. This action launches the User Creation Wizard.
  3. On the User information page, specify the following options:
    • Name: Enter a name for the user. This section is required.
    • Description: Enter a brief description for the user.
    • Email: Enter an email address for the user. System notifications, such as password reset messages, are sent to the address specified here.
    • Send a notification mail to the newly created user: If this option is enabled, the system will send a message to the user to notify him regarding the new account. This option requires enabling system email notifications at Main Menu > Control Panel > Notification > E-mail. By default the user's password is not displayed in the notification message. However, if you wish to display the password in the notification message, please check the Display user password in notification mail box.
    • Disallow the user to change account password: If this option is enabled, the user will not be able to change his password used to log into DSM.
  4. On the Group > Create > Group Creation Wizard page, specify the groups to which the new user shall belong (e.g. users, administrators, etc.).
  5. On the Assign shared folders permissions page, choose what shared folders the new user can access by modifying his access privileges. Privileges priority is as follows: No access > Read/Write > Read only.
    • Preview: This column displays the user’s actual shared folder access privileges, according to current settings and groups to which the user belongs.
    • Group permissions: This column displays shared folder access privileges assigned according to the user groups to which the user belongs.
  6. On the User quota setting page, you can control how much storage space can be used by the new user. To restrict the amount of space the user can use, check the box under Enable quota for each volume/shared folder. Then enter the maximum amount of space that can be used in the Quota field (e.g. 10 GB).
  7. On the Assign application permissions page, you can control which services the user can access.
  8. On the Group Speed Limit Setting page, you can enable a speed limit for different services (e.g. File Station, FTP, etc) to restrict the amount of bandwidth consumed by the user when transferring files. For each services, you can select one of the following:
    • None: The speed limit settings will be the same as the group to which the user belongs.
    • Constant: A fixed, constant speed limit will be applied. You can specify the upload and download speed limits in the fields to the left.
    • Variable: Two different speed limits can be specified and applied to the user according to a schedule. Click the Customize button to modify speed limit settings and the schedule.
  9. After confirming settings, click Apply to finish.

Username limitation:

  • The user name is not case sensitive and can include 1 to 64 Unicode characters, but cannot contain the following symbols:
    ! " # $ % & ' ( ) * + , / : ; < = > ? @ [ ] \ ^ ` { } | ~
  • The first character cannot be a minus sign or a space, and the last character cannot be a space.
  • The following user names are reserved for system use: mailer-daemon, postmaster.
  • The user description can include up to 64 displayable Unicode characters.

Password limitation:

  • The password is case sensitive and is limited up to 127 displayable characters, including letters, numbers, signs, and space.

Quota limitation:

  • The maximum storage usage quota that can be entered for each user is approximately 4 TB (4095 GB).

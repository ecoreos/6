LAN/インターネット

DiskStation のモデルとネットワーク設定に応じて、LAN やインターネットに接続するために DHCP の設定を行ってください。[コントロール パネル] > [ネットワーク] > [ネットワーク インターフェイス] の順に選択してください。DHCP とは自動 IP 割当てを行うネットワークプロトコルです。DHCP でシステム IP を割り当てるには、[編集] をクリックした後、[IPv4] タブで [DHCP サーバーのネットワーク設定を自動的に取得する] を選択します。

ネットワークサービスプロバイダの指定する IP アドレスを使用している場合は、[手動で設定する] を選択して、IP アドレスとサブネットマスクを入力します。DNS サーバーまたはデフォルトゲートウェイを利用しない場合は、未指定のままにしておいてください。

注：

  1. E メール通知用に適切な SMTP サーバー、有効な DNS サーバー IP が入力されているか確認してください。
  2. Jumbo Frames は、最大転送単位 (MTU) の標準 1500 バイトを超える Ethernet フレームで、サイズの大きいファイルの Ethernet 転送を効率よく行います。この機能は、ギガバイトのネットワーク環境でのみ有効です。Jumbo Frame が正しく機能するためには、ネットワーク経由で DiskStation にアクセスするすべてのコンピュータおよびデバイスが Jumbo Frame をサポートし、同じ MTU 値を使用していることを確認します。

VLAN

ネットワーク環境が VLAN 設定を必要とする場合は、「編集」画面の IPv4 タブで、VLAN オプションを有効にしてネットワーク環境の VLAN ID を入力します。

注：

  • ネットワーク スイッチが正しく構成されているかどうかを確認してください。そうしなければ、DiskStation のサービスにアクセスできなくなる場合があります。
  • 上記の設定を修正した後も DiskStation に接続できない場合は、DiskStation の [RESET] ボタンを押してデフォルト設定を読み込んでください。

IPv6 設定

DiskStation は IPv4/IPv6 デュアルスタックテーマをサポートしています。つまり、IPv4 および IPv6 の両方が DiskStation で同時に機能します。IPv6 のデフォルト設定は自動です。既存のネットワーク インターフェイスを選択し、[編集] ボタンをクリックして IPv6 タブを開き、設定を変更できます。

IPv6 の設定を行う：

  1. [コントロール パネル] > [ネットワーク] > [ネットワーク インターフェイス] の順に選択します。
  2. 既存のネットワーク インターフェイスを選択し、[編集] をクリックします。
  3. IPv6 タブの IPv6 セットアップからセットアップ タイプを選択する：
    • 自動：LAN Router Advertisement (RA) に応じて、DiskStation が自動的に IPv6 アドレスを生成します。
      • ステートレス モード：ルーターに接続されたデバイスが、IPv6 DNS サーバーを設定せずに IPv6 アドレスを自動設定できるようにします。
      • ステートレス DHCPv6 モード：ルーターに接続されたデバイスが、DHCPv6 から IPv6 アドレスを自動設定し、DNS 情報を取得できるようにします。
      • ステートフル モード：ルーターに接続されたデバイスが、DHCPv6 から IPv6 アドレスと DNS 情報を取得できるようにします。このモードでは、DHCPv6 サーバーを有効にして、 DHCPv6 の IP アドレスの先頭とIP アドレスの終了を設定する必要があります。
    • 手動：IPv6 アドレス、Prefix の長さ、およびデフォルトゲートウェイを入力して IPv6 を手動で設定します。
    • オフ：IPv6 サポートを無効にします。

IPv6 が完全に IPv4 に引き継がれるまで、IPv4 アドレスを持つホストが IPv6 サービスにアクセスできるように、IPv6 トランジション メカニズムが必要となります。異なる IPv6 トランジション メカニズムから選ぶ：

  • 6in4: トンネル ブローカーから IPv6 ネットワークに接続します。IPv6 アドレス、Prefix の長さ、Prefix、トンネル ブローカーの IPv4 アドレスを入力する必要があります。これにはパブリック IPv4 アドレスが必要です。
  • 6to4: エニーキャストを使って、トンネル ブローカーから IPv6 ネットワークに接続します。これにはパブリック IPv4 アドレスが必要です。
  • DHCP-PD: LAN 内の prefix delegation を介して、DHCP サーバーから IPv6 prefix を取得します。これには ISP サポートが必要であり、ルーター モードでのみ作動します。
  • IPv6 トンネリング (TSP)：[コントロール パネル] > [ネットワーク] > [全般] の順に選択してこのオプションを設定します。

802.1X

802.1X は、ユーザーが接続するネットワーク用に IEEE が開発した認証規格です。802.1X はクライアントを認証するためにネットワークで使用され、認証されなかったクライアントが保護されたネットワークにアクセスするのを阻止します。

Synology NAS を 802.1X 認証で保護されたネットワークに接続させることができます。

802.1X 認証を有効にする：

  1. [コントロールパネル] > [ネットワーク] > [ネットワーク インターフェイス] を選択します。
  2. 有線ネットワーク インターフェイス（LAN など）を選択し、[編集] をクリックします。
  3. [802.1X] タブを開き、[802.1X 認証を有効にする] にチェックマークを付けます。
  4. 認証ドロップダウン メニューで次のいずれかのプロトコルを選択します。最も適したプロトコルがわからない場合は、[自動] を選択してください。Â Â Â Â 
    • PEAP:セキュア接続を確立するには、CA 認証（任意）を選択します。このセキュア接続は、内部認証で使用されます。
    • TTLS:セキュア接続を確立するには、CA 認証（任意）を選択します。このセキュア接続は、内部認証で使用されます。
    • TLS: TLS (Transport Layer Security) を使って認証します。
  5. TTLS または PEAP を選択する場合は、ドロップダウン メニューから内部認証プロトコルの1つを選択してください。
    • MS CHAP v2:認証情報は、Microsoft CHAP v2 で暗号化されます。
    • GTC:認証情報は GTC (Generic Token Card) を使って暗号化されます。
  6. ユーザー名とパスワードを入力します。
  7. 必要であれば、[インポート] をクリックした後で [参照] をクリックし、Radius Server からエクスポートされた証明書を探してインポートしてください（例：ca.crt）。
  8. [OK] をクリックして、設定を保存します。

注意：

  • ワイヤレス ネットワーク用に 802.1X 認証を構成するには、[コントロールパネル] > [ワイヤレス] > [Wi-Fi] を選択します。
  • 認証設定は、Radius Server で設定したものと同じでなければなりません。詳しくは、Radius Server の管理者にお尋ねください。

ワイヤレス LAN*

DiskStation をワイヤレス ルーターとして機能させる場合、ワイヤレス LAN の次の設定を行うことができます。[コントロール パネル] > [ネットワーク] > [ネットワーク インターフェイス] を選択してください。

IPv4 設定

IP アドレスとサブネット マスク

ここでは、DiskStation の IP アドレスとサブネット マスクを設定できます。

DHCP Server

DHCP Server を有効にするオプションを有効にすると、DiskStation に DiskStation が提供するローカル ネットワークに接続するクライアント デバイスの IP アドレスを割り当てさせます。

PPPoE リレー

PPPoE リレーを有効にすると、DiskStation に接続されたデバイスに PPPoE 接続を介してインターネットにアクセスさせます。

UPnP インスタント ゲートウェイとして設定

UPnP 機能を有効にすると、DiskStation を UPnP インターネット ゲートウェイ デバイスとして機能させ、UPnP で DiskStation のローカル ネットワークに接続しているデバイスがポート転送設定を行えるようにします。

IPv6 設定

6in4、6to4、DHCP-PD で IPv6 アドレスと prefix を LAN/インターネットから取得できます。この情報があると、ここで DiskStation の IPv6 ルーティング オプションの設定を行うことができます。

  • IPv6 アドレス：LAN/インターネットから取得した IPv6 アドレスを入力します。
  • Prefix の長さ：LAN/インターネットから取得した IPv6 アドレスの prefix の長さを入力します。通常は 64 です。
  • プライマリ DNS：ルーターに接続したデバイスのプライマリ DNS サーバーです。これはステートレス モードで設定する必要はありません。
  • セカンダリ DNS：ルーターに接続したデバイスのセカンダリ DNS サーバーです。これはステートレス モードで設定する必要はありません。

3つの Router Advertisement (RA) モードがあります。

  • ステートレス モード：ルーターに接続されたデバイスが、IPv6 DNS サーバーを設定せずに IPv6 アドレスを自動設定できるようにします。
  • ステートレス DHCPv6 モード：ルーターに接続されたデバイスが、DHCPv6 から IPv6 アドレスを自動設定し、DNS 情報を取得できるようにします。
  • ステートフル モード：ルーターに接続されたデバイスが、DHCPv6 から IPv6 アドレスと DNS 情報を取得できるようにします。このモードでは、DHCPv6 サーバーを有効にして、 DHCPv6 のIP アドレスの先頭とIP アドレスの終了を設定する必要があります。

DHCP クライアント

ここでは、DHCP クライアントの一覧とネットワーク設定を見ることができます（MAC アドレス、IP アドレス、ホスト名、DHCP の有効期間の残り時間）。

IPv4/IPv6 クライアントをフィルターにかけて、それぞれ IPv4 と IPv6 を使用しているクライアントを見分けることができます。クライアントを選択し、アドレス予約に追加をクリックしてアドレス予約リストにクライアントを追加します。

DHCP 予約

DHCP リースの更新中にも、クライアントがいつも同じ IP アドレスを使用できるようにしたい場合は、そのクライアントをアドレス予約リストに追加することができます。

IPv4/IPv6 アドレス予約リストをフィルターにかけることができます。IPv4 アドレスは MAC アドレスによって認識され、IPv6 アドレスは DUID によって認識されます。

アドレス予約リストにクライアントを追加する：

  • クライアントを選択し、追加をクリックします。
  • クライアントの MAC アドレス、DUID、クライアントに割り当てたい IPv4/IPv6 アドレス、クライアントのホスト名を指定します。
  • OKをクリックして、設定を保存します。

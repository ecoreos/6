リソースモニター

リソース モニタを使うと、CPU 使用状況、メモリ使用状況、ディスクの使用、ネットワークフローなどが監視できます。リアルタイム監視、または以前のデータのいずれかを見ることができます。

リソース モニタを使うと、CPU 使用状況、メモリ使用状況、ネットワークフローなどが監視できます。リアルタイム監視、または以前のデータのいずれかを見ることができます。

注意：

  • [設定] タブで [使用履歴を有効にする] が選択されている場合に限り、以前のデータが表示されます。
  • Docker DSM の場合、CPU とメモリ は、Docker ホストについての使用量情報を示します。

性能

リソースの使用状況データを見るには、[性能] タブをクリックします。削除日時、チャートの情報は、マウスオーバーで表示されます。

CPU

ユーザー、システム、I/O 待機による全体的な CPU 使用量が表示されます。リソース モニタを初めて起動したときは、システムがリソースデータの収集と UI ページのロードを同時に実行するため CPU 負荷が高くなります。

ユーザーには、ユーザー アプリケーションの起動に使用する CPU のパーセンテージが表示されます。システムには、システムのコア、ハードウェア/ソフトウェアの IRQ で使用する CPU のパーセンテージが表示されます。I/O 待ちは、I/O 待ちで使用される CPU のパーセンテージを表示します。

使用状況は、ユーザーとシステムによる CPU の使用量の合計です。

1、5 から 15 分以内に、run-queue の長さを示す平均読み込みが表示されます（例：現在実行中の処理数と、処理待ちの合計）。

用語説明

  • I/O 待ち：I/O 待ちは、CPU がアイドル状態でプログラムで使用されていないことを意味しています。

    • I/O 待ちは正常な動作です。CPU の速度は I/O の速度よりもはるかに速いため、通常は CPU はディスクまたはネットワークの転送を待たなければなりません。しかし、I/O 待ちのパーセンテージが非常に高い場合は、CPU がほとんどの時間を I/O 待ちに費やすことになり、効率的に使用されない場合があります。より効率的なディスク グループ構成（例：SHR または RAID 5）またはネットワーク構成（例：ボンディング）を使用することで、システムの性能を高めることができます。

メモリ

物理メモリの全体的なメモリ使用量とスワップ スペースが表示されます。メモリ使用状況は、システムが頻繁にアクセスしたデータをキャッシュに保存するため高い状態が続きますが、そのため、データは、ハードディスクにアクセスする必要なく素早く取得することができます。キャッシュメモリは、全体的なメモリが不足すると解放されます。

スワップ スペースの使用量が高い場合は、システム メモリが不足していることを意味しており、システムの性能に影響を与えます。一番上のドロップダウン メニューから [スワップ] を選択すると、スワップインとスワップアウトのレートを確認することができます。

ネットワーク

ネットワークフロー図は、データの送受信速度を 1 秒間のデータ量 (kbp) で表示します。PPPoE を有効にすると、転送速度も表示されます。

ディスク

チャートの一番上にあるドロップダウン メニューをクリックすると、全体的なディスクの使用量、転送レート、または IOPS を見ることができます。一番下にある [すべて表示] をクリックすると、チェックボックスにチェックマークを付けてデータをチャートに追加すると、別のディスクも選択できます。

ボリューム

このページには、ボリュームの全体的な転送状態が表示されます。チャートの上のドロップダウン メニューをクリックすると、転送使用状況、転送レート、IOPS が表示されます。[すべて表示] ボタンをクリックして個々のボリュームや iSCSI LUN（ブロックレベル）を選択し、それらをチャートに追加してください。

iSCSI

このページには、現在起動中の iSCSI LUN の全体的な状態が表示されます。チャートの上にあるドロップダウン メニューをクリックして LUN を選択すると、その状態が表示されます。デフォルトではスループット、IOPS、レイテンシーの 3 つのチャートが表示されます。詳細は、[チャートの選択] ボタンをクリックして見たい他のチャートを選択してください。

プロセス

[プロセス] タブをクリックすると、各処理の CPU とメモリ使用量を見ることができます。

プロセスの状態には、作動中、スリープ中、停止があります。Linux でのプロセスの状態は、作動中、停止/トレーシング、スリープ中があります。

スリープ中は、次の文字でその種類が示されます。「D」はディスク スリープ、「Z」はゾンビ、「X」は死亡などです。

接続中のユーザー

このタブでは、現在システムに接続しているクライアンと、リソースにアクセスしているクライアントを表示、管理できます。このセクションには、HTTP、TELNET、SSH、SMB/CIFS、AFP、FTP、NFS を介して接続されているクライアントが表示されます。

現在の接続を管理するには、以下のいずれかを行います：

  • [保存] をクリックして、現在のリストを HTML ファイルにエクスポートします。
  • [最新の情報に更新] をクリックすして、リストを更新してください。
  • ユーザーがリソースにアクセスするのを禁止するには、[接続を遮断する] をクリックします。接続を解除されたユーザーは、もう一度認証を受けなければリソースにアクセスすることはできません。
  • ユーザーを無効にするには、[ユーザーを無効にする] をクリックします。再びそのユーザーを有効にするには、メイン メニュー > [コントロール パネル] > [ユーザー] の順に選択します。

注意：

  • [接続を解除する] オプションは、NFS クライアントでは使用できません。
  • NFS クライアントで共有フォルダが適切にアンマウントされていなければ、接続しているユーザーとして表示されます。
  • DiskStation が再起動すると、以前接続した NFS クライアントは接続しているユーザーとして表示されません。

速度制限

[速度制限] タブでは、DiskStation で速度制限が適用されているファイル転送処理のリストを見たり、管理したりすることができます。

リストを更新する：

[最新の情報に更新] をクリックして、ページを再ロードし、最新のログ記録を表示します。

ファイル転送を中止する：

[接続を解除する] をクリックするとファイル転送が中止されます。処理を完了するには、再びファイル転送を開始する必要があります。

設定

[設定] をクリックして、次のオプションを設定してください。

  • ドロップダウン メニューから [リアルタイムの時間] を分で選択します。
  • 以前のデータを見るには、[使用履歴を有効にする] にチェックマークを付けます。これを有効にすると、パフォーマンス チャートを表示しているときに、右上隅にあるドロップダウン メニューからリアルタイム、毎日、毎週、毎月、毎年の間で切り替えることができます。

[適用] をクリックして設定を保存します。

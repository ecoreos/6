バックアップ先

[バックアップと複製] を選択すると表示される [バックアップ先] タブでは、バックアップ タスクを作成する前に、バックアップ先としてローカル、ネットワーク、クラウドのいずれかを選択できます。

ローカル バックアップ先を作成する：

DiskStation からローカルの共有フォルダ、ボリューム、または外部 USB/eSATA ドライブにデータをバックアップするには、ローカル バックアップ先が必要です。バックアップ先とバックアップ元を同じにすることはできません。また、バックアップ先の容量がデータの合計よりも大きくなければなりません。

  1. [作成] をクリックして [ローカル バックアップ先] を選択した後で、[次へ] をクリックします。
  2. このバックアップ先に名前を付けて、バックアップ データを保存しておく場所を選択した上で、最新のバックアップ コピーだけを維持するのか、またはすべてのバックアップ バージョンを維持するのかを選択してください。

注意：

  • バックアップを実行しているときにバックアップ先のデータを削除すると、次回タスクを実行するときに完全バックアップが行われます。
  • 新しい外部ドライブをバックアップ先として使用するには、先に初期化しておく必要があります。
  • バックアップ タスクを作成する前に、外部ドライブが正しく DiskStation に接続されているか確認してください。[コントロール パネル] > [外部デバイス] を選択し、USB/eSATA ドライブの状態を確認してください。外部ドライブが初期化、フォーマット、故障していると、外部ドライブをバックアップ先として選択することはできません。
  • リモートでマウントした共有フォルダをバックアップ元またはバックアップ先として選択することはできません。
  • 暗号化された共有フォルダがマウントされていない場合は、バックアップ先として選択できません。
  • バックアップ先のファイル システムが NTFS/FAT の場合は、画像サムネイルをバックアップすることはできません。

ネットワーク バックアップ先を作成する：

DiskStation から別の DiskStation、または rsync 対応のサーバーにネットワークを介してデータをバックアップするには、ネットワーク バックアップ先が必要です。ネットワーク バックアップ先にバックアップする場合は、転送暗号化、圧縮、ブロックレベルのバックアップなど、各種オプションがあります。転送の暗号化など、一部のオプションを選択した場合は、バックアップ速度が遅くなることがあります。

  1. ネットワーク バックアップ先を作成する前に、バックアップ先のサーバーにまずログインして、[バックアップと複製] > [バックアップ サービス] を選択し、ネットワーク バックアップ サービスやネットワーク バックアップ ボリュームの保存先を有効にします。
  2. DiskStation に戻り、[バックアップと複製] > [バックアップ先] を選択します。[作成] をクリックし、[ネットワーク バックアップ先] を選択してウィザードの示す手順に従って設定を完了してください。
  3. [ネットワーク バックアップ先]（DiskStation サーバー） を選択すると、バックアップ データを共有フォルダに保存して最新のバックアップ コピーを維持することができます。または、ボリュームにバックアップ データを復元して、複数バージョンのバックアップ コピーを保持することができます。[ネットワークバックアップ先（rsync 対応サーバー）] を選択すると、最新のバックアップ コピーだけが維持されます。

注意：

  • ネットワーク バックアップ先として、rsync 対応サーバーにバックアップする場合は、次のモードを選択できます。「バックアップ モジュール」の横にあるドロップダウン メニューからバックアップ モジュール（rsync 対応サーバー上の共有フォルダ）を選択するときに使用する「rsync daemon モード」、「バックアップ モジュール」に絶対パス（[/] で始まります）を入力する時に有効になる「remote shell モード」があります。
  • Linux のユーザーの方は、バックアップに「admin」、「root」および「rsync」アカウントのみを実行することができます。
  • ホスト名を使ってバックアップ先サーバーにインターネットから接続する場合は、ホスト名は、「abc.bcd.com」のように FQDN（完全に記述したドメイン名）形式である必要があります。
  • DiskStation デバイスから別のデバイスにバックアップする場合は、バックアップ先でアカウントのユーザー名とパスワードを入力するよう求められます。特に、別の DiskStation のボリュームにバックアップしたい場合、administrators グループに属するアカウントを指定してください。共有フォルダを別の DiskStation にバックアップするには、フォルダのアクセス権の設定に応じてそれを行うことができます。特定のユーザーアカウントがアクセス権を持つ共有フォルダだけが表示されます。バックアップ先の共有フォルダがない場合は、アクセス権を確認してください。
  • バックアップの暗号化は、バックアップ先サーバーが暗号化ネットワークバックアップに対応し、認証パスワードが正しく入力されている場合に限り有効です。
  • Remote shell（リモートシェル）モードは暗号化バックアップにのみ対応しています。
  • rsync 対応サーバーにバックアップすると、rsync バックアップ タスクを作成した後は、rsync 暗号化モードを変更することはできなくなります。設定を変更するには、新しい rsync タスクを作成してください。
  • クライアントを介してファイルを DiskStation にバックアップする場合は、ファイル名が UTF-8 エンコード化されていなければなりません。
  • Rsync 対応クライアントを介して DiskStation にファイルをバックアップする場合は、DiskStation の「admin」､「root」､「rsync」アカウントでしか実行できません。
  • DiskStation から rsync 対応サーバーへファイルをバックアップする場合は、正しい rsync バックアップ アカウント名とパスワードを入力する必要があります。また書き込み権を持って正しいバックアップ モジュールを選択する必要もあります。

Amazon S3 バックアップ先を作成する：

DiskStation から Amazon S3 サーバーへデータをバックアップするには、Amazon S3 バックアップ先が必要です。

  1. [作成] をクリックし、[パブリッククラウドのバックアップ先] を選択します。その次に、[Amazon S3 バックアップ先] をクリックしてから、[次へ] をクリックします。
  2. このバックアップ先に名前を付けて、ログイン情報を入力した後で、必要な項目に記入してバックアップ タスクを設定します。

注意：

  • ネットワーク バックアップ先として Amazon S3 サーバーにバックアップすると、転送暗号化、Amazon S3 RRS (Reduced Redundancy Storage) へのバックアップなど、各種オプションを設定できます。転送の暗号化など、一部のオプションを選択した場合は、バックアップ速度が遅くなることがあります。
  • バックアップ ファイルは Amazon S3 サーバーで選択した Bucket に保管され、Set KEY の値が選択したディレクトリ名で始まり、ファイルの完全なファイルパスで終わります。
  • Amazon S3 のデータ構造は、Bucket とフォルダ構造の無い KEY のペアで構成します。空のフォルダをバックアップすることはできません。
  • バックアップしたファイルに変更がない場合は、Amazon S3 サーバーの同じ Bucket には再度バックアップされません。
  • Amazon S3 の各 Bucket には固有の名前が付けられます。また、名前付けには次のような規則があります。
    • 3 から 63 文字以内でなければなりません。
    • 小文字、数字、ピリオド(.)、ダッシュ(-)のみ使用できます。
    • 数字または文字で始まらなければなりません。また、ダッシュ (-) で終了したり、2 つ続けてピリオドを使用したり、ピリオドとダッシュを続けて使用したりすることはできません（[my-.bucket.com] はできませんが、[my.-bucket] は可能です）。
  • Amazon S3 KEY の最大長は 1024 文字です。Amazon S3 へのバックアップを成功させるためには、最大パス長を 1024 文字から「タスク作成時に指定したバックアップ先フォルダ名」の長さを引いた長さにしてください。
  • Amazon S3 RRS（Reduced Redundancy Storage）サポートが有効になっている場合は、オプションを有効にした後で変更したデータだけが RRS ストレージにバックアップされます。つまり、Amazon S3 の標準ストレージにすでにバックアップされたデータは、RRS にはバックアップされません。
  • バックアップしたファイルを Amazon S3 から Amazon Glacier に移動させることはできません。AWS Management コンソールを介してファイルを Amazon S3 から Amazon Glacier に移動させると、関連するバックアップ タスクが失敗します。
  • SSL で仮想的にホストされたバケツにファイルをバックアップすることはできません。
  • ファイルを転送する場合、ファイルはパーツに分割されます。パーツのサイズ制限は、[Multipart Upload パートサイズ（MB）] オプションで設定してください。デフォルト値は 512MB です。ネットワーク環境が安定しない場合は、このオプションを小さい数値に設定するようお薦めします。
  • 1アップロードあたりのパーツの最大数は 10000 個です。ファイルの最大サイズは、Multipart Upload パートサイズに 10000 を掛けた数値です。
  • ファイル名に非 UTF-8 またはコントロール文字を使用することはできません。

Microsoft Azure バックアップ先を作成する：

Microsoft Azure バックアップ先を使用するには、データを DiskStation から Microsoft Azure blob サーバーにバックアップする必要があります。

  1. [作成] をクリックし、[パブリッククラウドのバックアップ先] を選択します。その次に、[Microsoft Azure バックアップ先] をクリックしてから、[次へ] をクリックします。
  2. バックアップ先に名前を付けて、ログイン情報を入力した後で、必要な項目に記入してバックアップ タスクを設定します。

注意：

  • Microsoft Azure サーバーをネットワーク バックアップ先としてバックアップ タスクを作成する場合、さまざまなオプションがあります（転送暗号化など）。転送の暗号化など、一部のオプションを選択した場合は、バックアップ速度が遅くなることがあります。
  • バックアップ ファイルは Microsoft Azure サーバーで選択した Container に保管され、Blob 名が選択したディレクトリ名で始まり、ファイルの完全なファイルパスで終わります。
  • Microsoft Azure のデータ構造は、フォルダ構造のない Container と Blob のペアで構成します。空のフォルダをバックアップすることはできません。
  • バックアップしたファイルに変更がない場合は、Microsoft Azure サーバーの同じ Container には再度バックアップされません。
  • Microsoft Azure の各 Container には固有の名前が付けられます。また、名前付けには次のような規則があります。
    • 3 から 63 文字以内でなければなりません。
    • 小文字、数字、ダッシュ (-) を使用できます。
    • 先頭と末尾の文字が数字か文字であり、ダッシュ（-）を続けて使用することはできません。
  • Microsoft Azure Blob 名の最大長は、1024 文字です。Microsoft Azure へのバックアップを成功させるためには、最大パス長を 1024 文字から「タスク作成時に指定したバックアップ先フォルダ名」の長さを引いた長さにしてください。
  • ファイル名に非 UTF-8 またはコントロール文字を使用することはできません。
  • 認証情報が正確でない、ネットワーク接続に問題がある、または DiskStation とリモート サーバーの時間が一致しない場合は、認証失敗警告メッセージが表示されます。DiskStation で時間の設定を確認するには、[コントロール パネル] > [地域オプション] > [時間] を選択して NTP サーバーで時間を合わせてください。

hicloud S3 バックアップ先を作成する：

DiskStation から hicloud S3 サーバーへデータをバックアップするには、hicloud S3 バックアップ先が必要です。

  1. [作成] をクリックし、[パブリッククラウドのバックアップ先] を選択します。その次に、[hicloud S3 バックアップ先] をクリックします。
  2. このバックアップ先に名前を付けて、ログイン情報を入力した後で、必要な項目に記入してバックアップ タスクを設定します。

注意：

  • hicloud S3 サーバーをネットワーク バックアップ先としてバックアップ タスクを作成する場合、さまざまなオプションがあります（転送暗号化など）。.転送の暗号化など、一部のオプションを選択した場合は、バックアップ速度が遅くなることがあります。
  • バックアップ ファイルは hicloud S3 サーバーで選択したバケツに保管され、Set KEY の値が選択したディレクトリ名で始まり、ファイルの完全なファイルパスで終わります。
  • hicloud S3 のデータ構造は、フォルダ構造の無い Bucket と KEY のペアで構成します。空のフォルダをバックアップすることはできません。
  • バックアップしたファイルに変更がない場合は、hicloud S3 サーバーの同じ Bucket には再度バックアップされません。
  • hicloud S3 の各 Bucket には固有の名前が付けられます。また、名前付けには次のような規則があります。
    • 3 から 63 文字以内でなければなりません。
    • 小文字、数字、ピリオド(.)、ダッシュ(-)のみ使用できます。
    • 数字または文字で始まらなければなりません。また、ダッシュ (-) で終了したり、2 つ続けてピリオドを使用したり、ピリオドとダッシュを続けて使用したりすることはできません（[my-.bucket.com] はできませんが、[my.-bucket] は可能です）。
  • hicloud S3 KEY の最大長は 255 文字です。hicloud S3 へのバックアップを成功させるためには、最大パス長を 255 文字から「タスク作成時に指定したバックアップ先フォルダ名」の長さを引いた長さにしてください。
  • ファイルを転送する場合、ファイルはパーツに分割されます。パーツのサイズ制限は、[Multipart Upload パートサイズ（MB）] オプションで設定してください。デフォルト値は 512MB です。ネットワーク環境が安定しない場合は、このオプションを小さい数値に設定するようお薦めします。
  • Hicloud は1回の転送につき、最大 1000 個のパーツしかサポートしていません。ですから、ファイルサイズの合計が 1000 x パーツサイズを超えないようにしてください。たとえば、[Multipart Upload パートサイズ] を 8MB に設定した場合、ファイルサイズが 8000MB を超える場合はそのファイルはアップロードされません。
  • 暗号化された共有フォルダをバックアップすることはできません。
  • ファイル名に非 UTF-8 またはコントロール文字を使用することはできません。

バックアップ先を管理する：

既存のバックアップ先を選択し、次のボタンの 1 つをクリックします。

  • 編集：選択したバックアップ先の設定を変更します。
  • 削除：選択したバックアップ先を削除します。

共有フォルダにバックアップする

このタイプのバックアップ先には、最新のバックアップ コピーしか保存されません。デフォルトでは内部バックアップが完了すると、次のタスクが開始されると、バックアップ元から削除されたファイルはバックアップ先から削除されます。ただし、バックアップ元から削除されてもそれらのファイルをバックアップ先に保持しておきたい場合は、バックアップ タスクを作成するときに [目的のフォルダにバックアップファイルを維持する] オプションを有効にしてください。

ボリュームにバックアップする

このタイプのバックアップ先は、DiskStation デバイスのローカルまたはネットワーク ボリュームでなければなりません（外部ストレージ装置を除く）。複数バージョンのバックアップ コピーがアーカイブされますので、指定した時点から簡単にフォルダを復元できます。

バックアップタスクで保存したデータを削除する：

  1. [バックアップ先] タブで、削除したいタスクがあるバックアップ先を開きます。
  2. 選択したバックアップ タスクの [削除] アイコンをクリックし、ダイアログ ボックスで [削除] をクリックします。

バックアップ バージョンで保存したデータを削除する：

  1. [バックアップ先] タブで、削除したいタスク バージョンがあるバックアップ先を開きます。
  2. 選択したバックアップ タスクの [バックアップ バージョン情報] アイコンをクリックします。
  3. 削除したいバックアップ バージョンを選択し、[削除] アイコンをクリックします。

バックアップ ローテーション中にバックアップが削除されないようにロックする：

  1. [バックアップ先] タブで、ロックしたいタスク バージョンがあるバックアップ先を開きます。
  2. 選択したバックアップ タスクの [バックアップ バージョン情報] アイコンをクリックします。
  3. ロックしたいバックアップ バージョンを選択し、[ロック] アイコンをクリックします。

バックアップ先の権限を管理する：

  1. ローカル ボリュームと関連づけられているバックアップ先を選択し、[編集] をクリックします。
  2. [ホワイト リスト]、次に [追加] をクリックしてユーザーまたはグループをリストに追加します。

バックアップ先に所有者を設定する：

  1. ローカル ボリュームと関連付られているバックアップ先を選択し、選択した [バックアップ タスク] の [プロパティ] アイコンをクリックします。
  2. [バックアップ権] でドロップダウン メニューから [所有者] を選択します。

DiskStation でユーザーまたはグループを許可して保存したバックアップ タスクのデータを読み取り、復元する：

  1. ローカル ボリュームと関連付られているバックアップ先を選択し、選択した [バックアップ タスク] の [プロパティ] アイコンをクリックします。
  2. [ホワイト リスト]、次に [追加] をクリックしてユーザーまたはグループをリストに追加します。

注意：

  • バックアップ タスクのデフォルトの所有者は、タスクを作成したユーザーです。
  • バックアップ タスクの所有者しかタスクを実行できません。ただし、DSM 5.1 以前に作成されたバックアップ タスクにこのような制限がない場合は、administrators グループに属するユーザーなら誰でもこのバックアップ タスクを実行できます。
  • タスクの所有者と administrators グループに属するユーザーは、保存したバックアップ タスクのデータを削除することができます。
  • タスクの所有者、administrators グループに属するユーザー、ホワイト リストのユーザーは、保存したバックアップ タスクのデータを復元することができます。
  • Administrators グループに属するユーザーだけが、アクセス権を持つローカル バックアップ先を設定したり、所有者とホワイト リストのユーザーがローカル バックアップ先にバックアップできるように設定することができます。
  • 保存されたバックアップ タスクのデータは、タスク所有者のストレージ割り当てを使用します。
  • ボリュームにバックアップ先がない場合は、administrators グループに属するユーザーがそのボリュームに最初のバックアップ先を作成しなければなりません。
  • Administrators グループに属するユーザーだけが、バックアップ先全体の保存データを削除できます。
  • 各バックアップにはシステム構成とファイルのメタデータが含まれます。
  • 暗号化された共有フォルダをバックアップすることはできません。
  • バージョン ロックは、DSM 5.1 以下のバージョンではサポートされていません。

バックアップ

バックアップと復元は、DiskStation に保存されているさまざまな種類のデータ（システム構成、共有フォルダ、アプリケーション/パッケージ）を手動で、または設定した予定にしたがってバックアップします。バックアップ データは、ローカル共有フォルダ、リモート サーバー、パブリック クラウド（Amazon S3、Microsoft Azure、hicloud S3、SFR NAS BACKUP）に保管できます。バックアップと復元を活用すると、それぞれのバックアップ タスクは複数のバックアップ バージョンが維持され、スマート リサイクル ポリシーに基づいて循環されます。

データ バックアップ

バックアップと復元には、バックアップ先やデータ バックアップ タスクを設定する方法など、さまざまなオプションがあります。

データ バックアップタスクを作成する：

  1. 左下にある + をクリックして、[データ バックアップ タスク] を選択すると、バックアップ ウィザードが起動します。
  2. バックアップ先のタイプを選択：
    • ローカル/リモート共有フォルダまたは外部ストレージ デバイス
    • リモート rsync サーバー
    • パブリック クラウド (例：Amazon S3 および Microsoft Azure)
  3. バックアップ タスクのタイプを選択：
    • 新しくタスクを作成するには、[バックアップ タスクの作成] を選択します。
    • バックアップ先にある既存のバックアップ データを再利用するには、[既存のタスクに再リンク] を選択します。再リンクすると、別のタスクからのバックアップ データを直接利用することができます。
  4. ウィザードの指示に従って設定を完了してください。[クライアントサイドの暗号化を有効にする] を選択すると、データを送信する前に暗号化され、バックアップ先でのデータの安全性が確保されます。

注意：

  • タスクの名前には、1 から 32 個のユニコード文字を使用できます。ただし、次の文字は使用できません。
    ! " # $ % & ' ( ) * + , / :; < = > ?@ [ ] \ ^ ` { } | ~
  • バックアップのフルパスの最大長は 2048 文字です。
  • AFP を介して 32000 個以上のファイルかフォルダを EXT3 共有フォルダにアップロードすると、共有フォルダを含むすべてのバックアップ タスクが失敗します。
  • 暗号化された共有フォルダは、マウントされたときにバックアップできます。また、暗号化した/暗号化を解除したフォルダに復元することができます。ただし、暗号化/////////されて復元され、スタートアップ時には自動的にマウントされません。暗号化した共有フォルダの暗号化キーを入力するか、インポートしてマウントする必要があります。
  • 暗号化したフォルダを別の暗号化フォルダにバックアップすることはできません。
  • バックアップ ソースまたはバックアップ先としてリモート フォルダを使用することはできません。
  • 特定アプリケーションのバックアップに対応しています。特定のパッケージがバックアップ、バックアップ コンテンツ、制限に対応しているかどうかを確認するには、各アプリケーションのヘルプを参照してください。
  • DiskStation は、書き込み可能な EXT3/EXT4 外部デバイスがないアプリケーションのバックアップには対応していません。
  • DiskStation は、EXT3/EXT4 外部デバイスで作成した共有フォルダしかバックアップしません。

Windows ACL 設定のバックアップ：

バックアップ先のファイルシステムによっては、DiskStation 上のファイルの Windows ACL 設定もバックアップされる場合があります。

  • 同期先のファイルシステムに関係なく、 すべてのバックアップ先は Windows ACL 設定をバックアップできます。

バックアップ データをパブリック クラウドかリモート Synology NAS にエクスポートする:

大量のデータをパブリック クラウドにバックアップするときには、最初のバージョンを外部ストレージ デバイス（USB ドライブなど）にエクスポートして、デバイスをクラウドサービス プロバイダーに配送することをお勧めします。そのようにすると、デバイスを接続するだけで直接データをクラウドにアップロードすることができます。この方法なら、初回のバックアップにかかる時間を大幅に短縮することができます。

  1. 左下にある + をクリックし、リモート Synology NAS かクラウド サービス プロバイダーを選択します。
  2. [ローカル共有フォルダにエクスポート（外部ストレージ デバイスを含みます）] を選択し、設定を完了してください。要求されたら、[クライアント再度の暗号化を有効にする] を選択してバックアップデータを保護します。
  3. 外部ストレージ デバイスをリモート Synology NAS かクラウド サービス プロバイダーに送ります。外部ストレージ デバイスからエクスポートしたデータをインポートします。
  4. これを確認できたら、バックアップと復元に戻り、[今すぐ再リンク] をクリックします。
  5. 再リンクのウィザードに従って DiskStation をレポジトリにリンクします。データが暗号化された状態でエクスポートされた場合は、パスワード/暗号化キーが必要です。

注意：

  • タスクが一旦設定されると、クライアントサイドの暗号化を無効にしたり、パスワードを変更したりすることはできなくなります。
  • バックアップ レポジトリは、1個のバックアップ タスクとしかリンクできません。
  • バックアップ元のフォルダが暗号化された共有フォルダ、またはそのサブフォルダである場合は、バックアップ元のフォルダはバックアップ タスクを作成するときにマウントしたり、マウント解除したりすることができます。
  • 暗号化した共有フォルダをバックアップ先にすることはできません。
  • [クライアントサイドの暗号化を有効にする] を選択すると、このタスクでデータを復元するには、パスワード/暗号化キーが必要です。パスワード/暗号化キーを忘れてしまうと、データを復元することはできなくなります。

バックアップ タスクを管理する：

左側のパネルからバックアップ タスクを選択し、下のボタンをクリックして管理してください。

  • 今すぐバックアップ：選択したタスクのデータを今すぐバックアップします。
  • バックアップ エクスプローラ（拡大鏡アイコン）:すべてのバックアップ バージョンを表示し、含まれるファイルをコピー/復元/ダウンロードします。
  • バージョン リスト：すべてのバックアップ バージョンの作成時間と状態を表示し、特定のバックアップ バージョンを削除したり、安全性を高めるためにロックしたりします。
  • 編集:バックアップ元、バックアップ先、予約、その他の設定を変更できます。
  • 削除:バックアップタスクを削除します。
  • キャンセル:実行中のバックアップ タスクをキャンセルします。

バージョンを交代させる：

バックアップ ローテーションは、バージョン管理メカニズムです。古いものから、不要なバージョンやストレージスペースを空けることができそうなものを自動的に削除してバックアップ バージョンを管理します。この機能は [タスク設定] で有効にできます。または、バックアップ タスクを作成するときに設定できます。

  1. 左側のパネルからバックアップ タスクを選択し、[編集] または [設定]> [タスク設定] を選択します。
  2. [バックアップ ローテーションを有効にする] を選択し、ローテーション モードを選択します。
    • 古いバージョンから:バージョンの数が上限数を超えたときには、一番古いバージョンを削除します。
    • Smart Recycle:これらのポリシーに基づいてバックアップ バージョンを維持します。
      • 過去 24 時間分の時間ごとのバージョンを維持する:システムは、毎時作成した分から最初のバージョンを維持します。
      • 日時バージョンを過去 1 日～1 ヶ月分維持する:システムは、毎日作成した分から最初のバージョンを維持します。
      • 週次バージョンを 1 ヶ月分維持する:システムは、毎週作成した分から最初のバージョンを維持します。
  3. バージョンの最大数を指定します。バージョン数がこの数を超えると、最も古いバージョンからローテーションが開始されます。
  4. [予約] で繰り返し行うバックアップ予約を設定した場合は、バックアップ バージョンがどのように保持されるかを見たり、タイムラインで最初のリカバリー ポイントを確認したりすることができます。
  5. [OK] をクリックして、設定を保存します。

注意：

  • ロックしたバージョンは絶対に削除できません。
  • マルチ バージョンのバックアップ タスクにのみバージョン ローテーションを適用できます。
  • バージョン ローテーションは、DSM 5.1 以下のバージョンではサポートされていません。
  • ローテーションは、設定が変更されるかバックアップ タスクが完了するまでは実行されません。
  • 最大バージョン数を計算する際は、ロックしたバージョンは除外されます。

データ バックアップ タスクに関する注意

各タイプのバックアップ タスクについては、重要なセットアップの詳細と問題発生時の対処法が記載されていますので、次の注意書きをよくお読みください。

ローカル バックアップタスク:

  • バックアップを実行しているときにバックアップ先のデータを削除すると、次回タスクを実行するときに完全バックアップが行われます。
  • 新しい外部ドライブをバックアップ先として使用するには、先に初期化しておく必要があります。
  • バックアップ タスクを作成する前に、外部ドライブが正しく DiskStation に接続されているか確認してください。[コントロール パネル] > [外部デバイス] を選択し、USB/eSATA ドライブの状態を確認してください。外部ドライブが初期化、フォーマット、故障していると、外部ドライブをバックアップ先として選択することはできません。
  • リモートでマウントした共有フォルダをバックアップ元またはバックアップ先として選択することはできません。
  • 暗号化された共有フォルダがマウントされていない場合は、バックアップ先として選択できません。
  • バックアップ先のファイル システムが NTFS/FAT の場合は、画像サムネイルをバックアップすることはできません。

リモート rsync バックアップ タスク:

  • ネットワーク バックアップ先として、rsync 対応サーバーにバックアップする場合は、次のモードを選択できます。「バックアップ モジュール」の横にあるドロップダウン メニューからバックアップ モジュール（rsync 対応サーバー上の共有フォルダ）を選択するときに使用する「rsync daemon モード」、「バックアップ モジュール」に絶対パス（[/] で始まります）を入力する時に有効になる「remote shell モード」があります。
  • Linux のユーザーの方は、バックアップに 「admin」、「root」 および 「rsync」 アカウントのみを実行することができます。
  • ホスト名を使ってバックアップ先サーバーにインターネットから接続する場合は、ホスト名は、「abc.bcd.com」のように FQDN（完全に記述したドメイン名）形式である必要があります。
  • Synology NAS デバイスから別のデバイスにバックアップする場合は、バックアップ先でアカウントのユーザー名とパスワードを入力するよう求められます。共有フォルダを別の DiskStation にバックアップするには、フォルダのアクセス権の設定に応じてそれを行うことができます。特定のユーザーアカウントがアクセス権を持つ共有フォルダだけが表示されます。バックアップ先の共有フォルダがない場合は、アクセス権を確認してください。
  • 暗号化転送は、バックアップ先サーバーが暗号化ネットワークバックアップに対応し、認証パスワードが正しく入力されている場合に限り有効です。
  • Remote shell（リモートシェル）モードは暗号化転送にのみ対応しています。
  • クライアントを介してファイルを Synology NAS デバイスにバックアップする場合は、ファイル名が UTF-8 エンコード化されていなければなりません。
  • rsync 対応クライアントを介して DiskStation にファイルをバックアップする場合は、DiskStation の admin、root、および rsync アカウントでしか実行できません。
  • DiskStation から rsync 対応サーバーへファイルをバックアップする場合は、正しい rsync バックアップ アカウント名とパスワードを入力する必要があります。また書き込み権を持って正しいバックアップ モジュールを選択する必要もあります。

Amazon S3 バックアップタスク:

  • ネットワーク バックアップ先として Amazon S3 サーバーにバックアップすると、転送暗号化、Amazon S3 RRS (Reduced Redundancy Storage) へのバックアップなど、各種オプションを設定できます。転送の暗号化など、一部のオプションを選択した場合は、バックアップ速度が遅くなることがあります。
  • バックアップ ファイルは Amazon S3 サーバーで選択した Bucket に保管され、Set KEY の値が選択したディレクトリ名で始まり、ファイルの完全なファイルパスで終わります。
  • Amazon S3 のデータ構造は、Bucket とフォルダ構造の無い KEY のペアで構成します。空のフォルダをバックアップすることはできません。
  • バックアップしたファイルに変更がない場合は、Amazon S3 サーバーの同じ Bucket には再度バックアップされません。
  • Amazon S3 の各 Bucket には固有の名前が付けられます。また、名前付けには次のような規則があります。
    • 3 から 63 文字以内でなければなりません。
    • 小文字、数字、ピリオド(.)、ダッシュ(-)のみ使用できます。
    • 数字または文字で始まらなければなりません。また、ダッシュ (-) で終了したり、2 つ続けてピリオドを使用したり、ピリオドとダッシュを続けて使用したりすることはできません（[my-.bucket.com] はできませんが、[my.-bucket] は可能です）。
  • Amazon S3 KEY の最大長は 1024 文字です。Amazon S3 へのバックアップを成功させるためには、最大パス長を 1024 文字から「タスク作成時に指定したバックアップ先フォルダ名」の長さを引いた長さにしてください。
  • Amazon S3 RRS（Reduced Redundancy Storage）サポートが有効になっている場合は、オプションを有効にした後で変更したデータだけが RRS ストレージにバックアップされます。つまり、Amazon S3 の標準ストレージにすでにバックアップされたデータは、RRS にはバックアップされません。
  • バックアップしたファイルを Amazon S3 から Amazon Glacier に移動させることはできません。AWS Management コンソールを介してファイルを Amazon S3 から Amazon Glacier に移動させると、関連するバックアップ タスクが失敗します。
  • SSL で仮想的にホストされたバケツにファイルをバックアップすることはできません。
  • ファイルを転送する場合、ファイルはパーツに分割されます。パーツのサイズ制限は、[Multipart Upload パートサイズ（MB）] オプションで設定してください。デフォルト値は 512MB です。ネットワーク環境が安定しない場合は、このオプションを小さい数値に設定するようお薦めします。
  • アップロードあたりの最大パーツ数は 10000 台です。ファイルの最大サイズは [Multipart upload パートサイズ] ⅹ 10000 です。
  • ファイル名に非 UTF-8 またはコントロール文字を使用することはできません。

Microsoft Azure バックアップタスク:

  • Microsoft Azure サーバーをネットワーク バックアップ先としてバックアップ タスクを作成する場合、さまざまなオプションがあります（転送暗号化など）。転送の暗号化など、一部のオプションを選択した場合は、バックアップ速度が遅くなることがあります。
  • バックアップ ファイルは Microsoft Azure サーバーで選択した Container に保管され、Blob 名が選択したディレクトリ名で始まり、ファイルの完全なファイルパスで終わります。
  • Microsoft Azure のデータ構造は、フォルダ構造のない Container と Blob のペアで構成します。空のフォルダをバックアップすることはできません。
  • バックアップしたファイルに変更がない場合は、Microsoft Azure サーバーの同じ Container には再度バックアップされません。
  • Microsoft Azure の各 Container には固有の名前が付けられます。また、名前付けには次のような規則があります。
    • 3 から 63 文字以内でなければなりません。
    • 小文字、数字、ダッシュ (-) を使用できます。
    • 先頭と末尾の文字が数字か文字であり、ダッシュ（-）を続けて使用することはできません。
  • Microsoft Azure Blob 名の最大長は、1024 文字です。Microsoft Azure へのバックアップを成功させるためには、最大パス長を 1024 文字から「タスク作成時に指定したバックアップ先フォルダ名」の長さを引いた長さにしてください。
  • ファイル名に非 UTF-8 またはコントロール文字を使用することはできません。
  • 認証情報が正確でない、ネットワーク接続に問題がある、または DiskStation とリモート サーバーの時間が一致しない場合は、認証失敗警告メッセージが表示されます。DiskStation で時間の設定を確認するには、[コントロール パネル] > [地域オプション] > [時間] を選択して NTP サーバーで時間を合わせてください。

hicloud S3 バックアップタスク:

  • hicloud S3 サーバーをネットワーク バックアップ先としてバックアップ タスクを作成する場合、さまざまなオプションがあります（転送暗号化など）。.転送の暗号化など、一部のオプションを選択した場合は、バックアップ速度が遅くなることがあります。
  • バックアップ ファイルは hicloud S3 サーバーで選択した Bucket に保管され、Set KEY の値が選択したディレクトリ名で始まり、ファイルの完全なファイルパスで終わります。
  • hicloud S3 のデータ構造は、フォルダ構造の無い Bucket と KEY のペアで構成します。空のフォルダをバックアップすることはできません。
  • バックアップしたファイルに変更がない場合は、hicloud S3 サーバーの同じ Bucket には再度バックアップされません。
  • hicloud S3 の各 Bucket には固有の名前が付けられます。また、名前付けには次のような規則があります。
    • 3 から 63 文字以内でなければなりません。
    • 小文字、数字、ピリオド(.)、ダッシュ(-)のみ使用できます。
    • 数字または文字で始まらなければなりません。また、ダッシュ (-) で終了したり、2 つ続けてピリオドを使用したり、ピリオドとダッシュを続けて使用したりすることはできません（[my-.bucket.com] はできませんが、[my.-bucket] は可能です）。
  • hicloud S3 KEY の最大長は 255 文字です。hicloud S3 へのバックアップを成功させるためには、最大パス長を 255 文字から「タスク作成時に指定したバックアップ先フォルダ名」の長さを引いた長さにしてください。
  • ファイルを転送する場合、ファイルはパーツに分割されます。パーツのサイズ制限は、[Multipart Upload パートサイズ（MB）] オプションで設定してください。デフォルト値は 512MB です。ネットワーク環境が安定しない場合は、このオプションを小さい数値に設定するようお薦めします。
  • Hicloud は1回の転送につき、最大 1000 個のパーツしかサポートしていません。ですから、ファイルサイズの合計が 1000 x パーツサイズを超えないようにしてください。たとえば、[Multipart Upload パートサイズ] を 8MB に設定した場合、ファイルサイズが 8000MB を超える場合はそのファイルはアップロードされません。
  • 暗号化された共有フォルダをバックアップすることはできません。
  • ファイル名に非 UTF-8 またはコントロール文字を使用することはできません。

SFR NAS BACKUP バックアップ タスク:

  • SFR NAS BACKUP サーバーをネットワーク バックアップ先としてバックアップ タスクを作成する場合、さまざまなオプションがあります（転送暗号化など）。転送の暗号化など、一部のオプションを選択した場合は、バックアップ速度が遅くなることがあります。
  • バックアップ ファイルは SFR NAS BACKUP サーバーで選択した Bucket に保管され、Set KEY の値が選択したディレクトリ名で始まり、ファイルの完全なファイルパスで終わります。
  • SFR NAS BACKUP のデータ構造は、Bucket と KEY がペアとなるフォルダ構造で構成します。空のフォルダをバックアップすることはできません。
  • バックアップしたファイルに変更がない場合は、SFR NAS BACKUP サーバーの同じ Bucket には再度バックアップされません。
  • SFR NAS BACKUP の各 Bucket には固有の名前が付けられます。また、名前付けには次のような規則があります。
    • 3 から 63 文字以内でなければなりません。
    • 小文字、数字、ピリオド(.)、ダッシュ(-)のみ使用できます。
    • 数字または文字で始まらなければなりません。また、ダッシュ (-) で終了したり、2 つ続けてピリオドを使用したり、ピリオドとダッシュを続けて使用したりすることはできません（[my-.bucket.com] はできませんが、[my.-bucket] は可能です）。
  • SFR NAS BACKUP KEY の最大長は 1024 文字です。SFR NAS BACKUP へのバックアップを成功させるためには、最大パス長を 1024 文字から「タスク作成時に指定したバックアップ先フォルダ名」の長さを引いた長さにしてください。
  • ファイルを転送する場合、ファイルはパーツに分割されます。パーツのサイズ制限は、[Multipart Upload パートサイズ（MB）] オプションで設定してください。デフォルト値は 64MB です。ネットワーク環境が安定しない場合は、このオプションを小さい数値に設定するようお薦めします。
  • ファイル名に非 UTF-8 またはコントロール文字を使用することはできません。

その他の S3 対応バックアップ タスク:

  • 別の S3 対応クラウドを使用するには、その S3 対応クラウドの IP アドレス/ドメインを [S3 サーバー] 欄に入力してください。
  • 使用に問題がある場合は、関連するドキュメントをお読みになるか、ご使用の S3 対応クラウドのサービス プロバイダーにお問い合わせください。

LUN バックアップ*

[ローカル LUN バックアップ] を使用して DiskStation の iSCSI LUN をローカル共有フォルダにバックアップすることができます。また、「ネットワーク LUN バックアップ（Synology サーバー）」を使用すると、別の DiskStation にバックアップすることもできます。

iSCSI LUN をバックアップする：

  1. 左下にある + をクリックして、[LUN バックアップ タスク] を選択します。
  2. バックアップ先のタイプを選択するよう要求されたら、次のいずれかを選択してください。
    • ローカル LUN バックアップ:DiskStation からローカル共有フォルダ、または DiskStation の USB または eSATA ポートに接続された外部ストレージに LUN をバックアップします。
    • ネットワーク LUN バックアップ（Synology サーバー）:DiskStation から別の DiskStation に LUN をバックアップします。
  3. 要求されたらバックアップ先とソース LUN を指定して、バックアップ予定を設定します。
  4. ウィザードに従って設定を終了します。

バックアップ タスクが作成されたら、予定に基づいて DiskStation がバックアップを行い、バックアップ LUN をバックアップ先に保存します。バックアップ LUN はバックアップだけを目的として使用されます。バックアップ LUN にアクセスするには、「バックアップ LUN を復元する」を参照してください。

注意：

  • 通常ファイルとブロックレベル LUN の両方が、ファイル イメージとしてバックアップ先に保存されます。復元した場合は、ブロックレベル LUN はすべてファイル LUN に変更されます。
  • 各バックアップ タスクにはバックアップ ソースとして 1 つの LUN しか含まれません。
  • FAT、FAT32、HFS、HFS Plus、NTFS フォーマットで LUN を外部ディスクにバックアップすることはできません。
  • ネットワーク LUN バックアップを実行する前に、[コントロール パネル] > [バックアップ サービス] を選択してネットワーク バックアップ サービスを有効にして、バックアップ先の DiskStation にログインしてください。
  • DiskStation がルーターを介してネットワークに接続されている場合は、ネットワーク LUN バックアップを実行する前にルーター で iSCSI サービス（TCP ポート 3260）にポート転送規則が指定されていることを確認してください。

完全バックアップと増分バックアップについて

DiskStation は 1 回目は選択した LUN の完全バックアップを行い、次回からは増分バックアップを行います。ただし共有フォルダの名前やバックアップ先のディレクトリ名を変更した場合は、DiskStation は再び完全バックアップを行います。

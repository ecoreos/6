リモート接続

File Station では、さまざまなプロトコル（FTP、STFP、WebDAV、WebDAV HTTPS）を介してパブリック クラウド サービス（Box、Dropbox、Google Drive、Microsoft OneDrive）とファイル サーバーに接続することができます。

クラウド サービスに接続する：

  1. [ツール] > [リモート接続] > [接続設定] を選択します。
  2. アイコンをクリックしてクラウド サービスを選択し、[次へ] をクリックします。
  3. ポップアップ画面でウィザードの指示に従ってアカウントの認証を行ってください。
  4. 設定が完了したら、ターゲットのクラウド サービスが左パネルに表示されます。

クラウド特有の制約：

  • Box:
    • 他の協働者と共有しているカタログには、所有者のユーザー名が接尾辞として付けられます。
    • 他の協働者と共有しているカタログが同じ名前（所有者のユーザー名が接尾辞として使われたカタログ名）を使用している場合、カタログ等が1つだけ表示されます。
    • 次の拡張子のファイルは、Boxを介してのみ開くことができます。.gdoc、.gsheet、.boxnote.
    • ファイル名は大文字と小文字を区別します。
    • ファイアウォール設定を適切に行うには、この説明をお読みください。
  • Dropbox:
    • Dropbox に25000 個以上のファイルがある場合は、カタログは表示できません。
    • Dropbox に10000 個以上のファイルがある場合は、カタログの名前を変更することはできません。
    • 10 GB を超えるファイルはアップロードできません。
    • ターゲットのファイルがアップロードに24時間以上かかる場合は、アップロードできません。
    • ファイル名は大文字と小文字を区別します。
    • ファイアウォール設定を適切に行うには、この説明をお読みください。
  • Google ドライブ:
    • 2048 文字以上のパスを持つファイル/フォルダは表示されません。
    • 255 文字以上のファイル名を持つファイルは表示されません。
    • 同じ名前を持つファイル/カタログは、それぞれの作成時間で区別されます。
    • File Station で Google 専用ファイル（Google Docs や Google Slides）などを開くと、ターゲットのファイルにアクセスできるように URL が表示されます。
    • File Station で表示される[マイドライブ] と [評価済み] フォルダは、Google ドライブの [マイドライブ] と [評価済み] フォルダに相当します。
    • ファイアウォール設定を適切に行うには、この説明をお読みください。
  • Microsoft OneDrive:
    • OneNote メモ帳ファイルは表示されません。
    • 10 GB を超えるファイルはアップロードできません。
    • ファイル（例：.png、.jpg、.docx、.pdf）は、大きいファイル サイズで表示される場合があります（本来のサイズとサムネイル サイズ）。このこと自体は、元のファイルがダウンロードされても、そのサイズには影響しません。
    • ファイル名は大文字と小文字を区別します。

その他の制約については、当該クラウド サービスの公式ウェブサイトをご覧ください。

ファイル サーバーに接続する：

  1. [ツール] > [リモート接続] > [接続設定] を選択します。
  2. アイコンをクリックしてサーバー プロトコルを選択し、[次へ] をクリックします。
  3. 次の情報を提供してセットアップを完了してください。
    • ホスト名またはIP：ターゲットのファイル サーバーのホスト名または IP を入力します。
    • ポート：ターゲットのファイル サーバーのポート番号を入力します。
    • ユーザー と パスワード:ターゲットのファイル サーバーがアカウント/パスワードを必要とする場合は、このログイン情報を入力してください。
    • 文字コード：ファイル名を正しく表示するために、ターゲットのファイル サーバーの文字エンコードを選択します。
    • プロファイル名：このファイル サーバーを簡単に識別できるように、独自の名前を入力します。
    • WebDAV HTTPS 接続を使う:WebDAV プロトコルを設定するときと、クライアントとファイル サーバーの両方が WebDAV HTTPS に対応している場合に限り、このオプションを使用できます。
    • 1ユーザーにつき1接続のみ許可：FTP または SFTP プロトコルを設定する場合に限り、このオプションを使用できます。このオプションを有効にすると、サーバーに接続数について制約があるときに、接続を最適化します。
  4. [適用] をクリックして、設定を保存します。ターゲットのファイル サーバーは左パネルに表示されます。

既存のクラウド サービスまたはファイル サーバーのプロファイルを変更する：

  1. [ツール] > [リモート接続] > [接続リスト] を選択します。
  2. 次の管理を行うには、ターゲットのクラウド サービスをクリックするか、ファイル サーバーをクリックしてください。
    • 編集：[編集] をクリックして、現在のプロファイル構成を変更した後で [OK] をクリックします。
    • 接続解除：[接続解除] をクリックすると、接続が終了します。
    • 再接続：.［接続］ をクリックすると、再度接続します。(または、File Station の左パネルでターゲットをクリックすると再接続します)
    • 削除：[削除]をクリックして、プロファイルの破棄を確定してください。(または、File Station の左パネルで [削除] をクリックします)

注意：

  • ユーザーは、1人につき100件以上のクラウドサービスとファイルサーバーに同時接続することはできません。
  • 不測のエラーが発生しないように、必ず正しいサーバー コードページを選択してください。サーバーのコードページが分からない場合は、Unicode をお薦めします。
  • クラウド サービスまたはファイル サーバーに保存されたファイルでは、一部の File Station の機能（例：圧縮先、解凍、プレビュー）を使用できない場合があります。
  • 接続したクラウド サービスまたはファイル サーバーのそれぞれの転送速度は、ユーザーの速度制限設定によります。
  • 各ユーザーは、すべてのクラウド サービスおよびファイル サーバーの間で同時に3件のファイル転送しか実行できません。
  • WebDAV、WebDAV HTTPS、クラウド サービス プロキシの設定を行うには、[コントロールパネル] > [ネットワーク] > [全般] > [プロキシ] を選択します。クラウド サービスまたはファイル サーバーへの接続中にプロキシ設定を変更した場合は、一旦接続を解除してから再度接続してください。
  • 接続したクラウド サービスやファイル サーバーの情報を変更しても、接続状態には影響しません。
  • 接続リソースをリリースするには、含まれるファイルが30分間変更されずにいると、クラウド サービス/ファイル サーバーが自動的に接続解除されます。接続を再開するには、クラウド サービス/ファイル サーバーをクリックします。
  • 次の名前付けフォーマットが適用されたファイル/フォルダは表示できません。
    • ._ または .SYNOPPSDB で始まる
    • / を含む
    • という名前    ..    '    "    .DS_Store    @eaDir    @sharebin    @tmp    .SynologyWorkingDirectory
  • リモート接続は、高権限モードでのみサポートされています。この機能を使用するには、ホスト DSM で Docker パッケージを選択し、暗号化した共有フォルダを有効にする Docker DSM を選択します。次に、[編集] をクリックし、[高権限実行コンテナを使う] にチェックマークを付けます。

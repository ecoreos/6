Snapshots

Den här sidan låter dig ta, radera och redigera snapshots och även att konfigurera schema för snapshots och bevarande.

Du kan lagra upp till 256 kopior av snapshots på en gång för varje delad mapp eller iSCSI LUN. Snapshots minimerar diskförbrukning genom att bibehålla individuella block hellre än hela filer. Extra utrymme kan eventuellt förbrukas när filerna i delade mappar eller bock i iSCSI LUNs är uppdaterade eller raderade.

Delad mapp

Att ta ett snapshot för en delad mapp:

  1. Välj den delade mappen som du vill ta ett snapshot av.
  2. Klicka på Ta ett snapshot.
  3. Ange en beskrivning i Beskrivning för det snapshot som ska tas.
  4. Markera/avmarkera kryssrutan Lås för att avgöra om snapshotet ignorerar inställningarna för bevarande när automatisk borttagning är på väg att bli utlöst.
  5. Klicka på OK för att börja ta ett snapshot.

Obs:

  • Bland inställningarna på sidan Kontrollpanel > Delad mapp > Redigera blir endast behörigheterna för Windows ACL inspelade i ett snapshot.

Att bläddra bland snapshots i delade mappar:

  1. Välj en delad mapp som du vill bläddra bland snapshotsen i.
  2. Klicka på Snapshotlista.
  3. Välj ett snapshot som du vill bläddra och klicka på Bläddra.

Obs:

  • Bläddra bland snapshots är inte tillgängligt för krypterade delade mappar.
  • Om du inte kan bläddra snapshotet, vänligen gå till Kontrollpanel > Delad mapp > Redigera för att aktivera Göra snapshots synliga.

Att ta bort snapshots i delade mappar:

  1. Välj den delade mappen som du vill ta bort snapshotsen i.
  2. Klicka på Snapshotlista.
  3. Välj ett eller flera snapshots som du vill ta bort och klicka på Ta bort. (Tryck och håll nere skift eller ctrl medan du gör flera val.)
  4. Du kommer att se ett bekräftelsemeddelande. Klicka på Radera för att bekräfta borttagningen.

Att redigera snapshots i delade mappar:

  1. Välj den delade mappen som du vill redigera snapshotsen i.
  2. Klicka på Snapshotlista.
  3. Välj ett snapshot du vill redigera och klicka på Redigera.
  4. Du kan ändra beskrivningen av snapshotet i Beskrivning och markera/avmarkera kryssrutan Lås för att avgöra om snapshotet ignorerar inställningarna för bevarande när automatisk borttagning är på väg att bli utlöst.
  5. Klicka på OK för att spara dina inställningar.

Att konfigurera schemalagda snapshots:

Med schemalagda snapshots tar Snapshot & Replikering automatiskt snapshots vid den förutbestämda tiden och frekvensen och kommer därför att lätta på din arbetsbörda med att ta snapshots manuellt.

  1. Välj en delad mapp.
  2. Klicka på Lägg till schema.
  3. I fliken Schema, markera kryssrutan Aktivera schema för snapshots och ställ in tid och frekvens för att ta snapshots. För att avbryta schemalagda snapshots för den här delade mappen, vänligen avmarkera kryssrutan.
  4. Klicka på OK för att spara dina inställningar.
  5. Du kommer att se ett bekräftelsemeddelande. Klicka på OK för att bekräfta konfigurationen.

Obs:

  • Konfiguration av schemalagda snapshots inkluderas inte funktionen i Kontrollpanel > Uppdatera & Återställa > Säkerhetskopiering av konfiguration.

Att konfigurera inställningarna för bevarande i de schemalagda snapshotsen:

  1. Välj en delad mapp.
  2. Klicka på Lägg till schema.
  3. Välj fliken Bevarande.
  4. Konfigurera dina inställningar för bevarande. Välj något av följande alternativ:
    • Ta bort snapshots när gränsen är uppnådd: När det maximala antalet snapshots är uppnått blir de äldsta snapshotsen överskrivna av de som nyligen tagits.
    • Bevara alltid snapshots: När det maximala antalet snapshots har uppnåtts slutar Snapshot & Replikering ta nya snapshots utan att ta bort dem som redan tagits.
    • Bevara snapshots med följande policy: Du kan konfigurera regler för bevarande för specificerade tidsintervaller. För mer information, vänligen se noteringen om Bevarandeprincip.
  5. Klicka på OK för att spara dina inställningar.

Obs:

  • Att ändra inställningarna för bevarande påverkar motsvarande inställningar på sidan Replikering.

iSCSI LUN

Att ta ett snapshot för en iSCSI LUN:

  1. Välj den iSCSI LUN som du vill ta ett snapshot av.
  2. Klicka på Ta ett snapshot.
  3. Ange en beskrivning i Beskrivning för det snapshot som ska tas.
  4. Från rullgardinslistan Lås, välj Ja/Nej för att avgöra om snapshotet ignorerar inställningarna för bevarande när automatisk borttagning är på väg att bli utlöst.
  5. Klicka på OK för att börja ta ett snapshot.

Att ta bort ett snapshot för en iSCSI LUN:

  1. Välj den iSCSI LUN som du vill ta bort snapshotet för.
  2. Klicka på Snapshotlista.
  3. Välj ett eller flera snapshots som du vill ta bort och klicka på Ta bort. (Tryck och håll nere skift eller ctrl medan du gör flera val.)
  4. Du kommer att se ett bekräftelsemeddelande. Klicka på Ja för att bekräfta borttagningen.

Att redigera ett snapshot för en iSCSI LUN:

  1. Välj den iSCSI LUN som du vill redigera snapshotet för.
  2. Klicka på Snapshotlista.
  3. Välj ett snapshot du vill redigera och klicka på Redigera.
  4. Du kan göra något av följande för att redigera snapshotet:
    • Ändra beskrivningen för snapshotet i Beskrivning.
    • Välj Konsekvent tillstånd om applikationskonsekventa snapshots är aktiverat.
    • Växla mellan Ja/Nej i rullgardinslistan Lås för att avgöra om snapshotet ignorerar inställningarna för bevarande när automatisk borttagning är på väg att bli utlöst.
  5. Klicka på OK för att spara dina inställningar.

Att konfigurera schemalagda snapshots:

Med schemalagda snapshots tar Snapshot & Replikering automatiskt snapshots vid den förutbestämda tiden och frekvensen och kommer därför att lätta på din arbetsbörda med att ta snapshots manuellt.

  1. Välj en iSCSI LUN.
  2. Klicka på Lägg till schema.
  3. I fliken Schema, markera kryssrutan Aktivera schema för snapshots och ställ in tid och frekvens för att ta snapshots. För att avbryta schemalagda snapshots för den här delade mappen, vänligen avmarkera kryssrutan.
  4. Klicka på OK för att spara dina inställningar.

Att konfigurera inställningarna för bevarande i de schemalagda snapshotsen:

  1. Välj en iSCSI LUN.
  2. Klicka på Lägg till schema.
  3. Välj fliken Bevarande.
  4. Konfigurera dina inställningar för bevarande. Välj något av följande alternativ:
    • Ta bort snapshots när gränsen är uppnådd: När det maximala antalet snapshots är uppnått blir de äldsta snapshotsen överskrivna av de som nyligen tagits.
    • Bevara alltid snapshots: När det maximala antalet snapshots har uppnåtts slutar Snapshot & Replikering ta nya snapshots utan att ta bort dem som redan tagits.
    • Bevara snapshots med följande policy: Du kan konfigurera regler för bevarande för specificerade tidsintervaller. För mer information, vänligen se noteringen om Bevarandeprincip.
  5. Klicka på OK för att spara dina inställningar.
  6. Du kommer att se ett bekräftelsemeddelande. Klicka på OK igen för att bekräfta konfigurationen.

Att konfigurera konsekvent tillstånd för dina iSCSI LUN-snapshots:

  1. Välj en iSCSI LUN.
  2. Klicka på Lägg till schema.
  3. Välj fliken Applikation.
  4. Markera/avmarkera kryssrutan Aktivera applikationskonsekventa snapshots för att växla mellan följande typer av konsekventa tillstånd:
    • Applikationskonsekvent: Snapshots i det här konsekventa tillståndet kan bara tas när Synology Snapshot Manager är installerad på din Windows-server eller VMware vCenter Server, beroende på din driftsmiljö för att kommunicera med DSM. När ett applikationskonsekvent snapshot utlöses på DSM kommer applikationerna att meddelas och rensa alla data från minnet på de LUNs för vilka ett snapshot ska tas, på så sätt garanteras dataöverensstämmelse.

      Obs:

      • Om Synology Snapshot Manager inte har installerats på din driftsmiljö förblir applikationskonsekventa snapshots inaktiverade även om kryssrutan är markerad.
    • Kraschkonsekvent: Om Synology Snapshot Manager inte är installerad på någon av dina klienter kan du endast fånga kraschkonsekventa snapshots av LUNs. När kraschkonsekventa snapshots utlöses kommer applikationerna inte att meddelas om att rensa data från minnet till LUNs och följaktligen finns inte någon möjlighet att garantera dataöverensstämmelse.
  5. Klicka på OK för att spara dina inställningar.

Obs:

  • Att ändra inställningarna för konsekvens påverkar motsvarande inställningar på sidan Replikering.

Bevarandeprincip

Det här alternativet tillhandahåller en idealisk lösning för att hjälpa dig att specificera det maximala antalet versioner av snapshots att spara till ditt volymutrymme men du kan behöva bevara dina snapshots för en längre tidsperiod.

Synology använder bevarandeprincipen GFS dvs. Grandfather-Father-Son. Du kan konfigurera det maximala antalet versioner av snapshots som ska bevaras för följande tidsintervaller respektive: varje timme, varje dag, varje vecka, varje månad och varje år.

Snapshot & Replikering bevarar så många som det konfigurerade maximala antalet snapshots för varje tidsintervall. Om fler än en version av snapshotet finns inom ett tidsintervall behålls endast det tidigaste. Till exempel, om du ställer in en policy för 10 veckovisa snapshots bevarar Snapshot & Replikering tidigaste snapshotsen (om fler än ett snapshot tas på en vecka) för varje av de senaste 10 veckorna.

Dessutom behålls alla versioner av snapshots inom en timme efter det senaste snapshotet som standard, för din bekvämlighet att hitta och återställa från tidigare versioner av snapshots. Du kan också låsa ett snapshot för att förhindra att den automatiskt tas bort av regler för bevarande.

Replikering

Den här sidan gör att systemadministrators kan hantera replikeringar. Du kan skapa, redigera och radera replikeringsuppgifter. Du kan även köra/stoppa synkningen och visa informationen för dina replikeringsuppgifter.

Delad mapp

Att skapa en replikeringsuppgift för en delad mapp:

  1. Gå till fliken Delad mapp.
  2. Klicka på Skapa.
  3. Guiden Skapa replikeringsuppgift visas. Följ stegen i guiden för att skapa en replikering av delad mapp.

Obs:

  • Den delade mappen ”homes” kan inte replikeras.
  • Krypterade delade mappar kan inte replikeras.
  • Den delade mappen måste finnas på en Btrfs-volym.
  • Du kan skapa en-till-många-/utökade replikeringar för en delad mapp till olika målservrar. Du kan dock endast skapa en replikering till en målserver för varje delad mapp.

Att radera en replikering för en delad mapp:

  1. Välj den replikeringsuppgift för delad mapp som du vill radera.
  2. Klicka på Radera.
  3. Du kommer att se ett bekräftelsemeddelande. Klicka på Radera för att bekräfta raderingen.

Att redigera en replikering för en delad mapp:

  1. Välj den replikeringsuppgift för delad mapp som du vill redigera.
  2. Klicka på Redigera.
  3. Det visas ett fönster med följande flikar:
    • Partnerserver
    • Schema
    • Bevarande
  4. På fliken Partnerserver kan du redigera informationen om partnerservern, inklusive värdnamnet, porten, användarnamnet och lösenordet.
  5. I fliken Schema, markera kryssrutan Aktivera schema för snapshots och ställ in tid och frekvens för att ta snapshots. Du kan avmarkera kryssrutan för att avbryta de schemalagda snapshotsen för den delade mappen.
  6. På fliken Bevarande, välj något av följande alternativ:
    • Bibehåll de 256 senaste snapshoten: När det maximala antalet snapshots (256) är uppnått blir de äldsta snapshotsen överskrivna av de som nyligen tagits.
    • Bevara alltid snapshots: När det maximala antalet snapshots har uppnåtts slutar Snapshot & Replikering ta nya snapshots utan att ta bort dem som redan tagits.
    • Bevara snapshots med följande policy: Du kan konfigurera regler för bevarande för specificerade tidsintervaller.
  7. Klicka på OK för att använda de nya inställningarna.

Obs:

  • Du kan bara redigera schemat för replikeringsuppgifter på källservern.
  • Att ändra inställningarna för bevarande påverkar motsvarande inställningar på sidan Snapshot.

Att visa information för replikering av en delad mapp:

  1. Välj den replikeringsuppgift för delad mapp som du vill visa.
  2. Klicka på Info.
  3. Det visas ett fönster med följande information:
    • Status för den här replikeringen.
    • Statistik för den här replikeringen.

Att köra synkningen för replikering av en delad mapp:

  1. Välj replikering för delad mapp som du vill synka.
  2. Klicka på Kör.
  3. Ange en beskrivning för det snapshot som ska tas.
  4. Klicka på OK för att bekräfta åtgärden.

Att stoppa synkningen för replikering av en delad mapp:

  1. Välj replikering för delad mapp som du vill sluta synka.
  2. Klicka på Stopp.
  3. Du kommer att se ett bekräftelsemeddelande. Klicka på Stopp för att bekräfta åtgärden.

En-till-många-/utökad replikering

Du kan skapa flera replikeringar för en delad mapp/iSCSI LUN till flera målservrar vilket utgör ett en-till-många-schema. Du kan även skapa en utökad replikering för en replikerad delad mapp/iSCSI LUN som utgör ett utökat schema.

Obs:

  • Oavsett om det är ett ett-till-många-schema eller utökat schema kan varje replikeringsguppgift som mest ha tre målservrar.

Bevarandeprincip

Det här alternativet tillhandahåller en idealisk lösning för att hjälpa dig att specificera det maximala antalet versioner av snapshots att spara till ditt volymutrymme men du kan behöva bevara dina snapshots för en längre tidsperiod.

Synology använder bevarandeprincipen GFS dvs. Grandfather-Father-Son. Du kan konfigurera det maximala antalet versioner av snapshots som ska bevaras för följande tidsintervaller respektive: varje timme, varje dag, varje vecka, varje månad och varje år.

Dataskydd-hanterare bevarar så många som det konfigurerade maximala antalet snapshots för varje tidsintervall. Om fler än en version av snapshotet finns inom ett tidsintervall behålls endast det tidigaste. Till exempel, om du ställer in en policy för 10 veckovisa snapshot(s) behåller Dataskydd-hanterare tidigaste snapshotet (om fler än ett snapshot tas på en vecka) för varje av de senaste 10 veckorna.

Dessutom behålls alla versioner av snapshots inom en timme efter det senaste snapshotet som standard, för din bekvämlighet att hitta och återställa från tidigare versioner av snapshots. Du kan också låsa ett snapshot för att förhindra att den automatiskt tas bort av regler för bevarande.

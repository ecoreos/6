Återställning

Den här sidan gör att systemadministrators kan utföra återställning. För att möjliggöra för användare att få åtkomst till snapshots för delade mappar, vänligen aktivera Göra snapshots synliga för varje delad mapp så att användare kan få åtkomst till snapshoten via File Station eller Utforskaren.

Delad mapp

Att återställa individuella filer/mappar i en delad mapp:

  1. Välj den delade mapp som du vill återställa.
  2. Klicka på Återställ.
  3. Välj en version av snapshot som du vill återställa från.
  4. Klicka på Bläddra och kopiera filerna/mapparna som du vill återställa till målet.

Att återställa en delad mapp:

  1. Välj den delade mapp som du vill återställa.
  2. Klicka på Återställ.
  3. Välj en version av snapshot som du vill återställa från.
  4. Klicka på Åtgärd och välj Återställ på plats från rullgardinslistan.
  5. Du kommer att se ett bekräftelsemeddelande. Klicka på Ja för att bekräfta åtgärden.

För att ta ett snapshot till en ny delad mapp:

  1. Välj den delade mapp som du vill klona.
  2. Klicka på Återställ.
  3. Välj en version av snapshot som du vill klona till en ny delad mapp.
  4. Klicka på Åtgärd och välj Klona med ett nytt namn från rullgardinslistan.
  5. Du blir omdirigerad till sidan Klon i Kontrollpanel > Delad mapp > Skapa. Ange namnet på den klonade delade mappen och klicka på OK för att slutföra.

Obs:

  • Den delade mappen ”homes” kan inte återställas.
  • Den klonade delade mappen kan inte krypteras.
  • Om ett snapshot tas när överföring av några filer fortfarande pågår garanteras inte dataöverensstämmelse.
  • Begränsningar på krypterade delade mappar:
    • Ingen typ av återställning är tillgänglig för monterade krypterade delade mappar.
    • Klona med ett nytt namn är inte tillgängligt för krypterade delade mappar.
    • Omonterade krypterade delade mappar kan återställas från tidigare tagna snapshots genom att använda Återställ på plats.

Att utföra övergång för replikeringen av en delad mapp:

  1. Välj den delade mapp som du vill utföra övergång för.
  2. Klicka på Övergång.
  3. Välj den nya målservern om det finns flera replikeringsuppgifter för den delade mappen.
  4. Klicka på Övergång för att starta åtgärden.

Att utföra redundans för replikeringen av en delad mapp:

  1. Välj den delade mapp som du vill utföra redundans för.
  2. Klicka på Redundans > Redundans.
  3. Välj den version av snapshot som du vill återställa.
  4. Klicka på Redundans för att starta åtgärden.

Obs:

  • Du kan bara initiera en redundans om det är fel på källservern.

Att återskydda replikeringen av en delad mapp:

När redundans utförts för replikeringsuppgiften på målservern slutar replikeringsuppgifter synka data mellan de två servrarna. Du måste återskydda replikeringsuppgiften efter att den ursprungliga källservern är återställd.

  1. Välj den delade mapp som det utförts redundans för som du vill återskydda.
  2. Klicka på Redundans > Återskydda.
  3. Välj den nya källservern.
  4. Välj den nya målservern om det finns flera replikeringsuppgifter för den delade mappen.
  5. Klicka på Återskydda för att starta åtgärden.

Obs:

  • Data för den replikerade delade mappen som finns på den nya målservern skrivs över med den på den nya källservern.
  • Om den valda versionen av snapshot för redundans inte finns på partnerservern kan det ta längre tid att återskydda.

Att utföra en testredundans för en replikerad delad mapp:

  1. Välj den replikerade delade mapp som du vill testa.
  2. Klicka på Testredundans > Testredundans.
  3. Välj Testserver om det finns flera replikeringsuppgifter för den delade mappen.
  4. Ange Namn på testredundans för den delade mappen och välj en version av snapshot att testa.
  5. Klicka på Testredundans för att starta åtgärden.

Obs:

  • Att utföra övergång/redundans för en replikering rensar dess testredundans.
  • Du kan inte skapa en replikeringsuppgift för testredundans för en delad mapp.

Att rensa en testredundans för en replikerad delad mapp:

  1. Välj den replikerade delade mapp för vilken du vill rensa en testredundans.
  2. Klicka på Testredundans > Rensar testredundans.
  3. Markera den testredundans du vill rensa.
    Obs: Markera Radera testredundansdata om du inte längre vill behålla den delade mapp som skapats för testredundans.
  4. Klicka på Rensar testredundans för att starta åtgärden.

iSCSI LUN

  1. Välj den iSCSI LUN som du vill återställa.
  2. Klicka på Återställ.
  3. Välj en version av snapshot som du vill återställa från.
  4. Klicka på Åtgärd och välj Återställ på plats från rullgardinslistan.
  5. Du kommer att se ett bekräftelsemeddelande. Klicka på Ja för att bekräfta åtgärden.

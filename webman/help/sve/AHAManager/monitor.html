Övervaka

Den här sidan innehåller tre flikar som visar övervakningen för systemstatus, tjänster och nätverksgränssnitt. Denna sida låter dig övervaka specifika mål. Om det uppstår en felfunktion på ett övervakat mål på den aktiva servern, kommer systemet att utföra redundans till den passiva servern som är i beredskapsläge (förutsatt att den passiva servern fungerar normalt). Följande flikar finns på denna sida:

Fliken Systemstatus

Fliken Systemstatus övervakas som standard och kan inte avaktiveras. Redundans kommer att utlösas när systemfel inträffar på den aktiva servern. Följande avsnitt finns på denna flik:

  • Status för aktiv server: Redundans när den aktiva servern vid felfunktion inte kan tillhandahålla tjänster. Till exempel, felfunktion på båda strömenheterna för den aktiva servern.
  • Anslutningsbarhet för lagring: Redundans i en situation när en enhet är frånvarande på den aktiva servern men närvarande på den passiva servern.

Fliken Tjänster

Visar tjänster som kan övervakas och deras status. Redundans kommer att utlösas när en tjänst inte är normal. Följande avsnitt finns på denna flik:

  • Tjänst: Visar tjänster som kan övervakas.
  • Status: Visar huruvida tjänsterna körs.
  • Övervaka: Markera kryssrutan bredvid en tjänst för att börja övervaka den. Systemet utför redundans om det uppstår en felfunktion för den valda tjänsten.

Övervaka en tjänst:

  1. Lokalisera den tjänst du vill övervaka. Säkerställ att status för tjänsten är normal.
  2. Markera kryssrutan bredvid den.
  3. Klicka på Tillämpa för att spara inställningarna.

Obs:

Tjänster som kan övervakas är följande: Windows File Service, iSCSI Target, FTP, Mac File Service, NFS Service.

Fliken Nätverksgränssnitt

Visar alla logiska nätverksgränssnitt som kan övervakas. Redundans kommer att lösas ut om nätverksanslutningen inte är normal. (Se följande noteringar.) Följande avsnitt finns på denna flik:

  • Nätverksgränssnitt: Visar gränssnitten som kan övervakas.
  • Övervaka: Markera kryssrutan bredvid ett nätverksgränssnitt för att börja övervaka den.

Obs:

Om kryssrutan Övervaka är aktiverad kommer systemet utföra redundans till den passiva servern om det uppstår en felfunktion på eller om nätverksanslutningen slutar fungera för den aktiva servern och nätverksanslutningen på den passiva servern fungerar normalt.

  • Redundans kommer att utlösas endast om den passiva servern har fler anslutningar vid god hälsa. Till exempel, om den aktiva servern har tre anslutningar och en slutar fungera, kommer systemet att utlösa redundans endast om den passiva servern har tre eller fler anslutningar vid god hälsa.
  • Anslutningar som kombinerats med Link Aggregation räknas som en anslutning. Till exempel om två anslutningar är kombinerade och en av den slutar fungera, medan den andra fortfarande fungerar, kommer anslutningen fortfarande att betraktas som vid god hälsa.

Att övervaka ett nätverksgränssnitt:

  1. Lokalisera det nätverksgränssnitt du vill övervaka. Säkerställ att kablarna har blivit korrekt ansluta till nätverksgränssnitten på både den aktiva och den passiva servern.
  2. Markera kryssrutan bredvid den.
  3. Klicka på Tillämpa för att spara inställningarna.

iSCSI Target*

ISCSI är en typ av tjänst för lagringsnätverk (SAN) som erbjuder åtkomst till konsoliderad datalagring på blocknivå. Det underlättar dataöverföringar via intranät och lagringshantering på långa avstånd. Genom att mappa iSCSI LUNs till iSCSI Targets så kan klientservrar få tillgång till lagringsutrymmen via dess iSCSI-initiativtagare som om de vore lokala hårddiskar. Beroende på modell så stöder Synology-produkt skapande av flera iSCSI Targets som kan användas för att mappa iSCSI LUNs.

Obs: Antalet iSCSI targets som stöds varierar mellan dina Synology-produkter. Detaljerade produktspecifikationer hittar du på Synologys webbplats.

Skapa ett iSCSI Target:

  1. Klicka på knappen Skapa.
  2. Ange information för iSCSI Target, inklusive Namn, IQN och om CHAP-behörighetskontroll ska aktiveras.
  3. Klicka på Nästa.
  4. Ställ in iSCSI LUN-mappning. Välj Skapa ett nytt iSCSI LUN ellerMappa befintliga iSCSI LUN.
  5. Klicka på Nästa.
  6. Klicka på Bakåt för att ändra eller Tillämpa för att bekräfta inställningarna.

Förklaring av termer

  • IQN: IQN (iSCSI Qualified Name) är ett unikt namn för ett iSCSI Target. Det får inte hamna i konflikt med något IQN för ett iSCSI Target på andra servrar. IQN måste ha mindre än 128 tecken och kan innefatta bokstäver (ej skiftlägeskänsligt), siffror, kolon, streck och punkter. Föreslaget namnformat är iqn.[datum].[domän]:[enhetsidentifierare], till exempel: iqn.2000-04.com.synology:storage.diskarray.1.
  • CHAP-behörighetskontroll: Om du väljer CHAP, kommer servern att kunna verifiera klientens identitet. Användarnamnet kan utgöras av 1-12 bokstäver och siffror och lösenordet kan utgöras av 12-16 bokstäver och siffror.
  • Ömsesidig CHAP-behörighetskontroll: I Ömsesidig CHAP, kommer både klient och server att verifiera varandra. Ett CHAP-lösenord som används för klientautentisering får inte vara konfigurerat för autentisering av alla servrar, och ett CHAP-lösenord som används för serverautentisering får inte vara konfigurerat för autentisering av alla klienter.

Ta bort ett iSCSI Target:

Du kan ta bort ett iSCSI Target och alla mappade iSCSI LUN. Obs! När du tar bort ett iSCSI LUN kommer alla data på iSCSI LUN att raderas.

  1. Välj det iSCSI Target du vill ta bort.
  2. Kicka på Ta bort.
  3. Om du vill ta bort mappade iSCSI LUN väljer du Radera även mappade iSCSI LUN.
  4. Klicka på OK.

Redigera ett iSCSI Target:

  1. Välj det iSCSI Target du vill redigera.
  2. Klicka på Redigera.
  3. Redigera inställningarna enligt önskemål. Klicka på OK för att spara ändringarna.

Förklaring av termer

  • CRC-kontrollsumma: Aktivera CRC Checksum för att kontrollera om några fel uppstått under dataöverföring.
  • Flera sessioner: Tillåt flera sessioner från eller flera initiativtagare att ansluta till samma iSCSI Target på samma gång. För att skydda dina filer från skador eller oväntad dataförlust rekommenderas starkt att du använder ett system med delade filer eller klustersystem som t.ex. VMware Virtual Machine File System (VMFS) och Oracle Cluster File System (OCFS).
  • Maskning: Maskning gör det möjligt för specificerade iSCSI-initiativtagare-IQN att få exklusiv åtkomst till ett iSCSI Target.

Aktivera eller avaktivera ett iSCSI Target:

  1. Välj det iSCSI Target som du vill aktivera eller avaktivera.
  2. Klicka på Aktivera eller Avaktivera.

För att registrera iSCSI Targets med en iSNS-server

iSNS (Internet Storage Name Service) är mekanismen för centraliserad hantering. Du kan registrera alla iSCSI Targets med iSNS-server.

  1. Klicka på knappen iSNS och bocka för kryssrutanAktivera iSNS.
  2. Ange IP-adress för iSNS-servern. Klicka på OK för att tillämpa inställningen.

Resursskärm

Resursskärmen ger dig möjlighet att övervaka användningen av processorn, minnet, diskutnyttjande och nätverksflödet. Du kan välja att övervaka i realtid eller titta på tidigare data.

Resursskärm ger dig möjlighet att övervaka användningen av processorn, minnet och nätverksflödet. Du kan välja att övervaka i realtid eller titta på tidigare data.

Obs:

  • Tidigare data visas bara om Aktivera användningshistorik är markerat på fliken Avancerade inställningar.
  • För Docker DSM är CPU och minne det som visar information om användning för Docker Värd.

Prestanda

Klicka på fliken Prestanda för att visa data om resursanvändning. Detaljerad information för datum tid och diagram visas när du håller musen över.

CPU

Du kan se övergripande CPU-användning för Användare, System och I/O vänta. Processorbelastningen kan vara hög när Resursskärm först körs eftersom systemet måste hämta sina resursdata och ladda användargränssnittssidan samtidigt.

Användare visar procentsats av CPU som används för att köra användarapplikationer. Systemet visar procentsats av CPU som används för systemets kärna, hårdvaras/mjukvaras IRQ. I/O vänta visar procentsats av CPU som används för I/O vänta.

Utnyttjande är total CPU-användning för Användare och System.

Medelbelastningen inom 1, 5 och 15 minuter visas, och du kan se kör-kölängd (dvs. summan av antalet processer som för närvarande körs plus det antal som väntar på att få köra).

Förklaring av termer

  • I/O vänta: I/O vänta visar att CPU är inaktiv och inte används av något program.

    • I/O vänta är ett normalt tillstånd. Eftersom CPU:s hastighet är mycket snabbare den för I/O, så är det vanligt att CPU får vänta på att överföringen av hårddiskar eller nätverk ska slutföras. Om procentandelen för I/O är mycket stor och CPU oftast är i läget I/O vänta så kanske den inte korrekt utnyttjad. Därför kan du använda effektivare konfigurationer av hårddiskgrupper (t.ex. SHR eller RAID 5) eller nätverkskonfigurationer (t.ex. Bonding) för att uppnå bättre systemprestanda.

Minne

Den totala minnesanvändningen för fysiskt minne och swap space visas. Minnesanvändningen förblir hög eftersom systemet lagrar ofta förekommande data i cachen så att data snabbt kan erhållas utan att komma åt hårddisken. Cacheminnet används när det totala minnet inte räcker till.

Hög swap space pekar på för lite systemminne och påverkar även systemets prestanda. Du kan visa hastigheten vid swap in och swap ut genom att välja Swap från rullgardinsmenyn högst upp.

Nätverk

Nätverksflödesdiagrammet visar hastigheten, i KB per sekund, för data som skickas och mottas. Om PPPoE är aktiverat kommer dess överföringshastighet även att visas på diagrammet.

Hårddisk

Du kan välja att visa total diskanvändning, överföringshastighet eller IOPS genom att klicka på rullgardinsmenyn högst upp i diagrammet. Du kan klicka på Visa alla längst ned för att välja fler hårddiskar genom att markera kryssrutorna för att lägga till dess data till diagrammet.

Volym

Den här sidan visar den övergripande överföringsstatusen för volymer. Klicka på rullgardinsmenyn ovanför grafen för att visa överföringsutnyttjande, överföringshastighet eller IOPS. Klicka på knappen Visa alla för att välja enskilda volymer och lägg till dem i diagrammet.

iSCSI

Den här sidan visar den övergripande statusen för de iSCSI LUNs som för närvarande körs. Klicka på rullgardinsmenyn ovanför diagrammet för att välja en LUN och se dess status. Tre diagram visas som standard, de är Datakapacitet, IOPS och Fördröjning. För mer detaljerad information, klicka på knappen Välj diagram för att välja andra diagram som du vill visa.

Process

Klicka på fliken Process för att visa CPU- och minnesanvändning för varje process i sjunkande ordning baserat på CPU-belastning.

Processtatus omfattar Körs, Sover och Stoppad. Motsvarande processtatus för Linux är körs, stoppad/spårning och sover.

Sover kan följas av en bokstav med detaljerad information, t.ex. 'D' för hårddisk i viloläge, 'Z' för zombie och 'X' för död.

Anslutna användare

På denna flik kan du visa och hantera de klienter som för närvarande är anslutna till systemet och använder resurser. Detta avsnitt visar klienter som är anslutna via HTTP, TELNET, SSH, SMB/CIFS, AFP, FTP och NFS.

Gör något av följande för att hantera aktuella anslutningar:

  • Klicka på Spara för att exportera den aktuella listan till en HTML-fil.
  • Klicka på Uppdatera för att uppdatera listan.
  • Klicka på Avsluta anslutning för att förhindra användaren från att komma åt resursen. Den frånkopplade användaren måste auktorisera på nytt för att gå tillgång till resursen.
  • Klicka på Inaktivera användare för att inaktivera användaren. För att aktivera användaren igen, gå till Huvudmeny > Kontrollpanel > Användare.

Obs:

  • Alternativet Avsluta anslutning fungerar inte med NFS-klienter.
  • Om NFS-klienter inte avmonterar delade mappar ordentligt kan de fortfarande visas i listan över anslutna användare.
  • Om DiskStation startar om kommer inte NFS-klienter som tidigare var anslutna att visas i listan över ansluta användare.

Hastighetsbegränsning

Under fliken Hastighetsbegränsning kan du visa och hantera listan över filöverföringsprocesser som tillämpar hastighetsbegränsning på DiskStation.

Gör så här för att uppdatera listan:

Klicka på Uppdatera för att ladda om sidan och få de senaste posterna.

Gör så här för att stoppa en filöverföringsprocess:

Klicka på Avsluta anslutning för att stoppa filöverföringen. Du måste starta om filöverföringen för att framgångsrikt avsluta processen.

Inställningar

Klicka på Inställningar för att ställa in följande alternativ:

  • Välj Realtidslängd i minuter från rullgardinsmenyn.
  • Markera Aktivera användningshistorik om du vill se tidigare data. När det är aktiverat kan du växla mellan Realtid, 1 dag, 1 vecka, 1 Månad och 1 år i rullgardinsmenyn högst upp till höger när du tittar på prestandadiagram.

Klicka på Tillämpa för att spara inställningarna.

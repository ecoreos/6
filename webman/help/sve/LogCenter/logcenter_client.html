Loggsändning

Om du har en syslog-server kan du följa instruktionerna här nedanför för att konfigurera din DiskStation som en syslog-klient och överföra de lokala systemloggarna för din DiskStation till en syslog-server för centraliserad övervakning och hantering.

Skicka loggfiler till en syslog-server:

  1. Gå till Loggcentralen > Loggsändning.
  2. Markera kryssrutan Skicka loggfiler till en syslog-server.
  3. Ange värdnamnet eller IP-adressen för syslog-servern i Server.
  4. Specificera ett portnummer för att ta emot syslogmeddelanden i Port.
  5. Välj UDP eller TCP från Överföringsprotokoll. Det bör överensstämma med protokollet som används av syslog-servern.
  6. Välj BSD (RFC 3164) eller IETF (RFC 5424) från Loggformat för att sända loggfiler med det valda formatet.

    7. Obs:

    • Som standard är porten 514. Korrekta regler för vidarebefordring av port måste ställas in om du vill tilldela ett annat portnummer till syslog-server.
    • UDP (User Datagram Protocol) är ett enklare meddelandebaserat anslutningsprotokoll som är snabbare men saknar säkerhet och felkorrigering. TCP (Transmission Control Protocol) erbjuder garanterad leverans för vilket det använder flödeskontroll för att avgöra huruvida alla paket sänts med lyckat resultat innan nya bearbetas.
    • Säkra anslutningar (SSL) är bara tillgängligt vid användning av TCP.
    • Om du väljer TCP och vill säkra anslutningen för syslog med SSL kan du markera alternativet Aktivera säker anslutning (SSL). Klicka sedan på Importera certifikat och välj certifikat för att säkert överföra loggfiler till syslog-servern.
  7. Klicka på Skicka testlogg för att bekräfta att syslog-servern kan motta loggfilerna med lyckat resultat. Om syslog-servern inte mottar testloggen, vänligen kontrollera att dina inställningar är korrekta.
  8. Klicka på Tillämpa för att spara inställningarna.

Filtrera Loggfiler

På fliken Filter kan du specificera vilka typer av loggfiler som skickas till syslog-servern, i enlighet med kategori (dvs. system, anslutning, säkerhetskopiering) eller allvarlighetsgrad (dvs. varning, fel, information).

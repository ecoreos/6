Nätverksinställningar

Denna sida innehåller tre flikar som visar nätverksinställningar för high-availability-klustret, den aktiva medlemsservern och den passiva medlemsservern.

Fliken HA-kluster

När ett high-availability-kluster skapas genererar systemet en gemensam IP-adress för high-availability-klustrets IP för inkommande anslutningar. Ytterligare IP-adresser för high-availability-klustret kan skapas vid behov. IP-adresserna för de aktiverade nätverkskorten på den aktiva servern visas på denna flik (nätverkskortet som används av Heartbeat-anslutningen visas inte på denna sida och nämnda gränssnitt kommer att användas för datareplikering mellan de två servrarna). Denna flik visar information enligt nedanstående:

  • Nätverksgränssnitt: Aktiverat/aktiverade nätverkskort på den aktiva servern.
  • Anslutning: Visar huruvida detta gränssnitt är anslutet.
  • IP-adress: När ett high-availability-kluster skapas genererar systemet en gemensam IP-adress för high-availability-klustrets IP för inkommande anslutningar. Du kan ansluta till high-availability-klustret genom att använda denna IP-adress om du vill logga in på DiskStation Manager (DSM), oavsett vilken Synology-server som är aktiv server. Detta gör att anslutning inte behöver ske till annan IP-adress när en övergång görs.
  • Subnätmask: Den subnätmask som är associerad med aktuellt high-availability-klustrets IP-adress.

För att redigera IP-adress eller subnätmask:

  1. Välj den IP-adress eller subnätmask du vill redigera och klicka på Redigera eller dubbelklicka på fältet för att utföra redigering.
  2. Bocka för kryssrutorna Aktivera för att aktivera eller inaktivera respektive gränssnitt. Du kan aktivera mer än ett nätverkskort på klustret men nätverkskort som ursprungligen användes när klustret skapades kan inte inaktiveras.

Ändra servernamn för high-availability-klustret:

  1. Klicka på Ändra nätverksinställningar för att se namnet på high-availability-klustret och ändra dess inställningar.
  2. Klicka på Tillämpa för att spara inställningar med nuvarande namn och IP-adress för high-availability-klustret.
  3. För att kasta bort nuvarande ändringar klicka på Avbryt.

Fliken Aktiv och Passiv Server (visas som servernamn)

Dessa flikar visas status för nätverkskort på den aktiva eller passiva servern. Av dessa, använder Heartbeat-anslutningen ett nätverkskort vardera på den aktiva och den passiva servern. Dessa flikar visar följande information:

  • Nätverkskort: Aktiverat/aktiverade nätverkskort på servern.
  • IP-adress: Den IP-adress som tilldelats aktuellt nätverkskort.
  • Subnätmask: Den subnätmask som är associerad med aktuell IP-adress.
  • Beskrivning: Visar information om aktuellt nätverkskort och åtkomsten till dess tilldelade IP-adress. Detta fält kan visa följande möjliga tillstånd:
    1. Åtkomst till ursprunglig IP-adress: Den IP-adress som ursprungligen tilldelades gränssnittet på den aktiva servern är tillgänglig för data och tjänster.
    2. Ingen åtkomst till ursprunglig IP-adress: Den IP-adress som ursprungligen tilldelades gränssnittet på den aktiva servern är tillgänglig för data och tjänster.
    3. IP-anslutning för Heartbeat, inte tillgänglig från intranätet: Den ursprungliga IP-adressen som automatiskt tilldelats Heartbeat-anslutningen är inte tillgänglig för data och tjänster.

Säkerhetskopieringsmål

På fliken Säkerhetskopieringsmål i Säkerhetskopiering & Replikering kan du definiera ett säkerhetskopieringsmål lokalt, på nätverket eller i molnet, innan en säkerhetskopieringsuppgift skapas.

För att skapa ett lokalt säkerhetskopieringsmål:

Ett lokalt säkerhetskopieringsmål krävs för att säkerhetskopiera data från DiskStation till en lokalt delad mapp, volym eller en extern USB-/eSata-disk. Säkerhetskopieringsmålet måste skilja sig från källan och kapaciteten för målet måste vara större än den totala storleken av det data som ska säkerhetskopieras.

  1. Klicka på Skapa, välj Lokalt säkerhetskopieringsmål och klicka sedan på Nästa.
  2. Ge detta säkerhetskopieringsmål ett namn och välj målet där säkerhetskopierat data kommer att lagras och huruvida du vill behålla endast de senaste säkerhetskopiorna eller bevara alla versioner av säkerhetskopior.

Noteringar:

  • Om data på målet flyttas under tiden säkerhetskopieringsuppgiften utförs, kommer en fullständig säkerhetskopiering att utföras nästa gång uppgiften utförs.
  • Nya externa enheter kan behöva formateras innan de kan tas i bruk som säkerhetskopieringsmål.
  • Innan du skapar en säkerhetskopieringsuppgift, säkerställ att du redan har ordentligt har anslutit de externa diskarna till din DiskStation. Du kan gå till Kontrollpanel > Externa enheter för att kontrollera status för USB/eSATA-diskar. Externa diskar kan inte väljas som säkerhetskopieringsmål om den externa disken håller på att initialiseras, formateras eller är felaktig.
  • Fjärrmonterade utdelningar kan inte väljas som källa eller mål för säkerhetskopieringen.
  • Om en krypterad delad mapp avmonteras kan den inte väljas som mål vid en säkerhetskopiering.
  • Miniatyrbilder kan inte säkerhetskopieras när filsystemet på målet är NTFS/FAT.

För att skapa ett nätverksmål för säkerhetskopiering:

Ett nätverksmål för säkerhetskopiering krävs för säkerhetskopiering av data från DiskStation till en annan DiskStation eller rsync-kompatibel server över nätverket. Det finns flertalet alternativ tillgängliga när en säkerhetskopieringsuppgift med nätverksmål för säkerhetskopiering såsom överföringskryptering, komprimering, säkerhetskopiering på blocknivå med mera skapas. Vänligen observera att en del alternativ, såsom överföringskryptering kan minska hastigheten för säkerhetskopieringsuppgifter.

  1. Innan ett nätverksmål för säkerhetskopiering skapas, vänligen logga in på den server som är mål först, gå till Säkerhetskopiering & Replikering > Säkerhetskopieringstjänster och aktivera tjänsten nätverkssäkerhetskopiering och/eller volym som mål för nätverkssäkerhetskopiering.
  2. Tillbaka på din DiskStation, gå till Säkerhetskopiering & Replikering > Säkerhetskopieringsmål. Klicka på Skapa, välj Nätverksmål för säkerhetskopiering och följ guiden för att slutföra.
  3. Med Nätverksmål för säkerhetskopiering (DiskStation-server) kan du lagra säkerhetskopierad data i en delad mapp för att behålla den aktuella säkerhetskopian eller lagra dina säkerhetskopierade data på en volym för att behålla flera versioner av säkerhetskopior. Med Nätverksmål för säkerhetskopiering (rsync-kompatibel server) kan du bara behålla den aktuella säkerhetskopian.

Noteringar:

  • Följande lägen finns tillgängliga vid skapande av en säkerhetskopieringsuppgift med rsync-kompatibla servrar som nätverksmål för säkerhetskopiering: Läget rsync daemon som används när du väljer en modul för säkerhetskopiering (delade mappar på den rsync-kompatibla servern) från rullgardinsmenyn bredvid Säkerhetskopieringsmodul; och fjärrstyrt skalläge, vilket endast är aktiverat när du anger en absolut sökväg (startar med "/") i Säkerhetskopieringsmodul.
  • För Linux-användare är endast kontona admin, root och rsync tillåtna att utföra säkerhetskopieringsuppgifter.
  • Om du vill ha åtkomst till målservern över Internet genom att använda dess värdnamn måste värdnamnet vara en FQDN (Fully Qualified Domain Name, på svenska fullt kvalificerat domännamn), till exempel abc.bcd.com.
  • Vid säkerhetskopiering från en DiskStation-enhet till en annan blir du uppmanad att ange användarnamnet och lösenordet för ett konto på målservern. Om du vill säkerhetskopiera till en volym på en annan DiskStation behöver du särskilt specificera ett konto som hör till gruppen administrators. Vid säkerhetskopiering till delade mappar på en annan DiskStation kan du göra så i enlighet med mappens åtkomsträttigheter. Endast delade mappar till vilka den specifika användaren har åtkomstprivilegier kommer att synas. Om du inte kan hitta de önskade delade mapparna på målet, vänligen kontrollera dina åtkomstprivilegier.
  • Säkerhetskopieringskryptering har endast stöd när den server som är mål stöder krypterad nätverkssäkerhetskopiering och lösenordet för autentisering inte är tomt.
  • Fjärrstyrt skalläge stöder endast krypterad säkerhetskopiering.
  • Vid säkerhetskopiering till en rsync-kompatibel server kommer du inte längre att kunna ändra rsync-krypteringsläget efter att en säkerhetskopieringsuppgift för rsync skapats. För att ändra konfigurationen, vänligen skapa en ny rsync-uppgift.
  • Om du säkerhetskopierar filer till en DiskStation via en rsync-klient måste filnamnen vara i teckentabell UTF-8.
  • Om du säkerhetskopierar filer till en DiskStation via en rsync-kompatibel klient kan du endast utföra uppgiften med kontona admin, root och rsync på DiskStation.
  • Om du säkerhetskopierar filer till en DiskStation TILL en rsync-kompatibel server, måste du också mata in korrekt kontonamn och lösenord för rsync backup-konto. Du måste också välja rätt kopieringsmodul med skrivprivilegier.

Att skapa ett Säkerhetskopieringsmål Amazon S3:

Ett Säkerhetskopieringsmål Amazon S3 krävs för säkerhetskopiering av data från DiskStation till en Amazon S3-server.

  1. Klicka på Skapa, välj Säkerhetskopieringsmål Gemensamt moln och sedan Säkerhetskopieringsmål Amazon S3 och klicka sedan på Nästa.
  2. Ge detta säkerhetskopieringsmål ett namn, ange inloggningens autentiseringsuppgifter och fyll sedan i de obligatoriska fälten för att konfigurera en säkerhetskopieringsuppgift.

Noteringar:

  • När uppgifter för säkerhetskopiering med Amazon S3-server som nätverksmål för säkerhetskopiering skapas finns flertalet alternativ tillgängliga, såsom överföringskryptering, säkerhetskopiering till Amazon S3 RRS (Reduced Redundancy Storage) och andra. Vänligen observera att en del alternativ, såsom överföringskryptering kan minska hastigheten för säkerhetskopieringsuppgiften.
  • Säkerhetskopieringsfiler sparas i den Bucket du valde på Amazon S3-servern och Set KEY-värdet börjar med ett katalognamn som du väljer och slutar med den kompletta filsökvägen för filen.
  • Datastrukturen på Amazon S3 inkluderar ett par Bucket och KEY utan någon mappstruktur. Säkerhetskopiering av tomma mappar stöds inte.
  • Om en säkerhetskopierad fil inte har ändrats kommer den inte att säkerhetskopieras till samma Bucket på Amazon S3-server igen.
  • Varje Bucket på Amazon S3 måste ha ett unikt namn. Dessutom ska varje namn:
    • vara mellan 3 och 63 tecken långt.
    • endast innehålla gemena bokstäver, siffror, punkter (.) och streck (-).
    • börja och sluta med en siffra eller en bokstav och inte ha ett (-) i slutet, innehålla två på varandra följande punkter eller innehålla streck bredvid punkter (t.ex. är "my-.bucket.com" och "my.-bucket" ogiltiga).
  • En Amazon S3 KEY får inte överskrida 1024 tecken. För att säkerställa att det går att säkerhetskopiera till Amazon S3 är den längsta sökvägen 1024 minus namnet för målmappen som specificerades när uppgiften skapades.
  • Om du aktiverar stöd för Amazon S3 RRS (Reduced Redundancy Storage) kommer endast data som ändrats efter att detta alternativ har valts att säkerhetskopieras till RRS-lagring. Det innebär att data som redan säkerhetskopierats till Amazon S3:s standardlagring inte kommer att säkerhetskopieras igen.
  • Det finns inte stöd för att flytta säkerhetskopierade filer från Amazon S3 till Amazon Glacier. Om filer flyttas från Amazon S3 till Amazon Glacier via AWS hanteringskonsol, riskerar tillhörande säkerhetskopieringsuppgifter att misslyckas.
  • Säkerhetskopiering med SSL av filer till buckets med virtuell värd-stil stöds inte.
  • Vid överföring av en fil kommer filen att delas upp i delar. Storleken för delarna kan ställas in med alternativet Multipart Upload Part Size i uppgiftens inställning, vilket som standard är 512MB. Om nätverksmiljön inte är stabil rekommenderas att inställningen ställs in lägre.
  • Max antal samtidiga delar vid uppladdning är 10000. Den största filstorleken för en fil är Multipart Upload delstorlek gånger 10000.
  • Filnamnet får inte innehåll något tecken som inte är UTF-8 eller något kontrolltecken.

Att skapa ett Säkerhetskopieringsmål Microsoft Azure:

Ett Säkerhetskopieringsmål Microsoft Azure krävs för säkerhetskopiering av data från DiskStation till en Microsoft Azure blob-server.

  1. Klicka på Skapa, välj Säkerhetskopieringsmål Gemensamt moln och sedan Säkerhetskopieringsmål Microsoft Azure och klicka sedan på Nästa.
  2. Ge ditt säkerhetskopieringsmål ett namn, ange inloggningens autentiseringsuppgifter och fyll sedan i de obligatoriska fälten för att konfigurera din säkerhetskopieringsuppgift.

Noteringar:

  • När säkerhetskopieringsuppgifter skapas med en Microsoft Azure-server som nätverksmål för säkerhetskopiering finns olika alternativ tillgängliga (t.ex. krypterad överföring). Vänligen observera att en del alternativ, såsom överföringskryptering kan minska hastigheten för säkerhetskopieringsuppgiften.
  • Säkerhetskopieringsfiler sparas i den Container du valde på Microsoft Azure-servern och att det inställda Blob-värdet börjar med ett katalognamn som du väljer och slutar med den kompletta filsökvägen för filen.
  • Datastrukturen på Microsoft Azure inkluderar ett par Container och Blob utan någon mappstruktur. Säkerhetskopiering av tomma mappar stöds inte.
  • Om en säkerhetskopierad fil inte har ändrats kommer den inte att säkerhetskopieras till samma Container på Microsoft Azure-server igen.
  • Varje Container på Microsoft Azure måste ha ett unikt namn. Dessutom ska varje namn:
    • vara mellan 3 och 63 tecken långt.
    • endast innehålla gemena bokstäver, siffror och streck (-).
    • börja och sluta med en siffra eller en bokstav och det kan inte ha två på varandra följande streck.
  • Maximal längd på ett Microsoft Azure Blob-namn är 1024 tecken. För att säkerställa att det går att säkerhetskopiera till Microsoft Azure är den längsta sökvägen 1024 minus namnet för målmappen som specificerades när uppgiften skapades.
  • Filnamnet får inte innehåll något tecken som inte är UTF-8 eller något kontrolltecken.
  • Meddelandet om misslyckad autentisering kan orsakas av oriktigt autentiseringsinformation, ett problem med nätverksanslutning eller att tiden på din DiskStation och fjärrservern inte stämmer överens. För att kontrollera tidsinställningarna på din DiskStation, vänligen gå till Kontrollpanel > Nationella inställningar > Tid för att synkronisera tiden med NTP-servern.

Att hantera säkerhetskopieringsmål:

Välj ett befintlig säkerhetskopieringsmål och klicka på någon knapparna här nedanför:

  • Redigera: Ändra inställningar för det valt mål för säkerhetskopiering.
  • Radera: Radera valt mål för säkerhetskopiering.

Säkerhetskopiera till en Delad mapp

Detta typ av säkerhetskopieringsmål kommer endast att behålla den senaste säkerhetskopian. Som standard, efter att den initiala säkerhetskopieringen är slutförd, kommer filer som har tagits bort från källan att tas bort från målet när nästa uppgift utförs. Om du dock hellre vill behålla filerna på målet, även efter att de raderats från källan, vänligen aktivera alternativet Spara de säkerhetskopierade filerna på målenheten, när säkerhetskopieringsuppgifter skapas.

Säkerhetskopiera till volym

Den här typen av säkerhetskopieringsmål måste vara en lokal eller nätverksvolym på en DiskStation-enhet (förutom externa lagringsenheter). Den kommer att arkivera flera versioner av säkerhetskopior så att du enkelt kan återställa mappar från vald tidpunkt.

För att radera lagrat data för en säkerhetskopieringsuppgift:

  1. Gå till fliken Säkerhetskopieringsmål och expandera det säkerhetskopieringsmål som innehåller den uppgift som du vill radera.
  2. Klicka på ikonen Ta bort på din valda Säkerhetskopieringsuppgift och sedan på Ta bort i dialogrutan.

För att radera det lagrade datat för en version av säkerhetskopia:

  1. Gå till fliken Säkerhetskopieringsmål och expandera det säkerhetskopieringsmål som innehåller den versionen av uppgiften som du vill radera.
  2. Klicka på ikonen Versionsinformation för säkerhetskopiering för din valda Säkerhetskopieringsuppgift.
  3. Välj den version av säkerhetskopia som du vill radera och klicka på ikonen Radera.

För att låsa en version av säkerhetskopia för att förhindra att den tas bort vid rotation av säkerhetskopior:

  1. Gå till fliken Säkerhetskopieringsmål och expandera det säkerhetskopieringsmål som innehåller den versionen av uppgiften som du vill låsa.
  2. Klicka på ikonen Versionsinformation för säkerhetskopiering för din valda Säkerhetskopieringsuppgift.
  3. Välj den version av säkerhetskopia som du vill radera och klicka på ikonen Lås.

För att hantera privilegier för ett säkerhetskopieringsmål:

  1. Välj ett säkerhetskopieringsmål som är associerad med en lokal volym och klicka på Redigera.
  2. Klicka på Tillåten lista och sedan på Lägg till för att lägga till användaren eller gruppen till listan.

För att ställa in ett säkerhetskopieringsmål med en ägare:

  1. Välj ett säkerhetskopieringsmål som är associerat med en lokal volym och klicka på ikonen Egenskap för din valda Säkerhetskopieringsuppgift.
  2. Under Privilegier för säkerhetskopiering, välj en Ägare i rullgardinsmenyn.

För att tillåta en användare eller grupp på DiskStation att läsa och återställa lagrad data för en säkerhetskopieringsuppgift:

  1. Välj ett säkerhetskopieringsmål som är associerat med en lokal volym och klicka på ikonen Egenskap för din valda Säkerhetskopieringsuppgift.
  2. Klicka på Tillåten lista och sedan på Lägg till för att lägga till användaren eller gruppen till listan.

Noteringar:

  • Som standard blir den som skapade uppgiften ägaren till ägaren.
  • Endast ägaren av säkerhetskopieringsuppgiften kan utföra uppgiften. Dock har säkerhetskopieringsuppgifter som skapats innan DSM 5.1 inte någon sådan begränsning så alla användare som hör till gruppen administrators kan utföra denna säkerhetskopieringsuppgift.
  • Den som äger uppgiften och användare som hör till gruppen administrators kan radera lagrat data för en säkerhetskopieringsuppgift.
  • Den som äger uppgiften, användare som hör till gruppen administrators och användare som finns på tillåten lista kan alla återställa lagrat data för en säkerhetskopieringsuppgift.
  • Endast användare som hör till gruppen administrators kan konfigurera ett lokalt säkerhetsmål med åtkomstprivilegier och en säkerhetskopieringsuppgift på ett lokalt säkerhetskopieringsmål med en ägare och en tillåten lista.
  • Det lagrade datat för en säkerhetskopieringsuppgift kommer att använda lagringskvot för uppgiftens ägare.
  • Om en volym inte har något säkerhetskopieringsmål kan endast användare som hör till gruppen administrators skapa det första säkerhetskopieringsmålet på volymen.
  • Endast användare som hör till gruppen administrators kan radera lagrat data för ett helt säkerhetskopieringsmål.
  • Varje säkerhetskopiering innehåller systemkonfiguration och metadata för filer.
  • Säkerhetskopiering av krypterade delade mappar stöds inte.
  • Låsning av versioner har inte stöd i DSM 5.1 eller lägre.

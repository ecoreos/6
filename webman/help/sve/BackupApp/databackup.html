Säkerhetskopiering

Säkerhetskopiera & Återställ låter dig säkerhetskopiera olika sorts data (systemkonfigurationer, delade mappar och applikationer/filpaket) på din DiskStation, manuellt eller efter schema. Du kan lagra säkerhetskopierat data i lokala delade mappar, fjärrservrar och gemensamma moln (Amazon S3, Microsoft Azure, hicloud S3 och SFR NAS BACKUP). Med Säkerhetskopiera & Återställ kan varje säkerhetskopieringsuppgift bibehålla flera versioner av säkerhetskopia och rotera dem med policyer för Smart Recycle.

Datasäkerhetskopiering

Säkerhetskopiera & Återställ erbjuder ett antal olika alternativ för säkerhetskopieringsmål och sätt att iordningställa en datasäkerhetskopieringsuppgift.

Att skapa en datasäkerhetskopieringsuppgift:

  1. Klicka på + i det nedre vänstra hörnet och välj Datasäkerhetskopieringsuppgift för att starta Guiden Säkerhetskopiering.
  2. Välj den önskade typen av säkerhetskopieringsmål:
    • Delade lokala/fjärrmappar eller externa lagringseneheter
    • Fjärranslutna rsync-servrar
    • Gemensamma moln (t.ex. Amazon S3 och Microsoft Azure)
  3. Ställ in den önskade typen för din säkerhetskopieringsuppgift:
    • För att skapa en helt ny uppgift, välj Skapa säkerhetskopieringsuppgift.
    • För att återanvända befintligt säkerhetskopierat data på något mål, välj Omlänka till befintlig uppgift. Omlänkning hjälper dig att direkt dra nytta av säkerhetskopierat data från en annan uppgift.
  4. Följ guidens instruktioner för att avsluta inställningarna. Välj Aktivera kryptering på klientsidan för att kryptera data innan överföring för att säkerställa datasäkerhet på säkerhetskopieringsmålet.

Noteringar:

  • Namn på uppgifter kan innehålla 1 till 32 unicode-tecken men kan inte innehålla följande tecken:
    ! " # $ % & ' ( ) * + , / : ; < = > ? @ [ ] \ ^ ` { } | ~
  • Maximal längd för hela mappens sökväg för säkerhetskopiering är 2048 tecken.
  • Om mer än 32000 filer eller mappar laddas upp till en EXT3 delad mapp via AFP kommer alla säkerhetskopieringsuppgifter inklusive den delade mappen att misslyckas.
  • En krypterad delad mapp kan säkerhetskopieras när den är monterad och den kan återställas till en krypterad/okrypterad mapp. Den blir dock alltid återställd som okrypterad /////////och monteras inte automatiskt vid start. Du kommer att behöva ange eller importera krypteringsnyckeln till den delade mappen för att montera den.
  • Det finns inte stöd för att säkerhetskopiera en krypterad mapp till en annan krypterad mapp.
  • Det finns inte stöd för att använda fjärrmappar som källa eller mål för säkerhetskopieringen.
  • Säkerhetskopiering för vissa applikationer har stöd. För att se huruvida ett specifikt filpaket har stöd för säkerhetskopiering och säkerhetskopieringens innehåll samt dess begränsningar, vänligen se hjälpen för vardera applikation.
  • DiskStation har inte stöd för säkerhetskopiering av applikationer utan en skrivbar extern enhet med EXT3/EXT4.
  • DiskStation kan bara säkerhetskopiera delade mappar som skapats på externa enheter med EXT3/EXT4.

Säkerhetskopiera inställningar för Windows ACL:

Inställningar för Windows ACL för filerna på DiskStation kan också säkerhetskopieras, beroende på filsystemet på målet:

  • Alla säkerhetskopieringsmål kan säkerhetskopiera Windows ACL-inställningar oavsett filsystem på målet

Att exportera säkerhetskopierat data till gemensamma moln eller fjärransluten Synology NAS:

Vid säkerhetskopiering av stora mängder data till ett gemensamt moln rekommenderas det att du exporterar den initiala versionen till en extern lagringsenhet (t.ex. en USB-enhet) och levererar enheten till leverantören av molntjänsten. Där kan enheten anslutas och ladda upp data direkt till molnet. På det här sättet kan du signifikant minska tiden för den första säkerhetskopieringen.

  1. Klicka på + i det nedre vänstra hörnet och välj Fjärransluten Synology NAS eller den önskade leverantören av molntjänst.
  2. Välj Exportera till en lokal delad mapp (inklusive en extern lagringsenhet) och slutför inställningarna. Vid uppmaning, välj Aktivera kryptering på klientsidan för skyddad säkerhetskopiering a data.
  3. Leverera den externa lagringsenheten till fjärransluten Synology NAS eller leverantör av molntjänst. Importera exporterat data från den externa lagringsenheten.
  4. När det bekräftats, återgå till Säkerhetskopiera & Återställ och klicka på Omlänka nu.
  5. Följ instruktionerna i Guiden Omlänkning för att länka din DiskStation till dokumentsamlingen. Ett lösenord/krypteringsnyckel behövs om data exporterades krypterat.

Obs:

  • Kryptering på klientsidan kan inte avaktiveras när uppgiften väl har gjorts klar, inte heller kan lösenordet ändras.
  • En dokumentsamling för säkerhetskopiering kan bara länkas till en säkerhetskopieringsuppgift.
  • Om mappen i säkerhetskopieringskällan är en krypterad delad mapp eller en sådans undermapp kan källmappen vara monterad eller omonterad när du skapar säkerhetskopieringsuppgiften.
  • Krypterade delade mappar kan inte vara säkerhetskopieringsmål.
  • Om du väljer Aktivera kryptering på klientsidan krävs ditt lösenord/krypteringsnyckel för att återställa data i den här uppgiften. Förlust av lösenord/krypteringsnyckel orsakar permanent dataförlust.

Så här gör du för att hantera säkerhetskopieringsuppgifter:

Välj en önskad säkerhetskopieringsuppgift i den vänstra panelen och klicka på knapparna här nedanför för motsvarande hantering:

  • Säkerhetskopiera nu: Säkerhetskopiera omedelbart data för den valda uppgiften.
  • Säkerhetskopieringsutforskare (ikonen med ett förstoringsglas): Visar alla tillgängliga versioner av säkerhetskopior och kopierar/återställer/laddar ner de filer de innehåller.
  • Versionslista: Visar tiden den skapades och status för alla versioner av säkerhetskopior, raderar specifika versioner av säkerhetskopior eller låser upp specifika för framtvingad reservering.
  • Redigera: Ändra säkerhetskopieringskälla, -mål, -schema och andra inställningar.
  • Radera: Radera den valda säkerhetskopieringsuppgiften.
  • Avbryt: Stoppar den säkerhetskopieringsuppgift som för närvarande körs.

Rotera versioner:

Rotation av säkerhetskopior är en mekanism för versionshantering. Den hjälper till att hantera versioner av säkerhetskopiering genom att automatiskt radera äldre, onödiga versioner och potentiellt frigöra lagringsutrymme. Du kan aktivera den här funktionen i Inställningar för uppgifter eller konfigurera den när du skapar en säkerhetskopieringsuppgift.

  1. Välj den önskade säkerhetskopieringsuppgiften i den vänstra panelen och klicka på Redigera eller Inställningar > Inställningar för uppgifter.
  2. Välj Aktivera rotation av säkerhetskopior och välj det önskade rotationsläget:
    • Från de tidigaste versionerna: Radera de tidigaste versionerna av säkerhetskopior när tillgängliga versioner överskrider det fastställda antalet.
    • Smart Recycle: Behåller versioner av säkerhetskopior enligt dessa policyer:
      • Versioner som gjorts varje timme från de senaste i 24 timmarna: Behåll den tidigaste version som skapats varje timme.
      • Dagliga versioner från senaste 1 dag till 1 månad: Behåll den tidigaste version som skapats varje dag.
      • Versioner som gjorts varje vecka och är äldre än 1 månad: Behåll den tidigaste version som skapats varje vecka.
  3. Specificera det maximala antalet versioner. När antalet versioner överskrider det antalet roteras den äldsta versionen.
  4. Om du har konfigurerat ett repeterande schema för säkerhetskopiering i Planerat kan du förhandsgranska hur dina versioner av säkerhetskopior kommer att bibehållas och den tidigaste återställningspunkten på tidslinjen.
  5. Klicka på OK för att spara inställningarna.

Obs:

  • Låsta versioner tas aldrig bort.
  • Endast säkerhetskopieringsuppgifter i flera versioner tillhandahåller rotation av versioner.
  • Rotation av versioner har inte stöd i DSM 5.1 eller lägre.
  • Rotation kommer att bli utlöst direkt efter att inställningarna ändras eller säkerhetskopieringsuppgiften är slutförd.
  • Låsta versioner är exkluderade vid beräkning av maximalt antal versioner.

Anteckningar om datasäkerhetskopieringsuppgifter

För varje typ av säkerhetskopieringsuppgift, vänligen läs följande anteckningar för viktig konfigurationsinformation och tips för felsökning.

Lokala säkerhetskopieringsuppgifter:

  • Om data på målet flyttas under tiden säkerhetskopieringsuppgiften utförs, kommer en fullständig säkerhetskopiering att utföras nästa gång uppgiften utförs.
  • Nya externa enheter kan behöva formateras innan de kan tas i bruk som säkerhetskopieringsmål.
  • Innan du skapar en säkerhetskopieringsuppgift, säkerställ att du redan har ordentligt har anslutit de externa diskarna till din DiskStation. Du kan gå till Kontrollpanel > Externa enheter för att kontrollera status för USB/eSATA-diskar. Externa diskar kan inte väljas som säkerhetskopieringsmål om den externa disken håller på att initialiseras, formateras eller är felaktig.
  • Fjärrmonterade delade mappar kan inte väljas som källa eller mål för säkerhetskopieringen.
  • Om en krypterad delad mapp avmonteras kan den inte väljas som mål vid en säkerhetskopiering.
  • Miniatyrbilder kan inte säkerhetskopieras när filsystemet på målet är NTFS/FAT.

Fjärrmonterade rsync-säkerhetskopieringsuppgifter:

  • Följande lägen finns tillgängliga vid skapande av en säkerhetskopieringsuppgift med rsync-kompatibla servrar som nätverksmål för säkerhetskopiering: Läget rsync daemon som används när du väljer en modul för säkerhetskopiering (delade mappar på den rsync-kompatibla servern) från rullgardinsmenyn bredvid Säkerhetskopieringsmodul; och fjärrstyrt skalläge, vilket endast är aktiverat när du anger en absolut sökväg (startar med "/") i Säkerhetskopieringsmodul.
  • För Linux-användare är endast kontona admin, root och rsync tillåtna att utföra säkerhetskopieringsuppgifter.
  • Om du vill ha åtkomst till målservern över Internet genom att använda dess värdnamn måste värdnamnet vara en FQDN (Fully Qualified Domain Name, på svenska fullt kvalificerat domännamn), till exempel abc.bcd.com.
  • Vid säkerhetskopiering från en Synology NAS-enhet till en annan blir du uppmanad att ange användarnamnet och lösenordet för ett konto på målservern. Vid säkerhetskopiering till delade mappar på en annan DiskStation kan du göra så i enlighet med mappens åtkomsträttigheter. Endast delade mappar till vilka den specifika användaren har åtkomstprivilegier kommer att synas. Om du inte kan hitta de önskade delade mapparna på målet, vänligen kontrollera dina åtkomstprivilegier.
  • Överföringskryptering har endast stöd när den server som är mål stöder krypterad nätverkssäkerhetskopiering och lösenordet för autentisering inte är tomt.
  • Fjärrstyrt skalläge stöder endast krypterad överföring.
  • Om du säkerhetskopierar filer till en Synology NAS-enhet via en rsync-klient måste filnamnen vara i teckentabell UTF-8.
  • Om du säkerhetskopierar filer till en Synology NAS-enhet via en rsync-kompatibel klient kan du endast utföra uppgiften med kontona admin, root och rsync på DiskStation.
  • Om du säkerhetskopierar filer till din DiskStation TILL en rsync-kompatibel server, måste du också mata in korrekt kontonamn och lösenord för rsync backup-konto. Du måste också välja rätt kopieringsmodul med skrivprivilegier.

Amazon S3-säkerhetskopieringuppgifter:

  • När uppgifter för säkerhetskopiering med Amazon S3-server som nätverksmål för säkerhetskopiering skapas finns flertalet alternativ tillgängliga, såsom överföringskryptering, säkerhetskopiering till Amazon S3 RRS (Reduced Redundancy Storage) och andra. Vänligen observera att en del alternativ, såsom överföringskryptering kan minska hastigheten för säkerhetskopieringsuppgiften.
  • Säkerhetskopieringsfiler sparas i den Bucket du valde på Amazon S3-servern och Set KEY-värdet börjar med ett katalognamn som du väljer och slutar med den kompletta filsökvägen för filen.
  • Datastrukturen på Amazon S3 inkluderar ett par Bucket och KEY utan någon mappstruktur. Säkerhetskopiering av tomma mappar stöds inte.
  • Om en säkerhetskopierad fil inte har ändrats kommer den inte att säkerhetskopieras till samma Bucket på Amazon S3-server igen.
  • Varje Bucket på Amazon S3 måste ha ett unikt namn. Dessutom ska varje namn:
    • vara mellan 3 och 63 tecken långt.
    • endast innehålla gemena bokstäver, siffror, punkter (.) och streck (-).
    • börja och sluta med en siffra eller en bokstav och inte ha ett (-) i slutet, innehålla två på varandra följande punkter eller innehålla streck bredvid punkter (t.ex. är "my-.bucket.com" och "my.-bucket" ogiltiga).
  • En Amazon S3 KEY får inte överskrida 1024 tecken. För att säkerställa att det går att säkerhetskopiera till Amazon S3 är den längsta sökvägen 1024 minus namnet för målmappen som specificerades när uppgiften skapades.
  • Om du aktiverar stöd för Amazon S3 RRS (Reduced Redundancy Storage) kommer endast data som ändrats efter att detta alternativ har valts att säkerhetskopieras till RRS-lagring. Det innebär att data som redan säkerhetskopierats till Amazon S3:s standardlagring inte kommer att säkerhetskopieras igen.
  • Det finns inte stöd för att flytta säkerhetskopierade filer från Amazon S3 till Amazon Glacier. Om filer flyttas från Amazon S3 till Amazon Glacier via AWS hanteringskonsol, riskerar tillhörande säkerhetskopieringsuppgifter att misslyckas.
  • Säkerhetskopiering med SSL av filer till buckets med virtuell värd-stil stöds inte.
  • Vid överföring av en fil kommer filen att delas upp i delar. Storleken för delarna kan ställas in med alternativet Multipart Upload delstorlek i uppgiftens inställning, vilket som standard är 512MB. Om nätverksmiljön inte är stabil rekommenderas att inställningen ställs in lägre.
  • Max antal samtidiga delar vid uppladdning är 10000. Den största filstorleken för en fil är Multipart Upload delstorlek gånger 10000.
  • Filnamnet får inte innehåll något tecken som inte är UTF-8 eller något kontrolltecken.

Microsoft Azure-säkerhetskopieringsuppgifter:

  • När säkerhetskopieringsuppgifter skapas med en Microsoft Azure-server som nätverksmål för säkerhetskopiering finns olika alternativ tillgängliga (t.ex. krypterad överföring). Vänligen observera att en del alternativ, såsom överföringskryptering kan minska hastigheten för säkerhetskopieringsuppgiften.
  • Säkerhetskopieringsfiler sparas i den Container du valde på Microsoft Azure-servern och att det inställda Blob-värdet börjar med ett katalognamn som du väljer och slutar med den kompletta filsökvägen för filen.
  • Datastrukturen på Microsoft Azure inkluderar ett par Container och Blob utan någon mappstruktur. Säkerhetskopiering av tomma mappar stöds inte.
  • Om en säkerhetskopierad fil inte har ändrats kommer den inte att säkerhetskopieras till samma Container på Microsoft Azure-server igen.
  • Varje Container på Microsoft Azure måste ha ett unikt namn. Dessutom ska varje namn:
    • vara mellan 3 och 63 tecken långt.
    • endast innehålla gemena bokstäver, siffror och streck (-).
    • börja och sluta med en siffra eller en bokstav och det kan inte ha två på varandra följande streck.
  • Maximal längd på ett Microsoft Azure Blob-namn är 1024 tecken. För att säkerställa att det går att säkerhetskopiera till Microsoft Azure är den längsta sökvägen 1024 minus namnet för målmappen som specificerades när uppgiften skapades.
  • Filnamnet får inte innehåll något tecken som inte är UTF-8 eller något kontrolltecken.
  • Meddelandet om misslyckad autentisering kan orsakas av oriktigt autentiseringsinformation, ett problem med nätverksanslutning eller att tiden på din DiskStation och fjärrservern inte stämmer överens. För att kontrollera tidsinställningarna på din DiskStation, vänligen gå till Kontrollpanel > Nationella inställningar > Tid för att synkronisera tiden med NTP-servern.

hicloud S3-säkerhetskopieringuppgifter:

  • När säkerhetskopieringsuppgifter skapas med en hicloud S3-server som nätverksmål för säkerhetskopiering finns olika alternativ tillgängliga (t.ex. krypterad överföring). Vänligen observera att en del alternativ, såsom överföringskryptering kan minska hastigheten för säkerhetskopieringsuppgiften.
  • Säkerhetskopieringsfiler sparas i den Bucket du valde på hicloud S3-servern och Set KEY-värdet börjar med ett katalognamn som du väljer och slutar med den kompletta filsökvägen för filen.
  • Datastrukturen på hicloud S3 inkluderar ett par Bucket och KEY utan någon mappstruktur. Säkerhetskopiering av tomma mappar stöds inte.
  • Om en säkerhetskopierad fil inte har ändrats kommer den inte att säkerhetskopieras till samma Bucket på hicloud S3-server igen.
  • Varje Bucket på hicloud S3 måste ha ett unikt namn. Dessutom ska varje namn:
    • vara mellan 3 och 63 tecken långt.
    • endast innehålla gemena bokstäver, siffror, punkter (.) och streck (-).
    • börja och sluta med en siffra eller en bokstav och inte ha ett (-) i slutet, innehålla två på varandra följande punkter eller innehålla streck bredvid punkter (t.ex. är "my-.bucket.com" och "my.-bucket" ogiltiga).
  • En hicloud S3 KEY får inte överskrida 255 tecken. För att säkerställa att det går att säkerhetskopiera till hicloud S3 är den längsta sökvägen 255 minus namnet för målmappen som specificerades när uppgiften skapades.
  • Vid överföring av en fil kommer filen att delas upp i delar. Storleken för delarna kan ställas in med alternativet Multipart Upload delstorlek i uppgiftens inställning, vilket som standard är 512MB. Om nätverksmiljön inte är stabil rekommenderas att du ställer in ett lägre nummer.
  • Hicloud stöder endast de maximalt 1000 delarna per överföring. Det betyder att den totala storleken på filen inte kan överskrida 1000 multiplicerad med delstorlek. Om, till exempel, Multipart Upload Part Size är inställd på 8MB misslyckas uppladdning av en enstaka fil som är större 8000MB.
  • Säkerhetskopiering av krypterade delade mappar stöds inte.
  • Filnamnet får inte innehåll något tecken som inte är UTF-8 eller något kontrolltecken.

SFR NAS BACKUP-säkerhetskopieringsuppgifter:

  • När säkerhetskopieringsuppgifter skapas med en SFR NAS BACKUP-server som nätverksmål för säkerhetskopiering finns olika alternativ tillgängliga, såsom krypterad överföring med mera.   • Datastrukturen på SFR NAS BACKUP inkluderar ett par Bucket och Key utan någon mappstruktur. Säkerhetskopiering av tomma mappar stöds inte.
  • Om en säkerhetskopierad fil inte har ändrats kommer den inte att säkerhetskopieras till samma Bucket på SFR NAS BACKUP-server igen.
  • Varje Bucket på SFR NAS BACKUP-server måste ha ett unikt namn. Dessutom ska varje namn:
    • vara mellan 3 och 63 tecken långt.
    • endast innehålla gemena bokstäver, siffror, punkter (.) och streck (-).
    • börja och sluta med en siffra eller en bokstav och inte ha ett (-) i slutet, innehålla två på varandra följande punkter eller innehålla streck bredvid punkter (t.ex. är "my-.bucket.com" och "my.-bucket" ogiltiga).
  • En SFR NAS BACKUP KEY får inte överskrida 1024 tecken. För att säkerställa att det går att säkerhetskopiera till SFR NAS BACKUP är den längsta sökvägen 1024 minus namnet för målmappen som specificerades när uppgiften skapades.

Andra S3-kompatibla-säkerhetskopieringsuppgifter:

  • För att använda andra S3-kompatibla moln, ange vid uppmaning IP-adressen/domänen för det önskade S3-kompatibla molnet i fältet S3-server.
  • Om det föreligger något problem med användning, vänligen se den relevanta dokumentationen eller kontakta leverantören av det S3-kompatibla molnet.

LUN-säkerhetskopiering*

Du kan använda Lokal LUN-säkerhetskopiering för att säkerhetskopiera iSCSI LUNs på din DiskStation till lokala delade mappar. Du kan även använda Nätverks-LUN-säkerhetskopiering (Synology-server) för att säkerhetskopiera ytterligare en DiskStation.

Säkerhetskopiera en iSCSI LUN:

  1. Klicka på + i det nedre vänstra hörnet och välj LUN-säkerhetskopieringsuppgift.
  2. När du uppmanas att välja destinationstyp väljer du någon av följande:
    • Lokal LUN-säkerhetskopiering: Välj detta alternativ för att säkerhetskopiera LUN från din DiskStation till en lokal delad mapp eller en extern lagring som är ansluten till USB eller eSATA-porten på din DiskStation.
    • Nätverks-LUN-säkerhetskopiering (Synology server): Välj detta alternativ för att säkerhetskopiera LUN från din DiskStation till en annan DiskStation.
  3. När du uppmanas till det specificerar du säkerhetskopieringsdestinationen och käll-LUN och ställer in säkerhetskopieringsschemat.
  4. Följ guiden för att slutföra installationen.

Efter att säkerhetskopieringsuppgiften har skapats kommer DiskStation att utföra säkerhetskopieringsuppgiften enligt schema och lagra den säkerhetskopierade LUN på destinationen. Säkerhetskopierad LUN används endast för syftet säkerhetskopiering. För åtkomst till säkerhetskopierad LUN, se Återställa säkerhetskopierad LUN för mer information.

Obs:

  • Både vanliga fil- och blocknivå-LUN kommer att säkerhetskopieras som filavbilder på målet. Om återställda, kommer alla blocknivå-LUN att återställas till fil-LUN.
  • Varje säkerhetskopieringsuppgift kan endast inkludera en LUN som säkerhetskopieringskällan.
  • LUNS kan inte säkerhetskopieras till en extern disk med formatet FAT, FAT32, HFS, HFS Plus eller NTFS.
  • Innan Nätverks-LUN-säkerhetskopiering utförs, vänligen logga in på målet DiskStation för att säkerställa att tjänsten nätverkssäkerhetskopiering är aktiverad i Kontrollpanel > Säkerhetskopieringstjänst.
  • Om din DiskStation ansluts till nätverket via en router måste du se till att du har specificerat en regel för vidarebefordring av portar för iSCSI-tjänsten (TCP-port 3260) på routern, innan du utför Nätverks-LUN-säkerhetskopiering.

Om Fullständig säkerhetskopiering och inkrementell säkerhetskopiering:

DiskStation kommer att utföra en fullständig säkerhetskopiering för vald LUN första gången och därefter inkrementell säkerhetskopiering. Men om namnet på den delade mappen eller säkerhetskopieringsdestinationen ändras kommer DiskStation att utföra en fullständig säkerhetskopiering för uppgiften igen.

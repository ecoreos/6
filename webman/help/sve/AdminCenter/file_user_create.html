Skapa en användare

Du kan skapa och hantera flera användare på din DiskStation.

Skapa en användare:

  1. Gå till Kontrollpanel > Användare.
  2. Klicka på knappen Skapa och sedan på Skapa användare. Denna åtgärd startar Guiden Skapa Användare.
  3. På sidan för Användarinformation, specificera följande alternativ:
    • Namn: Ange ett namn för användaren. Detta avsnitt är obligatoriskt.
    • Beskrivning: Ange ett kortfattad beskrivning av användaren.
    • E-post: Ange en e-postadress för användaren. Systemmeddelanden, såsom meddelanden om återställning av lösenord, sänds till den adress som specificeras här.
    • Skicka ett meddelande till den nyskapade användaren: Om detta alternativ är aktiverat, kommer systemet att skicka ett meddelande till användaren för att meddela denna om det nya kontot. Detta alternativ kräver aktivering av systemmeddelanden via e-post i Huvudmeny > Kontrollpanel > Meddelande > E-post. Som standardinställning visas inte användarens lösenord i e-postmeddelandet. Om du emellertid vill visa lösenordet i meddelandet, vänligen markera rutan Visa användarens lösenord i e-postmeddelandet.
    • Förbjud användaren att ändra kontolösenord: Om detta alternativ är aktiverat, kommer användaren inte att kunna ändra lösenordet som används för att logga in på DSM.
  4. På sidan Grupp > Skapa > Guiden Skapa grupp, specificera de grupper till vilka den nya användaren ska tillhöra, (t.ex. användare, administrators, etc.).
  5. På sidan Tilldela behörigheter för delade mappar, välj vilka delade mappar den nya användaren har åtkomst till genom att ändra dennas åtkomsträttigheter. Prioriteten för privilegier är enligt följande: Ingen åtkomst > Läsa/skriva > Endast läsbar.
    • Förhandsgranska: Denna kolumn visar användarens faktiska åtkomsträttigheter till delade mappar, i enlighet med nuvarande inställningar och de grupper som användaren tillhör.
    • Gruppbehörigheter: Denna kolumn visar åtkomstprivilegier till delade mappar, i enlighet med de grupper som användaren tillhör.
  6. På sidan Användarkvot, kan du kontrollera hur mycket lagringsutrymme som kan användas av den nya användaren. För att begränsa den mängd utrymme användaren kan använda, markera rutan under Aktivera kvot, för varje volym/delad mapp. Ange sedan den maximala mängden utrymme som kan användas i fältet för Kvota, (t.ex. 10 GB).
  7. På sidan Tilldela behörigheter för applikationer kan du kontrollera till vilka tjänster användaren har åtkomst.
  8. På sidan Inställning för grupphastighetsbegränsning, kan du aktivera hastighetsbegränsning för olika typer av tjänster (t.ex. File Station, FTP, osv.) för att begränsa den mängd bandbredd användaren använder vid överföring av filer. För varje tjänst kan du välja en av de följande:
    • Ingen: Inställningen för hastighetsbegränsning kommer att vara samma som för gruppen användaren tillhör.
    • Konstant: En fast, konstant hastighetsbegränsning kommer att appliceras. Du kan specificera hastighetsbegränsningarna för uppladdning och nedladdning i fälten till vänster.
    • Variabel: Två olika hastighetsbegränsningar kan specificeras och appliceras på användaren, i enlighet med ett schema. Klicka på knappen Anpassa för att ändra inställningarna för hastighetsbegränsning och ställa in schemat.
  9. Efter att du ändrat inställningarna, klicka på Tillämpa för att slutföra.

Begränsningar för användarnamn:

  • Användarnamnet är inte skiftlägeskänsligt och kan innehålla mellan 1 och 64 Unicode-tecken, men kan inte innehålla följande symboler:
    ! " # $ % & ' ( ) * + , / : ; < = > ? @ [ ] \ ^ ` { } | ~
  • Det första tecknet får inte vara ett minustecken eller ett mellanslag och sista tecknet får inte vara ett mellanslag.
  • Följande användarnamn är reserverade för systembruk: mailer-daemon, postmaster.
  • Användarbeskrivningen kan innehålla upp till 64 Unicode-tecken.

Lösenordsbegränsningar:

  • Lösenordet är skiftlägeskänsligt och är begränsat till 127 tecken, inklusive bokstäver, siffror, tecken och mellanslag.

Kvotbegränsning:

  • Den högsta kvot för användning av lagring som kan anges för varje användare är ungefärligen 4 TB (4095 GB).

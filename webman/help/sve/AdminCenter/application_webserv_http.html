Webbprogram

Med den inbyggda funktionen Web Station och stöd för PHP kan du enkelt vara värd för en webbplats på din DiskStation och skapa dynamiska webbplatser för ditt företag eller din hobby. Dessutom tillhandahåller Paketcenter en mångfald av 3:e parts filpaket såsom Innehållshantering, Kundvårdshantering och e-butikssystem. En standard delad mapp med namnet web kommer att skapas för att lagra filerna för din webbpalts.

Aktivera Web Station:

  1. Gå till Kontrollpanel > Webbtjänster.
  2. Markera Aktivera Web Station.
  3. Klicka på Tillämpa.

Obs:

  • För bättre flexibilitet och säkerhet använder Web Station användargruppen http för att utföra uppgifter. Därför, för att kontrollera åtkomstbehörigheter för dina webbsidor, ändra vänligen åtkomstbehörigheter för gruppen http för varje motsvarande mapp och fil. Till exempel, om du vill göra filen sample.htm i den delade mappen web åtkomlig via Web Station behöver du tillse att gruppen http har tillbörliga läs-/skrivbehörigheter för den delade mappen web såväl som filen sample.htm.
  • Filen för välkomstsidan bör heta index.htm, index.html, eller index.php.
  • När du använder funktionen PHP mail() kommer systemet automatiskt att hämta och använda SMTP-serverns information som finns på sidan Meddelande. Se till att du har aktiverat e-postmeddelanden och angivit rätt information för SMTP-servern.
  • Den riktiga sökvägen för den delade mappen webb är /volume1/webb. Vissa applikationer kräver att användaren anger en sökväg i konfigurationsfilen.
  • Namnen blog, photo, photosrc och webdefault är reserverade mappnamn. Dessa namn kan inte användas av Web Station, och du bör därför inte skapa mappar med dessa namn under den delade mappen web.
  • Du kan anpassa felsidan genom att skapa en missing.html -fil, en missing.htm - eller en missing.php -fil. Aktivera först Web Station. Placera sedan filen missing.html, missing.htm, eller missing.php under den delade mappen web på servern. När användarna klickar på en länk till en sida som inte existerar kommer de att se denna anpassade felsida.
  • Maximalt antal aktiva HTTP-anslutningar: 256
  • Maximalt antal virtuella värdar: 30

Aktivera Virtuell värd att vara värd för webbsidor

Du kan vara värd för flera webbplatser på en enda server med funktionen virtuell värd. För att komma igång behöver du registrera några domännamn och konfigurera virtuell värd genom att följa stegen här nedanför.

Att aktivera Virtuell värd:

  1. Klicka på Virtuell värd.
  2. Klicka på Skapa.
  3. Ange ett namn på den undermapp i den delade mappen web som ska lagra webbplatsens filer.
  4. Ange ett värdnamn för webbplatsen.
  5. Ange en port för webbplatsen.
  6. Klicka på OK.

Aktivera personlig webbsida

Funktionen personlig webbsida är ett smidigt sätt för att tillåta lokala användare, domänanvändare och LDAP-användare att skapa sin egen personlig webbsida.

För lokala användare

Varje lokal användare får en unik webbadress server-IP/~användarnamn eller server-namn/~användarnamn. Till exempel:

  1. 192.168.1.1/~ken
  2. SynologyNAS/~ken

För domänanvändare

Varje domänanvändare får en unik webbadress server-IP/~domännamn%5canvändarnamn eller server-namn/~domännamn%5canvändarnamn. Till exempel:

  1. 192.168.1.1/~testdomän%5cken
  2. SynologyNAS/~testdomän%5cken

För LDAP-användare

Varje LDAP-användare får en unik webbadress server-IP/~användarnamn@domännamn eller server-name/~användarnamn@domännamn. Till exempel:

  1. 192.168.1.1/~ken@testdomän
  2. SynologyNAS/~ken@testdomän

Obs:

Innan aktivering av denna funktion, vänligen tillse att funktionen Användarhem har aktiverats. För lokala användare, gå till Kontrollpanel > Användare > Avancerat; för domänanvändare, gå till Kontrollpanel > Katalogtjänst > Domänanvändare > Användarhem; för LDAP-användare, gå till Kontrollpanel > Katalogtjänst > LDAP-användare > Användarhem.

Obs:

Innan aktivering av denna funktion, vänligen tillse att funktionen Användarhem har aktiverats. För lokala användare, gå till Kontrollpanel > Användare > Avancerad.

Att skapa en personlig webbsida:

  1. Om du har markerat Aktivera personlig webbsida kommer en mapp med namnet www automatiskt att skapas i användarens home-mapp.
  2. Användaren kan nu ladda upp filer som hör till webbplatsen (dvs. webbsidor och ljudfiler) till mappen www.
  3. Tillse att användaren har riktiga åtkomstbehörigheter för de uppladdade filerna.

Att anpassa felsidan:

  1. Spara felsidan som missing.html, missing.htm, eller missing.php.
  2. Ladda upp filen till mappen www.

När användare länkar till en sida som inte finns på den lokala användarens webbsida visas den anpassade felsidan.

Stöd och begränsningar:

Lokala användare tillåts endast åtkomst till data under följande:

  • Deras egna home
  • /var/tmp
  • /tmp
  • /volumeX/@tmp

HTTP-tjänst

Förutom standardportnummer 80 (för HTTP) eller 443 (för HTTPS) kan ytterligare portnummer läggas till för åtkomst till Photo Station och Web Station. Du kan lägga till ytterligare ett portnummer från 1 till 65535 för HTTP eller HTTPS.

För att lägga till ytterligare ett portnummer för HTTP- eller HTTPS-åtkomst:

  1. Markera Lägg till en ytterligare HTTP-port eller Lägg till en ytterligare HTTPS-port (alternativet Aktivera HTTPS-anslutning för webbtjänster måste vara aktiverat innan en ytterligare port för HTTPS läggs till).
  2. Ange det önskade portnumret i fältet.

Obs:

  • Följande portar bör undvikas: 20, 21, 22, 23, 25, 110, 137, 138, 139, 143, 199, 443, 445, 515, 543, 548, 587, 873, 993, 995, 3306, 3689, 5000, 5001, 5005, 5006, 5335, 5432, 6881, 8080, 7000, 7001, 8081, 9997, 9998, 9999, 50001, 50002, eMule-standardportar: 4662 (TCP), 4672(UDP), och standardportintervall för FTP: Det faktiska intervallet kan variera beroende på modell.

Applikationsportal

Applikationsportal låter dig konfigurera inställningarna för anslutning för olika applikationer så att du direkt kan få åtkomst till och köra dessa applikationer (t.ex. File Station) i oberoende flikar eller fönster i webbläsare.

Obs:

Du kan redigera inställningarna för applikationer som Audio Station, Download Station, Surveillance Station, Video Station, File Station och många andra.

Anpassa alias

Du kan tilldela ett alias för alla applikationer som utvecklats av Synology, exempelvis File Station. Med ett anpassat alias kan du snabbt öppna applikationen via en specialanpassad URL.

Att anpassa alias för applikationer:

  1. Gå till Kontrollpanel > Applikationsportal > Applikation.
  2. Välj en applikation.
  3. Klicka på Redigera > Allmänt.
  4. I popup-fönstret, välj Aktivera anpassade alias och specificera aliasen.
  5. Klicka på OK för att spara inställningarna.

Obs:

  • En del applikationer har sina standardalias: Audio Station (audio), Download Station (download), File Station (file), Surveillance Station (cam) och Video Station (video).
  • Aliasnamn kan inte vara desamma som är reserverade för systemet, webbläsare eller används av andra applikationer och antal tecken för dem måste begränsas till att vara mellan 2 och 20.
  • Ett alias kan bara bestå av blandade bokstäver, numeriska tecken och två specialtecken (-) och (_).
  • Ett alias kan inte börja eller sluta med något av specialtecknen (-) eller (_).

Att få åtkomst till applikationer med alias:

Efter att du har ställt in ett alias för en applikation (t.ex. File Station) kan du snabbt öppna applikationen genom att ange en URL i följande format: http://DS_IP_OR_SERVER_NAME/ALIAS/ eller https://DS_IP_OR_SERVER_NAME/ALIAS/

Till exempel kan du få snabb åtkomst till File Station via URL:erna: http://192.168.xx.xx/file/ eller https://MySynologyNAS/file/

Anpassa portar för HTTP/HTTPS

Du kan tilldela portar för HTTP/HTTPS för alla applikationer som utvecklats av Synology, exempelvis File Station. Med den anpassade porten kan du snabbt öppna applikationen via en specialanpassad URL.

Att anpassa portar för HTTP/HTTPS för applikationer:

  1. Gå till Kontrollpanel > Applikationsportal > Applikation.
  2. Välj en applikation.
  3. Klicka på Redigera > Allmänt.
  4. I popup-fönstret, markera Aktivera anpassad port (HTTP) eller Aktivera anpassad port (HTTPS).
  5. Specificera ett anpassat portnummer.

Obs:

  • Intervallet för portnumret kan vara mellan 1 och 65535.
  • Följande portar kan inte användas eftersom de är reserverade för systemanvändning:
    • 20, 21, 22, 23, 25, 80, 110, 137, 138, 139, 143, 199, 443, 445, 515, 543, 548, 587, 873, 993, 995, 3306, 3689, 5000, 5001, 5005, 5006, 5335, 5432, 6881, 8080, 7000, 7001, 8081, 9997, 9998, 9999, 50001, 50002
    • eMule-standardport: 4662 (TCP), 4672(UDP)
    • FTP-standardportintervall: Det faktiska intervallet kan variera beroende på modell.

Att få åtkomst till applikationer med HTTP-/HTTPS-portar:

Efter att du har ställt in ett alias för en applikation (t.ex. File Station) kan du snabbt öppna applikationen genom att ange en URL i följande format: http://DS_IP_OR_SERVER_NAME:HTTP_PORT eller https://DS_IP_OR_SERVER_NAME:HTTPS_PORT

Till exempel kan du få snabb åtkomst till File Station via URL:erna: http://192.168.xx.xx:7000 eller https://MySynologyNAS:7001

Anpassa domäner

Du kan matcha ett domännamn med en applikation som utvecklats av Synology, exempelvis File Station. Med den anpassade domänen kan du snabbt öppna applikationen via en specialanpassad URL.

Att anpassa domäner för applikationer:

  1. Gå till Kontrollpanel > Applikationsportal > Applikation.
  2. Välj en applikation.
  3. Klicka på Redigera > Allmänt.
  4. I popup-fönstret, välj Aktivera anpassad domän.
  5. Specificera en anpassad domän.
  6. För ytterligare inställningar kan du välja Aktivera HSTS eller Aktivera HTTP/2 för att motsvara dina behov.

Obs:

  • Innan den här funktionen ställs in måste du ansöka om domännamn från domänleverantörer så att användare kan få åtkomst till tjänsten från Internet.
  • För applikationer (t.ex. File Station) som används för uppladdning/nedladdning föreslår vi att Aktivera HTTP/2 inte väljs, eftersom sådana applikationer kan ha långsamma hastigheter för uppladdning och nedladdning.
  • Varje domännamn kan bara användas för en särskild applikation på din DiskStation.
  • Om domännamnet inte är längre än 15 tecken och överensstämmer med NetBIOS-namngivningskonventioner blir domännamnet automatiskt registrerat och utsänt av systemet.
  • När du får åtkomst till applikationer via webbadresser som använder domännamn går du genom HTTP-standardporten (80) eller HTTPS-standardporten (443).

Att få åtkomst till applikationer via domäner:

Efter att du har konfigurerat en domän (t.ex. file.example.com) för en applikation (t.ex. File Station) kan du snabbt öppna applikationen genom att ange en URL i följande format: http://APP_DOMAIN eller https://APP_DOMAIN

Till exempel kan du få snabb åtkomst till File Station via URL:erna: http://file.example.com eller https://file.example.com

Om domännamnet (t.ex. FileFile) för en applikation (t.ex. File Station) efterlyder NetBIOS-namngivningskonventioner kan Windows-användare få åtkomst till den på en Synology NAS på samma lokala nätverk med en liknande webbadress: http://FileFile eller https://FileFile

Anpassa omvända proxyregler

Din DiskStation kan agera som en omvänd proxyserver som överför förfrågningar från Internet till enheter på det lokala nätverket. Regler för omvänd proxy kan hjälpa dig att dölja känsliga portar från potentiella hot i de två scenarierna här nedanför:

Scenario 1: Anta att den känsliga porten är 80 som enligt brandväggsreglerna inte ska tillåta extern åtkomst. Du kan konfigurera regler för omvänd proxy för att tillåta betrodda användare att, från Internet, få åtkomst till den känsliga porten 80 via en annan öppen port (t.ex. 81). På det sättet kan de betrodda användarna kringgå brandväggen och fortfarande ha möjlighet till åtkomst till port 80.

Scenario 2: Anta att den känsliga porten är 80 som enligt inte ska tillåta extern åtkomst förutom från en specifik enhet (t.ex. en server med namnet ”MyTrustee”). Med omvända proxyregler kan du tillåta att endast trafik från MyTrustee kan nå porten 80 medan trafik från andra enheter inte kan det.

Konfigurera regler för omvänd proxy:

  1. Gå Kontrollpanel > Applikationsportal > Omvänd proxy.
  2. Klicka på Skapa och specificera de följande inställningarna:
    • Beskrivning: Specificera ett namn som hjälper dig att identifiera regelns funktion.
    • Specificera reglerna för Källa (enheten som skickar förfrågningar från Internet) och Mål (enheten i det lokala nätverket):
      • Protokoll: Protokollen HTTP och HTTPS används av både källa/mål
      • Värdnamn: Namnet på enhetenen som är källa/mål
      • Port: Porten som används av enheten som är källa/mål
      • Aktivera HSTS och Aktivera HTTP/2 (endast för källan)
  3. Klicka på OK för att spara inställningarna.

Obs:

Om du anger * som namn på källa eller lämnar tomt tillämpas regeln för alla källor.

FTP/FTPS

I Kontrollpanel > Filtjänster > FTP kan du aktivera FTP-tjänster för att tillåta klienter att få åtkomst till mappar och data på systemet. Före aktivering av FTP, tillse att följande TCP-portar på din router är vidarebefordrade till DiskStation-enheten: 21 (standardkontrollanslutning), 20 (dataanslutning för aktivt läge) and 55536-55567 (dataanslutning för passivt läge).

Aktivera FTP-tjänst:

  1. Aktivera en av följande:
    • Aktivera FTP-tjänst (utan kryptering): Detta är nätverksprotokollet som används som standard för att överföra filer. FTP tillhandahåller inte någon kryptering för att skydda information vid överföringssessioner, såsom lösenord, användarnamn eller filer. Dock är överföringshastigheten snabbare och kräver mindre systemresurser.
    • Aktivera krypteringstjänst för FTP SSL/TLS (FTPS): En förlängning av standard-FTP med ytterligare stöd för krypteringsprotokollen Transport Layer Security (TLS) och Secure Sockets Layer (SSL) som skyddar information under överföringssessioner, såsom lösenord, användarnamn eller filer. Dock blir överföringshastigheten mer långsam och mer systemresurser används vid kryptering.
  2. Klicka på Tillämpa.

Timeout

FTP-server kommer att koppla ifrån inaktiva användare efter den angivna tidsperioden.

Inställning av portnummer för FTP-tjänst

Detta avsnitt låter dig specificera den port som används för FTP kontrollkanal.

Obs:

Standardportnumret för FTP-kontrollkanal är 21. Den kan ställas in från 1 till 65 535 med undantag från följande nummer: 20, 22, 23, 25, 80, 110, 137, 138, 139, 143, 199, 443, 445, 515, 543, 548, 587, 873, 993, 995, 3306, 3689, 5000, 5001, 5005, 5006, 5335, 5432, 6881, 8080, 7000, 7001, 8081, 9997, 9998, 9999, 50001, 50002, och eMule standardportar: 4662 (TCP), 4672(UDP).

Portintervall för passiv FTP

Detta avsnitt låter dig specificera det portintervall som används för passiva FTP-anslutningar. Du kan använda standard portintervall eller specificera ett anpassat. Portintervallet kan ställas in från 1025 till 65535. Det kan innehålla upp till 128 portar.

Rapportera extern IP i PASV-läge

Genom att aktivera detta alternativ rapporterar servern sin externa IP-adress till FTP-klienterna. Den rapporterade IP-adressen kommer att visas i FTP-klientens anslutningslogg. Väl aktiverat, kan du välja mellan att automatiskt upptäcka den externa IP-adressen eller manuellt ange den externa IP-adressen för DiskStation.

Detta alternativ fungerar enbart om DiskStation befinner sig bakom NAT-funktion, och med FTP-klienter som finns i andra subnät än DiskStation. I de flesta fall är detta alternativ onödigt, men om FTP-klienter inte lyckas ansluta till DiskStation, kan du aktivera detta alternativ och försöka igen.

Aktivering av FXP

FXP står för File eXchange Protocol. Genom att aktivera detta alternativ kommer FTP-tjänsten att ha stöd för server-till-server-filöverföring.

Aktivering av FIPS kryptografisk modul

FIPS stöds av FTPS-tjänst som använder OpenSSL FIPS 140-2 validerad kryptografisk modul (Certifikat #1747). Genom att aktivera detta alternativ så använder FTPS-tjänsten extra säkra kryptografialgoritmer för att skapa säkrare sessioner.

Stöd för överföring i ASCII-läge

Aktivera detta alternativ för att ge stöd till överföring i ASCII-läge, vilket tillåter DiskStation att på rätt sätt överföra ASCII-formaterade enkla textfiler via FTP.

UTF-8 teckentabell

Det här alternativet kan användas för att hjälpa FTP-klienter att korrekt visa och hantera filer med flerspråkiga filnamn. För att uppfylla behoven hos olika FTP-klienter tillhandahålls följande tre alternativ:

  • Avaktivera: UTF-8 teckentabell används inte. Det här alternativet är avsett för klienter som använder Utforskaren som en FTP-klient för att ladda upp/ladda ner filer med flerspråkiga filnamn. Det är inte att rekommendera för normal användning.
  • Automatiskt: Låta DSM automatiskt avgöra hur UTF-8 teckentabell ska hanteras. Det här alternativet är inte att rekommendera för normal användning.
  • Framtvinga: Framtvinga användning av UTF-8 teckentabell Om du har aktiverat UTF-8 teckentabell på din FTP-klient, och om den fortfarande inte kan visa och hantera dina filer med flerspråkiga filnamn korrekt. kan du aktivera detta alternativ.

Anslutningsbegränsning

Detta avsnitt innehåller följande alternativ.

  • Begränsa anslutningar per IP: När den här funktionen är aktiverad begränsar servern det maximala antalet anslutningar per IP-adress. Aktivera denna funktion och begränsning av FTP-bandbredd för att effektivt kunna hantera bandbredd.
  • Aktivera begränsning i FTP-bandbredd: Aktivera denna funktion för att begränsa bandbredden för en enskild FTP-anslutning.

Obs:

  • Kontot "guest" kan inte logga in på servern via FTP.
  • Teckentabell inställning på FTP-klientens datorn måste vara samma som DiskStation för att kunna komma åt data på rätt sätt.
  • På vissa modeller stöder FTP-tjänsten anslutning med hjälp av SSL eller TLS.
  • Brandväggsinställningarna i din FTP-klient och två FTP-servrar ska vara korrekt konfigurerade, annars kommer FXP-funktionen inte att fungera bra.
  • Vissa FTP-klienter, såsom de som använder Internet Explorer 8/9 behöver logga in med sitt användarnamn och lösenord. Till exempel: ftp://username:password@your.ftp.com.

Ansluta till VPN

Du kan enkelt göra din DiskStation till en VPN-klient och ansluta till en befintlig VPN (Virtual Private Network)-server via protokollen PPTP, OpenVPN eller L2TP/IPSec. Om du har flera VPN-servrar kan du också skapa specifika VPN-profiler för varje server för att snabbt byta och ansluta till olika VPN-servrar med ett enkelt klick.

Vad är VPN?

Ett VPN eller virtual private network (på svenska virtuellt privat nätverk) är en lösning för att möta behovet av att på ett säkert sätt kunna få åtkomst till resurser på ditt privata nätverk från Internet. För företag och en del individer, som har behov av att utöka nätverkskapacitet som växer, kommer kostnaden för deras fysiska nätverkssystem och deras tekniska support att öka exponentiellt. När kostnadseffektivitet och långsiktigt underhåll övervägs är VPN en smart och allt mer attraktiv lösning. Med kryptering och andra säkerhetsmekanismer låter VPN-tekniken företagsmedlemmar att enkelt få åtkomst till företagets centrala nätverk och dra nytta av de resursers som finns i det, precis som i LAN. Individer kan också få åtkomst till resurser på deras LAN hemma, när de är långt hemifrån.

Obs:

  • DiskStation kan endast ansluta till OpenVPN-servrar som har stöd för tun-typen av tunnlar på lager 3.
  • Innan installation av filpaketet VPN, tillse att du har en VPN-server att ansluta till. För bättre kompabilitet föreslås att du ansluter till VPN-tjänster som finns på en DiskStation som kör filpaketet VPN Server. För att konfigurera en annan DiskStation som en VPN-server, vänligen se DSM hjälp > VPN Server > Konfigurera VPN Server för instruktioner.

Skapa VPN-profiler:

  1. Gå till Kontrollpanel > Nätverk.
  2. På fliken Nätverksgränssnitt, klicka på Skapa och välj Skapa VPN-profil.
  3. När en ny VPN-profil skapas kan du välja någon av de följande typerna av anslutning:
    • PPTP: PPTP (Point-to-Point Tunneling Protocol) är en vanlig VPN-lösning som stöds av de flesta klienter (inklusive Windows, Mac, Linux och mobilenheter).
    • OpenVPN: OpenVPN är en öppen källkodslösning för implementering av VPN-tjänsten. Den skyddar VPN-anslutningen med krypteringsmekanismen SSL/TLS.
    • L2TP/IPSec: L2TP (Layer 2 Tunneling Protocol) över IPSec tillhandahåller ökad säkerhet för virtuella privata nätverk och stöd finns hos de flesta klienter (såsom Windows, Mac, Linux, och mobila enheter).
  4. Se avsnitten här nedanför, för instruktionerna om att konfigurera någon av de tre typerna av VPN-anslutningar.

Skapa en PPTP-profil:

  1. Vid uppmaning att välja en metod för VPN-anslutning, välj PPTP och klicka sedan på Nästa.
  2. Välj ett namn för den nya profilen, specificera adress för den VPN-servern du vill ansluta med och ange ditt användarnamn och lösenord för servern. Klicka på Nästa för att fortsätta.

    3. Obs:

    • Användarnamnet, lösenordet och adressen för VPN-servern ska tillhandahållas av administratören för VPN-servern.
  3. Välj ett av de följande protokollen för autentisering från menyn Autentisering för att skydda VPN-klienters lösenord under autentiseringen:
    • PAP: Lösenordet krypteras inte vid autentisering.
    • CHAP: Lösenordet kommer att krypteras med CHAP (Challenge-Handshake Authentication Protocol).
    • MS CHAP: Lösenordet kommer att krypteras via Microsoft CHAP version 1.
    • MS CHAP v2: Lösenordet kommer att krypteras via Microsoft CHAP version 2.
  4. Om du väljer MS CHAP eller MS CHAP v2, välj då något av följande från menyn Kryptering för att kryptera VPN-anslutning:
    • Ingen MPPE: VPN-anslutningen kommer inte att skyddas med krypteringen Microsoft Point to Point.
    • Kräv MPPE (40/128 bit): VPN-anslutningen kommer att skyddas med 40-bitars eller 128-bitars kryptering med Microsoft Point to Point.
    • Maximal MPPE (128-bitars): VPN-anslutningen skyddas med en 128-bitars Microsoft Point to Point-kryptering, vilket ger högsta säkerhetsnivån.

    5. Obs:

    • Inställningar för autentisering och kryptering för VPN-klienter måste vara identiska med inställningarna som specificerats i VPN-servern. För mer information, kontakta administratören för VPN-servern.
  5. Markera någon av följande kryssrutor beroende på dina behov:
    • Använd standardgateway på fjärrnätverket: Aktivera detta alternativ för att dirigera nätverkstrafiken för DiskStation till den specificerade VPN-servern.
    • Tillåt andra nätverksenheter att ansluta genom denna Synology-servers Internetanslutning: Aktivera det här alternativet för att tillåta nätverksenheter som finns inom samma lokala nätverk som din DiskStation att ansluta till samma VPN-server.
    • Återanslut när VPN-anslutning har brutits: Om VPN-anslutningen oförväntat tappas kommer system att försöka återupprätta anslutningen fem gånger, ett försök var 30:e sekund.
  6. Klicka på Tillämpa.

Skapa en OpenVPN-profil:

  1. Vid uppmaning att välja en metod för VPN-anslutning, välj OpenVPN och klicka sedan på Nästa.
  2. Välj ett namn för den nya profilen, specificera adress för den VPN-servern du vill ansluta med och ange ditt användarnamn och lösenord för servern.
  3. I fältet Certifikat, klicka på knappen Bläddra för att lokalisera och importera certifikatfilen (t.ex. ca.crt) som exporterades från VPN-servern. Klicka på Nästa för att fortsätta.

    4. Obs:

    • Användarnamnet, lösenordet, adressen och certifikatet för VPN-servern ska tillhandahållas av administratören för VPN-servern.
  4. Markera någon av följande kryssrutor beroende på dina behov:
    • Aktivera komprimering på VPN-länken: Aktivera detta alternativ om du vill utföra datakomprimering före överföring av data via VPN. Detta alternativ kan komma att öka överföringshastigheten men kommer att använda mer systemresurser.
    • Använd standardgateway på fjärrnätverket: Aktivera detta alternativ för att dirigera nätverkstrafiken för DiskStation till den specificerade VPN-servern.
    • Tillåt andra nätverksenheter att ansluta genom denna Synology-servers Internetanslutning: Aktivera det här alternativet för att tillåta nätverksenheter som finns inom samma lokala nätverk som din DiskStation att ansluta till samma VPN-server.
    • Återanslut när VPN-anslutning har brutits: Om VPN-anslutningen oförväntat tappas kommer system att försöka återupprätta anslutningen fem gånger, ett försök var 30:e sekund.
  5. Klicka på Tillämpa.

Skapa en OpenVPN-anslutningsprofil (via .ovpn):

  1. Vid uppmaning att välja en metod för VPN-anslutning, välj OpenVPN (via import av en .ovpn-fil) och klicka sedan på Nästa.
  2. Välj ett namn på den nya profilen och, i fältet Importera .ovpn-fil, klicka på Bläddra för att välja och importera en .ovpn-fil som exporterats från VPN-server.
  3. Ange följande information som tillhandahålls av administratören för din VPN-server: ditt användarnamn, lösenord och importera CA-certifikatfilen (t.ex. ca.crt).
  4. Om leverantören av VPN-server även försett dig med en TLS-auth-nyckel, klientnyckel, klientcertifikat eller en lista över återkallade certifikat, vänligen klicka på Avancerade alternativ och importera dem till deras respektive fält.
  5. Klicka på Nästa för att fortsätta.

    6. Obs:

    • Användarnamnet, lösenordet och alla ytterligare filer, exempelvis certifikat eller nycklar ska tillhandahållas av administratören för VPN-servern.
  6. Markera någon av följande kryssrutor beroende på dina behov:
    • Aktivera komprimering på VPN-länken: Aktivera detta alternativ om du vill utföra datakomprimering före överföring av data via VPN. Detta alternativ kan komma att öka överföringshastigheten men kommer att använda mer systemresurser.
    • Använd standardgateway på fjärrnätverket: Aktivera detta alternativ för att dirigera nätverkstrafiken för DiskStation till den specificerade VPN-servern.
    • Tillåt andra nätverksenheter att ansluta genom denna Synology-servers Internetanslutning: Aktivera det här alternativet för att tillåta nätverksenheter som finns inom samma lokala nätverk som din DiskStation att ansluta till samma VPN-server.
    • Återanslut när VPN-anslutning har brutits: Om VPN-anslutningen oförväntat tappas kommer system att försöka återupprätta anslutningen fem gånger, ett försök var 30:e sekund.
  7. Klicka på Tillämpa.

Skapa en profil för L2TP/IPSec:

  1. Vid uppmaning att välja en metod för VPN-anslutning, välj L2TP/IPSec och klicka sedan på Nästa.
  2. Välj ett namn för den nya profilen, specificera adress för den VPN-servern du vill ansluta med och ange ditt användarnamn och lösenord för servern. Du behöver också ange den i förväg delade nyckeln för VPN-servern. Klicka sedan på Nästa.

    3. Obs:

    • Användarnamnet, lösenordet, adressen och i förväg delade nyckeln för VPN-servern ska tillhandahållas av administratören för VPN-servern.
  3. Välj ett av de följande protokollen för autentisering från menyn Autentisering för att skydda VPN-klienters lösenord under autentiseringen:
    • PAP: Lösenordet kommer inte att krypteras.
    • CHAP: Lösenordet kommer att krypteras med CHAP (Challenge-Handshake Authentication Protocol).
    • MS CHAP: Lösenordet kommer att krypteras via Microsoft CHAP version 1.
    • MS CHAP v2: Lösenordet kommer att krypteras via Microsoft CHAP version 2.

    4. Obs:

    • Inställningar för autentisering och kryptering för VPN-klienter måste vara identiska med inställningarna som specificerats i VPN-servern. För mer information, kontakta administratören för din VPN-servern.
  4. Markera någon av följande kryssrutor beroende på dina behov:
    • Använd standardgateway på fjärrnätverket: Aktivera detta alternativ för att dirigera nätverkstrafiken för DiskStation till den specificerade VPN-servern.
    • Tillåt andra nätverksenheter att ansluta genom denna Synology-servers Internetanslutning: Aktivera det här alternativet för att tillåta nätverksenheter som finns inom samma lokala nätverk som din DiskStation att ansluta till samma VPN-server.
    • Servern befinner sig bakom en NAT-enhet: Som standard tillåter inte protokollet L2TP/IPSec anslutning till en VPN-server som finns bakom en NAT-enhet. Aktivera detta alternativ för att förbigå denna begränsning och tillåta DiskStation att ansluta till en L2TP/IPSec VPN-server som finns bakom en NAT-enhet.
    • Återanslut när VPN-anslutning har brutits: Om VPN-anslutningen oförväntat tappas kommer system att försöka återupprätta anslutningen fem gånger, ett försök var 30:e sekund.
  5. Klicka på Tillämpa.

Ansluta eller koppla ifrån en VPN-server:

  1. Välj en VPN-profil.
  2. Klicka på Anslut eller Koppla ifrån.

Obs:

  • Endast en profil åt gången kan vara ansluten.

Ändra en VPN-profil:

  1. Välj en VPN-profil.
  2. Klicka på Redigera.

Avancerade inställningar för användare

För att ändra avancerade inställningar för användare kan du gå till Kontrollpanel > Användare > Avancerad. Nedanstående alternativ finns tillgängliga.

Tillåt användare att återställa glömda lösenord

Om du vill tillåta användare att återställa bortglömda lösenord via e-post kan du aktivera Tillåt användare som inte är admin att återställa glömda lösenord via e-post. När detta alternativ är aktiverat visas en länk med texten Glömt ditt lösenord? på inloggningssidan för DSM. Om en användare glömmer sitt lösenord, kan denne klicka på denna länk och ange sitt användarnamn. I så fall kommer systemet att skicka ett e-postmeddelande till användaren, med en länk till återställning av det glömda lösenordet.

Obs:

  • Innan aktivering av detta alternativ, vänlig tillse att du redan aktiverat SMTP för e-postmeddelanden. För att göra detta, gå till Kontrollpanel > Meddelande.
  • För att erhålla meddelande från systemet, måste användare ha angett en e-postadress i avsnittet för användarinformation i sitt konto.
  • Användare som hör till gruppen administrators kan inte återställa lösenord med detta alternativ.
  • Domänanvändare kan inte återställa lösenord med detta alternativ.
  • LDAP-användare kan återställa lösenord med detta alternativ, så länge som DiskStation agerar server för LDAP-katalogen.

Lösenordsstyrka

Du kan ställa in regler för lösenordsstyrka för användare av din DiskStation.

Obs:

Regler för lösenordsstyrka gäller endast nya lösenord. Till exempel, nya regler för lösenordsstyrka appliceras bara när ett nytt konto skapas eller när en befintligt användare byter sitt lösenord. Befintliga lösenord och de som hör till importerade användare är undantagna från nya lösenordsregler.

Aktivera regler för lösenordsstyrka:

  1. Markera kryssrutan Tillämpa regler för lösenordsstyrka om du vill aktivera lösenordsskydd.
  2. Välj fler än ett av dessa lösenordsskydd, från reglerna här nedanför:
    • Uteslut namn och beskrivning av användaren från lösenordet: Lösenordet får inte innehålla användarens namn eller användarens beskrivning. UTF-8-kodade tecken är uteslutna.
    • Inkludera både gemener och versaler: Lösenordet måste innehålla både gemena (små) och versala (stora) bokstäver.
    • Inkludera numeriskt tecken: Lösenordet måste innehålla minst ett numeriskt tecken (0~9).
    • Inkludera specialtecken: Lösenordet måste innehålla minst ett ASCII specialtecken (dvs. ~, `, !, @, #, $, %, ^, &, *, (, ), -, _, =, +, [, {, ], }, \, |, ;, :, ', ", <, >, /, ?).
    • Minsta längd på lösenord: Lösenordet måste vara minst det värde som specificeras i textfältet. Lösenordets längd ska vara mellan 6 och 127 tecken.
  3. Klicka på Tillämpa för att spara inställningarna.

2-stegsverifikation

2-stegsverifikation ger dig ökad säkerhet för användarei DSM. Du kan framtvinga administrators i DSM eller alla användare i DSM att aktivera deras 2-stegsverifikation för att ytterligare skydda deras konton.

För att aktivera 2-stegsverifikation för användare i DSM:

  1. Markera rutan Framtvinga 2-stegsverifikation för de följande användarna och välj vilka användare du vill tillämpa detta på.

    2. Obs:

    Om du fortfarande inte har konfigurerat 2-stegsverifikation i Inst. > Konto kommer installationsguiden för 2-stegsverifikation att startas för att hjälpa dig slutföra de erforderliga inställningarna innan du kan tillämpa detta framtvingande.

  2. Med detta framtvingande tillämpat kommer användare som inte har aktiverat sin 2-stegsverifikation att bli ombedda att slutföra inställningarna innan de kan logga in till DSM.

Obs:

Avaktivering av framtvingad 2-stegsverifikation här påverkar inte och avaktiverar heller inte inställningarna för 2-stegsverifikation för varje användare. Användare kan behålla deras inställningar för 2-stegsverifikation eller gå till Inst. > Konto för att avaktivera dessa inställningar.

Användarhem

Aktivera användarhem för att skapa en personlig mapp, home, för varje användare, utom för guest. Alla användare kan få åtkomst till sin egen home-mapp via CIFS, AFP, FTP eller File Station.

Användare som hör till gruppen administrators har åtkomst till alla personliga mappar som finns i den delade mappen homes som är standard. Namnet på home-mappen är det samma som användare.

Aktivera tjänsten Användarhem:

  1. Markera Aktivera tjänsten Användarhem.
  2. Om det finns flera volymer, välj var mappen homes ska lagras.
  3. Klicka på Tillämpa.

Obs:

När tjänsten användarhem för lokala användare avaktiveras kommer även tjänsten användarhem för domänanvändare att avaktiveras samtidigt.

Avaktivera tjänsten Användarhem:

  1. Avmarkera Aktivera tjänsten Användarhem.
  2. Klicka på Tillämpa.

Efter att tjänsten Användarhem avaktiverats kommer den delade mappen homes att bevaras för administratörs-åtkomst. Användare kommer att kunna få åtkomst till sina home-mappar igen när tjänsten för användarhem aktiveras igen.

Obs:

  1. För att ta bort den delade mappen homes måste först tjänsten användarhem avaktiveras.
  2. Tjänsten Användarhem för domänanvändare avaktiveras om tjänsten Användarhem för lokala användare avaktiveras.

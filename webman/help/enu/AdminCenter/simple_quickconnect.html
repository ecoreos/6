QuickConnect

QuickConnect allows client applications to connect to your DiskStation via the Internet without the hassle of setting up port forwarding rules.

Note:

QuickConnect can work with DSM and some Synology-developed packages, such as Audio Station, Video Station, Download Station, Surveillance Station, Photo Station, File Station, Cloud Station, and mobile applications.

To enable QuickConnect:

  1. Go to Control Panel (Simple Mode) > QuickConnect.
  2. Tick the box Enable QuickConnect.
  3. Enter your existing Synology Account information or register a new account.
  4. Specify a new QuickConnect ID. Make sure it's easy to remember, so you'll be able to access your DiskStation anytime.
  5. Click Apply.

Note:

  • Before enabling QuickConnect for client applications, make sure necessary DSM services are already enabled. For example, WebDAV must be enabled on the DiskStation before you can enable QuickConnect for DS file.
  • A custom QuickConnect ID can include letters, numbers, and hyphens ("-"), and must start with a letter.
  • For better QuickConnect performance, it is recommended that you go to Control Panel (Advanced Mode) > External Access > Router Configuration to configure port forwarding for each service:
    • DSM: 5000 (HTTP); 5001 (HTTPS)
    • Photo Station: 80 (HTTP); 443 (HTTPS)
    • Cloud Station：6690

Troubleshooting

If you see the Network error occurred message, this means QuickConnect service has stopped due to network errors. Please check the following:

  • Your DiskStation is connected to an active network (at Control Panel (Advanced Mode) > Network > Network Interface).
  • The DNS server and default gateway settings are properly configured (at Control Panel (Advanced Mode) > Network > General).
  • Make sure the DiskStation can access the Internet using the following destination ports: 22, 80, 443, 8888. 

When the network errors are fixed, QuickConnect service should resume automatically within a few minutes.

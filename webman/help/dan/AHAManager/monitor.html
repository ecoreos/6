Overvåg

Denne side indeholder tre faner, som viser overvågningen af systemstatus, tjeneste og netværksinterface. Denne side sætter dig i stand til at overvåge visse destinationer. Hvis en overvåget destination fejler på den aktive server, vil systemet failover til den passive standby-server (når den passive server fungerer normalt). Følgende faner vises på denne side:

Fanen Systemstatus

Systemstatus overvåges som standard og kan ikke deaktiveres. Failover bliver udløst, når systemetfejl forekommer på den aktive server. Følgende afsnit vises på denne fane:

  • Aktiv serverstatus: Failover ved fejl på den aktive server til at yde tjenester. F.eks. begge strømenheder på den aktive server fejlede.
  • Lagerforbindelse: Failover i den situation, hvor en disk mangler på den aktive server, men findes på den passive server.

Fanen Tjeneste

Viser tjenester, der kan overvåges og deres status. Failover bliver udløst, når tjenester er unormale. Følgende afsnit vises på denne fane:

  • Tjeneste: Viser tjenester, der an overvåges.
  • Status: Viser, om tjenester kører eller ej.
  • Overvågning: Marker afkrydsningsfeltet ved siden af en tjeneste for at starte dens overvågning. Systemet vil failover, hvis den valgte tjenester fejler.

Sådan overvåger du en tjeneste:

  1. Find den tjeneste, du vil overvåge. Sørg for, at tjenestestatus er normal.
  2. Marker afkrydsningsfeltet ved siden af den.
  3. Klik på Anvend for at gemme indstillingerne.

Bemærk:

Tjenester, der bliver overvåget, er følgende: Windows-filtjeneste, iSCSI Target, FTP, Mac File Service, NFS Service.

Fanen Netværksinterface

Viser alle logiske netværksinterfacer, der kan overvåges. Failover bliver udløst, hvis netværksforbindelsen er unormal. (der henvises til Noter som følger). Følgende afsnit vises på denne fane:

  • Netværksinterface: Viser alle interfacer, der kan overvåges.
  • Overvågning: Marker afkrydsningsfeltet ved siden af et interface for at starte dens overvågning.

Bemærk:

Hvis afkrydsningsfeltet Overvågning er markeret, vil systemet failover til den passive server, når netværksforbindelsen for den aktive server fejler, og netværksforbindelsen for den passive server stadig fungerer normalt.

  • Failover bliver kun udløst, hvis den passive server har sundere forbindelser. Hvis f.eks. den aktive server har tre forbindelser, og den ene fejler, vil systemet kun udløse en failover, hvis den passive server har tre eller flere sunde forbindelser.
  • Forbindelser kombineret med Link Aggregation tæller som én forbindelse. Hvis f.eks. to forbindelser kombineres, og en af dem fejler, men den anden fungerer endnu, vil forbindelsen stadig blive anset for sund.

Sådan overvåger du et interface:

  1. Find det interface, du vil overvåge. Sørg for, at kablerne er tilsluttet korrekt til interfacer på både den aktive og passive server.
  2. Marker afkrydsningsfeltet ved siden af den.
  3. Klik på Anvend for at gemme indstillingerne.

Opdelt hjerne

Når en high-availability-klynge fungerer normalt, bør kun en af medlemsserverne påtage sig rollen som aktiv server. I det tilfælde registrerer den passive server tilstedeværelsen af den aktive server via både heartbeat-forbindelsen og dataforbindelsen med de delte udvidelsesenheder, så switchover ikke udløses. Derudover registrerer den passive server det delte lagers status for at undgå at overtage rollen som aktiv server ved en fejl, når den aktuelt aktive server stadig normalt får adgang til de delte udvidelsesenheder.

Ved tab af heartbeat- og alle dataforbindelser eller ved ustabil status for delt lager og system, kan hver af de to servere af sig selv overtage rollen som aktiv server. Denne situation kaldes "split-brain". I dette tilfælde vil en server forsøge at fortsætte med at yde tjenester for at undgå skader på data pga. baggrundsadgang fra begge servere, mens den anden server vil genstarte og kræve forbindelseskontrol bagefter. Forbindelser fra klientbrugerne kan blive ustabile en tid i denne situation.

Programportal

Programportal muliggør konfiguration af forbindelsesindstillingerne til forskellige programmer, så du har direkte adgang til og kører disse programmer (f.eks. File Station) i uafhængige browserfaner eller vinduer.

Bemærk:

Du kan redigere adgangsindstillinger for programmer som Audio Station, Download Station, Surveillance Station, Video Station, File Station og mange andre.

Tilpas aliasser

Du kan tildele et alias til hvert Synology-udviklet program, f.eks. File Station. Med det tilpassede alias kan du hurtigt åbne programmet via en særlig URL.

Sådan tilpasser du programaliasser:

  1. Gå til Kontrolpanel > Programportal > Program i DSM.
  2. Vælg et program.
  3. Klik på Rediger > Generelt.
  4. I pop op-vinduet vælger du Aktiver tilpasset alias og angiver aliasset.
  5. Klik på OK for at gemme indstillingerne.

Bemærk:

  • Nogle programmer har deres standardaliasser: Audio Station (audio), Download Station (download), File Station (file), Surveillance Station (cam) og Video Station (video).
  • Aliasnavne kan ikke være de samme som dem, der er reserveret til systemet, browsere eller bruges af andre programmer, og antallet af tegn er begrænset til mellem 2 og 20.
  • Et alias kan kun indeholder store/små bogstaver, tal og to særlige tegn (-) og (_).
  • Et alias kan ikke begynde eller slutte med en af de særlige tegn (-) eller (_).

Sådan får du adgang til programmer med aliasser:

Når du har konfigureret et alias til et program (f.eks. File Station), kan du hurtigt åbne programmet ved at indtaste en URL i følgende formater: http://DS_IP_OR_SERVER_NAME/ALIAS/ eller https://DS_IP_OR_SERVER_NAME/ALIAS/

Du kan f.eks. hurtigt få adgang til File Station via URL'erne: http://192.168.xx.xx/file/ eller https://MySynologyNAS/file/

Tilpassede HTTP/HTTPS-porte

Du kan tildele en HTTP/HTTPS-port til hvert Synology-udviklet program, f.eks. File Station. Med den tilpassede port kan du hurtigt åbne programmet via en særlig URL.

Sådan tilpasser du programporte til HTTP/HTTPS:

  1. Gå til Kontrolpanel > Programportal > Program i DSM.
  2. Vælg et program.
  3. Klik på Rediger > Generelt.
  4. I pop op-vinduet markerer du Aktiver tilpasset port (HTTP) eller Aktiver tilpasset port (HTTPS).
  5. Angiv et tilpasset portnummer.

Bemærk:

  • Portintervallet kan ligge fra 1 til 65535.
  • Følgende porte kan ikke bruges, fordi de er reserveret til systembrug:
    • 20, 21, 22, 23, 25, 80, 110, 137, 138, 139, 143, 199, 443, 445, 515, 543, 548, 587, 873, 993, 995, 3306, 3689, 5000, 5001, 5005, 5006, 5335, 5432, 6881, 8080, 7000, 7001, 8081, 9997, 9998, 9999, 50001, 50002
    • eMule-standardporte: 4662 (TCP), 4672(UDP)
    • FTP-standardportinterval: Det faktiske interval kan variere, afhængigt af forskellige modeller.

Sådan får du adgang til programmer med HTTP/HTTPS-porte:

Når du har konfigureret et alias til et program (f.eks. File Station), kan du hurtigt åbne programmet ved at indtaste en URL i følgende formater: http://DS_IP_OR_SERVER_NAME:HTTP_PORT eller https://DS_IP_OR_SERVER_NAME:HTTPS_PORT

Du kan f.eks. hurtigt få adgang til File Station via URL'erne: http://192.168.xx.xx:7000 eller https://MySynologyNAS:7001

Tilpassede domæner

Du kan matche et domænenavn med et Synology-udviklet program, f.eks. File Station. Med det tilpassede domæne kan du hurtigt åbne programmet via en særlig URL.

Sådan tilpasser du programdomæner:

  1. Gå til Kontrolpanel > Programportal > Program i DSM.
  2. Vælg et program.
  3. Klik på Rediger > Generelt.
  4. I pop op-vinduet vælger du Aktiver tilpasset domæne.
  5. Angiv et tilpasset domæne.
  6. Af yderligere indstillinger kan du vælge Aktiver HSTS eller Aktiver HTTP/2 til at passe til dine behov.

Bemærk:

  • Før du konfigurerer denne funktion, skal du ansøge om domænenavne fra domæneudbydere, så brugere kan få adgang til tjenesten fra internettet.
  • Ved programmer (f.eks. File Station), der relaterer sig til filupload/-download, foreslår vi, at du ikke vælger Aktiver HTTP/2, da sådanne programmer kan have langsom upload- og downloadhastighed.
  • Hvert domænenavn kan kun bruges til ét særskilt program på din DiskStation.
  • Hvis domænenavnet ikke overstiger 15 tegn og overholder Navngivningskonventionerne for NetBIOS, vil systemet registrere og udsende domænenavnet automatisk.
  • Når du får adgang til programmer via URL'er, der bruger domænenavne, skal du gå gennem standard HTTP-porten (80) eller standard HTTPS-porten (443).

Sådan får du adgang til programmer via domæner:

Når du har konfigureret et domæne (f.eks. file.example.com) til et program (f.eks. File Station), kan du hurtigt åbne programmet ved at indtaste en URL i følgende formater: http://APP_DOMAIN eller https://APP_DOMAIN

Du kan f.eks. hurtigt få adgang til File Station via URL'erne: http://file.example.com eller https://file.example.com

Hvis domænenavnet (f.eks. FileFile) til et program (f.eks. File Station) overholder navngivningskonventionerne til NetBIOS, kan Windows-brugere få adgang til det på en Synology NAS på det samme netværk med en lignende URL: http://FileFile or https://FileFile

Tilpas omvendte proxyregler

Din DiskStation kan fungere som en omvendt proxyserver, der overfører anmodninger fra internettet til enheder på det lokale netværk. Omvendte proxyregler kan hjælpe dig med at skjule følsomme porte fra potentielle trusler som i de to nedenstående scenarier:

Scenarie 1: Antag, at den følsomme port er 80, som ikke skulle tillade ekstern adgang ifølge firewallreglen. Du kan konfigurere en omvendt proxyregel til at tillade brugere med tillid til fra interenettet at å den følsomme port 80 via en anden åben port (f.eks. 81). På den måde kan brugere med tillid til kommer udenom firewallen og stadig kunne få adgang til port 80.

Scenarie 2: Antag, at den følsomme port er 80, som ikke skulle tillade ekstern adgang bortset fra en bestemt enhed (f.eks. en server med navnet "MyTrustee"). Med omvendte proxyregler kan du tillade trafik alene fra MyTrustee til port 80, mens trafik fra andre enheder ikke får det.

Sådan konfigurerer du omvendte proxyregler:

  1. Gå til Kontrolpanel > Programportal > Omvendt proxy.
  2. Klik på Opret, og angiv følgende indstillinger:
    • Beskrivelse: Angiv et navn, der hjælper dig med at identificere regelfunktionen.
    • Angiv reglerne for the Kilde (den enheder, der sender anmodninger fra internettet) og Destination (enheden på det lokale netværk):
      • Protokol: Protokollerne HTTP eller HTTPS bruges af kilden/destinationen
      • Værtsnavn: Navnet på kilde-/destinationsenheden
      • Port: Den port, der bruges af kilde-/destinationsenheden
      • Aktiver HSTS og Aktiver HTTP/2 (kun for kilden)
  3. Klik på OK for at gemme indstillingerne.

Bemærk:

Hvis du indtaster * som værtsnavnet på kilde eller efterlader det tomt, bliver reglen anvendt på alle kilder.

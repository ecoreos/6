備份您的 Synology NAS

概述

硬碟毀損或天然災害可能造成資料意外遺失，如果缺乏可靠的備份方案，更對您的寶貴資料構成潛在威脅。Hyper Backup 提供符合使用直覺、完美整合的備份 / 還原解決方案，協助您保存 / 取回寶貴資料的複本，隨時隨地可在下列的本地及遠端儲存空間進行存取。下列儲存空間目的地皆可支援：

  • 本地共用資料夾 (含連接至 DiskStation 的外接儲存裝置)
  • 遠端 Synology NAS
  • 遠端 rsync 伺服器
  • 遠端 OpenStack Swift 伺服器
  • 公有雲端空間 (Amazon S3、Microsoft Azure 及 OpenStack Swift 等)

注意：

Hyper Backup 僅可備份特定的系統設定及應用程式。如需更多資訊，請參閱 Hyper Backup 的說明文件。

1. 建立本地備份任務至本地共用資料夾 / 外接儲存裝置

Hyper Backup 為不同類型的目的地提供類似的備份及還原流程。以下示範如何進行本地資料備份及還原，可作為其他類型任務的參考。

  1. 登入 DSM 並找到 Hyper Backup 套件。
  2. 按一下左下角的 +，選擇資料備份任務。
  3. 若要建立本地備份任務，選擇本地共用資料夾。
  4. 選擇建立備份任務，然後選擇共用資料夾或是連接至 DiskStation 的外接儲存裝置。
  5. 選擇來源端資料夾來備份內部資料。如果選擇加密共用資料夾，該資料夾在備份過程中必須維持掛載。如果來源端及目的地皆位於同一個儲存空間，來源端資料夾將出現橘色驚嘆號的標記。
  6. 選擇要備份的應用程式。
  7. 指定下列的備份設定，以符合使用需求：
    • 任務名稱：命名備份任務。
    • 啟動任務通知：選擇此選項，便可透過電子郵件、簡訊或推播服務取得任務狀態。
    • 啟動系統設定備份：備份系統設定 (例如使用者帳號密碼)。依照預設，此設定強制套用至所有的備份任務。
    • 壓縮備份資料：壓縮備份資料來減少目的地儲存空間的使用量。如果選取此選項，備份過程將需更多時間。
    • 啟動傳輸加密 (僅供遠端備份使用)：選擇此選項，備份資料在傳輸至遠端目的地 (例如 rsync 伺服器或公共雲端空間) 時便會受到加密保護。
    • Multipart Upload 檔案分割大小 (僅供 S3 相容伺服器使用)：依照指定的區塊大小分割備份資料。
    • 啟動用戶端加密：選擇此選項，便可透過密碼保護備份資料，使目的地資料免於不當存取。
    • 啟動備份排程：選擇此選項，便可規劃自動執行備份任務的排程。
  8. 選擇啟動自動刪除備份版本，便可設定自動刪除的配置方式：
    • 從最早版本開始刪除：一旦超出指定的版本數量，便會刪除現有的最早備份版本。
    • Smart Recycle：在超出指定的版本數量前，系統將會保存每項備份版本。觸發自動刪除後，完全不符合下列任一條件的版本，將被系統先行刪除；如果現存的所有版本皆符合下列條件，系統將自動刪除最早版本。
      • 過去 24 小時內，每小時保留一個版本：系統會保留每小時最早建立的備份版本。
      • 過去 1 天至 1 個月內，每天保留一個版本：系統會保留每天最早建立的備份版本。
      • 過去 1 個月之前，每週保留一個版本： 系統會保留每週最早建立的備份版本。
    • 版本數量：設定保留備份版本的數量上限。
    • 時間軸：依據備份排程及自動刪除配置，顯示可能的備份保留結果。
  9. 按一下套用來完成設定。
  10. 若要立即執行備份任務，請依照備份精靈的指示進行，然後按一下是。

2. 從本地共用資料夾 / 外接儲存裝置還原單份檔案

  1. 請前往 Hyper Backup，在左側面板選擇備份任務，按一下備份瀏覽器 (放大鏡圖示)。如果該任務受到用戶端加密保護，請輸入密碼。
  2. 找到並選取想要還原的檔案。更可使用時間軸橫桿選取想要還原的版本。
  3. 按一下還原，便可還原選取的檔案 / 資料夾。

3. 還原完整任務至特定時間點

  1. 按一下左下角的還原，然後按一下資料。如果該任務受到用戶端加密保護，請輸入密碼。
  2. 選擇想要還原的任務。若要自其他資料庫還原資料，請選擇從現存資料庫還原，然後找到要還原的任務。
  3. 選擇是否還原系統設定。如果選擇還原系統設定，在下方清單中，請選擇想要還原的特定設定。
  4. 找到並選取想要還原的檔案 / 任務。更可使用時間軸橫桿選取想要還原的版本。
  5. 選擇想要還原的應用程式。
  6. 基本的還原資訊便會顯示。
  7. 按一下套用來啟動還原。

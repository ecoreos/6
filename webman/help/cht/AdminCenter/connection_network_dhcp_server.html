DHCP 伺服器

DiskStation 可作為 DHCP 伺服器使用，並指派動態 IP 位址給區域網路內的 DHCP 用戶端。

若要啟動 DHCP 伺服器：

  1. 前往控制台 > DHCP 伺服器 > 網路介面。
  2. 選擇網路介面 (例如 LAN 1) 來指派 IP 位址，並按一下編輯。
  3. 在彈出視窗中，按一下 DHCP 伺服器並選擇啟動 DHCP Server。
  4. 指定下列的 DHCP 設定：
    • 位址租約時間 (分鐘)
    • 主要網域名稱伺服器 (DNS)
    • 次要網域名稱伺服器 (DNS) (選填)
    • 網域名稱 (選填)
    • 啟動 Web Proxy Automatic Discovery (選填)
  5. 設定至少一個子網路。 若要設定子網路，按一下新增來指定下列資訊：
    • 開始 IP 位址
    • 結束 IP 位址
    • 子網路遮罩
    • 預設閘道器 (gateway)
  6. 按一下新增，便可在清單中顯示新增的子網路。
  7. 新增足夠的子網路之後，請記得啟動至少一個子網路。
  8. 按一下確定。

注意：

  • 可依個人需求，建立任何數量的子網路。
  • 如果搭配 DHCP 轉送伺服器，作為 DHCP 伺服器使用的 DiskStation 可指派 IP 位址給其他子網路內的裝置。

DHCP 用戶端

可以找到 DHCP 用戶端清單及網路設定 (例如網路實體 (MAC) 位址、IP 位址、主機名稱、DHCP 租約到期前剩餘時間及保留狀態)。

  1. 請前往控制台 > DHCP 伺服器 > 網路介面，選擇需要的介面。
  2. 按一下編輯 > DHCP 用戶端清單，即可搜尋用戶端及相關細節。

若要管理 DHCP 用戶端的 IP 位址：

可以讓用戶端在租約更新時永遠使用相同 IP 位址，亦可進行用戶端管理。

  • 若要新增 DHCP 用戶端：按一下新增來指派 IP 位址給該用戶端裝置。
  • 若要保留 IP 位址：按一下鎖頭圖示，便可為目前裝置保留 IP 位址。
  • 若要取消位址保留：按一下鎖頭圖示，位址租約時間到期後，其他裝置便可使用目前的 IP 位址。

注意：

如果裝置可擁有 IPv4 及 IPv6 位址，可為該裝置保留一個 IPv4 及 IPv6 位址。

若要搜尋 DHCP 用戶端：

  1. 按一下右上角的漏斗圖示。
  2. 選擇下列一項搜尋條件：
    • 全部：列出所有 DHCP 用戶端及 DHCP 保留位址。
    • DHCP 用戶端：列出所有 DHCP 用戶端 (擁有保留或未保留的 IP 位址)。
    • DHCP 保留位址：列出所有保留的 IP 位址。保留的 IP 位址可能尚未與任何裝置配對。

DHCP 選項

指定 DHCP 選項後，作為 DHCP 伺服器使用的 DiskStation 即可傳送相關訊息至特定子網路中的 DHCP 用戶端。

若要為子網路指定 DHCP 選項：

  1. 前往控制台 > DHCP 伺服器 > 網路介面。
  2. 選擇需要的網路介面，按一下編輯。
  3. 前往 DHCP Server，再前往子網路清單。
  4. 執行下列任一動作：
    • 若要為現有子網路指定 DHCP 選項：選擇想要的介面，然後按一下編輯。
    • 若要新增子網路並指定 DHCP 選項：按一下新增，指定子網路設定。
  5. 選擇需要的 DHCP 選項；若要指定對應的輸入值，按一下空白欄位進行編輯。
  6. 按一下確定。

供應商

來自不同供應商的 DHCP 用戶端可能需要額外 DHCP 資訊。為供應商設定 DHCP 選項，DiskStation 即可提供用戶端相關細節。

若要瞭解各供應商支援的 DHCP 選項，請聯絡個別供應商以取得詳細資訊。

若要新增供應商：

  1. 請前往控制台 > DHCP 伺服器 > 供應商。
  2. 按一下新增來指定供應商及 DHCP 選項：
    • 名稱：指定供應商名稱。
    • 編碼：指定 DHCP 選項的數字編碼。
    • 輸入值：指定選項的輸入值。
    • +：新增 DHCP 選項。
    • -：刪除 DHCP 選項。
  3. 按一下新增來完成設定。

若要為供應商編輯 DHCP 選項：

  1. 前往控制台 > DHCP 伺服器 > 供應商。
  2. 選擇需要的供應商，按一下編輯來管理 DHCP 選項。
  3. 按一下確定。

PXE

DiskStation 可作為 DHCP 伺服器使用，進行 PXE (預先啟動執行環境) 的設定。PXE 為一種網路環境，允許用戶端裝置透過網路介面啟動，不須依賴資料儲存裝置 (例如硬碟)。

執行 PXE 時，開機載入檔可儲存在您的 DiskStation 或遠端 TFTP 伺服器上，用戶端裝置可透過網路使用此開機載入檔進行啟動。

若要使用 DiskStation 作為開機載入檔伺服器：

  1. 務必確認已選擇啟動 TFTP 服務 (位於控制台 > 檔案服務 > TFTP)。
  2. 選擇含有開機載入檔的資料夾作為 TFTP 根目錄。
  3. 請務必確認至少已啟動一個 DHCP 子網路 (位於控制台 > DHCP 伺服器 > 網路介面)。
  4. 前往控制台 > DHCP 伺服器 > PXE。
  5. 選擇啟動 PXE (預先啟動執行環境)。
  6. 選擇本機 TFTP 伺服器，然後選擇開機載入檔。
  7. 按一下套用來儲存設定。

若要使用遠端 TFTP 伺服器作為開機載入檔伺服器：

如果使用遠端 TFTP 伺服器作為開機載入伺服器，毋須更動目前網路架構，即可讓用戶端進行開機。

  1. 請務必確認至少已啟動一個 DHCP 子網路 (位於控制台 > DHCP 伺服器 > 網路介面)。
  2. 前往控制台 > DHCP 伺服器 > PXE。
  3. 選擇啟動 PXE (預先啟動執行環境)。
  4. 選擇遠端 TFTP 伺服器，指定伺服器 IP 位址及開機載入檔的檔案路徑。
  5. 按一下套用來儲存設定。

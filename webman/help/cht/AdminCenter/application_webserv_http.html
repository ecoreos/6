網頁應用程式

透過內建的 Web Station 功能及 PHP 支援，您可以輕鬆地在 DiskStation 上架設、開發雙向互動的網站來因應企業需求或個人興趣。此外，套件中心提供您眾多的第三方套件，例如內容管理、客戶關係管理及電子商務系統。系統會建立一個預設的共用資料夾「web」來讓您儲存您的網頁檔案。

若要啟動 Web Station：

  1. 請前往控制台 > 網頁服務。
  2. 勾選啟動 Web Station。
  3. 按一下套用。

注意：

  • 為提升使用上的彈性及安全性，Web Station 使用 http 使用者群組來執行任務。因此，若要控制網頁的存取權限，請變更 http 群組對於各個相對應資料夾及檔案的存取權限。例如，如果您想讓使用者透過 Web Station 存取 web 共用資料夾中的 sample.htm 檔案，您需確認 http 群組對於 web 共用資料夾及 sample.htm 檔案具有足夠的讀寫權限。
  • 網站首頁檔案應命名為 index.htm、index.html 或 index.php。
  • 當您使用 PHP mail() 功能時，系統會自動擷取並使用通知設定頁面的 SMTP 伺服器資訊。請確認您已啟動電子郵件通知並輸入正確的 SMTP 伺服器資訊。
  • web 共用資料夾的實際路徑為 /volume1/web。有些應用程式會要求使用者在設定檔設定這個路徑。
  • blog、photo、photosrc 及 webdefault 為系統保留的資料夾名稱。因為這些目錄下的檔案將無法被 Web Station 使用，所以請避免在 web 共用資料夾內建立這些目錄名稱。
  • 您可以建立 missing.html、missing.htm 或 missing.php 等檔案來自訂錯誤頁面。只要啟動Web Station，建立一個檔名為「missing」的.html、.htm、或是.php檔案，並上傳到 web 共用資料夾中，則當使用者嘗試連結到一個不存在的頁面時，即會看見您個人化的錯誤頁面。
  • 執行中的 HTTP 連線上限數：256
  • 虛擬主機上限數：30

啟動虛擬主機來架設許多網站

透過虛擬主機的功能，您可以在同一台伺服器上架設多個網站。首先，您需依照以下步驟來註冊數個網域名稱及設定虛擬主機。

若要啟動虛擬主機：

  1. 按一下虛擬主機。
  2. 按一下新增。
  3. 輸入 web 共用資料夾的子資料夾名稱，該子資料夾將用來存放網站檔案。
  4. 輸入網站的主機名稱。
  5. 輸入網站的連接埠。
  6. 按一下確定。

啟動個人網站

個人網站功能允許本機、網域及 LDAP 使用者方便地建立個人網站，並擁有個人專屬的網址。

針對本機使用者

每位本機使用者將會擁有專屬的網址，其格式為伺服器 IP/~使用者帳號或伺服器名稱/~使用者帳號。例如：

  1. 192.168.1.1/~ken
  2. SynologyNAS/~ken

針對網域使用者

每位網域使用者將會擁有專屬的網址，其格式為伺服器 IP/~網域名稱%5c使用者帳號或伺服器名稱/~網域名稱%5c使用者帳號。例如：

  1. 192.168.1.1/~testdomain%5cken
  2. SynologyNAS/~testdomain%5cken

針對 LDAP 使用者

每位 LDAP 使用者將會擁有專屬的網址，其格式為伺服器 IP/~使用者帳號@網域名稱或伺服器名稱/~使用者帳號@網域名稱。例如：

  1. 192.168.1.1/~ken@testdomain
  2. SynologyNAS/~ken@testdomain

注意：

在啟動此功能前，請先確定您已啟動使用者家目錄。本機使用者請至控制台 > 使用者帳號 > 進階設定啟動家目錄；網域使用者請至控制台 > 目錄服務 > 網域使用者 > 家目錄啟動；LDAP 使用者請至控制台 > 目錄服務 > LDAP 使用者 > 家目錄啟動。

注意：

在啟動此功能前，請先確定您已啟動使用者家目錄。本機使用者請至控制台 > 使用者帳號 > 進階設定啟動家目錄。

若要建立個人網站：

  1. 若您已勾選啟動個人網站，個人家目錄中便會自動新增一個名為 www 的資料夾。
  2. 使用者便可上傳網站相關檔案 (例如網頁及音效檔) 至 www 資料夾中。
  3. 務必確認使用者擁有適當的權限，可存取上傳的檔案。

若要個人化錯誤頁面：

  1. 將錯誤頁面存成 missing.html、missing.htm 或 missing.php。
  2. 上傳至 www 資料夾。

當使用者嘗試連結到個人網站裡頭一個不存在的頁面時，即會看見本機使用者個人化的錯誤頁面。

支援及限制：

本機使用者僅允許存取以下資料：

  • 個人家目錄
  • /var/tmp
  • /tmp
  • /volumeX/@tmp

HTTP 服務

除了預設的連接埠 80 (供 HTTP 用) 或 443 (供 HTTPS 用)，您也可以加入其他連接埠號碼來存取 Photo Station 及 Web Station。 您可以額外增加連接埠號碼 (1 到 65535) 供 HTTP 或 HTTPS 連線使用。

若要新增額外的 HTTP 或 HTTPS 連接埠：

  1. 勾選新增額外的 HTTP 連接埠或新增額外的 HTTPS 連接埠 (須先勾選啟動網頁服務的 HTTPS 安全連線才能新增 HTTPS 連接埠)。
  2. 在欄位中輸入想要的連接埠號碼。

注意：

  • 您不應使用下列埠：20, 21, 22, 23, 25, 110, 137, 138, 139, 143, 199, 443, 445, 515, 543, 548, 587, 873, 993, 995, 3306, 3689, 5000, 5001, 5005, 5006, 5335, 5432, 6881, 8080, 7000, 7001, 8081, 9997, 9998, 9999, 50001, 50002, 預設的 eMule 埠：4662 (TCP), 4672 (UDP), 以及預設的 FTP 埠範圍：實際範圍視機種而有所不同。

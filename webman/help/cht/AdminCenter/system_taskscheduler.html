任務排程表

任務排程表可讓您為系統服務及指令碼設定排程，並於預先定義的時段執行任務。

支援任務

可以新增任務來執行以下操作：

  • 使用者定義指令碼：執行使用者定義指令碼或命令。
  • 資源回收筒：清空特定共用資料夾的資源回收筒。此外，清空資源回收筒時，檔案保留規則設定可讓您決定哪些檔案應永久刪除。例如，依據大小、刪除時間或檔案格式，可決定檔案應刪除或保留。
  • 嗶聲控制：讓 DiskStation 發出嗶聲。
  • 系統服務：開啟或停止網路服務，列如 Windows 檔案服務、FTP、自動封鎖等等。
  • 使用者定義指令碼：執行使用者定義指令碼或命令。
  • 資源回收筒：清空特定共用資料夾的資源回收筒。此外，清空資源回收筒時，檔案保留規則設定可讓您決定哪些檔案應永久刪除。例如，依據大小、刪除時間或檔案格式，可決定檔案應刪除或保留。
  • 系統服務：開啟或停止網路服務，列如 Windows 檔案服務、FTP、自動封鎖等等。

管理任務

任務排程表的主要頁面顯示現有的全部任務，提供新增、編輯、執行任務的選項。

若要新增排程任務：

排程任務為在指定日期、時間或依指定頻率執行的任務。

  1. 前往控制台 > 任務排程表，按一下新增，然後選擇排程任務。
  2. 選擇想要排程的任務類型：使用者定義指令碼、資源回收筒、嗶聲控制或系統服務。
  3. 出現新增任務視窗時，按一下一般並修改下列設定：
    • 一般設定：輸入任務名稱、選擇使用時會執行任務的使用者，以及啟動或停用任務。
  4. 按一下排程並修改下列設定：
    • 日期：指定執行任務的時間。可安排任務於一週內的某些日子 (如：每天、週末、或每星期日等) 或特定日期 (如：2015/10/21) 執行。
    • 時間：指定任務每天執行的次數，以及開始和結束的時間。
  5. 按一下任務設定，便可訂定任務設定 (僅供特定任務使用)。
    • 若要接收任務執行細節，選擇透過電子郵件傳送執行細節。
    • 若希望只在異常發生時才傳送通知，選擇僅在指令碼異常終止時傳送執行細節。
    • 若要自訂任務內容，在使用者自定義指令碼下輸入指令。
    • 如果在步驟 2 選擇資源回收筒或系統服務，便可以訂定資源回收筒清空的方式及時間，或決定哪一項服務應啟動 / 停止。
  6. 按一下確定以儲存設定。

若要新增觸發任務：

觸發任務為在特定事件完成前 / 後才執行的任務。

  1. 前往控制台 > 任務排程表，按一下新增，然後選擇觸發任務。
  2. 選擇使用者定義指令碼。
  3. 出現新增任務視窗時，按一下一般並修改下列設定：
    • 一般設定：輸入任務名稱、選擇使用時會執行任務的使用者，選擇觸發的事件 / 先行任務，以及啟動或停用任務。
  4. 按一下任務設定，便可訂定任務設定 (僅供特定任務使用)。
    • 若要接收任務執行細節，選擇透過電子郵件傳送執行細節。
    • 若希望只在異常發生時才傳送通知，選擇僅在指令碼異常終止時傳送執行細節。
    • 若要自訂任務內容，在使用者自定義指令碼下輸入指令。
  5. 按一下確定以儲存設定。

注意：

  • 建立觸發任務時，可能須指定先行任務。先行任務執行完成後，才會執行建立的任務。
  • 啟動僅在指令碼異常終止時傳送執行細節後，如果指令碼結束時數值為 0，系統便不會寄送細節。

若要編輯任務設定：

  1. 選擇現有的任務。
  2. 按一下編輯。
  3. 在顯示的視窗中，修改任務設定。

若要刪除任務：

  1. 選擇現有的任務。
  2. 按一下動作 > 刪除。

若要啟動或停用任務：

  1. 在任務排程表的主要頁面，勾選或取消勾選已啟動下方的核取方塊來啟動或停用任務。
  2. 按一下儲存。

若要手動執行任務：

  1. 選擇單一或多個現有任務。
  2. 按一下執行。

任務排程表限制：

  • 任務名稱可包含最多 512 個可顯示字元。
  • 最大任務數：1024。
  • 指令碼或命令列的長度上限：4095。
  • 嗶聲持續時間可介於 1 至 600 秒。
  • 任務名稱可包含最多 512 個可顯示字元。
  • 最大任務數：1024。
  • 指令碼或命令列的長度上限：4095。

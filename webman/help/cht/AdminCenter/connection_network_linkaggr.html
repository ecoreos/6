Link Aggregation*

支援多重網路埠的 DiskStation 可讓您使用 Link Aggregation 技術合併多個區域網路介面。Link Aggregation 能聚合多個網路介面來增加 DiskStation 的頻寬，並能在連線中斷時提供流量容錯移轉來維持網路連線。

使用 Link Aggregation 合併區域網路介面之後，您將會在控制台 > 網路 > 網路介面中看到一個新的 Bond 介面，此介面的設定方式與其它介面 (如區域網路或 PPPoE) 雷同。

注意：

  • 啟動 Link Aggregation 後，每一個連接的裝置所使用的網路流量將會自動調整並達到平衡。
  • 若網路介面連接至多個交換器，請確認交換器連接埠皆在同一個區域網路或 VLAN 之中。

若要使用 Link Aggregation 合併多個區域網路：

  1. 請前往控制台 > 網路 > 網路介面。按一下新增 > 新增 Bond。
  2. 選擇您想要的模式。

    模式說明：

    • 適應性負載平衡：無論交換器是否支援 Link Aggregation，本模式能使 DiskStation 接收及傳送的網路流量達到最佳狀態。為避免未知錯誤發生，即使交換器支援 Link Aggregation，也請勿在交換器上啟動該功能。
    • IEEE 802.3ad Link Aggregation 動態模式：本模式能使 DiskStation 接收及傳送的網路流量達到最佳狀態，並且須在交換器上啟動 IEEE 802.3ad Link Aggregation (動態模式) (LACP, 802.1AX)。若使用多個交換器，所有交換器皆須為堆疊式交換器並且設定正確。
    • Balance XOR：本模式能平衡 DiskStation 接收與傳送的網路流量，並且須在交換器上啟動 Link Aggregation 靜態模式。若使用多個交換器，所有交換器皆須為堆疊式交換器並且設定正確。若交換器不支援 Link Aggregation，DiskStation 的傳送流量仍可達到平衡。使用本模式可偵測到中斷的連線，但無法偵測原因 (例如，乙太網路纜線已拔除或是交換器未正確設定)。
    • 啟動 / 待命模式：本模式能為 DiskStation 提供網路容錯功能。也就是，當可用的網路介面發生問題時，其餘的區域網路連接埠之一將接續服務來維持網路連線不中斷。選擇本模式時，毋須進行交換器設定，亦可使用支援 IEEE 802.3ad Link Aggregation 的交換器。
  3. 選擇實體網路介面來建立 Link Aggregation。
  4. 進行 IP 設定。如有需要，請啟動 VLAN。按一下套用來完成設定。
  5. 設定完成後，網路介面清單中會出現一個名為 Bond 1 的新介面。

若要編輯 Link Aggregation 設定：

  1. 請前往控制台 > 網路 > 網路介面 來變更 Link Aggregation 模式或檢視實體網路介面資訊。
  2. 選擇 Bond 1 介面並按一下編輯按鈕。
  3. 前往對應的頁籤來修改設定。

若要移除 Link Aggregation：

  1. 前往控制台 > 網路 > 網路介面。
  2. 選擇 Bond 1 介面並按一下刪除按鈕。
  3. 完成後，Bond 1 會被兩個 LAN 介面所取代。

* 僅適用於特定機型。

共用資料夾同步

透過共用資料夾同步服務，使用者可將共用資料夾從一台 Synology NAS 裝置 (來源) 同步至區域網路或網際網路上的其它 Synology NAS 裝置 (目的地)。

第一次進行同步時，共用資料夾同步功能會將所選的共用資料夾進行完整的同步。之後則會僅同步檔案修改過的部分。若在來源端移除檔案，目的地也會移除對應的檔案。

注意：

  • 共用資料夾同步提供單向同步功能，可將來源端檔案同步至目的地。若要使用雙向同步功能，請使用 Cloud Station ShareSync。
  • 如果目的地已存在共用資料夾，其名稱與來源端共用資料夾 (例如名稱為 SharedFolder) 相同，目的地資料夾將被命名為「SharedFolder_1」。
  • 如果目的地沒有與來源端共用資料夾同名的共用資料夾，將會在對應或第一個可用的儲存空間中建立同名的資料夾。
  • 依照預設，使用者皆無權限存取目的地共用資料夾。請連絡目的地 admin，請求權限。

啟動服務

若要啟動共用資料夾同步：

您必須在目的地 Synology NAS 上啟動此服務。

  1. 請前往控制台 > 檔案服務 > rsync。
  2. 選擇啟動 rsync 服務，便可啟動共用資料夾同步服務。
  3. 按一下套用。

若要取消共用資料夾與目的地 Synology NAS 的連結：

如果不同的來源端同時同步名稱相同的共用資料夾至相同目的地 DiskStation，將會產生衝突。在另一個來源端能夠同步資料夾至目的地前，須先在原先的來源端取消共用資料夾的連結。

  1. 登入目的地 Synology NAS，前往控制台 > 共用資料夾同步 > 連線清單。
  2. 選擇發生衝突的來源端 Synology NAS 連線，然後按一下取消連結，便可移除。

現在，另一個來源端便可以同步共用資料夾至目的地 DiskStation。

管理同步任務

若要新增同步任務：

  1. 登入來源端 Synology NAS，前往控制台 > 共用資料夾同步 > 任務清單。
  2. 按一下新增。
  3. 輸入任務的名稱。
  4. 選擇您要同步至目的地的共用資料夾。
  5. 指定目的地 Synology NAS 及下列同步選項：
    • 自訂 SSH 加密連接埠：指定 SSH 傳輸加密的連接埠。
    • 啟動 SSH 傳輸加密：傳輸資料時進行資料加密。此選項提供較高的安全性，但非加密傳輸的效能較佳。您可根據需要選擇。
    • 啟動傳輸壓縮：傳輸資料時進行資料壓縮。可節省頻寬，但會增加 CPU 工作量。
    • 啟動段落分塊 (block-level) 同步化：僅同步資料修改的部分，而非整個檔案。可節省頻寬，但會增加 CPU 工作量。
  6. 請依提示，選擇下列任一選項，決定何時要從來源端同步至目的地：
    • 資料異動時自動進行同步：來源端共用資料夾的資料改變時立即執行同步任務。
    • 手動執行同步化：手動從來源端共用資料夾執行同步任務。
    • 進階排程：按一下設定排程，指定何時執行同步任務。
  7. 按一下套用。現在，可在任務清單上看到同步任務，系統會根據所指定的排程執行任務。

若要管理同步任務：

  1. 在來源端 Synology NAS 上前往控制台 > 共用資料夾同步，按一下任務清單，然後選擇任務。
  2. 執行下列任何操作：
    • 按一下編輯來編輯任務。
    • 按一下刪除來移除任務。
    • 如果同步任務尚未進行，請按一下立即同步來立即執行任務。
    • 如果同步任務正在進行，請按一下取消來停止正在進行的任務。

      • 注意：

      如果同步任務的排程是設定為資料異動時自動進行同步，按一下取消會停止進行中的同步任務。然而，如果同步任務監控的任何共用資料夾內容改變，共用資料夾同步會繼續進行同步任務。

    • 按一下完整同步來同步所有的資料。第一次執行同步任務時，共用資料夾同步會執行完整同步；第一次同步完成之後，僅會同步變更的部分。按一下此按鈕可讓您再次手動同步所有資料。

更多資訊

  1. 任務名稱必須設定為 1 到 32 個 Unicode 字元，但不可為下列字元：
    ! " # $ % & ' ( ) * + , / : ; <= > ? @ [ ] \ ^ ` { } | ~
  2. 若您欲將 ext4 檔案系統的共用資料夾同步至 ext3 的檔案系統，同步任務的資料夾數量若超過 32,000 個時可能會發生錯誤。
  3. 若目的地端 DiskStation 上的共用資料夾曾與其它 DiskStation 進行同步，共用資料夾同步在進行任務時會執行完整的同步。
  4. 若要執行共用資料夾同步任務，您必須擁有目的地 DiskStation 的管理權限。您必須擁有 admin 帳號的密碼，或為管理群組內的一員。
  5. 透過網際網路同步至 DiskStation 時，您可在伺服器名稱欄位輸入其外部 IP 位址或主機名稱來連線。然而，若您選擇以主機名稱來連線，此主機名稱必須符合 FQDN (Full Qualified Domain Name) 格式，例如「www.synology.com」。
  6. 針對連接埠轉送，共用資料夾同步於加密傳輸時使用 22 埠，非加密傳輸時使用 873 埠。
  7. 共用資料夾同步不支援外接儲存裝置。
  8. 執行共用資料夾同步時，目的地的加密共用資料夾將被自動掛載；然而，若要存取內部資料，必須先在目的地掛載加密共用資料夾。
  9. 若所選的共用資料夾沒有任何變更，同步排程將不會在指定的時間執行。
  10. 為了達到最佳使用效能，建議每個同步任務中不要包含超過 200 個共用資料夾。
  11. 若來源端 DiskStation 將 homes 共用資料夾同步至目的地端 DiskStation，目的地端 DiskStation 的使用者家目錄功能會被停用。
  12. 針對搭載非 Intel 處理器的機種，您可以使用共用資料夾同步來同步最多 100,000 個共用資料夾 (包含子資料夾)。
  13. 共用資料夾同步在目的地資料系統為 ext4 時可備份 Windows ACL 設定。
  14. 若目的地端伺服器名稱和您的 DiskStation 相同，共用資料夾同步將會失敗。
  15. 來源進階共用資料夾權限將不會被同步至目的地。系統會在目的地自動設定對管理群組內的使用者的 read-only 權限。

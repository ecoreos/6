Windows 檔案服務

Windows 檔案服務可讓支援 SMB/CIFS 的 Windows、Mac 及 Linux 用戶端存取存放在 DiskStation 上的檔案。如果您想透過 Windows 檔案總管來瀏覽 DiskStation 上的檔案，或是掛載共用資料夾當作網路硬碟使用，請於控制台 > 檔案服務 > Win/Mac/NFS 啟動此選項。

若要啟動 Windows 檔案服務：

  1. 勾選啟動 Windows 檔案服務。
  2. 按一下套用。

注意：

啟動 Windows 檔案服務後，可透過 SMB 掛載已建立完整索引的資料夾至 Mac 電腦上，並可用 Finder 上的 Spotlight 搜尋掛載資料夾內的檔案。

工作群組

此選項可讓您將系統指定到一個特定的工作群組來作網路管理。

注意：

  • Windows 工作群組的名稱可以包含 1 至 15 個字元，但不可包含下列字元：[ ] ; : " < > * + = \ / | ? ,
  • 如果您是 Mac OS X 10.9 的使用者，且您的 DiskStation 已加入 Windows 網域，您可以在 DiskStation 本地帳號前加上 localhost\ 來登入，透過 Windows 檔案服務存取 DiskStation 上的檔案。

不允許存取先前版本

使用者可在 Windows 檔案總管中點選先前版本頁籤來存取資料夾或檔案的快照,不需透過管理者的協助即可還原或複製資料。若您選擇不允許存取先前版本，將不會在「先前版本」頁面中看到任何資訊。

注意：

  • 使用者可存取先前版本的條件為先前版本所對映的快照拍攝時，該使用者帳號已存在，並已具有該快照所拍攝之共用資料夾的讀取權限。

啟動傳輸日誌記錄

勾選這個選項，系統將會記錄透過 CIFS 或 SMB 協定 (即所謂的 Windows 檔案傳輸通訊協定) 存取資料的記錄，包含對檔案及資料夾的新增、移除、讀取、寫入、移動及更名。

進階設定

按一下 進階設定 按鈕來查看下列選項。

  • WINS 伺服器：Microsoft Windows 網路名稱服務 (WINS) 是一個「NetBIOS 名稱」對「IP 位址」的對應服務。 如果系統已設定為以 WINS 伺服器註冊，Windows 的使用者可以更容易在 TCP/IP 網路中尋找系統。
  • Windows 檔案服務優先使用的通訊協定：根據您的網路設定，您可以設定您的 DiskStation 所支援最高的通訊協定等級。
    • 啟動 SMB 2：從 Windows Vista 起已支援 SMB (Server Message Block) 2，此為 SMB 的增強版本。 SMB 2 增加將多個 SMB 動作複合成單一請求的能力，以減少網路封包數量並提昇效能。
    • 啟動 SMB 2 及 Large MTU：傳輸單元最大值 (MTU) 指的是可以透過通訊協定傳送的最大筆資料。若要利用 1 或 10 gigabit (GB) 乙太網路等較快速的網路介面，Large MTU 可將最大傳輸單位提升至 1 megabyte (MB)。啟動 Large MTU 選項可增強大型檔案的傳輸速度及效率，但同時會降低封包處理的數量。
    • 啟動 SMB 3：從 Windows 8 及 Windows Server 2012 起已支援 SMB 3。此為 SMB 2 的增強版本。SMB 3 支援以 AES 方式的檔案加密傳輸，因此可改善 P2P 檔案傳輸的安全性。

    注意：

    • 僅 Windows Vista、Windows Server 2008 R2、Windows 7 及以上版本支援 SMB 2。
    • 僅 Windows Server 2008 R2、Windows 7 及以上版本支援 Large MTU。
    • 若要使 Large MTU 設定在 Windows 電腦上生效，您必須修改電腦的登錄設定。如需更多詳細的說明，請參閱微軟技術支援的說明文件 (例如此文章)。
  • 加密傳輸模式：當 SMB 3 啟動時，Windows 檔案服務將新增加密傳輸來加強檔案傳送的安全性。
    • 停用：將不會套用加密傳輸。
    • 自動：加密傳輸將只會套用至支援此功能的用戶端。
    • 強制：加密傳輸將一律套用。這將導致不支援加密傳輸的用戶端無法使用 Windows 檔案服務。

    • 注意：

    • 僅 Windows 8、Windows Server 2012 及以上版本支援加密傳輸。
    • 啟動加密傳輸時將不支援匿名登入 Windows 檔案服務。
    • 啟動加密傳輸時，請確認未勾選停用 Opportunistic Locking 以避免應用程式逾時。
  • 停用 Opportunistic Locking：勾選這個選項將會停用 CIFS 中的「Opportunistic Locking」，並可提升資料庫程式經由網路存取 DiskStation 裝置共用資料夾上資料庫檔案的效能。
  • 啟動 Local Master Browser：當同一個子網路內存在多台 Windows 電腦時，它們會進行協商並選出一台電腦出任「Local Master Browser」。Local Master Browser 負責維護子網路內其他電腦的清單及共用資源，並和其他電腦分享這些清單。此選項可讓 DiskStation 成為 Local Master Browser。

    • 注意：

    啟動此功能會使得硬碟休眠模式被停用，並一併啟動沒有密碼的 guest 帳號。

  • 啟動 DirSort VFS 模組：將資料夾依照英文字母順序排列。啟動此選項會影響系統效能。
  • 禁用檔案：依照指定條件隱藏檔案。規則可以使用萬用字元 (*)，多個規則之間須以斜線符號 (/) 分隔。 例如：/abc*/*.txt/*.conf/。啟動此選項會影響系統效能。
  • 允許在共用資料夾中使用符號連結：啟動此選項可讓符號連結連結到同一共用資料夾下的檔案或資料夾。您可以透過 DSM 終端機建立符號連結。
  • 允許符號連結跨越共用資料夾：啟動此選項可讓符號連結連結到不同共用資料夾下的檔案或資料夾。 您可以透過 DSM 終端機建立符號連結。請注意，啟動此選項可能會導致不具存取權限的使用者有機會存取符號連結的目標共用資料夾。
  • 啟動 MSDFS VFS 模組：允許符號連結連結到其他 SMB 伺服器上的檔案或資料夾。在舊版本的 Windows 環境 (例如 Windows XP)，用戶端與 DiskStation 需要加入 Windows 網域才能啟動此選項。若要了解更多建立 MSDFS 連結的資訊，請參閱此處。 請注意，取消此功能需清除用戶端裝置的快取記錄，避免影響接下來的正常使用。
  • 停用同一 IP 位址的多重連線：此選項啟動之後，DiskStation 限制用戶端同一時間只能有單一 IP 位址使用 Windows 檔案服務。 當 IP 位址相同的多個用戶端同時使用此服務時，DiskStation 僅會保留最後一個連線並且踢除其他的使用者。在 NAT 環境下，用戶端可能只會使用單一 IP 位址，在此情況下啟動此選項可能會影響正常效能。
  • 收集偵錯記錄：此選項啟動之後，系統將會記錄詳盡的訊息作為除錯之用。啟動此選項會影響系統效能。
  • 套用 UNIX 預設權限：啟動此選項，則上傳或建立檔案及資料夾時會套用 UNIX 預設權限。 檔案的 UNIX 權限為 744，而資料夾的 UNIX 權限為 755。此選項停用時，檔案及資料夾的 UNIX 權限為 777。

    • 注意：

    • 啟動此選項可能會導致不同通訊協定間出現權限不一致的情況。若要避免不一致的情況，建議您停用此選項。

    注意：

    • 針對已啟動 Windows ACL 的共用資料夾 (所有共用資料夾，但不包含「photo」及外接硬碟上的共用資料夾)，請在您的 Linux 或 FTP 用戶端執行 chmod 指令來將資料夾及檔案權限類型從 Windows ACL 改為 UNIX。
    • 啟動此選項可能會導致不同通訊協定間出現權限不一致的情況。若要避免不一致的情況，建議您停用此選項。
  • 建立檔案時不要預留硬碟空間：啟動此選項後，在您的 DiskStation 上建立檔案時將不會預留硬碟空間。
  • 清除 SMB 快取：按一下此按鈕來刪除以下 SMB 快取檔案以作為疑難排解之用：
    • /var/cache/samba/netsamlogon_cache.tdb
    • /var/lib/samba/winbindd_cache.tdb
    • /var/lib/samba/share_info.tdb
    • /var/lib/samba/registry.tdb
    • /run/samba/connections.tdb
    • /run/samba/gencacche.tdb
    • /run/samba/gencache_notran.tdb

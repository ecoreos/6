DDNS

DDNS (Dynamic Domain Name Service，「動態網域名稱系統」) 透過將主機名稱對應到 IP 位址的方式，簡化了從網際網路連接至 DiskStation 的步驟。舉例來說，DDNS 可讓您使用網域名稱 (如：www.john.synology.me) 而無須記住 IP 位址 (如：172.16.254.1) 來存取 DiskStation。

設定 DDNS

您可以將現有的主機名稱指向 DiskStation 的 IP 位址，或者註冊一個 Synology 或其他 DDNS 供應商提供的主機名稱。請參閱供應商提供的資訊來取得主機名稱註冊的說明。

若要設定 DDNS 主機名稱：

  1. 按一下新增按鈕。
  2. 在顯示的視窗中，編輯下列設定：
    • 服務供應商：選擇服務供應商。若要註冊 Synology 提供的免費主機名稱，請從下拉式選單選擇 Synology，按一下立即註冊，並依指示輸入資訊。
    • 主機名稱：輸入已註冊的 DDNS 主機名稱，如：john.synology.me。
    • 使用者名稱：為您的 DDNS 供應商輸入使用者名稱。
    • 密碼：為您的 DDNS 供應商輸入密碼。
    • 外部位址：輸入 DiskStation 的外部 IP 位址，DiskStation 會使用這裡的主機名稱。
    • Heartbeat：啟動此選項來接收關於對應主機名稱的狀態通知 (限由 Synology 提供的主機名稱)。
  3. 按一下測試連線來檢查設定是否正確。
  4. 按一下確定來儲存並結束設定。

設定完成之後，您可以在網頁瀏覽器輸入 DDNS 主機名稱，透過網際網路來存取 DiskStation。

服務需求及限制

  • DDNS 服務供應商的伺服器需正常運作。
  • DiskStation 可以連線到網際網路。
  • DDNS 只支援不需透過 Proxy 代理伺服器連線的網路環境。
  • 每一個 DDNS 供應商只能對應到一個主機名稱。

管理主機名稱

請使用下列按鈕來管理現有的主機名稱項目：

  • 刪除：將所選的主機名稱項目從清單中移除。
  • 編輯：變更所選主機名稱項目的設定。
  • 立即更新：針對 DDNS 服務已啟動的主機名稱項目，更新所有項目的狀態。

建立自訂 DDNS 服務提供商

若您的 DDNS 服務提供者沒有出現在服務提供者清單裡，您可以新增自訂 DDNS 服務提供商。

若要建立自訂 DDNS 服務提供商：

  1. 點一下自訂。
  2. 在視窗裡輸入以下的設定：
    • 服務提供商： 輸入自訂服務提供商的名稱。
    • Query URL： 輸入自訂 服務提供商的 URL。
  3. 點一下儲存來新增自訂 DDNS 服務提供者至服務提供商清單。

服務需求及限制

  • 使用者須自行找到服務供應商在官網上所提供的 Query URL 進行設定。URL 是用來更新 IP 資訊到 DDNS 伺服器時所使用的。
  • Query URL 通常會包含以下四種參數。輸入 query URL 時請取代以下的參數在 Query URL 上的字串。
    • 使用者名稱：__USERNAME__
    • 密碼：__PASSWORD__
    • 主機名稱：__HOSTNAME__
    • 外部位址：__MYIP__
  • Query URL 不能包含空白及分號字元。

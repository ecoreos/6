終端機

於控制台 > 終端機 & SNMP > 終端機啟動選項後，您的 DiskStation 裝置便能支援 Telnet 及 SSH 命令列介面的服務。 您也可以變更 SSH 加密演算法的安全層級。

若要啟動 Telnet/SSH 服務：

  1. 勾選欲選擇的通訊協定 (SSH 或 Telnet) 旁的核取方塊。
  2. 按一下套用。

改變 SSH 服務連接埠：

  1. 在 連接埠輸入新的連接埠編號。
  2. 按一下套用。

有多種不同安全層級的加密演算法可供使用來保護 SSH 連線。 您可以決定使用最符合您需求的演算法。

若要變更 SSH 加密演算法安全層級：

  1. 按一下進階設定。
  2. 選擇一個加密演算法的安全層級。 若您要個別指定可使用的演算法，選擇自訂並從以下三個群組中選擇：加密方式、KEX (金鑰交換，Key Exchange)，及 MAC (訊息鑑別碼，Message Authentication Code)。
  3. 按一下套用。

部分機種支援特定加密演算法的硬體加速。 您可以勾選僅使用支援硬體加速的加密方式選項來改善 SSH 相關服務的傳輸速度，包含加密的網路備份及 SFTP 服務。 在進階設定頁面中，可使用硬體加速的演算法將顯示為紅色。

注意：

  • 部分服務 (例如：網路備份、SFTP) 會自動啟動 SSH 並開啟連接埠 22。 在這種情況下，您必須於控制台 > 終端機 & SNMP > 終端機啟動 SSH 服務，才能使用 SSH 登入 DSM。
  • SSH/Telnet 僅支援以 administrators 群組的帳號登入系統。若要切換為 root 帳號，請先透過 SSH/Telnet 使用 administrators 群組的帳號登入系統，接著執行指令 sudo -i，然後輸入登入用帳號的密碼。
  • 以 SSH/Telnet 登入系統時，密碼不得為空白。
  • 停用 SSH 服務後，所有的 SSH 連線都會終止，包含任何執行中的加密網路備份任務。 建議您在關閉 SSH 服務後再重新執行網路備份任務。
  • 任何不當操作或修改 DiskStation 都可能導致產品故障或資料遺失。
  • 選擇較高的安全層級或只使用硬體加速的加密方式可能導致某些 SSH 用戶端因相容問題而無法連線。
  • 根據選擇的安全層級，某些支援硬體加速的演算法可能無法使用。

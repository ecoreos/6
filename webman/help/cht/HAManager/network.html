網路設定

本頁包含三個分頁，顯示 high-availability 叢集、主伺服器、副伺服器的網路設定。

HA 叢集分頁

當 high-availability 叢集建立時，系統會產生一組 IP 位址來接收外來連線。若有需要，也可以增加更多 high-availability 叢集 IP 位址。此分頁會顯示主伺服器各個網路介面上的 IP 位址 (Heartbeat 連線所用的網路介面不會顯示於此頁面，該網路介面已用於兩台伺服器間的資料複製)。此分頁顯示下列資訊：

  • 網路介面：主伺服器上的網路介面。
  • 連線：顯示該網路介面的連線狀態。
  • IP 位址：當 high-availability 叢集建立時，系統會產生一組 IP 位址來接收外來連線。不論由哪台 Synology 伺服器擔任主伺服器，您都可以透過這組 IP 位址連到 high-availability 叢集登入 DiskStation Manager (DSM)。如此可以避免服務轉移後必須連到不同 IP 位址的情況。
  • 子網路遮罩：該 high-availability 叢集 IP 位址的子網路遮罩。

若要編輯 IP 位址和子網路遮罩：

  1. 選擇要編輯的 IP 位址或子網路遮罩，按下編輯或雙點擊來開始編輯。
  2. 勾選啟動核取方塊來啟動或停用各網路介面。您可啟動一個以上的叢集網路介面，但無法停用您在建立叢集時設定的網路介面。

若要變更 high-availability 叢集伺服器名稱：

  • 按下變更網路設定來檢視 high-availability 叢集名稱並更改其設定。
  • 按下套用來儲存目前的 high-availability 叢集名稱以及 IP 位址設定。
  • 若要放棄目前的變更，按一下取消。

主、副伺服器分頁 (以伺服器名稱顯示)

這些分頁顯示主、副伺服器上網路介面的使用情況。其中， Heartbeat 連線會於主伺服器與副伺服器上各佔用一個網路介面。這些分頁顯示以下資訊：

  • 網路介面：伺服器上的實體網路介面。
  • IP 位址：該網路介面的 IP 位址。
  • 子網路遮罩：該 IP 位址的子網路遮罩。
  • 敘述：顯示網路介面與其 IP 位址的資訊。此欄位會顯示下列可能狀態：
    1. 可存取的原始 IP 位址：主伺服器上網路介面的原始 IP 位址可用來存取資料與服務。
    2. 無法存取的原始 IP 位址：副伺服器上網路介面的原始 IP 位址無法用來存取資料與服務。
    3. Heartbeat 連線 IP，無法由內部網路存取：Heartbeat 連線的原始 IP 位址無法用來存取資料與服務。

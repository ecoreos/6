儲存空間狀態

本頁顯示 high-availability 叢集主、副伺服器的儲存空間、iSCSI LUN (段落分塊) 狀態。

由於主、副伺服器在對應的儲存空間、iSCSI LUN (段落分塊) 容量及設定必須一致，因此兩台伺服器上的儲存空間、iSCSI LUN (段落分塊) 統一由一個清單顯示。若您要檢視主、副伺服器的儲存空間、iSCSI LUN (段落分塊) 狀態，須前往儲存空間狀態頁面 [存取 high-availability 叢集時，在儲存空間管理員中僅能看到主伺服器的儲存空間、iSCSI LUN (段落分塊)]。

本頁面包含以下資訊：

  • 名稱：顯示儲存空間、iSCSI LUN (段落分塊) 名稱。
  • 已使用：顯示儲存空間、iSCSI LUN (段落分塊) 已使用的容量百分比。
  • 已使用/總容量：顯示儲存空間、iSCSI LUN (段落分塊) 總容量及已使用的容量。
  • 狀態：顯示儲存空間、iSCSI LUN (段落分塊) 及 SSD 快取的狀態。
  • 敘述：描述儲存空間、iSCSI LUN (段落分塊) 及 SSD 快取狀態異常時的原因。

若要修復狀態異常的儲存空間或 iSCSI LUN (段落分塊)：

  • 若其中一台伺服器的硬碟損毀，您可以前往硬碟狀態頁面查看哪一顆硬碟損毀而需要更換。在這樣的情況下，請進行下列步驟：
    1. 更換損毀的硬碟 (請參閱 Synology 產品隨附的快速安裝指南來了解硬碟安裝的相關步驟)。
    2. 前往儲存空間狀態頁面然後按一下修復。
  • 若副伺服器處於離線狀態，儲存空間、iSCSI LUN (段落分塊) 狀態會提示您發生錯誤，而硬碟狀態頁面不會顯示副伺服器的任何硬碟。在這樣的情況下，請進行下列步驟來排除問題：
    1. 確認副伺服器已開機。
    2. 檢查所有電源供應。
    3. 檢查所有連線。
  • 若其中一台伺服器的儲存空間、iSCSI LUN (段落分塊) 發生檔案系統問題，請進行下列步驟：
    • 前往儲存空間狀態頁面。
    • 按一下修復。

注意： High-availability 叢集最多支援 64 個儲存空間與 iSCSI LUN (段落分塊)。

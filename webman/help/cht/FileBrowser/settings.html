管理 File Station 設定

若要管理 File Station 設定，請開啟 File Station 並按一下設定按鈕。

一般

您可以在此啟動 File Station 日誌來記錄所有使用者的操作資訊。使用者的操作包含建立資料夾、上傳、下載、刪除、重新命名、移動、複製及屬性設定。

若要啟動 File Station 日誌：

  1. 如果您想查看所有使用者的操作記錄，請勾選啟動 File Station 日誌。
  2. 按一下確定。

若要查看 File Station 日誌：

  1. 按一下檢視日誌來前往日誌中心。
  2. 按一下檔案傳輸頁籤。

若要在使用拖拉上傳或快速鍵時自動覆寫檔案：

如果您要在拖拉電腦桌面的檔案到瀏覽器、或在 File Station 中使用快速鍵貼上檔案時自動覆寫檔名相同的檔案，請勾選將檔案拖拉至瀏覽器上傳或使用快速鍵貼上時總是覆寫檔案。

若要啟動智慧拖拉：

在 File Station 內拖拉檔案時，此功能將會自動將檔案傳輸的動作判定為複製或移動。

  1. 選擇啟動智慧拖拉。
  2. 如果同時也有其他使用者傳輸同名的檔案至相同地點，選擇任一動作便會在此情況下執行：
    • 覆寫：您傳輸的檔案將會覆寫同名檔案。
    • 略過：您傳輸的檔案將被略過，不會覆寫同名檔案。
  3. 按一下確定。

注意：

如果想要改變智慧拖拉功能判定的複製或移動動作，請使用下列快速鍵以符合需求：

  • 若要切換至複製：按住 Ctrl 並進行拖拉。
  • 若要切換至移動：按住 Shift 並進行拖拉。

若要使用 File Station 瀏覽並管理本機電腦上的檔案：

  1. 勾選使用 File Station 瀏覽本機電腦的檔案。
  2. 按一下確定。

若要分享 DSM 使用者清單給非 admin 使用者：

如果非 admin 使用者希望依特定 DSM 使用者 / 群組來搜尋所擁有的檔案，或和其他 DSM 使用者分享檔案連結，此選項可讓非 admin 使用者存取完整的 DSM 使用者清單。

  1. 選擇分享 DSM 使用者清單給非 admin 使用者。
  2. 按一下確定。

套用 UNIX 預設權限

按一下進階設定並勾選該選項來在上傳或建立檔案及資料夾時套用 UNIX 預設權限。所套用的權限和透過 UNIX 指令 umask 設定的權限完全相同。此選項啟動時，檔案的 UNIX 權限為 644，資料夾為 755。此選項停用時，檔案的 UNIX 權限為 666，資料夾為 777。預設的 umask 值為 022。

注意：

  • 針對已啟動 Windows ACL 的共用資料夾 (所有共用資料夾，但不包含「photo」及外接硬碟上的共用資料夾)，請在您的 Linux 或 FTP 用戶端執行 chmod 指令來將資料夾及檔案權限類型從 Windows ACL 改為 UNIX。
  • 啟動此選項可能會導致不同通訊協定間出現權限不一致的情況。若要避免不一致的情況，建議您停用此選項。

掛載 / 連線

根據預設，所有的使用者皆可掛載遠端資料夾或虛擬裝置。您可在此限制只有管理者才能擁有遠端資料夾或虛擬裝置的存取權限。

注意：

非管理者的使用者只能在他們的 home 資料夾中掛載遠端資料夾。

若要允許不同使用者掛載遠端資料夾或虛擬裝置：

  1. 在遠端資料夾或虛擬裝置區塊中選擇所有使用者或管理者。
  2. 按一下確定。

若要允許不同使用者掛載伺服器及雲端服務：

  1. 選擇所有使用者、管理者或特定使用者。
  2. 選擇特定使用者會開啟讓您為不同的使用者設定掛載權限的視窗。若要為特定使用者啟動權限，請確認已啟動欄位下的核取方塊已被勾選。若要為特定使用者停用權限，確定您取消勾選已啟動欄位下的方塊。調整完畢後，按一下確定來關閉本視窗。

    3. 注意：

    • 屬於 administrators 群組的使用者可以使用此功能，但無法進行任何變更。使用 guest 帳號登入的使用者無法使用此功能或進行任何變更。您也能在此為其他所有使用者設定權限。
  3. 按一下確定。

分享連結

根據預設，所有使用者皆可分享檔案連結。您可在此限制只有管理者才能使用分享連結的管理功能。

若要允許不同使用者分享檔案連結：

  1. 選擇所有使用者、管理者或特定使用者。
  2. 按一下確定。

若要以純文字開啟 HTML 檔案：

  1. 勾選總是以純文字格式開啟 HTML 檔案核取方塊。若啟動此選項，當使用者透過檔案分享連結開啟 HTML 檔案 (如 HTM、HTML、JSP、XHTML 檔案) 時，會以純文字格式顯示。此選項可協助避免惡意人士在 HTML 檔案中嵌入指令碼來劫持使用者的登入工作階段。
  2. 按一下確定。

速度限制

您可以啟動速度限制來控制使用者或群組透過 File Station 傳輸檔案時的最大可用頻寬。

若要啟動速度限制：

  1. 勾選啟動速度限制；或者，如果您希望速度限制僅在特定時間執行，勾選根據排程啟動速限。此設定會套用至全伺服器，且會覆寫個別使用者及群組的設定。選擇關閉速限將停用所有使用者及群組的速度限制。
  2. 按一下速度限制設定來修改個別使用者或群組的設定。針對每個使用者及群組，您可以選擇下列其一：
    • 套用群組設定：速度限制設定會與使用者所屬的群組相同。
    • 無：群組成員傳輸檔案時將不會套用任何速度限制。
    • 固定：固定的速度限制將會套用至使用者或群組成員。您可在左側欄位中指定上傳及下載之速度限制。
    • 變換：您可指定兩種不同速度限制，並根據排程套用至使用者或群組成員。按一下自訂按鈕來修改速度限制設定並設定排程。
  3. 按一下套用。

注意：

  • 如需更多群組速度限制設定的資訊，請參閱此處。
  • 為使用者設定速度限制時，根據使用者本身及群組設定所計算出的實際速度限制會顯示在結果區塊。例如，使用者「Achilles」本身的下載限制為 100 KB/s，但其所屬的群組下載限制為 500 KB/s，則結果區塊會顯示「0/500」。

安全性

您可在此處找到下列安全性設定。

若要以純文字開啟 HTML 檔案：

  1. 勾選總是以純文字格式開啟 HTML 檔案核取方塊。若啟動此選項，當使用者在 File Station 中開啟 HTML 檔案 (如 HTM、HTML、JSP、XHTML 檔案) 時，會以純文字格式顯示。此選項可協助避免惡意人士在 HTML 檔案中嵌入指令碼來劫持使用者的登入工作階段。

    2. 注意：

    • 此選項並非通用設定。每位使用者均可依據其個別需求，選擇是否要啟動此選項。
  2. 按一下確定。

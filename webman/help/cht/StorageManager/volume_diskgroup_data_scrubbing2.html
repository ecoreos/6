Data Scrubbing*

Data scrubbing 的功能為資料維護，用於檢測儲存空間或磁碟群組。Data scrubbing 有以下兩種類型：

  • RAID scrubbing：此功能可用於採用 SHR (由三顆以上的硬碟組成)、RAID 5 或 RAID 6 的儲存空間或磁碟群組。它可修復所偵測到的不一致資料。
  • 檔案系統 scrubbing：此功能可用於 Btrfs 檔案系統上的儲存空間。若偵測到任何與總和檢查碼不一致的資料，系統會將檔案路徑記錄於日誌中心。強烈建議您使用備份來修復資料。

建議您定期執行 data scrubbing 以確保資料的一致性，避免重要資料在硬碟故障時出現遺失的情形。

若要執行 data scrubbing：

  1. 開啟儲存空間管理員。
  2. 前往儲存空間或磁碟群組。
  3. 請選擇要執行 data scrubbing 的儲存空間或磁碟群組。
  4. 按一下管理。
  5. 選擇 Data scrubbing。
  6. 選擇想要執行的 data scrubbing 類型。(若只有一種類型可供選擇，系統會自動略過此步驟。)

注意：

  • Data scrubbing 僅能在儲存空間或磁碟群組狀態為正常的情況下執行。
  • 系統若在程序結束前關機，您需手動重新執行 data scrubbing。

____
* 僅適用於特定機型。

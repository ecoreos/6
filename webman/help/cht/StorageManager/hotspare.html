Hot Spare*

Hot spare 硬碟為備用硬碟，可自動更換損毀的硬碟來修復降級的儲存空間 / 磁碟群組 / iSCSI LUN。只要儲存空間 / 磁碟群組 / iSCSI LUN 符合下列條件，您就可將 hot spare 硬碟指定到 DiskStation 上的任何儲存空間 / 磁碟群組 / iSCSI LUN 來進行保護：

  • 儲存空間 / 磁碟群組 / iSCSI LUN 的 RAID 類型必須具有資料保護機制 (如：RAID 1、RAID 5、RAID 6、RAID 10)。
  • Hot spare 硬碟大小必須等於或大於儲存空間 / 磁碟群組 / iSCSI LUN 中最小的硬碟。

若要設定 hot spare 硬碟：

  1. 按一下管理。
  2. 在出現的視窗中，會顯示每個儲存空間 / 磁碟群組 / iSCSI LUN 的 hot spare 狀態以及可用硬碟。 儲存空間 / 磁碟群組 / iSCSI LUN 狀態包括下列：
    • 已指定：儲存空間 / 磁碟群組 / iSCSI LUN 受到至少一個已指定的 hot spare 硬碟保護。
    • 可用：有可用的 hot spare 硬碟可保護儲存空間 / 磁碟群組 / iSCSI LUN，但您尚未指定 hot spare 硬碟。
    • 未安裝：沒有可用的 hot spare 硬碟來保護儲存空間 / 磁碟群組 / iSCSI LUN。
  3. 可用硬碟區塊中顯示可以被指定為 hot spare 硬碟的硬碟。若要指定 hot spare 硬碟，請勾選欲作為 hot spare 的硬碟。
  4. 按一下套用來儲存設定。

注意：

  • 指定硬碟為 hot spare 硬碟後，所有在此硬碟上的資料都會消失。在您指定 hot spare 硬碟前，請確定硬碟上沒有重要資料。
  • SSD hot spare 硬碟無法自動修復 HDD 所組成的 RAID，反之亦然。

若要為儲存空間 / 磁碟群組 / iSCSI LUN 設定 hot spare：

  1. 按一下設定。
  2. Hot Spare 目標頁籤會列出所有可被 hot spare 硬碟保護的儲存空間 / 磁碟群組 / iSCSI LUN。若您取消勾選核取方塊，該儲存空間 / 磁碟群組 / iSCSI LUN 降級時，hot spare 硬碟將不會進行自動修復。
  3. 在跨裝置 Hot Spare 頁籤中，您可以允許位於一裝置上的 hot spare 硬碟修復位於另一裝置上降級的 RAID。
  4. 按一下確定來儲存設定。

* 僅適用於特定機型。

iSCSI LUN*

iSCSI 是一種可藉由區域網路或遠距離傳送資料的儲存空間網路服務。只要將 iSCSI LUN 連結至 iSCSI Target，用戶端便可以存取儲存空間，如同存取本機磁碟一樣。您可以將 iSCSI LUN (邏輯單元數) 連結至多個 iSCSI Targets 來進行檔案的讀寫。請前往儲存空間管理員 > iSCSI LUN 來管理 iSCSI LUN。

注意：

不同 DiskStation 機型支援的 iSCSI LUN 數不同。請參閱 www.synology.com 取得詳細產品規格訊息。

DiskStation 目前支援下列類型的 iSCSI LUN：

  • iSCSI LUN (一般檔案)：在一個儲存空間上劃分一塊空間而建立的 iSCSI LUN。您可在需要時擴充其容量。這類型的 iSCSI LUN 可額外提供具 Thin Provisioning 功能的彈性容量管理、VMware VAAI 硬體加速支援、Windows ODX 支援、LUN 快照和 LUN 複製功能。
  • iSCSI LUN (段落分塊) - 使用所有硬碟容量的 LUN：使用整個儲存空間建立而成的 iSCSI LUN。您可以修復、擴充或更改 RAID 類型，就像是一般儲存空間一樣。請參閱儲存空間頁面來瞭解詳細資訊。
  • iSCSI LUN (段落分塊) - 可彈性使用部份磁碟群組容量的 LUN：在一個磁碟群組上劃分一塊空間而建立的 iSCSI LUN。如果磁碟群組上還有可用的空間，您可依需要擴充 iSCSI LUN 的容量。請參閱磁碟群組頁面來瞭解詳細資訊。
  • iSCSI LUN (段落分塊)：在 RAID Group 上建立的 iSCSI LUN，提供動態容量管理以及最佳化存取效能。如果 RAID Group 上還有可用的空間，您可依需要擴充 iSCSI LUN 的容量。請參閱 RAID Group 頁面來瞭解詳細資訊。

注意：

具備進階 LUN 功能的 iSCSI LUN，目前僅支援於 ext4/Btrfs 檔案系統上使用。

若要新增 iSCSI LUN：

  1. 按一下新增。
  2. 選擇您想新增的 iSCSI LUN 類別。
  3. 跟隨精靈的指示來完成流程。

注意：

進階 LUN 功能僅可在創建 iSCSI LUN 時啟動。無法在事後進行變更。

您可以為某些應用程式伺服器功能，例如 Microsoft Exchange、SQL Server，或是為 Microsoft Windows ODX、VMware VAAI 等虛擬化伺服器和儲存加速技術，設定最佳化的進階 LUN 配置空間大小。請操作下列其中之一：

  • 在建立儲存空間的第一個進階 LUN 時，設定配置單元大小。
  • 在建立儲存空間的第一個進階 LUN 之前，在設定中設定配置單元大小。

在 DiskStation 上的 iSCSI 進階 LUN 從應用程式伺服器直接連線，或使用原始裝置對應 (RDM Pass-through) 模式時，建議您為應用程式設定適合的配置單元大小。若要利用 Synology 進階 LUN 技術搭配 Windows ODX 進行複製，建議您將 NTFS 儲存空間格式化為大於 (或等同於) 您在 DSM 上設定的配置單元大小。

注意：

每個進階 LUN 的配置單元大小只可設定一次。

若要移除 iSCSI LUN：

移除 iSCSI LUN 時，上面所有資料將會被刪除。iSCSI LUN 和 iSCSI Target 的連結也會遺失。

  1. 選擇您要移除的 iSCSI LUN。
  2. 按一下移除按鈕並跟隨精靈的指示來完成流程。

注意：

刪除 DiskStation 上的 VAAI LUN 並不會立即釋放硬碟空間。資料會被移至名為 @EP_trash 的系統保留資料夾，而後逐一刪除。例如，刪除 DiskStation 上 100 GB 的 VAAI LUN 後，大約需 15 到 20 分鐘來釋放此 100 GB 的硬碟儲存空間。

若要強制掛載 iSCSI LUN (段落分塊)：

若 iSCSI LUN (段落分塊) 上的快取遺失，系統開機時將不會掛載此 iSCSI LUN (段落分塊)。若快取無法復原，而您又希望存取此 iSCSI LUN (段落分塊)，您可以按一下強制掛載。

注意：

此功能僅適用於特定機種。

若要編輯 iSCSI LUN：

  1. 選擇您要編輯的 iSCSI LUN。
  2. 按一下編輯。
  3. 依需求編輯屬性。
  4. 按確定以儲存設定。

注意：

為避免編輯 iSCSI LUN 容量時造成檔案遺失，您只能將容量調整成比現在更大，而不能縮小。

若要拍攝 iSCSI LUN 快照：

iSCSI LUN 快照為 iSCSI LUN 在某一時間點的唯讀複本。

  1. 選擇您要拍攝快照的 iSCSI LUN。
  2. 按下快照並選擇拍攝快照。
  3. 輸入此快照的描述。
  4. 根據您的需求，選擇以下任一快照類型：
    • 應用程式一致：只有在您的 Windows 伺服器或 VMware vCenter 伺服器 (根據您用來連接 DSM 的操作環境) 上已安裝 Synology Snapshot Manager 時可拍攝此類型快照。DSM 觸發應用程式一致性快照時，應用程式會接到通知，並將所有記憶體資料排清到要拍攝快照的 LUN 上，以保持資料一致性。
    • 損毀一致：若您的用戶端尚未安裝 Synology Snapshot Manager，您將只能拍攝此類型的 LUN 快照。觸發損毀一致性快照時，將不會通知應用程式將記憶體的資料排清至 LUN，而無法確保資料一致。
  5. 按一下確定。

若要還原 iSCSI LUN：

選擇某一時間點的快照，將 iSCSI LUN 還原為當時的資料狀態。

  1. 選擇您要還原的 iSCSI LUN。
  2. 按下快照並選擇快照管理員。
  3. 選擇您要還原至哪份快照，並按下還原。

若要複製 iSCSI LUN 或快照：

為 iSCSI LUN 或 iSCSI LUN 快照建立可讀寫的複本。

  • 從清單中選擇 iSCSI LUN 並按一下複製。
  • 從快照管理員中選擇 iSCSI LUN 快照並按一下複製。

注意：

  • 針對特定機種，您須前往套件中心並安裝快照 & 複寫套件以取得 iSCSI LUN 還原、iSCSI LUN 快照拍攝及複製功能。
  • 如有任何 Windows Server 或 VMware ESXi Server 所使用的、已連接的 iSCSI 連線與您的 LUN 相關，為避免資料不一致，在您繼續之前，建議您安裝 Synology Snapshot Manager 外掛程式，否則您必須從啟動器中登出，並停止所有相關的應用程式。
  • 具備進階 LUN 功能的 iSCSI LUN 才能提供 LUN 快照、還原及複製功能。
  • 如果您在拍攝快照或建立複本後，發現其狀態為異常 / 已損毀，請刪除此快照 / 複本並重新建立。
  • 當您將 LUN 還原至某快照的時間點時，在此快照建立後您對 LUN 所作的變更將會全部遺失。
  • 若 iSCSI LUN 狀態變更為無法使用，表示在存取過程中發生資料錯誤。為了避免更嚴重的資料錯誤或遺失，暫時無法存取此 iSCSI LUN。請儘快聯繫 Synology 以取得技術支援。

詞彙解釋：

  • Thin Provisioning：Thin Provisioning (資源隨需分配) 是視需求、機動性地分配儲存空間以達到儲存使用優化的一種方式。這個設定只適用於 iSCSI LUN (一般檔案)。
  • VMware VAAI：支援 vStorage APIs for Array Integration (VAAI) 可讓 iSCSI 裝置分擔 VMware vSphere 環境中虛擬儲存陣列標準操作的工作量，並最佳化儲存空間效能。
  • Windows ODX：Windows 在 Windows Server 2012 及 8 新開發的資料傳輸技術，稱作卸載式資料傳輸 (Offload Data Transfer，ODX)。ODX 可讓儲存陣列分擔一些工作量，提升傳輸效能。
    若資料傳輸的來源與目的地 LUN 位於同一個儲存空間，複製過程會透過 Synology 進階 LUN 技術來處理，節省至多 99.9% 的儲存空間消耗，並加速複製的過程。
  • iSCSI Target 連結：將 iSCSI LUN 連結至一個或多個 iSCSI Targets。
  • Synology Snapshot Manager：Synology Snapshot Manager 是一款外掛程式，可讓您直接從 DiskStation Manager (DSM) 拍攝與應用程式一致的快照。
    • 在 VMware 環境下，您必須於已安裝 VMware vCenter Server 的 Windows 伺服器上安裝 Synology Snapshot Manager。DSM 觸發快照時，VMware vCenter Server 會接到通知，並將所有記憶體資料排清到要拍攝快照的 LUN 上，以保持資料一致性。快照拍攝完畢後，VMware vCenter Server 會繼續進行 VMware 資料存放區的正常 I/O 操作。適用於 VMware vCenter Server 的 Synology Snapshot Manager 支援 vSphere 5.1 或以上版本。
    • 在 Windows 環境下，您必須於 Windows 伺服器上安裝 Synology Snapshot Manager。DSM 觸發快照時，Synology Snapshot Manager 會運用 Microsoft Volume Shadow Copy Service (VSS) 技術來產生一致的資料時間點複本。適用於 Windows 的 Synology Snapshot Manager 支援 Windows Server 2008 R2、2012 及 2012 R2。
    • 您可於 www.synology.com 免費下載 Synology Snapshot Manager。

共用資料夾同步

共用資料夾同步服務可讓使用者將共用資料夾從一台 Synology 產品 (來源) 同步至區域網路或網際網路上的其它 Synology 產品 (目的地) 上。本功能是以共用資料夾為作業基礎。同步時，若目的地 Synology 產品已存在相同名稱的資料夾 (例如 SharedFolder)，則目的地端資料夾會被重新命名為「SharedFolder_1」。若尚未有同名資料夾，此功能會於對應的儲存空間或第一個可用的儲存空間中新增該資料夾。該資料夾的存取權限將會按照 DiskStation 預設，即未指定任何存取權限。

第一次進行同步時，共用資料夾同步功能會將所選的共用資料夾進行完整的同步。之後則會僅同步檔案修改過的部分。在來源端移除的檔案，在目的端也會被移除。

注意：

  • 共用資料夾同步是單向的同步服務。只有來源端的檔案變動會同步到目的端，相反則無。若要雙向同步功能，請使用 Cloud Station。

啟動服務

若要啟動共用資料夾同步：

您必須在目的地 DiskStation 上啟動此服務。請前往備份 & 同步 > 共用資料夾同步。在伺服器頁籤中，勾選啟動共用資料夾同步服務來啟動此功能。

若要將共用資料夾從來源端 Synology 產品連到目的地端 Synology 產品：

您可以將共用資料夾從一台來源端伺服器同步至多台不同的目的地端伺服器。此外，從 DSM 4.1 起，共用資料夾同步提供多來源共用資料夾同步功能，可讓多台用戶端同步共用資料夾至一台 DiskStation 上。若不同用戶端欲同時同步相同的共用資料夾至一台 DiskStation 上，會造成衝突，您必須先取消連結共用資料夾，其它來源端才能同步此資料夾到目的地端。若您想取消連結共用資料夾，請在目的地端前往共用資料夾同步 > 伺服器，按一下連線清單，選擇一個連線，然後按取消連結。

現在您可以開始將共用資料夾從來源端 DiskStation 同步到目的地端 DiskStation。請依照以下步驟繼續進行。

管理同步任務

透過共用資料夾同步精靈，您可輕鬆建立同步任務。

若要新增同步任務：

  1. 前往備份 & 同步 > 共用資料夾同步 > 用戶端。
  2. 按一下新增。
  3. 輸入任務的名稱。
  4. 選擇您要同步至目的地的共用資料夾。勾選下方的核取方塊以示確認，並按一下下一步以繼續。
  5. 設定目的地及下列同步選項：
    • 啟動傳輸加密：傳輸資料時進行資料加密。提供較高的安全性，但非加密傳輸的效能較佳。您可根據需要選擇。
    • 啟動傳輸壓縮：傳輸資料時進行資料壓縮。可節省頻寬，但會增加 CPU 負擔。
    • 啟動段落分塊 (block-level) 同步化：僅同步資料修改的部分，而非整個檔案。可節省頻寬，但會增加 CPU 負擔。
  6. 設定任務的排程：
    • 執行同步 每隔幾小時或執行同步 每隔幾分鐘：每隔特定數分鐘或數小時執行同步任務。時間由午夜開始計算，最大間隔不可超過 24 小時。
    • 每天自動執行同步於(小時:分)：每天特定時間執行同步任務。
    • 手動執行同步化：只有在手動執行時執行同步任務。
    • 資料異動時自動進行同步：來源共用資料夾的資料有改變時立即執行同步任務。
  7. 按一下套用。現在您可以在任務列表上看到同步任務，系統會根據所指定的排程執行任務。

若要管理同步任務：

  1. 在任務列表上選擇該任務。
  2. 執行下列任何操作：
    • 按一下編輯來編輯任務。
    • 按一下刪除來移除任務。
    • 如果同步任務尚未進行，請按一下立即同步來立即執行任務。
    • 如果同步任務正在進行，請按一下取消來停止正在進行的任務。

      • 注意：

      如果同步任務的排程是設定為資料異動時自動進行同步，按一下取消會停止進行中的同步任務。然而，如果同步任務監控的任何資料夾內容有發生異動，共用資料夾同步會繼續進行同步任務。

    • 按一下完整同步來同步所有的資料。共用資料夾同步第一次執行同步任務時會執行完整同步，並在第一次同步完成之後僅同步有修改的部分。按一下此按鈕可協助您以手動方式再度同步所有的資料。

更多資訊

  1. 任務名稱必須設定為 1 到 32 個 Unicode 字元，但不可為下列字元：
    !" # $ % & ' ( ) * + , / : ;<= >?@ [ ] \ ^ ` { } | ~
  2. 若您欲將 ext4 檔案系統的共用資料夾同步至 ext3 的檔案系統，同步任務的資料夾數量若超過 32,000 個時可能會發生錯誤。
  3. 若目的地端 DiskStation 上的共用資料夾曾與其它 DiskStation 進行同步，共用資料夾同步在進行任務時會執行完整的同步。
  4. 若要執行共用資料夾同步任務，您必須擁有目的地 DiskStation 的管理權限。您必須擁有 admin 帳號的密碼，或為管理群組內的一員。
  5. 當您透過網際網路同步至 DiskStation 時，您可在伺服器名稱欄位輸入其外部 IP 位址或主機名稱來連線。然而，若您選擇以主機名稱來連線，此主機名稱必須符合 FQDN (Full Qualified Domain Name) 格式，例如「www.synology.com」。
  6. 針對連接埠轉送，共用資料夾同步於加密傳輸時使用 22 埠，非加密傳輸時使用 873 埠。
  7. 共用資料夾同步不支援外接儲存裝置。
  8. 已加密的共用資料夾需要於目的地端手動使用加密金鑰掛載。
  9. 若所選的共用資料夾沒有任何變更，同步排程將不會在指定的時間執行。
  10. 為了達到最佳使用效能，建議每個同步任務中不要包含超過 200 個共用資料夾。
  11. 若來源端 DiskStation 將 homes 共用資料夾同步至目的地端 DiskStation，目的地端 DiskStation 的使用者家目錄功能會被停用。
  12. 針對搭載非 Intel 處理器的機種，您可以使用共用資料夾同步來同步最多 100,000 個共用資料夾 (包含子資料夾)。
  13. 共用資料夾同步在目的地資料系統為 ext4 時可備份 Windows ACL 設定。
  14. 若目的地端伺服器名稱和您的 DiskStation 相同，共用資料夾同步將會失敗。
  15. 來源進階共用資料夾權限將不會被同步至目的地。系統會在目的地自動設定對管理群組內的使用者的 read-only 權限。

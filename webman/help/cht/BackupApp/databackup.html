備份

備份 & 還原讓您可在 DiskStation 上手動或排程備份各種類型資料 (系統設定、共用資料夾、應用程式 / 套件)。您可以在本地共用資料夾、遠端伺服器、公共雲端空間 (Amazon S3、Microsoft Azure、hicloud S3 及 SFR NAS BACKUP) 儲存備份資料。透過備份 & 還原，各項備份任務皆可保留多備份版本，並透過 Smart Recycle 自動刪除。

資料備份

備份 & 還原提供多種備份目的地選項，以及設定資料備份任務的方式。

若要新增資料備份任務：

  1. 按一下左下角的 +，選擇資料備份任務來啟動備份精靈。
  2. 選擇希望的備份目的地類型：
    • 本地 / 遠端共用資料夾或外接儲存裝置
    • 遠端 rsync 伺服器
    • 公共雲端空間 (例如 Amazon S3 及 Microsoft Azure)
  3. 設定備份任務類型：
    • 若要建立全新任務，選擇建立備份任務。
    • 若要重新使用現存在任何目的地的備份資料，選擇重新連結至現存任務。重新連結可讓您直接利用不同任務的備份資料。
  4. 跟隨精靈指示完成步驟。選擇啟動用戶端加密來在傳輸前加密資料，便可保障資料在備份目的地的安全。

注意：

  • 任務名稱必須設定為 1 到 32 個 Unicode 字元，但不可為下列字元：
    ！ " # $ % & ' ( ) * + , / : ; < = > ? @ [ ] \ ^ ` { } | ~
  • 備份的完整資料夾路徑，最大長度限制為 2048 個字元。
  • 如果您透過 AFP 將超過 32000 個檔案或資料夾上傳到 EXT3 共用資料夾，則所有備份任務 (包含該共用資料夾) 將會失敗。
  • 加密共用資料夾掛載時可以備份，而且可以還原至加密 / 未加密資料夾。然後，資料夾只能被還原成未加密狀態，並且無法在開機時自動掛載。若要掛載共用加密資料夾，需要輸入或匯入該資料夾的加密金鑰。
  • 系統不支援將加密資料夾備份至另一個加密資料夾。
  • 系統不支援使用遠端資料夾作為備份來源或備份目的地。
  • 系統支援備份特定的套件。關於套件是否支援備份，以及備份的具體內容與限制，請參閱應用程式內各套件的說明。
  • 系統不支援 DiskStation 機種在沒有可寫入的 EXT3 或 EXT4 外接裝置的情況下備份應用程式。
  • DiskStation 機種僅能備份 EXT3 或 EXT4 外接裝置上的共用資料夾。

備份 Windows ACL 設定：

DiskStation 上檔案的 Windows ACL 設定可能也會一併備份 (視目的地檔案系統而定)：

  • 所有備份目的地皆可備份 Windows ACL 設定，不受限於目的地檔案系統。

若要匯出備份資料至公共雲端空間或遠端 Synology NAS：

備份大量資料至公共雲端空間時，建議您將最初備份版本匯出至外接儲存裝置 (例如 USB 硬碟)，寄送該裝置至雲端服務供應商。裝置便可在該處連接並直接上傳檔案至雲端空間。如此一來，便可大量減少第一次備份時間。

  1. 按一下左下角的+，選擇遠端 Synology NAS或想要的雲端服務提供商。
  2. 選擇匯出至本地共用資料夾 (包含外接儲存裝置)來完成設定。出現提醒時，選擇啟動用戶端加密來保護資料備份。
  3. 寄送外接儲存裝置至遠端 Synology NAS 或雲端服務提供商。從外接儲存裝置匯出資料。
  4. 確認後，請回到備份 & 還原，按一下立即重新連結。
  5. 依照重新連結精靈的指示來將 DiskStation 連結至資料庫。如果資料匯出時處於加密狀態，則需密碼 / 加密金鑰。

注意：

  • 一旦任務完成設定，用戶端任務加密無法被停用也無法再編輯密碼。
  • 重連已存在任務後，原任務則無法再進行備份。
  • 備份的來源資料夾，若是加密共用資料夾或是其子資料夾，在建立任務時可為 (未) 掛載狀態。
  • 您無法將加密共用資料夾選為備份目的地。
  • 如果選擇啟動用戶端加密，還原任務資料時需密碼 / 加密金鑰。若忘記密碼且遺失加密金鑰檔，資料將無法還原。

若要管理備份任務：

選擇左側面版的備份任務，按一下下列按鈕來執行相對操作：

  • 立即備份：立即備份所選任務。
  • 備份瀏覽器 (放大鏡圖示)：檢視所有可用的備份版本，也可複製 / 還原 / 下載內部檔案。
  • 版本清單：檢視所有備份任務的建立時間及狀態，刪除特定備份版本，或鎖定特定版本進行強制保存。
  • 編輯：修改所選任務的目的地、來源、排程及其它設定。
  • 刪除：刪除備份任務。
  • 取消：取消所選之正在執行的任務。

若要自動刪除版本：

自動刪除備份版本為版本管理的機制。它可有效的保管備份並避免您的儲存空間被過多的備份檔案佔據。您可以在任務設定頁面或在建立被分任務時啟動此功能。

  1. 在左側面版選擇備份版本，按一下編輯或設定 > 任務設定。
  2. 選擇自動刪除備份版本，選擇想要的自動刪除模式：
    • 從最早版本開始刪除：可用版本超過訂定的數量時，刪除最早備份版本。
    • Smart Recycle：依下列方式保存備份版本：
      • 過去 24 小時內，每小時保留一個版本：保留每小時最早建立的備份版本。
      • 過去 1 天至 1 個月內，每天保留一個版本：保留每天最早建立的備份版本。
      • 過去 1 個月之前，每週保留一個版本：保留每週最早建立的備份版本。
  3. 指定最高的版本數量。當版本數量超過該數字，則自動刪除最早版本。
  4. 如果在排程設定重複的備份排程，便可預覽備份版本未來保留狀況，以及時間軸上最早的還原點。
  5. 按一下確定以儲存設定。

注意：

  • 鎖定的版本不會被刪除。
  • 只有多版本備份任務提供自動刪除備份版本功能。
  • 備份目的地為 DSM 5.1 以下的系統無法使用自動刪除版本功能。
  • 備份任務設定改變後，自動刪除功能將立刻開始執行。
  • 系統計算最高版本數量時將不包含鎖定的版本。

資料備份任務注意事項

針對各類型備份任務，請詳閱下列注意事項來瞭解重要的設定細節及疑難排解方法。

本地備份任務：

  • 如果在備份任務進行時移除目的地端的資料，下一次執行備份任務時將會執行完整備份。
  • 新增的外接硬碟可能需要先進行格式化才可作為備份目的地。
  • 建立備份任務之前，確認外接硬碟是否已正確地連結至 DiskStation。前往控制台 > 外接裝置來檢查 USB/eSATA 硬碟狀態。如果外接硬碟剛剛插入、正在執行格式化或已故障，您無法將其選為備份目的地。
  • 您無法將遠端掛載的共用資料夾選為備份來源或目的地。
  • 如果加密共用資料夾已卸載，您無法將其選為備份目的地。
  • 如果目的地的檔案系統為 NTFS / FAT，您無法備份影像縮圖。

遠端 rsync 備份任務：

  • 備份檔案至 rsync 相容伺服器時，可選擇的模式如下：當您從備份模組旁的下拉式選單中選擇備份模組 (rsync 相容伺服器上的共用資料夾)，系統會使用 rsync daemon 模式；只有當您在備份模組中輸入絕對路徑 (以「/」開頭) 時，系統才會啟動 remote shell 模式。
  • 在 Linux 平台上，只有 admin、root 和 rsync 帳號可以執行備份任務。
  • 若您想要透過網際網路使用主機名稱存取目的地伺服器，則其主機名稱必須符合 FQDN (Fully Qualified Domain Name，完全合格的網域名稱) 格式，例如 abc.bcd.com。
  • 從一台 Synology NAS 裝置備份至另一台 Synology NAS 時，系統會要求您輸入一組目的地伺服器上的使用者帳號名稱及密碼。備份至另一台 DiskStation 上的共用資料夾時，您可以依資料夾存取權限設定來備份共用資料夾。畫面上只會顯示該使用者帳號具有存取權限的共用資料夾。若無法在目的地伺服器上找到您想要的共用資料夾，請檢查您的存取權限。
  • 若要使用傳輸加密，則目的地伺服器必須支援加密備份功能且使用者認證的密碼不得為空白。
  • Remote shell 模式只支援傳輸加密。
  • 若您使用 rsync 用戶端備份至 Synology NAS 裝置時，檔案名稱要以 UTF-8 編碼。
  • 若您使用 rsync 相容備份軟體備份至 Synology NAS 伺服器，則僅有 DiskStation 的 admin、root、rsync 帳號使用者可執行備份。
  • 若您使用 DiskStation 備份至 rsync 相容伺服器，您需要填寫正確的 rsync 備份帳號及密碼。您同時也需要選擇有寫入權限的備份模組。

Amazon S3 備份任務：

  • 建立備份任務至 Amazon S3 備份目的地時，可選擇的項目包含傳輸加密、備份至 Amazon S3 RＲS (少冗餘儲存空間) 等。請注意某些選項 (例如傳輸加密) 可能會減慢備份任務的速度。
  • 已備份的檔案會儲存在 Amazon S3 伺服器上您選擇的 Bucket 中，而 set Key 值的開頭為您所選的目錄名稱，結尾為該檔案的完整路徑。
  • Amazon S3 的資料架構為一組 Bucket 和 KEY，並不包含資料夾架構。 目前並不支援空資料夾的備份。
  • 如果所備份的檔案沒有修改，則不會再備份到 Amazon S3 伺服器上的同一個 Bucket 中。
  • Amazon S3 上每個 Bucket 的名稱不能重複。 此外，名稱必須：
    • 介於 3 至 63 個字元。
    • 只能包含小寫英文字母、數字、小數點 (.) 及破折號 (-)。
    • 開頭必須是數字或是小寫英文字母，結尾不可為破折號 (-)、不能含有兩個相鄰的小數點，或在小數點旁加上破折號 (例如「my-.bucket.com」和「my.-bucket」為無效格式)。
  • Amazon S3 KEY 值的長度上限為 1024 個字元。若要確保 Amazon S3 備份任務能順利執行，路徑的長度上限為：1024 減去建立任務時所選的目的地資料夾名稱的長度。
  • 如果您啟動 Amazon S3 RRS 支援，只有在此選項啟動之後修改的資料才會備份到 RRS 儲存空間上，也就是說，先前已備份至 Amazon S3 標準儲存空間的資料並不會再重新備份至 RRS。
  • 不支援將已備份的檔案從 Amazon S3 移至 Amazon Glacier。如果透過 AWS 管理控制台來將 Amazon S3 的檔案移至 Amazon Glacier，則相關的備份任務可能會失敗。
  • 不支援透過 SSL 備份檔案至虛擬主機類型的 bucket。
  • 在傳輸檔案的過程中，會將檔案切割成數個區塊分開傳輸。每個區塊大小上限可透過 Multipart upload 檔案分割大小 設定，目前預設為 512MB。如果遇到網路較為不穩定的環境，建議將 Multipart upload 檔案分割大小調小。
  • 上限為 10000 個區塊。檔案大小上傳上限為 Multipart upload 檔案分割大小 乘上 10000。
  • 檔案名稱不能包含非 UTF-8 字元及控制字元。

Microsoft Azure 備份任務：

  • 建立備份任務至 Microsoft Azure 備份目的地時，可選擇多種的項目 (例如傳輸加密)。請注意某些選項 (例如傳輸加密) 可能會減慢備份任務的速度。
  • 已備份的檔案會儲存在 Microsoft Azure 伺服器上您選擇的 Container 中，而 set blob 值的開頭為您所選的目錄名稱，結尾為該檔案的完整路徑。
  • Microsoft Azure 的資料架構為一組 Container 和 Blob，並不包含資料夾架構。目前並不支援空資料夾的備份。
  • 如果所備份的檔案沒有修改，則不會再備份到 Microsoft Azure 伺服器上的同一個 Container 中。
  • Microsoft Azure 上每個 Container 的名稱不能重複。此外，名稱必須：
    • 介於 3 至 63 個字元。
    • 只能包含小寫英文字母、數字、小數點 (.) 及破折號 (-)。
    • 開頭必須是數字或是小寫英文字母，結尾不可為兩個破折號 (-)。
  • Microsoft Azure Blob 名稱值的長度上限為 1024 個字元。若要確保 Microsoft Azure 備份任務能順利執行，路徑的長度上限為：1024 減去建立任務時所選的目的地資料夾名稱的長度。
  • 檔案名稱不能包含非 UTF-8 字元及控制字元。
  • 當發生認證資訊錯誤的警告訊息時，可能是因為認證資訊錯誤，網路連線出錯，或是 DiskStation 時間與遠端伺服器不符。如欲確認時間是否設定正確，請前往控制台 > 區域選項 >時間，來同步 DiskStation 與 NTP 伺服器的時間。

hicloud S3 備份任務：

  • 建立備份任務至 hicloud S3 備份目的地時，可選擇的項目包含傳輸加密等。請注意某些選項 (例如傳輸加密) 可能會減慢備份任務的速度。請注意某些選項 (例如傳輸加密) 可能會減慢備份任務的速度。

    • 開頭必須是數字或是小寫英文字母，結尾不可為破折號 (-)、不能含有兩個相鄰的小數點，或在小數點旁加上破折號 (例如「my-.bucket.com」和「my.-bucket」為無效格式)。

  • Hicloud 每次傳輸只能分成 1000 個區塊。也就是說，檔案總共的大小不能超過您設訂的區塊大小的 1000 倍。例如 Multipart Upload 分割大小設為 8MB 時，超過 8000MB 的檔案將不會被上傳。

  • 檔案名稱不能包含非 UTF-8 字元及控制字元。

其他 S3 相容備份任務：

  • 若要使用其他 S3 相容雲端空間，在 S3 伺服器輸入 S3 相容雲端空間的 IP 位址 / 網域名稱。
  • 如果遭遇任何使用問題，請參考相關文件或聯絡 S3 相容雲端空間服務提供商。

LUN 備份*

可使用本地 LUN 備份來備份 DiskStation 上的 iSCSI LUN 至本地共用資料夾。也可使用網路 LUN 備份 (Synology 伺服器)，便可備份至其他 DiskStation。

若要備份 iSCSI LUN：

  1. 按一下左下角的 +，選擇 LUN 備份任務。
  2. 當畫面上出現提示要求您選擇目的地類型時，請選擇以下其一：
    • 本地 LUN 備份：選擇此選項來將 LUN 從 DiskStation 備份至本機共用資料夾或 DiskStation 的 USB 或 eSATA 埠上連接的外接硬碟。
    • 網路 LUN 備份 (Synology 伺服器)：選擇此選項來將 LUN 從 DiskStation 備份到另一部 DiskStation。
  3. 出現提示時，指定備份目的地以及 LUN 來源，並設定備份排程。
  4. 依照設定精靈的指示完成設定。

建立備份任務後，DiskStation 會依排程執行備份任務，並將已備份的 LUN 儲存至目的地。您所備份的 LUN 僅供備份目的之使用。若要存取已備份的 LUN，請參閱若要還原所備份的 LUN 來取得更多資訊。

注意：

  • 一般檔案及段落分塊 LUN 皆會以檔案映像格式備份在目的地。若將 LUN 還原，段落分塊 LUN 將變更為一般檔案 LUN。
  • 每個備份任務的備份來源只能包含一個 LUN。
  • 您無法將 LUN 備份至 FAT、FAT32、HFS、HFS Plus 或 NTFS 格式的外接硬碟。
  • 執行網路 LUN 備份之前，請先登入目的地 DiskStation，然後確認其網路備份服務已啟動 (位於控制台 > 備份服務)。
  • 如果您的 DiskStation 是透過路由器連線至網路，在開始執行網路 LUN 備份之前，請先確定您已在路由器上為 iSCSI 服務 (TCP 3260 埠) 指定連接埠轉送規則。

關於完整備份及增量備份：

初次執行備份任務時，DiskStation 會為所選的 LUN 執行完整備份，並在之後執行增量備份。然而，如果備份目的地的共用資料夾名稱或目錄名稱有所變更，DiskStation 會再度為此任務執行完整備份。

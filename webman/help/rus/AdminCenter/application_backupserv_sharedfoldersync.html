Синхронизация общих папок

Служба синхронизации общих папок позволяет синхронизировать общие папки с одного устройства Synology NAS (источник) с другими устройствами Synology NAS (целевые) в локальной сети или через Интернет.

При первой синхронизации служба синхронизации общих папок выполняет полную синхронизацию выбранных папок. В дальнейшем синхронизироваться будут только измененные файлы. Файлы, которые были удалены из источника, удаляются также с целевых устройств.

Примечание:

  • Синхронизация общих папок представляет собой способ односторонней синхронизации файлов между исходным устройством и целевыми устройствами. Для двусторонней синхронизации используйте приложение Cloud Station ShareSync.
  • Если имя общей папки на целевом устройстве (например, SharedFolder) совпадает с именем общей папки-источника, папке назначения присваивается имя "SharedFolder_1".
  • Если такой папки нет на целевом устройстве, она будет создана с указанным именем в соответствующем разделе или первом доступном разделе.
  • По умолчанию у пользователей нет прав доступа к папке общего доступа на целевом устройстве. Для получения прав доступа обратитесь к администратору (admin) целевого устройства.

Включение службы

Включение синхронизации общих папок:

Сначала необходимо включить службу на целевом устройстве Synology NAS.

  1. Откройте Панель управления > Файловая служба > rsync.
  2. Выберите Включить службу rsync, чтобы включить службу синхронизации папки общего доступа.
  3. Нажмите Применить.

Отключение общих папок от целевого устройства Synology NAS:

Если разные источники одновременно выполняют синхронизацию одной и той же папки общего доступа с одним целевым устройством DiskStation, это вызовет конфликт. Потребуется удалить связь с общей папкой на стороне источника, чтобы другой источник мог выполнить ее синхронизацию с целевым устройством.

  1. Выполните вход в целевое устройство Synology NAS и выберите Панель управления > Синхронизация общих папок > Список подключений.
  2. Выберите конфликтующее подключение исходного устройства Synology NAS и выберите Удалить связь, чтобы удалить его.

Теперь другой источник может выполнить синхронизацию папки общего доступа с целевым устройством DiskStation.

Управление задачами синхронизации

Чтобы создать задачу синхронизации:

  1. Выполните вход в исходное устройство Synology NAS и выберите Панель управления > Синхронизация общих папок > Список задач.
  2. Нажмите Создать.
  3. Введите имя для задачи.
  4. Выберите общую папку, которую требуется синхронизировать с устройством назначения.
  5. Укажите целевое устройство Synology NAS и следующие параметры синхронизации:
    • Настроить порт шифрования SSH для синхронизации защищенной папки общего доступа: Укажите нужный порт шифрования для зашифрованной передачи SSH.
    • Включить защиту при передаче SSH: Шифрование данных при передаче. Этот параметр обеспечивает лучшую безопасность, в то время как нешифруемая передача более производительна. Вы можете выбрать параметр в соответствии со своими нуждами.
    • Включить сжатие при передаче: Сжатие данных при передаче. Этот параметр экономит пропускную способность сети, но повышает нагрузку на центральный процессор (CPU).
    • Включить синхронизацию на уровне блоков: Передача только отличающихся данных, вместо целых файлов. Этот параметр экономит пропускную способность сети, но повышает нагрузку на центральный процессор (CPU).
  6. При отображении запроса выберите следующие варианты, указав, когда необходимо выполнять синхронизацию с исходного на целевое устройство:
    • Запустить синхронизацию изменений: Запускать синхронизацию сразу после любых изменений в исходной общей папке.
    • Запустить синхронизацию вручную: Запускать задачу синхронизации вручную из общей папки-источника.
    • Детальное расписание: Нажмите Запланировать расписание, чтобы указать время выполнения задачи синхронизации.
  7. Нажмите Применить. Теперь вы увидите задачу синхронизации в списке задач, которые будут выполняться в соответствии с установленным расписанием.

Управление задачей синхронизации:

  1. На исходном устройстве Synology NAS выберите Панель управления > Синхронизация папки общего доступа, нажмите Список задач и выберите нужную задачу.
  2. Выполните одно из следующих действий:
    • Нажмите Изменить, чтобы изменить задачу.
    • Нажмите Удалить, чтобы удалить задачу.
    • Если задача синхронизации сейчас не выполняется, то нажмите Синхронизировать сейчас, чтобы выполнить эту задачу прямо сейчас.
    • Если задача синхронизации сейчас выполняется, то нажмите Отмена, чтобы остановить текущую задачу.

      • Примечание:

      Если для расписания задачи синхронизации выбрано значение Запустить синхронизацию изменений, то нажатие кнопки Отмена приведет к остановке текущей задачи синхронизации. Но если было изменено содержимое общих папок, контролируемых задачей синхронизации, то функция "Синхронизация общих папок" возобновит выполнение этой задачи.

    • Чтобы синхронизировать все данные, нажмите Полная синхронизация. Функция "Синхронизация общих папок" в первый раз проведет полную синхронизацию, а после первой синхронизации будет синхронизировать только измененные части. Нажмите эту кнопку, чтобы снова вручную синхронизировать все данные.

Дополнительная информация

  1. Название задачи должно состоять из 1–32 символов Unicode и не должно содержать следующие символы:
    ! " # $ % & ' ( ) * + , / : ; <= > ? @ [ ] \ ^ ` { } | ~
  2. Если задача включает более 32 000 папок, то при синхронизации общих папок из файловой системы ext4 в ext3 возможны ошибки.
  3. Если общие папки на целевом устройстве DiskStation были синхронизированы с другим устройством DiskStation ранее, синхронизация общих папок выполнит полную синхронизацию задачи.
  4. Для выполнения задач синхронизации общих папок необходимы права администратора (admin) для доступа к целевому устройству DiskStation. Пользователь должен иметь пароль учетной записи администратора (admin) либо быть членом группы администраторов этого устройства.
  5. При синхронизации на целевом устройстве DiskStation через Интернет можно ввести имя сервера либо в виде его внешнего IP-адреса, либо его имени хоста. Однако если вводить имя хоста, это имя должно быть в формате полного имени домена (FQDN), например "www.synology.com".
  6. Для переадресации портов служба синхронизации общих папок использует порт 22 для шифруемой передачи и порт 873 для нешифруемой передачи.
  7. Служба синхронизации общих папок не поддерживает внешние устройства хранения.
  8. При выполнении задач синхронизации общих папок зашифрованные общие папки в месте назначения будут автоматически подключаться. Однако для доступа к данным, находящимся внутри папок, сначала потребуется подключить зашифрованные общие папки в месте назначения.
  9. Запланированная синхронизация общих папок не будет выполнена в указанное время, если в выбранную папку не были внесены изменения.
  10. Для достижения наилучшей производительности рекомендуется включать в каждую задачу синхронизации не более 200 папок.
  11. Функция Главная папка пользователя будет отключена на целевом устройстве DiskStation, если папка homes синхронизирована с исходного устройства DiskStation на целевое устройство DiskStation.
  12. Для моделей с процессорами, отличными от Intel, можно синхронизировать до 100 000 общих папок (включая подпапки) с помощью службы синхронизации общих папок.
  13. Синхронизация общих папок позволяет синхронизировать настройки Windows ACL, если файловая система на целевом устройстве — ext4.
  14. Синхронизация общих папок не будет выполнена, если пытаться синхронизировать данные с целевым сервером, имеющим такое же имя, что и устройство DiskStation.
  15. Расширенные права доступа к общим папкам исходного устройства не будут синхронизированы с целевым устройством. Кроме того, на целевом устройстве для группы administrators будет установлен статус "только для чтения".

Link Aggregation*

DiskStation поддерживает несколько локальных сетей и позволяет объединять интерфейсы сетей с помощью технологии Link Aggregation. Link Aggregation увеличивает пропускную способность DiskStation с помощью объединения нескольких сетевых интерфейсов и обеспечивает переключение трафика для сохранения сетевого соединения в случае его потери.

После объединения интерфейсов локальных сетей новый интерфейс Bond отобразится в списке Панель управления > Сеть > Сетевой интерфейс, который можно настроить точно так же, как и другие интерфейсы (например, локальная сеть или PPPoE).

Примечание.

  • При активации Link Aggregation сетевой трафик будет автоматически настроен и сбалансирован для каждого подключенного устройства.
  • Если сетевые интерфейсы подключены к нескольким коммутаторам, убедитесь, что порты коммутаторов находятся в одной локальной сети или сети VLAN.

Объединение нескольких локальных сетей с помощью Link Aggregation:

  1. Выберите Панель управления > Сеть > Сетевой интерфейс. Щелкните Создать > Создать Bond.
  2. Выберите необходимый режим.

    Описание режимов:

    • Адаптивное распределение нагрузки (Adaptive Load Balancing): Этот режим оптимизирует сетевой трафик, принимаемый и отправляемый DiskStation, независимо от того, поддерживает ли коммутатор Link Aggregation или нет. Чтобы избежать непредвиденных ошибок, не включайте Link Aggregation на коммутаторе, даже если эта функция поддерживается им.
    • Динамический режим IEEE 802.3ad Link Aggregation: Этот режим оптимизирует сетевой трафик, принимаемый и отправляемый DiskStation и требует включения на коммутаторе (коммутаторах) динамического режима IEEE 802.3ad Link Aggregation (LACP, 802.1AX). Если используется несколько коммутаторов, они должны быть наращиваемыми и правильно настроенными.
    • Распределение XOR: Этот режим позволяет сбалансировать сетевой трафик, принимаемый и отправляемый DiskStation, и требует включения на коммутаторе (коммутаторах) статического режима Link Aggregation. Если используется несколько коммутаторов, они должны быть наращиваемыми и правильно настроенными. Если Link Aggregation не поддерживается коммутатором, трафик, отправляемый вашим устройством DiskStation, все равно можно сбалансировать. При помощи этого режима можно выявить потерю соединений, но не причину (например, отсоединение кабелей Ethernet или неправильная настройка коммутатора).
    • Активно/Режим ожидания: Этот режим обеспечит DiskStation отказоустойчивость сети (network fault tolerance). То есть, в случае сбоя активного сетевого интерфейса подключение к сети обеспечит один из других портов локальной сети. Можно выбрать этот режим без настройки коммутатора или при помощи коммутатора, поддерживающего IEEE 802.3ad Link Aggregation.
  3. Выберите интерфейсы для создания Link Aggregation.
  4. Настройте параметры IP. Здесь также можно включить VLAN. Нажмите кнопку Применить для завершения настройки.
  5. После завершения отобразится новый интерфейс Bond 1 в списке Сетевой интерфейс.

Для изменения настроек Link Aggregation выполните следующие действия:

  1. Чтобы изменить режим Link Aggregation или просмотреть информацию о физических устройствах, перейдите в раздел Панель управления > Сеть > Сетевой интерфейс.
  2. Выберите интерфейс Bond 1 и нажмите кнопку Изменить.
  3. Перейдите к соответствующим вкладкам для изменения требуемых настроек.

Для удаления Link Aggregation выполните следующие действия:

  1. Выберите Панель управления > Сеть > Сетевой интерфейс.
  2. Выберите интерфейс Bond 1 и нажмите кнопку Удалить.
  3. После завершения интерфейс Bond 1 будет заменен на два интерфейса сетей.

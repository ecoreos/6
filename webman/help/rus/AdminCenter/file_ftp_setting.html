FTP/FTPS

В пункте Панель управления > Файловые службы > FTP можно включить службы FTP для разрешения доступа клиентам к папкам и данным в системе. Перед активацией FTP убедитесь, что следующие порты TCP на маршрутизаторе переадресованы на устройство DiskStation: 21 (управляющее соединение по умолчанию), 20 (соединение с данными в активном режиме) и 55536-55567 (соединение с данными в пассивном режиме).

Включение службы FTP:

  1. Выполните одно из следующих действий:
    • Включить службу FTP (без защиты). Это стандартный сетевой протокол, используемый для передачи файлов. Во время сеанса передачи FTP не шифрует такую информацию, как пароли, имена пользователей или файлы, для ее защиты. Однако скорость передачи высокая, и требуется меньше системных ресурсов.
    • Включить службу защиты FTP SSL/TLS (FTPS). Продолжение стандартного протокола FTP с дополнительной поддержкой криптографических протоколов TLS и SSL, предназначенных для защиты информации во время передачи данных (например, паролей, имен пользователей или файлов). Однако из-за шифрования скорость передачи низкая, и требуется больше системных ресурсов.
  2. Нажмите Применить.

Время ожидания

Сервер FTP отключит неактивных пользователей после указанного периода времени.

Установка номера порта службы FTP

В данном разделе можно указать порт, который будет использоваться для канала управления FTP.

Примечание.

Номер порта по умолчанию для канала управления FTP — 21. Пользователь может задать диапазон от 1 до 65 535, исключая следующие номера: 20, 22, 23, 25, 80, 110, 137, 138, 139, 143, 199, 443, 445, 515, 543, 548, 587, 873, 993, 995, 3306, 3689, 5000, 5001, 5005, 5006, 5335, 5432, 6881, 8080, 7000, 7001, 8081, 9997, 9998, 9999, 50001, 50002 и порты eMule по умолчанию: 4662 (TCP), 4672 (UDP).

Диапазон портов пассивного FTP

В данном разделе можно указать порт, который будет использоваться для подключений пассивного FTP. Вы можете использовать диапазон портов по умолчанию или задать пользовательский диапазон. Диапазон портов можно установить от 1025 до 65535, и он может содержать до 128 портов.

Сообщение внешних IP-адресов только в PASV-режиме

В случае активации данной опции сервер будет сообщать свой внешний IP-адрес клиентам FTP. Полученный IP-адрес будет указан в журнале подключений клиента FTP. После включения вы можете выбрать автоматически обнаруженные внешние IP-адреса или вручную ввести внешний IP-адрес устройства DiskStation.

Данная функция работает только в том случае, если DiskStation находится в системе NAT, а клиенты FTP находятся в другой подсети, отличной от подсети DiskStation. В большинстве случаев данный параметр не используется, однако если клиентам FTP не удается подключиться к DiskStation, можно включить этот параметр и повторить попытку.

Включение FXP

FXP (File eXchange Protocol) — протокол обмена файлами. В случае активации данной опции служба FTP будет поддерживать функцию передачи файлов с сервера на сервер.

Включение криптографического модуля FIPS

FIPS поддерживается на службе FTPS с использованием криптографического модуля OpenSSL FIPS 140-2 (сертификат № 1747). При включении данного параметра службой FTPS будут использоваться дополнительные алгоритмы криптографии для защиты сеансов.

Поддержка режима передачи ASCII

Включите этот параметр для поддержки режима передачи ASCII, позволяющего устройству DiskStation корректно передавать простые текстовые файлы в формате ASCII по протоколу FTP.

Кодировка UTF-8

Этот параметр можно использовать, чтобы клиенты FTP правильно отображали и использовали имена файлов на различных языках. Чтобы все FTP-клиенты работали корректно, имеется три варианта:

  • Отключить. Кодировка UTF-8 не будет использоваться. Этот вариант предназначен для клиентов использующих в качестве FTP-клиента для отправки/загрузки файлов с именами на различных языках проводник Windows. Не рекомендуется для стандартного использования.
  • Авто. DSM Будет автоматически определять, что делать с кодировкой UTF-8. Рекомендуется для стандартного использования.
  • Принудительно. Принудительное использование кодировки UTF-8. Если вы активировали UTF-8 на FTP-клиенте и имена файлов на различных языках отображаются неверно, вы можете активировать данную опцию.

Ограничение подключения

В данном разделе содержатся следующие параметры.

  • Ограничивать число подключений по IP. Когда данная функция включена, сервер будет ограничивать максимальное число подключений на один IP-адрес. Для эффективного управления полосой пропускания включите эту функцию, а также активируйте параметр Ограничение полосы пропускания FTP.
  • Включить ограничение полосы пропускания FTP. Включите эту функцию, чтобы ограничить полосу пропускания для отдельного FTP-подключения.

Примечание.

  • Невозможно войти на сервер по протоколу FTP с помощью учетной записи "guest".
  • Для корректного доступа к данным настройки кодовой страницы на компьютере клиента FTP должны совпадать с настройками DiskStation.
  • На некоторых моделях служба FTP поддерживает подключение по протоколам SSL и TLS.
  • Настройки брандмауэра клиента FTP и двух серверов FTP должны быть выполнены правильно, в противном случае функция FXP может работать неисправно.
  • Некоторым клиентам FTP, например использующим Internet Explorer 8/9, для входа требуется вводить имя пользователя и пароль. Например: ftp://username:password@your.ftp.com.

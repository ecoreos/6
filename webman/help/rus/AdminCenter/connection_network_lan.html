ЛВС/Интернет

В зависимости от модели DiskStation и настроек сети можно настроить параметры DHCP для ЛВС или Интернета в пункте Панель управления > Сеть > Сетевой интерфейс. DHCP - это сетевой протокол для автоматической конфигурации IP. Чтобы DHCP мог присвоить IP-адрес системе, нажмите Редактировать и выберите Получить конфигурацию сети автоматически (DHCP) на вкладке IPv4.

Если используется конкретный IP-адрес, предоставленный провайдером услуг сети, выберите Конфигурировать вручную и вручную введите IP-адрес и маску подсети. Если недоступен DNS-сервер или шлюз по умолчанию, оставьте это поле пустым.

Примечание.

  1. Чтобы убедиться в том, что уведомления электронной почты через SMTP-сервер доставляются корректно, необходимо ввести действующий IP-адрес DNS-сервера.
  2. Jumbo-кадры – это интернет-кадры длиной более 1500 байт максимального размера блока (MTU), позволяющие эффективно передавать большие файлы по Ethernet. Их можно использовать только в сетевой среде Gigabit. Для обеспечения правильной работы Jumbo-кадров все компьютеры и устройства в сети, подключенные к DiskStation, должны поддерживать данную функцию и использовать одно и то же значение MTU.

VLAN

Если для сетевой среды требуются настройки VLAN, можно активировать параметр VLAN и ввести идентификатор VLAN для сетевой среды на вкладке IPv4 в окне "Редактировать".

Примечание.

  • Убедитесь, что коммутатор сети правильно настроен, в противном случае услуга DiskStation может стать недоступной.
  • Если вы не можете подключиться к DiskStation после изменения указанных выше настроек, нажмите кнопку RESET, расположенную на DiskStation, чтобы восстановить настройки по умолчанию.

Настройка IPv6

Устройства DiskStation поддерживают двойную схему блоков IPv4/IPv6. Поэтому IPv4 и IPv6 могут работать на DiskStation одновременно. По умолчанию в параметрах IPv6 установлено Авто. Выберите существующий сетевой интерфейс, нажмите кнопку Редактировать и выберите вкладку IPv6 для изменения настроек.

Настройка параметров IPv6:

  1. Выберите Панель управления > Сеть > Сетевой интерфейс.
  2. Выберите существующий сетевой интерфейс и нажмите Редактировать.
  3. На вкладке IPv6 выберите тип настройки в меню Настройка IPv6:
    • Авто: Устройство DiskStation автоматически создаст адрес IPv6 в зависимости от объявления маршрутизатора ЛВС (RA).
      • Режим без сохранения состояний: Устройства могут подключаться к маршрутизатору для автоматической настройки адреса IPv6 без настройки IPv6 DNS-сервера.
      • Режим DHCPv6 без сохранения состояний: Устройства могут подключаться к маршрутизатору для автоматической настройки адреса IPv6 и получения информации DNS через DHCPv6.
      • Режим отслеживания состояний: Устройства могут подключаться к маршрутизатору для получения адреса IPv6 и информации DNS через DHCPv6. В этом режиме можно активировать сервер DHCPv6 и настроить Начальный IP-адрес и Конечный IP-адрес сервера DHCPv6.
    • Вручную: Настройка IPv6 вручную с помощью ввода информации в поля Адрес IPv6, Длина префикса и Шлюз по умолчанию.
    • Выключить: Отключение поддержки IPv6.

Пока IPv6 полностью не вытеснит IPv4, механизмы перехода IPv6 требуются для доступа хостов с адресами IPv4 к службам IPv6. Выберите один из механизмов перехода IPv6:

  • 6in4: Выполните подключение к сети IPv6 с помощью сервиса туннелей. Вам необходимо ввести данные в поля Адрес IPv6, Длина префикса, Префикс и Адрес IPv4 сервиса туннелей. Для этого требуется публичный адрес IPv4.
  • 6to4: Выполните подключение к сети IPv6 через сервис туннелей с помощью произвольной рассылки. Для этого требуется публичный адрес IPv4.
  • DHCP-PD: Получите префикс IPv6 от сервера DHCP с помощью делегирования префикса в рамках ЛВС. Для этого требуется поддержка ISP; функция работает только в режиме маршрутизатора.
  • Туннелирование IPv6 (TSP): Выберите Панель управления > Сеть > Общие, чтобы настроить данный параметр.

802.1X

802.1X — это стандарт проверки подлинности, разработанный институтом IEEE для сетей с пользовательским подключением. 802.1X можно использовать в сетях для проверки подлинности клиента, чтобы не допустить доступ клиентов, не прошедших проверку подлинности, к защищенной сети.

Можно разрешить устройству Synology NAS подключаться к сети, в которой настроена проверка подлинности 802.1X.

Включение проверки подлинности 802.1X:

  1. Выберите Панель управления > Сеть > Сетевой интерфейс.
  2. Выберите проводной сетевой интерфейс (например, LAN) и выберите Редактировать.
  3. Откройте вкладку 802.1X и установите флажок Включить аутентификацию 802.1X.
  4. Выберите один из следующих протоколов из раскрывающегося меню Проверка подлинности: Если вы не знаете, какой протокол выбрать, выберите Авто.
    • Protected EAP (PEAP): Используйте сертификат Центра сертификации для установки защищенного соединения. Это защищенное соединение будет использоваться в процессе внутренней проверки подлинности.
    • Tunneled TLS (TTLS): Используйте сертификат Центра сертификации для установки защищенного соединения. Это защищенное соединение будет использоваться в процессе внутренней проверки подлинности.
    • TLS: используйте протокол TLS (Transport Layer Security) для проверки подлинности.
  5. При использовании TTLS или PEAP выберите один из внутренних протоколов проверки подлинности в раскрывающемся меню.
    • MS-CHAP v2: Аутентификационные данные будут зашифрованы по протоколу Microsoft CHAP версии 2.
    • GTC: Аутентификационные данные будут использовать протокол GTC (Generic Token Card) для шифрования.
  6. Введите имя пользователя и пароль.
  7. Если потребуется, нажмите Импорт > Обзор, чтобы найти и импортировать сертификат, экспортированный из Radius Server (например, ca.crt).
  8. Нажмите OK, чтобы сохранить изменения.

Примечание.

  • Для настройки проверки подлинности 802.1X для беспроводных сетей выберите Панель управления > Беспроводная сеть > Wi-Fi.
  • Настройки проверки подлинности должны совпадать с настройками Radius Server. Для получения дополнительной информации обратитесь к администратору Radius Server.

Беспроводная ЛВС*

Если DiskStation выступает в качестве беспроводного маршрутизатора, можно настроить следующие параметры для беспроводной локальной сети в пункте Панель управления > Сеть > Сетевой интерфейс.

Настройки IPv4

IP-адрес и маска подсети

Здесь вы можете установить IP-адрес и маску подсети для DiskStation.

DHCP Server

Включение параметра Включить DHCP Server позволяет DiskStation назначать IP-адреса клиентским устройствам, подключенным к локальной сети, предоставляемой DiskStation.

Ретранслятор PPPoE

Включите Ретранслятор PPPoE, чтобы разрешить устройствам, подключенным к DiskStation, подключаться к Интернету через подключение PPPoE.

Настроить как интернет-шлюз UPnP

Включите службу UPnP, чтобы использовать DiskStation как интернет-шлюз UPnP, чтобы устройства, подключенные к локальной сети DiskStation, могли настраивать параметры переадресации порта с помощью UPnP.

Настройки IPv6

Вы можете получить адрес IPv6 и префикс от ЛВС/Интернет с 6in4, 6to4 или DHCP-PD. С помощью данной информации можно настроить параметры маршрутизации IPv6 для DiskStation.

  • Адрес IPv6: Введите адрес IPv6, полученный от ЛВС/Интернет.
  • Длина префикса: Введите длину префикса адреса IPv6, полученного от ЛВС/Интернет, обычно 64.
  • Основной DNS: Основной сервер DNS для устройств, подключенных к маршрутизатору. Его не требуется настраивать в пункте Режим без сохранения состояний.
  • Дополнительный DNS: Дополнительный сервер DNS для устройств, подключенных к маршрутизатору. Его не требуется настраивать в пункте Режим без сохранения состояний.

Вы можете выбрать один из трех режимов объявления маршрутизатора (RA):

  • Режим без сохранения состояний: Устройства могут подключаться к маршрутизатору для автоматической настройки адреса IPv6 без настройки IPv6 DNS-сервера.
  • Режим DHCPv6 без сохранения состояний: Устройства могут подключаться к маршрутизатору для автоматической настройки адреса IPv6 и получения информации DNS через DHCPv6.
  • Режим отслеживания состояний: Устройства могут подключаться к маршрутизатору для получения адреса IPv6 и информации DNS через DHCPv6. В этом режиме можно активировать сервер DHCPv6 и настроить Начальный IP-адрес и Конечный IP-адрес серевера DHCPv6.

Клиенты DHCP

Здесь можно просматривать список клиентов DHCP и их сетевые конфигурации (например, MAC-адрес, IP-адрес, имя хоста и время до окончания срока действия аренды DHCP).

Вы можете отфильтровать клиентов IPv4/IPv6 в раскрывающемся списке для отображения клиентов, которые используют IPv4 и IPv6 соответственно. Выберите клиента и нажмите Добавить в зарезервированные адреса, чтобы добавить клиента в список зарезервированных адресов.

Резервирование DHCP

Чтобы клиент всегда получал тот же самый IP-адрес во время обновления аренды DHCP, можно добавить клиента в список зарезервированных адресов.

Вы можете отфильтровать раскрывающийся список зарезервированных адресов IPv4/IPv6. Адреса IPv4 идентифицируются с помощью MAC-адреса, а адреса IPv6 — с помощью DUID.

Добавление клиента в список зарезервированных адресов.

  • Выберите клиента и нажмите Добавить.
  • Укажите MAC-адрес или DUID-адрес клиента, адрес IPv4/IPv6, который необходимо назначить клиенту и имя хоста клиента.
  • Нажмите OK, чтобы сохранить изменения.

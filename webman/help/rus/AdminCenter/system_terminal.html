Терминал

Параметры, активированные в меню Панель управления > Терминал и SNMP > Терминал, позволяют DiskStation осуществлять поддержку служб Telnet и командной строки SSH. Можно изменить уровень безопасности алгоритма шифрования SSH.

Включение службы Telnet/SSH:

  1. Установите флажок рядом с протоколом (SSH или Telnet).
  2. Нажмите Применить.

Изменение порта службы SSH:

  1. В поле Порт введите новый номер порта.
  2. Нажмите Применить.

Для защиты подключений SSH используются различные алгоритмы шифрования с разными уровнями безопасности. Можно выбрать наиболее подходящие для вас алгоритмы.

Изменение уровня безопасности алгоритма шифрования SSH:

  1. Нажмите Дополнительные настройки.
  2. Выберите уровень безопасности алгоритмов шифрования. Чтобы указать, какие именно алгоритмы можно использовать, нажмите Настроить и выберите одну из трех групп: Cipher, KEX (Key Exchange — обмен ключами), и MAC (Message Authentication Code — код проверки подлинности сообщения).
  3. Нажмите Применить.

Некоторые модели поддерживают аппаратное ускорение определенных алгоритмов шифрования. Можно установить флажок Использовать только шифры аппаратного ускорения, чтобы повысить скорость передачи для служб SSH, включая службы SFTP и сетевого резервного копирования с шифрованием. На странице дополнительных настроек алгоритмы с поддержкой аппаратного ускорения показаны красным цветом.

Примечание.

  • Некоторые службы (например, сетевое резервное копирование, SFTP) автоматически активируют SSH и открывают порт 22. В этом случае необходимо активировать службу SSH в меню Панель управления > Терминал и SNMP > Терминал для входа в DSM с помощью SSH.
  • SSH/Telnet поддерживает вход в систему только под учетными записями, принадлежащими группе administrators (администраторы). Для переключения на учетную запись root выполните вход в систему как пользователь SSH/Telnet, принадлежащий к группе administrators (администраторы), запустите команду sudo -i и затем введите пароль от учетной записи, использовавшейся для входа.
  • Пароль не может быть пустым при входе в систему с помощью SSH/Telnet.
  • При отключении службы SSH отменяются все подключения SSH, включая выполняемые зашифрованные задачи сетевого резервного копирования. Рекомендуется выполнить задачи сетевого резервного копирования еще раз после отключения службы SSH.
  • Некорректное использование или изменение DiskStation могут привести к нарушению нормальной работы или потере данных.
  • Выбор более высокого уровня безопасности или использование только шифров аппаратного ускорения может привести к тому, что некоторые клиенты SSH не смогут подключиться из-за конфликтов совместимости.
  • В зависимости от выбранного уровня безопасности некоторые алгоритмы, поддерживающие аппаратное ускорение, могут быть недоступны.

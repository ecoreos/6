Дополнительные настройки пользователя

Чтобы изменить дополнительные настройки пользователя, выберите Панель управления > Пользователь > Дополнительно. Доступны следующие параметры:

Разрешить пользователям выполнять сброс забытого пароля

Если вы хотите разрешить пользователям выполнять сброс забытого пароля по электронной почте, установить флажок Разрешить пользователям выполнять сброс забытых паролей по электронной почте. При включении этого параметра на странице входа в DSM отобразится ссылка Забыли пароль?. Если пользователь забыл пароль, он может нажать эту ссылку и ввести имя пользователя. В этом случае система отправит пользователю электронное письмо с ссылкой для сброса забытого пароля.

Примечание.

  • Перед включением данного параметра убедитесь, что у вас включены уведомления электронной почты SMTP. Для этого выберите Панель управления > Уведомление.
  • Для получения сообщений от системы пользователи должны указать в своей учетной записи адрес электронной почты в разделе информации о пользователе.
  • Пользователи, принадлежащие группе administrators, не могут выполнять сброс пароля с помощью данного параметра.
  • Пользователи домена не могут выполнять сброс пароля с помощью данного параметра.
  • Пользователи LDAP могут выполнять сброс пароля с помощью данного параметра, если устройство DiskStation является сервером каталогов LDAP.

Надежность пароля

На DiskStation можно настроить правила надежности пароля для пользователей.

Примечание.

Правила надежности пароля применяются только к новым паролям. Например, правила надежности паролей будут применяться только при создании новой учетной записи пользователя или изменении пароля существующего пользователя. Правила новый паролей не действуют для существующих паролей и паролей, принадлежащих импортированным учетным записям пользователей.

Включение правил надежности пароля:

  1. Установите флажок Применить правила надежности пароля, чтобы включить ограничения паролей.
  2. Выберите более одного ограничения пароля из следующих правил:
    • Не включать в пароль имя и описание: Пароль не должен содержать имя пользователя и его описание. Символы в кодировке UTF-8 также исключаются.
    • Включить смешанный регистр: Пароль должен содержать буквы верхнего и нижнего регистра.
    • Использовать цифры: Пароль должен содержать минимум один числовой символ (0~9).
    • Использовать специальные символы: Пароль должен содержать минимум один особый символ ASCII (например, ~, `, !, @, #, $, %, ^, &, *, (, ), -, _, =, +, [, {, ], }, \, |, ;, :, ', ", <, >, /, ?).
    • Минимальная длина пароля: Пароль должен содержать не меньшее количество символов, чем указано в текстовом поле. Длина должна быть от 6 до 127.
  3. Нажмите Применить, чтобы сохранить настройки.

2-этапная проверка

Проверка в 2 этапа обеспечивает повышенную безопасность для вашей учетной записи DSM. Вы можете запросить администраторов DSM или всех пользователей DSM включить 2-этапную проверку для защиты учетных записей.

Принудительная 2-этапная проверка для пользователей DSM:

  1. Установите флажок Принудительно включить 2-этапную проверку для следующих пользователей и выберите пользователей, к которым будет применена данная настройка.

    2. Примечание.

    Если вы еще не настроили 2-этапную проверку в пункте Личные > Учетная запись, запустится мастер 2-этапной проверки для выполнения необходимых настроек перед применением данной функции.

  2. После применения данной функции пользователям, которые еще не активировали 2-этапную проверку, потребуется выполнить соответствующие настройку перед входом в DSM.

Примечание.

Отключение 2-этапной проверки здесь не повлияет на настройки 2-этапной проверки каждого пользователя. Пользователи могут сохранить настройки 2-этапной проверки или выбрать Личные > Учетная запись для отключения этих настроек.

Главная папка пользователя

Включите главную папку пользователя для создания личной папки home для каждого пользователя за исключением пользователей guest. Каждый пользователь может войти в личную папку через CIFS, AFP, FTP или File Station.

Пользователи, принадлежащие группе administrators, имеют доступ ко всем личным папкам в папке общего доступа homes. Имя главной папки является именем учетной записи пользователя.

Включение службы главной папки пользователя:

  1. Установите флажок Включить службу главной папки пользователя.
  2. Если присутствует более одного раздела, выберите раздел, где будет храниться папка homes.
  3. Нажмите Применить.

Примечание.

При отключении главной папки локального пользователя также будет отключена главная папка пользователя домена.

Отключение службы главной папки пользователя:

  1. Снимите флажок Включить службу главной папки пользователя.
  2. Нажмите Применить.

После отключения службы главной папки пользователя, папка общего доступа homes будет доступна только для администратора. Пользователи смогут получать доступ к главным папкам снова после того, как будет включена служба главной папки пользователя.

Примечание.

  1. Чтобы удалить папку общего доступа homes, необходимо отключить службу главной папки пользователя.
  2. Служба главной папки пользователя домена будет отключена, если отключена главная папка локального пользователя.

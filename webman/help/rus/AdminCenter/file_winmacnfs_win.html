Файловая служба Windows

Файловая служба Windows позволяет клиентам Windows, Mac и Linux с поддержкой SMB/CIFS получать доступ к данным, сохраненным на DiskStation. Чтобы просмотреть файлы на DiskStation с помощью Проводника Windows или подключить общие папки как сетевые диски, включите данный параметр в пункте Панель управления > Файловые службы > Win/Mac/NFS.

Включение файловой службы Windows:

  1. Установите флажок Включить файловую службу Windows.
  2. Нажмите Применить.

Примечание

После включения файловой службы Windows можно монтировать полностью проиндексированные папки на компьютере Mac по протоколу SMB и использовать Spotlight в Проводнике для поиска содержимого файлов в этих папках.

Рабочая группа

Данный параметр позволяет использовать систему как часть определенной рабочей группы для сетевого управления.

Примечания

  • Имя рабочей группы может содержать от 1 до 15 символов. Имя не может содержать следующие символы: [ ] ; : " < > * + = \ / | ? ,
  • При использовании Mac с OS X 10.9 и устройства DiskStation, подключенного к домену Windows, можно выполнить вход, добавив localhost\ перед локальным именем пользователя устройства DiskStation. После этого вы сможете получать доступ к файлам на DiskStation с помощью файловой службы Windows.

Запрет доступа к предыдущим версиям

В Проводнике Windows пользователи могут получить доступ к моментальным снимкам папок или файлов, открыв вкладку Предыдущие версии в общей папке и разрешив восстановление и копирование данных без помощи администратора. Если выбрать Запрет доступа к предыдущим версиям, на странице "Предыдущие версии" не будет отображаться никакой информации.

Примечание.

  • Пользователи могут получить доступ к предыдущей версии только в том случае, если во время создания моментального снимка, соответствующего предыдущей версии, учетные записи этих пользователей были созданы и им были назначены права на чтение общей папки, в которой был сделан моментальный снимок.

Включения журнала передачи

Выбор этого параметра включит ведение журнала операций доступа к файлам, включая создание, удаление, чтение, запись, перемещение и переименование файлов или папок по протоколу CIFS/SMB, известному как протокол передачи файлов Windows.

Дополнительные настройки

Нажмите кнопку Дополнительные настройки для просмотра указанных ниже параметров.

  • WINS-сервер: Microsoft Windows Internet Name Service (WINS) — это услуга сопоставления имени NetBIOS с IP-адресом. Пользователям Windows будет проще найти систему в сетях TCP/IP, если для системы настроена регистрация с WINS-сервером.
  • Макс. протокол для файловой службы Windows. В зависимости от конфигурации вашей сети вы можете установить высочайший уровень протокола, поддерживаемый DiskStation.
    • Включить SMB 2. SMB (Server Message Block) 2 поддерживается, начиная с версии Windows Vista, и является расширенной версией SMB. SMB 2 добавляет возможность объединять различные действия SMB в единый запрос для сокращения числа сетевых пакетов и повышения производительности.
    • Включить SMB 2 и Large MTU: Максимальным размером передаваемых данных (MTU) называется самый крупный блок данных, который можно передать по протоколу связи. Для использования преимуществ более быстрых сетевых интерфейсов, например 1- или 10-гигабитного Ethernet, Large MTU увеличивает максимальный размер передаваемых данных до 1 мегабайта (МБ). Включение Large MTU повышает скорость и эффективность передачи больших файлов, сокращая количество обрабатываемых пакетов.
    • Включить SMB 3. SMB 3 поддерживается, начиная с Windows 8 и Windows Server 2012. Это улучшенная версия SMB 2. Функция SMB 3 позволяет передавать файлы, зашифрованные по алгоритму AES, что повышает безопасность однорангового обмена файлами.

    Примечания

    • SMB 2 поддерживается на Windows Vista, Windows Server 2008 R2, Windows 7 и более поздних версиях.
    • Large MTU поддерживается на Windows Server 2008 R2, Windows 7 и более поздних версиях.
    • Чтобы обеспечить эффективную работу Large MTU с компьютером Windows, необходимо изменить настройки реестра компьютера. Дополнительную информацию можно найти на веб-сайте технической поддержки Microsoft (например, в этой статье).
  • Режим шифрования передачи: Когда функция SMB 3 включена, файловая служба Windows выполняет шифрование передачи для повышения безопасности при обмене файлами.
    • Отключить: шифрование передачи не будет использоваться.
    • Авто: шифрование передачи будет использоваться только для клиентов, поддерживающих данную функцию.
    • Принудительно: шифрование передачи будет использоваться всегда. Это приведет к тому, что клиенты, не поддерживающие функцию шифрования передачи, не смогут пользоваться файловой службой Windows.

    • Примечания

    • Шифрование передачи поддерживается в Windows 8, Windows Server 2012 и более поздних версиях.
    • Когда функция шифрования передачи включена, анонимный вход в файловую службу Windows не поддерживается.
    • При включении функции шифрования передачи не рекомендуется устанавливать флажок Отключить Opportunistic Locking во избежание превышения времени ожидания приложения.
  • Отключить Opportunistic Locking: данный параметр приведет к отключению уступающей блокировки "Opportunistic Locking" CIFS, что в результате приведет к повышению производительности доступа приложений баз данных к файлам базы данных в общей папке DiskStation по сети.
  • Включить Local Master Browser: если в одной подсети расположено несколько компьютеров Windows, среди них будет выбран один компьютер в качестве Local Master Browser. Local Master Browser содержит список других компьютеров в подсети и их общих ресурсов и обменивается данными списками с другими компьютерами. Данный параметр позволяет DiskStation выполнять роль Local Master Browser.

    • Примечание

    Включение данной функции приведет к отключению режима сна жестких дисков и включит гостевую учетную запись без пароля.

  • Включить модуль DirSort VFS. Сортировка папок в алфавитном порядке. Включение данного параметра влияет на производительность системы.
  • Запрещенные файлы. Скрытие файлов по определенным критериям. Критерии могут включать символы (*); несколько критериев необходимо разделять косой чертой (/). Например: /abc*/*.txt/*.conf/. Включение данного параметра влияет на производительность системы.
  • Разрешить символические ссылки в общих папках. Включение данного параметра разрешает использование символических ссылок для обозначения файлов или папок, расположенных в одной и той же общей папке. Символические ссылки можно создать с помощью терминала DSM.
  • Разрешить символические ссылки между различными общими папками. Данный параметр позволяет символическим ссылкам указывать на файлы и папки в различных общих папках. Символические ссылки можно создать с помощью терминала DSM. Обратите внимание, что включение данного параметра позволяет пользователям без прав доступа просматривать и осуществлять доступ к общим папкам, на которые указывают символические ссылки.
  • Включить модуль MSDFS VFS. Позволяет символическим ссылкам указывать на файлы и папки на других серверах SMB. В ранних версиях Windows, например Windows XP, перед включением данного параметра клиенту и DiskStation необходимо присоединиться к домену Windows. Дополнительную информацию о создании ссылок MSDFS см. здесь. Обратите внимание, что кэш клиентского устройства должен быть очищен после отключения данного параметра, в противном случае это может оказать влияние на производительность.
  • Отключить несколько подключений с одного IP-адреса. При включении данного параметра DiskStation будет поддерживать только одно подключение клиента на IP-адрес с помощью файловой службы Windows. Если несколько клиентов с одним IP-адресом используют файловую службу Windows, DiskStation будет использовать только последнее подключение и отключит предыдущие. В среде NAT клиент может использовать один IP-адрес, в этом случае включение данного параметра может повлиять на производительность.
  • Сохранение журналов отладки. При включении данного параметра в целях отладки будет выполняться сохранение подробных журналов. Включение данного параметра влияет на производительность системы.
  • Применение разрешений UNIX по умолчанию. Включение данного параметра позволяет применять разрешения UNIX по умолчанию при загрузке или создании папок и файлов. Для файлов используется разрешение UNIX 744, а для папок — 755. При отключении данного параметра для файлов и папок используется разрешение UNIX 777.

    • Примечание

    • Включение данного параметра может привести к несогласованности разрешений между различными протоколами. Во избежание несогласованности рекомендуется не включать данный параметр.

    Примечания

    • Для всех общих папок с включенными разрешениями Windows ACL (кроме папки "photo" и папок на внешних дисках) необходимо выполнить команду chmod на клиенте Linux или FTP. В результате типы разрешений для папок и файлов изменятся с Windows ACL на UNIX.
    • Включение данного параметра может привести к несогласованности разрешений между различными протоколами. Во избежание несогласованности рекомендуется не включать данный параметр.
  • Не резервировать дисковое пространство при создании файлов. Включение данного параметра позволяет не резервировать дисковое пространство при создании файлов на DiskStation.
  • Очистить кэш SMB. Нажмите данную кнопку, чтобы удалить следующие файлы кэша SMB для устранения неисправностей.
    • /var/cache/samba/netsamlogon_cache.tdb
    • /var/lib/samba/winbindd_cache.tdb
    • /var/lib/samba/share_info.tdb
    • /var/lib/samba/registry.tdb
    • /run/samba/connections.tdb
    • /run/samba/gencacche.tdb
    • /run/samba/gencache_notran.tdb

Управление трафиком

Управление трафиком позволяет управлять исходящим трафиком служб, работающих на DiskStation. После выбора службы необходимо настроить параметры Гарантированная ширина пропускания и Максимальная ширина пропускания для ограничения исходящего трафика.

Гарантированная ширина пропускания задает исходящий трафик, который данная служба гарантированно предоставляет при достаточном значении полосы пропускания всей системы.

Максимальная ширина пропускания задает исходящий трафик, который данная служба может предоставить при достаточном значении полосы пропускания всей системы и наличии Остаточной ширины пропускания системы. Системному менеджеру рекомендуется сначала рассчитать Исходящую ширину пропускания системы, а затем убедиться, что сумма Гарантированной ширины пропускания для каждой службы не превышает Исходящую ширину пропускания системы, в противном случае настройки могут работать неверно.

Формула настройки:

Исходящая ширина пропускания системы = сумма Гарантированной ширины пропускания каждой службы + Остаточная ширина пропускания системы

Гарантированная ширина пропускания каждой службы ≤ Максимальной ширины пропускания каждой службы

Примечание. Если исходящие пакеты не соответствуют правилам управления трафиком, ограничение установлено не будет. Вы можете добавить собственное правило управления трафиком по умолчанию в конце всех правил, выбрав Все порты и указав параметры Гарантированная ширина пропускания и Максимальная ширина пропускания.

Создание правила управления трафиком:

  1. Выберите Панель управления > Брандмауэр и Управление трафиком в левом столбце выберите сетевой интерфейс в соответствии с типом своего сетевого подключения.
  2. Нажмите Создать.
  3. В разделе Порты выберите для правила один из вариантов:
    • Выберите Все, чтобы применить правило ко всем портам.
    • Выберите Выбрать из списка встроенных приложений.
    • Выберите Пользовательский и нажмите Пользовательский, а затем введите не более 15 портов, разделяя их запятой, либо укажите диапазон портов.
    • Можно также указать протоколы.

Теперь вы можете просматривать созданное правило управления трафиком в списке правил.

Чтобы удалить правила управления трафиком, выполните следующие действия:

  • Снимите флажок правила в столбце Включено списка.
  • Удалите правило управления трафиком.
    Примечание.
  1. Вы можете создать до 100 правил для сетевого интерфейса.
  2. Чтобы перегруппировать правила, перетащите их. Правила получают приоритет согласно их положению в списке.
  3. Если в системе используются 2 сетевых порта, подключенных к одной подсети, правила брандмауэра могут работать неправильно.*

____
* Поддерживается только на некоторых моделях.

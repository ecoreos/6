Подключение к VPN

DiskStation можно легко использовать в качестве клиента виртуальной частной сети (VPN), подключаясь к существующему серверу VPN по протоколам PPTP, OpenVPN или L2TP/IPSec. Если в наличии имеется несколько серверов VPN, вы можете создать отдельные профили VPN для каждого сервера и переключаться между ними всего одним нажатием кнопки.

Что такое VPN?

Виртуальная частная сеть (VPN) — это решение, предназначенное для безопасного доступа к ресурсам частной сети через Интернет. С ростом потребностей расширения корпоративных и пользовательских сетей существенно увеличивается стоимость физических сетевых систем и их технического обслуживания. Если вы ищите систему с недорогим техническим обслуживанием, VPN — это именно то решение, которое вам подходит. Технология VPN поддерживает шифрование и другие механизмы защиты данных, благодаря чему пользователи компании могут получать доступ к централизованной сетевой системе и использовать ее ресурсы так же просто, как при использовании локальной сети. Простые пользователи также имеют доступ к ресурсам своих домашних локальных сетей, даже находясь вне дома.

Примечание.

  • DiskStation может подключаться только к серверам OpenVPN с поддержкой сетевых туннелей на уровне 3.
  • Перед установкой пакета VPN убедитесь, что у вас имеется сервер VPN для подключения. Для улучшения совместимости вы можете подключиться к службам VPN, расположенным на DiskStation с установленным пакетом VPN Server. Чтобы настроить другое устройство DiskStation в качестве сервера VPN, ознакомьтесь с инструкциями в разделе Справка DSM > VPN Server > Настроить VPN Server.

Для создания профилей VPN выполните следующие действия.

  1. Выберите Панель управления > Сеть.
  2. На вкладке Сетевой интерфейс нажмите Создать и выберите Создать профиль VPN.
  3. При создании нового профиля VPN можно выбрать один из следующих типов подключения:
    • PPTP: PPTP (туннельный протокол типа точка-точка) — это широко используемое VPN-решение, поддерживаемое большинством клиентов (включая Windows, Mac, Linux и мобильные устройства).
    • OpenVPN: OpenVPN — это созданное на основе открытого исходного кода решение для внедрения службы VPN. Оно защищает VPN-подключения с помощью механизма шифрования SSL/TLS.
    • L2TP/IPSec: L2TP (Туннельный протокол уровня 2) через IPSec предоставляет виртуальные частные сети с повышенным уровнем безопасности, поддерживаемые большинством клиентов (включая Windows, Mac, Linux и мобильные устройства).
  4. Инструкции по настройке всех трех типов подключения VPN см. в разделах ниже.

Создание профиля PPTP:

  1. При выборе способа подключения VPN нажмите PPTP, а затем нажмите Далее.
  2. Выберите имя нового профиля, укажите адрес сервера VPN, к которому вы хотите подключиться, и введите имя пользователя и пароль для сервера. Чтобы продолжить, нажмите Далее.

    3. Примечание.

    • Имя пользователя, пароль и адрес сервера VPN предоставляются администратором сервера VPN.
  3. Выберите любой из следующих протоколов проверки подлинности в раскрывающемся меню Проверка подлинности для защиты пароля клиента VPN во время проверки подлинности:
    • PAP: во время аутентификации пароль не будет шифроваться.
    • CHAP: пароль будет защищен с помощью протокола CHAP (Challenge-Handshake Authentication Protocol).
    • MS CHAP: пароль будет защищен с помощью Microsoft CHAP версии 1.
    • MS-CHAP v2: пароль будет защищен с помощью Microsoft CHAP версии 2.
  4. Если вы используете протокол MS CHAP или MS CHAP v2, то для защиты VPN-подключения в раскрывающемся меню Шифрование выберите один из следующих вариантов:
    • Без MPPE: VPN-подключение не будет защищено с помощью протокола MPPE (Microsoft Point to Point Encryption).
    • Требуется MPPE (40/128-разрядный): VPN-подключение будет защищено с помощью 40- или 128-разрядного протокола MPPE (Microsoft Point to Point Encryption).
    • Макс. MPPE (128-разрядный): VPN-подключение будет защищено 128-разрядным протоколом MPPE (Microsoft Point to Point Encryption), которое обеспечивает наивысший уровень защиты.

    5. Примечание.

    • Настройки проверки подлинности и шифрования должны совпадать с настройками сервера VPN. Для получения дополнительной информации обратитесь к администратору сервера VPN.
  5. Установите любой флажок в зависимости от ваших целей:
    • Использовать шлюз по умолчанию или удаленную сеть: включите данный параметр для перенаправления сетевого трафика DiskStation на указанный сервер VPN.
    • Разрешить другим устройствам выполнять подключение с помощью интернет-подключения данного сервера Synology: включите данный параметр, чтобы разрешить другим сетевым устройствам, находящимся в той же локальной сети, что и устройство DiskStation, выполнять подключение к одной виртуальной частной сети.
    • Возобновлять подключение после потери соединения с VPN: при потере VPN-подключения система выполнит пять попыток восстановления соединения через каждые 30 секунд.
  6. Нажмите Применить.

Создание профиля OpenVPN:

  1. В качестве метода подключения VPN выберите OpenVPN и нажмите Далее.
  2. Выберите имя нового профиля, укажите адрес сервера VPN, к которому вы хотите подключиться, и введите имя пользователя и пароль для сервера.
  3. В поле Сертификат нажмите кнопку Обзор и импортируйте файл сертификата (например, ca.crt), экспортированный с сервера VPN. Чтобы продолжить, нажмите Далее.

    4. Примечание.

    • Имя пользователя, пароль, адрес и сертификат сервера VPN должны быть предоставлены администратором сервера VPN.
  4. Установите любой флажок в зависимости от ваших целей:
    • Включить сжатие на подключении к VPN: включите данный параметр для выполнения сжатия данных перед отправкой с помощью VPN. Данный параметр может повысить скорость передачи, но одновременно может увеличиться потребление системных ресурсов.
    • Использовать шлюз по умолчанию или удаленную сеть: включите данный параметр для перенаправления сетевого трафика DiskStation на указанный сервер VPN.
    • Разрешить другим устройствам выполнять подключение с помощью интернет-подключения данного сервера Synology: включите данный параметр, чтобы разрешить другим сетевым устройствам, находящимся в той же локальной сети, что и устройство DiskStation, выполнять подключение к одной виртуальной частной сети.
    • Возобновлять подключение после потери соединения с VPN: при потере VPN-подключения система выполнит пять попыток восстановления соединения через каждые 30 секунд.
  5. Нажмите Применить.

Создание профиля OpenVPN (с использованием .ovpn):

  1. В качестве метода подключения VPN выберите OpenVPN (путем импорта файла .ovpn) и нажмите Далее.
  2. Укажите имя нового профиля и в поле Импорт файла .ovpn нажмите Обзор, чтобы выбрать и импортировать файл .ovpn, экспортированный с сервера VPN Server.
  3. Введите следующую информацию, предоставленную администратором VPN Server: ваши имя пользователя и пароль, а также импортируйте файл сертификата Центра сертификации (например, ca.crt).
  4. Если поставщик VPN Server также предоставил ключ аутентификации TLS, ключ клиента, сертификат клиента или список отзыва сертификатов, нажмите Дополнительные параметры и импортируйте эти файлы в соответствующие поля.
  5. Чтобы продолжить, нажмите Далее.

    6. Примечание.

    • Имя пользователя, пароль, адрес и любые дополнительные файлы, например сертификаты или ключи, должны быть предоставлены администратором VPN Server.
  6. Установите любой флажок в зависимости от ваших целей:
    • Включить сжатие на подключении к VPN: включите данный параметр для выполнения сжатия данных перед отправкой с помощью VPN. Данный параметр может повысить скорость передачи, но одновременно может увеличиться потребление системных ресурсов.
    • Использовать шлюз по умолчанию или удаленную сеть: включите данный параметр для перенаправления сетевого трафика DiskStation на указанный сервер VPN.
    • Разрешить другим устройствам выполнять подключение с помощью интернет-подключения данного сервера Synology: включите данный параметр, чтобы разрешить другим сетевым устройствам, находящимся в той же локальной сети, что и устройство DiskStation, выполнять подключение к одной виртуальной частной сети.
    • Возобновлять подключение после потери соединения с VPN: при потере VPN-подключения система выполнит пять попыток восстановления соединения через каждые 30 секунд.
  7. Нажмите Применить.

Создание профиля L2TP/IPSec:

  1. В качестве метода подключения VPN выберите L2TP/IPSec и нажмите Далее.
  2. Выберите имя нового профиля, укажите адрес сервера VPN, к которому вы хотите подключиться, и введите имя пользователя и пароль для сервера. Вам также понадобится ввести общий ключ для сервера VPN. Затем нажмите Далее.

    3. Примечание.

    • Имя пользователя, пароль, адрес и общий ключ VPN должны быть предоставлены администратором сервера VPN.
  3. Выберите любой из следующих протоколов проверки подлинности в раскрывающемся меню Проверка подлинности для защиты пароля клиента VPN во время проверки подлинности:
    • PAP: защита пароля обеспечиваться не будет.
    • CHAP: пароль будет защищен с помощью протокола CHAP (Challenge-Handshake Authentication Protocol).
    • MS CHAP: пароль будет защищен с помощью Microsoft CHAP версии 1.
    • MS-CHAP v2: пароль будет защищен с помощью Microsoft CHAP версии 2.

    4. Примечание.

    • Настройки проверки подлинности и шифрования должны совпадать с настройками сервера VPN. Для получения дополнительной информации обратитесь к администратору сервера VPN.
  4. Установите любой флажок в зависимости от ваших целей:
    • Использовать шлюз по умолчанию или удаленную сеть: включите данный параметр для перенаправления сетевого трафика DiskStation на указанный сервер VPN.
    • Разрешить другим устройствам выполнять подключение с помощью интернет-подключения данного сервера Synology: включите данный параметр, чтобы разрешить другим сетевым устройствам, находящимся в той же локальной сети, что и устройство DiskStation, выполнять подключение к одной виртуальной частной сети.
    • Сервер находится позади устройства NAT: по умолчанию протокол L2TP/IPSec не позволяет подключаться к серверу VPN, который находится позади устройства NAT. Чтобы обойти это ограничение, включите эту функцию, и DiskStation будет подключаться к серверу VPN, находящемуся позади NAT, по протоколу L2TP/IPSec.
    • Возобновлять подключение после потери соединения с VPN: при потере VPN-подключения система выполнит пять попыток восстановления соединения через каждые 30 секунд.
  5. Нажмите Применить.

Для подключения и отключения соединения сервера VPN выполните следующие действия.

  1. Выберите профиль VPN.
  2. Нажмите Подключиться или Отключиться.

Примечание.

  • Одновременно можно подключить только один профиль.

Для изменения профиля VPN выполните следующие действия.

  1. Выберите профиль VPN.
  2. Нажмите Редактировать.

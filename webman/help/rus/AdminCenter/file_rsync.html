rsync

Можно разрешить другим пользователям выполнять резервное копирование данных через rsync с устройства Synology NAS или rsync-совместимых клиентов на DiskStation.

Включение службы резервирования rsync

Включение службу rsync:

Выберите Панель управления > Файловые службы > rsync и выберите Включить службу rsync.

Назначение права доступа на резервирование rsync:

Назначьте пользователям соответствующие права доступа rsync с исходного устройства Synology NAS или rsync-совместимого сервера на целевое устройство.

  1. Выберите Панель управления > Права доступа, нажмите rsync (синхронизация папок общего доступа, резервирование LUN на поддерживаемых моделях) > Редактировать.
  2. Назначьте нужным пользователям права доступа Разрешить (пользователи DSM для зашифрованной службы rsync или учетные записи rsync для незашифрованной служб rsync). При необходимости укажите разрешенные IP-адреса для конкретных пользователей.

Примечание.

Если вы хотите сохранить информацию о владельце и группе исходных данных при выполнении резервирования rsync с DiskStation с версией DSM ниже 3.0 или с клиента, который не является DiskStation, необходимо добавить учетные записи rsync в группу administrators и выполнить резервное копирование данных в папку общего доступа NetBackup.

Включение учетных записей rsync

Если выполнить резервирование rsync из другого клиента (не Synology NAS), необходимо создать учетные записи rsync на целевом устройстве Synology NAS, чтобы пользователи rsync могли выполнить резервное копирование данных на устройство Synology NAS.

Включение учетных записей rsync:

  1. Выберите Панель управления > Файловые службы > rsync и выберите Включить учетную запись rsync.
  2. Нажмите Редактировать учетную запись rsync для настройки пользователей и соответствующих паролей для проверки подлинности.
  3. Нажмите Применить.

Примечание.

  • Если служба rsync зашифрована (то есть используется порт шифрования SSH), для учетных записей rsync используются имена пользователей DSM и их пароли.
  • Если служба rsync не зашифрована, для них требуется создать соответствующие учетные записи rsync.

Использование пользовательской конфигурации rsync

Можно использовать настраиваемую конфигурацию rsync и настроить права пользователя и целевую папку для архивирования на вашем устройстве DiskStation. Как следствие, другие люди смогут выполнять резервное копирование в режиме rsync со своих устройств DiskStation или клиентов, совместимых с rsync, в целевые общие папки (или "модули архивирования") на вашем устройстве DiskStation.

Использование пользовательской конфигурации rsync:

  1. Выполните вход в устройство DiskStation с помощью Telnet или SSH.
  2. В файле конфигурации rsync "/etc/rsyncd.conf" укажите модуль архивирования и авторизованного пользователя(-ей). Подробные сведения см. в разделе rsyncd.conf руководства клиента rsync или здесь.

Примечание.

Для модуля резервирования и ресурса папки общего доступа с одинаковыми именами используется настройка модуля резервного копирования.

Включение ограничения скорости

Позволяет управлять максимальной шириной используемой полосы резервного копирования rsync для пользователей и групп.

Включение ограничения скорости:

  1. Выберите Панель управления > Файловые службы > rsync. Выберите один из следующих вариантов:
    • Нет ограничения скорости
    • Включить ограничение скорости
    • Включить ограничение скорости согласно расписанию (выберите Настроить расписание для дальнейшего изменения.)

    Примечание.

    Эта настройка применяется на всех серверах и переопределяет настройки отдельных пользователей и групп. Например, если выбран параметр Нет ограничения скорости, ограничения скорости для пользователей и групп отсутствуют.

  2. Если ограничение скорости включено, нажмите Настройки ограничения скорости и выберите пользователя/группу для изменения настроек:
    • Нет. Ограничение скорости не действует, когда участники группы передают файлы с помощью этой службы.
    • Применить настройки группы: Пользователь и связанная группа используют одинаковые настройки ограничения скорости.
    • Постоянная. К отдельным пользователям или участникам группы будет применяться фиксированное постоянное ограничение скорости. В полях слева можно указать ограничения скорости при выгрузке и загрузке файлов.
    • Настраиваемая: К отдельным пользователям или участникам группы применяются два различных ограничения скорости, которые включаются по расписанию. Нажмите Настроить, чтобы изменить настройки ограничения скорости и настроить расписание.
  3. Нажмите Применить.

Примечание.

В разделе Результаты (UL/DL) отображается фактическое ограничение скорости передачи и загрузки пользователя с учетом индивидуальных и групповых настроек. Например, если для пользователя "Achilles" установлено индивидуальное ограничение скорости загрузки 100 КБ/с, но этот пользователь является участником группы с ограничением скорости загрузки 500 КБ/с, в разделе Результаты (UL/DL) будет отображаться значение "0/500".

Назначить права доступа NFS

Вы можете назначить права доступа NFS к любой папке общего доступа, позволяя пользователям Linux получать к ней доступ.

Назначение прав доступа NFS к папке общего доступа:

  1. В списке папок общего доступа выберите папку общего доступа, которую необходимо изменить.
  2. Нажмите Редактировать > Права доступа NFS.
  3. Нажмите Создать, чтобы добавить правило для NFS.
  4. Появится всплывающее окно. Укажите в нем следующие параметры.
  5. Имя хоста или IP-адрес: Введите IP-адрес клиента NFS, который будет иметь доступ к папке общего доступа. Имя хоста можно указать тремя разными способами:
    • Отдельный хост: Полное доменное имя или IP-адрес.
    • Групповые символы: *, *.synology.com
    • IP-сети: 203.74.205.32/255.255.252.0, /24
  6. Права доступа: Выберите для клиента NFS права на чтение и запись.
  7. Безопасность: Укажите необходимые флаги безопасности.
    • AUTH_SYS: Используйте UID (идентификатор пользователя) и GID (идентификатор группы) клиента NFS для проверки прав доступа.
    • Проверка подлинности Kerberos: Выполнение проверки подлинности Kerberos при подключении клиента NFS к папке общего доступа. Клиент может осуществить доступ к папке общего доступа только после выполнения проверки подлинности Kerberos.
    • Целостность Kerberos: Выполнение проверки подлинности Kerberos и обеспечение целостности пакетов при передаче данных.
    • Конфиденциальность Kerberos: Выполнение проверки подлинности Kerberos и защита пакетов NFS при передаче данных. Это позволяет предотвратить воздействие несанкционированных лиц на трафик NFS.
  8. Squash: Данное поле позволяет управлять правами доступа пользователей клиента NFS. Выберите один из следующих параметров:
    • Без сопоставления: Позволяет всем пользователям клиента NFS, включая привилегированных пользователей, сохранять исходные права доступа.
    • Сопоставить привилегированного пользователя с администратором: Назначение прав доступа привилегированным пользователям клиента NFS, эквивалентные правам доступа администратора системы.
    • Сопоставить привилегированного пользователя с гостем: Назначение прав доступа привилегированным пользователям клиента NFS, эквивалентные правам доступа гостя системы.
    • Сопоставить всех пользователей с администратором: Назначение прав доступа всем пользователям клиента NFS, эквивалентные правам доступа администратора системы.
  9. Включить асинхронизацию: Данный параметр позволяет устройству DiskStation отвечать на запросы клиентов NFS до завершения изменений файлов, что позволяет повысить производительность.
  10. Разрешить подключения с непривилегированных портов (выше 1024): Выбор данного параметра позволяет клиентам NFS использовать непривилегированные порты (т.е. порты выше 1024) при подключении к DiskStation.
  11. Разрешить пользователям доступ к подключенным подпапкам： Выбор данного параметра позволяет клиентам NFS осуществлять доступ к подключенным подпапкам.
  12. Для завершения нажмите OK.
  13. Нажмите кнопку OK, чтобы применить права доступа NFS.

Примечание.

  • Если имя сервера имеет формат *.domain, IP-адрес клиента NFS должен иметь соответствующую запись DNS PTR, чтобы устройство DiskStation могло находить имя *.domain, выполняя поиск соответствующего IP-адреса.

Флаги безопасности:

При доступе к папке общего доступа с помощью NFS с помощью определенной учетной записи:

  • Если внедрен флаг безопасности AUTH_SYS: Клиент должен иметь точно такой же числовой UID (идентификатор пользователя) и GID (идентификатор группа) на клиенте NFS и на DiskStation, иначе клиенту будут присвоены права доступа others при доступе к папке общего доступа. Во избежание конфликта разрешений выберите Сопоставить всех пользователей с admin в пункте Squash или предоставьте к папке общего доступа разрешения "Все".
  • Если внедрен флаг безопасности Kerberos (krb5, krb5i, krb5p): Выберите Win/Mac/NFS > Служба NFS Service > Настройки Kerberos > Сопоставление идентификатора для сопоставления клиента NFS с определенным пользователем, либо присоединения домена Windows /LDAP с соответствующей учетной записью пользователя; в противном случае клиенту будут присвоены права guest при доступе к папке общего доступа.
  • Если файловой системой внешнего устройства, на котором создана папка общего доступа, является система NTFS или FAT, параметр Сопоставить всех пользователей с администратором будет использован принудительно.

Для использования флагов безопасности Kerberos для подключения к DiskStation необходимо настроить проверку подлинности Kerberos в пункте Win/Mac/NFS > Служба NFS > Настройки Kerberos.

DDNS

DDNS (Dynamic Domain Name Service) позволяет упростить подключение к DiskStation по Интернету с помощью сопоставления имени хоста с IP-адресом. Например, DDNS предоставляет доступ к DiskStation с помощью имени домена (например, www.john.synology.me) без необходимости запоминания IP-адреса (например, 172.16.254.1).

Настройка DDNS

Вы можете указать существующее имя хоста IP-адреса устройства DiskStation, либо зарегистрировать новое имя, предоставленное Synology или другими провайдерами DDNS. Для получения дополнительной информации о регистрации имени хоста обратитесь к соответствующему провайдеру.

Настройка имени хоста DDNS:

  1. Нажмите кнопку Добавить.
  2. Отобразится диалоговое окно с предложением изменить следующие настройки:
    • Поставщик услуг: Выберите поставщика услуг. Для регистрации бесплатного имени хоста, предоставленного компанией Synology, выберите Synology в раскрывающемся меню, нажмите Зарегистрироваться сейчас и введите сведения в соответствии с подсказками.
    • Имя хоста: Введите зарегистрированное имя хоста DDNS, например john.synology.me.
    • Имя пользователя: Введите имя пользователя провайдера DDNS.
    • Пароль: Введите пароль провайдера DDNS.
    • Внешний адрес: Введите внешний IP-адрес устройства DiskStation, для которого будет использоваться имя хоста.
    • Heartbeat: Включите данный параметр для получения предупреждений о состоянии сопоставленного имени хоста (только для имен хостов, предоставленных компанией Synology).
  3. Нажмите Тестирование подключения для проверки правильности настроек.
  4. Нажмите OK, чтобы сохранить и завершить.

После завершения настройки вы сможете выполнить доступ к DiskStation по Интернету, введя имя хоста DDNS в веб-браузере.

Требования и ограничения

  • Серверы поставщика услуг DDNS работают нормально.
  • DiskStation может подключиться к Интернету.
  • DDNS будет работать только в сетевой среде, в которой не требуется прокси-сервер.
  • Вы можете ввести только одно имя хоста для каждого провайдера DDNS.

Управление именами хостов

Используйте следующие кнопки для управления существующими именами хостов:

  • Удаление: Удаление выбранного имени хоста из списка.
  • Редактирование: Изменение параметров выбранного имени хоста.
  • Обновить сейчас: Обновление состояния всех имен хостов, для которых включена служба DDNS.

Настройка пользовательского поставщика DDNS

Если ваш провайдер не появился в списке поставщиков услуг, вы можете добавить пользовательского поставщика DDNS.

Настройка пользовательского поставщика DDNS:

  1. Нажмите Настроить.
  2. Отобразится диалоговое окно с предложением ввести следующие настройки:
    • Поставщик услуг: Введите название пользовательского поставщика услуг.
    • URL-адрес запроса: Введите URL-адрес пользовательского поставщика услуг.
  3. Нажмите Сохранить, чтобы добавить пользовательского поставщика услуг в список поставщиков DDNS.

Требования и ограничения

  • URL-адрес запроса должен относиться к протоколу, указанному вашим поставщиком услуг. Данный URL-адрес используется для обновления информации об IP-адресе на DDNS-сервере вашего поставщика.
  • Как правило, URL-адрес запроса включает в себя четыре нижеприведенных параметра. При вводе URL-адреса запроса замените следующие параметры переменными величинами.
    • Имя пользователя: __USERNAME__
    • Пароль: __PASSWORD__
    • Имя хоста: __HOSTNAME__
    • Внешний адрес: __MYIP__
  • В URL-адресе запроса не должно быть пробелов или точек с запятой.

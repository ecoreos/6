Управление трафиком маршрутизатора

Управление трафиком маршрутизатора включено, когда DiskStation работает в режиме маршрутизатора. Управление трафиком маршрутизатора (в разделе Панель управления > Сеть > Управление трафиком маршрутизатора) позволяет контролировать трафик операций выгрузки и загрузки, выполняемых в сети DiskStation. При создании правил потребуется указать Гарантированная ширина пропускания и Максимальная ширина пропускания для ограничения трафика выгрузки и загрузки.

Объяснение терминов

  • Гарантированная ширина пропускания: задает исходящий трафик, который данная служба гарантированно предоставляет при достаточном значении полосы пропускания всей системы.
  • Максимальная ширина пропускания: определяет исходящий трафик, который может использоваться службой при достаточном значении ширины пропускания всей системы и наличии Остаточной ширины пропускания системы. Мы предлагаем сначала выполнить расчет Исходящей ширины пропускания системы. Затем следует убедиться, что Гарантированная ширина пропускания для каждой службы и приложения не превышает Исходящую ширину пропускания системы, иначе настройка может работать неправильно.

Формула настройки

  • Исходящая ширина пропускания системы = общей Гарантированной ширине пропускания всех служб + Остаточная ширина пропускания системы
  • Гарантированная ширина пропускания каждой службы ≤ Максимальной ширины пропускания каждой службы

Примечание.

Если исходящие пакеты не соответствуют правилам управления трафиком маршрутизатора, ограничение установлено не будет. Вы можете добавить собственное правило управления трафиком маршрутизатора по умолчанию в конце всех правил, выбрав Все порты и указав параметры Гарантированная ширина пропускания и Максимальная ширина пропускания.

Для создания правила управления трафиком маршрутизатора выполните следующие действия.

  1. Выберите сетевой интерфейс из раскрывающегося списка в правом верхнем углу.
  2. Нажмите Создать.
  3. В разделе Порты выберите один из вариантов:
    • Выберите Все, чтобы применить правило ко всем портам.
    • Выберите Пользовательский и нажмите Пользовательский. Введите не более 15 портов, разделяя их запятой, либо укажите диапазон портов, к которым необходимо применить это правило.
    • Можно также указать протокол.
  4. В разделе Исходный IP-адрес выберите один из вариантов:
    • Выберите Все, чтобы применить это правило ко всем IP-адресам.
    • Выберите Определенный IP-адрес и нажмите Выбрать. Можно указать IP-адреса или диапазон IP-адресов, к которым необходимо применить это правило.

Теперь вы можете просматривать созданное правило управления трафиком маршрутизатора в списке правил.

Чтобы удалить правила управления трафиком маршрутизатора, выполните следующие действия.

  • Снимите флажок правила в столбце Включено списка.
  • Удалите правило управления трафиком маршрутизатора.

Примечание.

  • Вы можете создать до 100 правил для каждого сетевого интерфейса.
  • Изменить порядок правил можно путем перетаскивания.
  • Приоритет правил соответствует их позициям в списке.
  • Указанные порты относятся к номерам портов интернет-служб за пределами локальной сети.
  • Исходные IP-адреса относятся к IP-адресам устройств, использующих Synology NAS в качестве маршрутизатора.

Домен

DiskStation можно присоединить к домену Windows. После присоединения к домену пользователи домена могут выполнять вход в DiskStation с помощью учетных записей и паролей домена, что позволяет им получать доступ к файлам и использовать приложения DSM без необходимости запоминания другого имени пользователя и пароля.

Чтобы присоединиться к домену, выполните следующие действия:

  1. Выберите Панель управления > Служба каталогов > Домен.
  2. Установите флажок Присоединиться к домену.
  3. Введите имя домена и укажите DNS-сервер в соответствующих полях.
  4. При необходимости установите флажок Дополнительные опции домена. (Дополнительную информацию см. ниже.)
  5. Нажмите Применить.
  6. Во всплывающем окне введите имя пользователя и пароль администратора на сервере домена.
  7. Нажмите OK.

DNS Server

Рекомендуется указать DNS-сервер для помощи DiskStation при поиске домена. Однако если домен не зарегистрирован на DNS-сервере, процедура присоединения к домену может закончиться неудачно.

Тип сервера домена

В этом поле отображается тип домена при присоединении DiskStation к домену. Домен может иметь тип "Домен AD".

Режим управления

Данный параметр определяет, как управлять правам пользователей и групп домена.

  • Доверенный домен: вы можете управлять пользователями и группами в присоединенном домене, а также в доверенных доменах. Пользователей и группы можно фильтровать в раскрывающемся списке домена.
  • Один домен с OU: в этом режиме отображаются только пользователи и группы. Этот режим позволяет отфильтровать пользователей домена/группы домена с помощью раскрывающегося списка OU (подразделений).

Дополнительные опции домена

В большинстве случаев дополнительные опции домена заполнять не требуется. Дополнительные опции домена необходимы только для определенных сред домена.

  • IP/FQDN контроллера домена: укажите IP-адрес или FQDN контроллера домена, и DiskStation выполнит попытку установить с ним связь. Если есть несколько IP-адресов или FQDN, разделяйте их запятыми. По желанию добавьте звездочку (*) после последнего IP-адреса или FQDN контроллера домена, и DiskStation выполнит попытку установить связь с другим контроллером домена, если связь с указанными контроллерами закончится неудачно. Необходимо добавить запятую между звездочкой и последним IP-адресом или FQDN.
  • Имя NetBIOS домена: укажите имя NetBIOS домена.
  • Домен FQDN (имя DNS): укажите FQDN (имя DNS) домена.
  • Зарегистрировать интерфейс DNS: При присоединении к домену указанная плата сетевого интерфейса будет зарегистрирована на сервере DNS. Обратите внимание: если имя хоста DiskStation содержит символ подчеркивания (_), регистрация не будет выполнена, так подчеркивания (_) не могут использоваться в DNS.
  • Обновлять список пользователей/групп: укажите, как часто DiskStation будет автоматически обновлять список пользователей/групп домена. Можно настроить выполнение обновлений ежедневно, еженедельно или ежемесячно. Кроме того, списки пользователей/групп доменов можно обновить вручную на вкладке Пользователь домена или нажав Обновить все домены. Обратите внимание, что автоматические обновления влияю на спящий режим системы.

Подключение к контроллеру домена только для чтения

Для присоединения DiskStation к контроллеру домена только для чтения (RODC) требуется указать полное имя компьютера (FQDN) контроллера домена для чтения и записи(RWDC) в поле DC IP/FQDN. Полное имя компьютера должно иметь следующий формат: ИмяКомпьютера.Домен, например, "ad01.synology.com".

Примечание.

  • Если ваше имя пользователя домена содержит символы "%" или "$", вы можете не получить доступ к своей главной папке. Попросите администратора домена предоставить вам новое имя пользователя.
  • Группы доменов Администраторы домена и Администраторы предприятия будут автоматически добавлены в локальную группу administrators. Как следствие, у пользователей домена в этих группах будут права администратора на DiskStation, включая доступ к файловым службам (CIFS/FTP/AFP/WebDAV) и другим приложениям DSM.
  • Вы также можете настроить права доступа пользователей домена к папкам общего доступа на DiskStation. Дополнительную информацию можно найти здесь.

Параметры домена

В данном разделе содержатся следующие дополнительные опции домена.

  • DiskStation будет выполнять синхронизацию с сервером NTP при каждом входе пользователя домена: Включение этого параметра приведет к принудительной синхронизации времени между DiskStation и NTP-сервером.
  • Включить подпись сервера: Если на клиенте включена подпись сервера, подпись сервера также должна быть включена на DiskStation для исправной работы служб домена.
  • Получить списки пользователей/групп в NT4-совместимом режиме: Включение этого параметра позволяет получить списки пользователей/групп в режиме NT4 RPC. Этот параметр можно включить, если определенные списки пользователей/групп домена не могут быть получены с использованием настроек по умолчанию.
  • Отключить права администратора для администраторов домена или корпоративных администраторов: Пользователь с правами администратора может установить полный контроль над DiskStation и его файлами. Включение этого параметра позволяет ограничить права администраторов домена или корпоративных администраторов, чтобы они совпадали с правами обычных пользователей, то есть настроить наследование всех прав доступ к файлам/папкам обычного пользователя.
  • Перечислить уровни вложенных групп: Указать количество участников вложенных групп домена, которые могут быть пронумерованы.

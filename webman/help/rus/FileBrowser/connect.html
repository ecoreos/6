Удаленное подключение

Используя File Station, можно подключаться к публичным облачным службам (Cloud Services) (Box, Dropbox, Google Drive и Microsoft OneDrive) и файловым серверам с помощью различных протоколов (FTP, STFP, WebDAV и WebDAV HTTPS).

Подключение к облачной службе:

  1. Выберите Инструменты > Удаленное подключение > Настройка подключения.
  2. Выбрав облачную службу, нажмите значок, а затем кнопку Далее.
  3. Выполните инструкции мастера для аутентификации учетной записи, которые отобразятся во всплывающем окне.
  4. После завершения настройки искомая облачная служба появится в левой панели.

Ограничения, связанные с облачными службами:

  • Box.
    • К каталогу, используемому совместно с другими сотрудниками, будет добавлен суффикс в виде имени пользователя владельца.
    • Если каталоги, используемые совместно с другими сотрудниками, имеют одинаковое имя (имя каталога с суффиксом в виде имени пользователя владельца), то будет отображаться только один такой каталог.
    • Только при помощи службы Box открываются файлы с такими расширениями: .gdoc, .gsheet и .boxnote.
    • Имена файлов не зависят от регистра.
    • Правильные настройки брандмауэра см. в данной статье.
  • Dropbox.
    • Каталог Dropbox не отображается, если в нем содержится более 25 000 файлов.
    • Каталог Dropbox нельзя переименовать, если в нем содержится более 10 000 файлов.
    • Невозможно отправлять файлы размером более 10 ГБ.
    • Задача отправки не будет выполнена, если отправка целевого файла занимает более 24 часов.
    • Имена файлов не зависят от регистра.
    • Правильные настройки брандмауэра см. в данной статье.
  • Google Drive.
    • Файлы/папки с именем более 2048 символов не будут отображаться.
    • Файлы с именем более 255 символов не будут отображаться.
    • На файлах/каталогах с одинаковыми именами будет указано время их создания.
    • При открытии Google-файла (к примеру, Google Docs или Google Slides) с помощью File Station будет отображаться URL-адрес целевого файла.
    • Папки My Drive и Starred в File Station соответствуют папкам "Мой диск" и "Важные" в Google Диске.
    • Правильные настройки брандмауэра см. в данной статье.
  • Microsoft OneDrive:
    • Файлы записной книги OneNote не отображаются.
    • Невозможно отправлять файлы размером более 10 ГБ.
    • Для файлов (например, .png, .jpg, .docx, .pdf) может отображаться больший размер (исходный размер плюс размер миниатюры). Это не влияет на размер исходных файлов при загрузке.
    • Имена файлов не зависят от регистра.

Подробнее узнать об ограничениях можно на официальном веб-сайте конкретной облачной службы.

Подключение к файловому серверу:

  1. Выберите Инструменты > Удаленное подключение > Настройка подключения.
  2. Чтобы выбрать протокол сервера, нажмите значок, а затем кнопку Далее.
  3. Чтобы завершить процесс установки, укажите следующую информацию.
    • Имя хоста или IP-адрес: Укажите имя хоста или IP-адрес целевого файлового сервера.
    • Порт. Укажите номер порта целевого файлового сервера.
    • Пользователь и Пароль. Введите эти идентификационные данные, если целевой файловый сервер запрашивает имя учетной записи/пароль.
    • Кодовая страница. Чтобы имена файлов отображались корректно, выберите кодировку символов целевого файлового сервера.
    • Имя профиля. Введите уникальное имя, чтобы легко идентифицировать данный файловый сервер.
    • Использовать WebDAV HTTPS-подключение: Данной функцией можно воспользоваться только при настройке протокола WebDAV, когда и клиент, и файловый сервер поддерживают WebDAV HTTPS.
    • Разрешать только одно подключение на пользователя. Данной функцией можно воспользоваться только при настройке протокола FTP или SFTP. Установите флажок на данном варианте для оптимизации соединения, если у сервера есть ограничения по количеству подключений.
  4. Нажмите Применить, чтобы сохранить настройки. Целевой файловый сервер появится на панели слева.

Редактирование существующего профиля облачной службы или файлового сервера:

  1. Выберите Инструменты > Удаленное подключение > Список подключений.
  2. Нажмите целевую облачную службу или файловый сервер, чтобы выполнить следующие действия.
    • Редактирование. Нажмите Редактировать, измените текущую конфигурацию профиля и щелкните OK.
    • Отключение. Чтобы прервать соединение, нажмите Отключить.
    • Повторное подключение. Чтобы возобновить соединение, нажмите Подключить. (Для повторного подключения можно также нажать на целевую службу или сервер на левой панели File Station.)
    • Удаление. Нажмите Удалить, а затем подтвердите намерение выйти из профиля. (Для удаления можно также нажать на целевую службу или сервер на левой панели File Station, а затем выбрать Удалить.)

Примечание.

  • Каждый пользователь может одновременно подключаться не более чем к 100 облачным службам (Cloud Services) и файловым серверам.
  • Во избежание непредвиденных ошибок выбирайте правильную кодовую страницу сервера. Если вы не знаете кодовую страницу сервера, рекомендуется кодировка Unicode.
  • Некоторые функции File Station (например, Сжать в, Извлечь, Предварительный просмотр и др.) недоступны для файлов, расположенных в облачных службах (Cloud Services) и на файловых серверах.
  • Индивидуальные скорости передачи для каждого подключенного файлового сервера или облачной службы могут изменяться в зависимости от настроек ограничений скорости.
  • Каждый пользователь может выполнять одновременно не более трех передач файлов между всеми облачными службами (Cloud Services) и файловыми серверами.
  • Для настройки прокси-сервера для WebDAV, WebDAV HTTPS и облачных служб (Cloud Services) выберите Панель управления > Сеть > Общее > Прокси-сервер. Если вы изменили настройки прокси-сервера при подключении к облачной службе или файловому серверу, отключитесь и заново подключитесь к ним.
  • Изменение информации о подключенных облачных службах (Cloud Services) или файловых серверах не повлияет на состояние соединения.
  • Чтобы сделать доступным ресурс подключения, облачная служба или файловый сервер автоматически отключается, если к содержащимся файлам в течение более 30 минут не осуществляется доступ. Щелкните облачную службу или файловый сервер, чтобы восстановить подключение.
  • Система не может отобразить файлы/папки с именами следующих форматов:
    • Начинающиеся с ._ или .SYNOPPSDB
    • Включающие /
    • С именем, совпадающим с .    ..    '    "    .DS_Store    @eaDir    @sharebin    @tmp    .SynologyWorkingDirectory
  • Удаленное подключение поддерживается только в режиме доступа высокого уровня. Чтобы использовать данную функцию, откройте пакет Docker в DSM хост-сервера, выберите Docker DSM, для которого необходимо включить зашифрованные общие папки, выберите Редактировать и установите флажок Использовать контейнер с правами доступа высокого уровня.

Резервирование Synology NAS

Обзор

Выход из строя жестких дисков или природные катастрофы представляют потенциальную угрозу потери самых важных данных, особенно при отсутствии надежного резервного копирования. Hyper Backup представляет собой интуитивно понятное и интегрированное решение для резервирования и восстановления, упрощающее сохранение/извлечение копий важных данных и восстановления конфигураций приложения и системы на определенный момент времени. Поддерживаются следующие хранилища для резервных копий:

  • Локальные общие папки (включая внешние устройства хранения, подключенные к DiskStation)
  • Удаленное устройство Synology NAS
  • Удаленные серверы rsync
  • Удаленные серверы OpenStack Swift
  • Публичные облачные хранилища (Amazon S3, Microsoft Azure, OpenStack Swift и др.)

Примечание

Hyper Backup создает резервные копии только определенных конфигураций системы и приложений. Подробнее см. справочную документацию по Hyper Backup.

1. Создание задач локального резервирования в локальной общей папке или на внешнем устройстве хранения

Инструмент Hyper Backup предлагает похожие процедуры резервирования и восстановления для разных типов целевых устройств. Ниже показано, как выполнить резервирование и восстановление данных локально. Этот пример можно использовать для других типов задач.

  1. Выполните вход в DSM и найдите пакет Hyper Backup.
  2. Нажмите + в левом нижнем углу и выберите Задача резервирования данных.
  3. Для настройки локальной задачи резервирования выберите Локальная папка общего доступа.
  4. Выберите Создать задачу резервирования и выберите локальную папку общего доступа или внешнее устройство хранения, подключенное к DiskStation.
  5. Выберите исходные папки для резервного копирования хранящихся в них данных. Выбранные зашифрованные папки общего доступа не отключаются в процессе резервного копирования. Если источник и место назначения находятся в одном томе, система отметит такие исходные папки оранжевыми восклицательными знаками.
  6. Выберите приложения для резервирования.
  7. Укажите следующие настройки резервирования в соответствии со своими потребностями:
    • Задача: присвойте имя задаче резервирования.
    • Включить уведомления о задачах: выберите способ получения уведомления о состоянии задачи: электронная почта, SMS-сообщения или push-служба.
    • Включить резервирование конфигурации: резервирование конфигураций системы (например, учетных данных пользователей). Этот параметр приводится в исполнение во всех задачах резервирования по умолчанию.
    • Сжатие данных резервного копирования: сжатие резервных данных для увеличения свободного пространства в хранилище. При активации этой опции потребуется больше времени на создание резервных копий
    • Включить шифрование при передаче (только для удаленного резервирования): Выберите для шифрования данных резервирования во время передачи в удаленное место назначения (например, сервер rsync или публичное облако).
    • Размер части при отправке несколькими частями (только для совместимых с S3 серверов): Разделить данные резервирования по указанному размеру блока.
    • Включить шифрование на стороне клиента: Выберите, чтобы защитить данные резервирования паролем от несанкционированного доступа на целевой стороне.
    • Включить резервирование по расписанию: Выберите, чтобы запланировать время автоматического выполнения задачи резервирования.
  8. Выберите Включить ротацию резервных копий, чтобы настроить схему ротации:
    • С самых ранних версий: Удалить самые ранние существующие версии резервирования при достижении указанного значения параметра Число версий.
    • Smart Recycle: Система будет сохранять все версии резервных копий, пока не будет достигнуто указанное количество версий. При запуске ротации система сначала выполняет ротацию версий, которые не удовлетворяют ни одному из условий; если все существующие версии отвечают приведенным ниже условиям, система выполнит ротацию самой ранней версии:
      • Ежечасные версии за последние 24 часа: Система хранит самую раннюю версию из создаваемых ежечасно.
      • Ежедневные версии от 1 прошедшего дня до 1 месяца: Система хранит самую раннюю версию из создаваемых ежедневно.
      • Еженедельная версия, созданная ранее 1 месяца назад: Система хранит самую раннюю версию из создаваемых еженедельно.
    • Количество версий: Установите максимальное количество сохраненных версий резервирования.
    • Временная шкала: Показывает возможный результат хранения резервных копий в соответствии с расписанием и схемой ротации:
  9. Нажмите кнопку Применить для завершения настройки.
  10. Для немедленного выполнения задачи резервирования следуйте инструкциям мастера резервирования и нажмите Да.

2. Восстановите файлы из локальной папки общего доступа/внешнего устройства хранения

  1. Откройте Hyper Backup, выберите нужную задачу резервирования на панели слева и нажмите Проводник резервирования (значок увеличительного стекла). Введите пароль, если задача защищена шифрованием на стороне клиента.
  2. Найдите и выберите нужный файл для восстановления. Также можно выбрать нужную версию для восстановления с помощью временной шкалы.
  3. Щелкните Восстановить, чтобы восстановить выбранный файл/выбранную папку.

3. Полностью восстановите задачи в нужные моменты времени

  1. Нажмите Восстановить в левом нижнем углу и нажмите Данные. Введите пароль, если задача защищена шифрованием на стороне клиента.
  2. Выберите задачу, которую необходимо восстановить. Для восстановления данных из других репозиториев выберите Восстановить из других репозиториев и найдите задачу для восстановления.
  3. Вы можете восстановить конфигурации системы. При выборе команды Восстановление системной конфигурации выберите определенные конфигурации для восстановления в приведенном ниже списке.
  4. Найдите и выберите целевой файл/целевую задачу для восстановления. Также можно выбрать нужную версию для восстановления с помощью временной шкалы.
  5. Выберите приложения для восстановления.
  6. Появится основная информация о восстановлении.
  7. Нажмите Применить, чтобы начать восстановление.

Восстановление

На этой странице системные администраторы могут выполнять восстановление. Чтобы разрешить пользователям доступ к моментальным снимкам общих папок, включите опцию Сделать моментальный снимок видимым для каждой общей папки. Пользователи получат доступ к снимкам через приложение File Station или Проводник Windows.

Общая папка

Восстановление отдельных файлов/папок в общей папке:

  1. Выберите общую папку, которую необходимо восстановить.
  2. Нажмите Восстановить.
  3. Выберите версию моментального снимка, из которого необходимо выполнить восстановление.
  4. Нажмите Обзор и скопируйте файлы/папки, которые необходимо восстановить.

Восстановление общей папки:

  1. Выберите общую папку, которую необходимо восстановить.
  2. Нажмите Восстановить.
  3. Выберите версию моментального снимка, из которого необходимо выполнить восстановление.
  4. Нажмите Действие и выберите в раскрывающемся списке Восстановить на месте.
  5. Появится сообщение с подтверждением. Нажмите Да для подтверждения действия.

Клонирование снимка в новую общую папку:

  1. Выберите общую папку, которую необходимо клонировать.
  2. Нажмите Восстановить.
  3. Выберите версию моментального снимка, которую вы хотите клонировать в новую общую папку.
  4. Нажмите Действие и выберите в раскрывающемся списке Клонировать с новым именем.
  5. Вы будете перенаправлены на страницу Клонирование в меню Панель управления > Общая папка > Создать. Введите имя клонированной общей папки и нажмите OK для завершения.

Примечание.

  • Общую папку "homes" восстановить нельзя.
  • Клонированные общие папки не могут быть зашифрованы.
  • Если моментальный снимок создан во время передачи файлов, целостность данных не гарантируется.
  • Ограничения на зашифрованные общие папки:
    • Для подключенных зашифрованных общих папок недоступен ни один тип восстановления данных.
    • Параметр Клонировать с новым именем для зашифрованных общих папок недоступен.
    • Отключенные защищенные общие папки могут быть восстановлены по сделанным ранее моментальным снимкам при помощи опции Восстановить на месте.

Переключение функций репликации общей папки:

  1. Выберите реплицированную общую папку, для которой необходимо выполнить переключение функций.
  2. Нажмите Переключение функций.
  3. Выберите новый целевой сервер при наличии нескольких задач репликации общей папки.
  4. Нажмите Переключение функций, чтобы начать действие.

Отработка отказа репликации общей папки:

  1. Выберите реплицированную общую папку, для которой необходимо выполнить отработку отказа.
  2. Выберите Отработка отказа > Отработка отказа.
  3. Выберите версию моментального снимка для восстановления.
  4. Нажмите Отработка отказа, чтобы начать действие.

Примечание.

  • Отработку отказа также можно инициировать в случае сбоя сервера-источника.

Повторная защита репликации общей папки:

При возникновении сбоя на целевом сервере задача репликации остановит синхронизацию данных между двумя серверами. Вам потребуется выполнить повторную защиту задачи репликации после восстановления исходного сервера-источника.

  1. Выберите общую папку, в которой произошел сбой и для которой необходимо выполнить повторную защиту.
  2. Выберите Отработка отказа > Повторная защита.
  3. Выберите новый сервер-источник.
  4. Выберите новый целевой сервер при наличии нескольких задач репликации общей папки.
  5. Нажмите Повторная защита, чтобы начать действие.

Примечание.

  • К этому моменту данные реплицированной общей папки, существующие на новом целевом сервере, будут перезаписаны на новом сервере-источнике.
  • Если выбранная версия моментального снимка для отработки отказа не существует на партнерском сервере, повторная защита может занять больше времени.

Выполнение тестовой отработки отказа для реплицированной общей папки:

  1. Выберите реплицированную общую папку, которую необходимо протестировать.
  2. Выберите Тестовая отработка отказа > Тестовая отработка отказа.
  3. Выберите Проверка сервера при наличии нескольких задач репликации общей папки.
  4. УкажитеИмя тестовой отработки отказа общей папки и выберите версию снимка для тестирования.
  5. Нажмите Тестовая отработка отказа, чтобы начать действие.

Примечание.

  • Выполнение переключения функций/отработки отказа репликации приведет к очистке тестовой отработки отказа.
  • Вы не можете создать задачу репликации для общей папки тестовой отработки отказа.

Очистка тестовой отработки отказа для реплицированной общей папки:

  1. Выберите реплицированную общую папку, в которой необходимо выполнить очистку тестовой отработки отказа.
  2. Выберите Тестовая отработка отказа > Очистка тестовой отработки отказа.
  3. Установите флажок напротив тестовой отработки отказа, которую необходимо очистить.
    Примечание. Установите флажок Удалить данные тестовой отработки отказа, чтобы не сохранять общую папку, созданную во время тестовой отработки отказа.
  4. Нажмите Очистка тестовой отработки отказа, чтобы начать действие.

iSCSI LUN

Восстановление iSCSI LUN:

  1. Выберите iSCSI LUN для восстановления.
  2. Нажмите Восстановить.
  3. Выберите версию моментального снимка, из которого необходимо выполнить восстановление.
  4. Нажмите Действие и выберите в раскрывающемся списке Восстановить на месте.
  5. Появится сообщение с подтверждением. Нажмите Да для подтверждения действия.

Клонирование моментального снимка в новый iSCSI LUN:

  1. Выберите iSCSI LUN, который необходимо клонировать.
  2. Нажмите Восстановить.
  3. Выберите версию моментального снимка, которую вы хотите клонировать в новый iSCSI LUN.
  4. Нажмите Действие и выберите в раскрывающемся списке Клонировать с новым именем.
  5. Вы будете перенаправлены на страницу Клонирование в меню Диспетчер хранилища > iSCSI LUN. Введите имя клонированного iSCSI LUN и нажмите OK для завершения.

Примечание.

  • Если моментальный снимок создан во время передачи файлов, целостность данных не гарантируется.

Переключение функций репликации общей папки:

  1. Выберите реплицированный iSCSI LUN для восстановления.
  2. Нажмите Переключение функций.
  3. Выберите новый целевой сервер при наличии нескольких задач репликации iSCSI LUN.
  4. Нажмите Переключение функций, чтобы начать действие.

Отработка отказа репликации iSCSI LUN:

  1. Выберите реплицированный iSCSI LUN для отработки отказа.
  2. Выберите Отработка отказа > Отработка отказа.
  3. Выберите версию моментального снимка для восстановления.
  4. Нажмите Отработка отказа, чтобы начать действие.

Примечание.

  • Отработку отказа также можно инициировать в случае сбоя сервера-источника.

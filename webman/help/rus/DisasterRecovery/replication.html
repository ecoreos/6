Репликация

На этой странице системные администраторы могут управлять репликациями. Можно создать, редактировать и удалять задачи репликации. Также можно запустить/остановить синхронизацию и посмотреть информацию о задачах репликации.

Общая папка

Создание задачи репликации общей папки:

  1. Откройте вкладку Папка общего доступа.
  2. Нажмите Создать.
  3. Откроется мастер Создать задачу репликации. Выполните действия мастера для создания репликации папки общего доступа.

Примечание.

  • Общую папку "homes" нельзя реплицировать.
  • Зашифрованную общую папку нельзя реплицировать.
  • Общая папка должна располагаться в разделе Btrfs.
  • Можно создать репликации "один ко многим"/расширенные репликации для общей папки на различных целевых серверах. Однако можно создать только одну репликацию на один целевой сервер для каждой общей папки.

Удаление репликации общей папки:

  1. Выберите задачу репликации общей папки, которую необходимо удалить.
  2. Нажмите Удалить.
  3. Появится сообщение с подтверждением. Нажмите Удалить для подтверждения удаления.

Редактирование репликации общей папки:

  1. Выберите задачу репликации общей папки, которую необходимо отредактировать.
  2. Нажмите Редактировать.
  3. Откроется окно со следующими вкладками:
    • Партнерский сервер
    • Расписание
    • Хранение
  4. На вкладке Партнерский сервер можно редактировать сведения о партнерском сервере, включая имя хоста, порт, имя пользователя и пароль.
  5. На вкладке Расписание установите флажок в поле Включить расписание создания моментальных снимков и установите время и частоту создания моментальных снимков. Можно снять флажок, чтобы отменить запланированные моментальные снимки для общей папки.
  6. На вкладке Хранение выберите один из следующих вариантов:
    • Сохранять 256 последних снимков: По достижению максимального количества (256) моментальных снимков самые старые снимки будут перезаписаны более новыми.
    • Всегда сохранять моментальные снимки: По достижении максимального количества моментальных снимков инструмент "Моментальные снимки и репликация" прекратит создание новых снимков и не будет удалять старые.
    • Сохранять моментальные снимки в соответствии со следующей политикой: Можно настроить правила хранения для установленных диапазонов времени.
  7. Нажмите кнопку OK, чтобы применить новые настройки.

Примечание.

  • На исходном сервере можно редактировать только расписание задачи репликации.
  • Изменение параметров хранения повлияет на эквивалентные параметры на странице Моментальный снимок.

Просмотр информации о репликации общих папок:

  1. Выберите задачу репликации общей папки, которую необходимо посмотреть.
  2. Нажмите Информация.
  3. Откроется окно со следующей информацией:
    • Статус этой репликации.
    • Статистика этой репликации.

Запуск синхронизации репликации общих папок:

  1. Выберите репликацию общей папки, которую необходимо синхронизировать.
  2. Нажмите Запустить.
  3. Введите описание снимка, который нужно сделать.
  4. Нажмите ОК для подтверждения действия.

Завершение синхронизации репликации общих папок:

  1. Выберите репликацию общей папки, синхронизацию которой необходимо завершить.
  2. Нажмите Остановить.
  3. Появится сообщение с подтверждением. Нажмите Остановить для подтверждения действия.

iSCSI LUN

Создание задачи репликации для iSCSI LUN:

  1. Откройте вкладку iSCSI LUN.
  2. Нажмите Создать.
  3. Откроется мастер Создать задачу репликации. Выполните действия мастера для создания репликации iSCSI LUN.

Примечание.

  • iSCSI LUN должна быть создана как расширенный LUN.
  • Можно создать репликации "один ко многим"/расширенные репликации для iSCSI LUN на различных целевых серверах. Однако можно создать только одну репликацию на один целевой сервер для каждого iSCSI LUN.

Удаление репликации iSCSI LUN:

  1. Выберите задачу репликации iSCSI LUN, которую необходимо удалить.
  2. Нажмите Удалить.
  3. Появится сообщение с подтверждением. Нажмите Удалить для подтверждения действия.

Репликация "один ко многим"/расширенная репликация

Для общей папки/iSCSI LUN можно создать несколько репликаций для общей папки/iSCSI LUN на несколько целевых серверов, сформировав схему один ко многим. Кроме того, можно создать расширенную репликацию для реплицированной общей папки/iSCSI LUN, сформировав расширенную схему.

Примечание.

  • Любая задача репликации может иметь не более трех целевых серверов как в схеме "один ко многим", так и в расширенной схеме.

Политика хранения

Это идеальное решение по определению максимального количества версий моментальных снимков для экономии места в томе, но при этом, возможно, придется хранить снимки дольше.

В компании Synology принята политика хранения данных GFS (Grandfather-Father-Son). Можно настроить хранение максимального количества версий моментальных снимков в течение следующих диапазонов времени соответственно: каждый час, ежедневно, еженедельно, ежемесячно и ежегодно.

Data Protection Manager будет сохранять установленное максимальное количество моментальных снимков для каждого диапазона времени. При наличии нескольких версий снимка в каком-либо временном диапазоне сохраняется только более ранняя. Например, если политикой предусмотрено 10 еженедельных снимков, Data Protection Manager будет сохранять самый ранний снимок (если в течение недели создано несколько снимков) для каждой из последних 10 недель.

Более того, все версии моментальных снимков в течение одного часа с момента создания последнего снимка будут сохраняться по умолчанию, чтобы упростить поиск и восстановление из последних версий моментальных снимков. Также можно заблокировать моментальный снимок, чтобы предотвратить его автоматическое удаление согласно правилам хранения.

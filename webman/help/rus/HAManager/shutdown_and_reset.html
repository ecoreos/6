Выключение и сброс

Указанные ниже инструкции помогут выполнить выключение, удаление или сброс кластера high-availability.

Выключение кластера high-availability с помощью Synology High Availability:

Если вы хотите выключить кластер high-availability для переноса серверов в новое физическое местоположение или выполнения обслуживания, но не хотите удалять конфигурации кластера high-availability, выполните следующие действия.

  1. Выполните вход в DSM.
  2. Откройте Synology High Availability.
  3. Нажмите кнопку Управление.
  4. Выберите Отключение кластера HA.
  5. Отобразится окно с подтверждением. Нажмите кнопку Да для подтверждения.

Выключение кластера high-availability вручную:

Кластер high-availability можно также выключить вручную с помощью кнопок питания, расположенных на каждом сервере.

  1. Нажмите и удерживайте кнопку питания на одном сервере (активном или пассивном), пока не раздастся звуковой сигнал.
  2. После полного выключения одного из серверов подождите 30 секунд (серверу в сети требуется время для сохранения настроек).
  3. Нажмите и удерживайте кнопку питания на одном сервере, пока не раздастся звуковой сигнал.
  4. Дождитесь полного отключения сервера.

Включение серверов:

Чтобы снова включить серверы, включите активный и пассивный сервер одновременно (или в течение пяти минут). Если вы включите только один сервер, будет выполнена попытка включения другого сервера с помощью сообщения Wake-on-LAN.

Нештатное выключение:

После создания кластера high-availability будет автоматически активирован перезапуск после сбоя питания на Панели управления. При внештатном отключении серверов из-за сбоя питания они будут автоматически включены после восстановления питания.

Удалить кластер high-availability:

Чтобы освободить оба сервера и удалить кластер high-availability для объединения с другим сервером, выполните следующие инструкции.

  1. Откройте страницу Обзор.
  2. Нажмите кнопку Управление.
  3. Выберите Удалить кластер HA.

После подтверждения система выполнит удаление кластера high-availability. В поле состояния будет отображаться ход выполнения удаления. Когда кластер high-availability будет удален, вы будете перенаправлены на исходный IP-адрес активного сервера. Состояние на обоих серверах изменится на Отдельный.

Сброс неисправного кластера high-availability:

Если вы не можете получить доступ к DSM или Synology High Availability из-за ошибки разделения центра управления или других неисправностей, вы можете освободить кластер high-availability, нажав кнопку Reset, расположенную на серверах.

  1. Нажмите и удерживайте кнопку Reset (обычно расположенную на задней панели) на одном сервере, пока не раздастся звуковой сигнал.
  2. Система удалит кластер high-availability и состояние обоих серверов изменится на Отдельный
  3. При нажатии кнопки Reset также будут восстановлены некоторые настройки по умолчанию, такие как IP-адрес, DNS и пароль администратора, а также следующие:
    • Пароль администратора будет пуст.
    • Гостевая учетная запись будет отключена.
    • DHCP будет активирован на IP-адресе, DNS, шлюзе и других сетевых интерфейсах.
    • JumboFrames будет отключено, а значение MTU установлено на 1500.
    • Значение порта управления будет установлено на 5000/5001.
    • PPPoE будет сброшен до значения по умолчанию.
    • Автоблокировка IP-адреса не будет отключена (заблокированный IP-адрес не будет удален).
    • Link Aggregation будет отключено.
    • Правила брандмауэра будут отключены (правила не будут удалены).

Примечание.

При удалении кластера high-availability могут образоваться незаконченные операции записи данных. Для обеспечения полноты данных выполните репликацию данных на основе активного сервера, поскольку на пассивном сервере могут отсутствовать некоторые обновления данных.

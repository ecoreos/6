Мастер High Availability Manager

Данный мастер поможет пройти процесс создания и настройки кластера high-availability.

Запуск мастера

  1. Откройте High Availability Manager.
  2. Перейдите на страницу Обзор.
  3. Нажмите Создать кластер HA.

Примечание.

  • Завершите настройку в мастере настройки кластера high-availability за отведенный временной интервал. По истечении установленного времени процедуру настройки придется начать заново.
  • Чтобы определить, какой сервер является активным для кластера high-availability, рекомендуется загрузить приложение Synology DS finder и использовать его функцию звукового сигнала, вынуждающую активный сервер издавать короткий звуковой сигнал.

Настройка пассивного сервера

Назначьте сервер Synology, который будет пассивным сервером. При нормальных условиях пассивный сервер не предоставляет служб. Однако в случае неисправности службы можно перенести с активного сервера на пассивный.

  1. IP-адрес: введите IP-адрес пассивного сервера.
  2. Имя пользователя: укажите учетную запись пользователя из группы administrator на сервере назначения.
  3. Пароль: укажите пароль для учетной записи из группы administrator.

Примечание.

  • После создания кластера high-availability все данные на пассивном сервере будут удалены. Убедитесь, что на пассивном сервере нет важных данных.

Настройка подключения Heartbeat

Подключите два сервера друг к другу. Это подключение будет выступать в качестве подключения Heartbeat между двумя серверами, упрощая связь и обеспечивая репликацию данных с активного сервера на пассивный. Чтобы обеспечить качество подключения Heartbeat, при создании кластера high-availability система будет проверять, соответствует ли подключение следующим критериям.

  • Между двумя серверами нет других устройств с IP-адресами, обнаруженными в этом же широковещательном домене. Если подключение Heartbeat проходит через коммутаторы, убедитесь, что коммутаторы поддерживают многоадресную передачу данных и Jumbo Frame.
  • Установлено сетевое подключение с низким уровнем задержек. Время ответа на запросы ping между двумя серверами не превышает 1 мс, скорость передачи составляет хотя бы 500 Мбит/с. Для MTU будет автоматически задано значение 9000.
  • На обоих серверах используется одинаковый сетевой интерфейс. Например если один конец кабеля подключен к ЛВС 1 на одном сервере, второй должен быть подключен к ЛВС 1 на другом сервере.
  • На обоих серверах используется самый быстрый сетевой интерфейс. Если серверы оснащены дополнительными платами сетевого интерфейса 10GbE, подключение использует их.

Убедившись в соблюдении перечисленных выше условий, перейдите к разделу Интерфейс подключения HA (Heartbeat) и выберите сетевой интерфейс на серверах, который будет выступать в качестве подключения Heartbeat.

Примечание.

  • Настройки подключения Heartbeat создадут два набора IP-адресов (например, 169.254.1.1 и 169.254.1.2) для репликации данных с активного сервера на пассивный. Убедитесь, что эти два IP-адреса не используются другим оборудованием в сети.
  • Параметр VLAN ID на портах подключения Heartbeat DiskStation не поддерживается.

Настройка Link Aggregation в подключении Heartbeat или подключении для передачи данных

  • Чтобы настроить Link Aggregation в подключении Heartbeat или подключении для передачи данных, настройте Link Aggregation на двух или более портах ЛВС в разделе Панель управления > Сеть > Сетевой интерфейс до запуска мастера создания high availability. Убедитесь, что настройки Link Aggregation совпадают на обоих серверах (т.е. если вы настроили Link Aggregation на портах 1 и 2 активного сервера, необходимо также настроить Link Aggregation на портах 1 и 2 пассивного сервера).
  • Подключение Heartbeat поддерживает отработку отказа и балансировку нагрузки, но подключения для передачи данных зависят от выбранного режима Link Aggregation.

Настройка параметров сетевого интерфейса в кластере high-availability

В качестве интерфейса для подключения к внешней сети кластер high-availability будет использовать IP-адрес кластера high-availability. Этот IP-адрес кластера high-availability можно использовать для подключения, чтобы использовать службы DSM независимо от того, какой сервер Synology активен. Это устраняет необходимость подключаться к разным IP-адресам после переключения с активного сервера на пассивный.

  1. Имя сервера кластера: имя кластера high-availability. Дополнительную информацию об ограничениях имен см. в правилах присвоения имен (в разделе Справка DSM > Панель управления > Подключение > Сеть > Общее).
  2. Сетевой интерфейс: сетевой интерфейс, который будет использоваться кластером high-availability. Необходимо выбрать порт ЛВС, отличный от порта, назначенного для репликации данных на предыдущем шаге.
  3. IP-адрес: IP-адрес, который будет использоваться кластером high-availability. Кластер можно подключить через этот IP-адрес независимо от того, какой сервер активно предоставляет службы.
  4. Маска подсети: маска подсети, которая будет использоваться кластером high-availability.

Назначение VLAN ID подключению для передачи данных

  • Чтобы назначить VLAN ID подключению для передачи данных, перейдите в раздел Панель управления > Сеть > Сетевой интерфейс до запуска мастера создания high-availability. Убедитесь, что настройки VLAN совпадают на обоих серверах (т.е. если для ЛВС 1 на активном сервере задан параметр VLAN ID 1, для ЛВС 1 на пассивном сервере также необходимо задать VLAN 1).

Подтверждение настроек

Проверьте настройки, которые будут использоваться кластером high-availability. Система начнет связывать кластер high-availability после нажатия кнопки Применить.

Примечание.

  • При создании кластера high-availability все данные на активном сервере необходимо дублировать на пассивный сервер. Если подключение к пассивному серверу утеряно более чем за 30 минут до завершения исходного процесса репликации, пассивный сервер вернется в Автономный режим. В этом случае потребуется повторно привязать его и снова выполнить полную репликацию данных.

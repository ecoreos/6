Параметры сети

На данной странице содержится три вкладки, на которых отображаются параметры сети кластера high-availability, активного сервера-участника и пассивного сервера-участника.

Вкладка кластера HA

После создания кластера high-availability система создаст IP-адрес кластера high-availability для получения входящих подключений. При необходимости можно создать дополнительные IP-адреса кластера high-availability. IP-адреса активированных сетевых интерфейсов на активном сервере будут отображены на данной вкладке (сетевой интерфейс, используемый подключением Heartbeat не будет отображаться на данной странице, поскольку он будет использоваться для репликации данных между двумя серверами). На данной вкладке отображается следующая информация:

  • Сетевой интерфейс: Активированный сетевой интерфейс на активном сервере.
  • Подключение: Отображает, подключен ли интерфейс или нет.
  • IP-адрес: После создания кластера high-availability система создаст IP-адрес кластера high-availability для получения входящих подключений. Вы можете подключиться к кластеру high-availability с помощью данного IP-адреса, когда вы хотите выполнить вход в DiskStation Manager (DSM) независимо от того, какой сервер Synology является активным. Это предотвращает подключение к другим IP-адресам при переключении функций.
  • Маска подсети: Связанная маска подсети для IP-адреса кластера high-availability.

Редактирование IP-адреса и маски подсети:

  1. Выберите IP-адрес или маску подсети, которую необходимо отредактировать, и нажмите Редактировать или дважды нажмите поле, чтобы начать редактирование.
  2. Установите флажок Включить, чтобы включить или отключить интерфейс. Вы можете активировать несколько сетевых интерфейсов кластера, однако исходный сетевой интерфейс, который был настроен при создании кластера, не может быть отключен.

Изменение имени сервера кластера high-availability:

  1. Нажмите Изменение параметров сети для просмотра имени кластера high-availability и управления его настройками.
  2. Нажмите Применить для сохранения текущего имени кластера high-availability и настроек IP-адреса.
  3. Чтобы отменить выполненные изменения, нажмите Отмена.

Вкладки активного и пассивного сервера (отображаемые как имя сервера)

Данные вкладки отображают текущее состояние сетевых интерфейсов на активном и пассивном сервере. Подключение Heartbeat будет использовать один сетевой интерфейс на каждом активном и пассивном сервере. На данных вкладках отображается следующая информация:

  • Сетевой интерфейс: Физический сетевой интерфейс на сервере.
  • IP-адрес: Присвоенный IP-адрес для определенного сетевого интерфейса.
  • Маска подсети: Связанная маска подсети для определенного IP-адреса.
  • Описание: Отображает сведения о сетевом интерфейсе и доступности его IP-адреса. В данном поле отображаются следующие возможные условия:
    1. Доступность оригинального IP-адреса: Оригинальный IP-адрес, назначенный интерфейсу активного сервера, доступен для данных и служб.
    2. Недоступность оригинального IP-адреса: Оригинальный IP-адрес, назначенный интерфейсу пассивного сервера, недоступен.
    3. IP-адрес подключения Heartbeat, недоступный через внутрикорпоративную сеть: Оригинальный IP-адрес, автоматически назначенный подключению Heartbeat, недоступен для данных и служб.

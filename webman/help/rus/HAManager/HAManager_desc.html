Диспетчер High Availability

Термин "High Availability" обозначает схему расположения сервера, которая позволяет снизить частоту перебоев служб, вызванных неисправностью сервера. В Synology High Availability два сервера используются для создания "кластера high-availability", в котором один сервер играет роль "активного сервера", а другой — "пассивного." Для внедрения и создания кластера high-availability потребуется два совместимых сервера Synology.

Кластер high-availability позволяет использовать следующие компоненты и концепции:

  • Концепция: Для использования технологии Synology High Availability необходимо два сервера Synology, чтобы создать кластер high-availability. Один сервер будет играть роль "активного сервера", а другой — "пассивного". После создания кластера high-availability будет выполняться непрерывная репликация данных с активного на пассивный сервер. Все файлы на активном сервере будут дублироваться на пассивном сервере. В случае неисправности активного сервера пассивный сервер будет выполнять функции активного сервера, поскольку содержит копии всех данных активного сервера. Таким образом кластер high-availability будет работать в обычном режиме, снижая время простоя сервера. Для получения дополнительной информации о переключении функций см. Переключение функций
  • Активный сервер: Активный сервер управляет предоставлением всех служб. В случае возникновения ошибки на активном сервере вы или система сможете выполнить переключение функций на пассивный сервер. В этом случае предоставление услуг будет передано на пассивный сервер, который при этом станет активным.
  • Пассивный сервер: При нормальных условиях пассивный сервер находится в режиме ожидания. Данные на активном сервере непрерывно дублируются на пассивный сервер. В случае неисправности работы активного сервера вы или система сможете выполнить переключение функций на пассивный сервер.
  • Подключение Heartbeat: Активный и пассивный серверы кластера high-availability связаны выделенным частным сетевым подключением, называемым "подключением Heartbeat". После создания кластера high-availability подключение Heartbeat позволяет выполнять синхронизацию данных и репликацию между активным и пассивным сервером. Если подключение между серверами утеряно в течение определенного периода времени, система определит необходимость переключения функций.
  • Основной раздел хранения кластера high-availability: Раздел хранения на активном сервере станет основным разделом хранения на кластере high-availability. Данные будут реплицированы на раздел хранения пассивного сервера с помощью подключения Heartbeat.
  • Запасной раздел хранения кластера high-availability: Раздел хранения на пассивном сервере станет запасным разделом хранения на кластере high-availability. Данные будут непрерывно реплицироваться с раздела хранения активного сервера на раздел хранения пассивного сервера с помощью подключения Heartbeat. Это обеспечивает единообразие содержимого файлов при переключении функций.
  • Имя сервера кластера high-availability: Серверы в кластере high-availability имеют одно имя. В сети вы можете подключаться с помощью данного имени сервера, чтобы все функции выполнялись активным сервером.
  • IP-адрес кластера high-availability: Поскольку каждый активный и пассивный сервер в кластере high-availability имеет собственный IP-адрес, то чтобы избежать замены IP-адреса на другой во время переключения функций, кластер high-availability создаст общий IP-адрес для доступа. Для доступа к службам DSM независимо от того, какой сервер Synology выполняет роль активного сервера, подключение можно выполнить с помощью следующих IP-адресов.

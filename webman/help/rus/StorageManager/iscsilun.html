iSCSI LUN*

iSCSI — это служба сетевого хранения, которая облегчает передачу данных по локальным сетям или на большие расстояния. Сопоставив iSCSI LUN с целями iSCSI Target, клиенты смогут работать с хранилищами так же, как и с локальными дисками. iSCSI LUN (логический номер устройства) можно сопоставить с несколькими целями iSCSI Target для предоставления услуг хранения iSCSI. Выберите Диспетчер хранилища > iSCSI LUN для управления iSCSI LUN.

Примечание.

Количество поддерживаемых номеров iSCSI LUN зависит от модели DiskStation. Для просмотра подробных характеристик продукта см. www.synology.com.

DiskStation в настоящее время поддерживает iSCSI LUN следующих типов:

  • iSCSI LUN (обычные файлы): iSCSI LUN создается путем назначения для него части тома. При необходимости позже его размер можно расширить. Этот тип iSCSI LUN обеспечивает гибкость динамического управления емкостью с помощью функции Thin Provisioning, поддержки аппаратного ускорения VMware VAAI, Windows ODX, мгновенных снимков LUN и клонирования LUN.
  • iSCSI LUN (уровень блоков) — один LUN на RAID: iSCSI LUN создается с использованием всего места в томе. Вы можете восстановить или расширить LUN либо изменить его тип RAID, как при работе с томами. Для получения дополнительной информации см. страницу Том.
  • iSCSI LUN (уровень блоков) — несколько LUN на RAID: iSCSI LUN создается с помощью выделения части объема в группе дисков. Если на группе дисков еще имеется доступное пространство, iSCSI LUN может быть позже расширен. Для получения дополнительной информации см. страницу Группа дисков.
  • iSCSI LUN (уровень блоков): iSCSI LUN этого типа создается на RAID Group и обеспечивает гибкое динамическое управление емкостью и оптимизированную производительность доступа. С помощью нераспределенного места на RAID Group данный тип iSCSI LUN может быть позже расширен. Для получения дополнительной информации см. страницу RAID Group.

Примечание.

iSCSI LUN с расширенными функциями LUN доступно только в файловой системе ext4/Btrfs.

Создание iSCSI LUN:

  1. Нажмите Создать.
  2. Выберите тип iSCSI LUN, который хотите создать.
  3. Следуйте инструкциям мастера по выполнению процесса.

Примечание.

Дополнительные функции LUN можно активировать при создании iSCSI LUN в первый раз. Данный параметр не может быть изменен впоследствии.

Размер единицы распределения расширенного LUN можно оптимизировать для определенных функций сервера приложений, например Microsoft Exchange или SQL Server либо серверов виртуализации и технологий ускорения хранения, например Microsoft Windows ODX или VMware VAAI, выполнив следующие действия:

  • Установите размер единицы распределения во время создания первого расширенного LUN тома.
  • Установите размер единицы распределения в пункте Настроить перед созданием первого расширенного LUN тома.

Рекомендуется указать для вашего приложения размер единицы распределения при использовании прямого подключения с сервера приложений или способов работы RDM (pass-through) с расширенными iSCSI LUN на DiskStation. Чтобы использовать технологию расширенного LUN от компании Synology, для клонирования Windows ODX рекомендуется отформатировать том NTFS с размером единицы распределения, равной (или превышающей) размер единицы, который вы установили в DSM.

Примечание.

Единицу распределения расширенного LUN каждого тома можно установить только один раз.

Удаление iSCSI LUN:

При удалении iSCSI LUN все данные на iSCSI LUN будут удалены. Подключение между iSCSI LUN и любыми сопоставленными целями iSCSI Target также будет утеряно.

  1. Выберите iSCSI LUN для удаления.
  2. Нажмите кнопку Удалить и следуйте инструкциям мастера.

Примечание.

Удаление VAAI LUN на DiskStation не приведет к немедленному освобождению пространства на диске. Данные будут перемещены в резервную системную папку @EP_trash и будут удаляться постепенно. Например, если VAAI LUN размером 100 ГБ удалить из DiskStation, освобождение 100 ГБ в томе диска может занять примерно 15–20 минут.

Принудительное подключение iSCSI LUN (уровень блоков):

Если кэш на iSCSI LUN (уровень блоков) отсутствует, система не выполнит подключение iSCSI LUN (уровень блоков) при загрузке. Если кэш нельзя восстановить и вы хотите осуществить доступ к iSCSI LUN (уровень блоков), нажмите Принудительное подключение.

Примечание.

Данная функция доступна только в некоторых моделях.

Изменение iSCSI LUN

  1. Выберите iSCSI LUN для редактирования.
  2. Нажмите Редактировать.
  3. Измените необходимые свойства.
  4. Нажмите кнопку OK, чтобы сохранить изменения.

Примечание.

Избежать потери данных при редактировании емкости iSCSI LUN можно, увеличив эту емкость.

Создание мгновенных снимков iSCSI LUN:

Мгновенный снимок iSCSI LUN представляет собой копию iSCSI LUN на определенный момент времени, доступную только для чтения.

  1. Выберите iSCSI LUN, мгновенный снимок которого вы хотите сделать.
  2. Нажмите Мгновенный снимок и выберите Сделать мгновенный снимок.
  3. Введите описание мгновенного снимка.
  4. Выберите один из следующих типов мгновенных снимков в соответствии со своими потребностями:
    • С учетом используемого приложения. Данный тип мгновенных снимков доступен, только если на сервере Windows или VMware vCenter Server установлен Synology Snapshot Manager для обмена данными с DSM. Когда мгновенные снимки с учетом используемого приложения запущены на DSM, на приложения отправится уведомление и все данные из памяти будут перенаправлены на LUN, на котором выполнен мгновенный снимок, для обеспечения единообразия данных.
    • С учетом сбоя. Если ни на одном клиенте не установлен Synology Snapshot Manager, вы можете создавать моментальные снимки LUN только с учетом сбоя. При их запуске на приложения не будет отправлено уведомление по перенаправлению данных из памяти на LUN, поэтому согласованность данных не гарантируется.
  5. Нажмите OK.

Восстановление iSCSI LUN:

Выберите мгновенный снимок на определенный момент времени для восстановления данных на iSCSI LUN.

  1. Выберите iSCSI LUN для восстановления.
  2. Нажмите Мгновенный снимок и выберите Snapshot Manager.
  3. Выберите мгновенный снимок для восстановления и нажмите кнопку Восстановить.

Клонирование iSCSI LUN или мгновенного снимка:

Создайте копию мгновенного снимка iSCSI LUN или iSCSI LUN с возможностью записи.

  • Выберите iSCSI LUN из списка и нажмите Клонировать.
  • Выберите мгновенный снимок iSCSI LUN в меню Snapshot Manager и нажмите Клонировать.

Примечание.

  • В некоторых моделях потребуется перейти в Центр пакетов и установить Моментальные снимки и репликация для использования таких функций, как восстановление iSCSI LUN, создание и клонирование снимков iSCSI LUN.
  • Если есть хотя бы один подключенный сеанс iSCSI, связанный с LUN, который используется Windows Server или VMware ESXi Server, рекомендуется установить плагин Synology Snapshot Manager во избежание несогласованности данных. В противном случае необходимо остановить все приложения и выполнить выход из сеансов для инициаторов перед продолжением.
  • Функции LUN по созданию мгновенного снимка, восстановлению и клонированию доступны только на iSCSI LUN с расширенными функциями LUN.
  • Если после создания мгновенного снимка или клона состояние изменится на Ненормально/Повреждение, удалите данный мгновенный снимок (клон) и создайте другой.
  • Все изменения LUN после создания мгновенного снимка будут утеряны при восстановлении LUN с помощью данного мгновенного снимка.
  • Если состояние iSCSI LUN меняется на Недоступно, значит при доступе к iSCSI LUN произошли ошибки, связанные с данными. Во избежание серьезной потери данных в системе этот iSCSI LUN будет временно недоступен. Обратитесь в компанию Synology для получения поддержки при первой же возможности.

Глоссарий:

  • Thin Provisioning: Thin Provisioning — это способ оптимизации места хранения с помощью распределения места динамическим способом или по требованию. Этот параметр доступен только для iSCSI LUN (обычные файлы).
  • VMware VAAI: функция VAAI (VMware vStorage APIs for Array Integration) позволяет перенести выполнение стандартных операций с виртуальных массивов хранения на устройство iSCSI в среде VMware vSphere и оптимизировать производительность системы хранения.
  • Windows ODX: новая технология передачи данных, разработанная Windows в Windows Server 2012 и Windows 8, называется Offload Data Transfer (ODX). ODX повышает скорость передачи за счет перераспределения части нагрузки на массив хранения.
    Если источник и целевое местоположение данных находятся в LUN, расположенных в одном томе, клонирование будет выполняться с помощью технологии расширенного LUN компании Synology, которая позволяет экономить до 99,9% места хранения и одновременно ускоряет клонирование.
  • Сопоставление iSCSI Target: сопоставление iSCSI LUN с одной или несколькими целями iSCSI Target.
  • Synology Snapshot Manager: Synology Snapshot Manager — это плагин программного обеспечения, который позволяет создавать согласованные с приложениями мгновенные снимки непосредственно в DiskStation Manager (DSM).
    • В среде VMware необходимо установить Synology Snapshot Manager на сервер Windows Server, где установлен VMware vCenter Server. Когда мгновенный снимок запускается на DSM, на VMware vCenter Server отправится уведомление и все данные из памяти будут перенаправлены на LUN, на котором выполнен мгновенный снимок, для обеспечения единообразия данных. После выполнения мгновенного снимка VMware vCenter Server восстановит обычную работу входа/выхода хранилища данных VMware. Synology Snapshot Manager for VMware vCenter Server поддерживает vSphere 5.1 или более новые версии.
    • В среде Windows необходимо установить Synology Snapshot Manager на Windows Server. Когда мгновенный снимок запускается на DSM, Synology Snapshot Manager будет использовать технологию Microsoft Volume Shadow Copy Service (VSS) для создания единообразных копий данных на определенный момент времени. Synology Snapshot Manager for Windows поддерживает Windows Server 2008 R2, 2012 и 2012 R2.
    • Synology Snapshot Manager можно загрузить бесплатно на сайте www.synology.com.

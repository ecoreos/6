Том

Тома являются базовыми пространствами хранения, на которых пользователь может создавать общие папки, iSCSI LUN (обычный файл), сохранять данные и устанавливать пакеты. Перед созданием томов убедитесь в наличии созданного блока RAID Group. Дополнительные сведения о RAID Group см. в разделе RAID Group.

Создание тома:

  1. Убедитесь, что на устройстве DiskStation существует RAID Group с распределяемым объемом.
  2. Нажмите кнопку Создать.
  3. Выберите блок RAID Group, на которого необходимо создать том.
  4. Измените необходимые настройки.
  5. Выберите размер для тома. (Распределение размера недоступно для томов, созданных на RAID Group для одного тома или iSCSI LUN)

Примечания

  • Максимальный выделенный размер тома — 200 ТБ.
  • Минимальный выделенный размер тома Btrfs должен составлять не менее 10 ГБ.
  • Тома Btrfs сохранят 4% пространства для метаданных.

Удаление тома:

  1. Выберите том, который необходимо удалить.
  2. Нажмите кнопку Удалить и следуйте инструкциям мастера.

Примечание

  • Удаление тома приведет к удалению всех данных на указанном томе.

Принудительное подключение тома:

Если кэш в томе отсутствует, система не выполнит подключение тома при загрузке. Если кэш нельзя восстановить и вы хотите получить доступ к тому, нажмите Принудительное подключение.

Примечание

  • Данная функция доступна только в некоторых моделях.

Восстановление тома:

Если у тома состояние Ограниченная функциональность, перейдите на вкладку RAID Group и найдите соответствующий блок RAID Group. Следуйте инструкциям по восстановлению.

Расширение размера тома

  • Для томов, созданных на RAID Group для нескольких томов или iSCSI LUN (уровень блоков).
    1. Выберите том, который необходимо расширить.
    2. Нажмите Редактировать.
    3. Введите новый размер и нажмите OK для расширения.
  • Для томов, созданных на RAID Group для одного тома или iSCSI LUN (уровень блоков), см. раздел Расширение размера RAID Group в RAID Group, размер тома будет увеличен после расширения соответствующего RAID Group.

SSD TRIM

Если том состоит только из SSD (твердотельных накопителей), рекомендуется включить SSD TRIM. Данная функция позволяет повысить скорость чтения и записи томов, созданных на SSD, что позволит повысить эффективность и срок службы SSD.

Включение SSD TRIM:

  1. Откройте Диспетчер хранения и выберите пункт Том.
  2. Выберите том, полностью состоящий из твердотельных дисков, и нажмите кнопку Настроить.
  3. В открывшемся окне перейдите на вкладку SSD TRIM и установите флажок Включить TRIM.
  4. Нажмите кнопку Настроить расписание, чтобы установить, когда система будет выполнять SSD TRIM.
  5. Нажмите OK.

Примечания

  • Функция SSD TRIM доступна только на некоторых моделях DiskStation и типах RAID.
  • Некоторые модели SSD не поддерживают выполнение SSD TRIM на томах RAID 5 и RAID 6. См. список совместимости на веб-сайте www.synology.com.
  • Невозможно включить SSD TRIM в следующих случаях.
    • Том подключен с помощью кэша SSD.
    • Установлен пакет Time Backup.
    • Создан кластер High Availability с помощью High Availability Manager.
  • SSD TRIM невозможно включить на iSCSI LUN уровня блоков.
  • Производительность системы будет снижена при использовании функции SSD TRIM. Время, необходимое для выполнения TRIM, зависит от размера тома и модели SSD.
  • Если для тома включена функция TRIM, она также будет активирована на других томах, созданных в той же группе дисков или RAID Group.

Исправление ошибок данных файловой системы*

Исправление ошибок данных файловой системы — это функция обслуживания данных, проверяющая тома. При обнаружении несоответствий данных с контрольной суммой путь доступа к файлу будет записан в Центр журналов. Рекомендуется исправить их с помощью резервной копии.

Для обеспечения согласованности данных рекомендуется регулярно выполнять исправление ошибок данных.

Исправление ошибок данных файловой системы:

  1. Откройте Диспетчер хранения и выберите пункт Том.
  2. Выберите том, на котором следует выполнить исправление ошибок данных файловой системы.
  3. Нажмите Управление.
  4. Выберите Исправление ошибок данных файловой системы.

Примечания

  • Исправление ошибок данных файловой системы можно выполнить только на томах в файловой системе Btrfs.
  • Исправление ошибок данных файловой системы можно выполнить, только когда том находится в состоянии Нормальный.
  • Не отключайте систему до полного завершения исправления ошибок данных. Если отключить систему до завершения процесса, потребуется вручную перезапустить исправление ошибок данных файловой системы с самого начала.

Дефрагментация файловой системы*

Если данные постоянно изменяются, объем фрагментированных файлов в файловой системе будет расти, что приведет к снижению скорости доступа к файлам на жестких дисках. Дефрагментация файловой системы поможет повысить скорость.

Дефрагментация файловой системы

  1. Откройте Диспетчер хранения и выберите пункт Том.
  2. Выберите том, на котором следует выполнить дефрагментацию файловой системы.
  3. Нажмите Управление.
  4. Выберите Дефрагментация файловой системы.

Примечания

  • Дефрагментацию файловой системы можно выполнить только на томах в файловой системе Btrfs.
  • Дефрагментацию файловой системы можно выполнить, только когда том находится в состоянии Нормальный.
  • Не отключайте систему до полного завершения дефрагментации. Если отключить систему до завершения процесса, потребуется вручную перезапустить дефрагментацию файловой системы с самого начала.
  • Использование тома может увеличиться, если в нем хранятся моментальные снимки папок общего доступа.

* Данная функция доступна только в некоторых моделях.

Кэш SSD*

Кэш SSD может повысить производительность произвольного доступа за счет сохранения часто используемых данных в кэше SSD. Кэш SSD может быть подключен в разделе или iSCSI LUN (уровень блоков). Существует два типа кэша SSD.

  • Кэш только для чтения может состоять из 1 или 12 твердотельных дисков (SSD), смонтированных в базовую конфигурацию или конфигурацию RAID 0 соответственно. Этот тип кэша повышает производительность произвольного доступа для чтения в хранилище, где он смонтирован.

    Примечание.

    Кэш только для чтения на твердотельных дисках копирует данные из раздела, что позволяет избежать повреждения данных даже в случае аварийного сбоя кэша.

  • Кэш чтения-записи можно использовать в конфигурации RAID 1 / RAID 5 / RAID 6 в зависимости от количества твердотельных дисков (до 12 дисков) для повышения производительности произвольного доступа для чтения и записи хранилища, в котором он установлен.

Оба типа кэша SSD используют алгоритм LRU (наиболее давно использовавшиеся) для замены данных в кэше.

Важно!

Перед установкой твердотельных накопителей (SSD) для использования в качестве кэша обратите внимание на следующие примечания.

  • Цель использования кэша SSD заключается в повышении производительности произвольного доступа к небольшим частям данных, которые часто используются в пространстве хранения. Например, схемы объемных последовательных операций чтения или записи (такие как потоковая обработка HD-видео) и чтения полностью случайных данных испытывают нехватку схем повторного чтения, и, таким образом, не получают значительных преимуществ от использования кэша SSD. Для основных областей использования рекомендуется включить параметр Пропустить последовательный ввод/вывод, с тем чтобы эта последовательная операция ввода/вывода могла по-прежнему проходить через накопители пространства хранения.
  • Чтобы узнать, какие твердотельные накопители можно использовать в качестве кэша на устройстве DiskStation, ознакомьтесь со списком совместимости на веб-сайте Synology.
  • Кэш SSD требует определенного объема системной памяти в зависимости от размера кэша, поэтому установка кэша SSD большой емкости может потребовать увеличения памяти на устройстве DiskStation. Для стабильной и быстрой работы системы для кэша SSD можно использовать не более 1/4 системной памяти. Каждый 1 ГБ кэша SSD требует примерно 416 КБ системной памяти (можно учитывать расширяемую память)**. Поэтому если конфигурация кэша только для чтения составляет 2 x 128 ГБ SSD, общий размер кэша будет составлять 256 ГБ и потребует минимум 104 МБ памяти; если конфигурация кэша чтения-записи составляет 2 x 128 ГБ SSD, то общий размер кэша будет составлять 128 ГБ и потребует минимум 52 МБ памяти. Если памяти недостаточно, создание кэша SSD будет запрещено, либо его размер будет ограничен.
  • Каждый 1 ГБ кэша SSD требует примерно 416 КБ системной памяти (учитывается и расширяемая память), а максимальный размер кэша составляет 930 ГБ. Поэтому если конфигурация кэша только для чтения составляет 2 x 128 ГБ SSD, общий размер кэша будет составлять 256 ГБ и потребует минимум 104 МБ памяти; если конфигурация кэша чтения-записи составляет 2 x 128 ГБ SSD, то общий размер кэша будет составлять 128 ГБ и потребует минимум 52 МБ памяти. Если памяти недостаточно, создание кэша SSD будет запрещено, либо его размер будет ограничен.
  • Чтобы узнать максимальный размер кэша SSD, поддерживаемый на вашем устройстве, DiskStation, обратитесь к разделу характеристик продуктов на веб-сайте Synology.
  • При ограничении функциональности кэша SSD чтения-записи функция кэширования будет отключена и система начнет записывать (т. е. синхронизировать) кэшированные данные с SSD на HDD-диски. После завершения синхронизации данных даже в случае повреждения кэша чтения-записи данные не будут утеряны.
  • Кэш чтения-записи нельзя использовать вместе с Time Backup. Например при создании кэша SSD для чтения-записи вы не сможете запустить Time Backup. И наоборот, если вы используете Time Backup, вы не сможете создать кэш SSD для чтения-записи.

Подключение кэша SSD:

  1. Установите твердотельные накопители (SSD) в DiskStation. Сначала см. указанные выше примечания.
  2. Выберите Диспетчер хранения > Кэш SSD. Нажмите Создать.
  3. Выберите режим кэша (доступно только для моделей, поддерживающих кэш чтения-записи).
  4. Откройте раскрывающееся меню и выберите том или iSCSI LUN (уровень блоков), к которому необходимо подключить кэш SSD.
  5. В списке выберите твердотельные накопители (SSD). Данные на твердотельных накопителях (SSD) будут удалены, поэтому убедитесь, что на них нет важных данных.
  6. Нажмите Применить.

Отмена кэширования последовательного ввода/вывода:

Если целевой том или iSCSI LUN (уровень блоков) имеет высокую эффективность последовательного ввода-вывода, можно остановить для кэша SSD кэширование последовательного ввода-вывода и получить доступ к данным непосредственно с дисков для повышения скорости и производительности.

  1. Нажмите Настроить, чтобы открыть окно конфигурации.
  2. Установите флажок Пропустить последовательный ввод/вывод.

Удаление кэша SSD:

  1. Нажмите Удалить, чтобы полностью удалить кэш SSD с устройства DiskStation.
  2. Кэш чтения-записи следует удалять только в указанном порядке (даже если устройство DiskStation выключено). В противном случае произойдет сбой тома, поскольку частичные данные находятся только в кэше.

Примечание.

  • При подключении или удалении кэша SSD система остановит все соответствующие службы (папки общего доступа, установленные пакеты, текущие соединения и др.)
  • Если кэш чтения-записи в хранилище при загрузке не обнаруживается, система не выполнит подключение этого хранилища. Найдите и переустановите твердотельные накопители (SSD) на устройстве DiskStation и выполните перезагрузку для подключения емкости хранения. Если вы не можете найти твердотельные накопители (SSD), необходимо принудительно подключить хранилище. См. Том или iSCSI LUN.

Восстановление кэша SSD:

  • Для кэша чтения-записи: Если один из твердотельных накопителей (SSD) поврежден, выполните указанные ниже действия для его восстановления.
    1. Замените неисправный твердотельный накопитель (SSD) новым. (Новый твердотельный накопитель (SSD) должен быть той же модели и размера, что и старый.)
    2. Нажмите кнопку Восстановить.
    3. Выберите твердотельный накопитель (SSD) из списка.
    4. Нажмите Применить.
  • Для кэш только для чтения: Если поврежден один из твердотельных накопителей (SSD), удалите кэш и замените неисправный твердотельный накопитель (SSD). Нажмите Создать для создания новой конфигурации кэша.

Справка по твердотельным кэш-накопителям

Данная функция позволяет вычислить и просмотреть общий размер и количество файлов на определенном томе, к которым в последнее время осуществлялся доступ. Если вы еще не создали кэши SSD в выбранном томе, данную информацию можно использовать в качестве справки при оценке количества кэшей SSD для вашей системы. Если вы ранее создали кэш SSD в выбранном томе, здесь будут также отображаться анализ и использование кэша.

Использование Справки по твердотельным кэш-накопителям:

  1. Нажмите Справка по твердотельным кэш-накопителям, чтобы открыть мастер.
  2. Выберите том для расчета и нажмите Далее.
  3. Мастер автоматически начнет подсчет общего размера и количества файлов, к которым в последнее время осуществлялся доступ в выбранном томе. Время расчета может отличаться в зависимости от среды.

* Кэш SSD доступен только в некоторых моделях.
** Это неприменимо кэшам SSD для чтения-записи, созданным в DSM 5.1 и более ранних версиях. То есть для каждого 1 ГБ кэша SSD, созданного в версии более ранней, чем DSM 5.2, будет по-прежнему требоваться 4 МБ системной памяти даже при использовании в среде DSM 5.2.

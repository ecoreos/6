Общие

Диспетчер хранения позволяет пользователям управлять всем объемом хранения в продуктах Synology. Чтобы упростить управление хранилищем и создавать гибкие объемы хранения с защитой данных, Synology DSM поддерживает несколько типов блоков RAID Group.

RAID Group

DiskStation позволяет объединять несколько жестких дисков в единый блок хранения под названием RAID Group. DiskStation предоставляет следующие типы блоков RAID Group.

  • RAID Group для одного тома или iSCSI LUN (уровень блоков):
    • Выделяет всю доступную емкость для одного тома или iSCSI LUN (уровень блоков).
    • Обеспечивает лучшую производительность, но меньшую гибкость управления хранилищем.
    • Позволяет создавать несколько iSCSI LUN (обычные файлы) на томах.
    • Поддерживает не более одного массива RAID.
  • RAID Group для нескольких томов или iSCSI LUN (уровень блоков):
    • Поддерживает создание нескольких томов или iSCSI LUN (уровень блоков) в RAID Group.
    • Обеспечивает большую гибкость в управлении хранилищем.
    • Позволяет создавать несколько iSCSI LUN (обычные файлы) на томах.
    • Поддерживает объединение нескольких массивов RAID в составе RAID (при настройке как RAID 5 или RAID 6).

Тип RAID

При создании RAID Group сначала выберите тип RAID. Различные типы RAID обеспечивают разные уровни защиты данных, возможности хранилища и производительность. В настоящий момент DiskStation поддерживает RAID следующих типов.

  • Базовый. Использует один жесткий диск для создания RAID Group. Тип Basic для RAID Group не обеспечивает избыточности данных.
  • JBOD*. Объединяет не менее двух жестких дисков для создания RAID Group. Блоки JBOD RAID Group не обеспечивают избыточности данных. Доступная емкость JBOD RAID Group равна общей емкости всех дисков, входящих в группу. JBOD поддерживает объединение жестких дисков различных размеров.
  • RAID 0*. Объединяет не менее двух жестких дисков для создания RAID Group. RAID 0 не обеспечивает избыточности данных. Если один из дисков отказывает, данные в RAID Group будут утеряны. Доступная емкость RAID 0 Group равняется общей емкости всех составляющих дисков. RAID 0 поддерживает объединение жестких дисков различных размеров.
  • RAID 1*. Объединяет 2–4 жестких диска для создания RAID Group. Система записывает идентичные данные на каждый из жестких дисков. RAID 1 обеспечивает защиту данных, пока хотя бы один диск нормально работает. Доступная емкость RAID 1 Group равняется емкости наименьшего из составляющих дисков.
  • RAID 5*. Объединяет как минимум 3 жестких диска для создания RAID Group и разделяет данные и информацию о четности между всеми дисками. RAID 5 обеспечивает избыточность данных. При сбое одного из дисков систему можно перестроить с помощью информации о четности, хранящейся на других дисках. Поддерживает объединение нескольких массивов RAID при создании на RAID Group для нескольких томов или iSCSI LUN (уровень блоков). Доступная емкость RAID 5 Group = общая емкость объединенных массивов RAID. Емкость массива RAID = (количество жестких дисков – 1) x (емкость наименьшего жесткого диска)
  • RAID 6*. Объединяет как минимум 4 жестких диска для создания RAID Group и разделяет данные и информацию о четности между всеми дисками. RAID 6 обеспечивает избыточность данных. При сбое двух дисков систему можно перестроить с помощью информации о четности, хранящейся на других дисках. Поддерживает объединение нескольких массивов RAID при создании на RAID Group для нескольких томов или iSCSI LUN (уровень блоков). Доступная емкость RAID 6 Group = общая емкость объединенных массивов RAID. Емкость массива RAID = (количество жестких дисков – 2) x (емкость наименьшего жесткого диска)
  • RAID 10*. Объединяет не менее четырех жестких дисков для создания RAID Group. Количество дисков должно быть четным. RAID 10 имеет производительность RAID 0 и уровень защиты данных RAID 1. Доступная емкость RAID 10 Group = (емкость наименьшего жесткого диска) X (количество жестких дисков/2)

Примечание. Типы RAID, помеченные символом "*", доступны только в моделях NAS с достаточным количеством отсеков жесткого диска.

Тип тома

В зависимости от соответствующего типа RAID Group есть два типа томов.

  • Том на RAID Group для одного тома или iSCSI LUN (уровень блоков)
    • Выделяет всю емкость хранилища на RAID Group для одного тома. Обеспечивает лучшую производительность, но меньшую гибкость управления.
    • Позволяет создавать несколько iSCSI LUN (обычные файлы) на томах.
  • Том на RAID Group для нескольких томов или iSCSI LUN (уровень блоков)
    • Выделяет определенную долю емкости хранилища на RAID Group для создания тома. Обеспечивает более высокую гибкость управления, позволяя при необходимости расширять размер тома или создавать дополнительные тома.
    • Позволяет создавать несколько iSCSI LUN (обычные файлы) на томах.

Формат тома

В зависимости от используемой файловой системы есть два формата томов.

  • Новый формат тома: этот формат использует файловую систему Btrfs. Минимальный размер тома в этом формате должен быть не менее 10 ГБ. Формат поддерживает следующие новые функции, упрощающие управление данными.
    • Фотоснимок папки общего доступа. Фотоснимок записывает все состояния данных на момент съемки. Если пользователь случайно изменит или удалит данные в томе со снимками, вы сможете быстро восстановить данные на тот момент, когда был сделан снимок.
    • Квота папки общего доступа. Позволяет задать максимальную емкость для каждой папки.
    • Квота пользователя на основе папки общего доступа. Позволяет управлять тем, сколько места в папке общего доступа может использовать каждый пользователь.
    • Усиленная защита целостности данных. Папка общего доступа защищена контрольной суммой CRC32 и стратегиями копирования при записи.
  • Традиционный формат тома. Этот формат использует файловую систему ext4. Минимальный размер тома в этом формате должен быть не менее 10 ГБ. Так как файловая система ext4 поддерживается в большинстве продуктов Synology, вы можете с легкостью переносить том в этом формате в другой продукт Synology.

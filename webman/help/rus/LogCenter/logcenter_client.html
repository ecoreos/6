Отправка журнала

Если у вас есть сервер syslog, выполните приведенные ниже инструкции, чтобы настроить DiskStation в качестве клиента syslog и передавать системные журналы DiskStation на сервер syslog для централизованного мониторинга и управления.

Отправка журналов на сервер syslog:

  1. Выберите Центр журналов > Отправка журнала.
  2. Установите флажок Отправка журналов на сервер syslog.
  3. Введите имя хоста IP-адрес сервера syslog в поле Сервер.
  4. Укажите номер порта для получения сообщений syslog в поле Порт.
  5. Выберите UDP или TCP в поле Протокол передачи. Это значение должно совпадать с протоколом, используемым на сервере syslog.
  6. Выберите BSD (RFC 3164) или IETF (RFC 5424) в разделе Формат журнала для отправки журналов в выбранном формате.

    7. Примечание.

    • По умолчанию значение порта — 514. Эффективные правила переадресации порта необходимо установить при необходимости переназначения другого номера порта для сервера syslog.
    • UDP (User Datagram Protocol) — это более простой протокол подключений на основе сообщений, который работает быстрее, однако имеет низкий уровень безопасности и устранения ошибок. TCP (Transmission Control Protocol) предлагает "гарантированную доставку", в которой используется управление потоком для определения всех пакетов, успешно отправленных перед обработкой новых пакетов.
    • Защищенное соединение (SSL) доступно только при использовании протокола TCP.
    • Чтобы выбрать TCP и защитить соединение syslog с помощью протокола SSL, установите флажок Включить безопасное соединение (SSL). Затем нажмите Импортировать сертификат и выберите сертификат для безопасной передачи журналов на сервер syslog.
  7. Нажмите Отправить тестовый журнал, чтобы подтвердить получение журналов на сервере syslog. Если сервер syslog не получает тестовый журнал, проверьте правильность настроек.
  8. Нажмите Применить, чтобы сохранить настройки.

Фильтрация журналов

На вкладке Фильтрация можно указать, какие типы журналов отправляются на сервер syslog в соответствии с категорией (например, система, подключение, резервное копирование) или уровнем серьезности (например, предупреждение, ошибка, информация).

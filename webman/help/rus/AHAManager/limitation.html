Ограничения и требования к системе

Внедрение Synology High Availability требует два одинаковых сервера Synology для использования в качестве активного и пассивного сервера. Перед созданием кластера high-availability из двух серверов система выполнит проверку следующего оборудования и ПО для обеспечения возможности создания кластера high-availability.

Требования к системе

Обратите внимание на следующие замечания о программном обеспечении и оборудовании:

  • Сервер Synology: Активный и пассивный сервер должны быть одинаковыми.
  • Устройство(-а) расширения: Активный и пассивный серверы должны быть подключены к одним и тем же устройствам расширения правильно.
  • Количество и типы дисков: Допустимый тип диска — только SAS. Перед установкой DSM в устройство расширения необходимо установить не менее трех дисков.
  • Внешнее устройство USB: Поддерживаются только USB-накопители. Прочие внешние USB-устройства (например, ИБП, динамики, принтер, а также аппаратные ключи Wi-Fi, Bluetooth и DVB) не поддерживаются. USB-накопитель может использоваться только на активном сервере и не может переключаться на пассивный сервер.

Сетевая среда

Обратите внимание на следующие замечания о параметрах сети при создании кластера high-availability.

  • Параметры сети: Подключение Heartbeat между двумя серверами должно быть правильно настроено. Оба сервера должны принадлежать одной подсети. Также необходимо настроить статические IP-адреса после установки.
  • Количество портов локальной сети: Оба сервера должны иметь одинаковое количество портов локальной сети. Если одно из устройств DiskStation оснащено дополнительными интерфейсами сетевой платы, такие же интерфейсы должны быть и на втором устройстве.
  • Ограничение подключения: Сеть Wi-Fi не поддерживается кластером high-availability.

Примечание:

  • Сервер NTP будет автоматически включен на активном сервере.

Ограничения для пакетов и программного обеспечения

См. ограничения для пакетов и программного обеспечения ниже:

  • Кластер high-availability не поддерживает следующие пакеты и функции:
    • Сервер DVBLink
    • Функция перезагрузки/завершения работы кластера high-availability в CMS
    • Time Backup
  • Кластер high-availability с общим хранилищем нельзя настроить для работы в качестве кластера high-availability на основе репликации или использовать в сочетании с таким кластером.

Особые устройства расширения

Активный и пассивный серверы должны быть подключены к определенным устройствам расширения. На каждом устройстве расширения есть два модуля.

Настройка/добавление устройств расширения:

  • В модуле есть два порта: входной порт находится на левой стороне, а выходной — на правой. Подсоедините кабель SAS от выходного порта модуля к входному порту модуля следующего устройства расширения.
  • Устройства расширения должны быть подключены последовательно, с размещением модулей на соответствующих сторонах каждого из серверов.
  • Активный и пассивный серверы должны быть подключены к общим устройствам расширения в одинаковом порядке.

Примечания:

Для получения подробной информации об установке устройства расширения см. руководство по быстрой установке, поставляемое с серверами и устройствами расширения.

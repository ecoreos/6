Разделение центра управления

Если кластер high-availability работает исправно, только один сервер-участник должен выполнять роль активного сервера. В этом случае пассивный сервер обнаруживает присутствие активного сервера с помощью подключения Heartbeat и соединения для передачи данных с общими устройствами расширения, чтобы не запустилось переключение функций. Кроме того, пассивный сервер определяет состояние общего хранилища, чтобы исключить ошибочный переход на роль активного сервера, когда текущий активный сервер сохраняет доступ к общим устройствам расширения.

В случае утери подключения Heartbeat и всех соединений для передачи данных или нестабильного состояния общего хранилища и системы любой из двух серверов может играть роль активного. Данная ситуация называется разделением центра управления. В этом случае, чтобы избежать повреждения данных из-за одновременного доступа обоих серверов, один сервер попытается продолжить предоставление услуг, в то время как другой перезагрузится и затем потребует проверки соединений. В этой ситуации соединения с пользователями-клиентами могут быть нестабильны в течение некоторого времени.

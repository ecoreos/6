Восстановление

В разделе Резервирование и восстановление выберите Восстановить в левом нижнем углу, чтобы восстановить данные из резервных копий.

Восстановление данных/LUN* из резервной копии

  1. Нажмите Восстановить в нижнем левом углу, выберите Данные или LUN*, затем выберите задачу резервного копирования для восстановления.
  2. Вам будет предложено выбрать системную конфигурацию, различные версии резервных данных и др. Это зависит от типа задачи резервного копирования, которую вы хотите восстановить.
  3. Если задача зашифрована, для восстановления необходим пароль или ключ шифрования.
  4. Для завершения восстановления следуйте инструкциям мастера.

Примечание.

  • При восстановлении пакета, который не установлен, система автоматически установит ту же или более новую версию приложения. Если версия установленного приложения ниже версии резервного копирования, система автоматически выполнит обновление до исходной скопированной версии.
  • Приложения, скопированные в DSM 5.1 или выше, не будут восстановлены в DSM 5.0 или более ранней версии.

Примечание.

  • После завершения восстановления номера LUN на уровне блоков и файлов будут восстановлены в LUN на уровне файлов.*
  • При восстановлении пакета, который не установлен, система автоматически установит ту же или более новую версию приложения. Если версия установленного приложения ниже версии резервного копирования, система автоматически выполнит обновление до исходной скопированной версии.
  • Приложения, скопированные в DSM 5.1 или выше, не будут восстановлены в DSM 5.0 или более ранней версии.

Восстановление отдельного файла или папки для задачи резервного копирования с учетом нескольких версий:

  1. Выберите необходимую задачу резервного копирования на левой панели и запустите Проводник резервирования (щелкнув значок лупы). Если задача резервного копирования была зашифрована, необходим пароль или ключ шифрования.
  2. Найдите нужную версию резервной копии с помощью временной шкалы или значка календаря.
  3. Выберите нужный файл или папку и нажмите Восстановить, Копировать в... или Загрузить для извлечения. Эти три варианта доступны также при щелчке на файле или папке правой кнопкой мыши.

Поиск конкретной версии:

Чтобы ускорить поиск файла, можно использовать ключевые слова для фильтра. Не соответствующие поисковым словам файлы или папки будут скрыты.

  1. Запустите Проводник резервирования и щелкните значок воронки в верхнем правом углу.
  2. В раскрывающемся меню можно фильтровать файлы и папки по следующим параметрам.
    • Ключевое слово. Введите имя любого файла или папки. Вы можете использовать специальные символы [*] (один или несколько символов) и [?] (один произвольный символ) в ключевых словах. Например, [a*n] может означать [action], [attention] или [an].
    • Тип файла. Выберите один из вариантов: Любой, Любая папка или Любой файл.
    • Размер (КБ). Выберите Любой либо равен, более чем, менее чем, а затем укажите значение.
    • Дата изменения. Выберите Сегодня, Вчера, Последняя неделя, Последний месяц или Настроить и введите начальную и конечную дату для поиска версий в этом диапазоне.
  3. Чтобы применить фильтры, нажмите кнопку Фильтр.
  4. Чтобы применить эти фильтры к другой версии резервной копии, переключитесь на нужную версию на временной шкале.
  5. Чтобы сбросить поиск по фильтру, нажмите кнопку Сброс.

Примечание.

  • Проводник резервирования доступен только для задач резервного копирования нескольких версий.
  • Функции проводника ограничены, если место назначения резервного копирования находится на устройстве DSM 5.1 или более ранней версии.
  • Конфигурацию системы и приложения можно восстановить только посредством выбора параметра Восстановить.
  • Функция фильтрования доступна только для не зашифрованных задач резервного копирования.
  • Исходные настройки прав доступа для файлов и папок, восстановленных с помощью параметров Копировать в... или Загрузить, будут утеряны.

Восстановление задач резервного копирования из других мест назначения резервного копирования:

  1. Нажмите Восстановить в нижнем левом углу и выберите Данные или LUN*.
  2. Можно выбрать и восстановить один из двух типов задач резервного копирования.
    • Восстановление данных из резервной копии Выберите Данные > Восстановить из существующих репозиториев, затем выберите необходимое место назначения резервной копии, из которой следует восстановить данные.
    • Восстановление LUN из резервной копии Нажмите LUN*.
  3. Следуйте инструкциям мастера. Если задача резервного копирования была зашифрована, необходим пароль или ключ шифрования.
  4. Нажмите Применить, чтобы завершить восстановление.

Примечание.

  • Embedded DataStation может восстанавливать только общие папки на внешние устройства ext3/ext4.
  • Embedded DataStation не поддерживает восстановление приложений без внешнего устройства ext3/ext4.

____
*Доступно только в некоторых моделях.

Место назначения резервирования

На вкладке Место назначения резервирования в меню Резервирование и репликация перед созданием задачи резервного копирования можно задать локальное, сетевое или облачное место назначения резервного копирования.

Создание локального места назначения резервного копирования:

Место назначения локального резервного копирования необходимо при выполнении резервного копирования данных с DiskStation в локальную папку общего доступа, том или внешний диск USB/eSATA. Место назначения резервного копирования должно отличаться от источника, а его емкость должна превышать общий размер копируемых данных.

  1. Нажмите Создать, выберите Место назначения локального резервирования и нажмите Далее.
  2. Присвойте имя этому месту назначения резервного копирования, выберите место назначения для сохранения данных, а также укажите, следует ли сохранять только последнюю резервную копию или все версии резервного копирования.

Примечания.

  • Если данные в месте назначения удалены при выполнении задачи резервного копирования, будет выполнено полное резервное копирование при следующем выполнении задачи.
  • Перед выполнением резервного копирования на новые внешние диски их может потребоваться отформатировать.
  • Перед созданием задачи резервного копирования убедитесь, что к DiskStation подключены внешние жесткие диски. Выберите Панель управления > Внешние устройства для проверки состояния дисков USB/eSATA. Внешние диски не могут использоваться как места назначения резервного копирования, если внешний диск инициализируется, форматируется или неисправен.
  • Удаленные подключенные папки общего доступа не могут являться источниками или местом назначения резервного копирования.
  • Если зашифрованная папка общего доступа отключена, ее невозможно будет выбрать в качестве места назначения резервного копирования.
  • Эскизы изображений не могут копироваться, если файловой системой места назначения является NTFS/FAT.

Создание места назначения сетевого резервного копирования:

Место назначения сетевого резервного копирования необходимо при копировании данных по сети с DiskStation на другое устройство DiskStation или сервер, совместимый с rsync. Доступны различные параметры при создании задачи резервного копирования с местом назначения в сети, например зашифрованная передача, сжатие, резервное копирование на уровне блоков и др. Обратите внимание, что некоторые параметры, например зашифрованная передача, могут снизить скорость резервного копирования.

  1. Перед созданием места назначения сетевого резервного копирования выполните вход на целевой сервер, выберите Резервирование и репликация > Службы резервирования и включите службу сетевого резервного копирования и место назначения тома сетевого резервного копирования.
  2. На DiskStation выберите Резервирование и репликация > Место назначения резервирования. Нажмите Создать, выберите Место назначения сетевого резервирования и следуйте инструкциям мастера.
  3. С помощью пункта Место назначения сетевого резервирования (DiskStation) вы можете хранить резервные данные в общей папке для получения последних резервных копий или сохранения резервных данных в томе для хранения нескольких версий резервных копий. С помощью пункта Место назначения сетевого резервирования (rsync-совместимый сервер) вы можете сохранять только последнюю резервную копию.

Примечания.

  • При создании задачи резервного копирования с помощью rsync-совместимых серверов в качестве места назначения сетевого резервного копирования доступны следующие режимы. режим rsync daemon, который используется при выборе модуля резервного копирования (папка общего доступа на rsync-совместимом сервере) в раскрывающемся меню, расположенном рядом с пунктом Модуль резервирования; удаленный режим shell, который активируется только при вводе полного пути (начинающегося с "/") в Модуле резервирования.
  • Для пользователей Linux только учетные записи admin, root и rsync могут выполнять задачи резервного копирования.
  • Если к целевому серверу осуществляется доступ через Интернет с помощью имени хоста, вводимое имя хоста должно быть в формате FQDN (полное доменное имя), например abc.bcd.com.
  • При резервном копировании с одного устройства DiskStation на другое вам будет предложено ввести имя пользователя и пароль учетной записи в месте назначения. В частности, если вы хотите выполнить резервное копирование в том, расположенный на другом устройстве DiskStation, необходимо указать учетную запись, принадлежащую группе administrators. При выполнении резервного копирования в общие папки на другом устройстве DiskStation это можно сделать в соответствии с настройками прав доступа к папкам. Будут отображаться только общие папки, к которым пользователь имеет права доступа. Если вы не можете найти необходимую папку общего доступа в месте назначения, проверьте ваши права доступа.
  • Резервное копирование с шифрованием поддерживается только в том случае, если целевой сервер также поддерживает сетевое резервное копирование с шифрованием, а пароль для аутентификации не пустой.
  • Удаленный режим shell поддерживает только резервное копирование с шифрованием.
  • При выполнении резервного копирования на rsync-совместимый сервер вы не сможете изменить режим шифрования rsync после создания задач резервного копирования rsync. Чтобы изменить конфигурацию, необходимо создать новую задачу rsync.
  • Если резервное копирование файлов выполняется на устройство DiskStation через rsync-совместимый клиент, имена файлов должны иметь кодировку UTF-8.
  • Если резервное копирование файлов выполняется на устройство DiskStation через rsync-совместимый клиент, задачу смогут выполнить только пользователи DiskStation с учетными записями admin, root и rsync.
  • Если резервное копирование файлов выполняется с устройства DiskStation на rsync-совместимый клиент, вы должны будете ввести правильное имя учетной записи и пароль rsync-резервного копирования. Необходимо также выбрать правильный модуль резервного копирования с правами записи.

Создание места назначения резервного копирования Amazon S3:

Место назначения резервного копирования Amazon S3 необходимо, если вы собираетесь выполнить резервное копирование данных с DiskStation на сервер Amazon S3.

  1. Нажмите Создать, выберите Место назначения резервирования облака общего доступа, выберите Место назначения резервирования Amazon S3 и нажмите Далее.
  2. Присвойте имя месту назначения резервного копирования, введите учетные данные для входа и заполните все обязательные поля для настройки задачи резервного копирования.

Примечания.

  • При создании задач резервного копирования с помощью сервера Amazon S3 в качестве места назначения сетевого резервного копирования доступны различные параметры, например зашифрованная передача, резервное копирование на Amazon S3 RRS (Reduced Redundancy Storage) и др. Обратите внимание, что некоторые параметры, например зашифрованная передача, могут снизить скорость выполнения задач резервного копирования.
  • Файлы резервного копирования сохраняются в Bucket, выбранном на сервере Amazon S3, а значение set Key начинается с имени каталога и заканчивается полным путем к данному файлу.
  • Структура данных Amazon S3 представляет собой пару Bucket и KEY без структуры папок. Резервное копирование пустых папок не поддерживается.
  • Если резервный файл не был изменен, он не будет скопирован на сервер Amazon S3 в тот же Bucket.
  • Каждый Bucket на Amazon S3 должен иметь уникальное имя. Кроме того, имя должно:
    • содержать от 3 до 63 символов;
    • содержать только буквы нижнего регистра, цифры, точки (.) и дефисы (-);
    • начинаться с цифры или буквы, не может заканчиваться дефисом (-), не может содержать две точки подряд или дефисы рядом с точкой (например, имена "my-.bucket.com" и "my.-bucket" являются недопустимыми).
  • Максимальная длина ключа Amazon S3 KEY составляет 1024 символа. Чтобы успешно выполнить резервное копирование на Amazon S3, максимальная длина пути должна составлять 1024 символа, не считая имени папки места назначения, указанного при создании задачи.
  • При включении поддержки Amazon S3 RRS только данные, измененные после включения параметра, будут скопированы в хранилище RRS, то есть данные уже скопированные в стандартное хранилище Amazon S3 не будут снова скопированы в RRS.
  • Перемещение резервных копий с Amazon S3 на Amazon Glacier не поддерживается. Если файлы перемещаются с Amazon S3 на Amazon Glacier с помощью консоли AWS Management, может произойти сбой связанных задач резервного копирования.
  • Резервное копирование файлов на виртуальные Bucket-хосты по протоколу SSL не поддерживается.
  • При передаче файла он будет разделен на несколько частей. Лимит размера части можно установить с помощью опции Размер части при отправке несколькими частями в настройках задачи. Размер по умолчанию равен 512 МБ. Если сетевое подключение неустойчиво, рекомендуется установить меньшее значение.
  • Максимальное количество частей на отправку — 10 000. Максимальный размер файла определяется значением в поле Размер части при отправке несколькими частями, умноженным на 10 000.
  • Имена файлов не должны содержать символы, отличные от UTF-8, или символы управления.

Создание места назначения резервного копирования Microsoft Azure:

Место назначения резервного копирования Microsoft Azure требуется для резервного копирования данных с DiskStation на BLOB-сервер Microsoft Azure.

  1. Нажмите Создать, выберите Место назначения резервирования облака общего доступа, выберите Место назначения резервирования Microsoft Azure и нажмите Далее.
  2. Присвойте имя месту назначения резервного копирования, введите учетные данные для входа и заполните все обязательные поля для настройки задачи резервного копирования.

Примечания.

  • При создании задач резервного копирования с сервером Microsoft Azure в качестве места назначения сетевого резервного копирования доступны различные параметры (например, зашифрованная передача). Обратите внимание, что некоторые параметры, например зашифрованная передача, могут снизить скорость выполнения задач резервного копирования.
  • Файлы резервного копирования сохраняются в Контейнере, выбранном на сервере Microsoft Azure, а установленное имя BLOB-объекта начинается с имени каталога и заканчивается полным путем к данному файлу.
  • Структура данных Microsoft Azure представляет собой пару контейнера и BLOB-объекта без структуры папок. Резервное копирование пустых папок не поддерживается.
  • Если резервный файл не был изменен, он не будет скопирован на сервер Microsoft Azure в тот же контейнер.
  • Каждый контейнер на Microsoft Azure должен иметь уникальное имя. Кроме того, имя должно:
    • содержать от 3 до 63 символов;
    • содержать только буквы нижнего регистра, цифры и дефисы (-);
    • начинаться и заканчивать буквой или цифрой и не должно содержать два дефиса подряд.
  • Максимальная длина имени BLOB-объекта Microsoft Azure составляет 1024 символа. Чтобы успешно выполнить резервное копирование на Microsoft Azure, максимальная длина пути должна составлять 1024 символа, не считая имени папки места назначения, указанного при создании задачи.
  • Имена файлов не должны содержать символы, отличные от UTF-8, или символы управления.
  • Сообщение о сбое аутентификации может быть вызвано неправильной информацией об аутентификации, проблемой сетевого подключения или несовпадением времени на DiskStation и удаленном сервере. Для проверки настроек времени на DiskStation выберите Панель управления > Региональные параметры > Время для синхронизации времени с сервером NTP.

Управление местом назначения резервного копирования:

Выберите существующее место назначения резервного копирования и нажмите одну из приведенных ниже кнопок:

  • Редактировать. Изменение настроек выбранного места назначения резервного копирования.
  • Удалить. Удаление выбранного места назначения резервного копирования.

Резервное копирование в папку общего доступа

Этот тип места назначения резервного копирования будет сохранять только последнюю резервную копию. По умолчанию после завершения исходного резервного копирования файлы, удаленные из источника, удаляются из места назначения при следующем выполнении задачи. Однако если вы хотите сохранить файлы в месте назначения даже после их удаления из источника, включите параметр Зарезервировать архивированные файлы в месте назначения при создании задач резервного копирования.

Резервное копирование в том

Этот тип места назначения резервного копирования должен быть локальным или сетевым томом на устройстве DiskStation (за исключением внешних устройств хранения). Будет выполнено архивирование нескольких версий резервных копий, чтобы можно было легко восстановить любую папку в любое время.

Удаление данных из задачи резервного копирования:

  1. Перейдите на вкладку Место назначения резервирования и раскройте место назначения резервного копирования, содержащее задачу, которую необходимо удалить.
  2. Нажмите значок Удалить на выбранной Задаче резервирования и выберите Удалить в диалоговом окне.

Удаление данных из версии резервного копирования:

  1. Перейдите на вкладку Место назначения резервирования и раскройте место назначения резервного копирования, содержащее версию задачи, которую необходимо удалить.
  2. Нажмите значок Информация о версии резервирования на выбранной Задаче резервирования.
  3. Выберите версию резервного копирования, которую необходимо удалить, и нажмите кнопку Удалить.

Блокирование версии резервного копирования во избежание ее удаления при ротации резервных копий:

  1. Перейдите на вкладку Место назначения резервирования и раскройте место назначения резервного копирования, содержащее версию задачи, которую необходимо заблокировать.
  2. Нажмите значок Информация о версии резервирования на выбранной Задаче резервирования.
  3. Выберите версию резервного копирования, которую необходимо заблокировать, и нажмите значок Заблокировать.

Управление правами доступа места назначения резервного копирования:

  1. Выберите место назначения резервного копирования, связанное с локальным томом, и нажмите Редактировать.
  2. Нажмите Список разрешенных и выберите Добавить, чтобы добавить пользователя или группу в список.

Настройка места назначения резервного копирования с владельцем:

  1. Выберите место назначения резервного копирования, связанное с локальным томом, и нажмите значок Свойства на выбранной Задаче резервирования.
  2. В разделе Права доступа на резервирование в раскрывающемся меню выберите Владелец.

Разрешение пользователю или группе на DiskStation выполнять чтение и восстанавливать сохраненные данные задачи резервного копирования:

  1. Выберите место назначения резервного копирования, связанное с локальным томом, и нажмите значок Свойства на выбранной Задаче резервирования.
  2. Нажмите Список разрешенных и выберите Добавить, чтобы добавить пользователя или группу в список.

Примечания.

  • Владельцем задачи резервного копирования по умолчанию является пользователь, создавший ее.
  • Только владелец задачи резервного копирования может выполнять эту задачу. Однако задачи резервного копирования, созданные в версии более ранней, чем DSM 5.1, не имеют такого ограничения, поэтому любой пользователь, принадлежащий группе administrators, сможет выполнить задачу резервного копирования.
  • Владелец задачи и пользователи, принадлежащие группе administrators, могут удалять сохраненные данные в задаче резервного копирования.
  • Владелец задачи и пользователи, принадлежащие группе administrators, а также пользователи в списке разрешенных могут восстанавливать сохраненные данные в задаче резервного копирования.
  • Только пользователи, принадлежащие группе administrators, могут настраивать место назначения локального резервного копирования с правами доступа, а также задачу резервного копирования в месте назначения локального резервного копирования с владельцем и списком разрешенных.
  • Сохраненные данные задачи резервного копирования занимают квоту хранения владельца задачи.
  • Если том не имеет места назначения резервного копирования, только пользователи, принадлежащие группе administrators, могут создавать первое место назначения резервного копирования в томе.
  • Только пользователи, принадлежащие группе administrators, могут удалять сохраненные данные во всем месте назначения резервного копирования.
  • Каждое резервное копирование содержит системную конфигурацию и метаданные файла.
  • Резервное копирование зашифрованных папок общего доступа не поддерживается.
  • Блокировка версий не поддерживается в DSM 5.1 и более старых версиях.

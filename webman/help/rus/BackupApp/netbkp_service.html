Службы резервирования

Параметры на странице Резервирование и репликация > Службы резервирования позволяют другим пользователям выполнить резервное копирование данных из устройств DiskStation или rsync-совместимых клиентов на DiskStation.

Включение службы сетевого резервирования

Если вы хотите выполнить резервное копирование данных с одного устройства DiskStation или rsync-совместимого сервера на другое устройство DiskStation или разрешить другому устройству DiskStation выполнить задачи удаленного резервирования и сохранить резервные данные в томе устройства DiskStation, необходимо включить службу сетевого резервирования на сервере назначения, прежде чем создать задачу сетевого резервирования. Это можно сделать на странице Панель управления > Права доступа, предоставив пользователю право Место назначения сетевого резервирования. Затем можно создать пользователя DSM с соответствующим приложением и правами доступа к общим папкам в DiskStation. После завершения настройки другие люди смогут использовать права этого пользователя для сетевого резервирования и со своих устройств DiskStation в целевые общие папки на DiskStation.

Включение службы резервирования одной версии.

  1. Выберите Службы резервирования > Место назначения сетевого резервирования и установите флажок Включить службу сетевого резервирования.
  2. Нажмите Применить.
  3. Откройте Панель управления > Папка общего доступа, выберите общую папку, нажмите Редактировать > Права доступа и предоставьте пользователю право на Чтение/запись. Повторите этот шаг, чтобы добавить другие общие папки для использования в качестве целевых папок для архивирования.

Включение службы резервирования нескольких версий.

  1. Выберите Службы резервирования > Место назначения тома сетевого резервирования и установите флажок напротив параметра Активировать место назначения тома сетевого резервирования.
  2. Выберите Место назначения резервирования и выберите место назначения резервирования, связанное с локальным томом.
  3. Нажмите Редактировать > Список разрешенных > Добавить, чтобы добавить пользователя или группу в список.

Примечание.

  • Если другие пользователи захотят выполнить сетевое резервирование с другого клиентского устройства, а не с DiskStation, то они смогут сделать это только в том случае, если на вашем устройстве DiskStation им предоставлены права admin или rsync. Если вы не предоставили другим пользователям права admin, то можно открыть Панель управления > Пользователь, чтобы создать пользователя с именем rsync, изменить его права и предоставить его права другим пользователям.
  • Если вы хотите сохранить информацию о владельце и группе исходных данных при выполнении сетевого резервирования с DiskStation с версией DSM ниже 3.0 или с клиента, который не является DiskStation, необходимо добавить учетную запись резервирования в группу administrators и выполнить резервное копирование данных в папку общего доступа NetBackup.

Включение службы сетевого резервирования с помощью конфигурации rsync

Можно включить службу сетевого резервирования с помощью настраиваемой конфигурации rsync и настроить права пользователя и целевую папку для архивирования на вашем устройстве DiskStation. После завершения настройки другие люди смогут выполнять резервное копирование в режиме rsync со своих устройств DiskStation или клиентов, совместимых с rsync, в целевые общие папки (или "модули архивирования") на вашем устройстве DiskStation.

Включение службы сетевого резервирования с помощью конфигурации rsync:

  1. Выберите Службы резервирования > Место назначения сетевого резервирования.
  2. Установите флажок Включить службу сетевого резервирования.
  3. Установите флажок Использовать пользовательскую конфигурацию rsync.
  4. Нажмите Применить.

Порядок настройки прав пользователя и модулей архивирования в вашем устройстве DiskStation:

  1. Выполните вход в устройство DiskStation с помощью Telnet или SSH.
  2. В файле конфигурации rsync "/etc/rsyncd.conf" укажите модуль архивирования и авторизованного пользователя(-ей). Подробные сведения см. в разделе rsyncd.conf руководства клиента rsync или здесь.

Примечание.

  • При использовании настраиваемой конфигурации rsync DiskStation больше не сможет служить в качестве сервера назначения для Синхронизация папки общего доступа, Time Backup и Резервирование LUN.

Включение ограничения скорости

Можно включить ограничение скорости, чтобы управлять максимальной используемой шириной пропускания сети для пользователей и групп, передающих данные с помощью сетевого резервирования.

Включение ограничения скорости:

  1. Выберите Службы резервирования > Место назначения сетевого резервирования. Установите флажок Включить ограничение скорости или Включить ограничение скорости согласно расписанию. В последнем случае ограничение скорости будет применяться только в определенное время. Эта настройка применяется на всех серверах и переопределяет настройки отдельных пользователей и групп. Если выбран параметр Нет ограничения скорости, ограничения скорости для пользователей и групп отсутствуют.
  2. Нажмите кнопку Настройки ограничения скорости, чтобы изменить настройки для отдельных пользователей или групп. Для каждого пользователя или группы можно выбрать один из следующих параметров.
    • Применить настройки группы: Настройки ограничения скорости будут совпадать с настройками группы, к которой принадлежит пользователь.
    • Нет. Ограничение скорости не действует, когда участники группы передают файлы с помощью этой службы.
    • Постоянная. К отдельным пользователям или участникам группы будет применяться фиксированное постоянное ограничение скорости. В полях слева можно указать ограничения скорости при выгрузке и загрузке файлов.
    • Переменная. К отдельным пользователям или участникам группы применяются два различных ограничения скорости, которые включаются по расписанию. Нажмите кнопку Настроить, чтобы изменить настройки ограничения скорости и настроить расписание.
  3. Нажмите Применить.

Примечание.

При установке ограничения скорости для пользователя в разделе Результат отображается фактическое ограничение скорости пользователя с учетом индивидуальных и групповых настроек. Например, если для пользователя "Achilles" установлено индивидуальное ограничение скорости загрузки 100 КБ/с, но этот пользователь является участником группы с ограничением скорости загрузки 500 КБ/с, в разделе Результаты будет отображаться значение "0/500."

Включение службы Time Backup

Служба Time Backup позволяет пользователям архивировать несколько версий папок с одного устройства DiskStation на это устройство DiskStation дистанционно. Для этого требуется включить службу сетевого резервирования и службу Time Backup.

Включение службы сетевого резервирования и службы Time Backup:

Выберите Службы резервирования > Time Backup. Установите соответствующий флажок.

Примечание.

Только пользователю с учетной записью администратора разрешено создавать подключение к удаленной службе Time Backup для резервного копирования.

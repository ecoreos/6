Синхронизация общих папок

Служба синхронизации общих папок позволяет синхронизировать общие папки с одного продукта Synology (источник) на другой продукт Synology (целевые) в локальной сети или через Интернет. Служба работает на уровне общих папок. Если при выполнении синхронизации папка с указанным именем (например, SharedFolder) уже существует на целевом продукте Synology, папка будет переименована в "SharedFolder_1". Если такой папки нет, она будет создана в соответствующем разделе или первом доступном разделе. Для доступа к этой папке на целевом устройстве DiskStation будут использоваться права доступа по умолчанию, то есть для данной папки права доступа не будут указаны.

При первой синхронизации служба синхронизации общих папок выполняет полную синхронизацию выбранных папок. В дальнейшем синхронизироваться будут только измененные файлы. Файлы, которые были удалены из источника, удаляются также в месте назначения.

Примечание.

  • Синхронизация общих папок представляет собой способ односторонней синхронизации, то есть файлы на исходном устройстве будут синхронизированы с целевым устройством, но не наоборот. Если вам необходима двусторонняя синхронизация, используйте приложение Cloud Station.

Включение службы

Включение синхронизации общих папок:

Необходимо включить службу на целевом устройстве DiskStation. Для этого выберите Резервное копирование и репликация > Синхронизация общих папок. На вкладке Сервер установите флажок Включить службу синхронизации папки общего доступа для ее включения.

Подключение общих папок от исходного продукта Synology к целевому:

Можно синхронизировать общие папки с источника на различные другие серверы. Кроме того, начиная с DSM 4.1, имеется функция синхронизации общих папок из нескольких источников, которая позволяет нескольким клиентам выполнять синхронизацию общих папок на одно устройство DiskStation. Если несколько клиентов одновременно выполняют синхронизацию одной и той же папки на одно устройство DiskStation, это вызовет конфликт, при котором потребуется удалить связь с общей папкой, чтобы другой клиент мог выполнить ее синхронизацию. Чтобы удалить связь с общей папкой, нажмите вкладку Синхронизация общих папок > Сервер, выберите Список подключений, укажите подключение и нажмите Удалить связь на целевой стороне.

Теперь можно начать синхронизацию общих папок с исходного на целевое устройство DiskStation. Для продолжения следуйте инструкциям из приведенных ниже разделов.

Управление задачами синхронизации

С помощью Мастера синхронизации общих папок можно легко создать задачу синхронизации.

Чтобы создать задачу синхронизации:

  1. Выберите Резервное копирование и репликация > Синхронизация общих папок > Клиент.
  2. Нажмите Создать.
  3. Введите имя для задачи.
  4. Выберите общую папку, которую требуется синхронизировать с устройством назначения. Установите ниже флажок, чтобы подтвердить выбор, и для продолжения нажмите Далее.
  5. Настройте место назначения и следующие параметры синхронизации:
    • Включить шифрование при передаче: Шифрование данных при передаче. Этот параметр обеспечивает лучшую безопасность, в то время как нешифруемая передача более производительна. Вы можете выбрать параметр в соответствии со своими нуждами.
    • Включить сжатие при передаче: Сжатие данных при передаче. Этот параметр экономит пропускную способность сети, но повышает нагрузку на центральный процессор (CPU).
    • Включить синхронизацию на уровне блоков: Передача только отличающихся данных, вместо целых файлов. Этот параметр экономит пропускную способность сети, но повышает нагрузку на центральный процессор (CPU).
  6. Настройте расписание для задачи:
    • Запускать синхронизацию каждые [число] часов или Запускать синхронизацию каждые [число] минут: Запускать синхронизацию через определенные интервалы (в минутах или часах). Отсчет времени начинается с полуночи. Интервал не может превышать 24 часа.
    • Запускать синхронизацию автоматически ежедневно в (чч:мм): Запускать задачу синхронизации ежедневно в определенное время.
    • Запустить синхронизацию вручную: Запускать синхронизацию только вручную при необходимости.
    • Запустить синхронизацию изменений: Запускать синхронизацию сразу после любых изменений в исходной общей папке.
  7. Нажмите Применить. Теперь вы увидите задачу синхронизации в списке задач, которые будут выполняться в соответствии с установленным расписанием.

Управление задачей синхронизации:

  1. Выберите задачу в списке задач.
  2. Выполните одно из следующих действий:
    • Нажмите Изменить, чтобы изменить задачу.
    • Нажмите Удалить, чтобы удалить задачу.
    • Если задача синхронизации сейчас не выполняется, то нажмите Синхронизировать сейчас, чтобы выполнить эту задачу прямо сейчас.
    • Если задача синхронизации сейчас выполняется, то нажмите Отмена, чтобы остановить текущую задачу.

      • Примечание.

      Если для расписания задачи синхронизации выбрано значение Запустить синхронизацию изменений, то нажатие кнопки Отмена приведет к остановке текущей задачи синхронизации. Но если было изменено содержимое общей папки(-ок), контролируемой задачей синхронизации, то функция "Синхронизация общих папок" возобновит выполнение этой задачи.

    • Чтобы синхронизировать все данные, нажмите Полная синхронизация. Функция "Синхронизация общих папок" в первый раз проведет полную синхронизацию, а после первой синхронизации будет синхронизировать только измененные части. Нажмите эту кнопку, чтобы снова вручную синхронизировать все данные.

Дополнительная информация

  1. Название задачи должно состоять из 1–32 символов Unicode и не должно содержать следующие символы:
    ! " # $ % & ' ( ) * + , / : ; <= > ? @ [ ] \ ^ ` { } | ~
  2. Если задача включает более 32 000 папок, то при синхронизации общих папок из файловой системы ext4 в ext3 возможны ошибки.
  3. Если общие папки на целевом устройстве DiskStation были синхронизированы с другим устройством DiskStation ранее, синхронизация общих папок выполнит полную синхронизацию задачи.
  4. Для выполнения задач синхронизации общих папок необходимы права администратора (admin) для доступа к целевому устройству DiskStation. Пользователь должен иметь пароль учетной записи администратора (admin) либо быть членом группы администраторов этого устройства.
  5. При синхронизации на устройстве DiskStation через Интернет можно ввести имя сервера либо в виде его внешнего IP-адреса, либо его имени хоста. Однако если вводить имя хоста, это имя должно быть в формате полного имени домена (FQDN), например "www.synology.com".
  6. Для переадресации портов служба синхронизации общих папок использует порт 22 для шифруемой передачи и порт 873 для нешифруемой передачи.
  7. Служба синхронизации общих папок не поддерживает внешние устройства хранения.
  8. Зашифрованные общие папки нужно подключать вручную на целевом устройстве при помощи ключа шифрования.
  9. Запланированная синхронизация общих папок не будет выполнена в указанное время, если в выбранную папку не были внесены изменения.
  10. Для достижения наилучшей производительности рекомендуется включать в каждую задачу синхронизации не более 200 папок.
  11. Функция Главная папка пользователя будет отключена на целевом устройстве DiskStation, если папка homes синхронизирована с исходного устройства DiskStation на целевое устройство DiskStation.
  12. Для моделей с процессорами, отличными от Intel, можно синхронизировать до 100 000 общих папок (включая подпапки) с помощью службы синхронизации общих папок.
  13. Синхронизация общих папок позволяет синхронизировать настройки Windows ACL, если файловая система на целевом устройстве — ext4.
  14. Синхронизация общих папок не будет выполнена, если пытаться синхронизировать данные с целевым сервером, имеющим такое же имя, что и устройство DiskStation.
  15. Расширенные права доступа к общим папкам исходного устройства не будут синхронизированы с целевым устройством. Кроме того, на целевом устройстве для пользователей из группы "administrators" будет установлен статус "только для чтения".

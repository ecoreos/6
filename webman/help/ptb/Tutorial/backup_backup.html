Faça backup do seu Synology NAS

Visão geral

Perda inesperada de dados devido a falhar no disco rígido ou desastres naturais representam ameaças em potencial para os seus principais dados, especialmente quando não há nenhuma solução de backup confiável disponível. O Hyper Backup é uma solução de backup/restauração intuitiva e integrada que ajuda a manter/recuperar cópias de seus dados valiosos e restaurar aplicativos e configurações do sistema para um estado anterior. Os destinos de armazenamento a seguir são suportados:

  • Pastas compartilhadas locais (incluindo dispositivos de armazenamento externos conectados ao seu DiskStation)
  • Synology NAS remoto
  • Servidores rsync remotos
  • Servidores OpenStack Swift remotos
  • Nuvens públicas (Amazon S3, Microsoft Azure e OpenStack Swift, etc.)

Nota:

O Hyper Backup só faz backup de alguns aplicativos e configurações de sistema específicos. Para obter mais informações, consulte a documentação de ajuda do Hyper Backup.

1. Crie tarefas de backup local em uma pasta compartilhada local//dispositivo de armazenamento externo

O Hyper Backup oferece procedimentos de backup e restauração semelhantes em diferentes tipos de destino. Abaixo demonstramos como fazer backup e restaurar dados localmente para sua referência de outros tipos de tarefas.

  1. Faça login no DSM e localize o pacote Hyper Backup.
  2. Clique em + no canto inferior esquerdo e selecione Tarefa de backup de dados.
  3. Para configurar uma tarefa de backup local, selecione Pasta compartilhada local
  4. Selecione Criar tarefa de backup e selecione uma pasta compartilhadas local ou um dispositivo de armazenamento externo conectado ao seu DiskStation.
  5. Selecione as pastas de origem para fazer backup dos dados contidos nelas. As pastas compartilhadas criptografadas, se selecionadas, devem permanecer montadas durante o processo de backup. Se a origem e o destino estiverem no mesmo volume, o sistema marcará tais pastas de origem com exclamações em laranja.
  6. Selecione os aplicativos para fazer backup.
  7. Especifique as configurações de backup abaixo conforme suas necessidades:
    • Tarefa: Nomeie a tarefa de backup.
    • Habilitar notificação de tarefa: Selecione para ser informado do status da tarefa por e-mail, SMS ou serviço push.
    • Habilitar backup de configuração: Backup das configurações do sistema (ex.: credenciais do usuário). Essa configurações é forçada em todas as tarefas de backup por padrão.
    • Compactar dados de backup: Compacte os dados de backup para reduzir o uso do armazenamento de destino. Se esta opção for selecionada, o tempo de backup será maior.
    • Habilitar criptografia de transferência (apenas para backup remoto): Selecione para criptografar dados de backup durante a transferência para um destino remoto (ex.: um servidor rsync ou nuvem pública).
    • Tamanho das partes para upload com diversas partes (apenas para servidores compatíveis com S3): Divide os dados de backup pelo tamanho especificado.
    • Habilitar criptografia do lado do cliente: Selecione para proteger os dados de backup com senha contra acessos indesejados no lado do destino.
    • Habilitar agendamento de backup: Selecione para agendar quando executar automaticamente a tarefa de backup.
  8. Selecione Habilitar rotação do backup para configurar o esquema de rotação:
    • Das versões mais antigas: Excluir as versões de backup existentes anteriormente quando o Número de versões especificado for ultrapassado.
    • Smart Recycle: O sistema manterá cada versão de backup até que o número de versões especificado seja ultrapassado. Quando a rotação for disparada, o sistema primeiro girará as versões que não atende nenhuma das condições, se todas as versões existentes atenderem as condições abaixo, o sistema girará a versão mais recente:
      • Versões horárias das últimas 24 horas: O sistema mantém a versão mais recente que foi criada em cada hora.
      • Versões diárias de 1 dia passado a 1 mês: O sistema mantém a versão mais recente que foi criada em cada dia.
      • Versão semanal com mais de 1 mês: O sistema mantém a versão mais recente que foi criada em cada semana.
    • Número de versões: Defina o limite superior das versões de backup possíveis mantidas.
    • Linha do tempo: Exibe o resultado possível da retenção de backup de acordo com sua programação de backup e esquema de alternância:
  9. Clique em Aplicar para completar a configuração.
  10. Para executar a tarefa de backup imediatamente, siga as instruções do Assistente de backup e clique em Sim.

2. Restaure arquivos únicos de uma pasta compartilhada local/dispositivo de armazenamento externo

  1. Vá para Hyper Backup, selecione a tarefa de backup desejada no painel esquerdo e clique em Explorador de backup (ícone da lupa). Digite a senha se a tarefa for protegida por criptografia do lado do cliente.
  2. Encontre e selecione o arquivo desejado para restaurar. Você também pode selecionar a versão desejada para restaurar com a barra da linha do tempo.
  3. Clique em Restaurar para restaurar o arquivo/pasta selecionado.

3. Restaure todas as tarefas para os pontos do tempo desejado

  1. Clique em Restaurar no canto inferior esquerdo e selecione Dados. Digite a senha se a tarefa for protegida por criptografia do lado do cliente.
  2. Selecionar uma tarefa para restaurar. Para restaurar dados de outros repositórios, selecione Restaurar de outros repositórios e localize a tarefa a ser restaurada.
  3. Selecione se deseja restaurar as configurações do sistema. Se você selecionar Restaurar configurações do sistema, selecione as configurações específicas na lista abaixo para restaurar.
  4. Encontre e selecione o arquivo alvo/tarefa desejado para restaurar. Você também pode selecionar a versão desejada para restaurar com a barra da linha do tempo.
  5. Selecionar aplicações para restaurar.
  6. Serão exibidas informações básicas de restauração.
  7. Clique em Aplicar para iniciar a restauração.

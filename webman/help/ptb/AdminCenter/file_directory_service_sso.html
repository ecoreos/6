Cliente SSO

Se o seu DiskStation tiver se unido ao mesmo domínio ou serviço LDAP do servidor SSO, você pode configurar seu DiskStation como um cliente SSO. Os usuários podem acessar os serviços fornecidos pelo seu DiskStation depois de fazerem login no servidor SSO com suas credenciais.

Antes de iniciar, certifique-se de que seu DiskStation tenha entrada no mesmo domínio ou serviço LDAP do servidor SSO.

Para configurar seu DiskStation como um cliente SSO:

  1. Vá para Painel de controle > Domínio/LDAP > Cliente SSO.
  2. Marque a caixa de verificação Ativar o serviço SSO.
  3. Especifique o URL do servidor SSO e o ID do aplicativo nos campos abaixo da caixa de seleção.

    Nota:

    Para obter o ID do aplicativo, selecione um dos métodos a seguir:

    • Método 1:
      1. Entre no DSM que está executando o servidor SSO.
      2. Adicione seu DiskStation que funcionará como o aplicativo do cliente na Lista de aplicativos do servidor SSO. Para as etapas detalhadas, consulte o artigo Ajuda do SSO Server > Lista de aplicativos.
      3. Copie e cole o ID do aplicativo alocado pelo SSO Server depois que o seu DiskStation for adicionado ao servidor SSO.
    • Método 2:
      1. Após inserir o URL do servidor SSO*, clique em Registro rápido ao lado do campo do ID do aplicativo.
      2. Na caixa de registro, digite as credenciais de uma conta de administrador do servidor SSO nos campos Administrador do servidor e Senha.
      3. Especifique o nome do aplicativo do seu cliente no campo Nome do aplicativo.
      4. Certifique-se de que Redirecionar URI refere-se exatamente à página de login de seu DiskStation.
      5. Clique em OK e o seu DiskStation será adicionado ao servidor SSO com o ID do aplicativo sendo preenchido automaticamente.

    ______

    * É necessário digitar o URL do SSO server idêntico (nome de host ou IP) ao configurar cada cliente SSO.

  4. Clique em Aplicar para salvar suas configurações.

Nota:

  • Se o servidor SSO usar protocolo HTTPS, o login do SSO pode ser bloqueado pelos navegadores por falta de um certificado confiável.
  • Quando o servidor SSO é acessado através do protocolo HTTP, se o seu DiskStation (definido como cliente SSO) usar a conexão HTTPS, os usuários precisarão ajustar as configurações de seus navegadores para permitir o uso de uma conexão não segura.
  • O endereço IP do servidor SSO no formato IPv6 não será suportado se você tiver marcado a caixa de seleção Aprimore a segurança com o cabeçalho da Política de segurança de conteúdo (CSP) HTTP no Painel de controle > Segurança > Segurança no seu DSM.

LAN/Internet

Dependendo do modelo e das configurações de rede do DiskStation, você pode configurar as configurações de DHCP para LAN ou Internet em Painel de Controle > Rede > Interface de Rede. DHCP é um protocolo de rede para configuração automática de IP. Para deixar o DHCP atribuir um IP para o sistema, clique em Editar e escolha Obter configuração de rede automaticamente (DHCP) na guia IPv4.

Se estiver usando um endereço específico de IP de seu provedor de serviços de rede, escolha Usar a configuração manual e digite manualmente o endereço IP e a máscara de sub-rede. Se não houver um servidor DNS ou gateway padrão disponível, deixe o campo em branco.

Observação:

  1. Para garantir que as notificações por e-mail serão enviadas corretamente através do servidor SMTP, deve ser digitado um IP válido de servidor de DNS.
  2. Jumbo Frames são quadros Ethernet com mais que os 1500 bytes padrão de Unidade Máxima de Transmissão (MTU), permitindo que a transmissão de grandes arquivos pela Ethernet seja mais eficiente. Eles só podem ser habilitados no ambiente de rede Gigabit. Para garantir que Jumbo Frame funcione de forma apropriada, todos os computadores e dispositivos da rede que acessam o DiskStation devem suportá-lo e usar o mesmo valor de MTU.

VLAN

Quando o ambiente de rede requerer configurações VLAN, você pode habilitar a opção VLAN e digitar um ID VLAN para o ambiente de rede na guia IPv4 na janela Editar.

Observação:

  • Certifique-se de que seu switch de rede esteja configurado adequadamente, caso contrário, o serviço de seu DiskStation também pode se tornar inacessível.
  • Se você não puder conectar seu DiskStation após modificar as configurações acima, pressione o botão RESET localizado em seu DiskStation para restaurar as configurações padrão.

Configuração de IPv6

Os dispositivos DiskStation suportam os esquemas de pilha dupla IPv4/IPv6. Isto significa que tanto IPv4 quanto IPv6 podem trabalhar em seu DiskStation ao mesmo tempo. A configuração padrão para IPv6 é Auto. Selecione uma interface de rede existente, clique no botão Editar e vá para a guia IPv6 para alterar as configurações.

Para definir as configurações de IPv6:

  1. Vá até Painel de Controle > Rede > Interface de Rede.
  2. Selecione uma interface de rede existente e clique em Editar.
  3. Na guia IPv6, selecione um tipo de configuração na Configuração de IPv6:
    • Auto: Seu DiskStation gerará automaticamente um endereço IPv6 dependendo do LAN Router Advertisement (RA).
      • Modo Stateless: Permite que dispositivos conectados ao roteador configurem automaticamente um endereço IPv6 sem configurar o DNS Server IPv6.
      • Modo DHCPv6 Stateless: Permite que dispositivos conectados ao roteador configurem automaticamente um endereço IPv6 e obtenha informações de DNS por meio de DHCPv6.
      • Modo Stateful: Permite que dispositivos conectados ao roteador obtenham um endereço IPv6 e informações de DNS por meio de DHCPv6. Nesse modo você tem que habilitar o servidor DHCPv6 e configurar o Endereço IP inicial e o Endereço IP final do servidor DHCPv6.
    • Manual: Configure manualmente o IPv6 inserindo informações em endereço IPv6, Comprimento do Prefixo e Gateway padrão.
    • Desativado: Desativa o suporte a IPv6.

Até o IPv6 suplantar completamente o IPv4, os mecanismos de transição de IPv6 são necessários para permitir que os hosts com endereços IPv4 se conectem a serviços IPv6. Escolha entre os diferentes mecanismos de transição de IPv6:

  • 6in4: Conecte-se à rede IPv6 por meio de um agente de túnel. Você precisará preencher Endereço IPv6, Comprimento do Prefixo, Prefixo e Endereço IPv4 do Agente de Túnel. Isso requer um endereço IPv4 público.
  • 6to4: Conecte-se à rede IPv6 por meio de um agente de túnel utilizando Anycast. Isso requer um endereço IPv4 público.
  • DHCP-PD: Obtenha um prefixo IPv6 do DHCP Server por meio da delegação de prefixo com a LAN. Isso requer suporte a ISP e funciona somente em modo de roteador.
  • Túnel IPv6 (TSP): Vá para Painel de controle > Rede > Geral para configurar esta opção.

802.1X

O 802.1X é um padrão de autenticação desenvolvido pelo IEEE para redes de conexão de usuários. O 802.1X pode ser usado em redes para autenticar o cliente, não permitindo que clientes que não passaram com sucesso pela autenticação acessem a rede protegida.

Você pode permitir que o Synology NAS se conecte a uma rede protegida pela autenticação 802.1X.

Para habilitar a autenticação 802.1X:

  1. Vá até Painel de Controle > Rede > Interface de Rede.
  2. Selecione uma interface de rede com fio (ex.: LAN) e clique em Editar.
  3. Vá para a guia 802.1X e marque Habilitar autenticação 802.1X.
  4. Selecione um dos seguintes protocolos do menu suspenso Autenticação. Se não tiver certeza qual protocolo é o mais adequado, selecione Auto. Â Â Â Â 
    • EAP protegido (PEAP): Use um certificado CA (opcional) para estabelecer uma conexão segura. Essa conexão segura será usada pela autenticação interna.
    • TLS em túnel (TTLS): Use um certificado CA (opcional) para estabelecer uma conexão segura. Essa conexão segura será usada pela autenticação interna.
    • TLS: use TLS (Segurança de camada de transporte) para autenticar.
  5. Se você escolher TTLS ou PEAP, você seleciona um dos protocolos de autenticação internos do menu suspenso.
    • MS CHAP v2: As informações de autenticação serão criptografadas com o Microsoft CHAP v2.
    • GTC: As informações de autenticação usarão o GTC (Cartão Token Genérico) para a criptografia.
  6. Digite seu nome de usuário e senha.
  7. Se necessário, clique em Importar, depois em Procurar para encontrar e importar um certificado exportado de um Radius Server (por exemplo ca.crt).
  8. Clique em OK para salvar suas configurações.

Nota:

  • Para configurar a autenticação 802.1X para redes sem fio, vá para Painel de controle > Sem fio > Wi-Fi.
  • As configurações de autenticação devem ser as mesmas das definidas no Radius Server. Para obter mais informações, entre em contato com o administrador do Radius Server.

LAN sem fio*

Se seu DiskStation atuar como roteador sem fio, será possível configurar as configurações a seguir para LAN sem fio em Painel de Controle > Rede > Interface de Rede.

Configurações de IPv4

Endereço IP e máscara de sub-rede

Você pode configurar o endereço IP e a máscara de sub-rede de seu DiskStation aqui.

DHCP Server

Verifique a opção Habilitar servidor DHCP para permitir que seu DiskStation atribua endereços IP aos dispositivos clientes para conexão à rede local fornecida por seu DiskStation.

Relé de PPPoE

Ative o Relé de PPPoE para permitir que os dispositivos conectados ao seu DiskStation façam a conexão à Internet via conexão PPPoE.

Configurar como um gateway de Internet UPnP

Ative o serviço UPnP para permitir que o DiskStation se torne um dispositivo de gateway de Internet de UPnP, para que os dispositivos conectados à rede local do seu DiskStation possam fazer as configurações de encaminhamento de porta com UPnP.

Configurações de IPv6

Você pode obter um endereço e um prefixo IPv6 na LAN/Internet com 6in4, 6to4 ou DHCP-PD. Com essas informações, é possível configurar as opções de roteamento IPv6 para seu DiskStation aqui.

  • Endereço IPv6: Digite o endereço IPv6 obtido na LAN/Internet.
  • Comprimento do Prefixo: Digite o comprimento do prefixo do endereço IPv6 obtido na LAN/Internet, normalmente 64.
  • DNS primário: O servidor DNS primário para os dispositivos conectados ao roteador. Isso não precisa ser configurado no Modo Stateless.
  • DNS secundário: O servidor DNS secundário para os dispositivos conectados ao roteador. Isso não precisa ser configurado no Modo Stateless.

Há três modos de Router Advertisement (RA) para selecionar:

  • Modo Stateless: Permite que dispositivos conectados ao roteador configurem automaticamente um endereço IPv6 sem configurar o DNS Server IPv6.
  • Modo DHCPv6 Stateless: Permite que dispositivos conectados ao roteador configurem automaticamente um endereço IPv6 e obtenha informações de DNS por meio de DHCPv6.
  • Modo Stateful: Permite que dispositivos conectados ao roteador obtenham um endereço IPv6 e informações de DNS por meio de DHCPv6. Nesse modo você tem que habilitar o servidor DHCPv6 e configurar o Endereço IP inicial e o Endereço IP final do servidor DHCPv6.

Clientes DHCP

Aqui você pode visualizar uma lista de clientes DHCP e suas configurações de rede (por exemplo, endereço MAC, endereço IP, nome de host e o tempo restante antes que a concessão do DHCP expire).

É possível filtrar a lista suspensa do cliente IPv4/IPv6 para mostrar clientes que estão usando IPv4 e IPv6, respectivamente. Escolha um cliente e clique em Adicionar à reserva de endereço para adicionar o cliente à lista de reserva de endereço.

Reserva de DHCP

Se você quiser que um cliente receba sempre o mesmo endereço IP durante a renovação da concessão do DHCP, você pode adicionar o cliente à lista de reserva do endereço.

É possível filtrar a lista suspensa de reserva de endereço IPv4/IPv6. Os endereços IPv4 são identificados por endereço MAC, enquanto endereços IPv6 são identificados por DUID.

Para adicionar um cliente à lista de reserva de endereço:

  • Selecione um cliente e clique em Adicionar.
  • Especifique o endereço MAC ou DUID do cliente, o endereço IPv4/IPv6 que você deseja atribuir ao cliente e o nome de host do cliente.
  • Clique em OK para salvar suas configurações.

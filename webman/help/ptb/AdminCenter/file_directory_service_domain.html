Domínio

O seu DiskStation pode ingressar um domínio Windows. Depois de ingressar no domínio, os usuários de domínio podem fazer login no DiskStation usando sua conta e senha de domínio, que permitem acessar arquivos e usar os aplicativos de DSM sem precisar lembrar outro conjunto de nomes de usuário e senhas.

Para ingressar em um domínio:

  1. Vá para Painel de controle > Serviço de diretório > Domínio.
  2. Marque a caixaIngressar domínio.
  3. Insira o nome de domínio e especifique o servidor DNS nos campos adequados.
  4. Se necessário, marque a caixa Opções avançadas de domínio. (Consulte abaixo para obter detalhes.)
  5. Clique em Aplicar.
  6. Na janela pop-up, digite o nome do usuário e senha do administrador do servidor do domínio.
  7. Clique em OK.

DNS Server

Recomendamos que especifique um DNS server para ajudar DiskStation a pesquisar o domínio. No entanto, se o domínio não estiver registrado no servidor DNS, poderá ocorrer falha no procedimento de ingresso no domínio.

Tipo de servidor de domínio

Esse campo mostra o tipo de domínio quando o DiskStation ingressa em um domínio. O tipo de domínio será "Domínio AD".

Modo de gerenciamento

Essa opção determinará como os privilégios dos usuários e grupos do domínio serão gerenciados.

  • Domínio confiável: Você pode gerenciar usuários e grupos no domínio ingressado, bem como domínios confiáveis. Os usuários e grupos podem ser filtrados com a lista suspensa de domínio.
  • Domínio único com UO: Somente usuários e grupos no domínio ingressado serão exibidos nesse modo. Esse modo permite filtrar usuários/grupos de domínio com a lista suspensa UO (unidade organizacional).

Opções avançadas de domínio

Na maioria dos casos, não é necessário preencher nenhuma das opções avançadas de domínio. As opções avançadas de domínio são necessárias apenas para ambientes de domínio específicos.

  • DC IP/FQDN: Especifique um endereço IP de DC (Controlador de domínio) ou FQDN e o DiskStation tentará comunicar-se com ele. Se houver mais de um endereço de IP ou FQDN, insira uma vírgula (,) entre cada um. Se quiser, adicione um asterisco (*) depois do último endereço IP de DC ou FQDN, e o DiskStation tentará comunicar-se com outro DC se houver erro em todos os especificados. Você deverá adicionar uma vírgula entre o asterisco e o último endereço IP ou FQDN.
  • Nome de domínio NetBIOS: Informe o nome NetBIOS do domínio.
  • FQDN do domínio (nome DNS): Informe o FQDN (nome DNS) do domínio.
  • Registrar interface DNS: Ao ingressar num domínio, a NIC (Placa de interface de rede) especificada aqui será registrada no servidor DNS. Observe que, se o nome do host do DiskStation incluir um sublinhado (_), o registro falhará, porque sublinhados (_) não podem ser usados com o DNS.
  • Atualizar lista de usuários/grupos: Especifique com que frequência o DiskStation atualiza automaticamente a lista de usuários/grupos do domínio. Você pode personalizar o período para realizar as atualizações, diariamente, semanalmente ou mensalmente. Além disso, as listas de usuários/grupos de domínio podem ser atualizadas manualmente indo para a guia Usuário do domínio e clicando em Atualizar todos os domínios. Observe que as atualizações automáticas afetarão a hibernação do sistema.

Ingresso do controlador de domínio de somente leitura

Para ingressar o dispositivo DiskStation a um controlador de domínio somente leitura (RODC), é necessário especificar o "nome completo do computador" (FQDN) do controlador de domínio de leitura e escrita (RWDC) no campo DC IP/FQDN. O nome completo do computador deve estar no seguinte formato: ComputerName.Domain, por exemplo, "ad01.synology.com".

Nota:

  • Se o nome de usuário do domínio inclui "%" e "$", você pode não conseguir acessar sua pasta base. Peça que o administrador do domínio atribua a você um novo nome de usuário.
  • Grupos de domínio Domain Admins e Enterprise Admins serão adicionados automaticamente ao grupo local ﻿Administrators. Portanto, os usuários do domínio desses grupos têm privilégios de direito administrativo sobre o DiskStation, incluindo acesso aos serviços de arquivos como CIFS/FTP/AFP/WebDAV e outros aplicativos de DSM.
  • Você também pode configurar os privilégios de acesso dos usuários do domínio às pastas compartilhadas no DiskStation. Veja aqui para obter mais informações.

Opções de domínio

Esta seção inclui as opções adicionais de domínio abaixo.

  • O DiskStation fará a sincronização com um servidor NTP sempre que o usuário de um domínio fizer logon: Ativar essa opção força a sincronização do tempo entre o DiskStation e o servidor de NTP.
  • Habilitar assinatura do servidor: Se o cliente ativou a assinatura do cliente, assinatura do servidor também deve ser ativada no DiskStation, para que os usuários possam usar os serviços de domínio normalmente.
  • Obter listas de usuários/grupos com o modo compatível ao NT4: Ativar essa opção permite que o sistema obtenha as listas de usuários/grupos usando o modo NT4 RPC. Essa opção pode ser ativada se certas listas de usuários/grupos do domínio não puderem ser obtidas usando as configurações padrão.
  • Desativar permissões de administrador para Administradores de Domínio e Administradores de Empresa: Usuários com permissões de administrador podem ter controle total de seu DiskStation e seus arquivos. Habilitar essa opção permite restringir as permissões de administradores de domínio e administradores de empresa para serem as mesmas dos usuários normais, herdando, portanto, todas as permissões de arquivos/pastas de usuários normais.
  • Enumerar os níveis de grupo aninhados: Especificar o número de membros de grupos de domínio aninhados que pode ser enumerado.

Atribuir Permissões NFS

Você pode atribuir permissões NFS a qualquer pasta compartilhada, para que os clientes Linux tenham acesso a ela.

Para atribuir permissões NFS a uma pasta compartilhada:

  1. Selecione a pasta compartilhada que deseja editar na lista de pastas compartilhadas.
  2. Clique em Editar > Permissões NFS.
  3. Clique em Criar para adicionar uma regra NFS.
  4. Aparece uma janela pop-up. Defina as opções abaixo.
  5. Nome de host ou IP: Digite o endereço IP do cliente NFS que acessará a pasta compartilhada. É possível especificar um host de três maneiras:
    • Host único: O nome de domínio totalmente qualificado ou um endereço IP.
    • Curingas: *, *.synology.com
    • Redes de IP: 203.74.205.32/255.255.252.0, /24
  6. Privilégio: Selecione as permissões de leitura/gravação do cliente NFS.
  7. Segurança: Especifique o tipo de segurança a ser implementado.
    • AUTH_SYS: Utilize o UID (identificador do usuário) e o GID (identificador do grupo) do cliente NFS para verificar as permissões de acesso.
    • Autenticação Kerberos: Realize a autenticação Kerberos quando o cliente NFS se conectar à pasta compartilhada. O cliente só pode acessar a pasta compartilhada depois de ser aprovado na autenticação Kerberos.
    • Integridade Kerberos: Realize a autenticação Kerberos e verifique a integridade dos pacotes durante a transferência de dados.
    • Privacidade Kerberos: Realize a autenticação Kerberos e criptografe os pacotes NFS durante a transferência de dados, evitando assim que partes mal-intencionadas adulterem o tráfego NFS.
  8. Squash: Este campo permite que você controle os privilégios de acesso dos usuários do cliente NFS. Selecione um dos seguintes:
    • Nenhum mapeamento: Permite que todos os usuários do cliente NFS, incluindo usuários raiz, mantenham os privilégios de acesso originais.
    • Mapear raiz para admin: Atribui privilégios de acesso a usuários raiz do cliente NFS equivalentes aos privilégios de acesso do usuário admin do seu sistema.
    • Mapear raiz para guest: Atribui privilégios de acesso a usuários raiz do cliente NFS equivalentes aos privilégios de acesso de guest do seu sistema.
    • Mapear todos os usuários para admin: Atribui privilégios de acesso a todos os usuários do cliente NFS equivalentes aos privilégios de acesso do usuário admin do seu sistema.
  9. Ativar assíncrono: Marcar esta opção permite que seu DiskStation responda a solicitações de clientes NFS antes de qualquer alteração ser feita em arquivos, proporcionando melhor desempenho.
  10. Permitir conexões a partir de portas sem privilégios (portas maiores que 1024): Marcar esta opção permite que os clientes NFS usem portas sem privilégios (ou seja, portas maiores que 1024) ao se conectarem ao DiskStation.
  11. Permitir o acesso dos usuários a subpastas montadas: Marcar esta opção permite que os clientes NFS acessem subpastas montadas.
  12. Clique em OK para concluir.
  13. Clique em OK para aplicar as permissões NFS.

Observação:

  • Quando o formato do nome do servidor for *.domain, o endereço IP do cliente NFS deve ter um registro DNS PTR correspondente, para permitir que o DiskStation encontre o nome *.domain buscando o endereço IP correspondente.

Recursos de Segurança:

Ao acessar uma pasta compartilhada via NFS com uma conta de usuário específica:

  • Se o recurso de segurança AUTH_SYS for implementado: O cliente deve ter exatamente o mesmo UID (identificador do usuário) numérico e GID (identificador do grupo) no cliente do NFS e no DiskStation, do contrário ele será atribuído à permissão outros ao acessar a pasta compartilhada. Para evitar qualquer conflito de permissões, você pode selecionar Mapear todos os usuários para admin em Squash ou dar permissões a "Todos" para a pasta compartilhada.
  • Se o recurso de segurança Kerberos (krb5, krb5i, krb5p) for implementado: Vá para Win/Mac/NFS > Serviço NFS > Configurações Kerberos > Mapeamento de ID para mapear o cliente do NFS a um usuário específico, ou entre em um domínio Windows /LDAP com a conta de usuário correspondente, do contrário ou cliente será atribuído às permissões de convidado ao acessar a pasta compartilhada.
  • Se o sistema de arquivos do dispositivo externo em que as pastas compartilhadas foram criadas for NTFS ou FAT, a opção Mapear todos os usuários para admin será aplicada à força.

Para usar os recursos de segurança do Kerberos para a conexão ao DiskStation, a autenticação do Kerberos deve ser configurada indo para Win/Mac/NFS > Serviço NFS > Configurações do Kerberos.

Conectar ao VPN

É possível tornar o DiskStation facilmente em um cliente de VPN e se conectar a VPN servers (Redes Privadas Virtuais) existentes por meio dos protocolos PPTP, OpenVPN ou L2TP/IPSec. Se você tiver vários servidores VPN, será possível criar perfis VPN específicos para cada servidor, para ativar e se conectar rapidamente a servidores VPN diferentes com um simples clique.

O que é VPN?

Um VPN ou rede privada virtual é uma solução para atender a necessidade de acessar com segurança os recursos de sua rede privada a partir da Internet. Para empresas ou alguns indivíduos que precisem ampliar as capacidades de redes em crescimento, o custo desse sistema de rede física e seu suporte técnico aumentará exponencialmente. Ao considerar o custo-benefício e as manutenções de longo prazo, o VPN é uma solução inteligente e cada vez mais atraente. Com a criptografia e outros mecanismos de segurança, a tecnologia VPN permite que os membros de empresas acessem facilmente a rede central da empresa e use seus recursos como na LAN. Os indivíduos também podem acessar os recursos em sua LAN local quando estiverem longe de casa.

Nota:

  • O DiskStation pode se conectar apenas a servidores OpenVPN que suportam túneis tun-style na Layer 3.
  • Antes de instalar o pacote de VPN, verifique se você tem um servidor VPN para se conectar. Para obter uma compatibilidade melhor, sugerimos a conexão aos serviços VPN hospedados no DiskStation executando o pacote VPN Server. Para configurar outro DiskStation como um VPN server, consulte Ajuda do DSM > VPN Server > Configurar VPN Server para obter instruções.

Para criar perfis de VPN:

  1. Vá até Painel de controle > ﻿Rede.
  2. Na guia Interface de Rede, clique em Criar e escolha Criar perfil de VPN.
  3. Ao criar um novo perfil VPN, é possível selecionar um dos seguintes tipos de conexão:
    • PPTP: PPTP (Protocolo de tunelamento ponto a ponto) é uma solução VPN usada comumente e suportada pela maioria dos clientes (incluindo Windows, Mac, Linux e dispositivos móveis).
    • OpenVPN: A OpenVPN é uma solução de fonte aberta para a implementação do serviço VPN. Ela protege a conexão VPN com o mecanismo de criptografia SSL/TLS.
    • L2TP/IPSec: O L2TP (Protocolo de tunelamento de camada 2) via IPSec fornece redes privadas virtuais com aumento de segurança e é suportado pela maioria dos clientes (como Windows, Mac, Linux e dispositivos móveis).
  4. Consulte as seções abaixo para obter instruções para configurar qualquer um dos três tipos de conexões VPN.

Para criar um perfil PPTP:

  1. Quando solicitado a escolher um método de conexão VPN, selecione PPTP e clique em Avançar.
  2. Escolha um nome para o novo perfil, especifique o endereço do servidor VPN ao qual deseja se conectar e digite seu nome de usuário e senha do servidor. Clique em Avançar para continuar.

    3. Nota:

    • O nome de usuário, senha e endereço do servidor VPN devem ser fornecidos pelo administrador do servidor VPN.
  3. Selecione um dos seguintes tipos de autenticação no menu Autenticação para proteger a senha do cliente VPN durante a autenticação:
    • PAP: A senha não será criptografada durante a autenticação.
    • CHAP: A senha será criptografada usando CHAP (Challenge-Handshake Authentication Protocol).
    • MS CHAP: A senha será criptografada usando o Microsoft CHAP versão 1.
    • MS CHAP v2: A senha será criptografada usando o Microsoft CHAP versão 2.
  4. Se você selecionar MS CHAP ou MS CHAP v2, selecione um dos itens a seguir no menu Criptografia para criptografar a conexão VPN:
    • Nenhum MPPE: A conexão VPN não será protegida com Microsoft Point to Point Encryption.
    • Requer MPPE (40/128 bits): A conexão VPN será protegida com Microsoft Point to Point Encryption de 40 ou 128 bits.
    • MPPE máximo (128 bits): A conexão VPN será protegida com a Criptografia Ponto a Ponto de 128 bits da Microsoft, que fornece o nível mais alto de segurança.

    5. Nota:

    • As configurações de autenticação e criptografia devem ser idênticas às configurações especificadas no servidor VPN. Para mais detalhes, entre em contato com o administrador do servidor VPN.
  5. Marque uma das caixas de seleção a seguir, de acordo com suas necessidades:
    • Usar gateway padrão na rede remota: Habilite essa opção para encaminhar o tráfego da rede do DiskStation para o servidor VPN especificado.
    • Permite que outros dispositivos da rede se conectem através da conexão de Internet do servidor Synology: Habilite essa opção para permitir que os dispositivos da rede que estejam na mesma rede local que seu DiskStation para se conectar ao mesmo servidor VPN.
    • Reconectar quando a conexão VPN for perdida: Se a conexão VPN for perdida inesperadamente, o sistema tentará restabelecer a conexão cinco vezes, uma vez a cada 30 segundos.
  6. Clique em Aplicar.

Para criar um perfil OpenVPN:

  1. Quando solicitado a escolher um método de conexão VPN, selecione OpenVPN e clique em Avançar.
  2. Escolha um nome para o novo perfil, especifique o endereço do servidor VPN ao qual deseja se conectar e digite seu nome de usuário e senha do servidor.
  3. No campo Certificado, clique no botão Procurar para localizar e importar o arquivo de certificado (por exemplo: ca.crt) que foi exportado pelo servidor VPN. Clique em Avançar para continuar.

    4. Nota:

    • O nome de usuário, senha, endereço e certificado do servidor VPN devem ser fornecidos pelo administrador do servidor VPN.
  4. Marque uma das caixas de seleção a seguir, de acordo com suas necessidades:
    • Habilitar compressão no link VPN: Habilite essa opção para realizar a compressão de dados antes da transferência pelo VPN. Essa opção pode aumentar a velocidade de transferência, mas vai consumir mais recursos do sistema.
    • Usar gateway padrão na rede remota: Habilite essa opção para encaminhar o tráfego da rede do DiskStation para o servidor VPN especificado.
    • Permite que outros dispositivos da rede se conectem através da conexão de Internet do servidor Synology: Habilite essa opção para permitir que os dispositivos da rede que estejam na mesma rede local que seu DiskStation para se conectar ao mesmo servidor VPN.
    • Reconectar quando a conexão VPN for perdida: Se a conexão VPN for perdida inesperadamente, o sistema tentará restabelecer a conexão cinco vezes, uma vez a cada 30 segundos.
  5. Clique em Aplicar.

Para criar um perfil OpenVPN (por meio do .ovpn):

  1. Quando solicitado a escolher um método de conexão VPN, selecione OpenVPN (por meio da importação de um arquivo .ovpn) e clique em Avançar.
  2. Selecione um nome para o novo perfil e, no campo Importar arquivo .ovpn, clique em Procurar para selecionar e importar um arquivo .ovpn exportado do servidor VPN.
  3. Insira as seguintes informações fornecidas a você pelo administrador do seu servidor VPN: nome de usuário, senha e importe o arquivo de certificado CA (por exemplo, ca.crt).
  4. Se o provedor do servidor VPN também tiver fornecido a você uma chave de autenticação TLS, chave de cliente, certificado de cliente ou lista de certificados revogados, clique em Opções avançadas e importe-os nos respectivos campos.
  5. Clique em Avançar para continuar.

    6. Nota:

    • O nome de usuário, a senha, o endereço e qualquer arquivo adicional, como certificados ou chaves, devem ser fornecidos pelo administrador do servidor VPN.
  6. Marque uma das caixas de seleção a seguir, de acordo com suas necessidades:
    • Habilitar compressão no link VPN: Habilite essa opção para realizar a compressão de dados antes da transferência pelo VPN. Essa opção pode aumentar a velocidade de transferência, mas vai consumir mais recursos do sistema.
    • Usar gateway padrão na rede remota: Habilite essa opção para encaminhar o tráfego da rede do DiskStation para o servidor VPN especificado.
    • Permite que outros dispositivos da rede se conectem através da conexão de Internet do servidor Synology: Habilite essa opção para permitir que os dispositivos da rede que estejam na mesma rede local que seu DiskStation para se conectar ao mesmo servidor VPN.
    • Reconectar quando a conexão VPN for perdida: Se a conexão VPN for perdida inesperadamente, o sistema tentará restabelecer a conexão cinco vezes, uma vez a cada 30 segundos.
  7. Clique em Aplicar.

Para criar um perfil L2TP/IPSec:

  1. Quando solicitado a escolher um método de conexão VPN, selecione L2TP/IPSec e clique em Avançar.
  2. Escolha um nome para o novo perfil, especifique o endereço do servidor VPN ao qual deseja se conectar e digite seu nome de usuário e senha do servidor. Também será necessário inserir a chave pré-compartilhada do servidor VPN. Em seguida, clique em Avançar.

    3. Nota:

    • O nome de usuário, senha, endereço e chave pré-compartilhada do servidor VPN devem ser fornecidos pelo administrador do servidor VPN.
  3. Selecione um dos seguintes tipos de autenticação no menu Autenticação para proteger a senha do cliente VPN durante a autenticação:
    • PAP: A senha não será criptografada.
    • CHAP: A senha será criptografada usando CHAP (Challenge-Handshake Authentication Protocol).
    • MS CHAP: A senha será criptografada usando o Microsoft CHAP versão 1.
    • MS CHAP v2: A senha será criptografada usando o Microsoft CHAP versão 2.

    4. Nota:

    • As configurações de autenticação e criptografia devem ser idênticas às configurações especificadas no servidor VPN. Para mais detalhes, entre em contato com o administrador de seu servidor VPN.
  4. Marque uma das caixas de seleção a seguir, de acordo com suas necessidades:
    • Usar gateway padrão na rede remota: Habilite essa opção para encaminhar o tráfego da rede do DiskStation para o servidor VPN especificado.
    • Permite que outros dispositivos da rede se conectem através da conexão de Internet do servidor Synology: Habilite essa opção para permitir que os dispositivos da rede que estejam na mesma rede local que seu DiskStation para se conectar ao mesmo servidor VPN.
    • O servidor está por trás do dispositivo NAT: Por padrão, o protocolo L2TP/IPSec não permite conexões a um servidor VPN por dispositivos NAT. Ative essa opção para contornar essa limitação, permitindo que o DiskStation se conecte a um servidor VPN L2TP/IPSec por dispositivos NAT.
    • Reconectar quando a conexão VPN for perdida: Se a conexão VPN for perdida inesperadamente, o sistema tentará restabelecer a conexão cinco vezes, uma vez a cada 30 segundos.
  5. Clique em Aplicar.

Para conectar ou desconectar de um VPN server:

  1. Selecione um perfil VPN.
  2. Clique em Conectar ou Desconectar.

Nota:

  • Apenas um perfil pode ser conectado ao mesmo tempo.

Para modificar um perfil VPN:

  1. Selecione um perfil VPN.
  2. Clique em Editar.

DDNS

O DDNS (Serviço de Nome de Domínio Dinâmico) simplifica a conexão com o DiskStation através da Internet mapeando um nome de host com seu endereço IP.Por exemplo, o DDNS permite que você acesse o DiskStation usando um nome de domínio (ex.: www.john.synology.me), sem necessidade de lembrar um endereço IP (ex.: 172.16.254.1).

Configuração do DDNS

Você pode apontar um nome de host existente para o endereço IP do DiskStation ou registrar um novo fornecido pela Synology ou vários outros provedores de DDNS.Consulte cada provedor para obter mais informações relativas ao registro do nome de host.

Para configurar um nome de host DDNS:

  1. Clique no botão Adicionar.
  2. Uma caixa de diálogo aparece pedindo que você edite as seguintes configurações:
    • Provedor de serviços:Selecione um provedor de serviços.Para se registrar para obter um nome de host grátis fornecido pela Synology, selecione Synology do menu suspenso, clique em Registrar agora e digite os detalhes conforme indicado.
    • Nome de host: Digite o nome de host do DDNS registrado, como john.synology.me.
    • Nome de usuário: Digite o nome de usuário do provedor de DDNS.
    • Senha: Digite a senha do provedor de DDNS.
    • Endereço externo: Digite o endereço IP externo do DiskStation que usará o nome de host.
    • Pulsação: Habilite essa opção para receber alertas sobre o status do nome de host mapeado (apenas para nomes de host fornecidos pela Synology).
  3. Clique em Testar conexão para verificar se as configurações estão corretas.
  4. Clique em OK para salvar e concluir.

Após concluir a configuração, você pode acessar o DiskStation pela Internet digitando o nome de host do DDNS em um navegador de Internet.

Requisitos e limitações

  • Os servidores do provedor de serviço de DDNS devem estar funcionando.
  • O DiskStation deve ser capaz de se conectar à Internet.
  • O DDSN funciona apenas em ambiente de rede onde nenhum servidor proxy seja necessário.
  • Você pode digitar apenas um nome de host para cada provedor de DDNS.

Gerenciamento de nomes de host

Use os botões a seguir para gerenciar as entradas de nomes de host existentes:

  • Excluir: Remove a entrada do nome de host selecionada da lista.
  • Editar: Altere as configurações da entrada de nome de host selecionada.
  • Atualizar agora: Atualiza o status de todas as entradas do nome de host que têm o serviço DDNS habilitado.

Configuração do Provedor DDNS personalizado

Você pode adicionar um provedor DDNS personalizado se o seu provedor já não aparecer na lista de provedores de serviço.

Para configurar um provedor DDNS personalizado:

  1. Clique em Personalizar.
  2. Uma caixa de diálogo aparece pedindo que você digite as seguintes configurações:
    • Provedor de serviços: Digite o nome de seu provedor de serviço personalizado.
    • URL da consulta: Digite o URL de seu provedor de serviço personalizado.
  3. Clique em Salvar para adicionar um provedor de serviço personalizado à lista de provedores de serviço de DDNS.

Requisitos e limitações

  • O URL da consulta deve seguir o protocolo especificado pelo seu provedor de serviço.Esse URL é usado para atualizar informações de IP do servidor de DDNS do seu provedor.
  • Um URL de consulta geralmente contém os quatro parâmetros a seguir.Ao digitar o URL da consulta, substitua os parâmetros a seguir pelas variáveis correspondentes.
    • Nome de usuário:__USERNAME__
    • Senha:__PASSWORD__
    • Nome de host:__HOSTNAME__
    • Endereço externo:__MYIP__
  • O URL da consulta não pode conter espaços ou pontos e vírgulas.

Criar uma pasta compartilhada

As pastas compartilhadas são os diretórios básicos onde você pode armazenar e gerenciar arquivos e pastas em seu DiskStation. Antes de armazenar arquivos, é preciso criar pelo menos uma pasta compartilhada no DSM.

Para criar uma pasta compartilhada:

  1. Vá até Painel de controle > Pasta compartilhada e clique em Criar.
  2. Especifique um nome e uma descrição para a pasta compartilhada.
  3. Selecione o armazenamento no qual deseja criar a pasta compartilhada (pule essa etapa se houver apenas um armazenamento).
  4. Se necessário, modifique as opções a seguir:
    • Oculte essa pasta compartilhada em "Meus Locais de Rede": evita que a pasta compartilhada apareça em "Rede" no Gerenciador de Arquivos do Windows. Essa opção não afeta os privilégios de acesso da pasta compartilhada.

      Nota:

      Ocultar uma pasta compartilhada em "Meus Locais de Rede" não afeta os direitos de acesso a ela. O direito de acesso da pasta compartilhada oculta permanecerá inalterado. Os usuários que tiverem os direitos de acesso corretos à pasta compartilhada ainda poderão acessá-la digitando "\\nomedoservidor\nome da pasta compartilhada".

    • Ocultar subpastas e arquivos de usuários sem permissões: Quando a opção Ocultar subpastas e arquivos de usuários sem permissões estiver habilitada, se um usuário sem privilégios de leitura tentar acessar uma pasta compartilhada através do protocolo de compartilhamento de arquivos do Windows, ele não poderá visualizar as subpastas ou arquivos dentro da pasta compartilhada. Isso impede que os usuários fiquem confusos quando se conectam aos produtos Synology, mas encontram um grande número de arquivos e subpastas que não podem abrir.

      Nota:

      • Esse recurso é suportado apenas no protocolo de compartilhamento de arquivos do Windows.
      • Você pode modificar as configurações de privilégio de cada pasta e arquivo no File Station clicando com o botão direito nele. Selecione Propriedades no menu suspenso e clique na guia Permissão.
      • A pasta compartilhada não será ocultada dos usuários sem permissões, permitindo Ocultar subpastas e arquivos de usuários sem permissão. Apenas as subpastas e arquivos dentro dessa pasta compartilhada serão ocultados.
  5. Habilitar Lixeira: Quando os arquivos na pasta compartilhada forem excluídos, eles serão movidos para uma pasta chamada #recycle. O acesso à lixeira pode ser limitado a usuários que pertencem ao grupo administrators.
  6. Habilitar cota da pasta compartilhada: Para especificar a capacidade máxima de cada pasta compartilhada, marque a caixa de seleção Habilitar cota da pasta compartilhada e digite a capacidade máxima (ex.: 10 GB) no seguinte campo.

    Nota:

    • Somente quando o armazenamento estiver usando o sistema de arquivos Btrfs a opção Habilitar cota da pasta compartilhada estará disponível para as pastas compartilhadas.
    • O uso do armazenamento combinado dos diretórios home de todos os usuários não pode ultrapassar a cota de "homes".
  7. Habilitar proteção avançada de integridade de dados: Para proteger uma pasta compartilhada com estratégias de soma de verificação CRC32 e cópia na gravação, você pode habilitar a proteção avançada de integridade de dados durante a criação da pasta compartilhada. A soma de verificação CRC32 é aplicada na verificação se os dados permanecerem idênticos a quando eles foram gravados originalmente e notificará o usuário quando for detectado qualquer corrompimento. A cópia na gravação ajuda a melhorar a consistência dos dados no desligamento anormal do sistema.

    Nota:

    • Essa opção está disponível apenas para os volumes que usam o sistema de arquivo Btrfs.
    • Você pode habilitar/desabilitar essa opção para cada pasta compartilhada durante sua criação. Essa opção não pode ser modificada quando a pasta compartilhada for criada.
    • Se arquivos como bancos de dados forem editados e atualizados constantemente, a quantidade de arquivos fragmentados pode aumentar devido à cópia em gravação. Recomenda-se desabilitar essa opção para melhorar o desempenho.
  8. Habilitar compactação de arquivo: Os dados na pasta compartilhada criada serão compactados automaticamente pelo sistema de arquivos Btrf para economizar o espaço de armazenamento. Quando recuperados para uso, os dados serão descompactados automaticamente.

    Nota:

    • Essa opção está disponível apenas quando você cria uma pasta compartilhada em um volume Btrfs ou quando a Proteção avançada de integridade de dados está habilitada.
    • Essa opção não pode ser modificada depois que a pasta compartilhada for criada.
    • A taxa de compactação difere conforme os tipos de dados: Os arquivos de texto contam com uma maior taxa de compactação possível, enquanto arquivos multimídia (imagens, músicas ou vídeos) não podem ser bem compactados pois já são compactados para a saída.
  9. Criptografar essa pasta compartilhada: Se desejar criptografar a pasta compartilhada, faça o seguinte:
    1. Especifique e confirme uma nova chave de criptografia. A chave de criptografia não pode incluir vírgulas (,) ou sinais de igual (=).
    2. Se necessário, marque a caixa Montar automaticamente ao iniciar para montar a pasta criptografada automaticamente quando o sistema for iniciado. Se essa opção estiver desativada, a pasta compartilhada criptografada será desmontada na próxima vez que o DiskStation reiniciar. Para exibir ou acessar uma pasta desmontada, será preciso montá-la manualmente importando ou digitando a chave de criptografia.
  10. Clique em OK para concluir.

Regras de Nomeação de Arquivos Compartilhados:

  • Os nomes das pastas compartilhadas não diferenciam maiúsculas de minúsculas e podem ter de 1 a 32 caracteres. Os nomes não podem ter os seguintes caracteres: ! " # $ % & ' ( ) * + , / : ; <= > ? @ [ ] \ ^ ` { } | ~
  • Os seguintes nomes são reservados para uso do sistema: ., .., global, home, homes, printers, public, satashare, surveillance, usbbackup, usbshare.
  • O primeiro caractere do nome de uma pasta compartilhada não pode ser um sinal de menos ou um espaço, e o último caractere não pode ser um espaço.
  • A descrição da pasta compartilhada diferencia maiúsculas e minúsculas e pode ter até 64 caracteres Unicode visíveis.

Criptografia de Pastas Compartilhadas:

A criptografia AES de 256 bits protege dados bloqueando todas as tentativas de acesso não autorizadas. Quando a pasta compartilhada for criptografada, as pessoas sem chave de criptografia não poderão acessar ou modificar os dados criptografados, mesmo se removerem os discos rígidos do DiskStation e montá-los em seu dispositivo.

Nota:

  • As pastas compartilhadas podem ser criptografadas apenas no momento da criação.
  • Os nomes dos arquivos e pastas dentro de pastas compartilhadas criptografadas não podem ultrapassar 143 caracteres ingleses ou 43 caracteres asiáticos.
  • As pastas compartilhadas criptografadas não podem ser acessadas através de NFS.
  • Se as senhas forem restauradas para o padrão usando o botão de restauração localizado no DiskStation, as pastas compartilhadas criptografadas serão desmontadas e a opção de montar automaticamente pastas compartilhadas criptografadas será desativada.
  • As pastas compartilhadas criptografadas são suportadas apenas no modo de alto privilégio. Para usar esse recurso, vá para o pacote Docker em seu DSM host e selecione o Docker DSM com o qual deseja habilitar as pastas compartilhadas criptografadas, clique em Editar e marque a caixa ao lado de Executar o contêiner usando altos privilégios.

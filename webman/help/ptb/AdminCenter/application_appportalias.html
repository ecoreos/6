Portal de aplicativo

O Portal de aplicativo permite que você ajuste as configurações de conexão de vários aplicativos para que possa acessar e executar esses aplicativos diretamente (ex.: File Station) em guias ou janelas independentes do navegador.

Nota:

Você pode editar as configurações de acesso de aplicativos como o Audio Station, Download Station, Surveillance Station, Video Station, File Station e vários outros.

Personalizar Aliases

Você pode atribuir um alias para pesquisa de aplicativos desenvolvidos pela Synology como o File Station. Com o alias personalizado, você pode abrir rapidamente o aplicativo através de um URL específico.

Para personalizar aliases de aplicativos:

  1. Vá para Painel de controle > Portal de aplicativo > Aplicativo.
  2. Selecione um aplicativo.
  3. Clique em Editar > Geral.
  4. Na janela pop-up, selecione Habilitar alias personalizado e especifique o alias.
  5. Clique em OK para salvar as configurações.

Nota:

  • Alguns aplicativos têm seus aliases padrão: Audio Station (audio), Download Station (download), File Station (file), Surveillance Station (cam) e Video Station (video).
  • Os nomes dos aliases não podem ser os mesmos daqueles reservados para o sistema, navegadores ou usados por outros aplicativos e seu número de caracteres deve ser limitado de 2 a 20.
  • Um alias pode ser composto somente de letras maiúsculas e minúsculas misturadas, caracteres numéricos e dois caracteres especiais (-) e (_).
  • Um alias não pode começar ou terminar com nenhum destes caracteres especiais: (-) ou (_).

Para acessar aplicativos com aliases:

Depois de configurar um alias para um aplicativo (ex.: File Station), você pode abrir rapidamente o aplicativo digitando um URL nos seguintes formatos: http://DS_IP_OR_SERVER_NAME/ALIAS/ ou https://DS_IP_OR_SERVER_NAME/ALIAS/

Por exemplo, você pode acessar o File Station rapidamente através dos URLs: http://192.168.xx.xx/file/ ou https://MySynologyNAS/file/

Personalizar portas HTTP/HTTPS

Você pode atribuir uma porta HTTP/HTTPS para cada aplicativo desenvolvido pela Synology como o File Station. Com a porta personalizada, você pode abrir rapidamente o aplicativo através de um URL específico.

Para personalizar as portas HTTP/HTTPS do aplicativo:

  1. Vá para Painel de controle > Portal de aplicativo > Aplicativo.
  2. Selecione um aplicativo.
  3. Clique em Editar > Geral.
  4. Na janela pop-up, selecione Habilitar porta personalizada (HTTP) ou Habilitar porta personalizada (HTTPS).
  5. Especifique um número de porta personalizada.

Nota:

  • O intervalo de números de porta pode ser de 1 a 65535.
  • As portas a seguir não podem ser usadas porque são reservadas para uso do sistema:
    • 20, 21, 22, 23, 25, 80, 110, 137, 138, 139, 143, 199, 443, 445, 515, 543, 548, 587, 873, 993, 995, 3306, 3689, 5000, 5001, 5005, 5006, 5335, 5432, 6881, 8080, 7000, 7001, 8081, 9997, 9998, 9999, 50001, 50002
    • Portas padrão do eMule: 4662 (TCP), 4672 (UDP)
    • Intervalo padrão de portas FTP: A faixa atual pode variar dependendo dos modelos diferentes.

Para acessar aplicativos com portas HTTP/HTTPS:

Depois de configurar um alias para um aplicativo (ex.: File Station), você pode abrir rapidamente o aplicativo digitando um URL nos seguintes formatos: http://DS_IP_OR_SERVER_NAME:HTTP_PORT ou https://DS_IP_OR_SERVER_NAME:HTTPS_PORT

Por exemplo, você pode acessar o File Station rapidamente através dos URLs: http://192.168.xx.xx:7000 ou https://MySynologyNAS:7001

Personalizar domínios

Você pode coincidir um nome de domínio com um aplicativo desenvolvido pela Synology, como o File Station. Com o domínio personalizado, você pode abrir rapidamente o aplicativo através de um URL específico.

Para personalizar domínios de aplicativos:

  1. Vá para Painel de controle > Portal de aplicativo > Aplicativo.
  2. Selecione um aplicativo.
  3. Clique em Editar > Geral.
  4. Na janela pop-up, selecione Habilitar domínio personalizado.
  5. Especifique um domínio personalizado.
  6. Para configurações adicionais, você pode selecionar Habilitar HSTS ou Habilitar HTTP/2 de acordo com suas necessidades.

Nota:

  • Antes de configurar esse recurso, é necessário se candidatar a nomes de domínio de provedores de domínio para que os usuários possam acessar o serviço pela Internet.
  • Para aplicativos (ex.: File Station) relacionados ao carregamento/download de arquivo, sugerimos que você não selecione Habilitar HTTP/2 pois tais aplicativos podem ter velocidades baixas de carregamento e download.
  • Cada nome de domínio pode ser usado somente para um aplicativo específico no seu DiskStation.
  • Se o nome de domínio não exceder 15 caracteres e respeitar as convenções de nomenclatura do NetBIOS, ele será registrado e transmitido pelo sistema automaticamente.
  • Ao acessar aplicativos com URLs que usam nomes de domínio, você passará pela porta HTTP padrão (80) ou pela porta HTTPS padrão (443).

Para acessar aplicativos através de domínios:

Depois de configurar um domínio (ex.: arquivo.exemplo.com) para um aplicativo (ex.: File Station), você pode abrir rapidamente o aplicativo digitando um URL nos seguintes formatos: http://APP_DOMAIN ou https://APP_DOMAIN

Por exemplo, você pode acessar o File Station rapidamente através dos URLs: http://file.example.com ou https://file.example.com

Se o nome de domínio (por exemplo, FileFile) de um aplicativo (por exemplo, File Station) respeitar as convenções de nomenclatura do NetBIOS, os usuários do Windows poderão acessá-lo em um Synology NAS na mesma rede local com uma URL semelhante: http://FileFile ou https://FileFile

Personalização de regras de proxy reverso

Seu DiskStation pode funcionar como um servidor de proxy reverso que transfere solicitações da Internet para dispositivos na rede local. As regras do proxy reverso podem ajudá-lo a ocultar portas secretas contra ameaças em potencial como as duas situações a seguir:

Situação 1: Suponha que a porta secreta seja a 80, que não deve permitir acesso externo de acordo com a regra do firewall. Você pode configurar uma regra do proxy reverso para permitir que usuários confiáveis da Internet acessem a porta secreta 80 através de uma outra porta aberta (ex.: 81). Desse modo, usuários confiáveis podem contornar o firewall e ainda conseguirem acessar a porta 80.

Situação 2: Suponha que a porta secreta seja a 80, que não deve permitir acesso externo, exceto de um dispositivo específico (ex.: um servidor com o nome "MeuConfiavel"). Com as regras do proxy reverso, você pode permitir que apenas o tráfego de MeuConfiavel acesse a porta 80 enquanto o tráfego de outros dispositivos não.

Para configurar as regras do proxy reverso:

  1. Vá para Painel de controle > Portal de aplicativo > Proxy reverso.
  2. Clique em Criar e especifique as seguintes configurações:
    • Descrição: Especifique um nome que o ajude a identificar a função da regra.
    • Especifique as regras da Origem (o dispositivo que envia solicitações da Internet) e Destino (o dispositivo na rede local).
      • Protocolo: Os protocolos HTTP ou HTTPS usados pela origem/destino
      • Nome do host: O nome do dispositivo de origem/destino
      • Porta: A porta usada pelo dispositivo de origem/destino
      • Habilitar HSTS e Habilitar HTTP/2 (apenas para a origem)
  3. Clique em OK para salvar as configurações.

Nota:

Se você digitar * como o nome de host de origem ou deixar em branco, a regra será aplicada para qualquer origem.

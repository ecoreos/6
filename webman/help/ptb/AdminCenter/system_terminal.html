Terminal

Ativar as opções em Painel de Controle > Terminal e SNMP > Terminal permite que o seu DiskStation suporte os serviços de interface de linha de comando Telnet e SSH. Você também pode alterar o nível de segurança do algoritmo de criptografia SSH.

Para ativar o serviço Telnet/SSH:

  1. Marque a caixa ao lado do protocolo desejado (SSH ou Telnet).
  2. Clique em Aplicar.

Para alterar a porta de serviço SSH:

  1. Em Porta, digite um novo número de porta.
  2. Clique em Aplicar.

Vários algoritmos de criptografia com diferentes níveis de segurança são usados para proteger as conexões SSH. Você pode decidir quais algoritmos são usados para atender melhor suas necessidades.

Para alterar o nível de segurança do algoritmo de criptografia SSH:

  1. Clique em Configurações avançadas.
  2. Selecione um nível de segurança para os algoritmos de criptografia. Se você quiser especificar exatamente quais algoritmos podem ser usados, selecione Personalizar para escolher de três grupos: Cipher (Codificação), KEX (Key Exchange (Troca de Chave)) e MAC (Message Authentication Code -(Código de autenticação de Mensagem)).
  3. Clique em Aplicar.

Alguns modelos suportam aceleração de hardware para certos algoritmos de criptografia. Você pode marcar a opção Usar apenas codificações aceleradas por hardware para aprimorar as velocidades de transferência de serviços relacionados ao SSH incluindo os serviços de backup de rede criptografada e SFTP. Na página avançada, os algoritmos capazes de usar a aceleração de hardware são exibidos em vermelho.

Nota:

  • Alguns serviços (ex.: backup de rede, SFTP) habilitam automaticamente o SSH e abrem a porta 22. Nesse caso, é necessário habilitar o serviço SSH em Painel de controle > Terminal e SNMP > Terminal para entrar no DSM usando SSH.
  • O SSH/Telnet suporta apenas login no sistema com contas que pertencem ao grupo administrators. Para alternar para uma conta root, faça login no sistema com SSH/Telnet como um usuário que pertence ao grupo administrators, execute o comando sudo -i e digite a senha da conta usada para fazer login.
  • A senha não pode ser deixada em branco ao fazer login no sistema com SSH/Telnet.
  • Quando o serviço SSH estiver desativado, todas as conexões SSH serão encerrados, incluindo todas as tarefas de backup de rede criptografada que estiverem sendo executadas. Sugerimos realizar tarefas de backup de rede novamente após desativar o serviço SSH.
  • A manipulação imprópria ou modificações do DiskStation podem resultar em mau funcionamento ou perda de dados.
  • A seleção de um nível de segurança maior ou usar apenas codificadores acelerador por hardware pode fazer com que alguns clientes SSH não consigam se conectar devido a conflitos de compatibilidade.
  • Dependendo do nível de segurança selecionado, alguns algoritmos que suportam a aceleração de hardware podem não estar disponíveis.

Geral

Em Painel de Controle > Serviços de arquivos > FTP, é possível habilitar o limite de velocidade, registro de transferência de arquivos, configurações de segurança e outras configurações avançadas na seção Geral.

Habilitar o Registro de Transferência de Arquivos por FTP

Clique em Configurações avançadas para habilitar essa função e gravar os registros de arquivos baixados ou enviados.

Nota:

Habilitar o registro de transferência de arquivos por FTP pode afetar o desempenho do sistema.

Alterando os diretórios raiz do usuário

Clique em Configurações avançadas e marque Alterar os diretórios raiz do usuário. Dessa forma, os usuários selecionados podem acessar apenas as pastas especificadas ao fazer login no DSM via FTP.

Para criar uma regra de alteração de diretório raiz:

  1. Em Configurações avançadas, marque Selecionar Usuário.
  2. Clique em Adicionar.
  3. Em Usuário ou grupo, selecione para quem essa regra será aplicada.
  4. Selecione qual diretório raiz estará disponível quando o usuário fizer login. Escolha entre:
    • Base do usuário: Os usuários chegam ao seu diretório base após o login.
    • Outro diretório: Selecione uma pasta compartilhada como diretório raiz e clique em Selecionar. Observe que as permissões da pasta para o usuário ou grupo devem ser marcadas primeiro. Um usuário ou grupo com permissões de pasta insuficientes não será capaz de fazer login.
  5. Clique em OK.
  6. Depois que todas as regras forem adicionadas, clique em Aplicar para salvar as regras.

Nota:

  • As regras são priorizadas de acordo com as suas posições na lista.
  • Arraste e solte para reorganizar as regras na lista.

Habilitar FTP Anônimo

Clique em Configurações Avançadas e marque a opção para permitir que qualquer pessoa faça login no serviço FTP do dispositivo DiskStation (com a conta anônima ou ftp).

Alterar Root Anônimo

Clique em Configurações avançadas e marque a opção para alterar o diretório de login do usuário FTP anônimo. Selecione uma pasta compartilhada no menu suspenso.

Aplicar Permissões UNIX Padrão

Marque Configurações avançadas e marque essa opção para aplicar as permissões UNIX padrão ao fazer upload ou criar arquivos e pastas. As permissões aplicadas são as mesmas das permissões aplicadas pelo comando UNIX umask. Quando essa opção está habilitada, as permissões UNIX são 644 para arquivos e 755 para pastas. Quando essa opção está desabilitada, as permissões UNIX são 666 para arquivos e 777 para pastas. O valor umask padrão é 022.

Nota:

  • Para pastas compartilhadas com a ACL do Windows habilitada (todos os compartilhamentos, exceto "photo" e compartilhamentos em unidades externas), execute o comando chmod no cliente Linux ou FTP para alterar os tipos de permissão de arquivos e pastas da ACL do Windows para UNIX.
  • Habilitar essa opção pode causar problemas de permissão inconsistente entre os diferentes protocolos. Para evitar inconsistências, sugerimos deixar essa opção desabilitada.

Ativar o limite de velocidade

Ative um limite de velocidade para controlar a largura de banda máxima usada pelos usuários e grupos que transferem dados via FTP.

Para ativar o limite de velocidade:

  1. Marque a caixa Ativar limite de velocidade ou Ativar limite de velocidade de acordo com uma programação se deseja implementar um limite de velocidade apenas em momentos específicos. Esta é uma configuração para todo o servidor e anulará as configurações de usuários e grupos individuais. Escolha Nenhum limite de velocidade para desativar as configurações de limite de velocidade de todos os usuários e grupos.
  2. Clique em Configurações do limite de velocidade para modificar as configurações para usuários ou grupos individuais. Para cada usuário e grupo, você pode selecionar uma das seguintes opções:
    • Aplicar configurações de grupo: As configurações do limite de velocidade serão iguais às do grupo ao qual o usuário pertence.
    • Nenhum: Nenhum limite de velocidade será aplicado quando os membros do grupo transferirem arquivos com o serviço.
    • Constante: Um limite de velocidade fixo e constante será aplicado ao usuário ou membros do grupo. Os limites de velocidade de upload e download podem ser especificados nos campos à esquerda.
    • Personalizado: Dois limites de velocidade diferentes podem ser especificados e aplicados ao usuário ou aos membros de grupo, de acordo com uma programação. Clique no botão Personalizar para modificar as configurações de limite de velocidade e também uma programação.
  3. Clique em Aplicar.

Nota:

  • Para obter mais detalhes sobre a dinâmica do limite de velocidade do grupo, clique aqui.
  • Ao configurar o limite de velocidade para um usuário, a seção Resultado exibe o limite de velocidade real do usuário de acordo com as configurações individuais dele e as do grupo. Por exemplo, se o usuário "Achilles" tiver um limite de download individual de 100 KB/s, mas pertence a um grupo com um limite de 500 KB/s, a seção Resultados mostrará "0/500".

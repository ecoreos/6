FTP/FTPS

Em Painel de controle > Serviços de arquivos > FTP você pode ativar os serviços FTP para permitir que os clientes acessem pastas e dados no sistema. Antes de habilitar o FTP, certifique-se de que as seguintes portas TCP de seu roteador estejam sendo enviadas para o dispositivo DiskStation: 21 (conexão de controle padrão), 20 (conexão de dados para o modo ativo) e 55536-55567 (conexão de dados para o modo passivo).

Para ativar o serviço FTP:

  1. Habilite um dos seguintes itens:
    • Habilitar serviço FTP (Sem criptografia): Esse é o protocolo de rede padrão usado para transferir arquivos. O FTP não fornece nenhuma criptografia para proteger informações durante as sessões de transferência como senhas, nomes de usuário ou arquivos. Entretanto, as velocidades de transferência são mais rápidas e exige menos recursos do sistema.
    • Habilitar serviço de criptografia FTP SSL/TLS (FTPS): Uma continuação do FTP padrão com o suporte adicional dos protocolos de criptografia Transport Layer Security (TLS) e Secure Sockets Layer (SSL), que protegem as informações durante as sessões de transferência, como senhas, nomes de usuário ou arquivos. Entretanto, as velocidades de transferência são menores e consome mais recursos do sistema devido à criptografia.
  2. Clique em Aplicar.

Tempo limite

O servidor de FTP desconectará os usuários ociosos depois do tempo especificado.

Configuração de número da porta do serviço FTP

Esta seção permite especificar a porta usada para o canal de controle do FTP.

Nota:

O número de porta padrão para o canal de controle do FTP é 21. Você pode defini-lo entre 1 e 65535, com exceção dos seguintes números: 20, 22, 23, 25, 80, 110, 137, 138, 139, 143, 199, 443, 445, 515, 543, 548, 587, 873, 993, 995, 3306, 3689, 5000, 5001, 5005, 5006, 5335, 5432, 6881, 8080, 7000, 7001, 8081, 9997, 9998, 9999, 50001, 50002 e portas padrão do eMule: 4662 (TCP), 4672 (UDP).

Extensão de porta de FTP Passivo

Esta seção permite especificar o intervalo de portas usado para as conexões do FTP Passivo. Você pode usar o intervalo padrão ou especificar um personalizado. O intervalo de portas pode ser definido entre 1025 e 65535, e pode conter até 128 portas.

Relatar IP externo no modo PASV

Quando essa função estiver ativada, o servidor relatará seu endereço IP externo aos clientes FTP. O endereço IP relatado será exibido no registro de conexão do cliente FTP. Uma vez ativado, você pode optar por detectar automaticamente o endereço IP externo ou inserir manualmente o do DiskStation.

Essa opção só funciona quando o DiskStation estiver por trás de um NAT, e os clientes FTP pertencerem a uma subrede diferente do DiskStation. Na maioria dos casos essa opção é desnecessária, mas se os clientes FTP não conseguirem se conectar ao DiskStation, você pode ativar esta opção e tentar novamente.

Ativar FXP

FXP significa Protocolo de troca de arquivos. Ao ativar essa opção, o serviço FTP suportará a função de transferência de arquivos de servidor para servidor.

Habilitar módulo criptográfico FIPS

FIPS é suportado no serviço FTPS usando-se o módulo de criptografia validado pelo OpenSSL FIPS 140-2 (Certificado No. 1747). Ao habilitar essa opção, os algoritmos de criptografia de segurança adicionais serão usados pelo serviço FTPS para proporcionar sessões mais seguras.

Modo de transferência ASCII de suporte

Ative essa opção para suportar o modo de transferência ASCII, permitindo que o DiskStation transfira corretamente os arquivos de texto simples no formato ASCII via FTP.

Codificação UTF-8

Essa opção pode ser usada para ajudar clientes FTP a exibir e manusear arquivos corretamente com nomes de arquivo em vários idiomas. Para atender as necessidades de vários clientes FTP, são fornecidas as três opções seguintes:

  • Desabilitar: A codificação UTF-8 não será usada. Essa opção destina-se a clientes que usam o Windows File Explorer como um cliente FTP para carregar/baixar arquivos com nomes em vários idiomas. Não é recomendada para uso normal.
  • Auto: Deixe que o DSM determine automaticamente como manusear a codificação UTF-8. Essa opção é recomendado para uso normal.
  • Forçar: Forçar usa a codificação UTF-8. Se você tiver habilitado a codificação UTF-8 em seu cliente FTP e ele ainda não conseguir exibir e manusear corretamente os nomes dos arquivos em vários idiomas, selecione essa opção.

Restrição na conexão

Esta seção inclui as seguintes opções:

  • Limitar conexões por IP: Quando esta função estiver ativada, o servidor limitará o número máximo de conexões por IP. Ativar essa função e a restrição de largura de banda de FTP para um gerenciamento eficiente da largura de banda.
  • Ativar restrição de largura de banda de FTP: Ative esta função para restringir a largura de banda de uma única conexão FTP.

Nota:

  • A conta "guest" não pode efetuar login no servidor por FTP.
  • A configuração da página de código do computador cliente de FTP deve ser a mesma de DiskStation para acessar os dados corretamente.
  • Em alguns modelos específicos, o serviço FTP suporta a conexão usando SSL ou TLS.
  • As configurações do firewall do seu cliente FTP e dois servidores FTP devem ser configuradas corretamente ou a função FXP pode não funcionar bem.
  • Alguns clientes de FTP, como aqueles que usam o Internet Explorer 8/9, precisam fazer login com seu nome de usuário e senha. Por exemplo: ftp://username:password@your.ftp.com.

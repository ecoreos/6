Geral

Opções de rede genéricas como gateway padrão, DNS Server, proxy e tunelamento de IPv6 estão disponíveis em Painel de Controle > Rede > Geral.

Renomear Dispositivo DiskStation

Para renomear o nome do dispositivo DiskStation, digite o novo nome do servidor e clique em Aplicar.

O nome de servidor não diferencia maiúsculas de minúsculas e pode estar na faixa de 1 a 15 caracteres, que podem consistir em letras, números, sublinhados e o sinal de menos. O primeiro caractere do nome deve ser uma letra.

Especificação do Gateway Padrão

Apenas digite o endereço IP do gateway padrão e clique em Aplicar para salvar a configuração.

Especificação do Endereço IP do DNS Server

Você pode escolher configurar manualmente o endereço IP do seu servidor DNS. Nesse caso, o endereço IP de seu servidor DNS preferido deve ser digitado. Um endereço IP do servidor DNS alternativo também pode ser digitado, fornecendo backup se o servidor DNS preferido apresentar problemas. Quando os endereços IP do servidor DNS estiverem configurados como desejado, clique em Aplicar para salvar as configurações.

Observação:

Os campos servidor DNS e gateway padrão serão definidos automaticamente e não podem ser alterados quando todas as interfaces de rede forem configurados automaticamente ou quando você estiver usando uma conexão PPPoE.

Conexão via Servidor Proxy

Para habilitar seu dispositivo DiskStation para se conectar via servidor proxy, marque Conectar-se por um servidor proxy, digite o Endereço e a Porta e clique em Aplicar.

Observação:

  • Clique em Configurações Avançadas para configurar o proxy HTTP, proxy HTTPS e as informações de autenticação.
  • Por padrão, as configurações de proxy HTTPS são as mesmas que as do proxy HTTP.
  • Se a opção Conectar-se por um servidor proxy estiver marcada, o DSM Update e o Centro de Pacotes, juntamente com alguns outros pacotes, sempre irão se conectar à Internet por um proxy.
  • Se o Servidor proxy para endereços locais estiver marcado, verifique se o DNS server usado pelo DiskStation pode atender consultas para dispositivos na rede local.

Habilitar Tunelamento de IPv6

Até o IPv6 suplantar completamente o IPv4, o tunelamento de IPv6 é necessário para permitir que os hosts com endereços IPv6 se conectem a serviços IPv4. Essa técnica fornece uma maneira de transportar tráfego IPv6 em redes IPv4 existentes.
Se não tiver certeza sobre como configurar IPv6 ou se tiver problemas de conexão, entre em contato com o Provedor de serviços de Internet.

Para habilitar o IPv6 Tunnel:

  1. Marque a caixa de seleção Ativar tunnel para ativar o suporte ao tunnel.
  2. Em Endereço do servidor, digite o nome de host ou o IP do provedor de serviços de tunnel, por exemplo: anonymous.freenet6.net.
  3. Escolha se quer Conectar anonimamente ou Conectar com uma conta existente.

Observação:

O endereço IPv6 externo pode ser atualizado automaticamente pelo provedor de serviços. Entretanto, o dispositivo DiskStation não removerá imediatamente o endereço IPv6 antigo.

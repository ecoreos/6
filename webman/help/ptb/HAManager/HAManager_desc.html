Gerenciador de High Availability

High Availability refere-se a uma solução de layout de servidor desenvolvida para reduzir interrupções de serviços causadas por falhas do servidor. O Synology High Availability usa dois servidores para formar um “cluster de high-availability”, onde um servidor assume o papel de “servidor ativo” e o outro servidor funciona como “servidor passivo” de espera. Para implementar e criar um grupo de high-availability, é necessário ter dois servidores Synology compatíveis.

Um conjunto de high-availability faz uso dos seguintes componentes e conceitos:

  • Conceito: A implementação do Synology High Availability requer dois servidores Synology para criar um conjunto de high-availability. Um servidor funciona como "servidor ativo" e o outro como "servidor passivo". Quando o conjunto de high-availability é criado, os dados são replicados continuamente a partir do servidor ativo até o servidor passivo. Todos os arquivos do servidor ativo existirão em duplicada no servidor passivo. Caso o servidor ativo apresente defeito, o servidor passivo estará pronto para assumir todos os serviços com uma cópia espelhada completa dos dados do servidor ativo. Assim, o conjunto de high-availability continuará fornecendo serviços normalmente e minimizando o tempo parado do servidor. Para obter mais informações sobre a mudança, consulte Mudança
  • Servidor ativo: O servidor ativo gerencia o fornecimento de todos os serviços. Caso ocorram erros no servidor ativo, você ou o sistema pode iniciar uma mudança para o servidor passivo. Nesse caso, o fornecimento de serviços é transferido para o servidor passivo, ponto em que o servidor ativo se torna o servidor passivo.
  • Servidor passivo: Sob condições normais, o servidor passivo permanece em modo de espera. Os dados do servidor ativo são replicados continuamente no servidor passivo. Caso o servidor ativo não possa fornecer serviços, você ou o sistema pode iniciar uma mudança para o servidor passivo.
  • Conexão de pulsação: Os servidores ativo e passivo de um conjunto de high-availability estão vinculados por uma conexão de rede privada dedicada e chamada de "Conexão de pulsação". Depois que o conjunto de high-availability é criado, a conexão de pulsação facilita a sincronização de dados e a replicação entre os servidores ativo e passivo. Se o contato entre os servidores se perder por um certo tempo, o sistema determinará a necessidade de uma mudança.
  • Volume de armazenamento principal do conjunto de High-availability: O volume de armazenamento do servidor ativo se tornará o volume de armazenamento principal do conjunto de high-availability. Os dados são replicados para o volume de armazenamento do servidor passivo através da conexão de pulsação.
  • Volume de armazenamento sobressalente do conjunto de high-availability: O volume de armazenamento do servidor passivo se tornará o volume de armazenamento sobressalente do conjunto de high-availability. Os dados são replicados continuamente do volume de armazenamento do servidor ativo para o volume de armazenamento passivo através da conexão de pulsação. Isso garante a consistência do conteúdo do arquivo quando ocorre uma mudança.
  • Nome do servidor do conjunto de high-availability: Os servidores no conjunto de high-availability compartilharão um nome de servidor. Na rede, você pode conectar usando esse nome de servidor e os serviços serão fornecidos pelo servidor ativo.
  • IP do conjunto de high-availability: Como cada um dos servidores ativo e passivo do conjunto de high-availability tem seu próprio endereço IP, para evitar a troca para outro endereço IP quando for realizada uma mudança, o conjunto de high-availability gerará endereços de IP comuns para acesso. Para acessar qualquer serviço DSM, independentemente do Synology estar funcionando como servidor ativo, você pode se conectar através desses endereços IP.

Desligamento e restabelecimento

As instruções abaixo o guiarão através do processo de desligamento, remoção ou restabelecimento de um conjunto de high-availability.

Para desligar o conjunto de high-availability usando o Synology High Availability:

Quando você desejar desligar um conjunto de high-availability para transportar os servidores para um novo local físico ou realizar manutenção, mas você não desejar remover as configurações do conjunto de high-availability, siga essas instruções.

  1. Faça login no DSM.
  2. Abra o Synology High Availability.
  3. Clique no botão marcado Gerenciamento.
  4. Selecione Desligar conjunto de HA.
  5. Uma mensagem de confirmação aparece. Clique em Sim para confirmar.

Para desligar manualmente um conjunto de high-availability:

O conjunto de high-availability também pode ser desligado manualmente usando-se os botões liga/desliga localizados em cada servidor.

  1. Pressione e segure o botão liga/desliga do servidor (ativo ou passivo) até que ele emita um bipe.
  2. Depois de um servidor ter se desligado totalmente, aguarde pelo menos 30 segundos (o servidor que ainda está on-line requer tempo para salvar as configurações).
  3. Pressione e segure o botão liga/desliga do outro servidor até que ele emita um bipe.
  4. Aguarde o servidor concluir o desligamento.

Ligação dos servidores:

Quando desejar ligar os servidores novamente, os servidores ativo e passivo devem ser ligados ao mesmo tempo (ou em cinco minutos um em relação ao outro). Se você ligar apenas um dos servidores, ele tentará ligar o outro servidor transmitindo uma mensagem Wake on LAN.

Desligamento anormal:

Quando um conjunto de high-availability é criado, a opção de reinício após a falha na ligação é habilitada automaticamente no Painel de controle. Se os servidores desligarem anormalmente devido a uma falha de energia, eles tentarão ligar automaticamente quando a energia for restaurada.

Para remover o conjunto de high-availability.

Quando você quiser desvincular os dois servidores e remover um conjunto de high-availability para vincular um servidor diferente, siga essas instruções.

  1. Vá para a página Visão geral.
  2. Clique no botão Gerenciamento.
  3. Selecione Remover conjunto de HA.

Após a confirmação o sistema remove o conjunto de high-availability. O campo status exibirá o andamento do processo de remoção. Quando o conjunto de high-availability tiver sido removido com sucesso, você será direcionado para o endereço IP original do servidor ativo. O status dos dois servidores mudará para Independente.

Para restabelecer um conjunto de high-availability com defeito:

Caso você não consiga acessar o DSM ou o Synology High Availability devido a um erro de redes separadas ou outros defeitos, você pode tentar forçar o conjunto de high-availability a se desvincular pressionando o botão Restabelecer em seus servidores.

  1. Pressione e segure o botão Restabelecer (localizado geralmente no painel traseiro do servidor) até que o servidor emita um bipe.
  2. O sistema removerá o conjunto de high-availability e o status dos dois servidores mudará para Independente
  3. Pressionar o botão Restabelecer também restaura algumas configurações para o padrão tais como o endereço IP, DNS e senha de administrador além do seguinte:
    • A senha do administrador será restabelecida para em branco.
    • A conta de convidado será desabilitada.
    • O DHCP será habilitado no IP, DNS, Gateway e outras interfaces de rede.
    • O JumboFrames será desabilitado e o MTU será definido como 1500.
    • A porta de gerenciamento será restabelecida para 5000/5001.
    • O PPPoE será restabelecido para o padrão.
    • O bloqueio automático de IP será desabilitado (o endereço IP bloqueado não será apagado).
    • Link Aggregation será desabilitada.
    • As regras do firewall serão desabilitadas (as regras não serão apagadas).

Observação:

Durante o procedimento de remoção de um conjunto de high-availability, deve haver algumas operações de gravação de dados incompletas. Para garantir a inteireza dos dados, execute a replicação dos dados com base no servidor ativo porque o servidor passivo pode estar sem algumas atualizações de dados.

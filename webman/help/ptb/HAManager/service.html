Monitoramento de serviços

Essa página permite o monitoramento de determinados serviços. Se um serviço monitorado falhar no servidor ativo, o sistema acionará o failover para o modo de espera, servidor passivo (desde que o servidor passivo esteja funcionando normalmente). As seguintes seções aparecem nesse página:

  • Serviço: Exibe os serviços que podem ser monitorados.
  • Status: Exibe se os serviços estão funcionando ou não.
  • Failover quando o serviço falhar: Marque a caixa de seleção do lado de um serviço para começar a monitorá-lo. O sistema ativará o failover se o serviço selecionado apresentar uma falha.

Para monitorar um serviço:

  1. Encontre o serviço que deseja monitorar. Marque a caixa de seleção ao lado dele.
  2. Clique em Aplicar para salvar as configurações.

Serviços Suportados

Os serviços que podem ser monitorados são os seguintes: Serviço de Arquivos Windows, iSCSI Target, FTP, Serviço de Arquivos Mac, Serviço NFS.

Failover para Falhas de Conexão de Rede

Se essa opção estiver ativada, o sistema ativará o failover para o servidor passivo quando a conexão de rede do servidor ativo apresentar falhas ou mau funcionamento, e a conexão de rede do servidor passivo ainda estiver funcionando normalmente.

  • O failover será acionado apenas se o servidor passivo tiver conexões em bom estado. Por exemplo, se o servidor ativo tiver três conexões e uma delas falhar, o sistema acionará um failover apenas se o servidor passivo tiver três ou mais conexões saudáveis.
  • Conexões combinadas com Link Aggregation contam como uma conexão. Por exemplo, se duas conexões forem combinadas e uma delas falhar, mas a outra ainda estiver funcionando, a conexão ainda será considerada saudável.

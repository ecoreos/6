Mudança

Para garantir o desempenho contínuo e minimizar o tempo parado, os serviços podem ser mudados do servidor ativo de um cluster de high-availability para o redundante, o servidor passivo. Esse recurso é chamado "Mudança" ou, em alguns casos, "Failover". Esta seção explica os elementos básicos desses dois conceitos.

Mudança

Mudança refere-se a uma mudança de um servidor ativo para o redundante, o servidor passivo de um cluster de high-availability. Sob condições normais, o servidor ativo gerencia todos os serviços. Entretanto, caso o servidor ativo apresente defeito, você pode iniciar uma mudança e transferir todos os serviços para o servidor passivo, considerando que o servidor passivo esteja funcionando normalmente. Nesse caso, o servidor ativo assume o papel de servidor passivo e o servidor passivo assume o papel do servidor ativo.

Para iniciar manualmente uma mudança:

  1. Vá para a guia Visão geral.
  2. Clique em Gerenciamento > Mudança.

Observação:

Quando uma mudança for iniciada, o servidor ativo primeiro assume o papel do servidor passivo em seguida, o servidor passivo se torna o servidor ativo. Assim, haverá um curto período em que os dois servidores se tornarão passivos e não poderão fornecer os serviços normalmente. O tempo para concluir uma mudança varia dependendo do número e do tamanho dos volumes ou iSCSI LUNs (nível de bloqueio) ou do número e do total de carga dos serviços do cluster de high-availability.

Failover

Quando ocorrem certos erros, o sistema pode iniciar automaticamente uma mudança dos serviços para manter o desempenho. Esse evento é chamado "failover". O sistema pode iniciar o failover nas seguintes situações:

  • Espaço de armazenamento travado: Um espaço de armazenamento (ex.: volume, Grupo de discos, RAID Group, cache de SSD, etc.) no servidor ativo travou, mas o espaço de armazenamento correspondente no servidor passivo está funcionando normalmente. Observe que o sistema não iniciará um failover se não houver nenhum volume ou iSCSI LUN (nível de bloqueio) no espaço de armazenamento travado.
  • Erro de serviço: Ocorreu um erro em um serviço monitorado.
  • Falha de energia: O servidor ativo foi desligado ou reinicializado; as duas unidades de energia do servidor ativo falharam ou a energia foi perdida.

Falha de mudança

A mudança ou failover pode falhar nas seguintes situações:

  • Replicação incompleta de dados: Quando você primeiro combina dois servidores para criar um cluster de high-availability, o sistema requer tempo para replicar todos os dados do servidor ativo para o servidor passivo.
  • Espaço de armazenamento travado no servidor passivo: Um espaço de armazenamento (ex.: volume, Grupo de discos, RAID Group, etc.) no servidor passivo travou.
  • Falha de energia: O servidor passivo foi desligado ou reinicializado; as duas unidades de energia do servidor passivo falharam ou a energia foi perdida.
  • Atualização do DSM: Ao instalar atualizações de DSM, todos os serviços serão interrompidos e retomados após a instalação da atualização do DSM ser concluída.

Se você iniciar manualmente uma mudança, mas os serviços não puderem ser mudados para o servidor passivo, então o sistema tentará mudar os serviços de volta para o servidor ativo.

Caso o failover seja iniciado, mas nenhum servidor seja capaz de assumir o papel do servidor ativo devido a erros, o cluster de high-availability não poderá fornecer os serviços normalmente. Se os servidores não voltarem automaticamente para o status Independente, pressione o botão Restaurar nos dois servidores para restaurar as configurações padrão. Quando as configurações padrão tiverem sido restauradas, você poderá combinar os servidores e criar um cluster de high-availability novamente.

Fonte de alimentação ininterrupta

Os clusters de high-availability podem ser conectados a fontes de alimentação ininterrupta (UPS) para reduzir o risco de perda de energia inesperada. Se a energia for perdida, os servidores receberão energia de backup da unidade UPS conectada a ele. Quando a unidade UPS conectada ao servidor ativo começar a ficar com pouca energia, o sistema tentará mudar os serviços para o servidor passivo. Consulte as diretrizes abaixo.

  • Se os servidores ativos e passivos estiverem ligados ao mesmo switch de rede, recomenda-se conectar os dois servidores à mesma fonte de energia do switch de rede.
  • Se os servidores ativos e passivos estiverem ligados a switches de rede diferentes que usam fontes de energia diferentes (por exemplo: eletricidade de diferentes empresas de energia), recomenda-se conectar os servidores ativos e passivos a fontes de energia correspondentes, as mesmas que os seus switches de rede conectados.
  • Se os servidores ativos e passivos estiverem ligados a diferentes switches de rede que utilizam diferentes unidades de no-break, recomenda-se conectar os dois servidores em uma das unidades de no-break, em vez de unidades de no-break diferentes. Quando a unidade de no-break perder energia, o servidor ativo será notificado e entrará em modo seguro, e o servidor passivo será desligado corretamente. Se os servidores estiverem conectados a unidades de no-break diferentes, o servidor ativo não será capaz de detectar se o no-break do servidor passivo está com pouca energia e, portanto, o servidor passivo poderá ser desligado inesperadamente.

Observação:

O DiskStation suporta a conexão a dispositivos locais e no-break SNMP. Para obter mais informações, consulte a ajuda em Painel de Controle > Hardware > No-break

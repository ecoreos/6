Redes divididas

Quando um conjunto de high-availability estiver funcionando normalmente, apenas um dos servidores membros assume o papel de servidor ativo. Nesse caso, o servidor passivo detecta a presença do servidor ativo por meio da conexão Heartbeat e da conexão de dados.

Se todas as conexões Heartbeat e de dados forem perdidas, os dois servidores poderão tentar assumir o papel de servidor ativo. Essa situação é chamada de erro de "redes divididas". Neste caso, as conexões com os endereços IP do conjunto de high-availability serão redirecionadas para um dos dois servidores e dados inconsistentes poderão ser atualizados ou gravados nos dois servidores.

Quando uma das conexões Heartbeat ou de dados for restabelecida, o sistema detectará o erro de redes divididas e a inconsistência dos dados nos dois servidores e entrará no modo de segurança de high-availability.

Caso ocorra um erro de redes dividas:

  1. Os serviços nos dois servidores e os endereços IP do conjunto de high-availability permanecerão indisponíveis até que o erro de redes divididas seja corrigido.
  2. Depois que os dois servidores entrarem no modo de segurança de high-availability, uma nova guia chamada Redes divididas aparecerá no painel esquerdo. Nessa guia, as seguintes informações estarão listadas: a diferença entre os arquivos nas pastas compartilhadas nos dois servidores, a hora em que os servidores se tornaram servidores ativos, bem como informações sobre a última conexão de iSCSI Target. Todas as outras guias continuarão sendo somente leitura.
  3. No modo de segurança de high-availability, o File Station ficará no modo apenas leitura e você conseguirá baixar ou visualizar os arquivos.
  4. Na guia Visão geral, você somente tem permissão para resolver erros de redes separadas ou desligar o servidor de login atual. Para resolver erros de redes separadas, execute uma das ações a seguir:
    • Escolha um servidor como o servidor ativo do conjunto de high-availability e o outro como o servidor passivo. Quando os dois servidores forem reiniciados, todos os dados e configurações diferentes no servidor ativo serão sincronizados com o servidor passivo. Os dados atualizados no servidor passivo durante o erro de redes divididas serão perdidos.
    • Escolha um servidor como o servidor ativo do conjunto de high-availability e desvincule o outro. Depois que os dois servidores forem reiniciados, o servidor ativo continuará no conjunto de high-availability e o servidor desvinculado reservará seus dados e voltará para o status Independente. Uma replicação completa será necessária para vincular um novo servidor passivo no futuro.
    • Desvincule os dois servidores, reserve seus dados individuais e deixe-os voltar para o status Independente.

Notas:

  • Quanto mais arquivos houver nas pastas compartilhadas, maior será o tempo necessário para listar as diferenças.
  • Antes de escolher qual servidor será o ativo/passivo, certifique-se de que os dois estejam ligados.

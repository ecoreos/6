Assistente do High Availability Manager

Esse assistente o guiará através dos processos de criação e configuração do cluster de high-availability.

Para abrir esse assistente:

  1. Abra o Gerenciador de High Availability.
  2. Vá para a página Visão geral.
  3. Clique em Criar cluster de HA.

Nota:

  • Conclua o assistente de configuração do cluster de high-availability dentro do tempo limite dado. Se o limite de tempo for ultrapassado, você precisará iniciar o processo de configuração novamente.
  • Se você precisar identificar qual servidor é o servidor ativo de um conjunto de high-availability, recomendamos que baixe o aplicativo DS finder da Synology e use sua função bipe, que pode fazer com que o servidor ativo emita um bipe curto.

Para configurar o servidor passivo:

Designe um servidor Synology como servidor passivo. Sob condições normais, o servidor passivo não fornece serviços. Entretanto, em caso de defeito, os serviços podem ser transferidos do servidor ativo para o servidor passivo.

  1. Endereço IP: Digite o endereço IP do servidor passivo.
  2. Nome do usuário: Forneça uma conta de usuário pertencente ao grupo administrator no servidor de destino.
  3. Senha: Forneça a senha para a conta pertencente ao grupo administrator.

Nota:

  • Quando o cluster de high-availability for criado, todos os dados do servidor passivo serão excluídos. Certifique-se de que não haja nenhum dado importante armazenado no servidor passivo.

Para configurar a conexão de Heartbeat:

Conecte os dois servidores. Esta será a conexão de "Heartbeat" entre os dois servidores, facilitando a comunicação e permitindo que os dados sejam replicados do servidor ativo para o passivo. Para garantir a qualidade da conexão Heartbeat, o sistema verificará se a conexão atende aos seguintes critérios ao criar o cluster de high-availability:

  • Entre os dois servidores, não há nenhum outro dispositivo com endereço IP encontrado no mesmo domínio de transmissão. Se essa conexão Heartbeat passar através de qualquer switch, você deve garantir que os switches suportem multicast e Jumbo Frame.
  • Tem uma conexão de rede de baixa latência. O tempo de resposta de Ping entre os dois servidores deve ser inferior a 1 ms e a velocidade de transmissão deve ser de pelo menos 500 Mbps. O MTU será configurado automaticamente para 9000.
  • Usar a mesma interface de rede nos dois servidores Por exemplo, se uma extremidade estiver conectada à LAN 1 em um servidor, a outra deve estar conectada à LAN 1 no outro servidor.
  • Usar a interface de rede mais rápida nos dois servidores Se os servidores estiverem equipados com placas de interface de rede de 10 GbE complementares, essa conexão deve usá-las.

﻿Depois de confirmar os itens acima, vá para a seção Interface de conexão HA (Heartbeat) e escolha a interface de rede nos servidores que irá servir como conexão Heartbeat.

Nota:

  • As configurações da conexão Heartbeat produzirão dois conjuntos de endereços IP (por exemplo, 169.254.1.1 e 169.254.1.2) para fins de replicação de dados do servidor ativo para o servidor passivo. Certifique-se de que esses dois endereços IP não sejam usados por nenhum outro equipamento da rede.
  • A habilitação do ID de VLAN das portas de conexão Heartbeat de seu DiskStation não é suportada.

Configurando o Link Aggregation nas conexões de dados ou Heartbeat:

  • Para configurar o Link Aggregation nas conexões de dados ou Heartbeat, configure-o em duas ou mais portas LAN em Painel de controle > Rede > Interface de rede antes de abrir o assistente de criação de high availability. As configurações do Link Aggregation devem ser idênticas nos dois servidores (por exemplo, se você configurá-lo nas portas 1 e 2 do servidor ativo, deve configurá-lo nas portas 1 e 2 do servidor passivo).
  • O Heartbeat é equipado com recursos de failover e balanceamento de carga, mas as conexões de dados dependem do modo de Link Aggregation que você escolher.

Para ajustar as configurações da interface de rede de um cluster de high-availability:

O cluster de high-availability usará um endereço IP de cluster de high-availability como sua interface de conexão com o meio externo. Você pode se conectar usando esse endereço IP de conjunto de high-availability quando quiser usar os serviços DSM, independente de qual Synology NAS for o servidor ativo. Isso evita ter que se conectar a endereços IP diferentes após realizar uma mudança de um servidor ativo para o servidor passivo.

  1. Nome do servidor do cluster: O nome do conjunto de high-availability. Consulte as regras de nomenclatura do servidor (em Ajuda do DSM > Painel de Controle > Conectividade > Rede > Geral) para obter informações sobre as restrições de nome.
  2. Interface de rede: A interface de rede a ser usada pelo conjunto de high-availability. É necessário selecionar uma porta LAN diferente daquela designada para a replicação de dados na etapa anterior.
  3. Endereço IP: O endereço IP a ser usado pelo conjunto de high-availability. Independentemente de qual servidor está fornecendo serviços ativamente, o cluster pode ser conectado através desse IP.
  4. Máscara de sub-rede: A máscara de sub-rede a ser usada pelo conjunto de high-availability.

Atribuindo um ID de VLAN à conexão de dados:

  • Para atribuir um ID de VLAN à conexão de dados, vá para Painel de controle > Rede > Interface de rede antes de abrir o assistente de criação de high-availability. As configurações de VLAN devem ser idênticas nos dois servidores (por exemplo, se a LAN 1 for configurada com ID de VLAN 1 no servidor ativo, a LAN 1 do servidor passivo também deve ser VLAN 1).

Para confirmar as configurações:

Verifique novamente as configurações a serem usadas pelo cluster de high-availability. O sistema iniciará a vinculação do cluster de high-availability quando você clicar em Aplicar.

Nota:

  • Ao criar um cluster de high-availability, todos os dados do servidor ativo devem ser replicados para o servidor passivo. Se a conexão para o servidor passivo for perdida por mais de 30 minutos antes que o processo de replicação inicial esteja concluído, o servidor passivo voltará para o modo Independente. Nesse caso, você precisará vinculá-lo novamente e repetir a replicação completa dos dados.

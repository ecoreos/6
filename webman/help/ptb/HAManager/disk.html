Status do disco

Essa página contém guias que exibem o status dos discos do conjunto de high-availability. A guia respectiva de cada servidor exibe as seguintes informações.

  • Número: Indica que slot cada disco ocupa. Os discos dos servidores ativo e passivo que ocupam os slots correspondentes devem ter a mesma capacidade.
  • Tamanho do disco: Indica a capacidade de cada disco.
  • Temperatura: Indica a temperatura do disco.
  • Status S.M.A.R.T.: Indica o status S.M.A.R.T. de cada disco.
  • Status: Exibe informações do disco e notificações de erro.

Para substituir discos com defeito:

  1. Verifique o número do disco.
  2. Substitua o disco com defeito por um novo. Para informar-se sobre os passos da instalação do disco rígido, consulte o Guia de instalação rápida do seu produto Synology.
  3. Vá para Status do armazenamento.
  4. Selecione o volume danificado.
  5. Clique em Reparar para reparar o volume.

Observação:

A capacidade dos discos rígidos de substituição deve ser igual ou maior do que o disco rígido com defeito.

Para adicionar novos discos a um conjunto de high-availability:

Os discos rígidos podem ser adicionados ao conjunto de high-availability para expandir a capacidade. Antes de adicionar qualquer disco rígido, veja abaixo.

  • Recomendamos adicionar discos quando os servidores ativo e passivo estiverem on-line.
  • A capacidade dos discos rígidos do servidor passivo deve ser igual ou maior do que a dos discos do servidor ativo.

Para adicionar discos rígidos:

  1. Insira os discos rígidos nos slots correspondentes dos servidores ativo e passivo. Por exemplo, se você inserir um disco rígido no slot 1 do servidor ativo, deverá inserir o disco rígido correspondente no slot 1 do servidor passivo.
  2. Vá para Gerenciador de armazenamento > Volume (or Grupo de discos/iSCSI LUN/RAID Group dependendo de sua situação e modelo). Selecione o volume que deseja expandir, clique no botão Gerenciar e siga as instruções para adicionar os discos rígidos.
  3. Quando os discos rígidos forem adicionados com sucesso, volte para Gerenciador de High Availability > Status do armazenamento para verificar se o status está Normal.

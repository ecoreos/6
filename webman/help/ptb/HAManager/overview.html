Visão geral

Essa página exibe informações básicas sobre o cluster de alta disponibilidade e os servidores membro. Além disso, essa página fornece ferramentas para criar, remover ou realizar a manutenção em clusters de high-availability e servidores membros.

Status do cluster de High Availability

Esse campo exibe o status geral do cluster de high-availability, alertando-o para qualquer erro. A seguir, uma tabela das condições possíveis exibidas nesse campo:

Status Descrição
Independente Os servidores não são combinados em um cluster de high-availability.
Boas condições A condição geral do cluster de high-availability é boa. A replicação de dados do servidor ativo para o servidor passivo está funcionando normalmente.
Preparando Os servidores foram combinados com sucesso em um cluster de high-availability, mas a replicação inicial de dados ainda não terminou. A mudança não pode ser realizada nesse momento. Para mais detalhes, consulte Mudança.
Carregando O sistema está carregando o status do cluster. A Mudança não pode ser realizada até que o carregamento do status seja concluído.
Aviso O status do cluster HA está anormal, mas os serviços ainda estão funcionando.
Perigo Os servidores ativo e passivo estão com defeito. O cluster de high-availability não pode funcionar normalmente. Verifique o status de todos os espaços de armazenamento (ex.: grupos RAID, grupos de disco, volumes e iSCSI LUNs).

Mensagens de advertência

A tabela a seguir mostra as mensagens de advertência que podem aparecer quando ocorre um erro no sistema. Quando mais de uma mensagem de advertência aparecer, você pode navegar entre elas com os botões de avanço/retrocesso.

Advertência/Erro Descrição
Erro de serviço monitorado Ocorreu em erro em um ou mais serviços monitorados.
Espaço de armazenamento degradado Um espaço de armazenamento (ex: RAID Group, Grupo de Disco, volume, iSCSI LUN e cache SSD) no servidor ativo ou passivo está degradado. Verifique a página Status de Armazenamento.
Espaço de armazenamento paralisado Um espaço de armazenamento (ex: RAID Group, Grupo de Disco, volume, iSCSI LUN e cache SSD) no servidor ativo ou passivo travou. Verifique a página Status de Armazenamento.
Falha ao detectar o servidor passivo Esse erro pode ocorrer porque o servidor passivo está offline ou os servidores ativo e passivo não podem se conectar pela conexão Heartbeat. Nesse caso, a mudança estará indisponível.
Os dados não podem ser replicados no servidor passivo Esse erro pode ocorrer devido ao volume de armazenamento ou falha do disco. Nesse caso, a mudança estará indisponível. Verifique a página Status de Armazenamento.
O servidor passivo não foi selecionado Se o servidor passivo estiver desvinculado do cluster de high-availability, esse status não será exibido. Nesse caso, a mudança estará indisponível. Se você quiser revincular o servidor passivo, clique no menu suspenso Manutenção e clique em Vincular. Quando se revincula o servidor passivo, é necessária uma replicação completa de dados.

Informações do cluster de High Availability

Este campo está localizado abaixo do cluster de high-availability e exibe as informações básicas sobre o cluster. Se esse cluster não existir, o campo fornece apenas uma explicação da função dos clusters de high-availability.

Esse campo inclui as seguintes informações:

  • Nome do servidor do cluster:O nome do servidor usado pelo conjunto de high-availability.
  • Endereço IP do cluster:O endereço IP usado pelo conjunto de high-availability. Independentemente de qual servidor está realizando os serviços, os usuários podem se conectar ao cluster de high-availability usando esse endereço IP.
  • Tempo interno: A data de criação do cluster de high-availability.

Botões de operação do cluster de High Availability

Os dois botões a seguir estão localizados abaixo das informações do cluster de high-availability:

  1. Criar cluster de HA: Quando não houver nenhum conjunto de high-availability, você pode clicar nesse botão para combinar dois servidores e criar um conjunto de high-availability. Para obter mais informações sobre a criação de um cluster de high-availability, consulte o Assistente do Synology High Availability.
  2. Gerenciamento: Esse botão está disponível apenas quando já existe um conjunto de high-availability. Ele inclui as seguintes opções:
    • Assistente de gerenciamento: Inicia um assistente que inclui instruções para diversos procedimentos de gerenciamento.
    • Mudança: Inicia uma mudança do servidor ativo para o servidor passivo. Durante esse processo, o servidor ativo deixa de prestar serviços e torna-se o novo servidor passivo. Em seguida, o antigo servidor passivo torna-se o novo servidor ativo e começa a prestar todos os serviços do cluster de high-availability. Esta opção não está disponível quando o servidor passivo está no status erro ou não está selecionado.
    • Atualização do DSM: Abre a página Atualização do DSM do Painel de controle. A partir daqui, os dois servidores podem ser atualizados.
    • Remover cluster de HA: Remove o cluster de high-availability.
    • Desvincular servidor passivo: Remove o servidor passivo do conjunto. Quando o servidor é adicionado ao cluster de high-availability novamente após a desvinculação, é necessária uma replicação completa de dados. Portanto, recomendamos o uso dessa opção apenas quando você desejar substituir o servidor passivo existente por um novo.
    • Vincular servidor passivo: Adiciona um servidor passivo ao conjunto. Essa opção está disponível apenas quando não existe um servidor passivo no cluster de high-availability.
    • Desligar cluster de HA: Desliga os servidores ativo e passivo.
    • Desligar servidor ativo: Desliga o servidor ativo.
    • Desligar servidor passivo: Desliga o servidor passivo.
    • Reiniciar servidor ativo: Reinicia o servidor ativo.
    • Reinicializar servidor passivo: Reinicia o servidor passivo.
    • Desativar o bipe do servidor passivo: Interrompe o som do bipe do servidor passivo quando ocorrer um erro. O servidor passivo ainda emite bipes se ocorrer outro erro mais tarde.

Nota:

  • Quando um cluster de high-availability é removido, será necessária a replicação completa de dados quando você quiser criar um novo cluster.
  • Se você desligar ou desvincular o servidor passivo, os serviços não podem ser mudados se ocorrerem erros no servidor ativo.

Status de replicação de dados e do servidor

Essa seção contém imagens que indicam o status dos servidores ativo e passivo, bem como o fluxo de replicação de dados entre os servidores.

A imagem da replicação de dados indica as seguintes condições possíveis:

  • Replicação de dados funcionando normalmente: Indicado pelo fluxo de dados do servidor ativo para o servidor passivo.
  • Nenhuma replicação de dados: Indicado pela ausência da imagem do fluxo de dados.

Informações sobre o servidor

Essa seção localiza-se embaixo das imagens do servidor. As informações relativas aos servidores ativo e passivo são exibidas nessa seção. Se nenhum servidor passivo foi selecionado, essa seção indicará N/A.

  • Nome do servidor: Nome do servidor do Synology.
  • Nome do modelo: Modelo do servidor Synology.
  • Número de série: Número de série do servidor Synology.
  • Status da ventoinha: Status das ventoinhas de refrigeração do servidor. Se uma ventoinha apresentar defeito, aparecerá um aviso. Nesse caso, entre em contato com o distribuidor local para obter assistência.
  • Temperatura: Temperatura do servidor Synology.
  • Status de energia: Indica se as fontes de alimentação estão funcionando normalmente no servidor ou não. Se o modelo do servidor for equipado com fontes de alimentação redundantes, aparecerá um aviso se uma delas estiver com defeito.

Visão geral

Essa página exibe informações básicas sobre o cluster de high-availability e os servidores membro. Além disso, essa página fornece ferramentas para realizar a manutenção em clusters de high-availability e servidores membros.

Status do cluster de High-Availability

Esse campo exibe o status geral do cluster de high-availability, alertando-o para qualquer erro. A seguir, uma tabela das condições possíveis exibidas nesse campo:

Status Descrição
Boas condições A condição geral do cluster de high-availability é boa.
Carregando O sistema está carregando o status do cluster. A Mudança não pode ser realizada até que o carregamento do status seja concluído. Consulte Mudança.
Configurando O sistema está sendo configurado. A Mudança não pode ser realizada até que a configuração esteja concluída.
Aviso O status do cluster HA está anormal, mas os serviços ainda estão funcionando. A Mudança pode não ser realizada nessa situação.
Perigo Os servidores ativo e passivo estão com defeito ou o armazenamento está danificado. O cluster de high-availability não pode funcionar normalmente. Verifique o status de todos os espaços de armazenamento (ex.: grupos RAID, grupos de disco, volumes e iSCSI LUNs). A Mudança não pode ser realizada nessa situação.

Mensagens de advertência

A tabela a seguir mostra as mensagens de advertência que podem aparecer quando ocorre um erro no sistema. Quando mais de uma mensagem de advertência aparecer, você pode navegar entre elas com os botões de avanço/retrocesso.

Advertência/Erro Descrição
Não há nenhum servidor passivo no grupo de high-availability As informações do servidor passivo não estão disponíveis. Certifique-se de que o servidor passivo esteja conectado corretamente. Nesse caso, a mudança estará indisponível.
Falha ao detectar o servidor passivo Esse erro pode ocorrer porque o servidor passivo está offline ou os servidores ativo e passivo não podem se comunicar através da conexão Heartbeat. Nesse caso, a mudança estará indisponível.
Falha ao executar failover Um failover foi disparado no servidor ativo mas falhou. Nesse caso, o servidor ativo não tentará outro failover. Para eliminar esse erro, siga as etapas abaixo:
  1. Tente solucionar os problemas no conjunto de high-availability.
  2. Reinicialize o servidor passivo.
  3. Execute uma mudança clicando em Gerenciar > Mudança.
Erro de alvo monitorado Ocorreu um erro em um ou mais alvos monitorados, incluindo status do sistema, serviço ou interface de rede.
Os erros ocorreram na conexão entre as unidades de expansão e o servidor ativo/passivo As unidades de expansão não estão conectadas corretamente ao servidor ativo ou passivo. Certifique-se de que a conexão com o servidor ativo esteja correta e configure a mesma conexão com o servidor passivo. Nesse caso, a mudança estará indisponível.
Um volume foi degradado Um espaço de armazenamento (ex: RAID Group, Grupo de Disco, volume, iSCSI LUN e cache SSD) está degradado. Verifique a página Status de Armazenamento.
Um volume foi danificado Um espaço de armazenamento (ex: RAID Group, Grupo de Disco, volume, iSCSI LUN e cache SSD) está danificado. Verifique a página Status de Armazenamento. O failover será executado automaticamente se o servidor passivo conseguir reconhecer os drives que o servidor não conseguiu acessar. Caso contrário, a mudança ficará indisponível.
A conexão Heartbeat foi perdida Esse erro pode ocorrer devido a uma falha de conexão Heartbeat. Nesse caso, a mudança estará indisponível.
Defeito no hardware Esse erro pode ocorrer devido a falha no hardware (ex.: unidade de alimentação ou ventoinha do sistema) no servidor ativo ou servidor passivo.
Tamanho físico de RAM diferente Esse erro pode ocorrer devido ao tamanho físico da RAM incompatível no servidor ativo e passivo. Nesse caso, a mudança estará indisponível.
Nenhum IP estático foi atribuído a esse grupo high-availability Esse erro pode ocorrer devido à falta do IP estático para o grupo de high-availability. Nesse caso, a mudança estará indisponível.
Não há IP estático na interface monitorada Esse erro pode ocorrer devido à falta do IP estático para a interface monitorada. Nesse caso, a mudança estará indisponível.
Recuperação do servidor passivo A recuperação do servidor passivo está em andamento. Nesse caso, a mudança estará indisponível.
Atualizando o servidor passivo O sistema está atualizando o servidor passivo. Nesse caso, a mudança estará indisponível.

Botão "Gerenciar" do Grupo de High-Availability

Esse botão localiza-se abaixo das informações do grupo de high-availability. Ele inclui as seguintes operações:

  • Mudança: Inicia uma mudança do servidor ativo para o servidor passivo. Durante esse processo, o servidor ativo deixa de prestar serviços e torna-se o novo servidor passivo. Em seguida, o antigo servidor passivo torna-se o novo servidor ativo e começa a prestar todos os serviços do cluster de high-availability. Esta opção não está disponível quando o servidor passivo está no status erro ou não está selecionado.
  • Atualização do DSM: Abre a página Atualização do DSM do Painel de controle. A partir daqui, os dois servidores podem ser atualizados.
  • Desligamento do grupo de high-availability: Desliga os servidores ativo e passivo.
  • Desligar servidor ativo: Desliga o servidor ativo. Os serviços serão alternados se o servidor passivo estiver íntegro.
  • Desligar servidor passivo: Desliga o servidor passivo.
  • Reiniciar servidor ativo: Reinicia o servidor ativo. Os serviços serão alternados se o servidor passivo estiver íntegro.
  • Reinicializar servidor passivo: Reinicia o servidor passivo.
  • Recupere o servidor desconhecido: Recupera o servidor passivo se o servidor passivo for substituído ou ainda for reconhecido como desconhecido.
  • Bipe do sistema desligado: Interrompe o som do bipe do servidor ativo e passivo quando ocorrer um erro. O sistema ainda emitirá um bipe no servidor ativo ou passivo se ocorrer outro erro posteriormente.

Observação:

  • Se você desligar o servidor passivo ou não houver nenhum servidor passivo, os serviços não podem ser alterados quando ocorrerem erros no servidor ativo.

Informações e diagrama do servidor

O diagrama do servidor mostra o status do servidor. O local corresponde ao lado dos módulos aos quais o servidor está conectado. Passe o mouse sobre ele para obter mais detalhes.

Ele inclui as seguintes situações:

  • Não há nenhum servidor passivo no grupo de high-availability: Exibe um quadro em branco.
  • Falha ao detectar o servidor passivo: Exibe uma imagem de servidor desfocada.
  • Ponto de exclamação: O hardware pode apresentar defeito ou as informações do servidor não estão disponíveis devido à falha da conexão Heartbeat.

Observação:

  • Se aparecer "desconhecido" no outro servidor, significa que ele não pertence ao grupo de high-availability. Clique em Gerenciar e selecione Recuperar servidor desconhecido para reparar esse problema.

As informações do servidor incluem as seguintes entradas. Elas mostram N/D se nenhum servidor passivo for detectado.

  • Número de série: Número de série do servidor Synology.
  • Status da ventoinha: Status das ventoinhas de refrigeração do servidor. Se uma das ventoinhas apresentar defeito, aparecerá um aviso. Nesse caso, entre em contato com o distribuidor local para obter assistência.
  • Temperatura: Temperatura do servidor Synology.
  • Status de energia: Indica se as fontes de alimentação do servidor estão todas funcionando normalmente. Se o servidor for equipado com fontes de alimentação redundantes, aparecerá um aviso se uma delas estiver com defeito.
  • Memória física: Tamanho físico da RAM do servidor Synology. A mudança não será realizada se os dois servidores não tiverem o mesmo tamanho físico de memória RAM.

Status de conectividade entre os servidores

Esta seção contém imagens entre os servidores ativo e passivo. Ele inclui as seguintes situações:

  • Nenhuma imagem: Indica que nenhum servidor passivo foi detectado.
  • Imagem estática: Indica a conexão Heartbeat incorreta.
  • Ponto de exclamação: Indica comunicação anormal entre os dois servidores ou conexão Heartbeat incorreta.

Status da interface da rede

Esta seção contém imagens entre os servidores e a rede de dados. Ele inclui as seguintes situações:

  • Nenhum ponto de exclamação: Indica que nenhuma falha de rede foi detectada.
  • Ponto de exclamação: Indica a ocorrência de falha na interface da rede monitorada.

Informações e diagrama do armazenamento compartilhado

O diagrama do armazenamento compartilhado mostra o status da conectividade entre os servidores e o armazenamento compartilhado. Passe o mouse sobre ele para obter informações básicas.

Ele inclui as seguintes situações:

  • Nenhum ponto de exclamação: Indica que nenhum erro foi detectado.
  • Ponto de exclamação: Indica status de conectividade anormal das unidades de expansão.

As informações básicas do armazenamento compartilhado incluem as seguintes entradas. Vá para Gerenciador de armazenamento para obter informações mais detalhadas.

  • Status da ventoinha: Exibe o status das ventoinhas das unidades de expansão. Se uma ventoinha apresentar defeito, aparecerá um aviso. Nesse caso, entre em contato com o distribuidor local para obter assistência.
  • Status de energia: Indica se as fontes de alimentação do servidor estão todas funcionando normalmente. Uma advertência aparecerá se um delas apresentar defeito.

Redes divididas

Quando um conjunto de high-availability estiver funcionando normalmente, apenas um dos servidores membros assume o papel de servidor ativo. Nesse caso, o servidor passivo detecta a presença do servidor ativo através da conexão Heartbeat bem como a conexão de dados com as unidades de expansão compartilhadas, para que a mudança não seja disparada. Além disso, o servidor passivo detecta o status do armazenamento compartilhado para evitar assumir erroneamente o papel do servidor ativo quando o servidor ativo atual ainda acessar normalmente as unidades de expansão compartilhadas.

Após a perda do Heartbeat e de todas as conexões de dados ou do status instável do armazenamento compartilhado e do sistema, cada um dos dois servidores pode considerar-se como o servidor ativo. Essa situação é chamada de "redes separadas". Nesse caso, para evitar o corrompimento dos dados devido a acessos concomitantes dos dois servidores, um servidor tentará continuar fornecendo os serviços enquanto o outro reiniciará e exigirá uma verificação posterior da conexão. As conexões dos usuários clientes podem ficar instáveis por um momento durante essa situação.

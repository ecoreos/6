Mudança

Para garantir o desempenho contínuo e minimizar o tempo parado, os serviços podem ser mudados do servidor ativo de um cluster de high-availability para a espera, o servidor passivo. Esse recurso é chamado "Mudança" ou, em alguns casos, "Failover". Esta seção contém informações básicas desses dois conceitos.

Mudança

Mudança refere-se a uma mudança de um servidor ativo para a espera, o servidor passivo de um cluster de high-availability. Sob condições normais, o servidor ativo gerencia todos os serviços. Entretanto, caso o servidor ativo apresente defeito, você pode iniciar uma mudança e transferir todos os serviços para o servidor passivo, considerando que o servidor passivo esteja funcionando normalmente. Nesse caso, o servidor ativo assume o papel do servidor passivo e vice-versa.

Para iniciar manualmente uma mudança:

  1. Vá para a página Visão geral.
  2. Clique em Gerenciar > Mudança.

Nota:

Antes da mudança manual, recomenda-se configurar um IP estático e fazer login no DSM com esse IP estático para evitar perda de página do DSM devido à alteração do IP após a mudança.

Nota:

Quando uma mudança for iniciada, o servidor ativo primeiro assume o papel do servidor passivo em seguida, o servidor passivo se torna o servidor ativo. Assim, haverá um curto período em que os dois servidores se tornarão passivos e não poderão fornecer os serviços normalmente. O tempo para concluir uma mudança varia dependendo do número e do tamanho dos volumes ou iSCSI LUNs (nível de bloqueio) ou do número e do total de carga dos serviços e pacotes do grupo de high-availability.

Failover

Quando ocorrem certos erros, o sistema pode iniciar automaticamente uma mudança dos serviços para minimizar o tempo parado de serviço. Esse evento é chamado "failover". O sistema pode iniciar o failover nas seguintes situações, considerando-se que o servidor passivo esteja funcionando normalmente:

  • Falha de energia:
    • O servidor ativo foi desligado ou reiniciado.
    • As unidades de alimentação do servidor ativo falharam.
    • Sem fonte de alimentação.
  • Erro na unidade: O número de caminhos da unidade no servidor ativo é menor do que do servidor passivo.
  • Erro de serviço: Ocorreu um erro em um serviço monitorado.
  • Erro de conexão: Ocorreu um erro de conexão em uma interface monitorada.

Falha de mudança

A mudança ou failover pode falhar nas seguintes situações:

  • Conexão Heartbeat anormal: O status da conexão do servidor ativo/passivo não está normal.
  • Tamanho de memória incompatível: Os servidores ativo e passivo não têm tamanhos de memória RAM idênticos.
  • Falha de energia:
    • O servidor passivo foi desligado ou reiniciado.
    • As unidades de alimentação do servidor passivo falharam.
    • Sem fonte de alimentação.
  • Erro de conexão: Ocorreu um erro de conexão em uma interface monitorada no servidor passivo.
  • Erro de serviço: Existem erros nos serviços monitorados no servidor passivo.
  • Erro de conexão da unidade de expansão: A conexão das unidades de expansão dos dois servidores não está na mesma ordem.
  • Erro na unidade: O número de caminhos da unidade no servidor passivo é menor do que do servidor ativo.
  • Atualização do DSM: Ao instalar atualizações de DSM, todos os serviços serão interrompidos e retomados após a instalação da atualização do DSM ser concluída.

Se você iniciar manualmente uma mudança, mas os serviços não puderem ser mudados para o servidor passivo, então o sistema tentará mudar os serviços de volta para o servidor ativo.

Fonte de alimentação ininterrupta

Os grupos de high-availability podem ser conectados a fontes de alimentação ininterruptas (UPS) de rede para reduzir o risco de perda de energia inesperada. Se a energia for perdida, os servidores receberão energia de backup da unidade UPS conectada a ele. Quando a unidade UPS conectada ao servidor ativo começar a ficar sem energia, o servidor passivo se desligará e o servidor ativo entrará no modo de segurança. Após a retomada da energia, o servidor ativo reiniciará e tentará ligar o outro servidor transmitindo uma mensagem Wake-on-LAN.

  • Recomenda-se conectar os dois servidores e as unidades de expansão na mesma UPS dos interruptores da rede.

Nota:

DiskStation suporta conexão apenas a dispositivos UPS de rede. Para obter mais informações, consulte a ajuda em Painel de Controle > Hardware > UPS.

Gerenciador de High Availability

High Availability refere-se a uma solução de layout de servidor desenvolvida para reduzir interrupções de serviços causadas por falhas do servidor.O Synology High Availability usa dois servidores para formar um “cluster de high-availability”, onde um servidor assume o papel de “servidor ativo” e o outro servidor funciona como “servidor passivo” de espera.Para implementar e criar um grupo de high-availability, é necessário ter dois servidores Synology compatíveis.

Um grupo de high-availability usa os seguintes componentes e conceitos:

  • Conceito: A implementação do Synology High Availability requer dois servidores Synology para criar um grupo de high-availability.Um servidor funciona como “servidor ativo” e o outro como “servidor passivo”.Os servidores ativo e passivo no grupo de high-availability são conectados às mesmas unidades de expansão e compartilham o mesmo armazenamento neles.Para garantir a integridade dos dados, apenas o servidor ativo pode manusear a carga de trabalho.Em caso de falha do servidor ativo, o servidor passivo estará pronto para assumir todos os serviços.Assim, o grupo de high-availability continuará fornecendo serviços normalmente, minimizando o tempo de inatividade do servidor.Para obter mais informações sobre as mudanças, consulte Mudança.
  • Servidor ativo: O servidor ativo gerencia o fornecimento de todos os serviços.Caso ocorram erros no servidor ativo, você ou o sistema podem iniciar uma mudança para o servidor passivo.Nesse caso, o fornecimento de serviços é transferido para o servidor passivo, ponto em que o servidor ativo se torna o servidor passivo.
  • Servidor passivo: Sob condições normais, o servidor passivo permanece em modo de espera.Se o servidor ativo não puder fornecer os serviços, você ou o sistema podem iniciar uma mudança para o servidor passivo.
  • Conexão Heartbeat: Os servidores ativo e passivo de um grupo de high-availability estão vinculados por uma conexão de rede privada dedicada conhecida como “Conexão Heartbeat”.Se o contato entre os servidores se perder por um certo tempo, o sistema determinará a necessidade de uma mudança.
  • Armazenamento compartilhado do grupo de high-availability: As unidades nas unidades de expansão conectadas pelos servidores ativo e passivo ao mesmo tempo funcionam como o armazenamento compartilhado no grupo de high-availability.Apenas o servidor ativo tem a permissão de gravação no armazenamento.
  • Nome do servidor do grupo de high-availability e configuração da rede: O nome do servidor e as configurações da rede do grupo de High-availability serão determinados pelas configurações correspondentes no DSM.Na rede, você pode se conectar usando esse nome de servidor e os serviços serão fornecidos pelo servidor ativo.Para obter mais informações sobre as configurações da rede, consulte aqui.

Monitor

Essa página contém três guias que exibem o monitor de status do sistema, o serviço e a interface de rede. Essa página permite o monitoramento de determinados alvos. Se um alvo monitorado falhar no servidor ativo, o sistema acionará o failover para o modo de espera, servidor passivo (desde que o servidor passivo esteja funcionando normalmente). As seguintes guias aparecem nessa página:

Guia de status do sistema

O status do sistema é monitorado por padrão e não pode ser desabilitado. O failover será disparado quando ocorrerem falhas de sistema no servidor ativo. As seguintes seções aparecem nessa guia:

  • Status do servidor ativo: Failover sobre a falha do servidor ativo para fornecer serviços. Por exemplo as duas unidades de energia do servidor ativo falham.
  • Conectividade do armazenamento: Failover sobre a situação em que uma unidade está ausente no servidor ativo, mas presente no servidor passivo.

Guia de serviço

Exibe os serviços que podem ser monitorados e seus status. O failover será disparado quando os serviços estiverem anormais. As seguintes seções aparecem nessa guia:

  • Serviço: Exibe os serviços que podem ser monitorados.
  • Status: Exibe se os serviços estão funcionando ou não.
  • Monitorar: Marque a caixa de seleção do lado de um serviço para começar a monitorá-lo. O sistema ativará o failover se o serviço selecionado apresentar uma falha.

Para monitorar um serviço:

  1. Encontre o serviço que deseja monitorar. Certifique-se de que o status de serviço esteja normal.
  2. Marque a caixa de seleção ao lado dele.
  3. Clique em Aplicar para salvar as configurações.

Nota:

Os serviços que podem ser monitorados são os seguintes: Serviço de Arquivos Windows, iSCSI Target, FTP, Serviço de Arquivos Mac, Serviço NFS.

Guia de Interface de rede

Exibe todas as interfaces de rede lógicas que podem ser monitoradas. O failover será disparado se a conexão de rede estiver anormal. (Consulte as Notas como a seguir). As seguintes seções aparecem nessa guia:

  • Interface de rede: Exibe as interfaces que podem ser monitoradas.
  • Monitorar: Marque a caixa de seleção do lado de uma interface para começar a monitorá-la.

Nota:

Se a caixa de seleção Monitorar estiver marcada, o sistema ativará o failover para o servidor passivo quando a conexão de rede do servidor ativo apresentar falhas ou mau funcionamento, e a conexão de rede do servidor passivo ainda estiver funcionando normalmente.

  • O failover será acionado apenas se o servidor passivo tiver conexões em bom estado. Por exemplo, se o servidor ativo tiver três conexões e uma delas falhar, o sistema acionará um failover apenas se o servidor passivo tiver três ou mais conexões saudáveis.
  • Conexões combinadas com Link Aggregation contam como uma conexão. Por exemplo, se duas conexões forem combinadas e uma delas falhar, mas a outra ainda estiver funcionando, a conexão ainda será considerada saudável.

Para monitorar uma interface:

  1. Encontre a interface que deseja monitorar. Certifique-se de que os cabos tenham sido conectados corretamente nas interfaces dos servidores ativo e passivo.
  2. Marque a caixa de seleção ao lado dele.
  3. Clique em Aplicar para salvar as configurações.

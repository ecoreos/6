Conexão remota

No File Station, você pode se conectar aos serviços na nuvem pública (Box, Dropbox, Google Drive e Microsoft OneDrive) e servidores de arquivos através de vários protocolos diferentes (FTP, STFP, WebDAV e WebDAV HTTPS).

Para se conectar a um serviço na nuvem:

  1. Vá para Ferramentas > Conexão remota > Configuração da conexão.
  2. Selecione um serviço na nuvem desejado clicando no ícone e clique em Avançar.
  3. Na janela pop-up, siga as instruções do assistente para a autenticação da conta.
  4. Quando a configuração estiver concluída, o serviço na nuvem alvo aparecerá no painel esquerdo.

Limitações específicas da nuvem:

  • Box:
    • Um catálogo compartilhado com outros colaboradores terá o sufixo do nome do usuário do proprietário.
    • Se os catálogos compartilhados com outros colaboradores usarem um nome idêntico (um nome de catálogo com o sufixo do nome de usuário do proprietário), apenas um catálogo será exibido.
    • Somente através do Box você pode abrir arquivos com estas extensões: .gdoc, .gsheet e .boxnote.
    • Os nomes de arquivos diferenciam maiúsculas de minúsculas.
    • Para informar-se sobre as configurações adequadas de firewall consulte esse artigo.
  • Dropbox:
    • Um catálogo do Dropbox não pode ser exibido se contiver mais de 25000 arquivos.
    • Um catálogo do Dropbox não pode ser renomeado se contiver mais de 10000 arquivos.
    • Os arquivos que ultrapassarem 10 GB não podem ser carregados.
    • Uma tarefa de carregamento falhará se o arquivo alvo demorar mais de 24 horas para ser carregado.
    • Os nomes de arquivos diferenciam maiúsculas de minúsculas.
    • Para informar-se sobre as configurações adequadas de firewall consulte esse artigo.
  • Google Drive:
    • Arquivos/pastas com um caminho com mais de 2048 caracteres não serão exibidos.
    • Arquivos com um nome de arquivo com mais de 255 caracteres não serão exibidos.
    • Arquivos/catálogos com um nome idêntico serão marcados com horários de criação individuais.
    • Se você abrir um arquivo específico do Google (e.g., Google Docs e Google Slides) através do File Station, um URL será exibido para que você acesse o arquivo alvo.
    • As pastas My Drive e Starred exibidas no File Station correspondem respectivamente às pastas "My Drive" e "Starred" no Google Drive.
    • Para informar-se sobre as configurações adequadas de firewall consulte esse artigo.
  • Microsoft OneDrive:
    • Os arquivos de bloco de anotações do OneNote não serão exibidos.
    • Os arquivos que ultrapassarem 10 GB não podem ser carregados.
    • Os arquivos (e.g., .png, .jpg, .docx, .pdf) podem exibir tamanhos de arquivo maiores (os tamanhos originais mais os tamanhos das miniaturas). Isso não afeta seus tamanhos de arquivo originais quando forem baixados.
    • Os nomes de arquivos diferenciam maiúsculas de minúsculas.

Para conhecer mais limitações, sugerimos que você acesse o site oficial do serviço de nuvem correspondente.

Para conectar a um servidor de arquivos:

  1. Vá para Ferramentas > Conexão remota > Configuração da conexão.
  2. Selecione um protocolo de servidor desejado clicando no ícone e clique em Avançar.
  3. Forneça as seguintes informações para concluir a configuração:
    • Nome de host ou IP: Digite o nome de host ou IP do servidor do arquivo alvo.
    • Porta: Digite o número da porta do servidor do arquivo alvo.
    • Usuário e Senha: Digite essas informações de login se o servidor de arquivo alvo exigir uma conta/senha.
    • Página de código: Selecione a codificação de caracteres do servidor de arquivo alvo para exibir os nomes de arquivo corretamente.
    • Nome do perfil: Digite um nome exclusivo para identificar facilmente este servidor de arquivos.
    • Uso da conexão WebDAV HTTPS: Essa opção está disponível apenas quando você está configurando um protocolo WebDAV e quando o cliente e o servidor de arquivos suportam WebDAV HTTPS.
    • Permitir apenas uma conexão por usuário: Essa opção está disponível apenas quando você está configurando um protocolo FTP ou SFTP. Marque essa opção para otimizar a conectividade quando o servidor tiver limitações no número de conexões.
  4. Clique em Aplicar para salvar as configurações. O servidor de arquivo alvo aparecerá no painel esquerdo.

Para modificar um serviço de nuvem existente ou perfil de servidor de arquivo:

  1. Vá para Ferramentas > Conexão remota > Lista de conexões.
  2. Clique no serviço de nuvem alvo ou servidor de arquivo para os seguintes gerenciamentos:
    • Edição: Clique em Editar, altera a configuração de perfil atual e clique em OK.
    • Desconexão: Clique em Desconectar para encerrar a conexão.
    • Reconexão: Clique em Conectar para recuperar a conexão. (Como alternativa, clique no alvo no painel esquerdo do File Station para se reconectar.)
    • Exclusão: Clique em Excluir e depois confirme para abandonar o perfil. (Como alternativa, clique no alvo no painel esquerdo do File Station e clique em Excluir para a exclusão.)

Nota:

  • Cada usuário pode se conectar simultaneamente a não mais do que 100 serviços de nuvem e servidores de arquivo.
  • Certifique-se de selecionar a página de códigos do servidor correto para evitar erros inesperados. Recomenda-se o Unicode caso você não saiba a página de código do servidor.
  • Determinadas funções do File Station (por exemplo, Compactar para, Extrair, Visualizar, etc.) não pode ser aplicadas aos arquivos armazenados em serviços na nuvem ou servidores de arquivos.
  • As velocidades individuais de transferência para cada serviço na nuvem ou servidor de arquivos conectados estão sujeitas às configurações de limite de velocidade do usuário.
  • Cada usuário está limitado a três transferências de arquivo simultâneas em todos os serviços na nuvem e servidores de arquivos.
  • Para ajustar as configurações WebDAV, WebDAV HTTPS e proxy de serviço de nuvem, vá para Painel de controle > Rede > Geral > Proxy. Se você alterou as configurações de proxy durante a conexão a um serviço de nuvem ou servidor de arquivos, desconecte-se e reconecte-se.
  • A modificação de informações de servidores de arquivos ou serviços de nuvem conectados não afetará o status da conexão.
  • Para liberar o recurso de conexão, um servidor de arquivo/serviço na nuvem se desconectará automaticamente se os arquivos contidos permanecerem intocados por mais de 30 minutos. Clique no servidor de arquivo/serviço na nuvem para recuperar a conexão.
  • O sistema não pode exibir arquivos/pastas que adotam os seguintes formatos de nomes:
    • Começando com ._ ou .SYNOPPSDB
    • Incluindo /
    • Nomeie exatamente como .    ..    '    "    .DS_Store    @eaDir    @sharebin    @tmp    .SynologyWorkingDirectory
  • A conexão remota é suportada apenas no modo de alto privilégio. Para usar esse recurso, vá para o pacote Docker em seu DSM host e selecione o Docker DSM com o qual deseja habilitar as pastas compartilhadas criptografadas, clique em Editar e marque a caixa ao lado de Executar o contêiner usando altos privilégios.

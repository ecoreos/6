Serviços de backup

As opções em Backup e Replicação > Serviços de Backup permitem aos outros usuários fazer backup de dados dos seus dispositivos DiskStation ou clientes compatíveis com rsync para seu DiskStation.

Ativar o serviço de backup de rede

Se você desejar fazer backup dos dados de um DiskStation ou de um servidor compatível de rsync em outro DiskStation ou permitir que outro dispositivo DiskStation execute tarefas de backup remoto e armazene dados de backup em um volume do seu dispositivo DiskStation, primeiro precisará ativar o serviço de backup de rede no servidor de destino antes de criar uma tarefa de backup de rede. Para fazer isso, basta ir para Painel de controle > Privilégios e conceder o privilégio Destino do Backup de Rede a um usuário. Em seguida, você poderá criar um usuário do DSM com os privilégios adequados de aplicativo e pasta compartilhada no seu DiskStation. Quando a configuração estiver completa, outras pessoas poderão ter os privilégios do usuário para realizar backup de rede do seu DiskStation para as pastas compartilhadas de destino em seu DiskStation.

Para ativar o serviço de backup de versão única:

  1. Vá para Serviços de Backup > Destino do Backup de Rede e marque Ativar serviço de backup de rede.
  2. Clique em Aplicar.
  3. Vá para Painel de controle > Pasta compartilhada, selecione uma pasta compartilhada, clique em Editar > Permissões e conceda o privilégio Leitura/gravação a um usuário. Repita essa etapa para incluir mais pastas compartilhadas que podem ser usadas como destinos de backup.

Para ativar o serviço de backup de várias versões:

  1. Vá para Serviços de backup > Destino do Volume de Backup da Rede e marque a caixa ao lado de Habilite o destino do volume de backup da rede.
  2. Vá para Destino do Backup e selecione um destino de backup associado a um volume local.
  3. Clique em Editar > Lista de permissão > Adicionar para adicionar o usuário ou o grupo à lista.

Nota:

  • Se outras pessoas quiserem realizar o backup de rede de um cliente que não seja o DiskStation, eles poderão fazê-lo apenas com os privilégios de admin ou rsync em seu DiskStation. Se não quiser compartilhar as suas credenciais admin com outras pessoas, acesse Painel de controle > Usuário para criar um usuário chamado rsync, modifique seus privilégios e, em seguida, compartilhe suas credenciais de usuário com outras pessoas.
  • Se você executar um backup de rede a partir de um DiskStation que executa uma versão do DSM anterior à 3.0 ou um cliente que não seja o DiskStation e desejar preservar as informações de proprietário e grupo dos dados de origem, precisará adicionar a conta de backup ao grupo administrators e fazer backup dos dados na pasta compartilhada NetBackup.

Ativar o serviço de backup da rede usando a configuração rsync personalizada

É possível ativar o serviço de backup da rede usando a configuração rsync personalizada e configurar os privilégios do usuário e o destino do backup em seu DiskStation. Quando a configuração estiver concluída, outras pessoas poderão realizar o backup rsync a partir de seus dispositivos DiskStation ou clientes compatíveis com rsync para pastas compartilhadas de destino (ou “módulos de backup”) em seu DiskStation.

Para ativar o serviço de backup da rede usando uma configuração de rsync personalizada:

  1. Vá para Serviços de Backup > Destino do Backup de Rede.
  2. Marque Ativar o serviço de backup de rede.
  3. Marque Usar configuração de rsync personalizada.
  4. Clique em Aplicar.

Para configurar os privilégios de usuário e os módulos de backup em seu DiskStation:

  1. Faça login no seu dispositivo DiskStation através de Telnet ou SSH.
  2. Especifique o módulo de backup e os usuários autorizados no arquivo de configuração do rsync “/etc/rsyncd.conf”. Para obter informações detalhadas, consulte a página do manual de rsyncd.conf no seu cliente rsync ou aqui.

Nota:

  • Se a configuração rsync personalizada for usada, o DiskStation não poderá mais servir como servidor de destino para a Sincronização da pasta compartilhada, Time Backup e Backup do LUN.

Ativar o limite de velocidade

Você pode habilitar um limite de velocidade para controlar a largura de banda máxima usada pelos usuários e grupos que transferem os dados via backup de rede.

Para ativar o limite de velocidade:

  1. Vá para Serviços de Backup > Destino do Backup de Rede. Marque a caixa Ativar limite de velocidade ou Ativar limite de velocidade de acordo com uma programação se deseja implementar um limite de velocidade apenas em momentos específicos. Esta é uma configuração para todo o servidor e anulará as configurações de usuários e grupos individuais. Escolha Nenhum limite de velocidade para desativar as configurações de limite de velocidade de todos os usuários e grupos.
  2. Clique em Configurações do limite de velocidade para modificar as configurações para usuários ou grupos individuais. Para cada usuário e grupo, você pode selecionar uma das seguintes opções:
    • Aplicar configurações de grupo: As configurações do limite de velocidade serão iguais às do grupo ao qual o usuário pertence.
    • Nenhum: Nenhum limite de velocidade será aplicado quando os membros do grupo transferirem arquivos com o serviço.
    • Constante: Um limite de velocidade fixo e constante será aplicado ao usuário ou membros do grupo. Os limites de velocidade de upload e download podem ser especificados nos campos à esquerda.
    • Variável: Dois limites de velocidade diferentes podem ser especificados e aplicados ao usuário ou aos membros de grupo, de acordo com uma programação. Clique no botão Personalizar para modificar as configurações de limite de velocidade e também uma programação.
  3. Clique em Aplicar.

Nota:

Ao configurar o limite de velocidade para um usuário, a seção Resultado exibe o limite de velocidade real do usuário de acordo com as configurações individuais dele e as do grupo. Por exemplo, se o usuário "Achilles" tiver um limite de download individual de 100 KB/s, mas pertence a um grupo com um limite de 500 KB/s, a seção Resultados mostrará "0/500".

Ativar o serviço Time Backup

O serviço Time Backup permite aos usuários fazer backup de várias versões de pastas de outro DiskStation para esse DiskStation remotamente. Isso requer que os serviço de backup de rede e o serviço Time Backup estejam ativados.

Para ativar os serviços de backup de rede e Time Backup:

Vá para Serviços de Backup > Time Backup. Marque as caixas de seleção apropriadas.

Nota:

Apenas a conta admin tem permissão para criar a conexão de backup para o Time Backup remoto.

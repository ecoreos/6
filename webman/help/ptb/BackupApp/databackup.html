Backup

Backup e restauração permite que você faça backup de vários tipos de dados (configurações do sistema, pastas compartilhadas e aplicativos/pacotes) em seu DiskStation, manualmente ou por programação. Você pode armazenar os dados de backup nas pastas locais compartilhadas, servidores remotos e nuvens públicas (Amazon S3, Microsoft Azure, hicloud S3 e SFR NAS BACKUP). Com o Backup e restauração, cada tarefa de backup pode reter várias versões de backup e girá-las com políticas de reciclagem inteligente.

Backup de dados

O Backup e restauração oferece várias opções de destinos de backup e maneiras de configurar uma tarefa de backup de dados.

Para criar uma tarefa de backup de dados:

  1. Clique em + no canto inferior esquerdo e selecione Tarefa de backup de dados para abrir o Assistente de backup.
  2. Selecione o tipo desejado de destino de backup:
    • Pastas compartilhadas locais/remotas ou dispositivos de armazenamento externo
    • Servidores rsync remotos
    • Nuvens públicas (ex.: Amazon S3 e Microsoft Azure)
  3. Configure o tipo desejado de sua tarefa de backup:
    • Para criar uma tarefa totalmente nova, selecione Criar tarefa de backup.
    • Para reutilizar os dados de backup em qualquer destino, selecione Revincular à tarefa existente. Revincular ajuda você a aproveitar diretamente os dados do backup de uma tarefa diferente.
  4. Siga as instruções do assistente para concluir as configurações. Selecione Habilitar a criptografia do lado do cliente para criptografar os dados antes da transmissão para garantir a segurança deles no destino do backup.

Notas:

  • Os nomes das tarefas podem incluir de 1 a 32 caracteres Unicode, mas não podem conter os seguintes caracteres:
    ! " # $ % & ' ( ) * + , / : ; < = > ? @ [ ] \ ^ ` { } | ~
  • O comprimento máximo do caminho completo da pasta para backup é de 2048 caracteres.
  • Se mais de 32.000 arquivos ou pastas forem carregados para uma pasta compartilhada EXT3 via AFP, todas as tarefas de backup que incluem a pasta compartilhada falharão.
  • Uma pasta compartilhada criptografada pode ter seu backup feito quando montada e ele pode ser restaurado em uma pasta criptografada/descriptografada. Entretanto, ela será sempre restaurada como descriptografada /////////e não será montada automaticamente na inicialização. O sistema solicitará que você digite ou importe a chave de criptografia da pasta compartilhada criptografada para montá-la.
  • Não é suportado fazer backup de uma pasta criptografada para outra pasta criptografada.
  • Não é suportado usar pastas remotas como fonte ou destino de backup.
  • O backup de determinados aplicativos é suportado. Para saber se um pacote específico suporta o backup e conhecer o conteúdo do backup e suas limitações, consulte a ajuda de cada aplicativo.
  • O DiskStation não suporta o backup de aplicativos sem um dispositivo externo EXT3/EXT4 gravável.
  • O DiskStation só pode fazer backup de pastas compartilhadas criadas em dispositivos externos EXT3/EXT4.

Para fazer backup de configurações de ACL do Windows:

Também é possível fazer o backup das configurações de ACL do Windows dos arquivos no DiskStation, dependendo dos sistemas de arquivos do destino:

  • Todos os destinos de backup podem fazer o backup das configurações do Windows ACL independentemente do sistema de arquivos de destino

Para exportar os dados de backup para nuvens públicas ou Synology NAS remoto:

Quando fizer backup de uma grande quantidade de dados para uma nuvem pública, recomendamos que você exporte a versão inicial para um dispositivo de armazenamento externo (ex: unidade USB) e entregue o dispositivo ao fornecedor de serviços de nuvem. Lá o dispositivo poderá ser conectado e os dados, carregados diretamente na nuvem. Desse modo você pode reduzir significativamente o tempo do primeiro backup.

  1. Clique em + no canto inferior esquerdo e selecione Synology NAS remoto ou o fornecedor de serviço de nuvem desejado.
  2. Selecione Exportar para uma pasta compartilhada local (incluindo um dispositivo de armazenamento externo) e conclua as configurações. Quando solicitado, selecione Habilitar criptografia do lado do cliente para obter um backup de dados protegido.
  3. Entregue o dispositivo de armazenamento externo para o Synology NAS remoto ou fornecedor de serviços de nuvem. Importe os dados exportados do dispositivo de armazenamento externo.
  4. Depois de confirmado, volte para Backup e restauração e clique em Revincular agora.
  5. Siga as instruções do Assistente de revinculação para vincular seu DiskStation ao repositório. É necessária uma senha/chave de criptografia se os dados foram exportados criptografados.

Observação:

  • A criptografia do lado do cliente não pode ser desabilitada quando a tarefa for configurada, nem a senha pode ser alterada.
  • Um repositório de backup pode ser vinculado apenas a uma tarefa de backup.
  • Se a pasta de origem do backup for uma pasta compartilhada criptografada ou sua subpasta, a pasta de origem pode ser montada ou desmontada quando você estiver criando a tarefa de backup.
  • As pastas compartilhadas criptografadas não podem ser destino de backup.
  • Se você selecionar Habilitar criptografia no lado do cliente sua senha/chave de criptografia será exigida para restaurar dados nessa tarefa. Perder a senha/chave de criptografia causará perda permanente de dados.

Para gerenciar as tarefas de backup:

Selecione uma tarefa de backup deseja no painel esquerdo e clique nos botões abaixo para obter o gerenciamento correspondente:

  • Fazer backup agora: Faz o backup da tarefa selecionada imediatamente.
  • Explorador de backup (o ícone da lupa): Visualize todas as versões de backup disponíveis e copie/restaure/baixe os arquivos contidos.
  • Lista de versões: Visualize o tempo e o status de criação de todas as versões de backup, exclua versões específicas de backup ou bloqueie as específicas para reserva forçada.
  • Editar: Modifique a origem do backup, o destino, a programação e outras configurações.
  • Excluir: Exclua a tarefa de backup.
  • Cancelar: Interrompa a tarefa de backup em execução atualmente.

Para rotacionar as versões:

A rotação de backup é um mecanismo de gerenciamento de versão. Ela ajuda a gerenciar as versões de backup excluindo automaticamente a mais antiga, as versões desnecessárias e liberando espaço de armazenamento em potencial. Você pode habilitar essa função em Configurações de tarefas ou configurá-la quando criar uma tarefa de backup.

  1. Selecione a tarefa de backup desejada no painel esquerdo e clique em Editar ou em Configurações > Configurações de tarefa.
  2. Selecione Habilitar rotação do backup e selecione o modo de rotação desejado:
    • Das versões mais antigas: Exclua as versões de backup mais antigas quando versões disponíveis ultrapassarem o número definido.
    • Smart Recycle: Mantenha as versões de backup de acordo com essas políticas:
      • Versões horárias das últimas 24 horas: Mantém a versão mais antiga criada a cada hora.
      • Versões diárias de 1 dia passado a 1 mês: Mantém a versão mais antiga criada a cada dia.
      • Versão semanal com mais de 1 mês: Mantém a versão mais antiga criada a cada semana.
  3. Especifique o número máximo de versões. Quando o número de versões ultrapassar esse número, a versão mais antiga será rotacionada.
  4. Se você configurou um programa de backup repetido em Programa, pode visualizar como suas versões de backup serão mantidas e o ponto de recuperação mais antigo na linha do tempo.
  5. Clique em OK para salvar as configurações.

Nota:

  • As versões bloqueadas nunca serão removidas.
  • Apenas as tarefas de backup multiversão fornecem funções de rotação de versão.
  • A rotação de versão não é suportada no DSM 5.1 ou inferior.
  • A rotação será disparada logo após a configuração ser alterada ou a tarefa de backup concluída.
  • As versões bloqueadas são excluídas ao se calcular o número máximo de versões.

Notas em tarefas de backups de dados

Para cada tipo de tarefa de backup, leia as notas a seguir para obter detalhes importantes de configuração e dicas de solução de problemas.

Tarefas de backup local:

  • Se os dados do destino forem removidos enquanto a tarefa de backup está sendo executada, um backup integral será executado na próxima vez em que a tarefa for executada.
  • Novas unidades externas podem precisar de formatação antes de servirem como destino de backup.
  • Antes de criar uma tarefa de backup, certifique-se de já ter anexado as unidades externas adequadamente para o seu DiskStation. Você pode ir para Painel de controle > Dispositivos Externos para verificar o estado das unidades USB/eSATA. Unidades externas não podem ser selecionadas como destino de backup se a unidade externa estiver sendo inicializada, formatada ou estiver funcionando incorretamente.
  • Pastas compartilhadas montadas remotamente não podem ser selecionadas como origem ou destino de backup.
  • Se uma pasta compartilhada criptografada estiver desmontada, ela não poderá ser escolhida como o destino do backup.
  • Não é possível fazer o backup de miniaturas de imagem quando o sistema de arquivos do destino for NTFS/FAT.

Tarefas de backup de rsync remoto:

  • Os modos a seguir estão disponíveis ao criar uma tarefa de backup com servidores compatíveis com rsync como o destino do backup em rede: modo de daemon rsync, que é usado quando você seleciona um módulo de backup (pasta compartilhada no servidor compatível com rsync) do menu suspenso próximo a Módulo de backup; e modo de shell remoto, que é usado somente quando você digita um caminho absoluto (iniciando por "/") no Módulo de backup.
  • Para usuários de Linux, somente as contas admin, root e rsync são autorizadas a executar tarefas de backup.
  • Para acessar o servidor de destino pela Internet usando o nome do host, o nome do host deve ser um FQDN (Fully Qualified Domain Name – Nome de domínio completamente qualificado), por exemplo abc.bcd.com.
  • Ao fazer backup de um dispositivo Synology NAS para outro, você será solicitado a inserir o nome de usuário e a senha de uma conta no destino. Ao fazer backup de pastas compartilhadas em outro DiskStation, é possível fazê-lo de acordo com as configurações de privilégios de acesso das pastas. Somente pastas compartilhadas para as quais a conta de usuário especificada possui privilégios de acesso estarão visíveis. Se você não localizar a pasta compartilhada desejada no destino, verifique seus privilégios de acesso.
  • A criptografia de transferência é suportada apenas se o servidor de destino suportar backup criptografado em rede e se a senha de autenticação não estiver vazia.
  • O modo shell remoto suporta somente a criptografia de transferência.
  • Se você estiver fazendo backup de arquivos para um Synology NAS por meio de um cliente rsync, os nomes dos arquivos devem estar em codificação UTF-8.
  • Se você estiver fazendo backup de arquivos em um Synology NAS por meio de um cliente compatível com rsync, só pode executar a tarefa com contas de admin, root e rsync do DiskStation.
  • Se você estiver fazendo backup de arquivos de seu DiskStation para um servidor compatível com rsync, você precisará digitar a senha e o nome da conta de backup rsync corretos. Você também precisará escolher o módulo de backup correto com o privilégio de Gravação.

Tarefas de backup Amazon S3:

  • Ao criar tarefas de backup em rede com um servidor Amazon S3 como destino de backup em rede, várias opções estão disponíveis, como criptografia de transferência, backup para Amazon S3 RSS (Reduced Redundancy Storage) e outras. Observe que algumas opções, como criptografia de transferência, podem reduzir a velocidade das tarefas de backup.
  • Os arquivos de backup são armazenados no Bucket que você selecionou no servidor Amazon S3 e o Valor Set KEY começa com um nome de diretório de sua escolha e termina com o caminho do arquivo completo.
  • A estrutura de dados do Amazon S3 inclui um par de Bucket e KEY sem nenhuma estrutura de pastas. Não há suporte para backup em pasta vazia.
  • Se um arquivo do qual foi feito backup foi alterado, seu backup não será feito novamente no mesmo Bucket para o servidor Amazon S3.
  • Cada Bucket no Amazon S3 deve ter um nome exclusivo. Além disso, o nome deve:
    • ter entre 3 e 63 caracteres.
    • conter letras minúsculas, números, pontos (.) e traços (-).
    • comece com um número ou letra, não podendo terminar com um hífen (-) ou conter duas frases adjacentes ou traços após pontos (por exemplo: "my-.bucket.com" e "my.-bucket" são nomes inválidos).
  • O comprimento máximo de um KEY Amazon S3 é de 1024 caracteres. Para assegurar um backup bem-sucedido para o Amazon S3, o comprimento máximo do caminho é 1024 menos o nome de pasta de destino especificado durante a criação da tarefa.
  • Se você habilitar o suporte a RRS (Armazenamento de Redundância Reduzida) do Amazon S3, será feito backup no armazenamento RRS apenas dos dados modificados após a opção ser habilitada, o que significa que não será feito backup no RRS novamente dos dados já copiados para o armazenamento padrão do Amazon S3.
  • Mover os arquivos de backup do Amazon S3 para o Amazon Glacier não é suportado. Se os arquivos forem movidos do Amazon S3 para o Amazon Glacier por meio do console de Gerenciamento do AWS, as tarefas de backup relacionadas poderão falhar.
  • Não há suporte para o backup de arquivos para buckets de estilo em host virtual com SSL.
  • Ao transferir um arquivo, o arquivo será dividido em partes. O tamanho de cada parte pode ser definido pela opção Tamanho das partes para upload com diversas partes nas configurações de tarefa, com o valor padrão de 512 MB. Se o ambiente de rede não for estável, recomenda-se que essa opção seja um número menor.
  • O número máximo de partes por carregamento é 10000. O tamanho máximo de um arquivo é o Tamanho das partes para upload com diversas partes multiplicado por 10000.
  • Os nomes dos arquivos não devem conter nenhum caractere que não seja UTF-8 ou de controle.

Tarefas de backup Microsoft Azure:

  • Ao criar tarefas de backup com o servidor Microsoft Azure como o destino de backup de rede, várias opções estão disponíveis (por exemplo: criptografia de transferência). Observe que algumas opções, como criptografia de transferência, podem reduzir a velocidade das tarefas de backup.
  • Os arquivos de backup são armazenados no Contêiner que você selecionou no servidor Microsoft Azure e o nome de configuração do Blob começa com um nome de diretório de sua escolha e termina com o caminho do arquivo completo.
  • A estrutura de dados do Microsoft Azure inclui um par de Contêiner e Blob sem nenhuma estrutura de pastas. Não há suporte para backup em pasta vazia.
  • Se um arquivo do qual foi feito backup não foi alterado, seu backup não será feito novamente no mesmo Contêiner para o servidor Microsoft Azure.
  • Cada Contêiner no Microsoft Azure deve ter um nome exclusivo. Além disso, o nome deve:
    • ter entre 3 e 63 caracteres.
    • conter letras minúsculas, números e traços (-).
    • começar e terminar com um número ou uma letra e não pode conter dois traços adjacentes.
  • O comprimento máximo de um Nome Blob do Microsoft Azure é de 1024 caracteres. Para assegurar uma tarefa de backup bem-sucedida para o Microsoft Azure, o comprimento máximo do caminho é 1024 menos o nome de pasta de destino especificado durante a criação da tarefa.
  • Os nomes dos arquivos não devem conter nenhum caractere que não seja UTF-8 ou de controle.
  • A mensagem de aviso de falha de autenticação pode ser gerada por informações de autenticação incorretas, problemas de conexão de rede, ou a hora em seu DiskStation e o servidor remoto não coincidem. Para verificar a configuração de hora em seu DiskStation, vá para Painel de Controle > Opções Regionais > Hora para sincronizar a hora com o servidor NTP.

Tarefas de backup hicloud S3:

  • Ao criar tarefas de backup com o servidor hicloud S3 como o destino de backup de rede, várias opções estão disponíveis (por exemplo: criptografia de transferência). Observe que algumas opções, como criptografia de transferência, podem reduzir a velocidade das tarefas de backup.
  • Os arquivos de backup são armazenados no Bucket que você selecionou no servidor hicloud S3 e o Valor Set KEY começa com um nome de diretório de sua escolha e termina com o caminho do arquivo completo.
  • A estrutura de dados do hicloud S3 inclui um emparelhamento de Bucket e KEY sem uma estrutura de pastas. Não há suporte para backup em pasta vazia.
  • Se um arquivo do qual foi feito backup não foi alterado, seu backup não será feito novamente no mesmo Bucket para o servidor hicloud S3.
  • Cada Bucket no hicloud S3 deve ter um nome exclusivo. Além disso, o nome deve:
    • ter entre 3 e 63 caracteres.
    • conter letras minúsculas, números, pontos (.) e traços (-).
    • comece com um número ou letra, não podendo terminar com um hífen (-) ou conter duas frases adjacentes ou traços após pontos (por exemplo: "my-.bucket.com" e "my.-bucket" são nomes inválidos).
  • O comprimento máximo de um KEY hicloud S3 é de 255 caracteres. Para assegurar um backup bem-sucedido para o hicloud S3, o comprimento máximo do caminho é 255 menos o nome de pasta de destino especificado durante a criação da tarefa.
  • Ao transferir um arquivo, o arquivo será dividido em partes. O tamanho de cada parte pode ser definido pela opção Tamanho das partes para upload com diversas partes nas configurações de tarefa, com o valor padrão de 512 MB. Se o ambiente de rede não for estável, recomenda-se selecionar um número menor.
  • O Hicloud suporta apenas 1000 partes por transferência no máximo. Isso significa que o tamanho total do arquivo não pode ultrapassar 1000 multiplicado pelo tamanho da parte. Por exemplo, se o tamanho da parte de carregamento multiparte for definido em 8MB, um único arquivo acima de 8000 MB não será carregado.
  • O backup de pastas compartilhadas criptografadas não é suportado.
  • Os nomes dos arquivos não devem conter nenhum caractere que não seja UTF-8 ou de controle.

Tarefas de backup SFR NAS BACKUP:

  • Ao criar tarefas de backup com o servidor SFR NAS BACKUP como o destino de backup de rede, várias opções estão disponíveis, como criptografia de transferência e outros. Observe que algumas opções, como criptografia de transferência, podem reduzir a velocidade das tarefas de backup.
  • Os arquivos de backup são armazenados no Bucket que você selecionou no servidor SFR NAS BACKUP e o Valor Set Key começa com um nome de diretório de sua escolha e termina com o caminho do arquivo completo.
  • A estrutura de dados do SFR NAS BACKUP inclui um emparelhamento de Bucket e KEY com uma estrutura de pastas. Não há suporte para backup em pasta vazia.
  • Se um arquivo do qual foi feito backup não foi alterado, seu backup não será feito no servidor SFR NAS BACKUP no mesmo Bucket novamente.
  • Cada Bucket no servidor SFR NAS BACKUP deve ter um nome exclusivo. Além disso, o nome deve:
    • ter entre 3 e 63 caracteres.
    • conter letras minúsculas, números, pontos (.) e traços (-).
    • comece com um número ou letra, não podendo terminar com um hífen (-) ou conter duas frases adjacentes ou traços após pontos (por exemplo: "my-.bucket.com" e "my.-bucket" são nomes inválidos).
  • O comprimento máximo de uma CHAVE DE BACKUP SFR NAS é de 1024 caracteres. Para assegurar um backup bem-sucedido para o SFR NAS BACKUP, o comprimento máximo do caminho é 1024 menos o nome de pasta de destino especificado durante a criação da tarefa.
  • Ao transferir um arquivo, o arquivo será dividido em partes. O tamanho de cada parte pode ser definido pela opção Tamanho das partes para upload com diversas partes nas configurações de tarefa, com o valor padrão de 64MB. Se o ambiente de rede não for estável, recomenda-se selecionar um número menor.
  • Os nomes dos arquivos não devem conter nenhum caractere que não seja UTF-8 ou de controle.

Outras tarefas de backup compatíveis com S3:

  • Para usar outras nuvens compatíveis com S3, digite o endereço IP/domínio da nuvem compatível com S3 desejada no campo Servidor S3 quando solicitado.
  • Se houver algum problema de uso, consulte a documentação relevante ou entre em contato com o fornecedor de serviço na nuvem compatível com S3.

Backup do LUN*

Você pode usar Backup de LUN local para fazer o backup de iSCSI LUNs em seu DiskStation em pastas compartilhadas locais. Você também pode usar Backup de LUN de Rede (servidor Synology) para fazer backup em outro DiskStation.

Para fazer backup do iSCSI LUN:

  1. Clique em + no canto inferior esquerdo e selecione Tarefa de backup de LUN.
  2. Quando solicitado, selecione um tipo de destino, selecione uma das seguintes opções:
    • Backup de LUN Local: Selecione essa opção para fazer backup do LUN do DiskStation para uma pasta compartilhada local ou uma unidade de armazenamento externa conectada à porta USB ou eSATA de seu DiskStation.
    • Backup de LUN de Rede (servidor Synology): Selecione essa opção para fazer backup do LUN de seu DiskStation para outro DiskStation.
  3. Quando solicitado, especifique o destino do backup e o LUN de origem e defina a programação de backup.
  4. Siga o assistente para concluir a configuração.

Depois da criação da tarefa de backup, o DiskStation executará a tarefa de backup de acordo com a programação e armazenará o backup do LUN no destino. O backup do LUN é usado apenas para fins de backup. Para acessar o backup do LUN, consulte Para restaurar o backup do LUN para obter mais informações.

Nota:

  • Os LUNs normais de arquivo e de nível de bloco serão incluídos no backup como imagens de arquivo no destino. Se restaurados, os LUNs de Nível de Bloco serão alterados para LUNs de Arquivo.
  • Cada tarefa de backup pode incluir apenas um LUN como origem de backup.
  • O backup dos LUNs não pode ser feito em um disco externo com o formato FAT, FAT32, HFS, HFS Plus ou NTFS.
  • Antes de executar o Backup do LUN da rede, faça login no DiskStation de destino para certificar-se de que o Serviço de backup de rede esteja habilitado em Painel de controle > Serviços de Backup.
  • Se o DiskStation se conecta a outra rede através de um roteador, certifique-se de ter especificado uma regra de encaminhamento de porta para o serviço iSCSI (porta TCP 3260) no roteador antes de realizar o backup do LUN da rede.

Sobre o backup completo e o backup incremental:

O DiskStation executará um backup completo do LUN selecionado pela primeira vez e posteriormente realizará backups incrementais. Entretanto, se o nome da pasta compartilhada ou o nome do diretório do destino do backup for alterado, o DiskStation realizará um backup completo da tarefa novamente.

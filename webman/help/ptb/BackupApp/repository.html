Destino de Backup

Na guia Destino de Backup em Backup e replicação é possível definir um destino de backup local, em rede ou na nuvem antes de criar a tarefa de backup.

Para criar um destino de backup local:

É necessário indicar um destino de backup local para fazer backup de dados do DiskStation para uma pasta compartilhada local, volume ou unidade USB/eSATA externa. O destino de backup deve ser diferente da origem, e a capacidade do destino deve ser maior do que o tamanho total dos dados a serem obtidos em backup.

  1. Clique em Criar, selecione Destino de Backup Local e clique em Próximo.
  2. Nomeie o destino de backup, selecione o destino onde os dados de backup serão armazenados e selecione se deseja manter apenas a cópia de backup mais atualizada ou se deseja manter todas as versões de backup.

Notas:

  • Se os dados do destino forem removidos enquanto a tarefa de backup está sendo executada, um backup integral será executado na próxima vez em que a tarefa for executada.
  • Novas unidades externas podem precisar de formatação antes de servirem como destino de backup.
  • Antes de criar uma tarefa de backup, certifique-se de já ter anexado as unidades externas adequadamente para o seu DiskStation. Você pode ir para Painel de controle > Dispositivos Externos para verificar o estado das unidades USB/eSATA. Unidades externas não podem ser selecionadas como destino de backup se a unidade externa estiver sendo inicializada, formatada ou estiver funcionando incorretamente.
  • Compartilhamentos montados remotamente não podem ser selecionados como origem ou destino de backup.
  • Se uma pasta compartilhada criptografada estiver desmontada, ela não poderá ser escolhida como o destino do backup.
  • Não é possível fazer o backup de miniaturas de imagem quando o sistema de arquivos do destino for NTFS/FAT.

Para criar um destino de backup em rede:

É necessário indicar um destino de backup em rede para fazer backup de dados do DiskStation para outro DiskStation ou para um servidor compatível de rsync na rede. Várias opções são disponibilizadas ao criar uma tarefa de backup com um destino de backup em rede, como criptografia de transferência, compressão, backup em nível de bloqueio, entre outras. Observe que algumas opções, como criptografia de transferência, podem reduzir a velocidade da tarefa de backup.

  1. Antes de criar um destino de backup de rede, primeiro faça login no servidor de destino, vá para Backup e replicação > Serviços de backup e habilite o serviço de backup de rede e/ou o destino do volume de backup da rede.
  2. Novamente no DiskStation, vá para Backup e replicação > Destino de Backup. Clique em Criar selecione Destino de backup de rede e siga o assistente para concluir.
  3. Com o Destino de Backup de Rede (servidor DiskStation) é possível armazenar dados de backup em uma pasta compartilhada para manter a cópia de segurança mais recente ou armazenar dados de backup em um volume para manter várias versões das cópias de segurança. Com o Destino de Backup de Rede (servidor compatível com rsync) é possível manter apenas a cópia de backup mais recente.

Notas:

  • Os modos a seguir estão disponíveis ao criar uma tarefa de backup com servidores compatíveis com rsync como o destino do backup em rede: modo de daemon rsync, que é usado quando você seleciona um módulo de backup (pasta compartilhada no servidor compatível com rsync) do menu suspenso próximo a Módulo de backup; e modo de shell remoto, que é usado somente quando você digita um caminho absoluto (iniciando por "/") no Módulo de backup.
  • Para usuários de Linux, somente as contas admin, root e rsync são autorizadas a executar tarefas de backup.
  • Para acessar o servidor de destino pela Internet usando o nome do host, o nome do host deve ser um FQDN (Fully Qualified Domain Name – Nome de domínio completamente qualificado), por exemplo abc.bcd.com.
  • Ao fazer backup de um dispositivo DiskStation para outro, você será solicitado a inserir o nome de usuário e a senha de uma conta no destino. Especialmente se você desejar fazer o backup para um volume em outro DiskStation, será preciso especificar uma conta pertencente ao grupo administrators. Ao fazer backup de pastas compartilhadas em outro DiskStation, é possível fazê-lo de acordo com as configurações de privilégios de acesso das pastas. Somente pastas compartilhadas para as quais a conta de usuário especificada possui privilégios de acesso estarão visíveis. Se você não localizar a pasta compartilhada desejada no destino, verifique seus privilégios de acesso.
  • A criptografia de backup é suportada apenas se o servidor de destino suportar backup criptografado em rede e se a senha de autenticação não estiver vazia.
  • O modo shell remoto suporta somente o backup criptografado.
  • Ao fazer backup de um servidor compatível com rsync, você não será capaz de alterar o modo de criptografia rsync depois que as tarefas de backup rsync forem criadas. Para alterar a configuração, crie uma nova tarefa rsync.
  • Se você estiver fazendo backup de arquivos para um DiskStation por meio de um cliente rsync, os nomes dos arquivos devem estar em codificação UTF-8.
  • Se você estiver fazendo backup de arquivos em um DiskStation por meio de um cliente compatível com rsync, só pode executar a tarefa com contas de admin, root e rsync do DiskStation.
  • Se você estiver fazendo backup de arquivos de um DiskStation para um servidor compatível com rsync, você precisará digitar a senha e o nome da conta de backup rsync corretos. Você também precisará escolher o módulo de backup correto com o privilégio de Gravação.

Para criar um destino de backup Amazon S3:

É necessário informar um destino de backup Amazon S3 para fazer backup dos dados do DiskStation para um servidor Amazon S3.

  1. Clique em Criar, selecione Destino de Backup de Nuvem Pública, Destino de Backup Amazon S3 e clique em Avançar.
  2. Dê um nome a esse destino de backup, digite as credenciais de login e preencha todos os campos necessários para configurar as tarefas de backup.

Notas:

  • Ao criar tarefas de backup em rede com um servidor Amazon S3 como destino de backup em rede, várias opções estão disponíveis, como criptografia de transferência, backup para Amazon S3 RRS (Reduced Redundancy Storage) e outras. Observe que algumas opções, como criptografia de transferência, podem reduzir a velocidade das tarefas de backup.
  • Os arquivos de backup são armazenados no Bucket que você selecionou no servidor Amazon S3 e o Valor Set KEY começa com um nome de diretório de sua escolha e termina com o caminho do arquivo completo.
  • A estrutura de dados do Amazon S3 inclui um par de Bucket e KEY sem nenhuma estrutura de pastas. Não há suporte para backup em pasta vazia.
  • Se um arquivo do qual foi feito backup foi alterado, seu backup não será feito novamente no mesmo Bucket para o servidor Amazon S3.
  • Cada Bucket no Amazon S3 deve ter um nome exclusivo. Além disso, o nome deve:
    • ter entre 3 e 63 caracteres.
    • conter letras minúsculas, números, pontos (.) e traços (-).
    • comece com um número ou letra, não podendo terminar com um hífen (-) ou conter duas frases adjacentes ou traços após pontos (por exemplo: "my-.bucket.com" e "my.-bucket" são nomes inválidos).
  • O comprimento máximo de um KEY Amazon S3 é de 1024 caracteres. Para assegurar um backup bem-sucedido para o Amazon S3, o comprimento máximo do caminho é 1024 menos o nome de pasta de destino especificado durante a criação da tarefa.
  • Se você habilitar o suporte a RRS (Armazenamento de Redundância Reduzida) do Amazon S3, será feito backup no armazenamento RRS apenas dos dados modificados após a opção ser habilitada, o que significa que não será feito backup no RRS novamente dos dados já copiados para o armazenamento padrão do Amazon S3.
  • Mover os arquivos de backup do Amazon S3 para o Amazon Glacier não é suportado. Se os arquivos forem movidos do Amazon S3 para o Amazon Glacier por meio do console de Gerenciamento do AWS, as tarefas de backup relacionadas poderão falhar.
  • Não há suporte para o backup de arquivos para buckets de estilo em host virtual com SSL.
  • Ao transferir um arquivo, o arquivo será dividido em partes. O tamanho de cada parte pode ser definido pela opção Tamanho das partes para upload com diversas partes nas configurações de tarefa, com o valor padrão de 512 MB. Se o ambiente de rede não for estável, recomenda-se que essa opção seja um número menor.
  • O número máximo de partes por carregamento é 10000. O tamanho máximo de um arquivo é o Tamanho das partes para upload com diversas partes multiplicado por 10000.
  • Os nomes dos arquivos não devem conter nenhum caractere que não seja UTF-8 ou de controle.

Para criar um destino de backup Microsoft Azure:

É necessário informar um destino de backup Microsoft Azure para fazer backup dos dados do DiskStation para um servidor blob Microsoft Azure.

  1. Clique em Criar, selecione Destino de Backup de Nuvem Pública, Destino de Backup Microsoft Azure e clique em Avançar.
  2. Dê um nome ao destino de backup, digite as credenciais de login e preencha todos os campos necessários para configurar a tarefa de backup.

Notas:

  • Ao criar tarefas de backup com o servidor Microsoft Azure como o destino de backup de rede, várias opções estão disponíveis (por exemplo: criptografia de transferência). Observe que algumas opções, como criptografia de transferência, podem reduzir a velocidade das tarefas de backup.
  • Os arquivos de backup são armazenados no Contêiner que você selecionou no servidor Microsoft Azure e o nome de configuração do Blob começa com um nome de diretório de sua escolha e termina com o caminho do arquivo completo.
  • A estrutura de dados do Microsoft Azure inclui um par de Contêiner e Blob sem nenhuma estrutura de pastas. Não há suporte para backup em pasta vazia.
  • Se um arquivo do qual foi feito backup não foi alterado, seu backup não será feito novamente no mesmo Contêiner para o servidor Microsoft Azure.
  • Cada Contêiner no Microsoft Azure deve ter um nome exclusivo. Além disso, o nome deve:
    • ter entre 3 e 63 caracteres.
    • conter letras minúsculas, números e traços (-).
    • começar e terminar com um número ou uma letra e não pode conter dois traços adjacentes.
  • O comprimento máximo de um Nome Blob do Microsoft Azure é de 1024 caracteres. Para assegurar uma tarefa de backup bem-sucedida para o Microsoft Azure, o comprimento máximo do caminho é 1024 menos o nome de pasta de destino especificado durante a criação da tarefa.
  • Os nomes dos arquivos não devem conter nenhum caractere que não seja UTF-8 ou de controle.
  • A mensagem de aviso de falha de autenticação pode ser gerada por informações de autenticação incorretas, problemas de conexão de rede, ou a hora em seu DiskStation e o servidor remoto não coincidem. Para verificar a configuração de hora em seu DiskStation, vá para Painel de Controle > Opções Regionais > Hora para sincronizar a hora com o servidor NTP.

Por exemplo, se o tamanho da parte de carregamento multiparte for definido em 8MB, um único arquivo acima de 8000 MB não será carregado.
  • O backup de pastas compartilhadas criptografadas não é suportado.
  • Os nomes dos arquivos não devem conter nenhum caractere que não seja UTF-8 ou de controle.

  • Os arquivos de backup são armazenados no Bucket que você selecionou no servidor SFR NAS BACKUP e o Valor Set Key começa com um nome de diretório de sua escolha e termina com o caminho do arquivo completo.
  • A estrutura de dados do SFR NAS BACKUP inclui um emparelhamento de Bucket e KEY com uma estrutura de pastas. Não há suporte para backup em pasta vazia.
  • Se um arquivo do qual foi feito backup não foi alterado, seu backup não será feito no servidor SFR NAS BACKUP no mesmo Bucket novamente.
  • Cada Bucket no servidor SFR NAS BACKUP deve ter um nome exclusivo. Além disso, o nome deve:
    • ter entre 3 e 63 caracteres.
    • conter letras minúsculas, números, pontos (.) e traços (-).

O tamanho de cada parte pode ser definido pela opção Tamanho das partes para upload com diversas partes nas configurações de tarefa, com o valor padrão de 64MB. Se o ambiente de rede não for estável, recomenda-se selecionar um número menor.
  • Os nomes dos arquivos não devem conter nenhum caractere que não seja UTF-8 ou de controle.

Para gerenciar o destino de backup:

Selecione um destino de backup existente e clique em um dos botões abaixo:

  • Editar: Modicar as configurações do destino de backup selecionado.
  • Excluir: Excluir o destino de backup selecionado.

Backup para Pastas Compartilhadas

Este tipo de destino de backup irá manter apenas a cópia de backup atualizada. Por padrão, após o backup inicial ser concluído, os arquivos que foram removidos da origem são removidos do destino na próxima vez em que uma tarefa for executada. Entretanto, se você preferir manter os arquivos no destino, mesmo depois de terem sido excluídos da origem, habilite a opção Reservar os arquivos com backup no destino quando criar as tarefas de backup.

Backup para Volume

Este tipo de destino de backup deve ser um volume local ou em rede em um dispositivo DiskStation (excluindo dispositivos de armazenamento externos). Ele irá arquivar várias versões das cópias de segurança, para que você possa restaurar facilmente as pastas a partir de determinado momento.

Para excluir os dados armazenados de uma tarefa de backup:

  1. Vá para a guia Destino do Backup e expanda o destino do backup que contém a tarefa que você deseja excluir.
  2. Clique no ícone Excluir na Tarefa de Backup selecionada e em Excluir na caixa de diálogo.

Para excluir os dados armazenados de uma versão de backup:

  1. Vá para a guia Destino do Backup e expanda o destino do backup que contém a versão da tarefa que você deseja excluir.
  2. Clique no ícone Informações de versão do backup em sua Tarefa de Backup selecionada.
  3. Selecione a versão de backup que deseja excluir e clique em Excluir.

Para bloquear uma versão de backup para impedi-la de ser removida durante a rotação do backup:

  1. Vá para a guia Destino do Backup e expanda o destino do backup que contém a versão da tarefa que você deseja bloquear.
  2. Clique no ícone Informações de versão do backup em sua Tarefa de Backup selecionada.
  3. Selecione a versão de backup que deseja bloquear e clique em bloquear.

Para gerenciar o privilégio de um destino do backup:

  1. Escolha um destino do backup associado a um volume local, clique em Editar.
  2. Clique em Lista de permissão e em Adicionar para adicionar o usuário ou o grupo à lista.

Para configurar um destino de backup com um proprietário:

  1. Selecione um destino de backup associado a um volume local e clique no ícone Propriedade na Tarefa de Backup selecionada.
  2. Em Privilégios de backup, selecione um Proprietário no menu suspenso.

Para permitir que um usuário ou grupo em DiskStation leia e restaure os dados armazenados de uma tarefa de backup:

  1. Selecione um destino de backup associado a um volume local e clique no ícone Propriedade na Tarefa de Backup selecionada.
  2. Clique em Lista de permissão e em Adicionar para adicionar o usuário ou o grupo à lista.

Notas:

  • O proprietário padrão de uma tarefa de backup é o usuário que criou a tarefa.
  • Somente o proprietário da tarefa de backup pode executar a tarefa. No entanto, as tarefas de backup criadas antes do DSM 5.1 não têm essa limitação, por isso qualquer usuário que pertença ao grupo administrators é capaz de realizar essa tarefa de backup.
  • O proprietário da tarefa e os usuários pertencentes ao grupo administrators podem excluir os dados armazenados de uma tarefa de backup.
  • O proprietário da tarefa, os usuários pertencentes ao grupo administrators e os usuários na lista de permissão podem restaurar os dados armazenados de uma tarefa de backup.
  • Somente usuários que pertençam ao grupo administrators podem configurar um destino de backup local com privilégios de acesso, e uma tarefa de backup em um destino de backup local com um proprietário e uma lista de permissão.
  • Os dados armazenados de uma tarefa de backup consomem a cota de armazenamento do proprietário da tarefa.
  • Se um volume não tiver um destino do backup, apenas os usuários que pertençam ao grupo administrators podem criar o primeiro destino do backup no volume.
  • Apenas os usuários pertencentes ao grupo administrators podem excluir os dados armazenados de uma tarefa de backup inteira.
  • Cada backup contém a configuração do sistema e os metadados do arquivo.
  • O backup de pastas compartilhadas criptografadas não é suportado.
  • O bloqueio da versão não é suportado no DSM 5.1 ou inferior.

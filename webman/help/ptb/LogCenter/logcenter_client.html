Envio de log

Se você tiver o syslog server, pode seguir as instruções abaixo para configurar o DiskStation como cliente syslog e transmitir os logs locais do sistema de seu DiskStation para um syslog server, para monitoramento e gerenciamento centralizados.

Para enviar logs para um syslog server:

  1. Vá para Log Center > Envio de log.
  2. Marque a caixa Enviar logs para um syslog server.
  3. Digite o nome de host ou endereço IP do syslog server em Servidor.
  4. Especifique um número de porta para receber mensagens do syslog em Porta.
  5. Selecione UDP ou TCP em Protocolo de transferência. Ele deve corresponder ao protocolo usado pelo syslog server.
  6. Selecione BSD (RFC 3164) ou IETF (RFC 5424) em Formato do log para enviar logs usando o formato selecionado.

    7. Nota:

    • Por padrão, a porta é 514. As regras de envio de porta adequadas devem ser definidas se você quiser transferir um número de porta diferente para o syslog server.
    • O UDP (Protocolo de Datagrama de Usuário) é um protocolo de conexão baseado em mensagens mais simples, sendo um protocolo mais rápido, mas que não tem correções de segurança e erro. O TCP (Protocolo de Controle de Transmissão) oferece uma entrega garantida, usando o controle de fluxo para determinar se todos os pacotes foram enviados com êxito antes que os novos pacotes sejam processados.
    • A conexão segura (SSL) está disponível apenas durante o uso do TCP.
    • Se você escolher TCP e desejar proteger a conexão syslog com SSL, pode marcar a opção Habilitar conexão segura (SSL). Em seguida, clique em Importar certificado e selecione o certificado para transferir logs com segurança para o syslog server.
  7. Clique em Enviar log de teste para confirmar que o syslog server pode receber o log com êxito. Se o syslog server não receber o log de teste, verifique se suas configurações estão corretas.
  8. Clique em Aplicar para salvar as configurações.

Filtrar logs

Na guia Filtro, você pode especificar quais tipos de logs são enviados para o syslog server, de acordo com a categoria (por exemplo, sistema, conexão, backup) ou gravidade (por exemplo, aviso, erro, informações).

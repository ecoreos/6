FTP / FTPS

I Kontrollpanel > Filtjenester > FTP kan du aktivere FTP-tjenester for å la klienter få tilgang til mapper og data på systemet. Før du aktiverer FTP, må du forsikre deg om at følgende TCP-porter på ruteren din blir videresendt til DiskStation-enheten: 21 (standard kontrolltilkobling), 20 (datatilkobling for aktiv modus) og 55536-55567 (datatilkobling for passiv modus).

Slik aktiverer du FTP-tjenesten:

  1. Aktiver én av følgende:
    • Aktiver FTP-tjeneste (ingen kryptering): Dette er standard nettverksprotokoll til å overføre filer. FTP tilbyr ikke noen kryptering for å beskytte informasjon under overføringsøkter, som passord, brukernavn eller filer. Men overføringshastigheten er raskere og det krever mindre systemressurser.
    • Aktiver FTP SSL/TLS-krypteringstjeneste (FTPS): En utvidelse av standard FTP med ytterligere støtte for krypteringsprotokollene Transport Layer Security (TLS) og Secure Sockets Layer (SSL), som beskytter informasjon under overføringsøkter, som passord, brukernavn eller filer. Men overføringshastigheten blir langsommere og det blir brukt mer systemressurser på grunn av kryptering.
  2. Klikk på Bruk.

Tidsavbrudd

FTP-server vil koble fra inaktive brukere etter angitt tidsperiode.

Portnummerinnstilling for FTP-tjeneste

Denne seksjonen lar deg spesifisere porten som brukes for FTP-kontrollkanalen.

Merk:

Standard portnummer for FTP-kontrollkanal er 21. Det kan være fra 1 til 65535, med unntak for følgende numre: 20, 22, 23, 25, 80, 110, 137, 138, 139, 143, 199, 443, 445, 515, 543, 548, 587, 873, 993, 995, 3306, 3689, 5000, 5001, 5005, 5006, 5335, 5432, 6881, 8080, 7000, 7001, 8081, 9997, 9998, 9999, 50001, 50002 og standard eMule-porter: 4662 (TCP), 4672(UDP).

Portområde for passiv FTP

Denne seksjonen lar deg spesifisere portområdet som brukes for passive FTP-tilkoblinger. Du kan bruke standard portområde eller spesifisere et egendefinert område. Portområdet kan være fra 1025 til 65535 og kan inneholde opp til 128 porter.

Rapportere ekstern IP i PASV-modus

Ved å aktivere dette alternativet, vil serveren rapportere sin eksterne IP-adresse til FTP-klienter. Den rapporterte IP-adressen vil registreres i FTP-klientens tilkoblingslogg. Når det er aktivert, kan du velge den automatisk registrerte eksterne IP-adressen eller angi den eksterne IP-adressen til DiskStation manuelt.

Dette alternativet fungerer kun når DiskStation er bak en NAT, og når FTP-klientene tilhører et annet delnett enn DiskStation. I de fleste tilfeller er dette alternativet overflødig, men hvis FTP-klienter ikke kan koble seg til DiskStation, kan du aktivere dette alternativet og prøve på nytt.

Aktivere FXP

FXP står for File eXchange Protocol. Ved å aktivere dette alternativet vil FTP-tjeneste støtte funksjonen for filoverføring fra server til server.

Aktivere FIPS-kryptografisk modul

FIPS støttes på FTPS-tjeneste med OpenSSL FIPS 140-2-godkjent kryptografimodul (Sertifikat #1747). Ved å aktivere dette alternativet, vil ytterligere sikre kryptografialgoritmer bli brukt av FTPS-tjenesten for å yte enda sikrere økter.

Støtte ASCII-overføringsmodus

Aktiver dette alternativet for å støtte ASCII-modus, slik at DiskStation kan overføre ASCII-formaterte vanlige tekstfiler via FTP uten problemer.

UTF-8-koding

Dette alternativet kan brukes til å hjelpe FTP-klienter med å vise og behandle filer med flerspråklige filnavn. For å møte behovene til ulike FTP-klienter, tilbys følgende tre alternativer:

  • Deaktiver: UTF-8-koding vil ikke bli brukt. Dette alternativet er for klienter som bruker Windows Filutforsker som FTP-klient for å laste opp/ned filer med flerspråklige navn. Det blir ikke anbefalt til normal bruk.
  • Auto: La DSM automatisk avgjøre hvordan UTF-8-koding skal håndteres. Dette alternativet er anbefalt til normal bruk.
  • Tving: Tving bruk av UTF-8-koding. Hvis du har aktivert UTF-8-koding på din FTP-klient og den likevel ikke viser korrekte filnavn og vil ikke behandle filer med flerspråklige filnavn, aktiverer du dette alternativet.

Tilkoblingsbegrensning

Denne seksjonen inkluderer følgende alternativer.

  • Begrens tilkoblinger per IP: Når denne funksjonen er aktivert, vil serveren begrense maksimalt antall tilkoblinger per IP. Aktiver denne funksjonen og FTP-båndbredderestriksjon for å få effektiv administrasjon av båndbredde.
  • Aktiver restriksjon på FTP-båndbredden: Aktiver denne funksjonen hvis du vil begrense båndbredden til en enkelt-FTP-tilkobling.

Merk:

  • Gjestekontoen kan ikke logge inn på serveren via FTP.
  • Kodesideinnstillingen på FTP-klientdatamaskinen må være de samme som for DiskStation for å oppnå korrekt tilgang til dataene.
  • På noen bestemte modeller støtter FTP-tjenesten SSL eller TLS tilkoblinger.
  • Brannmurinnstillingene for din FTP-klient og to FTP-servere må konfigureres på riktig måte, ellers vil FXP-funksjonen sannsynligvis ikke fungere på riktig måte.
  • Enkelte FTP-klienter, som de som bruker Internet Explorer 8/9, krever innlogging med brukernavn og passord. For eksempel: ftp://brukernavn:passord@din.ftp.com.

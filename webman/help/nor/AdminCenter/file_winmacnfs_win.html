Windows-filtjeneste

Windows-filtjeneste gir Windows-, Mac- og Linux-klienter med støtte for SMB/CIFS tilgang til data som er lagret på DiskStation. Hvis du vil bla gjennom filer på DiskStation via Windows Explorer eller montere delte mapper som nettverksstasjoner, aktiverer du dette alternativet i Kontrollpanel > Filtjenester > Win/Mac/NFS.

Aktiver Windows-filtjeneste:

  1. Merk av for Aktiver Windows-filtjeneste.
  2. Klikk på Bruk.

Merk:

Når du har aktivert Windows-filtjeneste, kan du montere indekserte mapper på Mac-datamaskinen din via SMB og bruke Spotlight i Finder til å søke etter filinnholdet i de monterte mappene.

Arbeidsgruppe

Dette alternativet lar deg tilordne systemet som et medlem av en bestemt arbeidsgruppe for nettverksadministrative formål.

Merk:

  • Navnet på en arbeidsgruppe kan inneholde fra 1 til 15 tegn. Navnet kan ikke inneholde følgende tegn: [ ] ; : " < > * + = \ / | ? ,
  • Hvis du bruker en Mac som kjører OS X 10.9, og din DiskStation er tilknyttet et Windows-domene, kan du logge inn ved å legge til localhost\ foran ditt lokale DiskStation-brukernavn for å få tilgang til filer som lagres på DiskStation via Windows-filtjeneste.

Nekt tilgang til Tidligere versjoner

I Windows Utforsker kan brukere aksessere snapshots av mapper eller filer ved å klikke på fanen Tidligere versjoner i en delt mappe, slik at de kan gjenopprette eller kopiere data uten hjelp fra administrator. Hvis du velger Nekt tilgang til Tidligere versjoner, vil ikke siden "Tidligere versjoner" vise noe informasjon.

Merk:

  • Brukerekan kun aksessere en tidligere versjon hvis brukerkontoen allerede er opprettet når snapshotet som korresponderer med den tidligere versjonen blir tatt og de har lesetillatelse for den delte mappen som det korresponderende snapshotet tilhører.

Aktivere overføringslogger

Hvis du merker av for dette alternativet, aktiveres logging av filtilgangsoperasjoner, inkludert oppretting, sletting, lesing, skriving, flytting og endring av navn på filer eller mapper gjennom CIFS/SMB-protokollen, også kjent som Windows-filoverføringsprotokoll.

Avanserte innstillinger

Klikk på knappen Avanserte innstillinger for å se alternativene nedenfor.

  • WINS-server: Microsoft Windows Internet Name Service (WINS) er en NetBIOS navn- til-IP-adresse-tilordningstjeneste. Windows-brukere kan lettere lokalisere systemet på TCP/IP-nettverk hvis systemet er konfigurert til å registrere hos en WINS-server.
  • Maks. protokoll for Windows-filtjeneste: Avhengig av nettverkskonfigurasjonen din, kan du angi det høyeste protokollnivået som støttes av din DiskStation.
    • Aktiver SMB 2: SMB (Server Message Block) 2 har vært støttet siden Windows Vista og er den forbedrede versjonen av SMB. SMB 2 tilfører evnen til å samle flere SMB-handlinger i én enkelt forespørsel for å redusere antallet nettverkspakker og forbedre ytelsen.
    • Aktiver SMB 2 med Large MTU: Maximum transmission unit (MTU, maksimum overføringsenhet) refererer til den største dataenheten som kan sendes over en kommunikasjonsprotokoll. For å dra nytte av raskere nettverksgrensesnitt, som 1- eller 10-gigabit Ethernet, øker Large MTU maksimum overføringsenhet (MTU) til 1 megabyte (MB). Aktivering av Large MTU forbedrer hastigheten og effektiviteten til store filoverføringer og reduserer antallet pakker som må behandles.
    • Aktiver SMB 3: SMB 3 har vært støttet siden Windows 8 og Windows Server 2012. Dette er den forbedrede versjonen av SMB 2. SMB 3 støtter AES-basert overføring av krypterte filer, og forbedrer dermed sikkerheten ved peer-to-peer-overføringer.

    Merk:

    • SMB 2 støttes av Windows Vista, Windows Server 2008 R2, Windows 7 og nyere.
    • Stor MTU støttes av Windows Server 2008 R2, Windows 7 og nyere.
    • For å gjøre stor MTU effektiv med din Windows-datamaskin, vil du måtte endre registerinnstillingene på datamaskinen din. For mer informasjon, se Microsofts tekniske kundestøtte (som denne artikkelen).
  • Transportkrypteringsmodus: Når SMB 3 er aktivert, vil Windows-filtjeneste legge til transportkryptering for å forbedre sikkerheten ved filoverføring.
    • Deaktiver: Ingen transportkryptering vil bli tatt i bruk.
    • Auto: Transportkryptering vil kun bli tatt i bruk for klienter som støtter denne funksjonen.
    • Tving: Transportkryptering vil alltid bli tatt i bruk. Dette vil føre til at klienter som ikke støtter transportkryptering ikke vil kunne bruke Windows-filtjeneste.

    • Merk:

    • Transportkryptering støttes på Windows 8, Windows Server 2012 og nyere.
    • Anonym pålogging for Windows-filtjeneste støttes ikke når transportkryptering er aktivert.
    • Når transportkryptering er aktivert, må du la Deaktiver Opportunistic Locking være uavmerket for å unngå at programmet blir tidsavbrutt.
  • Deaktiver Opportunistic Locking: Dersom du merker av dette alternativer, vil du deaktivere "Opportunistisk låsing" av CIFS, noe som medfører økt ytelse når databaseprogrammer søker tilgang til databasefiler i den delte mappen på DiskStation-enheten via nettverket.
  • Aktiver Local Master Browser: Når flere Windows-datamaskiner eksisterer på det samme delnettet, vil de forhandle og velge én datamaskin som "local master browser". Valgt Local Master Browser bevarer lister over de andre datamaskinene på delnettet og deres delte ressurser, og deler disse listene med de andre datamaskinene. Dette alternativet lar DiskStation ta rollen som Local Master Browser.

    • Merk:

    Aktivering av denne funksjonen deaktiverer dvalemodus for harddisker og aktiverer gjestekontoen uten passord.

  • Aktiver DirSort VFS-modul: Sorterer mappenavn og lister dem opp alfabetisk. Aktivering av dette alternativet påvirker systemytelsen.
  • Veto filer: Skjuler filer i henhold til spesifiserte kriterier. Kriterier kan inkludere jokertegn (*), og flere oppføringer må separeres med skråstrek (/). For eksempel: /abc*/*.txt/*.conf/. Aktivering av dette alternativet påvirker systemytelsen.
  • Tillat symbolske koblinger innad i delte mapper: Aktivering av dette alternativet lar symbolske koblinger peke mot filer eller mapper innad i den samme delte mappen. Symbolske koblinger kan opprettes ved hjelp av DSM-terminal.
  • Tillat symbolske koblinger mellom delte mapper: Aktivering av dette alternativet lar symbolske koblinger peke mot filer og mapper i andre delte mapper. Symbolske koblinger kan opprettes ved hjelp av DSM-terminal. Vær oppmerksom på at aktivering av dette alternativet kan la bukere uten tilgangsrettigheter vise og få tilgang til delte mapper som er mål for symbolske koblinger.
  • Aktiver MSDFS VFS-modul: Lar symbolske koblinger peke mot filer og mapper på andre SMB-servere. Med eldre versjoner av Windows, som Windows XP, må klienten og DiskStation tilknyttes et Windows-domene før du aktiverer dette alternativet. For å finne ut mer om oppretting av MSDFS-koblinger, se her. Vær oppmerksom på at bufferen til klientenheten bør tømmes etter deaktivering av dette alternativet, ellers kan normal ytelse bli påvirket.
  • Deaktiver flere tilkoblinger fra den samme IP-adressen: Når dette alternativet er aktivert, vil DiskStation kun tillate én tilkobling per IP-adresse fra klienter som bruker Windows-filtjeneste. Når flere klienter med den samme IP-adressen bruker Windows-filtjeneste, vil DiskStation beholde den nyligste tilkoblingen og avbryte alle andre tilkoblinger. I et NAT-miljø kan brukeren bruke en enkelt IP-adresse, og da vil aktivering av dette alternativet kunne påvirke den normale ytelsen.
  • Samle feilsøkingslogger: Når dette alternativet er aktivert, vil detaljerte logger bli lagret for feilsøkingsformål. Aktivering av dette alternativet påvirker systemytelsen.
  • Bruk standard UNIX-tillatelser: Aktiver dette alternativet for å ta i bruk standard UNIX-tillatelser når du laster opp eller oppretter filer og mapper. UNIX-tillatelsen vil være 744 for filer og 755 for mapper. Når dette alternativet er deaktivert, er UNIX-tillatelsen 777 for filer og mapper.

    • Merk:

    • Aktivering av dette alternativet kan forårsake problemer med inkonsekvente tillatelser mellom forskjellige protokoller. For å unngå inkonsekvens, foreslår vi at dette alternativet er deaktivert.

    Merk:

    • For Windows ACL-aktiverte delte mapper (alle delinger utenom "photo" og delinger på eksterne disker) kjører du chmod-kommandoen på Linux- eller FTP-klienten din for å endre tillatelsestypen for mapper og filer fra Windows ACL til UNIX.
    • Aktivering av dette alternativet kan forårsake problemer med inkonsekvente tillatelser mellom forskjellige protokoller. For å unngå inkonsekvens, foreslår vi at dette alternativet er deaktivert.
  • Ikke reserver diskplass ved oppretting av filer: Aktiver dette alternativet for å hindre at diskplass blir reservert ved oppretting av filer på din DiskStation .
  • Tøm SMB-buffer: Klikk på denne knappen for å slette følgende SMB-bufferfiler for feilsøkingsformål:
    • /var/cache/samba/netsamlogon_cache.tdb
    • /var/lib/samba/winbindd_cache.tdb
    • /var/lib/samba/share_info.tdb
    • /var/lib/samba/registry.tdb
    • /run/samba/connections.tdb
    • /run/samba/gencacche.tdb
    • /run/samba/gencache_notran.tdb

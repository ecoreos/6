Programportal

Programportal lar deg konfigurere tilkoblingsinnstillinger for ulike programmer slik at du kan aksessere og kjøre disse programmene (f.eks. File Station) i uavhengige nettleserfaner eller vinduer.

Merk:

Du kan redigere tilgangsinnstillinger for programmer som Audio Station, Download Station, Surveillance Station, Video Station, File Station og mange andre.

Egendefinere aliaser

Du kan tildele et alias til hvert Synology-utviklede program, f.eks. File Station. Med det egendefinerte aliaset får du hurtig tilgang til programmet via en spesialisert URL.

Slik egendefinerer du programaliaser:

  1. Gå til Kontrollpanel > Programportal > Program.
  2. Velg et program.
  3. Klikk på Rediger > Generelt.
  4. I vinduet som åpnes velger du Aktiver egendefinert alias og spesifiserer aliaset.
  5. Klikk på OK for å lagre innstillingene.

Merk:

  • Enkelte programmer har et standardalias: Audio Station (audio), Download Station (download), File Station (file), Surveillance Station (cam) og Video Station (video).
  • Aliasnavn kan ikke være de samme som dem som er reservert for systemet eller nettlesere eller brukes av andre programmer, og antall tegn må være begrenset til mellom 2 og 20.
  • Et alias kan kun bestå av store og små bokstaver, numeriske tegn og to spesialtegn (-) og (_).
  • Et alias kan ikke starte eller slutte med et spesialtegn, (-) eller (_).

Slik får du tilgang til programmer med aliaser:

Når du har satt opp et alias for et program (f.eks. File Station), får du hurtig tilgang til programmet ved å angi en URL i følgende formater: http://DS_IP_ELLER_SERVERNAVN/ALIAS/ or https://DS_IP_ELLR_SERVERNAVN/ALIAS/

Du kan for eksempel få hurtig tilgang til File Station via URL-ene: http://192.168.xx.xx/file/ or https://MinSynologyNAS/file/

Tilpassede HTTP/HTTPS-porter

Du kan tildele en HTTP/HTTPS-port til hvert Synology-utviklede program, f.eks. File Station. Med den tilpassede porten får du hurtig tilgang til programmet via en spesialisert URL.

Slik tilpasser du HTTP/HTTPS-porter for program:

  1. Gå til Kontrollpanel > Programportal > Program.
  2. Velg et program.
  3. Klikk på Rediger > Generelt.
  4. I vinduet som åpnes velger du Aktiver tilpasset port (HTTP) eller Aktiver tilpasset port (HTTPS).
  5. Spesifiser et tilpasset portnummer.

Merk:

  • Portnummerområdet kan være fra 1 til 65535.
  • Følgende porter kan ikke brukes fordi de er reservert for systembruk:
    • 20, 21, 22, 23, 25, 80, 110, 137, 138, 139, 143, 199, 443, 445, 515, 543, 548, 587, 873, 993, 995, 3306, 3689, 5000, 5001, 5005, 5006, 5335, 5432, 6881, 8080, 7000, 7001, 8081, 9997, 9998, 9999, 50001, 50002
    • eMule-standardporter: 4662 (TCP), 4672(UDP)
    • Standard FTP-portområde: Det faktiske området kan variere etter modell.

Slik får du tilgang til programmer med HTTP/HTTPS-porter:

Når du har satt opp et alias for et program (f.eks. File Station), får du hurtig tilgang til programmet ved å angi en URL i følgende formater: http://DS_IP_ELLER_SERVERNAVN:HTTP_PORT or https://DS_IP_ELLR_SERVERNAVN:HTTPS_PORT

Du kan for eksempel få hurtig tilgang til File Station via URL-ene: http://192.168.xx.xx:7000 or https://MinSynologyNAS:7001

Egendefinere domener

Du kan pare et domenenavn med et Synology-utviklet program, f.eks. File Station. Med det egendefinerte domenet får du hurtig tilgang til programmet via en spesialisert URL.

Slik egendefinerer du programdomener:

  1. Gå til Kontrollpanel > Programportal > Program.
  2. Velg et program.
  3. Klikk på Rediger > Generelt.
  4. I vinduet som åpnes velger du Aktiver egendefinert domene.
  5. Spesifiser et egendefinert domene.
  6. For ytterligere innstillinger kan du velge Aktiver HSTS eller Aktiver HTTP/2 i henhold til dine behov.

Merk:

  • Før du setter opp denne funksjonen må du søke om domenenavn fra en domeneleverandør slik at brukere kan få tilgang til tjenesten via Internett.
  • For programmer som er relatert til filopplasting/-nedlasting (f.eks. File Station), foreslår vi at du ikke velger Aktiver HTTP/2 fordi slike programmer kan få trege opplastings- og nedlastingshastigheter.
  • Hvert domenenvan kan kun brukes til ett bestemt formål på din DiskStation.
  • Hvis domenenavnet ikke overskrider 15 tegn og overholder NetBIOS-navnekonvensjonene, vil systemet automatisk registrere og kringkaste domenavnet.
  • Når du aksesserer programmer via URL-adresser som bruker domenenavn, går du gjennom standard HTTP-port (80) eller standard HTTPS-port (443).

Slik får du tilgang til programmer via domener:

Når du har satt opp et domene (f.eks. file.eksempel.com) for et program (f.eks. File Station), får du hurtig tilgang til programmet ved å angi en URL i følgende formater: http://PROGRAM_DOMENE eller https://PROGRAM_DOMENE

Du kan for eksempel få hurtig tilgang til File Station via URL-ene: http://file.eksempel.com or https://file.eksempel.com

Hvis domenenenavnet, (f.eks. FileFile) til et program (f.eks. File Station) overholder NetBIOS-navnekonvensjonene, kan Windows-brukere aksessere det på en Synology NAS på det samme lokale nettverket med en lignende URL-adresse: http://FileFile eller https://FileFile

Egendefinere regler for omvendt proxy

Din DiskStation kan fungere som en omvendt proxyserver som overfører forespørsler fra Internett til enheter på det lokale nettverket. Regler for omvendt proxy kan hjelpe deg med å skjule følsomme porter fra potensielle trusler, som i følgende scenarioer:

Scenario 1: La oss anta at den følsomme porten er 80, som ikke tillater ekstern tilgang i henhold til brannmurregelen. Du kan sette opp en regel for omvendt proxy for å la pålitelige brukere fra Internett bruke den følsomme porten (80) via en annen åpen port (f.eks. 81). På denne måten kan pålitelige brukere omgå brannmuren og få tilgang til port 80.

Scenario 2: La oss anta at den følsomme porten er 80, som ikke tillater ekstern tilgang med unntak av fra en spesifikk enhet (f.eks. en server som heter "MyTrustee"). Med regler for omvendt proxy kan du tillate at kun trafikk fra MyTrustee får tilgang til port 80, mens trafikk fra andre enheter ikke får tilgang.

Slik setter du opp regler for omvendt proxy:

  1. Gå til Kontrollpanel > Programportal > Omvendt proxy.
  2. Klikk på Opprett og spesifiser følgende innstillinger:
    • Beskrivelse: Spesifiser et navn som hjelper deg med å identifisere regelfunksjonen.
    • Spesifiser reglene for Kilden (enheten som sender forespørsler fra Internett) og Målet (enheten på det lokale nettverket):
      • Protokoll: HTTP- eller HTTPS-protokollene som brukes av kilden/målet.
      • Vertsnavn: Navnet på kilde-/målenheten.
      • Port: Porten som brukes av kilde-/målenheten
      • Aktiver HSTS og Aktiver HTTP/2 (kun for kilden)
  3. Klikk på OK for å lagre innstillingene.

Merk:

Hvis du angir * som kildevertsnavn eller lar det stå tomt, blir regelen tatt i bruk for alle kilder.

Windows fájlszolgáltatás

A Windows fájlszolgáltatás lehetővé teszi SMB/CIFS támogatással rendelkező Windows, Mac, és Linux kliensek számára a hozzáférést a(z) DiskStation készüléken tárolt adatokhoz. Ha Windows Intéző segítségével szeretne böngészni a(z) DiskStation eszközön tárolt fájlok között, illetve megosztott mappát szeretne hálózati meghajtóként csatlakoztatni, engedélyezze ezt a lehetőséget a Vezérlőpult > Fájlszolgáltatások > Win/Mac/NFS menüpontban.

Windows fájlszolgáltatás engedélyezése:

  1. Jelölje be a Windows fájlszolgáltatás engedélyezése jelölőnégyzetet.
  2. Kattintson az Alkalmaz gombra.

Megjegyzés:

A Windows fájlszolgáltatás engedélyezését követően az SMB-n keresztül csatlakoztathatja a teljesen indexelt mappákat Mac számítógépéhez, majd a Finder Spotlight funkciójának használatával a csatlakoztatott mappákban található fájlok tartalmában is végezhet keresést.

Munkacsoport

Ez a lehetőség hálózatkezelési okokból lehetővé teszi a rendszer meghatározott munkacsoport tagjaként való meghatározását.

Megjegyzés:

  • Az egyes munkacsoportok neve 1–15 karaktert tartalmazhat. A név nem tartalmazhatja a következő karaktereket: [ ] ; : " < > * + = \ / | ? ,
  • Ha OS X 10.9 operációs rendszert futtató Mac számítógépet használ, és DiskStation eszköze egy Windows tartományhoz csatlakozott, akkor a DiskStation eszközön tárolt fájlok Windows fájlszolgáltatás segítségével történő eléréséhez úgy jelentkezzen be, hogy a(z) DiskStation helyi felhasználóneve elé írja be a localhost\ szócskát.

Korábbi verziókhoz való hozzáférés letiltása

Windows Intézőben a felhasználók a megosztott mappákban található Korábbi verziók lapon férhetnek hozzá a mappákról és fájlokról készült pillanatfelvételekhez, így rendszergazda segítsége nélkül is visszaállíthatják és másolhatják adataikat. Ha bejelöli a Korábbi verziókhoz való hozzáférés letiltása lehetőséget, a „Korábbi verziók” oldalon semmilyen információ nem jelenik meg.

Megjegyzés:

  • A felhasználók csak abban az esetben férhetnek hozzá egy korábbi verzióhoz, ha az előző verziónak megfelelő pillanatfelvétel készítésekor a felhasználói fiókjuk már létre volt hozva, illetve megkapták azon megosztott mappa olvasási engedélyét, amelyről a megfelelő pillanatfelvétel készült.

Átviteli naplózás engedélyezése

Ennek a lehetőségnek a bejelölése engedélyezi a fájlhozzáférési műveletek naplózását, ideértve a fájlok és mappák CIFS/SMB-protokollon, azaz a Windows fájlátviteli protokollján keresztül történő létrehozását, törlését, olvasását, írását, áthelyezését vagy átnevezését.

Speciális beállítások

Az alábbi lehetőségek megtekintéséhez kattintson a Speciális beállítások gombra.

  • WINS szerver: A Microsoft Windows internetes névszolgáltatás (Microsoft Windows Internet Name Service – WINS) egy NetBIOS rendszert használó, gépnevek IP-címének feloldását végző szolgáltatás. A Windows felhasználói könnyebben találják meg a rendszert TCP/IP-hálózatokon, ha a rendszeren beállították a WINS szerverrel történő regisztrációt.
  • Max protokoll a Windows fájlszolgáltatáshoz: A hálózati konfiguráció függvényében beállíthatja a(z) DiskStation eszköz által támogatott legmagasabb protokollszintet.
    • SMB 2 engedélyezése: Az SMB (szerver üzenetblokk) 2 protokoll a Windows Vista operációs rendszer óta támogatott, ez az SMB továbbfejlesztett verziója. A hálózati csomagok számának csökkentése és a teljesítmény növelése érdekében az SMB 2 képes egyetlen kérelembe több SMB-művelet összefogni.
    • SMB 2 és Large MTU engedélyezése: A maximális átviteli egység (MTU) kifejezés azt a legnagyobb adategységet jelöli, amely egy kommunikációs protokoll segítségével átvihető. A gyorsabb hálózati interfészek, például az 1- vagy 10-gigabites Ethernet előnyeinek kihasználása érdekében a Large MTU 1 megabájtra (MB) növeli a maximális átviteli egységet. A Large MTU beállítás engedélyezése javítja a nagy fájlok átvitelének sebességét és hatékonyságát, miközben csökkenti a feldolgozandó csomagok számát.
    • SMB 3 engedélyezése: Az SMB 3 a Windows 8 és a Windows Server 2012 kiadásától kezdve támogatott. Ez az SMB 2 továbbfejlesztett változata. Az SMB 3 támogatja az AES-alapú titkosított fájlátvitelt, ezzel javítja a peer-to-peer fájlátvitel biztonságát.

    Megjegyzés:

    • Az SMB 2 protokollt a Windows Vista, Windows Server 2008 R2, Windows 7, valamint az ezeknél újabb operációs rendszerek támogatják.
    • A Large MTU funkciót a Windows Server 2008 R2, Windows 7, valamint az ezeknél újabb operációs rendszerek támogatják.
    • Annak érdekében, hogy a Large MTU az Ön Windows számítógépén is hatékonyan működjön, módosítani kell a számítógép beállításjegyzék-beállításait. További részletekért keresse fel a Microsoft műszaki támogatószolgálatát (például olvassa el ezt a cikket).
  • Átviteli titkosítási mód: SMB 3 engedélyezése esetén a Windows fájlszolgáltatás átviteli titkosítás alkalmazása révén javítja a fájlátvitel biztonságát.
    • Letilt: A rendszer nem alkalmaz átviteli titkosítást.
    • Automatikus: Az átviteli titkosítás kizárólag a funkciót támogató kliensek esetében kerül alkalmazásra.
    • Kényszerítés: Az átviteli titkosítás mindig alkalmazásra kerül. Ebben az esetben az átviteli titkosítást nem támogató kliensek nem fogják tudni használni a Windows fájlszolgáltatást.

    • Megjegyzés:

    • Az átviteli titkosítást a Windows 8, Windows 2012 és az újabb verziók támogatják.
    • Az átviteli titkosítás engedélyezése esetén nincs mód a Windows fájlszolgáltatásba való anonim bejelentkezésre.
    • Az átviteli titkosítás bekapcsolása esetén az alkalmazások időtúllépésének elkerülése érdekében ne jelölje be az Alkalmazkodó zárolás letiltása jelölőnégyzetet.
  • Alkalmazkodó zárolás letiltása: Az opció bejelölésével letiltja a CIFS „Opportunistic Locking” funkcióját, amely jobb teljesítményhez vezet az adatbázis-alkalmazások által a(z) DiskStation eszköz megosztott mappájában található adatbázisfájlok hálózaton keresztül történő elérése esetén.
  • Local Master Browser engedélyezése: Ha egyazon alhálózaton több Windows számítógép is megtalálható, az adatok egyeztetését követően az egyik számítógép megkapja a „helyi főtallózó” (local master browser) szerepkört. A helyi főtallózó (local master browser) listát tart fenn az alhálózat többi számítógépéről, illetve azok megosztott erőforrásairól, és megosztja ezeket a listákat a többi számítógéppel. Ez a lehetőség lehetővé teszi a(z) DiskStation számára, hogy felvegye a helyi főtallózó (local master browser) szerepkört.

    • Megjegyzés:

    Ennek a funkciónak az engedélyezése letiltja a merevlemez-hibernálást, valamint aktiválja a jelszó nélküli vendégfiókot.

  • DirSort VFS modul engedélyezése: Ez a modul a mappanevek rendezésére és ábécésorrendben történő felsorolására szolgál. Ennek az opciónak az engedélyezése hatással van a rendszerteljesítményre.
  • Fájlvétó: Fájlok meghatározott kritériumok szerinti elrejtése. A kritériumok közé tartozhatnak helyettesítő karakterek (*), a többszörös adatbevitel egyes tagjait pedig perjellel (/) kell elválasztani egymástól. Például: /abc*/*.txt/*.conf/. Ennek az opciónak az engedélyezése hatással van a rendszerteljesítményre.
  • Szimbolikus hivatkozások engedélyezése megosztott mappákon belül: Ezen opció engedélyezése lehetővé teszi, hogy szimbolikus hivatkozások mutathassanak az egyazon megosztott mappán belüli fájlokra vagy mappákra. A szimbolikus hivatkozások DSM terminál használatával hozhatók létre.
  • Szimbolikus hivatkozások engedélyezése megosztott mappák között: Ezen opció engedélyezése lehetővé teszi, hogy a szimbolikus hivatkozások különböző megosztott mappákban található fájlokra vagy mappákra mutassanak. A szimbolikus hivatkozások DSM terminál használatával hozhatók létre. Ne feledje, hogy ezen opció engedélyezése esetleg lehetővé teheti a hozzáférési jogosultsággal nem rendelkező felhasználók számára a szimbolikus hivatkozások céljaként szolgáló megosztott mappák megtekintését, valamint az azokhoz való hozzáférést.
  • MSDFS VFS modul engedélyezése: Lehetővé teszi, hogy a szimbolikus hivatkozások másik SMB-szervereken található fájlokra vagy mappákra mutassanak. A Windows régebbi verziói, például a Windows XP esetén a kliensnek és a(z) DiskStation készüléknek ezen opció engedélyezése előtt csatlakoznia kell egy Windows tartományhoz. Az MSDFS hivatkozások létrehozásával kapcsolatos további információkat lásd itt. Ne feledje, hogy ezen opció letiltása után a klienseszköz gyorsítótárát törölni kell, ellenkező esetben ez hatással lehet a megszokott teljesítményre.
  • Ugyanarról az IP-címről származó többszörös kapcsolat letiltása: Az opció engedélyezése esetén a(z) DiskStation készülék IP-címenként csak egy kapcsolatot engedélyez a Windows fájlszolgáltatást használó kliensekről. Ha több, ugyanazzal az IP-címmel rendelkező kliens használja a Windows fájlszolgáltatást, a(z) DiskStation a legutolsót tartja meg, minden régebbi kapcsolatot elvet. NAT környezetben előfordulhat, hogy a kliens esetleg egyetlen IP-címet használ, ebben az esetben az opció engedélyezése hatással lehet a megszokott teljesítményre.
  • Hibakeresési naplók gyűjtése: Az opció engedélyezése esetén a rendszer részletes, hibakeresési célokra használható naplókat tárol. Ennek az opciónak az engedélyezése hatással van a rendszerteljesítményre.
  • Alapértelmezett UNIX engedélyek alkalmazása: Az opció engedélyezése esetén a rendszer fájlok és mappák feltöltésekor vagy létrehozásakor az alapértelmezett UNIX engedélyeket alkalmazza. A UNIX engedély fájlok esetén 744, mappák esetén 755. Az opció letiltása esetén a UNIX engedély fájlok és mappák esetén is 777.

    • Megjegyzés:

    • Az opció engedélyezése inkonzisztens engedélyekkel kapcsolatos problémákat okozhat a különböző protokollok között. Az inkonzisztenciák elkerülése érdekében azt javasoljuk, hogy ne kapcsolja be ezt az opciót.

    Megjegyzés:

    • A Windows ACL-t használó megosztott mappák esetében (a külső meghajtókon lévő megosztásokon és a „photo” mappán kívül az összes megosztás) futtassa a chmod parancsot Linux rendszerén vagy FTP-kliensén, amennyiben a Windows ACL mappa- és fájlengedélyeit UNIX-ra szeretné állítani.
    • Az opció engedélyezése inkonzisztens engedélyekkel kapcsolatos problémákat okozhat a különböző protokollok között. Az inkonzisztenciák elkerülése érdekében azt javasoljuk, hogy ne kapcsolja be ezt az opciót.
  • Ne foglaljon le lemezterületet fájlok létrehozásánál: Az opció engedélyezése esetén a rendszer nem foglal le lemezterületet, amikor Ön fájlokat hoz létre DiskStation eszközén.
  • SMB gyorsítótár törlése: Kattintson erre a gombra, ha hibakeresési célból törölni szeretné a következő SMB gyorsítótárfájlokat:
    • /var/cache/samba/netsamlogon_cache.tdb
    • /var/lib/samba/winbindd_cache.tdb
    • /var/lib/samba/share_info.tdb
    • /var/lib/samba/registry.tdb
    • /run/samba/connections.tdb
    • /run/samba/gencacche.tdb
    • /run/samba/gencache_notran.tdb

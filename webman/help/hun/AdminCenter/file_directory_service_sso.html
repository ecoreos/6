SSO-kliens

Ha az Ön DiskStation alkalmazása azonos tartományhoz vagy LDAP szolgáltatáshoz csatlakozott, mint az SSO-szerver, beállíthatja a(z) DiskStation alkalmazását SSO-kliensként. A felhasználók hozzáférhetnek az Ön DiskStation eszköze által biztosított szolgáltatásokhoz, amint bejelentkeznek az SSO-szerverre a hitelesítő adataikkal.

Mielőtt elkezdené, győződjön meg róla, hogy az Ön DiskStation eszköze azonos tartományhoz vagy LDAP-szolgáltatáshoz csatlakozott, mint az SSO-szerver.

A(z) DiskStation beállítása SSO-kliensként.

  1. Lépjen be a Vezérlőpult > Tartomány/LDAP > SSO-kliens oldalra.
  2. Jelölje be az SSO-szolgáltatás engedélyezése jelölőnégyzetet.
  3. Adja meg az SSO-szerver URL-címe és az Alkalmazásazonosító értékeket a jelölőnégyzet alatti mezőkben.

    Megjegyzés:

    Az Alkalmazásazonosítót az alábbi módszerek valamelyikével tudhatja meg:

    • 1. módszer:
      1. Jelentkezzen be az SSO-szerverként működő DSM rendszerbe.
      2. Adja hozzá az Ön kliensalkalmazásként szolgáló DiskStation alkalmazását az Alkalmazás listához az SSO Server alkalmazásban. A részletes lépéseket lásd az SSO Server > Alkalmazáslista súgótémakörben.
      3. Miután az Ön DiskStation alkalmazása hozzáadásra került az SSO Server alkalmazáshoz, másolja ki és illessze be az SSO Server által kiosztott Alkalmazásazonosítót.
    • 2. módszer:
      1. Az SSO-szerver URL-címe* mező megadását követően kattintson a Gyors regisztráció lehetőségre az Alkalmazásazonosító mező mellett.
      2. A regisztrációs panelen írja be az SSO-szerver valamelyik rendszergazda fiókjának hitelesítő adatait a Szerver rendszergazdája és a Jelszó mezőkbe.
      3. Adja meg az kliensalkalmazás nevét az Alkalmazás neve mezőben.
      4. Győződjön meg róla, hogy az Átirányítási URI pontosan az Ön DiskStation alkalmazása bejelentkező oldalára utal.
      5. Kattintson az OK gombra, és az Ön DiskStation alkalmazása hozzáadásra kerül az SSO Server alkalmazáshoz, és az Alkalmazásazonosítót a program automatikusan kitölti.

    ______

    * Az SSO-szerver URL-címét (állomásnevet vagy IP-címet) azonosan kell megadni minden SSO-kliens konfigurálásakor.

  4. A beállítások mentéséhez kattintson az Alkalmaz gombra.

Megjegyzés:

  • Ha az SSO-szerver HTTPS-protokollt használ, az SSO-ba való bejelentkezést megbízható tanúsítvány hiányában a böngészők blokkolhatják.
  • Ha az SSO-szervert HTTP-protokollon keresztül éri el, és az Ön DiskStation alkalmazása (amely SSO-kliensként lett beállítva) HTTPS-kapcsolatot használ, a felhasználóknak be kell állítaniuk a böngészőjükben a nem biztonságos kapcsolat használatának engedélyezését.
  • Az SSO-szerver IPv6 formátumban megadott IP-címét a rendszer nem támogatja, amennyiben bejelölte a Biztonság javítása a HTTP Content Security Policy (CSP) fejléc segítségével jelölőnégyzetet a Vezérlőpult > Biztonság > Biztonság menüpontban a DSM rendszeren.

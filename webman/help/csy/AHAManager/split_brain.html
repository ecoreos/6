Split brain

Pokud cluster high-availability funguje správně, může úlohu aktivního serveru plnit pouze jeden ze členských serverů. V takovém případě pasivní server zjistí přítomnost aktivního serveru prostřednictvím připojení Heartbeat a datového připojení se sdílenými rozšiřovacími jednotkami, takže k přepnutí nedojde. Pasivní server dále zjistí stav sdíleného úložiště, aby nedošlo k nesprávnému převzetí úlohy aktivního serveru v době, kdy stávající aktivní server ještě pracuje se sdílenými rozšiřovacími jednotkami.

Po ztrátě připojení Heartbeat a všech datových připojení nebo v případě nestabilního stavu sdíleného úložiště a systému může kterýkoli z obou serverů vystupovat jako aktivní server. Tato situace se označuje jako chyba „split brain“. V takovém případě se doporučuje, aby jeden server pokračoval v poskytování služeb, zatímco u druhého se provede restart s požadavkem na následnou kontrolu spojení, aby nedošlo k poškození dat v důsledku souběžného přístupu z obou serverů. Za této situace mohou být připojení klientských uživatelů po určitou dobu nestabilní.

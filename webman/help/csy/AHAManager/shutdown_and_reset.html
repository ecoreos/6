Vypnutí a výměna serveru

Níže uvedené pokyny představují popis vypnutí nebo výměny serveru v clusteru high-availability.

Vypnutí clusteru high-availability pomocí nástroje High Availability Manager:

Chcete-li vypnout cluster high-availability z důvodu přemístění serverů do nového fyzického umístění nebo provedení údržby, postupujte podle následujících pokynů.

  1. Přihlaste se do systému DSM.
  2. Přejděte do nástroje High Availability Manager.
  3. Klikněte na tlačítko s označením Správa.
  4. Vyberte možnost Vypnout cluster high-availability.
  5. Zobrazí se potvrzovací zpráva. Potvrďte akci kliknutím na tlačítko OK.

Ruční vypnutí clusteru high-availability:

Cluster high-availability lze vypnout také ručně pomocí tlačítek napájení na obou serverech.

  1. Stiskněte a držte tlačítko napájení na pasivním serveru, dokud se neozve zvukový signál, a vyčkejte do celkového vypnutí serveru.
  2. Stiskněte a držte tlačítko napájení na aktivním serveru, dokud se neozve zvukový signál, a vyčkejte do celkového vypnutí serveru.

Poznámka:

  • Zapnutí serverů:
    Pokud chcete servery opět zapnout, je potřeba zapnout aktivní i pasivní server současně (případně s rozestupem do pěti minut). Pokud zapnete pouze jeden ze serverů, pokusí se zapnout druhý server odesláním zprávy Probuzení přes LAN.
  • Nestandardní vypnutí:
    Cluster high-availability umožňuje provést restart po výpadku napájení. Pokud dojde k vypnutí serverů z důvodu poruchy napájení, budou servery po obnovení napájení automaticky zapnuty.

Postup výměny serveru v clusteru high-availability:

Pokud chcete odebrat jeden server a nastavit jiný, postupujte podle následujících pokynů.

  1. Přejděte na stránku Přehled.
  2. Klikněte na tlačítko Správa.
  3. Volbou Vypnout aktivní server nebo Vypnout pasivní server vypněte server, který chcete vyměnit.
  4. Zkontrolujte připojení nového serveru. Po zapnutí nového serveru bude tento v nástroji High Availability Manager označen jako „neznámý“.
  5. Jděte opět na stránku Přehled.
  6. Klikněte na tlačítko Správa.
  7. Zvolte možnost Obnovit neznámý server a „neznámý server“ bude přidán do clusteru high-availability jakožto pasivní server.

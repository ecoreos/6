Tato stránka zobrazuje základní informace týkající se clusteru high-availability a členských serverů. Dále tato stránka nabízí nástroje určené k vytváření, odebírání nebo provádění údržby clusteru high-availability a členských serverů.

Stav clusteru high-availability

Toto pole zobrazuje celkový stav clusteru high-availability a upozorňuje na veškeré chyby. Následující tabulka představuje všechny možné podmínky zobrazené v tomto poli:

Stav Popis
Samostatný Servery nejsou zapojeny do clusteru high-availability.
V pořádku Celkový stav clusteru high-availability je dobrý. Replikace dat z aktivního serveru na pasivní server funguje normálně.
Příprava Servery byly úspěšně zapojeny do clusteru high-availability, ale počáteční replikace dat ještě nebyla dokončena. Přepnutí nelze nyní provést. Další informace najdete v části Přepnutí.
Načítání Systém načítá stav clusteru. Přepnutí lze provést až po dokončení načítání stavu.
Upozornění Cluster HA není v pořádku, služby však fungují standardně.
Výstraha Aktivní ani pasivní server nefunguje správně. Cluster high-availability nemůže správně fungovat. Zkontrolujte stav veškeré úložné kapacity (např. skupin RAID Group, skupin disků, svazků a jednotek iSCSI LUN).

Zprávy upozornění

Následující tabulka obsahuje zprávy upozornění, které se mohou zobrazit v případě chyby systému. Pokud se zobrazí více než jedna zpráva, můžete se v rámci těchto zpráv pohybovat tlačítky Dopředu/Zpět.

Upozornění/Chyba Popis
Chyba sledované služby V jedné nebo více ze sledovaných služeb došlo k chybě.
Omezený režim úložiště Došlo k omezenému režimu úložiště (např. skupiny RAID Group, skupiny disků, svazku, jednotky iSCSI LUN a mezipaměti SSD) na aktivním nebo pasivním serveru. Zkontrolujte stránku Stav úložiště.
Porucha úložiště Došlo k poruše úložiště (např. skupiny RAID Group, skupiny disků, svazku, jednotky iSCSI LUN a mezipaměti SSD) na aktivním nebo pasivním serveru. Zkontrolujte stránku Stav úložiště.
Pasivní server se nepodařilo nalézt K této chybě může dojít tehdy, pokud je pasivní server v režimu offline, anebo není možné připojit aktivní server k pasivnímu serveru pomocí připojení Heartbeat. V tomto případě nebude funkce přepnutí k dispozici.
Data nelze replikovat do pasivního serveru K této chybě může dojít v důsledku poruchy svazku úložiště nebo disku. V tomto případě nebude funkce přepnutí k dispozici. Zkontrolujte stránku Stav úložiště.
Pasivní server není vybrán Tento stav se zobrazí, když je pasivní server odpojen od clusteru high-availability. V tomto případě nebude funkce přepnutí k dispozici. Pokud chcete pasivní server znovu připojit, klikněte na rozevírací nabídku Údržba a vyberte možnost Připojit. Po připojení pasivního serveru bude vyžadována kompletní replikace dat.

Informace o clusteru high-availability

Toto pole se nachází pod stavem clusteru high-availability a zobrazuje základní informace týkající se clusteru. Pokud cluster high-availability neexistuje, bude toto pole obsahovat pouze vysvětlení funkce clusterů high-availability.

Toto pole obsahuje následující informace:

  • Název clusterového serveru: Název serveru používaný clusterem high-availability.
  • IP adresa clusteru: IP adresa používaná clusterem high-availability. Pomocí této IP adresy se mohou uživatelé připojit ke clusteru high-availability bez ohledu na to, na kterém serveru jsou služby spuštěny.
  • Čas sestavení: Datum vytvoření clusteru high-availability.

Ovládací tlačítka clusteru high-availability

Následující dvě tlačítka se nacházejí pod údaji o clusteru high-availability:

  1. Vytvořit cluster HA: Pokud neexistuje žádný cluster high-availability, můžete kliknutím na toto tlačítko propojit dva servery a vytvořit cluster high-availability. Další informace o vytvoření clusteru high-availability naleznete v článku Průvodce Synology High Availability Wizard.
  2. Správa: Toto tlačítko je k dispozici pouze v případě, že již cluster high-availability existuje. Obsahuje následující možnosti:
    • Průvodce správou: Spustí průvodce obsahujícího pokyny pro různé postupy správy.
    • Přepnutí: Zahájí proces přepnutí z aktivního serveru na pasivní server. Během tohoto procesu pozastaví aktivní server zajišťování služeb a bude plnit funkci nového pasivního serveru. Poté bude původní pasivní server plnit funkci nového aktivního serveru a zahájí provoz všech služeb clusteru high-availability. Tato funkce není k dispozici, pokud pasivní server vykazuje chybu nebo nebyl vybrán.
    • Aktualizovat DSM: Na stránce Ovládací panel otevře možnost Aktualizovat DSM. V této části lze aktualizovat oba servery.
    • Odebrat cluster HA: Odebere cluster high-availability.
    • Odpojit pasivní server: Odebere z clusteru pasivní server. Při opětovném přidání tohoto serveru do clusteru high-availability po jeho odpojení bude vyžadována kompletní replikace dat. Proto tuto možnost doporučujeme používat pouze v případě, že chcete nahradit existující pasivní server za nový.
    • Připojit pasivní server: Přidá do clusteru pasivní server. Tato možnost je k dispozici pouze pokud cluster high-availability neobsahuje žádný pasivní server.
    • Vypnout cluster HA: Vypne aktivní i pasivní server.
    • Vypnout aktivní server: Vypne aktivní server.
    • Vypnout pasivní server: Vypne pasivní server.
    • Restartovat aktivní server: Restartuje aktivní server.
    • Restartovat pasivní server: Restartuje pasivní server.
    • Deaktivovat zvukový signál pasivního serveru: Pozastaví zvukový signál pasivního serveru, pokud dojde k chybě. Pasivní server přesto vydá zvukový signál, pokud dojde později k chybě.

Poznámka:

  • Pokud po odebrání clusteru high-availability budete chtít vytvořit nový, bude vyžadována kompletní replikace dat.
  • Pokud pasivní server vypnete nebo odpojíte, nelze služby přepnout, pokud dojde k chybě aktivního serveru.

Stav serveru a replikace dat

Tato část obsahuje obrázky označující stav aktivního a pasivního serveru a také tok replikace dat mezi těmito servery.

Obrázek replikace dat označuje následující možné podmínky:

  • Replikace dat funguje normálně: Označeno daty proudícími z aktivního serveru směrem k pasivnímu serveru.
  • Bez replikace dat: Označeno absencí obrázku s proudícími daty.

Informace o serveru

Tato část se nachází pod obrázky serverů. V této části se zobrazují informace o aktivním a pasivním serveru. Pokud nebyl vybrán žádný pasivní server, bude v této části uvedeno Žádný.

  • Název serveru: Název serveru Synology.
  • Název modelu: Model serveru Synology.
  • Sériové číslo: Sériové číslo serveru Synology.
  • Stav ventilátoru: Stav chladících ventilátorů serveru. Pokud ventilátor nefunguje, bude zobrazeno upozornění. V tomto případě se obraťte na svého místního distributora.
  • Teplota: Teplota serveru Synology.
  • Stav napájení: Označuje, zda všechny napájecí zdroje serveru správně fungují. Pokud je model serveru vybaven redundantními napájecími zdroji, zobrazí se v případě nesprávného fungování upozornění.

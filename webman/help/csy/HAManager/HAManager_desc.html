Správce High Availability

Nástroj High Availability představuje řešení rozvržení serveru, které slouží k omezení přerušení služeb způsobených poruchou serveru. Nástroj Synology High Availability vytvoří pomocí dvou serverů tzv. „cluster high-availability“, v rámci kterého jeden ze serverů představuje „aktivní server“ a druhý server slouží jako „pasivní server“ v pohotovostním režimu. Implementace a vytvoření clusteru high-availability vyžaduje dva kompatibilní servery Synology.

Cluster high-availability využívá následující součásti a koncepty:

  • Koncept: Implementace technologie Synology High Availability vyžaduje k vytvoření clusteru high-availability dva servery Synology. Jeden ze severů představuje „aktivní server“, druhý pak „pasivní server“. Po vytvoření clusteru high-availability probíhá nepřetržité kopírování dat z aktivního serveru na pasivní server. Na pasivním serveru budou uloženy duplicitní kopie všech souborů z aktivního serveru. V případě poruchy aktivního serveru převezme pasivní server všechny služby a vytvoří kompletní zrcadlovou kopii dat uložených na aktivním serveru. Cluster high-availability tak bude i nadále zajišťovat běžný provoz služeb a sníží riziko výpadku serverů. Další informace o funkci přepnutí najdete v části Přepnutí
  • Aktivní server: Aktivní server provádí správu provozu všech služeb. V případě chyby aktivního serveru můžete vy nebo váš systém zahájit proces přepnutí služeb na pasivní server. V takovém případě bude provoz těchto služeb předán pasivnímu serveru a aktivní server bude poté plnit funkci pasivního serveru.
  • Pasivní server: Za běžných podmínek je pasivní server v pohotovostním režimu. Probíhá neustálé replikovaní dat z aktivního serveru na pasivní server. V případě, že aktivní server nemůže zajistit provoz služeb, můžete vy nebo váš systém zahájit proces přepnutí služeb na pasivní server.
  • Připojení prezenčního signálu: Aktivní a pasivní server clusteru high-availability jsou propojeny samostatným připojením privátní sítě, které se označuje jako „Připojení prezenčního signálu“. Po vytvoření clusteru high-availability bude na základě připojení prezenčního signálu zahájena synchronizace a replikace dat mezi aktivním a pasivním serverem. Pokud je kontakt mezi servery na nějakou dobu přerušen, bude systém zjišťovat, zda je nutné přepnutí.
  • Hlavní svazek úložiště pro cluster high-availability: Svazek úložiště aktivního serveru bude plnit funkci hlavního svazku úložiště pro cluster high-availability. Data budou synchronizována se svazkem úložiště pasivního serveru prostřednictvím připojení prezenčního signálu.
  • Záložní svazek úložiště pro cluster high-availability: Svazek úložiště pasivního serveru plní funkci záložního svazku úložiště pro cluster high-availability. Data budou neustále replikována ze svazku úložiště aktivního serveru na svazek úložiště pasivního serveru prostřednictvím připojení prezenčního signálu. Tím bude během přepnutí zajištěna konzistence obsahu souborů.
  • Název serveru clusteru high-availability: Servery v rámci clusteru high-availability budou mít stejný název. V rámci sítě se můžete připojit pomocí tohoto názvu serveru, provoz služeb pak bude zajišťovat aktivní server.
  • IP adresa clusteru high-availability: Vzhledem k tomu, že aktivní i pasivní servery v rámci clusteru high-availability mají vlastní IP adresu a při přepínání mezi jednotlivými servery je potřeba zabránit změně na jinou IP adresu, vytváří cluster high-availability obecnou přístupovou IP adresu. Chcete-li získat přístup k libovolným službám systému DSM bez ohledu na to, který server Synology plní funkci aktivního serveru, můžete se připojit pomocí těchto IP adres.

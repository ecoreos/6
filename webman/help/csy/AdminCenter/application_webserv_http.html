Webové aplikace

Díky integrované funkci Web Station a podpoře PHP můžete na zařízení DiskStation snadno hostovat webovou stránku a vytvářet dynamické webové stránky pro vaši podnikatelskou činnost nebo hobby. Kromě toho vám stránka Centrum balíčků nabízí celou řadu balíčků třetích stran, jako je například Správa obsahu, Správa vztahů se zákazníky a Systém e-commerce. Pro ukládání souborů webové stránky bude vytvořena výchozí sdílená složka web.

Postup povolení Web Station:

  1. Přejděte do části Ovládací panel > Webové služby.
  2. Zaškrtněte možnost Zapnout stanici Web Station.
  3. Klikněte na možnost Použít.

Poznámka:

  • Pro zajištění lepší flexibility a zabezpečení využívá Web Station pro provádění úloh skupinu uživatelů formát http. Chcete-li tedy spravovat přístupová oprávnění k vašim webovým stránkám, změňte přístupová oprávnění skupiny http pro jednotlivé příslušné složky a soubory. Chcete-li například zpřístupnit soubor sample.htm ve sdílené složce web prostřednictvím Web Station, bude potřeba zajistit, aby skupina http měla příslušná oprávnění čtení/zápisu ke sdílené složce web a také k souboru sample.htm.
  • Soubor domovské stránky by měl mít název index.htm, index.html nebo index.php.
  • Pokud použijete funkci PHP mail(), systém automaticky načte a použije informace o serveru SMTP na stránce Upozornění. Ověřte, zda jste povolili e-mailová upozornění a zadali správné informace serveru SMTP.
  • Skutečná cesta sdílené složky web je /volume1/web. Některé aplikace vyžadují nastavení cesty v konfiguračním souboru.
  • Názvy blog, photo, photosrc a webdefault jsou vyhrazené názvy složek. Web Station tyto názvy nemůže používat, proto ve sdílené složce web nevytvářejte složky s těmito názvy.
  • Chybovou stránku můžete upravit vytvořením souboru missing.html, missing.htm nebo missing.php. Nejdříve povolte Web Station. Potom umístěte soubor missing.html, missing.htm nebo missing.php do sdílené složky web přímo na serveru. Pokud budou uživatelé chtít otevřít stránku, která neexistuje, zobrazí se chybová stránka.
  • Maximální počet aktivních připojení HTTP: 256
  • Maximální počet virtuálních hostitelů: 30

Povolení virtuálního hostitele pro hostování webových stránek

Díky funkci virtuálního hostitele můžete hostovat několik webových stránek na jediném serveru. Dříve než začnete, bude nutné si podle níže uvedených kroků zaregistrovat několik názvů domény a nakonfigurovat virtuální hosting.

Postup povolení virtuálního hostitele:

  1. Klikněte na možnost Virtuální hostitel.
  2. Klikněte na možnost Vytvořit.
  3. Zadejte název pro podsložku sdílené složky web, ve které budou uloženy soubory webové stránky.
  4. Zadejte název hostitele webové stránky.
  5. Zadejte port webové stránky.
  6. Klikněte na možnost OK.

Povolení osobních webových stránek

Pomocí funkce osobní webové stránky mohou místní uživatelé, uživatelé domény a uživatelé LDAP vytvářet své vlastní osobní webové stránky.

Místní uživatelé

Jednotliví místní uživatelé získají jedinečnou adresu webové stránky ve formátu IP-adresa-serveru/~uživatelské jméno nebo název-serveru/~uživatelské jméno. Příklad:

  1. 192.168.1.1/~ken
  2. SynologyNAS/~ken

Uživatelé domény

Jednotliví místní uživatelé získají jedinečnou adresu webové stránky ve formátu IP-adresa-serveru/~názevdomény%5cuživatelské jméno nebo název-serveru/~názevdomény%5cuživatelské jméno. Příklad:

  1. 192.168.1.1/~zkušební doména%5cken
  2. SynologyNAS/~zkušební doména%5cken

Uživatelé LDAP

Jednotliví uživatelé LDAP získají jedinečnou adresu webové stránky ve formátu IP-adresa-serveru/~uživatelskéjméno@názevdomény nebo název-serveru/~uživatelskéjméno@názevdomény. Příklad:

  1. 192.168.1.1/~ken@zkušební doména
  2. SynologyNAS/~ken@zkušební doména

Poznámka:

Před povolením této funkce ověřte, zda jsou povoleny složky uživatelů. V případě místních uživatelů přejděte do části Ovládací panel > Uživatel > Rozšířené; v případě uživatelů domény přejděte do části Ovládací panel > Adresářová služba > Uživatelé domény > Složka uživatele; v případě uživatelů LDAP přejděte do části Ovládací panel > Adresářová služba > Uživatelé LDAP > Složka uživatele.

Poznámka:

Před povolením této funkce ověřte, zda jsou povoleny složky uživatelů. V případě místních uživatelů přejděte do části Ovládací panel > Uživatel > Upřesnit.

Postup vytvoření osobní webové stránky:

  1. Pokud jste zaškrtnuli možnost Povolit osobní webové stránky, bude v adresáři uživatele automaticky vytvořena složka www.
  2. Uživatel může nyní do složky www nahrávat soubory související s webovými stránkami (např. webové stránky a zvukové soubory).
  3. Ověřte, zda má uživatel nezbytná oprávnění k nahraným souborům.

Postup úpravy chybové stránky:

  1. Chybovou stránku uložte jako soubor missing.html, missing.htm nebo missing.php.
  2. Odešlete soubor do složky www.

Pokud budou uživatelé chtít otevřít stránku, která na webových stránkách místního uživatele neexistuje, zobrazí se zvláštní chybová stránka.

Podpora a omezení:

Místní uživatelé získají přístup pouze k datům v následujících složkách:

  • Vlastní složka
  • /var/tmp
  • /tmp
  • /volumeX/@tmp

HTTP služba

Kromě výchozích čísel portů 80 (pro protokol HTTP) nebo 443 (pro protokol HTTPS) lze pro přístup ke službám Photo Station a Web Station přidat další čísla portů. Pro protokol HTTP nebo HTTPS lze přidat další číslo portu v rozsahu od 1 do 65535.

Postup přidání dodatečného čísla portu pro přístup přes protokol HTTP nebo HTTPS:

  1. Zaškrtněte možnost Přidat dodatečný port HTTP nebo Přidat dodatečný port HTTPS (před přidáním portu pro protokol HTTPS je nutné zaškrtnout možnost Povolit připojení HTTPS pro webové služby.
  2. Do pole zadejte požadované číslo portu.

Poznámka:

  • Nelze použít následující porty: 20, 21, 22, 23, 25, 110, 137, 138, 139, 143, 199, 443, 445, 515, 543, 548, 587, 873, 993, 995, 3306, 3689, 5000, 5001, 5005, 5006, 5335, 5432, 6881, 8080, 7000, 7001, 8081, 9997, 9998, 9999, 50001, 50002, výchozí porty pro eMule: 4662 (TCP), 4672(UDP) a výchozí rozsah portů pro FTP: konkrétní rozsah se může u různých modelů lišit.
